## 趣味引入

同学们，你们有没有玩过爬楼梯的游戏？假设你正在爬一座有 `n` 级台阶的楼梯，每次你只能向上爬 1 级台阶或者 2 级台阶。现在，我问你，一共有多少种不同的方法可以爬到楼顶呢？

*   如果只有 1 级台阶，你只能 `1` 步爬上去，所以只有 **1** 种方法。
*   如果只有 2 级台阶，你可以 `1+1` 步爬上去，也可以 `2` 步爬上去，所以有 **2** 种方法。
*   那如果是 3 级台阶呢？
    *   你可以先爬到第 2 级，再走 1 级 (方法数与爬 2 级台阶相同)。
    *   你也可以先爬到第 1 级，再走 2 级 (方法数与爬 1 级台阶相同)。
    *   所以，爬 3 级台阶的方法数 = 爬 2 级台阶的方法数 + 爬 1 级台阶的方法数 = 2 + 1 = **3** 种！

同学们有没有发现，我们想知道爬 3 级台阶的方法数，竟然可以由爬 1 级和 2 级台阶的方法数“推导”出来！这就像一个“骨牌效应”，一个结果的得出，依赖于前面一些结果。这就是我们今天要学习的“递推算法”的魅力所在！它能帮助我们把大问题分解成小问题，一步步地解决它。

## 知识点讲解

### 什么是递推算法？
递推算法是一种特殊的算法思想，它通过已知的一些**初始条件**，以及一个**递推关系式**，从前往后一步步地计算出序列中的每一项，直到求得我们想要的结果。简单来说，就是“用前面已知的结果，推出当前的结果”。

### 递推算法的两个核心要素
1.  **初始条件（Base Case）**：
    *   就像盖房子需要地基一样，递推算法也需要一个或几个“起点”。这些起点是直接已知的，不需要通过递推关系式计算。
    *   在爬楼梯的例子中，爬 1 级台阶有 1 种方法，爬 2 级台阶有 2 种方法，它们就是初始条件。

2.  **递推关系式（Recurrence Relation）**：
    *   这是连接“过去”和“现在”的桥梁。它是一个数学公式，告诉我们如何根据前一项或前几项的值，计算出当前项的值。
    *   在爬楼梯的例子中，`f[i] = f[i-1] + f[i-2]` 就是递推关系式，其中 `f[i]` 表示爬 `i` 级台阶的方法数。

### 简单递推模型的例子
让我们来看几个经典的简单递推模型：

#### 1. 等差数列
*   **描述**：一个数列，从第二项起，每一项与它的前一项的差等于同一个常数。
*   **初始条件**：首项 `a[1]`
*   **递推关系式**：`a[i] = a[i-1] + d` (其中 `d` 是公差)
*   **例子**：1, 3, 5, 7, 9... (首项 `a[1]=1`，公差 `d=2`)

#### 2. 斐波那契数列 (Fibonacci Sequence)
*   **描述**：一个数列，从第三项起，每一项都等于前两项之和。
*   **初始条件**：`f[1] = 1`, `f[2] = 1` (或者 `f[0]=0, f[1]=1`，取决于定义)
*   **递推关系式**：`f[i] = f[i-1] + f[i-2]` (当 `i >= 3` 时)
*   **例子**：1, 1, 2, 3, 5, 8, 13, 21...

### 如何用代码实现递推？
我们通常会使用一个数组（或者几个变量）来存储已经计算出的结果，然后通过一个循环，从初始条件开始，一步步地计算出后续的结果。

**为什么不用递归？**
虽然递推关系式看起来很像递归函数的定义，但对于大规模问题，直接使用递归往往效率很低，因为它会重复计算很多次相同的值。而使用循环（迭代）配合数组存储结果，可以避免重复计算，大大提高效率。这种优化思想在算法中非常重要，我们称之为“动态规划”的初步思想。

## 代码示例

### 示例一：计算等差数列的第 n 项
**问题描述**：给定等差数列的首项 `a` 和公差 `d`，求该数列的第 `n` 项。

**分析**：
*   初始条件：`f[1] = a`
*   递推关系式：`f[i] = f[i-1] + d` (对于 `i > 1`)

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    // 输入首项 a，公差 d，和要查询的项数 n
    int a, d, n;
    cin >> a >> d >> n;

    // 如果 n 是 1，直接输出首项
    if (n == 1)
    {
        cout << a << endl;
        return 0;
    }

    // 创建一个数组来存储每一项的值
    // 注意：数组下标通常从 0 开始，如果题目从 1 开始计数，我们可以让数组大小为 n+1
    // 或者直接用一个变量滚动计算
    // 这里我们使用滚动变量来节省空间，因为我们只需要前一项的值
    long long current_term = a; // current_term 存储当前计算到的项的值

    // 从第 2 项开始计算，直到第 n 项
    for (int i = 2; i <= n; ++i)
    {
        current_term = current_term + d; // 根据递推关系式计算当前项
    }

    cout << current_term << endl; // 输出第 n 项

    return 0;
}
```

### 示例二：计算斐波那契数列的第 n 项
**问题描述**：斐波那契数列定义为 `f[1]=1, f[2]=1`，从第三项开始，每一项都等于前两项之和。求斐波那契数列的第 `n` 项。

**分析**：
*   初始条件：`f[1] = 1`, `f[2] = 1`
*   递推关系式：`f[i] = f[i-1] + f[i-2]` (对于 `i >= 3`)

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    // 输入要查询的项数 n
    int n;
    cin >> n;

    // 如果 n 是 1 或 2，直接输出初始值
    if (n == 1 || n == 2)
    {
        cout << 1 << endl;
        return 0;
    }

    // 创建一个数组来存储斐波那契数列的每一项
    // f[0] 不用，f[1] 存第1项，f[2] 存第2项... f[n] 存第n项
    // 由于 Level 4 限制不能使用 STL 容器，我们使用原生数组。
    // 数组大小需要足够大，一般根据题目最大 N 值来设定，这里假设 N 不超过 100。
    long long f[101]; // 存储斐波那契数列，注意数据类型，斐波那契数列增长很快

    // 初始化初始条件
    f[1] = 1;
    f[2] = 1;

    // 从第 3 项开始，根据递推关系式计算到第 n 项
    for (int i = 3; i <= n; ++i)
    {
        f[i] = f[i-1] + f[i-2]; // 递推关系式：当前项 = 前一项 + 前前一项
    }

    cout << f[n] << endl; // 输出第 n 项

    return 0;
}
```

### 小组任务：神奇细胞分裂
**任务描述**：
假设你有一个神奇的细胞，它每过一分钟就会分裂成两个一模一样的新细胞。一开始你只有一个细胞。请你和小组同学讨论：
1.  写出细胞数量的初始条件。
2.  写出细胞数量的递推关系式。
3.  如果用 `c[i]` 表示第 `i` 分钟末的细胞数量，那么 `c[5]` 会是多少？

**讨论指导**：
*   第 0 分钟末（刚开始）：有几个细胞？
*   第 1 分钟末：细胞分裂一次，有几个细胞？
*   第 2 分钟末：所有细胞都分裂一次，有几个细胞？
*   `c[i]` 和 `c[i-1]` 之间有什么关系？

**参考答案**：
1.  **初始条件**：`c[0] = 1` (第 0 分钟末有 1 个细胞)
2.  **递推关系式**：`c[i] = c[i-1] * 2` (第 `i` 分钟末的细胞数是第 `i-1` 分钟末细胞数的两倍)
3.  **计算 `c[5]`**：
    *   `c[0] = 1`
    *   `c[1] = c[0] * 2 = 1 * 2 = 2`
    *   `c[2] = c[1] * 2 = 2 * 2 = 4`
    *   `c[3] = c[2] * 2 = 4 * 2 = 8`
    *   `c[4] = c[3] * 2 = 8 * 2 = 16`
    *   `c[5] = c[4] * 2 = 16 * 2 = 32`
    *   所以 `c[5]` 是 **32**。

### 练习题目答案与解析

1.  **答案**：B
    **解析**：递推算法的核心就是从已知的初始条件出发，通过递推关系式一步步计算出结果。

2.  **答案**：C
    **解析**：
    *   `a[1] = 5`
    *   `a[2] = a[1] + 3 = 5 + 3 = 8`
    *   `a[3] = a[2] + 3 = 8 + 3 = 11`
    *   `a[4] = a[3] + 3 = 11 + 3 = 14`

3.  **答案**：B
    **解析**：
    *   `f[1] = 1`
    *   `f[2] = 1`
    *   `f[3] = f[2] + f[1] = 1 + 1 = 2`
    *   `f[4] = f[3] + f[2] = 2 + 1 = 3`
    *   `f[5] = f[4] + f[3] = 3 + 2 = 5`

4.  **答案**：C
    **解析**：递推关系式 `dp[i] = dp[i-1] + dp[i-2]` 需要前两项的值才能计算当前项。
    *   爬 1 级台阶：1 种方法 (1) -> `dp[1] = 1`
    *   爬 2 级台阶：2 种方法 (1+1, 2) -> `dp[2] = 2`
    所以需要 `dp[1]` 和 `dp[2]` 作为初始条件。

5.  **答案**：B
    **解析**：使用数组（或表格）来存储中间结果，可以避免重复计算，提高算法效率，这是动态规划思想的基础。
