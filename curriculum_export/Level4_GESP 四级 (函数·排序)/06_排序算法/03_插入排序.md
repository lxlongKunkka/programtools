## 趣味引入

同学们，你们有没有玩过扑克牌或者整理过书架呢？

想象一下，你手上拿着一叠杂乱无章的扑克牌，你想把它们按照从小到大的顺序排列好。你会怎么做呢？

通常，我们会从第二张牌开始，把它和前面已经排好的牌比较，然后找到一个合适的位置插进去。比如，你手上已经有 `[3, 7]` 这两张牌，现在你抓到一张 `5`。你会怎么放呢？是不是把 `5` 放到 `3` 和 `7` 之间，变成 `[3, 5, 7]`？

这就是我们今天要学习的**插入排序**的核心思想！它就像我们整理扑克牌一样，每次从待排序的元素中取出一个，然后插入到前面已经排好序的序列中的正确位置。是不是很简单又很形象呢？

让我们用一个例子来看看：
假设我们有这样一个数列：`[5, 2, 4, 6, 1, 3]`。
1. 我们先假设第一个元素 `5` 已经排好序了。
2. 接下来，我们拿起第二个元素 `2`。`2` 比 `5` 小，所以 `5` 往后挪，`2` 插到 `5` 的前面。序列变成 `[2, 5, 4, 6, 1, 3]`。
3. 再拿起第三个元素 `4`。`4` 比 `5` 小，`5` 往后挪；`4` 比 `2` 大，所以 `4` 插到 `2` 和 `5` 之间。序列变成 `[2, 4, 5, 6, 1, 3]`。
4. 看到这里，大家是不是已经明白插入排序是怎么工作的了？接下来我们一起来深入学习它的具体步骤！

## 知识点讲解

### 1. 插入排序的核心思想

插入排序（Insertion Sort）是一种简单直观的排序算法。它的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。

**核心思想**：就像整理扑克牌一样，每次从无序区取出一个元素，插入到有序区中的正确位置，使有序区仍然保持有序。

### 2. 插入排序的详细步骤

我们用一个数组 `a` 来说明：`[5, 2, 4, 6, 1, 3]`

1.  **初始化**：
    *   将数组的第一个元素视为一个已经排序好的序列（有序区）。
    *   从第二个元素开始，逐个取出元素（待插入元素）。

2.  **主循环**：从第二个元素 `a[1]` 开始，到最后一个元素 `a[n-1]` 结束。
    *   **取待插入元素**：将 `a[i]` 存储在一个临时变量 `key` 中。
    *   **寻找插入位置并移动元素**：
        *   从 `a[i-1]` 开始，向前（向左）遍历已排序序列。
        *   如果当前元素 `a[j]` 比 `key` 大，则将 `a[j]` 向右移动一位，即 `a[j+1] = a[j]`。
        *   继续向前比较，直到找到一个元素 `a[j]` 比 `key` 小（或等于），或者已经遍历到有序序列的开头（即 `j` 小于 `0`）。
    *   **插入元素**：将 `key` 插入到找到的正确位置 `a[j+1]`。

### 3. 步骤图解 (以 `[5, 2, 4, 6, 1, 3]` 为例)

*   **原始数组**: `[5, 2, 4, 6, 1, 3]`

*   **第 1 趟 (i=1, 待插入元素 `a[1]=2`)**:
    *   有序区: `[5]`
    *   `key = 2`
    *   比较 `key` (2) 和 `a[0]` (5): `2 < 5`。
    *   `5` 后移: `[5, 5, 4, 6, 1, 3]`
    *   插入 `key` 到 `a[0]`: `[2, 5, 4, 6, 1, 3]`
    *   **结果**: `[2, 5, 4, 6, 1, 3]`

*   **第 2 趟 (i=2, 待插入元素 `a[2]=4`)**:
    *   有序区: `[2, 5]`
    *   `key = 4`
    *   比较 `key` (4) 和 `a[1]` (5): `4 < 5`。
    *   `5` 后移: `[2, 5, 5, 6, 1, 3]`
    *   比较 `key` (4) 和 `a[0]` (2): `4 > 2`。
    *   找到位置，插入 `key` 到 `a[1]`: `[2, 4, 5, 6, 1, 3]`
    *   **结果**: `[2, 4, 5, 6, 1, 3]`

*   **第 3 趟 (i=3, 待插入元素 `a[3]=6`)**:
    *   有序区: `[2, 4, 5]`
    *   `key = 6`
    *   比较 `key` (6) 和 `a[2]` (5): `6 > 5`。
    *   `key` 比 `a[2]` 大，无需移动，直接插入到 `a[3]` (即它自己当前的位置)。
    *   **结果**: `[2, 4, 5, 6, 1, 3]`

*   **第 4 趟 (i=4, 待插入元素 `a[4]=1`)**:
    *   有序区: `[2, 4, 5, 6]`
    *   `key = 1`
    *   比较 `key` (1) 和 `a[3]` (6): `1 < 6`。`6` 后移: `[2, 4, 5, 6, 6, 3]`
    *   比较 `key` (1) 和 `a[2]` (5): `1 < 5`。`5` 后移: `[2, 4, 5, 5, 6, 3]`
    *   比较 `key` (1) 和 `a[1]` (4): `1 < 4`。`4` 后移: `[2, 4, 4, 5, 6, 3]`
    *   比较 `key` (1) 和 `a[0]` (2): `1 < 2`。`2` 后移: `[2, 2, 4, 5, 6, 3]`
    *   找到位置，插入 `key` 到 `a[0]`: `[1, 2, 4, 5, 6, 3]`
    *   **结果**: `[1, 2, 4, 5, 6, 3]`

*   **第 5 趟 (i=5, 待插入元素 `a[5]=3`)**:
    *   有序区: `[1, 2, 4, 5, 6]`
    *   `key = 3`
    *   比较 `key` (3) 和 `a[4]` (6): `3 < 6`。`6` 后移: `[1, 2, 4, 5, 6, 6]`
    *   比较 `key` (3) 和 `a[3]` (5): `3 < 5`。`5` 后移: `[1, 2, 4, 5, 5, 6]`
    *   比较 `key` (3) 和 `a[2]` (4): `3 < 4`。`4` 后移: `[1, 2, 4, 4, 5, 6]`
    *   比较 `key` (3) 和 `a[1]` (2): `3 > 2`。
    *   找到位置，插入 `key` 到 `a[2]`: `[1, 2, 3, 4, 5, 6]`
    *   **最终结果**: `[1, 2, 3, 4, 5, 6]`，排序完成！

### 4. 插入排序的时间复杂度分析

*   **最好情况**：当数组已经完全排好序时。
    *   内层循环每次只需比较一次（待插入元素比有序区的最后一个元素大），无需移动。
    *   总共需要进行 `N-1` 次比较。
    *   时间复杂度为 **O(N)**。
*   **最坏情况**：当数组是完全逆序时。
    *   内层循环每次都需要将待插入元素与有序区的所有元素进行比较，并移动所有元素。
    *   第 `i` 趟需要比较 `i` 次，移动 `i` 次。
    *   总比较和移动次数大约是 `1 + 2 + ... + (N-1)`，即 `N*(N-1)/2` 次。
    *   时间复杂度为 **O(N^2)**。
*   **平均情况**：
    *   平均时间复杂度也为 **O(N^2)**。

虽然插入排序在最坏和平均情况下是 O(N^2)，但它对于**基本有序**的数组排序效率非常高，并且它是一种**稳定**的排序算法（相等元素的相对顺序不会改变）。

## 代码示例

下面是插入排序的 C++ 实现。同学们可以仔细阅读代码，结合前面讲的步骤来理解。

```cpp
#include <bits/stdc++.h>

using namespace std;

// 打印数组的函数，方便我们观察排序过程
void print_array(int a[], int n)
{
    for (int i = 0; i < n; i++)
    {
        cout << a[i] << " ";
    }
    cout << endl;
}

// 插入排序函数
void insertion_sort(int a[], int n)
{
    // 从第二个元素开始遍历，因为第一个元素默认已经是有序的
    for (int i = 1; i < n; i++)
    {
        // current_element 存储待插入的元素
        int current_element = a[i]; 
        // j 是有序区的最后一个元素的下标
        int j = i - 1;

        // 循环条件：
        // 1. j >= 0 确保没有越界，还在有序区内
        // 2. a[j] > current_element 待插入元素比有序区中的元素小，需要将该元素后移
        while (j >= 0 && a[j] > current_element)
        {
            a[j + 1] = a[j]; // 将元素 a[j] 向右移动一位
            j--;             // 继续向前（向左）比较下一个元素
        }

        // 找到合适位置后，将 current_element 插入到 a[j+1]
        // 注意：循环结束后，j 已经减小了 1，所以正确的插入位置是 j+1
        a[j + 1] = current_element;

        // 每次插入完成后，打印数组状态，看看变化
        cout << "第 " << i << " 趟排序后: ";
        print_array(a, n);
    }
}

int main()
{
    int a[] = {5, 2, 4, 6, 1, 3}; // 待排序的数组
    int n = sizeof(a) / sizeof(a[0]); // 计算数组的长度

    cout << "原始数组: ";
    print_array(a, n);

    insertion_sort(a, n); // 调用插入排序函数

    cout << "最终排序结果: ";
    print_array(a, n);

    return 0;
}

```

**代码运行结果：**
```
原始数组: 5 2 4 6 1 3 
第 1 趟排序后: 2 5 4 6 1 3 
第 2 趟排序后: 2 4 5 6 1 3 
第 3 趟排序后: 2 4 5 6 1 3 
第 4 趟排序后: 1 2 4 5 6 3 
第 5 趟排序后: 1 2 3 4 5 6 
最终排序结果: 1 2 3 4 5 6 
```

4.  对于一个长度为 `N` 的数组，插入排序进行 `N-1` 趟。在第 `k` 趟（`k` 从 `1` 开始计数）中，待插入的元素是原始数组的第 ______ 个元素（下标从 0 开始）。

5.  在插入排序中，如果待插入的元素比有序区的所有元素都大，那么它将直接放到有序区的末尾，并且内层循环的比较次数最少是 ______ 次。

### 编程题

1.  **基本插入排序**
    编写一个 C++ 函数 `void insertion_sort_desc(int a[], int n)`，实现对整数数组进行**降序**插入排序。在 `main` 函数中测试你的函数，使用数组 `[10, 4, 8, 2, 6]`，并打印排序后的结果。

2.  **统计移动次数**
    修改插入排序函数，使其能统计并返回在排序过程中元素被**移动**的总次数（注意，是元素被挪动位置的次数，不是比较次数）。
    例如，数组 `[5, 2, 4]`：
    - `2` 插入时，`5` 移动一次。
    - `4` 插入时，`5` 移动一次。
    总移动次数为 2。
    使用数组 `[9, 1, 5, 8, 3, 7]` 测试你的函数。

---

### 练习题目答案与解析

1.  **答案：C**
    **解析**：当数组已经排好序时，插入排序的内层循环每次只需进行一次比较（待插入元素与有序区最后一个元素比较），无需移动。总共进行 `N-1` 次比较，因此时间复杂度是 O(N)。

2.  **答案：A**
    **解析**：
    *   原始数组: `[7, 3, 5, 1]`
    *   第一个元素 `7` 视为有序。有序区 `[7]`
    *   待插入 `3`：`3` 比 `7` 小，`7` 后移。插入 `3`。
    *   此时数组状态: `[3, 7, 5, 1]`。

3.  **答案：稳定**
    **解析**：插入排序在移动元素时，只有当待插入元素小于当前比较的元素时才会发生移动。如果待插入元素等于当前比较的元素，它会停在当前元素之后，从而保持了相等元素的相对顺序。

4.  **答案：k**
    **解析**：
    *   当 `i=1` 时，待插入的是 `a[1]`，对应第 `1` 趟。
    *   当 `i=2` 时，待插入的是 `a[2]`，对应第 `2` 趟。
    *   依此类推，当 `i=k` 时，待插入的是 `a[k]`，对应第 `k` 趟。

5.  **答案：1**
    **解析**：如果待插入元素比有序区的所有元素都大，它会从有序区最后一个元素开始比较。第一次比较时，就会发现待插入元素更大，此时内层循环条件 `a[j] > current_element` 不成立，循环终止。所以只会比较 1 次。
