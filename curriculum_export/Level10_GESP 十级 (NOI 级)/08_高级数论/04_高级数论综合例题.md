## 趣味引入

同学们，还记得我们之前学过的模运算吗？它就像一个无限循环的钟表，指针走了多少步，最终停在哪里。比如 `7 % 5 = 2`，就是指针从0开始走了7步，停在了2。

现在，我们把问题升级一下：

假设有一个神秘的保险箱，它的密码是 `A^B % M`。A和M是普通的数字，但B却是一个天文数字，比如有1000位那么长！你不可能真的把A连乘B次，那样宇宙的寿命都不够。有没有一种“魔法”能让我们快速算出这个天大的指数模M的结果呢？这就是我们今天要解决的第一个问题，它背后藏着一个强大的工具——**扩展欧拉定理**。

**场景二：神秘的数字**
村子里有三位智者，每个人都知道一个关于某个神秘数字X的线索：
*   智者A说：“X除以3余2。”
*   智者B说：“X除以5余3。”
*   智者C说：“X除以7余2。”
现在，你作为村子里最聪明的少年，能否找出这个最小的神秘数字X呢？如果智者们给出的线索更复杂，比如除以6余4，除以8余2，除以9余5，这些除数之间还有公约数，你还能找到X吗？这就是我们今天要探索的另一个强大工具——**中国剩余定理（CRT）**，以及它在模数不互质情况下的升级应用。

这两个问题，一个关于“无限次方的终点”，一个关于“多重约束下的唯一解”，都将带我们进入高级数论的殿堂，领略数学之美和算法之妙。准备好了吗？让我们一起揭开它们的神秘面纱！

## 深度知识点讲解

### 1. 扩展欧拉定理 (Extended Euler's Theorem)

#### 1.1 回顾欧拉定理 (Euler's Totient Theorem)
我们知道，如果 `gcd(a, m) = 1` (a和m互质)，那么 `a^phi(m) = 1 (mod m)`。其中 `phi(m)` 是欧拉函数，表示小于等于 `m` 且与 `m` 互质的正整数的个数。
这个定理告诉我们，当 `a` 和 `m` 互质时，`a` 的幂次在模 `m` 意义下是周期性变化的，周期就是 `phi(m)`。

#### 1.2 当 `gcd(a, m) != 1` 时怎么办？
如果 `a` 和 `m` 不互质，欧拉定理就不适用了。但幸运的是，数学家们找到了一个更普适的规律，这就是**扩展欧拉定理**。

**定理内容**：
对于任意正整数 `a, b, m`：
`a^b = a^(b % phi(m) + phi(m)) (mod m)`，当 `b >= phi(m)` 时。
`a^b = a^b (mod m)`，当 `b < phi(m)` 时。

**核心思想**：
这个定理的精髓在于，即使 `a` 和 `m` 不互质，在指数 `b` 足够大（`b >= phi(m)`）的时候，我们仍然可以把 `b` 对 `phi(m)` 取模，但为了保证结果正确，需要额外加上一个 `phi(m)`。这相当于给指数一个“偏移量”，确保幂次足够大，从而进入循环节。

**生活类比**：
想象你有一个智能灯泡，它有 `m` 种颜色（0到 `m-1`）。每次按按钮，颜色会从当前颜色 `C` 变成 `(C + a) % m`。现在你想知道按 `b` 次按钮后的颜色。
如果 `gcd(a, m) = 1`，那么按 `phi(m)` 次后，颜色会回到初始状态。
如果 `gcd(a, m) != 1`，比如 `m=6, a=2`，颜色序列是 `0, 2, 4, 0, 2, 4, ...`。它的周期是3，而不是 `phi(6)=2`。但如果按了很多次，比如按10次，它最终会在4。按 `10 % phi(6) + phi(6) = 10 % 2 + 2 = 0 + 2 = 2` 次，结果是2。按 `phi(6)` 次之后，它一定进入了循环。
扩展欧拉定理就是说，无论 `a` 和 `m` 是否互质，只要指数 `b` 足够大，它就一定会进入一个循环。这个循环的周期与 `phi(m)` 有关，并且为了确保进入循环，我们需要至少 `phi(m)` 这么大的指数。

**关键点剖析**：
1.  **计算 `phi(m)`**：
    *   欧拉函数 `phi(m)` 可以通过对 `m` 进行质因数分解来计算。
    *   如果 `m = p1^k1 * p2^k2 * ... * pn^kn` (其中 `p_i` 是质数)，
    *   那么 `phi(m) = m * (1 - 1/p1) * (1 - 1/p2) * ... * (1 - 1/pn)`
    *   或者 `phi(m) = (p1^k1 - p1^(k1-1)) * ... * (pn^kn - pn^(kn-1))`
    *   编程实现时，通常用第一种形式：遍历 `m` 的质因数 `p`，每次 `phi_val = phi_val / p * (p - 1)`。

2.  **处理指数 `b`**：
    *   由于 `b` 可能非常大（例如字符串形式），我们需要先用一个函数 `read_and_mod(string s, long long m)` 来计算 `b % phi(m)`。
    *   在计算 `b % phi(m)` 的同时，我们需要判断 `b` 是否真的大于等于 `phi(m)`。如果 `b` 本身就小于 `phi(m)`，那么直接用 `b` 作为指数；否则，使用 `b % phi(m) + phi(m)`。
    *   判断 `b >= phi(m)` 的技巧：在 `read_and_mod` 函数中，维护一个 `flag`。如果 `b` 在取模过程中，其值曾大于等于 `phi(m)`，就设置 `flag = true`。

3.  **模幂运算 (Binary Exponentiation / 快速幂)**：
    *   无论指数是否被处理过，最终的 `a^exp % m` 仍然需要使用快速幂算法来计算，以避免超时。

**常见误区**：
*   忘记 `b < phi(m)` 时直接用 `b`，`b >= phi(m)` 时才加 `phi(m)`。简单的 `b % phi(m)` 是不正确的。
*   `phi(m)` 计算错误，导致整个结果错误。
*   大整数 `b` 读取和取模时溢出或逻辑错误。

### 2. 中国剩余定理 (Chinese Remainder Theorem, CRT)

#### 2.1 CRT - 模数两两互质的情况
**定理内容**：
设 `m1, m2, ..., mk` 是两两互质的正整数，则同余方程组：
`x = a1 (mod m1)`
`x = a2 (mod m2)`
`...`
`x = ak (mod mk)`
在模 `M = m1 * m2 * ... * mk` 的意义下有唯一解。

**构造性解法**：
1.  计算 `M = m1 * m2 * ... * mk`。
2.  对于每个方程 `i`，计算 `Mi = M / mi`。
3.  计算 `t_i`，使得 `Mi * t_i = 1 (mod mi)`。这里的 `t_i` 就是 `Mi` 在模 `mi` 意义下的乘法逆元。由于 `mi` 与 `Mi` 互质（因为 `mi` 与其他 `mj` 互质），所以逆元一定存在，可以通过**扩展欧几里得算法**求得。
4.  最终解 `x = (a1 * M1 * t1 + a2 * M2 * t2 + ... + ak * Mk * tk) % M`。

**生活类比**：
假设你想找一个数字 `X`。
*   第一个朋友说：“`X` 加上 2 是 3 的倍数。” ( `X = 1 (mod 3)` )
*   第二个朋友说：“`X` 加上 3 是 5 的倍数。” ( `X = 2 (mod 5)` )
*   第三个朋友说：“`X` 加上 2 是 7 的倍数。” ( `X = 5 (mod 7)` )
每个朋友都给你一个“约束”，CRT 就像一个侦探，综合所有约束，找出符合所有条件的最小正整数 `X`。

#### 2.2 CRT - 模数不互质的情况 (合并同余方程)
当模数 `m1, m2, ..., mk` 不两两互质时，标准的 CRT 就不能直接使用了。但我们可以通过**逐步合并**的方式来解决。

**核心思想**：
将多个同余方程逐步合并成一个。每次合并两个方程：
`x = a1 (mod m1)`
`x = a2 (mod m2)`

1.  从第一个方程，我们知道 `x = k * m1 + a1`，其中 `k` 是某个整数。
2.  将 `x` 代入第二个方程：`k * m1 + a1 = a2 (mod m2)`。
3.  整理得到一个**线性同余方程**：`k * m1 = a2 - a1 (mod m2)`。
    *   设 `A = m1`, `B = (a2 - a1)` (注意 `B` 可能为负，需要调整为正)，`M = m2`。
    *   我们现在要解 `A * k = B (mod M)`。

#### 2.3 线性同余方程 `ax = b (mod m)` 的求解
**定理**：
方程 `ax = b (mod m)` 有解的充要条件是 `gcd(a, m)` 能够整除 `b`。
如果 `gcd(a, m)` 整除 `b`，则方程有 `gcd(a, m)` 个模 `m` 意义下的解。

**求解步骤**：
1.  计算 `g = gcd(a, m)`。
2.  如果 `b % g != 0`，则方程无解。
3.  如果 `b % g == 0`，则方程有解。我们可以将整个方程除以 `g`：
    ` (a/g) * x = (b/g) (mod m/g) `
    现在 `a/g` 和 `m/g` 互质了。
4.  设 `a' = a/g`, `b' = b/g`, `m' = m/g`。方程变为 `a' * x = b' (mod m')`。
5.  由于 `a'` 和 `m'` 互质，`a'` 在模 `m'` 意义下有乘法逆元 `inv(a', m')`。
6.  `x = b' * inv(a', m') (mod m')`。
7.  `inv(a', m')` 可以通过**扩展欧几里得算法**求解：
    *   `extended_gcd(a', m', x0, y0)` 会得到 `x0` 和 `y0` 使得 `a'*x0 + m'*y0 = gcd(a', m') = 1`。
    *   所以 `x0` 就是 `a'` 在模 `m'` 意义下的逆元。
    *   注意 `x0` 可能为负，需要调整为 `(x0 % m' + m') % m'`。
8.  最终，`x = (b' * x0) % m'` 是 `x` 的一个特解。
9.  这个特解是模 `m'` 意义下的唯一解。原始方程 `ax = b (mod m)` 在模 `m` 意义下有 `g` 个解：`x_0, x_0 + m', x_0 + 2m', ..., x_0 + (g-1)m'`。我们通常只需要一个最小非负解。

**回到合并同余方程**：
1.  解 `k * m1 = a2 - a1 (mod m2)` 得到 `k` 的一个特解 `k0` (模 `m2/gcd(m1, m2)` 意义下)。
2.  那么 `x = k0 * m1 + a1` 是一个特解。
3.  新的模数是 `lcm(m1, m2) = (m1 * m2) / gcd(m1, m2)`。
4.  所以合并后的方程是 `x = (k0 * m1 + a1) % lcm(m1, m2) (mod lcm(m1, m2))`。
5.  重复这个过程，直到所有方程合并完毕。

**常见误区**：
*   计算 `a2 - a1` 时可能出现负数，需要 `(a2 - a1) % m2` 之后再 `+ m2` 确保结果为正。
*   扩展欧几里得求逆元时，如果 `x0` 是负数，一定要 `(x0 % m + m) % m`。
*   合并方程时，新的模数是 `lcm(m1, m2)`，而不是 `m1 * m2`。
*   线性同余方程无解的情况判断。

### 3. 核心算法实现

*   **快速幂 (power)**: `long long power(long long b, long long e, long long m)`
*   **欧拉函数 (phi)**: `long long phi(long long n)` (质因数分解法)
*   **扩展欧几里得 (extended_gcd)**: `long long extended_gcd(long long a, long long b, long long &x, long long &y)`
*   **乘法逆元 (inv)**: 基于 `extended_gcd`
*   **大数读取并模 (read_and_mod)**: `long long read_and_mod(string s, long long m, bool &is_large)`
*   **CRT合并 (CRT_merge)**: `pair<long long, long long> CRT_merge(long long a1, long long m1, long long a2, long long m2)`

### 例题1: 超大指数模幂 (洛谷 P5091 / POJ 1995)

**题目大意**：
给定三个正整数 `a, m` 和一个可能非常大的正整数 `b`（以字符串形式给出，长度可达100000位）。求 `a^b % m` 的值。

**输入格式**：
一行，三个整数 `a, m`。
一行，一个字符串 `b`。

**输出格式**：
一个整数，表示 `a^b % m` 的结果。

**示例**：
输入
2 10
3
输出
8

输入
2 10
12
输出
4

**思路分析**：
这道题直接考察扩展欧拉定理。
1.  **计算 `phi(m)`**：使用质因数分解的方法计算 `m` 的欧拉函数值 `phi_m`。
2.  **处理指数 `b`**：由于 `b` 是字符串，我们需要将其转换为 `long long` 类型，并在转换过程中对 `phi_m` 取模。同时，需要判断原始的 `b` 是否大于等于 `phi_m`。
    *   定义一个 `read_and_mod` 函数，输入字符串 `b_str` 和模数 `mod_val`。
    *   在循环遍历 `b_str` 的每个字符时，`res = (res * 10 + (c - '0'))`。
    *   每次计算 `res` 后，如果 `res >= mod_val`，则 `res %= mod_val`，并设置一个 `is_large` 标志为 `true`。
    *   最终 `read_and_mod` 返回 `res` 和 `is_large` 标志。
3.  **应用扩展欧拉定理**：
    *   如果 `is_large` 为 `true` (即原始 `b` 大于等于 `phi_m`)，则实际使用的指数 `exp = read_and_mod(b_str, phi_m).first + phi_m`。
    *   如果 `is_large` 为 `false` (即原始 `b` 小于 `phi_m`)，则实际使用的指数 `exp = read_and_mod(b_str, phi_m).first`。
4.  **快速幂**：使用 `power(a, exp, m)` 计算最终结果。

**代码实现**：

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <cmath> // For sqrt

// 使用 long long 避免溢出
typedef long long ll;

// 快速幂函数: 计算 (b^e) % m
ll power(ll b, ll e, ll m) {
    ll r = 1;
    b %= m; // 提前取模，防止中间结果过大
    while (e > 0) {
        if (e % 2 == 1) { // 如果指数是奇数
            r = (r * b) % m;
        }
        b = (b * b) % m; // 基数平方
        e /= 2;          // 指数减半
    }
    return r;
}

// 欧拉函数 phi(n): 计算小于等于 n 且与 n 互质的正整数的个数
// 使用质因数分解法
ll phi(ll n) {
    ll r = n;
    for (ll i = 2; i * i <= n; ++i) {
        if (n % i == 0) { // i 是 n 的一个质因数
            r = r / i * (i - 1); // phi(n) = n * product(1 - 1/p)
            while (n % i == 0) {
                n /= i; // 除尽所有 i 因子
            }
        }
    }
    if (n > 1) { // 如果 n 还有剩余的质因数 (n本身就是质数或剩余质数)
        r = r / n * (n - 1);
    }
    return r;
}

// 读取大数字符串 b 并对 mod_val 取模，同时判断 b 是否大于等于 mod_val
// 返回 pair<取模结果, 是否大于等于>
std::pair<ll, bool> read_and_mod(const std::string& s, ll mod_val) {
    ll r = 0;
    bool is_large = false;
    for (char c : s) {
        r = r * 10 + (c - '0');
        if (r >= mod_val) { // 如果当前结果已经大于等于模数，说明原始数肯定大于等于模数
            is_large = true;
            r %= mod_val;
        }
    }
    return {r, is_large};
}

int main() {
    std::ios_base::sync_with_stdio(false); // 加速输入输出
    std::cin.tie(NULL);

    ll a, m;
    std::cin >> a >> m;
    std::string s;
    std::cin >> s;

    ll phi_m = phi(m); // 计算欧拉函数 phi(m)

    // 读取大数 b 并对 phi_m 取模，同时判断 b 是否大于等于 phi_m
    std::pair<ll, bool> b_info = read_and_mod(s, phi_m);
    ll exp_val = b_info.first; // b % phi_m 的结果
    bool b_is_ge_phi_m = b_info.second; // b 是否 >= phi_m

    // 根据扩展欧拉定理确定最终指数
    if (b_is_ge_phi_m) {
        exp_val += phi_m; // 如果 b >= phi_m，则指数需要加上 phi_m
    }

    // 计算 a^exp_val % m
    ll ans = power(a, exp_val, m);

    std::cout << ans << std::endl;

    return 0;
}

```

**复杂度分析**：
*   **`phi(m)` 函数**：质因数分解 `m`。最坏情况下 `m` 是质数，需要 `O(sqrt(m))` 时间。
*   **`read_and_mod` 函数**：遍历字符串 `s`，长度为 `L`。`O(L)` 时间。
*   **`power` 函数**：快速幂，时间复杂度为 `O(log(exp_val))`。由于 `exp_val` 最大为 `phi_m + phi_m`，所以是 `O(log(phi_m))`。
*   **总时间复杂度**：`O(sqrt(m) + L + log(phi_m))`。对于 `m` 10^9，`sqrt(m)` 约 3*10^4。`L` 10^5。`log(phi_m)` 约 60。所以主要取决于 `L` 和 `sqrt(m)` 中较大的一个。这在竞赛中是高效的。

### 例题2: 线性同余方程组 (洛谷 P4777 / SPOJ CRITICAL)

**题目大意**：
给定 `n` 组同余方程，形如 `xi = ai (mod bi)`。求满足所有方程的最小非负整数 `x`。
`n` 最大为 100000，`ai, bi` 都在 `10^12` 级别。

**输入格式**：
第一行一个整数 `n`。
接下来 `n` 行，每行两个整数 `ai, bi`。

**输出格式**：
一个整数，表示满足条件的最小非负整数 `x`。如果无解，输出 -1。

**示例**：
输入
3
2 3
3 5
2 7
输出
23

**思路分析**：
由于 `bi` 不一定两两互质，我们不能直接使用标准的 CRT 公式。我们需要使用逐步合并的方法。
假设我们已经合并了前 `i-1` 个方程，得到一个形如 `x = a_curr (mod m_curr)` 的解。
现在我们要合并第 `i` 个方程 `x = a_i (mod b_i)`。
这两个方程可以写成：
1.  `x = k * m_curr + a_curr`
2.  `x = l * b_i + a_i`
联立得到：`k * m_curr + a_curr = l * b_i + a_i`
移项：`k * m_curr - l * b_i = a_i - a_curr`
这是一个形如 `A*X + B*Y = C` 的不定方程，其中 `A = m_curr`, `X = k`, `B = -b_i`, `Y = l`, `C = a_i - a_curr`。
我们可以使用**扩展欧几里得算法**来求解 `k * m_curr + l * b_i = gcd(m_curr, b_i)`。
设 `g = gcd(m_curr, b_i)`。
1.  **判断有解条件**：如果 `(a_i - a_curr)` 不能被 `g` 整除，那么方程组无解。直接输出 -1 并退出。
2.  **求解 `k`**：
    *   使用 `extended_gcd(m_curr, b_i, x0, y0)` 得到 `x0, y0` 使得 `m_curr * x0 + b_i * y0 = g`。
    *   将方程两边乘以 `(a_i - a_curr) / g`，得到 `m_curr * (x0 * (a_i - a_curr) / g) + b_i * (y0 * (a_i - a_curr) / g) = a_i - a_curr`。
    *   此时 `k` 的一个特解是 `k0 = x0 * (a_i - a_curr) / g`。
    *   由于 `k` 是在模 `b_i / g` 意义下的，所以 `k = (k0 % (b_i / g) + (b_i / g)) % (b_i / g)`，确保 `k` 为最小非负解。
    *   这里需要注意 `a_i - a_curr` 可能是负数，所以 `(a_i - a_curr) / g` 也可能是负数。在计算 `k0` 后，需要对 `b_i/g` 取模并调整为正数。
3.  **更新 `x` 和 `m`**：
    *   新的 `x` 为 `k * m_curr + a_curr`。
    *   新的模数 `m_new = lcm(m_curr, b_i) = (m_curr / g) * b_i`。
    *   将 `x` 对 `m_new` 取模，得到 `a_new = (x % m_new + m_new) % m_new`。
    *   更新 `a_curr = a_new`, `m_curr = m_new`。
4.  **乘法防溢出**：由于 `m_curr` 和 `b_i` 都在 `10^12` 级别，它们的乘积可能超过 `long long` 的范围。在计算 `m_curr * x0` 和 `(m_curr / g) * b_i` 时，需要使用**龟速乘**（或者叫快速乘，`mul(A, B, M)` 计算 `(A * B) % M`）来防止溢出。

**代码实现**：

```cpp
#include <iostream>
#include <vector>
#include <numeric> // For std::gcd in C++17, otherwise implement manually

// 使用 long long 避免溢出
typedef long long ll;

// 扩展欧几里得算法: 计算 gcd(a, b)，并找到 x, y 使得 ax + by = gcd(a, b)
ll extended_gcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    ll g = extended_gcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}

// 快速乘 (龟速乘): 计算 (a * b) % m，防止中间结果溢出
ll mul(ll a, ll b, ll m) {
    ll r = 0;
    a %= m; // 提前取模
    while (b > 0) {
        if (b % 2 == 1) {
            r = (r + a) % m;
        }
        a = (a + a) % m;
        b /= 2;
    }
    return r;
}

// 合并两个同余方程 x = a1 (mod m1) 和 x = a2 (mod m2)
// 返回合并后的 (a_new, m_new)，如果无解则返回 (-1, -1)
std::pair<ll, ll> CRT_merge(ll a1, ll m1, ll a2, ll m2) {
    ll x0, y0;
    ll g = extended_gcd(m1, m2, x0, y0); // m1 * x0 + m2 * y0 = g

    // 如果 (a2 - a1) 不能被 g 整除，则无解
    if ((a2 - a1) % g != 0) {
        return {-1, -1};
    }

    // 求解 k * m1 = (a2 - a1) (mod m2)
    // 此时 k 的特解 x_k = x0 * ((a2 - a1) / g)
    // 注意 (a2 - a1) / g 可能是负数，所以 x_k 也要调整
    // 并且 k 是在模 (m2 / g) 意义下的
    ll k_mod = m2 / g;
    ll k = mul(x0, (a2 - a1) / g, k_mod); // 计算 k 的特解
    k = (k % k_mod + k_mod) % k_mod;     // 调整 k 为最小非负解

    // 计算新的 a 和 m
    // x = k * m1 + a1
    ll a_new = (mul(k, m1, m1 * k_mod) + a1); // x 的一个特解
    ll m_new = mul(m1 / g, m2, m1 * k_mod);   // 新的模数 lcm(m1, m2) = (m1 * m2) / g

    a_new = (a_new % m_new + m_new) % m_new; // 保证 a_new 为最小非负解

    return {a_new, m_new};
}

int main() {
    std::ios_base::sync_with_stdio(false); // 加速输入输出
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    ll current_a, current_m;
    std::cin >> current_a >> current_m; // 第一个方程

    for (int i = 1; i < n; ++i) {
        ll next_a, next_m;
        std::cin >> next_a >> next_m;

        std::pair<ll, ll> result = CRT_merge(current_a, current_m, next_a, next_m);

        if (result.first == -1) { // 如果合并过程中出现无解
            std::cout << -1 << std::endl;
            return 0;
        }
        current_a = result.first;
        current_m = result.second;
    }

    std::cout << current_a << std::endl;

    return 0;
}

```

**复杂度分析**：
*   **`extended_gcd` 函数**：`O(log(min(a, b)))`。
*   **`mul` (龟速乘) 函数**：`O(log(b))`。
*   **`CRT_merge` 函数**：主要由 `extended_gcd` 和 `mul` 决定，复杂度为 `O(log(min(m1, m2))) + O(log(k))`。
*   **主循环**：`n` 次调用 `CRT_merge`。
*   **总时间复杂度**：`O(n * (log(max(m_i)) + log(k_i)))`，其中 `k_i` 是乘法中的指数。由于模数可能达到 `10^12`，`log` 操作约 40-60 次。`n` 最大 100000。所以总复杂度约为 `10^5 * (60 + 60)`，在可接受范围内。
*   **空间复杂度**：`O(1)` (不考虑输入存储)。
