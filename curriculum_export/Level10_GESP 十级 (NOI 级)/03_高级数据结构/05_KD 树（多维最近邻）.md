## 深度知识点讲解

### 1. 什么是KD树？（K-Dimensional Tree）

KD树，全称K-Dimensional Tree，翻译过来就是“K维树”。它是一种用于组织K维空间中的点的二叉搜索树。

*   **核心用途：** 主要用来解决多维空间中的各种查询问题，最常见的就是“最近邻搜索”（Nearest Neighbor Search）和“范围搜索”（Range Search）。
*   **本质：** 我们可以把它想象成一个“多维二叉搜索树”。在普通的二叉搜索树中，我们根据一个键值（比如大小）来决定是去左子树还是右子树。在KD树中，我们是根据**点在某一特定维度上的坐标值**来决定去左子树还是右子树，从而把K维空间不断地一分为二。

### 2. 为什么需要KD树？

*   **一维问题：** 简单！用二叉搜索树（BST）就能搞定，`O(log N)` 查找。
*   **二维问题：** 四叉树（Quadtree）是一个不错的选择，它将二维平面递归地划分为四个象限。但四叉树每次都把空间分成2^K份，对于高维来说，分支因子会变得非常大（2^K），导致空间和时间效率降低。
*   **高维问题（K > 2）：** BST和四叉树都不太适用了。BST只能处理一维。四叉树在高维空间中，一个节点会有 2^K 个子节点，这会变得非常庞大和低效。我们需要一种更灵活、更通用的划分方式。
*   **KD树的优势：** 它巧妙地将高维空间划分问题，转化为一系列低维（一维）的二叉分割问题。每次只沿着一个维度进行分割，保持了二叉树的结构，使得其在高维空间中依然能保持相对高效的查询性能。

### 3. KD树的核心思想：交替维度分割与中位数划分

KD树的构建和查询都围绕着这两个核心思想：

#### 3.1 交替维度分割 (Alternating Dimensions)

*   **想象：** 你有一个大蛋糕，你第一次切，可以横着切。第二次切，就不能再横着切了，得竖着切。第三次，再横着切。
*   **KD树：** 在构建KD树时，我们不是每次都用同一个维度去分割空间。而是轮流选择不同的维度进行分割。
    *   根节点：可能根据第0维（比如x坐标）进行分割。
    *   根的子节点：可能根据第1维（比如y坐标）进行分割。
    *   再下一层：可能根据第2维（比如z坐标）进行分割。
    *   ...
    *   当所有K个维度都轮流一遍后，再回到第0维继续。
*   **好处：** 这样可以确保每个维度都被“照顾”到，使得空间在各个维度上都得到均匀的划分，避免出现“瘦高”或“扁平”的子树，从而保持树的平衡性。

#### 3.2 中位数划分 (Median Split)

*   **想象：** 你要切一堆披萨，为了让两边差不多大，你最好从中间切。
*   **KD树：** 当我们选择了一个维度（比如x轴）进行分割时，我们不是随便找个点切，而是找到当前所有点在这个维度上的**中位数**。以这个中位数点作为分割点，将比它小的点分到左子树，比它大的点分到右子树。
*   **好处：** 使用中位数可以最大程度地平衡左右子树的大小，使得KD树尽可能地保持平衡，从而保证查询效率接近 `O(log N)`。
*   **如何找中位数？**
    *   最直接的方法是先对所有点在当前维度上进行排序，然后取中间的那个点。
    *   更高效的方法是使用 `std::nth_element` 函数（C++标准库），它可以在线性时间 `O(N)` 内找到第k小的元素，并将其放到正确的位置，同时保证它左边的元素都小于等于它，右边的元素都大于等于它。这比完全排序 `O(N log N)` 快得多。

### 4. KD树的构建 (Build)

KD树的构建是一个递归过程：

1.  **输入：** 一组点 `P`，当前处理的维度 `d`。
2.  **终止条件：** 如果 `P` 为空，返回空节点。
3.  **选择分割维度：** `dim = d % K` (K是总维度数)。
4.  **选择分割点：**
    *   在 `P` 中，找到在 `dim` 维度上的中位数点 `m`。
    *   为了找到中位数，我们可以使用 `std::nth_element`。将 `P` 中的点按照 `dim` 维度上的坐标值进行比较，找到中位数，并将其交换到 `P` 的中间位置。
5.  **创建节点：** 创建一个KD树节点，存储点 `m`，并记录分割维度 `dim`。
6.  **递归构建左右子树：**
    *   左子树：由 `P` 中所有在 `dim` 维度上小于 `m` 的点构成，递归调用 `build(P_left, d+1)`。
    *   右子树：由 `P` 中所有在 `dim` 维度上大于 `m` 的点构成，递归调用 `build(P_right, d+1)`。
    *   将左右子树连接到当前节点。
7.  **返回节点。**

**注意：** 为了方便实现，我们通常会维护一个全局的点数组 `points[]`。`build` 函数接收一个点数组的范围 `[l, r]` 和当前维度 `d`。`std::nth_element` 直接作用在这个数组的 `[l, r]` 范围内。

### 5. KD树的查询 (Query)

最常用的查询是“最近邻查询”（Nearest Neighbor Search）。目标是找到离给定查询点 `target` 最近的点。

这是一个典型的递归搜索过程，关键在于**剪枝优化**。

1.  **输入：** 当前KD树节点 `u`，查询点 `target`，以及当前最小距离的平方 `ans_d2` (全局变量)。
2.  **计算距离：** 计算节点 `u` 存储的点 `p[u]` 到 `target` 的距离平方 `d2`。如果 `d2 < ans_d2`，则更新 `ans_d2` 和最近邻点。
3.  **确定搜索方向：**
    *   设当前节点的分割维度为 `dim = p[u].dim`。
    *   如果 `target` 在 `dim` 维度上的坐标小于 `p[u]` 在 `dim` 维度上的坐标，那么 `target` 更可能在左子树 `ls` 中。优先搜索 `ls`。
    *   否则，优先搜索右子树 `rs`。
4.  **递归搜索：**
    *   **优先搜索子树：** 递归调用 `query(p[u].ls, target)` 或 `query(p[u].rs, target)`。
    *   **剪枝判断（Pruning）：** 这是KD树查询效率的关键！
        *   在搜索完优先子树后，我们需要考虑是否需要搜索另一个子树。
        *   计算 `target` 点到当前节点 `u` 的分割超平面（即与 `dim` 维度垂直的平面）的**最短距离**。这个距离就是 `(target.coord[dim] - p[u].coord[dim])` 的绝对值。
        *   如果这个最短距离的平方 `(target.coord[dim] - p[u].coord[dim])^2` 已经**大于等于**当前的最小距离平方 `ans_d2`，说明另一侧的子树区域离 `target` 的最近距离都比 `ans_d2` 远，那么就没有必要去搜索另一侧的子树了，直接剪枝返回！
        *   否则，需要继续搜索另一个子树。

**更精细的剪枝（边界盒剪枝）：**
为了更精确地剪枝，每个KD树节点可以额外存储其所代表区域的**超矩形边界**（min/max坐标）。
1.  计算 `target` 点到当前节点所代表的超矩形区域的**最短距离平方** `min_dist_to_box_d2`。
2.  如果 `min_dist_to_box_d2 >= ans_d2`，则整个区域内的点都比 `ans_d2` 远，直接剪枝，无需进入当前节点的任何子树。
这种剪枝方式更强大，因为它可以提前剪掉整个不相关的区域，而不仅仅是基于分割超平面。

### 6. KD树的复杂度分析

*   **构建 (Build):**
    *   每一层都需要对点集进行 `nth_element` 操作。`nth_element` 的时间复杂度是线性的 `O(N)`。
    *   树的深度为 `log N`。
    *   因此，总的构建时间复杂度为 `O(N log N)`。
*   **查询 (Nearest Neighbor Search):**
    *   **平均情况：** 在随机数据下，KD树查询的平均时间复杂度通常是 `O(sqrt(N))` 或 `O(log N)` (对于低维数据)。这是因为剪枝操作大大减少了需要访问的节点数量。
    *   **最坏情况：** 如果数据分布非常不均匀（例如所有点都在一条直线上），KD树可能会退化成链表，查询时间复杂度会退化到 `O(N)`。
*   **空间复杂度：** 存储N个点，每个节点存储点的信息和子节点指针，因此空间复杂度为 `O(N)`。

### 7. KD树的优点与局限性

*   **优点：**
    *   适用于多维空间数据，特别是低维（2-5维）数据查询效率高。
    *   构建相对简单，平均查询性能优秀。
    *   支持多种查询类型（最近邻、范围查询）。
*   **局限性：**
    *   **维度灾难：** 当维度K非常高时（K > 20），KD树的性能会急剧下降，剪枝效果不明显，查询时间趋近 `O(N)`。
    *   **动态性差：** 插入和删除操作会导致树的结构失衡，需要进行复杂的重新平衡或重建，效率不高。通常用于静态数据集。
    *   **最坏情况：** 存在退化为链表的风险。

## 典型例题精讲

### 例题1：二维平面最近邻查询（基本构建与查询）

**题目描述：**
给定N个二维平面上的点 (x, y)。现在有M次查询，每次查询给定一个点 (qx, qy)，你需要找出这N个点中离 (qx, qy) 欧几里得距离最近的点，并输出其距离的平方。

**输入格式：**
第一行两个整数 N, M (1 <= N, M <= 100000)。
接下来 N 行，每行两个整数 x, y，表示一个点的坐标。
接下来 M 行，每行两个整数 qx, qy，表示一个查询点的坐标。

**输出格式：**
共 M 行，每行一个整数，表示对应查询点的最近邻距离平方。

**思路分析：**

1.  **数据结构选择：** N和M都比较大，每次查询如果遍历所有点是 `O(N)`，总查询 `O(M*N)` 会超时。我们需要一个更高效的数据结构。KD树正是解决这类多维最近邻查询的利器。
2.  **KD树的构建：**
    *   我们需要一个 `Point` 结构体来存储点的坐标（`x`, `y`）以及在KD树中的左右子节点索引 `ls`, `rs`。
    *   `build(l, r, d)` 函数：
        *   参数：`l` 和 `r` 表示当前处理的点在全局数组中的范围，`d` 表示当前分割的维度（0代表x，1代表y）。
        *   首先处理递归终止条件：如果 `l > r`，返回0（表示空节点）。
        *   计算中点 `mid = (l + r) / 2`。
        *   使用 `std::nth_element` 对 `points[l...r]` 范围内的点，根据 `d` 维度进行排序，将中位数点放到 `points[mid]` 的位置。
        *   创建新节点：`points[mid]` 就是当前子树的根节点。记录其分割维度 `d`。
        *   递归构建左右子树：`points[mid].ls = build(l, mid-1, (d+1)%2)`，`points[mid].rs = build(mid+1, r, (d+1)%2)`。
        *   返回 `mid` 作为当前子树的根节点索引。
3.  **最近邻查询：**
    *   `query(u, d)` 函数：
        *   参数：`u` 表示当前访问的KD树节点索引，`d` 表示当前节点 `u` 的分割维度。
        *   更新全局最小距离 `ans_d2`：计算 `points[u]` 到 `target` 的距离平方，如果更小就更新。
        *   判断 `target` 更偏向哪个子树：`target.p[d]` 与 `points[u].p[d]` 比较。
        *   优先递归搜索偏向的子树。
        *   **剪枝：** 在搜索完优先子树后，计算 `target` 到分割超平面（`points[u].p[d]`）的距离平方。如果这个距离平方小于 `ans_d2`，说明另一个子树区域内可能存在更近的点，需要继续搜索另一个子树。
        *   **注意：** 距离平方计算避免浮点数误差，用 `long long`。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

// 定义点的最大数量
const int MAXN = 100005;
const int K = 2; // 维度数量，这里是二维

// 定义点结构体
struct Point {
    long long p[K]; // 坐标，p[0]为x，p[1]为y
    int ls, rs;     // 左右子节点索引
    // 用于nth_element的比较函数，根据维度d进行比较
    bool operator<(const Point& o) const {
        return p[dim_cmp] < o.p[dim_cmp];
    }
} a[MAXN]; // 全局点数组，a[0]不用，从a[1]开始

int n_nodes; // KD树节点总数
int dim_cmp; // 当前比较的维度

// 计算两个点之间的欧几里得距离的平方
long long dist2(const Point& p1, const Point& p2) {
    long long res = 0;
    for (int i = 0; i < K; ++i) {
        res += (p1.p[i] - p2.p[i]) * (p1.p[i] - p2.p[i]);
    }
    return res;
}

// 构建KD树
// l, r: 当前处理点在a数组中的范围
// d: 当前分割维度 (0或1)
int build(int l, int r, int d) {
    if (l > r) {
        return 0; // 空节点
    }
    int mid = (l + r) / 2;
    dim_cmp = d; // 设置当前比较维度
    // 使用nth_element找到中位数，并将其放到mid位置
    std::nth_element(a + l, a + mid, a + r + 1);

    a[mid].ls = build(l, mid - 1, (d + 1) % K); // 递归构建左子树
    a[mid].rs = build(mid + 1, r, (d + 1) % K); // 递归构建右子树
    return mid; // 返回当前子树的根节点索引
}

Point target_q; // 查询目标点
long long min_d2; // 全局最小距离平方

// KD树最近邻查询
// u: 当前KD树节点索引
// d: 当前节点u的分割维度
void query(int u, int d) {
    if (u == 0) {
        return; // 空节点
    }

    // 1. 更新当前最小距离
    min_d2 = std::min(min_d2, dist2(target_q, a[u]));

    // 计算查询点target_q到当前节点a[u]的分割超平面的距离平方
    long long delta = target_q.p[d] - a[u].p[d];
    long long delta_d2 = delta * delta;

    int ls = a[u].ls, rs = a[u].rs;
    int next_d = (d + 1) % K; // 下一层节点的分割维度

    // 2. 确定搜索方向并剪枝
    if (target_q.p[d] < a[u].p[d]) { // 目标点在左子树区域
        query(ls, next_d); // 优先搜索左子树
        if (delta_d2 < min_d2) { // 如果另一侧可能存在更近的点 (剪枝判断)
            query(rs, next_d); // 搜索右子树
        }
    } else { // 目标点在右子树区域
        query(rs, next_d); // 优先搜索右子树
        if (delta_d2 < min_d2) { // 如果另一侧可能存在更近的点 (剪枝判断)
            query(ls, next_d); // 搜索左子树
        }
    }
}

int main() {
    // 提高输入输出效率
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i].p[0] >> a[i].p[1];
    }

    // 构建KD树，根节点为build(1, n, 0)返回的索引
    int root = build(1, n, 0);

    for (int i = 0; i < m; ++i) {
        std::cin >> target_q.p[0] >> target_q.p[1];
        min_d2 = LLONG_MAX; // 初始化最小距离为最大值
        query(root, 0); // 从根节点开始查询
        std::cout << min_d2 << "\n";
    }

    return 0;
}
```

**复杂度分析：**
*   **构建：** `O(N log N)`，因为有 `log N` 层递归，每层调用 `nth_element` 消耗 `O(N)`。
*   **查询：** 平均 `O(sqrt(N))`，最坏 `O(N)`。总查询 `O(M * sqrt(N))`。
*   **空间：** `O(N)`。

### 例题2：P4148 [AHOI2012] 树屋建造 (静态K-D Tree 范围查询)

**题目描述（简化，原题是动态的，我们只考虑静态部分）：**
给定N个二维平面上的点 (x, y)。现在有M次查询，每次查询给定一个矩形区域 (x1, y1) 到 (x2, y2)，你需要找出在这个矩形区域内有多少个点。

**输入格式：**
第一行一个整数 N (1 <= N <= 100000)。
接下来 N 行，每行两个整数 x, y，表示一个点的坐标。
接下来一个整数 M (1 <= M <= 100000)。
接下来 M 行，每行四个整数 x1, y1, x2, y2，表示一个查询矩形的左下角和右上角坐标。

**输出格式：**
共 M 行，每行一个整数，表示对应查询矩形内的点数量。

**思路分析：**

1.  **数据结构选择：** 同上，KD树适用于多维空间查询。这次是范围查询。
2.  **KD树的构建：** 同例题1，构建方法不变。
3.  **范围查询：**
    *   `query_range(u, d, x1, y1, x2, y2)` 函数：
        *   参数：`u` 当前节点索引，`d` 当前节点分割维度，`x1, y1, x2, y2` 查询矩形。
        *   **终止条件：** 如果 `u == 0`，返回0。
        *   **剪枝（关键）：**
            *   为了进行高效剪枝，每个KD树节点 `a[u]` 需要维护一个**边界盒**（Bounding Box），即以 `a[u]` 为根的子树所包含的所有点的最小 `x`、最大 `x`、最小 `y`、最大 `y`。我们可以在 `build` 函数的返回前计算并存储这些边界信息。
            *   判断当前节点 `a[u]` 所代表的边界盒是否**完全不相交**于查询矩形 `(x1, y1, x2, y2)`。如果不相交，则整个子树都不可能有满足条件的点，直接返回0。
            *   `a[u].min_x > x2 || a[u].max_x < x1 || a[u].min_y > y2 || a[u].max_y < y1`
        *   **统计当前点：** 判断 `a[u]` 点本身是否在查询矩形内。
            *   `a[u].p[0] >= x1 && a[u].p[0] <= x2 && a[u].p[1] >= y1 && a[u].p[1] <= y2`
            *   如果是，`count++`。
        *   **递归搜索子树：** 递归调用 `query_range(a[u].ls, (d+1)%K, x1, y1, x2, y2)` 和 `query_range(a[u].rs, (d+1)%K, x1, y1, x2, y2)`，并累加结果。

**额外结构体成员 (Bounding Box):**
在 `Point` 结构体中添加：
`long long min_p[K], max_p[K]; // 边界盒的最小/最大坐标`

**在 `build` 函数中更新边界盒：**
```cpp
// 在build函数中，mid节点构建完成后，计算并更新其边界盒
// 这需要递归地从子节点获取边界盒信息
void push_up(int u) {
    // 初始化为当前点
    for (int i = 0; i < K; ++i) {
        a[u].min_p[i] = a[u].max_p[i] = a[u].p[i];
    }
    // 合并左子树的边界盒
    if (a[u].ls) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].ls].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].ls].max_p[i]);
        }
    }
    // 合并右子树的边界盒
    if (a[u].rs) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].rs].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].rs].max_p[i]);
        }
    }
}

// build函数末尾：
// ...

**代码实现 (仅展示 `query_range` 部分和 `Point` 结构体修改)：**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;
const int K = 2; // 维度数量

struct Point {
    long long p[K];     // 坐标
    int ls, rs;         // 左右子节点索引
    long long min_p[K], max_p[K]; // 边界盒的最小/最大坐标

    bool operator<(const Point& o) const {
        return p[dim_cmp] < o.p[dim_cmp];
    }
} a[MAXN];

int dim_cmp;

void push_up(int u) {
    for (int i = 0; i < K; ++i) {
        a[u].min_p[i] = a[u].max_p[i] = a[u].p[i];
    }
    if (a[u].ls) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].ls].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].ls].max_p[i]);
        }
    }
    if (a[u].rs) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].rs].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].rs].max_p[i]);
        }
    }
}

int build(int l, int r, int d) {
    if (l > r) return 0;
    int mid = (l + r) / 2;
    dim_cmp = d;
    std::nth_element(a + l, a + mid, a + r + 1);

    a[mid].ls = build(l, mid - 1, (d + 1) % K);
    a[mid].rs = build(mid + 1, r, (d + 1) % K);
    push_up(mid); // 计算边界盒
    return mid;
}

// 检查点p是否在查询矩形内
bool check_point_in_range(const Point& p, long long x1, long long y1, long long x2, long long y2) {
    return p.p[0] >= x1 && p.p[0] <= x2 &&
           p.p[1] >= y1 && p.p[1] <= y2;
}

// KD树范围查询
// u: 当前KD树节点索引
// d: 当前节点u的分割维度
// x1, y1, x2, y2: 查询矩形的左下角和右上角坐标
int query_range(int u, int d, long long x1, long long y1, long long x2, long long y2) {
    if (u == 0) return 0;

    // 剪枝1: 如果当前节点所代表的区域完全在查询矩形之外，直接返回0
    if (a[u].max_p[0] < x1 || a[u].min_p[0] > x2 ||
        a[u].max_p[1] < y1 || a[u].min_p[1] > y2) {
        return 0;
    }

    // 剪枝2: 如果当前节点所代表的区域完全在查询矩形之内，则所有点都满足条件
    // 返回该子树包含的节点数量 (这里需要额外维护子树大小)
    // 为了简化，我们暂时不维护子树大小，而是继续递归。
    // 实际竞赛中，维护子树大小可以大幅加速。
    // if (a[u].min_p[0] >= x1 && a[u].max_p[0] <= x2 &&
    //     a[u].min_p[1] >= y1 && a[u].max_p[1] <= y2) {
    //     return a[u].size; // 假设a[u].size存储子树大小
    // }

    int count = 0;
    // 检查当前节点存储的点是否在查询矩形内
    if (check_point_in_range(a[u], x1, y1, x2, y2)) {
        count++;
    }

    // 递归搜索左右子树
    count += query_range(a[u].ls, (d + 1) % K, x1, y1, x2, y2);
    count += query_range(a[u].rs, (d + 1) % K, x1, y1, x2, y2);

    return count;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;
    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i].p[0] >> a[i].p[1];
    }

    int root = build(1, n, 0);

    int m;
    std::cin >> m;
    for (int i = 0; i < m; ++i) {
        long long x1, y1, x2, y2;
        std::cin >> x1 >> y1 >> x2 >> y2;
        std::cout << query_range(root, 0, x1, y1, x2, y2) << "\n";
    }

    return 0;
}
```
**复杂度分析：**
*   **构建：** `O(N log N)`。
*   **查询：** 平均 `O(sqrt(N))`，最坏 `O(N)`。总查询 `O(M * sqrt(N))`。
*   **空间：** `O(N)`。

**注意：** 如果要实现更强大的“完全包含剪枝”，即 `if (a[u].min_p[0] >= x1 && a[u].max_p[0] <= x2 && ...)`，需要每个节点额外维护一个 `size` 字段，表示以该节点为根的子树中点的数量。在 `push_up` 函数中更新 `a[u].size = 1 + a[a[u].ls].size + a[a[u].rs].size`。

## 代码实现模板

下面是一个更通用的KD树模板，适用于K维最近邻查询。

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件，如iostream, algorithm, limits等

// 定义维度数量，可以根据题目要求修改
const int K = 2; // 例如，二维是2，三维是3

// 定义点的最大数量
const int MAXN = 100005;

// 定义点结构体
struct Point {
    long long p[K]; // 存储K维坐标
    int ls, rs;     // 左右子节点在全局数组中的索引
    // 边界盒信息 (可选，但对于范围查询和更精细的最近邻剪枝很有用)
    long long min_p[K], max_p[K]; 

    // 用于nth_element的比较函数，根据全局变量dim_cmp指定的维度进行比较
    bool operator<(const Point& o) const {
        return p[dim_cmp] < o.p[dim_cmp];
    }
};

Point a[MAXN]; // 全局点数组，a[0]通常作为空节点或不使用
int dim_cmp;   // 全局变量，用于指示当前nth_element或比较的维度

// 计算两个K维点之间的欧几里得距离的平方
long long dist2(const Point& p1, const Point& p2) {
    long long res = 0;
    for (int i = 0; i < K; ++i) {
        res += (p1.p[i] - p2.p[i]) * (p1.p[i] - p2.p[i]);
    }
    return res;
}

// 更新节点u的边界盒信息
// 在build函数中，当左右子树构建完成后，调用此函数更新父节点的边界
void push_up(int u) {
    // 初始化为当前点a[u]的坐标
    for (int i = 0; i < K; ++i) {
        a[u].min_p[i] = a[u].max_p[i] = a[u].p[i];
    }
    // 如果存在左子树，合并左子树的边界盒
    if (a[u].ls) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].ls].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].ls].max_p[i]);
        }
    }
    // 如果存在右子树，合并右子树的边界盒
    if (a[u].rs) {
        for (int i = 0; i < K; ++i) {
            a[u].min_p[i] = std::min(a[u].min_p[i], a[a[u].rs].min_p[i]);
            a[u].max_p[i] = std::max(a[u].max_p[i], a[a[u].rs].max_p[i]);
        }
    }
}

// 构建KD树
// l, r: 当前处理点在a数组中的范围 [l, r]
// d: 当前分割维度 (0到K-1)
// 返回当前子树的根节点在a数组中的索引
int build(int l, int r, int d) {
    if (l > r) {
        return 0; // 范围无效，返回空节点索引
    }
    int mid = (l + r) / 2; // 选择中间点作为分割点
    dim_cmp = d; // 设置当前比较维度
    // 使用std::nth_element将第mid小的元素（中位数）放到a[mid]的位置
    // 并且保证a[l...mid-1]的元素都小于等于a[mid]，a[mid+1...r]都大于等于a[mid]
    std::nth_element(a + l, a + mid, a + r + 1);

    // 递归构建左右子树
    a[mid].ls = build(l, mid - 1, (d + 1) % K); // 左子树使用下一个维度
    a[mid].rs = build(mid + 1, r, (d + 1) % K); // 右子树使用下一个维度

    push_up(mid); // 更新当前节点的边界盒信息
    return mid; // 返回当前子树的根节点索引
}

// ----------- KD树最近邻查询相关全局变量 -----------
Point target_q; // 全局查询目标点
long long min_d2; // 全局最小距离平方

// KD树最近邻查询
// u: 当前KD树节点索引
// d: 当前节点u的分割维度
void query_nearest(int u, int d) {
    if (u == 0) {
        return; // 空节点，直接返回
    }

    // 1. 计算当前节点a[u]到查询点target_q的距离平方，更新全局最小距离
    min_d2 = std::min(min_d2, dist2(target_q, a[u]));

    // 计算查询点target_q到当前节点a[u]的分割超平面的距离
    long long delta = target_q.p[d] - a[u].p[d];
    long long delta_d2 = delta * delta; // 距离平方

    int ls = a[u].ls, rs = a[u].rs; // 左右子节点索引
    int next_d = (d + 1) % K;       // 下一层节点的分割维度

    // 2. 确定优先搜索方向
    if (target_q.p[d] < a[u].p[d]) { // 目标点在分割超平面左侧（或下侧、前侧）
        query_nearest(ls, next_d); // 优先搜索左子树
        // 剪枝判断：如果当前最小距离比到分割超平面的距离还小，
        // 说明另一侧的子树可能包含更近的点，需要搜索
        if (delta_d2 < min_d2) {
            query_nearest(rs, next_d);
        }
    } else { // 目标点在分割超平面右侧（或上侧、后侧）
        query_nearest(rs, next_d); // 优先搜索右子树
        // 剪枝判断
        if (delta_d2 < min_d2) {
            query_nearest(ls, next_d);
        }
    }
}

// ----------- KD树范围查询相关全局变量 -----------
long long query_x1, query_y1, query_x2, query_y2; // 全局查询矩形边界
int count_in_range; // 全局变量，记录在范围内的点数量

// 检查一个点是否
