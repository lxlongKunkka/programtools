## 深度知识点讲解

### 1. 闵可夫斯基和的定义：形状的“加法”

闵可夫斯基和（Minkowski Sum）听起来有点高深，但它的核心思想非常简单，就是把两个形状“加”起来。

**1.1 点集和的定义**

假设我们有两个点集 $A$ 和 $B$（点集可以是一个点、一条线段、一个多边形，甚至是一个圆），它们的闵可夫斯基和记作 $A \oplus B$，定义为：
$$ A \oplus B = \{ a + b \mid a \in A, b \in B \} $$
这里，$a$ 是集合 $A$ 中的任意一个点（向量），$b$ 是集合 $B$ 中的任意一个点（向量）。$a+b$ 就是这两个点的向量和。

**1.2 几何直观与生活类比**

*   **“刷漆”或“平移”的视角：**
    想象集合 $A$ 是一个画笔的“笔尖形状”，集合 $B$ 是这条画笔要“划过”的路径。那么 $A \oplus B$ 就是画笔划过路径后，所有被“刷到”的区域。
    更精确地说，你可以把 $B$ 中的每一个点都看作一个“原点”，然后把 $A$ 集合的原点平移到 $B$ 中的这个点上。所有这些平移后的 $A$ 的并集，就是 $A \oplus B$。
    反之亦然，你也可以把 $A$ 中的每一个点都看作一个“原点”，然后把 $B$ 集合的原点平移到 $A$ 中的这个点上。所有这些平移后的 $B$ 的并集，也是 $A \oplus B$。

    *   **例子1：点 + 形状**
        如果 $A$ 是一个点 $P_A$， $B$ 是一个多边形 $Q$。那么 $P_A \oplus Q$ 就相当于把多边形 $Q$ 整体平移，使其“原点”（通常指最低最左点，或任意一个参考点）移动到 $P_A$ 点。结果就是多边形 $Q$ 的一个平移版本。

    *   **例子2：线段 + 形状**
        如果 $A$ 是一条线段 $S$， $B$ 是一个多边形 $Q$。那么 $S \oplus Q$ 就是把多边形 $Q$ 沿着线段 $S$ 的方向，从 $S$ 的起点平移到终点，所有 $Q$ 扫过的区域。这就像扫地机器人（形状 $Q$）沿着一条直线路径（线段 $S$）清扫，它清扫过的总区域就是 $S \oplus Q$。

### 2. 闵可夫斯基和的性质：凸性与边的秘密

闵可夫斯基和有很多有趣的性质，其中最重要的就是关于凸性。

**2.1 凸性**

*   **定理：** 如果 $A$ 和 $B$ 都是凸集（例如凸多边形），那么它们的闵可夫斯基和 $A \oplus B$ 也一定是凸集。
*   **思考：** 为什么会这样呢？直观上，凸集没有“凹进去”的部分。把两个没有凹陷的形状“加”起来，也不会凭空产生凹陷。从定义上看，凸集的任意两点连线都在集内。对于 $A \oplus B$ 中的任意两点 $p_1 = a_1 + b_1$ 和 $p_2 = a_2 + b_2$，它们之间的连线上的点 $p_t = (1-t)p_1 + tp_2 = (1-t)(a_1+b_1) + t(a_2+b_2) = ((1-t)a_1+ta_2) + ((1-t)b_1+tb_2)$。由于 $A, B$ 都是凸集，$(1-t)a_1+ta_2 \in A$ 且 $(1-t)b_1+tb_2 \in B$，所以 $p_t \in A \oplus B$。

**2.2 凸多边形的边和顶点**

对于两个凸多边形 $P$ 和 $Q$，它们的闵可夫斯基和 $P \oplus Q$ 也是一个凸多边形。它的顶点和边有什么特点呢？

*   **边的性质：** $P \oplus Q$ 的边集是 $P$ 的边集和 $Q$ 的边集按极角（或方向）排序后合并而成的。
    *   这意味着 $P \oplus Q$ 的边的方向与 $P$ 或 $Q$ 的边的方向相同。
    *   $P \oplus Q$ 的边的长度等于对应 $P$ 或 $Q$ 的边的长度。
    *   $P \oplus Q$ 的总边数等于 $P$ 的边数加上 $Q$ 的边数。

*   **顶点的性质：** $P \oplus Q$ 的每一个顶点都是由 $P$ 的一个顶点和 $Q$ 的一个顶点相加得到的。
    *   具体来说，如果 $p_i$ 是 $P$ 的一个顶点，$q_j$ 是 $Q$ 的一个顶点，那么 $p_i + q_j$ 可能是 $P \oplus Q$ 的一个顶点。但不是所有这样的组合都是顶点，只有那些“最外围”的组合才是。

### 3. 凸多边形闵可夫斯基和的构造算法

基于上述边的性质，我们可以设计一个高效的算法来构造两个凸多边形 $P$ 和 $Q$ 的闵可夫斯基和。

**核心思想：**
我们可以把凸多边形看作一系列首尾相连的向量（边）。那么 $P \oplus Q$ 的周长，就是 $P$ 和 $Q$ 的所有边向量按照角度排序后，依次首尾相接所形成的。

**算法步骤：**

1.  **准备数据：**
    *   将凸多边形 $P$ 的顶点表示为 $p_0, p_1, \dots, p_{n-1}$。
    *   将凸多边形 $Q$ 的顶点表示为 $q_0, q_1, \dots, q_{m-1}$。
    *   计算出 $P$ 的所有边向量：$v_{P,i} = p_{(i+1)\%n} - p_i$。
    *   计算出 $Q$ 的所有边向量：$v_{Q,j} = q_{(j+1)\%m} - q_j$。
    *   注意：这些边向量在凸多边形中通常是按逆时针（或顺时针）顺序排列的。

2.  **寻找起始点：**
    $P \oplus Q$ 的一个顶点可以由 $P$ 的一个顶点和 $Q$ 的一个顶点相加得到。通常，我们选择 $P$ 中最低最左的顶点 $p_{start\_P}$ 和 $Q$ 中最低最左的顶点 $q_{start\_Q}$。那么 $P \oplus Q$ 的一个起始顶点就是 $s_0 = p_{start\_P} + q_{start\_Q}$。

3.  **合并边向量：**
    *   将 $P$ 的所有边向量和 $Q$ 的所有边向量收集起来。
    *   对这些向量进行排序，排序依据是它们的极角（与 $x$ 轴正方向的夹角）。极角相同的向量，可以任意顺序。
    *   这里需要注意，为了避免浮点数精度问题，通常不直接计算极角，而是使用叉积（`cross_product`）来比较两个向量的旋转方向。如果 `cross_product(v1, v2) > 0`，则 `v1` 在 `v2` 的逆时针方向。
    *   使用双指针或归并排序的思想，从 $P$ 的边向量列表和 $Q$ 的边向量列表的开头开始，每次选择极角最小的向量（或者说，最逆时针方向的向量）添加到结果边向量列表中。

4.  **生成结果多边形顶点：**
    *   从起始顶点 $s_0$ 开始。
    *   按照合并后的边向量列表的顺序，依次将每个向量加到当前顶点上，得到下一个顶点。
    *   例如，如果合并后的边向量序列是 $u_0, u_1, \dots, u_{n+m-1}$，那么 $P \oplus Q$ 的顶点序列就是 $s_0, s_1=s_0+u_0, s_2=s_1+u_1, \dots, s_{n+m-1}=s_{n+m-2}+u_{n+m-2}$。

**算法复杂度：**
*   寻找起始点：$O(N + M)$
*   计算边向量：$O(N + M)$
*   排序边向量：$O((N+M) \log (N+M))$ （如果使用 `std::sort` 对所有边向量排序）
    *   更优的方法：如果 $P$ 和 $Q$ 的边向量已经是按极角排序的（凸多边形通常如此），那么合并这两个已排序的列表只需要 $O(N+M)$ 的时间（类似于归并排序的合并步骤）。
*   生成顶点：$O(N+M)$
*   总复杂度：$O(N \log N + M \log M)$ 如果需要先对各自的边排序，或者 $O(N+M)$ 如果边已排序。考虑到输入通常是顶点列表，我们可能需要先提取边并对它们进行一次排序，因此是 $O(N \log N + M \log M)$。

### 4. 闵可夫斯基和的应用：碰撞检测

闵可夫斯基和的一个重要应用是碰撞检测。

**问题：** 两个凸多边形 $A$ 和 $B$ 是否相交？
**转化：** 我们可以将这个问题转化为“点是否在多边形内部”的问题。

定义 **闵可夫斯基差 (Minkowski Difference)**：
$A \ominus B = A \oplus (-B) = \{ a - b \mid a \in A, b \in B \}$
其中 $-B = \{ -b \mid b \in B \}$，即将 $B$ 中的所有点关于原点对称得到的新集合。

**定理：** 两个凸多边形 $A$ 和 $B$ 相交，当且仅当它们的闵可夫斯基差 $A \ominus B$ 包含原点 $(0,0)$。
*   **证明思路：**
    *   如果 $A$ 和 $B$ 相交，则存在一个点 $p$ 既在 $A$ 中也在 $B$ 中。
    *   那么 $p \in A$ 且 $p \in B$。
    *   我们可以写成 $p - p = 0$。
    *   根据闵可夫斯基差的定义，$p - p$ 就是一个 $a \in A$ 和一个 $b \in B$ 的差。所以 $0 \in A \ominus B$。
    *   反之，如果 $0 \in A \ominus B$，则存在 $a \in A$ 和 $b \in B$ 使得 $a - b = 0$，即 $a = b$。这意味着 $A$ 和 $B$ 有公共点，即它们相交。

所以，要检测两个凸多边形 $A$ 和 $B$ 是否相交，我们只需要计算它们的闵可夫斯基差 $A \ominus B$，然后判断原点 $(0,0)$ 是否在 $A \ominus B$ 内部。这个判断可以通过点在多边形内部测试算法（例如射线法或转角法）来完成。

## 典型例题精讲

为了遵守变量命名规则，我们将使用 `p` 结构体表示点，`v` 结构体表示向量，`n`, `m` 表示多边形顶点数，`i`, `j`, `k` 用于循环。

### 例题1：手算简单凸多边形的闵可夫斯基和

**题目：**
给定两个凸多边形 $P$ 和 $Q$ 的顶点。
$P = \{(0,0), (2,0), (1,1)\}$ (一个三角形)
$Q = \{(0,0), (1,0), (0,1)\}$ (另一个三角形)
请手动计算并画出它们的闵可夫斯基和 $P \oplus Q$。

**思路分析：**

1.  **提取顶点和边向量：**
    *   对于 $P$:
        *   顶点: $p_0=(0,0), p_1=(2,0), p_2=(1,1)$
        *   边向量:
            *   $v_{P,0} = p_1 - p_0 = (2,0)$
            *   $v_{P,1} = p_2 - p_1 = (-1,1)$
            *   $v_{P,2} = p_0 - p_2 = (-1,-1)$
    *   对于 $Q$:
        *   顶点: $q_0=(0,0), q_1=(1,0), q_2=(0,1)$
        *   边向量:
            *   $v_{Q,0} = q_1 - q_0 = (1,0)$
            *   $v_{Q,1} = q_2 - q_1 = (-1,1)$
            *   $v_{Q,2} = q_0 - q_2 = (0,-1)$

2.  **寻找起始点：**
    *   $P$ 的最低最左点是 $p_0=(0,0)$。
    *   $Q$ 的最低最左点是 $q_0=(0,0)$。
    *   $P \oplus Q$ 的起始顶点 $s_0 = p_0 + q_0 = (0,0) + (0,0) = (0,0)$。

3.  **排序和合并边向量：**
    所有边向量：
    *   $v_{P,0} = (2,0)$ (极角 0°)
    *   $v_{Q,0} = (1,0)$ (极角 0°)
    *   $v_{P,1} = (-1,1)$ (极角 135°)
    *   $v_{Q,1} = (-1,1)$ (极角 135°)
    *   $v_{Q,2} = (0,-1)$ (极角 270° 或 -90°)
    *   $v_{P,2} = (-1,-1)$ (极角 225° 或 -135°)

    按极角从小到大排序（逆时针）：
    1.  $(2,0)$ ($v_{P,0}$)
    2.  $(1,0)$ ($v_{Q,0}$)
    3.  $(-1,1)$ ($v_{P,1}$)
    4.  $(-1,1)$ ($v_{Q,1}$)
    5.  $(-1,-1)$ ($v_{P,2}$)
    6.  $(0,-1)$ ($v_{Q,2}$)

    合并后的边向量序列 $U$:
    $U = \{(2,0), (1,0), (-1,1), (-1,1), (-1,-1), (0,-1)\}$

4.  **生成 $P \oplus Q$ 的顶点：**
    *   $s_0 = (0,0)$
    *   $s_1 = s_0 + (2,0) = (2,0)$
    *   $s_2 = s_1 + (1,0) = (3,0)$
    *   $s_3 = s_2 + (-1,1) = (2,1)$
    *   $s_4 = s_3 + (-1,1) = (1,2)$
    *   $s_5 = s_4 + (-1,-1) = (0,1)$
    *   $s_6 = s_5 + (0,-1) = (0,0)$ (回到起点)

    所以 $P \oplus Q$ 的顶点是 $(0,0), (2,0), (3,0), (2,1), (1,2), (0,1)$。这是一个六边形。

**绘制：** (请自行在草稿纸上绘制)
P是一个直角三角形，Q也是一个直角三角形。
P+Q会是一个六边形，其形状是P在Q的每个顶点上平移，然后取并集。

### 例题2：编写程序计算凸多边形闵可夫斯基和

**题目：**
输入两个凸多边形 $P$ 和 $Q$ 的顶点坐标。
第一个多边形 $P$ 有 $n$ 个顶点，按逆时针顺序给出。
第二个多边形 $Q$ 有 $m$ 个顶点，按逆时针顺序给出。
输出 $P \oplus Q$ 的顶点数量和所有顶点的坐标，也按逆时针顺序。

**输入格式：**
第一行一个整数 $n$。
接下来 $n$ 行，每行两个整数 $x, y$ 表示 $P$ 的顶点坐标。
接下来一个整数 $m$。
接下来 $m$ 行，每行两个整数 $x, y$ 表示 $Q$ 的顶点坐标。

**输出格式：**
第一行一个整数 $k$ ($k=n+m$)，表示 $P \oplus Q$ 的顶点数量。
接下来 $k$ 行，每行两个整数 $x, y$ 表示 $P \oplus Q$ 的顶点坐标。

**样例输入：**
```
3
0 0
2 0
1 1
3
0 0
1 0
0 1
```

**样例输出：**
```
6
0 0
3 0
2 1
1 2
0 1
```
(注意：这里输出的顺序和例题1手算略有不同，因为起始点和排序可能导致最终顶点列表的起始点不同，但多边形本身是相同的。)

**思路分析 (结合代码实现)：**

1.  **定义点/向量结构体：**
    包含 `x`, `y` 坐标。重载加减法方便向量运算。
2.  **辅助函数：**
    *   `cross_product(p1, p2)`: 计算向量 $p1$ 和 $p2$ 的叉积，用于判断方向。
    *   `get_angle_comparison(v1, v2)`: 比较两个向量的极角。
3.  **读取输入：**
    *   使用原生数组 `ps[100005]` 和 `qs[100005]` 存储多边形顶点。
4.  **寻找最低最左顶点：**
    *   遍历 $P$ 和 $Q$ 的所有顶点，找到 $x$ 最小，若 $x$ 相同则 $y$ 最小的顶点。记为 `p_start_idx` 和 `q_start_idx`。
    *   计算 $P \oplus Q$ 的起始顶点 `res_start_p = ps[p_start_idx] + qs[q_start_idx]`。
5.  **提取边向量：**
    *   使用原生数组 `vs_p[100005]` 和 `vs_q[100005]` 存储边向量。
    *   对于 $P$，从 `p_start_idx` 开始，按逆时针顺序提取 $n$ 条边向量。
    *   对于 $Q$，从 `q_start_idx` 开始，按逆时针顺序提取 $m$ 条边向量。
    *   注意：这里是关键。由于输入的多边形是凸的且顶点按逆时针给出，其边向量也自然是按极角逆时针排序的（或者说，相对有序）。
6.  **合并边向量：**
    *   使用双指针 `i` 和 `j` 分别遍历 `vs_p` 和 `vs_q`。
    *   每次比较 `vs_p[i]` 和 `vs_q[j]` 的极角。选择极角较小的（更逆时针方向的）向量加入结果 `vs_res` 数组。
    *   如果极角相同，可以先加入 `vs_p[i]` 再加入 `vs_q[j]`，或者反之，不影响最终多边形形状，只影响顶点顺序。这里我们假设先加 `vs_p[i]`。
    *   直到一个列表遍历完，将另一个列表中剩余的向量全部加入 `vs_res`。
7.  **生成结果顶点：**
    *   将 `res_start_p` 作为 `res_ps[0]`。
    *   遍历 `vs_res`，依次累加向量，得到 `res_ps[k+1] = res_ps[k] + vs_res[k]`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 点/向量结构体
struct p {
    long long x, y;

    // 默认构造函数
    p() : x(0), y(0) {}
    // 带参数构造函数
    p(long long a, long long b) : x(a), y(b) {}

    // 向量加法
    p operator+(const p& o) const {
        return p(x + o.x, y + o.y);
    }
    // 向量减法
    p operator-(const p& o) const {
        return p(x - o.x, y - o.y);
    }
    // 向量相等判断
    bool operator==(const p& o) const {
        return x == o.x && y == o.y;
    }
};

// 全局数组，用于存储多边形顶点和边向量
p ps[100005]; // Polygon P vertices
p qs[100005]; // Polygon Q vertices
p vs_p[100005]; // Polygon P edge vectors
p vs_q[100005]; // Polygon Q edge vectors
p vs_res[200005]; // Resulting Minkowski Sum edge vectors
p res_ps[200005]; // Resulting Minkowski Sum vertices

// 叉积：判断向量 v1 到 v2 的旋转方向
// > 0 表示 v2 在 v1 的逆时针方向
// < 0 表示 v2 在 v1 的顺时针方向
// = 0 表示 v1, v2 共线
long long cross_product(p v1, p v2) {
    return v1.x * v2.y - v1.y * v2.x;
}

// 比较两个向量的极角
// 返回 true 如果 v1 的极角小于 v2 的极角 (或 v1 在 v2 的顺时针方向)
// 注意：这里我们希望按逆时针顺序排序，所以 v1 < v2 意味着 v1 应该在 v2 之前
// 也就是说，v2 相对 v1 逆时针旋转。
bool compare_vectors(p v1, p v2) {
    // 处理四个象限的特殊情况，确保正确排序
    // 象限 1 (x>=0, y>0)
    // 象限 2 (x<0, y>=0)
    // 象限 3 (x<=0, y<0)
    // 象限 4 (x>0, y<=0)
    // 这样处理可以避免 atan2 的浮点精度问题，并且能正确处理边界情况
    bool q1 = (v1.x >= 0 && v1.y > 0);
    bool q2 = (v1.x < 0 && v1.y >= 0);
    bool q3 = (v1.x <= 0 && v1.y < 0);
    bool q4 = (v1.x > 0 && v1.y <= 0);

    bool oq1 = (v2.x >= 0 && v2.y > 0);
    bool oq2 = (v2.x < 0 && v2.y >= 0);
    bool oq3 = (v2.x <= 0 && v2.y < 0);
    bool oq4 = (v2.x > 0 && v2.y <= 0);

    if (q1 && (oq2 || oq3 || oq4)) return true; // v1在第一象限，v2在后面象限
    if (q2 && (oq3 || oq4)) return true; // v1在第二象限，v2在后面象限
    if (q3 && oq4) return true; // v1在第三象限，v2在第四象限

    if ((q1 && oq1) || (q2 && oq2) || (q3 && oq3) || (q4 && oq4)) {
        // 同象限或轴上，使用叉积判断
        long long c = cross_product(v1, v2);
        if (c == 0) { // 共线
            // 长度小的优先 (可选，通常不影响凸包形状)
            // 这里为了保持一致性，如果共线且是同向，我们认为它们是等价的，
            // 保持原有的相对顺序即可，或者按x再按y排序
            // 题目中凸多边形保证了边是逆时针的，所以合并时只需考虑叉积
            return false; // 如果共线，则不认为 v1 < v2
        }
        return c > 0; // v1在v2的顺时针方向，即v1的极角小于v2
    }
    return false; // 默认情况，v1不在v2之前
}


int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;

    // 读取多边形 P
    std::cin >> n;
    int p_start_idx = 0;
    for (int i = 0; i < n; ++i) {
        std::cin >> ps[i].x >> ps[i].y;
        // 寻找最低最左的顶点
        if (ps[i].y < ps[p_start_idx].y || (ps[i].y == ps[p_start_idx].y && ps[i].x < ps[p_start_idx].x)) {
            p_start_idx = i;
        }
    }

    // 读取多边形 Q
    std::cin >> m;
    int q_start_idx = 0;
    for (int i = 0; i < m; ++i) {
        std::cin >> qs[i].x >> qs[i].y;
        // 寻找最低最左的顶点
        if (qs[i].y < qs[q_start_idx].y || (qs[i].y == qs[q_start_idx].y && qs[i].x < qs[q_start_idx].x)) {
            q_start_idx = i;
        }
    }

    // 计算 P 的边向量 (从起始点开始，逆时针顺序)
    for (int i = 0; i < n; ++i) {
        vs_p[i] = ps[(p_start_idx + i + 1) % n] - ps[(p_start_idx + i) % n];
    }

    // 计算 Q 的边向量 (从起始点开始，逆时针顺序)
    for (int i = 0; i < m; ++i) {
        vs_q[i] = qs[(q_start_idx + i + 1) % m] - qs[(q_start_idx + i) % m];
    }

    // 合并边向量
    int i = 0, j = 0, k = 0;
    while (i < n && j < m) {
        // compare_vectors(vs_p[i], vs_q[j]) 返回 true 意味着 vs_p[i] 的极角小于 vs_q[j]
        // 也就是说 vs_p[i] 更“靠前”或更“顺时针”
        // 我们要选择更“逆时针”的，所以当 cross_product(vs_p[i], vs_q[j]) >= 0 时
        // 意味着 vs_q[j] 在 vs_p[i] 的逆时针方向或共线，所以应该先取 vs_q[j]
        // 否则取 vs_p[i]
        if (cross_product(vs_p[i], vs_q[j]) >= 0) { // vs_q[j] 在 vs_p[i] 的逆时针方向或共线
            vs_res[k++] = vs_q[j++];
        } else { // vs_p[i] 在 vs_q[j] 的逆时针方向
            vs_res[k++] = vs_p[i++];
        }
    }

    // 添加剩余的边向量
    while (i < n) {
        vs_res[k++] = vs_p[i++];
    }
    while (j < m) {
        vs_res[k++] = vs_q[j++];
    }

    // 生成结果多边形的顶点
    res_ps[0] = ps[p_start_idx] + qs[q_start_idx]; // 起始顶点
    for (int idx = 0; idx < k; ++idx) {
        res_ps[idx + 1] = res_ps[idx] + vs_res[idx];
    }

    // 输出结果
    std::cout << k << std::endl;
    for (int idx = 0; idx < k; ++idx) {
        std::cout << res_ps[idx].x << " " << res_ps[idx].y << std::endl;
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 读取输入 $O(N+M)$。寻找起始点 $O(N+M)$。提取边向量 $O(N+M)$。合并边向量（双指针法，因为原始边向量已经按极角排序了）$O(N+M)$。生成结果顶点 $O(N+M)$。
    总时间复杂度为 $O(N+M)$。这是非常高效的。
*   **空间复杂度：** 存储顶点和边向量的数组，总共 $O(N+M)$。

**注意：**
1.  `compare_vectors` 函数在实际竞赛中通常不用，因为凸多边形的边本身就是按极角有序的。直接用叉积判断合并哪个更逆时针即可。
2.  `cross_product(v1, v2)` 的值如果大于0，表示 `v2` 在 `v1` 的逆时针方向，即 `v2` 的极角更大。如果小于0，表示 `v2` 在 `v1` 的顺时针方向，即 `v2` 的极角更小。如果等于0，表示共线。
3.  在合并边向量时，`if (cross_product(vs_p[i], vs_q[j]) >= 0)` 的判断逻辑是：如果 `vs_q[j]` 在 `vs_p[i]` 的逆时针方向（或共线），那么 `vs_q[j]` 的极角更大或相等，此时我们应该先取 `vs_p[i]`。
    *   **修正合并逻辑：** 应该是取极角更小的那个。如果 `cross_product(vs_p[i], vs_q[j]) < 0`，说明 `vs_p[i]` 相对 `vs_q[j]` 是顺时针的，即 `vs_p[i]` 极角更小，应该取 `vs_p[i]`。否则取 `vs_q[j]`。

    ```cpp
    // 修正后的合并边向量
    // 合并边向量
    i = 0, j = 0, k = 0;
    while (i < n && j < m) {
        // 如果 cross_product(vs_p[i], vs_q[j]) < 0，说明 vs_p[i] 的极角更小
        // 也就是 vs_p[i] 在 vs_q[j] 的顺时针方向
        if (cross_product(vs_p[i], vs_q[j]) < 0) { 
            vs_res[k++] = vs_p[i++];
        } else { // vs_q[j] 的极角更小或相等 (在 vs_p[i] 的顺时针方向或共线)
            vs_res[k++] = vs_q[j++];
        }
    }
    // ... 剩余部分不变
    ```
    经过测试，我提供的原始代码 `if (cross_product(vs_p[i], vs_q[j]) >= 0)` 导致 `vs_q[j]` 被优先选取，这实际上是选取了极角 **更大或相等** 的那个。为了得到逆时针顺序，我们应该选取极角 **更小** 的那个。所以修正后的逻辑是正确的。
    对于 `cross_product(v1, v2)`，如果 `v1` 逆时针转到 `v2`，叉积为正。
    如果 `v1` 顺时针转到 `v2`，叉积为负。
    我们需要按照逆时针顺序合并。
    比较 `v_p` 和 `v_q`。如果 `v_p` 比 `v_q` 更“逆时针”，那么 `cross_product(v_q, v_p) > 0`。
    或者说，如果 `cross_product(v_p, v_q) < 0`，表示 `v_p` 在 `v_q` 的顺时针方向，即 `v_p` 极角更小，先取 `v_p`。
    如果 `cross_product(v_p, v_q) > 0`，表示 `v_p` 在 `v_q` 的逆时针方向，即 `v_q` 极角更小，先取 `v_q`。
    如果 `cross_product(v_p, v_q) == 0`，表示共线，可以任意取一个。为了确定性，我们选择先取 `v_p`。
    所以，修正后的合并逻辑应该是：
    ```cpp
    if (cross_product(vs_p[i], vs_q[j]) < 0) { // vs_p[i] 极角更小
        vs_res[k++] = vs_p[i++];
    } else { // vs_q[j] 极角更小或相等
        vs_res[k++] = vs_q[j++];
    }
    ```
    这个修正后的逻辑是正确的。

### 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含常用头文件，如iostream, algorithm等

// 定义点/向量结构体
struct p {
    long long x, y; // 坐标使用 long long 防止溢出

    // 默认构造函数
    p() : x(0), y(0) {}
    // 带参数构造函数
    p(long long a, long long b) : x(a), y(b) {}

    // 向量加法
    p operator+(const p& o) const {
        return p(x + o.x, y + o.y);
    }
    // 向量减法 (用于计算边向量)
    p operator-(const p& o) const {
        return p(x - o.x, y - o.y);
    }
    // 向量相等判断 (可选，调试用)
    bool operator==(const p& o) const {
        return x == o.x && y == o.y;
    }
};

// 全局数组，用于存储多边形顶点和边向量
// 数组大小根据题目数据范围设定，这里假设最大顶点数为100000
p ps[100005];       // 多边形 P 的顶点
p qs[100005];       // 多边形 Q 的顶点
p vs_p[100005];     // 多边形 P 的边向量
p vs_q[100005];     // 多边形 Q 的边向量
p vs_res[200005];   // 闵可夫斯基和结果多边形的边向量 (最多 n+m 条)
p res_ps[200005];   // 闵可夫斯基和结果多边形的顶点 (最多 n+m 个)

// 叉积函数：计算向量 v1 和 v2 的叉积
// 叉积的符号表示 v1 逆时针旋转到 v2 的方向
// > 0: v2 在 v1 的逆时针方向 (v2 极角更大)
// < 0: v2 在 v1 的顺时针方向 (v2 极角更小)
// = 0: v1, v2 共线
long long cross_product(p v1, p v2) {
    return v1.x * v2.y - v1.y * v2.x;
}

int main() {
    // 开启快速IO
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n 为多边形 P 的顶点数，m 为多边形 Q 的顶点数

    // --- 1. 读取多边形 P 的顶点 ---
    std::cin >> n;
    int p_start_idx = 0; // 记录 P 中最低最左顶点的索引
    for (int i = 0; i < n; ++i) {
        std::cin >> ps[i].x >> ps[i].y;
        // 寻找最低最左的顶点作为起始点
        // 如果当前点 y 坐标更小，或者 y 相同但 x 坐标更小，则更新起始点
        if (ps[i].y < ps[p_start_idx].y || (ps[i].y == ps[p_start_idx].y && ps[i].x < ps[p_start_idx].x)) {
            p_start_idx = i;
        }
    }

    // --- 2. 读取多边形 Q 的顶点 ---
    std::cin >> m;
    int q_start_idx = 0; // 记录 Q 中最低最左顶点的索引
    for (int i = 0; i < m; ++i) {
        std::cin >> qs[i].x >> qs[i].y;
        // 寻找最低最左的顶点作为起始点
        if (qs[i].y < qs[q_start_idx].y || (qs[i].y == qs[q_start_idx].y && qs[i].x < qs[q_start_idx].x)) {
            q_start_idx = i;
        }
    }

    // --- 3. 提取多边形 P 的边向量 ---
    // 从 p_start_idx 开始，按逆时针顺序提取 n 条边向量
    for (int i = 0; i < n; ++i) {
        // ps[(p_start_idx + i + 1) % n] 是当前顶点 ps[(p_start_idx + i) % n] 的下一个顶点
        vs_p[i] = ps[(p_start_idx + i + 1) % n] - ps[(p_start_idx + i) % n];
    }

    // --- 4. 提取多边形 Q 的边向量 ---
    // 从 q_start_idx 开始，按逆时针顺序提取 m 条边向量
    for (int i =
