同学们，想象一下，如果有一天你们去医院看急诊，医生是按照“先来后到”的顺序看病吗？当然不是！医生会根据病情的紧急程度来决定谁先看，对不对？病情最严重的，哪怕他刚到，也要立刻安排治疗。

再比如，你们玩游戏的时候，是不是经常会有一个排行榜，显示谁的分数最高？当有新玩家加入或者老玩家分数变化时，这个排行榜是不是要实时更新，并且最厉害的玩家总是在最上面？

这些场景都有一个共同的特点：它们不是简单的“排队”（先进先出），也不是“插队”（后进先出），而是根据某个“重要性”或者“优先级”来决定谁先被处理。这种特殊的“队列”，我们把它叫做**优先队列（Priority Queue）**。

那么问题来了，我们怎么才能高效地管理这样一个“优先队列”呢？如果每次都要把所有元素找一遍才能找到最紧急/最重要的那个，那效率也太低了！别急，今天我们要学习一个神奇的数据结构，它就像一个“聪明”的管家，总能把最重要的事情安排在最前面，它就是——**二叉堆**！

想象一下，你有一堆玩具，你希望最喜欢的玩具总是在最容易拿到的地方。二叉堆就是这样，它能帮你把“最喜欢”的（优先级最高）的那个玩具，总是放在最顶上，而且每次你拿走一个，它都能迅速地把下一个“最喜欢”的玩具放到顶上！是不是很酷？

## 深度知识点讲解

### 1. 优先队列 (Priority Queue)

*   **本质与概念**：优先队列是一种特殊的队列。它不像普通队列那样，严格遵守“先进先出”（FIFO）的原则，而是根据元素的“优先级”来决定出队顺序。优先级最高的元素总是最先出队。
    *   **生活类比**：
        *   **医院急诊室**：病情危重的病人优先级最高。
        *   **操作系统任务调度**：紧急任务优先级高，先被CPU处理。
        *   **游戏排行榜**：分数最高的玩家优先级最高，排在榜首。
*   **基本操作**：
    *   `insert(element)`：将一个元素及其优先级加入队列。
    *   `top()`：查看队列中优先级最高的元素（不移除）。
    *   `pop()`：移除并返回队列中优先级最高的元素。
    *   `empty()`：判断队列是否为空。
*   **实现方式**：虽然可以用数组或链表实现优先队列，但效率不高（每次查找最高优先级元素可能需要O(N)时间）。最常用、最高效的实现方式就是我们今天要学习的——**二叉堆**。

### 2. 堆 (Heap)

堆是一种特殊的**完全二叉树**，它满足以下两个重要性质：

#### 2.1. 性质一：完全二叉树 (Complete Binary Tree)

*   **定义**：除了最后一层，其他所有层都被完全填充。最后一层的所有节点都尽可能地靠左填充。
*   **为什么是完全二叉树？**
    *   **效率**：完全二叉树结构紧凑，没有“空洞”，这使得它可以用数组进行高效存储，避免了指针带来的额外开销和内存碎片。
    *   **高度**：对于N个节点的完全二叉树，其高度始终是 `log N`（以2为底）。这保证了堆的大部分操作（如插入、删除）的时间复杂度为 `O(log N)`。
*   **生活类比**：想象一下盖楼房。完全二叉树的楼房，除了顶层，其他层都盖满了房子，而顶层的房子也都是从左边开始一间间盖好的，没有空着的地方。这样盖楼效率最高，占地面积最小。

#### 2.2. 性质二：堆序性 (Heap Property)

*   **定义**：
    *   **大顶堆 (Max Heap)**：每个父节点的值都大于或等于其子节点的值。这意味着堆中最大的元素总是在根节点。
    *   **小顶堆 (Min Heap)**：每个父节点的值都小于或等于其子节点的值。这意味着堆中最小的元素总是在根节点。
*   **为什么需要堆序性？**
    *   **快速查找最值**：有了堆序性，我们不需要遍历整个堆就能知道最大/最小的元素是谁——它就在根节点！这正是优先队列需要的核心功能。
*   **生活类比**：
    *   **大顶堆**：就像一个家族的族谱，族长（根节点）的辈分最高，他的孩子（子节点）辈分都比他低，孙子辈分又比孩子低。
    *   **小顶堆**：反过来，根节点是辈分最低的，子节点辈分都比它高。

#### 2.3. 堆的数组存储

由于堆是完全二叉树，我们可以非常巧妙地用一个**一维数组**来存储它，而不需要使用复杂的指针结构。这大大简化了实现，并提高了内存访问效率。

我们通常会选择从数组索引 `1` 开始存储堆的元素（索引 `0` 可以空着不用，也可以存储哨兵值）。这样做的好处是，父子节点之间的索引关系变得非常简洁：

*   **对于节点 `i`**：
    *   它的**父节点**的索引是 `i / 2` (整数除法)。
    *   它的**左子节点**的索引是 `2 * i`。
    *   它的**右子节点**的索引是 `2 * i + 1`。
*   **生活类比**：想象你把所有节点从上到下、从左到右依次编号（从1开始）。然后把这些编号直接对应到数组的索引上。父子关系就像一个简单的数学公式，一算便知。

**图示 (大顶堆)**：
假设数组 `h` 存储堆元素。
`h = [_, 100, 80, 90, 70, 75, 85, 60]` （`_` 表示索引0未使用）

```
                     100 (h[1])
                    /   \
                 80(h[2])  90(h[3])
                /  \     /  \
             70(h[4]) 75(h[5]) 85(h[6])  60(h[7])
```

*   节点 `h[3]` (值为90) 的父节点是 `h[3/2]` 即 `h[1]` (值为100)。
*   节点 `h[3]` (值为90) 的左子节点是 `h[2*3]` 即 `h[6]` (值为85)。
*   节点 `h[3]` (值为90) 的右子节点是 `h[2*3+1]` 即 `h[7]` (值为60)。

#### 2.4. 堆的基本操作

##### 2.4.1. 向上调整 (Sift Up / Heapify Up)

*   **目的**：当一个新元素被添加到堆的末尾时，或者某个元素的值被增加（在大顶堆中），它可能比它的父节点优先级更高，从而破坏了堆序性。向上调整操作就是将这个元素沿着路径向上移动，直到它找到合适的位置，恢复堆序性。
*   **过程**：
    1.  将新元素放到堆的最后一个位置 (数组末尾)。
    2.  比较这个新元素和它的父节点。
    3.  如果新元素比父节点“优先级更高”（大顶堆中是值更大，小顶堆中是值更小），则交换它们的位置。
    4.  重复步骤2和3，直到新元素到达根节点，或者它的父节点比它“优先级更高”（满足堆序性）。
*   **生活类比**：一个新来的小朋友插队，他发现自己比前面的哥哥姐姐更厉害，就一直往前挤，直到他前面的人都比他厉害，或者他已经排到了队伍的最前面。
*   **时间复杂度**：最坏情况下，元素需要从叶子节点一直移动到根节点，路径长度为 `log N`，所以时间复杂度是 `O(log N)`。

##### 2.4.2. 向下调整 (Sift Down / Heapify Down)

*   **目的**：当堆顶元素被移除后，或者某个元素的值被减少（在大顶堆中），它可能比它的子节点优先级更低，从而破坏了堆序性。向下调整操作就是将这个元素沿着路径向下移动，直到它找到合适的位置，恢复堆序性。
*   **过程**：
    1.  将堆顶元素移除。
    2.  将堆的最后一个元素（数组末尾的元素）放到堆顶位置。
    3.  比较这个新堆顶元素和它的子节点。
    4.  如果这个元素比它的任何一个子节点“优先级更低”（大顶堆中是值更小，小顶堆中是值更大），则与优先级最高的那个子节点交换位置。
    5.  重复步骤3和4，直到这个元素到达叶子节点，或者它的子节点都比它“优先级更低”（满足堆序性）。
*   **生活类比**：队伍最厉害的人走了，就把队伍最后的人拉到最前面来顶替。但这个人发现自己没有资格站在最前面，就一直往后退，直到他后面的人都比他弱，或者他已经退到了队伍的最后面。
*   **时间复杂度**：最坏情况下，元素需要从根节点一直移动到叶子节点，路径长度为 `log N`，所以时间复杂度是 `O(log N)`。

##### 2.4.3. 插入元素 (Insert)

*   直接调用 `siftUp`。
*   **过程**：
    1.  将新元素添加到数组的末尾，同时 `s`（堆大小）加1。
    2.  从新元素的位置开始，执行 `siftUp` 操作。
*   **时间复杂度**：`O(log N)`。

##### 2.4.4. 删除堆顶元素 (Pop)

*   直接调用 `siftDown`。
*   **过程**：
    1.  如果堆为空，则无法删除。
    2.  记录堆顶元素（即 `h[1]`）。
    3.  将数组的最后一个元素 (`h[s]`) 移动到堆顶 (`h[1]`)。
    4.  `s`（堆大小）减1。
    5.  从堆顶位置 (`1`) 开始，执行 `siftDown` 操作。
    6.  返回之前记录的堆顶元素。
*   **时间复杂度**：`O(log N)`。

##### 2.4.5. 建堆 (Build Heap)

*   **目的**：将一个无序的数组转换成一个符合堆序性的堆。
*   **过程**：
    1.  将所有元素按顺序放入数组中。
    2.  从最后一个非叶子节点开始（索引为 `s / 2`），依次向前遍历到根节点（索引 `1`）。
    3.  对每一个节点都执行 `siftDown` 操作。
*   **为什么从 `s / 2` 开始？**：因为索引大于 `s / 2` 的节点都是叶子节点，它们本身就是一个合法的堆（只有一个元素），无需调整。我们只需要从第一个有子节点的节点开始调整即可。
*   **时间复杂度**：虽然看起来有 `N/2` 次 `siftDown` 操作，每次 `siftDown` 是 `O(log N)`，但实际上，大部分 `siftDown` 操作的深度都很小。因此，建堆的平均时间复杂度是 `O(N)`。这是一个非常高效的操作。
*   **生活类比**：你有一堆乱七八糟的玩具，你先随便把它们堆成一个金字塔（完全二叉树），然后从金字塔的倒数第二层开始，把每个玩具都检查一遍，如果它比下面的玩具差，就把它往下挪，直到它找到合适的位置。这样一层层往上检查，最终整个金字塔就变成了一个有序的“堆”。

### 3. 常见误区与易错点

*   **数组索引从0还是1开始？**：虽然C++数组默认从0开始，但为了方便计算父子节点索引，堆通常从1开始存储。如果从0开始，父子索引关系会变得稍微复杂 (`(i-1)/2`, `2*i+1`, `2*i+2`)。建议初学者统一从1开始。
*   **`siftUp` 和 `siftDown` 的判断条件**：大顶堆是父节点大于等于子节点，小顶堆是父节点小于等于子节点。判断时不要混淆。
*   **`siftDown` 的子节点选择**：在 `siftDown` 中，如果当前节点需要与子节点交换，一定要选择“优先级更高”的那个子节点进行交换（大顶堆选值最大的子节点，小顶堆选值最小的子节点），否则可能无法一次性恢复堆序性。
*   **边界条件**：判断是否存在子节点时，要确保子节点的索引没有超出堆的有效范围 (`2*i <= s` 和 `2*i+1 <= s`)。

## 典型例题精讲

我们以**大顶堆**为例进行讲解。

**题目描述**：实现一个大顶堆，支持插入、删除堆顶元素操作。

### 例题一：大顶堆插入操作

**任务**：从一个空的堆开始，依次插入元素 `[5, 3, 8, 1, 6]`。请画出每一步堆的数组表示和逻辑结构。

**思路分析**：
每次插入新元素时，先将其添加到数组末尾，然后进行向上调整（`siftUp`），使其回到正确位置。

1.  **初始空堆**：`h = []`, `s = 0`
2.  **插入 5**：
    *   `h = [_, 5]`, `s = 1`
    *   `siftUp(1)`：根节点，无需调整。
    *   **结构**：`5`
3.  **插入 3**：
    *   `h = [_, 5, 3]`, `s = 2`
    *   `siftUp(2)`：`h[2]=3`，父节点 `h[1]=5`。`3 < 5`，满足大顶堆性质，无需交换。
    *   **结构**：
        ```
          5
         /
        3
        ```
4.  **插入 8**：
    *   `h = [_, 5, 3, 8]`, `s = 3`
    *   `siftUp(3)`：
        *   `h[3]=8`，父节点 `h[1]=5`。`8 > 5`，交换。
        *   `h = [_, 8, 3, 5]`
        *   当前位置 `p = 1`，已到根节点，停止。
    *   **结构**：
        ```
          8
         / \
        3   5
        ```
5.  **插入 1**：
    *   `h = [_, 8, 3, 5, 1]`, `s = 4`
    *   `siftUp(4)`：`h[4]=1`，父节点 `h[2]=3`。`1 < 3`，满足大顶堆性质，无需交换。
    *   **结构**：
        ```
          8
         / \
        3   5
       /
      1
        ```
6.  **插入 6**：
    *   `h = [_, 8, 3, 5, 1, 6]`, `s = 5`
    *   `siftUp(5)`：
        *   `h[5]=6`，父节点 `h[2]=3`。`6 > 3`，交换。
        *   `h = [_, 8, 6, 5, 1, 3]`
        *   当前位置 `p = 2`，父节点 `h[1]=8`。`6 < 8`，满足大顶堆性质，停止。
    *   **最终结构**：
        ```
          8
         / \
        6   5
       / \
      1   3
        ```
    *   **最终数组**：`h = [_, 8, 6, 5, 1, 3]`, `s = 5`

**复杂度分析**：每次插入操作，最坏情况下需要向上调整 `log N` 次，每次调整涉及常数次比较和交换。所以单次插入时间复杂度为 `O(log N)`。

### 例题二：大顶堆删除操作

**任务**：从例题一的最终堆 `[_, 8, 6, 5, 1, 3]` 中，删除堆顶元素两次。请画出每一步堆的数组表示和逻辑结构。

**思路分析**：
每次删除堆顶元素时，将堆尾元素移到堆顶，然后进行向下调整（`siftDown`），使其回到正确位置。

1.  **初始堆**：`h = [_, 8, 6, 5, 1, 3]`, `s = 5`
    *   **结构**：
        ```
          8
         / \
        6   5
       / \
      1   3
        ```
2.  **删除堆顶 (8)**：
    *   记录堆顶元素 `8`。
    *   将 `h[s]` (即 `h[5]=3`) 移到 `h[1]`。
    *   `s` 减1 (`s = 4`)。
    *   `h = [_, 3, 6, 5, 1]`, `s = 4`
    *   `siftDown(1)`：
        *   `h[1]=3`，子节点 `h[2]=6`, `h[3]=5`。最大子节点是 `h[2]=6`。`3 < 6`，交换。
        *   `h = [_, 6, 3, 5, 1]`
        *   当前位置 `p = 2`。子节点 `h[4]=1` (右子节点 `h[5]` 不存在)。最大子节点是 `h[4]=1`。`3 > 1`，满足大顶堆性质，无需交换。
        *   停止。
    *   **删除后结构**：
        ```
          6
         / \
        3   5
       /
      1
        ```
    *   **当前数组**：`h = [_, 6, 3, 5, 1]`, `s = 4`

3.  **再次删除堆顶 (6)**：
    *   记录堆顶元素 `6`。
    *   将 `h[s]` (即 `h[4]=1`) 移到 `h[1]`。
    *   `s` 减1 (`s = 3`)。
    *   `h = [_, 1, 3, 5]`, `s = 3`
    *   `siftDown(1)`：
        *   `h[1]=1`，子节点 `h[2]=3`, `h[3]=5`。最大子节点是 `h[3]=5`。`1 < 5`，交换。
        *   `h = [_, 5, 3, 1]`
        *   当前位置 `p = 3`。无子节点，停止。
    *   **最终结构**：
        ```
          5
         / \
        3   1
        ```
    *   **最终数组**：`h = [_, 5, 3, 1]`, `s = 3`

**复杂度分析**：每次删除操作，最坏情况下需要向下调整 `log N` 次，每次调整涉及常数次比较和交换。所以单次删除时间复杂度为 `O(log N)`。

### 例题三：建堆操作

**任务**：给定一个无序数组 `[3, 1, 4, 1, 5, 9, 2, 6]`，将其构建成一个大顶堆。

**思路分析**：
将数组元素直接放入堆数组，然后从最后一个非叶子节点开始，依次向上对其进行 `siftDown` 操作。

1.  **初始数组**：`a = [3, 1, 4, 1, 5, 9, 2, 6]`
2.  **放入堆数组**：`h = [_, 3, 1, 4, 1, 5, 9, 2, 6]`, `s = 8`
    *   **结构** (完全二叉树，但未满足堆序性)：
        ```
               3
             /   \
            1     4
           / \   / \
          1   5 9   2
         /
        6
        ```
3.  **建堆过程**：从最后一个非叶子节点 `s/2 = 8/2 = 4` (即 `h[4]=1`) 开始，向上到 `h[1]` 依次进行 `siftDown`。

    *   **`siftDown(4)`** (节点值1): 子节点 `h[8]=6`。`1 < 6`，交换。
        *   `h = [_, 3, 1, 4, 6, 5, 9, 2, 1]`
        *   `h[4]` 变为 `6`。
        *   结构：
            ```
                   3
                 /   \
                1     4
               / \   / \
              6   5 9   2
             /
            1
            ```
    *   **`siftDown(3)`** (节点值4): 子节点 `h[6]=9`, `h[7]=2`。最大子节点 `h[6]=9`。`4 < 9`，交换。
        *   `h = [_, 3, 1, 9, 6, 5, 4, 2, 1]`
        *   `h[3]` 变为 `9`。
        *   结构：
            ```
                   3
                 /   \
                1     9
               / \   / \
              6   5 4   2
             /
            1
            ```
    *   **`siftDown(2)`** (节点值1): 子节点 `h[4]=6`, `h[5]=5`。最大子节点 `h[4]=6`。`1 < 6`，交换。
        *   `h = [_, 3, 6, 9, 1, 5, 4, 2, 1]`
        *   `h[2]` 变为 `6`。
        *   结构：
            ```
                   3
                 /   \
                6     9
               / \   / \
              1   5 4   2
             /
            1
            ```
    *   **`siftDown(1)`** (节点值3): 子节点 `h[2]=6`, `h[3]=9`。最大子节点 `h[3]=9`。`3 < 9`，交换。
        *   `h = [_, 9, 6, 3, 1, 5, 4, 2, 1]`
        *   `h[1]` 变为 `9`。
        *   结构：
            ```
                   9
                 /   \
                6     3
               / \   / \
              1   5 4   2
             /
            1
            ```
    *   `h[1]` 变为 `9` 后，它的子节点 `h[2]=6`, `h[3]=3` 都比它小，满足堆序性。停止。

4.  **最终大顶堆数组**：`h = [_, 9, 6, 3, 1, 5, 4, 2, 1]`
    *   **最终结构**：
        ```
                   9
                 /   \
                6     3
               / \   / \
              1   5 4   2
             /
            1
        ```

**复杂度分析**：建堆操作的时间复杂度是 `O(N)`。

## 代码实现模板

这里我们实现一个基于数组的**大顶堆**，包含 `siftUp` (向上调整), `siftDown` (向下调整), `insert` (插入), `pop` (删除堆顶), `buildHeap` (建堆) 功能。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// 定义堆的最大容量，根据题目要求调整
const int MAXN = 100005;

// 全局数组 h 存储堆元素，s 记录堆当前大小
int h[MAXN];
int s; // current size of the heap

// 辅助函数：交换两个元素
void swap(int &a, int &b)
{
    int t = a;
    a = b;
    b = t;
}

// 向上调整操作 (siftUp)
// 当元素 p 的值可能比父节点大时，将其向上移动
void siftUp(int p)
{
    // p > 1 表示当前节点不是根节点
    // h[p] > h[p / 2] 表示当前节点比父节点大 (大顶堆)
    while (p > 1 && h[p] > h[p / 2])
    {
        swap(h[p], h[p / 2]); // 交换当前节点与父节点
        p /= 2;               // 移动到父节点位置，继续向上调整
    }
}

// 向下调整操作 (siftDown)
// 当元素 p 的值可能比子节点小时，将其向下移动
void siftDown(int p)
{
    // 当 p 节点有子节点时才需要调整
    // 至少有左子节点 2*p <= s
    while (2 * p <= s)
    {
        int j = 2 * p; // j 指向左子节点

        // 如果存在右子节点，并且右子节点比左子节点大，则 j 指向右子节点
        if (j + 1 <= s && h[j + 1] > h[j])
        {
            j++; // j 指向优先级更高的子节点
        }

        // 如果当前节点 h[p] 已经比子节点中最大的 h[j] 大，则满足堆序性，停止调整
        if (h[p] >= h[j])
        {
            break;
        }

        // 否则，交换当前节点与优先级最高的子节点
        swap(h[p], h[j]);
        p = j; // 移动到子节点位置，继续向下调整
    }
}

// 插入操作
void insert(int v)
{
    s++;        // 堆大小加1
    h[s] = v;   // 将新元素放到堆的末尾
    siftUp(s);  // 从新元素位置开始向上调整
}

// 删除堆顶元素操作
// 返回被删除的堆顶元素
int pop()
{
    if (s == 0)
    {
        // 堆为空，无法删除
        std::cerr << "Error: Heap is empty!" << std::endl;
        return -1; // 或者抛出异常
    }

    int top_value = h[1]; // 记录堆顶元素

    h[1] = h[s]; // 将最后一个元素移动到堆顶
    s--;         // 堆大小减1

    siftDown(1); // 从堆顶位置开始向下调整

    return top_value;
}

// 获取堆顶元素（不删除）
int top()
{
    if (s == 0)
    {
        std::cerr << "Error: Heap is empty!" << std::endl;
        return -1;
    }
    return h[1];
}

// 判断堆是否为空
bool empty()
{
    return s == 0;
}

// 从一个无序数组构建堆
// 假设数组元素已经提前存入 h[1] 到 h[initial_s]
void buildHeap(int initial_s)
{
    s = initial_s;
    // 从最后一个非叶子节点开始，向上遍历到根节点，依次进行向下调整
    // 叶子节点本身就是合法的堆，无需调整
    for (int i = s / 2; i >= 1; --i)
    {
        siftDown(i);
    }
}

int main()
{
    // 示例用法
    std::cout << "--- 大顶堆操作示例 ---" << std::endl;

    // 1. 插入操作
    std::cout << "插入元素: 5, 3, 8, 1, 6" << std::endl;
    insert(5);
    insert(3);
    insert(8);
    insert(1);
    insert(6);

    std::cout << "当前堆顶元素: " << top() << std::endl; // 应该是 8
    std::cout << "堆中元素 (数组表示): ";
    for (int i = 1; i <= s; ++i)
    {
        std::cout << h[i] << " ";
    }
    std::cout << std::endl; // 期望: 8 6 5 1 3

    // 2. 删除操作
    std::cout << "删除堆顶元素: " << pop() << std::endl; // 应该删除 8
    std::cout << "当前堆顶元素: " << top() << std::endl; // 应该是 6
    std::cout << "堆中元素 (数组表示): ";
    for (int i = 1; i <= s; ++i)
    {
        std::cout << h[i] << " ";
    }
    std::cout << std::endl; // 期望: 6 3 5 1

    std::cout << "删除堆顶元素: " << pop() << std::endl; // 应该删除 6
    std::cout << "当前堆顶元素: " << top() << std::endl; // 应该是 5
    std::cout << "堆中元素 (数组表示): ";
    for (int i = 1; i <= s; ++i)
    {
        std::cout << h[i] << " ";
    }
    std::cout << std::endl; // 期望: 5 1 3

    // 3. 建堆操作
    std::cout << "\n--- 建堆操作示例 ---" << std::endl;
    s = 0; // 重置堆
    int initial_elements[] = {0, 3, 1, 4, 1, 5, 9, 2, 6}; // 索引0空着，从1开始
    for (int i = 1; i <= 8; ++i)
    {
        h[i] = initial_elements[i];
    }
    buildHeap(8); // 从8个元素构建堆

    std::cout << "通过建堆构建后的堆顶元素: " << top() << std::endl; // 应该是 9
    std::cout << "堆中元素 (数组表示): ";
    for (int i = 1; i <= s; ++i)
    {
        std::cout << h[i] << " ";
    }
    std::cout << std::endl; // 期望: 9 6 4 1 5 3 2 1 (或类似，取决于具体调整路径)

    std::cout << "堆是否为空: " << (empty() ? "是" : "否") << std::endl;
    pop();
    pop();
    pop();
    std::cout << "删除3个元素后堆顶: " << top() << std::endl;
    std::cout << "堆是否为空: " << (empty() ? "是" : "否") << std::endl;

    return 0;
}
```

**复杂度分析总结**:

*   **空间复杂度**: `O(N)`，用于存储堆元素。
*   **时间复杂度**:
    *   `insert()`: `O(log N)`
    *   `pop()`: `O(log N)`
    *   `top()`: `O(1)`
    *   `empty()`: `O(1)`
    *   `buildHeap()`: `O(N)`
