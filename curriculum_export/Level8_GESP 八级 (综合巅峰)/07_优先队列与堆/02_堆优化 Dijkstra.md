## 深度知识点讲解

### 1. 朴素Dijkstra算法回顾与瓶颈分析

**核心思想**：Dijkstra算法是一种贪心算法，用于解决带非负权边的单源最短路径问题。它维护一个集合 `S`，表示已经确定最短路径的顶点。每次从不在 `S` 中的顶点中，选择一个当前离源点最近的顶点 `u`，将其加入 `S`。然后，以 `u` 为中转点，尝试“松弛”（更新）所有与 `u` 相邻的顶点 `v` 的距离。

**朴素Dijkstra算法步骤**：
1.  初始化：源点 `s` 到自身距离为0，到其他所有点距离为无穷大。
2.  循环 `V` 次（`V` 为顶点数）：
    a.  在所有尚未确定最短路径的顶点中，找到一个距离源点最近的顶点 `u`。
    b.  将 `u` 标记为已确定最短路径（加入 `S` 集合）。
    c.  遍历 `u` 的所有邻居 `v`，如果通过 `u` 到达 `v` 的路径更短（即 `dist[u] + weight(u,v) < dist[v]`），则更新 `dist[v]`。

**瓶颈分析**：
步骤 2a 是朴素Dijkstra算法的性能瓶颈。在最坏情况下，每次循环都需要遍历所有 `V` 个尚未确定最短路径的顶点来寻找最小值。这个查找操作的时间复杂度是 O(V)。由于外层循环执行 `V` 次，总的时间复杂度就至少有 O(V^2)。当图的顶点数 `V` 很大时，`V^2` 会变得非常巨大，导致算法效率低下。

### 2. 优先队列（Priority Queue）登场

**优先队列是什么？**
优先队列是一种抽象数据类型，它允许我们以特定的优先级来访问元素。最常见的操作是：
*   **插入 (Push)**：将一个元素放入队列。
*   **取出最小/最大 (Pop)**：移除并返回队列中优先级最高（最小或最大）的元素。
*   **查看最小/最大 (Top)**：返回队列中优先级最高（最小或最大）的元素，但不移除。

**生活类比**：
想象一个“VIP等候区”。普通队列是“先来先服务”，而VIP等候区则是“VIP优先”。你把人（元素）放进去，VIP等候区会自动把最尊贵的VIP（优先级最高的元素）安排在最前面。当我们说“优先队列”时，默认通常是“最小优先队列”（Min-Priority Queue），也就是总是能取出最小的元素。

**优先队列与堆 (Heap) 的关系**：
优先队列通常使用**堆 (Heap)** 来实现。堆是一种特殊的树形数据结构，它满足堆的性质：
*   **小根堆 (Min-Heap)**：每个父节点的值都小于或等于其子节点的值。这样，堆顶元素（根节点）就是整个堆中最小的元素。
*   **大根堆 (Max-Heap)**：每个父节点的值都大于或等于其子节点的值。堆顶元素是整个堆中最大的元素。

堆的插入和删除操作的时间复杂度都是 O(logN)，其中 N 是堆中元素的数量。正是这种高效的查找和维护最值的能力，让堆成为优化Dijkstra算法的理想选择。

### 3. 堆优化Dijkstra算法流程

有了优先队列这个“智能助手”，我们可以将朴素Dijkstra的步骤 2a 替换掉。

**核心思想**：我们不再手动遍历所有未访问节点寻找最小值，而是将所有“可能成为下一个访问点”的节点及其当前最短距离放入一个最小优先队列中。优先队列会自动保证每次取出的都是当前距离源点最近且未最终确定最短路径的节点。

**堆优化Dijkstra算法步骤**：
1.  **初始化**：
    *   `dist[s] = 0`，`dist[v] = INF` (无穷大) 对于所有 `v != s`。
    *   创建一个最小优先队列 `pq`。将 `(0, s)` 插入 `pq`，表示从源点 `s` 到自身的距离为0。
    *   创建一个 `visited` 数组（或 `bool` 数组）`vis[V]`，初始化为 `false`，用于标记节点是否已确定最短路径。

2.  **循环**：当优先队列不为空时，执行以下操作：
    a.  **取出最小**：从 `pq` 中取出优先级最高的元素 `(d, u)`。`d` 是当前从源点到 `u` 的最短距离，`u` 是节点编号。
    b.  **判断访问**：如果节点 `u` 已经被访问过（`vis[u]` 为 `true`），说明我们已经找到了 `u` 的最短路径并处理过了，或者 `pq` 中存在旧的、更长的路径信息，直接跳过。
    c.  **标记访问**：将 `u` 标记为已访问 `vis[u] = true`。此时，`dist[u]` 的值就是从源点到 `u` 的最终最短路径。
    d.  **松弛操作**：遍历 `u` 的所有邻居 `v`，设边权为 `w`：
        *   如果 `dist[u] + w < dist[v]`，说明通过 `u` 到达 `v` 的路径更短。
        *   更新 `dist[v] = dist[u] + w`。
        *   将新的距离信息 `(dist[v], v)` 插入优先队列 `pq`。

**为什么 `vis` 数组是必要的？**
在堆优化Dijkstra中，一个节点可能会多次被插入优先队列。例如，先通过路径A发现 `v` 的距离是10，`pq` 中有 `(10, v)`。后来通过路径B发现 `v` 的距离是5，`pq` 中又有 `(5, v)`。当我们取出 `(5, v)` 时，`v` 的最短路径被确定，此时 `vis[v]` 设为 `true`。之后如果又取出了 `(10, v)`，因为 `vis[v]` 已经是 `true`，我们就可以直接跳过，避免重复处理和不必要的松弛操作。

### 4. 时间复杂度分析

*   **图的存储**：通常使用邻接表存储图，对于有 `V` 个顶点和 `E` 条边的图，初始化邻接表的时间复杂度为 O(V+E)。
*   **初始化**：`dist` 数组和 `vis` 数组的初始化为 O(V)。
*   **优先队列操作**：
    *   每个顶点 `u` 最多会被从优先队列中取出一次（因为有 `vis` 数组的判断）。取出一个元素的操作是 O(logK)，其中 K 是优先队列中的元素数量。在最坏情况下，K 可以达到 O(E)。所以 V 次取出操作的总时间复杂度为 O(V log E)。
    *   每次松弛操作 (`dist[u] + w < dist[v]`) 都会导致一个元素被插入优先队列。在最坏情况下，每条边都可能触发一次松弛操作，所以最多会有 `E` 次插入操作。每次插入操作的时间复杂度也是 O(logK)，总时间复杂度为 O(E log E)。

**总时间复杂度**：O(V log E + E log E) = **O(E log E)**。
由于在连通图中，`E >= V-1`，且 `E <= V^2`。当图是稀疏图时，`E` 接近 `V`，则时间复杂度接近 `O(V log V)`。当图是稠密图时，`E` 接近 `V^2`，则时间复杂度接近 `O(V^2 log V^2)`，但通常我们更关注 `O(E log V)` 或 `O(E log E)`。更精确地说，因为堆中最多有 `E` 个元素，所以操作是 `log E`。又因为 `E` 最多是 `V^2`，所以 `log E` 最多是 `log V^2 = 2 log V`，因此也可以写成 `O(E log V)`。

**与朴素Dijkstra对比**：
*   朴素Dijkstra：O(V^2)
*   堆优化Dijkstra：O(E log V) 或 O(E log E)

显然，当 `E` 远小于 `V^2` (即稀疏图) 时，堆优化Dijkstra的性能提升非常显著。即使是稠密图，`log V` 也比 `V` 小得多，所以堆优化版本总是更优或至少不劣于朴素版本。

### 5. 常见误区与注意事项

*   **负权边问题**：Dijkstra算法（包括堆优化版）**不适用于**含有负权边的图。如果图中存在负权边，Dijkstra算法可能无法得到正确的最短路径。对于含有负权边的图，需要使用 Bellman-Ford 算法或 SPFA 算法（如果无负权环）。
*   **重边与自环**：邻接表天然支持重边（多条连接相同两个点的边）和自环（连接点自身的边）。Dijkstra算法能正确处理它们。
*   **无向图处理**：如果图是无向图，需要在邻接表中为每条边添加两个方向的记录（例如，从 `u` 到 `v` 和从 `v` 到 `u`）。
*   **`dist` 数组初始化为无穷大**：确保无穷大值足够大，能覆盖所有可能的路径长度，但又不能溢出整数类型。例如，`0x3f3f3f3f` 是一个常用的“无穷大”常量，它足够大（约10^9），且两个这样的值相加也不会溢出 `int`，同时也能作为 `memset` 的值。

## 典型例题精讲

### 例题一：最短路计数 (基本应用)

**题目描述**：
给定一个 `N` 个点 `M` 条边的有向图，每条边都有一个非负权值。请你求出从源点 `S` 到所有其他点的最短路径长度。

**输入格式**：
第一行包含三个整数 `N, M, S`，分别表示顶点数、边数和源点编号。
接下来 `M` 行，每行包含三个整数 `u, v, w`，表示存在一条从 `u` 到 `v` 权值为 `w` 的有向边。

**输出格式**：
共 `N` 行，每行一个整数，表示从源点 `S` 到对应点的最短路径长度。如果无法到达，则输出 `INF` (比如 `0x3f3f3f3f`)。

**思路分析**：
这是一道标准的单源最短路径问题，边权非负，因此可以直接套用堆优化Dijkstra算法。
1.  **图的表示**：使用邻接表存储图，`adj[u]` 存储一个 `pair<int, int>` 列表，表示与 `u` 相连的边 `(v, w)`。
2.  **距离数组**：`dist` 数组初始化为 `INF`，`dist[S] = 0`。
3.  **优先队列**：使用 `std::priority_queue`，存储 `pair<int, int>`，其中 `first` 是距离，`second` 是节点编号。为了实现最小优先队列，我们需要自定义比较器或存储负值。这里使用 `std::greater<std::pair<int, int>>`。
4.  **访问数组**：`vis` 数组记录节点是否已确定最短路径。
5.  **算法流程**：按照上述堆优化Dijkstra算法步骤实现。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// 定义无穷大常量，足够大且不会溢出
const int INF = 0x3f3f3f3f; 
const int MAXN = 100005; // 最大节点数

// 邻接表，存储图。adj[u] 存储 (v, w)，表示从 u 到 v 有一条权值为 w 的边。
std::vector<std::pair<int, int>> adj[MAXN]; 
int dist[MAXN]; // 存储源点到各个点的最短距离
bool vis[MAXN]; // 标记节点是否已确定最短路径

int main()
{
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL); // 解除cin和cout的绑定

    int n, m, s;
    std::cin >> n >> m >> s; // 读取顶点数N，边数M，源点S

    // 调整源点S为0-indexed，如果题目是1-indexed
    // 如果题目是1-indexed，可以不调整，但所有涉及节点编号的地方要保持一致
    // 这里假设题目是1-indexed，我们直接使用1-indexed
    
    for (int i = 0; i < m; ++i)
    {
        int u, v, w;
        std::cin >> u >> v >> w;
        adj[u].push_back({v, w}); // 添加有向边
    }

    // 初始化dist数组为INF，源点S到自身距离为0
    for (int i = 1; i <= n; ++i)
    {
        dist[i] = INF;
    }
    dist[s] = 0;

    // 优先队列，存储pair<距离, 节点编号>
    // std::greater<std::pair<int, int>> 使其成为最小优先队列
    std::priority_queue<std::pair<int, int>, 
                        std::vector<std::pair<int, int>>, 
                        std::greater<std::pair<int, int>>> pq;
    
    pq.push({0, s}); // 将源点及其距离0放入优先队列

    while (!pq.empty())
    {
        // 取出当前距离源点最近的节点
        int d = pq.top().first;
        int u = pq.top().second;
        pq.pop();

        // 如果u已经被访问过，或者当前取出的距离d比已知的dist[u]要大（说明是旧的、更长的路径信息），则跳过
        if (vis[u] || d > dist[u]) 
        {
            continue;
        }

        vis[u] = true; // 标记u为已访问，其最短路径已确定

        // 遍历u的所有邻居v
        for (auto& edge : adj[u])
        {
            int v = edge.first;
            int w = edge.second;

            // 如果通过u到达v的路径更短，则更新dist[v]并将其加入优先队列
            if (dist[u] + w < dist[v])
            {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }

    // 输出结果
    for (int i = 1; i <= n; ++i)
    {
        std::cout << dist[i] << (i == n ? "" : " "); // 格式化输出，最后一个数字后没有空格
    }
    std::cout << std::endl;

    return 0;
}
```

**时间复杂度**：
*   `N` 个顶点，`M` 条边。
*   图的初始化：O(N+M)。
*   `dist` 数组初始化：O(N)。
*   优先队列操作：`N` 次 `pop`，`M` 次 `push` (最坏情况)。每次操作 O(logK)，`K` 最大为 `M`。
*   总时间复杂度：O((N+M) log M) 或 O((N+M) log N) (通常写成 O(M log N) 或 O(M log M))。

### 例题二：特殊路径 (多起点/终点)

**题目描述**：
在一个国家里有 `N` 个城市，编号从 `1` 到 `N`。城市之间有 `M` 条单向道路，每条道路有其通行时间。现在有 `K` 个特殊城市，我们需要找到从**任意一个**特殊城市出发，到达**所有其他非特殊城市**的最短时间。如果一个城市无法到达，则认为时间为无穷大。

**输入格式**：
第一行包含三个整数 `N, M, K`。
第二行包含 `K` 个整数，表示 `K` 个特殊城市的编号。
接下来 `M` 行，每行包含三个整数 `u, v, w`，表示存在一条从 `u` 到 `v` 权值为 `w` 的有向边。

**输出格式**：
共 `N` 行，每行一个整数，表示从任意一个特殊城市出发到对应城市的最短时间。如果无法到达，输出 `INF`。

**思路分析**：
这道题是多源最短路径问题，但由于边权非负，我们可以将其转化为单源最短路径问题。
1.  **虚拟源点**：创建一个虚拟源点 `0`。从虚拟源点 `0` 到每个特殊城市 `S_i` 添加一条权值为 `0` 的有向边。
2.  **Dijkstra**：然后从这个虚拟源点 `0` 运行一次堆优化Dijkstra算法。这样，从虚拟源点 `0` 到任何一个城市 `v` 的最短路径，就相当于从任意一个特殊城市到 `v` 的最短路径。
3.  **结果处理**：输出 `dist` 数组中除了虚拟源点之外的所有城市的最短距离。

**代码实现**：

```cpp
#include <bits/stdc++.h>

const int INF = 0x3f3f3f3f;
const int MAXN = 100005; // 最大节点数，包括虚拟源点0

std::vector<std::pair<int, int>> adj[MAXN];
int dist[MAXN];
bool vis[MAXN];

int main()
{
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m, k;
    std::cin >> n >> m >> k;

    // 创建一个虚拟源点 0
    int virtual_s = 0; 

    // 将所有K个特殊城市作为虚拟源点的邻居，边权为0
    for (int i = 0; i < k; ++i)
    {
        int special_city;
        std::cin >> special_city;
        adj[virtual_s].push_back({special_city, 0});
    }

    // 读取M条边
    for (int i = 0; i < m; ++i)
    {
        int u, v, w;
        std::cin >> u >> v >> w;
        adj[u].push_back({v, w});
    }

    // 初始化dist数组，虚拟源点到自身距离为0
    for (int i = 0; i <= n; ++i) // 注意这里循环到n，包括虚拟源点0
    {
        dist[i] = INF;
    }
    dist[virtual_s] = 0;

    std::priority_queue<std::pair<int, int>, 
                        std::vector<std::pair<int, int>>, 
                        std::greater<std::pair<int, int>>> pq;
    
    pq.push({0, virtual_s}); // 将虚拟源点放入优先队列

    while (!pq.empty())
    {
        int d = pq.top().first;
        int u = pq.top().second;
        pq.pop();

        if (vis[u] || d > dist[u])
        {
            continue;
        }

        vis[u] = true;

        for (auto& edge : adj[u])
        {
            int v = edge.first;
            int w = edge.second;

            if (dist[u] + w < dist[v])
            {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }

    // 输出从任意特殊城市到其他城市的路径
    // 注意这里我们只关心1到N的城市，不关心虚拟源点0
    for (int i = 1; i <= n; ++i)
    {
        std::cout << dist[i] << (i == n ? "" : " ");
    }
    std::cout << std::endl;

    return 0;
}
```

**时间复杂度**：
与例题一类似，只是顶点数增加了一个虚拟源点。总时间复杂度仍为 O((N+M) log N) 或 O((N+M) log M)。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，如iostream, vector, queue等

// 定义无穷大常量，足够大且不会溢出。
// 0x3f3f3f3f 是一个常用的值，约10^9，两个相加不会超过int最大值。
const int INF = 0x3f3f3f3f; 
// 定义最大节点数，根据题目数据范围调整
const int MAXN = 100005; 

// 邻接表存储图
// adj[u] 存储一个pair的vector，每个pair表示 (v, w)，即从u到v有一条权值为w的边。
std::vector<std::pair<int, int>> adj[MAXN]; 

// 存储源点到各个点的最短距离
// 初始化时，dist[source] = 0，其他为INF
int dist[MAXN]; 

// 标记节点是否已确定最短路径
// vis[u] 为 true 表示节点u的最短路径已经找到并最终确定
bool vis[MAXN]; 

// Dijkstra算法函数，参数：顶点数n，边数m，源点s
void dijkstra(int n, int s)
{
    // 1. 初始化距离数组和访问数组
    for (int i = 1; i <= n; ++i) // 假设节点编号从1到n
    {
        dist[i] = INF; // 所有点到源点距离初始化为无穷大
        vis[i] = false; // 所有点都未访问
    }
    dist[s] = 0; // 源点到自身的距离为0

    // 2. 创建优先队列
    // 优先队列存储pair<距离, 节点编号>
    // std::greater<std::pair<int, int>> 使其成为最小优先队列
    // 即队首元素是距离最小的
    std::priority_queue<std::pair<int, int>, 
                        std::vector<std::pair<int, int>>, 
                        std::greater<std::pair<int, int>>> pq;
    
    // 3. 将源点及其距离0放入优先队列
    pq.push({0, s}); // pair的first是距离，second是节点编号

    // 4. 循环直到优先队列为空
    while (!pq.empty())
    {
        // a. 取出当前距离源点最近的节点
        // pq.top().first 是距离
        // pq.top().second 是节点编号
        int d = pq.top().first;
        int u = pq.top().second;
        pq.pop(); // 移除队首元素

        // b. 判断是否已访问过 或 是否是旧的/更长的路径信息
        // 如果u已经被访问过 (vis[u] == true)，说明u的最短路径已确定，跳过
        // 如果当前取出的距离d比已知的dist[u]要大，说明这是通过更长的路径到达u的旧信息，也跳过
        if (vis[u] || d > dist[u]) 
        {
            continue;
        }

        // c. 标记u为已访问，其最短路径已确定
        vis[u] = true; 

        // d. 遍历u的所有邻居v，进行松弛操作
        for (auto& edge : adj[u]) // 遍历adj[u]中的所有pair (v, w)
        {
            int v = edge.first;  // 邻居节点v
            int w = edge.second; // 边(u, v)的权值w

            // 如果通过u到达v的路径更短 (dist[u] + w < dist[v])
            if (dist[u] + w < dist[v])
            {
                dist[v] = dist[u] + w; // 更新v的最短距离
                pq.push({dist[v], v}); // 将新的距离信息(dist[v], v)插入优先队列
            }
        }
    }
}

int main()
{
    // 优化C++标准库输入输出流，加快速度
    std::ios_base::sync_with_stdio(false); 
    std::cin.tie(NULL); 

    int n, m, s; // n: 顶点数, m: 边数, s: 源点
    std::cin >> n >> m >> s;

    // 读取m条边，构建邻接表
    for (int i = 0; i < m; ++i)
    {
        int u, v, w;
        std::cin >> u >> v >> w;
        adj[u].push_back({v, w}); // 添加有向边 (u -> v 权值 w)
        // 如果是无向图，则需要再添加一条反向边：
        // adj[v].push_back({u, w}); 
    }

    // 调用Dijkstra算法
    dijkstra(n, s);

    // 输出从源点s到所有其他点的最短距离
    for (int i = 1; i <= n; ++i)
    {
        std::cout << dist[i] << (i == n ? "" : " "); // 格式化输出，最后一个数字后没有空格
    }
    std::cout << std::endl;

    return 0;
}
```

## 课堂互动

1.  **思考题 (回顾)**：同学们，还记得朴素Dijkstra算法在哪个步骤上最耗时吗？为什么它会慢？
    *   *预期回答*：在“从所有未访问节点中找到距离源点最近的节点”这个步骤。因为每次都要遍历所有未访问节点，时间复杂度是O(V)。

2.  **思考题 (引入)**：优先队列是如何解决这个问题的？它有什么“魔法”能让查找最小值变得高效？
    *   *预期回答*：优先队列（底层是堆）能自动维护元素的顺序，每次取出时都能直接拿到优先级最高的（最小的）。插入和删除都是O(logN)时间。

3.  **小组任务 (讨论)**：在堆优化Dijkstra算法中，我们可能会把同一个节点以不同的距离多次插入优先队列。例如，先插入 `(10, v)`，后来又发现一条更短的路径 `(5, v)` 并插入。当我们取出 `(10, v)` 时，它已经不是 `v` 的最短路径了。我们的代码中是如何处理这种情况的？`vis` 数组和 `d > dist[u]` 这个判断分别起到了什么作用？
    *   *预期回答*：
        *   `vis[u]`：一旦 `u` 的最短路径被确定，`vis[u]` 设为 `true`。之后即使队列里还有 `(旧的距离, u)`，也会因为 `vis[u]` 为 `true` 而被跳过。
        *   `d > dist[u]`：这个判断用于处理 `u` 节点在队列中存在旧的、更长的路径信息的情况。当取出 `(d, u)` 时，如果 `d` 已经比 `dist[u]` 大了（说明 `dist[u]` 已经被更短的路径更新过），那么这个 `(d, u)` 也是无用的，直接跳过。这两个判断共同确保了每个节点只被“真正处理”一次。

4.  **即时练习 (模拟)**：
    假设有如下有向图（节点1-4）：
    1 --(1)--> 2
    1 --(4)--> 3
    2 --(2)--> 3
    2 --(5)--> 4
    3 --(1)--> 4
    请你从源点 `S=1` 开始，手动模拟堆优化Dijkstra算法的前几步，并写出优先队列的变化和 `dist` 数组的更新。
    *   *提示*：
        *   `dist = [0, INF, INF, INF]` (假设节点从1开始)
        *   `pq = [(0, 1)]`
        *   取出 `(0, 1)`。`vis[1]=true`。
        *   松弛 `1` 的邻居：
            *   `2`：`dist[1]+1 < dist[2]` -> `0+1 < INF` -> `dist[2]=1`，`pq.push((1, 2))`
            *   `3`：`dist[1]+4 < dist[3]` -> `0+4 < INF` -> `dist[3]=4`，`pq.push((4, 3))`
        *   此时 `pq = [(1, 2), (4, 3)]` (顺序可能不同，但 `(1, 2)` 优先级最高)
        *   继续取出 `(1, 2)`... (让学生继续)

5.  **拓展讨论**：如果题目要求我们找到从源点到终点的最短路径，并且还要输出这条路径具体经过了哪些节点，我们应该如何修改Dijkstra算法来记录路径？
    *   *预期回答*：可以增加一个 `path[MAXN]` 数组（或 `parent` 数组），`path[v] = u` 表示到达 `v` 的最短路径是通过 `u` 来的。在进行松弛操作 `if (dist[u] + w < dist[v])` 时，除了更新 `dist[v]`，还要更新 `path[v] = u`。最后从终点倒着回溯 `path` 数组就能得到路径。

2.  **限时通行 (路径记录)**
    *   **思路**：
        1.  `dist` 数组存储 `pair<int, int>`，`dist[i] = {最短时间, 最少边数}`。
        2.  优先队列存储 `pair<pair<int, int>, int>`，即 `{{时间, 边数}, 节点编号}`。
        3.  比较逻辑：当 `dist[u] + edge_cost` 与 `dist[v]` 比较时，如果 `dist[u].first + w < dist[v].first`，则更新 `dist[v]`。如果 `dist[u].first + w == dist[v].first` 且 `dist[u].second + 1 < dist[v].second`，也更新 `dist[v]`。
        4.  优先队列的比较器需要自定义，或者将 `pair` 的元素顺序调整为 `{-时间, -边数}` (使用默认大根堆) 或 `pair<pair<int, int>, int>, vector<...>, greater<...>>` (使用小根堆)。
    *   **代码要点**：
        ```cpp
        std::vector<std::pair<int, int>> adj[MAXN];
        std::pair<int, int> dist[MAXN]; // first: time, second: edges
        bool vis[MAXN];

        void dijkstra_extended(int n, int s) {
            for (int i = 1; i <= n; ++i) {
                dist[i] = {INF, INF}; // 初始化时间为INF，边数为INF
                vis[i] = false;
            }
            dist[s] = {0, 0};

            // 优先队列存储 {{时间, 边数}, 节点编号}
            std::priority_queue<std::pair<std::pair<int, int>, int>,
                                std::vector<std::pair<std::pair<int, int>, int>>,
                                std::greater<std::pair<std::pair<int, int>, int>>> pq;
            
            pq.push({{0, 0}, s});

            while (!pq.empty()) {
                std::pair<int, int> current_cost = pq.top().first;
                int u = pq.top().second;
                pq.pop();

                if (vis[u] || current_cost > dist[u]) { // 注意这里pair的比较是按first再按second
                    continue;
                }
                vis[u] = true;

                for (auto& edge : adj[u]) {
                    int v = edge.first;
                    int w = edge.second;

                    // 新路径的 {时间, 边数}
                    std::pair<int, int> new_cost = {dist[u].first + w, dist[u].second + 1};

                    if (new_cost < dist[v]) { // pair的比较会按first再按second
                        dist[v] = new_cost;
                        pq.push({dist[v
