## 知识点讲解

### 1. 全源最短路径问题

我们今天解决的问题叫做“全源最短路径问题”（All-Pairs Shortest Path）。顾名思义，就是要求图中任意两个顶点之间的最短路径。

### 2. Floyd 算法的核心思想：动态规划

Floyd 算法的核心思想是动态规划。它通过逐步考虑允许哪些顶点作为“中间节点”来更新最短路径。

想象一下，我们想从城市 `i` 到城市 `j`。最初，我们只知道它们之间是否有直达航班，或者没有直达航班就认为距离无限大。

现在，我们引入一个概念：**中间节点**。
*   最开始，我们不允许任何城市作为中间节点，那么从 `i` 到 `j` 的最短路径就是它们之间的直达路径（如果有）。
*   接着，我们允许城市 1 作为中间节点。这时，从 `i` 到 `j` 的最短路径，可能是原来的直达路径，也可能是 `i` 先到城市 1，再从城市 1 到 `j` 的路径。我们取这两种情况的最小值。
*   然后，我们允许城市 1 和城市 2 作为中间节点。从 `i` 到 `j` 的最短路径，可能是之前只允许城市 1 作中间节点时的最短路径，也可能是 `i` 经过城市 2 再到 `j` 的路径（其中 `i` 到城市 2 和城市 2 到 `j` 的路径都只允许城市 1 作中间节点）。
*   以此类推，直到我们允许所有城市都作为中间节点。

### 3. 状态定义

我们用一个二维数组 `d[i][j]` 来表示从顶点 `i` 到顶点 `j` 的最短路径长度。

更精确地，我们可以定义 `d[k][i][j]` 表示：**只允许经过编号 1 到 `k` 的顶点作为中间节点时，从顶点 `i` 到顶点 `j` 的最短路径长度。**

### 4. 状态转移方程

根据上面的状态定义，当我们要计算 `d[k][i][j]` 时，有两种可能：

1.  从 `i` 到 `j` 的最短路径**不经过**顶点 `k`。那么这条路径仍然只允许经过编号 1 到 `k-1` 的顶点作为中间节点，所以它的长度是 `d[k-1][i][j]`。
2.  从 `i` 到 `j` 的最短路径**经过**顶点 `k`。那么这条路径可以拆分成两段：`i` 到 `k`，以及 `k` 到 `j`。这两段路径都只允许经过编号 1 到 `k-1` 的顶点作为中间节点。所以，路径长度是 `d[k-1][i][k] + d[k-1][k][j]`。

综合以上两种情况，我们得到状态转移方程：
`d[k][i][j] = min(d[k-1][i][j], d[k-1][i][k] + d[k-1][k][j])`

等等！这个三维数组是不是有点大？而且 `d[k][i][j]` 的计算只依赖于 `d[k-1][...]`，这意味着我们可以优化空间！我们发现，`k` 这一维可以被省略掉！
当我们计算 `d[i][j]` 时，`d[i][k]` 和 `d[k][j]` 实际上是“**当前已经考虑了所有编号小于 `k` 的节点作为中间节点**”时的最短路径。
所以，最终的简洁版状态转移方程是：
`d[i][j] = min(d[i][j], d[i][k] + d[k][j])`

这里的 `k` 就是当前允许作为中间节点的顶点编号。

### 5. 算法步骤

Floyd 算法是一个非常简洁的三重循环：

1.  **初始化：**
    *   对于任意顶点 `i`，`d[i][i] = 0`（自己到自己的距离为 0）。
    *   对于任意两个不同顶点 `i` 和 `j`：
        *   如果 `i` 和 `j` 之间有直连边，则 `d[i][j]` 初始化为边的权重。
        *   如果 `i` 和 `j` 之间没有直连边，则 `d[i][j]` 初始化为一个足够大的值（表示无穷大，例如 `0x3f3f3f3f`），表示目前无法直接到达。

2.  **核心计算：**
    使用三层嵌套循环，外层循环遍历中间节点 `k`，内层两层循环遍历起点 `i` 和终点 `j`。
    ```cpp
    for (int k = 1; k <= n; ++k) { // k 作为中间节点
        for (int i = 1; i <= n; ++i) { // i 作为起点
            for (int j = 1; j <= n; ++j) { // j 作为终点
                // 尝试通过 k 中转，看是否能缩短 i 到 j 的路径
                if (d[i][k] != 0x3f3f3f3f && d[k][j] != 0x3f3f3f3f) { // 确保路径存在
                    d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }
    ```
    **注意：** `k` 的循环必须在最外层！这是 Floyd 算法能够正确工作的关键。因为它确保了在计算 `d[i][j]` 经过 `k` 中转时，`d[i][k]` 和 `d[k][j]` 已经考虑了所有编号小于 `k` 的节点作为中间节点的最短路径。

### 6. 时间复杂度和空间复杂度

*   **时间复杂度：** 三层嵌套循环，每层循环 `n` 次，所以是 `O(N^3)`。
*   **空间复杂度：** 使用一个 `N x N` 的二维数组存储距离，所以是 `O(N^2)`。

### 7. 算法特点

*   **优点：**
    *   算法结构简单，易于理解和实现。
    *   可以处理带有负权重的边（只要没有负权环）。
    *   适用于稠密图（边数接近 N^2 的图）。
*   **缺点：**
    *   时间复杂度较高，为 `O(N^3)`，当 `N` 较大时（例如 `N > 500`），可能无法承受。
    *   空间复杂度为 `O(N^2)`。
    *   无法处理负权环。如果图中存在从某个顶点出发又回到该顶点的负权环，Floyd 算法会不断地尝试通过这个环来“缩短”路径，导致距离变为负无穷。

### 8. 负权环检测

如果算法结束后，发现 `d[i][i]` 小于 0，就说明图中存在负权环。这是因为从 `i` 经过负权环回到 `i` 的路径长度会是负值，比 `0` 更短。

## 代码示例

下面是一个使用 Floyd 算法计算全源最短路径的 C++ 示例代码。

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件
using namespace std;     // 使用标准命名空间

const int MAXN = 105;    // 图中最大顶点数，根据题目要求调整
const int INF = 0x3f3f3f3f; // 定义一个足够大的值表示无穷大

int d[MAXN][MAXN]; // d[i][j] 表示从顶点 i 到顶点 j 的最短路径长度
int n, m;          // n 为顶点数，m 为边数

void floyd() {
    // k 作为中间节点
    for (int k = 1; k <= n; ++k) {
        // i 作为起点
        for (int i = 1; i <= n; ++i) {
            // j 作为终点
            for (int j = 1; j <= n; ++j) {
                // 如果 d[i][k] 或 d[k][j] 是无穷大，说明无法到达，也就无法通过 k 中转
                // 必须保证 d[i][k] 和 d[k][j] 都不是 INF，才能进行加法运算，避免溢出
                if (d[i][k] != INF && d[k][j] != INF) {
                    d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }
}

int main() {
    // 读入顶点数 n 和边数 m
    cin >> n >> m;

    // 初始化距离矩阵
    // 1. 所有 d[i][j] 都初始化为 INF (无穷大)
    // 2. d[i][i] 初始化为 0 (自己到自己的距离为 0)
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (i == j) {
                d[i][j] = 0;
            } else {
                d[i][j] = INF;
            }
        }
    }

    // 读入 m 条边，并更新距离矩阵
    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w; // 从 u 到 v 有一条权重为 w 的边
        // 对于重边（多条连接相同两个顶点的边），只保留权重最小的
        d[u][v] = min(d[u][v], w);
        // 如果是无向图，还需要 d[v][u] = min(d[v][u], w);
        // 这里假设是简单的有向图，如果题目是无向图，请自行添加 d[v][u] 的更新
    }

    // 执行 Floyd 算法
    floyd();

    // 输出所有点对之间的最短路径
    cout << "所有点对之间的最短路径矩阵：" << endl;
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (d[i][j] == INF) {
                cout << "INF "; // 如果是无穷大，输出 INF
            } else {
                cout << d[i][j] << " ";
            }
        }
        cout << endl;
    }

    // 示例：查询特定路径
    // cout << "从 1 到 3 的最短路径是: " << d[1][3] << endl;

    return 0;
}
```

**代码注释说明：**
*   `MAXN`：定义了图中最大顶点的数量。在算法竞赛中，通常会给一个范围，你需要根据范围来设定这个值。
*   `INF`：代表无穷大的一个整数值。通常使用 `0x3f3f3f3f`，它是一个足够大的整数（大约 10^9），且两个 `INF` 相加不会溢出 `int` 类型，也不会与正常的路径长度冲突。
*   `d[MAXN][MAXN]`：二维数组，存储从 `i` 到 `j` 的最短距离。
*   初始化时，`d[i][i]` 设置为 0，表示自己到自己的距离是 0。其他 `d[i][j]` (i!=j) 设置为 `INF`，表示初始时不可达。
*   输入边时，如果 `u` 到 `v` 存在多条边，我们只保留权重最小的那条。
*   `if (d[i][k] != INF && d[k][j] != INF)`：这个判断非常重要！它确保了我们只在 `i` 到 `k` 和 `k` 到 `j` 都有通路（不是 `INF`）的情况下，才尝试通过 `k` 中转。否则，如果 `d[i][k]` 是 `INF`，`INF + d[k][j]` 可能会溢出或者得到一个不正确的值。

**选择题答案：**
1.  **C. O(N^3)**
    *   **解析：** Floyd 算法有三层嵌套循环，每层循环 N 次，因此时间复杂度为 N * N * N = O(N^3)。
2.  **D. 动态规划**
    *   **解析：** Floyd 算法通过逐步扩大允许的中间节点集合来更新最短路径，这是一种典型的动态规划思想。
3.  **C. 路径的中间节点**
    *   **解析：** `k` 循环在最外层，表示当前允许通过编号为 `k` 的节点作为中间节点。
4.  **A. 含有负权边的图，但无负权环**
    *   **解析：** Floyd 算法可以正确处理负权边，但如果存在负权环，它会陷入无限“缩短”的循环，导致结果不正确（通常表现为 `d[i][i] < 0`）。稀疏图通常更适合用多次 Dijkstra 或 SPFA。Floyd 既可以处理有向图也可以处理无向图。
5.  **B. O(N^2)**
    *   **解析：** Floyd 算法使用一个 N*N 的二维数组来存储所有点对之间的最短距离，因此空间复杂度为 O(N^2)。

**填空题答案：**
1.  **0**
    *   **解析：** 自己到自己的距离为 0。
2.  **小于**
    *   **解析：** 如果存在负权环，从 `x` 经过负权环回到 `x` 的路径长度会是负值，比 `0` 更短，因此 `d[x][x]` 会被更新为负数。
3.  **w**
    *   **解析：** `d[u][v]` 应该被初始化为这条直连边的权重 `w`。
