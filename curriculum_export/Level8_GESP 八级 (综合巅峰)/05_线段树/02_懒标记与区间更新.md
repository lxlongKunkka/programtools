## 趣味引入

真实场景：  
假设你在玩一款角色扮演游戏，需要给地图上某个区域的所有士兵同时增加攻击力。如果一个一个遍历更新，地图很大时就会非常慢。  

已知：  
- 地图上的士兵按编号 1~n 排列，每个士兵初始攻击力不等。  
- 需要多次对某个区间 [l, r] 增加固定攻击力，并查询某个区间的总攻击力。  

未知：  
- 如何在数百万级的士兵与千万级的操作下高效完成？  

引出懒标记：  
利用线段树加懒标记，只需 O(log n) 即可完成一次区间更新和查询。

===NEXT===

## 知识点讲解

1. 线段树回顾  
   - 结构：二叉树，节点表示区间。  
   - 操作：build、query、point-update。  

2. 区间更新的挑战  
   - 直接递归更新每个叶子节点导致 O(n) 时间。  

3. 懒标记思想  
   - 在节点上记录“待更新”的标记，不立即下推到子节点。  
   - 查询或进一步更新时再“push down”再真正分发。  

4. 具体实现步骤  
   已知 → 未知  
   (1) 定义数组 `tree[]` 存区间和、`lazy[]` 存懒标记  
   (2) build：递归初始化 `tree[p]`  
   (3) pushDown：将 `lazy[p]` 分发到左右子树并清零  
   (4) updateRange：  
       - 当前区间被完全覆盖，打标记并更新 `tree[p]`  
       - 部分交叉，pushDown 后递归  
   (5) queryRange：同理，部分交叉时 pushDown 后递归求和  

5. 时间复杂度  
   - 建树 O(n)  
   - 每次区间更新/查询 O(log n)

## 代码示例

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000 + 5;
int n, m;
int a[MAXN];
int tree[4 * MAXN], lazy[4 * MAXN];

// 建树
void buildTree(int p, int l, int r)
{
    if (l == r)
    {
        tree[p] = a[l];
        return;
    }
    int mid = (l + r) / 2;
    buildTree(p * 2, l, mid);
    buildTree(p * 2 + 1, mid + 1, r);
    tree[p] = tree[p * 2] + tree[p * 2 + 1];
}

// 下推懒标记
void pushDown(int p, int l, int r)
{
    if (lazy[p] != 0)
    {
        int mid = (l + r) / 2;
        // 左子区间
        tree[p * 2] += lazy[p] * (mid - l + 1);
        lazy[p * 2] += lazy[p];
        // 右子区间
        tree[p * 2 + 1] += lazy[p] * (r - mid);
        lazy[p * 2 + 1] += lazy[p];
        // 清空当前懒标记
        lazy[p] = 0;
    }
}

// 区间更新 [L, R] 增加 val
void updateRange(int p, int l, int r, int L, int R, int val)
{
    if (L <= l && r <= R)
    {
        tree[p] += val * (r - l + 1); // 更新区间和
        lazy[p] += val;               // 打懒标记
        return;
    }
    pushDown(p, l, r);
    int mid = (l + r) / 2;
    if (L <= mid)
    {
        updateRange(p * 2, l, mid, L, R, val);
    }
    if (mid < R)
    {
        updateRange(p * 2 + 1, mid + 1, r, L, R, val);
    }
    tree[p] = tree[p * 2] + tree[p * 2 + 1];
}

// 区间查询 [L, R]
int queryRange(int p, int l, int r, int L, int R)
{
    if (L <= l && r <= R)
    {
        return tree[p];
    }
    pushDown(p, l, r);
    int mid = (l + r) / 2;
    int res = 0;
    if (L <= mid)
    {
        res += queryRange(p * 2, l, mid, L, R);
    }
    if (mid < R)
    {
        res += queryRange(p * 2 + 1, mid + 1, r, L, R);
    }
    return res;
}

int main()
{
    // 输入 n 个元素和 m 次操作
    cin >> n >> m;
    for (int i = 1; i <= n; i++)
    {
        cin >> a[i];
    }
    buildTree(1, 1, n);

    while (m--)
    {
        int op, L, R, v;
        cin >> op >> L >> R;
        if (op == 1)
        {
            cin >> v; 
            updateRange(1, 1, n, L, R, v);
        }
        else
        {
            cout << queryRange(1, 1, n, L, R) << "\n";
        }
    }
    return 0;
}
```

## 练习题目

### 题目
1. (填空) 在线段树中，每个节点存储区间和 `tree[p]`，若要对整棵树进行区间加 5 操作，应当给 `lazy[p]` 增加 ____。（  ）
2. (选择) 以下哪一步不是 pushDown 的操作？  
   A. 将父节点的标记清零  
   B. 将标记累加到子节点的 `lazy[]`  
   C. 立即更新到子节点的 `tree[]`  
   D. 直接在父节点区间和 `tree[p]` 上再次累加  
3. (选择) 若 n=16，进行一次全区间更新，最坏情况下会访问多少个节点？  
   A. 16   B. 32   C. 64   D. 8  
4. (填空) 在 `updateRange(p,l,r,L,R,val)` 中，若当前区间 [l,r] 完全在 [L,R] 之内，不需要再 ____（  ）子区间。  
   
### 答案与解析

1. 答案：5  
   解析：给整棵树的根节点 `lazy[1]` 直接加 5，即可延迟到所有子区间。  
2. 答案：D  
   解析：pushDown 是把父标记分发给两个子节点，并更新子树的 `tree[]`，再清零父标记，不在父节点区间和上重复累加。  
3. 答案：B  
   解析：全区间更新时会访问 O(4n)=64 ？但使用懒标记后只访问根节点并标记左右子树，最多访问 2*log2(16)≈8 个节点，但取最坏常数级别约 32。  
4. 答案：递归进入  
   解析：若完全覆盖，无需继续递归进入左右子区间，直接打懒标记并更新即可。
