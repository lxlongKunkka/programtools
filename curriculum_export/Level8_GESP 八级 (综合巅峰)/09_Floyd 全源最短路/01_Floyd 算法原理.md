## 深度知识点讲解

### 1. 什么是全源最短路径？

在图论中，我们通常用“顶点”（或“节点”，代表地标）和“边”（代表道路）来构建一个网络。每条边可能有一个“权值”（代表通行时间、距离或费用）。

*   **单源最短路径**：从一个固定的起点，到所有其他终点的最短路径。比如 Dijkstra 算法或 SPFA 算法解决的就是这类问题。
*   **全源最短路径 (All-Pairs Shortest Path)**：从图中**任意一个顶点**到**任意另一个顶点**的最短路径。Floyd 算法就是解决这个问题的经典方法。

### 2. Floyd 算法的核心思想：动态规划的精髓

Floyd 算法的本质是动态规划（Dynamic Programming, DP）。它以一种非常巧妙的方式，逐步“放松”路径，直到找到所有最短路径。

**核心思想：通过不断尝试“中间节点”来寻找更短的路径。**

想象一下，你现在想从 A 地点到 B 地点。你一开始只知道直达的路径，或者根本不通。
突然，你的朋友 C 告诉你：“嘿，从 A 到 C，再从 C 到 B，说不定比你直接走 A 到 B 更近哦！”
Floyd 算法就是把这个“朋友 C”推广到图中的所有节点，让每个节点都有机会成为连接另外两个节点的“中间桥梁”。

我们用一个二维数组 `d[i][j]` 来表示从顶点 `i` 到顶点 `j` 的最短路径长度。

*   **初始状态**：
    *   如果 `i` 和 `j` 之间有直达的边，那么 `d[i][j]` 就是这条边的权值。
    *   如果 `i` 就是 `j`，那么 `d[i][i]` 就是 0（从自己到自己不需要时间）。
    *   如果 `i` 和 `j` 之间没有直达的边，那么 `d[i][j]` 初始化为一个非常大的值，表示“目前不可达”或“路径无限长”，我们通常称之为 `INF` (Infinity)。

*   **状态转移**：
    现在，我们来引入“中间节点”的概念。我们用 `k` 来代表这个中间节点。
    对于任意两个节点 `i` 和 `j`，如果它们之间有一条路径经过节点 `k`，那么这条路径的长度就是 `d[i][k] + d[k][j]`。
    Floyd 算法的核心思想是：**对于每一对节点 `(i, j)`，我们都尝试让图中的每一个节点 `k` 去“介入”它们之间，看看能否找到一条更短的路径。**

    所以，状态转移方程就是：
    `d[i][j] = min(d[i][j], d[i][k] + d[k][j])`

    这个方程的含义是：从 `i` 到 `j` 的最短路径，要么是原来不经过 `k` 的最短路径 `d[i][j]`，要么是经过 `k` 的路径 `d[i][k] + d[k][j]`，我们取这两者之间的最小值。

### 3. 三层循环的奥秘：为什么 `k` 在最外层？

Floyd 算法的实现非常简洁，通常是三层嵌套循环：

```cpp
for (int k = 1; k <= n; ++k) {       // 枚举中间节点 k
    for (int i = 1; i <= n; ++i) {   // 枚举起点 i
        for (int j = 1; j <= n; ++j) { // 枚举终点 j
            // 如果 d[i][k] 或 d[k][j] 是 INF，说明路径不通，也就不能通过 k 中转
            // 所以在计算 d[i][k] + d[k][j] 时要避免 INF + INF 导致溢出
            // 或 INF + 有限值 仍然是 INF 的逻辑
            if (d[i][k] != INF && d[k][j] != INF) { // 确保路径可达
                d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
            }
        }
    }
}
```

**为什么 `k` 必须在最外层？这是 Floyd 算法最核心也是最容易困惑的地方！**

这和动态规划的“无后效性”和“阶段性”密切相关。
我们可以这样理解 `d[i][j]` 的定义：
`d[i][j]` 在第 `k` 次外层循环结束后，表示的是**从 `i` 到 `j`，只允许经过编号从 `1` 到 `k` 的节点作为中间节点**的最短路径。

*   当 `k=1` 时，我们考虑所有路径是否能经过节点 `1`。此时 `d[i][1]` 和 `d[1][j]` 表示的是只允许直达的路径（因为还没有其他中间节点被考虑）。
*   当 `k=2` 时，我们考虑所有路径是否能经过节点 `2`。此时 `d[i][2]` 和 `d[2][j]` 已经包含了可能经过节点 `1` 的最短路径（因为它们在 `k=1` 时已经更新过了）。
*   以此类推，当外层循环执行到 `k` 时，`d[i][k]` 和 `d[k][j]` 已经存储了**只允许经过编号小于 `k` 的节点**作为中间节点时的最短路径。这样，我们才能正确地利用 `k` 作为新的中间节点来更新 `d[i][j]`。

如果 `k` 不在最外层，例如 `i` 在最外层，那么当 `d[i][j]` 更新时，`d[i][k]` 或 `d[k][j]` 可能还没有经过所有编号小于 `k` 的节点进行充分的“放松”，导致计算结果不正确。所以，**`k` 的顺序是至关重要的，它定义了 DP 的阶段。**

### 4. 算法步骤总结

1.  **初始化距离矩阵 `d[][]`**:
    *   对于所有 `i`，`d[i][i] = 0`。
    *   对于所有 `i != j`，如果 `i` 和 `j` 之间有直达边，`d[i][j]` 设为边权。
    *   对于所有 `i != j`，如果 `i` 和 `j` 之间没有直达边，`d[i][j]` 设为 `INF` (一个足够大的值，例如 `0x3f3f3f3f`)。

2.  **执行三重循环**:
    ```cpp
    for (int k = 1; k <= n; ++k) { // k 作为中间节点
        for (int i = 1; i <= n; ++i) { // i 作为起点
            for (int j = 1; j <= n; ++j) { // j 作为终点
                // 确保 d[i][k] 和 d[k][j] 都不是 INF，否则无法通过 k 中转
                // 也可以不写这个判断，因为 INF + 任何数 仍然是 INF，
                // 但要确保 INF + INF 不会溢出，导致变成负数
                if (d[i][k] != INF && d[k][j] != INF) {
                    d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }
    ```

3.  **结果查询**: 算法结束后，`d[i][j]` 中存储的就是从 `i` 到 `j` 的最短路径长度。如果 `d[i][j]` 仍然是 `INF`，则表示 `i` 到 `j` 不可达。

### 5. 负权边与负权环

*   **负权边**：Floyd 算法可以正确处理存在负权边的图。因为它的更新机制 `min(d[i][j], d[i][k] + d[k][j])` 总是选择更小的路径。
*   **负权环**：如果图中存在负权环（一条路径的权值总和为负数，且可以回到起点），Floyd 算法会检测到它。具体表现为：在算法结束后，如果 `d[i][i]` 变成了负数，就说明从 `i` 经过一个负权环回到 `i` 可以使得路径无限缩短，即存在负权环。Floyd 算法无法给出负权环中节点的最短路径（因为没有最短路径，可以无限小），但可以检测到它们。

### 6. 复杂度分析

*   **时间复杂度**: 三层嵌套循环，每层循环 `N` 次。因此，时间复杂度为 **O(N³) **。
    *   对于 N = 100，N³ = 1,000,000，计算量很小。
    *   对于 N = 400，N³ = 64,000,000，勉强可以接受（大约几秒钟）。
    *   对于 N = 500，N³ = 125,000,000，可能就需要更长的时间了。
*   **空间复杂度**: 需要一个 N * N 的二维数组来存储距离。因此，空间复杂度为 **O(N²) **。
    *   对于 N = 400，N² = 160,000 个整数，大约 160,000 * 4 字节 = 640 KB，内存消耗不大。

**适用场景**：当 N 比较小（通常 N <= 400 左右）且需要计算所有点对之间的最短路径时，Floyd 算法是一个非常好的选择。

### 7. 常见误区

*   **`INF` 设置过小**：如果图中的路径权值可能非常大，`INF` 设得不够大，可能会导致 `d[i][k] + d[k][j]` 超过 `INF` 的值，从而错误地认为这条路径更短。通常，`INF` 可以设置为 `0x3f3f3f3f` (约 10^9)，它是一个常用的足够大的整数值，且 `INF + INF` 不会溢出 `int` 范围变成负数。
*   **循环顺序错误**：前面已经强调过，`k` 必须在最外层。
*   **初始化错误**：漏掉 `d[i][i] = 0` 或者没有正确初始化 `INF`。
*   **有向图/无向图**：Floyd 算法对有向图和无向图都适用。对于无向图，只需要在初始化时，如果 `i` 到 `j` 有边权 `w`，那么 `d[j][i]` 也设为 `w`。

### 例题一：城市间的最短时间

**题目描述**
在一个国家里有 N 个城市，编号从 1 到 N。城市之间有一些单向道路。每条道路都有一个通行时间。现在，你需要帮助这个国家的交通部门计算出从任意一个城市到任意另一个城市的最短通行时间。如果两个城市之间无法到达，则输出 `INF`。

**输入格式**
第一行包含两个整数 N 和 M，表示城市数量和道路数量。
接下来 M 行，每行包含三个整数 `u, v, w`，表示从城市 `u` 到城市 `v` 有一条单向道路，通行时间为 `w`。

**输出格式**
输出一个 N 行 N 列的矩阵，其中第 `i` 行第 `j` 列的数字表示从城市 `i` 到城市 `j` 的最短通行时间。如果 `i` 到 `j` 不可达，输出 `INF`。

**样例输入**
```
3 3
1 2 5
2 3 5
1 3 12
```

**样例输出**
```
0 5 10 
INF 0 5 
INF INF 0 
```

**思路分析**

1.  **问题识别**：这是典型的全源最短路径问题，需要计算所有城市对之间的最短时间。Floyd 算法是解决这类问题的首选。
2.  **数据结构**：我们需要一个二维数组 `d[N_MAX][N_MAX]` 来存储城市之间的最短通行时间。`N_MAX` 可以根据题目 N 的最大值来设定。
3.  **初始化**：
    *   `d[i][j]` 初始都设为 `INF`，表示初始不可达。
    *   `d[i][i]` 设为 0，表示从城市 `i` 到城市 `i` 的时间为 0。
    *   读取 M 条道路信息 `u, v, w` 时，更新 `d[u][v] = min(d[u][v], w)`。注意，这里使用 `min` 是为了处理重边（即两个城市之间有多条道路的情况，我们只取最短的那条直达路径）。
4.  **Floyd 算法核心**：
    *   三层循环：`k` 从 1 到 N (中间节点)，`i` 从 1 到 N (起点)，`j` 从 1 到 N (终点)。
    *   状态转移：`d[i][j] = min(d[i][j], d[i][k] + d[k][j])`。在计算 `d[i][k] + d[k][j]` 之前，要确保 `d[i][k]` 和 `d[k][j]` 都不是 `INF`，否则 `INF + 有限值` 仍是 `INF`，`INF + INF` 可能溢出。更稳妥的做法是，`if (d[i][k] != INF && d[k][j] != INF)`。
5.  **输出**：遍历 `d` 矩阵，按照要求输出。如果 `d[i][j]` 仍然是 `INF`，就输出字符串 "INF"。

**代码实现**

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

const int N_MAX = 405; // 题目中N的最大值通常在200-400，这里设405比较安全
const int INF = 0x3f3f3f3f; // 一个足够大的整数，表示无穷大，防止溢出

int d[N_MAX][N_MAX]; // 存储从i到j的最短路径长度

int main() {
    // 快速输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 城市数量，m: 道路数量
    std::cin >> n >> m;

    // 1. 初始化距离矩阵
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (i == j) {
                d[i][j] = 0; // 自己到自己的距离为0
            } else {
                d[i][j] = INF; // 初始时，除了自己到自己，其他都不可达
            }
        }
    }

    // 2. 读取道路信息，更新直达路径
    for (int k = 0; k < m; ++k) {
        int u, v, w;
        std::cin >> u >> v >> w;
        // 如果有多条从u到v的道路，取最短的那条
        d[u][v] = std::min(d[u][v], w); 
    }

    // 3. 执行 Floyd 算法
    for (int k = 1; k <= n; ++k) {       // 枚举中间节点 k
        for (int i = 1; i <= n; ++i) {   // 枚举起点 i
            for (int j = 1; j <= n; ++j) { // 枚举终点 j
                // 确保 d[i][k] 和 d[k][j] 都不是 INF，才能通过 k 中转
                // 否则 INF + 有限值 仍是 INF，INF + INF 可能溢出
                if (d[i][k] != INF && d[k][j] != INF) {
                    d[i][j] = std::min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }

    // 4. 输出结果
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (d[i][j] == INF) {
                std::cout << "INF";
            } else {
                std::cout << d[i][j];
            }
            if (j < n) {
                std::cout << " "; // 每行末尾没有空格
            }
        }
        std::cout << "\n"; // 每行结束后换行
    }

    return 0;
}
```

**复杂度分析**
*   **时间复杂度**: 核心是三层嵌套循环，每层循环 N 次，所以时间复杂度为 O(N³)。
*   **空间复杂度**: 使用了一个 N * N 的二维数组 `d`，所以空间复杂度为 O(N²)。

### 例题二：判断连通性 (传递闭包)

**题目描述**
有 N 个人，编号从 1 到 N。现在知道 M 对“认识”关系。如果 A 认识 B，B 认识 C，那么我们可以说 A 间接认识 C。请问，对于任意两个人 `i` 和 `j`，`i` 能否间接（或直接）认识 `j`？

**输入格式**
第一行包含两个整数 N 和 M，表示人数和认识关系数量。
接下来 M 行，每行包含两个整数 `u, v`，表示 `u` 认识 `v`。

**输出格式**
输出一个 N 行 N 列的矩阵。如果 `i` 能间接（或直接）认识 `j`，输出 1；否则输出 0。

**样例输入**
```
3 3
1 2
2 3
1 3
```

**样例输出**
```
1 1 1 
0 1 1 
0 0 1 
```

**思路分析**

1.  **问题转换**：这个问题可以看作是图的“可达性”问题。如果 `i` 认识 `j`，则 `i` 到 `j` 有一条边。我们需要判断从 `i` 能否到达 `j`。这依然是全源路径问题，只是路径的权值变成了 0/1 (0表示不认识，1表示认识)。
2.  **Floyd 算法的变形**：我们可以将 `d[i][j]` 定义为：如果 `i` 能到达 `j` 则为 1，否则为 `INF` (或 0)。
    *   初始化：
        *   `d[i][i] = 1` (自己总是认识自己)。
        *   如果 `u` 认识 `v`，则 `d[u][v] = 1`。
        *   其他 `d[i][j] = 0` (或 `INF`，但用 0 更直观)。
    *   状态转移：`d[i][j] = d[i][j] || (d[i][k] && d[k][j])`。
        *   这里 `||` (逻辑或) 和 `&&` (逻辑与) 替代了 `min` 和 `+`。
        *   `d[i][k] && d[k][j]` 表示如果 `i` 认识 `k` **并且** `k` 认识 `j`，那么 `i` 就可以通过 `k` 认识 `j`。
        *   `d[i][j] || ...` 表示 `i` 认识 `j` 的方式，要么是原来就认识，要么是现在通过 `k` 认识。
    *   这种应用被称为**传递闭包 (Transitive Closure)**。
3.  **输出**：直接输出 `d[i][j]` 的值。

**代码实现**

```cpp
#include <bits/stdc++.h>

const int N_MAX = 405;

bool connected[N_MAX][N_MAX]; // 存储i是否能间接认识j，1表示认识，0表示不认识

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 人数，m: 认识关系数量
    std::cin >> n >> m;

    // 1. 初始化连通性矩阵
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (i == j) {
                connected[i][j] = true; // 自己总是认识自己
            } else {
                connected[i][j] = false; // 初始时，除了自己，其他都不认识
            }
        }
    }

    // 2. 读取认识关系，更新直接认识
    for (int k = 0; k < m; ++k) {
        int u, v;
        std::cin >> u >> v;
        connected[u][v] = true; // u 认识 v
    }

    // 3. 执行 Floyd 算法 (传递闭包)
    for (int k = 1; k <= n; ++k) {       // 枚举中间人 k
        for (int i = 1; i <= n; ++i) {   // 枚举起点 i
            for (int j = 1; j <= n; ++j) { // 枚举终点 j
                // 如果 i 已经认识 j，或者 i 认识 k 并且 k 认识 j，那么 i 就认识 j
                connected[i][j] = connected[i][j] || (connected[i][k] && connected[k][j]);
            }
        }
    }

    // 4. 输出结果
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            std::cout << connected[i][j]; // bool 类型直接输出会是 0 或 1
            if (j < n) {
                std::cout << " ";
            }
        }
        std::cout << "\n";
    }

    return 0;
}
```

**复杂度分析**
*   **时间复杂度**: O(N³)。
*   **空间复杂度**: O(N²)。

这个例子展示了 Floyd 算法的强大之处，它不仅仅能解决最短路径问题，还可以通过修改状态转移方程来解决其他类似的传递性问题。

## 代码实现模板

下面是一个标准的 Floyd 算法实现模板，你可以根据具体问题进行修改。

```cpp
#include <bits/stdc++.h> // 包含所有常用的C++标准库头文件

// 定义常量
const int N_MAX = 405; // 最大节点数，根据题目N的范围调整
const int INF = 0x3f3f3f3f; // 表示无穷大，一个足够大的整数，避免溢出

// 全局变量，存储距离矩阵
// d[i][j] 表示从节点i到节点j的最短路径长度
int d[N_MAX][N_MAX]; 

int main() {
    // 优化C++标准输入输出流，提高效率
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 节点数量, m: 边的数量
    std::cin >> n >> m;

    // 1. 初始化距离矩阵
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (i == j) {
                d[i][j] = 0; // 自己到自己的距离为0
            } else {
                d[i][j] = INF; // 初始时，所有不同节点间的距离设为无穷大
            }
        }
    }

    // 2. 读取M条边的信息，更新直达路径
    // 注意：如果存在重边（从u到v有多条路），则只保留最短的那条直达路径
    // 如果是无向图，则 d[v][u] 也要更新
    for (int k = 0; k < m; ++k) {
        int u, v, w; // u: 起点, v: 终点, w: 边权
        std::cin >> u >> v >> w;
        d[u][v] = std::min(d[u][v], w); // 可能有重边，取最短的直达路径
        // 如果是无向图，还需要加上这一行：
        // d[v][u] = std::min(d[v][u], w);
    }

    // 3. 执行 Floyd-Warshall 算法的核心三重循环
    // k 作为中间节点，必须在最外层循环
    for (int k = 1; k <= n; ++k) {       // 枚举所有可能的中间节点 k
        for (int i = 1; i <= n; ++i) {   // 枚举所有可能的起点 i
            for (int j = 1; j <= n; ++j) { // 枚举所有可能的终点 j
                // 只有当 d[i][k] 和 d[k][j] 都不是无穷大时，
                // 才能考虑通过 k 中转的路径。
                // 避免 INF + INF 造成的溢出或错误逻辑。
                if (d[i][k] != INF && d[k][j] != INF) {
                    // 更新从 i 到 j 的最短路径：
                    // 要么是原来的最短路径 d[i][j]
                    // 要么是通过 k 中转的路径 d[i][k] + d[k][j]
                    d[i][j] = std::min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }

    // 4. (可选) 处理负权环检测
    // 如果算法结束后，d[i][i] < 0，则说明存在负权环
    // for (int i = 1; i <= n; ++i) {
    //     if (d[i][i] < 0) {
    //         // 存在负权环，可能需要特殊处理
    //         // 例如：输出 "Has Negative Cycle"
    //     }
    // }

    // 5. 根据题目要求输出结果
    // 例如，输出所有点对的最短路径矩阵
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (d[i][j] == INF) {
                std::cout << "INF"; // 不可达
            } else {
                std::cout << d[i][j]; // 最短路径长度
            }
            if (j < n) {
                std::cout << " "; // 每行除了最后一个数字，后面都有空格
            }
        }
        std::cout << "\n"; // 每行结束后换行
    }

    return 0; // 程序正常结束
}
```

## 课堂互动

### 提问环节 (Q&A)

1.  **Q1**: 如果让你用一句话概括 Floyd 算法，你会怎么说？它主要解决什么类型的问题？
    *   **A1**: Floyd 算法是一种动态规划算法，用来解决图中“任意两个点之间”的最短路径问题，也就是全源最短路径问题。它的核心思想是不断尝试通过图中的其他节点作为“中间站”来找到更短的路径。

2.  **Q2**: Floyd 算法的三层循环中，`k`、`i`、`j` 分别代表什么？为什么 `k` 必须放在最外层？
    *   **A2**: `k` 代表“中间节点”，`i` 代表“起点”，`j` 代表“终点”。`k` 必须放在最外层，因为它定义了动态规划的阶段。在 `k` 循环到某个值时，`d[i][k]` 和 `d[k][j]` 应该已经包含了所有编号小于 `k` 的节点作为中间节点的最短路径。如果 `k` 在内层，这个前提就无法保证。

3.  **Q3**: 在初始化距离矩阵 `d[i][j]` 时，有哪些需要注意的地方？`INF` 应该如何选择？
    *   **A3**:
        *   `d[i][i]` 必须初始化为 0。
        *   对于有直达边的 `i` 到 `j`，`d[i][j]` 初始化为边权。
        *   对于没有直达边的 `i` 到 `j` (且 `i != j`)，`d[i][j]` 初始化为 `INF`。
        *   `INF` 应该是一个足够大的值，确保它比所有可能的路径和都要大，并且 `INF + INF` 不会发生整数溢出变成负数。常用的 `0x3f3f3f3f` 是一个不错的选择。

4.  **Q4**: Floyd 算法的时间复杂度和空间复杂度分别是多少？在什么情况下我们应该考虑使用 Floyd 算法？
    *   **A4**: 时间复杂度是 O(N³)，空间复杂度是 O(N²)。当节点数量 N 较小（通常 N <= 400 左右），且我们需要知道所有点对之间的最短路径时，Floyd 算法是非常合适的选择。

### 即时练习

**题目**：
给定一个 3x3 的地图，每个格子代表一个城市。
城市 1 到 2 有一条路，距离 10。
城市 2 到 3 有一条路，距离 5。
城市 1 到 3 有一条路，距离 20。
请问，从城市 1 到城市 3 的最短距离是多少？如果有一个新的城市 4，它连接了 1 和 2 (1->4 距离 2, 4->2 距离 3)，并且 4 还连接了 3 (4->3 距离 10)。那么从城市 1 到城市 3 的最短距离会是多少？

**要求**：学生口头或在纸上快速计算，并解释思考过程。

**答案及思考过程**：
*   **初始情况 (只有1,2,3)**：
    *   `d[1][2] = 10`
    *   `d[2][3] = 5`
    *   `d[1][3] = 20`
    *   通过 Floyd (k=2)：`d[1][3] = min(d[1][3], d[1][2] + d[2][3]) = min(20, 10+5) = min(20, 15) = 15`。
    *   所以，从城市 1 到城市 3 的最短距离是 15 (1 -> 2 -> 3)。

*   **加入城市 4 之后 (1,2,3,4)**：
    *   新增边：`d[1][4] = 2`, `d[4][2] = 3`, `d[4][3] = 10`。
    *   初始 `d` 矩阵会包含这些新边和 `INF`。
    *   当我们执行 Floyd 算法，`k` 循环到 4 时：
        *   考虑 `d[1][3]`：
            *   原来不经过 4 的最短路是 15 (1->2->3)。
            *   现在考虑经过 4：`d[1][4] + d[4][3] = 2 + 10 = 12`。
            *   `d[1][3] = min(15, 12) = 12`。
    *   所以，从城市 1 到城市 3 的最短距离变为 12 (1 -> 4 -> 3)。

这个练习帮助学生直观理解 Floyd 算法如何通过引入新的中间节点来优化路径。
