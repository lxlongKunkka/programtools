想象一下这样一个场景：我们班有 $N$ 个小朋友，小 A、小 B、小 C…… 他们想在一条直线上排队玩“木头人”。每个人都有一个理想的位置 $x_i$（比如离起点多远）。但是，这些小朋友可不是随便排的，他们有很多“小要求”：

1.  **“小明，你必须在小红的左边，而且你们俩的距离不能超过 3 个座位！”**

    *   “小明在小红左边”意味着 $x_{明} < x_{红}$。
    *   “距离不超过 3 个座位”意味着 $x_{红} - x_{明} \le 3$。
2.  **“小刚，你得在小李的右边，而且至少隔着 2 个座位！”**
    *   小刚位置 $x_{刚}$，小李位置 $x_{李}$。
    *   “小刚在小李右边”意味着 $x_{刚} > x_{李}$。
    *   “至少隔 2 个座位”意味着 $x_{刚} - x_{李} \ge 2$。
3.  **“小王和小张，你们俩是好朋友，必须紧挨着！”**
    *   小王位置 $x_{王}$，小张位置 $x_{张}$。
    *   “紧挨着”意味着 $x_{王} = x_{张}$。

现在，问题来了：我们能不能找到一组所有小朋友都满意的位置 $x_1, x_2, \dots, x_N$ 呢？如果能，这组位置是什么？如果不能，我们怎么知道呢？

这些“小要求”是不是都长得很像 $x_i - x_j \le c$ 这样的形式？没错！这就是我们今天的主角——**差分约束系统**！它的任务就是解决这类由一系列形如“两个变量之差不大于一个常数”的不等式组成的系统。

今天，我们就来学习如何把这个排队问题，变成一个我们熟悉的“找最短路”问题，然后用强大的 SPFA 算法把它解决掉！是不是很神奇？让我们一起揭开它的神秘面纱吧！

## 深度知识点讲解

### 1. 什么是差分约束系统？

差分约束系统（Difference Constraint System）是由一系列形如 $x_i - x_j \le c_k$ 的线性不等式组成的系统。其中 $x_i, x_j$ 是未知变量，$c_k$ 是一个常数。我们的目标是找到一组满足所有这些不等式的变量值 $x_1, x_2, \dots, x_N$，或者判断这样的解是否存在。

**核心特征**: 所有的约束条件都是关于两个变量之差的上限。

### 2. 差分约束与最短路：神奇的转化！

这是差分约束最最核心的思想，也是它最“魔法”的地方！我们怎么能把一个不等式问题，变成一个图论问题呢？

还记得我们学过的**最短路算法**吗？比如 Dijkstra 或者 SPFA。它们的核心思想是**三角形不等式**（或者叫松弛操作）：
对于图中的任意一条边 $u \to v$，权值为 $w(u, v)$，从源点 $S$ 到 $u$ 的最短距离是 $d[u]$，从源点 $S$ 到 $v$ 的最短距离是 $d[v]$。那么，一定有：
$$d[v] \le d[u] + w(u, v)$$

这个不等式是不是有点眼熟？我们把它变形一下：
$$d[v] - d[u] \le w(u, v)$$

哇！这不正是我们差分约束系统中的 $x_i - x_j \le c_k$ 吗？！
如果我们将每个变量 $x_i$ 看作是从一个虚拟的“源点”到节点 $i$ 的最短距离 $d[i]$，那么：
*   不等式 $x_i - x_j \le c_k$ 就可以解释为：从节点 $j$ 到节点 $i$ 有一条权值为 $c_k$ 的有向边。
*   如果我们能找到一组满足所有三角形不等式的 $d[i]$ 值，那么这组 $d[i]$ 就是差分约束系统的一组可行解！

**思考：为什么是 $j \to i$ 呢？**
因为 $d[i]$ 是目标节点， $d[j]$ 是起始节点。在松弛操作中，我们用 $d[j] + w(j, i)$ 来尝试更新 $d[i]$。所以，从 $j$ 到 $i$ 的边，对应的是 $d[i] \le d[j] + w(j, i)$。

### 3. 如何构建图？

有了上面的转化思想，构建图就变得直观了：

1.  **节点**: 为每个变量 $x_i$ 创建一个节点 $i$。如果变量有 $N$ 个（从 $x_1$ 到 $x_N$），那么图就有 $N$ 个节点。
2.  **边**:
    *   对于每一个形如 $x_i - x_j \le c_k$ 的约束，从节点 $j$ 向节点 $i$ 连接一条有向边，其权值为 $c_k$。
    *   **超级源点（虚拟源点）S**:
        为了确保所有变量 $x_i$ 都能被计算出相对值，并且能够判断是否存在负权环，我们通常会引入一个额外的**超级源点 $S$**（通常编号为 0）。
        从这个超级源点 $S$ 向所有变量节点 $i$（$1 \le i \le N$）连接一条权值为 0 的有向边：$S \to i$，边权为 0。
        这相当于增加了 $N$ 个约束条件：$x_i - x_S \le 0$，即 $x_i \le x_S$。
        如果我们设定 $x_S = 0$，那么这些约束就变成了 $x_i \le 0$。这样做的目的是给所有变量一个初始的、宽松的上限，并确保所有节点都能从源点“被访问到”，从而计算出它们到源点的最短距离。
        **为什么是权值 0？** 因为我们不希望这些额外的边对原有的约束产生影响，只是为了“连接”所有节点。

### 4. 求解算法与无解判断

构建好图之后，我们就可以使用最短路算法来求解了。由于边权 $c_k$ 可能是负数，我们不能使用 Dijkstra 算法（Dijkstra 要求边权非负）。我们需要使用能够处理负权边的算法：**SPFA (Shortest Path Faster Algorithm)** 或 **Bellman-Ford 算法**。

*   **SPFA 算法**:
    *   初始化：`dist[S] = 0`，其余 `dist[i] = INF`。将 $S$ 加入队列。
    *   松弛：当从队列中取出一个节点 $u$，遍历其所有出边 $u \to v$，如果 `dist[v] > dist[u] + w(u, v)`，则更新 `dist[v]` 并将 $v$ 加入队列。
    *   **负权环检测**: SPFA 算法可以通过记录每个节点入队的次数 `cnt[i]` 来检测负权环。如果任何节点 $i$ 的 `cnt[i]` 超过了图的节点总数 $N+1$（包括超级源点），则说明存在负权环。
        **为什么负权环意味着无解？**
        想象一个环 $A \to B \to C \to A$，边权分别为 $w_{AB}, w_{BC}, w_{CA}$。如果 $w_{AB} + w_{BC} + w_{CA} < 0$，那么从 $A$ 出发绕一圈回到 $A$，距离会变得更小。这意味着 $d[A]$ 可以无限小，那么 $x_A$ 就可以无限小，这导致系统没有一个确定的解，或者说，无论你给 $x_A$ 赋什么值，总能找到一个更小的值来满足条件。因此，存在负权环的差分约束系统是无解的。

*   **Bellman-Ford 算法**:
    *   迭代 $N$ 轮（节点数，包括超级源点）。每轮遍历所有边，尝试进行松弛操作。
    *   **负权环检测**: 如果在第 $N+1$ 轮（或者第 $N$ 轮，取决于算法实现）仍然有边可以被松弛，则说明存在负权环。

**最终解**:
如果 SPFA 算法成功运行完毕，没有检测到负权环，那么 `dist[i]` 的值就是一组满足所有约束的解。具体来说，$x_i = \text{dist}[i]$。
这组解通常是所有满足条件解中的“相对最小值”解。因为最短路算法寻找的是从源点到各点的最小距离，这相当于给变量 $x_i$ 赋了一个尽可能小的值，同时满足所有约束。

### 5. 其他形式的约束转化

差分约束不仅限于 $x_i - x_j \le c$。其他形式的约束也可以转化：

1.  **$x_i - x_j \ge c$**:
    *   转化为 $x_j - x_i \le -c$。
    *   建立边：从 $i$ 向 $j$ 连接一条权值为 $-c$ 的有向边。

2.  **$x_i = x_j$**:
    *   等价于 $x_i - x_j \le 0$ 且 $x_j - x_i \le 0$。
    *   建立两条边：从 $j$ 向 $i$ 连接权值为 0 的边，从 $i$ 向 $j$ 连接权值为 0 的边。

3.  **$x_i \le c$**: (变量的绝对上限)
    *   引入超级源点 $S$ (节点 0)，将 $x_S$ 视为 0。
    *   转化为 $x_i - x_S \le c$。
    *   建立边：从 $S$ 向 $i$ 连接一条权值为 $c$ 的有向边。

4.  **$x_i \ge c$**: (变量的绝对下限)
    *   引入超级源点 $S$ (节点 0)，将 $x_S$ 视为 0。
    *   转化为 $x_S - x_i \le -c$。
    *   建立边：从 $i$ 向 $S$ 连接一条权值为 $-c$ 的有向边。

**总结**: 记住核心的三角形不等式 $d[v] \le d[u] + w(u, v) \Rightarrow d[v] - d[u] \le w(u, v)$。任何能转化成这个形式的约束，都可以用差分约束解决！

### 例题 1: 最短路模型下的差分约束

**题目描述**:
有 $N$ 个变量 $x_1, x_2, \dots, x_N$。给定 $M$ 个约束条件，每个条件形如 $x_a - x_b \le k$。请找出一组满足所有约束的解，如果无解，输出 -1。

**输入**:
第一行两个整数 $N, M$ ($1 \le N \le 1000, 1 \le M \le 5000$)。
接下来 $M$ 行，每行三个整数 $a, b, k$，表示约束 $x_a - x_b \le k$。

**输出**:
如果存在解，输出 $x_1, x_2, \dots, x_N$ 的值，每个值占一行。
如果无解，输出 -1。

**思路分析**:

1.  **变量与节点**: $N$ 个变量 $x_1, \dots, x_N$ 对应图中的 $N$ 个节点 $1, \dots, N$。
2.  **约束与边**: 对于每个约束 $x_a - x_b \le k$，根据 $d[a] - d[b] \le w(b, a)$ 的对应关系，我们从节点 $b$ 向节点 $a$ 连接一条权值为 $k$ 的有向边。
3.  **超级源点**: 为了计算所有 $x_i$ 的相对值并检测负权环，我们需要引入一个超级源点（虚拟源点）0。从节点 0 向所有节点 $1, \dots, N$ 连接一条权值为 0 的有向边。这对应约束 $x_i - x_0 \le 0$，即 $x_i \le x_0$。我们设 $x_0 = 0$。
4.  **求解算法**: 图中可能存在负权边，所以使用 SPFA 算法来计算从超级源点 0 到所有节点的最短距离。
5.  **无解判断**: 在 SPFA 算法执行过程中，如果某个节点入队次数超过 $N+1$ 次（总节点数是 $N$ 个变量节点加上 1 个超级源点），则说明存在负权环，系统无解，输出 -1。
6.  **输出解**: 如果 SPFA 正常结束，`dist[i]` 就是一组可行解 $x_i$。

**代码实现**:
我们将使用邻接表（数组模拟）和队列来实现 SPFA 算法。

```cpp
#include <bits/stdc++.h> // 包含常用头文件

const int MAXN = 1005; // 最大节点数 N+1 (1000个变量 + 1个超级源点)
const int MAXM = 5005; // 最大边数 M + N (M个约束 + N条超级源点到变量的边)
const int INF = 0x3f3f3f3f; // 无穷大，用于初始化距离

// 邻接表结构
struct Edge {
    int v;      // 边的终点
    int w;      // 边的权值
    int next;   // 下一条边的索引
};

Edge e[MAXM * 2]; // 存储边，因为一个约束可能对应一条边，超级源点也连N条边，所以大概2*MAXM足够
int head[MAXN];   // 邻接表头
int idx;          // 边的索引

int n, m;         // 变量数N，约束数M
int dist[MAXN];   // 存储从源点到各点的最短距离
int cnt[MAXN];    // 记录每个节点入队的次数，用于判断负权环
bool vis[MAXN];   // 记录节点是否在队列中

std::queue<int> q; // SPFA 队列

// 添加边函数
void add(int u, int v, int w) {
    e[idx].v = v;
    e[idx].w = w;
    e[idx].next = head[u];
    head[u] = idx++;
}

// SPFA 算法
bool spfa(int s) { // s 是源点
    // 初始化
    for (int i = 0; i <= n; ++i) { // 节点编号从0到N，0是超级源点
        dist[i] = INF;
        cnt[i] = 0;
        vis[i] = false;
    }

    dist[s] = 0;
    vis[s] = true;
    q.push(s);
    cnt[s]++;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        vis[u] = false; // 节点出队

        // 遍历所有从u出发的边
        for (int i = head[u]; i != -1; i = e[i].next) {
            int v = e[i].v;
            int w = e[i].w;

            // 松弛操作
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                if (!vis[v]) { // 如果v不在队列中
                    q.push(v);
                    vis[v] = true;
                    cnt[v]++;
                    if (cnt[v] > n + 1) { // 注意：总节点数是N+1 (0~N)，所以超过N+1次表示有负权环
                        return false; // 存在负权环，无解
                    }
                }
            }
        }
    }
    return true; // 没有负权环，有解
}

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出
    std::cin.tie(NULL);

    std::cin >> n >> m;

    // 初始化邻接表头
    for (int i = 0; i <= n; ++i) {
        head[i] = -1;
    }

    // 添加约束边
    for (int i = 0; i < m; ++i) {
        int a, b, k;
        std::cin >> a >> b >> k;
        // x_a - x_b <= k  =>  从 b 到 a 连一条权值为 k 的边
        add(b, a, k);
    }

    // 添加超级源点 0 到所有变量节点的边
    // 约束 x_i - x_0 <= 0 (即 x_i <= x_0)，从 0 到 i 连权值为 0 的边
    // 这是为了确保所有节点都能从源点到达，且提供一个初始的松弛基础
    for (int i = 1; i <= n; ++i) {
        add(0, i, 0);
    }

    // 运行 SPFA，源点为 0
    if (!spfa(0)) {
        std::cout << -1 << std::endl; // 存在负权环，无解
    } else {
        // 输出解
        // dist[i] 表示 x_i 的值。
        // 注意：x_0 通常被认为是 0，所以这里输出的是 x_i - x_0 的值。
        // 如果题目要求的是绝对值，且允许负数，那么dist[i]就是答案。
        // 如果题目要求非负解，可能需要对所有dist[i]加上一个常数，使得最小的dist[i]变为0。
        // 这里直接输出dist[i]，因为差分约束的解通常是相对的。
        for (int i = 1; i <= n; ++i) {
            std::cout << dist[i] << std::endl;
        }
    }

    return 0;
}
```

**复杂度分析**:
*   **时间复杂度**: SPFA 算法在最好情况下接近 $O(E)$，但在最坏情况下可能达到 $O(N \cdot E)$，其中 $N$ 是节点数（包括超级源点），$E$ 是边数。本题中 $N \approx 1000, M \approx 5000$，边数 $E \approx M+N \approx 6000$。最坏情况 $1000 \times 6000 = 6 \times 10^6$，在时限内通常是可以接受的。
*   **空间复杂度**: 邻接表需要 $O(N+E)$ 空间，`dist`, `cnt`, `vis` 数组需要 $O(N)$ 空间。总体为 $O(N+E)$，在本题中约为 $O(1000+6000)$，也是可以接受的。

### 例题 2: 多种约束类型的差分约束

**题目描述**:
有 $N$ 个人，编号为 $1, \dots, N$。现在要给他们发工资，每个人 $i$ 的工资为 $s_i$。有 $M$ 条约束：
1.  $s_a - s_b \le k$
2.  $s_a - s_b \ge k$
3.  $s_a = s_b$
4.  $s_a \le k$ (相对某个基准的上限)
5.  $s_a \ge k$ (相对某个基准的下限)
请你为每个人确定一个工资，使得所有约束都得到满足。如果存在多组解，输出任意一组即可。如果无解，输出 -1。

**输入**:
第一行两个整数 $N, M$ ($1 \le N \le 1000, 1 \le M \le 5000$)。
接下来 $M$ 行，每行描述一个约束：
*   `1 a b k`: 表示 $s_a - s_b \le k$
*   `2 a b k`: 表示 $s_a - s_b \ge k$
*   `3 a b`: 表示 $s_a = s_b$
*   `4 a k`: 表示 $s_a \le k$
*   `5 a k`: 表示 $s_a \ge k$

**输出**:
如果存在解，输出 $s_1, s_2, \dots, s_N$ 的值，每个值占一行。
如果无解，输出 -1。

**思路分析**:

这道题是例题 1 的扩展，包含了更多种类的约束。我们需要将每种约束都转化成 $x_i - x_j \le c$ 的形式，然后构建图并运行 SPFA。

1.  **节点**: $N$ 个人对应 $N$ 个变量 $s_1, \dots, s_N$，即图中的节点 $1, \dots, N$。
2.  **超级源点**: 同样引入超级源点 0。
3.  **约束转化与建边**:
    *   `1 a b k`: $s_a - s_b \le k$
        *   从 $b$ 向 $a$ 连边，权值 $k$。
    *   `2 a b k`: $s_a - s_b \ge k$
        *   转化为 $s_b - s_a \le -k$。
        *   从 $a$ 向 $b$ 连边，权值 $-k$。
    *   `3 a b`: $s_a = s_b$
        *   等价于 $s_a - s_b \le 0$ 且 $s_b - s_a \le 0$。
        *   从 $b$ 向 $a$ 连边，权值 0。
        *   从 $a$ 向 $b$ 连边，权值 0。
    *   `4 a k`: $s_a \le k$
        *   引入超级源点 $s_0=0$。转化为 $s_a - s_0 \le k$。
        *   从 $0$ 向 $a$ 连边，权值 $k$。
    *   `5 a k`: $s_a \ge k$
        *   引入超级源点 $s_0=0$。转化为 $s_0 - s_a \le -k$。
        *   从 $a$ 向 $0$ 连边，权值 $-k$。
4.  **超级源点到所有变量的初始边**: 除了上述约束外，我们仍然需要从超级源点 0 向所有变量节点 $1, \dots, N$ 连接一条权值为 0 的边。这对应 $s_i - s_0 \le 0$，即 $s_i \le s_0$。这提供了一个所有变量的初始上限（如果 $s_0=0$，则 $s_i \le 0$），确保所有节点都能被 SPFA 算法访问到。
5.  **求解与判断**: 运行 SPFA 算法，源点为 0。如果检测到负权环，输出 -1。否则，`dist[i]` 就是 $s_i$ 的一组可行解。

**代码实现**:
基本框架与例题 1 相同，主要是 `main` 函数中添加边的逻辑有所不同。

```cpp
#include <bits/stdc++.h>

const int MAXN = 1005; // N个变量 + 1个超级源点
const int MAXM = 5005 * 2 + MAXN; // 最多M*2条边 (s_a=s_b) + N条超级源点边
const int INF = 0x3f3f3f3f;

struct Edge {
    int v;
    int w;
    int next;
};

Edge e[MAXM]; // 边数组
int head[MAXN];
int idx;

int n, m;
int dist[MAXN];
int cnt[MAXN];
bool vis[MAXN];

std::queue<int> q;

void add(int u, int v, int w) {
    e[idx].v = v;
    e[idx].w = w;
    e[idx].next = head[u];
    head[u] = idx++;
}

bool spfa(int s) {
    for (int i = 0; i <= n; ++i) { // 节点0到N
        dist[i] = INF;
        cnt[i] = 0;
        vis[i] = false;
    }

    dist[s] = 0;
    vis[s] = true;
    q.push(s);
    cnt[s]++;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        vis[u] = false;

        for (int i = head[u]; i != -1; i = e[i].next) {
            int v = e[i].v;
            int w = e[i].w;

            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                if (!vis[v]) {
                    q.push(v);
                    vis[v] = true;
                    cnt[v]++;
                    if (cnt[v] > n + 1) { // 节点总数 N+1
                        return false; // 负权环
                    }
                }
            }
        }
    }
    return true; // 无负权环
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m;

    for (int i = 0; i <= n; ++i) {
        head[i] = -1;
    }

    // 添加超级源点 0 到所有变量节点的初始边
    // 约束 s_i - s_0 <= 0 (即 s_i <= s_0)，从 0 到 i 连权值为 0 的边
    // 这确保了所有节点可达，并提供一个初始的相对值
    for (int i = 1; i <= n; ++i) {
        add(0, i, 0); // 默认所有 s_i <= 0，或说 s_i - s_0 <= 0
    }

    for (int i = 0; i < m; ++i) {
        int type, a, b, k;
        std::cin >> type;
        if (type == 1) { // s_a - s_b <= k
            std::cin >> a >> b >> k;
            add(b, a, k);
        } else if (type == 2) { // s_a - s_b >= k  =>  s_b - s_a <= -k
            std::cin >> a >> b >> k;
            add(a, b, -k);
        } else if (type == 3) { // s_a = s_b  =>  s_a - s_b <= 0 and s_b - s_a <= 0
            std::cin >> a >> b;
            add(a, b, 0);
            add(b, a, 0);
        } else if (type == 4) { // s_a <= k  =>  s_a - s_0 <= k
            std::cin >> a >> k;
            add(0, a, k);
        } else if (type == 5) { // s_a >= k  =>  s_0 - s_a <= -k
            std::cin >> a >> k;
            add(a, 0, -k);
        }
    }

    if (!spfa(0)) {
        std::cout << -1 << std::endl;
    } else {
        for (int i = 1; i <= n; ++i) {
            // 如果题目要求输出非负解，且dist[i]可能为负，则需要进行偏移
            // 例如：找到 min_dist = min(dist[1...n])
            // 然后输出 dist[i] - min_dist
            // 但通常差分约束求的是相对值，直接输出即可
            std::cout << dist[i] << std::endl;
        }
    }

    return 0;
}
```

**复杂度分析**:
与例题 1 类似。边数可能会略多，因为一个等式约束会产生两条边。但总体复杂度仍然是 SPFA 的 $O(N \cdot E)$ 最坏情况。

## 代码实现模板

这里提供一个通用的 SPFA 算法模板，用于解决差分约束问题。

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

const int MAX_NODES = 1005; // 根据题目 N 的范围调整，通常 N+1 (0~N)
const int MAX_EDGES = 5005 * 2 + MAX_NODES; // 根据 M 的范围和约束类型调整，可能需要更多
const int INF = 0x3f3f3f3f; // 表示无穷大，用于初始化距离

// 邻接表结构体，用于存储图的边
struct Edge {
    int v;      // 边的终点 (to node)
    int w;      // 边的权值 (weight)
    int next;   // 指向下一条边的索引，用于链式存储
};

Edge edges[MAX_EDGES]; // 存储所有边的数组
int head[MAX_NODES];   // 邻接表头，head[u] 存储从节点 u 出发的第一条边的索引
int edge_idx;          // 边的当前索引，每次添加新边时递增

int node_count;        // 图的实际节点数 (例如 N 个变量 + 1 个超级源点)
int dist[MAX_NODES];   // dist[i] 存储从源点到节点 i 的最短距离
int visit_count[MAX_NODES]; // visit_count[i] 记录节点 i 入队的次数，用于检测负权环
bool in_queue[MAX_NODES];   // in_queue[i] 标记节点 i 是否在队列中

std::queue<int> q; // SPFA 算法的队列

// 初始化图的函数
void init_graph(int n_nodes) {
    node_count = n_nodes; // 设置图的节点总数
    edge_idx = 0;         // 重置边索引
    for (int i = 0; i <= node_count; ++i) {
        head[i] = -1;     // 初始化邻接表头为-1，表示没有边
    }
}

// 添加有向边 (u -> v, 权值 w)
void add_edge(int u, int v, int w) {
    edges[edge_idx].v = v;
    edges[edge_idx].w = w;
    edges[edge_idx].next = head[u]; // 将新边添加到链表头部
    head[u] = edge_idx++;           // 更新链表头，并递增索引
}

// SPFA 算法
// 参数 s: 源点 (通常是超级源点 0)
// 返回值: true 表示有解，false 表示存在负权环（无解）
bool spfa(int s) {
    // 初始化所有节点的距离为无穷大，入队次数为0，不在队列中
    for (int i = 0; i <= node_count; ++i) {
        dist[i] = INF;
        visit_count[i] = 0;
        in_queue[i] = false;
    }

    // 源点s到自身的距离为0
    dist[s] = 0;
    // 将源点s加入队列
    q.push(s);
    in_queue[s] = true;
    visit_count[s]++;

    while (!q.empty()) {
        int u = q.front(); // 取出队头节点
        q.pop();           // 弹出队头节点
        in_queue[u] = false; // 标记节点u已出队

        // 遍历从节点u出发的所有边
        for (int i = head[u]; i != -1; i = edges[i].next) {
            int
