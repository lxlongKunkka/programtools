## 教学目标

### 知识目标
1.  **理解概念**：学生能够准确理解差分约束系统的定义，即由一系列形如 `x_i - x_j <= c` 的线性不等式组成的系统。
2.  **掌握转化**：学生能够将各类差分约束条件（如 `x_i - x_j >= c`, `x_i - x_j == c`, `x_i <= c` 等）统一转化为图论中的最短路或最长路问题模型。
3.  **理解核心原理**：学生能够深入理解差分约束与图论中三角不等式（最短路 `dist[v] <= dist[u] + w` 或最长路 `dist[v] >= dist[u] + w`）之间的对应关系。
4.  **掌握超级源点**：学生能够理解超级源点（Super Source）在构建图模型、统一初始条件和确保连通性方面的作用。
5.  **负权/正权环检测**：学生能够理解在差分约束系统中，负权环或正权环的存在分别意味着无解（矛盾）或无限解。

### 能力目标
1.  **问题建模能力**：学生能够分析实际应用问题，识别其中的差分约束关系，并将其抽象为数学不等式。
2.  **图论构建能力**：学生能够根据转化后的不等式，正确构建有向图（包括节点、边及其权重）。
3.  **算法应用能力**：学生能够熟练运用 SPFA（或 Bellman-Ford）算法解决差分约束问题，包括最短路和最长路两种情况。
4.  **结果解释能力**：学生能够根据 SPFA 算法的输出（最短/最长路径值、负权/正权环）正确判断问题是否有解，并解释解的含义。

### 素养目标
1.  **抽象思维**：培养学生将实际问题抽象为数学模型和图论模型的能力。
2.  **逻辑推理**：提升学生在分析不等式关系、推导图边构建规则时的逻辑推理能力。
3.  **计算思维**：通过差分约束的学习，让学生感受数学工具（不等式）与算法（图论最短路）结合解决复杂问题的精妙之处，激发对算法学习的兴趣。
4.  **严谨细致**：在问题转化和代码实现过程中，培养学生严谨细致的编程习惯。

## 深度知识点讲解

### 1. 差分约束系统：不等式中的“相对论”

**是什么？**
差分约束系统 (Difference Constraint System) 是一组由形如 `x_i - x_j <= c_k` 的线性不等式组成的系统。其中 `x_i` 和 `x_j` 是未知变量，`c_k` 是常数。
这些不等式描述的是变量之间的“相对关系”：`x_i` 相对于 `x_j` 的最大差值不能超过 `c_k`。

**为什么重要？**
就像我们引入的垃圾分类故事，很多实际问题都可以抽象成这种形式：
*   **时间安排**：任务A必须在任务B完成前多久，或任务C必须在任务D开始后多久。
*   **资源分配**：分配给项目A的资源不能超过项目B多少。
*   **位置关系**：物体A在物体B的左边不超过多少距离，或右边至少多少距离。
*   **高度/权重**：A的高度比B高多少，或低多少。

这些问题往往需要我们找到一组满足所有限制的变量值，或者判断是否存在这样的解。

**怎么想到的？——与最短路/最长路的奇妙联系**

还记得图论中的最短路算法吗？比如 Bellman-Ford 或 SPFA。它们的核心思想是“松弛操作”：
对于一条从 `u` 到 `v` 且权值为 `w` 的边，如果 `dist[u] + w < dist[v]`，我们就更新 `dist[v] = dist[u] + w`。
这个条件 `dist[u] + w < dist[v]` 也可以写成 `dist[v] - dist[u] > w`。
而松弛操作的最终目标是让所有边都满足“三角不等式”：`dist[v] <= dist[u] + w`。
把这个不等式移项，就得到了：`dist[v] - dist[u] <= w`。

有没有发现，这个 `dist[v] - dist[u] <= w` 的形式，和我们的差分约束系统 `x_i - x_j <= c` 简直一模一样！
这不就是说，如果把 `x_i` 看作是某个节点 `i` 的最短路径值 `dist[i]`，把 `x_j` 看作是节点 `j` 的最短路径值 `dist[j]`，那么一个差分约束条件 `x_i - x_j <= c` 就对应着图中的一条从 `j` 到 `i`，权值为 `c` 的有向边！

**核心原理：**
1.  **最短路模型**：对于不等式 `x_i - x_j <= c`，我们可以在图中添加一条从节点 `j` 到节点 `i` 的有向边，其权值为 `c`。运行最短路算法 (如 SPFA)，如果存在解，那么 `dist[i]` 就是一组满足所有约束的 `x_i` 的最小可能值。
2.  **最长路模型**：有些问题可能需要找到变量的最大可能值。对于不等式 `x_i - x_j >= c`，我们可以将其看作 `x_i >= x_j + c`。这与最长路的松弛条件 `dist[v] >= dist[u] + w` 相似。此时，我们可以在图中添加一条从节点 `j` 到节点 `i` 的有向边，其权值为 `c`。运行最长路算法 (SPFA 稍作修改)，如果存在解，那么 `dist[i]` 就是一组满足所有约束的 `x_i` 的最大可能值。

### 2. 差分约束的转化规则

我们通常会把所有不等式都转化为 `x_i - x_j <= c` 的形式，然后用最短路算法解决。如果问题要求最大值，也可以转化为 `x_i - x_j >= c` 的形式，用最长路算法解决。

**A. 统一转化为最短路模型 (`x_i - x_j <= c`)**
*   **`x_i - x_j <= c`**：直接添加一条从 `j` 到 `i` 的有向边，权值为 `c`。
*   **`x_i - x_j >= c`**：移项得 `x_j - x_i <= -c`。添加一条从 `i` 到 `j` 的有向边，权值为 `-c`。
*   **`x_i - x_j == c`**：这等价于两个不等式：`x_i - x_j <= c` 和 `x_i - x_j >= c`。
    *   `x_i - x_j <= c`：添加从 `j` 到 `i` 的边，权值 `c`。
    *   `x_i - x_j >= c` 转化为 `x_j - x_i <= -c`：添加从 `i` 到 `j` 的边，权值 `-c`。
*   **`x_i <= c`**：引入一个“超级源点” `s` (通常编号为 `0` 或 `N+1`)，并假设 `x_s = 0`。那么 `x_i <= c` 等价于 `x_i - x_s <= c`。添加一条从 `s` 到 `i` 的有向边，权值为 `c`。
*   **`x_i >= c`**：等价于 `x_s - x_i <= -c` (假设 `x_s = 0`)。添加一条从 `i` 到 `s` 的有向边，权值为 `-c`。

**B. 统一转化为最长路模型 (`x_i - x_j >= c` 或 `x_i >= x_j + c`)**
*   **`x_i - x_j >= c`**：直接添加一条从 `j` 到 `i` 的有向边，权值为 `c`。
*   **`x_i - x_j <= c`**：移项得 `x_j - x_i >= -c`。添加一条从 `i` 到 `j` 的有向边，权值为 `-c`。
*   **`x_i - x_j == c`**：等价于 `x_i - x_j >= c` 和 `x_j - x_i >= -c`。
    *   `x_i - x_j >= c`：添加从 `j` 到 `i` 的边，权值 `c`。
    *   `x_j - x_i >= -c`：添加从 `i` 到 `j` 的边，权值 `-c`。
*   **`x_i <= c`**：等价于 `x_s - x_i >= -c` (假设 `x_s = 0`)。添加一条从 `i` 到 `s` 的有向边，权值为 `-c`。
*   **`x_i >= c`**：等价于 `x_i - x_s >= c` (假设 `x_s = 0`)。添加一条从 `s` 到 `i` 的有向边，权值为 `c`。

**选择最短路还是最长路？**
*   如果问题要求找到一组 `x_i` 的**最小**可能值，或者判断是否存在解，通常使用最短路模型。
*   如果问题要求找到一组 `x_i` 的**最大**可能值，或者最大化某个表达式 (如 `x_N - x_1`)，通常使用最长路模型。

### 3. 超级源点 (Super Source)

在构建差分约束图时，我们通常会引入一个“超级源点” `s` (通常编号为 `0` 或 `N+1`)。
**作用：**
1.  **统一参考点**：为了给所有变量 `x_i` 提供一个统一的初始参考点。我们假设 `x_s = 0`。
2.  **确保连通性**：为了让所有变量 `x_i` 都能被最短路/最长路算法“访问”到。我们通常会从 `s` 向所有节点 `i` 添加一条边。
    *   **最短路模型**：添加 `s -> i` 的边，权值为 `0`。这意味着 `x_i - x_s <= 0`，即 `x_i <= x_s`。如果 `x_s = 0`，则 `x_i <= 0`。这可以为所有 `x_i` 提供一个上界。
    *   **最长路模型**：添加 `s -> i` 的边，权值为 `0`。这意味着 `x_i - x_s >= 0`，即 `x_i >= x_s`。如果 `x_s = 0`，则 `x_i >= 0`。这可以为所有 `x_i` 提供一个下界。

**重要提示：**
差分约束系统求得的解 `dist[i]` 只是其中一组可行解。如果 `dist[i]` 是一个解，那么 `dist[i] + C` (其中 `C` 是任意常数) 也是一个解，因为所有的约束都是关于变量的“差值”，平移整个解空间并不会改变差值。通常，我们求得的 `dist[i]` 是在 `x_s = 0` (或其他设定) 的前提下，对 `x_i` 的最小/最大界限。

### 4. 负权环与正权环：无解的信号

*   **最短路模型中的负权环**：如果在 SPFA 算法执行过程中检测到负权环，这意味着存在一组不等式 `x_a - x_b <= w1`, `x_b - x_c <= w2`, ..., `x_k - x_a <= wk`，它们相加后得到 `0 <= sum(w_i)`。如果 `sum(w_i)` 是一个负数，那么 `0 <= 负数` 显然是矛盾的。这意味着这组不等式系统无解。
*   **最长路模型中的正权环**：同理，如果在使用最长路模型时检测到正权环，也意味着这组不等式系统无解。因为正权环会导致路径长度无限增大，无法找到有限的最大值。

**检测方法**：在 SPFA 算法中，我们可以记录每个节点进入队列的次数 `cnt[i]`。如果某个节点 `i` 入队次数超过了节点总数 `N`，就说明图中存在一个环，并且该环是一个负权环 (最短路) 或正权环 (最长路)。

### 5. SPFA算法回顾

SPFA (Shortest Path Faster Algorithm) 是 Bellman-Ford 算法的一种优化，通常用于稀疏图中含有负权边的情况。
**核心思想**：
1.  维护一个队列 `q`，存储待松弛的节点。
2.  初始化 `dist[s] = 0`，其他 `dist[i] = INF` (最短路) 或 `-INF` (最长路)。
3.  `s` 入队。
4.  当队列不为空时：
    *   取出队头 `u`，标记 `u` 不在队列中。
    *   遍历 `u` 的所有邻居 `v`：
        *   **最短路松弛**：如果 `dist[u] + w(u, v) < dist[v]`，则更新 `dist[v] = dist[u] + w(u, v)`，并让 `v` 入队 (如果 `v` 不在队列中)。
        *   **最长路松弛**：如果 `dist[u] + w(u, v
