## 趣味引入

同学们，想象一下我们生活在一个神秘的“时钟世界”里。这个时钟只有12个小时，比如下午3点过5个小时是8点，过10个小时是1点。这就是我们熟悉的模运算（模12）！

在这个时钟世界里，加减法都很容易。比如，从3点开始，往前走5个小时（+5），是8点。从8点开始，往回走5个小时（-5），又回到了3点。加法和减法是一对“好伙伴”，一个操作可以被另一个操作“撤销”。

但是，如果我想在时钟上做“乘法”呢？比如，从1点开始，每次跳2个小时，跳了3次（1 * 2 * 3 = 6），变成了7点（1 + 2 + 2 + 2 = 7，或者直接算 `1 + (2 * 3)`，但这里我们强调的是“乘”的动作，比如 `3 * 5` 在时钟上会是 `(3+3+3+3+3)`）。假设我们现在是 `A` 点，想知道 `A` 乘以 `B` 之后会停在哪里。这很简单，就是 `(A * B) % P`。

那“除法”呢？如果我告诉你，我从 `X` 点开始，每次跳 `B` 个小时，跳了一段时间后，停在了 `Y` 点。你能告诉我，我总共跳了多少次吗？这不就是 `Y / B` 吗？
但在模运算的世界里，`Y / B` 怎么算呢？ `(Y / B) % P` 并不等于 `(Y % P) / (B % P)`！直接除法会出问题。
比如，在模7的世界里，`6 / 3` 是多少？我们知道 `2 * 3 = 6`，所以 `6 / 3 = 2`。
但是 `(6 / 3) % 7` 怎么用模运算来算呢？
我们不能直接写 `(6 % 7) / (3 % 7)`。

这就引出了我们今天的主角——**乘法逆元**！
乘法逆元就像是模运算世界里的“倒数”，它可以帮助我们把模运算中的除法变成乘法。
想象一下，在普通数学中，`A / B` 等于 `A * (1/B)`。这里的 `1/B` 就是 `B` 的倒数。
在模运算中，如果有一个数 `invB`，它满足 `(B * invB) % P = 1`，那么 `invB` 就叫做 `B` 在模 `P` 意义下的乘法逆元！
有了它，我们的除法 `(A / B) % P` 就可以变成 `(A * invB) % P` 了！是不是很神奇？

今天，我们就来学习两种寻找这个神奇“倒数”的方法！

## 深度知识点讲解

### 1. 什么是乘法逆元？

在普通的实数运算中，对于一个非零数 `b`，它的倒数是 `1/b`，满足 `b * (1/b) = 1`。我们知道 `a / b` 等价于 `a * (1/b)`。

在模运算中，我们希望找到一个类似的“倒数”概念，来帮助我们进行“除法”。
**定义：** 如果整数 `a` 和 `p` 互质（即 `gcd(a, p) = 1`），那么存在一个整数 `x`，使得 `(a * x) % p = 1`。这个 `x` 就称为 `a` 在模 `p` 意义下的**乘法逆元**，通常记作 `a^(-1)`。

**核心作用：** 当我们需要计算 `(A / B) % P` 时，如果 `B` 在模 `P` 意义下有乘法逆元 `invB` (即 `B^(-1)`)，那么我们可以将其转化为 `(A * invB) % P`。这避免了在模运算中直接进行除法操作的困难。

**为什么需要互质？**
如果 `gcd(a, p) != 1`，那么 `a` 在模 `p` 意义下就没有乘法逆元。
例如，在模6的世界里，`2` 的逆元是多少？
`2 * 1 = 2 % 6`
`2 * 2 = 4 % 6`
`2 * 3 = 0 % 6`
`2 * 4 = 2 % 6`
`2 * 5 = 4 % 6`
你看，`2` 乘以任何数都得不到 `1 % 6`。因为 `gcd(2, 6) = 2 != 1`。
所以，乘法逆元存在的前提是 `a` 和 `p` 互质。
**在GSP八级中，我们主要遇到的情况是模数 `p` 是一个质数。当 `p` 是质数时，除了 `p` 的倍数，任何数 `a` 都与 `p` 互质，因此都存在乘法逆元。**

### 2. 方法一：费马小定理求逆元 (当模数 `p` 是质数时)

**前置知识回顾：**
*   **模运算**：`a % p` 表示 `a` 除以 `p` 的余数。
*   **质数（素数）**：只能被1和它本身整除的正整数，例如2, 3, 5, 7, 11...
*   **快速幂**：一种高效计算 `a^b % p` 的算法，时间复杂度为 `O(log b)`。

**费马小定理：**
如果 `p` 是一个质数，并且整数 `a` 不是 `p` 的倍数（即 `a` 与 `p` 互质），那么有：
`a^(p-1) ≡ 1 (mod p)`

这个定理非常强大！我们如何用它来求逆元呢？
我们知道逆元的定义是 `a * x ≡ 1 (mod p)`。
费马小定理告诉我们 `a^(p-1) ≡ 1 (mod p)`。
我们可以把 `a^(p-1)` 写成 `a * a^(p-2)`。
所以，`a * a^(p-2) ≡ 1 (mod p)`。
对比逆元的定义 `a * x ≡ 1 (mod p)`，我们发现，当 `x = a^(p-2)` 时，等式成立！

**因此，当 `p` 是质数且 `a` 不是 `p` 的倍数时，`a` 在模 `p` 意义下的乘法逆元就是 `a^(p-2) % p`。**

**生活类比：密码锁**
想象你有一个密码锁，它的密码是 `a`。你每次输入密码后，锁都会把 `a` 乘以一个神秘的因子 `X`，然后取模 `p`，如果结果是 `1`，锁就打开了。费马小定理就像是告诉你，如果你把 `a` 连续输入 `p-2` 次，那么它就相当于那个神秘的因子 `X`，就能打开锁！

**优点：**
*   简单直观，容易理解和实现。
*   对于单个逆元的计算非常高效，时间复杂度为 `O(log p)`（取决于快速幂的复杂度）。

**缺点：**
*   **限制条件：模数 `p` 必须是质数。** 如果 `p` 不是质数，费马小定理不适用。
*   不适合批量计算 `1` 到 `N` 所有数的逆元，因为每次计算都需要 `O(log p)`。

**代码实现思路：**
使用快速幂函数 `power(base, exp, mod)` 计算 `a^(p-2) % p`。

```cpp
long long power(long long base, long long exp, long long mod) {
    long long res = 1;
    base %= mod; // 确保base在模范围内
    while (exp > 0) {
        if (exp % 2 == 1) { // 如果指数是奇数
            res = (res * base) % mod;
        }
        base = (base * base) % mod; // 基数平方
        exp /= 2; // 指数减半
    }
    return res;
}

// 求a在模p意义下的逆元 (p为质数)
long long inverse(long long a, long long p) {
    // 根据费马小定理，a的逆元是a^(p-2) % p
    return power(a, p - 2, p);
}
```

### 3. 方法二：线性时间求逆元 (当模数 `p` 是质数时)

在某些问题中，我们需要计算 `1` 到 `N` 之间所有整数在模 `p` 意义下的逆元（例如，计算组合数 `C(n, k)` 需要阶乘的逆元）。如果 `N` 很大，每次都用费马小定理计算 `O(N log p)` 会太慢。这时，我们可以利用线性时间 `O(N)` 的方法来预处理所有逆元。

**核心思想：**
利用递推关系，从 `inv[1]` 开始，依次计算 `inv[2], inv[3], ..., inv[N]`。
我们知道 `inv[1] = 1` (因为 `1 * 1 ≡ 1 (mod p)`)。

对于任意 `i > 1`，我们尝试找到 `inv[i]` 和 `inv[p % i]` 之间的关系。
设 `p = k * i + r`，其中 `k = p / i` (整数除法)，`r = p % i`。
显然 `0 <= r < i`。
将这个式子在模 `p` 意义下考虑：
`k * i + r ≡ 0 (mod p)`
将 `r` 移项：
`r ≡ -k * i (mod p)`
两边同时乘以 `inv[i] * inv[r]` (因为 `i` 和 `r` 都小于 `p` 且 `p` 是质数，所以它们都有逆元)：
`r * inv[r] * inv[i] ≡ -k * i * inv[i] * inv[r] (mod p)`
因为 `r * inv[r] ≡ 1 (mod p)` 且 `i * inv[i] ≡ 1 (mod p)`，所以：
`1 * inv[i] ≡ -k * 1 * inv[r] (mod p)`
`inv[i] ≡ -k * inv[r] (mod p)`
将 `k` 和 `r` 替换回 `p / i` 和 `p % i`：
`inv[i] ≡ -(p / i) * inv[p % i] (mod p)`

因为结果要是非负的，我们通常将其调整为：
`inv[i] = (p - p / i) * inv[p % i] % p`

**生活类比：多米诺骨牌**
想象 `inv[1]` 是第一块多米诺骨牌。我们推倒了它，它倒下时会触发第二块 `inv[2]` 倒下，而 `inv[2]` 倒下时又会触发 `inv[3]` 倒下... 每一块多米诺骨牌的倒下（计算）都依赖于它前面已经倒下的某一块 (`inv[p % i]`)。这样，我们只需要推倒第一块，就能让所有的多米诺骨牌在 `O(N)` 的时间里依次倒下。

**优点：**
*   **高效：** 可以在 `O(N)` 的时间复杂度内预处理出 `1` 到 `N` 所有数的逆元。
*   **应用广泛：** 尤其适用于组合数、阶乘逆元等需要大量逆元的计算。

**缺点：**
*   **限制条件：模数 `p` 必须是质数。**
*   需要额外的 `O(N)` 空间来存储逆元数组。

**代码实现思路：**
声明一个数组 `inv[N]`，首先初始化 `inv[1] = 1`，然后通过循环和递推公式填充 `inv[i]`。

```cpp
long long inv[1000005]; // 假设N最大为10^6

void precompute_inverses(int n, int p) {
    inv[1] = 1; // 1的逆元是1
    for (int i = 2; i <= n; ++i) {
        // inv[i] = (p - p / i) * inv[p % i] % p;
        // 注意：p/i是整数除法，p%i是取模运算
        // p - p / i 确保了 (p - p/i) * inv[p%i] 结果为正数，再取模
        inv[i] = (p - (p / i)) * inv[p % i] % p;
    }
}
```

### 4. 两种方法的比较与选择

| 特性       | 费马小定理求逆元                               | 线性时间求逆元                                  |
| :--------- | :--------------------------------------------- | :---------------------------------------------- |
| **适用条件** | 模数 `p` 必须是质数，`a` 与 `p` 互质。         | 模数 `p` 必须是质数。                           |
| **时间复杂度** | 单个逆元：`O(log p)`                          | 预处理 `1` 到 `N` 所有逆元：`O(N)`             |
| **空间复杂度** | `O(1)` (不考虑递归栈)                          | `O(N)` 存储逆元数组                             |
| **使用场景** | 只需要计算少数几个数的逆元，或者 `N` 远小于 `log P` 的情况。 | 需要计算 `1` 到 `N` 之间所有数的逆元，或者 `N` 较大时。 |
| **实现难度** | 简单，基于快速幂。                             | 稍复杂，需要理解递推公式。                      |

**总结：**
*   如果你只需要计算一个或几个数的逆元，并且模数是质数，用**费马小定理**。
*   如果你需要计算 `1` 到 `N` 范围内所有数的逆元（例如计算组合数 `C(n, k)` 涉及到阶乘逆元），并且模数是质数，用**线性时间求逆元**。
*   如果模数 `p` 不是质数，或者 `a` 与 `p` 不互质，那么以上两种方法都不能直接使用。此时需要使用**扩展欧几里得算法**，但这超出了本次课程的重点，我们暂时不深入探讨。

### 常见误区

1.  **忘记模数必须是质数**：费马小定理和线性逆元递推都强依赖于模数是质数这个条件。如果模数不是质数，这两个方法会给出错误的结果或无法使用。
2.  **快速幂中忘记取模**：在 `(res * base)` 或 `(base * base)` 中，中间结果可能会溢出 `long long`，每一步乘法后都必须立即取模。
3.  **线性逆元递推的初始条件**：`inv[1] = 1` 是递推的基石，不能忽略。
4.  **`p - p / i` 的处理**：在 `inv[i] = (p - p / i) * inv[p % i] % p` 中，`p - p / i` 是为了确保 `-(p / i)` 在模 `p` 意义下是正数，避免负数取模的平台差异（C++中负数取模结果可能为负）。

## 典型例题精讲

### 例题一：计算 `(A / B) % P`

**题目描述：**
给定三个正整数 `A`, `B`, `P`。其中 `P` 是一个质数，且 `P` 远大于 `A` 和 `B`。你需要计算 `(A / B)` 在模 `P` 意义下的结果。
例如：`A=10, B=3, P=7`。计算 `(10 / 3) % 7`。

**思路分析：**
1.  题目要求计算 `(A / B) % P`。我们知道在模运算中不能直接进行除法。
2.  由于 `P` 是质数，且 `B` 肯定不是 `P` 的倍数（因为 `P` 远大于 `B`），所以 `B` 在模 `P` 意义下的乘法逆元存在。
3.  根据费马小定理，`B` 的逆元是 `B^(P-2) % P`。
4.  我们可以用快速幂计算 `B^(P-2) % P` 得到 `invB`。
5.  最终结果就是 `(A * invB) % P`。

**具体步骤：**
1.  定义一个快速幂函数 `power(base, exp, mod)`。
2.  在主函数中，调用 `power(B, P - 2, P)` 得到 `B` 的逆元 `invB`。
3.  计算 `(A * invB) % P` 并输出。

**代码实现：**

```cpp
#include <bits/stdc++.h>

long long power(long long base, long long exp, long long mod) {
    long long res = 1;
    base %= mod; // 确保base在模范围内
    while (exp > 0) {
        if (exp % 2 == 1) { // 如果指数是奇数
            res = (res * base) % mod;
        }
        base = (base * base) % mod; // 基数平方
        exp /= 2; // 指数减半
    }
    return res;
}

int main() {
    long long a, b, p;
    std::cin >> a >> b >> p;

    // 1. 计算b在模p意义下的逆元
    // 根据费马小定理，b的逆元是 b^(p-2) % p
    long long invB = power(b, p - 2, p);

    // 2. 将除法转化为乘法
    long long ans = (a * invB) % p;

    std::cout << ans << std::endl;

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：主要取决于快速幂函数，为 `O(log P)`。
*   空间复杂度：`O(1)`。

### 例题二：计算组合数 `C(N, K) % P`

**题目描述：**
给定三个正整数 `N`, `K`, `P`。其中 `P` 是一个质数。你需要计算组合数 `C(N, K)` 在模 `P` 意义下的结果。
`C(N, K) = N! / (K! * (N-K)!)`
例如：`N=5, K=2, P=7`。计算 `C(5, 2) % 7`。 (`C(5,2) = 10`)

**思路分析：**
1.  组合数 `C(N, K)` 的定义是 `N! / (K! * (N-K)!)`。
2.  在模 `P` 意义下，我们需要计算 `(N! * (K!)^(-1) * ((N-K)!)^(-1)) % P`。
3.  这里需要 `K!` 和 `(N-K)!` 的逆元。由于 `K` 和 `N-K` 都在 `1` 到 `N` 的范围内，我们需要计算一系列数的逆元。
4.  由于 `P` 是质数，我们可以使用线性时间求逆元的方法，预处理出 `1` 到 `N` 之间所有数的逆元。
5.  更进一步，我们可以预处理出 `1` 到 `N` 之间所有数的阶乘 `fact[i]`，以及所有阶乘的逆元 `invFact[i]`。
    *   `fact[i] = i!`
    *   `invFact[i] = (i!)^(-1) % P`
6.  有了 `fact` 数组和 `invFact` 数组，`C(N, K) % P` 就可以表示为 `(fact[N] * invFact[K] % P * invFact[N-K] % P) % P`。

**具体步骤：**
1.  预处理阶乘数组 `fact[]`：`fact[i] = (fact[i-1] * i) % P`。
2.  预处理逆元数组 `inv[]`：使用线性方法 `inv[i] = (P - P / i) * inv[P % i] % P`。
3.  预处理阶乘逆元数组 `invFact[]`：`invFact[i] = (invFact[i-1] * inv[i]) % P`。
    或者，可以先用费马小定理计算 `invFact[N] = power(fact[N], P-2, P)`，然后倒推 `invFact[i-1] = (invFact[i] * i) % P`。这里我们用正向递推。
4.  计算 `(fact[N] * invFact[K] % P * invFact[N-K] % P) % P`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

const int MAXN = 1000005; // N的最大值
long long fact[MAXN];     // 存储阶乘
long long inv[MAXN];      // 存储逆元
long long invFact[MAXN];  // 存储阶乘的逆元

// 预处理阶乘、逆元、阶乘逆元
void precompute(int n, int p) {
    // 1. 预处理阶乘
    fact[0] = 1;
    for (int i = 1; i <= n; ++i) {
        fact[i] = (fact[i - 1] * i) % p;
    }

    // 2. 预处理1到n的逆元
    inv[1] = 1;
    for (int i = 2; i <= n; ++i) {
        inv[i] = (p - (p / i)) * inv[p % i] % p;
    }

    // 3. 预处理阶乘的逆元
    // invFact[i] = (i!)^(-1) % p
    // invFact[0] = 0!^(-1) = 1^(-1) = 1
    invFact[0] = 1; 
    for (int i = 1; i <= n; ++i) {
        invFact[i] = (invFact[i - 1] * inv[i]) % p;
    }
}

// 计算 C(n, k) % p
long long combinations(int n, int k, int p) {
    if (k < 0 || k > n) {
        return 0;
    }
    // C(n, k) = n! * (k!)^(-1) * ((n-k)!)^(-1) % p
    long long num = fact[n];
    long long den = (invFact[k] * invFact[n - k]) % p;
    return (num * den) % p;
}

int main() {
    int n, k, p;
    std::cin >> n >> k >> p;

    precompute(n, p); // 预处理到n的最大值

    long long ans = combinations(n, k, p);
    std::cout << ans << std::endl;

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：预处理阶段为 `O(N)`。每次查询 `C(N, K)` 只需要 `O(1)`。
*   空间复杂度：`O(N)` 用于存储 `fact`, `inv`, `invFact` 数组。

## 代码实现模板

### 1. 快速幂模板 (用于费马小定理求逆元)

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// 快速幂函数：计算 (base^exp) % mod
// 参数：
//   base: 基数
//   exp: 指数
//   mod: 模数
// 返回值：(base^exp) % mod 的结果
long long power(long long base, long long exp, long long mod) {
    long long res = 1;       // 结果初始化为1
    base %= mod;             // 基数先对模数取余，防止中间计算溢出
    while (exp > 0) {        // 当指数大于0时循环
        if (exp % 2 == 1) {  // 如果当前指数是奇数
            res = (res * base) % mod; // 将当前基数乘入结果
        }
        base = (base * base) % mod; // 基数平方，指数减半
        exp /= 2;
    }
    return res;              // 返回最终结果
}

// 费马小定理求乘法逆元：计算 a^(-1) % p
// 前提：p 必须是质数，且 a 不是 p 的倍数 (即 gcd(a, p) = 1)
// 参数：
//   a: 要计算逆元的数
//   p: 模数 (质数)
// 返回值：a 在模 p 意义下的乘法逆元
long long inverse_fermat(long long a, long long p) {
    // 根据费马小定理：a^(p-1) ≡ 1 (mod p)
    // 所以 a * a^(p-2) ≡ 1 (mod p)
    // 故 a 的逆元为 a^(p-2) % p
    return power(a, p - 2, p);
}

/*
// 示例用法
int main() {
    long long a = 3, p = 7; // 3在模7下的逆元
    long long inv_a = inverse_fermat(a, p);
```

### 2. 线性时间求逆元模板

```cpp
#include <bits/stdc++.h> // 包含常用头文件

const int MAX_N = 1000005; // 根据题目需求设置最大N值
long long inv[MAX_N];      // 存储 1 到 MAX_N 中每个数的逆元

// 线性时间预处理 1 到 N 中所有数的逆元
// 前提：p 必须是质数
// 参数：
//   n: 需要预处理逆元的上界 (1 到 n)
//   p: 模数 (质数)
void precompute_linear_inverses(int n, int p) {
    inv[1] = 1; // 1 的逆元在任何模 p 意义下都是 1
    for (int i = 2; i <= n; ++i) {
        // 递推公式：inv[i] = (p - p / i) * inv[p % i] % p
        // p / i 是整数除法，得到商 k
        // p % i 是取模运算，得到余数 r
        // 公式推导自：k*i + r = p ≡ 0 (mod p)
        // => r ≡ -k*i (mod p)
        // => r * inv[r] * inv[i] ≡ -k * i * inv[i] * inv[r] (mod p)
        // => inv[i] ≡ -k * inv[r] (mod p)
        // => inv[i] = (p - k) * inv[r] % p
        inv[i] = (p - (p / i)) * inv[p % i] % p;
    }
}

/*
// 示例用法
int main() {
    int n = 10, p = 7; // 预处理1到10在模7下的逆元
    precompute_linear_inverses(n, p);

    std::cout << "Linear inverses modulo " << p << ":" << std::endl;
    for (int i = 1; i <= n; ++i) {
        std::cout << "Inverse of " << i << " is " << inv[i] << std::endl;
        // 验证：(i * inv[i]) % p 应该等于 1
    }
    return 0;
}
*/
```

### 提问与思考

1.  **思考题1 (概念理解)**：
    *   小明说：“在模10的世界里，5的乘法逆元是2，因为 `5 * 2 = 10`，而 `10 % 10 = 0`。” 小明的说法对吗？为什么？如果不对，5在模10下有逆元吗？
    *   **预期回答：** 不对。乘法逆元的定义是 `a * x % p = 1`。`5 * 2 % 10 = 0` 不等于 `1`。5在模10下没有逆元，因为 `gcd(5, 10) = 5 != 1`。

2.  **思考题2 (方法选择)**：
    *   如果你需要计算 `123456789 ^ (-1) % 998244353` (其中 `998244353` 是一个质数)，你会选择哪种方法？为什么？

3.  **思考题3 (线性逆元推导)**：

    *   **预期回答：** (引导学生回顾推导过程) 从 `k*i + r ≡ 0 (mod p)` 开始，移项 `r ≡ -k*i (mod p)`，然后两边同乘 `inv[i] * inv[r]`，利用 `x * inv[x] ≡ 1 (mod p)` 化简。
