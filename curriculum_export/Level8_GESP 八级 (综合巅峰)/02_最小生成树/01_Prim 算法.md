## 知识点讲解

### 1. 什么是图？
在计算机科学中，图是一种由顶点（或节点）和边组成的抽象数据结构。
*   **顶点 (Vertex/Node)**：表示问题中的实体，比如我们例子中的居民区。
*   **边 (Edge)**：表示顶点之间的连接关系，比如居民区之间的电缆路径。
*   **权值 (Weight)**：边上附带的数值，表示连接的“成本”或“距离”，比如铺设电缆的费用。
*   **连通图 (Connected Graph)**：图中任意两个顶点之间都有路径相连。我们今天主要讨论连通图。

### 2. 什么是最小生成树 (Minimum Spanning Tree, MST)？
对于一个带权无向连通图，它的**生成树**是包含图中所有顶点，且只包含足以构成一棵树的边（即没有环）的子图。
**最小生成树 (MST)** 就是在所有可能的生成树中，边的权值之和最小的那棵生成树。

**MST 的特点：**
*   它是一棵树，所以有 `n` 个顶点的 MST 必然有 `n-1` 条边。
*   它连接了图中所有的顶点。
*   所有边的权值之和最小。

### 3. Prim 算法的基本思想
Prim 算法是一种**贪心算法**，它的思想有点像“圈地运动”或者“种树”。它从一个任意的起始顶点开始，逐步“生长”出一棵最小生成树。

**基本步骤：**
1.  **初始化**：
    *   选择图中的任意一个顶点作为起始点，将其加入到已构建的生成树集合 `T` 中。
    *   维护一个 `dist` 数组，`dist[i]` 表示顶点 `i` 到已构建的生成树 `T` 中任何顶点的最小距离。对于未加入 `T` 的顶点，`dist[i]` 初始化为到起始点的距离；对于已加入 `T` 的顶点，`dist[i]` 可以设为 0 或一个特殊值。
    *   维护一个 `visited` 数组，标记顶点是否已加入生成树。

2.  **循环生长**：重复 `n-1` 次（因为 MST 有 `n-1` 条边，每次找到一条边）：
    *   在所有**未加入**生成树的顶点中，找到一个顶点 `u`，使得 `dist[u]` 最小。这个 `dist[u]` 对应的边就是连接 `u` 和已生成树的权值最小的边。
    *   将顶点 `u` 加入到已构建的生成树集合 `T` 中，同时将 `u` 标记为已访问。
    *   将 `dist[u]` 的值累加到最小生成树的总权值中。
    *   **更新**：遍历所有与 `u` 相邻的顶点 `v`。如果 `v` 还没有被访问，并且 `u` 到 `v` 的边的权值小于当前的 `dist[v]`，那么就更新 `dist[v]` 为 `u` 到 `v` 的权值。这表示我们找到了从已生成树到 `v` 的更短路径。

3.  **结束**：当所有顶点都被加入到生成树中，算法结束。

**Prim 算法的贪心策略**：每一步都选择连接已生成树和未生成树的权值最小的边。这个策略之所以正确，是因为它遵循了“切分定理”：对于图中的任意一个切分（将顶点分为两个不相交的集合 `S` 和 `V-S`），连接 `S` 和 `V-S` 的权值最小的边，必然属于图的某棵最小生成树。Prim 算法正是利用了这一点。

### 4. Prim 算法的实现方式及时间复杂度

#### 方式一：朴素 Prim 算法 (邻接矩阵)
*   **适用场景**：边数较多，接近完全图，或者顶点数 `N` 较小（例如 `N <= 2000`）。
*   **数据结构**：
    *   邻接矩阵 `g[i][j]` 存储顶点 `i` 到顶点 `j` 的边的权值。
    *   `dist[i]`：顶点 `i` 到当前 MST 的最短距离。
    *   `visited[i]`：标记顶点 `i` 是否已加入 MST。
*   **时间复杂度**：
    *   外层循环 `N` 次（选择 `N` 个顶点）。
    *   内层循环 `N` 次（查找最小 `dist` 的顶点）。
    *   更新 `dist` 数组 `N` 次。
    *   总时间复杂度为 **O(N^2)**。

#### 方式二：优化 Prim 算法 (邻接表 + 优先队列)
*   **适用场景**：稀疏图（边数 `E` 远小于 `N^2`），或者顶点数 `N` 较大。
*   **数据结构**：
    *   邻接表 `adj[u]` 存储与 `u` 相邻的边（`{v, weight}`）。
    *   `dist[i]`：顶点 `i` 到当前 MST 的最短距离。
    *   `visited[i]`：标记顶点 `i` 是否已加入 MST。
    *   **优先队列 (priority_queue)**：存储 `{weight, vertex}` 对，每次取出权值最小的边所连接的顶点。
*   **时间复杂度**：
    *   外层循环 `N` 次（选择 `N` 个顶点）。
    *   每次从优先队列中取出元素：`log E` (最坏情况下，所有边都可能入队)。
    *   每次更新邻接点：对于每个顶点，其所有邻边都会被考虑一次，并可能插入优先队列。总共 `E` 条边，每次插入 `log E`。
    *   总时间复杂度为 **O(E log E)** 或 **O(E log V)** (因为优先队列中最多有 `V` 个不同顶点，或者 `E` 个边)。当 `E` 接近 `V^2` 时，`E log V` 接近 `V^2 log V`，不如朴素 Prim。但对于稀疏图，`E` 远小于 `V^2`，则优化 Prim 效率更高。

**小结：**
*   如果图是稠密的（边很多），朴素 Prim (O(N^2)) 更优。
*   如果图是稀疏的（边很少），优化 Prim (O(E log V)) 更优。

## 代码示例

我们将分别展示朴素 Prim 算法和优化 Prim 算法的 C++ 实现。

### 1. 朴素 Prim 算法 (邻接矩阵实现)

```cpp
#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f; // 表示无穷大，用于初始化距离

int n, m; // n: 顶点数, m: 边数
int g[505][505]; // 邻接矩阵存储图，g[i][j] 表示 i 到 j 的边的权值
int dist[505]; // dist[i] 表示顶点 i 到当前已生成树的最小距离
bool visited[505]; // visited[i] 为 true 表示顶点 i 已经加入到生成树中

int prim() {
    // 初始化所有顶点到生成树的距离为无穷大
    // 假设从 1 号顶点开始构建 MST
    for (int i = 1; i <= n; ++i) {
        dist[i] = INF;
        visited[i] = false;
    }

    dist[1] = 0; // 从 1 号顶点开始，它到自身的距离为 0
    int total_weight = 0; // 最小生成树的总权值

    for (int i = 0; i < n; ++i) { // 循环 n 次，每次选择一个顶点加入生成树
        int u = -1; // 找到当前距离生成树最近的未访问顶点
        for (int j = 1; j <= n; ++j) {
            if (!visited[j] && (u == -1 || dist[j] < dist[u])) {
                u = j;
            }
        }

        // 如果没有找到这样的顶点，说明图不连通
        if (u == -1 || dist[u] == INF) {
            return -1; // 返回 -1 表示图不连通，无法构建生成树
        }

        visited[u] = true; // 将顶点 u 加入到生成树
        total_weight += dist[u]; // 累加这条边的权值

        // 更新所有与 u 相邻的未访问顶点到生成树的距离
        for (int v = 1; v <= n; ++v) {
            if (!visited[v] && g[u][v] != INF) { // 如果 v 未访问且 u 到 v 有边
                dist[v] = min(dist[v], g[u][v]); // 更新 v 到生成树的最小距离
            }
        }
    }
    return total_weight; // 返回最小生成树的总权值
}

int main() {
    // 读入顶点数 n 和边数 m
    cin >> n >> m;

    // 初始化邻接矩阵，对角线为 0，其他为 INF
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            if (i == j) {
                g[i][j] = 0;
            } else {
                g[i][j] = INF;
            }
        }
    }

    // 读入 m 条边
    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        // 因为是无向图，所以要设置双向边
        g[u][v] = min(g[u][v], w); // 处理重边，只保留权值最小的
        g[v][u] = min(g[v][u], w);
    }

    int result = prim();
    if (result == -1) {
        cout << "图不连通，无法构建最小生成树" << endl;
    } else {
        cout << "最小生成树的总权值为: " << result << endl;
    }

    return 0;
}
```

### 2. 优化 Prim 算法 (邻接表 + 优先队列实现)

```cpp
#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f; // 表示无穷大

// 定义边的结构体，方便存储邻接表
struct Edge {
    int to;     // 边的目标顶点
    int weight; // 边的权值

    // 构造函数
    Edge(int _to, int _weight) : to(_to), weight(_weight) {}

    // 重载小于运算符，用于优先队列的排序（默认大顶堆，我们需要小顶堆）
    // priority_queue 默认是最大元素优先，所以我们希望 weight 越小优先级越高
    // 因此，需要定义 operator< 返回 true 表示 'this' 的优先级低于 'other'
    // 或者，直接存储 pair<int, int>，priority_queue 会默认按第一个元素排序
    // 这里我们用 pair<int, int> 来存储 {weight, to}，这样默认就是小顶堆了
};

int n, m; // n: 顶点数, m: 边数
vector<pair<int, int>> adj[100005]; // 邻接表，adj[u] 存储 {weight, v}
int dist[100005]; // dist[i] 表示顶点 i 到当前已生成树的最小距离
bool visited[100005]; // visited[i] 为 true 表示顶点 i 已经加入到生成树中

long long prim_optimized() {
    // 初始化所有顶点到生成树的距离为无穷大
    for (int i = 1; i <= n; ++i) {
        dist[i] = INF;
        visited[i] = false;
    }

    // 优先队列，存储 {当前边权值, 目标顶点}，会自动按权值从小到大排序
    // priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
    // 简写：
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;

    dist[1] = 0; // 从 1 号顶点开始，它到自身的距离为 0
    pq.push({0, 1}); // 将起始顶点加入优先队列 {权值, 顶点}

    long long total_weight = 0; // 最小生成树的总权值
    int edge_count = 0; // 统计已加入生成树的边数

    while (!pq.empty() && edge_count < n) { // 当优先队列不为空且未选择 n-1 条边时
        pair<int, int> current = pq.top();
        pq.pop();

        int d = current.first; // 当前边权值
        int u = current.second; // 当前顶点

        if (visited[u]) { // 如果 u 已经访问过，则跳过（因为可能通过不同路径多次入队）
            continue;
        }

        visited[u] = true; // 将顶点 u 加入到生成树
        total_weight += d; // 累加这条边的权值
        edge_count++; // 增加已加入生成树的边数

        // 遍历所有与 u 相邻的顶点
        for (auto& edge : adj[u]) {
            int v = edge.first; // 邻接点
            int w = edge.second; // u 到 v 的权值

            if (!visited[v] && w < dist[v]) { // 如果 v 未访问且通过 u 到 v 的路径更短
                dist[v] = w; // 更新 v 到生成树的最小距离
                pq.push({dist[v], v}); // 将更新后的 {距离, v} 加入优先队列
            }
        }
    }

    // 如果最终生成的边数不足 n-1 条（对于 n 个顶点），说明图不连通
    if (edge_count != n) { // 注意这里是顶点数 n，不是边数 n-1，因为我们是看有多少个顶点被加入
        return -1; // 返回 -1 表示图不连通
    }
    return total_weight; // 返回最小生成树的总权值
}

int main() {
    // 读入顶点数 n 和边数 m
    cin >> n >> m;

    // 读入 m 条边
    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        // 因为是无向图，所以要设置双向边
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    long long result = prim_optimized();
    if (result == -1) {
        cout << "图不连通，无法构建最小生成树" << endl;
    } else {
        cout << "最小生成树的总权值为: " << result << endl;
    }

    return 0;
}
```

#### 进阶练习：优化 Prim 算法
**题目描述**：
给定一个无向带权图，包含 `N` 个顶点和 `M` 条边。请你使用优化 Prim 算法（邻接表 + 优先队列）找出该图的最小生成树的总权值。如果图不连通，输出 -1。

**输入格式**：
第一行包含两个整数 `N` 和 `M` (1 <= N <= 100000, 1 <= M <= 500000)。
接下来 `M` 行，每行包含三个整数 `u, v, w`，表示顶点 `u` 和 `v` 之间存在一条权值为 `w` 的无向边 (1 <= u, v <= N, 1 <= w <= 10^9)。

**输出格式**：
输出一个整数，表示最小生成树的总权值。如果图不连通，输出 -1。

**样例输入**：
```
5 6
1 2 1
1 3 2
2 3 3
2 4 4
3 5 5
4 5 6
```

**样例输出**：
```
12
```

---

### 练习题目答案与解析

**选择题/填空题答案**

1.  **C** (解析：最小生成树中边的数量等于顶点数 `N-1`，而不是 `N`。A、B、D都是正确的描述。)
2.  **C** (解析：这是 Prim 算法的核心贪心策略，每次选择能将未访问顶点连接到已生成树的最小权值边。)
3.  **B** (解析：朴素 Prim 算法的两个嵌套循环导致其时间复杂度为 O(N^2)。)
4.  **C** (解析：使用邻接表存储图，优先队列优化查找最小边，时间复杂度为 O(E log E) 或 O(E log V)。)
5.  **7** (解析：
    *   从 1 开始，`dist = [0, 3, 1, INF]`。
    *   选择 (1,3,1)，3 加入 MST。`total_weight = 1`。更新 `dist`：`dist[2]` 还是 3 (1-2)，`dist[4]` 还是 INF。
    *   此时，2 到 MST 的距离是 3 (通过 1)，4 到 MST 的距离是 2 (通过 3)。
    *   选择 (3,2,2) (因为 3-2 比 1-2 权值小)。2 加入 MST。`total_weight = 1 + 2 = 3`。更新 `dist`：`dist[4]` 还是 4 (通过 2)。
    *   选择 (2,4,4)。4 加入 MST。`total_weight = 3 + 4 = 7`。
    *   所有顶点都已连接。总权值为 7。)

## 教学评价与作业

### 教学评价

*   **课堂参与度**：观察学生在趣味引入、知识点讲解和互动环节的积极性，是否主动提问或回答问题。
*   **小组任务表现**：评估小组在手动模拟 Prim 算法时的协作情况、逻辑严谨性以及最终结果的正确性。
*   **编程题理解**：通过学生完成编程题的代码质量、思路清晰度以及是否能正确处理边界情况（如图不连通）来评估对算法的掌握程度。

### 课后作业

1.  **复习与巩固**：
    *   回顾 Prim 算法的两种实现方式（朴素版和优化版）的代码，理解它们各自的优缺点和适用场景。
    *   尝试用自己的话向家人或朋友解释什么是最小生成树和 Prim 算法。

2.  **编程实践**：
    *   完成课堂上未能完成的编程题（基础版和进阶版），确保代码能够通过测试用例。
    *   尝试修改你的 Prim 算法代码，使其不仅能输出最小生成树的总权值，还能输出构成这棵树的具体是哪些边。

3.  **拓展思考**：
    *   思考 Prim 算法和 Kruskal 算法在解决同一问题时，哪一个在实际应用中更常见？为什么？
    *   查找资料，了解 Prim 算法在除了电力网络之外，还有哪些实际应用场景？比如网络路由、交通规划等。
