## 深度知识点讲解

### 1. 双链表（Doubly Linked List）

#### 1.1 概念本质：双向通行的高速公路

单链表就像一条单行道，每个节点只能指向下一个节点，所以我们只能往前走。如果想往回走，对不起，没路！这在某些情况下非常不方便，比如：
*   我们想快速找到某个节点的前一个节点。
*   我们想删除一个节点，但是只知道这个节点本身，不知道它的前一个节点。

为了解决这个问题，双链表应运而生！它就像一条双向通行的高速公路，每个节点不仅知道它“后面”是谁，还知道它“前面”是谁。

#### 1.2 结构特点：多一个“后视镜”

在单链表中，我们的节点结构是这样的：
```cpp
struct Node {
    int v;         // 数据域，存放节点的值
    Node *n;       // 指针域，指向下一个节点 (next)
};
```
而在双链表中，我们给每个节点多加了一个指针，用来指向它的前一个节点。
```cpp
struct Node {
    int v;         // 数据域，存放节点的值
    Node *p;       // 指针域，指向前一个节点 (previous)
    Node *n;       // 指针域，指向下一个节点 (next)
};
```
**生活类比**：
*   想象一下，你站在一排手拉手的小朋友中间。单链表是每个小朋友只拉着他前面小朋友的手，所以只能往前走。
*   双链表是每个小朋友既拉着前面小朋友的手，也拉着后面小朋友的手。这样，你想往前走可以，想往后走也可以！

**关键点剖析**：
*   **头节点和尾节点**：双链表的头节点的 `p` 指针指向 `NULL`，尾节点的 `n` 指针指向 `NULL`。这标志着链表的起点和终点。
*   **指针操作的复杂性**：由于每个节点有两个指针需要维护，所以在插入和删除操作时，我们需要修改的指针数量会比单链表多，需要格外小心。

#### 1.3 核心操作：步步为营，小心指针！

**a) 遍历（Traversal）**
从头节点（或尾节点）开始，沿着 `n` 指针（或 `p` 指针）依次访问每个节点，直到遇到 `NULL`。
*   **向前遍历**：`head -> n -> n -> ... -> NULL`
*   **向后遍历**：`tail -> p -> p -> ... -> NULL`
这比单链表多了一个向后遍历的能力，非常方便！

**b) 插入（Insertion）**
这是双链表最需要细心操作的地方。我们以在两个节点 `a` 和 `b` 之间插入新节点 `s` 为例。
假设我们想在节点 `a` 后面插入新节点 `s`。
原来的关系：`a <-> b` (即 `a->n = b`, `b->p = a`)
目标关系：`a <-> s <-> b`
步骤：
1.  新节点 `s` 的 `n` 指针指向 `a` 的下一个节点 (`b`)：`s->n = a->n;` (现在 `s->n` 指向 `b`)
2.  新节点 `s` 的 `p` 指针指向 `a`：`s->p = a;`
3.  `a` 的 `n` 指针指向 `s`：`a->n = s;`
4.  如果 `b` 存在（即 `a->n` 不是 `NULL`），那么 `b` 的 `p` 指针指向 `s`：`b->p = s;` (或 `a->n->p = s;`)

**思考题**：为什么我们需要检查 `b` 是否存在？如果 `a` 是尾节点，`a->n` 就是 `NULL`，直接 `a->n->p = s` 会发生什么？（会报错！）

**c) 删除（Deletion）**
删除一个节点 `s`（假设它不是头尾节点）。
原来的关系：`a <-> s <-> b` (即 `a->n = s`, `s->p = a`, `s->n = b`, `b->p = s`)
目标关系：`a <-> b`
步骤：
1.  `a` 的 `n` 指针指向 `s` 的下一个节点 (`b`)：`a->n = s->n;`
2.  `b` 的 `p` 指针指向 `s` 的前一个节点 (`a`)：`b->p = s->p;`
3.  最后，释放节点 `s` 的内存：`delete s;`

**常见误区**：
*   **指针遗漏**：在插入或删除时，忘记更新某个节点的 `p` 或 `n` 指针，导致链表断裂或出现错误连接。
*   **空指针判断**：在操作头节点、尾节点或空链表时，没有正确处理 `NULL` 指针的情况。

### 2. 循环链表（Circular Linked List）

#### 2.1 概念本质：永不停止的“击鼓传花”

单链表和双链表都有明确的起点（头节点）和终点（尾节点）。但有些场景下，我们希望数据结构是“循环”的，没有明确的开始和结束。比如：
*   操作系统中进程调度，轮流执行。
*   播放器中的循环播放列表。
*   “击鼓传花”游戏。

循环链表就是为了实现这种“循环”而设计的。它把链表的尾节点连接到头节点，形成一个闭环。

#### 2.2 结构特点：首尾相接，生生不息

循环链表可以是单向的（Singly Circular Linked List），也可以是双向的（Doubly Circular Linked List）。我们主要关注单向循环链表。

**单向循环链表的结构**：
和单链表一样，每个节点只有一个 `n` 指针。
```cpp
struct Node {
    int v;         // 数据域
    Node *n;       // 指针域，指向下一个节点
};
```
**不同之处在于**：
*   最后一个节点的 `n` 指针不再指向 `NULL`，而是指向链表的第一个节点（头节点）。
*   因此，循环链表中永远不会有 `NULL` 指针来表示链表的结束。

**生活类比**：
*   一队手拉手围成一个圈的小朋友。
*   一条没有尽头的环形跑道。

**关键点剖析**：
*   **遍历终止条件**：由于没有 `NULL`，遍历时需要特别注意，通常是从头节点开始，当再次回到头节点时停止。
*   **头指针或尾指针**：通常我们只维护一个指向链表任意一个节点的指针，或者为了方便操作，维护一个指向**尾节点**的指针 `tail`。为什么是尾节点？因为从尾节点出发，`tail->n` 就是头节点，`tail` 本身就是尾节点，这样我们就能同时访问到头和尾，非常方便！

#### 2.3 核心操作：循环往复，其乐无穷

**a) 遍历（Traversal）**
从一个起始节点 `s` 开始，沿着 `n` 指针一直走，直到再次回到 `s`。
```cpp
Node *current = head;
if (current != NULL) { // 检查链表是否为空
    do {
        // 访问 current->v
        current = current->n;
    } while (current != head); // 循环直到再次回到 head
}
```

**b) 插入（Insertion）**
以在链表头部插入新节点 `s` 为例：
假设 `tail` 指向链表的尾节点，`head` 是 `tail->n`。
1.  新节点 `s` 的 `n` 指针指向原来的头节点：`s->n = tail->n;` (即 `s->n = head;`)
2.  `tail` 的 `n` 指针指向新节点 `s`：`tail->n = s;`
如果链表为空，新节点 `s` 的 `n` 指针指向自己：`s->n = s;`，并且 `tail = s;`。

**c) 删除（Deletion）**
以删除链表头部节点为例：
假设 `tail` 指向尾节点，`head` 是 `tail->n`。
1.  如果链表只有一个节点（`head == tail`），直接 `delete head; tail = NULL;`。
2.  否则，`tail` 的 `n` 指针指向原来的头节点的下一个节点：`tail->n = head->n;`
3.  释放原来的头节点：`delete head;`
4.  更新新的头节点：现在 `tail->n` 就是新的头节点了。

**常见误区**：
*   **无限循环**：遍历时，如果终止条件判断不当，可能会陷入无限循环。
*   **空链表处理**：忘记处理链表为空或只有一个节点时的特殊情况。
*   **只维护头指针**：如果只维护头指针，插入尾部或删除尾部会很麻烦，因为需要遍历整个链表才能找到尾节点。维护尾指针会更高效。

### 3. 总结与比较

| 特性       | 单链表                 | 双链表                      | 循环链表（单向）            |
| :--------- | :--------------------- | :-------------------------- | :-------------------------- |
| 节点结构   | `v`, `next`            | `v`, `prev`, `next`         | `v`, `next`                 |
| 遍历方向   | 单向                   | 双向                        | 单向（但可循环）            |
| 尾节点     | `next` 指向 `NULL`     | `next` 指向 `NULL`          | `next` 指向头节点           |
| 头节点     | `prev` 指向 `NULL`     | `prev` 指向 `NULL`          | 无特殊 `prev` 指针          |
| 插入/删除  | 需前驱节点，O(N) 查找  | 知道节点本身即可，O(1)      | 需前驱节点，O(N) 查找       |
| 空间复杂度 | O(N)                   | O(2N) (多一个指针)          | O(N)                        |
| 优点       | 简单，节省空间         | 双向遍历，高效删除/插入     | 连续循环，方便轮询          |
| 缺点       | 查找前驱慢，删除慢     | 空间开销大，操作复杂易错    | 遍历终止条件复杂，查找前驱慢 |

**何时使用？**
*   **单链表**：数据量大，空间宝贵，且多为顺序访问或只在头部尾部操作。
*   **双链表**：需要频繁进行双向遍历，或在链表中间进行插入删除操作，且对空间不敏感。
*   **循环链表**：需要处理循环队列、任务调度、资源轮询等场景。

## 典型例题精讲

### 例题一：双链表中的插入与删除

**题目描述**：
实现一个双链表，支持以下操作：
1.  `insertAfter(Node* a, int val)`：在节点 `a` 之后插入一个值为 `val` 的新节点。
2.  `deleteNode(Node* s)`：删除指定的节点 `s`。
假设链表头节点为 `head`，尾节点为 `tail`。如果 `a` 或 `s` 是 `NULL`，则不进行操作。如果删除的是头尾节点，需要更新 `head` 或 `tail`。

**思路分析**：
这道题考察的是双链表核心的指针操作。关键在于理清每个指针应该指向哪里，并且考虑边界情况（插入到尾部、删除头节点、删除尾节点、链表只有一个节点等）。

**1. `insertAfter(Node* a, int val)`**
*   首先创建新节点 `s`，并赋值 `s->v = val;`。
*   如果 `a` 是 `NULL`，说明没有指定插入位置，直接返回。
*   **情况一：在链表尾部插入** (即 `a` 是当前尾节点，`a->n == NULL`)
    *   `s->p = a;`
    *   `s->n = NULL;`
    *   `a->n = s;`
    *   更新 `tail = s;`
*   **情况二：在链表中间插入** (即 `a` 不是尾节点)
    *   `s->p = a;`
    *   `s->n = a->n;`
    *   `a->n->p = s;`
    *   `a->n = s;`
*   注意：为了代码简洁，我们可以将情况一和情况二合并处理，只需在更新 `tail` 时做判断。

**2. `deleteNode(Node* s)`**
*   如果 `s` 是 `NULL`，说明没有指定删除节点，直接返回。
*   **情况一：删除的是头节点** (`s == head`)
    *   `head = s->n;`
    *   如果 `head` 不为 `NULL` (即链表不只一个节点)，则 `head->p = NULL;`
    *   如果链表只有一个节点（`s` 既是 `head` 又是 `tail`），则 `head = NULL; tail = NULL;`
*   **情况二：删除的是尾节点** (`s == tail`)
    *   `tail = s->p;`
    *   `tail->n = NULL;`
*   **情况三：删除的是中间节点**
    *   `s->p->n = s->n;`
    *   `s->n->p = s->p;`
*   最后，释放节点 `s` 的内存：`delete s;`

**C++ 代码**：

```cpp
#include <bits/stdc++.h>

// 双链表节点结构
struct Node {
    int v;
    Node *p; // previous
    Node *n; // next

    Node(int val) : v(val), p(NULL), n(NULL) {} // 构造函数
};

Node *head = NULL; // 链表头指针
Node *tail = NULL; // 链表尾指针

// 在指定节点 a 之后插入一个值为 val 的新节点
void insertAfter(Node *a, int val) {
    if (a == NULL && head != NULL) { // 如果a为NULL但链表不空，则无法插入
        std::cout << "Error: Cannot insert after a NULL node in a non-empty list." << std::endl;
        return;
    }

    Node *s = new Node(val); // 创建新节点

    if (head == NULL) { // 链表为空，新节点既是头也是尾
        head = s;
        tail = s;
    } else if (a == NULL) { // 链表不为空，但a为NULL，说明要插入到头部 (这里我们定义为insertAfter，所以a不能为NULL)
                            // 实际上，如果允许a为NULL表示插入到链表开头，需要特殊处理
                            // 但根据题目，a必须是现有节点。此处可按需求调整逻辑。
                            // 假设a为NULL时表示插入到头部，这里为了符合insertAfter语义，暂不处理这种情况。
        std::cout << "Error: 'insertAfter' requires a valid node 'a'. To insert at head, use a specific function." << std::endl;
        delete s; // 释放内存
        return;
    }
    else { // 在节点 a 之后插入
        s->p = a;
        s->n = a->n;

        if (a->n != NULL) { // 如果 a 不是尾节点
            a->n->p = s;
        } else { // 如果 a 是尾节点，s 成为新的尾节点
            tail = s;
        }
        a->n = s;
    }
}

// 删除指定节点 s
void deleteNode(Node *s) {
    if (s == NULL) {
        std::cout << "Error: Cannot delete a NULL node." << std::endl;
        return;
    }

    if (s->p != NULL) { // s 不是头节点
        s->p->n = s->n;
    } else { // s 是头节点
        head = s->n;
    }

    if (s->n != NULL) { // s 不是尾节点
        s->n->p = s->p;
    } else { // s 是尾节点
        tail = s->p;
    }

    delete s; // 释放内存
}

// 辅助函数：打印链表
void printList() {
    Node *current = head;
    std::cout << "List (forward): ";
    while (current != NULL) {
        std::cout << current->v << " ";
        current = current->n;
    }
    std::cout << std::endl;

    current = tail;
    std::cout << "List (backward): ";
    while (current != NULL) {
        std::cout << current->v << " ";
        current = current->p;
    }
    std::cout << std::endl;
}

int main() {
    // 初始链表
    insertAfter(NULL, 10); // 插入第一个节点
    insertAfter(head, 20); // 10 -> 20
    insertAfter(head->n, 30); // 10 -> 20 -> 30
    insertAfter(head, 15); // 10 -> 15 -> 20 -> 30

    printList(); // 期望: List (forward): 10 15 20 30, List (backward): 30 20 15 10

    // 删除节点 15
    Node *nodeToDelete = head->n; // 节点 15
    deleteNode(nodeToDelete); // 10 -> 20 -> 30
    std::cout << "After deleting 15:" << std::endl;
    printList(); // 期望: List (forward): 10 20 30, List (backward): 30 20 10

    // 删除头节点 10
    deleteNode(head); // 20 -> 30
    std::cout << "After deleting 10:" << std::endl;
    printList(); // 期望: List (forward): 20 30, List (backward): 30 20

    // 删除尾节点 30
    deleteNode(tail); // 20
    std::cout << "After deleting 30:" << std::endl;
    printList(); // 期望: List (forward): 20, List (backward): 20

    // 删除最后一个节点 20
    deleteNode(head); // 空链表
    std::cout << "After deleting 20:" << std::endl;
    printList(); // 期望: List (forward): , List (backward):

    return 0;
}
```

**时间复杂度分析**：
*   `insertAfter(Node* a, int val)`：由于我们已经有了节点 `a` 的指针，所以插入操作只需要进行固定数量的指针修改，时间复杂度为 **O(1)**。
*   `deleteNode(Node* s)`：同样，有了节点 `s` 的指针，删除操作也只需要进行固定数量的指针修改，时间复杂度为 **O(1)**。
相比单链表，如果要删除一个已知值的节点，单链表需要 O(N) 遍历找到其前驱，而双链表可以直接通过 `s->p` 找到前驱，因此效率更高。

### 例题二：循环链表模拟“报数游戏”

**题目描述**：
假设有 `N` 个人围成一个圈，从第一个人开始报数，报到 `M` 的人出列，下一个人继续从 1 开始报数，直到所有人都出列。请用循环链表模拟这个过程，并输出出列的顺序。

**思路分析**：
这正是循环链表的典型应用场景——约瑟夫环问题的一个简化版。
1.  **构建循环链表**：首先，创建 `N` 个节点，并将它们连接成一个单向循环链表。为了方便操作，我们可以维护一个 `tail` 指针，这样 `tail->n` 就是头节点。
2.  **模拟报数与出列**：
    *   从 `tail->n` (第一个人) 开始，沿着 `n` 指针移动 `M-1` 步，找到报到 `M` 的人。
    *   将这个人从链表中删除。删除时需要注意更新前一个节点的 `n` 指针。
    *   继续从出列人的下一个节点开始，重复报数过程，直到链表变空。

**C++ 代码**：

```cpp
#include <bits/stdc++.h>

// 循环链表节点结构
struct Node {
    int v;
    Node *n; // next

    Node(int val) : v(val), n(NULL) {}
};

// 模拟报数游戏
void josephus(int N, int M) {
    if (N <= 0 || M <= 0) {
        std::cout << "Invalid input." << std::endl;
        return;
    }

    // 1. 构建循环链表
    Node *head = new Node(1); // 第一个人
    head->n = head; // 只有一个节点时，自己指向自己
    Node *tail = head;

    for (int i = 2; i <= N; ++i) {
        Node *s = new Node(i);
        s->n = head; // 新节点指向头节点
        tail->n = s; // 当前尾节点指向新节点
        tail = s;    // 更新尾节点
    }

    // 2. 模拟报数与出列
    Node *current = head; // 从第一个人开始
    Node *prev = tail;    // current 的前一个节点 (初始时是尾节点)

    std::cout << "Order of elimination: ";
    while (current->n != current) { // 当链表中不止一个节点时
        // 移动 M-1 步，找到要出列的人
        for (int i = 1; i < M; ++i) {
            prev = current;
            current = current->n;
        }

        // current 是要出列的人
        std::cout << current->v << " ";

        // 从链表中删除 current
        prev->n = current->n; // prev 的 next 指向 current 的 next
        if (current == head) { // 如果删除的是头节点，更新 head
            head = current->n;
        }
        if (current == tail) { // 如果删除的是尾节点，更新 tail
            tail = prev;
        }

        Node *temp = current;
        current = current->n; // 下一个人从 current 的下一个开始报数
        delete temp; // 释放内存
    }

    // 最后一个出列的人
    std::cout << current->v << std::endl;
    delete current; // 释放最后一个节点的内存
}

int main() {
    std::cout << "N=5, M=2:" << std::endl;
    josephus(5, 2); // 期望: 2 4 1 5 3

    std::cout << "\nN=7, M=3:" << std::endl;
    josephus(7, 3); // 期望: 3 6 2 7 5 1 4

    return 0;
}
```

**时间复杂度分析**：
*   **构建链表**：创建 `N` 个节点，每个节点插入操作 O(1)，总共 **O(N)**。
*   **模拟报数与出列**：
    *   总共有 `N` 个人出列。
    *   每次出列，都需要移动 `M-1` 步找到目标节点，这个操作是 **O(M)**。
    *   删除操作是 **O(1)**。
    *   所以总的时间复杂度是 **O(N * M)**。

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// 双链表节点结构
struct Node {
    int v;         // 数据域，存放节点的值
    Node *p;       // 指针域，指向前一个节点 (previous)
    Node *n;       // 指针域，指向下一个节点 (next)

    // 构造函数，方便创建新节点
    Node(int val) : v(val), p(NULL), n(NULL) {}
};

// 全局头尾指针，方便操作（实际应用中通常封装在类里）
Node *head = NULL;
Node *tail = NULL;

// 初始化链表（插入第一个节点）
void initList(int val) {
    if (head != NULL) {
        std::cout << "List already initialized." << std::endl;
        return;
    }
    head = new Node(val);
    tail = head;
}

// 在链表尾部插入新节点
void insertTail(int val) {
    if (head == NULL) { // 如果链表为空
        initList(val);
        return;
    }
    Node *s = new Node(val);
    s->p = tail;
    tail->n = s;
    tail = s;
}

// 在链表头部插入新节点
void insertHead(int val) {
    if (head == NULL) { // 如果链表为空
        initList(val);
        return;
    }
    Node *s = new Node(val);
    s->n = head;
    head->p = s;
    head = s;
}

// 遍历并打印链表（向前）
void printListForward() {
    Node *current = head;
    std::cout << "Forward List: ";
    while (current != NULL) {
        std::cout << current->v << " ";
        current = current->n;
    }
    std::cout << std::endl;
}

// 遍历并打印链表（向后）
void printListBackward() {
    Node *current = tail;
    std::cout << "Backward List: ";
    while (current != NULL) {
        std::cout << current->v << " ";
        current = current->p;
    }
    std::cout << std::endl;
}

// 释放整个链表的内存
void clearList() {
    Node *current = head;
    while (current != NULL) {
        Node *temp = current;
        current = current->n;
        delete temp;
    }
    head = NULL;
    tail = NULL;
    std::cout << "List cleared." << std::endl;
}

int main() {
    // 示例用法
    initList(10);
    insertTail(20);
    insertTail(30);
    insertHead(5);
    printListForward();  // 期望: 5 10 20 30
    printListBackward(); // 期望: 30 20 10 5

    clearList();
    printListForward(); // 期望: List cleared. Forward List:
    return 0;
}
```

### 循环链表模板 (单向)

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// 循环链表节点结构
struct Node {
    int v;         // 数据域
    Node *n;       // 指针域，指向下一个节点

    // 构造函数
    Node(int val) : v(val), n(NULL) {}
};

// 全局尾指针，方便操作（通常维护尾指针，因为 tail->n 就是头节点）
Node *tail = NULL;

// 插入第一个节点（创建循环链表）
void initCircularList(int val) {
    if (tail != NULL) {
        std::cout << "List already initialized." << std::endl;
        return;
    }
    tail = new Node(val);
    tail->n = tail; // 自己指向自己，形成循环
}

// 在链表尾部插入新节点
void insertCircularTail(int val) {
    if (tail == NULL) { // 链表为空
        initCircularList(val);
        return;
    }
    Node *s = new Node(val);
    s->n = tail->n; // 新节点指向头节点
    tail->n = s;    // 原尾节点指向新节点
    tail = s;       // 更新尾节点
}

// 在链表头部插入新节点
void insertCircularHead(int val) {
    if (tail == NULL) { // 链表为空
        initCircularList(val);
        return;
    }
    Node *s = new Node(val);
    s->n = tail->n; // 新节点指向头节点 (tail->n)
    tail->n = s;    // 尾节点指向新节点
    // 注意：这里不用更新 tail，因为 tail 仍然是原来的尾节点
}

// 遍历并打印循环链表
void printCircularList() {
    if (tail == NULL) {
        std::cout << "Circular List is empty." << std::endl;
        return;
    }
    Node *head = tail->n; // 头节点
    Node *current = head;
    std::cout << "Circular List: ";
    do {
        std::cout << current->v << " ";
        current = current->n;
    } while (current != head); // 循环直到回到头节点
    std::cout << std::endl;
}

// 删除链表头节点
void deleteCircularHead() {
    if (tail == NULL) {
        std::cout << "Cannot delete from an empty list." << std::endl;
        return;
    }
    Node *head = tail->n; // 获取头节点
    if (head == tail) { // 只有一个节点
        delete head;
        tail = NULL;
    } else {
        tail->n = head->n; // 尾节点指向新的头节点
        delete head;
    }
}

// 释放整个循环链表的内存
void clearCircularList() {
    if (tail == NULL) {
        std::cout << "List is already empty." << std::endl;
        return;
    }
    Node *head = tail->n;
    Node *current = head;
    do {
        Node *temp = current;
        current = current->n;
        delete temp;
    } while (current != head);
    tail = NULL;
    std::cout << "Circular List cleared." << std::endl;
}

int main() {
    // 示例用法
    initCircularList(10);
    insertCircularTail(20);
    insertCircularTail(30);
    insertCircularHead(5);
    printCircularList(); // 期望: 5 10 20 30

    deleteCircularHead(); // 删除 5
    printCircularList(); // 期望: 10 20 30

    deleteCircularHead(); // 删除 10
    printCircularList(); // 期望: 20 30

    clearCircularList();
    printCircularList(); // 期望: Circular List cleared. Circular List is empty.
    return 0;
}
```
