## 趣味引入

真实场景  
想象你是学校食堂的数据统计员，需要每天统计每位同学前 1~100 天内的总消费。  
  
已知：每天的消费金额数组 a[i]（i=1…100）  
未知：如何在收到大量查询（如“第 20 天到第 50 天”的总消费）时快速得到答案？  

思考：  
- 直接累加每次查询 O(n) 会非常慢  
- 有没有一种“预处理”思路，让查询变成 O(1)？  

===NEXT===

## 知识点讲解

1. 前缀和定义  
   - 令 s[0]=0  
   - 对于 1≤i≤n，有 s[i]=a[1]+a[2]+…+a[i]  

2. 区间和公式  
   - 要求 a[l]…a[r] 的和，只需：  
     sum(l,r)=s[r]-s[l-1]  

3. 前缀和的计算  
   - 从 i=1 到 n，依次累加，时间复杂度 O(n)  
   - 构建后每次查询变为 O(1)  

4. 举例说明  
   - a = [3,1,4,1,5]  
   - s = [0, 3, 4, 8, 9, 14]  
   - 查询 sum(2,4)=s[4]-s[1]=9-3=6

## 代码示例

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    int n, q;
    // 读入数组大小和查询次数
    cin >> n >> q;

    static int a[100005];
    static int s[100005];
    s[0] = 0;

    // 读入原始数组并构建前缀和
    for (int i = 1; i <= n; i++)
    {
        cin >> a[i];
        s[i] = s[i - 1] + a[i];  // 累加得到前缀和
    }

    // 处理每个区间查询
    while (q--)
    {
        int l, r;
        cin >> l >> r;
        // 区间和 = s[r] - s[l-1]
        cout << s[r] - s[l - 1] << '\n';
    }

    return 0;
}
```

## 练习题目

### 基础题  
1. (填空) 给定 a=[2,4,1,3]，其前缀和 s[0…4] 分别是 ______ 。  
2. (填空) 已知 s=[0,2,6,7,10]，则 a[2…4] 的和是 ______ 。

### 进阶题  
3. (单选) 使用前缀和处理一次区间求和的时间复杂度是  
   A. O(1)  
   B. O(n)  
   C. O(log n)  
   D. O(n log n)  
4. (填空) 在以下代码片段中，缺少一行计算前缀和，请将“_____”补充完整：  
   ```cpp
   s[0] = 0;
   for (int i = 1; i <= n; i++)
   {
       cin >> a[i];
       _____;  // 请补充此处
   }
   ```

---  
### 答案与解析

1. s=[0,2,6,7,10]  
   解析：s[0]=0；s[1]=2；s[2]=2+4=6；s[3]=6+1=7；s[4]=7+3=10。  
2. sum(2,4)=s[4]-s[1]=10-2=8  
3. A  
   解析：前缀和构建后，每次查询直接用差值运算，时间复杂度为 O(1)。  
4. 填入 `s[i] = s[i-1] + a[i];`  
   解析：每步累加上一个前缀和和当前元素，即可得到 s[i]。
