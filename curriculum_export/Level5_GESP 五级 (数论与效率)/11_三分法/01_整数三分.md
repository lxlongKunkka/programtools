## 趣味引入

同学们，大家好！还记得我们之前学过的“二分法”吗？那就像是在一排整齐的、按身高排队的同学中，快速找到某个身高的同学，对不对？因为队伍是有序的，身高不是越来越高就是越来越矮。

但是，生活中的很多情况可不是这么简单的哦！

想象一下，你正在玩一个“扔纸飞机”的游戏。你想让纸飞机飞得最远。你发现：
*   如果扔的力气太小，飞机飞不远。
*   如果扔的力气太大，飞机可能一下子就冲到地上，也飞不远。
*   是不是存在一个“刚刚好”的力气，能让飞机飞得最远呢？

再比如，我们去爬山。从山脚开始，山路先是越来越陡（高度上升），然后到达山顶，接着又越来越缓（高度下降）。如果你想找到山顶的最高点，你会怎么找呢？你不可能一步一步地走过去，那样太慢了！

像这种“先上升后下降”或者“先下降后上升”的函数，我们称之为“单峰函数”或者“单谷函数”。它们的特点是，要么有一个最高点（山峰），要么有一个最低点（山谷），而且在这点两侧，函数的趋势是单一的。

二分法只能处理单调的（一直上升或一直下降）函数。那么对于这种“有山峰”或“有山谷”的函数，我们有没有一种像二分法一样高效的查找方法呢？

答案是：有！它就是我们今天要学习的——**三分法**！它就像是升级版的“找山顶/山谷”小能手，能帮助我们快速找到那个“刚刚好”的力气，或者那个“最高点”！

## 深度知识点讲解

### 1. 三分法的本质：寻找单峰（谷）函数的极值

**什么是单峰（谷）函数？**
*   **单峰函数 (Unimodal Function)**：在一个给定区间内，函数值先严格单调递增，达到一个最大值后，再严格单调递减。形状就像一座山峰。
    *   例如：`f(x) = -x^2 + 6x - 5` 在 `[0, 6]` 区间内。
*   **单谷函数 (Unimodal Function)**：在一个给定区间内，函数值先严格单调递减，达到一个最小值后，再严格单调递增。形状就像一个山谷。
    *   例如：`f(x) = x^2 - 6x + 5` 在 `[0, 6]` 区间内。

**核心思想：**
三分法就是用来在一个给定区间 `[L, R]` 内，高效地找到单峰（谷）函数的最大值或最小值。它和二分法一样，也是一种“分而治之”的策略。

**为什么不能用二分法？**
二分法要求函数是单调的（一直增或一直减）。如果函数是单峰或单谷的，你随便取一个中点 `mid`，比较 `f(mid)` 和 `f(mid+1)`，你无法确定极值点在 `mid` 的哪一侧，因为 `mid` 可能在上升段，也可能在下降段。

### 2. 三分法的工作原理：三等分区间

既然二分法不行，那我们怎么办呢？三分法给出了一个巧妙的解决方案：

1.  **划分区间**：我们将当前的搜索区间 `[L, R]` 平均分成三份。
    *   我们选择两个点 `m1` 和 `m2`。
    *   `m1 = L + (R - L) / 3`
    *   `m2 = R - (R - L) / 3` (或者 `m2 = m1 + (R - L) / 3`)
    *   这样，区间被分成了 `[L, m1]`, `[m1, m2]`, `[m2, R]` 三个部分。
    *   **注意整数三分**：由于是整数，`m1` 和 `m2` 可能会非常接近。当 `R - L` 小于某个值（比如小于 3）时，`m1` 和 `m2` 可能重合或计算有问题。这是整数三分法需要特别注意的地方。

2.  **比较函数值**：计算 `f(m1)` 和 `f(m2)` 的值，并进行比较。

3.  **缩小范围**：根据比较结果，我们可以排除掉三分之一的区间，将搜索范围缩小。

**以寻找最小值（单谷函数）为例：**

假设我们的函数 `f(x)` 是一个单谷函数（先下降后上升），我们要找它的最小值。

*   **情况一：`f(m1) < f(m2)`**
    *   这意味着 `m1` 处的函数值比 `m2` 处的函数值更小。
    *   如果最小值在 `[m2, R]` 这个区间里，那么 `f(m2)` 应该比 `f(m1)` 更小（因为 `m1` 在 `m2` 左边，如果最小值在 `[m2, R]`，那么 `m1` 肯定在下降段，`m2` 离谷底更近）。
    *   但现在 `f(m1) < f(m2)`，这说明最小值不可能在 `(m2, R]` 这个区间了。
    *   所以，我们可以安全地将右边界 `R` 更新为 `m2`。
    *   新的搜索区间变为 `[L, m2]`。

    *   **生活类比**：你在一个山谷里找最低点。你在 `m1` 和 `m2` 两个点测了海拔。发现 `m1` 比 `m2` 低。那最低点肯定在 `m2` 的左边或者就是 `m2`。所以，`m2` 右边的区域就可以不用找了。

*   **情况二：`f(m1) >= f(m2)`**
    *   这意味着 `m2` 处的函数值小于或等于 `m1` 处的函数值。
    *   如果最小值在 `[L, m1]` 这个区间里，那么 `f(m1)` 应该比 `f(m2)` 更小（因为 `m2` 在 `m1` 右边，如果最小值在 `[L, m1]`，那么 `m2` 肯定在上升段，`m1` 离谷底更近）。
    *   但现在 `f(m1) >= f(m2)`，这说明最小值不可能在 `[L, m1)` 这个区间了。
    *   所以，我们可以安全地将左边界 `L` 更新为 `m1`。
    *   新的搜索区间变为 `[m1, R]`。

    *   **生活类比**：你在一个山谷里找最低点。你在 `m1` 和 `m2` 两个点测了海拔。发现 `m2` 比 `m1` 低或者一样高。那最低点肯定在 `m1` 的右边或者就是 `m1`。所以，`m1` 左边的区域就可以不用找了。

**总结：**
*   **找最小值**：
    *   如果 `f(m1) < f(m2)`，则 `R = m2`。
    *   如果 `f(m1) >= f(m2)`，则 `L = m1`。
*   **找最大值**：
    *   如果 `f(m1) > f(m2)`，则 `R = m2`。
    *   如果 `f(m1) <= f(m2)`，则 `L = m1`。
    （注意：这里的等号处理可以灵活，只要保证不漏掉极值点即可。通常为了代码简洁，会把等号并入其中一个分支。）

### 3. 终止条件与边界处理

由于是整数三分，当区间 `[L, R]` 足够小时，例如 `R - L < 3` 时，`m1` 和 `m2` 的计算可能会出现问题（比如 `m1 == m2`）。
因此，我们通常设置循环的终止条件为 `L + 2 < R` (或者 `L + 3 < R`)。
当循环结束后，区间 `[L, R]` 中只剩下少数几个点（例如 `L, L+1, L+2`）。我们只需要对这几个点逐一计算函数值，然后比较出最终的最值即可。

### 4. 时间复杂度

每一步三分操作，我们都将搜索区间缩小到原来的 `2/3`。
假设初始区间长度为 `N`。
第一次操作后，区间长度变为 `N * (2/3)`。
第二次操作后，区间长度变为 `N * (2/3) * (2/3) = N * (2/3)^2`。
...
第 `k` 次操作后，区间长度变为 `N * (2/3)^k`。
当 `N * (2/3)^k` 接近于常数时，循环终止。
我们可以得到 `(2/3)^k = 1/N`，即 `k * log(2/3) = -log N`。
所以 `k = log N / log(3/2)`，也就是 `k = O(log_1.5 N)`。
通常我们简写为 `O(log N)`。
可见，三分法和二分法一样，都是非常高效的算法！

### 5. 常见误区

*   **函数非单峰/谷**：三分法的前提是函数必须是单峰或单谷的。如果函数有多个峰谷，三分法可能无法找到全局最值。
*   **整数除法问题**：`m1 = L + (R - L) / 3` 和 `m2 = R - (R - L) / 3` 在整数除法时可能会有精度损失，但对于整数三分法来说，这种计算方式是标准且正确的。关键是终止条件和对最后几个点的处理。
*   **比较符号错误**：寻找最大值和最小值时，比较符号是相反的。一定要仔细区分。
*   **区间更新错误**：`L = m1` 和 `R = m2` 是包含 `m1` 或 `m2` 本身的，这与二分法中 `L = mid + 1` 或 `R = mid - 1` 有所不同，因为极值点可能就在 `m1` 或 `m2` 上。

## 典型例题精讲

### 例题一：寻找二次函数最小值

**题目描述：**
给定一个二次函数 `f(x) = ax^2 + bx + c`，其中 `a > 0`。请你在整数区间 `[L, R]` 内，找到使得 `f(x)` 取得最小值的整数 `x`，并输出这个最小值。
数据范围：`a, b, c` 均为整数，`|a|, |b|, |c| <= 1000`。`L, R` 均为整数，`0 <= L <= R <= 10000`。

**思路分析：**
1.  **识别函数类型**：`f(x) = ax^2 + bx + c` 且 `a > 0`，这是一个开口向上的抛物线，它是一个典型的单谷函数。因此，可以使用三分法来寻找最小值。
2.  **定义 `check` 函数**：我们需要一个函数来计算给定 `x` 时的 `f(x)` 值。
    ```cpp
    long long calculate_f(int x, int a, int b, int c) {
        return (long long)a * x * x + (long long)b * x + c;
    }
    ```
    这里使用 `long long` 是为了防止 `a*x*x` 等中间结果溢出 `int` 范围。
3.  **三分法框架**：
    *   初始化 `l = L`, `r = R`。
    *   循环条件：`l + 2 < r`。
    *   在循环内计算 `m1 = l + (r - l) / 3`, `m2 = r - (r - l) / 3`。
    *   比较 `f(m1)` 和 `f(m2)`：
        *   如果 `f(m1) < f(m2)`，说明最小值可能在 `[l, m2]` 范围内，更新 `r = m2`。
        *   如果 `f(m1) >= f(m2)`，说明最小值可能在 `[m1, r]` 范围内，更新 `l = m1`。
4.  **处理剩余区间**：循环结束后，`l` 和 `r` 之间只有少数几个点（例如 `l, l+1, l+2`）。遍历这些点，找出最小值。

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 计算函数 f(x) = ax^2 + bx + c 的值
long long calculate_f(int x, int a, int b, int c) {
    return (long long)a * x * x + (long long)b * x + c;
}

int main() {
    // 禁用同步流，提高输入输出效率
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);
    std::cout.tie(0);

    int a, b, c; // 函数系数
    int l, r;    // 搜索区间 [l, r]

    std::cin >> a >> b >> c >> l >> r;

    // 三分法搜索
    while (l + 2 < r) { // 当区间长度大于2时继续三分
        int m1 = l + (r - l) / 3; // 第一个三等分点
        int m2 = r - (r - l) / 3; // 第二个三等分点

        // 比较 f(m1) 和 f(m2)
        // 寻找最小值，如果 f(m1) 更小，则最小值可能在 [l, m2] 区间
        if (calculate_f(m1, a, b, c) < calculate_f(m2, a, b, c)) {
            r = m2; // 缩小右边界
        } else { // 否则，最小值可能在 [m1, r] 区间
            l = m1; // 缩小左边界
        }
    }

    // 循环结束后，区间 [l, r] 只剩下少数几个点 (l, l+1, l+2 等)
    // 遍历这些点，找到最终的最小值
    long long min_val = calculate_f(l, a, b, c); // 初始最小值为 l 处的函数值
    for (int i = l + 1; i <= r; ++i) { // 遍历剩余点
        min_val = std::min(min_val, calculate_f(i, a, b, c));
    }

    std::cout << min_val << std::endl;

    return 0;
}
```

**时间复杂度：**
*   三分法循环：`O(log R)`，因为每次迭代将区间缩小到原来的 `2/3`。
*   循环结束后线性扫描：`O(1)`，因为剩余区间长度最大为 2 或 3 个点。
*   `calculate_f` 函数：`O(1)`。
*   总时间复杂度：`O(log R)`。对于 `R=10000`，`log_1.5 10000` 大约是 `log_1.5 (1.5^25)`，也就是 25 次左右，非常高效。

### 例题二：最小化最大值问题

**题目描述：**
给定 `N` 个整数 `p_1, p_2, ..., p_N`，它们代表了 `N` 个客户在一条数轴上的位置。现在你需要在数轴上选择一个整数点 `x` 来建立一个服务站，使得所有客户到服务站的最大距离最小。请你输出这个最小的最大距离。
例如：客户位置 `[1, 2, 7]`。
如果 `x=1`，最大距离是 `max(|1-1|, |2-1|, |7-1|) = max(0, 1, 6) = 6`。
如果 `x=2`，最大距离是 `max(|1-2|, |2-2|, |7-2|) = max(1, 0, 5) = 5`。
如果 `x=3`，最大距离是 `max(|1-3|, |2-3|, |7-3|) = max(2, 1, 4) = 4`。
如果 `x=4`，最大距离是 `max(|1-4|, |2-4|, |7-4|) = max(3, 2, 3) = 3`。
如果 `x=5`，最大距离是 `max(|1-5|, |2-5|, |7-5|) = max(4, 3, 2) = 4`。
数据范围：`N <= 100000`，客户位置 `p_i` 范围 `[-10^9, 10^9]`。

**思路分析：**
1.  **定义 `check` 函数**：对于一个给定的服务站位置 `x`，我们需要计算所有客户到它的最大距离。
    ```cpp
    long long calculate_max_dist(int x, int p[], int n) {
        long long max_d = 0;
        for (int i = 0; i < n; ++i) {
            max_d = std::max(max_d, std::abs((long long)p[i] - x));
        }
        return max_d;
    }
    ```
    这个函数 `f(x) = max_i(|p_i - x|)` 的图像是一个“山谷”形状的函数（先下降后上升），因此是单谷函数，可以使用三分法。
    *   **证明 (直观理解)**：`|p_i - x|` 是一个V形函数。多个V形函数的最大值，其图像也是一个V形（或者说，是多个V形函数上包络线的一部分）。
2.  **确定搜索区间 `[L, R]`**：
    *   `L` 可以取所有客户位置中的最小值。
    *   `R` 可以取所有客户位置中的最大值。
    *   更安全的做法是，取一个包含所有客户位置的足够大的区间，例如 `min(p_i) - 100` 到 `max(p_i) + 100`，或者直接用题目给出的最大/小位置范围。
    *   在本题中，客户位置 `p_i` 范围是 `[-10^9, 10^9]`，所以 `L = -1000000000`, `R = 1000000000` 是一个合理的搜索范围。
3.  **三分法框架**：与例题一类似，使用三分法在 `[L, R]` 范围内寻找 `calculate_max_dist` 的最小值。

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 数组 p 存储客户位置，n 为客户数量
int P[100005]; // 使用大写 P 避免与 int p[] 参数冲突

// 计算给定服务站位置 x 时，所有客户到它的最大距离
long long calculate_max_dist(int x, int p_arr[], int n) {
    long long max_d = 0;
    for (int i = 0; i < n; ++i) {
        max_d = std::max(max_d, std::abs((long long)p_arr[i] - x));
    }
    return max_d;
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);
    std::cout.tie(0);

    int n; // 客户数量
    std::cin >> n;

    int min_p = 2000000000; // 初始化为足够大的值
    int max_p = -2000000000; // 初始化为足够小的值

    for (int i = 0; i < n; ++i) {
        std::cin >> P[i];
        min_p = std::min(min_p, P[i]);
        max_p = std::max(max_p, P[i]);
    }

    // 搜索区间可以取所有客户位置的最小值和最大值
    // 也可以稍微扩大一点，以确保极值点不会被排除
    int l = min_p;
    int r = max_p;
    // 考虑到题目给的 p_i 范围，如果极值点可能在客户范围之外，需要扩大 L, R
    // 但对于此题，最优服务站位置 x 一定在 [min_p, max_p] 之间。
    // (因为如果 x 在 min_p 左边，将 x 往右移到 min_p，max_dist 不会增加；
    // 同理如果 x 在 max_p 右边，将 x 往左移到 max_p，max_dist 不会增加。)

    // 三分法搜索
    // 为了确保找到精确的整数解，可以设置循环次数，或者当区间足够小之后再线性扫描
    // 这里采用 L + 2 < R 的方式
    while (l + 2 < r) {
        int m1 = l + (r - l) / 3;
        int m2 = r - (r - l) / 3;

        // 注意这里 m1 和 m2 可能相等，如果相等，则跳过迭代，或者确保它们不同
        // 但通常情况下，当 l+2 < r 时，m1 和 m2 都会是不同的有效点
        if (m1 == m2) { // 避免死循环，如果区间太小导致 m1==m2，直接退出
            break;
        }

        // 寻找最小值，如果 calculate_max_dist(m1) 更小，则最小值可能在 [l, m2]
        if (calculate_max_dist(m1, P, n) < calculate_max_dist(m2, P, n)) {
            r = m2;
        } else { // 否则，最小值可能在 [m1, r]
            l = m1;
        }
    }

    // 循环结束后，l 和 r 之间只剩下少数几个点
    // 遍历这些点，找到最终的最小值
    long long min_max_dist = calculate_max_dist(l, P, n);
    for (int i = l + 1; i <= r; ++i) {
        min_max_dist = std::min(min_max_dist, calculate_max_dist(i, P, n));
    }

    std::cout << min_max_dist << std::endl;

    return 0;
}
```

**时间复杂度：**
*   读取输入并找到 `min_p`, `max_p`：`O(N)`。
*   三分法循环：`O(log(max_p - min_p))` 次。
*   每次循环内调用 `calculate_max_dist`：`O(N)`。
*   循环结束后线性扫描：`O(1)` 次调用 `calculate_max_dist`。
*   总时间复杂度：`O(N * log(max_p - min_p))`。
*   对于 `N=100000`，`log(2 * 10^9)` 大约是 `log_1.5 (2 * 10^9)` 约为 60 次。
*   总操作次数 `100000 * 60 = 6 * 10^6`，在可接受范围内。

// 这个函数就是我们需要找最值（最小或最大）的单峰/单谷函数
// 参数：x 是当前要计算的点，其他参数根据具体问题定义
// 返回值：x 处的函数值 (使用 long long 以防溢出)
long long calculate_value(int x, /* 其他问题相关参数 */) {
    // 假设这里是一个简单的二次函数 f(x) = x*x - 10*x + 30
    // 实际使用时，需要根据题目具体实现
    return (long long)x * x - 10LL * x + 30; 
}

int main() {
    // 禁用同步流，提高输入输出效率
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);
    std::cout.tie(0);

    // ================== 输入部分 ==================
    // 根据具体问题输入必要的参数，例如函数系数、搜索区间等
    // int a, b, c; // 如果是二次函数
    int l, r;    // 搜索区间的左右边界
    std::cin >> l >> r; 
    // ==============================================

    // ================== 三分法核心逻辑 ==================
    // 寻找最小值（如果是寻找最大值，比较符号需要反过来）
    // 循环条件：当区间长度 l+2 < r 时继续三分
    // 这个条件确保了 m1 和 m2 至少有一个点在 l 和 r 之间，且 l, m1, m2, r 之间有足够空间
    while (l + 2 < r) { 
        int m1 = l + (r - l) / 3; // 第一个三等分点
        int m2 = r - (r - l) / 3; // 第二个三等分点

        // 如果 m1 和 m2 相等，说明区间太小，无法再有效划分，直接跳出循环
        // (在 l + 2 < r 条件下，通常不会相等，但为了严谨性可以加上)
        if (m1 == m2) {
            break; 
        }

        // 比较 m1 和 m2 处的函数值
        // 如果 f(m1) < f(m2)，说明最小值在 [l, m2] 范围内，更新右边界
        // 注意：这里假设 calculate_value 函数只依赖 x，如果依赖其他参数，需要传入
        if (calculate_value(m1 /*, 其他参数 */) < calculate_value(m2 /*, 其他参数 */)) {
            r = m2; 
        } 
        // 否则 f(m1) >= f(m2)，说明最小值在 [m1, r] 范围内，更新左边界
        else { 
            l = m1; 
        }
    }

    // ================== 处理剩余区间 ==================
    // 循环结束后，l 和 r 之间只剩下少数几个点（例如 l, l+1, l+2, ... r）
    // 遍历这些点，找到最终的最小值
    long long min_v = calculate_value(l /*, 其他参数 */); // 初始最小值为 l 处的函数值
    for (int i = l + 1; i <= r; ++i) { // 遍历剩余点
        min_v = std::min(min_v, calculate_value(i /*, 其他参数 */));
    }
    // ==================================================

    // ================== 输出结果 ==================
    std::cout << min_v << std::endl;
    // ==============================================

    return 0;
}

### 提问环节

1.  **问：** 同学们，我们学过的二分法和今天的三分法，它们分别适用于哪种类型的函数呢？它们之间有什么本质的区别？
    *   **预期回答：** 二分法适用于单调函数（一直递增或一直递减）；三分法适用于单峰或单谷函数（先增后减或先减后增）。区别在于二分法只需要一个中间点，三分法需要两个中间点来判断极值点在哪一侧。
2.  **问：** 假设我们要找一个单峰函数（先上升后下降）的最大值。当 `f(m1)` 和 `f(m2)` 比较时，如果 `f(m1) > f(m2)`，我们应该怎么更新 `L` 和 `R` 呢？为什么？
    *   **预期回答：** 应该更新 `R = m2`。因为 `f(m1)` 比 `f(m2)` 大，说明极值点更可能在 `m2` 的左边（或者就是 `m2`），`m2` 右边的部分可以舍弃。
3.  **问：** 在整数三分法中，我们循环的终止条件通常是 `L + 2 < R`。为什么不是 `L < R` 或 `L + 1 < R` 呢？
    *   **预期回答：** 因为 `m1 = L + (R - L) / 3` 和 `m2 = R - (R - L) / 3` 在 `R - L` 很小时，可能会导致 `m1` 和 `m2` 重合，或者 `L, m1, m2, R` 之间没有足够多的不同整数点进行有效比较。`L + 2 < R` 可以确保至少有 `L, L+1, L+2, ... R` 这样的多个点，使得 `m1` 和 `m2` 能有效划分。最后剩下的少数几个点可以直接遍历。
4.  **问：** 三分法的时间复杂度是 `O(log N)`，这里的 `log` 是以几为底呢？它比二分法的 `log_2 N` 更快还是更慢？
    *   **预期回答：** 是以 `1.5` 为底（或者说 `3/2` 为底）。因为每一步区间缩小到原来的 `2/3`。`log_1.5 N` 比 `log_2 N` 要大（因为底数更小），所以三分法比二分法迭代次数略多，但都是对数级别，效率非常高。

### 即时练习

**题目：**
给定函数 `g(x) = -(x-3)^2 + 10`。请你在整数区间 `[0, 10]` 上，用三分法找到 `g(x)` 的最大值。
请写出你第一次迭代（即 `L=0, R=10` 时）的 `m1, m2` 以及 `f(m1), f(m2)` 的值，并说明 `L` 和 `R` 会如何更新。

**解答步骤：**
*   `L = 0, R = 10`
*   `m1 = L + (R - L) / 3 = 0 + (10 - 0) / 3 = 0 + 3 = 3`
*   `m2 = R - (R - L) / 3 = 10 - (10 - 0) / 3 = 10 - 3 = 7`
*   计算 `g(3) = -(3-3)^2 + 10 = 0 + 10 = 10`
*   计算 `g(7) = -(7-3)^2 + 10 = -(4)^2 + 10 = -16 + 10 = -6`
*   因为要找最大值，`g(m1) = 10` 远大于 `g(m2) = -6`。
*   所以最大值点在 `m2` 的左侧，更新 `R = m2`。
*   新的搜索区间变为 `[L, R] = [0, 7]`。
