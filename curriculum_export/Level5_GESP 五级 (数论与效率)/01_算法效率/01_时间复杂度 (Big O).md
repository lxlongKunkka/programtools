## 趣味引入

真实场景：  
小明在玩扑克牌时，需要把一副 52 张的牌按点数顺序排列。一次他和同学分别用了不同的方法排序：  
- 方法 A：每次随机交换两张牌，直到有序。  
- 方法 B：先把牌分成两堆，再递归地合并。  

小明很好奇：哪种方法在牌更多时（比如 1000 张、10000 张）表现更好？直接测算太麻烦，也不够准确，我们如何用“理论”来判断呢？  

已知：  
- 操作次数往往与数据规模（牌的数量）相关  
- 直接测量耗时受机器性能干扰  

未知：  
- 如何衡量算法随牌数增长的“趋势”  

引出本节新知：时间复杂度（Big O），用来度量“增长趋势”，与机器性能无关。

## 知识点讲解

1. 基本操作数  
   - 在普通计算机上，加减乘除、访问基本类型变量、赋值等可视为“基本操作”  
   - 我们通过计数或估测基本操作数来近似衡量算法用时  

2. 数据规模  
   - 通常用输入元素个数 \(n\) 来表示  
   - 随着 \(n\) 增大，算法用时也会变化  

3. 时间复杂度的定义  
   - 关注“用时随 \(n\) 增长的趋势”  
   - 忽略常数与低阶项，只保留影响最显著的部分  
   - 最坏时间复杂度：在所有可能输入中，所需操作数最多时的增长趋势  

4. 渐近符号（Big O）  
   - 记作 \(O(f(n))\)，表示算法用时上界与 \(f(n)\) 同阶  
   - 例如：若算法操作数为 \(3n^2 + 5n + 10\)，则为 \(O(n^2)\)  

5. 复杂度分析示例  
   - 顺序执行：\(O(1)\)  
   - 单层循环：  
     for (i = 0; i < n; ++i) 执行 \(n\) 次，\(O(n)\)  
   - 嵌套双重循环：  
     for (i = 0; i < n; ++i)  
       for (j = 0; j < n; ++j)  
     共执行 \(n \times n\) 次，\(O(n^2)\)  
   - 三重循环举例：\(O(n^2 m)\)

## 代码示例

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    int n, m;
    cin >> n >> m;              // 读取数据规模 n、m
    // 三重循环，计算时间复杂度 Θ(n^2 * m)
    for (int i = 0; i < n; ++i)
    {
        for (int j = 0; j < n; ++j)
        {
            for (int k = 0; k < m; ++k)
            {
                // 基本操作：一次输出，可认为是一次“基本操作”
                cout << "Hello" << "\n";  
            }
        }
    }
    return 0;
}
```

关键注释：  
- 外层两重循环共执行 \(n^2\) 次  
- 内层循环每次执行 \(m\) 次  
- 总操作≈\(n^2 \times m\)，记为 Θ(\(n^2m\))，同阶于 O(\(n^2m\))

## 练习题目

### 基础题（选择/填空）

1. 下列代码的时间复杂度是_______。  
   ```cpp
   for (int i = 0; i < n; ++i)
   {
       // 常数时间操作
       a = a + 1;
   }
   ```
   A. O(1)  B. O(n)  C. O(log n) D. O(n^2)  

2. 对于以下两段算法，当 n 很大时，哪一个更快？（多选）  
   Ⅰ. \(O(n \log n)\)  
   Ⅱ. \(O(n^2)\)  
   A. Ⅰ  B. Ⅱ  C. 一样快  

3. 填空：若算法运行步骤数为 \(5n^3 + 2n^2 + 100\)，其最坏时间复杂度是_________。

### 进阶题（选择/填空）

4. 下列代码的渐近时间复杂度是_______。  
   ```cpp
   for (int i = 0; i < n; ++i)
   {
       for (int j = 0; j < n; ++j)
       {
           for (int k = 0; k < n; ++k)
           {
               sum += i + j + k;
           }
       }
   }
   ```
   A. O(n^2)  B. O(n^3)  C. O(n^4)  D. O(n^3 log n)  

5. 填空：若算法由两部分组成，第一部分复杂度为 \(O(n)\)，第二部分复杂度为 \(O(n^2)\)，则合并后算法的时间复杂度是_________。

---

### 答案与解析

1. B。单层循环执行 n 次，故为 O(n)。  
2. A。\(n\log n\) 在大 n 下小于 \(n^2\)。  
3. O(n^3)。最高项多项式阶数决定渐近复杂度。  
4. B。三重循环共执行 \(n^3\) 次基本操作。  
5. O(n^2)。合并后取最高阶项。
