## 知识点讲解

### 1. 什么是空间复杂度？

**空间复杂度**（Space Complexity）是对一个算法在运行过程中**临时占用存储空间大小**的度量。

注意，这里强调的是**临时占用**和**额外**存储空间。它不包括输入数据本身所占用的空间，而是指算法在执行过程中为了存储变量、数组、递归调用栈等所额外申请的空间。

我们同样使用**大 O 记法**来表示空间复杂度，它描述的是算法的额外空间消耗与输入规模 `n` 之间的增长关系。

### 2. 空间复杂度的计算方法

在计算空间复杂度时，我们主要关注以下几个方面：

*   **变量：** 声明的普通变量（如 `int x;`）通常只占用常数空间。
*   **数组/数据结构：** 创建的数组、链表等数据结构会占用与其大小成比例的空间。
*   **递归：** 递归调用会占用“栈空间”，其大小与递归的深度成正比。

与时间复杂度类似，我们只关注**额外空间**中增长最快的项，并忽略常数项和低阶项。

### 3. 常见的空间复杂度类型

#### (1) O(1) - 常数空间复杂度

如果算法执行所需的额外存储空间不随输入数据量 `n` 的增大而增大，始终保持为一个常量，那么它的空间复杂度就是 O(1)。

**例子：** 交换两个变量的值。无论这两个变量的值多大，我们都只需要一个临时变量 `t` 来完成交换。

#### (2) O(n) - 线性空间复杂度

如果算法执行所需的额外存储空间与输入数据量 `n` 成正比，那么它的空间复杂度就是 O(n)。

**例子：** 为了存储输入数组的副本或者前缀和，我们创建了一个与输入数组大小相同的额外数组。

#### (3) O(n^2) - 平方空间复杂度

如果算法执行所需的额外存储空间与输入数据量 `n` 的平方成正比，那么它的空间复杂度就是 O(n^2)。

**例子：** 存储一个 `n * n` 的二维矩阵，比如图的邻接矩阵表示。

#### (4) 递归的栈空间

递归函数在调用时，每次调用都会在“调用栈”上分配一块空间，用于存储局部变量、函数参数和返回地址。如果递归深度为 `k`，那么它会占用 O(k) 的栈空间。在很多情况下，递归深度 `k` 与输入规模 `n` 相关，例如线性递归的深度为 `n`，则空间复杂度为 O(n)。

**例子：** 计算斐波那契数列的递归版本。

### 4. 总结

*   空间复杂度衡量的是算法运行过程中**额外**占用的存储空间。
*   使用大 O 记法表示，关注额外空间中增长最快的项。
*   常见类型有 O(1), O(n), O(n^2)，以及递归带来的栈空间占用。
*   在实际编程中，我们常常需要在时间复杂度和空间复杂度之间做出权衡。

## 代码示例

```cpp
#include <bits/stdc++.h>
using namespace std;

// 示例 1: O(1) 常数空间复杂度
// 功能: 交换两个整数的值
void swap_elements(int& a, int& b)
{
    // 额外只使用了常数个变量 (一个临时变量 t)
    int t = a;
    a = b;
    b = t;
} // 空间复杂度: O(1)

// 示例 2: O(1) 常数空间复杂度
// 功能: 计算一个数组所有元素的和
// 注意：输入数组 a 和其大小 n 不计入额外空间复杂度
long long sum_array(int a[], int n)
{
    long long s = 0; // 额外只使用了常数个变量 (s 和循环变量 i)
    for (int i = 0; i < n; i++)
    {
        s += a[i];
    }
    return s;
} // 空间复杂度: O(1)


// 示例 3: O(n) 线性空间复杂度
// 功能: 复制一个数组到新数组
// 注意：输入数组 a 和其大小 n 不计入额外空间复杂度
void copy_array(int a[], int n)
{
    // 创建了一个大小为 n 的新数组 b
    int b[100005]; // 假设 n 不会超过 100000
    for (int i = 0; i < n; i++)
    {
        b[i] = a[i];
    }
    // 这里我们只是演示，实际中可能会返回 b 或者对 b 进行其他操作
} // 空间复杂度: O(n)

// 示例 4: O(n) 线性空间复杂度 (递归栈空间)
// 功能: 递归计算斐波那契数列的第 n 项
// 注意：这个例子主要演示递归的栈空间占用
int fibonacci_recursive(int n)
{
    if (n <= 1)
    {
        return n;
    }
    // 每次递归调用都会在栈上创建一个新的栈帧
    // 递归深度为 n，所以栈空间为 O(n)
    return fibonacci_recursive(n - 1) + fibonacci_recursive(n - 2);
} // 空间复杂度: O(n) (栈空间)

// 示例 5: O(n^2) 平方空间复杂度
// 功能: 初始化一个 n*n 的二维数组 (邻接矩阵)
// 注意：n 假设最大为 1000
void init_matrix(int n)
{
    // 创建了一个 n*n 的二维数组 m
    int m[1005][1005]; // 假设 n 不会超过 1000
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            m[i][j] = 0; // 初始化为 0
        }
    }
} // 空间复杂度: O(n^2)

int main()
{
    int x = 10, y = 20;
    cout << "Before swap: x = " << x << ", y = " << y << endl;
    swap_elements(x, y);
    cout << "After swap: x = " << x << ", y = " << y << endl;

    int arr[] = {1, 2, 3, 4, 5};
    int len = 5;
    cout << "Sum of array elements: " << sum_array(arr, len) << endl;

    cout << "Copying array (O(n) space demo)..." << endl;
    copy_array(arr, len);

    int fib_n = 6;
    cout << "Fibonacci(" << fib_n << ") = " << fibonacci_recursive(fib_n) << endl;

    int matrix_size = 3;
    cout << "Initializing " << matrix_size << "x" << matrix_size << " matrix (O(n^2) space demo)..." << endl;
    init_matrix(matrix_size);

    return 0;
}
```

```cpp
#include <bits/stdc++.h>
using namespace std;

// 功能: 原地翻转数组
void reverse_array_inplace(int a[], int n)
{
    int l = 0;        // 左指针
    int r = n - 1;    // 右指针
    while (l < r)
    {
        // 交换 a[l] 和 a[r]
        int t = a[l];
        a[l] = a[r];
        a[r] = t;
        l++;
        r--;
    }
} // 空间复杂度: O(1)
```

### 编程题

1.  **题目：** 数组元素求平方
    **描述：** 给定一个整数数组 `a` 和其长度 `n`。请编写一个函数 `square_elements`，将数组中每个元素替换为其平方值。要求算法的额外空间复杂度为 O(1)。

    **描述：** 给定一个整数数组 `a` 和其长度 `n`。请编写一个函数 `find_max`，找出数组中的最大值。要求算法的额外空间复杂度为 O(1)。
    **输入：** `a` (整数数组), `n` (数组长度)

### 答案与解析

1.  **答案：B**
    **解析：**
    *   A. 遍历数组通常只用一个循环变量，O(1) 空间。
    *   B. 声明一个 `n * n` 的二维数组，会占用与 `n^2` 成正比的空间，所以是 O(n^2) 空间。
    *   C. 递归深度为 n 的递归函数，栈空间通常为 O(n)。
    *   D. 交换两个变量只用一个临时变量，O(1) 空间。

2.  **答案：A**
    **解析：**
    *   当 n 从 100 变为 1000 时 (10 倍)，内存从 10KB 变为 100KB (10 倍)。这符合 O(n) 的线性关系。
    *   所以当 n 从 1000 变为 10000 时 (10 倍)，内存也会增加 10 倍。
    *   100KB * 10 = 1000KB = 1MB。

3.  **答案：A**
    **解析：**
    *   尽管 `n` 可能很大，但数组 `m` 的大小是固定的 100，它不随 `n` 的增大而增大。
    *   `m[i % 100]` 只是在固定大小的数组中循环写入，没有额外创建新的空间。
    *   因此，额外空间占用是常数，空间复杂度为 O(1)。

4.  **答案：O(1)**
    **解析：** 常数个变量，无论输入规模多大，其占用空间都是固定的，所以是 O(1) 空间复杂度。

5.  **答案：O(n)**
    **解析：**
    *   函数中声明了一个 `s` 数组，其大小 `100005` 是为了容纳最大可能的 `n`。
    *   实际上，这个 `s` 数组用于存储 `n` 个前缀和，其占用的空间与输入数组的长度 `n` 成正比。
    *   因此，额外空间复杂度是 O(n)。
