再举个例子：你面前有一排排的怪兽，它们的战斗力各不相同。你作为一名勇士，想要挑战它们。但是为了节省体力，你决定只挑战一部分怪兽，并且你有一个特别的挑战策略：**你挑战的下一只怪兽的战斗力必须比上一只高！** 这样你就能不断提升自己的实力。现在，你的任务是：**找到一个最长的挑战序列，使得你挑战的怪兽战斗力是严格递增的。**

比如，怪兽的战斗力序列是：`[10, 22, 9, 33, 21, 50, 41, 60]`。
你能挑战的序列可以是：
*   `[10, 22, 33, 50, 60]` （长度为 5）
*   `[10, 21, 41, 60]` （长度为 4）
*   `[9, 21, 41, 60]` （长度为 4）

哪一个最长呢？答案是 `[10, 22, 33, 50, 60]`，长度为5！这就是我们今天要解决的“最长递增子序列”问题。它在计算机科学中可是个“明星”问题，有很多实际应用呢！

这个有趣的挑战，我们该如何用程序来解决它呢？暴力枚举所有可能的子序列肯定太慢了，因为序列可能会非常长。别担心，聪明的计算机科学家们早就想到了一个优雅而高效的方法——**动态规划 (Dynamic Programming)**！

## 深度知识点讲解

### 1. 什么是子序列？

在正式进入LIS之前，我们先来明确几个概念。
**子序列 (Subsequence)**：从一个序列中删除零个或多个元素，剩下的元素保持原有的相对顺序不变，所形成的新序列就是原序列的一个子序列。
**划重点**：元素可以不连续，但相对顺序不能变！

**例子**：
原序列：`A = [1, 3, 2, 4, 5]`
*   `[1, 2, 5]` 是 A 的一个子序列（删除了3和4，相对顺序 `1 < 2 < 5` 保持不变）。
*   `[3, 4]` 是 A 的一个子序列。
*   `[1, 3, 4, 5]` 是 A 的一个子序列。
*   `[5, 2]` **不是** A 的子序列（虽然元素都在 A 中，但 5 在 2 后面，原序列中 5 在 2 后面，相对顺序变了）。
*   `[1, 3, 5, 4]` **不是** A 的子序列（虽然 1,3,5 递增，但 4 在 5 后面，原序列中 4 在 5 前面，相对顺序变了）。

### 2. 什么是递增子序列？

**递增子序列 (Increasing Subsequence)**：一个子序列中，所有元素都是严格递增的（即后一个元素严格大于前一个元素）。

**例子**：
原序列：`A = [10, 22, 9, 33, 21, 50, 41, 60]`
*   `[10, 22, 33]` 是一个递增子序列。
*   `[9, 21, 41, 60]` 是一个递增子序列。
*   `[10, 9, 33]` **不是**递增子序列（因为 10 不小于 9）。

### 3. 最长递增子序列 (LIS) 问题

**最长递增子序列 (Longest Increasing Subsequence, LIS)**：在一个给定的序列中，找到一个长度最长的递增子序列。我们通常只要求输出这个最长递增子序列的长度。

### 4. 为什么用动态规划 (DP)？

*   **暴力枚举？太慢了！**
    一个长度为 `n` 的序列，理论上它的子序列有 `2^n` 种（每个元素都可以选择“取”或“不取”）。如果 `n=30`，`2^30` 大约是 10 亿，计算量太大了！我们需要更聪明的方法。
*   **重叠子问题 (Overlapping Subproblems)**：
    你会发现，当我们计算某个位置的 LIS 时，很可能会用到前面位置已经计算过的 LIS 结果。比如，要计算以 `a[i]` 结尾的 LIS，我们需要知道以 `a[j]` 结尾的 LIS (其中 `j < i`)。这些“以某个元素结尾的 LIS”就是重叠的子问题。
*   **最优子结构 (Optimal Substructure)**：
    一个大问题的最优解可以由其子问题的最优解构成。如果 `a[i]` 是某个 LIS 的最后一个元素，那么这个 LIS 的前一个部分（不包含 `a[i]`）也一定是以某个 `a[j]` (`j < i` 且 `a[j] < a[i]`) 结尾的最长递增子序列。

### 5. 动态规划核心三要素

**要素一：状态定义 (State Definition)**
这是DP最关键的一步！它决定了我们如何存储子问题的解。
对于LIS问题，我们定义 `dp[i]` 表示：**以序列中第 `i` 个元素 `a[i]` 结尾的最长递增子序列的长度。**

**为什么是“以 `a[i]` 结尾”？**
因为如果不是以 `a[i]` 结尾，我们很难确定 `dp[i]` 和 `dp[j]` 之间的关系。如果我们定义 `dp[i]` 是“前 `i` 个元素构成的序列的 LIS 长度”，那么 `dp[i]` 可能不包含 `a[i]`，这会使得状态转移变得复杂。而“以 `a[i]` 结尾”则清晰地指明了 `a[i]` 必须是子序列的最后一个元素，这样我们才能基于 `a[i]` 的值去连接前面的子序列。

**要素二：状态转移方程 (Transition Equation)**
当我们要求 `dp[i]` 时，`a[i]` 必须是子序列的最后一个元素。那么，它前面的那个元素 `a[j]` (`j < i`) 必须满足什么条件呢？
1.  `j` 必须在 `i` 之前，即 `0 <= j < i`。
2.  `a[j]` 必须小于 `a[i]`，这样 `a[j]` 才能作为 `a[i]` 的前一个元素，形成递增关系。

如果满足这两个条件，那么以 `a[i]` 结尾的 LIS 就可以由“以 `a[j]` 结尾的 LIS”加上 `a[i]` 自身构成。所以，它的长度就是 `dp[j] + 1`。
我们希望找到最长的，所以要遍历所有满足条件的 `j`，取 `dp[j] + 1` 的最大值。

**初始值**：每个元素 `a[i]` 自己就可以构成一个长度为 1 的递增子序列。所以，`dp[i]` 的初始值都设为 1。

**状态转移方程**：
对于 `i` 从 `0` 遍历到 `n-1`：
  `dp[i] = 1` (初始化：自身构成长度为1的LIS)
  对于 `j` 从 `0` 遍历到 `i-1`：
    如果 `a[i] > a[j]`：
      `dp[i] = max(dp[i], dp[j] + 1)`

**生活类比**：
想象你正在建造一排乐高塔。每块乐高积木都有一个编号（比如 `a[i]`）。你想知道，如果这块 `a[i]` 积木是塔的顶端，那么这塔最高能有多高？
你就会去看看前面已经建好的所有乐高塔（`j < i`）。如果某座塔 `dp[j]` 的顶端积木 `a[j]` 比你现在要放的 `a[i]` 小，那么你就可以把 `a[i]` 放在那座塔的顶端，形成一座新的塔，高度是 `dp[j] + 1`。你把所有能放的塔都试一遍，选一个最高的作为 `dp[i]` 的值。

**要素三：最终结果 (Final Result)**
我们定义的 `dp[i]` 是“以 `a[i]` 结尾的 LIS 长度”。题目要求的是整个序列的最长递增子序列长度，它可能以任何一个元素结尾。所以，最终结果就是 `dp` 数组中的最大值。
`ans = max(dp[0], dp[1], ..., dp[n-1])`

### 6. 算法步骤总结

1.  **定义**：`dp[i]` 表示以 `a[i]` 结尾的最长递增子序列的长度。
2.  **初始化**：所有 `dp[i]` 都初始化为 1。
3.  **遍历**：
    *   外层循环 `i` 从 `0` 到 `n-1`（表示当前处理的元素 `a[i]`）。
    *   内层循环 `j` 从 `0` 到 `i-1`（表示 `a[i]` 前面的所有元素 `a[j]`）。
    *   **判断**：如果 `a[i] > a[j]`，说明 `a[i]` 可以接在以 `a[j]` 结尾的 LIS 后面。
    *   **更新**：`dp[i] = max(dp[i], dp[j] + 1)`。
4.  **查找最大值**：遍历 `dp` 数组，找到其中的最大值，即为最终答案。

### 7. 时间复杂度分析

*   外层循环 `i` 运行 `n` 次。
*   内层循环 `j` 运行 `i` 次（平均 `n/2` 次）。
*   内部操作（比较、取最大值）是常数时间。
所以，总的时间复杂度是 `O(n * n) = O(n^2)`。
对于 `n` 范围在 `2000` 到 `5000` 的问题，`O(n^2)` 是可以接受的。

### 8. 常见误区

*   **`dp[i]` 的定义模糊**：很多同学容易把 `dp[i]` 定义成“前 `i` 个元素的 LIS 长度”，这样会导致状态转移非常困难。一定要记住，是“以 `a[i]` 结尾”！
*   **初始化错误**：忘记将 `dp[i]` 初始化为 1。如果某个元素 `a[i]` 找不到任何比它小的 `a[j]`，那么它自己就是长度为 1 的 LIS。
*   **循环边界错误**：内层循环 `j` 必须小于 `i`。

## 典型例题精讲

**例题 1：最长递增子序列长度**

**题目描述**：
给定一个整数序列 `a`，求它的最长递增子序列的长度。

**输入格式**：
第一行包含一个整数 `n`，表示序列的长度。
第二行包含 `n` 个整数 `a[0], a[1], ..., a[n-1]`，用空格隔开。

**输出格式**：
输出一个整数，表示最长递增子序列的长度。

**示例**：
**输入**
```
8
10 22 9 33 21 50 41 60
```
**输出**
```
5
```

**思路分析**：

1.  **明确目标**：求最长递增子序列的长度。
2.  **DP状态定义**：`dp[i]` 表示以 `a[i]` 结尾的最长递增子序列的长度。
3.  **DP初始化**：所有 `dp[i]` 都初始化为 1。因为至少每个元素本身就可以构成一个长度为 1 的递增子序列。
4.  **DP状态转移**：
    *   外层循环 `i` 从 `0` 到 `n-1`，处理每一个元素 `a[i]`。
    *   内层循环 `j` 从 `0` 到 `i-1`，检查 `a[i]` 之前的所有元素 `a[j]`。
    *   如果 `a[i] > a[j]`，说明 `a[i]` 可以接在以 `a[j]` 结尾的 LIS 后面。此时，以 `a[i]` 结尾的 LIS 长度可能是 `dp[j] + 1`。
    *   我们需要在所有满足条件的 `j` 中，选择能使 `dp[i]` 最大的那个，所以 `dp[i] = max(dp[i], dp[j] + 1)`。
5.  **最终结果**：遍历整个 `dp` 数组，找到其中的最大值，就是整个序列的最长递增子序列的长度。

**计算过程演示**：
序列 `a = [10, 22, 9, 33, 21, 50, 41, 60]`
长度 `n = 8`

初始化 `dp = [1, 1, 1, 1, 1, 1, 1, 1]`
`max_len = 1`

*   **`i = 0`, `a[0] = 10`**:
    *   `dp[0]` 保持 1。
    *   `max_len = max(1, 1) = 1`。
    `dp = [1, 1, 1, 1, 1, 1, 1, 1]`

*   **`i = 1`, `a[1] = 22`**:
    *   `j = 0`: `a[1] (22) > a[0] (10)`。
        `dp[1] = max(dp[1], dp[0] + 1) = max(1, 1 + 1) = 2`。
    *   `max_len = max(1, 2) = 2`。
    `dp = [1, 2, 1, 1, 1, 1, 1, 1]`

*   **`i = 2`, `a[2] = 9`**:
    *   `j = 0`: `a[2] (9) < a[0] (10)`。不满足 `a[i] > a[j]`。
    *   `j = 1`: `a[2] (9) < a[1] (22)`。不满足 `a[i] > a[j]`。
    *   `dp[2]` 保持 1。
    *   `max_len = max(2, 1) = 2`。
    `dp = [1, 2, 1, 1, 1, 1, 1, 1]`

*   **`i = 3`, `a[3] = 33`**:
    *   `j = 0`: `a[3] (33) > a[0] (10)`。`dp[3] = max(1, dp[0] + 1) = max(1, 1 + 1) = 2`。
    *   `j = 1`: `a[3] (33) > a[1] (22)`。`dp[3] = max(2, dp[1] + 1) = max(2, 2 + 1) = 3`。
    *   `j = 2`: `a[3] (33) > a[2] (9)`。`dp[3] = max(3, dp[2] + 1) = max(3, 1 + 1) = 3`。
    *   `max_len = max(2, 3) = 3`。
    `dp = [1, 2, 1, 3, 1, 1, 1, 1]`

*   **`i = 4`, `a[4] = 21`**:
    *   `j = 0`: `a[4] (21) > a[0] (10)`。`dp[4] = max(1, dp[0] + 1) = max(1, 1 + 1) = 2`。
    *   `j = 1`: `a[4] (21) < a[1] (22)`。不满足 `a[i] > a[j]`。
    *   `j = 2`: `a[4] (21) > a[2] (9)`。`dp[4] = max(2, dp[2] + 1) = max(2, 1 + 1) = 2`。
    *   `j = 3`: `a[4] (21) < a[3] (33)`。不满足 `a[i] > a[j]`。
    *   `max_len = max(3, 2) = 3`。
    `dp = [1, 2, 1, 3, 2, 1, 1, 1]`

*   **`i = 5`, `a[5] = 50`**:
    *   `j = 0`: `a[5] (50) > a[0] (10)`。`dp[5] = max(1, dp[0] + 1) = 2`。
    *   `j = 1`: `a[5] (50) > a[1] (22)`。`dp[5] = max(2, dp[1] + 1) = max(2, 2 + 1) = 3`。
    *   `j = 2`: `a[5] (50) > a[2] (9)`。`dp[5] = max(3, dp[2] + 1) = max(3, 1 + 1) = 3`。
    *   `j = 3`: `a[5] (50) > a[3] (33)`。`dp[5] = max(3, dp[3] + 1) = max(3, 3 + 1) = 4`。
    *   `j = 4`: `a[5] (50) > a[4] (21)`。`dp[5] = max(4, dp[4] + 1) = max(4, 2 + 1) = 4`。
    *   `max_len = max(3, 4) = 4`。
    `dp = [1, 2, 1, 3, 2, 4, 1, 1]`

*   **`i = 6`, `a[6] = 41`**:
    *   `j = 0`: `a[6] (41) > a[0] (10)`。`dp[6] = max(1, dp[0] + 1) = 2`。
    *   `j = 1`: `a[6] (41) > a[1] (22)`。`dp[6] = max(2, dp[1] + 1) = max(2, 2 + 1) = 3`。
    *   `j = 2`: `a[6] (41) > a[2] (9)`。`dp[6] = max(3, dp[2] + 1) = max(3, 1 + 1) = 3`。
    *   `j = 3`: `a[6] (41) > a[3] (33)`。`dp[6] = max(3, dp[3] + 1) = max(3, 3 + 1) = 4`。
    *   `j = 4`: `a[6] (41) > a[4] (21)`。`dp[6] = max(4, dp[4] + 1) = max(4, 2 + 1) = 4`。
    *   `j = 5`: `a[6] (41) < a[5] (50)`。不满足 `a[i] > a[j]`。
    *   `max_len = max(4, 4) = 4`。
    `dp = [1, 2, 1, 3, 2, 4, 4, 1]`

*   **`i = 7`, `a[7] = 60`**:
    *   `j = 0`: `a[7] (60) > a[0] (10)`。`dp[7] = max(1, dp[0] + 1) = 2`。
    *   `j = 1`: `a[7] (60) > a[1] (22)`。`dp[7] = max(2, dp[1] + 1) = max(2, 2 + 1) = 3`。
    *   `j = 2`: `a[7] (60) > a[2] (9)`。`dp[7] = max(3, dp[2] + 1) = max(3, 1 + 1) = 3`。
    *   `j = 3`: `a[7] (60) > a[3] (33)`。`dp[7] = max(3, dp[3] + 1) = max(3, 3 + 1) = 4`。
    *   `j = 4`: `a[7] (60) > a[4] (21)`。`dp[7] = max(4, dp[4] + 1) = max(4, 2 + 1) = 4`。
    *   `j = 5`: `a[7] (60) > a[5] (50)`。`dp[7] = max(4, dp[5] + 1) = max(4, 4 + 1) = 5`。
    *   `j = 6`: `a[7] (60) > a[6] (41)`。`dp[7] = max(5, dp[6] + 1) = max(5, 4 + 1) = 5`。
    *   `max_len = max(4, 5) = 5`。
    `dp = [1, 2, 1, 3, 2, 4, 4, 5]`

最终 `max_len = 5`。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// GESP 6级通常允许使用标准库函数，但禁止使用STL容器如vector, stack, queue。
// sort函数是可以使用的。

const int MAXN = 5005; // 定义最大数组长度，根据题目N的范围设定
int a[MAXN];          // 存储输入的序列
int dp[MAXN];         // dp[i] 表示以 a[i] 结尾的最长递增子序列的长度

int main() {
    // 禁用同步，加速输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 序列长度
    std::cin >> n;

    // 读取序列元素
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    int maxL = 0; // 记录最长递增子序列的总长度

    // 动态规划计算 LIS
    // 外层循环：遍历每一个元素 a[i]
    for (int i = 0; i < n; ++i) {
        // 初始化 dp[i] 为 1，因为每个元素本身就可以构成一个长度为 1 的递增子序列
        dp[i] = 1;

        // 内层循环：遍历 i 之前的所有元素 a[j]
        for (int j = 0; j < i; ++j) {
            // 如果 a[i] 大于 a[j]，说明 a[i] 可以接在以 a[j] 结尾的递增子序列后面
            if (a[i] > a[j]) {
                // 更新 dp[i]：取当前 dp[i] 和 (以 a[j] 结尾的 LIS 长度 + 1) 中的较大值
                // +1 是因为 a[i] 加入了子序列
                dp[i] = std::max(dp[i], dp[j] + 1);
            }
        }
        // 每次计算完 dp[i]，都更新一下全局的最长长度 maxL
        // maxL 记录的是所有 dp[k] 中的最大值
        maxL = std::max(maxL, dp[i]);
    }

    // 如果序列为空，或者只有一个元素，maxL 可能会是0或1。
    // 如果 n=0，maxL 初始为0，dp数组也不会被计算。
    // 如果 n=1，dp[0]会被设为1，maxL也会更新为1。
    // 因此，直接输出 maxL 即可。
    std::cout << maxL << std::endl;

    return 0;
}

```
**时间复杂度**：`O(N^2)`，其中 `N` 是序列的长度。
外层循环 `i` 运行 `N` 次，内层循环 `j` 运行 `i` 次，所以总操作次数大约是 `N * (N-1) / 2`，即 `O(N^2)`。
对于 `N` 达到 5000 的数据规模，`N^2` 大约是 `2.5 * 10^7`，在 1 秒的时限内是可接受的。

**例题 2：最长递减子序列长度**

**题目描述**：
给定一个整数序列 `a`，求它的最长递减子序列的长度。

**输入格式**：
同例题 1。

**输出格式**：
同例题 1。

**示例**：
**输入**
```
8
10 22 9 33 21 50 41 60
```
**输出**
```
3
```
(例如，`[22, 21]`, `[10, 9]`, `[33, 21]` 是递减子序列。最长的是 `[22, 21]`, `[50, 41]`, `[33, 21]` 等，但 `[50, 41]` 长度为2。实际上，`[22, 21]` 长度为2。`[60, 50, 41]` 长度为3。`[33, 21]` 长度为2。`[10, 9]` 长度为2。最长的是 `[60, 50, 41]`。)

**思路分析**：

这个题目是LIS问题的变种，只是把“递增”条件改成了“递减”。
1.  **DP状态定义**：`dp[i]` 表示以 `a[i]` 结尾的最长递减子序列的长度。
2.  **DP初始化**：所有 `dp[i]` 都初始化为 1。
3.  **DP状态转移**：
    *   外层循环 `i` 从 `0` 到 `n-1`。
    *   内层循环 `j` 从 `0` 到 `i-1`。
    *   **关键改变**：如果 `a[i] < a[j]`，说明 `a[i]` 可以接在以 `a[j]` 结尾的递减子序列后面。
    *   更新：`dp[i] = max(dp[i], dp[j] + 1)`。
4.  **最终结果**：遍历 `dp` 数组，找到最大值。

**代码实现** (仅展示 `main` 函数内部变化)：

```cpp
#include <bits/stdc++.h>

const int MAXN = 5005;
int a[MAXN];
int dp[MAXN];

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    int maxL = 0; // 记录最长递减子序列的总长度

    for (int i = 0; i < n; ++i) {
        dp[i] = 1;
        for (int j = 0; j < i; ++j) {
            // 唯一改变的条件：从 a[i] > a[j] 变为 a[i] < a[j]
            if (a[i] < a[j]) { // 如果 a[i] 小于 a[j]，则可以构成递减关系
                dp[i] = std::max(dp[i], dp[j] + 1);
            }
        }
        maxL = std::max(maxL, dp[i]);
    }
    
    // 特殊情况：如果序列是空的，或者只有一个元素，maxL 初始为0，或更新为1。
    // 如果 n=0，maxL 初始为0，dp数组也不会被计算。
    // 如果 n=1，dp[0]会被设为1，maxL也会更新为1。
    // 所以，如果 n > 0 且 maxL 最终为 0，说明序列中所有元素都相同，或者无法形成递减子序列（如 [1,2,3]），但单个元素本身就是长度为1的递减子序列。
    // 实际上 maxL 至少为1（除非 n=0）。
    if (n == 0) {
        std::cout << 0 << std::endl;
    } else {
        std::cout << maxL << std::endl;
    }
    

    return 0;
}
```
**时间复杂度**：`O(N^2)`，与LIS相同。

---

**总结**：通过这两个例子，我们看到动态规划解决这类问题时，关键在于正确定义 `dp` 状态和找出状态转移方程。对于变种问题，通常只需要微调转移方程中的条件即可。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，如iostream, algorithm等

// 定义常量，用于数组大小，避免魔术数字，方便修改
const int MAX_N = 5005; // 根据题目 N 的最大值设定，通常会预留一些冗余

// 定义全局数组，GESP 6级不允许使用STL容器如vector
int a[MAX_N];  // 存储输入的序列元素
int dp[MAX_N]; // dp[i] 表示以 a[i] 结尾的最长递增子序列的长度

int main() {
    // 优化C++标准库输入输出流，使其与C风格的scanf/printf速度接近
    // std::ios_base::sync_with_stdio(false) 关闭C++流与C标准流的同步
    // std::cin.tie(NULL) 解除cin与cout的绑定，cin不再刷新cout
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 序列的长度
    std::cin >> n;

    // 1. 读取输入序列
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    // 初始化最长递增子序列的长度为0
    // 如果序列为空，则长度为0；如果序列有元素，至少是1（单个元素也是LIS）
    int maxLength = 0; 
    if (n > 0) { // 避免空序列时maxLength仍为0，但dp[0]会被初始化为1的逻辑不一致
        maxLength = 1; // 至少有一个元素，LIS长度至少为1
    }

    // 2. 动态规划核心逻辑
    // 外层循环：遍历序列中的每一个元素 a[i]
    // i 从 0 到 n-1
    for (int i = 0; i < n; ++i) {
        // 初始化 dp[i]
        // 以 a[i] 结尾的 LIS 至少包含 a[i] 自己，所以长度至少为 1
        dp[i] = 1; 

        // 内层循环：遍历 i 之前的所有元素 a[j]
        // j 从 0 到 i-1
        for (int j = 0; j < i; ++j) {
            // 判断条件：如果 a[i] 可以接在以 a[j] 结尾的 LIS 后面
            // 即 a[i] 必须严格大于 a[j]
            if (a[i] > a[j]) {
                // 状态转移：
                // 以 a[i] 结尾的 LIS 长度，
                // 可以是当前的 dp[i] (即不接任何前面的序列，只包含a[i]自己)，
                // 也可以是 (以 a[j] 结尾的 LIS 长度 + 1)
                // 我们取两者中的最大值
                dp[i] = std::max(dp[i], dp[j] + 1);
            }
        }
        
        // 每次计算完 dp[i]，都更新全局的最大长度
        // maxLength 存储的是 dp 数组中所有元素的最大值，也就是整个序列的 LIS 长度
        maxLength = std::max(maxLength, dp[i]);
    }

    // 3. 输出最终结果
    // 注意：如果 n=0，maxLength 应该输出0。我们初始化时考虑了这种情况。
    // 如果 n > 0，maxLength 至少为1。
    std::cout << maxLength << std::endl;

    return 0; // 程序正常结束
}

```

**代码解释**：

1.  **`#include <bits/stdc++.h>`**：这是一个非标准的头文件，但它包含了几乎所有常用的C++标准库头文件，在竞赛编程中非常方便。
2.  **`const int MAX_N = 5005;`**：定义一个常量 `MAX_N` 来表示数组的最大尺寸。这是一种良好的编程习惯，方便管理和修改。
3.  **`int a[MAX_N];` `int dp[MAX_N];`**：声明两个全局数组。`a` 用于存储输入的数字序列，`dp` 用于存储动态规划的状态。全局数组默认初始化为0，但我们会在代码中显式初始化 `dp[i]` 为 1。
4.  **`std::ios_base::sync_with_stdio(false); std::cin.tie(NULL);`**：这两行是C++竞赛编程中常用的优化，用于加速输入输出，减少程序运行时间。
5.  **`int n; std::cin >> n;`**：读取序列长度 `n`。
6.  **`for (int i = 0; i < n; ++i) { std::cin >> a[i]; }`**：循环读取 `n` 个整数到数组 `a` 中。
7.  **`int maxLength = 0; if (n > 0) { maxLength = 1; }`**：初始化 `maxLength`。如果序列非空，则LIS长度至少为1（任何一个元素本身就是长度为1的递增子序列）。
8.  **外层循环 `for (int i = 0; i < n; ++i)`**：这个循环遍历序列中的每一个元素 `a[i]`。对于每个 `a[i]`，我们都要计算以它结尾的最长递增子序列的长度 `dp[i]`。
9.  **`dp[i] = 1;`**：这是 `dp[i]` 的初始值。即使 `a[i]` 之前没有比它小的元素，它自己也能构成一个长度为 1 的递增子序列。
10. **内层循环 `for (int j = 0; j < i; ++j)`**：这个循环遍历 `a[i]` 之前的所有元素 `a[j]`。
11. **`if (a[i] > a[j])`**：这是状态转移的关键条件。如果 `a[i]` 大于 `a[j]`，说明 `a[i]` 可以接在以 `a[j]` 结尾的递增子序列后面，形成一个新的更长的递增子序列。
12. **`dp[i] = std::max(dp[i], dp[j] + 1);`**：如果满足条件，我们就用 `dp[j] + 1` 来更新 `dp[i]`。`std::max` 函数确保 `dp[i]` 总是存储能达到的最大长度。
13. **`maxLength = std::max(maxLength, dp[i]);`**：每次计算出一个 `dp[i]`，都将其与当前的 `maxLength` 进行比较，更新 `maxLength` 为所有 `dp[k]` 中的最大值。
14. **`std::cout << maxLength << std::endl;`**：输出最终计算出的最长递增子序列的长度。
