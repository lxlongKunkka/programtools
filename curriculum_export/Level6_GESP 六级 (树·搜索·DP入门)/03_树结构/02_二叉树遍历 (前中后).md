## 知识点讲解

### 1. 二叉树回顾

还记得二叉树是什么吗？
-   每个节点最多有两个孩子，通常称为**左孩子**和**右孩子**。
-   没有孩子的节点称为**叶子节点**。
-   最顶端的节点称为**根节点**。

### 2. 什么是二叉树遍历？

**遍历**就是指沿着某条搜索路线，对树中每个节点访问一次且仅访问一次。访问的目的是为了处理节点上的数据。

### 3. 三种经典的遍历方式

这三种遍历方式都基于一个核心思想：对于任何一个节点，我们都遵循“左子树 -> 根节点 -> 右子树”的相对访问顺序，只是**根节点**什么时候被访问决定了遍历的类型。

#### (1) 前序遍历 (Pre-order Traversal)

-   **访问顺序：** **根节点** -> 左子树 -> 右子树
-   **记忆口诀：** “根在前，先访问根！”
-   **特点：** 总是先处理当前节点，再处理它的左边和右边。
-   **应用场景：** 常用于复制二叉树、构造表达式树（波兰式）。

**例子：**
假设有这样一棵二叉树（节点值为字符）：

```
      A
     / \
    B   C
   / \   \
  D   E   F
```

前序遍历的顺序是：
1.  访问根节点 `A`。
2.  对 `A` 的左子树（以 `B` 为根）进行前序遍历：
    1.  访问根节点 `B`。
    2.  对 `B` 的左子树（以 `D` 为根）进行前序遍历：
        1.  访问根节点 `D`。
        2.  `D` 没有左子树和右子树，遍历结束。
    3.  对 `B` 的右子树（以 `E` 为根）进行前序遍历：
        1.  访问根节点 `E`。
        2.  `E` 没有左子树和右子树，遍历结束。
3.  对 `A` 的右子树（以 `C` 为根）进行前序遍历：
    1.  访问根节点 `C`。
    2.  `C` 没有左子树。
    3.  对 `C` 的右子树（以 `F` 为根）进行前序遍历：
        1.  访问根节点 `F`。
        2.  `F` 没有左子树和右子树，遍历结束。

最终序列：`A B D E C F`

#### (2) 中序遍历 (In-order Traversal)

-   **访问顺序：** 左子树 -> **根节点** -> 右子树
-   **记忆口诀：** “根在中，中间访问根！”
-   **特点：** 总是先处理左边的所有子孙，再处理自己，最后处理右边的所有子孙。
-   **应用场景：** 如果二叉树是一棵**二叉搜索树 (BST)**，那么中序遍历可以得到一个**有序序列**，这非常有用！

**例子：**
使用上面的二叉树：

```
      A
     / \
    B   C
   / \   \
  D   E   F
```

中序遍历的顺序是：
1.  对 `A` 的左子树（以 `B` 为根）进行中序遍历：
    1.  对 `B` 的左子树（以 `D` 为根）进行中序遍历：
        1.  `D` 没有左子树。
        2.  访问根节点 `D`。
        3.  `D` 没有右子树，遍历结束。
    2.  访问根节点 `B`。
    3.  对 `B` 的右子树（以 `E` 为根）进行中序遍历：
        1.  `E` 没有左子树。
        2.  访问根节点 `E`。
        3.  `E` 没有右子树，遍历结束。
2.  访问根节点 `A`。
3.  对 `A` 的右子树（以 `C` 为根）进行中序遍历：
    1.  `C` 没有左子树。
    2.  访问根节点 `C`。
    3.  对 `C` 的右子树（以 `F` 为根）进行中序遍历：
        1.  `F` 没有左子树。
        2.  访问根节点 `F`。
        3.  `F` 没有右子树，遍历结束。

最终序列：`D B E A C F`

#### (3) 后序遍历 (Post-order Traversal)

-   **访问顺序：** 左子树 -> 右子树 -> **根节点**
-   **记忆口诀：** “根在后，最后访问根！”
-   **特点：** 总是先处理左边和右边的所有子孙，最后才处理自己。
-   **应用场景：** 常用于删除二叉树（先删除孩子，再删除自己）、计算表达式树（逆波兰式）。

**例子：**
使用上面的二叉树：

```
      A
     / \
    B   C
   / \   \
  D   E   F
```

后序遍历的顺序是：
1.  对 `A` 的左子树（以 `B` 为根）进行后序遍历：
    1.  对 `B` 的左子树（以 `D` 为根）进行后序遍历：
        1.  `D` 没有左子树。
        2.  `D` 没有右子树。
        3.  访问根节点 `D`。
    2.  对 `B` 的右子树（以 `E` 为根）进行后序遍历：
        1.  `E` 没有左子树。
        2.  `E` 没有右子树。
        3.  访问根节点 `E`。
    3.  访问根节点 `B`。
2.  对 `A` 的右子树（以 `C` 为根）进行后序遍历：
    1.  `C` 没有左子树。
    2.  对 `C` 的右子树（以 `F` 为根）进行后序遍历：
        1.  `F` 没有左子树。
        2.  `F` 没有右子树。
        3.  访问根节点 `F`。
    3.  访问根节点 `C`。
3.  访问根节点 `A`。

最终序列：`D E B F C A`

### 4. 递归思想

你会发现，无论是前序、中序还是后序遍历，它们都遵循一个非常相似的模式：
-   如果当前节点为空，就什么也不做（这是递归的**终止条件**）。
-   否则，对当前节点进行**访问操作**，然后**递归**地对它的左子树进行相同的遍历，再**递归**地对它的右子树进行相同的遍历。
-   **访问操作**的位置（在递归调用左子树之前、之间还是之后）决定了是哪种遍历。

递归就像照镜子一样，一个大问题被分解成更小的、相同类型的问题，直到问题小到可以直接解决（比如遇到空节点）。

## 代码示例

现在，让我们用 C++ 语言来实现这三种遍历方法吧！

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件
using namespace std;     // 使用标准命名空间

// 定义二叉树的节点结构
struct Node {
    char v;       // 节点存储的值，这里用字符表示
    Node *l;      // 指向左孩子的指针
    Node *r;      // 指向右孩子的指针

    // 节点的构造函数，方便创建节点
    Node(char data) {
        v = data;
        l = nullptr; // 初始化左孩子为空
        r = nullptr; // 初始化右孩子为空
    }
};

// 前序遍历函数：根 -> 左 -> 右
void preOrder(Node* r) {
    // 递归的终止条件：如果当前节点为空，则直接返回
    if (r == nullptr) {
        return;
    }
    // 1. 访问根节点
    cout << r->v << " ";
    // 2. 递归遍历左子树
    preOrder(r->l);
    // 3. 递归遍历右子树
    preOrder(r->r);
}

// 中序遍历函数：左 -> 根 -> 右
void inOrder(Node* r) {
    // 递归的终止条件
    if (r == nullptr) {
        return;
    }
    // 1. 递归遍历左子树
    inOrder(r->l);
    // 2. 访问根节点
    cout << r->v << " ";
    // 3. 递归遍历右子树
    inOrder(r->r);
}

// 后序遍历函数：左 -> 右 -> 根
void postOrder(Node* r) {
    // 递归的终止条件
    if (r == nullptr) {
        return;
    }
    // 1. 递归遍历左子树
    postOrder(r->l);
    // 2. 递归遍历右子树
    postOrder(r->r);
    // 3. 访问根节点
    cout << r->v << " ";
}

int main() {
    // 构造我们示例中的二叉树：
    //       A
    //      / \
    //     B   C
    //    / \   \
    //   D   E   F

    Node* a = new Node('A'); // 创建根节点 A
    Node* b = new Node('B');
    Node* c = new Node('C');
    Node* d = new Node('D');
    Node* e = new Node('E');
    Node* f = new Node('F');

    // 建立节点之间的父子关系
    a->l = b; // A 的左孩子是 B
    a->r = c; // A 的右孩子是 C
    b->l = d; // B 的左孩子是 D
    b->r = e; // B 的右孩子是 E
    c->r = f; // C 的右孩子是 F (注意 C 没有左孩子)

    cout << "二叉树的前序遍历序列为: ";
    preOrder(a); // 从根节点 A 开始前序遍历
    cout << endl;

    cout << "二叉树的中序遍历序列为: ";
    inOrder(a); // 从根节点 A 开始中序遍历
    cout << endl;

    cout << "二叉树的后序遍历序列为: ";
    postOrder(a); // 从根节点 A 开始后序遍历
    cout << endl;

    // 在实际编程中，我们还需要释放动态分配的内存，
    // 但在信奥比赛或本节课中，我们主要关注算法逻辑，
    // 可以暂时不考虑复杂的内存释放。

    return 0;
}
```

运行上面的代码，你会看到输出结果与我们手动推导的序列完全一致！是不是很神奇呢？

### 编程题

**任务：**
编写一个 C++ 程序，实现以下功能：
1.  定义一个二叉树节点结构 `struct Node`。
2.  手动构建一棵至少包含 6 个节点的二叉树，节点值可以是整数或字符。
3.  实现 `preOrder`、`inOrder`、`postOrder` 三个函数，分别输出这棵二叉树的前序、中序、后序遍历序列。

**进阶挑战：**
尝试在你的二叉树中创建一个只有左孩子或只有右孩子的节点，看看你的遍历算法是否依然正确。

---

### 答案与解析

**题目一答案：A**
**解析：**
前序遍历顺序是：根 -> 左 -> 右。
1.  根节点 1。
2.  左子树（根为 2）：访问 2，再访问 2 的左孩子 4。
3.  右子树（根为 3）：访问 3，再访问 3 的右孩子 5。
所以序列是 1 2 4 3 5。

**题目二答案：B**
**解析：**
中序遍历顺序是：左 -> 根 -> 右。
1.  左子树（根为 2）：先访问 2 的左孩子 4，再访问 2。
2.  根节点 1。
3.  右子树（根为 3）：先访问 3，再访问 3 的右孩子 5。
所以序列是 4 2 1 3 5。

**题目三答案：C**
**解析：**
后序遍历顺序是：左 -> 右 -> 根。
1.  左子树（根为 2）：先访问 2 的左孩子 4，再访问 2。
2.  右子树（根为 3）：先访问 3 的右孩子 5，再访问 3。
3.  根节点 1。
所以序列是 4 2 5 3 1。

**题目四答案：C**
**解析：**
`if (r == nullptr)` 是递归的终止条件，意味着当前子树是空的，没有节点可访问，所以直接返回，结束当前的递归调用。

**题目五答案：C**
**解析：**
已知前序遍历和中序遍历序列，可以唯一确定一棵二叉树的结构。前序遍历告诉我们根节点是谁，中序遍历告诉我们哪些节点在根节点的左边，哪些在右边，结合这两个信息就可以重建树。如果只知道前序和后序，或者中序和后序，则不能唯一确定（因为无法区分左右子树的边界）。
