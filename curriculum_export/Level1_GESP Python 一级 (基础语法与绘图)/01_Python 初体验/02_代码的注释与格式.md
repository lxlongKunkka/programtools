## 知识点讲解

### 1. 什么是注释？为什么要用注释？

*   **定义:** 注释是写在代码中，但不会被 Python 解释器执行的文字。它们是给程序员（也就是我们自己和看我们代码的人）看的说明。
*   **作用:**
    *   **解释代码:** 告诉别人（或未来的自己）这段代码是做什么的，为什么这么做。
    *   **帮助理解:** 当代码比较复杂时，注释能帮助我们快速理解代码的逻辑和意图。
    *   **临时禁用代码:** 有时候我们想测试某段代码的效果，或者暂时不需要它，但又不想删除，就可以把它注释掉。
    *   **记录信息:** 比如作者、日期、版本等重要信息。
*   **重要性:** 好的注释能让你的代码“活”起来，变得更容易阅读、理解和维护。没有注释的代码，就像没有地图的藏宝图，很难找到宝藏哦！

### 2. Python 中的注释怎么写？

Python 提供了两种主要的注释方式：

#### (1) 单行注释：使用 `#` (井号)

*   `#` 后面的所有内容直到行尾，都会被视为注释。
*   通常用于解释单行代码，或者在代码行的末尾添加简短说明。

```python
# 这是一个单行注释的例子
a = 10  # 变量 a 被赋值为 10
print(a) # 打印变量 a 的值
```

#### (2) 多行注释（或文档字符串 Docstring）：使用三对引号 `'''` 或 `"""`

*   当我们需要写比较长的说明，跨越多行时，可以使用三对单引号 `'''` 或三对双引号 `"""`。
*   严格来说，Python 并没有官方的“多行注释”语法。但这种三引号字符串，如果它们不是表达式的一部分，Python 解释器会忽略它们，所以常被用来实现多行注释的效果。
*   当它们出现在函数、类或模块的开头时，它们会被特殊处理为“文档字符串”（Docstring），用来详细说明这些代码块的功能。

```python
'''
这是一个多行注释的例子。
它可以跨越多行，用来写比较长的说明。
通常用于文件开头、函数或类的详细描述。
'''

"""
你也可以使用三对双引号来写多行注释。
效果和三对单引号是一样的。
这通常取决于个人习惯或团队规范。
"""

def add_two_numbers(n1, n2):
    """
    这个函数接收两个数字 n1 和 n2，
    然后返回它们的和。
    这是一个函数的文档字符串。
    """
    s = n1 + n2
    return s
```

### 3. 什么是代码格式化？为什么要格式化？

*   **定义:** 代码格式化就是让代码看起来整齐、美观、有条理，就像书籍的排版一样。
*   **作用:**
    *   **提高可读性:** 整齐的代码更容易阅读和理解，能大大减少眼睛的疲劳。
    *   **减少错误:** 规范的格式有助于我们更容易发现一些潜在的逻辑错误或语法错误。
    *   **统一风格:** 特别是在团队协作时，统一的代码风格能让所有人的代码看起来像一个人写的，方便大家互相理解。
*   **重要性:** 想象一下看一本没有标点符号、没有分段、所有字都挤在一起的书，是不是很崩溃？代码也是一样！

### 4. Python 中的基本格式化规则 (Level 1 重点)

#### (1) 缩进 (Indentation)：Python 的灵魂！

*   **Python 对缩进有严格要求！** 它不仅仅是为了好看，更是**代码结构的一部分**。
*   在其他编程语言中，我们常用大括号 `{}` 来表示代码块（比如 `if` 语句、循环）。
*   **Python 用缩进（空出来的部分）来表示代码块的层次结构。**
*   通常使用 **4 个空格**作为一个缩进级别。
*   **重要提示：** 严禁混合使用空格和 Tab 键进行缩进！这会导致代码错误。大多数现代代码编辑器都可以设置将 Tab 键自动转换为 4 个空格。

```python
# 正确的缩进示例
if True:
    print("条件为真")  # 这一行比 if 语句多了一个缩进 (4个空格)
    print("继续执行")  # 这一行也需要和上一行相同的缩进
else:
    print("条件为假")  # 这一行也比 else 语句多了一个缩进
print("程序结束")      # 这一行与 if/else 同一级，没有缩进
```

#### (2) 空格 (Whitespace)：让代码呼吸

*   **运算符两边通常加空格:** 比如 `+`, `-`, `*`, `/`, `=` 等，这样看起来更清晰，不那么拥挤。
    *   推荐：`a = b + c`
    *   不推荐：`a=b+c`
*   **逗号 `,` 后面通常加一个空格:**
    *   推荐：`print(x, y)`
    *   不推荐：`print(x,y)`
*   这些小小的空格，能让你的代码看起来更舒服、更专业。

## 代码示例

同学们，我们来一起写一些代码，看看注释和格式化是怎么用的吧！

```python
# 这是一个计算两个数之和的简单程序
# 作者：[你的名字]
# 日期：2023年10月27日

'''
这个程序会接收用户输入的两个整数，
然后计算它们的和，并将结果打印出来。
它还包含一个简单的条件判断，演示缩进。
'''

# 步骤 1: 获取第一个数字
# input() 函数用来从键盘获取用户输入，返回的是字符串
# int() 函数将输入的字符串转换为整数类型
n1 = int(input("请输入第一个数字: "))

# 步骤 2: 获取第二个数字
n2 = int(input("请输入第二个数字: "))

# 步骤 3: 计算它们的和
s = n1 + n2  # 将 n1 和 n2 相加，结果存储在变量 s 中

# 步骤 4: 打印结果
# 使用 f-string 格式化输出，将变量 s 的值嵌入到字符串中
print(f"你输入的两个数字的和是: {s}")

# 这是一个条件判断的例子，展示缩进的重要性
if s > 10:
    # 如果和大于 10，就打印以下两行
    print("和大于 10")    # 注意这里的缩进，它属于 if 代码块
    print("太棒了！")      # 这一行也需要与上一行相同的缩进
else:
    # 如果和不大于 10，就打印这一行
    print("和不大于 10")  # 注意这里的缩进，它属于 else 代码块

# 这个 print 语句与 if/else 同一级，所以没有缩进
print("程序运行结束。")
```

    *   **示例1 (注释错误):**
        ```python
        # print("你好") # 这句话是打印“你好”的
        print("Hello")
        ```
        *   **问题分析:** 注释 `这句话是打印“你好”的` 与实际代码 `print("Hello")` 不匹配。注释应该准确反映代码的功能。

        ```python
        a = 10
        if a > 5:
        print("a 大于 5")
        ```

    *   **示例3 (格式错误 - 空格):**
        ```python
        x=1+2
        print(x,y)
        ```
        *   **问题分析:** `x=1+2` 中 `=` 和 `+` 运算符两边缺少空格；`print(x,y)` 中 `,` 后面缺少空格。

## 练习题目

**选择题**

1.  在 Python 中，以下哪个符号用于创建单行注释？
    A.  `//`
    B.  `/* */`
    C.  `#`
    D.  `--`

2.  以下哪种方式常用于在 Python 中创建多行注释或文档字符串？
    A.  `##`
    B.  `''' '''` 或 `""" """`
    C.  `// //`
    D.  `<!-- -->`

3.  Python 代码中，以下哪一项通常用来表示代码块（如 `if` 语句或循环体）的层次结构？
    A.  大括号 `{}`
    B.  小括号 `()`
    C.  冒号 `:`
    D.  缩进

4.  阅读下面的 Python 代码片段：
    ```python
    a = 10
    # b = 20  这是一个被注释掉的行
    print(a)
    ```
    这段代码运行时会输出什么？
    A.  `10`
        `20`
    B.  `10`
    C.  `a`
        `b`
    D.  什么也不输出

5.  以下哪段代码的格式化是符合 Python 规范的（主要考虑缩进和运算符空格）？
    A.  ```python
        if True:
        print("Hello")
        ```
    B.  ```python
        x=5+3
        print(x)
        ```
    C.  ```python
        n = 10
        if n > 5:
            print("n is greater")
        ```
    D.  ```python
        y = 7
        if y < 10 :
        print(y)
        ```

**填空题**

1.  Python 中，注释是写给 _________ 看的，不会被计算机 _________。
2.  为了让代码看起来更整齐，运算符（如 `+`, `-`, `=`）的两边通常会加上 _________。
3.  在 Python 中，一个标准的缩进通常是使用 _________ 个空格。
4.  如果你想临时禁用一行代码，但又不想删除它，最好的方法是给它添加一个 _________。

---

**答案与解析**

**选择题答案**

1.  **C. `#`**
    *   解析：`#` 是 Python 中用于单行注释的符号。A、B、D 是其他语言的注释方式。
2.  **B. `''' '''` 或 `""" """`**
    *   解析：在 Python 中，三对单引号或三对双引号常用于表示多行字符串，当它们独立存在时，可以作为多行注释使用，也常用于编写函数的文档字符串（Docstring）。
3.  **D. 缩进**
    *   解析：Python 使用缩进来表示代码块的逻辑结构，这是它与其他许多编程语言的一个显著区别。
4.  **B. `10`**
    *   解析：被 `#` 注释掉的行不会被执行，所以 `b = 20` 不会产生任何输出。程序只会执行 `a = 10` 和 `print(a)`，因此输出 `10`。
5.  **C.**
    *   解析：
        *   A 选项：`print("Hello")` 没有缩进，格式错误。
        *   B 选项：`x=5+3` 运算符两边缺少空格，可读性差。
        *   C 选项：`n = 10` 运算符有空格，`if` 语句后的 `print` 有正确缩进，符合规范。
        *   D 选项：`print(y)` 没有缩进，格式错误。

**填空题答案**

1.  **程序员/人**，**执行**
    *   解析：注释是为了提高代码可读性，方便程序员理解和维护，计算机在执行时会忽略它们。
2.  **空格**
    *   解析：在运算符两边添加空格可以提高代码的可读性，使其看起来不那么拥挤。
3.  **4**
    *   解析：PEP 8（Python 官方编码风格指南）推荐使用 4 个空格作为一个缩进级别。
4.  **注释**
    *   解析：将代码行注释掉是一种常见的临时禁用代码的方法，这样可以方便后续恢复或测试。
