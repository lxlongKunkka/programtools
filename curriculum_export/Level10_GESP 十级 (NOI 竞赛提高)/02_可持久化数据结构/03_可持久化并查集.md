这就是我们今天要学习的——**可持久化并查集**！它能帮助我们的记录员，把每一次“朋友圈”的变化都完整地记录下来，而且还能高效地查询任何一个历史版本！

## 深度知识点讲解

### 1. 什么是可持久化数据结构？——“时间机器”的秘密

想象一下，你有一本相册。每次拍了新照片，你可以：
1.  **直接替换旧照片**：那么旧的瞬间就永远消失了，你只能看到最新的。这就是普通数据结构。
2.  **在相册后面加一页，贴上新照片**：旧照片还在，新照片也有了。但是如果你的相册有1000页，每次只加一张照片，相册会变得巨大无比。
3.  **聪明地组织相册**：如果你只是修改了相册里的一张照片（比如P图），你没必要重新打印整个相册。你只需要打印那张新照片，然后把它替换掉，并记住这个“新版本”相册的索引。对于那些没有被修改的照片，你仍然可以指向旧相册里的同一张照片。

**可持久化数据结构**，就是第三种情况！它的核心思想是：
*   **不破坏旧版本**：每次修改操作都不会直接改变旧版本的数据结构。
*   **创建新版本**：每次修改都会生成一个新的数据结构版本。
*   **共享未修改部分**：新版本尽可能地与旧版本共享那些没有被修改的部分，而不是全部复制一份。这大大节省了空间。

这种“写时复制”（Copy-on-Write）的策略，是实现“时间机器”的关键。

### 2. 并查集（DSU）回顾与持久化的挑战

我们来快速回顾一下并查集：
*   **用途**: 维护一些不相交的集合，支持两个基本操作：
    *   `find(x)`: 查找元素 `x` 所属集合的代表元素（根）。
    *   `union(x, y)`: 合并元素 `x` 和 `y` 所在的两个集合。
*   **实现**: 通常用一个 `parent` 数组 `p[x]` 存储 `x` 的父节点。如果 `p[x] == x`，则 `x` 是根节点。
*   **优化**:
    *   **路径压缩**: `find` 过程中，把路径上的所有节点直接连接到根节点，加速后续查询。
    *   **按秩/大小合并**: `union` 过程中，将小树连接到大树上，避免树变得过高，维持树的平衡。

**持久化的挑战**:
*   **`union` 操作**: 会修改两个节点的 `parent` 值（一个节点的父节点变成另一个根，另一个根的 `rank` 可能增加）。这只修改了少量节点，看起来还好。
*   **`find` 操作的路径压缩**: 这是最大的麻烦！路径压缩会修改从 `x` 到根节点路径上的所有节点的 `parent` 值。如果路径很长，一次 `find` 可能修改 $O(\log N)$ 个节点。为了持久化，我们就需要复制这条路径上的 $O(\log N)$ 个节点，这会使得 `find` 操作变得非常慢且占用大量空间。

**解决方案**:
为了简化可持久化，我们通常会**放弃路径压缩**！只保留按秩（或按大小）合并。这样，`find` 操作的复杂度会变成树高 $O(\log N)$，但它不会修改任何节点。`union` 操作仍然只修改最多两个节点（一个子树的根的 `parent`，一个根的 `rank`）。

### 3. 如何持久化一个“数组”？——可持久化线段树

现在问题变成了：我们需要一个能持久化地存储 `parent` 数组和 `rank` 数组的数据结构。每次 `union` 操作，我们更新数组中的一两个位置，并得到一个新的数组版本。

最适合这个任务的，就是**可持久化线段树（Persistent Segment Tree）**！
*   **线段树回顾**: 一种树形数据结构，每个节点代表一个区间。叶子节点代表数组中的一个元素。
*   **可持久化线段树**: 它的神奇之处在于，每次修改（例如更新数组中的 `pos` 位置的值），它不会修改旧的节点。它会从根节点到 `pos` 对应的叶子节点，把路径上的节点全部复制一遍，形成一条新的路径，并更新新路径上的节点信息。未修改的子树则直接共享旧版本的节点。
    *   **更新操作**: 每次更新，会创建 $O(\log N)$ 个新节点。
    *   **查询操作**: 和普通线段树一样，在指定版本的根节点上进行查询，复杂度 $O(\log N)$。
    *   **空间复杂度**: 如果有 $M$ 次更新，总共会创建 $O(M \log N)$ 个节点。

### 4. 可持久化并查集的工作原理

现在，我们把可持久化线段树和并查集结合起来：
1.  **维护信息**: 我们用一棵可持久化线段树来维护两个数组：`parent` 数组和 `rank` 数组。线段树的每个叶子节点 `i` 存储 `(parent[i], rank[i])`。
2.  **版本管理**: 每进行一次 `union` 操作，我们就生成一个新的线段树版本。我们用一个 `roots` 数组来存储每个版本的线段树的根节点索引：`roots[k]` 代表第 `k` 次操作后的并查集状态。
3.  **初始状态**:
    *   构建一个初始的可持久化线段树 (`roots[0]`)。
    *   线段树的每个叶子节点 `i` 存储 `(i, 0)`，表示元素 `i` 的父节点是 `i` 自己，秩为 `0`。
4.  **`find_root(version_idx, x)` 操作**:
    *   在 `roots[version_idx]` 对应的线段树版本中，查询 `x` 的父节点 `p[x]`。
    *   如果 `p[x] == x`，则 `x` 就是根节点，返回 `x`。
    *   否则，递归地调用 `find_root(version_idx, p[x])`。
    *   **注意**: 这里没有路径压缩，所以 `find` 过程只会查询，不会修改任何节点。
5.  **`union_set(prev_version_idx, x, y)` 操作**:
    *   这是最核心的部分。它会根据 `prev_version_idx` 版本的并查集状态，合并 `x` 和 `y`，并创建一个新的版本。
    *   **步骤**:
        1.  在 `roots[prev_version_idx]` 版本中，通过 `find_root` 找到 `x` 的代表元 `rx` 和 `y` 的代表元 `ry`。
        2.  如果 `rx == ry`，说明 `x` 和 `y` 已经在同一个集合中，不需要合并。直接 `roots[current_version] = roots[prev_version_idx]`。
        3.  否则，需要合并。根据按秩合并原则：
            *   在 `roots[prev_version_idx]` 版本中，查询 `rx` 的秩 `rank_rx` 和 `ry` 的秩 `rank_ry`。
            *   如果 `rank_rx < rank_ry`，则将 `rx` 挂到 `ry` 下面。需要更新 `parent[rx] = ry`。
            *   如果 `rank_ry < rank_rx`，则将 `ry` 挂到 `rx` 下面。需要更新 `parent[ry] = rx`。
            *   如果 `rank_rx == rank_ry`，则将 `ry` 挂到 `rx` 下面（或反之），并更新 `rank[rx]++`。
        4.  这些更新操作通过调用可持久化线段树的 `update` 函数来实现。每次 `update` 会返回一个新的线段树根节点索引。
            *   例如，要更新 `parent[rx] = ry`，就调用 `update(roots[prev_version_idx], rx, ry_parent_value, rank_rx_value)`。
            *   如果 `rank[rx]` 也需要更新，就再调用一次 `update`。
        5.  最终，得到一个新的线段树根节点，作为 `roots[current_version]`。

### 5. 复杂度分析

*   **时间复杂度**:
    *   `find_root` 操作: 因为没有路径压缩，每次查找根需要沿着父节点一直向上，最坏情况下树高为 $O(N)$（不使用按秩合并），但使用了按秩合并后，树高为 $O(\log N)$。在线段树上查询每个 `parent` 值都是 $O(\log N)$，所以 `find_root` 的复杂度是 $O(\log N \cdot \text{树高})$。实际上，`find_root` 是递归查询 `parent`，每次查询 `parent` 是 $O(\log N)$ (线段树查询)，递归深度是树高 $O(\log N)$ (并查集树高)，所以总复杂度是 $O(\log^2 N)$。
    *   `union_set` 操作: 包含两次 `find_root` ($O(\log^2 N)$)，以及最多三次线段树的 `update` 操作（修改一个 `parent`，可能修改一个 `rank`）。每次 `update` 都是 $O(\log N)$。所以 `union_set` 的总复杂度是 $O(\log^2 N)$。
    *   **优化**: `find_root` 实际上可以优化到 $O(\log N)$。因为我们每次在线段树上查询一个 `p[x]`，然后用 `p[x]` 作为新的 `x` 继续查询。这只是 `logN` 次线段树的单点查询，每次单点查询是 `logN`，所以是 `logN * logN`。但如果我们将 `parent` 和 `rank` 存储在同一个线段树节点中，那么一次 `query` 可以同时获取两者。而 `find_root` 本身只是沿着并查集的树结构走，每次走一步都需要通过线段树查询 `parent`。所以 `find_root` 的次数是 `logN`，每次查询是 `logN`，总共 `log^2N`。这对于 `N=10^5` 来说是 $10^5 \cdot (17)^2 \approx 3 \cdot 10^7$，可能有点慢。
    *   **更常见的实现**: 有时会用“可持久化数组”来描述，它不是线段树，而是基于树状结构（类似Trie）的数组。或者，直接用一个结构体 `Node { int p, rk; }` 存储在叶子节点，线段树只管索引。这样 `find` 操作的 $O(\log N)$ 是并查集树高，每次查询 `parent` 只需要 $O(\log N)$ 的线段树查询。所以总的 `find` 是 $O(\log N)$ * 并查集树高 $O(\log N)$。
    *   **总结**: 通常认为单次操作的时间复杂度为 $O(\log N)$ (如果并查集树高是 $O(\log N)$，且线段树查询是 $O(\log N)$，那么 `find_root` 就是 $O(\log N)$，`union` 也是 $O(\log N)$)。更严谨地，如果并查集树高是 $H$，线段树查询是 $O(\log N)$，那么 `find_root` 是 $O(H \log N)$，`union` 是 $O(H \log N + \log N)$。由于按秩合并，`H` 是 $O(\log N)$，所以是 $O(\log^2 N)$。在竞赛中，当 $N$ 很大时，`log^2 N` 可能会超时，但对于 $N \le 10^5$ 还是可以接受的。

*   **空间复杂度**:
    *   初始构建线段树需要 $O(N)$ 节点。
    *   每次 `union` 操作创建 $O(\log N)$ 个新的线段树节点。
    *   如果有 $M$ 次操作，总空间复杂度为 $O(N + M \log N)$。对于 $N, M \approx 10^5$，$\log N \approx 17$，那么节点总数是 $10^5 + 10^5 \times 17 \approx 1.8 \times 10^6$ 个节点。每个节点需要存储左右子节点索引、`parent` 值、`rank` 值，大约 4-5 个 `int`。总空间是 $1.8 \times 10^6 \times 4 \times 4 \text{ bytes} \approx 28 \text{MB}$，是可接受的。

### 深入思考：为什么放弃路径压缩？

路径压缩的目的是为了让下一次 `find` 更快，它直接修改了路径上的所有节点的 `parent`。
在可持久化数据结构中：
*   如果 `find` 时进行路径压缩，那么路径上的每个被修改的节点都需要被“复制”一份，形成新的节点。一条长度为 $H$ 的路径，需要复制 $H$ 个节点。
*   每次 `find` 都会产生 $O(H)$ 的空间开销，并且时间开销也是 $O(H \log N)$ (如果每次复制都需要线段树操作)。
*   这会使得 `find` 操作变得非常昂贵，不仅慢，而且空间爆炸。
*   因此，我们宁愿让 `find` 慢一点 ($O(H \log N)$，其中 $H=O(\log N)$)，但它不修改任何历史版本，从而避免了巨大的开销。

## 典型例题精讲

**例题：历史的足迹**

小明在玩一个策略游戏，游戏中有 $N$ 个不同的单位，编号从 $1$ 到 $N$。游戏过程中会发生 $M$ 次事件。每次事件可能是以下两种类型之一：
1.  **合并操作**: `1 a b`，表示单位 `a` 和单位 `b` 所在的队伍合并。
2.  **查询操作**: `2 k a b`，表示查询在第 `k` 次事件发生后（即第 `k` 个版本），单位 `a` 和单位 `b` 是否属于同一个队伍。

请你实现一个数据结构来处理这些操作。初始状态下，每个单位都独立属于自己的队伍。

**输入格式**:
第一行两个整数 $N, M$ ($1 \le N, M \le 10^5$)。
接下来 $M$ 行，每行表示一个操作。

**输出格式**:
对于每个查询操作，如果 `a` 和 `b` 属于同一个队伍，输出 `Yes`，否则输出 `No`。

**思路分析**:

这道题是典型的可持久化并查集应用。我们需要维护 $N$ 个元素的并查集，并能够查询任意历史版本。

1.  **维护数据**:
    *   我们需要一个可持久化线段树来存储 `parent` 数组和 `rank` 数组。
    *   线段树的每个叶子节点 `i` 存储 `parent[i]` 和 `rank[i]`。
    *   我们可以将 `parent` 和 `rank` 存储在一个结构体中，或者在线段树节点中存储两个值。为了简化，我们可以在线段树节点中直接存储 `parent` 值和 `rank` 值。

2.  **线段树结构**:
    *   每个线段树节点 `Node` 包含 `l_idx` (左子节点索引), `r_idx` (右子节点索引), `p_val` (当前节点代表的元素的父节点值), `rk_val` (当前节点代表的元素的秩值)。
    *   `nodes` 数组存储所有线段树节点。
    *   `roots` 数组存储每个操作版本对应的线段树根节点索引。`roots[0]` 是初始版本。

3.  **操作实现**:
    *   **`build(l, r)`**: 构建初始线段树。每个叶子节点 `i` 的 `p_val` 为 `i`，`rk_val` 为 `0`。
    *   **`update(prev_root_idx, l, r, pos, new_p, new_rk)`**: 在 `prev_root_idx` 版本上，更新 `pos` 位置的 `parent` 和 `rank`，返回新根的索引。这是一个标准的线段树单点更新操作，但每次复制路径上的节点。
    *   **`query(root_idx, l, r, pos)`**: 在 `root_idx` 版本上，查询 `pos` 位置的 `parent` 和 `rank`。
    *   **`find_root(root_idx, x)`**: 在 `root_idx` 版本的并查集中查找 `x` 的代表元。
        *   查询 `x` 的 `parent` 值 `p_x`。
        *   如果 `p_x == x`，则 `x` 是根，返回 `x`。
        *   否则，递归调用 `find_root(root_idx, p_x)`。
    *   **`union_set(prev_root_idx, x, y)`**: 合并 `x` 和 `y`。
        1.  调用 `find_root(prev_root_idx, x)` 得到 `rx`。
        2.  调用 `find_root(prev_root_idx, y)` 得到 `ry`。
        3.  如果 `rx == ry`，则它们已在同一集合，返回 `prev_root_idx`。
        4.  否则，查询 `rx` 的 `rank` `rank_rx` 和 `ry` 的 `rank` `rank_ry`。
        5.  按秩合并：
            *   如果 `rank_rx > rank_ry`，则将 `ry` 挂到 `rx` 下。新版本根 `cur_root` = `update(prev_root_idx, ry, rx_parent_value, rank_ry_value)` (更新 `ry` 的父节点为 `rx`)。
            *   如果 `rank_ry > rank_rx`，则将 `rx` 挂到 `ry` 下。新版本根 `cur_root` = `update(prev_root_idx, rx, ry_parent_value, rank_rx_value)`。
            *   如果 `rank_rx == rank_ry`，则将 `ry` 挂到 `rx` 下，并增加 `rx` 的秩。
                *   `cur_root` = `update(prev_root_idx, ry, rx_parent_value, rank_ry_value)` (更新 `ry` 的父节点为 `rx`)。
                *   `cur_root` = `update(cur_root, rx, rx_parent_value, rank_rx_value + 1)` (更新 `rx` 的秩)。
        6.  返回 `cur_root`。

**复杂度分析**:
*   时间复杂度: `find_root` $O(\log^2 N)$，`union_set` $O(\log^2 N)$。
*   空间复杂度: $O(N + M \log N)$。

**C++ 代码示例 (简化版，将 parent 和 rank 存储在叶子节点的数据中)**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;
const int MAXM = 100005;
const int MAX_NODES = MAXN + MAXM * 20; // N for initial + M * logN * 2 (for parent/rank updates)

struct Node {
    int l;      // 左子节点索引
    int r;      // 右子节点索引
    int p_val;  // 父节点值
    int rk_val; // 秩值
};

Node nodes[MAX_NODES];
int roots[MAXM + 1]; // roots[i] 存储第 i 次操作后的线段树根节点
int node_cnt;

// 构建初始线段树
// l, r: 当前节点代表的区间 [l, r]
// 返回新节点的索引
int build(int l, int r) {
    int k = ++node_cnt;
    if (l == r) {
        nodes[k].p_val = l; // 初始时，每个元素的父节点是自己
        nodes[k].rk_val = 0; // 初始时，秩为0
        return k;
    }
    int mid = (l + r) / 2;
    nodes[k].l = build(l, mid);
    nodes[k].r = build(mid + 1, r);
    return k;
}

// 在指定根 root_idx 的线段树中，更新 pos 位置的值
// l, r: 当前节点代表的区间 [l, r]
// pos: 要更新的位置
// new_p: pos 位置的新父节点值
// new_rk: pos 位置的新秩值
// 返回新根的索引
int update(int prev_root_idx, int l, int r, int pos, int new_p, int new_rk) {
    int k = ++node_cnt;
    nodes[k] = nodes[prev_root_idx]; // 复制旧节点信息
    
    if (l == r) {
        nodes[k].p_val = new_p;
        nodes[k].rk_val = new_rk;
        return k;
    }

    int mid = (l + r) / 2;
    if (pos <= mid) {
        nodes[k].l = update(nodes[prev_root_idx].l, l, mid, pos, new_p, new_rk);
    } else {
        nodes[k].r = update(nodes[prev_root_idx].r, mid + 1, r, pos, new_p, new_rk);
    }
    return k;
}

// 在指定根 root_idx 的线段树中，查询 pos 位置的父节点值
// l, r: 当前节点代表的区间 [l, r]
// pos: 要查询的位置
// 返回 pos 位置的父节点值
int query_parent(int root_idx, int l, int r, int pos) {
    if (l == r) {
        return nodes[root_idx].p_val;
    }
    int mid = (l + r) / 2;
    if (pos <= mid) {
        return query_parent(nodes[root_idx].l, l, mid, pos);
    } else {
        return query_parent(nodes[root_idx].r, mid + 1, r, pos);
    }
}

// 在指定根 root_idx 的线段树中，查询 pos 位置的秩值
// l, r: 当前节点代表的区间 [l, r]
// pos: 要查询的位置
// 返回 pos 位置的秩值
int query_rank(int root_idx, int l, int r, int pos) {
    if (l == r) {
        return nodes[root_idx].rk_val;
    }
    int mid = (l + r) / 2;
    if (pos <= mid) {
        return query_rank(nodes[root_idx].l, l, mid, pos);
    } else {
        return query_rank(nodes[root_idx].r, mid + 1, r, pos);
    }
}

// 在指定版本 root_idx 的并查集中查找 x 的代表元
// N_val: 元素总数
int find_root(int root_idx, int x, int N_val) {
    int p_x = query_parent(root_idx, 1, N_val, x);
    if (p_x == x) {
        return x;
    }
    return find_root(root_idx, p_x, N_val); // 没有路径压缩
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    roots[0] = build(1, n); // 构建初始版本

    for (int i = 1; i <= m; ++i) {
        int op, a, b, k;
        std::cin >> op;
        if (op == 1) { // 合并操作
            std::cin >> a >> b;
            int rx = find_root(roots[i-1], a, n);
            int ry = find_root(roots[i-1], b, n);

            if (rx == ry) {
                roots[i] = roots[i-1]; // 已经在同一个集合，版本不变
            } else {
                int rank_rx = query_rank(roots[i-1], 1, n, rx);
                int rank_ry = query_rank(roots[i-1], 1, n, ry);

                if (rank_rx < rank_ry) { // rx 挂到 ry 下
                    roots[i] = update(roots[i-1], 1, n, rx, ry, rank_rx);
                } else if (rank_ry < rank_rx) { // ry 挂到 rx 下
                    roots[i] = update(roots[i-1], 1, n, ry, rx, rank_ry);
                } else { // 秩相等，ry 挂到 rx 下，rx 秩增加
                    int temp_root = update(roots[i-1], 1, n, ry, rx, rank_ry); // 更新 ry 的父节点
                    roots[i] = update(temp_root, 1, n, rx, rx, rank_rx + 1); // 更新 rx 的秩
                }
            }
        } else { // 查询操作
            std::cin >> k >> a >> b;
            int rx = find_root(roots[k], a, n);
            int ry = find_root(roots[k], b, n);
            if (rx == ry) {
                std::cout << "Yes\n";
            } else {
                std::cout << "No\n";
            }
            roots[i] = roots[i-1]; // 查询操作不改变并查集状态，新版本沿用旧版本
        }
    }

    return 0;
}

## 代码实现模板

这里提供一个更清晰的模板，包含 `Node` 结构体和核心函数的实现。

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

// 常量定义，根据题目数据范围调整
const int MAX_ELEMENTS = 100005; // 并查集元素的最大数量 N
const int MAX_OPERATIONS = 100005; // 操作的最大数量 M

// 线段树节点的最大数量：
// 初始建树需要大约 2*N 个节点 (满二叉树叶子N, 内部N-1)
// 每次更新会创建 logN 个新节点
// 所以总节点数约为 MAX_ELEMENTS * 2 + MAX_OPERATIONS * log2(MAX_ELEMENTS) * 2 (因为可能更新parent和rank)
// log2(10^5) 约等于 17
// 10^5 * 2 + 10^5 * 17 * 2 = 20万 + 340万 = 360万
const int MAX_SEG_NODES = MAX_ELEMENTS * 2 + MAX_OPERATIONS * 40; 

// 线段树节点结构体
struct Node {
    int l_child_idx;    // 左子节点的索引
    int r_child_idx;    // 右子节点的索引
    int parent_val;     // 当前区间代表元素的父节点值 (如果为叶子节点)
    int rank_val;       // 当前区间代表元素的秩值 (如果为叶子节点)
};

Node seg_nodes[MAX_SEG_NODES]; // 所有线段树节点存储在这里
int root_versions[MAX_OPERATIONS + 1]; // 存储每个操作版本对应的线段树根节点索引
int current_node_count; // 当前已使用的线段树节点数量

// 1. 初始化线段树 (构建第0个版本)
// l, r: 当前线段树节点表示的区间范围 [l, r]
// 返回新创建的根节点的索引
int build_initial_tree(int l, int r) {
    int k = ++current_node_count; // 分配一个新的节点
    if (l == r) { // 叶子节点，存储并查集元素信息
        seg_nodes[k].parent_val = l; // 初始时，每个元素是自己的父节点
        seg_nodes[k].rank_val = 0;   // 初始时，秩为0
        return k;
    }
    int mid = (l + r) / 2;
    seg_nodes[k].l_child_idx = build_initial_tree(l, mid);
    seg_nodes[k].r_child_idx = build_initial_tree(mid + 1, r);
    return k;
}

// 2. 更新线段树 (创建新版本)
// prev_root_idx: 上一个版本的根节点索引
// l, r: 当前线段树节点表示的区间范围 [l, r]
// target_pos: 要更新的元素位置
// new_parent: target_pos 的新父节点值
// new_rank: target_pos 的新秩值
// 返回新版本的根节点索引
int update_tree(int prev_root_idx, int l, int r, int target_pos, int new_parent, int new_rank) {
    int k = ++current_node_count; // 分配一个新的节点
    seg_nodes[k] = seg_nodes[prev_root_idx]; // 复制上一个版本节点的通用信息

    if (l == r) { // 叶子节点，直接更新值
        seg_nodes[k].parent_val = new_parent;
        seg_nodes[k].rank_val = new_rank;
        return k;
    }

    int mid = (l + r) / 2;
    if (target_pos <= mid) {
        // 更新左子树，复制左子节点并递归
        seg_nodes[k].l_child_idx = update_tree(seg_nodes[prev_root_idx].l_child_idx, l, mid, target_pos, new_parent, new_rank);
    } else {
        // 更新右子树，复制右子节点并递归
        seg_nodes[k].r_child_idx = update_tree(seg_nodes[prev_root_idx].r_child_idx, mid + 1, r, target_pos, new_parent, new_rank);
    }
    return k;
}

// 3. 查询线段树中的某个元素值 (在指定版本中)
// current_root_idx: 当前查询的线段树根节点索引 (某个历史版本)
// l, r: 当前线段树节点表示的区间范围 [l, r]
// target_pos: 要查询的元素位置
// is_query_parent: true表示查询parent值，false表示查询rank值
// 返回 target_pos 对应的 parent_val 或 rank_val
int query_val(int current_root_idx, int l, int r, int target_pos, bool is_query_parent) {
    if (l == r) { // 叶子节点，直接返回对应值
        return is_query_parent ? seg_nodes[current_root_idx].parent_val : seg_nodes[current_root_idx].rank_val;
    }
    int mid = (l + r) / 2;
    if (target_pos <= mid) {
        return query_val(seg_nodes[current_root_idx].l_child_idx, l, mid, target_pos, is_query_parent);
    } else {
        return query_val(seg_nodes[current_root_idx].r_child_idx, mid + 1, r, target_pos, is_query_parent);
    }
}

// 4. 在指定版本的并查集中查找元素的代表元
// version_root_idx: 要查询的并查集版本对应的线段树根节点索引
// x: 要查找代表元的元素
// max_elements_count: 并查集中的元素总数 (范围 [1, max_elements_count])
// 返回 x 的代表元
int find_root_in_version(int version_root_idx, int x, int max_elements_count) {
    // 查询 x 的父节点
    int p_x = query_val(version_root_idx, 1, max_elements_count, x, true);
    if (p_x == x) { // 如果父节点是自己，则 x 是根节点
        return x;
    }
    // 否则，递归向上查找，没有路径压缩
    return find_root_in_version(version_root_idx, p_x, max_elements_count);
}

    std::cin >> n >> m;

    // 初始化第0个版本
    root_versions[0] = build_initial_tree(1
