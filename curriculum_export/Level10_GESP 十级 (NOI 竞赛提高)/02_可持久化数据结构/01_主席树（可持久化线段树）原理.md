## 深度知识点讲解

### 1. 线段树：我们的老朋友

在深入主席树之前，我们先快速回顾一下线段树。线段树是一种二叉树，用于在区间上进行高效的查询和修改。每个节点代表一个区间，叶子节点代表单个元素，非叶子节点存储其子节点区间的某种聚合信息（比如和、最大值等）。
*   **特点：** 单点修改或区间修改 `O(log N)`，区间查询 `O(log N)`。
*   **局限：** 每次修改都会直接改变树的结构或节点值。如果你想查询“修改前”的数据状态，普通线段树就无能为力了。它没有“历史版本”的概念。

### 2. 可持久化：让数据结构拥有“记忆”

“可持久化”（Persistent）是什么意思？
简单来说，一个数据结构是可持久化的，意味着它的每一次修改操作都不会破坏原有的数据结构，而是会创建一个新的版本，同时保留所有旧版本。我们可以随时访问任何一个历史版本。

**朴素的可持久化：**
最直接的想法是，每次修改时，把整个数据结构复制一份。
*   **优点：** 简单粗暴，确实保留了历史版本。
*   **缺点：** 空间复杂度爆炸！如果对一个有 `N` 个元素的线段树进行 `M` 次修改，每次都复制一份，总空间将达到 `O(M * N)`，时间也会是 `O(M * N)`。这显然是不可接受的。

**核心思想：共享节点（Sharing Nodes）**
既然大部分内容是相同的，我们为什么不共享它们呢？
想象一下，你有一本电子书，你每次修改它，软件都不会直接覆盖旧文件，而是创建一个新的“版本”。但是，如果只修改了其中一页，难道要把整本书都复制一份吗？当然不！聪明的设计会只复制被修改的那一页，并且让新版本指向旧版本中那些未被修改的页面。

这正是可持久化的精髓：**只复制被修改的路径上的节点，未被修改的节点则与旧版本共享。**

### 3. 主席树（Persistent Segment Tree）的诞生：区间第K小的利器

主席树，又称可持久化线段树，就是将线段树进行可持久化改造的产物。它最经典的用途是解决“静态区间第K小”问题。

**主席树解决静态区间第K小问题的基本思路：**

1.  **离散化 (Discretization)：**
    *   **为什么需要？** 静态区间第K小问题通常涉及的数值范围非常大（例如 `1` 到 `10^9`），而我们关注的往往是这些数值的相对大小关系，而不是绝对大小。如果直接以数值作为线段树的区间，会导致线段树的区间范围过大，节点数过多。
    *   **怎么做？** 将所有可能出现的数值（例如输入数组 `a` 中的所有元素）收集起来，去重，排序。然后，将原始数值映射到它们在排序后数组中的下标。这样，一个值域很大的问题就被转化成了值域较小（`1` 到 `N`）的问题。
    *   例如：`[100, 1, 50, 1000]` 离散化后可能变为 `[3, 1, 2, 4]` (假设 `1` -> `1`, `50` -> `2`, `100` -> `3`, `1000` -> `4`)。

2.  **构建主席树：版本累积与共享**
    *   主席树不是一次性建好一棵树，而是通过一系列单点更新操作，逐步构建出 `N` 个版本的线段树。
    *   我们定义 `root[i]` 为一个线段树的根节点，它代表了原数组 `a` 中前 `i` 个元素构成的“统计信息”。
    *   `root[0]`：一个空的线段树，所有节点计数为0。
    *   `root[i]`：在 `root[i-1]` 的基础上，插入第 `i` 个元素 `a[i]`（离散化后的值）。
        *   **关键：** 当我们在 `root[i-1]` 上插入 `a[i]` 时，沿着 `a[i]` 对应的路径向下走，这条路径上的所有节点都需要创建新的副本。而其他未受影响的节点，则直接指向 `root[i-1]` 中对应的节点。
        *   每个节点存储的是它所代表区间内元素出现的**次数**。

    **具体构建过程 (以插入 `pos` 为例)：**
    ```cpp
    // update(pre_node_idx, current_range_left, current_range_right, value_to_insert_pos)
    int update(int pre, int tl, int tr, int pos) {
        int cur = ++cnt; // 创建新节点
        tree[cur] = tree[pre]; // 复制旧节点信息
        tree[cur].sum++; // 计数增加

        if (tl == tr) { // 到达叶子节点
            return cur;
        }

        int mid = (tl + tr) / 2;
        if (pos <= mid) { // 插入到左子树
            tree[cur].l = update(tree[pre].l, tl, mid, pos); // 新的左子树
            // 右子树不变，直接指向旧节点的右子树
        } else { // 插入到右子树
            tree[cur].r = update(tree[pre].r, mid + 1, tr, pos); // 新的右子树
            // 左子树不变，直接指向旧节点的左子树
        }
        return cur;
    }
    ```
    *   `root[i]` 就是 `update(root[i-1], 1, max_val, a[i])` 的结果。
    *   每次 `update` 操作都会创建 `O(log N)` 个新节点。因此，构建 `N` 个版本的主席树总共需要 `O(N log N)` 的空间和时间。

3.  **查询：利用版本差分**
    *   现在我们有了 `N` 个版本的主席树，`root[i]` 代表了前 `i` 个元素的信息。
    *   如何查询区间 `[L, R]` 内的第K小值？
    *   **核心思想：** `root[R]` 记录了 `a[1...R]` 中所有元素的统计信息，`root[L-1]` 记录了 `a[1...L-1]` 中所有元素的统计信息。
    *   那么，`root[R]` 和 `root[L-1]` 的“差值”，就代表了区间 `[L, R]` 内元素的统计信息！
    *   具体来说，对于线段树的某个区间 `[tl, tr]`，它在 `root[R]` 中的计数是 `tree[root[R]].sum`，在 `root[L-1]` 中的计数是 `tree[root[L-1]].sum`。
    *   所以，在区间 `[L, R]` 内，`[tl, tr]` 这个子区间中元素的总数就是 `tree[root[R]].sum - tree[root[L-1]].sum`。

    **查询第K小值的过程 `query(u, v, tl, tr, k)`：**
    其中 `u` 是 `root[R]` 中的对应节点，`v` 是 `root[L-1]` 中的对应节点。
    ```cpp
    // query(node_R, node_L_minus_1, current_range_left, current_range_right, k)
    int query(int u, int v, int tl, int tr, int k) {
        if (tl == tr) { // 到达叶子节点，找到K小值
            return tl;
        }

        int mid = (tl + tr) / 2;
        // 计算左子树区间 [tl, mid] 在原区间 [L, R] 中的元素个数
        int count_left = tree[tree[u].l].sum - tree[tree[v].l].sum;

        if (k <= count_left) { // 如果K在左子树中
            return query(tree[u].l, tree[v].l, tl, mid, k);
        } else { // 如果K在右子树中
            return query(tree[u].r, tree[v].r, mid + 1, tr, k - count_left);
        }
    }
    ```
    *   每次 `query` 操作都在树上走一条路径，因此时间复杂度是 `O(log N)`。

### 4. 常见误区与注意点：

*   **离散化是关键：** 大多数主席树问题都需要离散化。忘记离散化会导致值域过大，数组越界或内存超限。
*   **节点池：** 由于每次更新都会创建新节点，需要一个足够大的节点数组来存储。通常是 `MAXN * LOGN`。
*   **根节点数组：** `root[i]` 存储的是第 `i` 个版本的根节点索引，而不是整个树。
*   **`root[0]` 的作用：** 作为所有空树的基准，方便后续更新。
*   **`update` 和 `query` 的参数：** 仔细区分是旧节点索引还是新节点索引，是当前线段树的区间还是查询的原始区间。

### 5. 时间空间复杂度总结：

*   **离散化：** `O(N log N)` (排序) 或 `O(N)` (哈希表，但竞赛中常直接排序去重)。
*   **建树：** `N` 次更新，每次更新 `O(log N)` 个节点。总时间 `O(N log N)`。
*   **查询：** 每次查询 `O(log N)`。
*   **空间：** `N` 次更新，每次创建 `O(log N)` 个节点。总空间 `O(N log N)`。
    *   其中 `N` 是原始数组的长度，`log N` 是线段树的深度。例如，如果 `N = 10^5`，`log N` 大约是 `17`。那么节点总数约为 `10^5 * 17`，大概 `1.7 * 10^6` 个节点。每个节点存储 `l, r, sum` 三个 `int`，内存消耗在可接受范围内。

## 典型例题精讲

**例题：静态区间第K小**

**题目描述：**
给定一个包含 N 个整数的序列 `a[1...N]`。有 `M` 次查询，每次查询给定 `L, R, K`，要求输出区间 `[L, R]` 中第 `K` 小的数。

**输入格式：**
第一行两个整数 `N, M`。
第二行 `N` 个整数，表示序列 `a`。
接下来 `M` 行，每行三个整数 `L, R, K`。

**输出格式：**
对于每次查询，输出一行一个整数，表示答案。

**示例输入：**
```
5 3
1 5 2 6 3
1 5 3
2 4 1
3 5 2
```

**示例输出：**
```
3
2
5
```

---

**思路分析：**

1.  **离散化：**
    *   将原始序列 `a` 中的所有元素复制到一个辅助数组 `b` 中。
    *   对 `b` 进行排序并去重，得到所有出现过的不同数值。
    *   建立一个映射关系：原始值 -> 离散化后的下标。例如，排序后的 `b` 中第 `i` 个元素，其离散化后的下标就是 `i`。
    *   将原始序列 `a` 中的每个元素替换为其离散化后的下标。
    *   记录离散化后的最大值 `max_val`，作为线段树的区间上限。

2.  **构建主席树：**
    *   创建一个空的根节点 `root[0]`。
    *   循环 `i` 从 `1` 到 `N`：
        *   `root[i] = update(root[i-1], 1, max_val, a[i] 离散化后的值);`
        *   `update` 函数会沿着路径创建 `O(log max_val)` 个新节点，并更新它们的 `sum` 值。

3.  **处理查询：**
    *   对于每个查询 `(L, R, K)`：
        *   调用 `query(root[R], root[L-1], 1, max_val, K)`。
        *   `query` 函数会比较左右子树在 `root[R]` 和 `root[L-1]` 上的元素数量差值。
        *   根据 `K` 的大小，递归进入左子树或右子树。
        *   当到达叶子节点时，返回该叶子节点对应的离散化下标。
        *   将这个离散化下标再映射回原始值（从排序去重后的 `b` 数组中取值），即为答案。

---

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// Allman 风格大括号

const int MAXN = 100005; // 原始数组最大长度
const int LOGN = 18;     // N=10^5, log2(10^5) 约等于 16.6，所以取 18 或 20
const int MAX_NODES = MAXN * LOGN; // 主席树节点总数

struct Node {
    int l, r;    // 左右子节点在 tree 数组中的索引
    int sum;     // 当前节点代表区间内元素的数量
} tree[MAX_NODES];

int root[MAXN]; // root[i] 存储第 i 个版本（即处理完前 i 个元素）的线段树根节点索引
int cnt = 0;    // 当前已使用的节点数量，作为新节点的索引

int a[MAXN];          // 原始输入数组
int b[MAXN];          // 用于离散化的辅助数组
int values_count;     // 离散化后不同值的数量 (即线段树的实际值域上限)

// 离散化函数
// 返回离散化后的值域上限
int discretize(int n) {
    for (int i = 1; i <= n; ++i) {
        b[i] = a[i]; // 复制原始数据
    }
    std::sort(b + 1, b + n + 1); // 排序
    values_count = std::unique(b + 1, b + n + 1) - (b + 1); // 去重，返回不同元素的数量
    // 将原始数组 a 中的元素替换为离散化后的下标
    for (int i = 1; i <= n; ++i) {
        a[i] = std::lower_bound(b + 1, b + values_count + 1, a[i]) - b;
    }
    return values_count; // 返回离散化后的最大值（即不同元素的数量）
}

// 构建空树或更新节点
// pre: 上一个版本的节点索引
// tl, tr: 当前节点代表的区间范围 (离散化后的值域)
// pos: 要插入的元素在离散化后的值域中的位置
int update(int pre, int tl, int tr, int pos) {
    int cur = ++cnt; // 创建一个新节点
    tree[cur] = tree[pre]; // 复制上一个版本节点的信息 (包括左右子节点指针和sum)
    tree[cur].sum++; // 当前区间内的元素数量加1

    if (tl == tr) { // 如果到达叶子节点
        return cur;
    }

    int mid = (tl + tr) / 2;
    if (pos <= mid) { // 如果要插入的元素在左子区间
        tree[cur].l = update(tree[pre].l, tl, mid, pos); // 递归更新左子树
        // 右子树不变，直接指向 pre 节点的右子树 (tree[cur].r 已经复制了 tree[pre].r)
    } else { // 如果要插入的元素在右子区间
        tree[cur].r = update(tree[pre].r, mid + 1, tr, pos); // 递归更新右子树
        // 左子树不变，直接指向 pre 节点的左子树 (tree[cur].l 已经复制了 tree[pre].l)
    }
    return cur; // 返回新节点的索引
}

// 查询区间 [L, R] 中的第 K 小值
// u: root[R] 中对应当前区间的节点索引
// v: root[L-1] 中对应当前区间的节点索引
// tl, tr: 当前节点代表的区间范围 (离散化后的值域)
// k: 要查找的第 K 小值
int query(int u, int v, int tl, int tr, int k) {
    if (tl == tr) { // 如果到达叶子节点，说明找到了第 K 小的离散化值
        return tl;
    }

    int mid = (tl + tr) / 2;
    // 计算左子区间 [tl, mid] 在原查询区间 [L, R] 中的元素数量
    // 等于 root[R] 的左子树 sum 减去 root[L-1] 的左子树 sum
    int count_left = tree[tree[u].l].sum - tree[tree[v].l].sum;

    if (k <= count_left) { // 如果第 K 小值在左子区间
        return query(tree[u].l, tree[v].l, tl, mid, k);
    } else { // 如果第 K 小值在右子区间
        // 注意：K 需要减去左子区间中的元素数量
        return query(tree[u].r, tree[v].r, mid + 1, tr, k - count_left);
    }
}

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i];
    }

    int max_val = discretize(n); // 离散化，并得到离散化后的值域上限

    // 构建主席树
    root[0] = 0; // root[0] 代表空树，其节点索引为 0
    // 注意：tree[0] 是一个默认的空节点，其 sum=0, l=0, r=0
    // 这样，当 update 递归到空子树时，tree[pre].l 或 tree[pre].r 就会是 0，
    // tree[0].sum 仍然是 0，不会影响计算。
    for (int i = 1; i <= n; ++i) {
        root[i] = update(root[i-1], 1, max_val, a[i]);
    }

    // 处理 M 次查询
    for (int i = 0; i < m; ++i) {
        int l, r, k;
        std::cin >> l >> r >> k;
        int result_idx = query(root[r], root[l-1], 1, max_val, k);
        std::cout << b[result_idx] << "\n"; // 将离散化后的结果映射回原始值并输出
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：**
    *   离散化：`O(N log N)` (排序和去重)
    *   构建主席树：`N` 次更新，每次 `O(log max_val)`。总计 `O(N log max_val)`。由于 `max_val` 最大为 `N`，所以是 `O(N log N)`。
    *   查询：`M` 次查询，每次 `O(log max_val)`。总计 `O(M log N)`。
    *   综合：`O((N+M) log N)`。
*   **空间复杂度：**
    *   存储原始数组 `a` 和离散化数组 `b`：`O(N)`。
    *   主席树节点：`N` 次更新，每次创建 `O(log max_val)` 个节点。总计 `O(N log max_val)`，即 `O(N log N)`。
    *   综合：`O(N log N)`。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // GESP 竞赛常用，包含大部分标准库

// 定义常量，方便修改和阅读
const int MAXN = 100005; // 原始数组的最大长度
const int LOGN = 18;     // log2(MAXN) 的近似值，用于估算线段树深度
const int MAX_NODES = MAXN * LOGN; // 主席树节点池的最大容量，每个元素更新会创建 logN 个节点

// 主席树节点结构体
struct Node {
    int l, r;    // 左右子节点的索引，0表示空节点
    int sum;     // 当前节点所代表区间内元素的计数
} tree[MAX_NODES]; // 节点池，用数组模拟链式结构

int root[MAXN]; // root[i] 存储处理完前 i 个元素后的主席树根节点索引
int cnt = 0;    // 当前已分配的节点数量，作为新节点的唯一索引

int a[MAXN];          // 原始输入数组
int b[MAXN];          // 用于离散化的辅助数组
int values_count;     // 离散化后不同值的数量 (即线段树的实际值域上限)

// 离散化函数
// 参数 n: 原始数组的长度
// 返回值: 离散化后的值域上限（即不同元素的数量）
int discretize(int n) {
    for (int i = 1; i <= n; ++i) {
        b[i] = a[i]; // 复制原始数据到辅助数组
    }
    std::sort(b + 1, b + n + 1); // 对辅助数组排序
    // 去重，并更新 values_count 为不同元素的数量
    values_count = std::unique(b + 1, b + n + 1) - (b + 1);
    // 将原始数组 a 中的元素替换为它们在排序去重数组 b 中的下标 (1-based)
    for (int i = 1; i <= n; ++i) {
        a[i] = std::lower_bound(b + 1, b + values_count + 1, a[i]) - b;
    }
    return values_count; // 返回离散化后的最大值域
}

// 主席树的单点更新函数
// pre: 上一个版本的根节点索引 (或当前路径上的父节点索引)
// tl, tr: 当前节点管理的离散化值域区间 [tl, tr]
// pos: 要插入的元素在离散化值域中的位置
// 返回值: 新创建的节点索引
int update(int pre, int tl, int tr, int pos) {
    int cur = ++cnt; // 分配一个新的节点索引
    tree[cur] = tree[pre]; // 复制上一个版本节点的所有信息
    tree[cur].sum++; // 当前区间内的元素计数加一

    if (tl == tr) { // 如果到达叶子节点，表示成功插入
        return cur;
    }

    int mid = (tl + tr) / 2;
    if (pos <= mid) { // 如果要插入的元素在左子区间
        tree[cur].l = update(tree[pre].l, tl, mid, pos); // 递归更新左子树
        // 右子树保持不变，因为 tree[cur] 已经复制了 tree[pre]，所以 tree[cur].r 已经指向 tree[pre].r
    } else { // 如果要插入的元素在右子区间
        tree[cur].r = update(tree[pre].r, mid + 1, tr, pos); // 递归更新右子树
        // 左子树保持不变
    }
    return cur; // 返回新节点的索引
}

// 主席树的区间第K小查询函数
// u: 区间右端点 R 对应的根节点 (root[R]) 或其子节点
// v: 区间左端点 L-1 对应的根节点 (root[L-1]) 或其子节点
// tl, tr: 当前节点管理的离散化值域区间 [tl, tr]
// k: 要查找的第 K 小值
// 返回值: 第 K 小值在离散化值域中的位置
int query(int u, int v, int tl, int tr, int k) {
    if (tl == tr) { // 如果到达叶子节点，说明找到了结果
        return tl;
    }

    int mid = (tl + tr) / 2;
    // 计算左子区间 [tl, mid] 在原始查询区间 [L, R] 中的元素数量
    // 即 (root[R] 的左子树中的 sum) - (root[L-1] 的左子树中的 sum)
    int count_left = tree[tree[u].l].sum - tree[tree[v].l].sum;

    if (k <= count_left) { // 如果第 K 小值在左子区间
        return query(tree[u].l, tree[v].l, tl, mid, k);
    } else { // 如果第 K 小值在右子区间
        // K 需要减去左子区间已经包含的元素数量
        return query(tree[u].r, tree[v].r, mid + 1, tr, k - count_left);
    }
}

int main() {
    // 优化 C++ 标准输入输出流
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 序列长度, m: 查询次数
    std::cin >> n >> m;

    // 读取原始序列
    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i];
    }

    // 离散化处理，并获取离散化后的最大值域
    int max_val = discretize(n);

    // 构建主席树的各个版本
    root[0] = 0; // root[0] 代表一个空的线段树，其索引为0
    // tree[0] 作为一个默认的空节点，其 sum=0, l=0, r=0。
    // 这样在 update 或 query 访问到空子树时 (如 tree[pre].l == 0)，
    // tree[0].sum 仍为 0，不会影响计算。
    for (int i = 1; i <= n; ++i) {
        root[i] = update(root[i-1], 1, max_val, a[i]);
    }

    // 处理 M 次查询
    for (int i = 0; i < m; ++i) {
        int l, r, k; // 查询区间 [l, r] 中的第 k 小值
        std::cin >> l >> r >> k;
        // 调用查询函数，传入 root[r] (表示前 r 个元素的树) 和 root[l-1] (表示前 l-1 个元素的树)
        int result_idx = query(root[r], root[l-1], 1, max_val, k);
        // 将查询结果（离散化后的下标）映射回原始值并输出
        std::cout << b[result_idx] << "\n";
    }

    return 0;
}
```

    *   **提示：** 主席树的节点 `sum` 存储的是区间内所有数字的总计数。如果想查询某个特定数字 `X` 的出现次数，可以考虑将 `update` 函数修改为，只在 `pos == X` 的叶子节点上增加计数，或者查询时 `query(root[R], root[L-1], X, X, 1)`。更直接的方法是，主席树的 `sum` 依然是总数，但查询时，如果 `X` 在 `[tl, tr]` 区间内，且 `tl==tr==X`，那么 `tree[u].sum - tree[v].sum` 就是 `X` 的出现次数。

1.  **题目：静态区间查询小于等于 K 的数的个数**
    给定一个包含 N 个整数的序列 `a[1...N]`。有 `M` 次查询，每次查询给定 `L, R, K`，要求输出区间 `[L, R]` 中有多少个数字是小于或等于 `K` 的。
    *   **提示：** 这比查询第K小更直接。在 `query(u, v, tl, tr, K_val)` 函数中，如果当前区间 `[tl, tr]` 完全在 `K_val` 范围内（即 `tr <= K_val`），则直接返回 `tree[u].sum - tree[v].sum`。如果 `K_val` 在 `mid` 和 `tr` 之间，则需要累加左子树的全部计数，并递归查询右子树。

**拓展挑战：**

1.  **题目：静态区间查询不同数的个数 (较难，仅作思考)**
    给定一个包含 N 个整数的序列 `a[1...N]`。有 `M` 次查询，每次查询给定 `L, R`，要求输出区间 `[L, R]` 中有多少个不同的数字。
    *   **提示：** 这是一个经典的主席树变种问题，需要更复杂的构建方式。通常的处理方法是，对于每个元素 `a[i]`，除了记录它本身，还要记录它**上一次出现的位置 `pre[i]`**。构建主席树时，在 `root[i]` 中，对 `i` 位置的 `a[i]` 增加计数，同时对 `pre[i]` 位置的 `a[i]` 减少计数。这样，查询 `[L, R]` 时，只有在 `[L, R]` 中第一次出现的 `a[i]` 会被计数为 1，重复出现的会被抵消。这需要对主席树的 `update` 函数进行修改，使其支持单点增减。

### 练习题目答案 (仅供参考，不包含代码)

**基础巩固：**

1.  **静态区间查询出现次数：**
    *   **思路：** 离散化所有可能的值。构建主席树时，每个节点 `sum` 仍然记录它所代表区间内所有元素的计数。
    *   **查询：** `query(root[R], root[L-1], 1, max_val, X_discretized)`。这个 `query` 函数不是找第 K 小，而是找 `X_discretized` 这个值在 `[L, R]` 中出现了多少次。它会递归到 `X_discretized` 所在的叶子节点，然后返回 `tree[u].sum - tree[v].sum`。
    *   **具体查询函数：**
        ```cpp
        int query_count(int u, int v, int tl, int tr, int target_pos) {
            if (tl == tr) {
                return tree[u].sum - tree[v].sum; // 叶子节点直接返回差值
            }
            int mid = (tl + tr) / 2;
            if (target_pos <= mid) {
                return query_count(tree[u].l, tree[v].l, tl, mid, target_pos);
            } else {
                return query_count(tree[u].r, tree[v].r, mid + 1, tr, target_pos);
            }
        }
        ```
        然后调用 `query_count(root[r], root[l-1], 1, max_val, X_discretized)`。

**能力提升：**

1.  **静态区间查询小于等于 K 的数的个数：**
    *   **思路：** 离散化所有值。构建主席树同例题。
    *   **查询：** `query_less_equal(root[R], root[L-1], 1, max_val, K_discretized)`。
    *   **具体查询函数：**
        ```cpp
        int query_less_equal(int u, int v, int tl, int tr, int k_val_discretized) {
            if (u == 0 || v == 0) return 0; // 边界情况，如果节点不存在，返回0
            if (tr <= k_val_discretized) { // 如果当前区间完全小于等于 K_val
                return tree[u].sum - tree[v].sum;
            }
            if (tl > k_val_discretized) { // 如果当前区间完全大于 K_val
                return 0;
            }

            int mid = (tl + tr) / 2;
            int count = 0;
            // 考虑左子树
            count += query_less_equal(tree[u].l, tree[v].l, tl, mid, k_val_discretized);
            // 考虑右子树
            count += query_less_equal(tree[u].r, tree[v].r, mid + 1, tr, k_val_discretized);
            return count;
        }
        ```
        然后调用 `query_less_equal(root[r], root[l-1], 1, max_val, K_discretized)`。
