## 趣味引入

同学们，想象一下我们生活在一个充满魔法的森林里。这片森林里的树木可不是普通的树，它们是“变形金刚树”！

平时，每棵树都有自己的根，枝叶繁茂。但是，森林里会发生各种奇妙的事情：
1.  **树枝会突然变粗变细**：有些连接会变得非常“实”，像高速公路，而有些连接则变得“虚”，像羊肠小道。
2.  **树根会随机变化**：今天这棵树的根是A节点，明天可能就变成了B节点！
3.  **秘密通道**：我们经常需要快速找到从任意一个节点到它当前树根的“秘密通道”，并且这条通道要变成“高速公路”。
4.  **改变全树方向**：有时，为了某个魔法仪式，我们需要把一棵树的某个节点 `x` 变成新的树根，让所有路径都以 `x` 为起点。

传统的树结构，比如我们学过的重链剖分（HLD），虽然能处理路径查询，但它有个缺点：树的结构是固定的！如果树根变了，或者枝干的虚实关系变了，HLD就得重新计算，这太慢了！

那么，有没有一种魔法工具，能够让我们轻松驾驭这些“变形金刚树”，快速响应各种动态变化呢？
有！它就是我们今天要学习的强大魔法——**Link-Cut Tree (LCT)**！
LCT就像是森林的智能管家，能动态调整树的结构，高效地处理各种“变形”操作。今天，我们就先从LCT的“两大基本变形术”——`Access` 和 `makeRoot`——学起！

## 深度知识点讲解

### 1. LCT 是什么？为什么需要它？

**LCT (Link-Cut Tree)**，中文常称作“动态树”，是一种用于维护一个森林（或多棵树）的数据结构，它能高效地支持树的连接、断开、换根、路径查询、子树查询等一系列动态操作，且所有操作的均摊时间复杂度都是 $O(\log n)$。

**为什么需要 LCT？**
我们之前学过很多静态树的算法，比如 LCA、树上差分、重链剖分 (HLD)。这些算法在树的结构不变时非常高效。但是，如果树的结构会动态变化呢？
*   **边的增删**：比如，连接两棵树，或者断开一条边。
*   **换根操作**：将树的某个节点设置为新的根。
*   **路径信息查询/修改**：查询或修改从 `u` 到 `v` 路径上的信息。
*   **子树信息查询/修改**：查询或修改以 `u` 为根的子树信息。

对于这些动态操作，静态的算法往往需要重新计算，效率低下。LCT 就是为了解决这些问题而诞生的。它通过一种巧妙的方式，将复杂的树操作转化为一系列 Splay 树操作，从而达到高效的目的。

### 2. LCT 的核心思想：实链与虚链

LCT 的核心思想是把一棵树分解成若干条“实链”和“虚链”。
*   **实链 (Preferred Path)**：由一些节点通过“实边”连接而成，这些节点在原树中是父子关系。每条实链都是从一个节点向上延伸到其祖先，但不会包含所有祖先。
*   **虚链 (Non-Preferred Path)**：连接不同实链的边。一个节点的父节点，如果它不是当前节点实链上的直接父节点，那么它们之间就是虚边连接。

**Splay 树的引入：**
LCT 使用 Splay 树（伸展树）来维护这些实链。
*   **每条实链对应一棵 Splay 树**：Splay 树的中序遍历结果就是这条实链上从上到下（或从下到上，取决于你如何定义）的节点顺序。
*   **Splay 树的根节点**：Splay 树的根节点是它所维护的实链中深度最小的那个节点。
*   **虚边连接**：Splay 树的根节点通过一个特殊的指针 `pa`（或 `p`，指向原树的父节点）指向它在原树中的父节点。这个父节点不在当前 Splay 树中，它属于另一条实链（或者就是另一条实链的根）。
*   **节点在 Splay 树中的父子关系与原树的父子关系**：Splay 树中的左右孩子关系与原树中的父子关系是独立的。Splay 树只是用来维护一条链上的相对顺序。

**生活类比：**
想象我们的“变形金刚树”被切割成许多段“高速公路”（实链），每段高速公路都由 Splay 树来管理。这些高速公路之间，则由一些“小路”（虚链）连接。LCT 的操作就是动态地将“小路”升级为“高速公路”，或者将“高速公路”降级为“小路”。

### 3. LCT 节点结构

每个 LCT 节点 `i` 通常需要维护以下信息：
*   `c[0], c[1]`：Splay 树中的左右孩子。`c[0]` 是左孩子，`c[1]` 是右孩子。
*   `p`：Splay 树中的父节点。
*   `pa`：**最重要的区别！** 这个 `pa` 指针指向的是当前节点在**原树**中的父节点。如果当前节点是它所在 Splay 树的根，那么 `pa` 指向的就是通过虚边连接的那个父节点。如果当前节点不是 Splay 树的根，那么 `pa` 就没有作用（或者可以指向 `0`）。
*   `rev`：懒标记，表示当前 Splay 树需要进行翻转操作。这用于 `makeRoot` 等操作。
*   `v`：节点自身的值（如果需要维护节点信息）。
*   `s`：子树信息（如果需要维护路径/子树和等）。

```cpp
struct Node {
    int c[2]; // Splay树中的左右孩子：c[0]是左孩子，c[1]是右孩子
    int p;    // Splay树中的父节点
    int pa;   // 在原树中，当前Splay树的根通过虚边连接的父节点
    bool rev; // 翻转标记
    // int v; // 节点值（如果需要）
    // int s; // 子树信息（如果需要）
};
Node t[N]; // N为节点总数
```

**辅助函数：**
*   `isRoot(x)`：判断节点 `x` 是否是它所在 Splay 树的根。当 `x` 的父节点 `t[x].p` 为 0，或者 `t[t[x].p].c[0] != x && t[t[x].p].c[1] != x` 时，`x` 就是 Splay 树的根。后一个条件是因为 Splay 树的根通过 `pa` 指针连接到原树的父节点，而不是通过 `c[0]/c[1]` 连接。
*   `pushup(x)`：更新节点 `x` 的信息（如子树和 `s`），通常由其孩子节点的信息合并得到。
*   `pushdown(x)`：下传节点 `x` 的懒标记（如 `rev` 翻转标记）。

```cpp
void pushup(int x) {
    // t[x].s = t[t[x].c[0]].s ^ t[t[x].c[1]].s ^ t[x].v; // 示例：异或和
}

void pushdown(int x) {
    if (t[x].rev) {
        swap(t[x].c[0], t[x].c[1]); // 翻转左右孩子
        if (t[x].c[0]) t[t[x].c[0]].rev ^= 1; // 传递翻转标记给左孩子
        if (t[x].c[1]) t[t[x].c[1]].rev ^= 1; // 传递翻转标记给右孩子
        t[x].rev = 0; // 清除当前节点的翻转标记
    }
}

bool isRoot(int x) {
    return t[x].p == 0 || (t[t[x].p].c[0] != x && t[t[x].p].c[1] != x);
}

// 获取x是其父节点哪个孩子 (0为左，1为右)
int getChildType(int x) {
    return t[t[x].p].c[1] == x;
}

void rotate(int x) {
    int y = t[x].p, z = t[y].p;
    int k = getChildType(x); // x是y的左孩子还是右孩子
    
    if (!isRoot(y)) { // 如果y不是Splay树的根，则z是y的父节点
        t[z].c[getChildType(y)] = x;
    } else { // 如果y是Splay树的根，则x继承y的虚边父节点
        t[x].pa = t[y].pa; // 虚边父节点转移
    }
    t[x].p = z; // x的父节点现在是z

    t[y].c[k] = t[x].c[k ^ 1]; // 将x的另一侧孩子变为y的孩子
    if (t[x].c[k ^ 1]) {
        t[t[x].c[k ^ 1]].p = y;
    }
    
    t[x].c[k ^ 1] = y; // y成为x的另一侧孩子
    t[y].p = x; // y的父节点现在是x
    
    pushup(y); // 更新y的信息（因为y的孩子变了）
    pushup(x); // 更新x的信息（因为x的孩子变了）
}

void splay(int x) {
    // 自底向上pushdown，确保路径上的懒标记都已下传
    int s[N], top = 0;
    s[++top] = x;
    for (int i = x; !isRoot(i); i = t[i].p) {
        s[++top] = t[i].p;
    }
    while (top) {
        pushdown(s[top--]);
    }

    // Splay操作
    while (!isRoot(x)) {
        int y = t[x].p, z = t[y].p;
        if (!isRoot(y)) { // 如果y不是Splay树的根
            if (getChildType(x) == getChildType(y)) { // 之字形
                rotate(y);
            } else { // 一字形
                rotate(x);
            }
        }
        rotate(x);
    }
}
```

### 4. `Access(x)` 操作：打通秘密通道

`Access(x)` 是 LCT 中最核心、最复杂的单点操作。它的目的是：
**打通从当前整棵 LCT 维护的森林的根（或者说，从 `x` 所在树的根）到节点 `x` 的路径，使这条路径成为一条实链，并且 `x` 成为这条实链所对应的 Splay 树的根。**

**生活类比：**
想象你要从某个地方（根）找到一条直达你家（`x`）的“高速公路”。这条路上可能有很多弯弯绕绕的小路，`Access` 的作用就是把这些小路都“拉直”，变成一条畅通无阻的实链高速公路，并且把你家放在这条高速公路的终点（Splay 树的根）。

**操作原理：**
`Access(x)` 的过程是一个自底向上、不断切换实虚链的过程。
它通过不断地将 `x` 所在的 Splay 树与 `x` 的虚边父节点所在的 Splay 树连接，来逐步构建这条从根到 `x` 的实链。

1.  **初始化 `y = 0`**：`y` 用于记录当前 `x` 的实链右孩子（在Splay树中）。
2.  **循环向上**：从 `x` 开始，不断向上遍历到其虚边父节点 `t[x].pa`。
    *   **Splay `x` 到其所在 Splay 树的根**：确保 `x` 是当前实链的“顶端”（Splay 树的根）。
    *   **断开 `x` 的右孩子，连接 `y` 为 `x` 的右孩子**：
        *   `t[x].c[1]` (即 `x` 的右孩子) 原本是 `x` 所属实链中比 `x` 深度更大的节点。现在我们要把 `x` 变为新的实链的“终点”，所以原来 `x` 的右孩子就不能再是它的实链孩子了，它变成了虚边连接。因此，`t[t[x].c[1]].p = 0;` (断开父子关系)，`t[x].c[1] = 0;` (清空右孩子)。
        *   然后，将之前处理过的下一段实链的根 `y` 连接为 `x` 的右孩子。`t[x].c[1] = y;`，`t[y].p = x;`。这样，`y` 变成了 `x` 的实链孩子。
    *   **更新 `x` 和 `y`**：`y = x;` （当前 `x` 成为下一轮的 `y`），`x = t[x].pa;` （`x` 向上跳到虚边父节点）。
3.  **最终 Splay `y`**：循环结束后，`y` 就是最初的那个目标节点。此时它已经是整条路径的 Splay 树的根了，但可能由于路径连接导致其父子关系被改变，所以最后再 `splay(y)` 一次，确保 `y` 处于 Splay 树的根部。

```cpp
// Access操作：打通从根到x的路径，使x成为Splay树的根
void Access(int x) {
    // 从根到x的路径会变成一条实链，x会是这条实链Splay树的根
    // y用于记录当前x的实链右孩子（在Splay树中）
    for (int y = 0; x; y = x, x = t[x].pa) {
        splay(x); // 将x旋转到它所在Splay树的根

        // 断开x与其原右孩子的实链，将其变为虚链
        // 新的实链将从x向下延伸到y（如果y存在）
        if (t[x].c[1]) {
            t[t[x].c[1]].pa = x; // 原右孩子的虚父现在是x
            t[t[x].c[1]].p = 0;  // 断开Splay树中的父子关系
            t[x].c[1] = 0;       // x的右孩子设为0
        }
        
        // 将y（之前处理过的下一段实链的根）连接为x的右孩子
        // 这表示x现在是y的实链父节点
        if (y) {
            t[x].c[1] = y;
            t[y].p = x;
            t[y].pa = 0; // y现在是实链孩子，不再通过pa连接
        }
        pushup(x); // 更新x的信息
    }
}
```

**`Access` 的关键点：**
*   **自底向上**：从 `x` 向上，逐段处理。
*   **Splay 树的根与虚边父节点**：`t[x].pa` 指针是联系原树结构的关键。当 `x` 是 Splay 树的根时，`t[x].pa` 指向其在原树中的父节点（通过虚边连接）。当 `x` 不是 Splay 树的根时，`t[x].pa` 通常是 `0` 或者不使用。
*   **断开右孩子，连接 `y`**：这是实现实链切换的核心。当 `x` 向上跳时，它会把之前处理好的下一段实链 `y` 连接到自己下方，形成一条更长的实链。

### 5. `makeRoot(x)` 操作：改变全树方向

`makeRoot(x)` 的目的是：
**将节点 `x` 设为它所在树的新的根。**

**生活类比：**
你找到了一条直通你家的高速公路（`Access(x)`），现在你要施展魔法，把这条高速公路的起点和终点颠倒过来，这样你家就变成了这条路的起点，整棵树的“方向”也随之改变了。

**操作原理：**
1.  **`Access(x)`**：首先，调用 `Access(x)`。这会将从原树根到 `x` 的路径打通，使 `x` 成为这条路径所对应 Splay 树的根。此时，`x` 没有右孩子（因为所有比它深度大的节点都被断开了）。
2.  **`splay(x)`**：再次 `splay(x)`，确保 `x` 仍然是 Splay 树的根。这一步在 `Access` 内部的循环结束后已经完成，但为了确保，可以再调用一次。
3.  **翻转 Splay 树**：现在 `x` 是 Splay 树的根，且这条 Splay 树代表了从原树根到 `x` 的路径。如果我们要把 `x` 变成新的树根，那么原树根就变成了 `x` 的“子孙”。这意味着这条路径上所有节点的深度关系都反过来了。在 Splay 树中，这意味着这条 Splay 树的节点顺序需要反转。我们通过设置 `t[x].rev = !t[x].rev` 来标记 `x` 所在的 Splay 树需要翻转。
4.  **`pushdown(x)`**：下传 `rev` 标记，实际执行翻转操作。

```cpp
// makeRoot操作：将x设为所在树的根
void makeRoot(int x) {
    Access(x); // 将x到当前根的路径打通，x成为这条路径Splay树的根
    splay(x);  // 确保x是Splay树的根

    // 翻转x所在的Splay树。
    // 因为x现在是Splay树的根，且没有右孩子。
    // 翻转后，x将变成路径的“起点”，即原树的根
    t[x].rev ^= 1;
    pushdown(x); // 下传翻转标记，实际执行翻转
}
```

**`makeRoot` 的关键点：**
*   `Access(x)` 确保 `x` 成为一条从原树根到 `x` 的实链的 Splay 树根。
*   `rev` 标记的巧妙应用：通过翻转这条实链，我们改变了这条路径上所有节点的相对深度关系，从而实现了“换根”的效果。

**总结：**
`Access` 和 `makeRoot` 是 LCT 的基石。`Access` 负责动态地将一条路径转化为 Splay 树维护的实链，而 `makeRoot` 则在此基础上，通过翻转 Splay 树，实现了树的换根操作。理解这两个操作的细节，是掌握 LCT 的第一步。

### 例题1：Access 操作模拟

**题目描述：**
给定一棵包含 5 个节点的树，结构为：`1-2`, `2-3`, `3-4`, `4-5`。初始时，所有边都是虚边，或者说节点都在各自的 Splay 树中。现在执行 `Access(5)` 操作。请详细描述 `Access(5)` 的过程，画出关键步骤的 Splay 树结构和虚实链变化。

**初始状态（概念上）：**
所有节点都是独立的 Splay 树的根。
1: p=0, pa=0, c={0,0}
2: p=0, pa=1, c={0,0}
3: p=0, pa=2, c={0,0}
4: p=0, pa=3, c={0,0}
5: p=0, pa=4, c={0,0}

**思路分析：**
我们从 `x = 5` 开始，`y = 0`。
1.  **第一次循环：`x = 5`**
    *   `splay(5)`：5已经是Splay树的根，无需操作。
    *   `t[5].c[1]` 为 0，无需断开。
    *   `y = 0`，无需连接。
    *   `pushup(5)`。
    *   `y = 5`, `x = t[5].pa = 4`。
2.  **第二次循环：`x = 4`**
    *   `splay(4)`：4已经是Splay树的根，无需操作。
    *   `t[4].c[1]` 为 0，无需断开。
    *   `y = 5`，连接 5 为 4 的右孩子：`t[4].c[1] = 5`, `t[5].p = 4`, `t[5].pa = 0`。
    *   `pushup(4)`。
    *   `y = 4`, `x = t[4].pa = 3`。
    *   **Splay 树结构：** `4` 是根，`5` 是 `4` 的右孩子。`[4, 5]` 形成一条实链。
3.  **第三次循环：`x = 3`**
    *   `splay(3)`：3已经是Splay树的根，无需操作。
    *   `t[3].c[1]` 为 0，无需断开。
    *   `y = 4`，连接 4 为 3 的右孩子：`t[3].c[1] = 4`, `t[4].p = 3`, `t[4].pa = 0`。
    *   `pushup(3)`。
    *   `y = 3`, `x = t[3].pa = 2`。
    *   **Splay 树结构：** `3` 是根，`4` 是 `3` 的右孩子，`5` 是 `4` 的右孩子。`[3, 4, 5]` 形成一条实链。
4.  **第四次循环：`x = 2`**
    *   `splay(2)`：2已经是Splay树的根，无需操作。
    *   `t[2].c[1]` 为 0，无需断开。
    *   `y = 3`，连接 3 为 2 的右孩子：`t[2].c[1] = 3`, `t[3].p = 2`, `t[3].pa = 0`。
    *   `pushup(2)`。
    *   `y = 2`, `x = t[2].pa = 1`。
    *   **Splay 树结构：** `2` 是根，`3` 是 `2` 的右孩子，`4` 是 `3` 的右孩子，`5` 是 `4` 的右孩子。`[2, 3, 4, 5]` 形成一条实链。
5.  **第五次循环：`x = 1`**
    *   `splay(1)`：1已经是Splay树的根，无需操作。
    *   `t[1].c[1]` 为 0，无需断开。
    *   `y = 2`，连接 2 为 1 的右孩子：`t[1].c[1] = 2`, `t[2].p = 1`, `t[2].pa = 0`。
    *   `pushup(1)`。
    *   `y = 1`, `x = t[1].pa = 0`。
    *   **Splay 树结构：** `1` 是根，`2` 是 `1` 的右孩子，... `5` 是 `4` 的右孩子。`[1, 2, 3, 4, 5]` 形成一条实链。
6.  **循环结束**：`x` 变为 0。
7.  **`splay(y)`**：`splay(5)`。由于 `5` 已经是最右边的节点，`splay(5)` 会将 `5` 旋转到 Splay 树的根。
    *   这会通过多次 `rotate` 操作实现。最终 Splay 树的结构会变成：`5` 是根，`4` 是 `5` 的左孩子，`3` 是 `4` 的左孩子，`2` 是 `3` 的左孩子，`1` 是 `2` 的左孩子。

**最终 Splay 树结构 (Access(5) 后)：**
```
      5
     /
    4
   /
  3
 /
2
/
1
```
其中，`t[5].p=0`, `t[5].pa=0`。`t[4].p=5`, `t[5].c[0]=4`。`t[3].p=4`, `t[4].c[0]=3`，以此类推。
所有节点 `1,2,3,4,5` 都在同一棵 Splay 树中，且 `5` 是这棵 Splay 树的根。从 `5` 向左遍历就是 `4,3,2,1`。

**代码 (Access 部分):**
见上面的深度知识点讲解中的 `Access` 函数。

**复杂度分析：**
`Access(x)` 操作的均摊时间复杂度为 $O(\log n)$。这得益于 Splay 树的均摊分析特性。每次 `splay` 操作会将节点 `x` 移动到 Splay 树的根，从而平摊了路径上的访问成本。

### 例题2：makeRoot 操作模拟

**题目描述：**
在例题1的基础上，假设现在树的结构已经是 `Access(5)` 后的状态（即 `1-2-3-4-5` 形成一条实链，`5` 是 Splay 树的根，`1` 是 Splay 树的最左孩子）。现在执行 `makeRoot(1)` 操作。请详细描述 `makeRoot(1)` 的过程。

**初始状态 (Access(5) 后):**
Splay 树结构：
```
      5
     /
    4
   /
  3
 /
2
/
1
```
其中，`5` 是 Splay 树的根，`t[5].c[0]=4`, `t[4].c[0]=3`, ... `t[2].c[0]=1`。
`t[1].p=2`, `t[2].p=3`, `t[3].p=4`, `t[4].p=5`, `t[5].p=0`。
所有 `pa` 指针均为 `0`。

**思路分析：**
执行 `makeRoot(1)`。
1.  **`Access(1)`**：
    *   `Access(1)` 会将从当前根（`5`）到 `1` 的路径打通，使 `1` 成为 Splay 树的根。
    *   在 `Access` 的循环中，`x` 从 `1` 向上遍历到 `5`。
    *   `splay(x)` 操作会把 `x` 旋转到当前它所在 Splay 树的根。
    *   断开 `t[x].c[1]` 并连接 `y` 的操作。
        *   当 `x=1` 时，`y=0`。`splay(1)` 会把 `1` 旋转到 Splay 树的根。此时 Splay 树变为：`1` 是根，`2` 是 `1` 的右孩子，`3` 是 `2` 的右孩子，... `5` 是 `4` 的右孩子。
        *   当 `x=2` 时，`y=1`。`splay(2)` 会把 `2` 旋转到 Splay 树的根。`t[2].c[1]` (即 `3`) 被断开，`t[3].pa=2`, `t[3].p=0`。然后 `1` 连接为 `2` 的右孩子。
        *   这个过程会一直向上进行，直到 `x=5`。
    *   最终 `Access(1)` 结束后，`1` 会成为 Splay 树的根，并且其下方的节点（`2,3,4,5`）会以左孩子的形式连接。
        ```
              1
             /
            2
           /
          3
         /
        4
       /
      5
        ```
        此时 `t[1].p=0`, `t[1].pa=0`。`t[2].p=1`, `t[1].c[0]=2`。`t[3].p=2`, `t[2].c[0]=3`，以此类推。
2.  **`splay(1)`**：再次 `splay(1)`，确保 `1` 是 Splay 树的根。此时 `1` 已经是根，无需操作。
3.  **`t[1].rev ^= 1`**：设置 `1` 的翻转标记为 `true`。
4.  **`pushdown(1)`**：下传翻转标记。
    *   `pushdown(1)` 会 `swap(t[1].c[0], t[1].c[1])`。
        *   原 `t[1].c[0]` 是 `2`，`t[1].c[1]` 是 `0`。
        *   交换后，`t[1].c[0]` 变为 `0`，`t[1].c[1]` 变为 `2`。
    *   同时，`t[2].rev ^= 1`。
    *   此时，Splay 树结构变为：
        ```
        1
         \
          2
         /
        3  (rev=1)
       /
      4
     /
    5
        ```
        （注意，`pushdown` 只是改变了 `1` 的孩子，`2` 及其子树的 `rev` 标记被设置为 `1`，但 `2` 的子树还没有实际翻转。当下次访问 `2` 或其子树时，`pushdown(2)` 会被调用，进一步下传标记。）
    *   **从原树的角度看**：`makeRoot(1)` 后，`1` 成为了新的根。路径 `1-2-3-4-5` 的方向反转了，现在 `5` 是 `1` 的最深子节点。

**最终状态 (makeRoot(1) 后):**
逻辑上，`1` 是树的根。
Splay 树的结构在 `1` 处被翻转：
```
     1 (rev=0)
      \
       2 (rev=1)
        \
         3 (rev=1)
          \
           4 (rev=1)
            \
             5 (rev=1)
```
**注意：** 实际的 `pushdown` 是惰性的。当 `t[x].rev` 为 `true` 时，`swap(t[x].c[0], t[x].c[1])` 发生，并且 `t[x].c[0]` 和 `t[x].c[1]` 的 `rev` 标记被翻转。只有当我们需要访问 `t[x].c[0]` 或 `t[x].c[1]` 时，才会对它们进行 `pushdown`。所以，上述图示中的 `rev=1` 表示的是标记已传递下去，但实际的 `swap` 尚未发生。

**代码 (makeRoot 部分):**
见上面的深度知识点讲解中的 `makeRoot` 函数。

**复杂度分析：**
`makeRoot(x)` 的时间复杂度主要由 `Access(x)` 决定，也是均摊 $O(\log n)$。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // GESP 10 允许使用此头文件

const int N = 100005; // 最大节点数

struct Node {
    int c[2]; // Splay树中的左右孩子：c[0]是左孩子，c[1]是右孩子
    int p;    // Splay树中的父节点
    int pa;   // 在原树中，当前Splay树的根通过虚边连接的父节点
    bool rev; // 翻转标记
    // int v; // 节点值（如果需要）
    // int s; // 子树信息（如果需要）
};
Node t[N]; // N为节点总数

// GESP 1-5级限制STL，LCT内部结构通常用原生数组，符合要求
// 对于输入，可以使用 vector<int> adj[N] 来构建邻接表，但这里不涉及读图，只关注LCT操作

// 辅助函数：更新节点信息
// 根据题目需要，这里可以更新子树和、最大值等
void pushup(int x) {
    if (!x) return;
    // t[x].s = t[x].v; // 假设节点值就是自身信息
    // if (t[x].c[0]) t[x].s ^= t[t[x].c[0]].s; // 示例：异或和
    // if (t[x].c[1]) t[x].s ^= t[t[x].c[1]].s;
}

// 辅助函数：下传懒标记
void pushdown(int x) {
    if (!x || !t[x].rev) return; // 无需下传或节点不存在
    
    // 交换左右孩子，实现翻转
    std::swap(t[x].c[0], t[x].c[1]);
    
    // 向孩子传递翻转标记
    if (t[x].c[0]) t[t[x].c[0]].rev ^= 1;
    if (t[x].c[1]) t[t[x].c[1]].rev ^= 1;
    
    t[x].rev = 0; // 清除当前节点的翻转标记
}

// 判断节点x是否是它所在Splay树的根
// Splay树的根的父节点是0，或者它的父节点的孩子指针不指向它
// 后者情况是因为Splay树的根通过pa指针连接，而不是c[0]/c[1]
bool isRoot(int x) {
    return t[x].p == 0 || (t[t[x].p].c[0] != x && t[t[x].p].c[1] != x);
}

// 获取x是其父节点哪个孩子 (0为左，1为右)
int getChildType(int x) {
    return t[t[x].p].c[1] == x;
}

// 旋转操作
void rotate(int x) {
    int y = t[x].p; // x的父节点
    int z = t[y].p; // y的父节点
    int k = getChildType(x); // x是y的左孩子(0)还是右孩子(1)

    // 如果y不是Splay树的根，则y的父节点z的孩子指针需要更新
    if (!isRoot(y)) {
        t[z].c[getChildType(y)] = x;
    } else {
        // 如果y是Splay树的根，那么x在旋转后将继承y的pa指针
        t[x].pa = t[y].pa;
    }
    t[x].p = z; // x的父节点更新为z

    // 将x的k^1侧孩子变为y的k侧孩子
    t[y].c[k] = t[x].c[k ^ 1];
    if (t[x].c[k ^ 1]) {
        t[t[x].c[k ^ 1]].p = y;
    }
    
    // y成为x的k^1侧孩子
    t[x].c[k ^ 1] = y;
    t[y].p = x;

    // 更新信息
    pushup(y);
    pushup(x);
}

// Splay操作：将x旋转到它所在Splay树的根
void splay(int x) {
    // 自底向上pushdown，确保路径上的所有懒标记都已下传
    // 使用一个栈来存储需要pushdown的节点，确保顺序正确
    int s[N], top = 0;
    s[++top] = x;
    for (int i = x; !isRoot(i); i = t[i].p) {
        s[++top] = t[i].p;
    }
    while (top) {
        pushdown(s[top--]);
    }

    // 执行Splay旋转
    while (!isRoot(x)) {
        int y = t[x].p;
        int z = t[y].p;
        if (!isRoot(y)) { // 如果y不是Splay树的根
            // 判断是之字形还是一字形旋转
            if (getChildType(x) == getChildType(y)) { // 之字形 (zig-zig 或 zag-zag)
                rotate(y); // 先旋转y
            } else { // 一字形 (zig-zag 或 zag-zig)
                rotate(x); // 先旋转x
            }
        }
