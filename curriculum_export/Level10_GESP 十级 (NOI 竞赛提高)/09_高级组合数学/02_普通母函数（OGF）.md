## 深度知识点讲解

### 1. 普通母函数的本质：序列的魔法目录

**定义：**
对于一个序列 $a_0, a_1, a_2, \dots, a_n, \dots$，它的**普通母函数 (Ordinary Generating Function, OGF)** 是一个形式幂级数：
$A(x) = a_0 + a_1x + a_2x^2 + \dots + a_nx^n + \dots = \sum_{n=0}^{\infty} a_nx^n$

**本质剖析：**
*   **$x$ 是什么？** 在普通母函数中，$x$ 通常被视为一个**占位符**，它本身没有数值意义，我们不关心它的具体取值。我们关心的只是 $x$ 的不同幂次所对应的系数。
*   **系数 $a_n$ 是什么？** $a_n$ 是序列的第 $n$ 项。在组合数学中，$a_n$ 往往代表“某种方式下，总和为 $n$ 的计数”。例如，在糖果问题中，$a_n$ 就是凑出 $n$ 元钱的方案数。
*   **“魔法目录”的类比：** 想象一下，你有一本目录，第 $n$ 页写着“凑出 $n$ 元钱有 $a_n$ 种方法”。普通母函数就是把这整本目录的信息，浓缩成了一个代数表达式！当你需要知道凑出 $N$ 元钱的方法数时，你只需要找到这个表达式中 $x^N$ 的系数 $a_N$ 就行了。

**为什么它很“普通”？**
因为它的系数直接对应序列的项。我们之后还会学到指数母函数（EGF），它处理的是排列问题，系数需要除以 $n!$，所以普通母函数相对更“直接”。

### 2. 构建普通母函数的基本砖块

构建普通母函数，就像搭乐高积木一样，我们需要一些基本砖块。

**砖块1：选择一个物品**
假设你有一件价值为 $k$ 的物品，你可以选择拿 0 件或 1 件。
*   拿 0 件：贡献 0 价值，对应 $x^0 = 1$
*   拿 1 件：贡献 $k$ 价值，对应 $x^k$
所以，选择一件价值为 $k$ 的物品的 OGF 是 $1 + x^k$。

**砖块2：选择一种物品，数量不限**
假设你有一种价值为 $k$ 的物品，你可以选择拿 0 件，1 件，2 件，... 任意多件。
*   拿 0 件：贡献 0 价值，对应 $x^0 = 1$
*   拿 1 件：贡献 $k$ 价值，对应 $x^k$
*   拿 2 件：贡献 $2k$ 价值，对应 $x^{2k}$
*   ...
所以，选择一种价值为 $k$ 的物品（数量不限）的 OGF 是 $1 + x^k + x^{2k} + x^{3k} + \dots = \sum_{j=0}^{\infty} (x^k)^j$。
这是一个等比数列求和，当 $|x^k| < 1$ 时，其和为 $\frac{1}{1 - x^k}$。在形式幂级数中，我们直接用这个表达式。

**砖块3：选择一种物品，数量有限制**
假设你有一种价值为 $k$ 的物品，最多只能拿 $m$ 件。
OGF 是 $1 + x^k + x^{2k} + \dots + x^{mk} = \frac{1 - (x^k)^{m+1}}{1 - x^k}$。

**常见序列的 OGF：**
*   **常数序列 $1, 1, 1, \dots$**：$1 + x + x^2 + \dots = \frac{1}{1-x}$
*   **序列 $1, 0, 1, 0, \dots$**：$1 + x^2 + x^4 + \dots = \frac{1}{1-x^2}$
*   **序列 $1, a, a^2, a^3, \dots$ (等比序列)**：$1 + ax + (ax)^2 + \dots = \frac{1}{1-ax}$
*   **序列 $\binom{k}{0}, \binom{k}{1}, \dots, \binom{k}{k}, 0, 0, \dots$ (二项式系数)**：$\sum_{i=0}^k \binom{k}{i} x^i = (1+x)^k$

### 3. 普通母函数的神奇运算与组合意义

**3.1 OGF 加法：**
如果 $A(x)$ 生成序列 $a_n$， $B(x)$ 生成序列 $b_n$。
那么 $A(x) + B(x) = (a_0+b_0) + (a_1+b_1)x + \dots$
**组合意义：** 如果 $a_n$ 是做任务 A 得到 $n$ 个结果的方法数，$b_n$ 是做任务 B 得到 $n$ 个结果的方法数，并且任务 A 和任务 B 是**互斥**的（不能同时发生），那么 $a_n+b_n$ 就是做任务 A 或任务 B 得到 $n$ 个结果的方法数。

**3.2 OGF 乘法：**
这是普通母函数最核心、最强大的运算！
如果 $A(x) = a_0 + a_1x + a_2x^2 + \dots$
$B(x) = b_0 + b_1x + b_2x^2 + \dots$
那么 $A(x)B(x) = (a_0b_0) + (a_0b_1+a_1b_0)x + (a_0b_2+a_1b_1+a_2b_0)x^2 + \dots$
一般地， $A(x)B(x)$ 中 $x^n$ 的系数是 $\sum_{k=0}^n a_k b_{n-k}$。

**组合意义（最重要的！）：**
如果 $a_k$ 是从**第一组物品**中选择，总价值为 $k$ 的方法数；
$b_j$ 是从**第二组物品**中选择，总价值为 $j$ 的方法数。
那么， $A(x)B(x)$ 中 $x^n$ 的系数 $\sum_{k=0}^n a_k b_{n-k}$，就是**从第一组和第二组物品中各选择一部分，使得总价值恰好为 $n$ 的方法数**。
这完美对应了我们“凑钱”的例子：选择彩虹糖是一个 OGF，选择流星糖是另一个 OGF，把它们乘起来，就能得到同时选择这两种糖果的总方法数！

**3.3 OGF 求导与积分 (了解其组合意义即可，应用较少)**
*   **求导：** $A'(x) = a_1 + 2a_2x + 3a_3x^2 + \dots = \sum_{n=1}^{\infty} n a_n x^{n-1}$
    *   $x A'(x) = \sum_{n=1}^{\infty} n a_n x^n$。这个系数 $n a_n$ 常用于计算平均值或带权重的计数。
*   **积分：** $\int A(x) dx = a_0x + \frac{a_1}{2}x^2 + \frac{a_2}{3}x^3 + \dots = \sum_{n=0}^{\infty} \frac{a_n}{n+1} x^{n+1}$
    *   组合意义通常不那么直观，但有时可用于解决特定类型的递推关系。

### 4. 提取系数的魔法：从 OGF 中找到答案

一旦我们构建出了普通母函数，我们的目标就是找到特定 $x^n$ 的系数。

**4.1 广义二项式定理 (Generalized Binomial Theorem)：**
这是提取系数的利器！
$(1+y)^\alpha = \sum_{n=0}^{\infty} \binom{\alpha}{n} y^n$
其中 $\binom{\alpha}{n} = \frac{\alpha(\alpha-1)\dots(\alpha-n+1)}{n!}$。
当 $\alpha$ 是负整数时，这个定理特别有用。例如，如果 $\alpha = -k$ ( $k$ 是正整数)：
$(1-x)^{-k} = \sum_{n=0}^{\infty} \binom{-k}{n} (-x)^n = \sum_{n=0}^{\infty} (-1)^n \binom{k+n-1}{n} (-1)^n x^n = \sum_{n=0}^{\infty} \binom{n+k-1}{n} x^n = \sum_{n=0}^{\infty} \binom{n+k-1}{k-1} x^n$
这个公式非常重要！它表示从 $k$ 种物品中，每种数量不限，选取 $n$ 件物品的方案数。

**4.2 部分分式分解：**
当普通母函数是一个有理分式 $\frac{P(x)}{Q(x)}$ 时，如果 $Q(x)$ 可以分解成线性因子的乘积，我们可以使用部分分式分解将其拆分成更简单的形式，例如 $\frac{A}{1-ax} + \frac{B}{1-bx} + \dots$。然后利用 $\frac{1}{1-ax} = \sum (ax)^n$ 的形式来提取系数。

**4.3 暴力多项式乘法 (系数较小)：**
如果 $N$ 比较小，或者母函数的阶数不高，我们可以直接进行多项式乘法，然后找出 $x^N$ 的系数。这在编程实现中通常转化为动态规划。

### 5. 常见误区提醒

*   **$x$ 的意义：** 再次强调，$x$ 只是占位符，不要轻易代入数值（除非是求和或特定恒等式）。
*   **OGF vs. EGF：** OGF 用于计数无序选择（组合），EGF 用于计数有序排列（排列）。不要混淆。
*   **广义二项式定理的符号：** 记住 $(1-x)^{-k} = \sum_{n=0}^{\infty} \binom{n+k-1}{k-1} x^n$ 的形式，特别是指数上的负号和组合数中的 $k-1$。

通过这些知识点的学习，我们就可以开始用普通母函数解决实际问题了！

## 典型例题精讲

### 例题1：凑钱问题（基础应用）

**题目：** 你有无限数量的1元、2元和3元硬币。问凑成 $N$ 元钱有多少种不同的方法？

**思路分析：**
1.  **为每种硬币构建 OGF：**
    *   1元硬币：可以取0个、1个、2个...，对应的价值是 $0, 1, 2, \dots$。其 OGF 为 $1 + x^1 + x^2 + x^3 + \dots = \frac{1}{1-x}$。
    *   2元硬币：可以取0个、1个、2个...，对应的价值是 $0, 2, 4, \dots$。其 OGF 为 $1 + x^2 + x^4 + x^6 + \dots = \frac{1}{1-x^2}$。
    *   3元硬币：可以取0个、1个、2个...，对应的价值是 $0, 3, 6, \dots$。其 OGF 为 $1 + x^3 + x^6 + x^9 + \dots = \frac{1}{1-x^3}$。
2.  **组合所有选择：** 由于选择每种硬币是独立的，且我们希望得到总价值为 $N$，所以我们将这三个 OGF 相乘。
    总的 OGF $A(x) = \frac{1}{(1-x)(1-x^2)(1-x^3)}$。
3.  **提取系数：** 我们需要找到 $A(x)$ 中 $x^N$ 的系数。
    *   对于较小的 $N$，我们可以通过多项式乘法展开。
    *   对于较大的 $N$，这通常涉及到部分分式分解，或者更常用的是**动态规划**。动态规划是计算这种 OGF 系数的一种高效方法。

**代码实现（动态规划模拟系数提取）：**
```cpp
#include <bits/stdc++.h> // GESP 10 允许使用此头文件

int dp[1000 + 1]; // dp[j] 表示凑成 j 元钱的方法数

void solve() {
    int n; // 目标金额
    std::cin >> n;

    // 初始化：凑成 0 元钱只有 1 种方法（不取任何硬币）
    dp[0] = 1;

    // 硬币面值
    int c[] = {1, 2, 3}; // 题目中给出的硬币面值
    int m = sizeof(c) / sizeof(c[0]); // 硬币种类数量

    // 遍历每种硬币
    for (int i = 0; i < m; ++i) { // i 遍历硬币种类
        int v = c[i]; // 当前硬币的面值
        // 遍历所有可能的金额 j，从当前硬币面值开始
        // 为什么从 v 开始？因为凑 j 元需要至少一个当前硬币 v
        // 为什么是 dp[j] += dp[j - v]？
        // 想象我们现在考虑使用硬币 v。
        // 凑成 j 元的方法数，可以分为两部分：
        // 1. 不使用硬币 v 的方法数 (这是在考虑当前硬币 v 之前的 dp[j] 值)
        // 2. 使用至少一个硬币 v 的方法数 (这部分是 dp[j-v] 的基础上再加一个 v)
        // 这种累加方式正是 OGF 乘法的组合意义：
        // (1 + x^v + x^2v + ...) * (之前硬币的OGF)
        // 乘法展开后，x^j 的系数是 sigma(x^(j-kv) * x^kv)
        // 对应 dp[j] = sum(dp_prev[j-kv])
        // 这里的 dp[j] += dp[j-v] 相当于在更新 dp 数组时，
        // 考虑了当前硬币 v 可以用任意次的情况。
        // 对于每一个 j，dp[j] 存储的是考虑了前面所有硬币和当前硬币 v 之前所有硬币的情况。
        // 当我们加上 dp[j-v] 时，dp[j-v] 已经包含了凑成 j-v 元的所有方法。
        // 那么在这些方法的基础上再加一个硬币 v，就得到了凑成 j 元的新方法。
        // 由于 dp 数组是原地更新的，dp[j-v] 已经包含了使用当前硬币 v 的信息。
        // 这种DP的顺序保证了每种凑法只被计数一次，且硬币的顺序不影响结果（组合）。
        for (int j = v; j <= n; ++j) {
            dp[j] = dp[j] + dp[j - v]; // Allman style
        }
    }

    std::cout << dp[n] << std::endl;
}

/*
int main() {
    solve();
    return 0;
}
*/
```
**复杂度分析：**
*   时间复杂度：$O(M \cdot N)$，其中 $M$ 是硬币种类数， $N$ 是目标金额。
*   空间复杂度：$O(N)$，用于存储 `dp` 数组。

### 例题2：解决线性递推关系（进阶应用）

**题目：** 已知一个序列 $a_n$ 满足递推关系 $a_n = 3a_{n-1} - 2a_{n-2}$，且初始条件为 $a_0=1, a_1=3$。请用普通母函数的方法求出 $a_n$ 的通项公式。

**思路分析：**
1.  **构建 OGF：** 设序列 $a_n$ 的普通母函数为 $A(x) = \sum_{n=0}^{\infty} a_nx^n$。
2.  **代入递推关系：** 将递推关系式 $a_n = 3a_{n-1} - 2a_{n-2}$ 乘以 $x^n$，并对 $n \ge 2$ 求和。
    $\sum_{n=2}^{\infty} a_nx^n = 3 \sum_{n=2}^{\infty} a_{n-1}x^n - 2 \sum_{n=2}^{\infty} a_{n-2}x^n$
3.  **将和式转换为 $A(x)$ 的形式：**
    *   左边：$\sum_{n=2}^{\infty} a_nx^n = (a_0 + a_1x + a_2x^2 + \dots) - a_0 - a_1x = A(x) - a_0 - a_1x$
    *   中间项：$3 \sum_{n=2}^{\infty} a_{n-1}x^n = 3x \sum_{n=2}^{\infty} a_{n-1}x^{n-1} = 3x \sum_{k=1}^{\infty} a_kx^k = 3x(A(x) - a_0)$
    *   右边项：$2 \sum_{n=2}^{\infty} a_{n-2}x^n = 2x^2 \sum_{n=2}^{\infty} a_{n-2}x^{n-2} = 2x^2 \sum_{k=0}^{\infty} a_kx^k = 2x^2 A(x)$
4.  **代入初始条件并解出 $A(x)$：**
    $A(x) - a_0 - a_1x = 3x(A(x) - a_0) - 2x^2 A(x)$
    代入 $a_0=1, a_1=3$:
    $A(x) - 1 - 3x = 3x(A(x) - 1) - 2x^2 A(x)$
    $A(x) - 1 - 3x = 3xA(x) - 3x - 2x^2 A(x)$
    $A(x) - 1 = 3xA(x) - 2x^2 A(x)$
    $A(x) (1 - 3x + 2x^2) = 1$
    $A(x) = \frac{1}{1 - 3x + 2x^2}$
5.  **部分分式分解并提取系数：**
    分母可以因式分解：$1 - 3x + 2x^2 = (1-x)(1-2x)$
    所以 $A(x) = \frac{1}{(1-x)(1-2x)}$
    进行部分分式分解：
    $\frac{1}{(1-x)(1-2x)} = \frac{C_1}{1-x} + \frac{C_2}{1-2x}$
    通分后 $1 = C_1(1-2x) + C_2(1-x)$
    *   令 $x=1$: $1 = C_1(1-2) + C_2(1-1) \Rightarrow 1 = -C_1 \Rightarrow C_1 = -1$
    *   令 $x=1/2$: $1 = C_1(1-1) + C_2(1-1/2) \Rightarrow 1 = C_2(1/2) \Rightarrow C_2 = 2$
    所以 $A(x) = \frac{-1}{1-x} + \frac{2}{1-2x}$
    现在利用等比数列的 OGF 形式：
    $\frac{1}{1-x} = \sum_{n=0}^{\infty} x^n$
    $\frac{1}{1-2x} = \sum_{n=0}^{\infty} (2x)^n = \sum_{n=0}^{\infty} 2^n x^n$
    因此，$A(x) = - \sum_{n=0}^{\infty} x^n + 2 \sum_{n=0}^{\infty} 2^n x^n = \sum_{n=0}^{\infty} (-1 + 2 \cdot 2^n) x^n = \sum_{n=0}^{\infty} (2^{n+1} - 1) x^n$
6.  **得到通项公式：**
    $a_n = 2^{n+1} - 1$

**复杂度分析：**
*   这是一个纯粹的数学推导过程，没有直接的计算代码复杂度。代数操作的“复杂度”取决于表达式的复杂性。

### 例题3：受限制的物品选择（综合应用）

**题目：** 有红、黄、蓝三种颜色的球。
*   红球：每只价值1元，最多选2只。
*   黄球：每只价值2元，最多选1只。
*   蓝球：每只价值3元，数量无限。
问：总共选择价值为 $N$ 元球的方法数。请写出其普通母函数，并求出当 $N=4$ 时的方法数。

**思路分析：**
1.  **为每种球构建 OGF：**
    *   红球（价值1元，最多2只）：$1 + x^1 + x^2$
    *   黄球（价值2元，最多1只）：$1 + x^2$
    *   蓝球（价值3元，数量无限）：$1 + x^3 + x^6 + \dots = \frac{1}{1-x^3}$
2.  **组合所有选择：** 将它们相乘。
    总的 OGF $A(x) = (1 + x + x^2)(1 + x^2) \frac{1}{1-x^3}$
3.  **求 $N=4$ 时的方法数（即 $x^4$ 的系数）：**
    我们可以直接展开多项式来找到 $x^4$ 的系数。
    首先计算 $(1 + x + x^2)(1 + x^2)$：
    $(1 + x + x^2)(1 + x^2) = 1(1+x^2) + x(1+x^2) + x^2(1+x^2)$
    $= (1+x^2) + (x+x^3) + (x^2+x^4)$
    $= 1 + x + 2x^2 + x^3 + x^4$
    现在 $A(x) = (1 + x + 2x^2 + x^3 + x^4) (1 + x^3 + x^6 + \dots)$
    我们需要找到 $x^4$ 的系数。从第一个括号中取出 $x^k$，从第二个括号中取出 $x^{4-k}$。
    *   当 $k=1$ 时，从第一个括号取 $x^1$，从第二个括号取 $x^3$ ($1 \cdot x^3$)。贡献 $1 \cdot 1 = 1$。
    *   当 $k=2$ 时，从第一个括号取 $2x^2$，从第二个括号取 $x^2$ (不可能，蓝球只能是 $x^0, x^3, x^6, \dots$)。贡献 $0$。
    *   当 $k=4$ 时，从第一个括号取 $x^4$，从第二个括号取 $x^0$ ($1 \cdot x^0$)。贡献 $1 \cdot 1 = 1$。
    *   当 $k=0$ 时，从第一个括号取 $1$，从第二个括号取 $x^4$ (不可能)。贡献 $0$。
    *   当 $k=3$ 时，从第一个括号取 $x^3$，从第二个括号取 $x^1$ (不可能)。贡献 $0$。

    让我们重新仔细计算 $x^4$ 的系数：
    $A(x) = (1 + x + 2x^2 + x^3 + x^4) \cdot (1 + x^3 + x^6 + \dots)$
    要得到 $x^4$，我们可以有以下组合：
    1.  从第一个括号取 $1$ ($x^0$)，从第二个括号取 $x^4$ (不可能，因为蓝球只能是 $x^0, x^3, x^6, \dots$)。
    2.  从第一个括号取 $x^1$，从第二个括号取 $x^3$。系数是 $1 \times 1 = 1$。
    3.  从第一个括号取 $2x^2$，从第二个括号取 $x^2$ (不可能)。
    4.  从第一个括号取 $x^3$，从第二个括号取 $x^1$ (不可能)。
    5.  从第一个括号取 $x^4$，从第二个括号取 $x^0$。系数是 $1 \times 1 = 1$。

    所以， $x^4$ 的系数是 $1 + 1 = 2$。
    方法数：2种。
    具体是哪两种呢？
    *   1个红球 (1元) + 1个蓝球 (3元) = 4元
    *   4个红球 (不可能，最多2个)
    *   2个红球 (2元) + 1个黄球 (2元) = 4元
    *   1个黄球 (2元) + 2个蓝球 (不可能，蓝球只能3的倍数)
    *   1个蓝球 (3元) + 1个红球 (1元) = 4元
    *   1个蓝球 (3元) + 2个红球 (不可能)
    *   1个蓝球 (3元) + 1个黄球 (不可能)

    重新梳理一下组合：
    1. 红球2个 (2元) + 黄球1个 (2元) = 4元
    2. 红球1个 (1元) + 蓝球1个 (3元) = 4元
    所以答案是 2。我们的计算是正确的。

**代码实现（直接模拟多项式乘法或DP）：**
对于这种小 $N$ 的情况，直接模拟多项式乘法比DP更直观。
我们可以用一个数组 `p` 来表示多项式的系数。
`p[k]` 存储 $x^k$ 的系数。

```cpp
#include <bits/stdc++.h>

// 辅助函数：将两个多项式相乘，返回结果多项式
// p1: 第一个多项式系数数组
// s1: 第一个多项式有效最高次幂
// p2: 第二个多项式系数数组
// s2: 第二个多项式有效最高次幂
// res: 结果多项式系数数组
// GESP 10 规定用单字母变量，这里为了清晰，对参数名做了特殊处理
void multiplyPolynomials(int p1[], int s1, int p2[], int s2, int res[], int& rs) {
    // 结果多项式的最高次幂是两个多项式最高次幂之和
    rs = s1 + s2;
    // 初始化结果多项式为0
    for (int i = 0; i <= rs; ++i) {
        res[i] = 0;
    }

    // 执行多项式乘法
    for (int i = 0; i <= s1; ++i) {
        for (int j = 0; j <= s2; ++j) {
            res[i + j] = res[i + j] + p1[i] * p2[j]; // Allman style
        }
    }
}

void solve_example3() {
    int n = 4; // 目标总价值

    // 红球 OGF: (1 + x + x^2)
    int p_red[3] = {1, 1, 1}; // 系数: x^0, x^1, x^2
    int s_red = 2; // 最高次幂

    // 黄球 OGF: (1 + x^2)
    int p_yellow[3] = {1, 0, 1}; // 系数: x^0, x^1, x^2
    int s_yellow = 2;

    // 蓝球 OGF: 1 / (1 - x^3) = 1 + x^3 + x^6 + ...
    // 对于有限的 N，我们可以只考虑直到 N 的项
    int p_blue[1000 + 1]; // 足够大
    int s_blue = n; // 只需要考虑最高到 N 的项
    for (int i = 0; i <= s_blue; ++i) {
        p_blue[i] = (i % 3 == 0) ? 1 : 0; // Allman style
    }

    // 1. 先乘红球和黄球的 OGF
    int p_temp1[5]; // 最高次幂 2+2=4
    int s_temp1;
    multiplyPolynomials(p_red, s_red, p_yellow, s_yellow, p_temp1, s_temp1);
    // p_temp1 此时是 (1 + x + x^2)(1 + x^2) = 1 + x + 2x^2 + x^3 + x^4

    // 2. 再乘上蓝球的 OGF (只考虑直到 N 的项)
    int p_final[1000 + 1]; // 最终结果
    int s_final;
    multiplyPolynomials(p_temp1, s_temp1, p_blue, n, p_final, s_final);

    std::cout << "凑成 " << n << " 元的方法数: " << p_final[n] << std::endl;
}

**复杂度分析：**
*   多项式乘法的复杂度是 $O(D_1 \cdot D_2)$，其中 $D_1, D_2$ 是两个多项式的最高次幂。
*   本例中，最终的乘法是 $O(N \cdot N)$，因为蓝球 OGF 我们只取到 $x^N$ 项。
*   空间复杂度：$O(N)$。

## 代码实现模板

由于普通母函数通常用于数学推导或理论分析，其“代码实现”往往体现在两种场景：
1.  **多项式乘法**：当母函数由若干个多项式相乘得到，且多项式次数不高时。
2.  **动态规划**：当母函数是形如 $\frac{1}{(1-x^{c_1})(1-x^{c_2})\dots}$ 的形式，需要计算特定系数时。这本质上是多项式乘法的优化实现。

这里提供一个通用的**动态规划模板**，用于解决“凑钱问题”或“整数划分”等经典 OGF 问题，这相当于在计算 OGF 乘积的某个系数。

```cpp
#include <bits/stdc++.h> // GESP 10 允许使用此头文件

const int MAX_N = 1000; // 目标总和的最大值
const int MAX_ITEMS = 100; // 物品种类或硬币种类最大值

// dp[j] 表示凑成金额 j 的方法数
int dp[MAX_N + 1];

// 物品的价值或面值
int item_values[MAX_ITEMS];
int num_items; // 物品种类数量

void initialize_dp(int target_n) {
    // 初始化 dp 数组
    // 凑成 0 元（或总和为 0）只有 1 种方法：什么都不选
    for (int j = 1; j <= target_n; ++j) {
        dp[j] = 0; // 初始时，除了 0 之外，其他金额都无法凑成
    }
    dp[0] = 1;
}

// 核心函数：计算凑成目标金额 n 的方法数
// 对应于 OGF A(x) = product_i (1 + x^(value_i) + x^(2*value_i) + ...)
// 的 x^n 的系数
int calculate_ways(int target_n) {
    initialize_dp(target_n);

    // 遍历每种物品/硬币
    for (int i = 0; i < num_items; ++i) {
        int v = item_values[i]; // 当前物品的价值

        // 从当前物品的价值开始，遍历所有可能的目标金额
        // 对于每个目标金额 j，尝试使用当前物品 v
        // dp[j] = dp[j] (不使用当前物品 v 的方法数) + dp[j - v] (使用至少一个当前物品 v 的方法数)
        // 这个循环确保了每种物品可以被使用任意多次
        for (int j = v; j <= target_n; ++j) {
            dp[j] = dp[j] + dp[j - v]; // Allman style
        }
    }

    return dp[target_n];
}

// 如果物品有数量限制，例如每种物品最多使用 k 次
// OGF 为 product_i (1 + x^(value_i) + ... + x^(k*value_i))
// 动态规划需要调整：
int calculate_ways_with_limit(int target_n, int limits[]) {
    initialize_dp(target_n);

    for (int i = 0; i < num_items; ++i) {
        int v = item_values[i]; // 当前物品的价值
        int k = limits[i];      // 当前物品的最多使用次数

        // 临时数组，存储上一轮（不考虑当前物品或只考虑更少次当前物品）的 dp 值
        // 避免在同一轮内，dp[j-v] 已经包含了使用当前物品 v 的信息，导致重复计数
        // 对于有限制的情况，我们需要用一个辅助数组来确保每一轮的计算是基于上一轮的结果
        // 这是一个更复杂的DP，或者说，有限制的问题直接用多项式乘法更直观
        // 但如果只有一种物品有限制，其他无限，可以用 OGF 乘法加 DP
        // 为了简化，这里不提供通用的有限制DP模板，因为通常直接用多项式乘法或更复杂的DP
        // 对于普通母函数，有限制通常直接体现在多项式因子上，然后通过多项式乘法解决。
        // 如果题目要求用DP，那通常是“背包问题”的变种。

        // 但如果非要用DP
