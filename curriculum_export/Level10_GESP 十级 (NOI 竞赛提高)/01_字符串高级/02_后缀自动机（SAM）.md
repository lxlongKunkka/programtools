## 趣味引入

同学们，想象一下我们有一个巨大的“子串迷宫”！迷宫里有各种各样的路径，每条路径都代表一个子串。现在，我们想做几件事：
1.  找出迷宫里所有不同的子串路径有多少条？
2.  给定两个迷宫，它们之间最长的公共路径是哪一条？
3.  某个特定的路径（模式串）在迷宫里出现了多少次？

传统的字符串算法，比如KMP、Z算法、Manacher，就像是迷宫里的一个个小工具，各自有擅长的领域，但都不能一劳永逸地解决所有问题。

今天，我们要介绍一个“超级导航系统”——**后缀自动机（Suffix Automaton，SAM）**！它就像一个神奇的地图，能够把一个字符串的所有子串信息，以最简洁、最有效的方式储存起来。有了它，上面提到的所有问题，以及更多复杂的字符串问题，都能迎刃而解！它不仅能识别所有后缀，更能识别所有子串，而且是**最小的**确定性有限状态自动机，效率非常高！是不是听起来很酷？

## 深度知识点讲解

### 1. SAM 是什么？

后缀自动机（SAM）是一个**最小的确定性有限状态自动机 (DFA)**，它能够识别给定字符串 `S` 的**所有后缀**。听起来有点绕口，但它的真正强大之处在于，它实际上能识别 `S` 的**所有子串**！
*   **为什么能识别所有子串？** 因为任何一个子串，都可以看作是 `S` 的某个后缀的一个前缀。如果SAM能识别所有后缀，并且能从起点走到任意一个状态，那它就能识别所有子串。
*   **“最小的DFA”** 意味着它用最少的节点和边，完成了识别所有后缀的任务，非常高效且紧凑。

**生活类比**：想象SAM是一个超级智能的“字符串百科全书”。你输入任何一个子串，它都能立刻告诉你这个子串在原字符串中是否存在。更厉害的是，这本百科全书的索引系统（SAM本身）是经过高度优化的，没有任何冗余信息，用最小的篇幅涵盖了所有子串的知识。

### 2. SAM 的核心构成

一个SAM由一系列**状态**和**转移边**组成，同时还有一个特殊的**后缀链接**结构。

#### 2.1 状态 (State)

*   SAM的节点就是状态。每个状态都代表一个或多个子串。
*   **endpos 集合**：这是理解SAM状态的关键！对于字符串 `S` 中的任意一个子串 `t`，它的 `endpos(t)` 集合定义为 `t` 在 `S` 中所有出现结束位置的集合。
    *   **例如**：`S = "ababa"`
        *   `endpos("a")` = {1, 3, 5} （索引从1开始）
        *   `endpos("b")` = {2, 4}
        *   `endpos("aba")` = {3, 5}
        *   `endpos("ababa")` = {5}
*   **状态的本质**：SAM中的每个状态都对应着一个唯一的 `endpos` 集合。所有 `endpos` 集合相同的子串，都会被同一个状态表示。它们是“等价”的。
    *   **类比**：`endpos` 集合就像是这些子串的“共同出生地”或“共同经历”。拥有完全相同经历的子串被归为一类，住在一个“状态房间”里。

#### 2.2 转移边 (Transition)

*   从状态 `u` 沿字符 `c` 到状态 `v`，表示如果 `u` 对应的某个子串后面接上字符 `c`，就能形成 `v` 对应的某个子串。
*   SAM的转移边构成了一个**有向无环图 (DAG)**。从起始状态（通常是状态0，代表空串）沿着转移边走，可以生成 `S` 的所有子串。
*   **类比**：就像地铁线路图。从一个站台（状态）乘坐一趟车（字符），就能到达另一个站台（状态），这个过程就像是把当前子串“延长”了一个字符。

#### 2.3 后缀链接 (Suffix Link)

*   每个状态 `u` 都有一个后缀链接 `link[u]`，指向另一个状态 `v`。
*   **核心性质**：`link[u]` 指向的状态 `v` 代表的 `endpos` 集合，是 `u` 代表的 `endpos` 集合的**真超集**。
    *   这意味着 `v` 代表的子串是 `u` 代表的子串的**最长后缀**，并且这个后缀的 `endpos` 集合与 `u` 的 `endpos` 集合不同。
*   **`len` 属性**：每个状态 `u` 还会存储它所代表的**最长子串的长度 `len[u]`**。
    *   那么，`link[u]` 对应的最长子串长度就是 `len[link[u]]`。
    *   状态 `u` 实际上代表了所有长度在 `(len[link[u]], len[u]]` 范围内的子串。这些子串的 `endpos` 集合都与 `u` 的 `endpos` 集合相同。
*   **类比**：后缀链接就像家族树里的“父子关系”或“祖先关系”。一个状态 `u` 代表的子串 `T`，它的后缀链接 `link[u]` 就代表 `T` 的“父辈”子串——它是 `T` 的一个后缀，而且是“最长”的那个后缀，但又不能和 `T` 完全一样（即 `endpos` 集合不同）。它提供了一条“快速通道”，让我们从一个长子串的代表状态，快速找到它的更短的后缀的代表状态。

### 3. SAM 的在线构建算法 (增量法)

SAM最神奇的地方在于它可以**在线构建**：从空字符串开始，每次添加一个字符，就能逐步构建出最终的SAM。这个过程被称为**增量法**。

**核心思想**：每次添加一个新字符 `c`，我们都要在当前SAM中找到所有以 `c` 结尾的新子串，并将它们正确地加入到SAM中。

**构建步骤分解 (以添加字符 `c` 为例)**：
假设我们已经构建了字符串 `S` 的SAM，现在要添加字符 `c`，形成 `S+c`。

1.  **创建新状态 `cur`**:
    *   `cur = sz++` (创建一个新状态，`sz` 是当前状态总数)。
    *   `sam[cur].len = sam[last].len + 1`。（`last` 是上次添加字符后代表整个字符串 `S` 的状态）。
    *   新状态 `cur` 将代表 `S+c` 这个新字符串，以及所有以 `S+c` 结尾的新子串。

2.  **处理后缀链接**:
    *   从 `last` 状态开始，沿着后缀链接 `p = sam[last].link` 向上跳。
    *   **循环条件**: `p != -1` (还没跳到根节点上方) 并且 `sam[p].ch[c]` 不存在 (状态 `p` 没有字符 `c` 的转移)。
    *   **循环内操作**: `sam[p].ch[c] = cur`（将 `p` 的字符 `c` 转移指向 `cur`）。然后 `p = sam[p].link`（继续向上跳）。
    *   **循环结束后的判断**:
        *   **情况 A: `p == -1`** (跳到了根节点上方)。
            *   这说明字符 `c` 是一个全新的前缀，`cur` 没有更短的后缀可以共用 `endpos` 集合，所以 `sam[cur].link = 0` (指向根节点，代表空串)。
        *   **情况 B: `p != -1`** (找到了一个状态 `p`，它有字符 `c` 的转移)。
            *   设 `q = sam[p].ch[c]`。`q` 是 `p` 经过 `c` 转移到的状态。
            *   **子情况 B1: `sam[q].len == sam[p].len + 1`**
                *   这表示 `q` 状态所代表的最长子串，正好就是 `p` 状态所代表的最长子串后面接上 `c` 得到的。这意味着 `q` 已经是一个“完美”的后缀链接目标了，它的 `endpos` 集合恰好是 `cur` 对应的某些短后缀的 `endpos` 集合。
                *   所以，`sam[cur].link = q`。
            *   **子情况 B2: `sam[q].len > sam[p].len + 1`**
                *   这说明 `q` 状态所代表的最长子串，比 `p` 状态所代表的最长子串后面接上 `c` 得到的要长。这意味着 `q` 状态包含了太多的子串，它的 `endpos` 集合比我们需要的要大（或者说，它不仅包含了 `p` 接 `c` 得到的子串，还包含了更长的、`endpos` 集合不同的子串）。
                *   这时，我们需要**分裂状态 `q`**。
                    *   创建一个新状态 `nq = sz++`。
                    *   `sam[nq].len = sam[p].len + 1`。
                    *   `nq` 复制 `q` 的所有转移边：`sam[nq].ch = sam[q].ch`。
                    *   `nq` 的后缀链接指向 `q` 的后缀链接：`sam[nq].link = sam[q].link`。
                    *   **关键步骤**: 调整后缀链接：`sam[q].link = nq`，`sam[cur].link = nq`。现在 `q` 和 `cur` 都指向了新的、更精确的 `nq`。
                    *   **继续向上跳**: 从 `p` 状态开始，沿着后缀链接继续向上跳。只要 `sam[p'].ch[c] == q` (即 `p'` 通过 `c` 转移到了旧的 `q` 状态)，就将 `sam[p'].ch[c]` 改为 `nq`。直到 `p' == -1` 或 `sam[p'].ch[c] != q`。

3.  **更新 `last`**: `last = cur`。

**为什么是 `len[q] == len[p] + 1`？为什么需要分裂？**
*   **`len[q] == len[p] + 1`**: 表示状态 `q` 恰好代表了所有以 `p` 对应的最长子串加上 `c` 结尾的子串。它的 `endpos` 集合是精确的，可以直接作为 `cur` 的后缀链接。
*   **`len[q] > len[p] + 1`**: 表示 `q` 状态代表的子串比 `p` 接 `c` 得到的子串要长。`q` 的 `endpos` 集合包含了 `p` 接 `c` 得到的子串的 `endpos` 集合，可能还包含了其他子串的 `endpos` 集合。为了保持SAM的“最小性”和 `endpos` 集合的“纯粹性”，我们必须把 `q` 分裂。
    *   `nq` 状态将专门负责 `p` 接 `c` 得到的子串，以及所有 `endpos` 集合与它相同的短子串。
    *   `q` 状态则继续负责那些更长的、`endpos` 集合不同的子串。
    *   **类比**：想象 `q` 像一个大盒子，里面装着不同长度的子串。现在 `p` 接 `c` 形成了一个新的子串，它应该和 `q` 盒子里的某些短子串放在一起。如果 `q` 盒子里的最长子串比 `p` 接 `c` 得到的子串长太多，这个 `q` 盒子就“不纯了”。我们必须把 `q` 盒子里的短子串拿出来，放到一个新盒子 `nq` 里，让 `nq` 成为 `cur` 和 `q` 的“共同祖先”，这样才能保证每个盒子（状态）里的子串都严格符合 `endpos` 集合的定义。

**常见误区**

*   **混淆 `len` 和 `endpos` 集合大小**：`len[u]` 是状态 `u` 代表的最长子串的长度，而不是 `endpos` 集合的大小。
*   **混淆后缀链接和转移边**：后缀链接构成一棵树（或森林），而转移边构成一个DAG。它们是两个不同的结构。
*   **状态分裂的逻辑错误**：这是SAM构建中最复杂的部分。理解 `len[q] == sam[p].len + 1` 和 `len[q] > sam[p].len + 1` 两种情况的本质区别至关重要。

### **典型例题精讲**

**例题 1: 统计不同子串的数量**

*   **题目**: 给定一个字符串 `S`，求它有多少个不同的子串。
*   **思路分析**:
    *   SAM的每个状态 `u` 代表一个唯一的 `endpos` 集合。这个集合中的所有子串，都是不同的。
    *   一个状态 `u` 实际代表的子串长度范围是 `(sam[sam[u].link].len, sam[u].len]`。
    *   因此，状态 `u` 贡献了 `sam[u].len - sam[sam[u].link].len` 个不同的子串。
    *   将所有状态（除了起始状态0，因为它代表空串）贡献的子串数量加起来，就是总的不同子串数量。
*   **代码实现 (核心部分)**:
    ```cpp
    // 假设SAM已经构建完成，sz是状态总数
    long long ans = 0;
    for (int i = 1; i < sz; ++i) { // 状态0代表空串，不计入，所以从1开始
        ans += (long long)sam[i].len - sam[sam[i].link].len;
    }
    // ans 即为不同子串的数量
    ```
*   **复杂度**: SAM 构建 O(N)，遍历 O(N)。总 O(N)。

**例题 2: 两个字符串的最长公共子串**

*   **题目**: 给定两个字符串 `S1` 和 `S2`，求它们的最长公共子串。
*   **思路分析**:
    1.  对 `S1` 构建 SAM。这个SAM包含了 `S1` 的所有子串信息。
    2.  用 `S2` 在 `S1` 的 SAM 上进行匹配。
        *   我们维护两个变量：`p` 表示当前在 `S1` 的 SAM 中匹配到的状态，`l` 表示当前匹配到的长度。
        *   遍历 `S2` 的每个字符 `c`：
            *   **尝试匹配**: 如果当前状态 `p` 有字符 `c` 的转移边 `sam[p].ch[c]`，我们就沿着这条边走：`p = sam[p].ch[c]`，`l++`。
            *   **无法匹配**: 如果 `p` 没有字符 `c` 的转移边，说明当前匹配的子串 `S2[...i]` 无法通过 `c` 延长。我们需要寻找 `p` 的一个后缀，看它是否能与 `c` 匹配。
                *   沿着 `p` 的后缀链接向上跳：`p = sam[p].link`。
                *   每跳一次，当前匹配长度 `l` 也要相应地更新为 `sam[p].len`。
                *   重复这个过程，直到找到一个 `p'` 有字符 `c` 的转移边，或者 `p'` 跳到根节点（`p' == 0`）。
                *   如果找到 `p'` 有 `c` 的转移：`p = sam[p'].ch[c]`，`l = sam[p'].len + 1`。
                *   如果跳到根节点 `0` 都没有找到：说明 `c` 无法作为当前子串的延续，需要从 `S1` 的SAM的根节点重新开始匹配：`p = 0`，`l = 0`。
            *   在每一步，我们都更新 `max_l = max(max_l, l)`，记录当前找到的最长公共子串长度。
*   **代码实现 (核心部分)**:
    ```cpp
    // 假设SAM for S1 已经构建完成
    // S2 是要匹配的字符串
    std::string s2_str; // 从用户输入获取 S2
    // std::cin >> s2_str; 

    int p = 0; // 当前在SAM中的状态，初始为根节点
    int l = 0; // 当前匹配到的长度
    int max_l = 0; // 记录最长公共子串的长度
    
    for (char c_char : s2_str) {
        int c_idx = c_char - 'a'; // 假设是小写字母
        // 如果当前状态p没有c_idx的转移，或者转移到的状态不合法（这里用0判断）
        // 就沿着后缀链接向上跳，回溯匹配长度
        while (p != 0 && sam[p].next_ch[c_idx] == 0) { 
            p = sam[p].link;
            l = sam[p].len; // 匹配长度回溯到后缀链接状态的最长长度
        }
        if (sam[p].next_ch[c_idx] != 0) { // 如果找到了c_idx的转移
            p = sam[p].next_ch[c_idx];
            l++; // 匹配长度增加1
        } else { // 没找到任何匹配，从根节点重新开始
            p = 0;
            l = 0;
        }
        max_l = std::max(max_l, l); // 更新最长公共子串的长度
    }
    // max_l 即为最长公共子串的长度
    ```
*   **复杂度**: SAM 构建 O(N1)，匹配 O(N2)。总 O(N1 + N2)。

### **代码实现模板**

```cpp
#include <iostream>
#include <string>
#include <vector> // GESP 10级允许使用vector，但SAM核心结构我们会尽量用原生数组模拟
#include <algorithm> // for std::max

// 使用MAX_STATES定义最大状态数，防止越界
const int MAX_LEN = 100005; // 原始字符串最大长度
const int MAX_STATES = MAX_LEN * 2; // SAM状态数最多是2 * MAX_LEN - 1，留足空间

// SAM节点结构体
struct State {
    int len; // 状态表示的最长子串的长度
    int link; // 后缀链接
    int next_ch[26]; // 转移边，用固定大小数组模拟map，0表示没有该转移

    // 构造函数，初始化状态
    State() : len(0), link(0) {
        for (int i = 0; i < 26; ++i) {
            next_ch[i] = 0;
        }
    }
};

State sam[MAX_STATES]; // SAM的状态数组，全局变量
int sz; // 当前SAM中的状态数量，0是初始状态
int last; // 上一次添加字符后，代表整个字符串的状态

// 初始化SAM
void sam_init() {
    sz = 1; // 初始时只有一个状态0，代表空字符串
    last = 0; // last指向初始状态0
    sam[0].len = 0;
    sam[0].link = -1; // 根节点（状态0）的后缀链接通常设为-1或特殊值，表示无更短后缀
    // sam[0].next_ch 在构造函数中已清空
}

// 向SAM中添加一个字符c_char
void sam_extend(char c_char) {
    int c_idx = c_char - 'a'; // 将字符映射到0-25的数字索引

    int cur = sz++; // 创建新状态，编号为sz，并递增sz
    sam[cur].len = sam[last].len + 1; // 新状态的len是last状态的len+1

    int p = last; // 从last状态开始沿着后缀链接向上跳
    // 循环条件：p未跳到根节点上方（-1）且p没有字符c_idx的转移
    while (p != -1 && sam[p].next_ch[c_idx] == 0) {
        sam[p].next_ch[c_idx] = cur; // 为p添加指向cur的字符c_idx转移
        p = sam[p].link; // 继续向上跳
    }

    if (p == -1) { // 如果p跳到了根节点上方（-1），说明c_idx是全新的前缀
        sam[cur].link = 0; // cur的后缀链接指向根节点0
    } else {
        // 找到了一个状态p，它有字符c_idx的转移
        int q = sam[p].next_ch[c_idx]; // q是p通过c_idx转移到的状态
