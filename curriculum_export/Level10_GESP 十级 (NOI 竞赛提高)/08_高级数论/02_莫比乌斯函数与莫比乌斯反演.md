## 深度知识点讲解

### 1. 莫比乌斯函数 (Möbius Function, μ)

莫比乌斯函数 `mu(n)` 是数论中一个非常重要的函数，它以19世纪德国数学家奥古斯特·费迪南德·莫比乌斯的名字命名。它看起来定义有些古怪，但其背后蕴藏着巨大的能量。

**定义：**
对于正整数 `n`，莫比乌斯函数 `mu(n)` 的定义如下：
1.  **`mu(1) = 1`**
2.  **`mu(n) = 0`** 如果 `n` 含有平方因子。
    *   什么叫平方因子？就是 `n` 可以被某个质数的平方整除。
    *   例如：`mu(4)=0` (因为 `2^2 | 4`)，`mu(12)=0` (因为 `2^2 | 12`)，`mu(18)=0` (因为 `3^2 | 18`)。
3.  **`mu(n) = (-1)^k`** 如果 `n` 是 `k` 个不同质数的乘积。
    *   也就是说，`n` 是一个“无平方因子数”（square-free number），可以表示成 `n = p1 * p2 * ... * pk`，其中 `p1, p2, ..., pk` 都是互不相同的质数。
    *   例如：
        *   `mu(2) = -1` (2是1个质数)
        *   `mu(3) = -1` (3是1个质数)
        *   `mu(6) = mu(2*3) = (-1)^2 = 1` (6是2个不同质数乘积)
        *   `mu(30) = mu(2*3*5) = (-1)^3 = -1` (30是3个不同质数乘积)

**生活类比：**
你可以把 `mu(n)` 想象成一个“质因数质量检测器”：
*   如果 `n` 是 `1`，它给你一个“完美”的 `1` 分。
*   如果 `n` 里面混入了“重复的质因数”（比如 `4 = 2*2`，`12 = 2*2*3`），它认为这个数“不纯粹”，直接给 `0` 分。
*   如果 `n` 是由“不同”的质因数构成的（比如 `6 = 2*3`，`30 = 2*3*5`），它会根据质因数的数量给你打分：奇数个质因数就扣分 (`-1`)，偶数个质因数就加分 (`+1`)。

**关键性质：**
莫比乌斯函数最核心、最强大的性质是：
对于任意正整数 `n`，有：
`sum_{d|n} mu(d) = [n=1]`

其中 `[n=1]` 是艾弗森括号，表示当 `n=1` 时值为 `1`，否则为 `0`。
这个性质是莫比乌斯反演的基石，它就像一个“魔法开关”：当 `n=1` 时，它启动；当 `n > 1` 时，它关闭。

**为什么这个性质成立？**
1.  **当 `n = 1` 时：**
    `sum_{d|1} mu(d) = mu(1) = 1`。这符合 `[n=1]` 的定义。

2.  **当 `n > 1` 时：**
    设 `n` 的标准分解式为 `n = p1^a1 * p2^a2 * ... * pk^ak`，其中 `p1, ..., pk` 是互不相同的质数，`ai >= 1`。
    要计算 `sum_{d|n} mu(d)`，我们只需要考虑那些 `mu(d)` 不为 `0` 的因子 `d`。根据定义，`mu(d)` 不为 `0` 当且仅当 `d` 是一个无平方因子数。
    这意味着 `d` 只能是 `p1, p2, ..., pk` 这些质数中，选取若干个相乘得到的。
    例如，如果 `n = p1^a1 * p2^a2`，那么 `d` 可能是 `1`, `p1`, `p2`, `p1*p2`。
    所以，`sum_{d|n} mu(d)` 实际上等于 `sum_{d | (p1*p2*...*pk)} mu(d)`。
    我们来列出这些 `d`：
    *   选择 `0` 个质数：`d=1`，`mu(1) = 1`。
    *   选择 `1` 个质数：`d=p1, p2, ..., pk`，共有 `C(k, 1)` 种，每个 `mu(d) = -1`。总和为 `C(k, 1) * (-1)`。
    *   选择 `2` 个质数：`d=p1*p2, ...`，共有 `C(k, 2)` 种，每个 `mu(d) = (-1)^2 = 1`。总和为 `C(k, 2) * 1`。
    *   ...
    *   选择 `k` 个质数：`d=p1*p2*...*pk`，共有 `C(k, k)` 种，每个 `mu(d) = (-1)^k`。总和为 `C(k, k) * (-1)^k`。

    因此，`sum_{d|n} mu(d) = C(k, 0)*1 + C(k, 1)*(-1) + C(k, 2)*1 + ... + C(k, k)*(-1)^k`
    这正是二项式展开 `(1 + x)^k` 在 `x = -1` 时的结果：
    `sum_{i=0 to k} C(k, i) * (-1)^i = (1 + (-1))^k = 0^k`。
    由于 `n > 1`，所以 `k >= 1`，因此 `0^k = 0`。
    所以，对于 `n > 1`，`sum_{d|n} mu(d) = 0`。

    这个性质非常巧妙，它能够“检测”一个数是否为 `1`，这在数论求和问题中有着魔术般的作用。

**莫比乌斯函数的计算 (线性筛法)：**
由于 `mu(n)` 是一个**积性函数**（即当 `gcd(a, b) = 1` 时，`mu(a*b) = mu(a) * mu(b)`），我们可以利用线性筛在 `O(N)` 的时间复杂度内预处理出 `1` 到 `N` 范围内的所有 `mu` 值。

回忆线性筛素数的过程：每个合数只会被它的最小质因子筛掉一次。
我们可以在筛素数的同时计算 `mu` 值：
*   `mu[1] = 1`。
*   当 `i` 是素数 `p` 时：`mu[p] = -1`。
*   当 `i` 是合数 `i = j * p` (其中 `p` 是 `i` 的最小质因子) 时：
    *   **情况1：`j` 能被 `p` 整除 (即 `p | j`)。** 这意味着 `i` 含有 `p^2` 因子。根据定义，`mu[i] = 0`。
    *   **情况2：`j` 不能被 `p` 整除 (即 `p !| j`)。** 这意味着 `gcd(j, p) = 1`。因为 `mu` 是积性函数，所以 `mu[i] = mu[j * p] = mu[j] * mu[p] = mu[j] * (-1)`。

### 2. 莫比乌斯反演 (Möbius Inversion)

莫比乌斯反演是一种**反转求和关系**的强大工具。它通常用于解决这样一类问题：我们知道一个函数 `F(n)` 是另一个函数 `f(d)` 在 `n` 的所有因子上的求和，但我们想知道 `f(n)` 是什么。莫比乌斯函数就是实现这种“反转”的关键。

**基本形式一：**
如果存在两个数论函数 `f(n)` 和 `F(n)`，满足关系：
`F(n) = sum_{d|n} f(d)` (对 `n` 的所有因子 `d` 求和)

那么，我们可以通过莫比乌斯函数反演出 `f(n)`：
`f(n) = sum_{d|n} mu(d) F(n/d)` (对 `n` 的所有因子 `d` 求和)

**推导：**
我们尝试将 `sum_{d|n} mu(d) F(n/d)` 展开，看它是否真的等于 `f(n)`。
`sum_{d|n} mu(d) F(n/d)`
将 `F(n/d)` 替换为它的定义 `sum_{k|(n/d)} f(k)`：
`= sum_{d|n} mu(d) sum_{k|(n/d)} f(k)`

现在，我们交换求和顺序。外层循环 `d` 遍历 `n` 的所有因子，内层循环 `k` 遍历 `n/d` 的所有因子。
这等价于：`k` 遍历 `n` 的所有因子，`d` 遍历 `n/k` 的所有因子，且 `d` 必须是 `n` 的因子。
`= sum_{k|n} f(k) sum_{d|(n/k) 且 d|n} mu(d)`
由于 `k|n` 且 `d|(n/k)`，那么 `d` 必然是 `n` 的因子，所以 `d|n` 这个条件是冗余的。
`= sum_{k|n} f(k) sum_{d|(n/k)} mu(d)`

现在，我们看到了熟悉的部分：`sum_{d|(n/k)} mu(d)`！
根据莫比乌斯函数的核心性质，`sum_{d|(n/k)} mu(d) = [n/k = 1]`。
所以，只有当 `n/k = 1` 时（即 `k = n` 时），内层求和的结果才为 `1`；否则为 `0`。
`= sum_{k|n} f(k) * [n/k = 1]`

当 `k = n` 时，这一项是 `f(n) * 1 = f(n)`。
当 `k != n` 时，这一项是 `f(k) * 0 = 0`。
所以，整个求和只剩下 `f(n)`。
`= f(n)`

完美！我们成功地从 `F(n)` 反演出了 `f(n)`。

**核心思想：**
莫比乌斯反演就像一个“解密器”，当你有一个函数 `F(n)` 包含了 `f(d)` 的信息（通过因子求和的方式），它能帮你把纯粹的 `f(n)` 给“提取”出来。在很多计数问题中，我们很难直接计算 `f(n)`，但容易计算 `F(n)`（通常 `F(n)` 涉及的是倍数关系，而非严格的 `gcd` 关系），这时莫比乌斯反演就派上大用场了。

**基本形式二 (倍数形式)：**
如果存在两个数论函数 `f(n)` 和 `F(n)`，满足关系：
`F(n) = sum_{n|d, d<=M} f(d)` (对 `n` 的所有倍数 `d` 求和，且 `d` 不超过 `M`)

那么，我们可以通过莫比乌斯函数反演出 `f(n)`：
`f(n) = sum_{n|d, d<=M} mu(d/n) F(d)`

这个形式在一些问题中也很有用，它的推导与形式一类似，留给同学们课后思考。在实际应用中，形式一更为常见，或者通过变形，许多形式二的问题也可以用形式一的思路解决。

### 3. 应用模式：解决与 `gcd` 相关的求和问题

莫比乌斯反演最经典的应用场景就是解决涉及 `gcd(i, j) = k` 或 `gcd(i, j)` 的求和问题。

**常见转化技巧：**
遇到 `sum_{i=1 to N} sum_{j=1 to M} [gcd(i, j) == k]` 这种形式，我们可以：
1.  **设 `f(x)`：** 定义 `f(x)` 为满足 `gcd(i, j) = x` 的 `(i, j)` 对的数量。
2.  **设 `F(x)`：** 定义 `F(x)` 为满足 `x | gcd(i, j)` 的 `(i, j)` 对的数量。
    *   注意 `x | gcd(i, j)` 等价于 `x | i` 且 `x | j`。
    *   那么 `F(x)` 的计算就变得简单了：有多少个 `i` ( `1 <= i <= N`) 是 `x` 的倍数？`floor(N/x)` 个。
    *   有多少个 `j` ( `1 <= j <= M`) 是 `x` 的倍数？`floor(M/x)` 个。
    *   所以 `F(x) = floor(N/x) * floor(M/x)`。
3.  **建立 `F(x)` 和 `f(d)` 的关系：**
    如果 `x | gcd(i, j)`，那么 `gcd(i, j)` 可能是 `x, 2x, 3x, ...` 等 `x` 的倍数。
    所以，`F(x) = sum_{x|d} f(d)`。
    这个关系正好是莫比乌斯反演的第二种形式！
4.  **反演得到 `f(x)`：**
    `f(x) = sum_{x|d} mu(d/x) F(d)`
    `f(x) = sum_{x|d} mu(d/x) * floor(N/d) * floor(M/d)`
    通常我们要求的是 `f(1)` (即 `gcd(i, j) == 1` 的对数)，那么：
    `f(1) = sum_{d=1 to min(N,M)} mu(d) * floor(N/d) * floor(M/d)`

**优化：数论分块 (整除分块)**
在上面的公式中，我们注意到 `floor(N/d)` 和 `floor(M/d)` 的值是分段常数。对于 `d` 在 `1` 到 `N` 的范围内，`floor(N/d)` 最多只有 `2 * sqrt(N)` 种取值。我们可以利用这个性质，将求和区间分成若干块，每块内部 `floor(N/d)` 和 `floor(M/d)` 的值都相同，从而快速计算它们的和。
具体来说，对于一个给定的 `d`，`floor(N/d)` 的值在 `[d, N / floor(N/d)]` 这个区间内都是相同的。我们可以找到下一个 `d'` 使得 `floor(N/d')` 发生变化，即 `d' = N / floor(N/d) + 1`。对 `M` 也做同样处理，取两者 `d'` 的最小值作为下一个分块的起点。这样，可以在 `O(sqrt(N))` 的时间内完成求和。

## 典型例题精讲

**例题1：求 `sum_{i=1 to n} sum_{j=1 to m} [gcd(i, j) == 1]`**
（统计 `1 <= i <= n, 1 <= j <= m` 中，`gcd(i, j) = 1` 的对数）

**思路分析：**
1.  **定义函数：**
    设 `f(x)` 为满足 `gcd(i, j) = x` 的 `(i, j)` 对数。
    设 `F(x)` 为满足 `x | gcd(i, j)` 的 `(i, j)` 对数。
2.  **建立关系：**
    显然，`F(x) = sum_{x|d} f(d)`。
    因为如果 `x | gcd(i, j)`，那么 `gcd(i, j)` 肯定是 `x` 的某个倍数 `d`。
3.  **计算 `F(x)`：**
    `x | gcd(i, j)` 等价于 `x | i` 且 `x | j`。
    在 `1` 到 `n` 中，`x` 的倍数有 `floor(n/x)` 个。
    在 `1` 到 `m` 中，`x` 的倍数有 `floor(m/x)` 个。
    所以 `F(x) = floor(n/x) * floor(m/x)`。
4.  **莫比乌斯反演：**
    根据莫比乌斯反演的第二种形式，由 `F(x) = sum_{x|d} f(d)` 可得：
    `f(x) = sum_{x|d} mu(d/x) F(d)`
    我们要求的答案是 `f(1)` (即 `gcd(i, j) = 1` 的对数)。
    `f(1) = sum_{d=1 to min(n,m)} mu(d/1) F(d)`
    `f(1) = sum_{d=1 to min(n,m)} mu(d) * floor(n/d) * floor(m/d)`
5.  **优化：数论分块**
    这个求和式子可以使用数论分块进行优化，将 `O(min(n,m))` 的复杂度降低到 `O(sqrt(min(n,m)))`。

**代码实现 (伪代码):**

```cpp
const int MAXN = 1e7 + 5; // 根据N, M的范围设定
int mu[MAXN];
int primes[MAXN];
bool is_prime[MAXN];
int cnt_primes = 0;

void sieve(int n) { // 线性筛计算mu函数
    mu[1] = 1;
    for (int i = 0; i <= n; ++i) {
        is_prime[i] = true;
    }
    is_prime[0] = is_prime[1] = false;

    for (int i = 2; i <= n; ++i) {
        if (is_prime[i]) {
            primes[cnt_primes++] = i;
            mu[i] = -1; // i是素数，只有一个质因子
        }
        for (int j = 0; j < cnt_primes && i * primes[j] <= n; ++j) {
            is_prime[i * primes[j]] = false;
            if (i % primes[j] == 0) { // primes[j]是i的最小质因子，所以i*primes[j]有平方因子
                mu[i * primes[j]] = 0;
                break;
            } else { // primes[j]不是i的因子，gcd(i, primes[j]) = 1
                mu[i * primes[j]] = mu[i] * mu[primes[j]]; // 积性函数性质
            }
        }
    }
}

long long solve_example1(int n, int m) {
    if (n > m) { // 保证n <= m，便于处理
        int temp = n;
        n = m;
        m = temp;
    }
    
    // 预处理mu函数，最大范围取max(n, m)
    // 假设sieve(MAXN-1)已在主函数或初始化时调用
    
    long long ans = 0;
    for (int l = 1, r; l <= n; l = r + 1) { // 数论分块
        r = min(n / (n / l), m / (m / l)); // 计算当前块的右边界
        ans += (long long)(mu[r] - mu[l - 1]) * (n / l) * (m / l); // 注意mu需要前缀和
    }
    return ans;
}
```
**复杂度分析：**
*   线性筛预处理 `mu`：`O(MAXN)`。
*   求解 `f(1)`：数论分块，`O(sqrt(min(n,m)))`。
*   总复杂度：`O(MAXN + sqrt(min(n,m)))`。

**例题2：求 `sum_{i=1 to n} sum_{j=1 to m} gcd(i, j)`**
（计算 `1 <= i <= n, 1 <= j <= m` 中，所有 `gcd(i, j)` 的和）

**思路分析：**
1.  **转化求和：**
    `Ans = sum_{k=1 to min(n,m)} k * (满足 gcd(i,j)=k 的 (i,j) 对数)`
    这里 `(满足 gcd(i,j)=k 的 (i,j) 对数)` 正是例题1中的 `f(k)`。
    所以 `Ans = sum_{k=1 to min(n,m)} k * f(k)`。
2.  **代入反演公式：**
    `f(k) = sum_{k|d} mu(d/k) * floor(n/d) * floor(m/d)`
    将 `f(k)` 代入 `Ans`：
    `Ans = sum_{k=1 to min(n,m)} k * sum_{k|d} mu(d/k) * floor(n/d) * floor(m/d)`
3.  **交换求和顺序 (关键一步)：**
    为了方便计算，我们交换 `k` 和 `d` 的求和顺序。
    `d` 的范围是从 `1` 到 `min(n,m)`。
    对于一个确定的 `d`，`k` 必须是 `d` 的因子 (因为 `k|d`)，且 `k` 必须小于等于 `min(n,m)`。
    令 `T = d/k`，那么 `d = k * T`。
    原式变为：
    `Ans = sum_{d=1 to min(n,m)} floor(n/d) * floor(m/d) * sum_{k|d} k * mu(d/k)`

    **解释交换过程：**
    *   原式是先枚举 `k`，再枚举 `k` 的倍数 `d`。
    *   现在我们先枚举 `d` (作为 `gcd` 的倍数)，再枚举 `d` 的因子 `k`。
    *   对于每个 `d`，`floor(n/d) * floor(m/d)` 是一个公共因子。
    *   内层求和 `sum_{k|d} k * mu(d/k)` 是一个关于 `d` 的函数。
4.  **内层求和的化简：**
    令 `g(D) = sum_{k|D} k * mu(D/k)`。
    这个 `g(D)` 是两个函数的狄利克雷卷积：`id * mu`。
    其中 `id(k) = k` (恒等函数)。
    我们知道欧拉函数 `phi(n)` 的性质是 `sum_{d|n} phi(d) = n` (即 `phi * 1 = id`)。
    将 `phi * 1 = id` 两边同时卷积 `mu`：
    `(phi * 1) * mu = id * mu`
    `phi * (1 * mu) = id * mu`
    我们知道 `1 * mu = epsilon` (单位元函数，`epsilon(1)=1`, `epsilon(n)=0` for `n>1`)。
    所以 `phi * epsilon = id * mu`。
    而 `phi * epsilon = phi`。
    因此，`id * mu = phi`。
    也就是说，`g(D) = sum_{k|D} k * mu(D/k) = phi(D)` (欧拉函数)。
5.  **最终公式：**
    `Ans = sum_{D=1 to min(n,m)} phi(D) * floor(n/D) * floor(m/D)`
6.  **优化：数论分块**
    这个公式同样可以使用数论分块进行优化。我们需要预处理 `phi` 函数和 `phi` 的前缀和。
