## 趣味引入

同学们，想象一下，你是一个拥有神奇力量的炼金术士！你手上有两种神秘的魔法药水：
第一种药水A，它的配方是 `A(x) = a_0 + a_1x + a_2x^2`，其中 `a_i` 代表第 `i` 种魔法材料的用量。
第二种药水B，它的配方是 `B(x) = b_0 + b_1x + b_2x^2`。
现在，你的任务是把这两种药水混合，生成一种全新的超级药水C！新药水C的配方 `C(x)` 是由 `A(x)` 和 `B(x)` 相乘得到的：`C(x) = A(x) * B(x)`。

**思考一下：** 如果用我们小学学过的乘法，`C(x)` 的每个系数 `c_k` 会怎么算呢？
例如，`c_0` 是 `a_0 * b_0`。
`c_1` 是 `a_0 * b_1 + a_1 * b_0`。
`c_2` 是 `a_0 * b_2 + a_1 * b_1 + a_2 * b_0`。

看起来，每个 `c_k` 都是由一些 `a_i * b_j` 的和组成的，而且这些 `i` 和 `j` 总是满足 `i + j = k`。
如果我们的药水配方很长，有成千上万种材料（多项式项数很高），我们一个一个地这样计算，是不是太慢了？这就像用普通加法器计算1000位数的乘法，会非常耗时。

今天，我们就来学习一种“魔法棒”——快速傅里叶变换（FFT）和数论变换（NTT），它们能以惊人的速度完成多项式乘法，就像瞬间炼制出超级药水一样！这不仅仅是为了炼金，在很多计算机科学和数学问题中，比如信号处理、数据分析、密码学，甚至是在我们熟悉的组合计数问题中，这种“魔法”都无处不在。

## 深度知识点讲解

### 1. 多项式乘法与卷积的本质

我们先从最基础的定义开始。
一个多项式 `A(x)` 可以表示为 `A(x) = a_0x^0 + a_1x^1 + a_2x^2 + ... + a_{n-1}x^{n-1}`。我们通常只关心它的系数序列 `[a_0, a_1, ..., a_{n-1}]`。
当两个多项式 `A(x)` 和 `B(x)` 相乘得到 `C(x)` 时：
`C(x) = A(x) * B(x) = (a_0 + a_1x + ...)(b_0 + b_1x + ...)`
展开后，`C(x) = c_0 + c_1x + c_2x^2 + ...`
其中，`c_k` 的计算公式是：
`c_k = \sum_{i=0}^{k} a_i \cdot b_{k-i}`

这个求和的形式 `c_k = \sum_{i=0}^{k} a_i \cdot b_{k-i}`，就是我们常说的**卷积 (Convolution)**。
**生活类比：**
*   **掷骰子求和：** 假设你掷两个骰子，第一个骰子点数 `i` 的概率是 `P_A(i)`，第二个骰子点数 `j` 的概率是 `P_B(j)`。那么，两个骰子点数之和为 `k` 的概率 `P_C(k)` 就是 `\sum_{i=1}^{k-1} P_A(i) \cdot P_B(k-i)`。这正是卷积的形式！我们可以用多项式 `A(x) = \sum P_A(i)x^i` 和 `B(x) = \sum P_B(j)x^j` 来表示，它们的乘积 `C(x)` 的系数 `c_k` 就是和为 `k` 的概率。
*   **商品组合：** 假设你有两堆商品，第一堆有 `a_i` 种价格为 `i` 的商品，第二堆有 `b_j` 种价格为 `j` 的商品。如果你从两堆中各取一件，总价格为 `k` 的组合有多少种？答案就是 `c_k = \sum a_i \cdot b_{k-i}`。

**朴素算法：** 直接按照定义计算 `c_k`，每个 `c_k` 需要 `O(k)` 次乘法和加法。如果 `A(x)` 和 `B(x)` 都有 `N` 项，那么 `C(x)` 最多有 `2N-1` 项，总时间复杂度是 `O(N^2)`。当 `N` 达到 `10^5` 时，`N^2` 就是 `10^{10}`，这太慢了！

### 2. FFT/NTT的魔法：从系数到点值

FFT/NTT 解决 `O(N^2)` 瓶颈的关键在于：**多项式有两种表示方法**。
1.  **系数表示 (Coefficient Representation)**：就是我们熟悉的 `A(x) = a_0 + a_1x + ... + a_{n-1}x^{n-1}`，用系数序列 `[a_0, a_1, ..., a_{n-1}]` 表示。
2.  **点值表示 (Point-Value Representation)**：一个 `N-1` 次多项式可以由 `N` 个不同的点 `(x_0, y_0), (x_1, y_1), ..., (x_{N-1}, y_{N-1})` 唯一确定，其中 `y_i = A(x_i)`。

**为什么点值表示很棒？**
如果 `A(x)` 在 `x_0, x_1, ..., x_{N-1}` 处的点值是 `[A(x_0), A(x_1), ..., A(x_{N-1})]`。
如果 `B(x)` 在 `x_0, x_1, ..., x_{N-1}` 处的点值是 `[B(x_0), B(x_1), ..., B(x_{N-1})]`。
那么，它们的乘积 `C(x) = A(x) * B(x)` 在这些点处的点值是多少呢？
很简单！`C(x_i) = A(x_i) * B(x_i)`。
所以，在点值表示下，多项式乘法就变成了**对应点值相乘**，这只需要 `O(N)` 次乘法！

**FFT/NTT 的核心思路：**
1.  **求值 (Evaluate)**：将 `A(x)` 和 `B(x)` 从系数表示转换为点值表示。这个过程就是**离散傅里叶变换 (DFT)**，由FFT快速完成，时间复杂度 `O(N log N)`。
2.  **点乘 (Pointwise Multiply)**：将 `A(x)` 和 `B(x)` 的点值表示在对应点上相乘，得到 `C(x)` 的点值表示。时间复杂度 `O(N)`。
3.  **插值 (Interpolate)**：将 `C(x)` 从点值表示转换回系数表示。这个过程是**逆离散傅里叶变换 (IDFT)**，也由FFT快速完成，时间复杂度 `O(N log N)`。

整个过程的总时间复杂度为 `O(N log N)`，相比 `O(N^2)` 是一个巨大的提升！

### 3. 快速傅里叶变换 (FFT)

FFT 是实现 DFT/IDFT 的高效算法。它的“快”体现在以下几个方面：
*   **巧妙选择求值点**：FFT 选择了一组特殊的点来求值——**复数单位根**。这些点具有周期性、对称性等优良性质，使得分治算法成为可能。
*   **分治思想**：FFT 算法利用单位根的性质，将一个 `N` 点的 DFT 分解为两个 `N/2` 点的 DFT，一个处理偶数项系数，一个处理奇数项系数，然后将结果合并。
*   **复数的力量**：在FFT中，我们需要用到复数。一个复数 `z = a + bi` 可以看作二维平面上的一个点 `(a, b)` 或一个向量。复数的乘法可以理解为长度的缩放和角度的旋转。`e^{i\theta} = \cos\theta + i\sin\theta` 描述了单位圆上的点，这些就是单位根。

**核心原理概述：**
假设我们要计算多项式 `A(x)` 在 `N` 个 `N` 次单位根 `\omega_N^0, \omega_N^1, ..., \omega_N^{N-1}` 处的值。`\omega_N^k = e^{2\pi ik/N}` 是主 `N` 次单位根的 `k` 次幂。
`A(x) = a_0 + a_1x + a_2x^2 + ... + a_{N-1}x^{N-1}`
我们将 `A(x)` 按照指数的奇偶性分为两部分：
`A_{even}(x) = a_0 + a_2x + a_4x^2 + ...`
`A_{odd}(x) = a_1 + a_3x + a_5x^2 + ...`
那么 `A(x) = A_{even}(x^2) + x \cdot A_{odd}(x^2)`。
利用单位根的性质 `(\omega_N^k)^2 = \omega_{N/2}^k` 和 `\omega_N^{k+N/2} = -\omega_N^k`，我们可以将 `A(x)` 在 `\omega_N^k` 处的值分解为 `A_{even}((\omega_N^k)^2)` 和 `A_{odd}((\omega_N^k)^2)` 的组合。这样，通过递归地计算 `N/2` 规模的问题，最终合并结果。这就是 FFT 的分治思想。

**迭代实现（蝴蝶操作）：** 实际编程中，为了避免递归开销，FFT通常采用迭代实现。它需要预先进行**位逆序置换 (Bit-Reversal Permutation)**，将数组中的元素重新排列，使得分治过程中需要合并的元素在物理内存上相邻。然后，通过 `log N` 轮循环，每轮处理一个特定长度的子区间，完成“蝴蝶”操作。

**FFT的局限性：**
*   **浮点数精度问题**：复数运算使用 `double` 类型，可能会有精度误差，对于结果要求是精确整数的问题不适用。
*   **模数限制**：如果问题需要在模意义下进行，FFT的复数运算无法直接进行模运算。

### 4. 数论变换 (NTT)

为了解决FFT的浮点数精度和模数限制问题，科学家们提出了**数论变换 (NTT)**。
**核心思想：**
NTT 是在模 `P` 意义下的离散傅里叶变换。它用**原根 (Primitive Root)** 的幂次代替了复数单位根。
*   **原根**：在一个模 `P` 的剩余类环中，如果存在一个数 `g`，使得 `g^0, g^1, ..., g^{P-2}` 模 `P` 的结果互不相同，则 `g` 称为模 `P` 的一个原根。
*   **模 `P` 意义下的单位根**：我们选取一个原根 `g`，并用 `g^{(P-1)/N}` 替代FFT中的主 `N` 次单位根 `\omega_N`。这样，所有运算都在模 `P` 意义下进行，结果是精确的整数。
*   **模数要求**：NTT要求模数 `P` 必须是**质数**，且 `N` 是 `P-1` 的因子（通常 `N` 是 `2` 的幂次，所以 `P-1` 必须能被 `2` 的足够高次幂整除）。常见的NTT模数有 `998244353` ( `P-1 = 998244352 = 119 * 2^{23}` )，其原根为 `3`。

**NTT的优缺点：**
*   **优点**：结果精确，没有浮点数误差；适用于模意义下的计算。
*   **缺点**：对模数有严格要求；需要找到合适的原根。

在实际竞赛中，NTT 因为其精确性，通常比FFT更受欢迎，尤其是在涉及到大整数或模运算的问题中。但从原理上理解，FFT是基础，NTT是其在数论领域的变种。今天我们主要以FFT为例进行讲解。

**关键点总结：**
1.  **长度补齐**：为了应用FFT的分治结构，多项式的项数 `N` 必须是 `2` 的幂次。如果原始多项式长度为 `L`，我们需要找到最小的 `2^k \ge L` 作为新的 `N`，并用 `0` 补齐多项式的高次项。
2.  **位逆序置换**：FFT迭代实现前，需要对系数数组进行位逆序置换，以便后续的蝴蝶操作能正确地进行。
3.  **复数运算**：理解复数的加减乘除是实现FFT的基础。`std::complex<double>` 是C++标准库中方便的复数类型。

**常见误区：**
*   忘记将多项式长度补齐到 `2` 的幂次。
*   FFT和IDFT的系数没有正确归一化（IDFT后需要除以 `N`）。
*   NTT选择的模数不满足要求，或者原根选择错误。
*   对于一些计数问题，`c_k = \sum a_i b_{k-i}` 可能会有 `i=k-i` (即 `2i=k`) 的情况，需要特殊处理，例如计算 `A(x)*A(x)` 时，`c_k` 包含了 `a_i*a_{k-i}` 和 `a_{k-i}*a_i`，对于 `i \ne k-i` 它们是两项，对于 `i = k-i` 它们是同一项。

## 典型例题精讲

### 例题1：标准多项式乘法

**题目描述：**
给定两个多项式 `A(x)` 和 `B(x)`，它们的系数分别为 `a_0, a_1, ..., a_{n-1}` 和 `b_0, b_1, ..., b_{m-1}`。请计算它们的乘积 `C(x) = A(x) * B(x)` 的系数 `c_0, c_1, ..., c_{n+m-2}`。
例如：`A(x) = 1 + 2x`, `B(x) = 3 + 4x`。
`C(x) = (1+2x)(3+4x) = 3 + 4x + 6x + 8x^2 = 3 + 10x + 8x^2`。
输出系数 `[3, 10, 8]`。

**思路分析：**
这是一个最直接的多项式乘法问题，完全符合卷积的定义。
1.  确定结果多项式的最大次数为 `(n-1) + (m-1) = n+m-2`，所以结果多项式 `C(x)` 的系数数组长度至少为 `n+m-1`。
2.  选取一个足够大的 `N`，它是 `2` 的幂次，且 `N \ge n+m-1`。
3.  将 `A` 和 `B` 的系数数组补齐到长度 `N`。
4.  对 `A` 和 `B` 进行 FFT，得到它们的点值表示 `A'` 和 `B'`。
5.  将 `A'` 和 `B'` 对应位置的元素相乘，得到 `C'`。
6.  对 `C'` 进行 IDFT，得到 `C` 的系数表示。
7.  由于FFT使用了浮点数，结果可能不是精确整数，需要四舍五入到最近的整数。

**代码实现 (FFT版本)：**

```cpp
#include <iostream>
#include <vector>
#include <complex>
#include <cmath>
#include <algorithm> // For std::reverse

const double PI = acos(-1.0); // 定义圆周率

// FFT核心函数
// a: 待变换的系数数组
// inv: true表示进行IDFT (逆变换)，false表示进行DFT (正变换)
void fft(std::vector<std::complex<double>>& a, bool inv) {
    int n = a.size();

    // 位逆序置换 (Bit-reversal permutation)
    // 目的是让分治过程中需要合并的元素在物理内存上相邻
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        while (j >= bit) {
            j -= bit;
            bit >>= 1;
        }
        j += bit;
        if (i < j) {
            std::swap(a[i], a[j]);
        }
    }

    // 蝴蝶操作 (Butterfly operation)
    // len: 当前处理的子区间长度，从2开始，每次翻倍
    for (int len = 2; len <= n; len <<= 1) {
        // wn: 当前子区间的主单位根
        // inv ? -1 : 1 决定了是正变换还是逆变换
        std::complex<double> wn(cos(2 * PI / len), sin(2 * PI / len * (inv ? -1 : 1)));
        
        // i: 遍历每个子区间
        for (int i = 0; i < n; i += len) {
            // w: 当前单位根的幂次
            std::complex<double> w(1);
            // j: 遍历子区间的前半部分
            for (int j = 0; j < len / 2; j++) {
                // 蝴蝶操作的核心：
                // u: 子区间前半部分的元素
                // v: 子区间后半部分的元素
                std::complex<double> u = a[i + j];
                std::complex<double> v = a[i + j + len / 2] * w;
                a[i + j] = u + v;
                a[i + j + len / 2] = u - v;
                w *= wn; // 更新单位根的幂次
            }
        }
    }

    // 如果是逆变换，需要除以N进行归一化
    if (inv) {
        for (int i = 0; i < n; i++) {
            a[i] /= n;
        }
    }
}

// 多项式乘法函数
std::vector<long long> multiply(const std::vector<int>& pA, const std::vector<int>& pB) {
    int n = pA.size();
    int m = pB.size();
    int len = 1; // 结果多项式长度至少为 n+m-1
    while (len < n + m - 1) {
        len <<= 1; // 补齐到2的幂次
    }

    std::vector<std::complex<double>> fA(len);
    std::vector<std::complex<double>> fB(len);

    // 将系数复制到复数向量中
    for (int i = 0; i < n; i++) fA[i] = pA[i];
    for (int i = 0; i < m; i++) fB[i] = pB[i];

    // DFT (正变换)
    fft(fA, false);
    fft(fB, false);

    // 点值相乘
    for (int i = 0; i < len; i++) {
        fA[i] *= fB[i];
    }

    // IDFT (逆变换)
    fft(fA, true);

    // 提取结果系数，并四舍五入
    std::vector<long long> res(n + m - 1);
    for (int i = 0; i < n + m - 1; i++) {
        res[i] = round(fA[i].real());
    }
    return res;
}

// 主函数示例
int main() {
    std::vector<int> a = {1, 2}; // A(x) = 1 + 2x
    std::vector<int> b = {3, 4}; // B(x) = 3 + 4x

    std::vector<long long> c = multiply(a, b);

    std::cout << "多项式 A(x) 的系数: ";
    for (int x : a) std::cout << x << " ";
    std::cout << std::endl;

    std::cout << "多项式 B(x) 的系数: ";
    for (int x : b) std::cout << x << " ";
    std::cout << std::endl;

    std::cout << "乘积 C(x) 的系数: ";
    for (long long x : c) std::cout << x << " ";
    std::cout << std::endl; // 预期输出: 3 10 8

    // 另一个例子
    std::vector<int> a2 = {1, 0, 1}; // A(x) = 1 + x^2
    std::vector<int> b2 = {1, 1, 0, 1}; // B(x) = 1 + x + x^3

    std::vector<long long> c2 = multiply(a2, b2);
    // (1 + x^2)(1 + x + x^3) = 1 + x + x^3 + x^2 + x^3 + x^5
    // = 1 + x + x^2 + 2x^3 + x^5
    // 系数: [1, 1, 1, 2, 0, 1] (长度 n+m-1 = 3+4-1 = 6)

    std::cout << "多项式 A2(x) 的系数: ";
    for (int x : a2) std::cout << x << " ";
    std::cout << std::endl;

    std::cout << "多项式 B2(x) 的系数: ";
    for (int x : b2) std::cout << x << " ";
    std::cout << std::endl;

    std::cout << "乘积 C2(x) 的系数: ";
    for (long long x : c2) std::cout << x << " ";
    std::cout << std::endl; // 预期输出: 1 1 1 2 0 1

    return 0;
}
```

**复杂度分析：**
*   长度补齐：`O(N)`
*   FFT/IDFT：`O(N log N)`
*   点值相乘：`O(N)`
*   总时间复杂度：`O(N log N)`，其中 `N` 是补齐后的多项式长度。
*   空间复杂度：`O(N)`

### 例题2：组合计数 - 寻找和为K的数对

**题目描述：**
给定 `N` 个整数 `a_1, a_2, ..., a_N` (这些整数可能重复，且值域在 `[0, M]` 之间)。请找出有多少对 `(a_i, a_j)` (其中 `i \ne j`) 满足 `a_i + a_j = K`。

**思路分析：**
这个问题可以转化为卷积问题。
1.  **构建多项式**：我们构建一个多项式 `P(x)`，其中 `P(x) = \sum_{k=0}^{M} c_k x^k`。`c_k` 表示数字 `k` 在给定序列中出现的次数。
    例如，如果序列是 `[1, 2, 2, 3]`，那么 `P(x) = 1x^1 + 2x^2 + 1x^3` (假设 `M` 足够大)。
2.  **计算卷积**：我们计算 `Q(x) = P(x) * P(x)`。
    `Q(x) = \sum_{k=0}^{2M} q_k x^k`。
    根据卷积定义，`q_k = \sum_{i=0}^{k} c_i \cdot c_{k-i}`。
    `q_k` 的含义是：所有可能的 `(i, j)` 对，使得 `i+j=k`，并且 `i` 出现了 `c_i` 次，`j` 出现了 `c_{k-i}` 次。那么 `c_i \cdot c_{k-i}` 就是 `i` 和 `j` 组合成 `k` 的方案数。
    所以，`q_K` 就是所有和为 `K` 的数对 `(i, j)` 的总数。
3.  **特殊处理**：
    *   **`i \ne j` 的限制**：题目要求 `i \ne j`。然而 `q_K` 包含了 `i+j=K` 的所有情况，包括 `i=j` 的情况（即 `2i=K`）。
        对于 `i \ne j` 的情况，`c_i \cdot c_j` 和 `c_j \cdot c_i` 都贡献给了 `q_K`，所以每个 `(i, j)` 对被计数了两次。
        对于 `i=j` 的情况（即 `2i=K`），`c_i \cdot c_i` 只被计数了一次，但它实际上代表了从 `c_i` 个 `i` 中选择两个 `i` 的组合数，即 `c_i * (c_i - 1)` 对。而 `q_K` 中的 `c_i * c_i` 仅仅表示 `i` 和 `i` 组成 `K` 的“有序”对数。
    *   **修正**：
        `q_K` 统计的是“有序对” `(i, j)` 使得 `i+j=K`。
        我们需要的是“无序对” `(a_x, a_y)` 使得 `a_x+a_y=K` 且 `x \ne y`。
        如果 `a_x = i, a_y = j` 且 `i \ne j`，那么 `(i, j)` 这样的有序对在 `q_K` 中被计算了 `c_i \cdot c_j` 次，` (j, i)` 这样的有序对被计算了 `c_j \cdot c_i` 次。由于 `i \ne j`，`c_i \cdot c_j = c_j \cdot c_i`，所以 `q_K` 中 `i \ne j` 的贡献是 `2 \cdot (\text{无序对数量})`。
        如果 `a_x = i, a_y = i` 且 `2i=K`，那么 `q_K` 中 `c_i \cdot c_i` 统计的是 `c_i` 个 `i` 和 `c_i` 个 `i` 组成 `K` 的有序对数。而实际上，我们是从 `c_i` 个 `i` 中选出两个不同的 `i`，其组合数为 `c_i \cdot (c_i-1)`。`q_K` 中的 `c_i \cdot c_i` 实际上比我们需要的 `c_i \cdot (c_i-1)` 多了 `c_i`。
        所以，最终答案是 `(q_K - \sum_{2i=K} c_i) / 2`。
        `q_K` 是通过 `P(x)*P(x)` 得到的 `K` 次项系数。
        `\sum_{2i=K} c_i` 是在 `K` 为偶数时，`c_{K/2}` 的值。如果 `K` 是奇数，这一项为 `0`。

**示例代码核心：**

```cpp
// 假设 FFT 模板已经定义好了 (如例题1中的fft函数)
// const double PI = acos(-1.0); 等也已定义
// std::vector<long long> multiply(const std::vector<int>& pA, const std::vector<int>& pB) 也已定义

// 寻找和为K的数对
// nums: 原始整数序列
// M: 序列中最大整数的值
// K: 目标和
long long find_pairs_with_sum_K(const std::vector<int>& nums, int M, int K) {
    // 1. 构建多项式 P(x)
    std::vector<int> counts(M + 1, 0); // counts[i] 存储数字 i 出现的次数
    for (int x : nums) {
        counts[x]++;
    }

    // 2. 计算 P(x) * P(x)
    // 这里我们直接使用counts作为pA和pB传入multiply函数
    // 注意：multiply函数返回的是long long，因为系数可能很大
    std::vector<long long> q_coeffs = multiply(counts, counts);

    // 3. 提取 q_K
    long long q_K = 0;
    if (K < q_coeffs.size()) {
        q_K = q_coeffs[K];
    }

    // 4. 处理 i=j 的情况
    // q_K 包含了所有有序对 (i, j) 使得 i+j=K
    // 对于 i != j 的情况，每个无序对 (i, j) 被计算了两次 (i,j) 和 (j,i)
    // 对于 i == j 的情况 (即 2i=K)，每个 i 被计算了 c_i * c_i 次
    // 但我们实际上需要的是从 c_i 个 i 中选出两个 i 的组合数，即 c_i * (c_i - 1)
    // q_K 中这一项是 c_i * c_i，比 c_i * (c_i - 1) 多了 c_i
    // 所以需要减去所有 2i=K 的情况下的 c_i

    if (K % 2 == 0) { // 如果 K 是偶数，可能存在 i=j 的情况
        int i_val = K / 2;
        if (i_val <= M) { // 确保 i_val 在多项式系数范围内
            q_K -= counts[i_val]; // 减去多算的 c_i (因为 q_K 统计的是有序对)
        }
    }

    // 最终结果除以2，因为 q_K (修正后) 统计的是有序对，每个无序对被计算了两次
    return q_K / 2;
}

int main() {
    std::vector<int> nums1 = {1, 2, 2, 3};
    int M1 = 3; // 序列中最大值为3
    int K1 = 4; // 目标和为4
    // 预期结果： (1,3), (2,2)
    // 但题目要求 i != j，所以 (2,2) 这种不计入
    // 答案是 1 (1,3)
    long long ans1 = find_pairs_with_sum_K(nums1, M1, K1);
    std::cout << "序列 [1, 2, 2, 3] 中和为 " << K1 << " 且下标不同 (i!=j) 的数对数量: " << ans1 << std::endl;
    // 验证：
    // counts: [0, 1, 2, 1] (0,1,2,3出现次数)
    // P(x) = x + 2x^2 + x^3
    // Q(x) = P(x)*P(x) = (x + 2x^2 + x^3)(x + 2x^2 + x^3)
    // q_4 的项:
    // x^1 * 2x^3 (c_1*c_3 = 1*1=1)
    // 2x^2 * 2x^2 (c_2*c_2 = 2*2=4)
    // x^3 * x^1 (c_3*c_1 = 1*1=1)
    // q_4 = 1+4+1 = 6
    // K=4 是偶数，i_val=2。counts[2]=2。
    // 修正 q_4 = 6 - counts[2] = 6 - 2 = 4
    // 最终 ans = 4 / 2 = 2。
    // 为什么和预期不同？因为题意 i!=j 指的是原始序列中的索引，而不是数值。
    // 原始序列 [a_0, a_1, a_2, a_3] = [1, 2, 2, 3]
    // (a_0, a_3) = (1,3) -> 和为4
    // (a_1, a_2) = (2,2) -> 和为4 (但 a_1 和 a_2 是不同位置的2)
    // (a_2, a_1) = (2,2) -> 和为4
    // 共有 3 对。
    // 
    // 我的修正逻辑是针对“数值” i != j 的，但题目是“下标” i != j。
    // 如果是下标 i != j, 那么 q_K 其实是所有 (a_x, a_y) 且 x != y 的有序对之和。
    // q_K = sum_{i+j=K} c_i c_j
    // 对于 i != j, 它贡献了 c_i c_j 对。
    // 对于 i = j, 且 2i = K, 它贡献了 c_i c_i 对。
    // 但实际上，c_i c_i 应该代表从 c_i 个 i 中选出两个不同的 i 的组合数，即 c_i * (c_i - 1)。
    // 所以，当 i=j 时，q_K 中多算了 c_i。
    // 因此，修正仍然是减去 c_{K/2}。
    // 修正后 q_K 统计的是所有 (a_x, a_y) 且 x != y 的有序对。
    // 所以最终答案仍然是 q_K / 2。
    // 结果 2 是对的。 (1,3) 有1对。 (2,2) 有2对 (a_1, a_2) 和 (a_2, a_1) 算作 1 对。
    // (1,3) 1对，(2,2) 1对。总共2对。

    std::vector<int> nums2 = {1, 1, 1, 1}; // 4个1
    int M2 = 1;
    int K2 = 2; // 目标和为2
    // 预期结果：从4个1中选2个，C(4,2) = 6 对
    long long ans2 = find_pairs_with_sum_K(nums2, M2, K2);
    std::cout << "序列 [1, 1, 1, 1] 中和为 " << K2 << " 且下标不同 (i!=j) 的数对数量: " << ans2 << std::endl;
    // 验证：
    // counts: [0, 4]
    // P(x) = 4x
    // Q(x) = P(x)*P(x) = (4x)*(4x) = 16x^2
    // q_2 = 16
    // K=2 是偶数，i_val=1。counts[1]=4。
    // 修正 q_2 = 16 - counts[1] = 16 - 4 = 12
    // 最终 ans = 12 / 2 = 6。 正确。

    return 0;
}
```

**复杂度分析：**
*   构建 `counts` 数组：`O(N + M)`
*   多项式乘法：`O(M' log M')`，其中 `M'` 是补齐后的 `2M+1` 长度。
*   总时间复杂度：`O(N + M' log M')`。
*   空间复杂度：`O(M')`。

### 例题3：多项式快速幂（进阶）

**题目描述：**
给定一个多项式 `A(x)` 和一个整数 `k`，计算 `A(x)^k` 的系数，结果需要在模 `P` 意义下输出。

**思路分析：**
直接计算 `A(x)` 乘以自身 `k` 次，每次都需要 `O(N log N)`，总复杂度是 `O(k \cdot N log N)`，如果 `k` 很大，会超时。
我们可以使用**快速幂 (Binary Exponentiation)** 的思想。
`A(x)^k` 可以通过 `A(x)^{k/2} \cdot A(x)^{k/2}` (如果 `k` 是偶数) 或者 `A(x)^{k-1} \cdot A(x)` (如果 `k` 是奇数) 来计算。
这样，我们只需要进行 `log k` 次多项式乘法。
由于题目要求模 `P` 意义下，这里应该使用 NTT 来避免浮点精度问题。
