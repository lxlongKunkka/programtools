## 趣味引入

同学们，你们有没有玩过搭积木或者堆石头的游戏？想象一下，你有一堆大小不一的石头，要把它们堆成一排，每堆石头的重量平方和就是这一堆的“不稳定值”，你的目标是让所有堆的“不稳定值”之和最小。

这听起来是不是有点像我们之前学过的动态规划问题？比如，我们要把 $N$ 个物品分成若干组，每组有一个代价，求总代价最小。通常我们设 $dp[i]$ 表示前 $i$ 个物品的最优分组代价，然后用一个循环去枚举第 $i$ 个物品所在的组是从哪里开始的：
$dp[i] = \min_{0 \le k < i} (dp[k] + \text{cost}(k+1, i))$

这里 $k$ 就是我们的“决策点”，表示第 $i$ 个物品和前面的哪些物品分成一组。这个 $k$ 循环通常会带来一个 $O(N)$ 的复杂度，使得整个 DP 变成 $O(N^2)$。如果 $N$ 很大，比如 $10^5$，那么 $N^2$ 就是 $10^{10}$，这可就太慢了！

有没有什么办法能让这个 $k$ 循环跑得更快呢？就像我们在玩“猜数字”游戏，如果知道数字在某个范围，每次猜都能排除一半，那是不是很快就能猜到？

今天，我们要学习的“四边形不等式优化”和“决策单调性”，就是 DP 优化中的“秘密武器”！它能帮我们找到这个“猜数字”的规律，让 $N^2$ 的 DP 变成 $N \log N$，甚至更快！它就像是给我们的 DP 算法装上了一对“千里眼”，能提前看到最优决策点的大致位置，从而大大加速计算！

## 深度知识点讲解

### 1. 动态规划的常见形式

我们今天主要关注两种常见的动态规划形式，它们是四边形不等式和决策单调性优化的主要对象：

**形式一：区间 DP (2D DP)**
$dp[i][j] = \min_{i \le k < j} (dp[i][k] + dp[k+1][j] + C[i][j])$
这类 DP 通常用于解决区间合并问题，例如矩阵链乘法、最优二叉搜索树等。其朴素实现的时间复杂度通常是 $O(N^3)$。

**形式二：线性 DP (1D DP)**
$dp[i] = \min_{0 \le k < i} (dp[k] + C[k][i])$
这类 DP 通常用于解决将前 $i$ 个元素划分成若干段的问题，例如最小代价分段、多边形裁剪等。其朴素实现的时间复杂度通常是 $O(N^2)$。

这两种形式的共同点是：都有一个内层循环 $k$ 来寻找最优决策点。我们的目标就是优化这个 $k$ 的寻找过程。

### 2. 四边形不等式 (Quadrilateral Inequality, QI)

四边形不等式是关于代价函数 $C[i][j]$ 的一个数学性质。它描述了合并区间的“成本”变化规律。

**定义：**
对于一个定义在整数区间上的函数 $C[i][j]$，如果对于任意 $a \le b \le c \le d$，都满足：
$C[a][c] + C[b][d] \le C[a][d] + C[b][c]$
那么我们称函数 $C[i][j]$ 满足四边形不等式。

**形象理解：**
想象一下，你有四段区间 $A=[a,b]$, $B=[b,c]$, $C=[c,d]$。
*   左边 $C[a][c] + C[b][d]$ 对应的是合并 $[a,c]$（包含 $A,B$）和合并 $[b,d]$（包含 $B,C$）。这两个合并的区间是“交叉”的，它们都包含了 $B$ 段。
*   右边 $C[a][d] + C[b][c]$ 对应的是合并 $[a,d]$（包含 $A,B,C$）和合并 $[b,c]$（包含 $B$）。这两个合并的区间是“嵌套”的。

四边形不等式告诉我们一个规律：**交叉合并的代价，不大于嵌套合并的代价。**
或者说，把一个大区间分成两个小区间，不如把两个“内层”的小区间合并起来，把两个“外层”的大区间合并起来。合并的“边际成本”随着区间长度的增加而减小。

**等价形式（更常用，更容易验证）：**
如果对于任意 $i \le j$，都满足：
$C[i][j] + C[i+1][j+1] \le C[i][j+1] + C[i+1][j]$
那么函数 $C[i][j]$ 也满足四边形不等式。
这个形式更直观：一个“小四边形”的顶点 $C[i][j], C[i+1][j+1], C[i][j+1], C[i+1][j]$，它表示的是合并小区间 $[i,j]$ 和大区间 $[i+1,j+1]$ 的代价之和，不大于合并 $[i,j+1]$ 和 $[i+1,j]$ 的代价之和。

**例子：**
常见的满足四边形不等式的代价函数有：
*   $C[i][j] = (S[j] - S[i-1])^2$ (其中 $S$ 是前缀和)
*   $C[i][j] = \max(A[i \dots j])$
*   $C[i][j] = \min(A[i \dots j])$
*   $C[i][j] = \text{常数}$
*   如果 $C_1, C_2$ 都满足四边形不等式，那么 $C_1+C_2$ 也满足。

### 3. 决策单调性 (Decision Monotonicity, DM)

决策单调性是指在动态规划中，最优决策点 $k$ 随着 DP 状态的改变而呈现出单调不减的趋势。

**定义：**
对于 1D DP 问题 $dp[i] = \min_{0 \le k < i} (dp[k] + C[k][i])$，设 $p[i]$ 为使得 $dp[i]$ 取得最小值的最优决策点 $k$。如果对于任意 $i < j$，都有 $p[i] \le p[j]$，那么我们称该 DP 具有决策单调性。

**形象理解：**
这就像一个“最佳伙伴”的选择。当我们要为第 $i$ 个任务寻找最佳搭档 $k$ 时，我们找到了 $p[i]$。当我们要为第 $i+1$ 个任务寻找最佳搭档 $k'$ 时，这个 $k'$ 肯定不会比 $p[i]$ 更靠前，它只会等于 $p[i]$ 或者比 $p[i]$ 更靠后。最佳搭档只会“前进”，不会“后退”。

**四边形不等式如何推导出决策单调性？**
这是一个非常重要的性质！如果代价函数 $C[k][i]$ 满足四边形不等式，那么对应的 DP 问题就具有决策单调性。

**证明思路（直观理解）：**
我们假设 $C[k][i]$ 满足四边形不等式。
考虑 $dp[i] = \min_{0 \le k < i} (dp[k] + C[k][i])$。
设 $p[i]$ 是 $dp[i]$ 的最优决策点。
我们想证明 $p[i] \le p[i+1]$。
假设 $p[i+1] < p[i]$ (即最优决策点“后退”了)，我们来尝试导出矛盾。
设 $k_1 = p[i+1]$ 且 $k_1 < p[i]$。
根据 $k_1$ 是 $dp[i+1]$ 的最优决策点，我们有：
$dp[k_1] + C[k_1][i+1] \le dp[p[i]] + C[p[i]][i+1]$ (1)

根据 $p[i]$ 是 $dp[i]$ 的最优决策点，我们有：
$dp[p[i]] + C[p[i]][i] \le dp[k_1] + C[k_1][i]$ (2)

现在我们利用四边形不等式。令 $a=k_1, b=p[i], c=i, d=i+1$。
因为 $k_1 < p[i] < i < i+1$，所以 $a \le b \le c \le d$ 成立。
四边形不等式给出：
$C[k_1][i] + C[p[i]][i+1] \le C[k_1][i+1] + C[p[i]][i]$ (3)

将 (1) 和 (2) 相加：
$dp[k_1] + C[k_1][i+1] + dp[p[i]] + C[p[i]][i] \le dp[p[i]] + C[p[i]][i+1] + dp[k_1] + C[k_1][i]$
整理后得到：
$C[k_1][i+1] + C[p[i]][i] \le C[p[i]][i+1] + C[k_1][i]$
这与四边形不等式 (3) 的形式恰好相反！
(3) 是 $C[k_1][i] + C[p[i]][i+1] \le C[k_1][i+1] + C[p[i]][i]$
而我们从假设得到的却是 $C[k_1][i+1] + C[p[i]][i] \le C[p[i]][i+1] + C[k_1][i]$
这就意味着，如果 $p[i+1] < p[i]$ 成立，那么四边形不等式 (3) 必须取等号。
但是，如果 (3) 不取等号（即严格小于），那么我们的假设 $p[i+1] < p[i]$ 就会导致矛盾。
更严谨地说，如果 $p[i+1] < p[i]$，则必然有 $C[k_1][i+1] + C[p[i]][i] < C[p[i]][i+1] + C[k_1][i]$ (因为 $p[i]$ 和 $k_1$ 是最优的)。这与四边形不等式 $C[k_1][i] + C[p[i]][i+1] \le C[k_1][i+1] + C[p[i]][i]$ 矛盾。
因此，假设不成立，决策单调性 $p[i] \le p[i+1]$ 成立。

**总结：** 四边形不等式是决策单调性的充分条件。只要代价函数满足 QI，我们就可以放心地使用决策单调性进行优化！

### 4. 优化算法：分治优化 (Divide and Conquer Optimization)

分治优化是利用决策单调性来加速 1D DP 的主要方法，可以将 $O(N^2)$ 的 DP 优化到 $O(N \log N)$。

**适用范围：**
主要用于形如 $dp[i] = \min_{0 \le k < i} (dp[k] + C[k][i])$ 的 1D DP，且代价函数 $C[k][i]$ 满足四边形不等式，从而导致决策单调性 $p[i] \le p[i+1]$。

**核心思想：**
分治优化的关键在于：如果我们知道区间 $[L, R]$ 中某个 $dp[mid]$ 的最优决策点 $p[mid]$，那么对于 $dp[L \dots mid-1]$ 的最优决策点，一定在 $p[L \dots mid-1]$ 的范围内；对于 $dp[mid+1 \dots R]$ 的最优决策点，一定在 $p[mid+1 \dots R]$ 的范围内。
更具体地，如果 $p[mid]$ 是 $dp[mid]$ 的最优决策点，那么：
*   对于所有 $i < mid$，其最优决策点 $p[i]$ 必定在 $p[L \dots mid-1]$ 范围内。
*   对于所有 $i > mid$，其最优决策点 $p[i]$ 必定在 $p[mid \dots R]$ 范围内。
由于决策单调性 $p[i] \le p[j]$ (当 $i<j$)，我们可以进一步缩小搜索区间：
*   对于 $dp[i]$ (其中 $L \le i < mid$)，其最优决策点 $p[i]$ 必然在 $optL$ 到 $p[mid]$ 之间。
*   对于 $dp[i]$ (其中 $mid < i \le R$)，其最优决策点 $p[i]$ 必然在 $p[mid]$ 到 $optR$ 之间。

**算法步骤 (递归函数 `solve(L, R, optL, optR)`):**
这个函数的目标是计算 $dp[i]$ 对于所有 $i \in [L, R]$ 的值。
我们知道这些 $dp[i]$ 的最优决策点 $p[i]$ 都在 $[optL, optR]$ 这个范围内。

1.  **递归终止条件：** 如果 $L > R$，说明区间为空，直接返回。
2.  **找到中点：** 设 $mid = (L+R)/2$。
3.  **计算 $dp[mid]$：**
    *   为了计算 $dp[mid]$，我们需要找到它的最优决策点 $k$。
    *   根据决策单调性，这个 $k$ 必定在 $[optL, \min(mid-1, optR)]$ 这个范围内。
    *   我们在这个缩小的范围内暴力枚举 $k$，计算 $dp[k] + C[k][mid]$ 的最小值，从而得到 $dp[mid]$ 和其最优决策点 $p[mid]$。
    *   **注意：** $dp[k]$ 必须是已经计算好的值。通常，我们从 $dp[0]$ 开始计算，或者在 `solve` 函数中保证 $k$ 对应的 $dp[k]$ 已经计算。在分治优化中，递归左区间时，`optK` 已经计算，所以 `dp[optK]` 是可用的。
4.  **递归处理左半部分：**
    *   调用 `solve(L, mid-1, optL, p[mid])`。
    *   这意味着对于 $dp[L \dots mid-1]$，其最优决策点 $k$ 范围缩小到 $[optL, p[mid]]$。
5.  **递归处理右半部分：**
    *   调用 `solve(mid+1, R, p[mid], optR)`。
    *   这意味着对于 $dp[mid+1 \dots R]$，其最优决策点 $k$ 范围缩小到 $[p[mid], optR]$。

**时间复杂度分析：**
设 $T(N)$ 为计算 $dp[1 \dots N]$ 的总时间。
在每层递归中，我们选择一个 $mid$，然后遍历一个长度为 $optR - optL$ 的区间来找到 $p[mid]$。这个区间最大长度是 $N$。
假设计算 $C[k][i]$ 的时间是 $O(1)$。
那么计算 $dp[mid]$ 的时间是 $O(optR - optL)$。
总的递归关系大致是：$T(N) = 2T(N/2) + O(N)$。
根据主定理 (Master Theorem)，这会得到 $T(N) = O(N \log N)$ 的时间复杂度。

### 5. Knuth 优化 (Knuth Optimization)

Knuth 优化主要用于优化区间 DP，形如 $dp[i][j] = \min_{i \le k < j} (dp[i][k] + dp[k+1][j] + C[i][j])$。
如果代价函数 $C[i][j]$ 满足四边形不等式，那么最优决策点 $p[i][j]$ 也满足决策单调性：
$p[i][j-1] \le p[i][j] \le p[i+1][j]$

**核心思想：**
在计算 $dp[i][j]$ 时，我们不再需要遍历所有的 $k \in [i, j-1]$。
我们只需要在 $p[i][j-1]$ 到 $p[i+1][j]$ 这个更小的范围内寻找最优决策点。
由于 $p[i][j-1]$ 和 $p[i+1][j]$ 都是在计算更短区间时得到的，所以它们的值是已知的。

**算法步骤：**
1.  外层循环 `len` 从 1 到 $N-1$ (区间长度)。
2.  内层循环 `i` 从 1 到 $N-len$ (区间左端点)。
3.  计算 `j = i + len` (区间右端点)。
4.  确定 $k$ 的搜索范围：`k_min = p[i][j-1]`，`k_max = p[i+1][j]`。
    *   对于 $len=1$ 的情况（即 $j=i+1$），$dp[i][i+1]$ 的 $k$ 范围是 $[i, i]$，所以 $p[i][i+1]=i$。可以设置 $p[i][i]=i$, $p[i][i-1]=i-1$ 作为边界。
5.  在 `[k_min, k_max]` 范围内枚举 $k$，计算 $dp[i][k] + dp[k+1][j] + C[i][j]$ 的最小值，更新 $dp[i][j]$ 和 $p[i][j]$。

**时间复杂度：**
虽然内层 $k$ 循环的范围缩小了，但总的循环次数仍然是 $O(N^2)$。
不过，由于 $k$ 的搜索范围是单调递增的，总的 $k$ 循环的迭代次数会大大减少。
具体来说，对于 $dp[i][j]$， $p[i][j]$ 的搜索范围是 $[p[i][j-1], p[i+1][j]]$。
总的 $k$ 循环迭代次数是 $\sum_{len=1}^{N-1} \sum_{i=1}^{N-len} (p[i+1][i+len] - p[i][i+len-1] + 1)$。
这个和可以通过巧妙的求和技巧证明为 $O(N^2)$。
因此，Knuth 优化将 $O(N^3)$ 的区间 DP 优化到了 $O(N^2)$。

**总结：**
*   **分治优化** 适用于 1D DP，将 $O(N^2)$ 优化到 $O(N \log N)$。
*   **Knuth 优化** 适用于 2D DP，将 $O(N^3)$ 优化到 $O(N^2)$。
两者都依赖于四边形不等式带来的决策单调性。

### 常见误区：
1.  **误以为所有 DP 都能用：** 四边形不等式和决策单调性是特定 DP 结构和代价函数性质才能使用的优化。
2.  **混淆 1D 和 2D DP 的优化方法：** 分治优化主要针对 1D DP，Knuth 优化主要针对 2D DP。
3.  **忽略边界条件：** 在分治优化中，`optL`, `optR` 以及 `mid-1` 等边界需要仔细考虑，确保 $k$ 的范围合法。
4.  **未正确验证四边形不等式：** 在使用这些优化前，务必验证代价函数是否满足四边形不等式，否则优化可能导致错误结果。

### 示例：验证 $C[k][i] = (S[i] - S[k])^2$ 满足四边形不等式

这是最经典的例子之一。设 $S[x]$ 为前缀和数组。
我们需要验证对于任意 $a \le b \le c \le d$，是否满足：
$C[a][c] + C[b][d] \le C[a][d] + C[b][c]$

代入代价函数：
$(S[c] - S[a])^2 + (S[d] - S[b])^2 \le (S[d] - S[a])^2 + (S[c] - S[b])^2$

为了简化，我们令 $x = S[a]$, $y = S[b]$, $z = S[c]$, $w = S[d]$。
由于 $a \le b \le c \le d$ 且 $S[x]$ 是非递减的（假设元素非负），所以 $x \le y \le z \le w$。
原不等式变为：
$(z-x)^2 + (w-y)^2 \le (w-x)^2 + (z-y)^2$

展开各项：
$z^2 - 2zx + x^2 + w^2 - 2wy + y^2 \le w^2 - 2wx + x^2 + z^2 - 2zy + y^2$

消去两边相同的项 ($x^2, y^2, z^2, w^2$)：
$-2zx - 2wy \le -2wx - 2zy$

两边同时除以 $-2$ (注意不等号方向反转)：
$zx + wy \ge wx + zy$

将所有项移到一边：
$zx - wx - zy + wy \ge 0$

提取公因式：
$x(z-w) - y(z-w) \ge 0$
$(x-y)(z-w) \ge 0$

现在我们检查这个表达式：
*   因为 $x = S[a]$ 且 $y = S[b]$，且 $a \le b$，所以 $x \le y$，因此 $(x-y) \le 0$。
*   因为 $z = S[c]$ 且 $w = S[d]$, 且 $c \le d$，所以 $z \le w$，因此 $(z-w) \le 0$。

所以，$(x-y)$ 是一个非正数，$(z-w)$ 也是一个非正数。
两个非正数的乘积 $(x-y)(z-w)$ 必定是一个非负数（$\ge 0$）。
因此，$(x-y)(z-w) \ge 0$ 成立。

这就证明了 $C[k][i] = (S[i] - S[k])^2$ 满足四边形不等式。

## 典型例题精讲

**例题：最小费用分割问题**

**题目描述：**
给定一个包含 $N$ 个正整数的序列 $A_1, A_2, \dots, A_N$。你需要将这个序列分割成若干个连续的子序列。对于每一个子序列 $A_k, \dots, A_i$，其花费为 $(Sum(k, i))^2$，其中 $Sum(k, i)$ 表示 $A_k$ 到 $A_i$ 的元素之和。你的目标是找到一种分割方案，使得所有子序列的花费之和最小。

**输入格式：**
第一行一个整数 $N$ ($1 \le N \le 10^5$)。
第二行 $N$ 个整数 $A_1, A_2, \dots, A_N$ ($1 \le A_i \le 100$)。

**输出格式：**
一个整数，表示最小的总花费。

**样例输入：**
```
4
1 2 3 4
```

**样例输出：**
```
30
```
（分割方案：[1], [2], [3], [4]，花费 $1^2+2^2+3^2+4^2 = 1+4+9+16 = 30$）

**思路分析：**

1.  **定义 DP 状态：**
    设 $dp[i]$ 表示将前 $i$ 个元素 $A_1, \dots, A_i$ 分割成若干子序列的最小总花费。
    我们的目标是计算 $dp[N]$。

2.  **DP 转移方程：**
    为了计算 $dp[i]$，我们可以考虑最后一个子序列。如果最后一个子序列是从 $A_{k+1}$ 到 $A_i$，那么它的花费是 $Sum(k+1, i)^2$。而前 $k$ 个元素 $A_1, \dots, A_k$ 的最小花费是 $dp[k]$。
    所以，DP 转移方程为：
    $dp[i] = \min_{0 \le k < i} (dp[k] + (Sum(k+1, i))^2)$
    基础情况：$dp[0] = 0$ (前 0 个元素花费为 0)。
    为了方便计算 $Sum(k+1, i)$，我们可以预处理前缀和数组 $S[x] = \sum_{j=1}^x A_j$。那么 $Sum(k+1, i) = S[i] - S[k]$。
    DP 转移方程变为：
    $dp[i] = \min_{0 \le k < i} (dp[k] + (S[i] - S[k])^2)$

3.  **朴素 DP 的复杂度：**
    外层循环 $i$ 从 1 到 $N$。
    内层循环 $k$ 从 0 到 $i-1$。
    每次计算 $dp[i]$ 需要 $O(i)$ 时间。
    总时间复杂度为 $O(N^2)$。
    当 $N=10^5$ 时，$N^2 = 10^{10}$，会超时。

4.  **识别优化机会：**
    这个 DP 形式是典型的 1D DP：$dp[i] = \min_{0 \le k < i} (dp[k] + C[k][i])$。
    其中 $C[k][i] = (S[i] - S[k])^2$。
    在之前的讲解中，我们已经证明了 $C[k][i] = (S[i] - S[k])^2$ 满足四边形不等式。
    因此，这个 DP 具有决策单调性，可以使用分治优化。

5.  **应用分治优化：**
    我们使用 `solve(L, R, optL, optR)` 函数来计算 $dp[i]$ for $i \in [L, R]$，并且这些 $dp[i]$ 的最优决策点 $k$ 都在 $[optL, optR]$ 范围内。

    *   **预处理：** 计算前缀和数组 $S$。
    *   **初始化：** $dp[0] = 0$，其他 $dp[i]$ 初始化为无穷大。
    *   **调用：** `solve(1, N, 0, N-1)`。

    `solve(L, R, optL, optR)` 的实现：
    1.  如果 $L > R$，返回。
    2.  `mid = L + (R-L)/2`。
    3.  `minCost = LLONG_MAX`, `optK_mid = -1`。
    4.  遍历 $k$ 从 `optL` 到 `min(mid-1, optR)`：
        *   计算当前花费 `currentCost = dp[k] + (S[mid] - S[k])^2`。
        *   如果 `currentCost < minCost`，更新 `minCost = currentCost`, `optK_mid = k`。
    5.  `dp[mid] = minCost`。
    6.  递归调用 `solve(L, mid-1, optL, optK_mid)`。
    7.  递归调用 `solve(mid+1, R, optK_mid, optR)`。

6.  **时间复杂度：**
    预处理前缀和 $O(N)$。
    分治优化 $O(N \log N)$。
    总时间复杂度 $O(N \log N)$，空间复杂度 $O(N)$。
    对于 $N=10^5$， $N \log N \approx 10^5 \times 17$，是可接受的。

**样例推演（简略）：**
$N=4$, $A = [1, 2, 3, 4]$
$S = [0, 1, 3, 6, 10]$ (S[0]=0, S[1]=1, S[2]=1+2=3, S[3]=1+2+3=6, S[4]=1+2+3+4=10)
$dp[0] = 0$

`solve(1, 4, 0, 3)`
  `mid = 2`
  计算 $dp[2]$：$k \in [0, \min(2-1, 3)] = [0, 1]$
    $k=0: dp[0] + (S[2]-S[0])^2 = 0 + (3-0)^2 = 9$
    $k=1: dp[1]$ (未计算) $dp[1]$ 必须在 $dp[2]$ 之前计算。
