## 趣味引入

同学们，想象一下我们现在是超级图书馆的管理员！这个图书馆可不一般，它有好多层（比如1到100层），每一层又有好多排书架（比如1到100排）。每本书都有一个固定的位置`(层数, 排数)`。

现在，我们面临几个挑战：
1.  **新书入库！** 有一本新书到了，我们把它放到`(第X层, 第Y排)`。
2.  **找书！** 有读者想知道：在“第 `X1` 层到 `X2` 层之间，并且在`Y1`排到`Y2`排之间”的区域里，一共有多少本书？

这是一个典型的**二维查询**问题！如果只有一层（一维），我们用普通的线段树就能轻松搞定：线段树的叶子节点代表书架，非叶子节点代表一段书架区域的书籍总数。

但现在是二维的！我们怎么才能快速找到一个矩形区域里的书呢？

*   **方案一：二维数组** `book[MAX_X][MAX_Y]`。新书入库直接 `book[X][Y]++`。查询时，双重循环遍历矩形区域求和。
    *   **优点**：简单粗暴。
    *   **缺点**：如果层数和排数都很大（比如1到10亿），数组根本开不下！而且查询效率是 `O(宽度 * 高度)`，太慢了！

*   **方案二：一维线段树，但每个节点存什么？**
    *   如果线段树管X坐标，每个节点要怎么存Y坐标的信息才能快速查询？
    *   如果线段树管Y坐标，每个节点要怎么存X坐标的信息才能快速查询？

同学们，我们能不能把“一维线段树”这个好用的工具“套娃”起来，让它既能管X，又能管Y呢？这就是我们今天要学习的**树套树**——用一个线段树来管理另一个线段树！就像图书馆里，有一个总的目录（外层线段树）管着所有楼层，而每一层又有一个自己的详细目录（内层线段树）管着那一层的所有书架。

## 深度知识点讲解

### 1. 什么是树套树？（线段树套线段树）

**本质：** 树套树是一种高级的数据结构设计思想，它通过将一个数据结构作为另一个数据结构的“节点内容”来，从而解决更高维度的问题。最常见和最实用的是**线段树套线段树**，用于处理二维（或多维）的区间查询和更新问题。

**结构：**
想象一下，我们有一个“外层线段树”，它负责把我们问题的第一个维度（比如X轴坐标）进行区间划分。这和我们平时用的线段树一模一样。
但特别的是，外层线段树的**每一个节点**，不再简单地存储一个数字（比如区间和），而是存储**另一个完整的线段树**！这个“内层线段树”负责处理我们问题的第二个维度（比如Y轴坐标）。

所以，结构就像这样：
`外层线段树 (X轴)`
  `├── 节点 [x_low, x_mid]`
  `│   └── 内层线段树 (Y轴) - 存储所有 X 落在 [x_low, x_mid] 范围内的点的 Y 坐标信息`
  `└── 节点 [x_mid+1, x_high]`
      `└── 内层线段树 (Y轴) - 存储所有 X 落在 [x_mid+1, x_high] 范围内的点的 Y 坐标信息`

### 2. 为什么要用树套树？

*   **维度提升：** 单一的线段树只能处理一维的区间问题。当问题涉及两个独立的维度（例如二维平面上的点，或数组下标和数值大小同时受限的查询），普通的线段树就力不从心了。树套树通过嵌套，将一维的强大能力扩展到二维。
*   **解决2D问题：** 它可以高效地处理二维平面上的矩形区域查询（如统计点数、求和、求最值），以及一维数组上同时对下标和数值范围有限制的查询（如查询 `[L, R]` 区间内，数值在 `[V_min, V_max]` 范围内的元素个数）。
*   **兼顾更新和查询：** 对于静态的二维问题，我们可能有很多其他方法（如二维前缀和、KD树）。但当问题需要支持单点更新时，树套树能够保持较好的时间复杂度。

### 3. 工作原理：以“二维平面点计数”为例

**问题描述：** 在一个二维平面上，初始没有点。我们有两种操作：
1.  `add(x, y)`: 在坐标 `(x, y)` 处添加一个点。
2.  `query(x1, y1, x2, y2)`: 查询矩形区域 `(x1, y1)` 到 `(x2, y2)` 内有多少个点。
（假设 `x, y` 坐标范围都在 `[1, MAX_COORD]` 内）

**核心思想：**
*   **外层线段树：** 负责划分X轴的区间 `[1, MAX_COORD]`。每个节点 `u` 代表一个X轴区间 `[L_x, R_x]`。
*   **内层线段树：** 外层线段树的每个节点 `u` 内部都维护着一个独立的线段树。这个内层线段树负责划分Y轴的区间 `[1, MAX_COORD]`。它统计的是**所有X坐标落在 `[L_x, R_x]` 区间内的点，在Y轴上的分布情况**。

#### 3.1. 坐标离散化 (Coordinate Compression)

在实际问题中，`MAX_COORD` 可能会非常大（例如 `10^9`），直接开这么大的线段树是不现实的。但是，实际出现的 `x` 和 `y` 坐标的数量往往远小于 `MAX_COORD`。
**解决方案：** 坐标离散化。
1.  收集所有可能出现的 `x` 坐标和 `y` 坐标。
2.  对这些坐标进行排序并去重。
3.  将原始坐标映射到它们在排序后的数组中的索引。这样，原本 `1` 到 `10^9` 的坐标范围就被压缩成了 `1` 到 `N` （或 `M`）的范围，其中 `N` 和 `M` 是不同坐标值的数量。
**重要提示：** 离散化后，线段树操作的范围就变成了 `[1, N_compressed]` 和 `[1, M_compressed]`。

#### 3.2. 构建 (Build)

对于一个初始为空的二维平面，我们不需要显式地“构建”一个巨大的树套树。相反，节点是**动态创建**的。
当执行 `add(x, y)` 或 `query(x1, y1, x2, y2)` 时，如果访问到了一个之前未创建的节点，我们就创建它。这种按需创建的方式，可以有效节省内存。

#### 3.3. 单点更新 `add(x, y)`

1.  **外层更新：** 首先，我们像一维线段树一样，在外层线段树上从根节点开始，沿着 `x` 坐标所在的路径向下遍历。这个路径上会有 `O(log MAX_X)` 个节点。
2.  **内层更新：** 对于外层线段树路径上的每一个节点 `u`，我们需要更新它所包含的内层线段树。在内层线段树中，我们对 `y` 坐标进行单点更新（例如，将 `y` 对应位置的计数加1）。
    *   **关键：** 由于内层线段树是存储在**外层线段树的每个节点中**的，所以当外层节点 `u` 被访问时，我们实际上是在更新 `u` 内部的那个内层线段树。
    *   每次内层线段树的更新，都会沿着 `y` 坐标的路径创建 `O(log MAX_Y)` 个新节点（或更新旧节点）。

**时间复杂度：** 外层 `O(log MAX_X)` 步，每一步都进行一次内层线段树的 `O(log MAX_Y)` 更新。总计 `O(log MAX_X * log MAX_Y)`。
**空间复杂度：** 每次 `add` 操作会创建 `O(log MAX_X * log MAX_Y)` 个新的内层线段树节点。如果总共有 `Q` 次 `add` 操作，总空间复杂度将是 `O(Q * log MAX_X * log MAX_Y)`。

#### 3.4. 区间查询 `query(x1, y1, x2, y2)`

1.  **外层查询：** 我们像一维线段树一样，在外层线段树上进行区间查询，找到所有完全覆盖 `[x1, x2]` 区间的 `O(log MAX_X)` 个外层节点。
2.  **内层查询：** 对于找到的每一个外层节点 `u`，我们查询其内部的内层线段树，在其中进行 `[y1, y2]` 区间的查询（例如，求和）。
    *   内层线段树的区间查询，会沿着 `y` 坐标的路径进行 `O(log MAX_Y)` 步。
    *   将所有这些内层查询的结果累加起来，就是最终答案。

**时间复杂度：** 外层 `O(log MAX_X)` 步，每一步都进行一次内层线段树的 `O(log MAX_Y)` 查询。总计 `O(log MAX_X * log MAX_Y)`。

### 4. 内存管理与节点分配

由于我们不能使用 `std::vector` 等STL容器来动态管理节点，我们需要手动分配节点。
1.  定义一个全局的 `Node` 结构体数组：`Node tree[MAX_TOTAL_NODES];`
2.  使用一个全局变量 `node_idx` 来记录当前已分配的节点数量，每次需要新节点时就 `++node_idx` 并返回 `node_idx`。
3.  **关键：`MAX_TOTAL_NODES` 的大小如何确定？**
    *   如果有 `Q` 次 `add` 操作，每次 `add` 操作会创建 `O(log MAX_X * log MAX_Y)` 个节点。
    *   所以 `MAX_TOTAL_NODES` 大约是 `Q * log MAX_X * log MAX_Y`。
    *   如果 `Q = 10^5`，`MAX_COORD = 10^5`，那么 `log MAX_COORD` 大约是 `17`。
    *   `10^5 * 17 * 17 ≈ 2.89 * 10^7` 个节点。每个节点存储 `ls, rs, cnt` 三个 `int`。
    *   `2.89 * 10^7 * 3 * 4 bytes ≈ 346 MB`。这个内存量较大，但在NOI级别竞赛中是可能出现的。务必精确计算并留有余量。

### 5. 与持久化线段树的区别

树套树和持久化线段树都能解决一些二维问题，尤其是Kth问题。
*   **持久化线段树：** 每次更新会创建一条新的路径，但共享未修改的节点。它更像是在历史版本上进行操作，适合处理“基于某个历史状态的查询”。通常内存 `O(N log N)`，查询 `O(log N)`。
*   **线段树套线段树：** 每个外层节点拥有独立的内层线段树（或通过合并而来）。它更直接地将问题分解为两个维度，适合处理动态的二维点更新和矩形查询。内存通常 `O(Q log N log M)`。

它们各有适用场景，理解其内在机制是关键。

### 6. 常见误区与注意事项

*   **内存爆炸：** 最常见的错误是 `MAX_TOTAL_NODES` 设置太小，导致运行时数组越界。务必根据 `Q` 和坐标范围精确估算。
*   **坐标离散化：** 忘记或错误地进行坐标离散化，导致线段树范围过大或映射错误。
*   **节点分配：** 手动分配节点时，忘记 `node_idx++` 或根节点指向错误。
*   **递归边界：** 内外层线段树的递归边界和空节点处理。
*   **根节点：** 外层线段树的每个节点都应该有一个独立的内层线段树的根节点索引。

## 典型例题精讲

**例题：[二维数点] 矩形区域点计数**

**题目描述：**
给定一个 `W * H` 的二维平面（`W, H` 最大为 `10^5`）。初始时平面上没有点。
你需要支持 `Q` (`Q` 最大为 `10^5`) 次操作：
1.  `1 x y`: 在 `(x, y)` 位置添加一个点。
2.  `2 x1 y1 x2 y2`: 查询矩形区域 `(x1, y1)` 到 `(x2, y2)` 内有多少个点。
**注意：** 点可以重复添加，每次添加都算作一个新点。

**输入格式：**
第一行三个整数 `W, H, Q`。
接下来 `Q` 行，每行一个操作。

**输出格式：**
对于每个类型2操作，输出一个整数表示答案。

---

**思路分析：**

1.  **问题分析：** 典型的二维点操作问题，涉及单点更新和矩形查询。由于 `W, H` 较大，不能使用二维数组。`Q` 次操作，每次 `O(log^2 N)` 的复杂度是可接受的。

2.  **数据结构选择：** 线段树套线段树是解决此类问题的标准方法。
    *   **外层线段树：** 负责X轴的区间 `[1, W]`。
    *   **内层线段树：** 存储在外层线段树的每个节点中，负责Y轴的区间 `[1, H]`。

3.  **坐标离散化：** `W, H` 都是 `10^5`，可以直接用作线段树的范围，无需离散化。如果 `W, H` 达到 `10^9`，则需要对X和Y坐标分别进行离散化。这里假设 `W, H` 都是 `10^5` 级别。

4.  **节点定义：**
    *   我们需要一个 `Node` 结构体来表示线段树的节点，包含左孩子 `ls`，右孩子 `rs`，以及当前区间内的点数量 `cnt`。
    *   `ls` 和 `rs` 存储的是子节点的索引，而不是直接的指针。
    *   我们需要一个全局数组 `Node tree[MAX_TOTAL_NODES]` 和一个 `node_idx` 来手动分配节点。

5.  **外层线段树：**
    *   `root_outer`: 整个外层线段树的根节点索引。
    *   `update_outer(u, L_x, R_x, target_x, target_y, val)`:
        *   从 `u` 开始，找到 `target_x` 所在的路径。
        *   对于路径上的每个节点 `u'`，都需要更新其内部的内层线段树。
        *   `u` 内部的内层线段树的根节点索引是 `tree[u].inner_root` (或者我们直接传递 `tree[u].ls` 和 `tree[u].rs` 到内层更新函数)。
    *   `query_outer(u, L_x, R_x, query_x1, query_x2, query_y1, query_y2)`:
        *   从 `u` 开始，找到所有与 `[query_x1, query_x2]` 有交集的外层节点。
        *   如果外层节点 `u` 的区间 `[L_x, R_x]` 完全落在 `[query_x1, query_x2]` 内，则对 `u` 内部的内层线段树进行 `[query_y1, query_y2]` 的查询。
        *   否则，递归查询左右子节点。

6.  **内层线段树：**
    *   `update_inner(u, L_y, R_y, target_y, val)`:
        *   在以 `u` 为根的内层线段树中，对 `target_y` 进行单点更新（`cnt += val`）。
        *   如果 `u` 不存在，就创建一个新节点。
    *   `query_inner(u, L_y, R_y, query_y1, query_y2)`:
        *   在以 `u` 为根的内层线段树中，查询 `[query_y1, query_y2]` 区间的点数。
        *   如果 `u` 不存在，返回 `0`。

7.  **内存估算：**
    *   `Q = 10^5`, `W = 10^5`, `H = 10^5`。
    *   `log W ≈ 17`, `log H ≈ 17`。
    *   `MAX_TOTAL_NODES = Q * log W * log H = 10^5 * 17 * 17 ≈ 2.89 * 10^7`。
    *   每个 `Node` 结构体需要存储 `ls, rs, cnt` 三个 `int`。
    *   总内存 `2.89 * 10^7 * 3 * sizeof(int) ≈ 346 MB`。这个数值是可行的，但需要注意C++默认栈空间可能不足，需要将线段树定义为全局变量或使用 `new`。

**代码实现提示：**
*   手动分配节点时，`new_node()` 函数返回新节点的索引，并初始化 `ls=0, rs=0, cnt=0`。
*   `update_outer` 函数需要传递内层线段树的根节点引用，以便在递归过程中更新它。
*   当内层线段树的节点不存在时（`u == 0`），表示该区间没有点，直接返回 `0`。

```cpp
#include <bits/stdc++.h>

// GESP 十级 (NOI 竞赛提高)
// 主题: 树套树（线段树套线段树）
// 题目: 二维矩形区域点计数 (带点更新)
// 遵循核心原则：无STL容器，原生数组实现，单字母变量

// 假设 W, H 最大 10^5
const int MAX_COORD = 100005; // 坐标范围 [1, MAX_COORD]
const int MAX_Q = 100005;     // 查询/更新操作数量

// 内存估算: Q * log(MAX_COORD) * log(MAX_COORD)
// 10^5 * log(10^5) * log(10^5) ≈ 10^5 * 17 * 17 ≈ 2.89 * 10^7
// 每个节点 3个 int (ls, rs, cnt)。
// 2.89 * 10^7 * 3 = 8.67 * 10^7 ints.
// 8.67 * 10^7 * 4 bytes/int ≈ 346 MB。
// 这是一个巨大的数组，但对于NOI级别是可以接受的。
// 注意：实际节点数可能略小于理论最大值，但预留足够空间非常重要。
const int MAX_TOTAL_NODES = 90000000; // 预留足够的节点空间

// 线段树节点结构
// ls: 左孩子索引
// rs: 右孩子索引
// cnt: 当前节点代表区间内的点数量
struct Node {
    int ls, rs, cnt;
};

Node tree[MAX_TOTAL_NODES]; // 所有线段树节点都存储在这里
int node_idx = 0;           // 用于分配新节点的索引

// 创建一个新节点并返回其索引
int new_node() {
    node_idx++;
    tree[node_idx].ls = 0;
    tree[node_idx].rs = 0;
    tree[node_idx].cnt = 0;
    return node_idx;
}

// 内层线段树的更新操作 (Y轴)
// u: 当前内层线段树节点的索引 (传引用，因为可能创建新节点)
// ly, ry: 当前节点代表的Y轴区间
// y: 要更新的目标Y坐标
// v: 更新值 (+1 或 -1)
void update_inner(int &u, int ly, int ry, int y, int v) {
    if (u == 0) { // 如果节点不存在，就创建一个
        u = new_node();
    }
    tree[u].cnt += v; // 更新当前节点的计数

    if (ly == ry) { // 到达叶子节点
        return;
    }

    int mid = ly + (ry - ly) / 2;
    if (y <= mid) { // 目标在左子区间
        update_inner(tree[u].ls, ly, mid, y, v);
    } else { // 目标在右子区间
        update_inner(tree[u].rs, mid + 1, ry, y, v);
    }
}

// 内层线段树的查询操作 (Y轴)
// u: 当前内层线段树节点的索引
// ly, ry: 当前节点代表的Y轴区间
// qy1, qy2: 查询的Y轴区间
// 返回: 查询区间内的点数
int query_inner(int u, int ly, int ry, int qy1, int qy2) {
    if (u == 0 || qy1 > ry || qy2 < ly) { // 节点不存在 或 查询区间与当前区间无交集
        return 0;
    }
    if (qy1 <= ly && ry <= qy2) { // 当前区间完全包含在查询区间内
        return tree[u].cnt;
    }

    int mid = ly + (ry - ly) / 2;
    int ans = 0;
    ans += query_inner(tree[u].ls, ly, mid, qy1, qy2);
    ans += query_inner(tree[u].rs, mid + 1, ry, qy1, qy2);
    return ans;
}
