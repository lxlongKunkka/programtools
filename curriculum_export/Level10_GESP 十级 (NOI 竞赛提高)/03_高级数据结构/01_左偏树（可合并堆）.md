## 趣味引入

同学们，想象一下，你是一位管理着两个巨大宝藏堆的国王。每个宝藏堆里的宝藏都按照价值从低到高（或者从高到低）排列着，最珍贵的宝藏总是在最顶端（就像一个优先级队列，最优先级的东西总能立刻拿到）。

有一天，你的两个邻国合并了，他们各自的宝藏堆也需要合并成一个大宝藏堆。你希望合并之后，依然能快速地找出所有宝藏中最珍贵的那一个，并且随时可以把最珍贵的宝藏取走。

如果只是简单地把两个宝藏堆堆在一起，然后重新整理，那可太费劲了！对于一个有N个宝藏的堆，重新整理可能需要O(N log N)的时间。如果我们要频繁地合并和取走最珍贵的宝藏，这种效率是远远不够的。

我们学过的二叉堆（或者`std::priority_queue`）可以高效地插入和删除最值，但是它有一个缺点：**合并两个堆的效率很低**。通常的合并方法是把一个堆的所有元素逐个插入到另一个堆中，这需要O(N log N)的时间。

那么，有没有一种“魔法”数据结构，既能像普通堆一样快速找到最值和删除最值，又能像变魔术一样，**在很短的时间内把两个宝藏堆合二为一呢？**

今天，我们就来学习这种“魔法”数据结构——**左偏树（Leftist Tree）**，它是一种神奇的**可合并堆**！它能让我们优雅而高效地解决国王的烦恼，把两个宝藏堆合并成一个，并且每次操作都只需要对数级别的时间！

## 深度知识点讲解

### 1. 左偏树的本质：可合并的优先级队列

左偏树（Leftist Tree），顾名思义，它“偏向左边”。它是一种特殊的二叉树，同时满足两个重要的性质：
1.  **堆性质 (Heap Property)**：和我们熟悉的二叉堆一样，它要么是最小堆（父节点的值小于等于子节点的值），要么是最大堆（父节点的值大于等于子节点的值）。我们通常以**最小堆**为例讲解。
2.  **左偏性质 (Leftist Property)**：这是左偏树的灵魂所在！它保证了树的右子树总是“最短”的，从而使得合并操作能够沿着右子树高效进行。

### 2. 核心概念：零路径长度 (Null Path Length, dist值/s-value/rank)

要理解左偏性质，我们首先要理解“零路径长度”（Null Path Length，简称`dist`值）。

*   **定义**：对于树中的任意一个节点`p`，它的`dist`值定义为从节点`p`到最近的**外部节点（即空节点`NULL`）**的路径上，**只经过右子节点的边数**。
    *   一个空节点（`NULL`）的`dist`值定义为 **-1**。
    *   一个叶子节点（左右子节点都为空）的`dist`值定义为 **0**。
    *   对于非空节点`p`，其`dist`值等于 `min(dist(p->l), dist(p->r)) + 1`。

    **为什么是“只经过右子节点的边数”？**
    想象你从一个节点出发，每次只能选择向右走一步，直到遇到一个空节点。这条路径上你走了多少步，就是这个节点的`dist`值。
    例如：
    *   节点`A`，如果它的左子节点`L`的`dist`是2，右子节点`R`的`dist`是1。那么`A`的`dist`就是`min(2, 1) + 1 = 2`。

*   **`dist`值的意义**：`dist`值衡量的是从当前节点到最近的外部节点的“最短右路径”。它其实代表了以当前节点为根的子树中，**最短的从根到任意外部节点的路径长度**。这个值在左偏树中起着至关重要的作用。

### 3. 左偏性质 (Leftist Property)

有了`dist`值的概念，左偏性质就很好理解了：
*   对于左偏树中的任意一个节点`p`，其**左子节点`p->l`的`dist`值 总是 大于等于 右子节点`p->r`的`dist`值。**
    即：`dist(p->l) >= dist(p->r)`。

    **为什么叫“左偏”？**
    因为左子树总是“更深”或“更重”的，它的`dist`值更大，而右子树的`dist`值总是相对较小。这就像一个天平，左边总是比右边重（或一样重），所以天平的右臂总是相对较短。

*   **左偏性质的威力**：这个性质保证了**树的右子树的深度是 O(log N) 的**。
    *   设一个节点的`dist`值为`k`，那么它至少有`2^k - 1`个节点。
    *   因为`dist(p->l) >= dist(p->r)`，所以我们总是沿着右子树走，右子树的`dist`值会快速减小。从根节点到最右边的叶子节点的路径长度（也就是右子树的最大深度）最多是`log N`。
    *   **这正是左偏树能高效合并的关键！** 所有的合并操作都将集中在较短的右子树上，从而保证了合并操作的对数时间复杂度。

### 4. 核心操作：合并 (Merge)

合并是左偏树最核心、最精妙的操作。所有的其他操作（插入、删除最小值）都可以通过合并来实现。

假设我们要合并两个左偏树 `h1` 和 `h2`（都是最小堆）。
**合并函数 `merge(h1, h2)` 的递归思路：**

1.  **基本情况（递归终止条件）**：
    *   如果 `h1` 为空，直接返回 `h2`。
    *   如果 `h2` 为空，直接返回 `h1`。
    *   （如果 `h1` 和 `h2` 都为空，上面的情况会处理，最终返回 `NULL`）

2.  **维护堆性质**：
    *   确保 `h1` 的根节点的值小于等于 `h2` 的根节点的值。如果不是，就交换 `h1` 和 `h2`，使得 `h1` 成为值较小的那个根。这样，最终合并后的根节点一定是两个原树中最小的那个。

3.  **递归合并右子树**：
    *   将 `h1` 的右子树 `h1->r` 与 `h2` 进行递归合并。合并后的结果作为 `h1` 的新右子树：`h1->r = merge(h1->r, h2)`。

4.  **维护左偏性质**：
    *   合并完成后，`h1` 的右子树可能变得比左子树“更深”（即`dist`值更大），这违反了左偏性质。
    *   因此，如果 `dist(h1->l) < dist(h1->r)`，我们需要**交换 `h1` 的左右子树**，使得左偏性质重新得到满足。

5.  **更新`dist`值**：
    *   `h1` 的左右子树可能已经改变，所以需要重新计算并更新 `h1` 的`dist`值：`h1->dist = min(getDist(h1->l), getDist(h1->r)) + 1`。

6.  **返回`h1`**：
    *   返回处理后的 `h1` 作为新的合并树的根。

**合并过程图示（以最小堆为例）：**

假设要合并 `h1` (根为2) 和 `h2` (根为3)。
```
h1:       2                 h2:       3
        / \                         / \
       5   8                       6   7
      /                           /
     9                           10
```

1.  `h1->key (2) < h2->key (3)`，堆性质满足。
2.  `h1->r` (8) 与 `h2` (3) 递归合并：`h1->r = merge(8, 3)`。
    *   在 `merge(8, 3)` 内部：`3->key (3) < 8->key (8)`，交换，变成 `merge(3, 8)`。
    *   `3->r` (7) 与 `8` 递归合并：`3->r = merge(7, 8)`。
        *   在 `merge(7, 8)` 内部：`7->key (7) < 8->key (8)`，堆性质满足。
        *   `7->r` (NULL) 与 `8` 递归合并：`7->r = merge(NULL, 8)`，返回 `8`。
        *   `7->r` 变为 `8`。此时 `7->l` (NULL) 的 `dist` 是 -1，`7->r` (8) 的 `dist` 是 0。`dist(NULL) < dist(8)` 违反左偏性质。
        *   **交换 `7` 的左右子树**。`7->l` 变为 `8`，`7->r` 变为 `NULL`。
        *   更新 `7` 的 `dist` 值：`min(dist(8), dist(NULL)) + 1 = min(0, -1) + 1 = 0`。
        *   返回 `7`。
    *   `3->r` 变为 `7`。此时 `3->l` (6) 的 `dist` 是 0，`3->r` (7) 的 `dist` 是 0。`dist(6) >= dist(7)` 满足左偏性质。
    *   更新 `3` 的 `dist` 值：`min(dist(6), dist(7)) + 1 = min(0, 0) + 1 = 1`。
    *   返回 `3`。
3.  `h1->r` 变为 `3`。
    ```
    h1:       2
            / \
           5   3
          /   / \
         9   6   7
            /
           10
    ```
4.  检查 `h1` 的左右子树：`h1->l` (5) 的 `dist` 是 0，`h1->r` (3) 的 `dist` 是 1。`dist(5) < dist(3)` 违反左偏性质。
5.  **交换 `h1` 的左右子树**。`h1->l` 变为 `3`，`h1->r` 变为 `5`。
    ```
    h1:       2
            / \
           3   5
          / \ /
         6   7 9
        /
       10
    ```
6.  更新 `h1` 的 `dist` 值：`min(dist(3), dist(5)) + 1 = min(1, 0) + 1 = 1`。
7.  返回 `h1` (根为2)。

最终合并后的树：
```
          2 (dist=1)
        /          \
       3 (dist=1)   5 (dist=0)
      / \          /
     6   7        9
    /
   10
```

**时间复杂度分析**：
由于每次递归合并都是沿着右子树进行的，而左偏性质保证了右子树的深度是O(log N)的，所以`merge`操作的时间复杂度是**O(log N)**，其中N是两个树中节点总数。

### 5. 其他操作（基于Merge）

*   **插入 (Insert)**：
    *   要插入一个新值`v`，可以把这个新值看作一个只包含一个节点的左偏树。
    *   然后，将这个单节点树与原有的左偏树进行合并。
    *   时间复杂度：O(log N)。

*   **删除最小值 (DeleteMin)**：
    *   对于最小堆，根节点就是最小值。
    *   删除根节点后，它原来的左右子树就变成了两个独立的左偏树。
    *   将这两个子树进行合并，作为新的左偏树。
    *   时间复杂度：O(log N)。

### 6. 常见误区

*   **混淆`dist`和树的深度/高度**：`dist`不是树的深度，也不是树的高度。它是“最短右路径长度”。
*   **合并时忘记交换根节点以维护堆性质**：如果 `h1->key > h2->key` 却不交换 `h1` 和 `h2`，那么较小的根节点就会被“埋没”在子树中，破坏了堆性质。
*   **合并后忘记更新`dist`值或交换左右子树以维护左偏性质**：这是最常见的错误。`dist`值必须在左右子树可能发生变化后重新计算，并且如果左偏性质被破坏，必须交换左右子树。

## 典型例题精讲

### 例题1：可合并的优先级队列

**题目描述**：
你需要实现一个数据结构，支持以下操作：
1.  `insert x`：向当前集合中插入元素 `x`。
2.  `get_min`：查询当前集合中的最小元素。
3.  `delete_min`：删除当前集合中的最小元素。
4.  `merge h1 h2`：将编号为 `h1` 和 `h2` 的两个集合合并成一个新的集合，并删除原来的 `h1` 和 `h2`。返回新集合的编号。

假设初始有 `M` 个空的集合，编号从 `1` 到 `M`。

**思路分析**：
这道题是左偏树的经典应用。每个集合就是一个左偏树。
*   为了管理多个集合，我们可以使用一个数组 `roots[M+1]` 来存储每个集合的根节点指针。初始时都为 `NULL`。
*   `insert x`：创建一个新节点 `x`，然后将它与 `roots[current_set_id]` 合并。
*   `get_min`：直接访问 `roots[current_set_id]->key`。注意判空。
*   `delete_min`：调用 `deleteMin` 函数，更新 `roots[current_set_id]`。
*   `merge h1 h2`：调用 `merge(roots[h1], roots[h2])`，将结果赋值给一个新的集合编号（或者直接覆盖其中一个，并清空另一个）。

**代码实现（核心部分）**：
```cpp
#include <iostream>
#include <algorithm> // For std::min

// 定义左偏树节点结构
struct Node {
    int key;    // 节点存储的值
    int dist;   // 零路径长度
    Node *l, *r; // 左右子节点指针

    // 构造函数
    Node(int v) : key(v), dist(0), l(nullptr), r(nullptr) {}
};

// 获取节点的dist值，空节点为-1
int getDist(Node* p) {
    if (p == nullptr) {
        return -1;
    }
    return p->dist;
}

// 合并两个左偏树 h1 和 h2
Node* merge(Node* h1, Node* h2) {
    // 基本情况：如果其中一个为空，返回另一个
    if (h1 == nullptr) {
        return h2;
    }
    if (h2 == nullptr) {
        return h1;
    }

    // 维护堆性质：确保 h1 的根节点值小于等于 h2
    if (h1->key > h2->key) {
        std::swap(h1, h2);
    }

    // 递归合并 h1 的右子树与 h2
    h1->r = merge(h1->r, h2);

    // 维护左偏性质：如果左子树的dist小于右子树的dist，交换左右子树
    if (getDist(h1->l) < getDist(h1->r)) {
        std::swap(h1->l, h1->r);
    }

    // 更新 h1 的dist值
    h1->dist = getDist(h1->r) + 1;

    return h1;
}

// 插入操作：将新值 v 插入到以 h 为根的左偏树中
Node* insert(Node* h, int v) {
    Node* new_node = new Node(v); // 创建一个新节点
    return merge(h, new_node);     // 将新节点视为一个单节点左偏树与 h 合并
}

// 删除最小值操作：删除以 h 为根的左偏树的最小值
Node* deleteMin(Node* h) {
    if (h == nullptr) { // 空树无法删除
        return nullptr;
    }
    Node* left_child = h->l;
    Node* right_child = h->r;
    delete h; // 释放根节点内存
    return merge(left_child, right_child); // 合并左右子树
}

// 主函数模拟
const int MAX_HEAP_COUNT = 1005; // 假设最多1000个堆
Node* roots[MAX_HEAP_COUNT]; // 存储每个堆的根节点

int main() {
    // 初始化所有堆为空
    for (int i = 0; i < MAX_HEAP_COUNT; ++i) {
        roots[i] = nullptr;
    }

    int Q; // 操作次数
    std::cin >> Q;

    for (int q = 0; q < Q; ++q) {
        int op_type;
        std::cin >> op_type;
        if (op_type == 1) { // insert x to heap_id
            int heap_id, x;
            std::cin >> heap_id >> x;
            roots[heap_id] = insert(roots[heap_id], x);
        } else if (op_type == 2) { // get_min from heap_id
            int heap_id;
            std::cin >> heap_id;
            if (roots[heap_id] == nullptr) {
                std::cout << "EMPTY\n";
            } else {
                std::cout << roots[heap_id]->key << "\n";
            }
        } else if (op_type == 3) { // delete_min from heap_id
            int heap_id;
            std::cin >> heap_id;
            if (roots[heap_id] == nullptr) {
                std::cout << "EMPTY\n"; // 无法删除，但通常不报错，只是不做操作
            } else {
                roots[heap_id] = deleteMin(roots[heap_id]);
            }
        } else if (op_type == 4) { // merge h1 and h2 into new_h (e.g., h1 becomes new_h, h2 is cleared)
            int h1_id, h2_id;
            std::cin >> h1_id >> h2_id;
            // 合并到 h1_id，并将 h2_id 清空
            roots[h1_id] = merge(roots[h1_id], roots[h2_id]);
            roots[h2_id] = nullptr; // h2_id 对应的堆现在是空的
        }
    }

    // 注意：实际竞赛中可能需要更完善的内存管理，例如在程序结束时释放所有节点
    // 但对于大多数竞赛题，不显式释放内存通常不会导致TLE或MLE，除非节点数量非常大。

    return 0;
}
```

**复杂度分析**：
*   `insert`：O(log N)，N为当前堆的元素数量。
*   `get_min`：O(1)。
*   `delete_min`：O(log N)，N为当前堆的元素数量。
*   `merge`：O(log N)，N为两个被合并堆的元素总数。

### 例题2：Kruskal算法的优化（合并连通块的最小边权）

**题目描述**：
给定一个无向图，包含N个节点和M条边。每条边有一个权值。现在需要执行一系列操作：
1.  `query u`：查询节点 `u` 所在的连通块中，所有边的权值的最小值。
2.  `merge u v`：合并节点 `u` 和 `v` 所在的连通块。

**思路分析**：
这道题结合了**并查集**和**左偏树**。
*   **并查集**：用来维护节点的连通性。每个集合的代表元可以存储该连通块的信息。
*   **左偏树**：每个连通块维护一个左偏树，存储该连通块中所有边的权值。
    *   初始时，每个节点是一个独立的连通块，其左偏树为空。
    *   当读取边 `(u, v, w)` 时，如果 `u` 和 `v` 已经在同一个连通块中，则这条边形成环，不加入。否则，将这条边 `w` 加入到 `u` 所在连通块的左偏树中。
    *   `query u`：找到 `u` 的代表元 `root_u`，然后查询 `roots[root_u]` 左偏树的最小值。
    *   `merge u v`：
        1.  找到 `u` 和 `v` 的代表元 `root_u` 和 `root_v`。
        2.  如果 `root_u == root_v`，则它们已在同一连通块，无需操作。
        3.  否则，将 `roots[root_u]` 和 `roots[root_v]` 进行合并：`roots[root_u] = merge(roots[root_u], roots[root_v])`。
        4.  然后通过并查集的 `union` 操作将 `root_v` 合并到 `root_u`（或反之），并清空 `roots[root_v]`。

**代码片段（展示结合点）**：
```cpp
#include <iostream>
#include <vector>
#include <numeric> // For std::iota
#include <algorithm>

// 左偏树节点和操作（同例题1）
struct Node {
    int key;
    int dist;
    Node *l, *r;
    Node(int v) : key(v), dist(0), l(nullptr), r(nullptr) {}
};
int getDist(Node* p) { /* ... */ return (p == nullptr) ? -1 : p->dist; }
Node* merge(Node* h1, Node* h2) { /* ... */ }
Node* insert(Node* h, int v) { /* ... */ }
Node* deleteMin(Node* h) { /* ... */ }

// 并查集
std::vector<int> parent;
std::vector<Node*> heap_roots; // 每个连通块的左偏树根节点

int find_set(int v) {
    if (v == parent[v])
        return v;
    return parent[v] = find_set(parent[v]);
}

void union_sets(int a, int b) {
    a = find_set(a);
    b = find_set(b);
    if (a != b) {
        // 合并左偏树：将 b 的堆合并到 a 的堆
        heap_roots[a] = merge(heap_roots[a], heap_roots[b]);
        heap_roots[b] = nullptr; // b 的堆现在为空
        parent[b] = a; // 并查集合并
    }
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n_nodes, m_edges;
    std::cin >> n_nodes >> m_edges;

    parent.resize(n_nodes + 1);
    std::iota(parent.begin(), parent.end(), 0); // 初始化并查集

    heap_roots.resize(n_nodes + 1, nullptr); // 初始化每个节点的堆为空

    // 假设边是预先给定的，或者逐个处理
    // Kruskal 算法通常是先排序边，这里简化处理
    // 对于此题，可能先将所有边加入到对应连通块的堆中
    // 假设输入格式是边列表，然后是查询/合并操作
    for (int i = 0; i < m_edges; ++i) {
        int u, v, w;
        std::cin >> u >> v >> w;
        // 将边权 w 加入到 u 所在连通块的堆中
        int root_u = find_set(u);
        heap_roots[root_u] = insert(heap_roots[root_u], w);
        // 如果 v 和 u 不在同一连通块，并且这条边不是为了连接两个连通块，
        // 而是作为连通块内部的边，则也要加入到 v 所在连通块的堆中
        // 但如果目标是Kruskal，通常是选边时才加入。
        // 这里假设每条边都属于它连接的两个节点的连通块，直到它们合并。
        // 更准确的做法是，Kruskal过程中，当选择一条边连接两个连通块时，
        // 才将这条边加入到新合并的连通块的堆中。
        // 为了简化，我们假设边权是连通块的“属性”，一开始就加入。
        // 实际应用中，如果仅是查询连通块中的“边权最小值”，那么
        // 1. 对于每条边(u,v,w)，将其权值w加入到u所在连通块的左偏树中。
        // 2. 如果u,v不在同一连通块，也将w加入到v所在连通块的左偏树中。
        // 3. 当union(u,v)时，合并u和v连通块的左偏树。
    }


    int Q; // 操作次数
    std::cin >> Q;
    for (int q = 0; q < Q; ++q) {
        std::string op;
        std::cin >> op;
        if (op == "query") {
            int u;
            std::cin >> u;
            int root_u = find_set(u);
            if (heap_roots[root_u] == nullptr) {
                std::cout << "EMPTY\n";
            } else {
                std::cout << heap_roots[root_u]->key << "\n";
            }
        } else if (op == "merge") {
            int u, v;
            std::cin >> u >> v;
            union_sets(u, v); // 合并并查集和左偏树
        }
    }

    // 内存释放：对于heap_roots中的所有Node，需要递归释放。
    // 在NOI/GESP竞赛中，通常不强调显式释放，但好的习惯是考虑。

    return 0;
}
```

**复杂度分析**：
*   **并查集操作** (`find_set`, `union_sets`)：在路径压缩和按秩合并（或按大小合并）优化下，接近O(α(N))，其中α是反阿克曼函数，一个极小的常数。
*   **左偏树操作** (`insert`, `deleteMin`, `merge`)：O(log N)，N为当前相关左偏树的节点总数。
*   **总复杂度**：每次`query`或`merge`操作的时间复杂度主要取决于左偏树的O(log N)部分，加上并查集的O(α(N))。因此单次操作复杂度为**O(log N)**。

## 代码实现模板

```cpp
#include <iostream>
#include <algorithm> // For std::swap and std::min
// #include <bits/stdc++.h> // 竞赛中常用，包含所有标准库

// 左偏树节点结构体
struct Node {
    int key;    // 节点存储的值 (这里以最小堆为例，存储优先级)
    int dist;   // 零路径长度 (Null Path Length)
    Node *l, *r; // 左右子节点指针

    // 构造函数：初始化新节点
    Node(int v) : key(v), dist(0), l(nullptr), r(nullptr) {}
};

// 辅助函数：获取节点的dist值，空节点定义为-1
// 这是为了简化计算，避免在merge中频繁判空
int getDist(Node* p) {
    if (p == nullptr) {
        return -1;
    }
    return p->dist;
}

// 核心操作：合并两个左偏树 h1 和 h2
// 返回合并后新树的根节点指针
Node* merge(Node* h1, Node* h2) {
    // 1. 基本情况：如果其中一个堆为空，直接返回另一个
    if (h1 == nullptr) {
        return h2;
    }
    if (h2 == nullptr) {
        return h1;
    }

    // 2. 维护堆性质（以最小堆为例）：
    //    确保 h1 的根节点值小于等于 h2 的根节点值。
    //    如果不是，则交换 h1 和 h2，保证 h1 始终是根值较小的那个。
    if (h1->key > h2->key) {
        std::swap(h1, h2);
    }

    // 3. 递归合并：
    //    将 h1 的右子树与 h2 进行递归合并。
    //    合并后的结果作为 h1 的新右子树。
    h1->r = merge(h1->r, h2);

    // 4. 维护左偏性质：
    //    如果合并后 h1 的左子树的 dist 值小于右子树的 dist 值，
    //    说明左偏性质被破坏，需要交换 h1 的左右子树。
    if (getDist(h1->l) < getDist(h1->r)) {
        std::swap(h1->l, h1->r);
    }

    // 5. 更新 dist 值：
    //    左右子树可能已改变，重新计算并更新 h1 的 dist 值。
    //    dist(p) = min(dist(p->l), dist(p->r)) + 1
    //    由于我们已经保证了 dist(p->l) >= dist(p->r)，
    //    所以简写为 dist(p) = dist(p->r) + 1
    h1->dist = getDist(h1->r) + 1;

    // 6. 返回合并后的树的根节点
    return h1;
}

// 插入操作：将新值 v 插入到以 h 为根的左偏树中
// 返回插入后新树的根节点指针
Node* insert(Node* h, int v) {
    // 创建一个只包含新值 v 的单节点左偏树
    Node* new_node = new Node(v);
    // 将这个单节点树与原有的左偏树 h 进行合并
    return merge(h, new_node);
}

// 删除最小值操作：删除以 h 为根的左偏树的最小值（即根节点）
// 返回删除后新树的根节点指针
Node* deleteMin(Node* h) {
    // 如果树为空，无法删除，直接返回空
    if (h == nullptr) {
        return nullptr;
    }

    // 保存根节点的左右子树
    Node* left_child = h->l;
    Node* right_child = h->r;

    // 释放原根节点的内存
    delete h;

    // 合并原根节点的左右子树，作为新的左偏树
    return merge(left_child, right_child);
}

// 示例：遍历（这里简单打印，实际左偏树不常用遍历）
void inorderTraversal(Node* p) {
    if (p == nullptr) {
        return;
    }
    inorderTraversal(p->l);
    std::cout << p->key << " (dist:" << p->dist << ") ";
    inorderTraversal(p->r);
}

// 示例主函数
int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    Node* heap1 = nullptr; // 初始化一个空堆
    heap1 = insert(heap1, 10);
    heap1 = insert(heap1, 20);
    heap1 = insert(heap1, 5);
    std::cout << "Heap 1 after insertions: ";
    inorderTraversal(heap1);
    std::cout << "\n";
    std::cout << "Min of Heap 1: " << heap1->key << "\n";

    Node* heap2 = nullptr; // 初始化另一个空堆
    heap2 = insert(heap2, 15);
    heap2 = insert(heap2, 3);
    heap2 = insert(heap2, 25);
    std::cout << "Heap 2 after insertions: ";
    inorderTraversal(heap2);
    std::cout << "\n";
    std::cout << "Min of Heap 2: " << heap2->key << "\n";

    Node* merged_heap = merge(heap1, heap2);
    std::cout << "Merged Heap: ";
    inorderTraversal(merged_heap);
    std::cout << "\n";
    std::cout << "Min of Merged Heap: " << merged_heap->key << "\n";

    merged_heap = deleteMin(merged_heap);
    std::cout << "Merged Heap after deleteMin: ";
    inorderTraversal(merged_heap);
    std::cout << "\n";
    std::cout << "Min of Merged Heap: " << merged_heap->key << "\n";

    // 注意：在实际竞赛中，如果创建了大量节点，需要考虑内存释放，
    // 但通常在程序退出时操作系统会自动回收。
    // 对于左偏树，如果需要手动释放，需要一个递归的 deleteTree 函数。
    // void deleteTree(Node* p) {
    //     if (p == nullptr) return;
    //     deleteTree(p->l);
    //     deleteTree(p->r);
    //     delete p;
    // }
    // deleteTree(merged_heap); // 在程序结束前调用

    return 0;
}
```
