## 趣味引入

同学们，想象一下，你是一位海岛探险家，发现了一座形状不规则的无人岛屿。现在，你需要在岛屿上选择两个地点建立瞭望塔，这两个瞭望塔之间的距离要尽可能远，这样才能覆盖最大的海域。这座岛屿上有很多特殊的地点，你可以把它们看作一个个点。

【思考题1】如果岛屿上只有10个地点，你会怎么找最远的两个？
（学生可能会回答：把所有点对的距离都算一遍，然后找出最大的。）

没错，这是最直接的方法，我们称之为“暴力枚举”。如果有 `n` 个地点，我们需要计算大约 `n * (n-1) / 2` 对点的距离。如果 `n` 很大，比如有10万个地点，那这个计算量就非常恐怖了（10万的平方是100亿！）。我们的电脑可受不了！

【思考题2】仔细观察这座岛屿，你觉得最远的两个瞭望塔会建在岛屿的“内部”还是“边缘”呢？
（学生可能会直观地认为在边缘。）

对！聪明！如果一个点在岛屿的“内部”，我们总能找到一个在它“外面”的点，使得距离更远。所以，最远的两个瞭望塔，一定会建在岛屿的“最外围”的那些点上。这些点围成的区域，我们叫做**凸包（Convex Hull）**。就像用一根橡皮筋把所有点都圈起来，橡皮筋绷紧后的形状就是凸包。

那么，问题就变成了：已知一个凸包（一个凸多边形），怎么快速找到上面最远的一对点呢？今天，我们就来学习一个超级酷炫的算法——**旋转卡壳（Rotating Calipers）**，它能像“旋转的尺子”一样，帮你迅速找到答案！

## 深度知识点讲解

### 1. 凸包（Convex Hull）与最远点对

在深入旋转卡壳之前，我们先巩固一个重要的结论：**一个点集中任意两点之间的最远距离，一定发生在它的凸包上的两个顶点之间。**

**为什么？**
假设最远点对 `(A, B)` 中有一个点 `A` 不在凸包上。这意味着 `A` 在凸包的内部。那么，我们可以在凸包上找到一条边 `PQ`，使得 `A` 位于 `PQ` 和凸包内部的某条线段之间。
现在，我们把 `A` 移动到凸包上的某个点 `A'`。例如，我们可以找到一条穿过 `A` 且垂直于 `AB` 的直线 `L`。然后，将 `A` 沿着 `L` 的方向移动到它与凸包的交点 `A'`。
由于 `A'` 在凸包上且方向上可能更远离 `B`，或者至少不会更近，所以 `dist(A', B)` 可能会大于 `dist(A, B)`，或者等于 `dist(A, B)`。
这说明，如果最远点对中有不在凸包上的点，我们可以通过移动它到凸包上，使得距离至少不减小。因此，最远点对一定都在凸包上。
**结论：** 我们可以先求出所有点的凸包，然后只在凸包的顶点上寻找最远点对。
（假设同学们已经掌握了凸包的构建算法，如单调链法或Graham扫描法，这里我们直接使用凸包作为输入。）

### 2. 对踵点对（Antipodal Pair）

这是旋转卡壳算法的核心概念。
**定义：** 在一个凸多边形上，如果存在两条平行线，它们都与多边形相切，并且多边形完全位于这两条平行线之间，那么这两条切线与多边形的交点（或边）所组成的点对，就叫做**对踵点对**。

**生活类比：** 想象你有一个形状不规则的石头（凸多边形），你想把它夹在一把卡尺（Calipers）的两臂之间。当你旋转这块石头时，卡尺的两个臂始终保持平行，并且紧贴着石头。卡尺两臂接触到的点，就是一对对踵点对。

**为什么对踵点对很重要？**
**定理：** 凸多边形上任意两点之间的最远距离，一定发生在某一对对踵点对之间。

**为什么？**
假设 `(P, Q)` 是凸多边形上的最远点对。
1. 我们可以通过 `P` 和 `Q` 分别作一条垂直于线段 `PQ` 的直线 `L1` 和 `L2`。
2. 由于 `(P, Q)` 是最远点对，这意味着多边形的所有点都必须位于 `L1` 和 `L2` 之间。
3. 如果多边形有任何点 `R` 位于 `L1` 的左侧或 `L2` 的右侧，那么 `dist(R, Q)` 或 `dist(P, R)` 就会大于 `dist(P, Q)`，这与 `(P, Q)` 是最远点对的假设矛盾。
4. 因此，`L1` 和 `L2` 实际上就是多边形的两条平行支撑线，它们通过 `P` 和 `Q`。所以 `(P, Q)` 是一对对踵点对。
**结论：** 我们只需要检查所有的对踵点对，就能找到最远点对。

### 3. 旋转卡壳（Rotating Calipers）算法原理

旋转卡壳算法就是一种高效地枚举所有对踵点对的方法。

**核心思想：**
想象我们有两把“卡尺”（两条平行线）。我们让它们绕着凸多边形旋转。在旋转过程中，我们只关注卡尺边缘接触到的点，这些点就是当前的对踵点对。由于凸多边形的性质，当一条卡尺线沿着一条边旋转时，另一条卡尺线也会相应地移动，并且这个移动是单向的。这使得我们可以在线性时间复杂度 `O(N)` 内遍历所有对踵点对。

**具体步骤：**
假设凸多边形有 `m` 个顶点，按逆时针顺序存储在数组 `h[0...m-1]` 中。

1.  **初始化：**
    *   找到一个初始的对踵点对。一个简单的方法是找到 `y` 坐标最小的点 `h[i]` 和 `y` 坐标最大的点 `h[j]`。或者更通用的，找到 `x` 坐标最小的点 `h[i]` 和 `x` 坐标最大的点 `h[j]`。
    *   将 `max_d` 初始化为 `dist2(h[i], h[j])` (为了避免浮点数误差和开方运算，我们通常比较距离的平方)。

2.  **旋转过程：**
    *   我们让两条卡尺线，一条通过 `h[i]` 和 `h[(i+1)%m]` 这条边，另一条通过 `h[j]`。
    *   我们希望找到 `h[j]` 使得它离 `h[i]h[(i+1)%m]` 这条线段最远。这可以通过比较叉积来实现。
    *   **如何移动 `j`？**
        对于当前的边 `h[i] -> h[(i+1)%m]`，我们希望找到点 `h[j]`，使得三角形 `h[i]h[(i+1)%m]h[j]` 的面积最大（因为底边 `h[i]h[(i+1)%m]` 固定，面积最大意味着高最大，即 `h[j]` 离这条边最远）。
        我们可以比较 `cross(h[i], h[(i+1)%m], h[j])` 和 `cross(h[i], h[(i+1)%m], h[(j+1)%m])`。
        如果 `cross(h[i], h[(i+1)%m], h[(j+1)%m])` 更大，说明 `h[(j+1)%m]` 离 `h[i]h[(i+1)%m]` 更远，那么我们就将 `j` 推进到 `(j+1)%m`。
        我们不断推进 `j`，直到 `cross(h[i], h[(i+1)%m], h[j])` 达到最大值，或者说 `cross(h[i], h[(i+1)%m], h[(j+1)%m])` 不再比 `cross(h[i], h[(i+1)%m], h[j])` 大。
    *   **更新最远距离：**
        一旦找到了与边 `h[i]h[(i+1)%m]` 对应的“最远”点 `h[j]`，我们就更新 `max_d = max(max_d, dist2(h[i], h[j]))` 和 `max_d = max(max_d, dist2(h[(i+1)%m], h[j]))`。这是因为 `h[j]` 可能是离 `h[i]` 最远，也可能是离 `h[(i+1)%m]` 最远。
    *   **移动 `i`：**
        然后，我们将 `i` 推进到 `(i+1)%m`，继续下一条边的处理。
    *   **终止条件：**
        当 `i` 遍历完所有边（例如 `i` 从 `0` 到 `m-1`），整个过程就结束了。因为 `j` 也是单向前进，并且总共前进的距离不超过 `2m`，所以总时间复杂度是 `O(m)`。

### 4. 几何原语：叉积

叉积在计算几何中非常有用，可以判断点的相对位置，也常用于计算面积。
对于二维向量 `OA` 和 `OB`，它们的叉积定义为 `OA.x * OB.y - OA.y * OB.x`。
如果点 `O`, `A`, `B` 的坐标分别为 `(Ox, Oy)`, `(Ax, Ay)`, `(Bx, By)`，那么向量 `OA` 为 `(Ax-Ox, Ay-Oy)`，向量 `OB` 为 `(Bx-Ox, By-Oy)`。
`cross(O, A, B)` = `(Ax - Ox) * (By - Oy) - (Ay - Oy) * (Bx - Ox)`。

*   **几何意义：** 叉积的绝对值等于以 `OA` 和 `OB` 为邻边的平行四边形的面积。
*   **方向判断：**
    *   `cross(O, A, B) > 0`：`OB` 在 `OA` 的逆时针方向（或 `B` 在 `OA` 的左侧）。
    *   `cross(O, A, B) < 0`：`OB` 在 `OA` 的顺时针方向（或 `B` 在 `OA` 的右侧）。
    *   `cross(O, A, B) == 0`：`O`, `A`, `B` 三点共线。

在旋转卡壳中，`cross(h[i], h[(i+1)%m], h[j])` 的值表示 `h[j]` 到线段 `h[i]h[(i+1)%m]` 距离的“两倍”乘上底边长度。所以比较叉积大小可以直接比较距离。

### 5. 常见误区与注意事项

*   **坐标类型：** 题目中的坐标通常是整数，但距离平方和叉积的结果可能非常大，需要使用 `long long` 类型存储。
*   **浮点数误差：** 避免直接使用 `sqrt` 进行距离比较，而是比较距离的平方 `dist2`。
*   **凸包的顺序：** 凸包的顶点必须是逆时针或顺时针有序的，通常是逆时针。
*   **边界条件：** 模运算 `(x+1)%m` 要正确处理，确保索引在 `[0, m-1]` 范围内。
*   **共线点：** 如果凸包上有三点共线，可能会导致叉积为0。旋转卡壳算法通常对共线点有良好的容忍度，但凸包构建阶段需要注意处理。

## 典型例题精讲

**例题：最远点对**

给定平面上的 `N` 个点，求这些点中距离最远的一对点的距离。

**输入格式：**
第一行一个整数 `N`。
接下来 `N` 行，每行两个整数 `x, y`，表示一个点的坐标。

**输出格式：**
一个整数，表示最远点对距离的平方。

**输入样例：**
```
4
0 0
1 0
0 1
1 1
```

**输出样例：**
```
2
```
（因为 `(0,0)` 到 `(1,1)` 的距离平方是 `(1-0)^2 + (1-0)^2 = 1+1=2`）

**思路分析：**

1.  **构建凸包：** 首先，我们需要从给定的 `N` 个点中构建出它们的凸包。这里我们使用 Monotone Chain（单调链法）来构建，它能得到逆时针有序的凸包顶点。
    *   对所有点按 `x` 坐标升序排序，如果 `x` 相同则按 `y` 坐标升序排序。
    *   构建下凸壳：从左到右遍历排序后的点，使用栈（或数组模拟栈）维护当前凸壳。每加入一个点时，如果它与栈顶的两个点构成“右转”或共线，则弹出栈顶（因为它们不是凸壳的一部分）。
    *   构建上凸壳：从右到左遍历排序后的点，同样方法构建。
    *   合并上下凸壳，并去除重复点，得到最终的凸包 `h`。

2.  **旋转卡壳：** 获得凸包 `h` (假设有 `m` 个点) 后，应用旋转卡壳算法。
    *   初始化 `max_d = 0`。
    *   初始化两个指针 `i = 0`, `j = 1`。
    *   如果 `m < 2`，最远距离为0。如果 `m == 2`，最远距离就是这两个点的距离。
    *   主循环 `i` 从 `0` 到 `m-1`：
        *   内层 `while` 循环：不断移动 `j`，直到 `h[j]` 是当前边 `h[i]h[(i+1)%m]` 的最远点。即 `while (cross(h[i], h[(i+1)%m], h[j]) < cross(h[i], h[(i+1)%m], h[(j+1)%m]))` 成立时，`j = (j+1)%m`。
        *   更新 `max_d`：`max_d = max(max_d, dist2(h[i], h[j]))` 和 `max_d = max(max_d, dist2(h[(i+1)%m], h[j]))`。
    *   返回 `max_d`。

**时间复杂度分析：**

*   **凸包构建：** `O(N log N)` (排序) + `O(N)` (单调链扫描) = `O(N log N)`。
*   **旋转卡壳：** `i` 指针从 `0` 到 `m-1` 遍历 `m` 次。在 `i` 的每次迭代中，`j` 指针也只会单向前进，总共前进的次数不超过 `2m` 次。每次操作（叉积、距离计算）都是 `O(1)`。所以旋转卡壳部分是 `O(m)`。
*   **总复杂度：** `O(N log N) + O(m)`。由于 `m <= N`，所以总复杂度为 `O(N log N)`。这比暴力枚举的 `O(N^2)` 大大提高效率。

**代码实现（伪代码结构，完整代码见模板）：**

```cpp
// Point 结构体
struct Point {
    long long x, y;
};

// 叉积函数
long long cross(Point o, Point a, Point b) {
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x);
}

// 距离平方函数
long long dist2(Point p1, Point p2) {
    return (p1.x - p2.x) * (p1.x - p2.x) + (p1.y - p2.y) * (p1.y - p2.y);
}

// 凸包函数 (Monotone Chain)
// 输入 n 个点，输出凸包 h (m 个点，逆时针顺序)
// (此处省略具体实现，假设已完成或作为前置知识)
void convexHull(Point ps[], int n, Point h[], int& m) {
    // ... Monotone Chain implementation ...
}


int main() {
    // ... 读取 N 个点到 ps[] ...

    // 1. 构建凸包
    Point h[N_MAX]; // 存储凸包点
    int m = 0;      // 凸包点数量
    convexHull(ps, n, h, m);

    // 2. 旋转卡壳
    if (m < 2) {
        printf("0\n"); // 0或1个点，最远距离为0
        return 0;
    }
    if (m == 2) {
        printf("%lld\n", dist2(h[0], h[1])); // 只有2个点，直接计算
        return 0;
    }

    long long max_d = 0;
    int j = 1; // 初始j可以随便取，比如1，或者找到y最大的点

    for (int i = 0; i < m; ++i) {
        // 确保 j 领先于 i，并且 j 不等于 i
        // 也可以将 j 初始化为 i+1
        while (j < i) { // 保证j至少在i的后面一个点，或者在它前面转了一圈
            j = (j + 1) % m;
        }

        // 寻找与 h[i]h[(i+1)%m] 边对应的最远点 h[j]
        // 只要 h[(j+1)%m] 离当前边更远，就推进 j
        while (cross(h[i], h[(i + 1) % m], h[j]) < cross(h[i], h[(i + 1) % m], h[(j + 1) % m])) {
            j = (j + 1) % m;
        }
        
        // 更新最远距离
        max_d = std::max(max_d, dist2(h[i], h[j]));
        max_d = std::max(max_d, dist2(h[(i + 1) % m], h[j])); // h[j]可能与h[i+1]距离最远
    }

    printf("%lld\n", max_d);

    return 0;
}
```

**关于 `j` 的初始化和推进：**
更严谨的 `j` 初始化和推进方式：
初始化 `j = 1`。
主循环 `i` 从 `0` 到 `m-1`。
在 `i` 循环内部，`j` 始终保持在 `i` 的前面，并且 `j` 的总移动次数是 `O(m)`。
`while (j < m && cross(h[i], h[(i+1)%m], h[j]) < cross(h[i], h[(i+1)%m], h[(j+1)%m])) { j++; }`
这里 `j` 不再取模，而是让 `j` 可以跑到 `m` 甚至 `2m-1`。然后使用 `h[j%m]` 访问点。
这样可以避免 `j < i` 的判断，直接让 `j` 持续前进。

```cpp
// 旋转卡壳更标准的 j 推进方式
// 假设 m >= 2
long long max_d = 0;
int j = 1; // 初始 j 可以是 1
// 找到初始的 j，使其成为 h[0]h[1] 边的最远点
while (j < m - 1 && cross(h[0], h[1], h[j]) < cross(h[0], h[1], h[j+1])) {
    j++;
}

for (int i = 0; i < m; ++i) {
    // 确保 j 总是指向对踵点
    // 当 i 移动到下一个点时，j 也会向前移动，维持其对踵点性质
    // 注意这里 h[i] 和 h[(i+1)%m] 形成一条边
    // 寻找与这条边平行的最远点 h[j]
    while (cross(h[i], h[(i + 1) % m], h[j]) < cross(h[i], h[(i + 1) % m], h[(j + 1) % m])) {
        j = (j + 1) % m;
    }
    
    // 更新最远距离
    max_d = std::max(max_d, dist2(h[i], h[j]));
    max_d = std::max(max_d, dist2(h[(i + 1) % m], h[j])); 
}
```
这个 `j` 推进逻辑是正确的，它保证了 `j` 总是“追赶” `i`，并且总共遍历了 `2m` 次。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用库，符合GESP要求

// 定义点结构体
struct Point {
    long long x, y; // 坐标可能较大，使用 long long
};

// 全局变量，用于存储原始点和凸包点
const int N_MAX = 100005; // 最大点数
Point ps[N_MAX];       // 存储原始输入点
Point h[N_MAX];        // 存储凸包上的点

// 比较函数，用于点排序（先按 x 坐标，再按 y 坐标）
bool cmp(Point p1, Point p2) {
    if (p1.x != p2.x) {
        return p1.x < p2.x;
    }
    return p1.y < p2.y;
}

// 计算向量 OA 和 OB 的叉积 (OA x OB)
// 用于判断 B 在 OA 的左侧、右侧还是共线
// 返回值 > 0 表示逆时针 (左侧)
// 返回值 < 0 表示顺时针 (右侧)
// 返回值 = 0 表示共线
long long cross(Point o, Point a, Point b) {
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x);
}

// 计算两点之间距离的平方
// 避免浮点数运算和开方，提高精度和效率
long long dist2(Point p1, Point p2) {
    return (p1.x - p2.x) * (p1.x - p2.x) + (p1.y - p2.y) * (p1.y - p2.y);
}

// 构建凸包 (Monotone Chain 算法)
// 输入: ps[] 原始点数组, n 原始点数量
// 输出: h[] 凸包点数组 (逆时针有序), m 凸包点数量
void convexHull(Point ps[], int n, Point h[], int& m) {
    // 1. 对所有点进行排序
    std::sort(ps, ps + n, cmp);

    m = 0; // 凸包点数量初始化为 0

    // 2. 构建下凸壳
    for (int i = 0; i < n; ++i) {
        // 当 m >= 2 且新加入的点与栈顶的两个点构成非逆时针转弯时，弹出栈顶
        // 叉积 <= 0 表示顺时针转弯或共线
        while (m >= 2 && cross(h[m - 2], h[m - 1], ps[i]) <= 0) {
            m--;
        }
        h[m++] = ps[i]; // 将当前点加入凸壳
    }

    // 记录下凸壳的最后一个点索引 (上凸壳的起始点)
    int lower_hull_size = m;

    // 3. 构建上凸壳 (从右向左遍历)
    // 注意：ps[n-1] 已经在下凸壳中，所以从 n-2 开始
    // ps[0] 也会被加入，所以循环到 0
    for (int i = n - 2; i >= 0; --i) {
        // 当 m >= lower_hull_size + 1 且新加入的点与栈顶的两个点构成非逆时针转弯时，弹出栈顶
        while (m > lower_hull_size && cross(h[m - 2], h[m - 1], ps[i]) <= 0) {
            m--;
        }
        h[m++] = ps[i]; // 将当前点加入凸壳
    }

    // 4. 去除重复的起始点 (ps[0] 和 ps[n-1] 可能会被加入两次)
    // 如果凸包有多个点，且首尾点重复，则 m--
    if (m > 1 && h[0].x == h[m-1].x && h[0].y == h[m-1].y) {
        m--;
    }
}

// 旋转卡壳算法，计算凸包 h 上的最远点对距离的平方
long long rotatingCalipers(Point h[], int m) {
    // 凸包点数小于2，最远距离为0
    if (m < 2) {
        return 0;
    }
    // 凸包点数等于2，最远距离就是这两点距离
    if (m == 2) {
        return dist2(h[0], h[1]);
    }

    long long max_d = 0; // 记录最大距离平方
    int j = 1;           // 指针 j，初始指向 h[1]

    // 遍历凸包的每一条边 h[i] -> h[(i+1)%m]
    for (int i = 0; i < m; ++i) {
        // 寻找与当前边 h[i]h[(i+1)%m] 对应的最远点 h[j]
        // cross(h[i], h[(i+1)%m], h[j]) 表示三角形 h[i]h[(i+1)%m]h[j] 面积的两倍
        // 面积越大，点 h[j] 离边 h[i]h[(i+1)%m] 越远
        // 这里的 (j+1)%m 确保 j 循环前进
        while (cross(h[i], h[(i + 1) % m], h[j]) < cross(h[i], h[(i + 1) % m], h[(j + 1) % m])) {
            j = (j + 1) % m; // 推进 j
        }
        
        // 更新最大距离
        // h[j] 是与边 h[i]h[(i+1)%m] 最远的对踵点
        // 最远点对可能是 (h[i], h[j]) 或 (h[(i+1)%m], h[j])
        max_d = std::max(max_d, dist2(h[i], h[j]));
        max_d = std::max(max_d, dist2(h[(i + 1) % m], h[j])); 
    }

    return max_d;
}

int main() {
    // 优化输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 原始点数量
    std::cin >> n;

    for (int i = 0; i < n; ++i) {
        std::cin >> ps[i].x >> ps[i].y;
    }

    int m; // 凸包点数量
    convexHull(ps, n, h, m); // 构建凸包

    long long result = rotatingCalipers(h, m); // 运行旋转卡壳

    std::cout << result << std::endl; // 输出结果

    return 0;
}
```

2.  **提问2：** 在 `cross(o, a, b)` 函数中，如果 `cross` 值为 0，代表什么？这在构建凸包和旋转卡壳时有什么特殊意义吗？
    *   **预期回答：** 代表 `o, a, b` 三点共线。
        *   在构建凸包时，如果 `cross(h[m-2], h[m-1], ps[i]) <= 0`，我们弹出 `h[m-1]`。这意味着如果 `ps[i]` 与 `h[m-2], h[m-1]` 共线，`h[m-1]` 也会被弹出。这会得到一个“严格”的凸包，不包含共线点。如果允许共线点在边上，则使用 `< 0`。
        *   在旋转卡壳中，如果叉积为 0，说明 `h[j]` 在 `h[i]h[(i+1)%m]` 延长线上。这表示 `h[j]` 刚好是这条边的一个顶点，或者 `h[j]` 位于这条边上（这种情况在凸多边形中通常不会发生，除非 `h[i]h[(i+1)%m]` 本身就是凸包的一条边，且 `h[j]` 是 `h[i]` 或 `h[i+1]`）。更常见的是，如果 `cross(h[i], h[(i+1)%m], h[j]) == cross(h[i], h[(i+1)%m], h[(j+1)%m])`，说明 `h[j]` 和 `h[(j+1)%m]` 都离 `h[i]h[(i+1)%m]` 一样远，它们可能都是与该边平行的对踵点。

3.  **提问3：** 为什么旋转卡壳算法的 `j` 指针可以只向前移动，而不需要回溯？它的总移动次数为什么是 `O(m)`？
    *   **预期回答：** 因为凸多边形的边是按逆时针（或顺时针）顺序排列的，其角度是单调变化的。当 `i` 指针从 `h[i]` 移动到 `h[(i+1)%m]` 时，卡尺的“方向”也发生了逆时针旋转，那么与新方向平行的对踵点 `h[j]` 也必然会沿逆时针方向前进或保持不变，而不会后退。所以 `j` 指针只需要单向前进。由于 `i` 遍历 `m` 条边，`j` 最多也只循环一圈多一点，所以总移动次数是 `O(m)`。
