## 趣味引入

同学们，大家有没有遇到过这样的情况：

1.  你在手机上打字，不小心把“苹果”打成了“pingguo”，但输入法却立刻帮你纠正成了“苹果”。它是怎么知道你打错了，而且知道你最可能想打的是“苹果”呢？
2.  你在写作文时，把“漂亮”写成了“漂酿”，或者把“开心”写成了“开森”，然后Word文档下面出现了红色的波浪线，提示你可能拼写错误。它又是怎么判断的呢？
3.  玩游戏时，你输入了一个指令“go north”，结果打成了“go noth”，游戏提示你指令无效。但如果游戏能聪明一点，识别出你只是少打了一个'r'，是不是会更方便？

这些例子背后，都藏着一个非常有趣的算法概念——**编辑距离**！

想象一下，我们有两个单词，比如 `s1 = "kitten"` 和 `s2 = "sitting"`。我们的目标是把 `s1` 变成 `s2`。每次操作，我们可以选择：
*   **插入** 一个字符 (Insert)
*   **删除** 一个字符 (Delete)
*   **替换** 一个字符 (Replace)

每进行一次操作，我们就花费一点“魔法能量”。我们的任务就是，用最少的“魔法能量”，把 `s1` 完全变成 `s2`。这个最少的“魔法能量”消耗，就是它们之间的“编辑距离”。

比如 `kitten` 变成 `sitting`：
1.  `k` -> `s` （替换）
2.  `i` -> `i` （匹配，不操作）
3.  `t` -> `t` （匹配，不操作）
4.  `t` -> `t` （匹配，不操作）
5.  `e` -> `i` （替换）
6.  `n` -> `n` （匹配，不操作）
7.  最后 `g` （插入）

这样一共用了3次操作：替换'k'为's'，替换'e'为'i'，插入'g'。编辑距离就是3。

这听起来是不是像一个寻找最短路径的迷宫游戏？没错！而这种“求最值”的问题，往往就是我们动态规划（Dynamic Programming，简称DP）大展身手的好机会！今天，我们就一起来揭秘编辑距离的魔法！

## 深度知识点讲解

### 1. 编辑距离的本质：字符串的“相似度”量化

编辑距离，也叫莱文斯坦距离（Levenshtein Distance），它衡量的是将一个字符串转换成另一个字符串所需的最少单字符编辑操作次数。这些操作包括：插入、删除、替换。

**为什么是“最少”操作？**
因为我们可能有很多种方法把 `s1` 变成 `s2`，但我们只关心代价最小的那一种。比如把 "cat" 变成 "cut"：
*   方法1: 替换 'a' 为 'u' (1次操作)
*   方法2: 删除 'a'，插入 'u' (2次操作)
显然，方法1更好。

### 2. 动态规划的思考：大问题分解小问题

DP的核心思想是：把一个大问题拆解成相互关联的小问题，然后从小问题的解推导出大问题的解。
对于编辑距离，我们怎么拆分呢？

假设我们有两个字符串 `s1` 和 `s2`，长度分别是 `n` 和 `m`。
我们想知道 `s1` 的前 `i` 个字符 (`s1[0...i-1]`) 转换成 `s2` 的前 `j` 个字符 (`s2[0...j-1]`) 的最小编辑距离。

**状态定义：**
我们用 `dp[i][j]` 来表示 `s1` 的前 `i` 个字符与 `s2` 的前 `j` 个字符之间的最小编辑距离。

**生活类比：厨师做菜**
想象你是一个厨师，你手里有 `s1` 的前 `i` 份食材，你的食谱要求你做出 `s2` 的前 `j` 份菜品。
你现在考虑的是 `s1` 的第 `i` 份食材 (也就是 `s1[i-1]`) 和 `s2` 的第 `j` 份菜品 (也就是 `s2[j-1]`)。

### 3. 边界条件（Base Cases）：DP表的起点

在开始“做菜”之前，总得有个基础。
*   **`dp[0][0]`：** 空字符串变成空字符串，不需要任何操作，所以 `dp[0][0] = 0`。
*   **`dp[i][0]`：** `s1` 的前 `i` 个字符变成空字符串。这意味着要把 `s1` 的这 `i` 个字符全部删除。需要 `i` 次删除操作。
    所以，`dp[i][0] = i` (对于 `i` 从 `1` 到 `n`)。
*   **`dp[0][j]`：** 空字符串变成 `s2` 的前 `j` 个字符。这意味着要插入 `s2` 的这 `j` 个字符。需要 `j` 次插入操作。
    所以，`dp[0][j] = j` (对于 `j` 从 `1` 到 `m`)。

这就像是：如果你手里有食材但食谱是空的，你就得扔掉所有食材；如果食谱要求你做菜但你手里没有食材，你就得去采购所有食材。

### 4. 状态转移方程：三种“魔法”操作的选择

现在，我们来考虑如何从 `dp` 表中已经计算出的“小问题”答案，推导出 `dp[i][j]` 的答案。
当我们要计算 `dp[i][j]` 时，我们关注的是 `s1[i-1]` (s1的第i个字符) 和 `s2[j-1]` (s2的第j个字符)。

**情况一：`s1[i-1]` 和 `s2[j-1]` 相同 (字符匹配)**
如果 `s1[i-1]` 和 `s2[j-1]` 这两个字符本来就一样，那真是太棒了！我们不需要对它们进行任何操作。
这时，`s1` 的前 `i` 个字符转换成 `s2` 的前 `j` 个字符的最小代价，就等于 `s1` 的前 `i-1` 个字符转换成 `s2` 的前 `j-1` 个字符的最小代价。
`dp[i][j] = dp[i-1][j-1]`
（这相当于，最后一步我们发现两个字符一样，就不用管它们了，直接看前面部分的转换。）

**情况二：`s1[i-1]` 和 `s2[j-1]` 不相同 (字符不匹配)**
如果它们不相同，我们就必须使用一次操作来处理它们。这时有三种选择，我们选择代价最小的那一种：

1.  **删除 `s1[i-1]` 操作：**
    我们把 `s1` 的第 `i` 个字符 (`s1[i-1]`) 删掉。
    那么问题就变成了：`s1` 的前 `i-1` 个字符转换成 `s2` 的前 `j` 个字符。
    这个代价是 `dp[i-1][j]`。再加上我们这次删除操作的代价 `1`。
    总代价：`dp[i-1][j] + 1`

2.  **插入 `s2[j-1]` 操作：**
    我们在 `s1` 的末尾（也就是 `s1[i-1]` 之后）插入一个字符，使它变成 `s2[j-1]`。
    那么问题就变成了：`s1` 的前 `i` 个字符转换成 `s2` 的前 `j-1` 个字符。
    这个代价是 `dp[i][j-1]`。再加上我们这次插入操作的代价 `1`。
    总代价：`dp[i][j-1] + 1`

3.  **替换 `s1[i-1]` 为 `s2[j-1]` 操作：**
    我们把 `s1` 的第 `i` 个字符 (`s1[i-1]`) 替换成 `s2` 的第 `j` 个字符 (`s2[j-1]`)。
    那么问题就变成了：`s1` 的前 `i-1` 个字符转换成 `s2` 的前 `j-1` 个字符。
    这个代价是 `dp[i-1][j-1]`。再加上我们这次替换操作的代价 `1`。
    总代价：`dp[i-1][j-1] + 1`

因此，当 `s1[i-1]` 和 `s2[j-1]` 不相同时，`dp[i][j]` 就等于这三种操作中代价最小的那一个：
`dp[i][j] = min(dp[i-1][j] + 1, dp[i][j-1] + 1, dp[i-1][j-1] + 1)`

**综合状态转移方程：**
我们可以把两种情况合并起来。定义一个 `cost` 变量：
如果 `s1[i-1] == s2[j-1]`，则 `cost = 0` (不需要额外操作)
如果 `s1[i-1] != s2[j-1]`，则 `cost = 1` (需要一次替换操作)

那么，最终的转移方程就是：
`dp[i][j] = min(dp[i-1][j] + 1,        // 删除 s1[i-1]`
              `dp[i][j-1] + 1,        // 插入 s2[j-1]`
              `dp[i-1][j-1] + cost);  // 替换 s1[i-1] 为 s2[j-1] 或 匹配`

### 5. 常见误区与注意事项

*   **字符串索引与DP状态索引的对应：** DP状态 `dp[i][j]` 通常表示前 `i` 个和前 `j` 个字符，对应字符串的索引是 `s1[0...i-1]` 和 `s2[0...j-1]`。因此，在访问 `s1` 的第 `i` 个字符时，应该用 `s1[i-1]`。这是最容易出错的地方！
*   **DP表的初始化：** 确保 `dp[0][j]` 和 `dp[i][0]` 都正确初始化。
*   **循环顺序：** `i` 和 `j` 的循环必须从 `1` 开始，因为 `dp[i][j]` 依赖于 `dp[i-1][...]` 和 `dp[...][j-1]`。

### 6. 时间复杂度和空间复杂度

*   **时间复杂度：** 我们需要填充一个 `(n+1) * (m+1)` 大小的DP表，每个单元格的计算是常数时间。所以总时间复杂度是 **O(n * m)**。
*   **空间复杂度：** 我们需要一个 `(n+1) * (m+1)` 大小的二维数组来存储DP状态。所以总空间复杂度是 **O(n * m)**。
    （对于某些题目，可以优化空间到 O(min(n, m))，但对于 GESP 7 级，先掌握 O(n*m) 的标准解法即可。）

## 典型例题精讲

**例题：计算两个单词的最小编辑距离**

**题目描述：**
给定两个单词 `word1` 和 `word2`，计算将 `word1` 转换成 `word2` 所使用的最少操作数。
你可以对一个单词进行如下三种操作：
1.  插入一个字符
2.  删除一个字符
3.  替换一个字符

**输入：**
两行字符串，分别代表 `word1` 和 `word2`。
例如：
`horse`
`ros`

**输出：**
一个整数，表示最小编辑距离。
例如：
`3`

**思路分析：**

1.  **定义DP状态：** `dp[i][j]` 表示 `word1` 的前 `i` 个字符 (`word1[0...i-1]`) 转换成 `word2` 的前 `j` 个字符 (`word2[0...j-1]`) 的最小编辑距离。
    设 `n` 为 `word1` 的长度，`m` 为 `word2` 的长度。我们的目标是计算 `dp[n][m]`。

2.  **初始化DP表：**
    *   `dp[0][0] = 0` (空字符串转空字符串)
    *   `dp[i][0] = i` (对于 `i` 从 `1` 到 `n`，`word1` 的前 `i` 个字符转空字符串需要 `i` 次删除)
    *   `dp[0][j] = j` (对于 `j` 从 `1` 到 `m`，空字符串转 `word2` 的前 `j` 个字符需要 `j` 次插入)

3.  **填充DP表（状态转移）：**
    从 `i = 1` 到 `n`，从 `j = 1` 到 `m` 遍历。
    对于每个 `dp[i][j]`：
    *   **计算 `cost`：** 如果 `word1[i-1] == word2[j-1]`，则 `cost = 0`；否则 `cost = 1`。
    *   **转移方程：**
        `dp[i][j] = min(dp[i-1][j] + 1,        // 删除 word1[i-1]`
                      `dp[i][j-1] + 1,        // 插入 word2[j-1]`
                      `dp[i-1][j-1] + cost);  // 替换 word1[i-1] 或 匹配`

4.  **最终结果：** `dp[n][m]` 即为所求。

**以 `word1 = "horse"`, `word2 = "ros"` 为例，手动模拟DP表：**

`word1` 长度 `n = 5`，`word2` 长度 `m = 3`。
DP表大小 `(n+1) x (m+1)` 即 `6 x 4`。

| :--------- | :-------- | :-------- | :-------- | :-------- |
| ` ` (i=0)  | 0         | 1         | 2         | 3         |
| `h` (i=1)  | 1         | `dp[1][1]` | `dp[1][2]` | `dp[1][3]` |
| `o` (i=2)  | 2         | `dp[2][1]` | `dp[2][2]` | `dp[2][3]` |

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 假设字符串的最大长度为1000
const int MAXN = 1005;
char s1[MAXN];
char s2[MAXN];
int dp[MAXN][MAXN];

int main()
{
    // 禁用同步，加速输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> s1 + 1; // 从索引1开始存储，方便DP状态对应
    std::cin >> s2 + 1; // 从索引1开始存储

    int n = strlen(s1 + 1); // s1的实际长度
    int m = strlen(s2 + 1); // s2的实际长度

    // 初始化边界条件
    // s1的前i个字符变成空字符串，需要i次删除
    for (int i = 0; i <= n; ++i)
    {
        dp[i][0] = i;
    }

    // 空字符串变成s2的前j个字符，需要j次插入
    for (int j = 0; j <= m; ++j)
    {
        dp[0][j] = j;
    }

    // 填充DP表
    for (int i = 1; i <= n; ++i)
    {
        for (int j = 1; j <= m; ++j)
        {
            // 如果当前字符匹配，cost为0；否则为1（替换操作）
            int cost = (s1[i] == s2[j]) ? 0 : 1;

            // 1. 删除 s1[i]：s1的前i-1个字符转s2的前j个字符，然后删除s1[i]
            int delete_op = dp[i - 1][j] + 1;

            // 2. 插入 s2[j]：s1的前i个字符转s2的前j-1个字符，然后插入s2[j]
            int insert_op = dp[i][j - 1] + 1;

            // 3. 替换 s1[i] 为 s2[j] (如果不同) 或 匹配 (如果相同)
            int replace_or_match_op = dp[i - 1][j - 1] + cost;

            // 取三种操作中的最小值
            dp[i][j] = std::min({delete_op, insert_op, replace_or_match_op});
        }
    }

    // 最终结果是s1的全部字符转s2的全部字符的编辑距离
    std::cout << dp[n][m] << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 两个嵌套循环，分别从 `1` 到 `n` 和 `1` 到 `m`。循环体内操作为常数时间。所以总时间复杂度为 `O(n * m)`。
*   **空间复杂度：** 使用了一个 `(n+1) * (m+1)` 大小的二维数组 `dp`。所以总空间复杂度为 `O(n * m)`。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，方便竞赛使用

// 定义字符串的最大长度，根据题目要求调整
const int MAX_LEN = 1005; 

// 使用字符数组存储字符串，符合GESP要求
char s1[MAX_LEN];
char s2[MAX_LEN];

// 定义DP数组，dp[i][j] 表示 s1的前i个字符 与 s2的前j个字符 的最小编辑距离
// 数组大小为 (MAX_LEN * MAX_LEN)
int dp[MAX_LEN][MAX_LEN]; 

int main()
{
    // 提升C++标准库输入输出流的效率，在竞赛中常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    // 读取两个字符串
    // 注意：s1+1 和 s2+1 是为了让字符串从数组的索引1开始存储
    // 这样 s1[i] 对应 s1的第i个字符，与dp状态 dp[i][j] 更自然地对应
    std::cin >> s1 + 1;
    std::cin >> s2 + 1;

    // 获取字符串的实际长度
    // strlen(s1+1) 计算的是从 s1[1] 开始的字符串长度
    int n = strlen(s1 + 1); 
    int m = strlen(s2 + 1);

    // --- 步骤1: 初始化DP表的边界条件 ---

    // 将空字符串 (s1的前0个字符) 转换为 s2的前 j 个字符，需要 j 次插入操作
    for (int j = 0; j <= m; ++j)
    {
        dp[0][j] = j;
    }

    // 将 s1的前 i 个字符 转换为 空字符串 (s2的前0个字符)，需要 i 次删除操作
    for (int i = 0; i <= n; ++i)
    {
        dp[i][0] = i;
    }

    // --- 步骤2: 填充DP表，进行状态转移 ---

    // 外层循环遍历 s1 的每个字符 (从第1个到第n个)
    for (int i = 1; i <= n; ++i)
    {
        // 内层循环遍历 s2 的每个字符 (从第1个到第m个)
        for (int j = 1; j <= m; ++j)
        {
            // 计算当前字符是否匹配的代价
            // 如果 s1的第i个字符 (s1[i]) 与 s2的第j个字符 (s2[j]) 相同，cost为0
            // 如果不同，cost为1 (表示需要一次替换操作)
            int cost = (s1[i] == s2[j]) ? 0 : 1;

            // 考虑三种可能的转换操作，并取其最小值：

            // 1. 删除 s1[i]：
            //    相当于将 s1的前 i-1 个字符 转换为 s2的前 j 个字符，
            //    然后再删除 s1的第 i 个字符。
            //    代价为 dp[i-1][j] + 1 (1是删除操作的代价)
            int delete_op = dp[i - 1][j] + 1;

            // 2. 插入 s2[j]：
            //    相当于将 s1的前 i 个字符 转换为 s2的前 j-1 个字符，
            //    然后再插入 s2的第 j 个字符。
            //    代价为 dp[i][j-1] + 1 (1是插入操作的代价)
            int insert_op = dp[i][j - 1] + 1;

            // 3. 替换 s1[i] 为 s2[j] (如果不同) 或 匹配 (如果相同)：
            //    相当于将 s1的前 i-1 个字符 转换为 s2的前 j-1 个字符，
            //    然后再处理 s1的第 i 个字符 和 s2的第 j 个字符。
            //    代价为 dp[i-1][j-1] + cost (cost为0或1，取决于是否匹配)
            int replace_or_match_op = dp[i - 1][j - 1] + cost;

            // dp[i][j] 取这三种操作中的最小代价
            dp[i][j] = std::min({delete_op, insert_op, replace_or_match_op});
        }
    }

    // --- 步骤3: 输出最终结果 ---
    // 最终结果是 s1 的全部字符 (前 n 个) 转换为 s2 的全部字符 (前 m 个) 的最小编辑距离
    std::cout << dp[n][m] << std::endl;

    return 0; // 程序正常退出
}

```
