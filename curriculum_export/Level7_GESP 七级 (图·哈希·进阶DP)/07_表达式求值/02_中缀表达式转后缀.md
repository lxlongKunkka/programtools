# 中缀表达式转后缀

## 教学目标

本节课我们将深入学习计算机如何理解和处理复杂的算术表达式，特别是中缀表达式向后缀表达式的转换。

1.  **理解**：掌握中缀表达式、前缀表达式和后缀表达式（逆波兰表达式）的概念、特点及它们之间的区别。
2.  **掌握**：理解为什么需要将中缀表达式转换为后缀表达式，以及后缀表达式在计算机处理中的优势。
3.  **熟悉**：掌握中缀表达式转换为后缀表达式的基本算法流程，特别是运算符优先级和括号的处理规则。
4.  **了解**：学习如何使用原生数组模拟栈数据结构，以应对GESP考试中对STL容器的限制。

### 能力目标

2.  **能够**：利用C++语言，使用数组模拟栈实现中缀表达式到后缀表达式的转换算法。
3.  **能够**：对给定的中缀表达式进行正确解析，区分操作数和运算符。

### 素养目标
1.  **培养**：逻辑思维能力和抽象建模能力，理解将实际问题（表达式求值）转化为计算机可处理形式的思路。

3.  **激发**：对数据结构和算法的兴趣，为后续更深入的学习打下基础。

## 趣味引入

同学们，你们有没有想过，我们平时写算式 `(2 + 3) * 5` 这种形式，计算机是怎么理解并计算出结果的呢？

想象一下，小明和小红是好朋友。小明是个“直性子”，他拿到算式就想立刻从左到右算。比如 `2 + 3 * 5`，他可能会先算 `2 + 3 = 5`，再算 `5 * 5 = 25`。但小红是个“聪明蛋”，她知道要“先乘除后加减”，所以她会先算 `3 * 5 = 15`，再算 `2 + 15 = 17`。

你看，同一个算式，因为计算顺序不同，结果就可能不一样！这都是因为我们平时写的算式——叫做**中缀表达式**，它有复杂的“潜规则”：
1.  **优先级**：乘除法要比加减法“先做”。
2.  **结合性**：同级运算从左到右（或从右到左）。
3.  **括号**：括号里面的要“优先”计算。

这些规则对我们人类来说很自然，但对于“一根筋”的计算机来说，可就太麻烦了！计算机可不喜欢“潜规则”，它喜欢“明规则”：**从左到右，一步一步，绝不回头**。

那么，有没有一种表达式，能让计算机不用管优先级和括号，直接从左到右就能算出结果呢？当然有！这就是我们今天要学习的**后缀表达式**，也叫**逆波兰表达式**。它就像给计算机设计了一套“傻瓜式”的计算流程，让计算机能高效地工作！

所以，这节课的目标就是，教会我们的计算机——或者说，教会我们自己——如何把人类习惯的“中缀表达式”翻译成计算机喜欢的“后缀表达式”。这是一个非常经典且实用的算法哦！

## 深度知识点讲解

### 1. 表达式的三种表示法：中缀、前缀、后缀

在讲解转换算法之前，我们先来认识一下表达式的三种常见表示方法。

*   **中缀表达式 (Infix Expression)**
    *   **特点**：运算符在两个操作数**中间**。
    *   **例子**：`a + b`, `(a - b) * c`, `2 + 3 * 5`
    *   **优点**：符合人类的自然思维习惯，易于阅读。
    *   **缺点**：计算机处理困难，需要考虑运算符优先级、结合性和括号。

*   **前缀表达式 (Prefix Expression)**，也叫**波兰表达式**
    *   **特点**：运算符在两个操作数**前面**。
    *   **例子**：`+ a b`, `* - a b c`, `+ 2 * 3 5` (对应 `2 + 3 * 5`)
    *   **优点**：计算机求值时无需考虑优先级和括号，从右向左扫描即可。
    *   **缺点**：不符合人类阅读习惯。

*   **后缀表达式 (Postfix Expression)**，也叫**逆波兰表达式 (Reverse Polish Notation, RPN)**
    *   **特点**：运算符在两个操作数**后面**。
    *   **例子**：`a b +`, `a b - c *`, `2 3 5 * +` (对应 `2 + 3 * 5`)
    *   **优点**：
        1.  **无需括号**：所有运算顺序由操作符的位置确定。
        2.  **无需优先级**：所有运算符都被“推迟”到其操作数之后，计算机只需从左到右扫描，遇到运算符就对栈顶的两个操作数进行运算。
        3.  **易于计算机求值**：是计算机内部处理表达式的理想形式。
    *   **缺点**：不符合人类阅读习惯。

**举例对比：**
中缀: `(A + B) * C`
前缀: `* + A B C`
后缀: `A B + C *`

**思考：为什么后缀表达式易于计算机求值？**
想象一个计算器，它有一个“待处理”的区域（我们称之为**栈**）。
当它读到 `2 3 5 * +` 时：
1.  读到 `2`，压入栈：[2]
2.  读到 `3`，压入栈：[2, 3]
3.  读到 `5`，压入栈：[2, 3, 5]
4.  读到 `*`，这是一个运算符！它知道要取出栈顶的两个数（`5` 和 `3`），进行乘法运算 `3 * 5 = 15`。然后把结果 `15` 压回栈：[2, 15]
5.  读到 `+`，又是一个运算符！取出栈顶的两个数（`15` 和 `2`），进行加法运算 `2 + 15 = 17`。把结果 `17` 压回栈：[17]
表达式结束，栈中剩下的就是最终结果 `17`。
整个过程是不是非常清晰，没有任何“犹豫”和“回头”？这就是后缀表达式的魅力！

### 2. 中缀表达式转后缀表达式的算法核心思想

将中缀表达式转换为后缀表达式，最核心的工具就是**栈 (Stack)**。栈是一种“后进先出” (Last In, First Out, LIFO) 的数据结构，非常适合用来“暂时存放”那些需要等待的运算符。

我们可以把栈想象成一个“运算符等待区”。运算符们排队进入这个等待区，但它们不是随便排的，要讲究“资历”和“辈分”：

*   **资历高（优先级高）**的运算符，可以“插队”进入等待区。
*   **资历低（优先级低）**的运算符，如果等待区里有比它“资历高”或“资历一样高”的，就得等它们先出去。
*   **括号**就像“VIP通道”或“独立包厢”，左括号进去后，会阻止外面的运算符进来，直到遇到右括号，这个“包厢”里的运算符才能出来。

**算法步骤（基于栈的转换）：**

1.  **初始化**：
    *   创建一个空的**运算符栈**（我们用数组来模拟）。
    *   创建一个空字符串，用于存放转换后的**后缀表达式**。

2.  **从左到右扫描中缀表达式的每个字符** `c`：

    *   **如果 `c` 是操作数**（数字或变量，例如 `a, b, c, 1, 2, 3`）：
        *   直接将 `c` 追加到后缀表达式字符串中。
        *   为了区分操作数和运算符，通常会在每个操作数或运算符后添加一个空格。

    *   **如果 `c` 是左括号 `(`**：
        *   将其压入运算符栈。

    *   **如果 `c` 是右括号 `)`**：
        *   不断地从栈顶弹出运算符并追加到后缀表达式字符串中，直到遇到栈顶是左括号 `(`。
        *   将这个左括号 `(` 也从栈中弹出，但**不追加**到后缀表达式中（括号只用于控制顺序，不参与运算）。

    *   **如果 `c` 是运算符**（`+, -, *, /`）：
        *   **比较优先级**：
            *   **情况1**：如果栈为空，或者栈顶是左括号 `(`，或者当前运算符 `c` 的优先级**高于**栈顶运算符的优先级，则将 `c` 压入栈。
            *   **情况2**：否则（即栈不为空，栈顶不是左括号，且当前运算符 `c` 的优先级**小于等于**栈顶运算符的优先级），则不断地从栈顶弹出运算符并追加到后缀表达式字符串中，直到满足情况1的条件。
            *   **然后**，将当前运算符 `c` 压入栈。

3.  **扫描结束后**：
    *   如果运算符栈中还有剩余的运算符，将它们依次弹出并追加到后缀表达式字符串中。

**运算符优先级定义：**
我们通常定义 `*` 和 `/` 的优先级高于 `+` 和 `-`。
`(` 在栈中时，其优先级最低（或特殊处理，因为它不参与比较，只是一个屏障）。

| 运算符 | 优先级 |
| :----: | :----: |
| `+`, `-` | 1 |
| `*`, `/` | 2 |
| `(` | 0 (在栈内，作为屏障) |

### 3. 如何用数组模拟栈 (GESP 7级重点)

GESP 7级明确要求不能使用C++标准库中的 `std::stack` 等容器。这意味着我们需要自己动手，用基本的数组来实现栈的功能。

栈的本质就是一个可以“只从一端”存取数据的线性结构。我们用一个数组 `char s[MAXN]` 来存放栈的元素，再用一个整数变量 `int t`（通常表示 `top`）来指示栈顶元素的位置。

*   **栈顶指针 `t` 的含义**：
    *   如果 `t = -1`，表示栈为空。
    *   如果 `t = 0`，表示栈中有一个元素在 `s[0]`。
    *   以此类推，`s[t]` 就是栈顶元素。

*   **基本操作实现：**

    1.  **`push(char x)`：入栈**
        *   将 `t` 加1，然后将 `x` 存入 `s[t]`。
        *   `s[++t] = x;`

    2.  **`pop()`：出栈**
        *   先取出 `s[t]` 的值，然后将 `t` 减1。
        *   `char val = s[t]; t--; return val;`

    3.  **`top()`：查看栈顶元素**
        *   直接返回 `s[t]` 的值，`t` 不变。
        *   `return s[t];`

    4.  **`isEmpty()`：判断栈是否为空**
        *   检查 `t` 是否等于 `-1`。
        *   `return t == -1;`

**示例：**
假设 `MAXN = 10`
初始状态：`t = -1` (栈空)
`push('A')` -> `t = 0`, `s = ['A', ?, ?, ..., ?]`
`push('B')` -> `t = 1`, `s = ['A', 'B', ?, ..., ?]`
`top()` -> `'B'`
`pop()` -> `'B'`, `t = 0`, `s = ['A', ?, ?, ..., ?]`
`isEmpty()` -> `false`
`pop()` -> `'A'`, `t = -1`, `s = [?, ?, ?, ..., ?]`
`isEmpty()` -> `true`

**注意事项：**
*   **栈溢出**：当 `t` 达到 `MAXN - 1` 时，如果继续 `push` 就会发生栈溢出。在实际编程中需要检查。
*   **栈下溢**：当 `t` 为 `-1` 时，如果继续 `pop` 或 `top` 就会发生栈下溢。同样需要检查。
*   在我们的转换算法中，栈的深度不会超过中缀表达式的长度，所以 `MAXN` 足够大即可。

## 典型例题精讲

### 例题1：简单表达式转换 (无括号)

**中缀表达式：** `a + b * c`

**思路分析：**

1.  **初始化**：栈空，后缀表达式 `ans = ""`。
2.  **扫描 `a`**：是操作数，`ans += 'a '`。 `ans = "a "`。
3.  **扫描 `+`**：是运算符。栈空，将 `+` 压入栈。 栈：[`+`]
4.  **扫描 `b`**：是操作数，`ans += 'b '`。 `ans = "a b "`。
5.  **扫描 `*`**：是运算符。当前 `*` 优先级 (`2`) > 栈顶 `+` 优先级 (`1`)。将 `*` 压入栈。 栈：[`+`, `*`]
6.  **扫描 `c`**：是操作数，`ans += 'c '`。 `ans = "a b c "`。
7.  **扫描结束**：栈不为空。
    *   弹出栈顶 `*`，`ans += '* '`。 `ans = "a b c * "`。 栈：[`+`]
    *   弹出栈顶 `+`，`ans += '+ '`。 `ans = "a b c * + "`。 栈：[]
8.  **最终结果**：`a b c * +`

**C++ 代码实现：**
（结合代码模板，这里只展示主转换逻辑，栈操作函数已在模板中定义）

```cpp
#include <iostream>
#include <string>
#include <algorithm> // For std::reverse if needed, but not here.
#include <vector> // GESP 7 禁用此行，仅为演示。实际代码中不包含。
#include <bits/stdc++.h> // 允许使用这个万能头文件

// === 数组模拟栈的实现 START ===
const int MAXN = 1005; // 表达式最大长度
char s[MAXN]; // 模拟栈的数组
int t = -1; // 栈顶指针，-1表示空栈

void push(char c) {
    if (t >= MAXN - 1) {
        // std::cerr << "Error: Stack overflow!" << std::endl;
        return; // 实际竞赛中可能直接忽略或假设不会溢出
    }
    s[++t] = c;
}

char pop() {
    if (t == -1) {
        // std::cerr << "Error: Stack underflow!" << std::endl;
        return '\0'; // 返回空字符表示错误或空
    }
    return s[t--];
}

char top() {
    if (t == -1) {
        // std::cerr << "Error: Stack is empty!" << std::endl;
        return '\0';
    }
    return s[t];
}

bool isEmpty() {
    return t == -1;
}
// === 数组模拟栈的实现 END ===

// 获取运算符优先级
int getPriority(char op) {
    if (op == '+' || op == '-') return 1;
    if (op == '*' || op == '/') return 2;
    if (op == '(') return 0; // 左括号在栈内时优先级最低
    return -1; // 非法字符或未知运算符
}

// 判断是否是运算符
bool isOperator(char c) {
    return c == '+' || c == '-' || c == '*' || c == '/';
}

// 判断是否是操作数 (这里假设是单字母变量或单数字)
bool isOperand(char c) {
    return (c >= 'a' && c <= 'z') || (c >= 'A' && c <= 'Z') || (c >= '0' && c <= '9');
}

std::string infixToPostfix(std::string exp) {
    std::string ans = "";
    t = -1; // 每次转换前重置栈

    for (char c : exp) {
        if (c == ' ') continue; // 忽略空格

        if (isOperand(c)) {
            ans += c;
            ans += ' '; // 操作数后加空格
        } else if (c == '(') {
            push(c);
        } else if (c == ')') {
            while (!isEmpty() && top() != '(') {
                ans += pop();
                ans += ' '; // 运算符后加空格
            }
            if (!isEmpty() && top() == '(') { // 弹出左括号
                pop();
            }
        } else if (isOperator(c)) {
            while (!isEmpty() && top() != '(' && getPriority(c) <= getPriority(top())) {
                ans += pop();
                ans += ' '; // 运算符后加空格
            }
            push(c);
        }
    }

    // 处理栈中剩余的运算符
    while (!isEmpty()) {
        ans += pop();
        ans += ' '; // 运算符后加空格
    }

    // 移除末尾多余的空格
    if (!ans.empty() && ans.back() == ' ') {
        ans.pop_back();
    }
    return ans;
}

int main() {
    std::string infix_exp = "a+b*c";
    std::cout << "中缀表达式: " << infix_exp << std::endl;
    std::string postfix_exp = infixToPostfix(infix_exp);
    std::cout << "后缀表达式: " << postfix_exp << std::endl; // 预期输出: a b c * +

    // 再次调用，确保栈被正确重置
    infix_exp = "2+3*5";
    std::cout << "中缀表达式: " << infix_exp << std::endl;
    postfix_exp = infixToPostfix(infix_exp);
    std::cout << "后缀表达式: " << postfix_exp << std::endl; // 预期输出: 2 3 5 * +

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度**：O(N)，其中 N 是中缀表达式的长度。每个字符只会被扫描一次，并且每个运算符最多被压入栈一次，弹出栈一次。栈操作 (push, pop, top, isEmpty) 都是 O(1) 操作。
*   **空间复杂度**：O(N)，在最坏情况下（例如 `((((a))))`），栈的深度可能与表达式长度成正比。

### 例题2：带括号的表达式转换

**中缀表达式：** `(a + b) * c`

**思路分析：**

1.  **初始化**：栈空，后缀表达式 `ans = ""`。
2.  **扫描 `(`**：左括号，压入栈。 栈：[`(`]
3.  **扫描 `a`**：操作数，`ans += 'a '`。 `ans = "a "`。
4.  **扫描 `+`**：运算符。栈顶是 `(`，优先级最低，将 `+` 压入栈。 栈：[`(`, `+`]
5.  **扫描 `b`**：操作数，`ans += 'b '`。 `ans = "a b "`。
6.  **扫描 `)`**：右括号。
    *   弹出 `+`，`ans += '+ '`。 `ans = "a b + "`。 栈：[`(`]
    *   栈顶是 `(`，弹出 `(`（不追加）。 栈：[]
7.  **扫描 `*`**：运算符。栈空，将 `*` 压入栈。 栈：[`*`]
8.  **扫描 `c`**：操作数，`ans += 'c '`。 `ans = "a b + c "`。
9.  **扫描结束**：栈不为空。
    *   弹出 `*`，`ans += '* '`。 `ans = "a b + c * "`。 栈：[]
10. **最终结果**：`a b + c *`

### 例题3：复杂表达式转换 (优先级和括号混合)

**中缀表达式：** `a + (b - c) * d / e`

**思路分析：**

1.  **初始化**：栈空，`ans = ""`。
2.  `a`：操作数，`ans = "a "`。
3.  `+`：运算符，栈空，`push('+')`。 栈：[`+`]
4.  `(`：左括号，`push('(')`。 栈：[`+`, `(`]
5.  `b`：操作数，`ans = "a b "`。
6.  `-`：运算符。栈顶是 `(`，`push('-')`。 栈：[`+`, `(`, `-`]
7.  `c`：操作数，`ans = "a b c "`。
8.  `)`：右括号。
    *   `pop()` 栈顶 `-`，`ans += '- '`。 `ans = "a b c - "`。 栈：[`+`, `(`]
    *   `pop()` 栈顶 `(`。 栈：[`+`]
9.  `*`：运算符。当前 `*` 优先级 (`2`) > 栈顶 `+` 优先级 (`1`)。`push('*')`。 栈：[`+`, `*`]
10. `d`：操作数，`ans = "a b c - d "`。
11. `/`：运算符。当前 `/` 优先级 (`2`) <= 栈顶 `*` 优先级 (`2`)。
    *   `pop()` 栈顶 `*`，`ans += '* '`。 `ans = "a b c - d * "`。 栈：[`+`]
    *   `push('/')`。 栈：[`+`, `/`]
12. `e`：操作数，`ans = "a b c - d * e "`。
13. **扫描结束**：栈不为空。
    *   `pop()` 栈顶 `/`，`ans += '/ '`。 `ans = "a b c - d * e / "`。 栈：[`+`]
    *   `pop()` 栈顶 `+`，`ans += '+ '`。 `ans = "a b c - d * e / + "`。 栈：[]
14. **最终结果**：`a b c - d * e / +`

## 代码实现模板

```cpp
#include <bits/stdc++.h> // GESP 竞赛允许使用，包含大部分常用头文件

// 定义常量，例如表达式的最大长度
const int MAXN = 1005; 

// ====== 数组模拟栈的实现 ======
char s[MAXN]; // 用于存储栈元素的数组
int t = -1;   // 栈顶指针，初始化为 -1 表示栈为空

// 入栈操作
void push(char c) {
    if (t >= MAXN - 1) { // 检查栈是否已满，防止溢出
        // std::cerr << "Error: Stack overflow!" << std::endl;
        return; 
    }
    s[++t] = c; // 先将栈顶指针加1，再存入元素
}

// 出栈操作
char pop() {
    if (t == -1) { // 检查栈是否为空，防止下溢
        // std::cerr << "Error: Stack underflow!" << std::endl;
        return '\0'; // 返回空字符或其他标识符表示错误
    }
    return s[t--]; // 先返回栈顶元素，再将栈顶指针减1
}

// 查看栈顶元素（不弹出）
char top() {
    if (t == -1) { // 检查栈是否为空
        // std::cerr << "Error: Stack is empty!" << std::endl;
        return '\0';
    }
    return s[t]; // 返回栈顶元素
}

// 判断栈是否为空
bool isEmpty() {
    return t == -1;
}

// ====== 辅助函数 ======

// 获取运算符的优先级
// 返回值越大，优先级越高
// 左括号在栈内时优先级最低，用于阻挡其他运算符
int getPriority(char op) {
    if (op == '+' || op == '-') return 1; // 加减优先级最低
    if (op == '*' || op == '/') return 2; // 乘除优先级较高
    if (op == '(') return 0;             // 左括号在栈中时优先级最低，作为屏障
    return -1; // 其他字符（例如空格、非法字符）
}

// 判断字符是否是运算符
bool isOperator(char c) {
    return c == '+' || c == '-' || c == '*' || c == '/';
}

// 判断字符是否是操作数（这里简化为单字母或单数字）
// 实际应用中可能需要处理多位数字或更复杂的变量名
bool isOperand(char c) {
    return (c >= 'a' && c <= 'z') || (c >= 'A' && c <= 'Z') || (c >= '0' && c <= '9');
}

// ====== 中缀表达式转后缀表达式的核心函数 ======
std::string infixToPostfix(std::string exp) {
    std::string ans = ""; // 存储转换后的后缀表达式
    t = -1;               // 每次转换前，重置栈，确保它是空的

    // 遍历中缀表达式的每一个字符
    for (char c : exp) {
        if (c == ' ') continue; // 忽略输入中的空格

        if (isOperand(c)) {
            // 如果是操作数，直接追加到结果字符串
            ans += c;
            ans += ' '; // 操作数后添加空格，方便区分
        } else if (c == '(') {
            // 如果是左括号，直接压入栈
            push(c);
        } else if (c == ')') {
            // 如果是右括号，不断弹出栈顶运算符直到遇到左括号
            while (!isEmpty() && top() != '(') {
                ans += pop();
                ans += ' '; // 运算符后添加空格
            }
            if (!isEmpty() && top() == '(') { // 弹出匹配的左括号，但不追加到结果
                pop();
            }
            // 理论上，如果栈空了还没找到'('，说明括号不匹配，此处可加入错误处理
        } else if (isOperator(c)) {
            // 如果是运算符
            // 比较当前运算符与栈顶运算符的优先级
            // 如果栈不为空，栈顶不是左括号，且当前运算符优先级小于等于栈顶运算符
            while (!isEmpty() && top() != '(' && getPriority(c) <= getPriority(top())) {
                ans += pop(); // 弹出栈顶运算符并追加到结果
                ans += ' ';   // 运算符后添加空格
            }
            push(c); // 将当前运算符压入栈
        }
    }

    // 表达式扫描结束后，将栈中所有剩余的运算符依次弹出并追加到结果
    while (!isEmpty()) {
        ans += pop();
        ans += ' '; // 运算符后添加空格
    }

    // 移除结果字符串末尾可能多余的空格
    if (!ans.empty() && ans.back() == ' ') {
        ans.pop_back();
    }

    return ans;
}

// ====== main 函数用于测试 ======
int main() {
    std::string infix_exp1 = "a+b*c";
    std::cout << "中缀表达式: " << infix_exp1 << std::endl;
    std::cout << "后缀表达式: " << infixToPostfix(infix_exp1) << std::endl << std::endl;

    std::string infix_exp2 = "(a+b)*c";
    std::cout << "中缀表达式: " << infix_exp2 << std::endl;
    std::cout << "后缀表达式: " << infixToPostfix(infix_exp2) << std::endl << std::endl;

    std::string infix_exp3 = "a+(b-c)*d/e";
    std::cout << "中缀表达式: " << infix_exp3 << std::endl;
    std::cout << "后缀表达式: " << infixToPostfix(infix_exp3) << std::endl << std::endl;

    std::string infix_exp4 = "1+2*3-4/5";
    std::cout << "中缀表达式: " << infix_exp4 << std::endl;
    std::cout << "后缀表达式: " << infixToPostfix(infix_exp4) << std::endl << std::endl;

    return 0;
}
```

1.  **提问**：为什么在将中缀表达式转换为后缀表达式时，栈顶遇到左括号 `(` 时，即使当前运算符优先级低于栈顶的 `(`，我们也要直接将当前运算符压入栈，而不是弹出 `(`？
    *   **引导**：回忆括号的作用，它像一个“屏障”，保护内部的运算优先级。
    *   **参考答案**：左括号 `(` 在栈中起着特殊的“优先级最低”的屏障作用。它保证了括号内的运算会先进行。当遇到左括号时，它会阻止任何外部运算符弹出，直到遇到其匹配的右括号。所以，当栈顶是左括号时，任何运算符（包括低优先级的）都可以直接压入栈，因为它们属于括号内的运算。

2.  **提问**：如果中缀表达式中出现了像 `a * (b + c)` 这样的情况，当处理到 `*` 运算符时，栈中只有 `(`，此时是直接压入 `*` 还是有其他操作？
    *   **引导**：结合优先级规则和左括号的特殊性。
    *   **参考答案**：当处理到 `*` 运算符时，栈顶是 `(`。根据规则，如果栈顶是左括号，当前运算符可以直接压入栈。所以 `*` 会被直接压入栈。栈变为 `( *`。

3.  **小组任务**：设计一个 `getPriority` 函数，除了 `+ - * /` 之外，再加入一个幂运算 `^`（优先级最高，例如 `a^b`）。讨论它的优先级应该设置成多少？
    *   **引导**：幂运算通常比乘除优先级还高。
    *   **参考答案**：可以设置 `getPriority('^')` 返回 `3`，比乘除的 `2` 更高。
