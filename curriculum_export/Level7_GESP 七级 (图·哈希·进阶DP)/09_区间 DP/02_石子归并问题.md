## 趣味引入

同学们，大家好！今天我们要玩一个特别的“合并游戏”！

想象一下，你面前有一排五颜六色的魔法石子，它们排成一队：
`[5] [3] [8] [2] [7]`

你的任务是把这些石子全部合并成一堆。但是，每次你只能选择**相邻**的两堆石子进行合并。
比如，你可以选择 `[5]` 和 `[3]` 合并，它们就变成了一堆 `[8]`。合并的代价就是这两堆石子的数量之和（5 + 3 = 8）。
现在石子变成了：`[8] [8] [2] [7]`

你又可以选 `[8]` 和 `[8]` 合并，代价是 8 + 8 = 16。
石子变成：`[16] [2] [7]`

然后 `[16]` 和 `[2]` 合并，代价 16 + 2 = 18。
石子变成：`[18] [7]`

最后 `[18]` 和 `[7]` 合并，代价 18 + 7 = 25。
石子变成：`[25]`

所有石子都合为一堆了！这次合并的总代价是：8 + 16 + 18 + 25 = 67。

但是，等等！如果我们换一种合并顺序呢？
`[5] [3] [8] [2] [7]`
先合并 `[2]` 和 `[7]`，代价 2 + 7 = 9。
`[5] [3] [8] [9]`
再合并 `[3]` 和 `[8]`，代价 3 + 8 = 11。
`[5] [11] [9]`
再合并 `[5]` 和 `[11]`，代价 5 + 11 = 16。
`[16] [9]`
最后合并 `[16]` 和 `[9]`，代价 16 + 9 = 25。
总代价是：9 + 11 + 16 + 25 = 61。

哇！换一种顺序，总代价竟然变小了！
现在问题来了：对于任意一排石子，我们怎么才能找到一种合并顺序，使得总代价最小（或者最大）呢？
这就是我们今天要学习的经典问题——**石子归并问题**，它将带我们走进一个强大的算法思想：**区间动态规划 (Interval DP)**！

## 深度知识点讲解

### 1. 什么是区间动态规划 (Interval DP)？

在之前的动态规划学习中，我们可能遇到过 `dp[i]` 表示前 `i` 个元素的某种最优解，或者 `dp[i][j]` 表示从 `i` 到 `j` 两个维度上的最优解。
**区间动态规划** 是一种特殊的动态规划，它的状态通常定义为 `dp[i][j]`，表示解决从第 `i` 个元素到第 `j` 个元素（也就是一个“区间”或“子串”）的问题所得到的某种最优解（比如最小代价、最大价值等）。

**核心思想：**
区间 DP 的问题通常具有这样的特点：一个大区间的解，可以由它内部的**两个**更小区间的解**合并**而来。就像拼图一样，一块大拼图可以由两块小拼图拼成。
为了找到大区间的最优解，我们需要尝试所有可能的“分界点”`k`，把大区间 `[i, j]` 分成两个小区 `[i, k]` 和 `[k+1, j]`，然后结合这两个小区间的最优解，再加上合并这两个小区间所需的代价，来更新大区间的解。

### 2. 石子归并问题：深入剖析

让我们以最小化合并代价的石子归并问题为例，来理解 Interval DP 的具体实现。

**问题描述：**
给定 `N` 堆石子，排成一行，每堆石子的数量已知。每次只能合并**相邻**的两堆石子，合并的代价是这两堆石子的数量之和。求将所有石子合并成一堆的最小总代价。

**2.1 状态定义**
这是 DP 问题最关键的一步！
我们定义 `dp[i][j]` 表示：将从第 `i` 堆石子到第 `j` 堆石子（闭区间 `[i, j]`）合并成一堆所需的**最小**总代价。
*   `i` 和 `j` 是石子堆的编号，通常从 1 开始。

**2.2 思考“为什么”和“怎么想到的”**

*   **为什么是 `dp[i][j]`？**
    因为合并操作是“相邻”的，最终形成一堆石子，这个“最终一堆”就是某个连续的区间 `[i, j]`。如果我们能计算出所有小区间 `[i, j]` 的最小合并代价，那么最终 `dp[1][N]` 就是我们想要的答案。
*   **怎么才能从 `dp[i][j]` 得到 `dp[i][j+1]` 呢？**
    这不像普通的线性 DP 那样，从 `dp[i-1]` 推出 `dp[i]` 那么直接。
    对于区间 `[i, j]`，它的最终合并操作一定是将某个 `[i, k]` 区间合并后的结果，与 `[k+1, j]` 区间合并后的结果，再进行一次合并。
    想象一下，你已经把 `[i, k]` 合并成了一大堆石子，代价是 `dp[i][k]`。
    你也已经把 `[k+1, j]` 合并成了一大堆石子，代价是 `dp[k+1][j]`。
    现在，这两大堆石子是相邻的，你需要把它们合并起来。这次合并的代价是多少呢？
    **关键点：** 这次合并的代价是 `[i, k]` 这堆石子的总数量 加上 `[k+1, j]` 这堆石子的总数量。而 `[i, k]` 这堆石子的总数量，就是原始的第 `i` 堆到第 `k` 堆石子的数量之和！同理，`[k+1, j]` 的总数量是原始的第 `k+1` 堆到第 `j` 堆石子的数量之和。所以，**最后一次合并的代价就是整个大区间 `[i, j]` 中所有石子的数量之和！**

**2.3 状态转移方程**

为了求 `dp[i][j]`，我们需要考虑所有可能的最后一次合并的分界点 `k`。
`k` 可以从 `i` 取到 `j-1`。
对于每一个 `k`，我们将区间 `[i, j]` 分为 `[i, k]` 和 `[k+1, j]`。
那么，将 `[i, j]` 合并的最小总代价就是：
`dp[i][j] = min { dp[i][k] + dp[k+1][j] + sum(i, j) }`
其中 `k` 的取值范围是 `i <= k < j`。
`sum(i, j)` 表示从第 `i` 堆石子到第 `j` 堆石子的数量总和。

**2.4 基础情况 (Base Case)**

*   当 `i == j` 时，表示只有一个石子堆。把它合并成一堆的代价是 0。
    所以，`dp[i][i] = 0`。

**2.5 计算区间和的优化：前缀和**

每次计算 `sum(i, j)` 都遍历一遍 `a[i]` 到 `a[j]` 会非常慢 (`O(N)` 时间)。
为了快速求出任意区间的和，我们可以使用**前缀和**。
定义 `s[x]` 为前 `x` 堆石子的总数量，即 `s[x] = a[1] + a[2] + ... + a[x]`。
那么，`sum(i, j) = a[i] + ... + a[j] = s[j] - s[i-1]`。
（注意：如果石子堆从 0 开始编号，那么 `s[x] = a[0] + ... + a[x-1]`，`sum(i, j) = s[j+1] - s[i]`）
这样，计算 `sum(i, j)` 只需要 `O(1)` 时间。

**2.6 循环顺序**

区间 DP 的一个重要特点是，计算大区间的 `dp` 值，需要依赖小区间的 `dp` 值。
所以，我们不能像常规 DP 那样简单地从 `i` 循环到 `N`。
我们需要先计算长度为 2 的区间，再计算长度为 3 的区间，以此类推，直到计算长度为 `N` 的区间（即 `[1, N]`）。

1.  **外层循环：区间长度 `len`**
    `len` 从 2 遍历到 `N` (因为 `len=1` 已经是 base case)。
2.  **中层循环：区间起始点 `i`**
    `i` 从 1 遍历到 `N - len + 1`。
3.  **确定区间结束点 `j`**
    `j = i + len - 1`。
4.  **内层循环：分界点 `k`**
    `k` 从 `i` 遍历到 `j - 1`。

```cpp
for (int len = 2; len <= n; ++len) { // 区间长度
    for (int i = 1; i <= n - len + 1; ++i) { // 区间起点
        int j = i + len - 1; // 区间终点
        // 初始化 dp[i][j] 为一个非常大的值，因为我们要找最小值
        dp[i][j] = 0x3f3f3f3f; // 这是一个常用的表示“无穷大”的整数值
        for (int k = i; k < j; ++k) { // 分界点
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + (s[j] - s[i-1]));
        }
    }
}
```

**2.7 常见误区**

1.  **忘记加 `sum(i, j)`：** 很多同学在写转移方程时，只写 `dp[i][k] + dp[k+1][j]`，而忘记了最后一次合并 `[i, k]` 和 `[k+1, j]` 所需的代价 `sum(i, j)`。这是最常见的错误！
2.  **`dp` 数组初始化不当：** 求解最小值时，`dp` 数组除了 `dp[i][i]` 之外，其他值需要初始化为足够大的数（例如 `0x3f3f3f3f`）。求解最大值时，则需要初始化为足够小的数（例如 `0xc0c0c0c0`）。
3.  **循环边界错误：** `len`, `i`, `j`, `k` 的循环范围需要仔细推敲，确保不会越界，并且覆盖所有有效情况。特别是 `k` 的范围，是 `i` 到 `j-1`。

### 3. 时间与空间复杂度

*   **时间复杂度：**
    *   `len` 循环：`N` 次
    *   `i` 循环：`N` 次
    *   `k` 循环：`N` 次
    *   每次计算 `sum(i, j)` 是 `O(1)`
    *   总时间复杂度：`O(N * N * N) = O(N^3)`
*   **空间复杂度：**
    *   `dp[N][N]` 数组：`O(N^2)`
    *   `s[N]` 前缀和数组：`O(N)`
    *   总空间复杂度：`O(N^2)`

对于 GESP 七级，`N` 的范围通常在 200-500 之间，`N^3` 的算法是可接受的 (`500^3 = 1.25 * 10^8`，勉强通过；`200^3 = 8 * 10^6`，轻松通过)。

## 典型例题精讲

**例题：石子归并 (最小代价)**

**题目描述：**
有 `N` 堆石子排成一行，第 `i` 堆石子有 `a[i]` 个。每次合并只能选择两堆相邻的石子，合并的代价是两堆石子的总数量。请你找出一种合并方案，使得将所有石子合并成一堆的总代价最小。

**输入格式：**
第一行一个整数 `N` (1 <= N <= 300)。
第二行 `N` 个整数，表示每堆石子的数量 `a[1], a[2], ..., a[N]`。

**输出格式：**
一个整数，表示最小的总代价。

**样例输入：**
```
4
10 20 30 40
```

**样例输出：**
```
210
```

**思路分析：**

1.  **问题识别：** 这是一个典型的石子归并问题，要求合并相邻元素，目标是最小化总代价。符合区间 DP 的特征。
2.  **状态定义：** `dp[i][j]` 表示将第 `i` 堆到第 `j` 堆石子合并成一堆的最小代价。
3.  **预处理：** 为了快速计算区间 `[i, j]` 的石子总和，我们需要计算前缀和数组 `s`。
    `s[x]` 表示前 `x` 堆石子的数量总和。
    `s[0] = 0`
    `s[x] = s[x-1] + a[x]` (对于 `x` 从 1 到 `N`)
    那么 `sum(i, j) = s[j] - s[i-1]`。
4.  **初始化：**
    *   `dp` 数组所有元素初始化为足够大的值，例如 `0x3f3f3f3f` (代表正无穷)。
    *   `dp[i][i] = 0` (合并一堆石子的代价为 0)。
5.  **循环顺序：**
    *   外层循环 `len` (区间长度) 从 2 到 `N`。
    *   中层循环 `i` (区间起点) 从 1 到 `N - len + 1`。
    *   计算 `j = i + len - 1` (区间终点)。
    *   内层循环 `k` (分界点) 从 `i` 到 `j - 1`。
6.  **状态转移：**
    `dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + (s[j] - s[i-1]))`
7.  **最终答案：** `dp[1][N]`。

**代码实现：**

```cpp
#include <iostream>
#include <algorithm> // 包含 min 函数

const int INF = 0x3f3f3f3f; // 定义一个足够大的整数，表示无穷大

int a[305];    // 存储每堆石子的数量
int s[305];    // 存储前缀和
int dp[305][305]; // dp[i][j] 表示合并第i堆到第j堆石子的最小代价

int main()
{
    // 禁用同步，加速I/O
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    // 读取石子数量并计算前缀和
    s[0] = 0; // s[0] 为0，方便计算s[j] - s[i-1]
    for (int i = 1; i <= n; ++i)
    {
        std::cin >> a[i];
        s[i] = s[i - 1] + a[i];
    }

    // 初始化dp数组
    // dp[i][j] 默认为无穷大，表示尚未计算或不可达
    // dp[i][i] 初始化为0，因为合并一堆石子不需要代价
    for (int i = 1; i <= n; ++i)
    {
        for (int j = 1; j <= n; ++j)
        {
            if (i == j)
            {
                dp[i][j] = 0;
            }
            else
            {
                dp[i][j] = INF;
            }
        }
    }

    // 区间DP的核心循环
    // len 表示当前合并的区间长度，从2开始（长度为1的区间已初始化）
    for (int len = 2; len <= n; ++len)
    {
        // i 表示区间的起始点
        for (int i = 1; i <= n - len + 1; ++i)
        {
            int j = i + len - 1; // j 表示区间的结束点

            // k 是分界点，将区间 [i, j] 分为 [i, k] 和 [k+1, j]
            for (int k = i; k < j; ++k)
            {
                // 状态转移方程：
                // dp[i][j] = min(dp[i][k] + dp[k+1][j] + sum_of_stones_in_interval_ij)
                // sum_of_stones_in_interval_ij = s[j] - s[i-1]
                dp[i][j] = std::min(dp[i][j], dp[i][k] + dp[k + 1][j] + (s[j] - s[i - 1]));
            }
        }
    }

    // 最终答案是合并所有石子 (从第1堆到第n堆) 的最小代价
    std::cout << dp[1][n] << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 三重循环，每层循环最大 `N` 次，所以是 `O(N^3)`。
    *   `N=300` 时，`300^3 = 2.7 * 10^7`，在可接受范围内。
*   **空间复杂度：** `dp[N][N]` 数组和 `a[N]`、`s[N]` 数组，所以是 `O(N^2)`。
    *   `N=300` 时，`300*300 = 9 * 10^4`，大约 `90000 * 4` 字节 (`int`)，约 `360KB`，在可接受范围内。

## 代码实现模板

这里提供一个通用的区间DP模板，帮助同学们快速构建解决方案。

```cpp
#include <iostream>
#include <algorithm> // 用于 std::min 或 std::max
// #include <vector> // GESP 7级通常禁用STL容器，所以我们用原生数组

// 定义常量，方便修改和理解
const int MAXN = 305; // 根据题目 N 的最大值设定
const int INF_MIN = 0x3f3f3f3f; // 用于求最小值时，表示“无穷大”
const int INF_MAX = 0xc0c0c0c0; // 用于求最大值时，表示“无穷小” (负数)

int a[MAXN];    // 存储原始数据（例如石子数量）
int s[MAXN];    // 存储前缀和（如果需要快速计算区间和）
int dp[MAXN][MAXN]; // dp[i][j] 表示区间 [i, j] 的最优解

int main()
{
    // 优化I/O，GESP竞赛常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 数据规模
    std::cin >> n;

    // 1. 数据读取与预处理 (例如计算前缀和)
    s[0] = 0; // 前缀和s[0]通常设为0
    for (int i = 1; i <= n; ++i)
    {
        std::cin >> a[i];
        s[i] = s[i - 1] + a[i]; // 计算前缀和
    }

    // 2. 初始化dp数组
    // 对于求最小值的问题，dp[i][j] 初始化为 INF_MIN
    // 对于求最大值的问题，dp[i][j] 初始化为 INF_MAX
    // base case (通常是长度为1的区间) 需要特殊处理
    for (int i = 1; i <= n; ++i)
    {
        for (int j = 1; j <= n; ++j)
        {
            if (i == j)
            {
                dp[i][j] = 0; // 例如：合并单个石子堆的代价为0
            }
            else
            {
                dp[i][j] = INF_MIN; // 初始化为无穷大，准备求最小值
            }
        }
    }

    // 3. 区间DP的核心循环
    // len: 区间长度，从2开始，因为长度为1的区间已作为base case
    for (int len = 2; len <= n; ++len)
    {
        // i: 区间起点，从1开始
        // n - len + 1 是最后一个可能的起点，确保 j 不会越界 (j = i + len - 1)
        for (int i = 1; i <= n - len + 1; ++i)
        {
            int j = i + len - 1; // j: 区间终点

            // k: 分割点，将区间 [i, j] 分割成 [i, k] 和 [k+1, j]
            // k 从 i 遍历到 j-1，确保两个子区间都有至少一个元素
            for (int k = i; k < j; ++k)
            {
                // 状态转移方程：
                // 根据具体问题修改此处，通常是：
                // dp[i][j] = std::min(dp[i][j], dp[i][k] + dp[k+1][j] + cost_to_merge_subproblems);
                // cost_to_merge_subproblems 通常是整个区间 [i, j] 的总和
                
                // 示例（石子归并最小代价）：
                dp[i][j] = std::min(dp[i][j], dp[i][k] + dp[k + 1][j] + (s[j] - s[i - 1]));
            }
        }
    }

    // 4. 输出最终结果
    // 通常是 dp[1][n] (如果问题是合并整个序列)
    std::cout << dp[1][n] << std::endl;

    return 0;
}
```

**注释说明：**

*   `MAXN`: 数组最大尺寸，通常比题目给的 `N` 最大值稍大，防止越界。
*   `INF_MIN`/`INF_MAX`: 用于初始化 `dp` 数组的“无穷大”或“无穷小”值。`0x3f3f3f3f` 是一个常用的整数值，它足够大，两倍也不会溢出 `int`，且 `memset` 初始化后每个字节都是 `0x3f`。
*   `std::ios_base::sync_with_stdio(false); std::cin.tie(NULL);`: 这是 C++ 竞赛中常用的 IO 优化，可以显著加快输入输出速度。
*   `s[0] = 0`: 前缀和的常见设置，使得 `s[j] - s[i-1]` 在 `i=1` 时也能正确计算 (`s[j] - s[0]`)。
*   `len` 循环从 2 开始：因为 `len=1` 的情况（`dp[i][i]`）通常是 base case，直接初始化为 0。
*   `i` 循环到 `n - len + 1`: 确保 `j = i + len - 1` 不会超过 `n`。
*   `k` 循环到 `j - 1`: 确保 `[k+1, j]` 这个子区间至少有一个元素。

## 课堂互动

**思考题与讨论：**

1.  **Q1: 合并顺序的重要性**
    *   **老师：** 刚才我们通过例子看到，`[5] [3] [8] [2] [7]` 合并成一堆，不同的合并顺序会导致不同的总代价。你能再举一个简单的例子（比如三堆石子 `[1] [2] [3]`），手动计算所有可能的合并顺序，看看它们的总代价分别是多少吗？
    *   **学生讨论：** (引导学生画出合并树或列出步骤)
        *   (1,2) -> 3; (3,3) -> 6. Total = (1+2) + (3+3) = 3 + 6 = 9.
        *   (2,3) -> 5; (1,5) -> 6. Total = (2+3) + (1+5) = 5 + 6 = 11.
    *   **老师：** 看来合并顺序确实很重要，我们不能只凭直觉。

2.  **Q2: 为什么 `dp[i][i]` 是 0？**
    *   **老师：** 在我们的 `dp[i][j]` 定义中，`dp[i][i]` 表示什么？它的值为什么是 0 呢？
    *   **学生思考：** `dp[i][i]` 是把第 `i` 堆石子合并成一堆的最小代价。因为它本身就是一堆了，不需要再合并，所以代价是 0。
    *   **老师：** 非常正确！这就是我们 DP 的“最基础”的子问题。

3.  **Q3: 为什么需要前缀和？**
    *   **老师：** 我们在计算 `dp[i][j]` 时，会用到一个 `sum(i, j)`，也就是从 `i` 到 `j` 的石子总数量。如果我们不用前缀和，每次都去循环求和，会有什么问题？
    *   **学生思考：** 每次求和都要遍历 `len` 次，总共是 `N^3 * N` 变成 `N^4`，太慢了！
    *   **老师：** 没错！`N^4` 的算法通常在 `N` 超过 50 就会超时。前缀和把 `O(N)` 的求和操作优化成了 `O(1)`，从而让整个算法保持在 `O(N^3)`。这就是算法优化带来的巨大收益！

4.  **Q4: 状态转移方程中的“加号”**
    *   **老师：** 回忆一下状态转移方程 `dp[i][j] = min(dp[i][k] + dp[k+1][j] + (s[j] - s[i-1]))`。
    *   **老师：** `dp[i][k]` 和 `dp[k+1][j]` 分别代表了什么？ `(s[j] - s[i-1])` 又代表了什么？为什么它们之间是加号？
    *   **学生思考：** `dp[i][k]` 是合并左边小区的最小代价，`dp[k+1][j]` 是合并右边小区的最小代价。`s[j] - s[i-1]` 是最后一次合并这两个小区（即 `[i, k]` 合并后的大堆和 `[k+1, j]` 合并后的大堆）的代价。总代价就是把这些合并代价加起来。
    *   **老师：** 非常棒！这个 `sum(i, j)` 是最容易被遗漏的部分，大家一定要记住！

**小组任务：**

*   **任务：** 假设有 4 堆石子，数量分别为 `[10, 20, 30, 40]`。请大家以小组为单位，用我们今天学到的区间 DP 的思路，手动模拟计算 `dp[1][4]` 的最小代价。
    *   提示：从 `len=2` 开始，逐步填写 `dp` 表格。
    *   `dp[1][1]=0, dp[2][2]=0, dp[3][3]=0, dp[4][4]=0`
    *   计算 `len=2` 的：`dp[1][2], dp[2][3], dp[3][4]`
    *   计算 `len=3` 的：`dp[1][3], dp[2][4]`
    *   计算 `len=4` 的：`dp[1][4]`
*   **讨论重点：** 在计算 `dp[i][j]` 时，`k` 的取值以及对应的 `dp[i][k] + dp[k+1][j] + sum` 的值。

**即时练习：**

*   **题目：** 有 3 堆石子，数量分别为 `[4, 5, 6]`。请你计算将它们合并成一堆的最小代价。
*   **学生动手：**
    *   `a = [4, 5, 6]`
    *   `s = [0, 4, 9, 15]`
    *   `dp[1][1]=0, dp[2][2]=0, dp[3][3]=0`
    *   `len=2`:
        *   `dp[1][2]` (i=1, j=2): k=1. `dp[1][1]+dp[2][2]+sum(1,2) = 0+0+(s[2]-s[0]) = 4+5 = 9`. So `dp[1][2]=9`.
        *   `dp[2][3]` (i=2, j=3): k=2. `dp[2][2]+dp[3][3]+sum(2,3) = 0+0+(s[3]-s[1]) = 5+6 = 11`. So `dp[2][3]=11`.
    *   `len=3`:
        *   `dp[1][3]` (i=1, j=3): `sum(1,3) = s[3]-s[0] = 15`.
            *   k=1: `dp[1][1]+dp[2][3]+15 = 0+11+15 = 26`.
            *   k=2: `dp[1][2]+dp[3][3]+15 = 9+0+15 = 24`.
            *   取最小值，`dp[1][3]=24`.
*   **答案：** 24

## 分层练习题目

### 基础巩固 (Level 1)

1.  **题目：** 有 5 堆石子，数量分别为 `[1, 2, 3, 4, 5]`。请计算将它们合并成一堆的最小代价。
    *   **提示：** 严格按照模板和循环顺序，逐步填写 `dp` 表格。

2.  **题目：** 假设石子归并的规则是每次合并两堆相邻石子，代价是两堆石子数量之和。现在要求将所有石子合并成一堆的**最大**总代价。
    *   输入：`N` 堆石子数量 `a[1...N]`。
    *   输出：最大总代价。
    *   **提示：** 只需要修改 DP 数组的初始化值（无穷小）和 `min` 运算符为 `max` 运算符。

### 能力提升 (Level 2)

1.  **题目：** **石子归并 II (同时求最小和最大代价)**
    *   输入：`N` 堆石子数量 `a[1...N]`。
    *   输出：最小总代价 和 最大总代价。
    *   **提示：** 你需要维护两个 `dp` 数组，一个 `min_dp[i][j]`，一个 `max_dp[i][j]`。它们的转移方程几乎一样，只是取 `min` 或 `max`，以及初始化值不同。

2.  **题目：** **环形石子归并**
    *   **描述：** `N` 堆石子围成一个圈。每次合并两堆相邻的石子，代价是两堆石子数量之和。求将所有石子合并成一堆的最小总代价。
    *   **输入：** `N` 堆石子数量 `a[1...N]`。
    *   **输出：** 最小总代价。
    *   **提示：**
        *   环形问题通常的处理方式是将链式数组复制一份，变成 `a[1], ..., a[N], a[1], ..., a[N]`，这样就得到了一个长度为 `2N` 的链式数组。
        *   然后在这个 `2N` 长度的数组上跑标准的区间 DP。
        *   最终的答案是所有长度为 `N` 的区间的最小值。例如，`min(dp[1][N], dp[2][N+1], ..., dp[N][2N-1])`。
        *   `dp` 数组需要开到 `2N * 2N`。

### 拓展挑战 (Level 3)

1.  **题目：** **带权石子归并 (变种)**
    *   **描述：** 仍然是合并相邻两堆石子，但合并代价不再是简单的数量之和，而是 `(a + b) * C`，其中 `C` 是一个额外的常数权重，或者 `a * b` 等更复杂的计算方式。
    *   **提示：** 只需要修改状态转移方程中 `cost_of_merge` 的计算方式即可。
    *   例如，如果代价是 `(a + b) * C`，那么 `(s[j] - s[i-1]) * C`。

**基础巩固**

1.  `[1, 2, 3, 4, 5]`
    *   `s = [0, 1, 3, 6, 10, 15]`
    *   `len=2`: `dp[1][2]=3`, `dp[2][3]=5`, `dp[3][4]=7`, `dp[4][5]=9`
    *   `len=3`:
        *   `dp[1][3] = min(dp[1][1]+dp[2][3]+6, dp[1][2]+dp[3][3]+6) = min(0+5+6, 3+0+6) = 9`
        *   `dp[2][4] = min(dp[2][2]+dp[3][4]+9, dp[2][3]+dp[4][4]+9) = min(0+7+9, 5+0+9) = 14`
        *   `dp[3][5] = min(dp[3][3]+dp[4][5]+12, dp[3][4]+dp[5][5]+12) = min(0+9+12, 7+0+12) = 19`
    *   `len=4`:
        *   `dp[1][4] = min(dp[1][1]+dp[2][4]+10, dp[1][2]+dp[3][4]+10, dp[1][3]+dp[4][4]+10)`
            `= min(0+14+10, 3+7+10, 9+0+10) = min(24, 20, 19) = 19`
        *   `dp[2][5] = min(dp[2][2]+dp[3][5]+14, dp[2][3]+dp[4][5]+14, dp[2][4]+dp[5][5]+14)`
            `= min(0+19+14, 5+9+14, 14+0+14) = min(33, 28, 28) = 28`
    *   `len=5`:
        *   `dp[1][5] = min(dp[1][1]+dp[2][5]+15, dp[1][2]+dp[3][5]+15, dp[1][3]+dp[4][5]+15, dp[1][4]+dp[5][5]+15)`
            `= min(0+28+15, 3+19+15, 9+9+15, 19+0+15)`
            `= min(43, 37, 33, 34) = 33`
    *   **答案：33**

2.  **最大代价：**
    *   `dp` 数组初始化为 `INF_MAX` (例如 `0xc0c0c0c0`)。
    *   `dp[i][i] = 0` (不变)。
    *   状态转移方程：`dp[i][j] = std::max(dp[i][j], dp[i][k] + dp[k+1][j] + (s[j] - s[i-1]))`

**能力提升**

1.  **石子归并 II**
    *   需要定义 `min_dp[MAXN][MAXN]` 和 `max_dp[MAXN][MAXN]` 两个数组。
    *   `min_dp` 初始化为 `INF_MIN`，`max_dp` 初始化为 `INF_MAX`。
    *   两个数组的 `dp[i][i]` 都设为 0。
    *   在相同的三重循环中，分别更新 `min_dp` 和 `max_dp`。
    *   `min_dp[i][j] = std::min(min_dp[i][j], min_dp[i][k] + min_dp[k+1][j] + (s[j] - s[i-1]));`
    *   `max_dp[i][j] = std::max(max_dp[i][j], max_dp[i][k] + max_dp[k+1][j] + (s[j] - s[i-1]));`
    *   最终输出 `min_dp[1][N]` 和 `max_dp[1][N]`。

2.  **环形石子归并**
    *   将 `a` 数组复制一份：`a[i+N] = a[i]` (对于 `i` 从 1 到 `N`)。
    *   此时实际石子堆数为 `2N-1` 堆 (因为 `a[N]` 和 `a[N+1]` 实际上是同一堆的两种表示，但为了处理方便，可以看作 `2N` 堆，然后取 `N` 长度的区间)。
    *   前缀和 `s` 数组也要计算到 `s[2N]`。
    *   `dp` 数组开到 `MAXN*2` 尺寸，例如 `dp[605][605]`。
    *   循环 `len` 从 2 到 `N`。
    *   循环 `i` 从 1 到 `2N - len + 1`。
    *   循环 `k` 从 `i` 到 `j-1`。
    *   最终答案是 `min(dp[i][i+N-1])`，其中 `i` 从 1 到 `N`。
