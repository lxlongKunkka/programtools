## 深度知识点讲解

### 1. 树上路径距离的定义

在树中，任意两个节点 `u` 和 `v` 之间都存在一条**唯一**的**简单路径**（不重复经过任何节点和边）。这条路径上所包含的边的数量，就是 `u` 和 `v` 之间的**树上路径距离**，我们通常记作 `dist(u, v)`。

比如，从节点1到节点5的距离是2（经过1->2->5），从节点3到节点6的距离是3（经过3->1->2->6）。

### 2. LCA与树上路径距离的奥秘：A + B - 2C 公式

好了，现在我们有了LCA这个强大的工具，它能帮我们找到两个节点的“最近公共祖先”。那么，它和距离有什么关系呢？

我们来揭示一个非常精妙的公式：
**`dist(u, v) = depth[u] + depth[v] - 2 * depth[LCA(u, v)]`**

其中：
*   `depth[x]` 表示节点 `x` 的深度，即从根节点到 `x` 的路径上的边数。我们通常约定根节点的深度为0。

**为什么这个公式成立呢？** 让我们用一个形象的比喻来理解它。

想象一下，树的根节点是“天花板”上的一个挂钩。每个节点都是一个“小灯泡”，每条边都是一根“电线”。从天花板（根节点）到每个小灯泡（节点）都有一条唯一的电线路径。`depth[x]` 就是从天花板到小灯泡 `x` 所需要的电线总长度。

现在，你手里拿着 `u` 和 `v` 两个小灯泡。你想知道把 `u` 和 `v` 连接起来需要多长的电线。

1.  **`depth[u]`：** 这是从天花板到 `u` 的电线长度。
2.  **`depth[v]`：** 这是从天花板到 `v` 的电线长度。

如果我们将这两段电线加起来 (`depth[u] + depth[v]`)，会发生什么？
你可能会发现，从天花板到 `LCA(u, v)` 的那段电线，被我们**重复计算了两次**！

*   路径1：天花板 -> ... -> `LCA(u, v)` -> ... -> `u`
*   路径2：天花板 -> ... -> `LCA(u, v)` -> ... -> `v`

看，从天花板到 `LCA(u, v)` 的这段路径，在 `depth[u]` 里算了一次，在 `depth[v]` 里又算了一次。
这段公共路径的长度正好是 `depth[LCA(u, v)]`。

所以，为了得到 `u` 到 `v` 的真实路径长度，我们需要把多算的那部分减掉。因为多算了两次，所以要减去 `2 * depth[LCA(u, v)]`。

这样，剩下的就是：
`(天花板 -> ... -> LCA(u, v))` + `(LCA(u, v) -> ... -> u)` +
`(天花板 -> ... -> LCA(u, v))` + `(LCA(u, v) -> ... -> v)`
- `2 * (天花板 -> ... -> LCA(u, v))`

结果就变成了：
`(LCA(u, v) -> ... -> u)` + `(LCA(u, v) -> ... -> v)`
这不就是 `u` 到 `v` 的路径吗？！完美！

**特例思考：**
如果 `u` 就是 `LCA(u, v)` 怎么办？也就是说 `u` 是 `v` 的祖先。
那么 `depth[LCA(u, v)]` 就是 `depth[u]`。
公式变成：`dist(u, v) = depth[u] + depth[v] - 2 * depth[u] = depth[v] - depth[u]`。
这正是 `u` 作为 `v` 祖先时，它们之间的距离！公式依然成立！太棒了！

### 3. 倍增法求LCA回顾 (预处理与查询)

由于LCA是计算距离的关键，我们快速回顾一下倍增法求LCA的步骤。

**核心思想：**
利用“二进制拆分”的思想，预先计算出每个节点向上跳 `2^k` 步能到达的祖先。这样在查询时，可以快速地让两个节点跳到同一深度，然后同步向上跳，直到它们是同一个节点的子节点（或就是同一个节点）。

**预处理阶段 (DFS + 动态规划)：**
1.  **DFS遍历树：**
    *   从根节点（通常是1号节点）开始DFS。
    *   在DFS过程中，记录每个节点的深度 `depth[u]`。
    *   记录每个节点的直接父节点 `p[u][0]`（即向上跳 `2^0 = 1` 步的祖先）。
    *   `depth[root] = 0`。
    *   `p[root][0] = root` (或者一个特殊值如0，表示没有父节点，具体看题目约定)。

2.  **动态规划计算 `p[u][k]`：**
    *   `p[u][k]` 表示节点 `u` 向上跳 `2^k` 步能到达的祖先。
    *   递推关系：`p[u][k] = p[ p[u][k-1] ][k-1]`。
        *   这表示从 `u` 向上跳 `2^k` 步，等价于先从 `u` 向上跳 `2^(k-1)` 步到达一个中间节点，再从这个中间节点向上跳 `2^(k-1)` 步。
    *   `k` 的范围：从 `1` 到 `LOGN`（`LOGN` 是 `log2(MAXN)` 的上界，通常取 `18` 或 `20`）。

**查询阶段 (`getLCA(u, v)` 函数)：**
1.  **调整深度：** 先将 `u` 和 `v` 中较深的那个节点向上跳，直到它们处于同一深度。
    *   如果 `depth[u] < depth[v]`，交换 `u` 和 `v`。
    *   让 `u` 向上跳 `depth[u] - depth[v]` 步。利用二进制拆分，从大到小遍历 `k`，如果 `u` 向上跳 `2^k` 步后还在 `v` 的深度之下，就跳。
2.  **如果 `u == v`，则 `u` (或 `v`) 就是LCA。**
3.  **同步向上跳：** 如果 `u != v`，说明它们还不是同一个节点。
    *   从 `LOGN-1` 到 `0` 遍历 `k`。
    *   如果 `p[u][k] != p[v][k]`，说明它们共同的LCA还在更上方，让 `u` 和 `v` 同时向上跳 `2^k` 步。
4.  **最终LCA：** 经过上述操作后，`u` 和 `v` 会变成LCA的直接子节点。此时，`p[u][0]` (或 `p[v][0]`) 就是它们的LCA。

**时间复杂度：**
*   预处理：`O(N log N)` (N个节点，每个节点计算LOGN个祖先)。
*   查询：`O(log N)` (每次查询向上跳LOGN次)。
*   总计：`O(N log N + Q log N)`，其中 `Q` 是查询次数。

**空间复杂度：**
*   `O(N log N)` (存储 `p[N][LOGN]` 数组)。

### 4. 总结：解决树上路径距离问题的步骤

1.  **读取输入：** 构建树的邻接表。
2.  **预处理DFS：**
    *   从根节点（通常为1）开始DFS。
    *   计算每个节点的 `depth`。
    *   记录每个节点的直接父节点 `p[u][0]`。
3.  **预处理倍增数组：**
    *   利用 `p[u][k] = p[ p[u][k-1] ][k-1]` 填充 `p` 数组。
4.  **实现 `getLCA(u, v)` 函数。**
5.  **实现 `getDistance(u, v)` 函数：**
    *   调用 `getLCA(u, v)` 得到 `lca_node`。
    *   返回 `depth[u] + depth[v] - 2 * depth[lca_node]`。
6.  **处理查询：** 对于每个查询 `(u, v)`，调用 `getDistance(u, v)` 并输出结果。

## 典型例题精讲

### 例题1：魔法森林距离查询

**题目描述：**
你在一片有 `N` 棵树的魔法森林中探险。这些树木之间有 `N-1` 条路连接，构成了一个标准的树形结构。现在，你需要回答 `Q` 次查询，每次查询给出两棵树的编号 `u` 和 `v`，你需要计算从 `u` 走到 `v` 最短需要经过多少条路。树的编号从 `1` 到 `N`。根节点默认为 `1` 号树。

**输入格式：**
第一行包含两个整数 `N` 和 `Q`，分别表示树的数量和查询次数。
接下来 `N-1` 行，每行包含两个整数 `x` 和 `y`，表示树 `x` 和树 `y` 之间有一条路。
接下来 `Q` 行，每行包含两个整数 `u` 和 `v`，表示一次查询。

**输出格式：**
对于每次查询，输出 `u` 和 `v` 之间的路径距离。

**输入样例：**
```
7 3
1 2
1 3
2 4
2 5
3 6
3 7
4 7
5 6
2 3
```

**输出样例：**
```
3
4
2
```

**思路分析：**

1.  **构建树：** 用邻接表 `adj` 存储树的结构。
2.  **预处理DFS：**
    *   从节点1（根节点）开始DFS。
    *   `dfs(u, p_u, d)` 函数：
        *   `p[u][0] = p_u` (记录父节点)。
        *   `depth[u] = d` (记录深度)。
        *   对 `u` 的每个子节点 `v`（除了 `p_u`），递归调用 `dfs(v, u, d+1)`。
3.  **预处理倍增数组：**
    *   外层循环 `k` 从 `1` 到 `LOGN-1`。
    *   内层循环 `i` 从 `1` 到 `N`。
    *   `p[i][k] = p[p[i][k-1]][k-1]`。
4.  **LCA查询函数 `getLCA(u, v)`：**
    *   确保 `u` 的深度不小于 `v` 的深度（如果 `depth[u] < depth[v]`，交换 `u, v`）。
    *   将 `u` 向上跳 `depth[u] - depth[v]` 步，使其与 `v` 同深。
    *   如果此时 `u == v`，则 `u` 就是LCA。
    *   否则，从 `LOGN-1` 到 `0` 循环 `k`：
        *   如果 `p[u][k] != p[v][k]`，则 `u = p[u][k]`, `v = p[v][k]`。
    *   最终，`p[u][0]` 就是LCA。
5.  **距离计算函数 `getDistance(u, v)`：**
    *   调用 `getLCA(u, v)` 得到 `lca_node`。
    *   返回 `depth[u] + depth[v] - 2 * depth[lca_node]`。
6.  **主函数：**
    *   读取 `N, Q`。
    *   读取 `N-1` 条边，填充 `adj`。
    *   调用 `dfs(1, 1, 0)` 进行预处理（根节点1，父节点设为自己，深度0）。
    *   调用 `init_lca_table()` 填充倍增数组。
    *   循环 `Q` 次，每次读取 `u, v`，调用 `getDistance(u, v)` 并输出。

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 定义最大节点数和最大层数 (log2(MAXN))
const int MAXN = 100005; // 允许的节点数量
const int LOGN = 18;     // ceil(log2(MAXN)), 2^17 约等于 13万

// 邻接表存储树
std::vector<int> adj[MAXN];

// 存储每个节点的深度
int depth[MAXN];

// p[i][k] 表示节点 i 向上跳 2^k 步能到达的祖先
int p[MAXN][LOGN];

int n, q; // 节点数和查询次数

// DFS预处理：计算深度和直接父节点
// u: 当前节点, pr: u的父节点, d: u的深度
void dfs(int u, int pr, int d) {
    depth[u] = d;
    p[u][0] = pr; // u 的 2^0 步祖先就是它的直接父节点
    for (int v : adj[u]) {
        if (v == pr) { // 避免访问父节点，防止死循环
            continue;
        }
        dfs(v, u, d + 1);
    }
}

// 初始化倍增数组
void init_lca_table() {
    // 从 k=1 开始，计算所有 2^k 步祖先
    for (int k = 1; k < LOGN; ++k) {
        for (int i = 1; i <= n; ++i) {
            // p[i][k] 是 p[i][k-1] 的 2^(k-1) 步祖先的 2^(k-1) 步祖先
            // 也就是 p[i][k-1] 向上跳 2^(k-1) 步
            p[i][k] = p[p[i][k - 1]][k - 1];
        }
    }
}

// 获取节点 u 和 v 的最近公共祖先 (LCA)
int getLCA(int u, int v) {
    // 1. 确保 u 的深度不小于 v 的深度
    if (depth[u] < depth[v]) {
        std::swap(u, v);
    }

    // 2. 将 u 向上跳，使其与 v 处于同一深度
    // 从大到小遍历 k，利用二进制拆分
    for (int k = LOGN - 1; k >= 0; --k) {
        if (depth[u] - (1 << k) >= depth[v]) { // 如果 u 向上跳 2^k 步后仍在 v 或之上
            u = p[u][k];
        }
    }

    // 此时 u 和 v 在同一深度
    if (u == v) { // 如果 u 和 v 已经重合，说明 v (或 u) 就是LCA
        return u;
    }

    // 3. u 和 v 同步向上跳，直到它们的父节点相同 (即LCA的子节点)
    // 从大到小遍历 k
    for (int k = LOGN - 1; k >= 0; --k) {
        if (p[u][k] != p[v][k]) { // 如果 2^k 步祖先不同，则继续向上跳
            u = p[u][k];
            v = p[v][k];
        }
    }

    // 此时 u 和 v 是 LCA 的直接子节点，它们的父节点就是 LCA
    return p[u][0];
}

// 获取节点 u 和 v 之间的路径距离
int getDistance(int u, int v) {
    int lca_node = getLCA(u, v);
    // 距离公式：depth[u] + depth[v] - 2 * depth[LCA(u,v)]
    return depth[u] + depth[v] - 2 * depth[lca_node];
}

int main() {
    // 优化输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> q;

    // 构建邻接表
    for (int i = 0; i < n - 1; ++i) {
        int u, v;
        std::cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // DFS预处理：从根节点1开始，父节点设为1，深度为0
    // 注意：如果根节点没有父节点，p[1][0] 可以设为1或0，只要后续LCA逻辑能处理即可
    // 这里设为1，表示根节点向上跳1步还是自己，这在LCA计算中是安全的
    dfs(1, 1, 0); 
    
    // 初始化倍增LCA表
    init_lca_table();

    // 处理查询
    for (int i = 0; i < q; ++i) {
        int u, v;
        std::cin >> u >> v;
        std::cout << getDistance(u, v) << "\n";
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：**
    *   构建邻接表：`O(N)`
    *   DFS预处理深度和 `p[u][0]`：`O(N)`
    *   初始化倍增数组 `p[u][k]`：`O(N * LOGN)`
    *   每次LCA查询：`O(LOGN)`
    *   每次距离查询：`O(LOGN)` (因为调用了LCA)
    *   总时间复杂度：`O(N + N * LOGN + Q * LOGN) = O((N + Q) * LOGN)`
*   **空间复杂度：**
    *   邻接表 `adj`：`O(N)`
    *   深度数组 `depth`：`O(N)`
    *   倍增数组 `p`：`O(N * LOGN)`
    *   总空间复杂度：`O(N * LOGN)`

### 例题2：树上最远节点对（变种）

**题目描述：**
给定一棵 `N` 个节点的树，以及一个起点 `S`。现在有 `Q` 次操作，每次操作给定一个节点 `T`。你需要找出从 `S` 到 `T` 的路径上，距离 `S` 最远的节点是哪个节点？如果路径上有多个节点距离 `S` 最远，输出深度最小的那个。

**输入格式：**
第一行包含两个整数 `N` 和 `Q`，以及起点 `S`。
接下来 `N-1` 行，每行包含两个整数 `x` 和 `y`，表示树 `x` 和树 `y` 之间有一条路。
接下来 `Q` 行，每行包含一个整数 `T`，表示一次查询。

**输出格式：**
对于每次查询，输出路径 `S` 到 `T` 上距离 `S` 最远的节点编号。

**输入样例：**
```
7 3 1
1 2
1 3
2 4
2 5
3 6
3 7
4
7
5
```

**输出样例：**
```
4
7
5
```

**思路分析：**

这题稍微有点不同，它要求在 `S` 到 `T` 的路径上找到一个节点 `X`，使得 `dist(S, X)` 最大。
观察路径 `S -> ... -> LCA(S, T) -> ... -> T`。
对于路径上的任意一个节点 `X`：
*   如果 `X` 在 `S` 到 `LCA(S, T)` 的路径上，那么 `dist(S, X) = depth[X] - depth[S]` (假设 `S` 是 `X` 的祖先)。
*   如果 `X` 在 `LCA(S, T)` 到 `T` 的路径上，那么 `dist(S, X) = dist(S, LCA(S, T)) + dist(LCA(S, T), X)`。

实际上，从 `S` 到 `T` 的路径，距离 `S` 最远的节点，一定是 `T` 本身！因为 `T` 就在路径的另一端。
所以，这题的陷阱在于，它问的是“路径上距离 `S` 最远的节点”，而这个节点就是 `T`。
但是，为了考察LCA和距离的应用，我们可以理解为：找到一个节点 `X`，使得 `dist(S, X)` 最大，且 `X` 在 `S` 到 `T` 的路径上。

**更严谨的思考：**
路径 `S` 到 `T` 上的所有节点 `X`，`dist(S, X)` 的最大值必然发生在 `X = T` 的时候。因为 `S` 到 `T` 的路径是唯一的简单路径，`T` 是这条路径上距离 `S` 最远的节点。

这题的考点可能在于：
1.  **理解题意：** 路径上距离 `S` 最远的节点就是 `T`。
2.  **LCA和距离的结合：** 即使答案是 `T`，但如果题目问的是“路径上距离 `S` 最远的节点是哪个，并且这个节点是 `S` 的子孙”，那可能就需要更复杂的逻辑。
3.  **如果题目问的是"路径上距离 `LCA(S,T)` 最远的节点"：** 那就是 `S` 或 `T` 中的一个，取决于 `depth[S] - depth[LCA(S,T)]` 和 `depth[T] - depth[LCA(S,T)]` 哪个更大。

对于这道题，直接输出 `T` 即可。但这并不能很好地体现LCA和距离的复杂应用。

**我们把题目稍微改一下，让它更有趣，更符合LCA距离的应用：**
**新题目描述：**
给定一棵 `N` 个节点的树。现在有 `Q` 次查询，每次查询给出两个节点 `u` 和 `v`。你需要找到路径 `u` 到 `v` 上，距离 `u` 最远的节点是哪个？如果距离相同，输出深度更小的那个。

**思路分析（针对新题目）：**
路径 `u` 到 `v` 上的节点 `X`，距离 `u` 最远，那 `X` 必然是 `v`。
但是，如果题目是问：找出路径 `u` 到 `v` 上，**深度最大**的节点是哪个？
或者：找出路径 `u` 到 `v` 上，**距离根节点最远**的节点是哪个？那答案就是 `u` 和 `v` 中深度更大的那个。

这道题目的原始描述，其实直接输出 `T` 就可以，因为 `T` 就在路径的另一端，它就是距离 `S` 最远的节点。这有点像一个“送分题”或者“理解题意”的题。
但如果我们要利用LCA和距离公式，可以这样来理解：
路径 `S` 到 `T` 包含两部分：`S` 到 `LCA(S,T)`，以及 `LCA(S,T)` 到 `T`。
对于路径上的任何节点 `X`：
*   如果 `X` 在 `S` 到 `LCA(S,T)` 之间（包括 `S` 和 `LCA(S,T)`），那么 `dist(S, X) = depth[S] - depth[X]`。
*   如果 `X` 在 `LCA(S,T)` 到 `T` 之间（包括 `LCA(S,T)` 和 `T`），那么 `dist(S, X) = dist(S, LCA(S,T)) + (depth[X] - depth[LCA(S,T)])`。

显然，在第二种情况下，`dist(S,X)` 会随着 `X` 离 `LCA(S,T)` 越远而越大，直到 `X=T`。
所以，距离 `S` 最远的节点确实就是 `T`。

如果题目想考察更深入的LCA应用，可能会问：
**"给定 `u, v`，求 `u` 到 `v` 路径上，距离根节点最远的节点是哪个？"**
这其实就是比较 `depth[u]` 和 `depth[v]`，深度更大的那个就是答案。
如果 `depth[u] == depth[v]`，那么 `u` 和 `v` 哪个都可以，或者按照题目要求输出深度更小的（这和距离根节点最远矛盾）。通常会要求输出编号最小的。

所以，这道题的原始描述，最佳答案就是 `T`。
我们仍旧使用例题1的代码框架，只是 `getDistance` 函数的返回值可能不是直接需要的。

**总结：**
对于例题2（原版），不需要复杂的LCA或距离计算，直接输出 `T` 即可。
如果题目要考察路径上其他性质，比如“路径上深度最大的节点”，那就是 `u` 和 `v` 中深度更大的那个。
LCA和距离公式主要用于计算两点之间的边数。

我们还是回到例题1这种直接计算距离的题目，它更直接地体现LCA距离公式的应用。
例题2的变种如果想考察LCA+距离，通常会是以下形式：
**例题2（修订版）：**
给定一棵 `N` 个节点的树。现在有 `Q` 次查询，每次查询给出两个节点 `u` 和 `v`。你需要找出路径 `u` 到 `v` 上，距离 `LCA(u,v)` 最远的节点是哪个？如果距离相同，输出编号较小的那个。

**思路分析（修订版）：**
1.  首先，像例题1一样，预处理出所有节点的深度 `depth` 和倍增祖先 `p` 数组。
2.  对于每次查询 `(u, v)`：
    *   计算 `lca_node = getLCA(u, v)`。
    *   计算 `u` 到 `lca_node` 的距离：`dist_u_lca = depth[u] - depth[lca_node]`。
    *   计算 `v` 到 `lca_node` 的距离：`dist_v_lca = depth[v] - depth[lca_node]`。
    *   比较 `dist_u_lca` 和 `dist_v_lca`：
        *   如果 `dist_u_lca > dist_v_lca`，则 `u` 距离 `lca_node` 更远，答案是 `u`。
        *   如果 `dist_v_lca > dist_u_lca`，则 `v` 距离 `lca_node` 更远，答案是 `v`。
        *   如果 `dist_u_lca == dist_v_lca`，则 `u` 和 `v` 距离 `lca_node` 相同，根据题目要求输出编号较小的，即 `std::min(u, v)`。

**代码实现（修订版）：**
（基于例题1的代码，只修改 `main` 函数的查询部分）

```cpp
// ... (例题1的头文件、全局变量、dfs、init_lca_table、getLCA 函数不变) ...

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int s_node; // 这里的s_node是原题的起点，修订版中不再需要
    std::cin >> n >> q >> s_node; // s_node 被读入但未被使用

    // 构建邻接表
    for (int i = 0; i < n - 1; ++i) {
        int x, y;
        std::cin >> x >> y;
        adj[x].push_back(y);
        adj[y].push_back(x);
    }

    dfs(1, 1, 0); // 从根节点1开始预处理
    init_lca_table();

    // 处理查询
    for (int i = 0; i < q; ++i) {
        int u, v; // 查询的两个节点
        std::cin >> u >> v;

        int lca_node = getLCA(u, v);

        int dist_u_lca = depth[u] - depth[lca_node];
        int dist_v_lca = depth[v] - depth[lca_node];

        if (dist_u_lca > dist_v_lca) {
            std::cout << u << "\n";
        } else if (dist_v_lca > dist_u_lca) {
            std::cout << v << "\n";
        } else { // 距离相等，输出编号较小的
            std::cout << std::min(u, v) << "\n";
        }
    }

    return 0;
}
```

## 代码实现模板

这里提供一个完整的C++代码模板，包含了LCA倍增法和路径距离计算的所有核心部分。你可以直接使用这个模板来解决相关问题。

```cpp
#include <bits/stdc++.h> // 包含大部分常用库

// 定义最大节点数和最大层数 (log2(MAXN))
const int MAXN = 100005; // 允许的节点数量
const int LOGN = 18;     // ceil(log2(MAXN)), 2^17 约等于 13万，足够处理 10^5 节点

// 邻接表存储树的结构
std::vector<int> adj[MAXN];

// 存储每个节点的深度 (从根节点到当前节点的边数，根节点深度为0)
int depth[MAXN];

// p[i][k] 表示节点 i 向上跳 2^k 步能到达的祖先
// p[i][0] 是 i 的直接父节点
int p[MAXN][LOGN];

int n; // 树的节点数量

// DFS预处理：计算每个节点的深度和直接父节点
// u: 当前正在访问的节点
// pr: u 的父节点
// d: u 的深度
void dfs(int u, int pr, int d) {
    depth[u] = d;        // 设置当前节点的深度
    p[u][0] = pr;        // 设置当前节点的直接父节点 (2^0 步祖先)
    
    // 遍历当前节点 u 的所有邻居
    for (int v : adj[u]) {
        if (v == pr) {   // 如果邻居是父节点，则跳过，避免回溯和死循环
            continue;
        }
        dfs(v, u, d + 1); // 递归访问子节点，深度加1
    }
}

// 初始化倍增数组 p[i][k]
void init_lca_table() {
    // 从 k=1 开始，计算所有 2^k 步祖先
    // p[i][k] 是 p[i][k-1] 的 2^(k-1) 步祖先的 2^(k-1) 步祖先
    // 也就是先跳 2^(k-1) 步到中间节点，再从中间节点跳 2^(k-1) 步
    for (int k = 1; k < LOGN; ++k) {
        for (int i = 1; i <= n; ++i) { // 遍历所有节点
            p[i][k] = p[p[i][k - 1]][k - 1];
        }
    }
}

// 获取节点 u 和 v 的最近公共祖先 (LCA)
int getLCA(int u, int v) {
    // 1. 确保 u 的深度不小于 v 的深度
    // 如果 depth[u] < depth[v]，则交换 u 和 v，保证 u 始终是较深的那个节点
    if (depth[u] < depth[v]) {
        std::swap(u, v);
    }

    // 2. 将较深的节点 u 向上跳，使其与 v 处于同一深度
    // 从 LOGN-1 迭代到 0，利用二进制拆分，快速向上跳
    for (int k = LOGN - 1; k >= 0; --k) {
        // 如果 u 向上跳 2^k 步后仍然在 v 或 v 的下方
        if (depth[u] - (1 << k) >= depth[v]) {
            u = p[u][k]; // u 向上跳 2^k 步
        }
    }

    // 此时 u 和 v 已经在同一深度
    if (u == v) { // 如果 u 和 v 已经重合，说明它们就是彼此的LCA
        return u;
    }

    // 3. u 和 v 同步向上跳，直到它们的父节点相同（即LCA的直接子节点）
    // 从 LOGN-1 迭代到 0
    for (int k = LOGN - 1; k >= 0; --k) {
        // 如果 u 和 v 的 2^k 步祖先不同，说明LCA还在更上方
        // 此时让 u 和 v 同时向上跳 2^k 步
        if (p[u][k] != p[v][k]) {
            u = p[u][k];
            v = p[v][k];
        }
    }

    // 经过上述操作后，u 和 v 已经是 LCA 的直接子节点
    // 它们的父节点 p[u][0] (或 p[v][0]) 就是它们的LCA
    return p[u][0];
}

// 获取节点 u 和 v 之间的路径距离
int getDistance(int u, int v) {
    int lca_node = getLCA(u, v); // 首先找到 u 和 v 的LCA
    // 应用距离公式：dist(u, v) = depth[u] + depth[v] - 2 * depth[LCA(u, v)]
    return depth[u] + depth[v] - 2 * depth[lca_node];
}

int main() {
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int q; // 查询次数
    std::cin >> n >> q; // 读取节点数和查询次数

    // 构建树的邻接表
    for (int i = 0; i < n - 1; ++i) {
        int u, v_node; // 避免变量名冲突，使用 v_node
        std::cin >> u >> v_node;
        adj[u].push_back(v_node);
        adj[v_node].push_back(u);
    }

    // DFS预处理：从根节点1开始，父节点设为1，深度为0
    // 注意：根节点的父节点可以设为自身，或0（如果0号节点不参与编号）
    // 这里设为1，LCA逻辑仍能正确处理
    dfs(1, 1, 0); 
    
    // 初始化倍增LCA表
    init_lca_table();

    // 处理所有查询
    for (int i = 0; i < q; ++i) {
        int u, v_node; // 查询的两个节点
        std::cin >> u >> v_node;
        std::cout << getDistance(u, v_node) << "\n"; // 输出计算出的距离
    }

    return 0;
}
```
