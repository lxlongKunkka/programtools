## 四、Level 3 重构（当前：7专题 → 建议：7专题）

### 主要问题：进制与位运算重叠

| 原专题3 | 原专题6 | 重叠分析 |
|---------|---------|---------|
| 二进制基础 | 原码/反码/补码 | 都在讲整数内存表示 |
| 八进制/十六进制转换 | 位运算 (<<, >>, &) | 关联性强但割裂 |

### 建议：合并为"整数的二进制世界（进制+位运算）"专题
```
原专题3 + 原专题6 → 新专题3「整数的二进制世界」
  01. 进制的本质（十进制 ↔ 二进制 ↔ 十六进制）
  02. 原码、反码、补码（整数在内存中的表示）
  03. 位运算魔法（<<, >>, &, |, ^）
  04. 位运算应用例题（判断奇偶、交换、子集枚举）
```
原来拆成 2个专题 6章 → 合并为 1个专题 4章，**节省 2章**，内容更连贯。

### 新增专题：递归思想入门
> Level 3 是引入递归的最佳位置（已有函数基础）

```
新专题 8（或挂在专题4函数末尾）「递归思想入门」
  01. 递归的本质——函数调自己
  02. 递归五步法（终止条件、递归调用、回溯）
  03. 经典递归：阶乘、斐波那契、数字和
  04. 递归画图思维（递归树）
```

## 五、Level 4 重构（当前：7专题 → 建议：7专题，调整 1 个）

### 主要问题：指针专题过早引入

**现状：** 专题6「指针与引用」（3章）放在 Level4（四级，约小学高年级）  
**问题：** 指针涉及内存地址概念，GESP 三四级竞赛基本不考，小学生理解困难  
**建议：** 替换为「字符串深入」专题

```
删除：专题6 指针与引用（3章）
替换为：专题6 字符串深入（3章）
  01. string 类型与常用操作（length, find, substr）
  02. 字符串与数字互转（stoi, to_string）
  03. 字符串综合例题
```

> 指针内容建议移至 Level5 或以上，与结构体/链表一起教学。

### 算法复杂度专题优化

现有「算法复杂度」2章（空）+ Level4 思维不够连贯：
```
建议专题7「算法效率入门」扩展到 3 章：
  01. 时间复杂度概念与 O 记号（小学生版：数体操步骤数）
  02. 常见排序的复杂度对比（O(n²) vs O(n log n)）
  03. 空间复杂度与内存限制
```

## 六、新课程总体路径图

```
Level 1 ─── 基础语法（IO/变量/运算/if/while）
    │
Level 2 ─── 逻辑进阶（字符/数组/函数初步/枚举/模拟）
    │
Level 3 ─── 算法萌芽（二进制/字符串/排序/贪心/递归入门）  ← 新增递归
    │
Level 4 ─── 进阶思维（STL字符串/复杂排序/搜索初步/复杂度）
    │
Level 5 ─── 图论思维（BFS/DFS/树/指针链表）  ← 指针推迟到这里
```

## 八、无题章节问题推荐策略

### 背景
部分章节有教案但 `## 必做题` 为空。题目 ID 格式为 `LevelN:M`。

### 推荐方法
对于 Level1-4 无题章节，建议采用以下策略：

**策略A（相邻题号延续）：**
找同专题上下章节最大已使用题号，按序分配 3-5 道题。

**策略B（AI内容匹配）：**
将章节教案内容发给 AI，要求推荐 3 道难度匹配的 Hydro 题目关键词。

**当前已用题号统计（Level1）：**
- 专题1：Level1:1 ~ Level1:9（已用）
- 专题2：Level1:10 ~ Level1:15（推测）
- 后续章节题号需从 Hydro 平台导出确认

**行动建议：**
1. 先运行 `batch_gen_lessons.py` 完成教案生成
2. 再由教研人员在 Hydro 平台手动匹配题目 ID
3. 或使用 `import_to_db.js` 更新数据库后在管理后台批量绑定
