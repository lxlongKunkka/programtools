# while 与 do-while 循环

## 教学目标

*   **知识目标**：
    *   理解循环结构的基本概念，知道为什么需要循环。
    *   掌握 `while` 循环的语法结构及其执行流程，理解“先判断，后执行”的特点。
    *   掌握 `do-while` 循环的语法结构及其执行流程，理解“先执行，后判断”的特点。
    *   能够识别 `while` 和 `do-while` 循环在执行次数上的主要区别。
*   **能力目标**：
    *   能够独立编写简单的 `while` 循环程序，解决重复执行特定任务的问题。
    *   能够独立编写简单的 `do-while` 循环程序，解决至少执行一次再判断的问题。
    *   在给定情境下，能够选择合适的循环类型（`while` 或 `do-while`）来解决问题。
*   **素养目标**：
    *   培养计算思维，认识到计算机在处理重复性任务上的高效和优势。
    *   提高问题分析能力，学会将重复性问题抽象为循环结构。
    *   激发对编程的兴趣，享受用代码解决问题的乐趣。

## 趣味引入

同学们，大家有没有玩过这样的游戏：小明说“我要数星星，数到100颗就睡觉！”或者小红说“我要跳绳，跳到绳子断了才休息！”

在这些故事里，小明和小红都在做一件事情，那就是“**重复**”地做某件事，直到某个条件满足了才停下来。

想一想，如果我们想让电脑打印100次“你好！”，要怎么写代码呢？是不是要写100行 `cout << "你好！" << endl;` 呢？天哪，那也太累了吧！如果想打印10000次呢？那我们岂不是要写一万行代码？

别担心！电脑可比我们聪明多了，它有一个“**自动重复机**”，我们只需要告诉它“重复做什么”和“什么时候停下来”，它就能自己乖乖地一直重复下去，直到条件满足！这个神奇的“自动重复机”在编程里就叫做“**循环**”！

今天，我们就来学习两种非常常用的“自动重复机”—— `while` 循环和 `do-while` 循环，看看它们是怎么帮助我们省力气，让电脑变得更聪明的！

## 深度知识点讲解

### 1. 什么是循环？

就像我们刚才说的，循环就是让计算机**重复执行**一段代码的方法。它就像一个勤劳的小机器人，你告诉它一个任务和停止的条件，它就会一直做，直到满足停止的条件为止。

**为什么需要循环？**
*   **省力气**：避免写重复的代码。
*   **高效率**：计算机执行重复任务的速度非常快。
*   **灵活**：可以根据不同的条件来决定重复的次数。

### 2. `while` 循环：先检查，再行动！

想象一下，你准备去游泳池游泳。你会怎么做？是不是先看看水温高不高，水深不深，确定一切OK了，你才跳下去？如果水太冷，你可能就直接回家了，根本不会下水！

`while` 循环就是这样的“**先检查，再行动**”的机器人。它会先检查一个条件，如果条件是真的（符合要求），它才会去执行里面的代码；如果条件一开始就是假的（不符合要求），那么它连一次都不会执行！

**生活类比**：
*   **过马路**：`while (红灯亮着) { 等待; }` 当红灯亮着时，你就一直等待。如果绿灯一开始就亮着，你就直接走过去，根本不用等待。
*   **吃薯片**：`while (薯片袋里还有薯片) { 拿一片吃掉; }` 只要袋子里还有薯片，你就继续吃。如果袋子是空的，你就一片也吃不到。

**`while` 循环的语法结构**：

```cpp
while (条件) {
    // 循环体：当条件为真时，会重复执行这里的代码
    // 记得在循环体内部改变条件，否则可能会无限循环！
}
```

*   `while` 是关键字，意思是“当……的时候”。
*   `条件` 是一个表达式，结果只能是 `true` (真) 或 `false` (假)。
*   `{ }` 里面的代码块叫做“循环体”。

**`while` 循环的执行流程**：
1.  首先，检查 `条件` 是否为 `true`。
2.  如果 `条件` 为 `true`：
    *   执行 `{ }` 里面的代码（循环体）。
    *   执行完循环体后，再次回到第1步，重新检查 `条件`。
3.  如果 `条件` 为 `false`：
    *   跳出 `while` 循环，继续执行 `while` 循环后面的代码。

**关键点剖析**：
*   **0 次或多次执行**：`while` 循环的特点是，它的循环体**可能一次都不执行**，如果一开始条件就是 `false`。
*   **改变条件**：在循环体内部，一定要有代码能够改变 `条件` 的结果，否则 `条件` 可能会一直为 `true`，导致程序陷入“**无限循环**”（Infinite Loop），就像机器人一直重复工作停不下来一样，程序就“卡住”了！

### 3. `do-while` 循环：先行动，再检查！

现在，再想象一个场景：你想让你的朋友尝尝你做的神秘饮料。你会怎么做？是不是先给他一杯，让他尝一口，然后才问他“好喝吗？”？就算他一开始就说“我不想喝”，你可能也会坚持让他尝一口对不对？

`do-while` 循环就是这样的“**先行动，再检查**”的机器人。它会**至少先执行一次**循环体里面的代码，然后再去检查条件。如果条件是真的，它就继续执行；如果条件是假的，它就停下来。

**生活类比**：
*   **玩游戏**：`do { 玩一局游戏; } while (还想玩);` 不管三七二十一，先玩一局再说！玩完之后，如果还想玩，就继续下一局。
*   **点菜**：`do { 推荐一个菜; } while (顾客不满意);` 服务员先推荐一个菜，如果顾客不满意，就继续推荐下一个。

**`do-while` 循环的语法结构**：

```cpp
do {
    // 循环体：至少会执行一次这里的代码
    // 记得在循环体内部改变条件！
} while (条件); // 注意！这里有一个分号！
```

*   `do` 关键字，意思是“做”。
*   `{ }` 里面的代码块是循环体。
*   `while (条件);` 意思是“当……的时候，继续做”。

**`do-while` 循环的执行流程**：
1.  首先，**无条件执行一次** `{ }` 里面的代码（循环体）。
2.  执行完循环体后，检查 `条件` 是否为 `true`。
3.  如果 `条件` 为 `true`：
    *   再次回到第1步，执行循环体。
4.  如果 `条件` 为 `false`：
    *   跳出 `do-while` 循环，继续执行 `do-while` 循环后面的代码。

**关键点剖析**：
*   **至少一次执行**：`do-while` 循环的特点是，它的循环体**至少会执行一次**，即使条件一开始就是 `false`。
*   **分号**：`do-while` 循环的 `while (条件)` 后面**必须有一个分号** `;`，这是它语法上一个特殊的地方，很多同学会忘记！

### 4. `while` 与 `do-while` 的区别总结

| 特性       | `while` 循环                               | `do-while` 循环                               |
| :--------- | :----------------------------------------- | :-------------------------------------------- |
| **执行次数** | 0 次或多次                                 | 1 次或多次                                    |
| **判断时机** | 先判断条件，再执行循环体                   | 先执行循环体，再判断条件                      |
| **适用场景** | 当循环体可能不需要执行时 (例如：文件可能为空) | 当循环体至少需要执行一次时 (例如：用户输入校验) |
| **语法特点** | `while (条件) { ... }`                     | `do { ... } while (条件);` (注意分号)         |

**常见误区**：
*   **无限循环**：忘记在循环体内修改条件，导致条件一直为真。
*   **`do-while` 的分号**：忘记在 `do-while` 语句末尾添加分号。
*   **选择错误**：当需要至少执行一次时，误用了 `while` 循环，导致逻辑错误。

## 典型例题精讲

### 例题 1: 使用 `while` 循环打印数字 1 到 5

**题目描述**：请编写一个程序，使用 `while` 循环从 1 开始，依次打印出 1、2、3、4、5，每个数字占一行。

**思路分析**：
1.  我们需要一个变量来记住当前打印的数字，我们叫它 `n`。
2.  `n` 应该从 1 开始。
3.  我们要一直打印，直到 `n` 变成 6（也就是 `n` 小于等于 5 的条件不再满足）。
4.  每次打印完 `n` 之后，`n` 都要增加 1，才能打印下一个数字。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含常用的头文件

int main() {
    int n = 1; // 定义一个整数变量n，并初始化为1，作为计数器

    // 当 n 小于等于 5 的时候，就一直重复执行循环体里面的代码
    while (n <= 5) {
        std::cout << n << std::endl; // 打印当前 n 的值，并换行
        n = n + 1;                   // 让 n 增加 1，准备打印下一个数字
    }

    std::cout << "循环结束！" << std::endl; // 循环结束后打印一句话
    return 0;
}
```

**运行结果**：
```
1
2
3
4
5
循环结束！
```

**复杂度分析**：
*   时间复杂度：O(N)，其中 N 是循环执行的次数。在这个例子中，循环执行了 5 次，所以是 O(5)，也就是常数时间，但我们通常用 N 来表示循环次数。
*   空间复杂度：O(1)，只使用了常数个变量。

### 例题 2: 使用 `do-while` 循环确保用户输入有效

**题目描述**：请编写一个程序，要求用户输入一个 1 到 10 之间的数字。如果用户输入的数字不在这个范围内，就一直提示用户重新输入，直到输入一个有效的数字为止。

**思路分析**：
1.  我们至少要提示用户输入一次，所以 `do-while` 循环很适合。
2.  我们需要一个变量来存储用户输入的数字，我们叫它 `a`。
3.  用户输入后，我们要检查这个数字是否在 1 到 10 之间。
4.  如果 `a` 小于 1 或者 `a` 大于 10，说明输入无效，需要继续循环。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含常用的头文件

int main() {
    int a; // 定义一个整数变量a，用于存储用户输入的数字

    // do-while 循环会先执行一次循环体，再判断条件
    do {
        std::cout << "请输入一个 1 到 10 之间的数字: "; // 提示用户输入
        std::cin >> a; // 读取用户输入的数字到变量 a

        // 如果输入的数字不在 1 到 10 之间，就提示错误
        if (a < 1 || a > 10) {
            std::cout << "输入无效！请重新输入。" << std::endl;
        }

    } while (a < 1 || a > 10); // 当 a 小于 1 或者 a 大于 10 时，条件为真，继续循环

    std::cout << "你输入的有效数字是: " << a << std::endl; // 打印用户输入的有效数字
    return 0;
}
```

**运行结果示例**：
```
请输入一个 1 到 10 之间的数字: 0
输入无效！请重新输入。
请输入一个 1 到 10 之间的数字: 11
输入无效！请重新输入。
请输入一个 1 到 10 之间的数字: 5
你输入的有效数字是: 5
```

**复杂度分析**：
*   时间复杂度：O(K)，其中 K 是用户输入无效数字的次数。在最坏情况下，用户可能一直输入无效数字，理论上是无限的，但在实际使用中 K 通常很小。
*   空间复杂度：O(1)，只使用了常数个变量。

## 代码实现模板

### `while` 循环模板

```cpp
#include <bits/stdc++.h> // 包含所有标准库头文件，方便使用

int main() {
    // 1. 初始化循环控制变量（通常是计数器）
    int i = 初始值; 

    // 2. while 关键字，后面跟着小括号里的循环条件
    //    只要条件为真 (true)，就一直执行大括号里的代码块
    while (i 符合某个条件) { 
        // 3. 循环体：需要重复执行的代码
        //    例如：打印、计算等
        std::cout << "这是第 " << i << " 次循环" << std::endl;

        // 4. 更新循环控制变量：这是最关键的一步！
        //    确保在某个时候条件会变为假 (false)，避免无限循环
        i = i + 1; // 或者 i++; i--; i = i * 2; 等
    }

    std::cout << "while 循环结束。" << std::endl;

    return 0;
}
```

### `do-while` 循环模板

```cpp
#include <bits/stdc++.h> // 包含所有标准库头文件，方便使用

int main() {
    // 1. 初始化循环控制变量（通常是计数器或用户输入变量）
    int j = 初始值; 

    // 2. do 关键字，后面跟着大括号里的循环体
    //    这里的代码至少会执行一次
    do {
        // 3. 循环体：需要重复执行的代码
        //    例如：提示用户输入、进行计算等
        std::cout << "这是第 " << j << " 次循环" << std::endl;

        // 4. 更新循环控制变量：确保在某个时候条件会变为假 (false)
        j = j + 1; // 或者 j++; j--; 等

    // 5. while 关键字，后面跟着小括号里的循环条件，以及一个分号！
    //    当条件为真 (true) 时，会继续执行 do 里面的代码
    } while (j 符合某个条件); // 注意这里的分号！

    std::cout << "do-while 循环结束。" << std::endl;

    return 0;
}
```

```cpp
#include <bits/stdc++.h>

int main() {

}

        }
```
```cpp
        i = i + 1;
    }
    return 0;
}
```

```cpp
#include <string> // 引入string头文件

int main() {
    std::string s; // 定义一个字符串变量 s 存储用户输入的密码
    std::string p = "123"; // 预设密码

    do {
        std::cout << "请输入密码: ";
        std::cin >> s; // 读取用户输入的字符串

        if (s != p) { // 如果输入的密码不等于预设密码
            std::cout << "密码错误，请重试！" << std::endl;
        }
    } while (s != p); // 当密码不正确时，继续循环

    std::cout << "密码正确，欢迎进入！" << std::endl;
    return 0;
}
```

## 教学评价与作业

**教学评价**：

*   **课堂参与**：观察学生在趣味引入、知识点讲解和课堂互动环节的积极性和参与度。
*   **理解程度**：通过提问、即时练习和小组讨论，评估学生对 `while` 和 `do-while` 循环语法、执行流程及区别的掌握情况。
*   **编程能力**：检查学生在即时练习中编写代码的正确性、逻辑性和规范性。
*   **思维发展**：评价学生是否能将循环思想应用到生活场景中，并尝试解决简单重复问题。

**作业布置**：

1.  **必做题**：完成“分层练习题目”中的所有“基础巩固”和“能力提升”任务，并确保代码能够正确运行。
2.  **选做题**：尝试完成“拓展挑战”中的“密码输入”任务。
3.  **创意题**：
    *   请同学们思考一个生活中需要重复做的事情，然后尝试用文字描述出来，并说明应该用 `while` 还是 `do-while` 循环，为什么？
    *   例如：每天早上刷牙直到牙齿干净、玩游戏直到闯关成功、给花浇水直到盆里有水溢出。
4.  **预习任务**：预习下节课要学习的 `for` 循环，思考它和 `while` 循环有什么相似和不同之处。
