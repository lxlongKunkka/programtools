## 深度知识点讲解

### 1. 四舍五入：让数字更“友好”的魔法

**什么是四舍五入？**
四舍五入是一种非常常用的近似方法，它可以让一个小数变得更简洁，更容易理解和使用。想象一下，你在量身高，量出来是150.4厘米，你会说你大概150厘米高；如果量出来是150.6厘米，你就会说你大概151厘米高。这就是四舍五入在生活中的应用！

**核心规则：看“邻居”**
要将一个小数四舍五入到**整数**，我们只需要看它**小数点后第一位**的数字：
*   如果这位数字是 `0, 1, 2, 3, 4` （也就是小于5），我们就把小数部分全部**“舍”**掉，整数部分不变。
*   如果这位数字是 `5, 6, 7, 8, 9` （也就是大于等于5），我们就把小数部分全部**“舍”**掉，同时整数部分要**“进1”**。

**生活类比：决定去留的门槛**
你可以把小数点后第一位数字想象成一个“门槛”。
*   如果你的分数是 `64.9` 分，你想四舍五入到整数，看小数点后第一位是 `9`，它跨过了 `5` 这个门槛，所以整数部分 `64` 就要“进1”，变成 `65` 分。
*   如果你的分数是 `64.3` 分，看小数点后第一位是 `3`，它没有跨过 `5` 这个门槛，所以小数部分就“舍”掉，整数部分 `64` 不变，还是 `64` 分。

**C++代码实现到整数 (针对正数)**
在C++中，我们可以用一个巧妙的小技巧来实现四舍五入到整数：**`static_cast<int>(x + 0.5)`**。

*   **为什么是 `+ 0.5`？**
    *   `static_cast<int>(...)` 的作用是**向下取整**（或者叫“截断”），它会直接把小数部分扔掉，只保留整数部分。
        *   `static_cast<int>(3.1)` 得到 `3`
        *   `static_cast<int>(3.9)` 得到 `3`
        *   `static_cast<int>(4.0)` 得到 `4`
    *   现在我们来加上 `0.5` 看看效果：
        *   如果 `x = 3.1` (小数点后第一位是 `1`，应该舍去)：
            `x + 0.5 = 3.1 + 0.5 = 3.6`
            `static_cast<int>(3.6)` 得到 `3`。**正确！**
        *   如果 `x = 3.4` (小数点后第一位是 `4`，应该舍去)：
            `x + 0.5 = 3.4 + 0.5 = 3.9`
            `static_cast<int>(3.9)` 得到 `3`。**正确！**
        *   如果 `x = 3.5` (小数点后第一位是 `5`，应该进位)：
            `x + 0.5 = 3.5 + 0.5 = 4.0`
            `static_cast<int>(4.0)` 得到 `4`。**正确！**
        *   如果 `x = 3.9` (小数点后第一位是 `9`，应该进位)：
            `x + 0.5 = 3.9 + 0.5 = 4.4`
            `static_cast<int>(4.4)` 得到 `4`。**正确！**

    这个方法是不是很神奇？它完美地利用了 `static_cast<int>` 的向下取整特性，实现了四舍五入！
*   **小提示：** 这个方法主要适用于**正数**。对于负数，四舍五入的规则会稍微复杂一些，在GESP一级我们暂时不深入讨论。

### 2. 计算机的“小数烦恼”——浮点数精度问题

计算机在处理整数时通常非常准确，但处理小数（我们称之为**浮点数**）时，有时会遇到一些小麻烦，产生微小的误差。这就是**浮点数精度问题**。

**本质原因：二进制与十进制的“翻译”困难**
我们人类习惯用十进制（0-9这十个数字）来表示数字。但计算机内部只认识二进制（0和1）。
就像我们把 `1/3` 写成 `0.33333...` 永远写不完一样，有些十进制小数（比如 `0.1`）在转换成二进制时，也会变成一个无限循环的小数！
`0.1` 的二进制表示是 `0.00011001100110011...` (无限循环的 `0011`)。

**生活类比：有限的刻度尺**
想象你有一把刻度尺，上面只有毫米的刻度（最小单位）。现在你想要量一个东西，它的真实长度是 `1.23456789` 米。你的尺子能精确地量出来吗？不能！你最多只能说它大概是 `1.235` 米，或者 `1.23` 米。
计算机也是一样，它用来存储小数的空间是有限的，就像你的刻度尺的刻度是有限的。当遇到无限循环的二进制小数时，它只能“截断”一部分，存储一个近似值。

**后果：微小的误差**
这些微小的误差在大多数情况下我们感受不到，但有时候它们会悄悄地影响我们的计算结果。
例如，你可能认为 `0.1 + 0.2` 应该严格等于 `0.3`。但在计算机内部，由于 `0.1` 和 `0.2` 都可能被存储为略有偏差的近似值，它们相加的结果可能不是精确的 `0.3`，而是一个非常接近 `0.3` 的数，比如 `0.30000000000000004`。

**常见误区：** 以为计算机能够精确存储和计算所有小数。

**如何解决（GESP 1 级别）？**

1.  **使用 `double` 类型：**
    C++中有两种常用的浮点数类型：`float`（单精度）和 `double`（双精度）。`double` 类型的存储空间更大，能表示的小数位数更多，所以它的精度比 `float` 更高，可以减少误差。在竞赛编程中，通常推荐使用 `double`。

2.  **“放大”变整数法 (处理小数位数时非常有效)：**
    当我们需要精确计算小数，或者对小数进行四舍五入到指定的位数时，我们可以利用“放大”的技巧，把小数变成整数来处理，最后再变回小数。
    *   **例如：** 要把 `3.14` 四舍五入到小数点后一位。
        1.  先乘以 `10`，把它放大：`3.14 * 10 = 31.4`。现在，我们想保留的小数点后第一位 `1` 变成了整数部分。
        2.  对 `31.4` 进行四舍五入到整数：`static_cast<int>(31.4 + 0.5) = 31`。
        3.  再除以 `10.0` 还原：`31 / 10.0 = 3.1`。
    *   这个方法巧妙地避开了小数直接计算的精度问题，因为它把关键的四舍五入操作转移到了整数上！
    *   如果要保留两位小数，就乘以 `100`，最后再除以 `100.0`。依此类推。

## 典型例题精讲

### 例题一：四舍五入到整数

**题目描述：**
输入一个小数 `x`，请你编写程序，输出 `x` 四舍五入到整数后的结果。

**输入样例：**
`3.7`

**输出样例：**
`4`

**输入样例：**
`2.4`

**输出样例：**
`2`

**思路分析：**
这道题直接考察我们刚刚学到的四舍五入到整数的方法。对于正数 `x`，我们只需要使用 `static_cast<int>(x + 0.5)` 即可。
1.  首先，需要定义一个 `double` 类型的变量来存储输入的小数 `x`。
2.  然后，计算 `x + 0.5`。
3.  最后，将结果强制转换为 `int` 类型，并输出。

**代码实现：**

```cpp
#include <iostream> // 引入输入输出流库

int main() {
    double x; // 声明一个双精度浮点数变量 x
    std::cin >> x; // 从键盘读取一个小数

    // 使用四舍五入的核心技巧：加 0.5 再强制转换为整数
    // 例如：
    // 如果 x 是 3.7，那么 x + 0.5 = 4.2，static_cast<int>(4.2) = 4
    // 如果 x 是 2.4，那么 x + 0.5 = 2.9，static_cast<int>(2.9) = 2
    int y = static_cast<int>(x + 0.5);

    std::cout << y << std::endl; // 输出四舍五入后的整数结果

    return 0; // 程序正常结束
}
```

**时间复杂度：**
O(1) - 无论输入什么数字，程序执行的步骤都是固定的，所以时间复杂度是常数级别。

### 例题二：四舍五入到指定小数位数 (保留一位小数)

**题目描述：**
输入一个小数 `x`，请你编写程序，将其四舍五入到小数点后一位并输出。

**输入样例：**
`3.14`

**输出样例：**
`3.1`

**输入样例：**
`3.15`

**输出样例：**
`3.2`

**思路分析：**
这道题需要我们将小数四舍五入到小数点后一位。这时候，直接用 `+0.5` 的方法就不够了。我们需要用到前面提到的“放大再缩小”的技巧，同时也要注意输出格式。

1.  **放大：** 要保留一位小数，意味着我们要看小数点后第二位来决定第一位是“舍”还是“入”。所以，我们先把 `x` 乘以 `10.0`，让原来小数点后第一位变成整数部分，原来小数点后第二位变成新的小数点后第一位。
    *   `3.14 * 10.0 = 31.4`
    *   `3.15 * 10.0 = 31.5`
2.  **四舍五入到整数：** 对放大后的结果（`temp_x`）进行四舍五入到整数。
    *   `static_cast<int>(31.4 + 0.5) = 31`
    *   `static_cast<int>(31.5 + 0.5) = 32`
3.  **缩小还原：** 将四舍五入后的整数结果再除以 `10.0`，还原到原来小数位。
    *   `31 / 10.0 = 3.1`
    *   `32 / 10.0 = 3.2`
4.  **输出格式：** 为了确保输出结果正好显示一位小数，我们需要使用 `iomanip` 库中的 `std::fixed` 和 `std::setprecision(1)`。

**代码实现：**

```cpp
#include <iostream> // 引入输入输出流库
#include <iomanip>  // 引入用于格式化输出的库，特别是设置小数点位数

int main() {
    double x; // 声明一个双精度浮点数变量 x
    std::cin >> x; // 从键盘读取一个小数

    // 步骤1: 放大。要保留小数点后 1 位，所以乘以 10.0
    // 例如：x = 3.14 -> temp_x = 31.4
    // 例如：x = 3.15 -> temp_x = 31.5
    double temp_x = x * 10.0;

    // 步骤2: 对放大后的数进行四舍五入到整数
    // 例如：31.4 + 0.5 = 31.9 -> static_cast<int>(31.9) = 31
    // 例如：31.5 + 0.5 = 32.0 -> static_cast<int>(32.0) = 32
    int rounded_int = static_cast<int>(temp_x + 0.5);

    // 步骤3: 缩小还原。将结果除以 10.0
    // 例如：31 / 10.0 = 3.1
    // 例如：32 / 10.0 = 3.2
    double final_result = rounded_int / 10.0;

    // 设置输出格式：
    // std::fixed 确保输出是固定小数位数模式
    // std::setprecision(1) 设置小数点后显示 1 位
    std::cout << std::fixed << std::setprecision(1) << final_result << std::endl;

    return 0; // 程序正常结束
}
```

**时间复杂度：**
O(1) - 程序的执行步骤是固定的，与输入值大小无关。

## 代码实现模板

### 1. 四舍五入到整数 (适用于正数)

这个模板是最基础也是最常用的四舍五入方法，记住它就像记住一个魔法咒语！

```cpp
#include <iostream> // GESP 1 常用输入输出库

int main() {
    double x; // 声明一个双精度浮点数变量 x，用来存储小数
    std::cin >> x; // 从键盘读取你想要四舍五入的小数

    // 四舍五入到整数的核心魔法：加 0.5 再强制转换为整数
    // 记住这个小技巧：x + 0.5 会让所有 >= X.5 的数变成 >= (X+1).0
    // 而所有 < X.5 的数会变成 < (X+1).0
    // 然后 static_cast<int> 会把小数部分直接“砍掉”，实现四舍五入！
    int y = static_cast<int>(x + 0.5);

    std::cout << y << std::endl; // 输出四舍五入后的整数结果
    return 0; // 程序运行成功
}
```

### 2. 四舍五入到小数点后 N 位 (以 N=2 为例)

当你需要保留更多小数位时，就需要“放大再缩小”的技巧了。这里的 N 代表你想保留的小数位数。

```cpp
#include <iostream> // 输入输出库
#include <iomanip>  // 格式化输出库，用于控制小数点位数

int main() {
    double x; // 声明一个双精度浮点数变量 x
    std::cin >> x; // 从键盘读取一个小数

    // 假设我们要四舍五入到小数点后 2 位 (N=2)
    // 步骤1: 放大！乘以 10 的 N 次方 (这里 N=2，所以乘以 100.0)
    // 这样做是为了把我们想保留的小数部分“挪到”整数部分，方便四舍五入。
    double temp_x = x * 100.0;

    // 步骤2: 对放大后的数进行四舍五入到整数
    // 这和上面的四舍五入到整数的魔法是一样的！
    int rounded_int = static_cast<int>(temp_x + 0.5);

    // 步骤3: 还原！除以 10 的 N 次方 (这里 N=2，所以除以 100.0)
    // 把整数结果再变回小数，但此时它已经经过了四舍五入。
    double final_result = rounded_int / 100.0;

    // 输出结果，并设置输出精度为 N 位小数
    // std::fixed 告诉 cout 按照固定小数位数模式输出
    // std::setprecision(2) 告诉 cout 输出小数点后 2 位
    std::cout << std::fixed << std::setprecision(2) << final_result << std::endl;
    return 0; // 程序运行成功
}
```

**小提示：** 如果你需要四舍五入到小数点后 3 位，就把代码中的 `100.0` 改成 `1000.0`，`std::setprecision(2)` 改成 `std::setprecision(3)`，以此类推！
