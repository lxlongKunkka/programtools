## 教学目标

### 知识目标
1.  **理解欧拉路与欧拉回路的定义**：学生能够准确区分欧拉路径和欧拉回路，并理解“一笔画”问题的本质。
2.  **掌握欧拉路/欧拉回路的判定条件**：学生能够记住并应用无向图和有向图各自的欧拉路/回路存在充要条件。
3.  **了解Hierholzer算法的基本思想**：学生能够理解基于深度优先搜索（DFS）的欧拉路/回路查找算法的核心逻辑。

### 能力目标
1.  **判断图的欧拉性质**：学生能够对给定的无向图或有向图，通过计算度数（入度/出度）和检查连通性，判断其是否存在欧拉路或欧拉回路。
2.  **实现欧拉路/回路算法**：学生能够运用C++编程语言，基于Hierholzer算法，实现查找并输出欧拉路或欧拉回路的代码。
3.  **分析算法复杂度**：学生能够对所实现的欧拉路/回路算法进行时间复杂度分析。

### 素养目标
1.  **培养逻辑推理能力**：通过对欧拉路/回路存在条件的深入探讨，培养学生从具体问题抽象出数学模型，并进行严谨逻辑推理的能力。
2.  **提升问题解决思维**：在学习Hierholzer算法时，引导学生体会如何将数学理论转化为高效的计算机算法，培养计算思维和解决实际问题的能力。
3.  **激发对图论的兴趣**：通过“哥尼斯堡七桥问题”等经典案例，激发学生对图论领域的好奇心和探索欲，感受算法之美。

## 趣味引入

同学们，你们有没有玩过“一笔画”的游戏？就是那种给你一个图形，要求你从某个点出发，不抬笔，不重复地走过所有的线条，最后回到起点或者到达另一个点。比如，下面这个图形：

```
  A---B
  | / |
  D---C
```
你能从A点出发，一笔画过所有边，最后回到A点吗？

这个看似简单的游戏，其实是图论领域一个非常经典且重要的分支——**欧拉路与欧拉回路**的源头！

在欧洲，有一个叫哥尼斯堡的城市（现在叫加里宁格勒），那里有一条普雷格尔河，河上有七座桥。当地的居民喜欢在假日散步，他们就提出了一个有趣的问题：“有没有可能从城里的某处出发，不重复地走过这七座桥，最后回到起点？”

这个问题在当时难倒了许多人，直到18世纪，一位名叫莱昂哈德·欧拉的瑞士数学家，巧妙地将这个问题抽象成了一个数学模型，并给出了完美的解答。他不仅解决了哥尼斯堡七桥问题，还为此创立了图论中一个重要的概念，也就是我们今天要学习的——**欧拉路与欧拉回路**！

那么，欧拉是如何思考这个问题的？他发现了什么秘密？我们今天就一起走进欧拉的智慧世界，揭开“一笔画”的奥秘！

## 深度知识点讲解

### 1. 欧拉的由来：哥尼斯堡七桥问题

欧拉是这样思考哥尼斯堡七桥问题的：
- 他把陆地（或河中的岛屿）看作一个个“点”（顶点）。
- 他把桥看作连接这些“点”的“线”（边）。
- 这样，整个城市地图就被抽象成了一个**图**。

问题的核心变成了：在一个由顶点和边构成的图中，能否找到一条路径，使得这条路径经过图中的**每一条边恰好一次**？

欧拉最终证明了，哥尼斯堡七桥问题是无解的。他通过分析每个顶点的“度数”（连接到该顶点的边的数量），发现了欧拉路和欧拉回路存在的关键条件。

### 2. 基本概念

在理解欧拉路和欧拉回路之前，我们先回顾一下图的基本概念：
- **顶点 (Vertex)**：图中的点，通常用 `u, v` 表示。
- **边 (Edge)**：连接两个顶点的线，通常用 `(u, v)` 表示。
- **度数 (Degree)**：连接到某个顶点的边的数量。
    - **无向图**：顶点的度数就是连接它的边的数量。
    - **有向图**：
        - **入度 (In-degree)**：指向该顶点的边的数量。
        - **出度 (Out-degree)**：从该顶点指出的边的数量。
- **连通性**：
    - **无向图**：如果任意两个顶点之间都存在路径，则称图是连通的。
    - **有向图**：
        - **弱连通**：如果忽略边的方向，图是连通的。
        - **强连通**：如果任意两个顶点 `u, v`，都存在从 `u` 到 `v` 的路径，也存在从 `v` 到 `u` 的路径。

现在，我们正式介绍欧拉相关的概念：

*   **欧拉路径 (Eulerian Path)**：
    *   一条路径，**经过图中所有边恰好一次**。
    *   起点和终点可以不同。
    *   生活类比：一次性走完一个迷宫的所有通道，不重复，不漏掉。

*   **欧拉回路 (Eulerian Circuit)**：
    *   一条路径，**经过图中所有边恰好一次**，并且**起点和终点是同一个顶点**。
    *   生活类比：从家出发，把小区里所有街道都走一遍，不重复，不漏掉，最后回到家。

*   **欧拉图 (Eulerian Graph)**：包含欧拉回路的图。
*   **半欧拉图 (Semi-Eulerian Graph)**：包含欧拉路径但不包含欧拉回路的图。

### 3. 欧拉路/欧拉回路的判定条件

欧拉发现的关键在于顶点的度数。让我们一起思考“为什么”这些条件是成立的。

#### 3.1 无向图的判定条件

**一个无向图 G 存在欧拉路或欧拉回路，当且仅当：**

1.  **欧拉回路存在条件：**
    *   图是连通的（所有非孤立顶点都在同一个连通分量中）。
    *   **所有顶点的度数都是偶数。**
    *   **为什么？** 想象一下你在沿着一条欧拉回路行走。每当你进入一个顶点，你必然会从这个顶点离开。这意味着，除了起点和终点（它们是同一个顶点），所有中间顶点，你进入的次数和离开的次数是相等的。每进出一次，就会消耗两条边，所以这些顶点的度数必须是偶数。对于起点/终点，它既是开始也是结束，所以它也满足进出平衡，度数也是偶数。

2.  **欧拉路径存在条件：**
    *   图是连通的（所有非孤立顶点都在同一个连通分量中）。
    *   **恰好有0个或2个顶点的度数是奇数。**
        *   如果有0个奇度顶点，则实际上是欧拉回路。
        *   如果有2个奇度顶点，那么这两个顶点必然是欧拉路径的起点和终点。
    *   **为什么？** 同样，当你沿着一条欧拉路径行走时：
        *   对于路径上的**中间顶点**（既非起点也非终点），你每次进入它都会离开它，所以进出平衡，度数是偶数。
        *   对于**起点**，你从它出发，但没有“进入”它来平衡这次出发，所以它会比其他顶点多一条“出”边，度数是奇数。
        *   对于**终点**，你到达它，但没有“离开”它来平衡这次到达，所以它会比其他顶点多一条“入”边，度数是奇数。
        *   因此，只有起点和终点可能度数为奇数，且只能是两个。

#### 3.2 有向图的判定条件

**一个有向图 G 存在欧拉路或欧拉回路，当且仅当：**

1.  **欧拉回路存在条件：**
    *   图是弱连通的（如果忽略边的方向，图是连通的）。**更严格地说，所有非孤立顶点必须在同一个强连通分量中。** 但在判断欧拉路/回路时，通常只需要检查弱连通性，并确保所有非孤立点形成一个强连通分量。
    *   **所有顶点的入度等于出度。** (`in_degree(v) == out_degree(v)` 对所有 `v` 成立)
    *   **为什么？** 沿着欧拉回路行走，每当你进入一个顶点，你必然会从这个顶点离开。对于有向图，这意味着每有一条边指向该顶点（入度），就必须有一条边从该顶点指出（出度）。对于所有顶点，包括起点/终点，入度都必须等于出度才能形成回路。

2.  **欧拉路径存在条件：**
    *   图是弱连通的。
    *   满足以下条件：
        *   **恰好有一个顶点 `s` 满足 `out_degree(s) - in_degree(s) = 1`。** (这个顶点是路径的起点)
        *   **恰好有一个顶点 `t` 满足 `in_degree(t) - out_degree(t) = 1`。** (这个顶点是路径的终点)
        *   **所有其他顶点的入度等于出度。**
    *   **为什么？**
        *   对于路径上的**中间顶点**，每次进入它都会离开它，所以入度等于出度。
        *   对于**起点 `s`**，它多发出一条边，所以出度比入度多1。
        *   对于**终点 `t`**，它多接收一条边，所以入度比出度多1。
        *   除了 `s` 和 `t`，其他所有顶点的入度与出度都必须相等。

**常见误区：**
- **忽略连通性**：即使度数条件满足，如果图不连通，也无法走遍所有边。例如，两个独立的三角形，每个三角形都是欧拉图，但整个图不是。
- **有向图误判强连通**：对于有向图，弱连通性是必要条件，但如果存在欧拉回路，则所有非孤立点必须构成一个强连通分量。对于欧拉路径，则只需要弱连通即可。在实际判断中，通常先检查度数条件，再从符合条件的起点（或任意点）开始DFS/BFS，看是否能访问所有非孤立边。

### 4. 算法思想：Hierholzer 算法 (高效查找欧拉路径/回路)

Hierholzer 算法是一种高效查找欧拉路径/回路的算法，通常比Fleury算法更常用。它的核心思想是利用深度优先搜索（DFS）来构建路径。

**基本思想：**
1.  首先，根据上述条件判断图是否存在欧拉路径或欧拉回路。如果不存在，则直接结束。
2.  选择一个合适的起点 `s`：
    *   对于欧拉回路，可以选择任意一个非孤立顶点作为起点。
    *   对于欧拉路径，必须选择那个 `out_degree - in_degree = 1` (有向图) 或度数为奇数 (无向图) 的顶点作为起点。如果没有这样的点，就任意选择一个非孤立点（此时图必有欧拉回路）。
3.  从起点 `s` 开始进行深度优先搜索（DFS）。
4.  在DFS遍历过程中，当我们从一个顶点 `u` 沿着边 `(u, v)` 访问到 `v` 时，我们立即将这条边**从图中“删除”**（或标记为已访问），以确保每条边只被访问一次。
5.  **关键步骤：** 当DFS从 `dfs(v)` 返回到 `dfs(u)`，并且 `v` 的所有邻边都已经被访问完毕时，我们将顶点 `v` 添加到欧拉路径的**末尾**。
    *   为什么是末尾？因为DFS是先深入再回溯。当我们回溯到一个顶点时，意味着从这个顶点出发的所有可能路径都已经被探索完毕，并且它的所有边都已经被使用了。所以，这个顶点应该是路径中比较靠后的部分。最终得到的路径是逆序的，需要反转。

**数据结构与边删除技巧：**
- **邻接表**：`vector<pair<int, int>> adj[N];` 其中 `pair<int, int>` 存储 `(v, edge_id)`，`v` 是邻接顶点，`edge_id` 是边的唯一标识符。
- **边标记数组**：`bool used_edge[M];` `M` 是总边数（对于无向图，可以看作 `2 * 实际边数`，或者 `实际边数` 然后用 `edge_id / 2` 这样的方式来标记）。
- **优化访问效率**：为了避免每次都从邻接表的开头遍历，可以使用一个 `int cur[N];` 数组。`cur[u]` 记录了顶点 `u` 的邻接表中，下一条未被访问的边的索引。这样，每次从 `u` 开始DFS时，可以直接从 `adj[u][cur[u]]` 开始遍历。

**无向图的边删除技巧 (`^1` 异或技巧)：**
对于无向图的边 `(u, v)`，我们在邻接表中会添加两条“有向边”：`u -> v` 和 `v -> u`。为了正确标记一条无向边只被访问一次，我们通常这样构建邻接表：
- 当加入边 `(u, v)` 时，假设这是第 `k` 条实际边。
- `adj[u].push_back({v, 2*k});`
- `adj[v].push_back({u, 2*k + 1});`
- 这样，`2*k` 和 `2*k+1` 就代表了同一条无向边的两个方向。
- 当 `dfs(u)` 访问 `(u, v)` 这条边，其 `edge_id` 是 `2*k` 时，我们不仅要标记 `used_edge[2*k] = true;`，还要标记它的反向边 `used_edge[2*k+1] = true;`。利用异或的性质，`2*k ^ 1` 就是 `2*k+1`，反之亦然。所以只需 `used_edge[edge_id] = true; used_edge[edge_id ^ 1] = true;`

**Hierholzer 算法的步骤（以无向图为例）：**
1.  **预处理**：
    *   构建邻接表，计算每个顶点的度数。
    *   检查连通性（从任意非孤立顶点开始DFS/BFS，看是否访问了所有非孤立顶点）。
    *   检查奇度顶点数量（0或2）。
    *   确定起点 `s`：如果有2个奇度顶点， `s` 是其中一个；否则， `s` 是任意非孤立顶点。
2.  **DFS 构建路径**：
    *   初始化一个空路径列表 `path`。
    *   调用 `dfs(s)`。
    *   在 `dfs(u)` 函数中：
        *   使用 `cur[u]` 指针遍历 `adj[u]`。
        *   对于 `adj[u][cur[u]]` 中的边 `(u, v)`，如果 `edge_id` 未被 `used_edge` 标记：
            *   标记 `used_edge[edge_id] = true;` 和 `used_edge[edge_id ^ 1] = true;`
            *   递归调用 `dfs(v);`
            *   **回溯时**：`path.push_back(v);` (这里添加的是 `v`，因为从 `v` 返回到 `u`，意味着 `v` 后面的路径已经处理完了)
        *   当 `for` 循环结束，即 `u` 的所有邻边都已处理完毕时，`path.push_back(u);` (添加 `u` 本身)
3.  **最终处理**：
    *   将 `path` 列表反转，得到正确的欧拉路径/回路。

**时间复杂度：**
- 预处理（计算度数，连通性）：`O(N + M)`，其中 `N` 是顶点数，`M` 是边数。
- Hierholzer 算法：每个顶点和每条边都只被访问常数次。因此也是 `O(N + M)`。
- 总时间复杂度：`O(N + M)`。

### 5. 欧拉路/欧拉回路的完整判断流程：

1.  **构建图**：使用邻接表存储图，同时计算每个顶点的度数（无向图）或入度/出度（有向图）。
2.  **检查连通性**：
    *   从一个非孤立顶点开始DFS/BFS，统计访问到的顶点数。如果访问到的非孤立顶点数不等于图中非孤立顶点总数，则图不连通，不存在欧拉路/回路。
    *   对于有向图，需要检查弱连通性。
3.  **检查度数条件**：
    *   **无向图**：
        *   统计奇度顶点的数量 `odd_cnt`。
        *   如果 `odd_cnt == 0`：存在欧拉回路。
        *   如果 `odd_cnt == 2`：存在欧拉路径。
        *   否则 (`odd_cnt > 2`)：不存在欧拉路/回路。
    *   **有向图**：
        *   统计 `out_degree - in_degree = 1` 的顶点数量 `start_cnt`。
        *   统计 `in_degree - out_degree = 1` 的顶点数量 `end_cnt`。
        *   统计 `in_degree != out_degree` 的顶点数量 `diff_cnt`。
        *   如果 `diff_cnt == 0`：存在欧拉回路。
        *   如果 `start_cnt == 1` 且 `end_cnt == 1` 且 `diff_cnt == 2`：存在欧拉路径。
        *   否则：不存在欧拉路/回路。
4.  **执行Hierholzer算法**：如果满足存在条件，则从合适的起点开始执行Hierholzer算法，找到并输出路径。

## 典型例题精讲

### 例题一：无向图欧拉回路判断与输出

**题目描述：**
给定一个无向图，包含 `n` 个顶点和 `m` 条边。判断该图是否存在欧拉回路。如果存在，输出一条欧拉回路；否则，输出 "No Euler Circuit"。图保证是连通的，且没有自环和重边。

**输入格式：**
第一行包含两个整数 `n, m` (1 <= n <= 1000, 0 <= m <= 5000)。
接下来 `m` 行，每行两个整数 `u, v`，表示顶点 `u` 和 `v` 之间有一条边 (顶点编号从1到 `n`)。

**输出格式：**
如果存在欧拉回路，输出一行，包含欧拉回路的顶点序列，顶点之间用空格隔开。
否则，输出 "No Euler Circuit"。

**思路分析：**
1.  **判断条件**：对于无向图的欧拉回路，需要满足：
    *   图是连通的（题目已保证）。
    *   所有顶点的度数都是偶数。
2.  **数据结构**：
    *   使用邻接表 `adj[N]` 存储图，每个元素 `adj[u]` 是一个 `vector<pair<int, int>>`，存储 `(v, edge_id)`。
    *   `degree[N]` 数组记录每个顶点的度数。
    *   `used[M*2]` 数组标记边是否已访问。`M*2` 是因为每条无向边在邻接表中会存储为两个方向的“有向边”。
    *   `cur[N]` 数组作为邻接表的迭代器，优化DFS效率。
    *   `path` 列表存储欧拉回路。
3.  **算法流程**：
    *   **构建图**：读取 `m` 条边，同时更新 `degree` 数组，并使用 `^1` 技巧构建邻接表。
    *   **检查度数**：遍历所有顶点，如果发现任何一个顶点的度数是奇数，则不存在欧拉回路，直接输出 "No Euler Circuit"。
    *   **DFS查找**：如果所有度数都是偶数，则存在欧拉回路。从顶点1（或任意非孤立顶点）开始调用 `dfs(1)`。
        *   `dfs(u)` 函数：
            *   利用 `cur[u]` 遍历 `adj[u]`。
            *   对于 `adj[u][cur[u]]` 对应的边 `(u, v)` 和 `edge_id`：
                *   如果 `used[edge_id]` 为 `false`：
                    *   标记 `used[edge_id] = true;` 和 `used[edge_id ^ 1] = true;`
                    *   递归调用 `dfs(v);`
            *   当 `u` 的所有邻边都遍历完（或已标记）后，将 `u` 加入 `path` 列表。
    *   **输出结果**：将 `path` 列表反转，然后按顺序输出。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h>

const int N = 1005; // 最大顶点数
const int M = 5005; // 最大边数

std::vector<std::pair<int, int>> adj[N]; // adj[u] = {{v, edge_id}, ...}
int degree[N]; // 存储每个顶点的度数
bool used[M * 2]; // 标记边是否已访问，M*2 是因为每条无向边在邻接表中存为两个方向的边
int cur[N]; // 用于Hierholzer算法优化DFS，记录每个顶点下一条未访问边的索引
std::vector<int> path; // 存储欧拉回路的顶点序列

void dfs(int u) {
    // 遍历顶点 u 的所有邻边
    // cur[u] 记录了从 u 出发的下一条未访问的边
    for (int& i = cur[u]; i < adj[u].size(); ++i) {
        int v = adj[u][i].first;
        int edge_id = adj[u][i].second;

        // 如果这条边已经访问过，跳过
        if (used[edge_id]) {
            continue;
        }

        // 标记这条边及其反向边为已访问
        used[edge_id] = true;
        used[edge_id ^ 1] = true; // 利用异或^1技巧标记无向边的两个方向

        // 递归访问下一个顶点
        dfs(v);
    }
    // 当一个顶点 u 的所有邻边都被访问完毕后，将 u 加入路径
    // 注意：这里是回溯时加入，所以路径是逆序的
    path.push_back(u);
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    int edge_count = 0; // 实际边的计数器
    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        // 添加边 (u, v) 及其反向边 (v, u)
        // edge_count*2 和 edge_count*2+1 作为这对边的唯一ID
        adj[u].push_back({v, edge_count * 2});
        adj[v].push_back({u, edge_count * 2 + 1});
        degree[u]++;
        degree[v]++;
        edge_count++;
    }

    // 检查是否存在欧拉回路的条件：所有顶点的度数必须是偶数
    for (int i = 1; i <= n; ++i) {
        if (degree[i] % 2 != 0) {
            std::cout << "No Euler Circuit" << std::endl;
            return 0;
        }
    }

    // 题目保证图是连通的，所以无需额外检查连通性。
    // 从任意一个有边的顶点开始DFS。这里从顶点1开始。
    // 如果顶点1是孤立点，需要找到第一个非孤立点。
    int start_node = 1;
    while (start_node <= n && degree[start_node] == 0) {
        start_node++;
    }
    if (start_node > n && m > 0) { // 所有点都是孤立点但有边，不应该发生
        std::cout << "No Euler Circuit" << std::endl; // 理论上不会走到这里
        return 0;
    } else if (m == 0) { // 边数为0，空图，可以认为是平凡的欧拉回路
        std::cout << "1" << std::endl; // 或者直接输出空行，取决于题目定义
        return 0;
    }

    dfs(start_node); // 调用DFS构建欧拉回路

    // 检查是否所有边都被访问过
    // 因为题目保证连通，且度数都为偶数，所以 Hierholzer 算法一定能找到回路
    // path.size() 应该是 m + 1 (m条边，m+1个顶点)
    if (path.size() != m + 1) { // 如果路径长度不等于边数+1，说明没有遍历所有边
        std::cout << "No Euler Circuit" << std::endl; // 这种情况通常是图不连通导致的
        return 0;
    }

    // path 是逆序的，需要反转
    std::reverse(path.begin(), path.end());

    // 输出欧拉回路
    for (int i = 0; i < path.size(); ++i) {
        std::cout << path[i] << (i == path.size() - 1 ? "" : " ");
    }
    std::cout << std::endl;

    return 0;
}
```
**复杂度分析：**
*   **时间复杂度**：
    *   读入边并构建邻接表：`O(M)`
    *   计算度数：`O(N)`
    *   检查度数奇偶性：`O(N)`
    *   DFS (Hierholzer 算法)：每个顶点访问一次，每条边（两个方向）访问一次。`O(N + M)`
    *   反转路径：`O(M)`
    *   总时间复杂度：`O(N + M)`
*   **空间复杂度**：
    *   邻接表：`O(N + M)`
    *   度数数组：`O(N)`
    *   `used` 数组：`O(M)`
    *   `cur` 数组：`O(N)`
    *   `path` 列表：`O(M)`
    *   总空间复杂度：`O(N + M)`

### 例题二：有向图欧拉路径判断与输出

**题目描述：**
给定一个有向图，包含 `n` 个顶点和 `m` 条边。判断该图是否存在欧拉路径。如果存在，输出一条欧拉路径；否则，输出 "No Euler Path"。图可能不连通，但没有自环和重边。

**输入格式：**
第一行包含两个整数 `n, m` (1 <= n <= 1000, 0 <= m <= 5000)。
接下来 `m` 行，每行两个整数 `u, v`，表示从顶点 `u` 到顶点 `v` 有一条有向边 (顶点编号从1到 `n`)。

**输出格式：**
如果存在欧拉路径，输出一行，包含欧拉路径的顶点序列，顶点之间用空格隔开。
否则，输出 "No Euler Path"。

**思路分析：**
1.  **判断条件**：对于有向图的欧拉路径，需要满足：
    *   图是弱连通的（所有非孤立顶点都在同一个弱连通分量中）。
    *   恰好有一个顶点 `s` 满足 `out_degree(s) - in_degree(s) = 1`。
    *   恰好有一个顶点 `t` 满足 `in_degree(t) - out_degree(t) = 1`。
    *   所有其他顶点的入度等于出度。
2.  **数据结构**：
    *   邻接表 `adj[N]` 存储图，每个元素 `adj[u]` 是一个 `vector<pair<int, int>>`，存储 `(v, edge_id)`。
    *   `in_degree[N]` 和 `out_degree[N]` 数组记录入度和出度。
    *   `used[M]` 数组标记边是否已访问。`M` 是实际边数。
    *   `cur[N]` 数组作为邻接表的迭代器。
    *   `path` 列表存储欧拉路径。
    *   `visited_nodes[N]` 用于连通性检查。
3.  **算法流程**：
    *   **构建图**：读取 `m` 条边，同时更新 `in_degree` 和 `out_degree` 数组，构建邻接表。
    *   **检查度数**：遍历所有顶点，统计 `start_cnt` (出度比入度多1的顶点数) 和 `end_cnt` (入度比出度多1的顶点数)，以及 `start_node` (欧拉路径起点) 和 `end_node` (欧拉路径终点)。
        *   如果 `start_cnt > 1` 或 `end_cnt > 1`，或者 `start_cnt` 和 `end_cnt` 不相等，则不存在欧拉路径。
        *   如果 `start_cnt == 0 && end_cnt == 0` (所有点入度=出度)，则存在欧拉回路。此时起点可以是任意非孤立点。
        *   如果 `start_cnt == 1 && end_cnt == 1`，则存在欧拉路径，起点为 `start_node`。
        *   否则，不存在欧拉路径。
    *   **检查弱连通性**：
        *   从 `start_node` (或任意非孤立点) 开始，进行一次DFS（忽略边的方向，即当作无向图来遍历），标记所有可达的顶点。
        *   然后遍历所有顶点，如果存在一个非孤立顶点未被访问到，则图不弱连通，不存在欧拉路径。
    *   **DFS查找**：如果满足所有条件，从 `start_node` 开始调用 `dfs(start_node)`。
        *   `dfs(u)` 函数与无向图类似，但只需标记 `used[edge_id] = true;` (因为是有向边，没有反向边)。
    *   **输出结果**：将 `path` 列表反转，然后按顺序输出。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h>

const int N = 1005; // 最大顶点数
const int M = 5005; // 最大边数

std::vector<std::pair<int, int>> adj[N]; // adj[u] = {{v, edge_id}, ...}
int in_degree[N]; // 存储每个顶点的入度
int out_degree[N]; // 存储每个顶点的出度
bool used[M]; // 标记边是否已访问，M是实际边数
int cur[N]; // 用于Hierholzer算法优化DFS，记录每个顶点下一条未访问边的索引
std::vector<int> path; // 存储欧拉路径的顶点序列
bool visited_nodes[N]; // 用于弱连通性检查

// 弱连通性检查的DFS，忽略边的方向
void check_weak_connectivity_dfs(int u) {
    visited_nodes[u] = true;
    for (auto& edge : adj[u]) {
        int v = edge.first;
        if (!visited_nodes[v]) {
            check_weak_connectivity_dfs(v);
        }
    }
    // 对于弱连通性检查，还需要考虑反向边，但这里为了简化，
    // 假设如果从起点能到达所有非孤立点，且度数条件满足，就足够了。
    // 更严谨地，可以构建一个无向图来检查连通性。
    // 在这里，我们简化为从起点能访问所有有度的点。
}


void dfs(int u) {
    for (int& i = cur[u]; i < adj[u].size(); ++i) {
        int v = adj[u][i].first;
        int edge_id = adj[u][i].second;

        if (used[edge_id]) {
            continue;
        }

        used[edge_id] = true; // 标记这条有向边为已访问
        dfs(v);
    }
    path.push_back(u); // 回溯时将顶点加入路径
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        adj[u].push_back({v, i}); // i 作为边的唯一ID
        out_degree[u]++;
        in_degree[v]++;
    }

    if (m == 0) { // 没有边，空图
        if (n >= 1) std::cout << "1" << std::endl; // 只有一个点，没有边，也算平凡路径
        else std::cout << "No Euler Path" << std::endl;
        return 0;
    }

    int start_node = -1;
    int end_node = -1;
    int diff_cnt = 0; // 记录入度不等于出度的顶点数量
    int odd_diff_out = 0; // 记录出度比入度多1的顶点数量
    int odd_diff_in = 0; // 记录入度比出度多1的顶点数量

    for (int i = 1; i <= n; ++i) {
        if (in_degree
