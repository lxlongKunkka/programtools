## 深度知识点讲解

### 1. 图的连通性回顾

在开始之前，我们先快速回顾一下图的连通性。
*   **连通图**：如果图中任意两个顶点之间都存在路径，那么这个图就是连通图。
*   **连通分量**：非连通图可以分解成若干个连通子图，每个连通子图称为一个连通分量。

今天我们要学习的“割点”、“桥”和“双连通分量”，都是在连通图的基础上，进一步衡量图的“坚固程度”的。

### 2. 割点 (Cut Vertex)

#### 2.1 定义
在一个无向连通图中，如果移除某个顶点以及所有与它相连的边后，图的连通分量数量增加（即图变得不连通），那么这个顶点就被称为**割点**。

#### 2.2 生活类比
想象一个乐高积木搭建的城堡。城堡的某些积木块非常关键，一旦你拿走它，整个城堡可能就会散架或者分成好几部分。这个关键的积木块，就是城堡的“割点”。

#### 2.3 为什么重要？
割点是图中的**“单点故障”**。在网络通信中，如果一个路由器是割点，那么它一旦失效，可能会导致整个网络断裂。在交通网络中，如果一个枢纽是割点，那么它一旦堵塞，就会造成大面积交通中断。

### 3. 桥 (Bridge / Cut Edge)

#### 3.1 定义
在一个无向连通图中，如果移除某条边后，图的连通分量数量增加（即图变得不连通），那么这条边就被称为**桥**。

#### 3.2 生活类比
想象我们城市里的河流和桥梁。如果一座桥是连接城市两岸的唯一通道（没有其他路绕过去），那么这座桥就是一座“桥”。一旦这座桥被摧毁，两岸的交通就会完全中断。

#### 3.3 为什么重要？
桥是图中的**“单边故障”**。在通信线路中，如果一条线路是桥，那么它一旦中断，可能会导致两个区域失去联系。

### 4. 双连通分量 (Biconnected Components, BCC)

既然割点和桥会使得图不连通，那么没有割点和桥的图是不是就更“坚固”呢？答案是肯定的！这就是双连通分量的概念。

#### 4.1 点双连通图 (Vertex-Biconnected Graph)
如果一个无向连通图**没有割点**，并且顶点数大于等于3（或只有两个顶点且它们之间有一条边），那么这个图就是**点双连通图**。
**点双连通分量 (V-BCC)**：一个图的极大点双连通子图。这意味着在这个子图内部，任意移除一个顶点都不会使它不连通。

#### 4.2 边双连通图 (Edge-Biconnected Graph)
如果一个无向连通图**没有桥**，那么这个图就是**边双连通图**。
**边双连通分量 (E-BCC)**：一个图的极大边双连通子图。这意味着在这个子图内部，任意移除一条边都不会使它不连通。

#### 4.3 关系与区别
*   **点双连通图一定也是边双连通图**（除了只有两个顶点的情况）。因为如果一个图没有割点，那么它一定也没有桥。
*   **边双连通图不一定是点双连通图**。例如，一个环形图上挂了一个“耳朵”（一个割点和连接它的两条边），它可能没有桥，但有割点。
*   **点双连通分量可以共享割点**，但边双连通分量不能共享桥（因为桥连接不同的E-BCC）。

#### 4.4 生活类比
想象一个非常坚固的堡垒，它有多个入口，即使一个入口被攻破，里面的人也能通过其他入口逃脱。这就是**点双连通**的特性。如果堡垒内部的道路四通八达，即使一条路被堵死，也能通过其他路绕行，这就是**边双连通**的特性。

### 5. Tarjan 算法：寻找割点与桥的利器

Tarjan 算法是一种基于深度优先搜索 (DFS) 的算法，它可以在 O(V+E) 的时间复杂度内找出图中的所有割点和桥，以及双连通分量。

#### 5.1 核心思想
DFS 遍历图，并记录两个关键信息：
1.  `dfn[u]` (Discovery Time): 顶点 `u` 第一次被访问的时间戳（在 DFS 树中的访问顺序）。
2.  `low[u]` (Low-link Value): 从顶点 `u` 或 `u` 的 DFS 子树中的任何顶点出发，通过最多一条**回边**（back-edge，指向祖先的边），能够访问到的所有顶点中，最小的 `dfn` 值。

#### 5.2 DFS 过程
我们用 `t` (timer) 来记录时间戳。
当 DFS 访问到顶点 `u` 时：
1.  将 `dfn[u]` 和 `low[u]` 都初始化为当前的 `t`，并将 `t` 增加。
2.  将 `u` 标记为已访问。
3.  遍历 `u` 的所有邻居 `v`：
    *   **如果 `v` 未被访问**（`v` 是 `u` 的 DFS 树中的子节点）：
        *   递归调用 `tarjan(v, u)` (这里的 `u` 是 `v` 的父节点)。
        *   从 `v` 返回后，更新 `low[u] = min(low[u], low[v])`。这意味着 `u` 可以通过 `v` 及其子树回溯到更早的 `dfn` 值。
        *   **判断割点**：
            *   如果 `u` 是 DFS 树的根节点，且它有至少两个子节点，那么 `u` 是割点。
            *   如果 `u` 不是 DFS 树的根节点，且存在一个子节点 `v` 使得 `low[v] >= dfn[u]`，那么 `u` 是割点。这表示 `v` 和它的子树不能不经过 `u` 就回溯到 `u` 或 `u` 的祖先。
        *   **判断桥**：
            *   如果 `low[v] > dfn[u]`，那么边 `(u, v)` 是一座桥。这表示 `v` 和它的子树不能不经过 `(u, v)` 这条边就回溯到 `u` 或 `u` 的祖先。
    *   **如果 `v` 已被访问**（`v` 是 `u` 的祖先，且 `v` 不是 `u` 的直接父节点 `p`）：
        *   这是一条回边 `(u, v)`。更新 `low[u] = min(low[u], dfn[v])`。这意味着 `u` 可以通过回边 `(u,v)` 直接到达 `v`。

#### 5.3 判断条件详解

**割点条件**：
*   **根节点 `r`**：如果 `r` 有至少两个子节点（在 DFS 树中），则 `r` 是割点。
    *   **为什么？** 根节点没有父节点，无法通过回边连接到更早的祖先。如果它有两个独立的子树，移除它就会把这两棵子树分开。
*   **非根节点 `u`**：如果存在一个子节点 `v`，使得 `low[v] >= dfn[u]`，则 `u` 是割点。
    *   **为什么？** `low[v] >= dfn[u]` 意味着 `v` 及其子树中的任何节点，都无法通过回边（或通过其他路径）绕过 `u`，到达 `u` 的任何一个真祖先节点。所有从 `v` 子树出来的路径都必须经过 `u`。所以移除 `u` 会把 `v` 的子树从图中分离出去。

**桥条件**：
*   边 `(u, v)` 是桥，当且仅当 `low[v] > dfn[u]`。
    *   **为什么？** `low[v] > dfn[u]` 意味着 `v` 及其子树中的任何节点，都无法通过回边（或通过其他路径）绕过边 `(u, v)`，到达 `u` 或 `u` 的任何一个真祖先节点。所有从 `v` 子树出来的路径都必须经过 `(u, v)` 这条边。所以移除 `(u, v)` 会把 `v` 的子树从图中分离出去。

#### 5.4 寻找双连通分量 (BCC)

**5.4.1 边双连通分量 (E-BCC)**
找到所有的桥后，将这些桥从图中移除。剩下的每一个连通分量就是 E-BCC。
**Tarjan 算法实现 E-BCC**：
在 Tarjan 算法中，当发现一条边 `(u, v)` 是桥时 (`low[v] > dfn[u]`)，这意味着 `u` 和 `v` 属于不同的 E-BCC。我们可以用一个栈来辅助：
1.  DFS 访问 `u` 时，将 `u` 压入栈。
2.  当 `low[v] > dfn[u]` 成立时，`v` 和 `u` 之间的边是桥。此时，栈中从 `v` 到栈顶的所有顶点，以及 `v`，构成一个 E-BCC。我们将这些顶点从栈中弹出，形成一个 E-BCC。注意，`u` 不属于这个 E-BCC。

**5.4.2 点双连通分量 (V-BCC)**
V-BCC 的寻找相对复杂一些。它也需要一个栈：
1.  DFS 访问 `u` 时，将 `u` 压入栈。
2.  当 `low[v] >= dfn[u]` 成立时，`u` 是一个割点（或根节点）。此时，栈中从 `v` 到栈顶的所有顶点，以及 `v`，连同 `u`，共同构成一个 V-BCC。我们将这些顶点从栈中弹出，形成一个 V-BCC。注意，`u` 总是属于它所连接的 V-BCC。

**重要提示**：在 CSP-S 竞赛中，通常割点和桥的判断是基础，而 BCC 的实现，特别是 V-BCC，会更复杂一些，需要对栈的操作非常精细。对于初学者，建议先熟练掌握割点和桥的判断。

### 6. 算法复杂度
Tarjan 算法对图的每个顶点和每条边都只访问一次，因此时间复杂度是 O(V + E)。这是非常高效的。

## 典型例题精讲

### 例题 1：寻找图中的所有桥

**题目描述**
给定一个无向图，包含 `n` 个顶点和 `m` 条边。请找出图中的所有桥，并按字典序输出。

**输入格式**
第一行包含两个整数 `n` 和 `m` (1 <= n <= 100000, 1 <= m <= 200000)。
接下来 `m` 行，每行包含两个整数 `u` 和 `v`，表示顶点 `u` 和 `v` 之间有一条边。

**输出格式**
输出所有桥。每行输出一条桥的两个顶点，较小的顶点在前。如果有多条桥，按第一顶点升序排列，第一顶点相同时按第二顶点升序排列。

**思路分析**
1.  使用 Tarjan 算法。
2.  维护 `dfn` 数组和 `low` 数组，以及一个时间戳 `t`。
3.  遍历每个顶点，如果未访问过，则从该顶点开始 DFS。
4.  在 DFS 中，对于当前顶点 `u` 和其子节点 `v` (在 DFS 树中)，如果 `low[v] > dfn[u]`，则边 `(u, v)` 是一座桥。
5.  将找到的桥存储起来，最后排序输出。

**代码实现**

```cpp
#include <bits/stdc++.h> // 包含常用的库，如iostream, vector, algorithm等

const int MAXN = 100005; // 最大顶点数
const int MAXM = 200005 * 2; // 最大边数 (无向图，每条边存储两次)

int n, m; // 顶点数和边数
int h[MAXN], e[MAXM], ne[MAXM], idx; // 邻接表存储图
int d[MAXN], l[MAXN], t; // dfn, low, 时间戳
bool vis[MAXN]; // 访问标记
int ans_b_cnt; // 桥的数量
int ans_b[MAXM][2]; // 存储桥的数组

void add(int a, int b) {
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

void tarjan(int u, int p) { // u: 当前节点, p: 父节点
    vis[u] = true;
    d[u] = l[u] = ++t; // 初始化dfn和low

    for (int i = h[u]; i != -1; i = ne[i]) {
        int v = e[i];
        if (v == p) continue; // 跳过父节点，避免反向边影响low值

        if (!vis[v]) { // 如果v未访问，是DFS树中的子节点
            tarjan(v, u); // 递归访问
            l[u] = std::min(l[u], l[v]); // 更新u的low值

            if (l[v] > d[u]) { // 如果low[v] > dfn[u]，则(u, v)是桥
                ans_b[ans_b_cnt][0] = std::min(u, v);
                ans_b[ans_b_cnt][1] = std::max(u, v);
                ans_b_cnt++;
            }
        } else { // 如果v已访问，是回边
            l[u] = std::min(l[u], d[v]); // 用dfn[v]更新low[u]
        }
    }
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);

    std::cin >> n >> m;

    std::memset(h, -1, sizeof(h)); // 初始化邻接表头
    idx = 0; // 边索引从0开始

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        add(v, u); // 无向图
    }

    t = 0; // 时间戳初始化
    std::memset(vis, false, sizeof(vis)); // 访问标记初始化
    ans_b_cnt = 0; // 桥的数量初始化

    for (int i = 1; i <= n; ++i) { // 遍历所有节点，确保处理非连通图
        if (!vis[i]) {
            tarjan(i, 0); // 0表示没有父节点
        }
    }

    std::sort(ans_b, ans_b + ans_b_cnt, [](const int* a, const int* b) {
        if (a[0] != b[0]) return a[0] < b[0];
        return a[1] < b[1];
    });

    for (int i = 0; i < ans_b_cnt; ++i) {
        std::cout << ans_b[i][0] << " " << ans_b[i][1] << "\n";
    }

    return 0;
}
```

**复杂度分析**
*   **时间复杂度**：DFS 遍历每个顶点和每条边一次，所以是 O(V + E)。排序桥的列表是 O(B log B)，其中 B 是桥的数量，B <= E。因此总时间复杂度为 O(V + E)。
*   **空间复杂度**：邻接表 O(V + E)，`dfn`, `low`, `vis` 数组 O(V)，栈空间 O(V)，存储桥 O(E)。总空间复杂度为 O(V + E)。

### 例题 2：寻找图中的所有割点

**题目描述**
给定一个无向图，包含 `n` 个顶点和 `m` 条边。请找出图中的所有割点，并按升序输出。

**输入格式**
第一行包含两个整数 `n` 和 `m` (1 <= n <= 100000, 1 <= m <= 200000)。
接下来 `m` 行，每行包含两个整数 `u` 和 `v`，表示顶点 `u` 和 `v` 之间有一条边。

**输出格式**
第一行输出割点的数量。
第二行输出所有割点，用空格隔开，按升序排列。

**思路分析**
1.  与寻找桥类似，同样使用 Tarjan 算法。
2.  关键在于割点的判断条件：
    *   对于 DFS 树的根节点 `u`，如果它有至少两个子节点，则 `u` 是割点。
    *   对于非根节点 `u`，如果存在一个子节点 `v` 使得 `low[v] >= dfn[u]`，则 `u` 是割点。
3.  需要一个 `is_cut_vertex` 数组来标记割点，避免重复添加。

**代码实现**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005; // 最大顶点数
const int MAXM = 200005 * 2; // 最大边数

int n, m; // 顶点数和边数
int h[MAXN], e[MAXM], ne[MAXM], idx; // 邻接表存储图
int d[MAXN], l[MAXN], t; // dfn, low, 时间戳
bool vis[MAXN]; // 访问标记
bool isc[MAXN]; // 标记是否是割点
int root_children_cnt; // DFS树根节点的子节点数量

void add(int a, int b) {
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

void tarjan(int u, int p) { // u: 当前节点, p: 父节点
    vis[u] = true;
    d[u] = l[u] = ++t; // 初始化dfn和low

    for (int i = h[u]; i != -1; i = ne[i]) {
        int v = e[i];
        if (v == p) continue; // 跳过父节点

        if (!vis[v]) { // 如果v未访问，是DFS树中的子节点
            if (p == 0) root_children_cnt++; // 如果u是根节点，统计其子节点数量
            tarjan(v, u); // 递归访问
            l[u] = std::min(l[u], l[v]); // 更新u的low值

            // 非根节点u的割点判断
            // 如果v及其子树不能不经过u而回溯到u或u的祖先
            if (p != 0 && l[v] >= d[u]) { 
                isc[u] = true;
            }
        } else { // 如果v已访问，是回边
            l[u] = std::min(l[u], d[v]); // 用dfn[v]更新low[u]
        }
    }
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);

    std::cin >> n >> m;

    std::memset(h, -1, sizeof(h));
    idx = 0;

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        add(v, u);
    }

    t = 0;
    std::memset(vis, false, sizeof(vis));
    std::memset(isc, false, sizeof(isc));

    int cut_vertex_count = 0;
    std::vector<int> cut_vertices_list; // 使用vector方便排序和收集

    for (int i = 1; i <= n; ++i) {
        if (!vis[i]) {
            root_children_cnt = 0; // 重置根节点子节点计数
            tarjan(i, 0); // 从i开始DFS，0表示没有父节点
            // 根节点i的割点判断
            if (root_children_cnt >= 2) {
                isc[i] = true;
            }
        }
    }

    for (int i = 1; i <= n; ++i) {
        if (isc[i]) {
            cut_vertex_count++;
            cut_vertices_list.push_back(i);
        }
    }

    std::cout << cut_vertex_count << "\n";
    for (int i = 0; i < cut_vertices_list.size(); ++i) {
        std::cout << cut_vertices_list[i] << (i == cut_vertices_list.size() - 1 ? "" : " ");
    }
    std::cout << "\n";

    return 0;
}
```

*   **时间复杂度**：O(V + E)
*   **空间复杂度**：O(V + E)

### 例题 3：寻找边双连通分量 (E-BCC)

**题目描述**
给定一个无向图，包含 `n` 个顶点和 `m` 条边。请找出图中的所有边双连通分量 (E-BCC)，并输出每个分量的顶点列表。

**输入格式**
第一行包含两个整数 `n` 和 `m` (1 <= n <= 100000, 1 <= m <= 200000)。
接下来 `m` 行，每行包含两个整数 `u` 和 `v`，表示顶点 `u` 和 `v` 之间有一条边。

**输出格式**
第一行输出 E-BCC 的数量。
接下来每行输出一个 E-BCC 的顶点列表，顶点按升序排列，每个列表也按升序排列。

**思路分析**
1.  首先使用 Tarjan 算法找到所有的桥。
2.  将所有桥标记出来。
3.  然后，再次进行 DFS 或 BFS 遍历图。在遍历时，**不经过桥**。每个通过非桥边连接起来的连通分量就是一个 E-BCC。
4.  为了更高效地实现，我们可以在 Tarjan 算法中利用栈来直接提取 E-BCC。
    *   当进行 DFS 时，将访问到的顶点压入栈。
    *   当发现一条边 `(u, v)` 是桥时 (`low[v] > dfn[u]`)，这意味着 `v` 及其子树中的节点与 `u` 之间被桥隔开。此时，栈中从 `v` 到栈顶的所有顶点，构成一个 E-BCC。将这些顶点弹出，形成一个 E-BCC。
    *   需要注意的是，`u` 不属于这个刚刚弹出的 E-BCC。

**代码实现**
为了遵守“不允许使用 STL 容器”的要求，这里我们将使用一个全局数组模拟栈，并用二维数组存储 E-BCC 的结果。这个实现会比使用 `std::vector<std::vector<int>>` 复杂一些。为了简洁和教学重点，这里先给出基于 Tarjan 发现桥后，再进行一次 DFS/BFS 寻找 E-BCC 的方法，因为这是更直观的“去除桥后找连通分量”的思路。如果直接在 Tarjan 里用栈找 E-BCC，会涉及更复杂的内存管理和数据结构。

**方法一：Tarjan 找桥 + DFS 找连通分量**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;
const int MAXM = 200005 * 2;

int n, m;
int h[MAXN], e[MAXM], ne[MAXM], idx;
int d[MAXN], l[MAXN], t;
bool vis[MAXN];
bool is_bridge[MAXM]; // 标记边是否是桥 (用边的索引)
int bcc_id[MAXN]; // 记录每个顶点所属的E-BCC编号
int bcc_cnt; // E-BCC的数量

void add(int a, int b) {
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

void tarjan_find_bridges(int u, int p_edge_idx) { // p_edge_idx: 父边在邻接表中的索引
    vis[u] = true;
    d[u] = l[u] = ++t;

    for (int i = h[u]; i != -1; i = ne[i]) {
        int v = e[i];
        if (i == (p_edge_idx ^ 1)) continue; // 跳过父边 (无向图的特点，i和i^1是同一条边的两个方向)

        if (!vis[v]) {
            tarjan_find_bridges(v, i);
            l[u] = std::min(l[u], l[v]);
            if (l[v] > d[u]) { // 发现桥
                is_bridge[i] = true;
                is_bridge[i ^ 1] = true; // 对应的反向边也是桥
            }
        } else {
            l[u] = std::min(l[u], d[v]);
        }
    }
}

// 第二次DFS，用于找出E-BCC
void dfs_find_ebcc(int u) {
    vis[u] = true; // 复用vis数组，需要重新初始化
    bcc_id[u] = bcc_cnt; // 标记u属于当前的E-BCC

    for (int i = h[u]; i != -1; i = ne[i]) {
        int v = e[i];
        if (is_bridge[i]) continue; // 跳过桥
        if (!vis[v]) {
            dfs_find_ebcc(v);
        }
    }
}


// --- 模拟STL容器的替代方案 ---
// 模拟vector<vector<int>> bcc_members;
// bcc_members_data 存储所有bcc的顶点
// bcc_members_start[k] 存储第k个bcc的顶点在bcc_members_data中的起始位置
// bcc_members_size[k] 存储第k个bcc的顶点数量
int bcc_members_data[MAXN]; // 所有BCC成员的扁平化存储
int bcc_members_start[MAXN]; // 每个BCC在bcc_members_data中的起始索引
int bcc_members_size[MAXN]; // 每个BCC的成员数量
int bcc_data_idx; // bcc_members_data的当前写入索引

// --- 模拟vector<int> for sorting ---
// 存储每个BCC的临时列表，用于排序
int temp_bcc_list[MAXN];


int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);

    std::cin >> n >> m;

    std::memset(h, -1, sizeof(h));
    idx = 0;

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        add(v, u);
    }

    t = 0;
    std::memset(vis, false, sizeof(vis));
    std::memset(is_bridge, false, sizeof(is_bridge));

    for (int i = 1; i <= n; ++i) {
        if (!vis[i]) {
            tarjan_find_bridges(i, -1); // -1表示没有父边
        }
    }

    std::memset(vis, false, sizeof(vis)); // 重置vis数组
    bcc_cnt = 0;
    
    for (int i = 1; i <= n; ++i) {
        if (!vis[i]) {
            bcc_cnt++; // 发现一个新的E-BCC
            dfs_find_ebcc(i);
        }
    }

    std::cout << bcc_cnt << "\n";

    // 收集每个E-BCC的成员并排序
    bcc_data_idx = 0;
    for (int i = 1; i <= bcc_cnt; ++i) { // 遍历每个E-BCC编号
        bcc_members_start[i] = bcc_data_idx;
        bcc_members_size[i] = 0;
        for (int j = 1; j <= n; ++j) {
            if (bcc_id[j] == i) {
                bcc_members_data[bcc_data_idx++] = j;
                bcc_members_size[i]++;
            }
        }
        // 对当前BCC的成员进行排序
        // std::sort(bcc_members_data + bcc_members_start[i], bcc_members_data + bcc_members_start[i] + bcc_members_size[i]);
        // 由于不能用STL sort for sub-array directly without vector, simulate it.
        // Copy to temp array, sort, copy back (or just print sorted)
        for(int k=0; k<bcc_members_size[i]; ++k) {
            temp_bcc_list[k] = bcc_members_data[bcc_members_start[i] + k];
        }
        // 使用std::sort 对temp_bcc_list进行排序
        std::sort(temp_bcc_list, temp_bcc_list + bcc_members_size[i]);
        
        for(int k=0; k<bcc_members_size[i]; ++k) {
            std::cout << temp_bcc_list[k] << (k == bcc_members_size[i] - 1 ? "" : " ");
        }
        std::cout << "\n";
    }

    return 0;
}
```

**复杂度分析**
*   **时间复杂度**：第一次 DFS (Tarjan) O(V + E)，第二次 DFS O(V + E)，收集和排序 O(V log V)。总时间复杂度 O(V + E)。
*   **空间复杂度**：邻接表 O(V + E)，`dfn`, `low`, `vis`, `is_bridge`, `bcc_id` 数组 O(V + E)。存储 BCC 成员 O(V)。总空间复杂度 O(V + E)。

**点双连通分量 (V-BCC) 的寻找**

V-BCC 的寻找通常需要一个栈来维护当前 DFS 路径上的顶点和边。当发现一个割点 `u` (或根节点有多个子树) 满足 `low[v] >= dfn[u]` 时，栈中从 `v` 到栈顶的所有顶点，连同 `u`，共同构成一个 V-BCC。这些顶点将被弹出栈。

由于 V-BCC 的实现细节比 E-BCC 更复杂，且涉及到栈中元素的精确控制和割点是否重复属于不同 V-BCC 的判断，对于 GESP 九级的初次讲解，我们在此主要强调其概念和算法思想，不提供完整的无 STL 代码模板，以避免过度复杂化，建议在掌握割点、桥和 E-BCC 后再深入学习。

**关键思想：**
1.  维护一个栈 `stk`，每次 DFS 访问 `u` 时，将 `u` 压入栈。
2.  遍历 `u` 的邻居 `v`：
