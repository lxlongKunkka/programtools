## 趣味引入

同学们，还记得我们之前学过的欧几里得算法（辗转相除法）吗？它就像一个“最大公约数探测器”，能够快速帮我们找到两个数的最大公约数 `gcd(a, b)`。比如 `gcd(24, 18)` 是 `6`。

但是，如果我们想再进一步，问一个更有趣的问题：有没有可能找到两个整数 `x` 和 `y`，使得 `24 * x + 18 * y = 6` 呢？或者更一般地，对于任意两个整数 `a` 和 `b`，我们能不能找到 `x` 和 `y`，使得 `a * x + b * y = gcd(a, b)` 呢？

想象一下，你是一个魔法药剂师，需要精确地量出 `gcd(a, b)` 毫升的药水。但是你手边只有两个“魔法量杯”，一个容量是 `a` 毫升，另一个容量是 `b` 毫升。你既可以往瓶子里倒满 `a` 毫升或 `b` 毫升（对应 `x` 或 `y` 为正），也可以从瓶子里取出 `a` 毫升或 `b` 毫升（对应 `x` 或 `y` 为负）。你能不能通过反复使用这两个量杯（倒进倒出），最终精确地得到 `gcd(a, b)` 毫升的药水呢？如果能，你是怎么操作的？

今天，我们就来揭开这个“魔法操作”的秘密，学习一个强大的工具——**扩展欧几里得算法**，以及它背后重要的数学原理——**裴蜀定理**！

## 深度知识点讲解

### 1. 欧几里得算法回顾

在深入学习扩展欧几里得算法之前，我们先快速回顾一下普通的欧几里得算法，它是扩展版本的基石。

欧几里得算法的核心思想是：两个正整数 `a` 和 `b` 的最大公约数等于 `b` 和 `a` 除以 `b` 的余数 `a % b` 的最大公约数。
即：`gcd(a, b) = gcd(b, a % b)`。
递归的终止条件是当 `b = 0` 时，`gcd(a, 0) = a`。

**生活类比：** 想象你在玩一个游戏，目标是把两堆石子（数量分别为 `a` 和 `b`）变成一样多，并且尽可能多。每次操作，你只能从多的那堆里取出和少的那堆一样多的石子。直到两堆石子一样多，或者其中一堆变成空的，另一堆就是最大公约数。

### 2. 裴蜀定理 (Bezout's Identity)

**定理内容：**
对于任意不全为零的两个整数 `a` 和 `b`，存在一对整数 `x` 和 `y`，使得 `ax + by = gcd(a, b)`。
并且，`gcd(a, b)` 是 `a` 和 `b` 的所有形如 `ax + by`（`x, y` 为整数）的线性组合中，最小的正整数。

**为什么是这样？**
我们可以从欧几里得算法的角度来理解。
假设 `g = gcd(a, b)`。
1.  首先，`g` 肯定是 `a` 的约数，也是 `b` 的约数。那么 `ax + by` 肯定能被 `g` 整除（因为 `g | ax` 且 `g | by`）。所以 `ax + by` 的任何结果都是 `g` 的倍数。这意味着 `ax + by` 的最小正值不小于 `g`。
2.  裴蜀定理告诉我们，这个最小的正值**恰好就是** `g`。也就是说，我们一定能找到 `x, y` 使得 `ax + by` 的结果正好是 `g`。

**推论：**
方程 `ax + by = c` 有整数解的充要条件是 `c` 是 `gcd(a, b)` 的倍数。
**解释：** 根据裴蜀定理，`ax + by` 能够表示出的所有值都是 `gcd(a, b)` 的倍数。所以如果 `c` 不是 `gcd(a, b)` 的倍数，那么 `ax + by = c` 就不可能有整数解。反之，如果 `c` 是 `gcd(a, b)` 的倍数，设 `c = k * gcd(a, b)`。我们知道存在 `x0, y0` 使得 `ax0 + by0 = gcd(a, b)`。那么将等式两边同乘以 `k`，就有 `a(kx0) + b(ky0) = k * gcd(a, b) = c`。所以 `x = kx0, y = ky0` 就是一组整数解。

**生活类比：**
你只有面值为 3 元和 5 元的硬币。你能凑出哪些金额？
`gcd(3, 5) = 1`。根据裴蜀定理，你可以凑出任何 `1` 的倍数，也就是任何正整数金额！
例如：
`3 * 2 + 5 * (-1) = 6 - 5 = 1`
`3 * (-1) + 5 * 1 = -3 + 5 = 2`
`3 * 1 + 5 * 0 = 3`
`3 * 0 + 5 * 1 = 5`
这表明，如果你能用 `a` 和 `b` 凑出 `gcd(a, b)`，你就能凑出 `gcd(a, b)` 的所有倍数。

### 3. 扩展欧几里得算法 (Extended Euclidean Algorithm)

既然裴蜀定理告诉我们 `x, y` 一定存在，那我们怎么找到它们呢？这就是扩展欧几里得算法的任务！

**核心思想：**
扩展欧几里得算法是在普通欧几里得算法的基础上，通过递归回溯的方式，一步步推导出 `x` 和 `y` 的值。它利用了这样一个事实：
`gcd(a, b) = gcd(b, a % b)`。
同时，如果我们在子问题 `gcd(b, a % b)` 中找到了 `x'` 和 `y'`，使得 `bx' + (a % b)y' = gcd(b, a % b)`，那么我们就可以利用 `x'` 和 `y'` 来推导出原问题 `ax + by = gcd(a, b)` 的 `x` 和 `y`。

**推导过程：**
1.  **假设：** 我们已经递归地解决了子问题 `gcd(b, a % b)`，并且找到了对应的 `x'` 和 `y'`，使得：
    `bx' + (a % b)y' = gcd(b, a % b)`  --- (式1)
2.  **已知：** 根据欧几里得算法，`gcd(a, b) = gcd(b, a % b)`。
    所以，我们希望找到 `x, y` 使得：
    `ax + by = gcd(a, b)` --- (式2)
3.  **联立：** 将 (式1) 和 (式2) 的右侧相等，得到：
    `ax + by = bx' + (a % b)y'`
4.  **替换 `a % b`：** 我们知道 `a % b` 可以表示为 `a - (a / b) * b` (这里 `/` 是整数除法)。
    将 `a % b` 代入上式：
    `ax + by = bx' + (a - (a / b) * b)y'`
5.  **展开并整理：**
    `ax + by = bx' + ay' - (a / b) * by'`
    `ax + by = ay' + b(x' - (a / b) * y')`
6.  **系数比较：** 比较等式两边的 `a` 和 `b` 的系数：
    `x = y'`
    `y = x' - (a / b) * y'`

**递归基 (Base Case)：**
当 `b = 0` 时，`gcd(a, 0) = a`。
此时方程变为 `ax + 0y = a`。
显然，我们可以取 `x = 1, y = 0` (或者 `y` 为任意整数，但为了简化通常取 `0`)。
所以，当递归到 `b = 0` 时，我们返回 `a` 作为 `gcd`，并且设置 `x = 1, y = 0`。

**生活类比：倒着推火车**
想象你有一列火车，你想把它从终点站推回起点站。扩展欧几里得算法就是这样：
*   **起点：** 你想找到 `ax + by = gcd(a, b)` 的 `x, y`。
*   **前进（递归）：** 你把问题转换为更小的 `bx' + (a % b)y' = gcd(b, a % b)`。
*   **终点（递归基）：** 当 `b = 0` 时，问题变得非常简单：`ax + 0y = a`，此时 `x = 1, y = 0`。这就是火车的终点站，你知道怎么处理了。
*   **后退（回溯）：** 从终点站开始，你一步步根据 `x = y'` 和 `y = x' - (a / b) * y'` 的关系，把 `x', y'` 转换为 `x, y`，就像把火车一节节地推回起点一样，直到你回到了最初的问题，并得到了答案。

**`x, y` 解的通式：**
扩展欧几里得算法找到的 `x, y` 只是方程 `ax + by = gcd(a, b)` 的一组特解。
它的通解是：
`x = x0 + k * (b / gcd(a, b))`
`y = y0 - k * (a / gcd(a, b))`
其中 `x0, y0` 是算法找到的一组特解，`k` 是任意整数。

**时间复杂度：**
扩展欧几里得算法的递归层数与普通欧几里得算法相同，都是 `O(log(min(a, b)))`。因此，它的时间复杂度也是 `O(log(min(a, b)))`。

### 4. 扩展欧几里得算法的应用

#### 应用1: 求解线性丢番图方程 `ax + by = c`

**问题：** 给出整数 `a, b, c`，求一组整数解 `x, y` 使得 `ax + by = c`。
**步骤：**
1.  首先计算 `g = gcd(a, b)`。
2.  **判断是否有解：** 如果 `c` 不能被 `g` 整除（即 `c % g != 0`），则方程无整数解。
3.  **求解特解：** 如果 `c % g == 0`，则方程有解。
    *   将原方程两边同除以 `g`：`(a/g)x + (b/g)y = (c/g)`。
    *   现在我们要求解 `(a/g)x + (b/g)y = 1`。注意，此时 `gcd(a/g, b/g) = 1`。
    *   使用扩展欧几里得算法求解 `(a/g)x0 + (b/g)y0 = 1`，得到一组特解 `x0, y0`。
    *   那么原方程 `ax + by = c` 的一组特解就是 `x = x0 * (c/g)`，`y = y0 * (c/g)`。
4.  **调整解的范围 (如果需要)：** 有时题目会要求 `x` 在某个特定范围内（例如最小正整数解）。
    *   `x` 的通解是 `x = x_particular + k * (b / g)`。
    *   为了找到最小正整数 `x`，我们可以让 `x = (x_particular % (b/g) + (b/g)) % (b/g)`。
    *   然后根据新的 `x` 值计算对应的 `y` 值：`y = (c - ax) / b`。

#### 应用2: 求解模逆元 `ax ≡ 1 (mod m)`

**问题：** 给出整数 `a` 和 `m`，求 `x` 使得 `ax ≡ 1 (mod m)`。这里的 `x` 称为 `a` 在模 `m` 意义下的逆元。
**条件：** 模逆元存在的充要条件是 `gcd(a, m) = 1`。
**步骤：**
1.  将同余方程 `ax ≡ 1 (mod m)` 转换为线性丢番图方程：
    `ax + my = 1` (其中 `y` 是某个整数，表示 `(ax - 1)` 是 `m` 的倍数)。
2.  检查 `gcd(a, m)` 是否为 `1`。如果不是，则模逆元不存在。
3.  如果 `gcd(a, m) = 1`，则使用扩展欧几里得算法求解 `ax + my = 1`，得到一组特解 `x, y`。
4.  此时得到的 `x` 就是一个模逆元。由于模逆元通常要求在 `[0, m-1]` 范围内，需要进行调整：
    `x = (x % m + m) % m`。

**重要提示：** `x` 可能为负数，所以 `(x % m)` 结果也可能为负数（C++中的 `%` 运算符对负数行为特殊），因此通常写成 `(x % m + m) % m` 来确保结果是正的且在 `[0, m-1]` 范围内。

## 典型例题精讲

### 例题1: 求解 `ax + by = gcd(a, b)`

**题目描述：**
给定两个正整数 `a` 和 `b`，请你利用扩展欧几里得算法求出 `gcd(a, b)`，以及一组整数 `x, y` 使得 `ax + by = gcd(a, b)`。

**输入：**
两个整数 `a, b` (1 <= a, b <= 10^9)

**输出：**
一行，包含三个整数 `x, y, gcd(a, b)`，用空格隔开。

**思路分析：**
直接套用扩展欧几里得算法的递归推导过程。
1.  定义一个 `exgcd` 函数，接收 `a, b`，并用引用传递 `x, y` 来更新它们的值。
2.  **递归基：** 当 `b = 0` 时，`gcd(a, 0) = a`。此时 `x = 1, y = 0`。返回 `a`。
3.  **递归过程：**
    *   递归调用 `exgcd(b, a % b, x', y')`，得到子问题的 `gcd` (它就是原问题的 `gcd`) 和 `x', y'`。
    *   利用推导出的公式更新 `x` 和 `y`：
        `x = y'`
        `y = x' - (a / b) * y'`
    *   返回 `gcd`。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// 扩展欧几里得算法
// 返回 gcd(a, b)
// 同时通过引用参数 x, y 返回一组特解，使得 ax + by = gcd(a, b)
long long exgcd(long long a, long long b, long long &x, long long &y) {
    // 递归终止条件：当 b 为 0 时
    if (b == 0) {
        x = 1; // 此时 ax + 0y = a，所以 x = 1, y = 0
        y = 0;
        return a; // gcd(a, 0) = a
    }

    // 递归调用：解决子问题 gcd(b, a % b)
    // 假设我们已经得到了子问题的解 x_prime, y_prime
    long long g = exgcd(b, a % b, y, x); // 注意这里 x 和 y 的顺序，是为了方便后面直接赋值

    // 根据子问题的解 (x_prime, y_prime) 更新当前问题的解 (x, y)
    // 原始推导是：x = y_prime, y = x_prime - (a / b) * y_prime
    // 由于我们在递归调用时，将 y 传给了 x_prime 的位置，x 传给了 y_prime 的位置
    // 所以现在 y 存储的是原来的 x_prime, x 存储的是原来的 y_prime
    // 那么：
    // new_x = current_y_from_recursion
    // new_y = current_x_from_recursion - (a / b) * current_y_from_recursion
    // 所以，我们直接计算 y 的值即可，x 已经在上一行被更新为 y_prime 了
    y = y - (a / b) * x;

    return g; // 返回最大公约数
}

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出
    std::cin.tie(NULL);

    long long a, b;
    std::cin >> a >> b;

    long long x, y;
    long long g = exgcd(a, b, x, y);

    std::cout << x << " " << y << " " << g << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** `O(log(min(a, b)))`，与欧几里得算法相同，因为递归调用的次数与辗转相除的步数一致。
*   **空间复杂度：** `O(log(min(a, b)))`，主要来自递归栈的深度。

### 例题2: 求解线性丢番图方程 `ax + by = c`

**题目描述：**
给定三个整数 `a, b, c` (1 <= a, b, c <= 10^9)，求解方程 `ax + by = c` 的一组整数解 `x, y`。如果无解，输出 `-1`。

**输入：**
一行，包含三个整数 `a, b, c`。

**输出：**
一行，如果无解输出 `-1`。否则输出两个整数 `x, y`，用空格隔开。

**思路分析：**
1.  首先，使用扩展欧几里得算法求出 `g = gcd(a, b)`。
2.  判断是否有解：如果 `c % g != 0`，则无解，输出 `-1`。
3.  如果 `c % g == 0`，则有解。
    *   调用 `exgcd(a, b, x0, y0)` 得到 `ax0 + by0 = g` 的一组特解 `x0, y0`。
    *   由于我们要求 `ax + by = c`，而 `c = (c/g) * g`，所以将 `x0, y0` 分别乘以 `(c/g)` 即可得到 `x = x0 * (c/g)` 和 `y = y0 * (c/g)`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    long long g = exgcd(b, a % b, y, x);
    y = y - (a / b) * x;
    return g;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    long long a, b, c;
    std::cin >> a >> b >> c;

    long long x0, y0;
    long long g = exgcd(a, b, x0, y0); // 求解 ax0 + by0 = g

    if (c % g != 0) { // 如果 c 不是 g 的倍数，则无解
        std::cout << -1 << std::endl;
    } else {
        // 计算倍数 factor
        long long factor = c / g;
        // 得到一组特解
        long long x = x0 * factor;
        long long y = y0 * factor;
        std::cout << x << " " << y << std::endl;
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** `O(log(min(a, b)))`。
*   **空间复杂度：** `O(log(min(a, b)))`。

### 例题3: 求解模逆元 `ax ≡ 1 (mod m)`

**题目描述：**
给定两个正整数 `a` 和 `m` (1 <= a, m <= 10^9)，求 `a` 在模 `m` 意义下的逆元 `x`。即 `ax ≡ 1 (mod m)`。如果逆元不存在，输出 `-1`。如果存在，输出最小的正整数逆元。

**输入：**
一行，包含两个整数 `a, m`。

**输出：**
一行，如果逆元不存在输出 `-1`。否则输出最小的正整数逆元 `x`。

**思路分析：**
1.  将同余方程 `ax ≡ 1 (mod m)` 转换为线性丢番图方程 `ax + my = 1`。
2.  求解 `ax + my = 1` 的前提是 `gcd(a, m) = 1`。
3.  使用 `exgcd(a, m, x0, y0)` 求解，得到 `x0`。
4.  如果 `gcd(a, m) != 1`，则逆元不存在，输出 `-1`。
5.  如果 `gcd(a, m) == 1`，则 `x0` 是一个逆元。我们需要将其调整为 `[0, m-1]` 范围内的最小正整数。
    `x = (x0 % m + m) % m`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    long long g = exgcd(b, a % b, y, x);
    y = y - (a / b) * x;
    return g;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    long long a, m;
    std::cin >> a >> m;

    long long x, y;
    long long g = exgcd(a, m, x, y); // 求解 ax + my = g

    if (g != 1) { // 如果 gcd(a, m) 不为 1，则逆元不存在
        std::cout << -1 << std::endl;
    } else {
        // 调整 x 使其在 [0, m-1] 范围内
        // C++ 的 % 运算符对负数结果可能为负，所以需要 (x % m + m) % m 来确保结果为正
        long long inverse_x = (x % m + m) % m;
        std::cout << inverse_x << std::endl;
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** `O(log(min(a, m)))`。
*   **空间复杂度：** `O(log(min(a, m)))`。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，如 iostream, algorithm 等

// 功能：扩展欧几里得算法
// 参数：
//   a, b: 两个正整数，用于计算 gcd(a, b)
//   x, y: 引用参数，用于返回一组整数解，使得 ax + by = gcd(a, b)
// 返回值：
//   long long: a 和 b 的最大公约数 gcd(a, b)
long long exgcd(long long a, long long b, long long &x, long long &y) {
    // 递归终止条件
    // 当 b 等于 0 时，gcd(a, 0) = a
    // 此时方程变为 ax + 0y = a，显然 x = 1, y = 0 是一组解
    if (b == 0) {
        x = 1; // 设置 x 为 1
        y = 0; // 设置 y 为 0
        return a; // 返回 gcd(a, 0) = a
    }

    // 递归调用
    // 解决子问题：exgcd(b, a % b, x_prime, y_prime)
    // 假设我们已经通过递归得到了子问题的解 x_prime 和 y_prime
    // 使得 b * x_prime + (a % b) * y_prime = gcd(b, a % b)
    // 注意：在调用时，我们将 x 传给 y_prime 的位置，y 传给 x_prime 的位置。
    // 这样做是为了在回溯时，可以直接利用这些引用来更新值，避免额外的临时变量。
    long long g = exgcd(b, a % b, y, x); // 这里的 y 和 x 对应的是子问题中的 x_prime 和 y_prime

    // 回溯阶段：根据子问题的解 (x_prime, y_prime) 推导出当前问题的解 (x, y)
    // 原始推导公式为：
    // x = y_prime
    // y = x_prime - (a / b) * y_prime
    //
    // 结合上面的递归调用：
    // 当前的 x 实际上是子问题中的 y_prime
    // 当前的 y 实际上是子问题中的 x_prime
    //
    // 所以，我们只需要更新 y 的值即可：
    // y = (子问题中的 x_prime) - (a / b) * (子问题中的 y_prime)
    // y = 当前的 y - (a / b) * 当前的 x
    y = y - (a / b) * x;

    // 返回最大公约数
    // 注意：gcd(a, b) = gcd(b, a % b)，所以子问题返回的 g 就是当前问题的 gcd
    return g;
}

int main() {
    // 示例用法：求解 ax + by = gcd(a, b)
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL); // 解除 cin 和 cout 的绑定

    long long a_val = 24, b_val = 18; // 待计算的 a 和 b
    long long x_res, y_res; // 存储结果 x 和 y

    long long gcd_val = exgcd(a_val, b_val, x_res, y_res);

    std::cout << "对于 a=" << a_val << ", b=" << b_val << ":" << std::endl;
    std::cout << "gcd(a, b) = " << gcd_val << std::endl;
    std::cout << "一组解 (x, y) 为: " << x_res << ", " << y_res << std::endl;
    std::cout << "验证: " << a_val << "*" << x_res << " + " << b_val << "*" << y_res << " = "
              << a_val * x_res + b_val * y_res << std::endl;

    // 示例用法2：求解模逆元 5x = 1 (mod 13)
    long long a_mod = 5, m_mod = 13;
    long long x_inv, y_dummy; // y_dummy 是一个占位符，因为我们只关心 x
    long long g_inv = exgcd(a_mod, m_mod, x_inv, y_dummy);

    std::cout << "\n对于模逆元 " << a_mod << "x = 1 (mod " << m_mod << "):" << std::endl;
    if (g_inv != 1) {
        std::cout << "逆元不存在，因为 gcd(" << a_mod << ", " << m_mod << ") != 1" << std::endl;
    } else {
        // 调整 x 使其在 [0, m-1] 范围内
        long long inverse = (x_inv % m_mod + m_mod) % m_mod;
        std::cout << a_mod << " 在模 " << m_mod << " 意义下的逆元是: " << inverse << std::endl;
        std::cout << "验证: (" << a_mod << " * " << inverse << ") % " << m_mod << " = "
                  << (a_mod * inverse) % m_mod << std::endl;
    }

    return 0;
}
```
