## 趣味引入

同学们，大家好！想象一下，你正在玩一个寻宝游戏。你手里有一张藏宝图（模式串），上面画着一个特殊的符号。现在，你面前是一堵长长的墙壁（文本串），上面密密麻麻地刻满了各种图案。你的任务就是在这堵墙上找到所有和藏宝图上一样的特殊符号。

### 困境：笨拙的“地毯式搜索”

最简单的方法是什么呢？你可能会从墙壁的第一个图案开始，和你的藏宝图符号逐个比较。如果发现不匹配，比如藏宝图符号是“ABC”，墙壁上是“ABX”，在“X”这里不匹配了。你怎么办？你只好把藏宝图往右移动一位，从墙壁的第二个图案开始，重新和“ABC”比较。如果墙壁是“AAAAAAAAB”，藏宝图是“AAB”，你会发现：

墙壁：`A A A A A A A A B`
藏宝图：`A A B` (第一次尝试，匹配到第二个A，第三个B不匹配)
墙壁：`A A A A A A A A B`
藏宝图：`  A A B` (藏宝图向右移动一位，又从头开始比较)
墙壁：`A A A A A A A A B`
藏宝图：`    A A B` (再移动一位，再从头开始比较)

这样来来回回，是不是很麻烦？每次不匹配，都要把藏宝图“退回去”，再从头开始比较。这就像你在地毯上找一粒米，只能用手一个格子一个格子地摸，摸到不匹配就退回起点再摸下一个格子。如果墙壁很长，而藏宝图又很短，但有很多重复的图案，这种方法就会非常非常慢！我们把这种方法叫做**朴素匹配算法**。它最坏情况下可能需要 $O(N \times M)$ 的时间，N是墙壁长度，M是藏宝图长度。

### 曙光：KMP 的“火眼金睛”

难道就没有更聪明的方法吗？当然有！今天我们要学习的 KMP 算法，就像给你的眼睛装上了“火眼金睛”。它能一眼看穿藏宝图自身的特点，当发现不匹配时，它不会笨拙地从头开始，而是像开了“透视”一样，瞬间知道藏宝图应该往右移动到哪个位置，才能跳过那些肯定不会匹配的地方，继续高效地搜索！

KMP 算法的核心思想就是：**当匹配失败时，模式串（藏宝图）不回溯，只有文本串（墙壁）的指针可能回溯，或者说，模式串的指针根据预处理的信息进行“聪明地跳跃”**。这样，墙壁上的图案我们永远只看一遍！是不是很酷？

## 深度知识点讲解

### 1. 朴素匹配算法的“痛点”

我们先回顾一下朴素匹配算法的“痛点”。
假设文本串 `S = "ABCABCA"`，模式串 `P = "ABCAB"`。

1.  `S`: `A B C A B C A`
    `P`: `A B C A B`
    `S[0...4]` 与 `P[0...4]` 匹配成功。
    这时，我们发现了一个匹配！

2.  如果模式串是 `P = "ABABD"`，文本串是 `S = "ABABCABABD"`
    `S`: `A B A B C A B A B D`
    `P`: `A B A B D`
    `S[0]` vs `P[0]` -> `A` vs `A` (匹配)
    `S[1]` vs `P[1]` -> `B` vs `B` (匹配)
    `S[2]` vs `P[2]` -> `A` vs `A` (匹配)
    `S[3]` vs `P[3]` -> `B` vs `B` (匹配)
    `S[4]` vs `P[4]` -> `C` vs `D` (不匹配！)

    此刻，朴素算法会将模式串 `P` 整体右移一位，然后从 `S[1]` 重新开始比较 `P[0]`。
    `S`: `A B A B C A B A B D`
    `P`: `  A B A B D`
    这很低效！因为我们已经知道 `S[1...3]` 是 `"BAB"`，而 `P[0]` 是 `'A'`，肯定不会匹配。我们甚至知道 `S[1]` 肯定不会匹配 `P[0]`。这些信息都被浪费了。

**朴素匹配的缺点：** 当发生不匹配时，文本串的指针 `i` 和模式串的指针 `j` 都可能回溯，导致大量的重复比较。

### 2. KMP 算法的核心思想：不回溯的艺术

KMP 算法的精髓在于：**当模式串 `P` 在文本串 `S` 的某个位置发生不匹配时，我们不需要回溯文本串 `S` 的指针 `i`。相反，我们只调整模式串 `P` 的指针 `j`，让它跳到一个“聪明”的位置，继续与 `S[i]` 进行比较。**

这个“聪明”的位置是怎么来的呢？它来源于对模式串 `P` 自身的分析。当 `S[i]` 与 `P[j]` 不匹配时，说明 `P[0...j-1]` 这一段已经成功匹配了 `S[i-j...i-1]`。现在，我们需要找到 `P` 的一个子串，它既是 `P[0...j-1]` 的后缀，又是 `P` 的前缀，这样我们就可以把模式串向右滑动，让这个最长的公共部分重新对齐，从而避免重复比较。

听起来有点绕？没关系，我们用一个生活中的例子来理解。

**生活类比：拼图高手**

你正在拼一个图案很复杂的拼图。你有一块大拼图板（文本串 `S`），上面已经拼好了一部分。你手里拿着一块形状特殊的拼图块（模式串 `P`），想把它放到板子上。

1.  你把 `P` 放到 `S` 的某个位置，从左到右逐个小块地比较图案。
2.  当你比较到第 `j` 个小块时，发现 `P` 的第 `j` 个小块和 `S` 的第 `i` 个小块图案不符！
3.  如果你是朴素算法，你会把 `P` 整个拿起来，向右移动一格，然后从 `P` 的第一个小块重新比较。
4.  但如果你是 KMP 算法，你会怎么做？你是个拼图高手！你知道 `P` 的前 `j-1` 个小块是匹配的。你会立刻检查 `P` 自己的特性：`P` 的前 `j-1` 个小块中，有没有某个后缀（尾巴部分）和 `P` 的某个前缀（开头部分）是完全一样的？
    *   如果有，你就能直接把 `P` 移动到让这个“公共部分”对齐的位置，然后从 `P` 的 `j` 指针继续比较，而 `S` 的 `i` 指针根本不需要动！
    *   如果没有（或者只有很短的公共部分），那就只能移动多一点，或者从 `P` 的开头重新开始比较。

这个“预先知道 `P` 自身特性”的信息，就存储在一个叫做 `next` 数组里。

### 3. `next` 数组（最长公共前后缀长度）

`next` 数组是 KMP 算法的灵魂。它存储了模式串 `P` 的每个前缀（子串）的最长“**公共前后缀**”的长度。

**定义：** 对于模式串 `P`，`next[j]` 表示 `P` 的前缀 `P[0...j]` 中，最长的**真前缀**（不等于自身）同时又是它的**真后缀**（不等于自身）的长度。

**真前缀：** `P[0...k-1]`，其中 `k < j+1`。
**真后缀：** `P[j-k+1...j]`，其中 `k < j+1`。

**举个例子：** 模式串 `P = "ABABCAB"`
*   `P[0] = "A"`: 最长公共前后缀长度为 0。`next[0] = 0`。
*   `P[0...1] = "AB"`: 最长公共前后缀长度为 0。`next[1] = 0`。
*   `P[0...2] = "ABA"`: 前缀有 "A", "AB"；后缀有 "A", "BA"。最长公共前后缀是 "A"，长度为 1。`next[2] = 1`。
*   `P[0...3] = "ABAB"`: 前缀有 "A", "AB", "ABA"；后缀有 "B", "AB", "BAB"。最长公共前后缀是 "AB"，长度为 2。`next[3] = 2`。
*   `P[0...4] = "ABABC"`: 前缀有 "A", "AB", "ABA", "ABAB"；后缀有 "C", "BC", "ABC", "BABC"。没有公共前后缀。长度为 0。`next[4] = 0`。
*   `P[0...5] = "ABABCA"`: 前缀有 "A", "AB", ... "ABABC"；后缀有 "A", "CA", ... "BABCA"。最长公共前后缀是 "A"，长度为 1。`next[5] = 1`。
*   `P[0...6] = "ABABCAB"`: 前缀有 "A", "AB", ... "ABABC A"；后缀有 "B", "AB", ... "BABCA B"。最长公共前后缀是 "AB"，长度为 2。`next[6] = 2`。

所以，对于 `P = "ABABCAB"`，它的 `next` 数组是 `[0, 0, 1, 2, 0, 1, 2]`。

### 4. `next` 数组的计算（“模式串自我匹配”）

计算 `next` 数组的过程本身就像 KMP 匹配过程的一个简化版，模式串 `P` 和它自己进行匹配。

我们用两个指针 `j` 和 `k`：
*   `j`：当前正在计算 `next[j]` 的模式串的下标（从 `1` 到 `m-1`）。
*   `k`：表示 `P[0...j-1]` 中，最长公共前后缀的长度。它也同时是模式串 `P` 的一个前缀的末尾下标（`P[k-1]`）。

**计算步骤：**

1.  初始化 `next[0] = 0`。
2.  `k = 0` (表示当前匹配的前缀长度)。
3.  从 `j = 1` 开始遍历模式串 `P` 的每个字符到 `m-1`。
    *   **如果 `P[j]` 和 `P[k]` 匹配：**
        这说明我们可以把公共前后缀的长度再增加 1。所以 `k++`。
    *   **如果 `P[j]` 和 `P[k]` 不匹配：**
        这时，我们需要寻找更短的公共前后缀。`k` 应该“回溯”到 `next[k-1]` 的位置。这意味着我们尝试用 `P` 的次长公共前后缀来继续匹配。这个回溯过程可能会多次发生，直到 `k` 变为 0（表示没有公共前后缀），或者找到一个 `k` 使得 `P[j]` 和 `P[k]` 匹配。
    *   将当前的 `k` 值赋给 `next[j]`。

**示例：计算 `P = "ABABA"` 的 `next` 数组**

`P` 的长度 `m = 5`。
`next` 数组初始化：`[0, ?, ?, ?, ?]`
`k = 0`

*   `j = 1` (`P[1] = 'B'`)
    *   `P[1] ('B') != P[k] (P[0] = 'A')`
    *   `k` 当前为 0，不能再回溯 (`k > 0` 为假)。
    *   `next[1] = k = 0`。
    `next = [0, 0, ?, ?, ?]`

*   `j = 2` (`P[2] = 'A'`)
    *   `P[2] ('A') == P[k] (P[0] = 'A')`
    *   `k++` (k 变为 1)。
    *   `next[2] = k = 1`。
    `next = [0, 0, 1, ?, ?]`

*   `j = 3` (`P[3] = 'B'`)
    *   `P[3] ('B') == P[k] (P[1] = 'B')`
    *   `k++` (k 变为 2)。
    *   `next[3] = k = 2`。
    `next = [0, 0, 1, 2, ?]`

*   `j = 4` (`P[4] = 'A'`)
    *   `P[4] ('A') == P[k] (P[2] = 'A')`
    *   `k++` (k 变为 3)。
    *   `next[4] = k = 3`。
    `next = [0, 0, 1, 2, 3]`

最终 `next` 数组为 `[0, 0, 1, 2, 3]`。

### 5. KMP 匹配过程

有了 `next` 数组，我们就可以高效地进行匹配了。
我们用两个指针 `i` 和 `j`：
*   `i`：文本串 `S` 的当前匹配位置（从 `0` 到 `N-1`）。
*   `j`：模式串 `P` 的当前匹配位置（从 `0` 到 `M-1`）。

**匹配步骤：**

1.  初始化 `i = 0`, `j = 0`。
2.  当 `i < N` 且 `j < M` 时循环：
    *   **如果 `S[i]` 和 `P[j]` 匹配：**
        两个指针都向前移动：`i++`, `j++`。
    *   **如果 `S[i]` 和 `P[j]` 不匹配：**
        *   **如果 `j > 0`：** 这表示模式串 `P` 已经匹配了一部分 `P[0...j-1]`。现在 `P[j]` 不匹配，我们需要利用 `next` 数组来“聪明地跳跃”模式串。我们将 `j` 更新为 `next[j-1]`。这意味着我们尝试用 `P` 的次长公共前后缀来重新对齐，而 `i` 保持不变。
        *   **如果 `j == 0`：** 这表示模式串的第一个字符 `P[0]` 就没有匹配成功。此时模式串无法再向后“跳跃”，只能将文本串的指针 `i` 向前移动一位，继续从 `S` 的下一个字符开始尝试匹配 `P[0]`。`i++`。

3.  **匹配成功：** 如果循环结束后 `j == M`，说明模式串 `P` 已经完全匹配。此时 `S[i-M ... i-1]` 就是一个匹配。
    *   **注意：** 如果要查找所有匹配，找到一个匹配后，`j` 应该更新为 `next[j-1]`（或者 `next[M-1]`），然后继续循环，寻找下一个可能出现的匹配。`i` 不变。

**示例：`S = "ABABCABABD"`, `P = "ABABD"`**

我们已经知道 `P = "ABABD"` 的 `next` 数组是 `[0, 0, 1, 2, 0]`。
(`P[0]=A, P[1]=B, P[2]=A, P[3]=B, P[4]=D`)

`i = 0, j = 0`

| `i` | `S[i]` | `j` | `P[j]` | 匹配情况 | `next[j-1]` | 动作 |
| :-- | :----- | :-- | :----- | :------- | :---------- | :--- |
| 0   | `A`    | 0   | `A`    | 匹配     | -           | `i++`, `j++` |
| 1   | `B`    | 1   | `B`    | 匹配     | -           | `i++`, `j++` |
| 2   | `A`    | 2   | `A`    | 匹配     | -           | `i++`, `j++` |
| 3   | `B`    | 3   | `B`    | 匹配     | -           | `i++`, `j++` |
| 4   | `C`    | 4   | `D`    | **不匹配** | `next[3]=2` | `j = next[3] = 2` (`i` 不变) |
| 4   | `C`    | 2   | `A`    | **不匹配** | `next[1]=0` | `j = next[1] = 0` (`i` 不变) |
| 4   | `C`    | 0   | `A`    | **不匹配** | -           | `j` 为 0，`i++` (`j` 不变) |
| 5   | `A`    | 0   | `A`    | 匹配     | -           | `i++`, `j++` |
| 6   | `B`    | 1   | `B`    | 匹配     | -           | `i++`, `j++` |
| 7   | `A`    | 2   | `A`    | 匹配     | -           | `i++`, `j++` |
| 8   | `B`    | 3   | `B`    | 匹配     | -           | `i++`, `j++` |
| 9   | `D`    | 4   | `D`    | 匹配     | -           | `i++`, `j++` |
| 10  | -      | 5   | -      | `j == M` | -           | **匹配成功！** 模式串从 `S[10-5] = S[5]` 开始。 |

### 6. 时间复杂度分析

*   **`next` 数组的计算：** 模式串的长度为 `M`。`j` 指针从 `1` 走到 `M-1`，总共移动 `M-1` 次。`k` 指针也会向前移动（`k++`），总共最多移动 `M` 次。当 `k` 回溯时（`k = next[k-1]`），它每次移动的距离都小于它之前前进的距离，而且 `k` 的总前进距离是有限的。所以，`next` 数组的计算时间复杂度是 $O(M)$。
*   **KMP 匹配过程：** 文本串的长度为 `N`，模式串的长度为 `M`。`i` 指针从 `0` 走到 `N-1`，总共移动 `N` 次。`j` 指针也会向前移动（`j++`），总共最多移动 `N` 次。当 `j` 回溯时（`j = next[j-1]`），它每次移动的距离都小于它之前前进的距离，而且 `j` 的总前进距离是有限的。所以，KMP 匹配过程的时间复杂度是 $O(N)$。

**综上所述，KMP 算法的总时间复杂度是 $O(N+M)$。**
这比朴素匹配算法的 $O(N \times M)$ 要高效得多，尤其是在 `N` 和 `M` 都很大时。

### 7. 常见误区与小细节

*   **`next` 数组的下标和含义：** 务必搞清楚 `next[j]` 对应的是 `P[0...j]` 这个前缀，还是 `P[0...j-1]` 这个前缀。本教案采用 `P[0...j]` 的定义。
*   **`next[0]` 的值：** 通常设置为 0。
*   **`j` 回溯时的处理：** 当 `S[i]` 与 `P[j]` 不匹配，且 `j > 0` 时，`j = next[j-1]`。这里 `j-1` 是因为 `next` 数组存储的是前缀 `P[0...j-1]` 的信息，而当前不匹配的是 `P[j]`。
*   **`j == 0` 且不匹配时的处理：** 此时 `P[0]` 就没匹配上，模式串无法再向后“跳跃”，只能 `i++`，让文本串指针移动，继续从 `S` 的下一个字符开始尝试匹配 `P[0]`。
*   **找到匹配后的处理：** 如果目标是找到所有匹配，那么找到一个匹配后，`j` 应该更新为 `next[j-1]` （或者 `next[M-1]`，取决于 `next` 数组的定义和实现方式），然后继续扫描，而不是将 `j` 直接重置为 0。

## 典型例题精讲

### 例题 1: 计算 `next` 数组

**题目：** 给定模式串 `P = "partial"`，请计算其 `next` 数组。

**思路分析：**
严格按照 `next` 数组的计算步骤来。
`P = "partial"`，长度 `m = 7`。
`next` 数组大小为 `m`。
`next[0] = 0`。
使用 `k` 指针记录当前匹配的前缀长度。

**步骤：**

1.  `P = "partial"`
    `next = [?, ?, ?, ?, ?, ?, ?]`
    `next[0] = 0`
    `k = 0`

2.  `j = 1` (`P[1] = 'a'`)
    *   `P[1] ('a') != P[k] (P[0] = 'p')`
    *   `k` 为 0，不回溯。
    *   `next[1] = 0`
    `next = [0, 0, ?, ?, ?, ?, ?]`

3.  `j = 2` (`P[2] = 'r'`)
    *   `P[2] ('r') != P[k] (P[0] = 'p')`
    *   `k` 为 0，不回溯.
    *   `next[2] = 0`
    `next = [0, 0, 0, ?, ?, ?, ?]`

4.  `j = 3` (`P[3] = 't'`)
    *   `P[3] ('t') != P[k] (P[0] = 'p')`
    *   `k` 为 0，不回溯.
    *   `next[3] = 0`
