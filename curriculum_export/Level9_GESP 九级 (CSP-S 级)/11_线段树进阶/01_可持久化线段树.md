## 趣味引入

想象在一个文档编辑器里，每次修改都希望保留历史版本，以便随时“撤销”或对比。  
同样地，我们对一个数字序列做多次更新，却希望能在任意一次更新后，依然查询之前版本的区间和。  
持久化线段树正是为此而生：它像“时间旅行机”，让我们自由在历史版本间来回穿梭并进行查询。

## 知识点讲解

1. 复习线段树  
   - 节点表示区间 [l, r] 的和  
   - 建树时间 O(n)，单次更新/查询 O(log n)  

2. 持久化思想  
   - 每次更新只“复制”从根到叶的那条路径上的节点  
   - 旧版本根节点不变，新版本根节点指向新复制的路径  
   - 保证每次更新空间增 O(log n)，查询时间 O(log n)  

3. 节点结构与全局管理  
   - struct Node { int sum; Node *l, *r; }  
   - 使用 new 分配新节点  
   - 维护一个版本根数组 root[]  

4. 核心算法  
   - build(l, r)：初始化一棵空树或根据初始数组建树  
   - update(prev, l, r, idx, val)：根据 prev 版本复制节点并更新  
   - query(cur, l, r, ql, qr)：在指定版本 cur 上查询区间和  

5. 复杂度分析  
   - 建树：O(n)  
   - 每次更新：O(log n)  
   - 每次查询：O(log n)  
   - 空间：O((n + m) * log n)，m 为更新次数

## 代码示例

```cpp
#include <bits/stdc++.h>

using namespace std;

struct Node
{
    int sum;
    Node *l;
    Node *r;
};

const int MAXN = 100000;
int n, m;
int a[MAXN + 5];
Node* root[MAXN * 20];
int cntNode = 0;

// 新建一个节点，并初始化
Node* newNode()
{
    Node* p = new Node();
    p->sum = 0;
    p->l = NULL;
    p->r = NULL;
    return p;
}

// 根据初始数组建树
Node* build(int l, int r)
{
    Node* p = newNode();
    if (l == r)
    {
        p->sum = a[l];
        return p;
    }
    int mid = (l + r) >> 1;
    p->l = build(l, mid);
    p->r = build(mid + 1, r);
    p->sum = p->l->sum + p->r->sum;
    return p;
}

// 在 prev 树上更新 idx 位置为 val，返回新版本根
Node* update(Node* prev, int l, int r, int idx, int val)
{
    Node* p = newNode();
    *p = *prev; // 复制 prev 节点
    if (l == r)
    {
        p->sum = val;
        return p;
    }
    int mid = (l + r) >> 1;
    if (idx <= mid)
    {
        p->l = update(prev->l, l, mid, idx, val);
    }
    else
    {
        p->r = update(prev->r, mid + 1, r, idx, val);
    }
    p->sum = p->l->sum + p->r->sum;
    return p;
}

// 在版本 cur 上查询区间 [ql, qr]
int query(Node* cur, int l, int r, int ql, int qr)
{
    if (ql <= l && r <= qr)
    {
        return cur->sum;
    }
    int mid = (l + r) >> 1;
    int res = 0;
    if (ql <= mid)
    {
        res += query(cur->l, l, mid, ql, qr);
    }
    if (qr > mid)
    {
        res += query(cur->r, mid + 1, r, ql, qr);
    }
    return res;
}

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> n >> m;
    for (int i = 1; i <= n; i++)
    {
        cin >> a[i];
    }

    // 建立第0个版本
    root[0] = build(1, n);

    for (int i = 1; i <= m; i++)
    {
        int op, version, x, y;
        cin >> op;
        if (op == 1)
        {
            // 1 version idx val：在指定 version 基础上更新
            cin >> version >> x >> y;
            root[i] = update(root[version], 1, n, x, y);
        }
        else
        {
            // 2 version l r：在指定 version 上查询区间和
            cin >> version >> x >> y;
            cout << query(root[version], 1, n, x, y) << "\n";
            root[i] = root[version];
        }
    }

    return 0;
}
```

## 练习题目

基础题  
1. 选择题：持久化线段树在一次点更新操作中，复制的节点数量是  
   A. O(1)  
   B. O(log n)  
   C. O(n)  
   D. O(n log n)  

2. 填空：持久化线段树查询区间时，需要访问的节点数量与区间长度无关，主要受 ____ 决定。  

进阶题  
3. 选择题：若有 m 次更新，持久化线段树总体空间复杂度为  
   A. O(n + m)  
   B. O((n + m) log n)  
   C. O(n log m)  
   D. O(m log m)  

4. 填空：在 C++ 中，持久化线段树常用的节点内存分配方式是使用 ____ 操作符动态申请。  

5. 选择题：以下哪种场景最适合使用持久化线段树？  
   A. 只需要最新版本的区间和查询  
   B. 需要多个版本的历史数据回溯与查询  
   C. 只做一次建树，不做更新  
   D. 线下批量区间求和，无需动态操作  

——答案与解析——

1. B。更新只复制从根到叶的 O(log n) 个节点。  
2. 深度。查询复杂度由树的高度（≈log n）决定。  
3. B。每次更新需要 O(log n) 个新节点，共 O((n + m) log n) 空间。  
4. new。使用 `new Node()` 动态分配节点。  
5. B。持久化线段树主要用于保留历史版本并查询。
