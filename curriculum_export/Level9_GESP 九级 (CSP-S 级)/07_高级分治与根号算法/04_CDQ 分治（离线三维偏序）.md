现在，游戏里有一个“英雄榜”，上面有N个英雄。我们想知道一个秘密：对于每个英雄A，有多少个其他英雄B被它“完全碾压”？

什么叫“完全碾压”呢？就是说，英雄A的攻击力 **大于等于** 英雄B的攻击力，同时，英雄A的防御力 **大于等于** 英雄B的防御力，并且，英雄A的生命值 **大于等于** 英雄B的生命值。只有三项能力都满足，才算“完全碾压”。

*   **思考一下：**
    *   如果我们只有“攻击力”这一个能力值（一维问题），怎么知道一个英雄能碾压多少个其他英雄？
        *   很简单！把所有英雄按攻击力从低到高排序，那么一个英雄前面所有攻击力比它低的英雄，都被它碾压了！
    *   如果我们有“攻击力”和“防御力”两个能力值（二维问题），又该怎么算呢？
        *   嗯... 我们可以先按攻击力排序，这样第一维就满足了。然后呢？对于防御力，我们可能需要一个数据结构，比如树状数组，来快速统计第二维的符合条件的英雄数量。
    *   那现在有“攻击力”、“防御力”、“生命值”三个能力值（三维问题）！这可就复杂了！我们如果还是简单地排序一维，然后用树状数组处理第二维，那第三维怎么办呢？好像没有现成的工具能直接解决！

别担心！今天，我们就来学习一个超级强大的“降维打击”算法——CDQ分治！它能帮助我们将三维问题巧妙地转化为二维，甚至一维问题，最终漂亮地解决它！就像给我们的超级英雄配备了一个“维度转换器”！

## 深度知识点讲解

### 1. CDQ分治是什么？

CDQ分治（Chen Daqi Divide and Conquer）是一种离线算法，名字来源于中国选手陈大奇。它的核心思想可以概括为：**分治处理，合并计算**。它和我们熟悉的归并排序非常相似，都是将一个大问题分成左右两半，递归解决，然后在合并（merge）阶段处理“跨越中点”的贡献。

*   **本质：** CDQ分治是一种巧妙的“降维”技巧。它通过强制一维有序，然后递归处理，在合并时利用左右两部分数据的特性，将高维问题转化为低维问题，再结合其他数据结构（如树状数组或线段树）来解决。
*   **与归并排序的类比：**
    *   归并排序：分治排序，合并时计算逆序对。
    *   CDQ分治：分治处理，合并时计算左区间对右区间的贡献。

### 2. 为什么需要CDQ分治？

在多维偏序问题中，简单的排序只能解决一维的偏序关系。比如，对于点 `P1(x1, y1, z1)` 和 `P2(x2, y2, z2)`，如果 `P1` 支配 `P2`，意味着 `x1 >= x2` 且 `y1 >= y2` 且 `z1 >= z2`。

*   **一维偏序：** 排序即可。
*   **二维偏序：** 先按 `x` 排序，然后用树状数组或线段树处理 `y` 维度。
*   **三维偏序：** 如果我们先按 `x` 排序，那么 `x` 维度的偏序关系就确定了。但 `y` 和 `z` 维度仍然是乱序的，树状数组只能处理一维。这时候，CDQ分治就派上用场了！它能帮我们把三维问题“降维”到二维，再用树状数组解决。

### 3. 三维偏序问题定义

给定 `N` 个点，每个点有三个坐标 `(x, y, z)`。对于每个点 `P_i(x_i, y_i, z_i)`，我们需要统计有多少个点 `P_j(x_j, y_j, z_j)` 满足：
`x_j <= x_i` 且 `y_j <= y_i` 且 `z_j <= z_i`。

**关键点：**

*   **去重与计数：** 如果存在多个完全相同的点，它们对答案的贡献是相同的。通常的做法是，将这些相同的点合并成一个点，并记录它们的出现次数 `cnt`。最后计算答案时，需要乘以这个 `cnt`。
*   **坐标范围：** `x, y, z` 的坐标值可能很大，但树状数组的下标通常需要是连续的小整数。对于 `y` 和 `z` 坐标，如果范围过大，需要进行**离散化**。

### 4. CDQ分治处理三维偏序的步骤

我们以统计“每个点被多少个其他点支配”为例（即 `x_j <= x_i, y_j <= y_i, z_j <= z_i`）。

#### 步骤一：预处理

1.  **数据结构定义：** 定义一个结构体 `Point`，包含 `x, y, z` 坐标，原始 `id`（用于存储最终答案），`cnt`（表示该点在输入中出现的次数），以及 `ans`（存储被支配的点的数量）。
2.  **去重与计数：**
    *   将所有输入的点存储在一个数组 `a` 中。
    *   对 `a` 数组按 `x` 升序，`y` 升序，`z` 升序进行排序。
    *   遍历排序后的数组，将相同的点合并，更新 `cnt` 值。
    *   例如：`p1(1,2,3), p2(1,2,3), p3(1,2,4)` 会被处理成 `P'(1,2,3, cnt=2), P''(1,2,4, cnt=1)`。
3.  **z坐标离散化（如果需要）：** 收集所有点的 `z` 坐标，排序去重，然后用 `std::lower_bound` 映射到 `[1, K]` 的范围。

#### 步骤二：外层排序

对处理后的点数组 `a`，再次按 `x` 坐标升序排序。如果 `x` 相同，则按 `y` 升序；如果 `y` 相同，则按 `z` 升序。
**注意：** 这里的排序是全局的，确保了 `x` 维度上的偏序关系在后续处理中得到满足。

#### 步骤三：CDQ分治函数 `solve(l, r)`

这个函数是CDQ分治的核心。它负责处理 `[l, r]` 区间内的点。

1.  **Base Case (递归终止条件):**
    *   如果 `l == r`，表示只有一个点，直接返回。
2.  **Divide (分):**
    *   `mid = (l + r) / 2`。
    *   递归处理左半部分：`solve(l, mid)`。
    *   递归处理右半部分：`solve(mid + 1, r)`。
    *   **思考：** 经过这两步，`[l, mid]` 内部的偏序关系已经处理完毕，`[mid+1, r]` 内部的偏序关系也处理完毕。现在，我们只剩下处理“左半部分对右半部分的贡献”了。
        *   即，对于右半部分的一个点 `P_R`，有多少个左半部分的点 `P_L` 满足 `x_L <= x_R, y_L <= y_R, z_L <= z_R`。

3.  **Conquer (合并与计算):** 这是最关键的一步，也是CDQ分治“降维”的精髓。

    *   **目标：** 计算 `[l, mid]` 中的点对 `[mid+1, r]` 中的点的贡献。
    *   **维度处理：**
        *   由于 `solve(l, r)` 是在 `x` 维度全局排序的基础上进行的，且 `[l, mid]` 和 `[mid+1, r]` 是原数组的左右两半，所以对于任何 `P_L` (在 `[l, mid]`) 和 `P_R` (在 `[mid+1, r]`)，天然满足 `x_L <= x_R`。第一维的偏序关系已经满足了！
        *   现在问题简化为：在 `x` 满足偏序的前提下，统计 `y_L <= y_R` 且 `z_L <= z_R` 的点对数量。这是一个二维偏序问题！我们可以用双指针 + 树状数组来解决。
    *   **具体操作：**
        1.  **复制到临时数组：** 将 `[l, r]` 区间的点复制到一个临时数组 `t` 中。这一步是为了排序 `y` 轴时不会影响原数组 `a` 的 `x` 轴排序。
        2.  **双指针与树状数组：**
            *   初始化树状数组。
            *   设置两个指针：`i = l` (指向左半部分) 和 `j = mid + 1` (指向右半部分)。
            *   遍历 `k` 从 `l` 到 `r` (遍历 `t` 数组)：
                *   **如果 `j > r` (右半部分已处理完) 或者 (`i <= mid` 且 `t[i].y <= t[j].y`)：**
                    *   说明 `t[i]` (左半部分的点) 的 `y` 坐标更小或相等。
                    *   将 `t[i]` 的 `z` 坐标和 `cnt` 值加入树状数组 `bit.update(t[i].z, t[i].cnt)`。
                    *   `i++`。
                *   **否则 (`i > mid` 或者 `t[j].y < t[i].y`)：**
                    *   说明 `t[j]` (右半部分的点) 的 `y` 坐标更小。
                    *   此时，树状数组中所有已经添加的左半部分的点 `P_L'` 都满足 `P_L'.x <= t[j].x` (因为 `P_L'` 在左半部分，`t[j]` 在右半部分) 和 `P_L'.y <= t[j].y` (因为我们是按 `y` 升序处理的)。
                    *   所以，我们只需要在树状数组中查询 `query(t[j].z)`，得到的就是满足 `x_L <= x_R, y_L <= y_R, z_L <= z_R` 的 `P_L` 的总 `cnt`。
                    *   将这个值加到 `t[j].ans` 上：`t[j].ans += bit.query(t[j].z)`。
                    *   `j++`。
        3.  **清理树状数组：** 在上述双指针循环结束后，树状数组中可能还留有左半部分点的贡献。我们需要将它们移除，确保下次调用 `solve` 时树状数组是空的。最简单的做法是记录所有 `update` 操作，然后反向 `update` 一次。
        4.  **归并排序的最后一步：** 将临时数组 `t` 中 `[l, r]` 区间的点，按照 `y` 坐标的升序顺序，复制回原数组 `a` 的 `[l, r]` 区间。这一步是为了确保在更高层级的CDQ分治调用中，`[l, r]` 区间内部的 `y` 坐标是有序的，从而能正确处理跨区间的贡献。

#### 步骤四：收集答案

CDQ分治结束后，每个点的 `ans` 字段就存储了它被多少个点支配。
需要注意的是，这里的 `ans` 是指被**不同**的三元组支配的数量。如果题目要求统计被**原始输入点**支配的数量，我们需要将 `ans` 乘以它自己的 `cnt`，然后还原到原始 `id` 上。

### 5. 树状数组 (Fenwick Tree) 的作用

在CDQ分治的合并阶段，树状数组用于维护 `z` 维度的前缀和。
*   `update(idx, val)`: 在 `idx` 位置增加 `val`。
*   `query(idx)`: 查询 `[1, idx]` 区间的和。
*   **注意：** 树状数组的下标通常从1开始，且不能为0。如果 `z` 坐标是0或者有负数，需要做适当的偏移或离散化。

### 6. 时间复杂度分析

*   外层排序：`O(N log N)`。
*   CDQ分治：`T(N) = 2T(N/2) + O(N log Z_max)`。
    *   `2T(N/2)` 是递归处理左右两半的时间。
    *   `O(N log Z_max)` 是合并阶段的时间：
        *   双指针遍历 `N` 个点：`O(N)`。
        *   每个点最多进行一次树状数组 `update` 或 `query` 操作，每次操作 `O(log Z_max)`。
    *   所以总时间复杂度为 `O(N log N log Z_max)`。如果 `Z_max` 经过离散化，则为 `O(N log N log N)`。
*   总复杂度：`O(N log^2 N)`。

### 7. 生活类比

你可以把CDQ分治想象成一个“三维筛选机”。

1.  **第一层筛选（X轴）：** 我们先把所有货物（点）按照它们的“长度”（X坐标）排好队。这样，任何左边的货物，长度都比右边的短或相等。这个“长度”维度就搞定了。
2.  **分治：** 然后我们把这批货物一分为二，让两个小助手分别去处理，他们也用同样的方法，先处理好自己区域内的货物。
3.  **第二层筛选（Y轴 + Z轴，合并阶段）：** 当两个小助手各自处理完后，他们要把两批货物合并。这时候，他们拿出另一个“宽度”（Y坐标）的筛选器。
    *   他们会把左边和右边的货物都按照“宽度”从小到大再排一遍队（临时排序）。
    *   然后，他们用一个“Z轴计数器”（树状数组）。当左边的一个货物 `L` 的“宽度”比右边一个货物 `R` 的“宽度”小或相等时，就把 `L` 的“高度”（Z坐标）信息扔进“Z轴计数器”里。
    *   当右边的一个货物 `R` 的“宽度”比左边的都小，或者左边的货物都处理完了，这时候，小助手就会去问“Z轴计数器”：“现在有多少货物的‘高度’比 `R` 的‘高度’小或相等？”计数器马上给出答案！这个答案就是 `R` 被多少个左边的货物碾压了。
    *   所有右边的货物都问完后，小助手会清空“Z轴计数器”，然后把临时排好“宽度”的货物们放回原来的大货架，为下一层筛选做准备。

这样一层层筛选下来，每个货物最终就知道自己被多少个其他货物碾压了。是不是很神奇？

## 典型例题精讲

**题目描述：**
给定 `N` 个三元组 `(x, y, z)`，统计对于每个三元组 `(x_i, y_i, z_i)`，有多少个其他三元组 `(x_j, y_j, z_j)` 满足 `x_j <= x_i` 且 `y_j <= y_i` 且 `z_j <= z_i`。请输出每个原始输入点对应的答案。

**输入格式：**
第一行一个整数 `N` (1 <= N <= 100000)。
接下来 `N` 行，每行三个整数 `x, y, z` (1 <= x, y, z <= 100000)。

**输出格式：**
共 `N` 行，每行一个整数，表示对应原始输入点的答案。

**思路分析：**

1.  **结构体定义：**
    我们需要一个结构体 `Point` 来存储每个点的信息。
    *   `int x, y, z;`：点的三维坐标。
    *   `int id;`：原始输入的索引，用于最后输出答案。
    *   `int cnt;`：该点在输入中出现的次数（用于去重）。
    *   `int ans;`：该点被多少个其他点支配。
    *   `int res;`：最终每个原始点的答案，`res = ans * cnt`。

2.  **去重与计数：**
    *   将 `N` 个输入点读入到一个 `Point` 数组 `a` 中。
    *   对 `a` 数组进行排序：先按 `x` 升序，再按 `y` 升序，最后按 `z` 升序。
    *   遍历排序后的 `a` 数组，将相邻的相同点合并。例如，如果 `a[i]` 和 `a[i-1]` 完全相同，则将 `a[i]` 的 `cnt` 加到 `a[i-1]` 上，并标记 `a[i]` 为无效。最后，构建一个新的有效点数组 `p`。

3.  **`z` 坐标离散化：**
    *   题目中 `z` 的范围是 `1e5`，可以直接作为树状数组的下标，无需离散化。如果 `z` 范围更大，则需要离散化。

4.  **CDQ分治 `solve(l, r)`：**
    *   **外层排序：** 第一次调用 `solve(0, M-1)` 之前，需要对 `p` 数组（去重后的点）按照 `x` 升序，`y` 升序，`z` 升序进行排序。这是为了保证 `x` 维度的偏序关系。
    *   **递归：** `solve(l, mid)` 和 `solve(mid + 1, r)`。
    *   **合并（关键）：**
        *   使用一个临时数组 `t` 存储 `[l, r]` 区间的点。
        *   双指针 `i = l`, `j = mid + 1`。
        *   初始化树状数组 `bit`。
        *   遍历 `k` 从 `l` 到 `r`：
            *   如果 `j > r` 或者 (`i <= mid` 且 `t[i].y <= t[j].y`)：
                *   将 `t[i]` 的 `z` 值和 `cnt` 加入树状数组：`bit.update(t[i].z, t[i].cnt)`。
                *   `i++`。
            *   否则：
                *   查询树状数组：`t[j].ans += bit.query(t[j].z)`。
                *   `j++`。
        *   **清理树状数组：** 遍历 `k` 从 `l` 到 `r`，如果 `t[k]` 是左半部分的点（`k < i_end_of_left_part`），则执行 `bit.update(t[k].z, -t[k].cnt)` 撤销贡献。
        *   **归并：** 将临时数组 `t` 中的 `[l, r]` 区间按 `y` 坐标排序后的结果，复制回原数组 `p` 的 `[l, r]` 区间。这一步是为了保持 `y` 维度的有序性，方便上层递归调用。

5.  **最终答案：**
    *   CDQ分治结束后，每个点 `p[k].ans` 存储的是它被多少个“去重后”的点支配。
    *   我们需要将这个 `ans` 加上它自己的 `cnt - 1`（因为自己也算支配自己一次，但题目要求“其他三元组”，所以要减去1），再乘以它自己的 `cnt`，得到最终对原始点的贡献。
    *   `res[p[k].id] = (p[k].ans + p[k].cnt - 1) * p[k].cnt;`
    *   注意：题目中说的“支配”通常不包含自己。如果包含自己，那么 `ans` 就不需要减 `1`。这里我们按“不包含自己”来理解，所以一个点 `P` 支配 `P` 自身，但我们只计算 `P` 支配其他点。如果 `P` 有 `cnt` 个副本，那么 `P` 的每个副本都会被 `ans` 个其他点支配，加上 `cnt-1` 个自己的副本支配。所以总共是 `(ans + cnt - 1) * 1`。
    *   更准确的理解是：`p[k].ans` 是被“不同的其他点”支配的次数。那么 `p[k]` 这个点（有 `p[k].cnt` 个原始副本）被支配的总次数是 `(p[k].ans + p[k].cnt - 1)`。因为 `p[k]` 的 `cnt-1` 个副本也支配 `p[k]` 自己。
    *   所以，对于原始的第 `id` 个点，它被支配的总数是 `p[k].ans` （被其他不同点支配）`+ (p[k].cnt - 1)` （被自己的其他副本支配）。这个值就是 `ans_for_original_id[id]`。
    *   **修正：** 题目问的是“有多少个其他三元组 `(x_j, y_j, z_j)` 满足...”。这里的 `ans` 已经统计了 `x_j <= x_i` 且 `y_j <= y_i` 且 `z_j <= z_i` 且 `j != i` 的情况。
        *   `p[k].ans` 存储的是被 `p[k]` 左边的、满足条件的 **不同** 点的数量的总 `cnt`。
        *   所以对于 `p[k]` 自身，它的 `ans` 应该是 `p[k].ans` (来自其他不同点) `+ (p[k].cnt - 1)` (来自它自己的其他副本)。
        *   最终，每个原始 `id` 的点的答案就是 `p[k].ans + p[k].cnt - 1`。
        *   如果题目要求的是“有多少个点（包括自己）支配自己”，那么就是 `p[k].ans + p[k].cnt`。
        *   这里按“不包含自己”来理解，所以 `final_ans[p[k].id] = p[k].ans + p[k].cnt - 1`。

**示例代码 (伪代码，完整代码见模板)：**

```cpp
struct Point {
    int x, y, z;
    int id; // 原始下标
    int cnt; // 出现次数
    int ans; // 被支配点数量
};

Point p[MAX_N], t[MAX_N]; // p是原始点数组，t是临时数组
int final_ans[MAX_N]; // 存储每个原始id的最终答案
int bit[MAX_Z]; // 树状数组

int MAX_Z_COORD = 100000; // z的最大值

void update(int idx, int val) {
    for (; idx <= MAX_Z_COORD; idx += idx & -idx) {
        bit[idx] += val;
    }
}

int query(int idx) {
    int sum = 0;
    for (; idx > 0; idx -= idx & -idx) {
        sum += bit[idx];
    }
    return sum;
}

// 比较函数，用于x优先的排序
bool cmp1(const Point& A, const Point& B) {
    if (A.x != B.x) return A.x < B.x;
    if (A.y != B.y) return A.y < B.y;
    return A.z < B.z;
}

// 比较函数，用于y优先的排序
bool cmp2(const Point& A, const Point& B) {
    if (A.y != B.y) return A.y < B.y;
    return A.z < B.z;
}

void solve(int l, int r) {
    if (l == r) {
        return;
    }

    int mid = (l + r) / 2;
    solve(l, mid);
    solve(mid + 1, r);

    // 将[l, r]复制到临时数组t
    for (int k = l; k <= r; ++k) {
        t[k] = p[k];
    }

    int i = l, j = mid + 1;
    int current_updates_count = 0; // 记录本次merge中对bit的操作次数，用于清理

    // 合并并计算贡献
    for (int k = l; k <= r; ++k) {
        if (j > r || (i <= mid && t[i].y <= t[j].y)) {
            // 左半部分的点，y坐标更小或相等，将其z和cnt加入树状数组
            update(t[i].z, t[i].cnt);
            current_updates_count++; // 记录一次update
            i++;
        } else {
            // 右半部分的点，y坐标更大，查询树状数组
            t[j].ans += query(t[j].z);
            j++;
        }
    }

    // 清理树状数组
    // 另一种清理方法是，在循环结束后，将所有在[l, mid]范围内被update的点撤销
    for (int k_cleanup = l; k_cleanup < i; ++k_cleanup) { // 注意这里是 < i, 因为i已经++了
        update(t[k_cleanup].z, -t[k_cleanup].cnt);
    }
    // 或者更简单的，记录所有update操作的z坐标和值，然后反向操作

    // 归并回原数组p，保持y的有序性，方便上层递归
    // 注意：这里需要根据y排序，而不是x。
    // std::sort(p + l, p + r + 1, cmp2); // 这一步实际上可以省掉，因为下面的inplace_merge_sort_like_copy会完成排序
    // 更准确的做法是，将t数组按y排序，然后复制回p
    // std::inplace_merge(p + l, p + mid + 1, p + r + 1, cmp2);
    // 这里的合并操作其实就是归并排序的合并，但我们已经用双指针做了，所以只需要把t复制回p
    // 重新排序p[l...r] by y
    std::sort(t + l, t + r + 1, cmp2);
    for (int k = l; k <= r; ++k) {
        p[k] = t[k];
    }
}
```

**复杂度分析：**
*   去重与初始排序：`O(N log N)`
*   CDQ分治：`solve(N)` 递归调用 `2 * solve(N/2)`，合并阶段 `O(N log Z_max)`。
    *   总时间复杂度：`O(N log N log Z_max)`。
    *   在本题中 `Z_max = 100000`，所以是 `O(N log^2 N)`。
*   空间复杂度：`O(N)` (临时数组 `t` 和树状数组 `bit`)。
