# 分块（根号分治）

## 教学目标

### 知识目标
1.  **理解分块思想的本质**：掌握将大型数据结构（如数组）划分为若干个子块，并对块内、块间进行不同处理的策略。
2.  **掌握分块算法的基本操作**：能够设计和实现区间查询和区间修改操作，包括处理不完整块和完整块的策略。
3.  **理解块大小选择的数学原理**：明确为什么选择 `sqrt(N)` 作为块大小能够达到最优时间复杂度。

### 能力目标
1.  **问题分解与抽象**：能够将一个复杂的区间操作问题分解为分块模型，并设计相应的块内、块间处理逻辑。
2.  **代码实现能力**：能够使用原生数组和基础循环结构，独立实现基于分块的数据结构，解决GSP九级难度的相关问题。
3.  **时间复杂度分析**：能够准确分析分块算法在不同操作下的时间复杂度，并理解其优化效果。

### 素养目标
1.  **平衡复杂度思维**：体会“以空间换时间”和“均摊复杂度”的思想，培养在不同算法之间权衡利弊的优化思维。
2.  **算法设计与创新**：在面对传统数据结构难以解决或效率低下时，能够尝试从分块等“根号算法”角度思考解决方案。
3.  **严谨编程习惯**：注重边界条件、数据更新和懒惰标记的处理，培养细致严谨的编程习惯。

## 趣味引入

同学们，想象一下我们有一个超大的图书馆，里面有10万本书，排成一长串。现在，有两位馆员小A和小B，他们接到了一些任务：

**任务一：** 领导想知道，从第1000本书到第2000本书之间，一共有多少本关于“人工智能”的书？（**区间查询**）
**任务二：** 领导又说，从第5000本书到第6000本书之间，所有关于“历史”的书，全部换成“地理”类。（**区间修改**）

如果图书馆只有一排书架，小A馆员每次都得从头到尾一本书一本书地找，或者一本书一本书地换标签。这显然太慢了！10万本书，每次操作都要走完这么多，很快就累趴下了。

小B馆员是个聪明人，他想：“这不行啊！我得想个办法！”他观察了一下，发现书架虽然长，但可以分成很多小段。于是他把这10万本书，每100本分成一个“书区”，每个书区都有一个区长来管理。

*   当任务是查询或修改一小段书（比如从第105本到第115本）时，他直接找到对应的书区，让区长去处理，很快。
*   当任务是查询或修改一大段书（比如从第1000本到第5000本）时，他发现这会跨越很多个“书区”。他会让两边的**不完整书区**的区长去处理各自小段的任务，而中间的那些**完整书区**，他直接告诉这些书区的区长：“你们整个区都按这个要求来！” 区长们会记下来，下次有人问起这个区的情况，他们就能快速回答，而不用真的每本书都去处理一遍。

这就是我们今天要学习的“分块”思想！它就像图书馆的区长制度一样，把一个大问题拆分成许多小块，对不同的块采用不同的处理方法，从而大大提高效率！

## 深度知识点讲解

### 1. 分块思想的本质：化整为零，分而治之
分块，也叫根号分治，是一种处理区间操作的算法思想。它的核心在于：
*   **将一个大的序列（通常是数组 `a[1...N]`）划分为若干个大小相近的子序列，我们称之为“块”**。
*   **对这些块进行不同的处理：**
    *   **块内的操作：** 如果查询或修改的区间只涉及一小段数据，或者只涉及某个块的一部分，我们就直接暴力遍历这些数据。
    *   **块间的操作：** 如果查询或修改的区间覆盖了多个完整的块，我们就利用每个块预先存储的某种“概括信息”（比如块内所有数的和、最大值、或者一个统一的修改标记），快速处理这些完整的块。

这种思想的精妙之处在于，它在“暴力”和“精细”之间找到了一个平衡点。当高级数据结构（如线段树、树状数组）难以实现或不适用时，分块提供了一种简单而有效的替代方案。

### 2. 为什么需要分块？
1.  **传统暴力解法太慢：** 对于长度为 `N` 的数组，如果每次区间查询或修改都需要 `O(N)` 的时间，那么 `Q` 次操作的总时间复杂度将是 `O(Q*N)`。当 `N` 和 `Q` 都很大（例如 `N, Q = 10^5`）时，`10^{10}` 次操作是无法接受的。
2.  **高级数据结构有时受限：** 线段树和树状数组虽然强大，但它们通常只能处理具有“结合律”的操作（如求和、求最大最小值）。对于某些复杂操作（例如区间开方、区间排序），线段树和树状数组就显得力不从心了。此时，分块可能会提供一种更灵活、更容易实现的方案。
3.  **实现简单，常数小：** 相比于线段树等，分块的实现相对简单，代码量小，常数因子通常也较小，有时能跑得更快。

### 3. 如何分块？
假设我们有一个长度为 `N` 的数组 `a`，下标从 `0` 到 `N-1` 或 `1` 到 `N`。
1.  **确定块的大小 `S`：** 这是分块最核心的参数。后面我们会详细解释为什么 `S` 通常取 `sqrt(N)`。
2.  **计算块的数量 `num_blocks`：** `num_blocks = (N + S - 1) / S` (向上取整)。
3.  **划分块：**
    *   第 `k` 个块（从 `0` 或 `1` 开始编号）包含的元素范围是：`[k*S, (k+1)*S - 1]`。
    *   对于数组中的任意一个元素 `a[i]`，它所属的块的编号 `block_id = i / S` (如果下标从0开始)。
4.  **维护每个块的信息：**
    *   **块的起始和结束索引：** `block_start[k]` 和 `block_end[k]`。
    *   **块的概括信息：** 根据具体问题而定，例如：
        *   `block_sum[k]`：第 `k` 个块内所有元素的和。
        *   `block_max[k]`：第 `k` 个块内所有元素的最大值。
        *   `block_add[k]`：第 `k` 个块的懒惰标记，表示整个块都被加上了一个值。

### 4. 分块操作的核心：区间查询与区间修改

假设我们要处理的区间是 `[L, R]` (下标从 `0` 开始)。
首先，确定 `L` 和 `R` 分别属于哪个块：
`block_L = L / S`
`block_R = R / S`

#### 4.1. 区间查询 (Query)
目标是计算区间 `[L, R]` 内的某个值（如和、最大值）。
1.  **情况一：`block_L == block_R` (L和R在同一个块内)**
    *   直接暴力遍历 `a[L]` 到 `a[R]`，并累加（或比较）结果。
    *   如果块有懒惰标记 `block_add[block_L]`，在计算每个 `a[i]` 的真实值时，要加上 `block_add[block_L]`。
    *   时间复杂度：`O(S)`。

2.  **情况二：`block_L != block_R` (L和R跨越多个块)**
    *   **处理 `L` 所在的不完整块：** 从 `a[L]` 遍历到 `block_end[block_L]`。每个 `a[i]` 加上 `block_add[block_L]` 后参与计算。
    *   **处理 `R` 所在的不完整块：** 从 `block_start[block_R]` 遍历到 `a[R]`。每个 `a[i]` 加上 `block_add[block_R]` 后参与计算。
    *   **处理中间的完整块：** 从 `block_L + 1` 到 `block_R - 1` 遍历这些完整的块。对于每个块 `k`，直接使用它预先存储的概括信息（如 `block_sum[k]` 或 `block_max[k]`），并考虑其懒惰标记 `block_add[k]`。
        *   例如，求和时，一个完整块 `k` 的贡献是 `block_sum[k] + block_add[k] * S`。
    *   时间复杂度：`O(S + N/S)`。

#### 4.2. 区间修改 (Update)
目标是修改区间 `[L, R]` 内的元素（如加上一个值 `x`）。
1.  **情况一：`block_L == block_R` (L和R在同一个块内)**
    *   直接暴力遍历 `a[L]` 到 `a[R]`，每个 `a[i]` 加上 `x`。
    *   **重要：** 修改后，需要**重新计算** `block_L` 块的概括信息（如 `block_sum[block_L]`）。
    *   时间复杂度：`O(S)`。

2.  **情况二：`block_L != block_R` (L和R跨越多个块)**
    *   **处理 `L` 所在的不完整块：** 从 `a[L]` 遍历到 `block_end[block_L]`，每个 `a[i]` 加上 `x`。
        *   **注意：** 如果 `block_L` 有懒惰标记 `block_add[block_L]`，在暴力修改前，需要将 `block_add[block_L]` 的效果“下沉”到块内所有元素上（即 `a[i] += block_add[block_L]`），然后清空 `block_add[block_L]`。这样才能保证 `a[i]` 存储的是真实值的一部分，后续的暴力修改才正确。修改后，重新计算 `block_sum[block_L]`。
    *   **处理 `R` 所在的不完整块：** 类似地，从 `block_start[block_R]` 遍历到 `a[R]`，每个 `a[i]` 加上 `x`。同样需要先“下沉” `block_add[block_R]`，再暴力修改，最后重新计算 `block_sum[block_R]`。
    *   **处理中间的完整块：** 从 `block_L + 1` 到 `block_R - 1` 遍历这些完整的块。对于每个块 `k`，直接更新它的懒惰标记 `block_add[k] += x`。同时，如果该块维护了 `block_sum[k]`，也要更新 `block_sum[k] += x * S`。
    *   时间复杂度：`O(S + N/S)`。

### 5. 块大小的选择：为什么是 `sqrt(N)`？
这是分块算法最核心的数学优化。
设块的大小为 `S`，那么总共有 `N/S` 个块。
每次查询或修改操作，最坏情况下：
*   需要遍历 `L` 和 `R` 所在的两端不完整块，这需要 `O(S)` 的时间（最坏是 `2*S` 个元素）。
*   需要处理中间的完整块，这需要 `O(N/S)` 的时间（最坏是 `N/S` 个块）。
因此，单次操作的总时间复杂度为 `T(S) = O(S + N/S)`。

为了让 `T(S)` 最小，我们需要找到 `S` 的最佳值。
我们可以把 `f(S) = S + N/S` 看作一个函数。根据均值不等式，当 `S = N/S` 时，即 `S^2 = N`，`S = sqrt(N)` 时，`S + N/S` 取得最小值。
此时，`T(S) = O(sqrt(N) + N/sqrt(N)) = O(sqrt(N) + sqrt(N)) = O(sqrt(N))`。

所以，将块大小设置为 `sqrt(N)`，可以使得单次查询和修改操作的时间复杂度达到最优的 `O(sqrt(N))`。对于 `Q` 次操作，总时间复杂度为 `O(Q * sqrt(N))`。

### 6. 常见误区
1.  **忘记更新块内信息：** 当对不完整块进行暴力修改后，必须重新计算该块的 `block_sum`、`block_max` 等概括信息。
2.  **懒惰标记处理不当：** 当暴力修改一个有懒惰标记的块时，需要先将懒惰标记“下沉”到块内所有元素，才能保证 `a[i]` 的值是正确的。
3.  **边界条件错误：** 数组下标从 `0` 开始还是从 `1` 开始，以及块的起始和结束索引，都容易出错。要仔细检查 `L`, `R`, `block_start`, `block_end` 的计算。
4.  **块大小计算：** `num_blocks = (N + S - 1) / S` 是常用的向上取整方法，确保所有元素都被分到块中。

## 典型例题精讲

### 例题1：区间加，单点查询

**问题描述：**
给定一个长度为 `N` 的数组 `a`，支持 `Q` 次操作。
1.  `1 L R x`：将区间 `[L, R]` 内的所有元素都加上 `x`。
2.  `2 k`：查询 `a[k]` 的值。
（`N, Q <= 100000`，`0 <= L, R < N`，`|x| <= 10000`，`|a[i]| <= 10^9`）

**思路分析：**
这是一个经典的区间修改、单点查询问题。如果使用线段树，可以做到 `O(log N)`。这里我们用分块来解决。
*   **分块策略：** 将数组 `a` 分成 `sqrt(N)` 个块，每个块大小为 `S = sqrt(N)`。
*   **每个块维护的信息：**
    *   `a[i]`：原始数组，存储每个元素的当前值（在没有懒惰标记的情况下）。
    *   `block_add[k]`：第 `k` 个块的懒惰标记，表示整个块都被加上了 `block_add[k]`。

**操作实现：**

1.  **`update(L, R, x)` (区间加)：**
    *   确定 `L` 和 `R` 所在的块 `block_L` 和 `block_R`。
    *   **如果 `block_L == block_R`：** (L和R在同一个块)
        *   直接遍历 `a[i]` 从 `L` 到 `R`，将 `a[i] += x`。
        *   **注意：** 此时 `a[i]` 的真实值是 `a[i] + block_add[block_L]`。但由于我们是单点查询，`block_add` 会在查询时计算，所以这里直接修改 `a[i]` 即可。
    *   **如果 `block_L != block_R`：** (L和R跨越多个块)
        *   **处理 `L` 所在的不完整块 (`block_L`)：**
            *   遍历 `a[i]` 从 `L` 到 `block_end[block_L]`，将 `a[i] += x`。
        *   **处理 `R` 所在的不完整块 (`block_R`)：**
            *   遍历 `a[i]` 从 `block_start[block_R]` 到 `R`，将 `a[i] += x`。
        *   **处理中间的完整块 (`block_L + 1` 到 `block_R - 1`)：**
            *   对于每个完整块 `k`，直接更新它的懒惰标记：`block_add[k] += x`。

2.  **`query(k)` (单点查询)：**
    *   确定 `k` 所在的块 `block_k`。
    *   `a[k]` 的真实值就是 `a[k] + block_add[block_k]`。
    *   时间复杂度：`O(1)`。

**复杂度分析：**
*   初始化：`O(N)`。
*   单次 `update` 操作：遍历不完整块 `O(S)`，更新完整块懒惰标记 `O(N/S)`。总计 `O(S + N/S)`。
*   单次 `query` 操作：`O(1)`。
*   取 `S = sqrt(N)`，单次 `update` 为 `O(sqrt(N))`。
*   总时间复杂度：`O(N + Q * sqrt(N))`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;

long long a[MAXN]; // 原始数组
long long block_add[MAXN]; // 块的懒惰标记
int block_id[MAXN]; // 元素 i 属于哪个块
int block_start[MAXN]; // 每个块的起始索引
int block_end[MAXN]; // 每个块的结束索引

int n, q, block_size, num_blocks;

void build() {
    block_size = sqrt(n); // 块大小
    if (block_size == 0) block_size = 1; // 避免N很小导致块大小为0
    num_blocks = (n + block_size - 1) / block_size; // 块的数量

    for (int i = 0; i < n; ++i) {
        block_id[i] = i / block_size;
    }

    for (int i = 0; i < num_blocks; ++i) {
        block_start[i] = i * block_size;
        block_end[i] = (i + 1) * block_size - 1;
        if (block_end[i] >= n) { // 最后一个块可能不满
            block_end[i] = n - 1;
        }
    }
}

// 区间 [L, R] 加 x
void update(int L, int R, int x) {
    int bl = block_id[L];
    int br = block_id[R];

    // L和R在同一个块内
    if (bl == br) {
        for (int i = L; i <= R; ++i) {
            a[i] += x;
        }
    } else {
        // 处理L所在的不完整块
        for (int i = L; i <= block_end[bl]; ++i) {
            a[i] += x;
        }
        // 处理R所在的不完整块
        for (int i = block_start[br]; i <= R; ++i) {
            a[i] += x;
        }
        // 处理中间的完整块
        for (int k = bl + 1; k < br; ++k) {
            block_add[k] += x;
        }
    }
}

// 查询 a[k] 的值
long long query(int k) {
    return a[k] + block_add[block_id[k]];
}

int main() {
    // 快速IO，CSP竞赛常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> q;

    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    build(); // 初始化分块信息

    for (int op_idx = 0; op_idx < q; ++op_idx) {
        int op_type;
        std::cin >> op_type;
        if (op_type == 1) {
            int L, R, x;
            std::cin >> L >> R >> x;
            // 题目通常是1-indexed，这里转换成0-indexed
            update(L - 1, R - 1, x);
        } else {
            int k;
            std::cin >> k;
            // 题目通常是1-indexed，这里转换成0-indexed
            std::cout << query(k - 1) << "\n";
        }
    }

    return 0;
}
```

### 例题2：区间加，区间求和

**问题描述：**
给定一个长度为 `N` 的数组 `a`，支持 `Q` 次操作。
1.  `1 L R x`：将区间 `[L, R]` 内的所有元素都加上 `x`。
2.  `2 L R`：查询区间 `[L, R]` 内所有元素的和。
（`N, Q <= 100000`，`0 <= L, R < N`，`|x| <= 10000`，`|a[i]| <= 10^9`）

**思路分析：**
这个例子比上一个更典型，因为区间修改和区间查询都需要维护块的概括信息。
*   **分块策略：** 同例题1，块大小 `S = sqrt(N)`。
*   **每个块维护的信息：**
    *   `a[i]`：原始数组，存储每个元素的当前值（在没有懒惰标记的情况下）。
    *   `block_add[k]`：第 `k` 个块的懒惰标记。
    *   `block_sum[k]`：第 `k` 个块内所有元素的和（不包含懒惰标记的值）。

**操作实现：**

1.  **`update(L, R, x)` (区间加)：**
    *   确定 `L` 和 `R` 所在的块 `bl` 和 `br`。
    *   **如果 `bl == br`：**
        *   遍历 `a[i]` 从 `L` 到 `R`，`a[i] += x`。
        *   **重要：** 更新 `block_sum[bl]`。由于 `a[i]` 已经被修改，直接 `block_sum[bl] += (R - L + 1) * x` 即可。
    *   **如果 `bl != br`：**
        *   **处理 `L` 所在的不完整块 (`bl`)：**
            *   遍历 `a[i]` 从 `L` 到 `block_end[bl]`，`a[i] += x`。
            *   **重要：** 更新 `block_sum[bl]`。
        *   **处理 `R` 所在的不完整块 (`br`)：**
            *   遍历 `a[i]` 从 `block_start[br]` 到 `R`，`a[i] += x`。
            *   **重要：** 更新 `block_sum[br]`。
        *   **处理中间的完整块 (`bl + 1` 到 `br - 1`)：**
            *   对于每个完整块 `k`：
                *   更新 `block_add[k] += x`。
                *   更新 `block_sum[k] += x * (block_end[k] - block_start[k] + 1)`。

2.  **`query(L, R)` (区间求和)：**
    *   确定 `L` 和 `R` 所在的块 `bl` 和 `br`。
    *   初始化 `total_sum = 0`。
    *   **如果 `bl == br`：**
        *   遍历 `a[i]` 从 `L` 到 `R`，累加 `a[i]`。
        *   **重要：** 别忘了加上 `block_add[bl]` 对这个区间的贡献：`total_sum += (R - L + 1) * block_add[bl]`。
    *   **如果 `bl != br`：**
        *   **处理 `L` 所在的不完整块 (`bl`)：**
            *   遍历 `a[i]` 从 `L` 到 `block_end[bl]`，累加 `a[i]`。
            *   **重要：** 加上 `block_add[bl]` 对这部分区间的贡献：`total_sum += (block_end[bl] - L + 1) * block_add[bl]`。
        *   **处理 `R` 所在的不完整块 (`br`)：**
            *   遍历 `a[i]` 从 `block_start[br]` 到 `R`，累加 `a[i]`。
            *   **重要：** 加上 `block_add[br]` 对这部分区间的贡献：`total_sum += (R - block_start[br] + 1) * block_add[br]`。
        *   **处理中间的完整块 (`bl + 1` 到 `br - 1`)：**
            *   对于每个完整块 `k`：
                *   累加 `block_sum[k]`。
                *   **重要：** 加上 `block_add[k]` 对这个块的贡献：`total_sum += block_add[k] * (block_end[k] - block_start[k] + 1)`。

**复杂度分析：**
*   初始化：`O(N)`。
*   单次 `update` 操作：`O(S + N/S)`。
*   单次 `query` 操作：`O(S + N/S)`。
*   取 `S = sqrt(N)`，单次操作为 `O(sqrt(N))`。
*   总时间复杂度：`O(N + Q * sqrt(N))`。

**代码实现：**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;

long long a[MAXN]; // 原始数组
long long block_add[MAXN]; // 块的懒惰标记
long long block_sum[MAXN]; // 块内元素的和 (不包含懒惰标记)
int block_id[MAXN]; // 元素 i 属于哪个块
int block_start[MAXN]; // 每个块的起始索引
int block_end[MAXN]; // 每个块的结束索引

int n, q, block_size, num_blocks;

// 重新计算某个块的block_sum
void recalc_block_sum(int k) {
    block_sum[k] = 0;
    for (int i = block_start[k]; i <= block_end[k]; ++i) {
        block_sum[k] += a[i];
    }
}

void build() {
    block_size = sqrt(n);
    if (block_size == 0) block_size = 1;
    num_blocks = (n + block_size - 1) / block_size;

    for (int i = 0; i < n; ++i) {
        block_id[i] = i / block_size;
    }

    for (int i = 0; i < num_blocks; ++i) {
        block_start[i] = i * block_size;
        block_end[i] = (i + 1) * block_size - 1;
        if (block_end[i] >= n) {
            block_end[i] = n - 1;
        }
        recalc_block_sum(i); // 初始化每个块的和
    }
}

// 区间 [L, R] 加 x
void update(int L, int R, int x) {
    int bl = block_id[L];
    int br = block_id[R];

    if (bl == br) {
        for (int i = L; i <= R; ++i) {
            a[i] += x;
        }
        recalc_block_sum(bl); // 暴力修改后，重新计算块的和
    } else {
        // 处理L所在的不完整块
        for (int i = L; i <= block_end[bl]; ++i) {
            a[i] += x;
        }
        recalc_block_sum(bl);

        // 处理R所在的不完整块
        for (int i = block_start[br]; i <= R; ++i) {
            a[i] += x;
        }
        recalc_block_sum(br);

        // 处理中间的完整块
        for (int k = bl + 1; k < br; ++k) {
            block_add[k] += x;
            block_sum[k] += (long long)x * (block_end[k] - block_start[k] + 1);
        }
    }
}

// 查询区间 [L, R] 的和
long long query(int L, int R) {
    long long total_sum = 0;
    int bl = block_id[L];
    int br = block_id[R];

    if (bl == br) {
        for (int i = L; i <= R; ++i) {
            total_sum += a[i];
        }
        total_sum += (long long)(R - L + 1) * block_add[bl]; // 加上懒惰标记的贡献
    } else {
        // 处理L所在的不完整块
        for (int i = L; i <= block_end[bl]; ++i) {
            total_sum += a[i];
        }
        total_sum += (long long)(block_end[bl] - L + 1) * block_add[bl];

        // 处理R所在的不完整块
        for (int i = block_start[br]; i <= R; ++i) {
            total_sum += a[i];
        }
        total_sum += (long long)(R - block_start[br] + 1) * block_add[br];

        // 处理中间的完整块
        for (int k = bl + 1; k < br; ++k) {
            total_sum += block_sum[k];
            total_sum += (long long)(block_end[k] - block_start[k] + 1) * block_add[k];
        }
    }
    return total_sum;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> q;

    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    build();

    for (int op_idx = 0; op_idx < q; ++op_idx) {
        int op_type;
        std::cin >> op_type;
        if (op_type == 1) {
            int L, R, x;
            std::cin >> L >> R >> x;
            update(L - 1, R - 1, x);
        } else {
            int L, R;
            std::cin >> L >> R;
            std::cout << query(L - 1, R - 1) << "\n";
        }
    }

    return 0;
}
```

```cpp
#include <bits/stdc++.h> // 包含常用头文件，CSP竞赛风格

const int MAXN = 100005; // 数组最大长度，根据题目N的范围调整

long long a[MAXN];         // 原始数组，存储每个元素的值
long long block_info[MAXN]; // 每个块的概括信息（例如：块内元素之和、最大值等）
long long block_lazy[MAXN]; // 每个块的懒惰标记（例如：整个块要加上的值）

int block_id[MAXN];       // 元素 i 属于哪个块
int block_start[MAXN];    // 每个块的起始索引
int block_end[MAXN];      // 每个块的结束索引

int n;                     // 数组长度
int q;                     // 操作次数
int block_size;            // 块的大小，通常取 sqrt(N)
int num_blocks;            // 块的数量

// 函数：重新计算某个块的 block_info
// 当块内元素被暴力修改后，需要调用此函数更新块的概括信息
void recalc_block_info(int k) {
    // 根据具体问题实现，例如求和：
    block_info[k] = 0;
    for (int i = block_start[k]; i <= block_end[k]; ++i) {
        block_info[k] += a[i];
    }
    // 例如求最大值：
    // block_info[k] = -2e18; // 足够小的初始值
    // for (int i = block_start[k]; i <= block_end[k]; ++i) {
    //     if (a[i] > block_info[k]) {
    //         block_info[k] = a[i];
    //     }
    // }
}

// 函数：初始化分块结构
void build() {
    block_size = sqrt(n); // 块大小通常取 sqrt(N)
    if (block_size == 0) { // 防止 N 过小导致 block_size 为 0
        block_size = 1;
    }
    num_blocks = (n + block_size - 1) / block_size; // 计算块的数量，向上取整

    // 确定每个元素所属的块ID
    for (int i = 0; i < n; ++i) {
        block_id[i] = i / block_size;
    }

    // 确定每个块的起始和结束索引
    for (int k = 0; k < num_blocks; ++k) {
        block_start[k] = k * block_size;
        block_end[k] = (k + 1) * block_size - 1;
        if (block_end[k] >= n) { // 最后一个块可能不满，修正其结束索引
            block_end[k] = n - 1;
        }
        recalc_block_info(k); // 初始化每个块的概括信息
    }
}

// 函数：区间 [L, R] 修改操作 (例如：区间加 x)
// L, R 为 0-indexed
void update_range(int L, int R, int x) {
    int bl = block_id[L]; // L 所在的块
    int br = block_id[R]; // R 所在的块

    // 如果 L 和 R 在同一个块内
    if (bl == br) {
        for (int i = L; i <= R; ++i) {
            a[i] += x; // 暴力修改数组元素
        }
        recalc_block_info(bl); // 暴力修改后，重新计算块的概括信息
    } else {
        // 处理 L 所在的不完整块
        for (int i = L; i <= block_end[bl]; ++i) {
            a[i] += x;
        }
        recalc_block_info(bl); // 重新计算

        // 处理 R 所在的不完整块
        for (int i = block_start[br]; i <= R; ++i) {
            a[i] += x;
        }
        recalc_block_info(br); // 重新计算

        // 处理中间的完整块
        for (int k = bl + 1; k < br; ++k) {
            block_lazy[k] += x; // 更新懒惰标记
            // 根据具体操作，更新 block_info
            // 例如：区间加，区间求和时，block_info[k] += x * (block_end[k] - block_start[k] + 1);
            // 例如：区间加，区间求最大值时，block_info[k] 可能需要特殊处理或不更新
            block_info[k] += (long long)x * (block_end[k] - block_start[k] + 1); // 示例：区间加对和的影响
        }
    }
}

// 函数：区间 [L, R] 查询操作 (例如：区间求和)
// L, R 为 0-indexed
long long query_range(int L, int R) {
    long long ans = 0; // 存储查询结果
    int bl = block_id[L]; // L 所在的块
    int br = block_id[R]; // R 所在的块

    // 如果 L 和 R 在同一个块内
    if (bl == br) {
        for (int i = L; i <= R; ++i) {
            ans += a[i]; // 累加数组元素
        }
        ans += (long long)(R - L + 1) * block_lazy[bl]; // 加上懒惰标记的贡献
    } else {
        // 处理 L 所在的不完整块
        for (int i = L; i <= block_end[bl]; ++i) {
            ans += a[i];
        }
        ans += (long long)(block_end[bl] - L + 1) * block_lazy[bl]; // 加上懒惰标记的贡献

        // 处理 R 所在的不完整块
        for (int i = block_start[br]; i <= R; ++i) {
            ans += a[i];
        }
        ans += (long long)(R - block_start[br] + 1) * block_lazy[br]; // 加上懒惰标记的贡献

        // 处理中间的完整块
        for (int k = bl + 1; k < br; ++k) {
            ans += block_info[k]; // 累加块的概括信息
            ans += (long long)(block_end[k] - block_start[k] + 1) * block_lazy[k]; // 加上懒惰标记的贡献
        }
    }
    return ans;
}


int main() {
    // 优化C++标准库IO，在CSP/NOIP中常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> q;

    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    build(); // 初始化分块结构

    for (int op_idx = 0; op_idx < q; ++op_idx) {
        int op_type;
        std::cin >> op_type;
        if (op_type == 1) { // 示例：区间加操作
            int L, R, x;
            std::cin >> L >> R >> x;
            update_range(L - 1, R - 1, x); // 假设题目是1-indexed，这里转换为0-indexed
        } else { // 示例：区间求和操作
            int L, R;
            std::cin >> L >> R;
            std::cout << query_range(L - 1, R - 1) << "\n"; // 假设题目是1-indexed，这里转换为0-indexed
        }
    }

    return 0;
}
```
