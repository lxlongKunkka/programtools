# 迭代加深搜索（IDA*）

## 教学目标

*   **知识目标:**
    *   理解迭代加深搜索（IDDFS）的基本原理，及其如何结合DFS的低空间复杂度和BFS的最短路径特性。
    *   掌握A\*算法的核心思想，特别是启发式函数 `h(n)` 在评估节点优劣中的作用。
    *   深入理解迭代加深A\*搜索（IDA\*）的算法流程，包括其迭代深度（或成本上限）的确定、启发式函数的选择与设计，以及剪枝策略。
    *   明确IDA\*算法的适用场景和优缺点。
*   **能力目标:**
    *   能够分析给定搜索问题，判断是否适合使用IDA\*算法。
    *   独立设计并实现满足问题的状态表示、状态转移函数以及关键的启发式函数 `h(n)`。
    *   熟练编写IDA\*的核心搜索函数 `dfs`，并正确实现剪枝逻辑。
    *   能够对IDA\*算法的时间和空间复杂度进行初步分析。
*   **素养目标:**
    *   培养学生在面对复杂搜索问题时，权衡不同算法优劣（时间、空间、最优解）的计算思维。
    *   提升抽象问题、建模问题和设计高效解决方案的能力。
    *   激发对算法优化的兴趣，体会“组合”不同算法思想（DFS、BFS、A\*）以解决难题的智慧。

## 深度知识点讲解

### 1. 回顾：DFS与BFS的优劣

*   **深度优先搜索 (DFS):**
    *   **优点:** 空间复杂度低，因为它只需要存储当前路径上的节点。
    *   **缺点:**
        *   不保证找到最短路径。如果目标在很深的某个分支，DFS会先探索完这个分支，即使旁边有很浅的路径。
        *   可能陷入无限循环或过深的路径，尤其是在有环的图中。
*   **广度优先搜索 (BFS):**
    *   **优点:** 保证找到最短路径（如果每条边的权重相同）。因为它总是先探索距离起点近的节点。
    *   **缺点:** 空间复杂度高，因为它需要存储所有已访问但未扩展的节点（队列）。对于状态空间巨大的问题，这常常是不可接受的。

### 2. IDDFS：迭代加深深度优先搜索 (Iterative Deepening DFS)

IDDFS 是 DFS 和 BFS 思想的巧妙结合。它模拟了 BFS 寻找最短路径的能力，同时保持了 DFS 的低空间复杂度。

**核心思想:**
IDDFS 不断地增加一个深度限制 `max_d`，然后在这个深度限制内进行 DFS。
1.  首先，执行一个深度限制为 `d=1` 的 DFS。
2.  如果没找到目标，就执行一个深度限制为 `d=2` 的 DFS。
3.  如果还没找到，就执行一个深度限制为 `d=3` 的 DFS。
4.  ...
5.  直到找到目标为止。

**生活类比：**
就像你在一个多层迷宫里找出口。
*   DFS：你一头扎进一个岔路，一直走到底，直到无路可走才回头。可能绕远路，也可能陷入死胡同。
*   BFS：你把所有岔路都探索一遍，先走一步能到的地方，再走两步能到的地方……直到找到出口。但你得记住所有岔路口，脑子要爆炸。
*   IDDFS：你先规定自己只能走1步，看看有没有出口。没有？好，再规定自己只能走2步，看看有没有出口。还没有？再规定只能走3步……直到找到出口。这样每次探索的深度有限，不会走太远，而且每一步都从头开始，不用记住太多信息（省内存）。虽然每次都“重复”探索了浅层路径，但由于搜索树的叶子节点数量远大于内部节点，重复的开销相对较小，而且找到了最短路径！

**IDDFS的优点:**
*   空间复杂度低，和 DFS 一样，为 O(d)，d 为找到最短路径的深度。
*   能找到最短路径，和 BFS 一样。

**IDDFS的缺点:**
*   对于每个深度限制 `d`，都会从头开始搜索，重复计算了许多节点。但通常这种重复计算的开销可以接受。
*   在没有启发式函数的情况下，仍然是盲目搜索，效率可能不高。

### 3. A\* 算法的核心：启发式搜索

在介绍 IDA\* 之前，我们先简单回顾一下 A\* 算法。A\* 算法是一种启发式搜索算法，它在 BFS 的基础上，加入了对未来代价的“预判”。

**核心公式:** `f(n) = g(n) + h(n)`
*   `n`: 当前节点。
*   `g(n)`: 从起点到当前节点 `n` 的实际代价（已走过的步数或成本）。
*   `h(n)`: 从当前节点 `n` 到目标节点的估算代价（启发式函数）。
*   `f(n)`: 从起点经过 `n` 到目标节点的总估算代价。

A\* 算法总是优先扩展 `f(n)` 值最小的节点。

**启发式函数 `h(n)` 的重要性：**
*   `h(n)` 必须是**可采纳的 (Admissible)**：它永远不能高估从 `n` 到目标节点的实际代价。如果 `h(n)` 高估了，A\* 算法可能错过最优解。
    *   例如：在寻路问题中，曼哈顿距离或欧几里得距离就是可采纳的启发式，因为直线距离或轴向距离总是小于等于实际路径距离。
*   `h(n)` 越准确 (越接近实际代价但又不超过)，A\* 搜索效率越高。如果 `h(n)` 总是 0，A\* 就退化成了 BFS。

### 4. IDA\*：迭代加深A\*搜索 (Iterative Deepening A\*)

IDA\* 是 IDDFS 与 A\* 算法的结合体。它在 IDDFS 的基础上引入了 A\* 的启发式函数，使得搜索不再是盲目的，而是带有“方向感”的。

**核心思想:**
IDA\* 不再简单地限制搜索的深度 `d`，而是限制搜索的**总估算代价 `max_f`**。这个 `max_f` 就是 A\* 算法中的 `f(n)` 值。

**算法流程:**
1.  **初始化 `max_f`：** 从起点的 `f(start) = g(start) + h(start)` 开始，通常 `g(start) = 0`，所以 `max_f = h(start)`。
2.  **迭代搜索：**
    *   在一个循环中，每次迭代增加 `max_f` 的值。
    *   在每次迭代中，执行一个**深度优先搜索** `dfs(u, current_g, max_f)`。
    *   在 `dfs` 函数中，我们不仅检查深度限制，更重要的是检查**估算总代价限制**：
        *   如果 `current_g + h(u) > max_f`，则立即剪枝，不再探索该分支。这表示即使从当前节点 `u` 走最短的估算路径，也已经超出了当前迭代允许的总代价上限 `max_f`。
        *   如果 `current_g + h(u)` 达到目标，且 `current_g` 是当前迭代中找到的最小 `g` 值，则找到解。
    *   如果当前迭代的 `dfs` 找到了目标，则算法结束，当前 `max_f` 就是最短路径的代价。
    *   如果当前迭代的 `dfs` 没有找到目标，但遇到了被剪枝的节点（即 `current_g + h(u)` 超过了 `max_f`），记录所有这些超限的 `f(n)` 值中的最小值，作为下一次迭代的 `max_f`。
3.  **更新 `max_f`：** 如果当前迭代没有找到解，将 `max_f` 更新为上一次迭代中所有被剪枝的 `f(n)` 值中的最小值（这个最小值是所有超出当前 `max_f` 的 `f(n)` 中最接近 `max_f` 的那个）。这保证了下一次迭代会探索到刚好超过当前 `max_f` 的那些节点。

**生活类比：**
你还是在找迷宫出口，但这次你带了一个“预判器”。
*   你设定一个“总花费上限” `max_f`。
*   你开始探索，每走到一个地方，你就计算：`已走步数(g) + 预计到达出口还需要多少步(h)`。
*   如果 `g + h` 超过了你设定的 `max_f`，你立刻掉头，不再往这条路走了（剪枝）。因为你预判这条路无论如何都会超支。
*   如果所有路都走完了，还没找到出口，你就把 `max_f` 稍微提高一点点（提高到上次探索中，那些“差点就够了”的路径的 `g+h` 的最小值），再重新探索。
*   这样，你既不会走太远（DFS的优点），又能沿着最有希望的路径前进（A\*的优点），而且每次迭代的 `max_f` 都是递增的，确保你最终会找到最短路径。

**IDA\*的优点:**
*   **空间复杂度低:** 和 DFS 一样，为 O(d)，d 为找到最短路径的深度。
*   **能找到最短路径:** 和 BFS/A\* 一样。
*   **效率高:** 结合了启发式函数，比 IDDFS 更快地找到目标。

**IDA\*的缺点:**
*   需要一个好的、可采纳的启发式函数 `h(n)`。启发式函数的设计是 IDA\* 成败的关键。
*   每次迭代会重复计算浅层节点，但通常其效率仍高于纯 BFS。

**关键点总结:**
1.  **状态表示:** 如何将问题抽象成计算机可以处理的状态。
2.  **状态转移:** 如何从一个状态到达另一个状态（即“移动”）。
3.  **启发式函数 `h(n)`:**
    *   **可采纳性 (Admissibility):** `h(n)` 必须永远不大于从 `n` 到目标节点的实际最短代价。这是保证找到最优解的前提。
    *   **效率:** `h(n)` 的计算速度要快。
    *   **准确性:** `h(n)` 越接近真实代价，搜索效率越高。
4.  **`dfs` 函数的参数:** 通常包括 `当前状态`、`当前已走代价g`、`当前迭代的f值上限max_f`。
5.  **剪枝条件:** `if (current_g + h(u) > max_f) return;`

**常见误区:**
*   **启发式函数不可采纳:** 如果 `h(n)` 高估了真实代价，IDA\* 可能在达到最优解之前就剪枝了最优路径，导致找不到最优解或找到次优解。
*   **`max_f` 的更新逻辑错误:** `max_f` 应该从 `h(start)` 开始，每次迭代更新为上一次搜索中，所有剪枝点的 `g(n) + h(n)` 的最小值。
*   **忘记处理目标状态:** 在 `dfs` 函数中，达到目标状态时需要立即返回或记录解。
*   **状态重复访问问题:** 如果问题存在环，且 `g(n)` 不仅仅是步数，而是累加的成本，需要考虑如何避免重复访问或处理环。对于纯步数问题，IDA\* 的迭代加深特性本身就能处理循环，因为 `g(n)` 不断增加，会超过 `max_f` 限制。但如果搜索空间很小，或者状态的哈希代价不高，也可以考虑用 `std::set` 或哈希表记录已访问状态。不过对于GSP 9级，通常状态空间巨大，哈希表会爆内存，IDA\* 自身的剪枝足够。

## 典型例题精讲

### 例题一：八数码问题 (8-Puzzle)

**题目描述:**
在一个 3x3 的棋盘上，摆放着 1~8 这 8 个数字，还有一个空格。你可以将空格旁边的数字移动到空格中。目标是将打乱的棋盘通过最少步数移动，使其变为目标状态（例如：
1 2 3
4 5 6
7 8 _
）。

**输入:**
一个 3x3 的矩阵，表示初始状态。例如：
2 8 3
1 6 4
7 _ 5
（其中 `_` 表示空格）

**输出:**
最少移动步数。如果无解，输出 -1。

**思路分析:**

1.  **状态表示:**
    *   一个 3x3 的 `char` 数组 `a[3][3]` 或 `int` 数组 `a[3][3]` 来表示棋盘状态。
    *   为了方便处理和哈希（如果需要，但IDA\*通常不显式哈希），也可以将其展平为一个长度为 9 的字符串或 `int` 数组。例如，将棋盘按行连接成 "2831647_5"。
    *   需要记录空格的位置 `(x, y)`。

2.  **目标状态:**
    *   `12345678_` 或对应的数组。

3.  **为什么IDA\*？**
    *   BFS 可以找到最短路径，但 3x3 棋盘的状态数是 9! / 2 = 181440，如果扩展到 4x4 (15-puzzle)，状态数是 16! / 2，BFS 队列会爆内存。
    *   DFS 虽然省内存，但无法保证找到最短路径。
    *   IDA\* 结合了两者优点，且八数码问题存在很好的启发式函数。

4.  **启发式函数 `h(n)` 的设计 (关键):**
    *   **曼哈顿距离 (Manhattan Distance):** 对于棋盘上的每个数字，计算它当前位置到目标位置的水平距离和垂直距离之和。所有数字的曼哈顿距离之和就是 `h(n)`。
    *   例如，数字 `1` 目标位置是 `(0,0)`。如果它当前在 `(1,1)`，那么它的曼哈顿距离是 `|1-0| + |1-0| = 2`。
    *   **为什么可采纳？** 每次移动，只能将一个数字从一个格子移动到相邻的格子。每次移动最多只能减少一个数字的曼哈顿距离（当该数字向目标位置靠近时），或者保持不变，或者增加。因此，所有数字的曼哈顿距离之和，总是小于等于达到目标状态所需的实际移动步数。所以它是可采纳的。
    *   **计算 `h(n)`:** 遍历 1-8 每个数字，找到它当前在棋盘上的位置 `(cx, cy)`，然后找到它在目标棋盘上的位置 `(tx, ty)`，累加 `abs(cx - tx) + abs(cy - ty)`。

5.  **IDA\* 算法流程:**
    *   **迭代上限 `max_f`:** 从 `h(initial_state)` 开始，每次增加。
    *   **`dfs(current_state, current_g, max_f)` 函数:**
        *   `current_g`: 当前已走步数。
        *   `current_h`: 计算 `current_state` 的曼哈顿距离。
        *   **剪枝条件:** `if (current_g + current_h > max_f) return;`
        *   **目标判断:** `if (current_h == 0) { found_solution = true; return; }` （当所有数字都在正确位置时，`h` 为 0）
        *   **扩展节点:** 找到空格位置 `(sx, sy)`。尝试向上、下、左、右移动空格（即交换空格和相邻数字）。对于每种移动，生成新状态，递归调用 `dfs`。
        *   **记录下一次迭代的 `max_f` 最小值:** 在 `dfs` 返回时，如果 `found_solution` 为 `false`，则记录所有 `current_g + current_h` 超过 `max_f` 的最小值 `next_max_f`。
    *   **主循环:**
        ```cpp
        for (max_f = h(initial_state); ; max_f = next_max_f) {
            next_max_f = 2e9; // 足够大的数
            found_solution = false;
            // 找到空格位置
            // 调用 dfs(initial_state, 0, max_f)
            if (found_solution) {
                // 输出 max_f (其实是 current_g)
                break;
            }
            if (next_max_f == 2e9) { // 意味着所有分支都走完了，但没找到解
                // 无解，或者 max_f 已经很大了，可以认为无解
                break;
            }
        }
        ```
    *   **判断无解:** 八数码问题有奇偶性判断。一个状态能否到达目标状态，取决于其逆序对的数量（忽略空格）加上空格所在行号（从0开始）的奇偶性。如果初始状态和目标状态的奇偶性不同，则无解。对于 3x3 棋盘，如果逆序对数量为奇数，则无解；如果为偶数，则有解。

**代码实现 (简化版，仅展示核心结构):**

```cpp
#include <bits/stdc++.h>

using namespace std;

// 定义棋盘状态，用一个长度为9的char数组表示
// 例如 "12345678x" (x代表空格)
char a[9];
char t[9] = {'1', '2', '3', '4', '5', '6', '7', '8', 'x'}; // 目标状态

int dx[] = {-1, 1, 0, 0}; // 移动方向：上、下、左、右
int dy[] = {0, 0, -1, 1};

int limit; // IDA*的f值上限 (g + h)
int next_limit; // 下一次迭代的f值上限
bool found;     // 是否找到解

// 预计算每个数字的目标位置
int target_pos[9][2]; // target_pos[num-'0'][0]是行, [num-'0'][1]是列

// 计算当前状态的启发式函数 h(n) - 曼哈顿距离
int h() {
    int s = 0;
    for (int i = 0; i < 9; ++i) {
        if (a[i] == 'x') continue;
        int num = a[i] - '0';
        // 当前位置 (i/3, i%3)
        // 目标位置 (target_pos[num][0], target_pos[num][1])
        s += abs(i / 3 - target_pos[num][0]) + abs(i % 3 - target_pos[num][1]);
    }
    return s;
}

// IDA*的DFS部分
// k: 当前步数 g(n)
// p: 空格当前在a数组中的位置 (0-8)
void dfs(int k, int p) {
    // k是当前g(n), h()是当前h(n)
    int current_f = k + h();

    // 剪枝条件
    if (current_f > limit) {
        next_limit = min(next_limit, current_f); // 记录下一次迭代的最小f值
        return;
    }

    // 找到目标状态
    if (h() == 0) {
        found = true;
        return;
    }

    // 尝试移动空格
    int sx = p / 3, sy = p % 3; // 空格的当前行列坐标

    for (int i = 0; i < 4; ++i) {
        int nx = sx + dx[i];
        int ny = sy + dy[i];

        if (nx >= 0 && nx < 3 && ny >= 0 && ny < 3) {
            int np = nx * 3 + ny; // 相邻位置在a数组中的索引

            // 交换空格和相邻数字
            swap(a[p], a[np]);

            // 递归搜索
            dfs(k + 1, np);

            // 如果已经找到解，则直接返回，无需回溯更多分支
            if (found) return;

            // 回溯：恢复棋盘状态
            swap(a[p], a[np]);
        }
    }
}

            }
        }
    }
    return s;
}

    }

    // IDA*主循环
    limit = h(); // 初始f值上限为初始状态的h值
```

**复杂度分析:**
*   **时间复杂度:** 很难给出精确的数学表达式。在最坏情况下，IDA\* 可能会重复访问许多节点，但由于启发式函数的剪枝作用，其效率通常远高于纯 IDDFS。对于 8 数码问题，通常可以在毫秒级完成。对于 15 数码问题，可能需要几秒到几十秒。它取决于启发式函数的准确性。
*   **空间复杂度:** O(d)，其中 `d` 是最短路径的深度。这是因为 `dfs` 递归栈的深度最多为 `d`。相比于 BFS 的 O(b^d) (b是分支因子)，IDA\* 在空间上具有巨大优势。

### 例题二：魔板 (Magic Board)

**题目描述:**
有一个 2x4 的魔板，共 8 个格子，里面放着 1~8 的数字。允许三种操作：
A: 上下两行交换。
B: 最右边一列移动到最左边。
C: 中间四格顺时针旋转。
给定初始状态，求最少步数达到目标状态 `12348765`。

**输入:**
一行 8 个数字，表示初始状态。例如 `86417253`。

**输出:**
最少步数。

**思路分析:**

1.  **状态表示:**
    *   一个长度为 8 的 `char` 数组或 `string` 来表示魔板状态。例如 `s[0..7]`。
    *   目标状态通常是 `12348765`。

2.  **为什么IDA\*？**
    *   魔板的状态数是 8! = 40320。这个数量虽然不大，BFS 也能跑，但是如果问题规模稍微大一点，或者操作更复杂，BFS 的内存压力就会出现。这里用 IDA\* 来训练启发式函数设计和更复杂的 DFS 状态转移。
    *   BFS 虽然能过，但 IDA\* 仍然是学习启发式搜索的好例子。

3.  **操作函数:**
    *   `opA(s)`: 交换 `s[0], s[1], s[2], s[3]` 与 `s[7], s[6], s[5], s[4]`。
    *   `opB(s)`: `s[3]` 移到 `s[0]`，`s[0]` 移到 `s[1]`，`s[1]` 移到 `s[2]`，`s[2]` 移到 `s[3]`； `s[4]` 移到 `s[5]`，`s[5]` 移到 `s[6]`，`s[6]` 移到 `s[7]`，`s[7]` 移到 `s[4]`。
    *   `opC(s)`: `s[1]` 移到 `s[2]`，`s[2]` 移到 `s[6]`，`s[6]` 移到 `s[5]`，`s[5]` 移到 `s[1]`。

4.  **启发式函数 `h(n)` 的设计 (关键):**
    *   **错位个数 (Misplaced Tiles):** 计算当前状态中有多少个数字不在目标位置上。
    *   **为什么可采纳？** 每一步操作，最多只能让 8 个数字中的一部分移动到正确位置。所以，错位数字的个数总是小于等于实际所需步数。它是可采纳的。
    *   **计算 `h(n)`:** 遍历当前状态 `s`，与目标状态 `t` 逐位比较，统计 `s[i] != t[i]` 的个数。

5.  **IDA\* 算法流程:**
    *   与八数码类似，只是状态转移函数和启发式函数不同。
    *   由于状态数相对较小，为了避免重复搜索，可以在 `dfs` 中增加一个 `last_op` 参数，避免连续执行互逆操作（例如，A操作后紧接着执行A操作）。
    *   由于状态数 8! = 40320，可以考虑用 `std::map<string, int>` 或 `std::unordered_map<string, int>` 记录 `visited` 状态，以避免重复搜索（虽然IDA\*本身通过 `g+h > limit` 剪枝，但对于这类问题，记录 `visited` 可以进一步加速）。但根据题目要求，**不能使用 STL 容器**。那么就只能依靠 `g+h` 剪枝和操作限制来优化。

**代码实现 (仅展示核心结构，省略部分细节):**

```cpp
#include <bits/stdc++.h>

using namespace std;

char initial_s[9]; // 初始状态
char target_s[9] = {'1', '2', '3', '4', '8', '7', '6', '5'}; // 目标状态

int limit; // IDA*的f值上限 (g + h)
int next_limit; // 下一次迭代的f值上限
bool found;     // 是否找到解

// h(n) - 错位个数
int h(char s[9]) {
    int cnt = 0;
    for (int i = 0; i < 8; ++i) {
        if (s[i] != target_s[i]) {
            cnt++;
        }
    }
    return cnt;
}

// 操作A：上下两行交换
void opA(char s[9]) {
    for (int i = 0; i < 4; ++i) {
        swap(s[i], s[7 - i]);
    }
}

// 操作B：最右边一列移动到最左边
void opB(char s[9]) {
    char temp_s[9];
    for(int i=0; i<8; ++i) temp_s[i] = s[i];
    
    // 上行循环右移
    s[0] = temp_s[3];
    s[1] = temp_s[0];
    s[2] = temp_s[1];
    s[3] = temp_s[2];

    // 下行循环右移
    s[4] = temp_s[7];
    s[5] = temp_s[4];
    s[6] = temp_s[5];
    s[7] = temp_s[6];
}

// 操作C：中间四格顺时针旋转
void opC(char s[9]) {
    char temp_s[9];
    for(int i=0; i<8; ++i) temp_s[i] = s[i];

    s[1] = temp_s[5];
    s[2] = temp_s[1];
    s[6] = temp_s[2];
    s[5] = temp_s[6];
}

// IDA*的DFS部分
// k: 当前步数 g(n)
// s: 当前状态
// last_op: 上一步操作的类型，用于避免冗余操作 (0:无, 1:A, 2:B, 3:C)
void dfs(int k, char s[9], int last_op) {
    int current_h = h(s);
    int current_f = k + current_h;

    // 剪枝条件
    if (current_f > limit) {
        next_limit = min(next_limit, current_f);
        return;
    }

    // 找到目标状态
    if (current_h == 0) {
        found = true;
        return;
    }

    char original_s[9]; // 保存当前状态，用于回溯
    for(int i=0; i<8; ++i) original_s[i] = s[i];

    // 尝试操作A
    opA(s);
    dfs(k + 1, s, 1);
    if (found) return;
    for(int i=0; i<8; ++i) s[i] = original_s[i]; // 回溯

    // 尝试操作B
    opB(s);
    dfs(k + 1, s, 2);
    if (found) return;
    for(int i=0; i<8; ++i) s[i] = original_s[i]; // 回溯

    // 尝试操作C
    opC(s);
    dfs(k + 1, s, 3);
    if (found) return;
    for(int i=0; i<8; ++i) s[i] = original_s[i]; // 回溯
}

int main() {
    for (int i = 0; i < 8; ++i) {
        cin >> initial_s[i];
    }
    initial_s[8] = '\0'; // 字符串结束符

    // IDA*主循环
    limit = h(initial_s); // 初始f值上限为初始状态的h值

    while (!found) {
        next_limit = 2e9; // 初始化下一次迭代的最小f值
        char current_s_copy[9];
        for(int i=0; i<9; ++i) current_s_copy[i] = initial_s[i];
        
        dfs(0, current_s_copy, 0); // 从0步开始搜索, last_op=0表示无操作

        if (found) {
            cout << limit << endl; // 找到解，输出当前limit (即最短路径g值)
            break;
        }
        if (next_limit == 2e9) { // 如果next_limit没有被更新，说明所有分支都低于当前limit或已经探索完，无解
            cout << -1 << endl; // 魔板问题一般有解，但为了通用性保留
            break;
        }
        limit = next_limit; // 更新f值上限
    }

    return 0;
}
```

**复杂度分析:**
*   **时间复杂度:** 同样依赖于启发式函数的准确性。对于 8! 这种规模，可以非常快地找到解。
*   **空间复杂度:** O(d)，`d` 为最短路径深度。
