## 深度知识点讲解

### 1. 什么是贪心算法？（快速回顾）

我们之前学过贪心算法，它的核心思想是：在每一步都做出当前看起来最优的选择，希望这些局部最优选择能够最终导向全局最优解。
**生活类比：** 比如你饿了，面前有各种食物。简单贪心就是：哪个食物看起来最诱人，就先吃哪个！

### 2. 贪心算法为什么会“失败”？

虽然贪心算法很直观，但在很多情况下，局部最优并不等于全局最优。
**举个例子：**
你有一个任务，需要完成3个子任务A、B、C。
*   任务A：耗时1小时，奖励100元。
*   任务B：耗时2小时，奖励10元。
*   任务C：耗时2小时，奖励200元。
你总共只有3小时时间。
**简单贪心策略（按单位时间奖励最高）：**
1.  任务A (1小时, 100元)。剩余2小时。
2.  任务C (2小时, 200元)。剩余0小时。
总奖励：100 + 200 = 300元。

**现在换一种策略：**
1.  任务C (2小时, 200元)。剩余1小时。
2.  任务A (1小时, 100元)。剩余0小时。
总奖励：200 + 100 = 300元。

**再看一个可能失败的例子：**
你有一个背包，最多装 10kg。有三件物品：
*   物品A：重量 5kg，价值 10元。
*   物品B：重量 6kg，价值 100元。
*   物品C：重量 5kg，价值 10元。
**简单贪心策略（每次选择价值最高的）：**
1.  选择物品B (6kg, 100元)。背包剩余 4kg。
2.  无法再选择物品A或C，因为它们都超过 4kg。
总价值：100元。

**如果不是这样选呢？**
1.  选择物品A (5kg, 10元)。背包剩余 5kg。
2.  选择物品C (5kg, 10元)。背包剩余 0kg。
总价值：10 + 10 = 20元。

在这个例子中，简单贪心（每次选价值最高的）虽然没有失败，但如果我们换成另一种贪心策略（比如先选价值密度最高的），或者背包容量稍微调整一下，就可能出现问题。

**更典型的失败例子：**
你有一些零钱：[1, 5, 10, 25] 美分硬币。现在要凑 30 美分。
**简单贪心（每次选择面值最大的）：**
1.  选择 25 美分。还需要 5 美分。
2.  选择 5 美分。凑齐 30 美分。
总共 2 枚硬币。这是最优解。

**如果硬币是：[1, 5, 12] 美分，凑 15 美分呢？**
**简单贪心：**
1.  选择 12 美分。还需要 3 美分。
2.  选择 1 美分。还需要 2 美分。
3.  选择 1 美分。还需要 1 美分。
4.  选择 1 美分。凑齐 15 美分。
总共 4 枚硬币 (12+1+1+1)。

**但最优解是：** 3 枚 5 美分硬币 (5+5+5)。
看，简单贪心在这里就“失败”了！因为它在选择 12 美分时，没有考虑到后续选择的局限性。

### 3. 什么是反悔贪心？

当简单贪心无法得到全局最优解时，我们就要考虑“反悔”机制了。
**反悔贪心的核心思想是：**
1.  **先做决定：** 按照某种贪心策略，做出当前的局部最优决策。
2.  **留有余地：** 但这个决策不是“一锤定音”的，我们保留了“撤销”它的可能性。
3.  **择优反悔：** 当后续出现一个更好的选择，并且这个新选择能够弥补（甚至超过）之前决策的“损失”时，我们就撤销之前的某个决策（通常是之前做出的最差决策），转而选择新的。

**生活类比：**
就像我们一开始的“宝物探险”故事。你先捡了一件宝物，因为它当时是最好的。但后来你发现了一件更好的，为了最大化总价值，你决定把背包里最差的宝物扔掉，换上这件新宝物。这就是“反悔”！你“后悔”了之前捡最差宝物的决定，并用更好的替代了它。

### 4. 如何实现反悔机制？——优先队列（堆）登场！

要实现“反悔”，关键在于：我们如何高效地找到之前“最不优”的决策并替换它？
这里，**优先队列（Priority Queue）**就成了我们的得力助手！

*   **优先队列是什么？**
    你可以把它想象成一个特殊队列，每次取出的不是最早进入的元素，而是优先级最高的元素。
    *   **大根堆 (Max-Heap)：** 优先级最高的元素是数值最大的，每次取出最大值。
    *   **小根堆 (Min-Heap)：** 优先级最高的元素是数值最小的，每次取出最小值。

*   **在反悔贪心中的应用：**
    假设我们要最大化总收益，并且有容量限制（比如只能选 K 件商品）。
    1.  我们遍历所有商品，每遇到一件商品，我们先“假装”把它选上。
    2.  如果当前已选商品数量还没达到 K，那就直接选。
    3.  如果已经达到 K 件了，这时新商品来了，我们就要比较：新商品的价值 vs. 已选商品中价值最低的那个。
        *   如果新商品价值更高，我们就“反悔”：把已选商品中价值最低的那个“扔掉”，换成新商品。
        *   为了高效地找到并移除已选商品中价值最低的那个，我们就需要一个**小根堆**！小根堆的堆顶元素就是当前已选商品中价值最低的。

**C++ `std::priority_queue` 的使用：**
*   默认是大根堆：`std::priority_queue<int> pq;`
*   小根堆：`std::priority_queue<int, std::vector<int>, std::greater<int>> pq;`
    *   `std::vector<int>` 是底层容器。
    *   `std::greater<int>` 是比较器，表示“更大”的元素优先级更低（即数值小的优先级高），从而形成小根堆。

### 5. 关键点剖析

*   **识别问题：**
    *   问题通常涉及在有限资源（如时间、数量 K）下，最大化或最小化某个目标。
    *   简单贪心策略在局部看起来最优，但可能导致错过全局最优。
    *   决策具有“可撤销性”，即一个已做的决策可以被另一个更好的决策替换。
*   **确定反悔条件：**
    *   什么时候“反悔”？比如，当新出现的元素比已选择元素中的某个更优时。
    *   “反悔”的代价是什么？通常是移除一个旧元素，增加一个新元素。
*   **选择数据结构：**
    *   通常使用优先队列来维护已做出的决策集合。
    *   如果要“反悔”掉最小的元素，用小根堆。
    *   如果要“反悔”掉最大的元素（这种情况较少见），用大根堆。

### 6. 常见误区

*   **过度使用反悔贪心：** 并非所有贪心问题都需要反悔，有些问题简单贪心就够了。要先尝试证明简单贪心的正确性，如果失败再考虑反悔。
*   **优先队列类型选择错误：** 如果目标是最大化收益，通常用小根堆来存储已选元素（以便替换掉最差的）。如果目标是最小化成本，则可能用大根堆来存储已选元素（以便替换掉最贵的）。
*   **反悔逻辑不清晰：** 什么时候入队？什么时候出队？什么时候比较？这些步骤必须非常明确。

## 典型例题精讲

### 例题1：挑选商品 (Maximize Value with K items)

**问题描述：**
你逛商场，有 N 件商品，每件商品有一个价值 `v_i`。你的购物车最多可以装 K 件商品。商品是按顺序一件一件展示给你的。每展示一件商品，你必须立刻决定是购买它还是放弃它。如果你购买它，并且购物车已满 K 件，你可以选择把购物车里价值最低的商品退掉，换成这件新商品。你的目标是最大化购买商品的总价值。

**输入：**
第一行两个整数 N 和 K (1 <= K <= N <= 100,000)。
第二行 N 个整数，表示 N 件商品的价值 `v_i` (1 <= `v_i` <= 1,000,000)。

**输出：**
一个整数，表示能获得的最大总价值。

**思路分析：**
这是一个非常典型的反悔贪心问题。
1.  **贪心策略：** 每次遇到一件商品，我们都希望尽可能把它加入购物车，因为它的价值是正的。
2.  **反悔机制：**
    *   如果购物车还没满 K 件，那就直接买，把它加入购物车。
    *   如果购物车已经满了 K 件，这时新来的商品 `v_new` 怎么办？我们不能直接扔掉它，也不能直接放弃。我们需要比较 `v_new` 和购物车里“最不划算”的那件商品（也就是价值最低的 `v_min`）。
        *   如果 `v_new > v_min`，那么我们就可以进行“反悔”操作：把 `v_min` 扔掉，换成 `v_new`。这样总价值会增加 `v_new - v_min`。
        *   如果 `v_new <= v_min`，说明新商品不如购物车里最差的，那就直接放弃 `v_new`。
3.  **数据结构：** 为了高效地找到购物车里价值最低的商品，我们需要一个**小根堆**。小根堆的堆顶元素就是最小价值的商品。

**步骤：**
1.  初始化一个 `long long` 类型的变量 `totalValue` 为 0，用于记录总价值。
2.  创建一个小根堆 `std::priority_queue<int, std::vector<int>, std::greater<int>> pq;`。
3.  遍历 N 件商品：
    a.  读取当前商品价值 `v`。
    b.  如果 `pq.size() < k` (购物车未满)：
        *   将 `v` 加入 `pq`。
        *   `totalValue += v`。
    c.  如果 `pq.size() == k` (购物车已满)：
        *   比较 `v` 和 `pq.top()` (购物车中价值最低的商品)。
        *   如果 `v > pq.top()`：
            *   `totalValue -= pq.top()` (从总价值中减去被替换掉的商品价值)。
            *   `pq.pop()` (移除价值最低的商品)。
            *   `totalValue += v` (将新商品价值加到总价值)。
            *   `pq.push(v)` (将新商品加入购物车)。
4.  最终 `totalValue` 就是最大总价值。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

void solveExample1() {
    int n, k;
    std::cin >> n >> k; // 读取商品数量 N 和购物车容量 K

    long long totalValue = 0; // 使用 long long 防止总价值溢出
    // 定义一个小根堆，用于存储已购买商品的价值
    // std::priority_queue<Type, Container, Compare>
    // Type: 存储元素的类型
    // Container: 底层容器，默认为 std::vector
    // Compare: 比较函数对象，std::greater<Type> 表示小根堆
    std::priority_queue<int, std::vector<int>, std::greater<int>> pq;

    for (int i = 0; i < n; ++i) {
        int v;
        std::cin >> v; // 读取当前商品的价值

        // 情况1：购物车未满 K 件
        if (pq.size() < k) {
            pq.push(v);         // 直接购买，加入购物车
            totalValue += v;    // 累加到总价值
        }
        // 情况2：购物车已满 K 件
        else {
            // 如果当前商品价值 v 比购物车里最便宜的商品 (pq.top()) 还要高
            if (v > pq.top()) {
                totalValue -= pq.top(); // 从总价值中减去最便宜的商品价值
                pq.pop();               // 移除最便宜的商品
                totalValue += v;        // 将新商品价值加到总价值
                pq.push(v);             // 将新商品加入购物车
            }
            // 否则（新商品价值不比最便宜的更高），放弃当前商品，不进行任何操作
        }
    }

    std::cout << totalValue << std::endl; // 输出最大总价值
}

// int main() {
//     solveExample1();
//     return 0;
// }
```

**复杂度分析：**
*   **时间复杂度：** 遍历 N 件商品，每件商品最多进行一次优先队列的 `push` 或 `pop` 操作。优先队列的这两种操作的时间复杂度都是 O(log M)，其中 M 是优先队列中元素的数量。在这里，M 最多为 K。所以总时间复杂度为 O(N log K)。
*   **空间复杂度：** 优先队列最多存储 K 个元素，所以空间复杂度为 O(K)。

### 例题2：任务调度 (Job Sequencing with Deadlines)

**问题描述：**
你有 N 份工作，每份工作 `i` 有一个截止日期 `d_i` 和一份报酬 `p_i`。你每天只能完成一份工作。在截止日期前完成工作可以获得报酬，否则报酬为0。你的目标是最大化总报酬。

**输入：**
第一行一个整数 N (1 <= N <= 100,000)。
接下来 N 行，每行两个整数 `d_i` 和 `p_i` (1 <= `d_i` <= N, 1 <= `p_i` <= 1,000,000)。

**输出：**
一个整数，表示能获得的最大总报酬。

**思路分析：**
这个经典的“工作调度”问题也可以用反悔贪心来解决。
1.  **贪心策略：** 我们希望尽可能安排高报酬的工作。
2.  **反悔机制：**
    *   首先，将所有工作按照它们的截止日期 `d_i` **从小到大**排序。这样，我们在处理工作时，总是优先考虑那些截止日期较早的工作。
    *   遍历排序后的工作。对于当前工作 `(d, p)`：
        *   我们尝试安排它。如果当前已经安排的工作数量 `pq.size()` 小于当前工作的截止日期 `d`，说明我们可以在 `d` 之前找到一个空闲的日子来完成它。所以，我们直接安排它，并将其报酬 `p` 加入到一个小根堆 `pq` 中。
        *   如果 `pq.size()` 已经等于 `d`，这意味着在截止日期 `d` 之前，我们已经安排了 `d` 份工作。此时，如果当前工作 `p` 的报酬比 `pq` 中最小的报酬（即已安排工作中报酬最低的）还要高，那么我们就可以“反悔”：将 `pq` 中报酬最低的工作移除，然后安排当前工作 `p`。
    *   这样，我们就能保证在每个截止日期 `d` 之前，我们总是选择了报酬最高的 `d` 份工作。
3.  **数据结构：** 同样，为了高效地找到并替换掉已安排工作中报酬最低的那个，我们需要一个**小根堆**。

**步骤：**
1.  定义一个结构体 `Job` 来存储 `d` 和 `p`。
2.  将所有工作读入 `std::vector<Job>` 中。
3.  对 `jobs` 向量按截止日期 `d` **从小到大**排序。
4.  初始化一个 `long long` 类型的变量 `totalProfit` 为 0。
5.  创建一个小根堆 `std::priority_queue<int, std::vector<int>, std::greater<int>> pq;`。
6.  遍历排序后的每份工作 `j` (截止日期 `j.d`，报酬 `j.p`)：
    a.  如果 `pq.size() < j.d` (当前已安排工作数量小于截止日期)：
        *   将 `j.p` 加入 `pq`。
        *   `totalProfit += j.p`。
    b.  如果 `pq.size() == j.d` (当前已安排工作数量已达到截止日期限制)：
        *   比较 `j.p` 和 `pq.top()` (已安排工作中报酬最低的)。
        *   如果 `j.p > pq.top()`：
            *   `totalProfit -= pq.top()`。
            *   `pq.pop()`。
            *   `totalProfit += j.p`。
            *   `pq.push(j.p)`。
7.  最终 `totalProfit` 就是最大总报酬。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// 定义工作结构体
struct Job {
    int d; // deadline 截止日期
    int p; // profit 报酬
};

// 比较函数，用于按截止日期从小到大排序
bool compareJobs(const Job& a, const Job& b) {
    return a.d < b.d;
}

void solveExample2() {
    int n;
    std::cin >> n; // 读取工作数量 N

    std::vector<Job> jobs(n);
    for (int i = 0; i < n; ++i) {
        std::cin >> jobs[i].d >> jobs[i].p; // 读取每份工作的截止日期和报酬
    }

    // 将工作按截止日期从小到大排序
    std::sort(jobs.begin(), jobs.end(), compareJobs);

    long long totalProfit = 0; // 使用 long long 防止总报酬溢出
    // 定义一个小根堆，用于存储已安排工作的报酬
    std::priority_queue<int, std::vector<int>, std::greater<int>> pq;

    for (int i = 0; i < n; ++i) {
        // 获取当前工作
        int currentDeadline = jobs[i].d;
        int currentProfit = jobs[i].p;

        // 情况1：当前已安排的工作数量小于当前工作的截止日期
        // 意味着在截止日期前还有空闲时间可以安排这个工作
        if (pq.size() < currentDeadline) {
            pq.push(currentProfit);     // 直接安排，加入优先队列
            totalProfit += currentProfit; // 累加到总报酬
        }
        // 情况2：当前已安排的工作数量已经达到当前工作的截止日期
        // 意味着在截止日期前已经安排了 currentDeadline 份工作
        else {
            // 如果当前工作的报酬比已安排工作中报酬最低的还要高
            if (currentProfit > pq.top()) {
                totalProfit -= pq.top(); // 从总报酬中减去报酬最低的工作
                pq.pop();                // 移除报酬最低的工作
                totalProfit += currentProfit; // 将当前工作报酬加到总报酬
                pq.push(currentProfit);      // 将当前工作加入优先队列
            }
            // 否则（当前工作报酬不比最低的更高），放弃当前工作，不进行任何操作
        }
    }

    std::cout << totalProfit << std::endl; // 输出最大总报酬
}
```

**复杂度分析：**
*   **时间复杂度：**
    *   读取输入和存储：O(N)。
    *   排序：O(N log N)。
    *   遍历 N 份工作，每份工作最多进行一次优先队列的 `push` 或 `pop` 操作。优先队列中元素的数量最多为 N (因为截止日期 `d_i` 最多为 N)。所以这部分时间复杂度为 O(N log N)。
    *   总时间复杂度为 O(N log N)。
*   **空间复杂度：** `jobs` 向量存储 N 个工作，优先队列最多存储 N 个报酬，所以空间复杂度为 O(N)。
