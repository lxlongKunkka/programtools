## 趣味引入

同学们，你们有没有玩过那种“魔法数字”的游戏？比如说，给你几个数字，像 `1`、`2`、`4`，然后让你用它们通过异或运算，组合出其他数字。
`1 ^ 2 = 3`
`1 ^ 4 = 5`
`2 ^ 4 = 6`
`1 ^ 2 ^ 4 = 7`
看，我们用 `1, 2, 4` 这三个数字，居然能组合出 `1, 2, 3, 4, 5, 6, 7` 所有的数字！这就像它们是这个数字世界的“基因”，通过不同的组合（异或），就能创造出各种“形态”的数字。

但如果我给你的数字是 `1, 2, 3` 呢？
`1`
`2`
`3` (可以由 `1 ^ 2` 得到)
`1 ^ 2 = 3`
`1 ^ 3 = 2`
`2 ^ 3 = 1`
`1 ^ 2 ^ 3 = 0`
你会发现，`3` 这个数字其实并不是“独特”的，它可以用 `1` 和 `2` 异或出来。我们只需要 `1` 和 `2`，就能生成 `1, 2, 3` 能生成的所有数字，甚至更多！

那么问题来了，如果我给你一堆乱七八糟的数字，比如 `6, 3, 10, 5, 9`，我们怎么才能找到一个最小的、最“独特”的数字集合，让它们能通过异或运算，组合出原集合所有可能异或和？而且，如果我想从这些数字里，选出一些，让它们的异或和最大，又该怎么做呢？

今天，我们就来学习一种神奇的“数字基因库”技术——**异或线性基**！它能帮我们找到那个最精简、最核心的“基因”集合，并且利用它解决各种复杂的异或问题！

## 深度知识点讲解

### 1. 异或线性基 (XOR Linear Basis) 是什么？

我们先从一个更抽象的角度理解。在数学的线性代数中，有一个概念叫做“基”（Basis）。它指的是一个向量空间中，一组线性无关的向量，并且这个空间中的任何向量都可以由这组向量线性组合得到。

对于异或运算来说，我们可以把它看作是在GF(2)（伽罗瓦域2）上的加法。每个数字的二进制表示可以看作是一个向量，每一位是向量的一个分量。异或运算就是在这个域上的“向量加法”。
那么，**异或线性基**就是从一个给定的整数集合 `S` 中，找到一个最小的子集 `B = {b_1, b_2, ..., b_k}`，满足以下两个条件：
1.  **线性无关性（异或无关性）**：在 `B` 中，任何一个 `b_i` 都不能由 `B` 中其他元素的异或组合得到。
2.  **生成性**：原集合 `S` 中任何元素，或者 `S` 中任意元素的异或和，都可以由 `B` 中元素的异或组合得到。
3.  `B` 是满足上述条件的所有集合中，元素个数最少的。

简单来说，异或线性基就是原集合 `S` 的一个“压缩版”或“精华版”。它包含了原集合所有“独特”的异或信息，并且没有冗余。

### 2. 异或线性基的性质

1.  **最小性**：异或线性基的大小是唯一的，且是原集合能生成的所有异或和中，所需最少元素个数的集合。
2.  **生成性**：原集合中所有能通过异或组合得到的数字，都可以由线性基中的元素异或组合得到。反之亦然。
3.  **唯一表示（简化形式）**：如果我们将线性基表示为“简化行阶梯形矩阵”（类似高斯消元后的形式），那么每个能被表示的数，其表示方式是唯一的。通常，我们会让每个基元素 `b_i` 的最高有效位（MSB）是独一无二的。

### 3. 异或线性基的构造方法（高斯消元思想）

构造异或线性基的核心思想，类似于高斯消元法。我们维护一个基数组 `basis[]`，其中 `basis[i]` 存储的是最高位为第 `i` 位的基元素（如果有的话）。

假设我们处理的数字最大不超过 `2^D - 1`（例如，`int` 类型 `D=31`，`long long` 类型 `D=63`）。那么我们的基数组 `basis` 就可以开 `D` 位。`basis[i]` 代表一个最高位在第 `i` 位的基元素。

**算法步骤：**
对于集合中的每一个数字 `x`：
1.  从最高位 `D-1` 遍历到最低位 `0`。
2.  检查 `x` 的第 `j` 位是否为 `1`。如果不是，跳过。
3.  如果 `x` 的第 `j` 位是 `1`：
    *   **情况1：** 如果 `basis[j]` 还没有元素（即 `basis[j] == 0`），那么 `x` 可以作为第 `j` 位的基元素。将 `basis[j] = x`，然后跳出循环，处理下一个数字。
    *   **情况2：** 如果 `basis[j]` 已经有元素了，说明 `basis[j]` 已经“占据”了第 `j` 位。为了保持基的线性无关性，我们需要用 `basis[j]` 来“消除” `x` 的第 `j` 位。将 `x = x ^ basis[j]`。然后继续检查 `x` 的下一位（`j-1`位）。
这个过程会一直进行，直到 `x` 变为 `0`（表示 `x` 可以被现有基组合出来），或者 `x` 找到了一个它独有的最高位并被加入到基中。

**生活类比：**
想象你有一堆不同型号的“万能钥匙”（数字 `x`），每把钥匙能开不同型号的锁（二进制位）。你想要建立一个最精简的“万能钥匙库”（`basis` 数组）。
*   当你拿到一把新钥匙 `x` 时，你从最高级的锁（最高位）开始尝试。
*   如果 `x` 能开 `j` 型号的锁，并且钥匙库里还没有能开 `j` 型号锁的钥匙 (`basis[j] == 0`)，那么 `x` 就成了你的“`j` 型万能钥匙”，把它放进钥匙库 `basis[j]`。
*   如果钥匙库里已经有 `j` 型万能钥匙 `basis[j]` 了，那么 `x` 的 `j` 型功能就是多余的。你可以用 `basis[j]` 来“改造” `x`，让 `x` 失去 `j` 型功能（`x = x ^ basis[j]`）。然后，你带着改造后的 `x`，继续尝试它能开的下一个最高级锁（`j-1` 型），看看它还有没有其他独特的功能。
*   如果 `x` 经过一系列改造，最后变得什么锁都开不了了 (`x == 0`)，说明它所有的功能都被钥匙库里的钥匙替代了，它不是一把“独特”的钥匙，就不需要加入钥匙库了。

### 4. 异或线性基的应用：求最大异或和

有了线性基，求一个集合能生成的最大异或和就变得非常简单了。

**算法步骤：**
1.  首先，按照上述方法构造出异或线性基 `basis`。
2.  初始化 `res = 0`。
3.  从最高位 `D-1` 遍历到最低位 `0`。
4.  对于 `basis[j]`：
    *   如果 `res ^ basis[j]` 能够比当前的 `res` 更大，那么就更新 `res = res ^ basis[j]`。
    *   如何判断 `res ^ basis[j]` 是否更大？其实就是看 `res` 当前的最高位是不是 `basis[j]` 的最高位。如果 `basis[j]` 的最高位还没被 `res` 覆盖，那么异或上 `basis[j]` 肯定能让 `res` 变得更大。
    *   更直观的判断是：`res = max(res, res ^ basis[j])`。由于我们是从高位往低位贪心，只要 `basis[j]` 能让 `res` 的某个高位从 `0` 变成 `1`，那么就一定取它。
    *   所以，实际上是：`res = res ^ basis[j]`。 (这里需要注意，`basis` 数组里的元素可能不是“纯粹”的，即 `basis[j]` 的最高位是 `j`，但它可能还有更低的位，这些低位可能与 `res` 已有的低位冲突。正确的贪心策略是：从高位到低位遍历 `basis[j]`，如果 `res` 的第 `j` 位是 `0` 且 `basis[j]` 存在，那么 `res = res ^ basis[j]`。但更通用且正确的做法是：`res = max(res, res ^ basis[j])`，因为 `basis[j]` 已经被处理成只有最高位 `j` 是独有的了。)

**正确的贪心策略是：**
从最高位 `D-1` 遍历到最低位 `0`。
如果 `basis[j]` 存在（不为0）：
    `res = max(res, res ^ basis[j])`。
为什么要这样？因为我们希望尽可能让结果的高位是 `1`。当我们遍历到 `basis[j]` 时，它保证了在第 `j` 位是 `1`。如果 `res` 异或上 `basis[j]` 后，能让 `res` 的第 `j` 位从 `0` 变为 `1`，那么这个操作就是有利的。更一般地，如果 `res ^ basis[j]` 比 `res` 大，就说明 `basis[j]` 能帮助 `res` 获得更高的位权。

### 5. 时间复杂度分析

*   **构造线性基**：假设有 `N` 个数字，每个数字最大为 `MAX_VAL`。`MAX_VAL` 的二进制位数是 `D = log2(MAX_VAL)`。
    *   对于每个数字，我们需要在 `D` 位上进行循环和异或操作。
    *   所以，构造线性基的时间复杂度是 `O(N * D)`，即 `O(N * log MAX_VAL)`。
*   **查询最大异或和**：线性基构造完成后，只需要遍历 `D` 位，进行 `D` 次异或操作。
    *   时间复杂度是 `O(D)`，即 `O(log MAX_VAL)`。

对于 `int` 类型，`D` 大约是 31；对于 `long long` 类型，`D` 大约是 63。这是一个非常高效的算法。

### 6. 常见误区

*   **基元素的顺序**：在插入数字时，从高位到低位遍历 `j` 是为了确保 `basis[j]` 存储的是最高位为 `j` 的基元素，并且它能帮助我们处理其他数字的第 `j` 位。
*   **线性基的简化**：一些实现中会将 `basis` 进一步简化，使得 `basis[j]` 异或上 `basis[k]` (k < j) 来消除 `basis[j]` 的第 `k` 位。但对于求最大异或和，这种额外的简化并不是必须的，因为贪心策略 `res = max(res, res ^ basis[j])` 已经足够强大。我们这里采用的是“半简化”形式，即 `basis[j]` 的最高位是 `j`，且这个 `j` 位是它独有的（通过其他元素消除）。

## 典型例题精讲

### 例题 1：构造异或线性基并求集合所有子集的最大异或和

**题目描述：**
给定 `n` 个非负整数 `a_1, a_2, ..., a_n`。求从这 `n` 个数中选择任意个数，使得它们的异或和最大。

**输入格式：**
第一行一个整数 `n` (1 <= n <= 50)。
第二行 `n` 个整数 `a_i` (0 <= a_i <= 10^18)。

**输出格式：**
一个整数，表示最大异或和。

**思路分析：**
1.  **问题转化**：题目要求从给定集合中选择任意个数，使它们的异或和最大。这正是异或线性基的经典应用之一。我们只需要先构造出原集合的异或线性基，然后利用线性基来贪心地构造最大异或和。
2.  **数据范围**：`a_i` 最大 `10^18`，这超出了 `int` 的范围，需要使用 `long long`。`n` 比较小，`50` 个数。
3.  **线性基构造**：
    *   定义一个 `long long basis[64]` 数组，初始化为 `0`。`64` 是因为 `long long` 有 `64` 位。
    *   遍历输入的 `n` 个数字 `x`。
    *   对于每个 `x`，从 `62` 位（`long long` 的最高位通常是 `63`，但一般从 `62` 或 `61` 开始处理，因为 `0` 到 `63` 共 `64` 位，最高位是 `63`，为了避免索引越界或习惯性从 `62` 开始）向下遍历到 `0` 位。
    *   如果 `x` 的第 `j` 位是 `1`：
        *   如果 `basis[j]` 为 `0`，则 `basis[j] = x`，并跳出内层循环。
        *   如果 `basis[j]` 不为 `0`，则 `x = x ^ basis[j]`。
4.  **最大异或和查询**：
    *   初始化 `res = 0`。
    *   从 `62` 位向下遍历到 `0` 位。
    *   如果 `basis[j]` 不为 `0`：
        *   `res = max(res, res ^ basis[j])`。这个贪心策略是正确的：我们从高位开始，如果 `basis[j]` 能让 `res` 的第 `j` 位从 `0` 变为 `1`，那么 `res` 异或上 `basis[j]` 肯定会更大。即使 `basis[j]` 可能会影响 `res` 的低位，但高位的影响更大，所以这种贪心是有效的。

**C++ 代码实现：**

```cpp
#include <iostream>
#include <algorithm> // for std::max

// GESP 9级不允许使用vector等STL容器，所以使用原生数组
// 线性基数组，long long 类型有64位，所以数组大小设为64
long long basis[64];

// 函数：插入一个数到线性基中
void insert(long long x) {
    // 从最高位（62位）开始遍历到最低位（0位）
    // long long 最大值约 2^63 - 1，所以最高位是63位。
    // 通常从62或63开始遍历，这里我们从62开始，涵盖所有可能位。
    for (int j = 62; j >= 0; --j) {
        // 如果 x 的第 j 位是 0，则跳过
        if (!((x >> j) & 1)) {
            continue;
        }

        // 如果 basis[j] 还没有元素，说明 x 可以作为第 j 位的基
        if (basis[j] == 0) {
            basis[j] = x;
            return; // 插入成功，处理下一个数字
        } else {
            // 如果 basis[j] 已经有元素，则用 basis[j] 消去 x 的第 j 位
            x ^= basis[j];
        }
    }
}

// 函数：查询集合中子集的最大异或和
long long query_max_xor_sum() {
    long long res = 0;
    // 从最高位开始遍历线性基
    for (int j = 62; j >= 0; --j) {
        // 如果 basis[j] 存在（不为0）
        if (basis[j] != 0) {
            // 贪心策略：如果异或上 basis[j] 能让结果更大，就异或它
            // 这样能保证我们尽可能地让高位为1
            res = std::max(res, res ^ basis[j]);
        }
    }
    return res;
}

int main() {
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    // 清空线性基数组，确保每次运行都是新的状态
    // GESP 允许使用 memset。这里手动清零也行，因为是全局变量，默认就是0
    for (int i = 0; i < 64; ++i) {
        basis[i] = 0;
    }

    for (int i = 0; i < n; ++i) {
        long long a;
        std::cin >> a;
        insert(a); // 将每个数字插入线性基
    }

    long long max_xor = query_max_xor_sum(); // 查询最大异或和
    std::cout << max_xor << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度**：
    *   初始化 `basis` 数组：`O(D)`，其中 `D` 是 `long long` 的位数 (64)。
    *   插入 `N` 个数字：每个数字插入 `O(D)`。总共 `O(N * D)`。
    *   查询最大异或和：`O(D)`。
    *   总时间复杂度：`O(N * D)`。对于 `N=50, D=64`，大约 `50 * 64 = 3200` 次操作，非常快。
*   **空间复杂度**：
    *   `basis` 数组：`O(D)`。非常小，`64 * sizeof(long long)`。

### 例题 2：判断一个数是否能被异或线性基表示

**题目描述：**
给定 `n` 个非负整数 `a_1, a_2, ..., a_n`。再给定 `q` 个查询，每个查询给出一个数字 `x`，判断 `x` 是否能由 `a_1, ..., a_n` 中的某些数异或得到。

**输入格式：**
第一行一个整数 `n` (1 <= n <= 50)。
第二行 `n` 个整数 `a_i` (0 <= a_i <= 10^18)。
第三行一个整数 `q` (1 <= q <= 100)。
接下来 `q` 行，每行一个整数 `x` (0 <= x <= 10^18)。

**输出格式：**
对于每个查询，如果能表示则输出 "YES"，否则输出 "NO"。

**思路分析：**
1.  **问题转化**：一个数 `x` 能否被原集合的元素异或得到，等价于 `x` 能否被异或线性基的元素异或得到。
2.  **线性基构造**：同例题1，先构造出异或线性基 `basis`。
3.  **查询判断**：
    *   对于每个查询的数字 `x`，尝试用线性基中的元素去“消灭” `x` 的每一位。
    *   从最高位 `62` 向下遍历到 `0`。
    *   如果 `x` 的第 `j` 位是 `1` 并且 `basis[j]` 存在，则 `x = x ^ basis[j]`。
    *   如果经过所有位的消灭后，`x` 最终变成了 `0`，说明 `x` 可以被线性基中的元素异或得到，输出 "YES"。
    *   否则，如果 `x` 最终不为 `0`，说明 `x` 无法被线性基表示，输出 "NO"。

**C++ 代码实现：**

```cpp
#include <iostream>
#include <algorithm> // for std::max

long long basis[64];

void insert(long long x) {
    for (int j = 62; j >= 0; --j) {
        if (!((x >> j) & 1)) {
            continue;
        }
        if (basis[j] == 0) {
            basis[j] = x;
            return;
        } else {
            x ^= basis[j];
        }
    }
}

// 函数：判断一个数是否能被线性基表示
bool can_represent(long long x) {
    // 尝试用线性基中的元素消去 x 的每一位
    for (int j = 62; j >= 0; --j) {
        // 如果 x 的第 j 位是 1，并且 basis[j] 存在
        if (((x >> j) & 1) && (basis[j] != 0)) {
            x ^= basis[j]; // 用 basis[j] 消去 x 的第 j 位
        }
    }
    // 如果最终 x 变为 0，说明可以被表示
    return x == 0;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    for (int i = 0; i < 64; ++i) {
        basis[i] = 0;
    }

    for (int i = 0; i < n; ++i) {
        long long a;
        std::cin >> a;
        insert(a);
    }

    int q;
    std::cin >> q;
    for (int i = 0; i < q; ++i) {
        long long x;
        std::cin >> x;
        if (can_represent(x)) {
            std::cout << "YES\n";
        } else {
            std::cout << "NO\n";
        }
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度**：
    *   构造线性基：`O(N * D)`。
    *   `Q` 次查询：每次查询 `O(D)`。总共 `O(Q * D)`。
    *   总时间复杂度：`O(N * D + Q * D)`。对于 `N=50, Q=100, D=64`，大约 `(50 + 100) * 64 = 9600` 次操作，非常快。
*   **空间复杂度**：`O(D)`。

## 代码实现模板

```cpp
#include <iostream> // 用于输入输出
#include <algorithm> // 用于 std::max

// 定义一个全局的线性基数组
// long long 类型最大到 2^63 - 1，所以需要 64 位来表示
// 数组索引可以从 0 到 63，共 64 个元素
long long basis[64]; 

// 常量定义，表示数字的最大位数
// 对于 long long，通常是 63 或 62，取决于你如何处理最高位
// 这里我们从 62 位向下遍历，所以最高有效位可以看作是 62
const int MAX_BITS = 62; 

// 函数：初始化线性基
// 通常全局变量默认初始化为 0，但显式初始化是个好习惯
void init_basis() {
    for (int i = 0; i <= MAX_BITS; ++i) {
        basis[i] = 0;
    }
}

// 函数：将一个数字 x 插入到线性基中
// 参数 x: 要插入的非负整数
void insert_num(long long x) {
    // 从数字的最高位开始，向下遍历到最低位
    // 这样可以确保 basis[j] 存储的是最高位为 j 的基元素
    for (int j = MAX_BITS; j >= 0; --j) {
        // 检查 x 的第 j 位是否为 1
        // (x >> j) & 1 会得到第 j 位的二进制值 (0 或 1)
        if (!((x >> j) & 1)) {
            continue; // 如果第 j 位是 0，则跳过，检查下一位
        }

        // 如果 basis[j] 为 0，说明目前还没有基元素能够“占据”第 j 位
        // 那么 x 就是一个独一无二的基元素，将其放入 basis[j]
        if (basis[j] == 0) {
            basis[j] = x;
            return; // 插入成功，这个数字 x 已经找到了它的“归宿”，结束
        } else {
            // 如果 basis[j] 不为 0，说明第 j 位已经被 basis[j] 占据了
            // 为了保持线性基的“线性无关性”，我们需要用 basis[j] 来“消除” x 的第 j 位
            // x = x ^ basis[j] 相当于用 basis[j] 来改造 x，让 x 的第 j 位变为 0
            x ^= basis[j];
            // 改造后的 x 可能在更低位有新的最高位，所以继续循环检查更低位
        }
    }
    // 如果循环结束后 x 变为 0，说明 x 可以完全由现有线性基中的元素异或得到
    // 也就是说，x 是“多余”的，不需要加入线性基
}

// 函数：查询由线性基元素能组合出的最大异或和
// 返回值: 最大异或和
long long query_max_xor_sum() {
    long long res = 0; // 初始化结果为 0
    // 从最高位开始，向下遍历线性基
    // 贪心策略：尽可能让结果的高位为 1
    for (int j = MAX_BITS; j >= 0; --j) {
        // 如果 basis[j] 不为 0 (即存在一个基元素其最高位在 j)
        if (basis[j] != 0) {
            // 尝试将当前结果 res 与 basis[j] 异或
            // 如果 res ^ basis[j] 比当前的 res 更大，说明异或上 basis[j] 是有利的
            // 因为 basis[j] 的最高位是 j，它能帮助 res 在高位上获得 1
            res = std::max(res, res ^ basis[j]);
        }
    }
    return res; // 返回最终的最大异或和
}

// 函数：判断一个数字 x 是否能被线性基表示
// 参数 x: 要判断的数字
// 返回值: true 如果能表示，false 如果不能
bool can_represent_num(long long x) {
    // 尝试用线性基中的元素消去 x 的所有位
    for (int j = MAX_BITS; j >= 0; --j) {
        // 如果 x 的第 j 位是 1 且 basis[j] 存在
        if (((x >> j) & 1) && (basis[j] != 0)) {
            x ^= basis[j]; // 用 basis[j] 消去 x 的第 j 位
        }
    }
    // 经过所有消去操作后，如果 x 最终变为 0，则说明 x 可以被线性基表示
    return x == 0;
}
```

## 课堂互动

1.  **思考题：异或的奥秘**
    *   “同学们，我们知道 `a ^ a = 0`，`a ^ 0 = a`。如果给你两个数 `A` 和 `B`，它们的异或和是 `C` (`A ^ B = C`)。那么 `A ^ C` 会是什么？`B ^ C` 呢？这说明异或运算有什么神奇的性质？”
    *   **引导**：`A ^ C = A ^ (A ^ B) = (A ^ A) ^ B = 0 ^ B = B`。`B ^ C = B ^ (A ^ B) = (B ^ B) ^ A = 0 ^ A = A`。这说明异或运算具有“可逆性”和“自反性”，非常适合用于编码和消除。

2.  **小组任务：手动构建线性基**
    *   “现在，我们来玩一个手动构建线性基的游戏。请每组同学拿到以下数字：`{6, 3, 10}`。请按照我们刚刚学到的高斯消元法，一步步地构建出这个集合的异或线性基。”
    *   **提示**：`6` (0110), `3` (0011), `10` (1010)。请记录下每一步 `x` 和 `basis` 的变化。
    *   **预期过程**：
        *   `basis = {0,0,0,0...}`
        *   插入 `6` (0110): `basis[2] = 6`
        *   插入 `3` (0011): `x=3`。`j=2`: `3` 的第 2 位是 `0`。`j=1`: `3` 的第 1 位是 `1`。`basis[1]==0`，所以 `basis[1]=3`。
        *   插入 `10` (1010): `x=10`。`j=3`: `10` 的第 3 位是 `1`。`basis[3]==0`，所以 `basis[3]=10`。
        *   最终线性基可能为 `{basis[3]=10, basis[2]=6, basis[1]=3}`。

3.  **即时练习：最大异或和**
    *   “基于刚才你们小组构建的线性基 `{basis[3]=10, basis[2]=6, basis[1]=3}`，请计算这个集合能产生的最大异或和是多少？”
    *   **引导**：从高位到低位贪心。`res=0`。
        *   `j=3`: `basis[3]=10` (1010)。`res = max(0, 0^10) = 10`。
        *   `j=2`: `basis[2]=6` (0110)。`res = max(10, 10^6)`。`10` (1010) `^` `6` (0110) = `12` (1100)。`res = 12`。
        *   `j=1`: `basis[1]=3` (0011)。`res = max(12, 12^3)`。`12` (1100) `^` `3` (0011) = `15` (1111)。`res = 15`。
        *   `j=0`: `basis[0]=0`。
    *   最终结果：`15`。

4.  **讨论环节：为什么贪心法求最大异或和是正确的？**
    *   “为什么我们从高位到低位遍历线性基，每次都选择 `res = max(res, res ^ basis[j])` 就能得到最大异或和？有没有可能我们为了让低位变成 `1` 而放弃了高位，结果反而更大？”
    *   **引导**：在线性基中，每个 `basis[j]` 都是一个“独一无二”的基元素，它的最高位是 `j`。当我们从高位 `j` 开始处理时，如果 `res ^ basis[j]` 能让 `res` 的第 `j` 位从 `0` 变为 `1`，那么这个操作对结果的贡献是最大的（因为 `2^j` 是当前最高位能获得的最高权值）。即使这个操作可能会影响 `res` 的低位，但高位的权值远大于所有低位的权值之和。所以，先处理高位，确保高位尽可能为 `1`，是正确的贪心策略。

1.  **编程题：最大异或路径**
    给定一个无向图，每条边上有一个权值。从起点 `S` 到终点 `T` 的一条路径的异或和定义为路径上所有边权值的异或。现在允许你走任意多圈（即可以重复经过边和点）。求从 `S` 到 `T` 的所有可能路径中，最大异或和是多少。
    **提示**：图中的任意环路（异或和）都可以加入到线性基中。从 `S` 到 `T` 找到一条任意路径 `P`，其异或和为 `val_P`。那么所有其他路径 `S->T` 的异或和都可以表示为 `val_P ^ (所有环路的异或组合)`。所以，我们只需要找到一条 `S` 到 `T` 的路径，并把图中所有环路的异或和加入线性基，然后用 `val_P` 去查询线性基能得到的最大异或和。
    **输入样例：**
    `2 2` (n=2点, m=2边)
    `1 2 5` (边1-2，权值5)
    `1 2 3` (边1-2，权值3)
    `1 2` (查询S=1, T=2)
    **输出样例：**
    `7` (路径1-2权值5，路径1-2权值3。环路：5^3=6。任意路径5，与线性基{6}组合，max(5, 5^6) = max(5, 3) = 5。如果路径是3，与线性基{6}组合，max(3, 3^6) = max(3, 5) = 5。不对。应该把路径5和环路6都插入线性基。线性基是{5, 6}。最大异或和是 max(0, 0^5)=5, max(5, 5^6)=7。所以是7。)
    **更正提示**：将所有边权值插入线性基，然后找到所有环路的异或和也插入线性基。最后查询最大异或和。

1.  **编程题：K-th 小的异或和**

**基础巩固 1：**
a) `S = {1, 2, 4, 8, 15}`
    *   `basis = {0, ...}`
    *   插入 `1` (0001): `basis[0] = 1`
    *   插入 `2` (0010): `basis[1] = 2`
    *   插入 `4` (0100): `basis[2] = 4`
    *   插入 `8` (1000): `basis[3] = 8`

        *   `j=3`: `15` 的第 3 位是 `1`，`basis[3]=8`。`x = 15 ^ 8 = 7` (0111)
        *   `j=2`: `7` 的第 2 位是 `1`，`basis[2]=4`。`x = 7 ^ 4 = 3` (0011)
        *   `j=1`: `3` 的第 1 位是 `1`，`basis[1]=2`。`x = 3 ^ 2 = 1` (0001)
        *   `j=0`: `1` 的第 0 位是 `1`，`basis[0]=1`。`x = 1 ^ 1 = 0`
    *   最终线性基：`basis = {basis[0]=1, basis[1]=2, basis[2]=4, basis[3]=8}` (其实就是 `1, 2, 4, 8`，`15` 可以由它们组合得到)
b) 最大异或和：
    `res=0`
    `j=3`: `basis[3]=8`. `res = max(0, 0^8) = 8`.
    `j=2`: `basis[2]=4`. `res = max(8, 8^4) = 12`.
    `j=1`: `basis[1]=2`. `res = max(12, 12^2) = 14`.
    `j=0`: `basis[0]=1`. `res = max(14, 14^1) = 15`.
    最大异或和为 `15`。
c) 判断 `7` 和 `13`：
    *   `7` (0111):
        *   `j=3`: `7` 的第 3 位是 `0`。
        *   `j=2`: `7` 的第 2 位是 `1`，`basis[2]=4`. `x = 7 ^ 4 = 3`.
        *   `j=1`: `3` 的第 1 位是 `1`，`basis[1]=2`. `x = 3 ^ 2 = 1`.
        *   `j=0`: `1` 的第 0 位是 `1`，`basis[0]=1`. `x = 1 ^ 1 = 0`.
        *   最终 `x=0`，所以 `7` 可以被表示。
    *   `13` (1101):
        *   `j=3`: `13` 的第 3 位是 `1`，`basis[3]=8`. `x = 13 ^ 8 = 5` (0101).
        *   `j=2`: `5` 的第 2 位是 `1`，`basis[2]=4`. `x = 5 ^ 4 = 1` (0001).
        *   `j=1`: `1` 的第 1 位是 `0`。
        *   `j=0`: `1` 的第 0 位是 `1`，`basis[0]=1`. `x = 1 ^ 1 = 0`.
        *   最终 `x=0`，所以 `13` 可以被表示。

```cpp
#include <iostream>
#include <algorithm>

long long basis[64];
const int MAX_BITS = 62;

void init_basis() {
    for (int i = 0; i <= MAX_BITS; ++i) {
        basis[i] = 0;
    }
}

void insert_num(long long x) {
    for (int j = MAX_BITS; j >= 0; --j) {
        if (!((x >> j) & 1)) {
            continue;
        }
        if (basis[j] == 0) {
            basis[j] = x;
            return;
        } else {
            x ^= basis[j];
        }
    }
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    init_basis();

    int n;
    std::cin >> n;

    for (int i = 0; i < n; ++i) {
        long long a;
        std::cin >> a;
        insert_num(a);
    }

    for (int j = MAX_BITS; j >= 0; --j) {
        if (basis[j] != 0) {
            std::cout << basis[j] << " ";
        }
    }
    std::cout << std::endl;

    return 0;
}
```
