## 深度知识点讲解

### 1. 公平游戏 (Impartial Games) 的本质

首先，我们得知道什么是“公平游戏”。
**定义**：公平游戏是一种双人游戏，满足以下条件：
1.  **轮流操作**：两人交替进行操作。
2.  **信息完全**：所有游戏信息对双方都是可见的（没有隐藏信息）。
3.  **无随机性**：没有骰子、抽牌等随机因素。
4.  **有限步数结束**：游戏会在有限步内结束，不会出现平局。
5.  **可选操作相同**：在任何给定的局面下，当前玩家可选择的操作集合与轮到谁操作无关（这是“公平”的关键！）。
6.  **正常游戏规则 (Normal Play)**：最后一个能进行操作的玩家获胜。

**例子**：
*   **是公平游戏**：Nim游戏、取石子游戏（每次取1-k个）、Wythoff游戏。
*   **不是公平游戏**：象棋、围棋（黑白双方棋子不同）、扑克牌（有隐藏信息）、大富翁（有骰子）。

### 2. P-位置与N-位置：胜负的DNA

在公平游戏中，每个局面都可以被标记为两种状态之一：
*   **P-位置 (Previous Player Winning Position)**：如果你面对这个局面，那么先前的那个玩家（也就是你的对手）有必胜策略。对你来说，这是一个必败局面。
*   **N-位置 (Next Player Winning Position)**：如果你面对这个局面，那么当前玩家（也就是你）有必胜策略。对你来说，这是一个必胜局面。

**怎么判断P/N位置呢？**
我们可以用递归的方式来定义它们：
1.  **终局是P-位置**：任何不能再进行操作的局面都是P-位置（因为当前玩家无法操作，所以前一个玩家是最后一个操作的，从而获胜）。
2.  **N-位置的定义**：如果一个局面能通过一次操作到达至少一个P-位置，那么它就是N-位置。
3.  **P-位置的定义**：如果一个局面无论如何操作，都只能到达N-位置，那么它就是P-位置。

**生活类比**：想象你和朋友在玩一个“陷阱”游戏。P-位置就像一个陷阱，你掉进去了，无论怎么挣扎，都只能进入下一个陷阱。N-位置就像你站在陷阱边缘，你总能找到一条路，把你的对手推到陷阱里。

### 3. Nim 游戏与 Nim 和 (XOR Sum)

现在，我们回到经典的Nim游戏：有n堆石子，每堆有a_i颗。两人轮流操作，每次从一堆中取走任意正数颗石子。最后一个取走石子的人获胜。

**Nim和 (Nim Sum)**：Nim游戏的胜负判断有一个神奇的工具，叫做Nim和。它就是所有石子堆大小的**异或和**。
`Nim Sum = a_1 ^ a_2 ^ ... ^ a_n` (其中 `^` 表示异或操作)

**Nim游戏的必胜策略**：
*   如果当前局面的Nim和不为0，那么当前玩家有必胜策略（这是一个N-位置）。
*   如果当前局面的Nim和为0，那么当前玩家没有必胜策略（这是一个P-位置）。

**为什么Nim和能决定胜负？**
这是Nim游戏最核心的理论，由Bouton在1901年证明。我们不展开严格的数学证明，但可以从直观上理解其精髓：

1.  **终局**：当所有石子堆都为空时，局面是 `0, 0, ..., 0`，Nim和是 `0 ^ 0 ^ ... ^ 0 = 0`。这是一个P-位置（因为当前玩家无子可取，前一个玩家取走了最后一颗）。
2.  **从Nim和非0到Nim和为0**：如果当前Nim和 `S != 0`，当前玩家总能找到一种操作，使得操作后的Nim和变为0。
    *   **怎么找？** 找到 `S` 的最高位1，假设这一位是第 `k` 位。那么至少有一堆石子 `a_i` 的第 `k` 位也是1。
    *   我们选择这样一堆 `a_i`，并且将其变为 `a_i' = a_i ^ S`。
    *   因为 `a_i` 和 `S` 的第 `k` 位都是1，所以 `a_i ^ S` 的第 `k` 位是0，并且 `a_i ^ S < a_i`（因为 `S` 的最高位1在 `k` 位，而 `a_i` 的第 `k` 位也是1，异或后这一位变成0，意味着 `a_i ^ S` 比 `a_i` 小）。
    *   所以，从 `a_i` 中取出 `a_i - a_i'` 颗石子，就能让当前Nim和变成 `(a_1 ^ ... ^ a_i') ^ ... ^ a_n = (a_1 ^ ... ^ (a_i ^ S)) ^ ... ^ a_n = S ^ S = 0`。
3.  **从Nim和为0到Nim和非0**：如果当前Nim和 `S = 0`，无论当前玩家如何操作，从某一堆 `a_i` 中取出一些石子变为 `a_i'`（`a_i' < a_i`），新的Nim和 `S' = S ^ a_i ^ a_i' = 0 ^ a_i ^ a_i' = a_i ^ a_i'` 必然不为0。
    *   **为什么？** 如果 `a_i ^ a_i' = 0`，则 `a_i = a_i'`，但这与“取走了石子” (`a_i' < a_i`) 矛盾。

**总结**：Nim和为0的局面是P-位置，Nim和不为0的局面是N-位置。当前玩家总是试图将Nim和变为0，并将这个P-位置留给对手。

### 4. Sprague-Grundy 定理与 SG 函数：通用的 Nim 魔法

Nim游戏的原理很美，但它只适用于Nim游戏。如果游戏规则变了呢？比如，每次只能取1颗、2颗或3颗石子，或者取当前堆石子数量的某个因子？这时候，我们就需要更通用的“魔法”——**SG函数**！

**核心思想**：Sprague-Grundy定理（简称SG定理）指出，任何一个**公平游戏**都等价于一堆特定数量的Nim石子。这个“特定数量”就是这个游戏局面的**SG值**。

**SG函数定义**：
对于一个游戏局面 `x`，它的SG值 `sg(x)` 定义为：
`sg(x) = mex({sg(y) | y 是从 x 能够到达的所有局面})`

**mex (Minimum Excluded value) 运算**：
`mex` 运算是“最小的非负整数”的缩写。
对于一个非负整数集合 `S`，`mex(S)` 就是不属于 `S` 的最小非负整数。
**例子**：
*   `mex({0, 1, 3}) = 2`
*   `mex({1, 2, 4}) = 0`
*   `mex({}) = 0` (空集的mex是0)
*   `mex({0, 1, 2, 3}) = 4`

**SG函数的计算步骤**：
1.  **确定终局**：所有不能再进行操作的局面，其SG值都为0。这是递归的终止条件。
2.  **逆向计算**：从终局开始，逐步计算所有能到达终局的局面，再计算能到达这些局面的局面，直到计算出所有需要的SG值。通常采用**记忆化搜索（Memoized DFS）**或**动态规划（DP）**的方式。
3.  对于一个局面 `x`：
    *   枚举所有从 `x` 可以到达的局面 `y_1, y_2, ..., y_k`。
    *   递归或查询 `sg(y_1), sg(y_2), ..., sg(y_k)`。
    *   将这些SG值组成一个集合 `S = {sg(y_1), sg(y_2), ..., sg(y_k)}`。
    *   `sg(x) = mex(S)`。

**SG定理的威力**：
如果一个复杂的公平游戏 `G` 可以被分解成多个独立的子游戏 `G_1, G_2, ..., G_n` （比如，Nim游戏就是多堆石子的组合，每堆石子都是一个独立的子游戏），那么整个游戏 `G` 的SG值就是所有子游戏的SG值的异或和：
`sg(G) = sg(G_1) ^ sg(G_2) ^ ... ^ sg(G_n)`

**必胜/必败判断**：
*   如果一个游戏局面 `x` 的 `sg(x) != 0`，那么这个局面是N-位置（必胜态）。
*   如果一个游戏局面 `x` 的 `sg(x) == 0`，那么这个局面是P-位置（必败态）。

**SG函数与Nim游戏的联系**：
其实，Nim游戏就是SG定理的一个特例！
对于一堆有 `k` 颗石子的Nim游戏，它的SG值是多少呢？
*   `sg(0) = 0` (终局)
*   `sg(1) = mex({sg(0)}) = mex({0}) = 1`
*   `sg(2) = mex({sg(0), sg(1)}) = mex({0, 1}) = 2`
*   `sg(k) = mex({sg(0), sg(1), ..., sg(k-1)}) = mex({0, 1, ..., k-1}) = k`
所以，一堆 `k` 颗石子的SG值就是 `k`。
那么，对于多堆石子的Nim游戏 `a_1, a_2, ..., a_n`，根据SG定理：
`sg(Nim游戏) = sg(a_1) ^ sg(a_2) ^ ... ^ sg(a_n) = a_1 ^ a_2 ^ ... ^ a_n`
这不就是我们之前说的Nim和吗！所以，SG定理是Nim和的推广，它适用于所有公平游戏。

**常见误区**：
1.  **mex计算错误**：一定要从0开始找最小的未出现整数。
2.  **SG定理应用范围**：只适用于公平游戏，且游戏可以分解为独立子游戏。
3.  **记忆化不足**：计算SG值时，如果不对已计算过的状态进行记忆化，会导致重复计算，效率低下甚至超时。

**生活类比**：SG函数就像一个“游戏强度评估器”。每个游戏局面都有一个“强度值”（SG值）。我们的目标就是每次操作都让游戏的“总强度”（所有子游戏SG值的异或和）从非零变为零，从而把一个“弱势局面”留给对手。

## 典型例题精讲

### 例题 1：取石子游戏 (Take Stones Game)

**题目描述**：有 `n` 堆石子，每堆 `a_i` 颗。两个人轮流操作，每次可以从**任意一堆**石子中取走 1 颗、2 颗或 3 颗石子。最后一个取走石子的人获胜。判断给定局面是先手必胜还是先手必败。

**思路分析**：
这是一个典型的公平游戏，且可以分解为 `n` 个独立的子游戏（每堆石子是一个子游戏）。我们可以先计算单个石子堆的SG值，然后根据SG定理将所有子游戏的SG值异或起来得到总游戏的SG值。

1.  **计算单个石子堆的SG值 `sg(k)`**：
    *   `k` 表示当前石子堆的石子数量。
    *   终局：`k=0` 时，不能操作，`sg(0) = 0`。
    *   从 `k` 可以到达的局面：`k-1` (取1颗)，`k-2` (取2颗)，`k-3` (取3颗)。需要保证 `k-1 >= 0`, `k-2 >= 0`, `k-3 >= 0`。
    *   `sg(k) = mex({sg(k-1), sg(k-2), sg(k-3)})`

2.  **逐步计算**：
    *   `sg(0) = 0`
    *   `sg(1) = mex({sg(0)}) = mex({0}) = 1`
    *   `sg(2) = mex({sg(1), sg(0)}) = mex({1, 0}) = 2`
    *   `sg(3) = mex({sg(2), sg(1), sg(0)}) = mex({2, 1, 0}) = 3`
    *   `sg(4) = mex({sg(3), sg(2), sg(1)}) = mex({3, 2, 1}) = 0`
    *   `sg(5) = mex({sg(4), sg(3), sg(2)}) = mex({0, 3, 2}) = 1`
    *   `sg(6) = mex({sg(5), sg(4), sg(3)}) = mex({1, 0, 3}) = 2`
    *   `sg(7) = mex({sg(6), sg(5), sg(4)}) = mex({2, 1, 0}) = 3`
    *   `sg(8) = mex({sg(7), sg(6), sg(5)}) = mex({3, 2, 1}) = 0`

    我们可以发现规律：`sg(k) = k % 4`。

3.  **应用SG定理**：
    对于 `n` 堆石子 `a_1, a_2, ..., a_n`，总游戏的SG值 `TotalSG = sg(a_1) ^ sg(a_2) ^ ... ^ sg(a_n)`。
    因为 `sg(a_i) = a_i % 4`，所以 `TotalSG = (a_1 % 4) ^ (a_2 % 4) ^ ... ^ (a_n % 4)`。
    如果 `TotalSG != 0`，先手必胜；否则，先手必败。

**代码实现 (伪代码)**：
```cpp
// 预处理sg值
const int MAXN = 100005; // 假设石子堆最大数量
int sg[MAXN];
bool vis[MAXN]; // 用于mex计算

void precompute_sg(int limit) {
    sg[0] = 0;
    for (int i = 1; i <= limit; ++i) {
        // 计算 mex({sg(i-1), sg(i-2), sg(i-3)})
        // 清空 vis 数组
        for (int j = 0; j < limit; ++j) { // mex的值不会超过limit
            vis[j] = false;
        }

        if (i - 1 >= 0) vis[sg[i-1]] = true;
        if (i - 2 >= 0) vis[sg[i-2]] = true;
        if (i - 3 >= 0) vis[sg[i-3]] = true;

        int m = 0;
        while (vis[m]) {
            m++;
        }
        sg[i] = m;
    }
}

// 主逻辑
int main() {
    // precompute_sg(MAXN - 1); // 如果需要预处理所有可能值
    // 或者直接使用 sg[k] = k % 4;

    int n; // 堆数
    scanf("%d", &n);
    int totalSG = 0;
    for (int i = 0; i < n; ++i) {
        int a; // 每堆石子数
        scanf("%d", &a);
        totalSG ^= (a % 4); // 直接使用规律
        // totalSG ^= sg[a]; // 如果用预处理的sg数组
    }

    if (totalSG != 0) {
        printf("First player wins\n");
    } else {
        printf("Second player wins\n");
    }
    return 0;
}
```

**复杂度分析**：
*   预处理SG值：如果最大石子数是 `M`，每个SG值的计算需要 `O(K)` (K为可取石子种类数，这里是3) 查找 `mex`，总复杂度 `O(M * K)`。
*   主逻辑：读取 `n` 堆石子，计算 `n` 次异或，复杂度 `O(n)`。
*   由于本例有 `sg(k) = k % 4` 的简单规律，预处理甚至可以省略，直接 `O(n)` 计算。

### 例题 2：取石子游戏 II (Take Stones Game II)

**题目描述**：有 `n` 堆石子，每堆 `a_i` 颗。两个人轮流操作，每次可以从**任意一堆**石子中取走 `x` 颗，其中 `x` 必须是当前堆石子数量 `k` 的一个**正因子**（即 `k % x == 0`）。最后一个取走石子的人获胜。判断给定局面是先手必胜还是先手必败。

**思路分析**：
同样是公平游戏，可以分解。关键是计算单个石子堆 `k` 的SG值。
1.  **计算单个石子堆的SG值 `sg(k)`**：
    *   `k` 表示当前石子堆的石子数量。
    *   终局：`k=0` 时，不能操作，`sg(0) = 0`。
    *   从 `k` 可以到达的局面：枚举 `k` 的所有正因子 `x`。取走 `x` 颗后，石子数量变为 `k-x`。
    *   `sg(k) = mex({sg(k-x) | x 是 k 的正因子})`

2.  **逐步计算（需要记忆化搜索或DP）**：
    *   `sg[0] = 0`
    *   `sg[1] = mex({sg(1-1)}) = mex({sg(0)}) = mex({0}) = 1` (1的因子只有1)
    *   `sg[2] = mex({sg(2-1), sg(2-2)}) = mex({sg(1), sg(0)}) = mex({1, 0}) = 2` (2的因子有1, 2)
    *   `sg[3] = mex({sg(3-1), sg(3-3)}) = mex({sg(2), sg(0)}) = mex({2, 0}) = 1` (3的因子有1, 3)
    *   `sg[4] = mex({sg(4-1), sg(4-2), sg(4-4)}) = mex({sg(3), sg(2), sg(0)}) = mex({1, 2, 0}) = 3` (4的因子有1, 2, 4)
    *   `sg[5] = mex({sg(5-1), sg(5-5)}) = mex({sg(4), sg(0)}) = mex({3, 0}) = 1` (5的因子有1, 5)
    *   `sg[6] = mex({sg(6-1), sg(6-2), sg(6-3), sg(6-6)}) = mex({sg(5), sg(4), sg(3), sg(0)}) = mex({1, 3, 1, 0}) = mex({0, 1, 3}) = 2` (6的因子有1, 2, 3, 6)

3.  **应用SG定理**：
    对于 `n` 堆石子 `a_1, a_2, ..., a_n`，总游戏的SG值 `TotalSG = sg(a_1) ^ sg(a_2) ^ ... ^ sg(a_n)`。
    如果 `TotalSG != 0`，先手必胜；否则，先手必败。

**代码实现 (使用记忆化搜索)**：
```cpp
#include <bits/stdc++.h>

const int MAXA = 1005; // 假设每堆石子最大数量
int sg[MAXA];
bool visited_mex[MAXA]; // 用于mex计算，注意每次调用sg函数时要清空
bool computed_sg[MAXA]; // 标记sg[i]是否已经计算过

// 计算一个数的所有因子
void get_divisors(int n, int divisors[], int& count) {
    count = 0;
    for (int i = 1; i * i <= n; ++i) {
        if (n % i == 0) {
            divisors[count++] = i;
            if (i * i != n) {
                divisors[count++] = n / i;
            }
        }
    }
}

int calculate_sg(int k) {
    if (k == 0) {
        return 0;
    }
    if (computed_sg[k]) { // 如果已经计算过，直接返回
        return sg[k];
    }

    // 清空 mex 集合标记
    // 注意：这里的MAXA需要足够大，以覆盖所有可能的sg值。
    // sg值最大不会超过k，所以这里可以设为k+1
    for (int i = 0; i <= k; ++i) { // mex的值不会超过 k
        visited_mex[i] = false;
    }

    int divisors[MAXA]; // 存储因子
    int count;
    get_divisors(k, divisors, count); // 获取k的所有因子

    for (int i = 0; i < count; ++i) {
        int d = divisors[i];
        if (k - d >= 0) { // 确保操作合法
            visited_mex[calculate_sg(k - d)] = true;
        }
    }

    int m = 0;
    while (visited_mex[m]) {
        m++;
    }
    sg[k] = m;
    computed_sg[k] = true;
    return m;
}

int main() {
    // 初始设置
    memset(sg, 0, sizeof(sg));
    memset(computed_sg, false, sizeof(computed_sg));
    computed_sg[0] = true; // sg(0) = 0 已经确定

    int n; // 堆数
    scanf("%d", &n);
    int totalSG = 0;
    for (int i = 0; i < n; ++i) {
        int a; // 每堆石子数
        scanf("%d", &a);
        totalSG ^= calculate_sg(a);
    }

    if (totalSG != 0) {
        printf("First player wins\n");
    } else {
        printf("Second player wins\n");
    }
    return 0;
}
```

**复杂度分析**：
*   `get_divisors(k)` 的复杂度大约是 `O(sqrt(k))`。
*   `calculate_sg(k)`：对于每个 `k`，它会计算所有因子的 `sg(k-d)`，并进行 `mex` 运算。`mex` 运算最多需要 `O(k)`。因子数量最多 `O(sqrt(k))` 个。
*   总体的SG函数计算，由于记忆化搜索，每个 `sg[k]` 只会被真正计算一次。所以总计算量是 `sum_{k=1 to MAXA} (sqrt(k) + k)`，大约是 `O(MAXA * sqrt(MAXA))` 或者 `O(MAXA * log MAXA)` 如果因子数量更少。
*   主逻辑：`O(n)` 次调用 `calculate_sg`，由于记忆化，实际是 `O(n + MAXA * (sqrt(MAXA) + MAX_SG_VALUE))`。
*   其中 `MAX_SG_VALUE` 大约是 `MAXA`。

### 例题 3：组合游戏 (Combined Game)

**题目描述**：有两类石子堆。
第一类有 `N` 堆，每堆 `a_i` 颗，每次可以取走 1, 2 或 3 颗。
第二类有 `M` 堆，每堆 `b_j` 颗，每次可以取走 `x` 颗，`x` 必须是当前堆石子数量 `k` 的一个正因子。
两个人轮流操作，从任意一堆石子中取走石子。最后一个取走石子的人获胜。判断给定局面是先手必胜还是先手必败。

**思路分析**：
这是一个由两种不同规则的子游戏组成的复合游戏。根据SG定理，我们可以分别计算两类子游戏的SG值，然后将它们全部异或起来。

1.  **计算第一类石子堆的SG值 `sg1(k)`**：
    与例题1相同，`sg1(k) = k % 4`。

2.  **计算第二类石子堆的SG值 `sg2(k)`**：
    与例题2相同，需要使用记忆化搜索计算 `sg2(k) = mex({sg2(k-x) | x 是 k 的正因子})`。

3.  **应用SG定理**：
    总游戏的SG值 `TotalSG = (sg1(a_1) ^ ... ^ sg1(a_N)) ^ (sg2(b_1) ^ ... ^ sg2(b_M))`。
    如果 `TotalSG != 0`，先手必胜；否则，先手必败。

**代码实现**：
```cpp
#include <bits/stdc++.h>

const int MAXA = 1005; // 假设石子最大数量
int sg1[MAXA]; // 第一类游戏的sg值
int sg2[MAXA]; // 第二类游戏的sg值
bool visited_mex2[MAXA]; // 用于sg2的mex计算
bool computed_sg2[MAXA]; // 标记sg2[i]是否已经计算过

// 计算一个数的所有因子
void get_divisors(int n, int divisors[], int& count) {
    count = 0;
    for (int i = 1; i * i <= n; ++i) {
        if (n % i == 0) {
            divisors[count++] = i;
            if (i * i != n) {
                divisors[count++] = n / i;
            }
        }
    }
}

// 计算第二类游戏的sg值
int calculate_sg2(int k) {
    if (k == 0) {
        return 0;
    }
    if (computed_sg2[k]) {
        return sg2[k];
    }

    for (int i = 0; i <= k; ++i) { // mex的值不会超过 k
        visited_mex2[i] = false;
    }

    int divisors[MAXA];
    int count;
    get_divisors(k, divisors, count);

    for (int i = 0; i < count; ++i) {
        int d = divisors[i];
        if (k - d >= 0) {
            visited_mex2[calculate_sg2(k - d)] = true;
        }
    }

    int m = 0;
    while (visited_mex2[m]) {
        m++;
    }
    sg2[k] = m;
    computed_sg2[k] = true;
    return m;
}

int main() {
    // 预处理 sg1 (k % 4)
    for (int i = 0; i < MAXA; ++i) {
        sg1[i] = i % 4;
    }

    // 预处理 sg2 (或在需要时计算)
    memset(sg2, 0, sizeof(sg2));
    memset(computed_sg2, false, sizeof(computed_sg2));
    computed_sg2[0] = true; // sg2(0) = 0 已经确定

    int N, M; // 两类堆的数量
    scanf("%d %d", &N, &M);

    int totalSG = 0;

    // 处理第一类石子堆
    for (int i = 0; i < N; ++i) {
        int a;
        scanf("%d", &a);
        totalSG ^= sg1[a];
    }

    // 处理第二类石子堆
    for (int i = 0; i < M; ++i) {
        int b;
        scanf("%d", &b);
        totalSG ^= calculate_sg2(b); // 记忆化搜索会自动处理
    }

    if (totalSG != 0) {
        printf("First player wins\n");
    } else {
        printf("Second player wins\n");
    }
    return 0;
}
```

**复杂度分析**：
整体复杂度是两种子游戏计算复杂度的叠加，即 `O(N + M + MAXA * (sqrt(MAXA) + MAX_SG_VALUE))`。

## 代码实现模板

这里提供一个通用的SG函数计算模板，适用于需要记忆化搜索的场景。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

const int MAX_STATE_VALUE = 1005; // 假设游戏状态的最大值（例如，石子数量的最大值）
int sg[MAX_STATE_VALUE];          // 存储每个状态的SG值
bool computed_sg[MAX_STATE_VALUE]; // 标记sg[i]是否已经计算过
bool visited_mex[MAX_STATE_VALUE]; // 用于mex计算的辅助数组，每次计算sg时清空

// =================================================================
// 核心函数：计算状态 k 的 SG 值
// =================================================================
int calculate_sg(int k) {
    // 1. 终局条件：如果 k 是游戏结束状态，其 SG 值为 0
    if (k == 0) {
        return 0;
    }

    // 2. 记忆化搜索：如果 k 的 SG 值已经计算过，直接返回
    if (computed_sg[k]) {
        return sg[k];
    }

    // 3. 清空 mex 辅助数组：为当前状态 k 的 mex 运算做准备
    // 注意：mex 的值不会超过 k (或者 k+1)，所以清空范围可以设为 k+1
    // 实际应用中，为了安全，可以清空到 MAX_STATE_VALUE
    for (int i = 0; i <= k; ++i) { // 确保覆盖所有可能的SG值
        visited_mex[i] = false;
    }

    // 4. 枚举所有可达状态并计算其 SG 值
    // 这一部分是根据具体游戏规则修改的地方！
    // 假设游戏规则是：从 k 可以移动到 k-1, k-2, ..., k-m
    // 或者从 k 可以移动到 k-d (d是k的因子) 等等
    
    // 示例：可以从 k 移动到 k-1, k-2, k-3
    if (k - 1 >= 0) {
        visited_mex[calculate_sg(k - 1)] = true;
    }
    if (k - 2 >= 0) {
        visited_mex[calculate_sg(k - 2)] = true;
    }
    if (k - 3 >= 0) {
        visited_mex[calculate_sg(k - 3)] = true;
    }

    // 示例2：可以从 k 移动到 k-d (d是k的因子)
    /*
    int divisors[MAX_STATE_VALUE]; // 临时存储因子
    int count = 0;
    for (int i = 1; i * i <= k; ++i) {
        if (k % i == 0) {
            divisors[count++] = i;
            if (i * i != k) {
                divisors[count++] = k / i;
            }
        }
    }
    for (int i = 0; i < count; ++i) {
        int d = divisors[i];
        if (k - d >= 0) {
            visited_mex[calculate_sg(k - d)] = true;
        }
    }
    */

    // 5. 计算 mex 值
    int m = 0;
    while (visited_mex[m]) {
        m++;
    }

    // 6. 存储并标记已计算
    sg[k] = m;
    computed_sg[k] = true;
    return m;
}

// =================================================================
// 主函数
// =================================================================
int main() {
    // 初始化 SG 数组和计算标记
    memset(sg, 0, sizeof(sg));
    memset(computed_sg, false, sizeof(computed_sg));
    computed_sg[0] = true; // 状态 0 的 SG 值已知为 0

    // 读取游戏输入
    int n; // 游戏组件（石子堆）的数量
    scanf("%d", &n);

    int total_sg_sum = 0; // 所有组件 SG 值的异或和

    for (int i = 0; i < n; ++i) {
        int a; // 当前组件（石子堆）的大小
        scanf("%d", &a);
        total_sg_sum ^= calculate_sg(a); // 将每个组件的 SG 值异或起来
    }

    // 判断先手胜负
    if (total_sg_sum != 0) {
        printf("First player wins\n");
    } else {
        printf("Second player wins\n");
    }

    return 0;
}
```

**详细注释**：
*   `MAX_STATE_VALUE`：根据题目中石子数量的最大值来设定，要留出足够的空间。
*   `sg[]`：存储状态 `i` 的SG值。
*   `computed_sg[]`：布尔数组，`computed_sg[i]` 为 `true` 表示 `sg[i]` 已经计算过，可以直接使用，避免重复计算（记忆化）。
*   `visited_mex[]`：布尔数组，用于辅助 `mex` 运算。在计算 `sg(k)` 时，我们会把所有从 `k` 可达状态的SG值 `sg(y)` 标记为 `visited_mex[sg(y)] = true`。然后从0开始找第一个 `visited_mex[m]` 为 `false` 的 `m`，它就是 `mex` 值。**每次计算新的 `sg(k)` 时，这个数组都需要清空！**
*   `calculate_sg(k)`：这是递归计算SG值的核心函数。
    *   **基准情况**：`k=0` 是终局，`sg(0)=0`。
    *   **记忆化**：如果 `computed_sg[k]` 为真，直接返回 `sg[k]`。
    *   **构建可达状态的SG值集合**：根据具体的游戏规则，枚举所有从 `k` 可以一步到达的局面 `y`，然后递归调用 `calculate_sg(y)` 得到其SG值，并将 `visited_mex[sg(y)]` 标记为 `true`。
    *   **计算mex**：从0开始递增查找，直到找到第一个 `m` 使得 `visited_mex[m]` 为 `false`。
    *   **存储结果**：将 `m` 存入 `sg[k]` 并标记 `computed_sg[k]` 为 `true`。
*   `main` 函数：
    *   初始化 `sg` 数组和 `computed_sg` 数组。
    *   读取游戏组件数量 `n`。
    *   循环读取每个组件的大小 `a`，调用 `calculate_sg(a)` 得到其SG值，并与 `total_sg_sum` 进行异或。
    *   根据 `total_sg_sum` 的值判断胜负。
