## 深度知识点讲解

### 1. 概率基础回顾（热身运动！）

在深入概率DP之前，我们先快速回顾一下概率的基本概念，它们是概率DP的基石。

*   **事件 (Event)**：在一次试验中可能发生的结果。比如掷骰子，掷出“3”是一个事件，掷出“偶数”也是一个事件。
*   **样本空间 (Sample Space)**：一次试验所有可能结果的集合。掷骰子的样本空间是 {1, 2, 3, 4, 5, 6}。
*   **概率 (Probability)**：某个事件发生的可能性大小。如果所有基本事件发生的可能性相等，那么事件 A 的概率 P(A) = (事件 A 包含的基本事件数) / (样本空间中基本事件总数)。比如，掷骰子掷出“3”的概率是 1/6。
*   **独立事件 (Independent Events)**：两个事件的发生互不影响。比如你第一次掷骰子和第二次掷骰子，结果是独立的。
*   **互斥事件 (Mutually Exclusive Events)**：两个事件不能同时发生。比如掷骰子，不可能同时掷出“1”和“2”。如果事件 A 和 B 互斥，那么 P(A 或 B) = P(A) + P(B)。

### 2. 数学期望：平均值的秘密（核心概念！）

我们引入的寻宝游戏，最终要计算的是“平均掷骰子次数”，这正是数学期望的应用。

*   **什么是数学期望？**
    *   数学期望，简单来说，就是**平均值**。它表示在一个随机事件中，某个变量所有可能取值的加权平均。
    *   **生活类比**：
        *   **班级平均分**：所有同学分数之和除以人数。如果看成随机变量“一个同学的分数”，那么每个分数出现的频率就是它的“概率”，平均分就是期望。
        *   **彩票中奖**：买一张彩票，可能中100万，可能中10块，也可能不中。每种结果都有它的概率。你的“期望收益”就是每种结果的奖金乘以其对应的概率，然后加起来。
    *   **数学定义**：
        如果一个离散随机变量 X 可能取值为 `x1, x2, ..., xk`，并且取这些值的概率分别为 `p1, p2, ..., pk`，那么 X 的数学期望 E(X) 定义为：
    `E(X) = x1*p1 + x2*p2 + ... + xk*pk = Σ (xi * pi)`

*   **期望的线性性质（超级重要！）**
    *   这是解决期望DP问题的“瑞士军刀”！
    *   **性质 1**：`E(X + Y) = E(X) + E(Y)`
        *   两个随机变量的和的期望，等于它们各自期望的和。
        *   **理解**：你玩两个游戏，游戏 A 期望赢 5 块钱，游戏 B 期望赢 3 块钱，那么你玩这两个游戏总共期望赢 5+3=8 块钱。这不管两个游戏是不是独立都成立！
    *   **性质 2**：`E(cX) = cE(X)`
        *   随机变量乘以常数的期望，等于期望乘以这个常数。
        *   **理解**：如果你期望每天吃 2 个苹果，那么期望 3 天吃 3 * 2 = 6 个苹果。

### 3. 概率DP的核心思想（DP遇上随机性！）

现在，我们把DP和概率期望结合起来。

*   **DP三要素回顾**：状态、转移方程、边界条件。
*   **概率DP的特点**：
    *   DP状态 `dp[i]` 通常表示：
        1.  达到某种状态 `i` 的**概率**。
        2.  从状态 `i` 到达目标状态的**期望**。
        3.  达到状态 `i` 时，某个量的**期望**。
    *   DP转移方程的构建：
        *   **计算概率**：通常从初始状态向目标状态**正向推导**。`dp[i]` 会从 `dp[j]` 转移过来，其中 `j` 是能够到达 `i` 的前一个状态。 `dp[i] = Σ (dp[j] * P(j->i))`。
        *   **计算期望**：通常从目标状态向初始状态**逆向推导**。`dp[i]` 表示从状态 `i` 到达目标状态的期望。`dp[i]` 会依赖于 `dp[j]`，其中 `j` 是从 `i` 能够到达的下一个状态。
            *   `dp[i] = (从 i 到 j 的代价) + Σ (P(i->j) * dp[j])`。
            *   **为什么期望DP常常逆推？**
                *   因为在期望DP中，我们通常想知道从当前状态 `i` 到达终点状态 `E` 的期望。如果我们正向推，从 `i` 到 `j`，再到 `k`... 路径是不确定的，不好计算。
                *   但是如果我们逆向推，从 `E` 往回推，目标 `E` 的期望是已知的（通常是 0，因为已经到达了），那么我们就可以计算 `E` 的前一个状态 `k` 的期望，再计算 `k` 的前一个状态 `j` 的期望，直到初始状态。这就像“倒着看电影”，更容易理清剧情走向。

*   **边界条件**：
    *   概率DP：通常是初始状态的概率为 1 (从这里开始)，或目标状态的概率为 0 (尚未达到)。
    *   期望DP：通常是目标状态的期望为 0 (因为已经到达，不需要额外步数/代价)。

### 4. 常见误区提醒！

*   **混淆概率和期望**：概率是一个0到1之间的值，表示可能性；期望是一个平均值，可能大于1，也可能小于0。
*   **转移方程方向错误**：尤其是在期望DP中，务必思考清楚是正向推还是逆向推。
*   **未考虑所有可能事件**：在构建转移方程时，要确保覆盖了所有可能的后续状态及其概率。
*   **浮点数精度**：使用 `double` 或 `long double` 类型存储概率和期望值。输出时注意格式化，保留足够的小数位数。

## 典型例题精讲

### 例题 1：掷硬币游戏（期望DP入门）

**题目**：你有一个均匀的硬币，每次抛掷出现正面或反面的概率都是 `0.5`。请问，你**期望**抛掷多少次才能第一次连续出现 `n` 次正面？

**分析**：
这是一个典型的期望DP问题。我们要求的是一个“平均次数”，且涉及随机过程。

1.  **确定目标**：求从初始状态（没有连续正面）到“连续出现 `n` 次正面”的期望次数。
2.  **状态定义**：由于我们要求的是从某个状态到目标状态的期望，所以通常采用逆推。
    *   `dp[i]`：表示**已经**连续出现了 `i` 次正面，还需要**期望**多少次抛掷才能达到连续 `n` 次正面。
    *   我们的最终答案是 `dp[0]`。
3.  **边界条件**：
    *   当已经连续出现了 `n` 次正面时，目标已经达成，不需要再抛了。
    *   所以 `dp[n] = 0`。
4.  **转移方程**：
    考虑 `dp[i]` (当前已经连续 `i` 次正面，`0 <= i < n`)。
    我们进行下一次抛掷，会发生什么？
    *   **情况 1：抛到正面 (H)**
        *   概率：`0.5`
        *   结果：连续正面的次数增加 `1`，变为 `i+1` 次。
        *   需要额外期望次数：`dp[i+1]`
    *   **情况 2：抛到反面 (T)**
        *   概率：`0.5`
        *   结果：连续正面的链断裂，需要从头开始，连续正面次数变为 `0`。
        *   需要额外期望次数：`dp[0]`

    所以，从状态 `i` 到达目标状态的期望次数 `dp[i]`，就是“进行这次抛掷的 1 次”加上“抛掷后进入新状态所需的期望次数”。
    根据期望的线性性质：
    `dp[i] = 1 + P(H) * dp[i+1] + P(T) * dp[0]`
    `dp[i] = 1 + 0.5 * dp[i+1] + 0.5 * dp[0]` (对于 `0 <= i < n`)

    注意，这个方程中 `dp[i]` 依赖于 `dp[0]`，形成了一个方程组。
    我们可以把 `dp[0]` 移项，或者从 `n-1` 倒着推，最终会发现 `dp[0]` 会出现在右边。
    我们有 `n` 个未知数 `dp[0], dp[1], ..., dp[n-1]` 和 `n` 个方程。
    `dp[n-1] = 1 + 0.5 * dp[n] + 0.5 * dp[0] = 1 + 0.5 * 0 + 0.5 * dp[0] = 1 + 0.5 * dp[0]`
    `dp[n-2] = 1 + 0.5 * dp[n-1] + 0.5 * dp[0]`
    ...
    `dp[0] = 1 + 0.5 * dp[1] + 0.5 * dp[0]`

    从 `dp[0]` 的方程：`0.5 * dp[0] = 1 + 0.5 * dp[1]`
    `dp[0] = 2 + dp[1]`

    我们发现，如果把 `dp[0]` 作为一个变量，可以从 `dp[n-1]` 开始向上递推，每次都用 `dp[i]` 表示成 `A + B * dp[0]` 的形式。
    更直接的方法是，我们可以把 `dp[0]` 设为 `x`，然后从 `dp[n-1]` 往回计算，最后得到一个关于 `x` 的方程。

    让我们换个思路，直接从 `dp[n-1]` 往回算：
    `dp[n] = 0`
    `dp[n-1] = 1 + 0.5 * dp[n] + 0.5 * dp[0] = 1 + 0.5 * dp[0]`
    `dp[n-2] = 1 + 0.5 * dp[n-1] + 0.5 * dp[0]`
    `dp[n-2] = 1 + 0.5 * (1 + 0.5 * dp[0]) + 0.5 * dp[0] = 1 + 0.5 + 0.25 * dp[0] + 0.5 * dp[0] = 1.5 + 0.75 * dp[0]`
    ...

    这看起来有点复杂。其实，我们可以直接求解 `dp[0]`。
    假设我们知道 `dp[0]`，那么所有 `dp[i]` 都可以算出来。
    我们可以把 `dp[0]` 提出来单独看。
    考虑 `dp[i]`：
    `dp[i] = 1 + 0.5 * dp[i+1] + 0.5 * dp[0]`

    我们知道 `dp[0] = 1 + 0.5 * dp[1] + 0.5 * dp[0]`
    `0.5 * dp[0] = 1 + 0.5 * dp[1]`
    `dp[0] = 2 + dp[1]` (方程 A)

    我们再看 `dp[i]` 和 `dp[i+1]` 的关系：
    `dp[i] - dp[i+1] = (1 + 0.5 * dp[i+1] + 0.5 * dp[0]) - dp[i+1]`
    `dp[i] - dp[i+1] = 1 - 0.5 * dp[i+1] + 0.5 * dp[0]`

    这个解法需要一点技巧。一个更常用的技巧是：
    `dp[i]` 表示已经连续 `i` 个 H，还需要多少次抛掷的期望。
    当抛到第 `i` 个 H 时，如果下一个是 H，则状态变为 `i+1`。如果下一个是 T，则状态变为 `0`。
    `dp[i] = 1 + 0.5 * dp[i+1] + 0.5 * dp[0]` (1是当前这次抛掷)
    `dp[n] = 0`

    我们可以从 `dp[n-1]` 倒推到 `dp[0]`。
    `dp[n-1] = 1 + 0.5 * dp[n] + 0.5 * dp[0] = 1 + 0.5 * 0 + 0.5 * dp[0] = 1 + 0.5 * dp[0]`
    `dp[n-2] = 1 + 0.5 * dp[n-1] + 0.5 * dp[0] = 1 + 0.5 * (1 + 0.5 * dp[0]) + 0.5 * dp[0]`
    `dp[n-2] = 1 + 0.5 + 0.25 * dp[0] + 0.5 * dp[0] = 1.5 + 0.75 * dp[0]`
    ...
    `dp[0] = 1 + 0.5 * dp[1] + 0.5 * dp[0]`
    `0.5 * dp[0] = 1 + 0.5 * dp[1]`
    `dp[0] = 2 + dp[1]`

    我们发现，`dp[i]` 可以表示为 `A_i + B_i * dp[0]` 的形式。
    `dp[n] = 0 = 0 + 0 * dp[0]` (所以 `A_n=0, B_n=0`)
    `dp[i] = 1 + 0.5 * (A_{i+1} + B_{i+1} * dp[0]) + 0.5 * dp[0]`
    `dp[i] = (1 + 0.5 * A_{i+1}) + (0.5 * B_{i+1} + 0.5) * dp[0]`
    所以，`A_i = 1 + 0.5 * A_{i+1}`
    `B_i = 0.5 * B_{i+1} + 0.5`

    我们从 `i=n-1` 倒推到 `i=0`：
    `A_n = 0, B_n = 0`
    `A_{n-1} = 1 + 0.5 * A_n = 1`
    `B_{n-1} = 0.5 * B_n + 0.5 = 0.5`
    `A_{n-2} = 1 + 0.5 * A_{n-1} = 1 + 0.5 * 1 = 1.5`
    `B_{n-2} = 0.5 * B_{n-1} + 0.5 = 0.5 * 0.5 + 0.5 = 0.25 + 0.5 = 0.75`
    ...
    `A_0 = 1 + 0.5 * A_1`
    `B_0 = 0.5 * B_1 + 0.5`

    最后，我们得到 `dp[0] = A_0 + B_0 * dp[0]`
    `dp[0] * (1 - B_0) = A_0`
    `dp[0] = A_0 / (1 - B_0)`

    这个方法非常通用，当期望方程出现环形依赖（如 `dp[i]` 依赖 `dp[0]`）时，可以用这种方式解。

    **时间复杂度**：O(N)，因为我们只需要线性地计算 `A_i` 和 `B_i`。

```cpp
#include <bits/stdc++.h>

using namespace std;

int main() {
    // 题目：期望抛掷多少次才能第一次连续出现 n 次正面
    // n 表示连续正面的次数
    int n;
    cin >> n;

    // dp[i] 表示已经连续出现了 i 次正面，还需要期望多少次抛掷才能达到连续 n 次正面。
    // 为了处理 dp[0] 的环形依赖，我们使用 A_i 和 B_i
    // dp[i] = A_i + B_i * dp[0]
    double a[n + 1]; // 存储 A_i
    double b[n + 1]; // 存储 B_i

    // 边界条件：dp[n] = 0
    // 对应 A_n = 0, B_n = 0
    a[n] = 0.0;
    b[n] = 0.0;

    // 从 i = n-1 倒推到 i = 0
    // 转移方程：dp[i] = 1 + 0.5 * dp[i+1] + 0.5 * dp[0]
    // 代入 dp[i+1] = A_{i+1} + B_{i+1} * dp[0]
    // dp[i] = 1 + 0.5 * (A_{i+1} + B_{i+1} * dp[0]) + 0.5 * dp[0]
    // dp[i] = (1 + 0.5 * A_{i+1}) + (0.5 * B_{i+1} + 0.5) * dp[0]
    // 所以：
    // A_i = 1 + 0.5 * A_{i+1}
    // B_i = 0.5 * B_{i+1} + 0.5
    for (int i = n - 1; i >= 0; --i) {
        a[i] = 1.0 + 0.5 * a[i + 1];
        b[i] = 0.5 + 0.5 * b[i + 1];
    }

    // 最终我们要求的是 dp[0]
    // dp[0] = A_0 + B_0 * dp[0]
    // dp[0] - B_0 * dp[0] = A_0
    // dp[0] * (1 - B_0) = A_0
    // dp[0] = A_0 / (1 - B_0)
    double ans = a[0] / (1.0 - b[0]);

    // 输出结果，保留小数点后两位
    cout << fixed << setprecision(2) << ans << endl;

    return 0;
}
```
**复杂度分析**：
*   **时间复杂度**：循环从 `n-1` 到 `0`，共 `n` 次迭代，每次迭代常数时间操作，所以总时间复杂度为 `O(n)`。
*   **空间复杂度**：使用了两个大小为 `n+1` 的数组 `a` 和 `b`，所以空间复杂度为 `O(n)`。

### 例题 2：寻宝大冒险（期望DP经典）

**题目**：你从格子 `1` 开始，目标是到达格子 `n`。每次掷一个 `m` 面的骰子 (点数从 `1` 到 `m`)，掷出几点就前进几步。如果掷出的点数让你超过了格子 `n`，你将停留在原地，不前进也不后退。问从格子 `1` 到达格子 `n` 的期望步数。

**分析**：
这是我们引入时提到的寻宝游戏。

1.  **确定目标**：求从格子 `1` 到格子 `n` 的期望步数。
2.  **状态定义**：
    *   `dp[i]`：表示从格子 `i` 出发，到达格子 `n` 的**期望**步数。
    *   我们的最终答案是 `dp[1]`。
3.  **边界条件**：
    *   当已经在格子 `n` 时，目标已经达成，不需要再走。
    *   所以 `dp[n] = 0`。
4.  **转移方程**：
    考虑 `dp[i]` (当前在格子 `i`，`1 <= i < n`)。
    我们进行下一次掷骰子，会发生什么？
    *   掷骰子一次，这本身算作 1 步。
    *   骰子有 `m` 个面，每个面出现的概率是 `1/m`。
    *   如果掷出 `j` (从 `1` 到 `m`)：
        *   新的位置是 `i + j`。
        *   **如果 `i + j > n`**：根据题目规则，停留在原地 `i`。所以需要从 `dp[i]` 重新计算。
        *   **如果 `i + j <= n`**：移动到格子 `i + j`。需要从 `dp[i+j]` 继续计算。
        *   为了统一，我们可以使用 `min(i+j, n)` 来表示实际到达的格子。当 `i+j > n` 时，`min(i+j, n)` 仍为 `n`，但题目说停留在原地 `i`。所以这里需要特别处理。

    正确的处理方式：
    `dp[i] = 1 + (1/m) * Σ_{j=1}^{m} ( 从 i 走 j 步后的期望 )`
    对于每个 `j`：
    *   如果 `i + j >= n` (即到达或超过终点 `n`)：
        *   如果 `i + j == n`，到达终点，期望是 `dp[n] = 0`。
        *   如果 `i + j > n`，停留在 `i`，期望是 `dp[i]`。
    *   如果 `i + j < n` (未到达终点)：
        *   到达格子 `i + j`，期望是 `dp[i+j]`。

    所以，
    `dp[i] = 1 + (1/m) * ( Σ_{j=1}^{n-i-1} dp[i+j] + Σ_{j=n-i}^{m} dp[i] )`
    这个方程中 `dp[i]` 依赖于 `dp[i]` 自身，需要移项。
    `dp[i] = 1 + (1/m) * ( Σ_{j=1}^{n-i-1} dp[i+j] + (m - (n-i) + 1) * dp[i] )`
    `dp[i] = 1 + (1/m) * Σ_{j=1}^{n-i-1} dp[i+j] + ( (m - (n-i) + 1) / m ) * dp[i]`

    设 `k = m - (n-i) + 1` 为停留在 `i` 的骰子点数个数。
    `dp[i] * (1 - k/m) = 1 + (1/m) * Σ_{j=1}^{n-i-1} dp[i+j]`
    `dp[i] * ((m-k)/m) = 1 + (1/m) * Σ_{j=1}^{n-i-1} dp[i+j]`
    `dp[i] = (m / (m-k)) * (1 + (1/m) * Σ_{j=1}^{n-i-1} dp[i+j])`
    `dp[i] = m / (m-k) + (1 / (m-k)) * Σ_{j=1}^{n-i-1} dp[i+j]`

    其中 `m-k` 是能够前进的骰子点数个数。

    DP顺序：从 `n-1` 递减到 `1`。

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXN = 1005; // 最大格子数
double dp[MAXN];       // dp[i] 表示从格子 i 到格子 n 的期望步数

int main() {
    // n: 总格子数，m: 骰子面数
    int n, m;
    cin >> n >> m;

    // 边界条件：到达格子 n 时，期望步数为 0
    dp[n] = 0.0;

    // 从 n-1 倒推到 1
    for (int i = n - 1; i >= 1; --i) {
        double sum_next_dp = 0.0; // 存储能够前进到不同格子时的期望和
        int stay_count = 0;       // 存储掷骰子后停留在当前格子 i 的情况数

        for (int j = 1; j <= m; ++j) { // 掷出 j 点
            if (i + j >= n) {
                // 如果掷出 j 点后到达或超过格子 n
                // 1. 如果 i + j == n，刚好到达终点，期望是 dp[n] = 0
                // 2. 如果 i + j > n，停留在原地 i，期望是 dp[i]
                if (i + j == n) {
                    sum_next_dp += dp[n]; // dp[n] 是 0，所以这里加 0
                } else { // i + j > n，停留在 i
                    stay_count++;
                }
            } else { // i + j < n，前进到 i+j
                sum_next_dp += dp[i + j];
            }
        }

        // 转移方程：dp[i] = 1 + (1/m) * (sum_next_dp + stay_count * dp[i])
        // 移项求解 dp[i]:
        // dp[i] - (stay_count / m) * dp[i] = 1 + (1/m) * sum_next_dp
        // dp[i] * (1 - stay_count / m) = 1 + (1/m) * sum_next_dp
        // dp[i] * ((m - stay_count) / m) = 1 + (1/m) * sum_next_dp
        // dp[i] = (1 + (1.0 / m) * sum_next_dp) * (m / (m - stay_count))
        // 注意：如果 m - stay_count 为 0，说明所有情况都停留在原地，这是死循环，期望无限大。
        // 题目保证能到达终点，所以 m - stay_count 必然 > 0。
        
        // 确保分母不为0，题目保证能到达终点，所以 m - stay_count > 0
        if (m - stay_count == 0) {
            // 这意味着所有投掷都会停留在原地，无法前进，期望为无穷大
            // 实际竞赛中，这种情况通常不会出现，或者会给出特殊说明
            // 为了安全，可以输出一个特殊值或报错
            // 这里我们假设题目保证能到达终点，所以 m - stay_count > 0
            // 但为了代码健壮性，可以加上判断
            dp[i] = 1e18; // 设为一个很大的值表示无穷大
        } else {
            dp[i] = (1.0 + (1.0 / m) * sum_next_dp) * (1.0 * m / (m - stay_count));
        }
    }

    // 最终答案是 dp[1]
    cout << fixed << setprecision(6) << dp[1] << endl;

    return 0;
}
```
**复杂度分析**：
*   **时间复杂度**：外层循环从 `n-1` 到 `1`，共 `n-1` 次迭代。内层循环从 `1` 到 `m`，共 `m` 次迭代。所以总时间复杂度为 `O(n * m)`。
*   **空间复杂度**：使用了一个大小为 `n+1` 的数组 `dp`，所以空间复杂度为 `O(n)`。

    *   **预期回答**：
        *   `dp[2]`：从格子 `2` 出发。
            *   掷 `1`：到 `2+1=3` (终点)，期望 `dp[3]=0`。
            *   掷 `2`：到 `2+2=4` (>3，停留在 `2`)，期望 `dp[2]`。
            *   所以 `dp[2] = 1 + 0.5 * dp[3] + 0.5 * dp[2]`
            *   `dp[2] = 1 + 0.5 * 0 + 0.5 * dp[2]`
            *   `0.5 * dp[2] = 1`
            *   `dp[2] = 2`
        *   `dp[1]`：从格子 `1` 出发。
            *   掷 `1`：到 `1+1=2`，期望 `dp[2]`。
            *   掷 `2`：到 `1+2=3` (终点)，期望 `dp[3]=0`。
            *   所以 `dp[1] = 1 + 0.5 * dp[2] + 0.5 * dp[3]`
            *   `dp[1] = 1 + 0.5 * 2 + 0.5 * 0`
            *   `dp[1] = 1 + 1 + 0 = 2`
        *   所以从格子 1 到格子 3 的期望步数是 2。

2.  **小组讨论 (10分钟)**：
