## 深度知识点讲解

### 1. 什么是计数DP？

我们之前学习的动态规划，很多时候都是在解决“最大值”、“最小值”这类问题，比如背包问题中求能装下的最大价值，最长公共子序列中求最大长度。这类问题我们称为**最优化DP**。

而**计数DP**，顾名思义，它的目标是计算满足某种条件的方案总数。它不关心“哪个方案最好”，只关心“一共有多少个这样的方案”。

**核心思想：** 计数DP依然遵循动态规划的“无后效性”和“重叠子问题”原则。只不过，这里的“最优子结构”变成了“计数子结构”——一个大问题的方案总数，可以由它所有子问题的方案总数累加得到。这就像你统计一个班有多少人，你可以统计男生人数加上女生人数，或者统计戴眼镜的人数加上不戴眼镜的人数，只要分类不重不漏，总数就是子类数的和。

### 2. 为什么动态规划可以用于计数？

1.  **加法原理的体现**：计数DP的转移方程通常是加法，因为我们把一个复杂问题分解成若干个更小的、互不重叠的子问题，每个子问题的方案数加起来就是原问题的方案数。这完美符合组合数学中的加法原理。
2.  **避免重复计算**：DP表存储了每个子问题的计算结果，当我们需要某个子问题的方案数时，直接查表即可，避免了重复计算，大大提高了效率。
3.  **系统性枚举**：DP的迭代计算过程，实际上是一种系统性的、不重不漏的枚举所有可能方案的方式。通过精心设计的状态和转移，它能确保所有合法方案都被考虑，且每个方案只被计算一次。

### 3. 计数DP的关键步骤

计数DP的解题思路与最优化DP非常相似，但有一些细节需要特别注意：

#### 步骤一：定义DP状态 (`dp[i]` 或 `dp[i][j]`)

这是计数DP最关键的一步。状态要能完整地描述一个子问题，并且足够支撑后续的决策。
*   `dp[i]` 通常表示“前 `i` 个元素（或长度为 `i` 的序列）满足某种条件的方案总数”。
*   `dp[i][j]` 通常表示“前 `i` 个元素，且第 `i` 个元素处于状态 `j` 时，满足某种条件的方案总数”。
    *   **思考：** 为什么计数DP常常需要额外的状态维度 `[j]`？
        *   因为仅仅知道“前 `i` 个元素”的方案数可能不足以决定第 `i+1` 个元素如何放置。例如，如果题目规定不能有两个相邻的1，那么在计算 `dp[i+1]` 时，我们需要知道第 `i` 个元素是0还是1，才能决定第 `i+1` 个元素能否放1。这个“第 `i` 个元素的状态”就需要作为DP状态的一个维度。

#### 步骤二：确定边界条件（Base Cases）

DP表的初始值。通常是 `dp[0]` 或 `dp[1]`，它们代表最小、最简单情况下的方案数。
*   `dp[0]` 常常表示空序列或没有选择任何东西，其方案数可能为1（表示“不选”本身也是一种方案）或0，这取决于具体问题。

#### 步骤三：推导转移方程

这是如何从已知的子问题方案数，推导出当前问题的方案数。
*   **核心：加法原理。** `dp[i]` 是所有能达到状态 `i` 的前一个状态的方案数之和。
*   仔细分析：要达到 `dp[i]` 这个状态，它可能通过哪些前一个状态 `dp[k]` 转移过来？把所有合法的转移路径的方案数加起来。
*   例如，如果 `dp[i]` 可以由 `dp[i-1]` 加上一个元素得到，也可以由 `dp[i-2]` 加上两个元素得到，那么 `dp[i] = dp[i-1] + dp[i-2]`。

#### 步骤四：确定计算顺序

通常是从小到大，从简单到复杂，或者按照DP状态的维度顺序（例如，先遍历 `i`，再遍历 `j`）。

#### 步骤五：处理大数（模运算）

计数DP的方案数往往非常庞大，容易超出 `long long` 的范围。因此，题目通常会要求将结果对一个大质数（如 `10^9 + 7`）取模。
*   **模运算的性质：** `(a + b) % M = ((a % M) + (b % M)) % M`。
*   这意味着在每次加法操作后，都应该立即对结果取模，以防止中间结果溢出。
*   **注意：** 减法和乘法也有类似性质，但除法没有。在计数DP中，通常只涉及加法和乘法（如果存在组合数等）。

### 4. 生活类比：搭桥过河

想象一下，你站在河的一岸，要通过一系列跳板跳到对岸。每个跳板都有一个编号。
*   **最优化DP：** 问你最少跳几步能到对岸？（求最小值）
*   **计数DP：** 问你一共有多少种不同的跳法能到对岸？（求方案数）

假设你每次可以向前跳1步或2步。
*   `dp[i]` 表示跳到第 `i` 个跳板的方案总数。
*   要跳到第 `i` 个跳板，你可以：
    1.  从第 `i-1` 个跳板跳1步过来。方案数是 `dp[i-1]`。
    2.  从第 `i-2` 个跳板跳2步过来。方案数是 `dp[i-2]`。
*   所以，`dp[i] = dp[i-1] + dp[i-2]`。
*   边界条件：`dp[0] = 1` (站在起点，算一种方案)，`dp[1] = 1` (从起点跳1步到第一个跳板)。
*   这正是斐波那契数列！这个简单的例子就完美体现了计数DP的思路。

### 5. 常见误区

*   **状态定义不完整**：遗漏了某些关键信息，导致无法正确转移。
*   **转移方程错误**：漏算了某些情况（导致计数偏低），或者重复计算了某些情况（导致计数偏高）。
*   **边界条件设置不当**：`dp[0]` 或 `dp[1]` 的初始值错误。
*   **未及时取模**：导致中间结果溢出，最终答案错误。
*   **数组越界**：DP数组开得不够大，或者索引计算错误。

## 典型例题精讲

### 例题一：铺瓷砖问题

**题目描述：**
给定一个 `1 x N` 的地板，现在要用两种瓷砖去铺：一种是 `1 x 1` 的小方砖，另一种是 `1 x 2` 的长条砖。请问，一共有多少种不同的方法可以将这个 `1 x N` 的地板铺满？
答案可能很大，请将结果对 `10^9 + 7` 取模。

**输入：** 一个整数 `N` ( `1 <= N <= 100000` )
**输出：** 铺满 `1 x N` 地板的方法总数

**思路分析：**
1.  **定义DP状态：**
    设 `dp[i]` 表示铺满 `1 x i` 长的地板的方案总数。

2.  **确定边界条件：**
    *   当 `i = 0` 时（铺满一个长度为0的地板），我们认为有一种“不铺”的方案，即 `dp[0] = 1`。这在后续计算中作为基础。
    *   当 `i = 1` 时（铺满一个长度为1的地板），只能放一个 `1 x 1` 的瓷砖，所以 `dp[1] = 1`。

3.  **推导转移方程：**
    考虑铺满 `1 x i` 的地板的最后一块瓷砖。它可能是：
    *   **一块 `1 x 1` 的小方砖：** 如果最后放的是 `1 x 1` 瓷砖，那么前面 `1 x (i-1)` 的地板必须已经铺好。铺满 `1 x (i-1)` 地板的方案数是 `dp[i-1]`。
    *   **一块 `1 x 2` 的长条砖：** 如果最后放的是 `1 x 2` 瓷砖，那么前面 `1 x (i-2)` 的地板必须已经铺好。铺满 `1 x (i-2)` 地板的方案数是 `dp[i-2]`。
    这两种情况是互不重叠的，所以根据加法原理，`dp[i]` 等于这两种情况的方案数之和。
    `dp[i] = dp[i-1] + dp[i-2]`

4.  **计算顺序：**
    从 `i = 2` 开始，依次计算到 `N`。

5.  **模运算：**
    每次加法后都要对 `10^9 + 7` 取模。

**代码实现：**

```cpp
#include <iostream> // 使用 iostream 而不是 bits/stdc++.h，因为 GESP 9 可能更注重精简
// #include <bits/stdc++.h> // 根据要求，也可以使用这个

const int MOD = 1e9 + 7; // 定义模数

int dp[100005]; // 定义DP数组，N最大100000，所以开100005

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出速度
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    // 边界条件
    dp[0] = 1; // 长度为0的地板，视为1种铺法（什么都不铺）
    dp[1] = 1; // 长度为1的地板，只能放一个1x1瓷砖，1种铺法

    // 递推计算
    for (int i = 2; i <= n; ++i) {
        dp[i] = (dp[i - 1] + dp[i - 2]) % MOD; // 状态转移方程，并取模
    }

    std::cout << dp[n] << std::endl; // 输出最终结果

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 循环从 `2` 到 `N`，每次循环执行常数次操作，所以时间复杂度为 `O(N)`。
*   **空间复杂度：** 使用了一个大小为 `N+1` 的数组 `dp`，所以空间复杂度为 `O(N)`。

### 例题二：不含连续1的二进制串

**题目描述：**
给定一个正整数 `N`，求长度为 `N` 的所有二进制串中，不包含连续两个 `1` 的方案总数。
例如，当 `N=3` 时，合法的串有："000", "001", "010", "100", "101"。共 5 种。
答案可能很大，请将结果对 `10^9 + 7` 取模。

**输入：** 一个整数 `N` ( `1 <= N <= 100000` )
**输出：** 长度为 `N` 的不含连续 `1` 的二进制串总数

**思路分析：**
1.  **定义DP状态：**
    由于题目要求不能有连续的 `1`，这意味着当前位的选择会受到前一位的影响。如果前一位是 `1`，当前位就不能是 `1`。这提示我们需要在DP状态中记录前一位的信息。
    *   `dp[i][0]`：表示长度为 `i` 的二进制串，以 `0` 结尾，且不含连续 `1` 的方案总数。
    *   `dp[i][1]`：表示长度为 `i` 的二进制串，以 `1` 结尾，且不含连续 `1` 的方案总数。

2.  **确定边界条件：**
    对于长度 `i = 1` 的二进制串：
    *   `dp[1][0] = 1` (串为 "0")
    *   `dp[1][1] = 1` (串为 "1")

3.  **推导转移方程：**
    考虑构建长度为 `i` 的串：
    *   **计算 `dp[i][0]` (以 `0` 结尾)：**
        如果第 `i` 位是 `0`，那么第 `i-1` 位可以是 `0` 也可以是 `1`，只要前面的 `i-1` 位构成的串是合法的。
        所以，`dp[i][0] = (dp[i-1][0] + dp[i-1][1]) % MOD`。
    *   **计算 `dp[i][1]` (以 `1` 结尾)：**
        如果第 `i` 位是 `1`，那么第 `i-1` 位**只能是 `0`**，否则就会出现连续的 `1`。
        所以，`dp[i][1] = dp[i-1][0] % MOD`。

4.  **计算顺序：**
    从 `i = 2` 开始，依次计算到 `N`。

5.  **最终答案：**
    长度为 `N` 的所有合法串，包括以 `0` 结尾和以 `1` 结尾的。
    所以，最终答案是 `(dp[N][0] + dp[N][1]) % MOD`。

**代码实现：**

```cpp
#include <iostream>

const int MOD = 1e9 + 7;

int dp[100005][2]; // dp[i][0] 表示以0结尾，dp[i][1] 表示以1结尾

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    // 边界条件
    dp[1][0] = 1; // "0"
    dp[1][1] = 1; // "1"

    // 递推计算
    for (int i = 2; i <= n; ++i) {
        // 第i位是0：前一位可以是0或1
        dp[i][0] = (dp[i - 1][0] + dp[i - 1][1]) % MOD;
        // 第i位是1：前一位必须是0
        dp[i][1] = dp[i - 1][0] % MOD;
    }

    // 最终结果是所有以0结尾和以1结尾的合法串之和
    int ans = (dp[n][0] + dp[n][1]) % MOD;
    std::cout << ans << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 循环从 `2` 到 `N`，每次循环执行常数次操作，所以时间复杂度为 `O(N)`。
*   **空间复杂度：** 使用了一个大小为 `(N+1) x 2` 的二维数组 `dp`，所以空间复杂度为 `O(N)`。

### 例题三：带障碍的路径计数

**题目描述：**
一个机器人位于一个 `m x n` 的网格的左上角 `(0, 0)`。
机器人每次只能向下或向右移动一步。
现在网格中有一些障碍物。障碍物用 `1` 表示，空单元格用 `0` 表示。
请问机器人从左上角到右下角 `(m-1, n-1)` 共有多少条不同的路径？
答案可能很大，请将结果对 `10^9 + 7` 取模。

**输入：**
第一行两个整数 `m, n` ( `1 <= m, n <= 1000` )
接下来 `m` 行，每行 `n` 个整数，表示网格 `g[i][j]` ( `0` 或 `1` )。

**输出：**
路径总数

**思路分析：**
1.  **定义DP状态：**
    `dp[i][j]` 表示从左上角 `(0, 0)` 到达单元格 `(i, j)` 的路径总数。

2.  **确定边界条件：**
    *   如果起点 `g[0][0]` 就是障碍物（`g[0][0] == 1`），那么无法开始，路径数为 `0`。
    *   否则，`dp[0][0] = 1` (到达起点只有1种方式)。
    *   对于第一行 `(0, j)`：如果 `(0, j)` 不是障碍物，且 `(0, j-1)` 可达，则 `dp[0][j] = dp[0][j-1]`。如果遇到障碍物，则其及之后的所有 `dp[0][k]` 都为 `0`。
    *   对于第一列 `(i, 0)`：同理，如果 `(i, 0)` 不是障碍物，且 `(i-1, 0)` 可达，则 `dp[i][0] = dp[i-1][0]`。如果遇到障碍物，则其及之后的所有 `dp[k][0]` 都为 `0`。

3.  **推导转移方程：**
    要到达 `(i, j)`，机器人只能从 `(i-1, j)` (向下移动) 或 `(i, j-1)` (向右移动) 过来。
    所以，`dp[i][j] = (dp[i-1][j] + dp[i][j-1]) % MOD`。
    **重要：** 在计算前，需要先判断当前单元格 `(i, j)` 是否是障碍物。如果是障碍物，`dp[i][j]` 直接为 `0`。

4.  **计算顺序：**
    从 `(0,0)` 开始，按行或按列遍历，确保计算 `dp[i][j]` 时，`dp[i-1][j]` 和 `dp[i][j-1]` 已经计算完毕。

5.  **最终答案：**
    `dp[m-1][n-1]`。

**代码实现：**

```cpp
#include <iostream>

const int MOD = 1e9 + 7;

int g[1005][1005]; // 存储网格
int dp[1005][1005]; // DP数组

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int m, n;
    std::cin >> m >> n;

    for (int i = 0; i < m; ++i) {
        for (int j = 0; j < n; ++j) {
            std::cin >> g[i][j];
        }
    }

    // 如果起点是障碍物，则无法到达，路径数为0
    if (g[0][0] == 1) {
        std::cout << 0 << std::endl;
        return 0;
    }

    // 边界条件：起点
    dp[0][0] = 1;

    // 填充第一行
    for (int j = 1; j < n; ++j) {
        if (g[0][j] == 0) { // 如果当前格不是障碍物
            dp[0][j] = dp[0][j - 1]; // 只能从左边过来
        } else { // 如果是障碍物，则无法到达
            dp[0][j] = 0;
        }
    }

    // 填充第一列
    for (int i = 1; i < m; ++i) {
        if (g[i][0] == 0) { // 如果当前格不是障碍物
            dp[i][0] = dp[i - 1][0]; // 只能从上边过来
        } else { // 如果是障碍物，则无法到达
            dp[i][0] = 0;
        }
    }

    // 填充剩余部分
    for (int i = 1; i < m; ++i) {
        for (int j = 1; j < n; ++j) {
            if (g[i][j] == 0) { // 如果当前格不是障碍物
                // 可以从上方或左方过来，将两种路径数相加并取模
                dp[i][j] = (dp[i - 1][j] + dp[i][j - 1]) % MOD;
            } else { // 如果是障碍物，则无法到达
                dp[i][j] = 0;
            }
        }
    }

    std::cout << dp[m - 1][n - 1] << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 读入网格 `O(M*N)`，填充DP表 `O(M*N)`。总时间复杂度为 `O(M*N)`。
*   **空间复杂度：** 使用了两个 `M x N` 的二维数组 `g` 和 `dp`，所以空间复杂度为 `O(M*N)`。

## 课堂互动

### 1. 思考题：状态的含义

**提问：**
在“不含连续1的二进制串”问题中，我们定义了 `dp[i][0]` 和 `dp[i][1]`。
如果我只定义 `dp[i]` 为“长度为 `i` 的不含连续1的二进制串总数”，你觉得能顺利写出转移方程吗？为什么？
（给学生3-5分钟思考，可以小组讨论）

**预期回答：**
不能。因为只知道 `dp[i-1]`（前 `i-1` 个串的总数），我们无法确定第 `i` 位是放 `0` 还是 `1`。如果第 `i-1` 位是 `1`，那么第 `i` 位就不能放 `1`；如果第 `i-1` 位是 `0`，那么第 `i` 位可以放 `0` 也可以放 `1`。这种依赖于前一位具体值的情况，迫使我们把前一位的状态也加入到DP状态中。

### 2. 小组任务：设计转移方程

**任务描述：**
假设现在有一种新的铺瓷砖问题：地板是 `1 x N`，瓷砖有三种：`1 x 1`，`1 x 2`，`1 x 3`。
请你和小组同学一起，设计 `dp[i]` 的状态定义，并推导出铺满 `1 x i` 地板的转移方程。
（给学生5-7分钟讨论，并请小组代表上台板书或口述）

**预期回答：**
*   **状态定义：** `dp[i]` 表示铺满 `1 x i` 地板的方案总数。
*   **边界条件：**
    *   `dp[0] = 1` (空地板，一种方案)
    *   `dp[1] = 1` (一个1x1瓷砖)
    *   `dp[2] = 2` (两个1x1，或一个1x2)
*   **转移方程：**
    考虑最后一块瓷砖：
    1.  最后一块是 `1 x 1`：前面铺好 `1 x (i-1)`，方案数 `dp[i-1]`。
    2.  最后一块是 `1 x 2`：前面铺好 `1 x (i-2)`，方案数 `dp[i-2]`。
    3.  最后一块是 `1 x 3`：前面铺好 `1 x (i-3)`，方案数 `dp[i-3]`。
    所以，`dp[i] = (dp[i-1] + dp[i-2] + dp[i-3]) % MOD`。

### 3. 即时练习：快速判断与纠错

**题目：**
小明写了一个DP代码来解决一个计数问题，他想计算一个长度为 `N` 的序列，其中每个元素是 `1` 或 `2`，要求相邻元素不能相同。他定义的DP状态和转移方程如下：
`dp[i]` 表示长度为 `i` 的满足条件的序列总数。
`dp[i] = dp[i-1] * 2` (因为第 `i` 位可以放 `1` 或 `2`，只要和 `i-1` 位不同)
`dp[1] = 2` ("1", "2")
请问小明的DP状态和转移方程是否正确？如果不对，问题出在哪里？应该如何修正？

**预期回答：**
不正确。
**问题：** `dp[i-1]` 只是前 `i-1` 个序列的总数，它没有记录第 `i-1` 位具体是什么。所以无法直接用 `dp[i-1] * 2` 来计算。如果前一位是 `1`，当前位只能是 `2`；如果前一位是 `2`，当前位只能是 `1`。这两种情况各有多少种？`dp[i-1]` 无法提供这个信息。
**修正：**
需要增加状态维度，记录最后一位的值。
*   `dp[i][1]`：长度为 `i`，以 `1` 结尾的合法序列数。
*   `dp[i][2]`：长度为 `i`，以 `2` 结尾的合法序列数。
*   **转移方程：**
    *   `dp[i][1] = dp[i-1][2]` (因为第 `i` 位是 `1`，前一位只能是 `2`)
    *   `dp[i][2] = dp[i-1][1]` (因为第 `i` 位是 `2`，前一位只能是 `1`)
*   **边界条件：**
    *   `dp[1][1] = 1` ("1")
    *   `dp[1][2] = 1` ("2")
*   **最终答案：** `(dp[N][1] + dp[N][2]) % MOD`。

这个练习旨在巩固学生对状态定义重要性的理解，以及如何通过增加状态维度来解决复杂约束。

### 4. 讨论环节：模运算的重要性

**提问：**
为什么在计数DP中，每次加法操作后都要立即取模，而不是等到最后计算出总和再取一次模？
（引导学生思考整数溢出的问题）

**预期回答：**
因为中间结果可能会非常大，超出 `int` 甚至 `long long` 的最大表示范围。一旦溢出，结果就会变得不准确，即使最后再取模也无法恢复正确值。所以，为了确保每次运算都在有效范围内，必须在每次加法后都立即取模。

### 参考答案（请学生独立完成后再核对）

#### 基础巩固答案

1.  **爬楼梯**
    *   `dp[i]`：爬到第 `i` 级楼梯的方法数。
    *   `dp[i] = dp[i-1] + dp[i-2]`
    *   `dp[0] = 1` (或 `dp[1]=1, dp[2]=2`，看具体定义)
    *   `dp[1] = 1`, `dp[2] = 2`
    *   时间复杂度 `O(N)`，空间复杂度 `O(N)`。

2.  **不同路径**
    *   `dp[i][j]`：到达 `(i, j)` 的路径数。
    *   `dp[i][j] = dp[i-1][j] + dp[i][j-1]`
    *   `dp[0][0] = 1`
    *   第一行和第一列的 `dp` 值都为 `1` (只要没有障碍)。
    *   时间复杂度 `O(M*N)`，空间复杂度 `O(M*N)`。

#### 能力提升答案

1.  **排列计数**
    *   `dp[i][last][second_last]`：长度为 `i`，倒数第一个数字是 `last`，倒数第二个数字是 `second_last` 的方案数。
    *   `last` 和 `second_last` 范围是 `1` 到 `K`。
    *   转移：`dp[i][cur][last] = sum(dp[i-1][last][prev_last])`，其中 `cur != last` 或 `cur == last` 但 `last != second_last`。
    *   更简洁的：`dp[i][last]` 表示长度为 `i`，末尾是 `last` 的合法序列数。
        *   `dp[i][cur] = sum(dp[i-1][prev])` for all `prev != cur`
        *   `dp[i][cur] += dp[i-1][cur]` if `i-1` 位的 `cur` 之前不是 `cur`。
        *   这需要更精细的状态：`dp[i][cur][count]` 表示长度为 `i`，末尾是 `cur`，且 `cur` 已经连续出现了 `count` 次的方案数。
            *   `dp[i][cur][1] = sum(dp[i-1][prev][x])` for all `prev != cur`
            *   `dp[i][cur][2] = dp[i-1][cur][1]`
            *   `dp[i][cur][3]` 为 `0` (因为不允许连续三次)
    *   时间复杂度 `O(N * K * K)` 或 `O(N * K)` (取决于状态压缩方式)，空间 `O(N * K * K)` 或 `O(N * K)`。

2.  **环形房屋染色**
    *   破环为链：先计算 `N` 个房屋排成一列，首尾颜色不相同的方案数。
    *   设 `dp[i][0]` 为第 `i` 个房屋与第 `i-1` 个房屋颜色不同，且第 `i` 个房屋与第 `1` 个房屋颜色不同的方案数。
    *   设 `dp[i][1]` 为第 `i` 个房屋与第 `i-1` 个房屋颜色不同，且第 `i` 个房屋与第 `1` 个房屋颜色相同的方案数。
    *   更直接的方法：
        1.  固定第一个房屋的颜色（例如，红色）。总方案数是固定第一个房屋颜色后计算出的方案数乘以 `3`。
        2.  对于链式结构，`dp[i][color]` 表示第 `i` 个房屋染 `color` 色的方案数。
            `dp[i][R] = dp[i-1][B] + dp[i-1][G]`
            `dp[i][B] = dp[i-1][R] + dp[i-1][G]`
            `dp[i][G] = dp[i-1][R] + dp[i-1][B]`
        3.  最终答案需要考虑第一个和最后一个房屋的颜色关系。
            *   情况1：第一个房屋和最后一个房屋颜色不同。
            *   情况2：第一个房屋和最后一个房屋颜色相同。
            这需要 `dp[i][last_color][first_color]` 或者更巧妙的递推。
    *   一个常见的方法是：
        *   假设第一个房屋为颜色 `C1`。
        *   计算 `N-1` 个房屋的链式染色方案，使得第 `N-1` 个房屋的颜色不等于 `C1`。
        *   设 `dp[i]` 为长度为 `i` 的链式，首尾颜色不同的方案数。
        *   `dp[i]` = `(dp[i-1] * (K-2) + dp[i-2] * (K-1))`
        *   对于环形，总方案数 `K * ( (K-1)^(N-1) + (-1)^N * (K-1) ) / K` （这是组合数学公式）
        *   或者 DP 方式：
            `dp[i][0]`：前 `i` 个房屋，第 `i` 个和第 `1` 个颜色相同，且中间合法。
            `dp[i][1]`：前 `i` 个房屋，第 `i` 个和第 `1` 个颜色不同，且中间合法。
            `dp[i][0] = dp[i-1][1]` (第 `i` 个和第 `i-1` 个颜色不能相同，所以第 `i-1` 个必须是不同于 `1` 的颜色，且第 `i` 个是 `1` 的颜色，这样第 `i`
