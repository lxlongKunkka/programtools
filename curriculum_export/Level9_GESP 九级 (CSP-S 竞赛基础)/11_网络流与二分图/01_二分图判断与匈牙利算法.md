## 深度知识点讲解

### 1. 二分图 (Bipartite Graph)

#### 1.1 什么是二分图？

**本质：** 二分图是一种特殊的图，它的所有顶点可以被分成两个互不相交的集合 U 和 W，并且图中所有的边都只连接 U 中的一个顶点和 W 中的一个顶点。也就是说，U 集合内部的顶点之间没有边相连，W 集合内部的顶点之间也没有边相连。

**生活类比：**
*   **男生和女生：** 你可以把所有男生看作一个集合 U，所有女生看作一个集合 W。如果一条边表示“互有好感”，那么边只能连接男生和女生，男生之间不能互相有好感（这里指配对意义上的），女生之间也不能。
*   **项目经理和程序员：** 一个项目组，项目经理负责管理，程序员负责写代码。经理和程序员之间有工作关系（边），但经理之间没有直接的工作关系（除非是上下级，但在这个模型里我们简化），程序员之间也没有。
*   **棋盘上的格子：** 国际象棋棋盘，可以把所有黑格子看作一个集合 U，所有白格子看作一个集合 W。如果一个棋子（比如马）从一个格子跳到另一个格子，它总是从黑格子跳到白格子，或者从白格子跳到黑格子。

#### 1.2 二分图的性质：没有奇数长度的环

**关键点：** 判断一个图是否是二分图，有一个非常重要的性质：一个图是二分图，当且仅当它不包含奇数长度的环（Odd Cycle）。

**为什么？**
我们来思考一下：
*   如果一个环的长度是偶数（比如 4 个顶点 A-B-C-D-A），我们尝试给它染色：A染1色，B染2色，C染1色，D染2色。回到A，发现A是1色，与D的2色不同，所以没问题。
*   如果一个环的长度是奇数（比如 3 个顶点 A-B-C-A），我们尝试染色：A染1色，B染2色，C染1色。现在，C要和A相连，但是C和A都是1色！这就矛盾了，说明无法分成两部分。
所以，一旦图中出现奇数环，这个图就肯定不是二分图。

#### 1.3 如何判断二分图？（染色法）

我们利用“没有奇数环”这个性质来判断。最常用的方法是**染色法**，它基于广度优先搜索（BFS）或深度优先搜索（DFS）。

**染色法步骤 (以BFS为例)：**
1.  **初始化：** 创建一个 `color` 数组，所有顶点颜色初始化为0（表示未染色）。
2.  **遍历顶点：** 遍历图中的每一个顶点 `u`。如果 `u` 尚未被染色：
    a.  将 `u` 染成颜色1，并将其加入队列。
    b.  **BFS遍历：** 当队列不为空时，取出队头顶点 `curr`。
        i.  遍历 `curr` 的所有邻居 `v`。
        ii. 如果 `v` 尚未被染色：将 `v` 染成与 `curr` 不同的颜色（如果 `curr` 是颜色1，`v` 染颜色2；如果 `curr` 是颜色2，`v` 染颜色1），并将 `v` 加入队列。
        iii. 如果 `v` 已经被染色：检查 `v` 的颜色是否与 `curr` 的颜色相同。如果相同，说明我们找到了一个奇数环（因为 `curr` 和 `v` 都是同色，且它们之间有边），那么这个图就**不是二分图**，立即返回 `false`。
3.  如果所有顶点都成功染色，且没有发现颜色冲突，那么这个图就是**二分图**，返回 `true`。

**常见误区：**
*   **孤立顶点：** 孤立的顶点（没有边的顶点）可以被认为是二分图的一部分，它不会影响二分图的性质。
*   **非连通图：** 一个非连通图也可以是二分图，只要它的每个连通分量都是二分图。染色法需要遍历所有未染色的顶点，确保处理了所有连通分量。

### 2. 匹配与最大匹配 (Matching & Maximum Matching)

#### 2.1 什么是匹配？

**本质：** 在一个图中，匹配是边的集合，其中任意两条边都没有公共端点。也就是说，每条边都“独占”它的两个顶点。

**生活类比：**
*   **情侣配对：** 一对情侣就是一条匹配边。如果小明和小红配对成功，那么小明就不能再和小丽配对，小红也不能再和小刚配对。
*   **社团招新：** 一个学生加入一个社团，就是一条匹配边。这个学生不能再加入其他社团，这个社团也不能再招收其他学生。

**概念：**
*   **匹配 M：** 图中边的子集，M ⊆ E，且对于任意 e1, e2 ∈ M，e1 和 e2 没有公共顶点。
*   **匹配边：** 属于匹配 M 的边。
*   **匹配点：** 匹配边的端点。
*   **非匹配点（或未匹配点）：** 不属于任何匹配边的顶点。

#### 2.2 什么是最大匹配？

**本质：** 在一个图中，边数最多的匹配就是最大匹配。我们通常的目标就是找到最大匹配。

**生活类比：** 在相亲节目中，我们希望促成尽可能多的情侣，这个“最多”就是最大匹配。

#### 2.3 什么是完美匹配？

**本质：** 如果一个匹配 M 包含了图中的所有顶点，使得所有顶点都是匹配点，那么这个匹配 M 就是完美匹配。

**注意：** 完美匹配的要求非常高，只有在特定条件下（例如图的顶点数是偶数，且每个顶点都能找到唯一的匹配）才可能存在。最大匹配是更普遍的概念。

### 3. 匈牙利算法 (Hungarian Algorithm)

匈牙利算法是一种解决二分图最大匹配问题的经典算法。它的核心思想是**寻找增广路径**。

#### 3.1 什么是增广路径 (Augmenting Path)？

**本质：** 一条连接两个非匹配点的路径，路径上的边交替地属于匹配 M 和不属于 M。

**仔细拆解：**
*   **起点和终点：** 都是非匹配点。
*   **路径上的边：** `非匹配边 -> 匹配边 -> 非匹配边 -> 匹配边 -> ... -> 非匹配边`
    *   第一条边必须是非匹配边。
    *   最后一条边也必须是非匹配边。

**生活类比（相亲大会）：**
假设我们已经有一些情侣配对了（匹配 M），但还有一些男生是单身（非匹配点）。我们想看看能不能让更多的男生找到对象。

1.  **单身男生 A** 看上了 **单身女生 X**。太好了！他们直接配对，匹配数+1。
2.  **单身男生 A** 看上了 **女生 B**，但 **女生 B** 已经和 **男生 C** 配对了（边 A-B 是非匹配边，边 B-C 是匹配边）。
    *   男生 A 对 B 说：“跟我在一起吧！”
    *   女生 B 说：“可是我有C了啊！”
    *   男生 A 说：“没关系，你去问问C，他能不能找到新的对象？”
    *   **男生 C** 去找对象：
        *   如果 **C** 找到了 **单身女生 Y**。太好了！C 和 Y 配对。这样，B 就单身了。于是 A 和 B 配对。总匹配数+1。
        *   如果 **C** 看上了 **女生 D**，但 **D** 已经和 **男生 E** 配对了…… （这个过程可以一直进行下去，形成一条链）
    *   这条“链式反应”就是增广路径。

**增广路径的魔力：**
一旦我们找到一条增广路径，我们就可以“翻转”这条路径上的所有边：
*   把原来属于 M 的边从 M 中移除。
*   把原来不属于 M 的边加入 M。
这样操作之后，匹配的边数会**增加 1**。
比如路径是 `u -- v1 == v2 -- v3 == v4 -- w` ( `--` 是非匹配边，`==` 是匹配边)。
翻转后变成 `u == v1 -- v2 == v3 -- v4 == w`。新的匹配边数比原来多了一条。

#### 3.2 匈牙利算法流程 (DFS 实现)

匈牙利算法的核心就是不断地寻找增广路径，直到找不到为止。

**数据结构：**
*   `adj[u]`：邻接表，存储左部顶点 `u` 可以连接的右部顶点列表。
*   `match[v]`：一个数组，`match[v] = u` 表示右部顶点 `v` 当前与左部顶点 `u` 匹配。如果 `v` 未匹配，则 `match[v] = -1` (或 0，取决于数组下标从1还是0开始)。
*   `vis[v]`：一个布尔数组，在每次 DFS 寻找增广路径时使用。`vis[v] = true` 表示右部顶点 `v` 在当前这次 DFS 搜索中已经被访问过。**注意：每次尝试为新的左部顶点 `u` 寻找增广路径时，`vis` 数组都需要清空！**

**算法步骤：**
1.  **初始化：** `match` 数组所有元素设为 -1 (表示所有右部顶点都未匹配)。`res = 0` (当前最大匹配数)。
2.  **遍历左部顶点：** 对于左部图的每个顶点 `i` (从 0 到 `N-1`)：
    a.  **清空 `vis` 数组：** 将 `vis` 数组所有元素设为 `false`。这是非常关键的一步！因为 `vis` 记录的是当前这次 DFS 搜索中右部顶点的访问状态，不同左部顶点的增广路径搜索是独立的。
    b.  **尝试寻找增广路径：** 调用 DFS 函数 `dfs(i)`。
    c.  如果 `dfs(i)` 返回 `true` (表示成功为左部顶点 `i` 找到了增广路径，并更新了匹配)，则 `res++`。
3.  最终 `res` 就是最大匹配数。

**DFS 函数 `dfs(u)` 的实现 (尝试为左部顶点 `u` 寻找匹配)：**
1.  **遍历 `u` 的所有邻居 `v` (右部顶点)：**
2.  **避免重复访问：** 如果 `v` 在当前 DFS 搜索中已经被访问过 (`vis[v]` 为 `true`)，则跳过 `v`。
3.  **标记访问：** 设置 `vis[v] = true`。
4.  **检查 `v` 的匹配状态：**
    a.  **如果 `v` 是非匹配点** (`match[v] == -1`)：
        *   太好了！`u` 可以和 `v` 直接配对。
        *   更新匹配：`match[v] = u`。
        *   返回 `true` (表示为 `u` 成功找到了匹配)。
    b.  **如果 `v` 已经匹配了** (`match[v]` 不为 -1)，且 `v` 的匹配对象是 `match[v]`：
        *   我们尝试让 `match[v]` 去寻找新的匹配。
        *   **递归调用 `dfs(match[v])`：** 如果 `match[v]` 成功找到了新的匹配（即 `dfs(match[v])` 返回 `true`）：
            *   那么 `v` 就可以“抛弃” `match[v]`，转而和 `u` 配对。
            *   更新匹配：`match[v] = u`。
            *   返回 `true` (表示为 `u` 成功找到了匹配)。
5.  如果遍历完 `u` 的所有邻居 `v` 都未能成功找到匹配，则返回 `false`。

**时间复杂度：**
*   二分图的判断（BFS/DFS）：O(V + E)，其中 V 是顶点数，E 是边数。
*   匈牙利算法：对于每个左部顶点，最多进行一次 DFS 搜索，每次 DFS 搜索最坏情况下会遍历所有边。所以总复杂度是 O(N * (V + E))，其中 N 是左部顶点的数量。如果使用邻接矩阵，则是 O(N * V^2)。在竞赛中，通常 V 和 E 比较大，但由于实际图结构和剪枝，DFS 效率较高，常表现为 O(N * E) 甚至更好。

**常见误区：**
*   **`vis` 数组不清空：** 这是最常见的错误！如果 `vis` 数组不清空，一次 DFS 结束后，所有被访问过的右部顶点都会被标记为 `true`，导致后续的 DFS 无法再次访问它们，从而漏掉增广路径。
*   **`match` 数组初始化不当：** 必须初始化为表示“未匹配”的值（如 -1）。
*   **非二分图使用匈牙利算法：** 匈牙利算法是针对二分图的，如果不是二分图，结果可能是错误的。

### 4. 总结与思考

*   二分图是解决“配对”问题的基础模型。
*   染色法是判断二分图的利器，核心在于检测奇数环。
*   匈牙利算法通过不断寻找增广路径来逐步增加匹配数，直到达到最大。增广路径是其灵魂！

理解了这些，你就可以解决很多实际生活中的配对问题了！

## 典型例题精讲

### 例题1：判断二分图

**题目描述：**
给定一个无向图，包含 N 个顶点和 M 条边。请判断这个图是否是二分图。
输入格式：
第一行包含两个整数 N 和 M (1 <= N <= 100, 0 <= M <= N*(N-1)/2)。
接下来 M 行，每行包含两个整数 u, v (1 <= u, v <= N)，表示 u 和 v 之间有一条边。
输出格式：
如果图是二分图，输出 "YES"，否则输出 "NO"。

**思路分析：**
这道题是二分图判断的经典应用。我们采用染色法，使用 BFS 来实现。
1.  **数据结构：** 需要一个邻接表 `adj` 来存储图，一个 `color` 数组来存储每个顶点的颜色（0表示未染色，1和2表示两种颜色）。
2.  **遍历：** 图可能不连通，所以我们需要遍历所有顶点。如果某个顶点 `i` 尚未被染色，就从 `i` 开始进行 BFS 染色。
3.  **BFS过程：**
    *   将起始顶点 `s` 染成颜色1，并加入队列。
    *   当队列不为空时，取出队头 `u`。
    *   遍历 `u` 的所有邻居 `v`：
        *   如果 `v` 未染色，将 `v` 染成与 `u` 不同的颜色，并加入队列。
        *   如果 `v` 已染色，且 `v` 的颜色与 `u` 的颜色相同，则说明发现奇数环，图不是二分图，立即返回 `false`。
4.  如果所有连通分量都成功染色，没有冲突，则图是二分图，返回 `true`。

**C++ 代码实现：**

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <string> // For outputting "YES" or "NO"

// GESP 1-5级严禁使用 vector/stack/queue 等STL容器，必须用原生数组。
// GESP 9级 (CSP-S 竞赛基础) 允许使用vector和queue等STL容器，但为了保持与低级别风格一致，这里仍然使用原生数组模拟，
// 或者在教案中明确说明对于九级可以放宽STL限制。根据要求，这里使用原生数组模拟。
// 但考虑到九级，用vector<vector<int>> adj; 更加符合实际竞赛习惯。
// 按照核心原则：STL限制 GESP 1-5级严禁，这里是9级，所以可以使用vector。
// 变量命名 a-z 单字母变量，但adj, color, q, n, m 这种常用短名是允许的。

const int MAXN = 105; // N <= 100
int n, m;
int adj[MAXN][MAXN]; // 邻接矩阵，表示是否有边，1表示有边
int degree[MAXN];    // 存储每个顶点的度，用于邻接表模拟
int graph[MAXN][MAXN]; // 模拟邻接表，graph[u][k] 表示 u 的第 k 个邻居
int color[MAXN];     // 0: 未染色, 1: 颜色1, 2: 颜色2

// 模拟队列
int q[MAXN];
int head, tail;

bool isBipartite() {
    for (int i = 1; i <= n; ++i) {
        color[i] = 0; // 初始化所有顶点未染色
    }

    for (int s = 1; s <= n; ++s) { // 遍历所有顶点，处理非连通图
        if (color[s] == 0) { // 如果当前顶点未染色，开始一次BFS
            head = 0;
            tail = 0;
            q[tail++] = s;
            color[s] = 1; // 染成颜色1

            while (head < tail) {
                int u = q[head++];

                // 遍历 u 的所有邻居
                for (int k = 0; k < degree[u]; ++k) {
                    int v = graph[u][k]; // u 的第 k 个邻居

                    if (color[v] == 0) { // 如果邻居 v 未染色
                        color[v] = 3 - color[u]; // 染成与 u 不同的颜色 (1变2, 2变1)
                        q[tail++] = v;
                    } else if (color[v] == color[u]) { // 如果邻居 v 已染色，且颜色与 u 相同
                        return false; // 发现奇数环，不是二分图
                    }
                }
            }
        }
    }
    return true; // 所有顶点都成功染色，是二分图
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m;

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        // 模拟邻接表添加边
        graph[u][degree[u]++] = v;
        graph[v][degree[v]++] = u;
    }

    if (isBipartite()) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** O(V + E)。每个顶点和每条边都会被访问常数次。其中 V 是顶点数 N，E 是边数 M。
*   **空间复杂度：** O(V + E)。用于存储图的邻接表和 `color` 数组、队列。

### 例题2：二分图最大匹配（匈牙利算法）

**题目描述：**
给定一个二分图，左部有 N 个顶点，右部有 M 个顶点。有 K 条边连接左部和右部的顶点。请找出这个二分图的最大匹配数。
输入格式：
第一行包含三个整数 N, M, K (1 <= N, M <= 100, 0 <= K <= N*M)。
接下来 K 行，每行包含两个整数 u, v (1 <= u <= N, 1 <= v <= M)，表示左部顶点 u 和右部顶点 v 之间有一条边。
输出格式：
输出最大匹配数。

**思路分析：**
这是一个标准的二分图最大匹配问题，直接使用匈牙利算法解决。
1.  **数据结构：**
    *   `adj[u]`：存储左部顶点 `u` 的邻居（右部顶点）列表。这里使用原生数组模拟邻接表。
    *   `match[v]`：存储右部顶点 `v` 当前匹配的左部顶点。初始化为 -1。
    *   `vis[v]`：布尔数组，在每次 DFS 中标记右部顶点 `v` 是否已被访问。每次 DFS 前清空。
2.  **算法流程：**
    *   初始化 `max_match = 0`。
    *   遍历左部所有顶点 `i` (从 1 到 `N`)。
    *   在每次遍历前，将 `vis` 数组全部置为 `false`。
    *   调用 `dfs(i)` 尝试为左部顶点 `i` 寻找增广路径。
    *   如果 `dfs(i)` 返回 `true`，说明成功找到并更新了匹配，`max_match` 增加 1。
3.  最终 `max_match` 即为最大匹配数。

**C++ 代码实现：**

```cpp
#include <iostream>
// #include <vector> // GESP 9级允许，但为了保持一致性，继续使用原生数组模拟

const int MAXN_LEFT = 105;  // N <= 100
const int MAXN_RIGHT = 105; // M <= 100
const int MAX_EDGE_PER_NODE = 105; // 每个左部顶点最多连接 M 个右部顶点

int n_left, n_right, k_edges; // 左部顶点数, 右部顶点数, 边数
int adj[MAXN_LEFT][MAX_EDGE_PER_NODE]; // adj[u][idx] 存储左部顶点 u 的第 idx 个邻居
int degree[MAXN_LEFT];              // degree[u] 存储左部顶点 u 的邻居数量

int match[MAXN_RIGHT]; // match[v] = u 表示右部顶点 v 匹配了左部顶点 u
bool vis[MAXN_RIGHT];  // vis[v] = true 表示右部顶点 v 在当前DFS中已被访问

// DFS 函数：尝试为左部顶点 u 寻找一个匹配
bool dfs(int u) {
    // 遍历左部顶点 u 的所有邻居 v (右部顶点)
    for (int i = 0; i < degree[u]; ++i) {
        int v = adj[u][i];

        if (vis[v]) { // 如果右部顶点 v 在当前DFS路径中已被访问，跳过
            continue;
        }
        vis[v] = true; // 标记 v 为已访问

        // 如果 v 未被匹配，或者 v 的匹配对象可以找到新的匹配
        if (match[v] == -1 || dfs(match[v])) {
            match[v] = u; // u 和 v 匹配成功
            return true;  // 找到增广路径
        }
    }
    return false; // 未能为 u 找到匹配
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n_left >> n_right >> k_edges;

    // 初始化邻接表和 match 数组
    for (int i = 1; i <= n_left; ++i) {
        degree[i] = 0; // 初始化每个左部顶点的邻居数量
    }
    for (int i = 1; i <= n_right; ++i) {
        match[i] = -1; // 初始化所有右部顶点未匹配
    }

    // 读取边
    for (int i = 0; i < k_edges; ++i) {
        int u, v;
        std::cin >> u >> v;
        adj[u][degree[u]++] = v; // 添加边 (u, v)
    }

    int max_match = 0;
    // 遍历所有左部顶点，尝试为它们寻找匹配
    for (int i = 1; i <= n_left; ++i) {
        // 每次DFS前，清空 vis 数组，因为每次DFS都是一次独立的增广路径搜索
        for (int j = 1; j <= n_right; ++j) {
            vis[j] = false;
        }
        if (dfs(i)) { // 如果成功找到增广路径
            max_match++;
        }
    }

    std::cout << max_match << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** O(N * (V + E))，其中 N 是左部顶点数，V 是总顶点数 (N_left + N_right)，E 是边数 K。在最坏情况下，每次 DFS 都需要遍历所有边。对于邻接表，每次 DFS 是 O(E)，总共 N 次，所以是 O(N * E)。
*   **空间复杂度：** O(V + E)。用于存储邻接表、`match` 数组和 `vis` 数组。

### 模板1：二分图判断 (BFS 染色法)

```cpp
#include <iostream>
// 使用原生数组模拟邻接表和队列
const int MAXN = 105; // 最大顶点数 N

int n, m; // N个顶点，M条边
int adj[MAXN][MAXN]; // 邻接表：adj[u][k] 存储 u 的第 k 个邻居
int degree[MAXN];    // degree[u] 存储顶点 u 的邻居数量
int color[MAXN];     // 0: 未染色, 1: 颜色1, 2: 颜色2

// 模拟队列
int q[MAXN];
int head, tail;

// 函数：判断图是否为二分图
bool isBipartite() {
    // 初始化所有顶点未染色
    for (int i = 1; i <= n; ++i) {
        color[i] = 0;
    }

    // 遍历所有顶点，处理可能存在的非连通图
    for (int s = 1; s <= n; ++s) {
        if (color[s] == 0) { // 如果当前顶点 s 未染色
            head = 0; // 队列头指针
            tail = 0; // 队列尾指针
            q[tail++] = s; // s 入队
            color[s] = 1;  // 将 s 染成颜色1

            while (head < tail) {
                int u = q[head++]; // u 出队

                // 遍历 u 的所有邻居 v
                for (int k = 0; k < degree[u]; ++k) {
                    int v = adj[u][k];

                    if (color[v] == 0) { // 如果 v 未染色
                        color[v] = 3 - color[u]; // 染成与 u 不同的颜色 (1变2, 2变1)
                        q[tail++] = v; // v 入队
                    } else if (color[v] == color[u]) { // 如果 v 已染色，且颜色与 u 相同
                        return false; // 发现奇数环，不是二分图
                    }
                }
            }
        }
    }
    return true; // 所有顶点都成功染色，是二分图
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m;

    // 构建邻接表
    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        adj[u][degree[u]++] = v;
        adj[v][degree[v]++] = u; // 无向图，两条边
    }

    if (isBipartite()) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0;
}
```

### 模板2：二分图最大匹配 (匈牙利算法 DFS 实现)

```cpp
#include <iostream>

const int MAXN_LEFT = 105;  // 左部最大顶点数 N
const int MAXN_RIGHT = 105; // 右部最大顶点数 M
const int MAX_EDGE_PER_NODE = 105; // 每个左部顶点最多连接 M 个右部顶点

int n_left, n_right, k_edges; // 左部顶点数 N, 右部顶点数 M, 边数 K

// 邻接表：adj[u][k] 存储左部顶点 u 的第 k 个邻居 (右部顶点)
int adj[MAXN_LEFT][MAX_EDGE_PER_NODE]; 
int degree[MAXN_LEFT]; // degree[u] 存储左部顶点 u 的邻居数量

// match[v] = u 表示右部顶点 v 当前匹配了左部顶点 u
// 如果 v 未匹配，则 match[v] = -1
int match[MAXN_RIGHT]; 

// vis[v] = true 表示右部顶点 v 在当前DFS搜索中已被访问
// 每次为新的左部顶点 u 寻找增广路径时，vis 数组都需要清空
bool vis[MAXN_RIGHT]; 

// DFS 函数：尝试为左部顶点 u 寻找一个匹配
// 返回 true 表示成功找到增广路径并更新匹配，返回 false 表示未能找到
bool dfs(int u) {
    // 遍历左部顶点 u 的所有邻居 v (右部顶点)
    for (int i = 0; i < degree[u]; ++i) {
        int v = adj[u][i];

        // 如果右部顶点 v 在当前DFS路径中已被访问过，跳过
        // 避免重复访问和死循环
        if (vis[v]) {
            continue;
        }
        vis[v] = true; // 标记 v 为已访问 (在当前DFS搜索中)

        // 两种情况可以形成增广路径：
        // 1. 如果 v 尚未被匹配 (match[v] == -1)
        // 2. 如果 v 已经被匹配，但 v 的匹配对象 (match[v]) 可以找到新的匹配
        if (match[v] == -1 || dfs(match[v])) {
            match[v] = u; // 成功找到增广路径，将 u 和 v 配对
            return true;  // 返回 true，表示为 u 找到了匹配
        }
    }
    return false; // 遍历完所有邻居，未能为 u 找到匹配
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n_left >> n_right >> k_edges;

    // 初始化邻接表和 match 数组
    for (int i = 1; i <= n_left; ++i) {
        degree[i] = 0; // 初始化每个左部顶点的邻居数量为0
    }
    for (int i = 1; i <= n_right; ++i) {
        match[i] = -1; // 初始化所有右部顶点都未匹配 (用 -1 表示)
    }

    // 读取并构建二分图的边
    for (int i = 0; i < k_edges; ++i) {
        int u, v; // u 是左部顶点，v 是右部顶点
        std::cin >> u >> v;
        adj[u][degree[u]++] = v; // 将 v 添加到 u 的邻居列表中
    }

    int max_match = 0; // 记录最大匹配数

    // 遍历所有左部顶点，尝试为它们寻找匹配
    for (int i = 1; i <= n_left; ++i) {
        // 在每次调用 dfs(i) 之前，必须清空 vis 数组
        // 这是因为每次 dfs 都是一次独立的增广路径搜索，需要从头开始标记右部顶点的访问状态
        for (int j = 1; j <= n_right; ++j) {
            vis[j] = false;
        }

        // 如果 dfs(i) 返回 true，表示成功为左部顶点 i 找到了一个匹配
        // (通过找到一条增广路径并更新了匹配)
        if (dfs(i)) {
            max_match++; // 匹配数加 1
        }
    }

    std::cout << max_match << std::endl; // 输出最终的最大匹配数

    return 0;
}
```

    *   （引导学生思考集合划分和边的连接方式）
    *   **预期回答：** 顶点可以分成两组，且所有边都只连接不同组的顶点，同组顶点之间没有边。普通图则没有这个限制。
2.  **提问二：为什么一个图如果包含奇数长度的环，就一定不是二分图？你能用染色法来解释一下吗？**
    *   （引导学生模拟染色过程，发现颜色冲突）
    *   **预期回答：** 假设从环上一个点开始染1色，它的邻居染2色，再邻居染1色……如果是奇数环，最后回到起点时，起点和它最后一个邻居的颜色会一样，但它们之间有边，所以矛盾。
3.  **提问三：增广路径的“增广”是什么意思？为什么找到一条增广路径就能让匹配数加1？**
    *   （引导学生回想相亲大会的例子，理解路径翻转的原理）
    *   **预期回答：** “增广”就是增加匹配的意思。因为它连接了两个非匹配点，路径上的边交替地属于匹配和非匹配。翻转路径上的边后，非匹配边变成匹配边，匹配边变成非匹配边，由于路径两端都是非匹配边，所以最终会多一条匹配边。
4.  **提问四：在匈牙利算法的 DFS 函数中，`vis` 数组的作用是什么？为什么每次从新的左部顶点开始 DFS 搜索时，都必须把 `vis` 数组清空？**
    *   （强调 `vis` 的局部性，避免错过增广路径）
    *   **预期回答：** `vis` 数组用来标记在当前这次 DFS 搜索中，右部顶点是否已经被访问过，防止重复访问和死循环。每次为新的左部顶点找增广路径时，都需要清空 `vis`，因为不同左部顶点的增广路径搜索是独立的，不能因为上次搜索访问过某个右部顶点，就认为这次也不能访问它。
