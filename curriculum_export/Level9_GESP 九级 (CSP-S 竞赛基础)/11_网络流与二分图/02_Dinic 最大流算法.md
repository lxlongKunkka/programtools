## 趣味引入

同学们，想象一下我们都是一家大型快递公司的老板！我们的任务是把尽可能多的包裹从总仓库（源点 S）运送到客户手中（汇点 T）。

但是，运输过程中有很多限制：
1.  **道路容量**：每条道路（边）都有它能承受的最大包裹量（容量）。比如，一条小路一天最多运100个包裹，高速公路一天能运10000个。
2.  **包裹不能凭空消失，也不能凭空出现**：除了总仓库和客户，其他中转站（节点）收到的包裹必须全部送走，不能囤积，也不能自己变出来。
3.  **我们的目标**：在这些限制下，我们一天能从总仓库运出多少包裹，送到客户手中？这就是我们今天要解决的“最大流”问题！

传统的快递公司可能每次只找一条路运送一车包裹，然后重新找路，效率有点低。今天，我们要学习一种更“聪明”的策略——Dinic 算法，它能像一个超级大脑一样，高效地规划出最大的运输量！它会怎么做呢？

## 深度知识点讲解

### 1. 网络流基础回顾 (快速通道)

*   **流网络 (Flow Network)**：一个有向图 G = (V, E)，每条边 (u, v) ∈ E 有一个非负的容量 c(u, v) ≥ 0。图中包含一个源点 S 和一个汇点 T。
*   **流 (Flow)**：对于每条边 (u, v)，有一个流量 f(u, v)，满足：
    *   **容量限制**：0 ≤ f(u, v) ≤ c(u, v) (流量不能超过容量)
    *   **流量守恒**：除了源点 S 和汇点 T，任意中间节点 u，流入 u 的总流量等于流出 u 的总流量。
    *   **反对称性**：f(u, v) = -f(v, u) (反向边流量定义)
*   **最大流 (Maximum Flow)**：在满足上述条件的前提下，从源点 S 流向汇点 T 的总流量的最大值。

**思考**：为什么需要反向边？
想象一下，你从 A 运了 5 个包裹到 B，但后来发现这条路其实不应该运这么多，或者说，从 A 到 B 的 5 个包裹，其中 2 个更应该走 A 到 C 再到 B 的路。此时，我们不能直接“撤销”A 到 B 的 2 个包裹。在残量网络中，我们可以理解为从 B 到 A 有 2 个“空位”，可以“逆向”地调整 2 个包裹的路线，这等价于 A 到 B 的流量减少了 2。这是一种**反悔机制**，让算法能找到更优的路径。

### 2. 残量网络 (Residual Network) 和增广路 (Augmenting Path)

*   **残量网络 G_f**：在任意时刻，基于当前流量 f，我们可以构建一个残量网络。它表示每条边还可以增加多少流量，或者可以“反悔”多少流量。
    *   对于原图中的一条边 (u, v)，容量为 c(u, v)，当前流量为 f(u, v)：
        *   **正向残量边**：(u, v) 的残余容量为 `c(u, v) - f(u, v)`。如果这个值 > 0，说明 (u, v) 还可以增加流量。
        *   **反向残量边**：(v, u) 的残余容量为 `f(u, v)`。如果这个值 > 0，说明可以“撤销” (u, v) 上的 `f(u, v)` 流量，这等价于在 (v, u) 上增加 `f(u, v)` 的流量。
*   **增广路**：在残量网络中，一条从 S 到 T 的路径，其中所有边的残余容量都大于 0。沿着这条路径，我们可以增加流量。增广路上所有边的最小残余容量就是这条路径可以增广的流量。

**Ford-Fulkerson 算法的核心思想**：不断在残量网络中寻找任意一条增广路，并沿其增广，直到找不到增广路为止。
**缺点**：如果每次都找到很长的增广路，或者选择不当，效率会非常低。

### 3. Dinic 算法：高效的秘诀

Dinic 算法是对 Ford-Fulkerson 算法的重大优化，它通过引入两个核心思想来提高效率：**分层图** 和 **阻塞流**。

#### 核心思想一：分层图 (Layered Network) - BFS 的力量

Dinic 算法的第一步是使用 **BFS (广度优先搜索)** 从源点 S 开始，遍历残量网络，为所有可达节点计算它们到 S 的最短距离（边数）。这个距离我们称为节点的“层级”或“深度” (`level[u]`)。

*   `level[S] = 0`。
*   对于任意边 (u, v)，如果 `level[v]` 未被访问且 `c(u, v) - f(u, v) > 0` (即正向残余容量大于0)，则 `level[v] = level[u] + 1`。
*   **关键点**：BFS 只会沿着残余容量大于 0 的边前进。如果 BFS 无法到达汇点 T，说明 S 和 T 之间已经没有增广路了，算法结束。
*   **为什么分层？**：分层图确保我们每次寻找增广路时，只考虑那些从当前层 `level[u]` 到下一层 `level[u]+1` 的边。这样，我们找到的增广路都是“最短”的（按边数），并且避免了在残量网络中随意乱走，提高了搜索效率。

#### 核心思想二：阻塞流 (Blocking Flow) - DFS 的力量

一旦分层图构建完成（即 `level` 数组计算完毕），Dinic 算法的第二步是使用 **DFS (深度优先搜索)** 在这个分层图上寻找增广路。

*   DFS 从 S 开始，只沿着满足 `level[v] == level[u] + 1` 且残余容量大于 0 的边前进。
*   当 DFS 找到一条 S 到 T 的路径时，计算这条路径上可增广的最小容量 `min_cap`。
*   将 `min_cap` 增加到总流量中，并更新路径上所有边的流量：正向边 `f(u, v)` 增加 `min_cap`，反向边 `f(v, u)` 增加 `min_cap` (即其残余容量减小 `min_cap`)。
*   **关键点**：DFS 会持续地在当前分层图上寻找增广路，直到从 S 出发，再也找不到任何一条路径能到达 T。此时，我们称找到了一条“阻塞流”。一条阻塞流可能由多条不相交的增广路组成。
*   **`cur` 数组优化**：为了避免重复搜索已经没有增广能力的边，我们为每个节点 `u` 维护一个 `cur[u]` 指针，表示 `u` 下一次应该从哪条边开始尝试 DFS。当一条边被完全用尽（容量为 0）或者无法通过它到达 T 时，`cur[u]` 就指向 `u` 的下一条边。这是一种非常重要的剪枝优化。

#### Dinic 算法的整体流程：

1.  初始化总流量 `max_flow = 0`。
2.  **循环**：
    a.  使用 BFS 构建分层图。如果 BFS 无法从 S 到达 T，则算法结束，`max_flow` 即为结果。
    b.  初始化 `cur` 数组（`cur[u]` 指向 `u` 的第一条出边）。
    c.  使用 DFS 在分层图上寻找阻塞流。每次 DFS 返回一个增广量 `d`。将 `d` 加到 `max_flow`。
    d.  重复 c，直到从 S 出发无法再找到增广路。
3.  返回 `max_flow`。

**生活类比**：
*   **BFS 分层**：就像快递公司先用地图规划出所有从总仓库到客户的“最短路径”等级。比如，第一层是总仓库，第二层是距离总仓库最近的中转站，第三层是再远一点的中转站，等等。我们只沿着这些“层级递增”的路径走。
*   **DFS 阻塞流**：在规划好的“层级地图”上，快递公司派出所有的车辆，尽可能多地一次性把包裹运走。只要有路能走，就走！直到这条“层级地图”上的所有通道都被塞满或者没有路可走了，才停止。然后，我们重新规划下一张“层级地图”（因为一些路被占用了，一些新的路可能变得可用）。
*   **`cur` 数组**：就像每个中转站的调度员，他知道哪些发货口已经没法用了，下次直接从下一个发货口开始尝试，不用每次都从头检查。

#### 时间复杂度

Dinic 算法的理论时间复杂度通常为 O(V²E)，在一些特殊图（如单位容量网络）上可以达到 O(min(V^(2/3), E^(1/2)) * E)。对于竞赛题目，O(V²E) 是一个安全的估计，足以处理 V=1000, E=10000 左右的数据规模。

### 4. 残量网络中的边更新

当通过一条正向边 (u, v) 增广了 `delta` 流量时：
*   正向边 (u, v) 的残余容量 `c(u, v) - f(u, v)` 减少 `delta`。
*   反向边 (v, u) 的残余容量 `f(u, v)` 增加 `delta`。
    *   在代码实现中，我们通常直接修改 `cap` 数组。如果 `cap[edge_idx]` 是 (u, v) 的容量，那么 `cap[edge_idx] -= delta`，同时 `cap[edge_idx^1] += delta` (其中 `edge_idx^1` 是反向边的索引，因为我们总是成对添加边)。

### 例题 1：简单最大流

**题目描述**：
给定一个流网络，包含 4 个节点和 5 条边。源点 S=0，汇点 T=3。求从 S 到 T 的最大流。
边及其容量：
*   S(0) -> A(1): 容量 10
*   S(0) -> B(2): 容量 10
*   A(1) -> B(2): 容量 2
*   A(1) -> T(3): 容量 4
*   B(2) -> T(3): 容量 8

**思路分析**：

1.  **初始化**：最大流 `max_flow = 0`。
2.  **第一次 BFS**：
    *   `level[0] = 0`
    *   从 S(0) 出发：
        *   S(0) -> A(1) (容量10): `level[1] = 1`
        *   S(0) -> B(2) (容量10): `level[2] = 1`
    *   从 A(1) 出发：
        *   A(1) -> B(2) (容量2): `level[2]` 已访问，且 `level[2]` 不等于 `level[1]+1` (都是1)，跳过。
        *   A(1) -> T(3) (容量4): `level[3] = 2`
    *   从 B(2) 出发：
        *   B(2) -> T(3) (容量8): `level[3]` 已访问，且 `level[3]` 等于 `level[2]+1` (2 == 1+1)。
    *   BFS 结束，分层图为：`level[0]=0, level[1]=1, level[2]=1, level[3]=2`。T 可达。
3.  **第一次 DFS (寻找阻塞流)**：
    *   从 S(0) 开始 DFS。
    *   路径 1: S(0) -> A(1) -> T(3)
        *   S(0) -> A(1) (容量10, level[0]->level[1])
        *   A(1) -> T(3) (容量4, level[1]->level[2])
        *   增广量 `min_cap = min(10, 4) = 4`。
        *   更新流量：S->A 流量+4 (剩余6)，A->T 流量+4 (剩余0)。残量网络中，A->S 容量+4，T->A 容量+4。
        *   `max_flow = 4`。
    *   路径 2: S(0) -> B(2) -> T(3)
        *   S(0) -> B(2) (容量10, level[0]->level[1])
        *   B(2) -> T(3) (容量8, level[1]->level[2])
        *   增广量 `min_cap = min(10, 8) = 8`。
        *   更新流量：S->B 流量+8 (剩余2)，B->T 流量+8 (剩余0)。残量网络中，B->S 容量+8，T->B 容量+8。
        *   `max_flow = 4 + 8 = 12`。
    *   路径 3: S(0) -> A(1) -> B(2) -> T(3)
        *   S(0) -> A(1) (剩余容量6, level[0]->level[1])
        *   A(1) -> B(2) (容量2, level[1]->level[1]) -> **注意这里！BFS分层时A->B的边被跳过了因为level[A]和level[B]都是1，不满足level[v] == level[u]+1。所以这条路径在当前分层图上不可行。** (这个点很重要，体现了分层图的严格性)
        *   再看S(0) -> A(1) (剩余容量6, level[0]->level[1])
        *   A(1) 已经没有指向 T(3) 的有效增广路了 (A->T 容量已为0)。
        *   S(0) -> B(2) (剩余容量2, level[0]->level[1])
        *   B(2) 已经没有指向 T(3) 的有效增广路了 (B->T 容量已为0)。
    *   当前分层图上，S 已经无法到达 T，DFS 结束，第一次阻塞流找到，总流量 `max_flow = 12`。

4.  **第二次 BFS**：
    *   S(0) -> A(1) (剩余容量6): `level[1] = 1`
    *   S(0) -> B(2) (剩余容量2): `level[2] = 1`
    *   从 A(1) 出发：A(1) -> B(2) (容量2)。`level[B]` 是1，`level[A]` 是1。不满足 `level[v] == level[u]+1`。
    *   从 B(2) 出发：B(2) -> T(3) (容量0)。无法通过。
    *   **重要**：A(1) -> T(3) 已经没有容量了。
    *   此时，BFS 无法到达 T(3)。算法结束。

5.  **最终结果**：最大流为 12。

**复杂度分析**：
对于这个小图，手动模拟的步骤较少。Dinic 算法的复杂度在理论上是 O(V²E)。V=4, E=5，计算量非常小。

### 例题 2：带有反向边调整的复杂图

**题目描述**：
给定一个流网络，包含 4 个节点和 5 条边。源点 S=0，汇点 T=3。求从 S 到 T 的最大流。
边及其容量：
*   S(0) -> A(1): 容量 7
*   S(0) -> B(2): 容量 5
*   A(1) -> C(2): 容量 6  (注意，这里 C 实际上是节点2，和 B 是同一个节点)
*   A(1) -> T(3): 容量 4
*   B(2) -> T(3): 容量 8

为了避免混淆，我们把节点 A 叫做 1，B 叫做 2，T 叫做 3。
S(0) -> 1: 容量 7
S(0) -> 2: 容量 5
1 -> 2: 容量 6
1 -> 3: 容量 4
2 -> 3: 容量 8

**思路分析**：

1.  **初始化**：`max_flow = 0`。
2.  **第一次 BFS**：
    *   `level[0] = 0`
    *   `q = [0]`
    *   弹出 0：
        *   0 -> 1 (cap 7): `level[1] = 1`
        *   0 -> 2 (cap 5): `level[2] = 1`
    *   弹出 1：
        *   1 -> 2 (cap 6): `level[2]` 已访问，且 `level[2]` 不等于 `level[1]+1` (都是1)，跳过。
        *   1 -> 3 (cap 4): `level[3] = 2`
    *   弹出 2：
        *   2 -> 3 (cap 8): `level[3]` 已访问，且 `level[3]` 等于 `level[2]+1` (2 == 1+1)。
    *   BFS 结束。`level[0]=0, level[1]=1, level[2]=1, level[3]=2`。T 可达。
3.  **第一次 DFS (寻找阻塞流)**：
    *   从 S(0) 开始 DFS。
    *   路径 1: S(0) -> 1 -> 3
        *   S(0) -> 1 (剩余7)
        *   1 -> 3 (剩余4)
        *   增广量 `min_cap = min(7, 4) = 4`。
        *   更新流量：S->1 流量+4 (剩余3)，1->3 流量+4 (剩余0)。反向边 S<-1 容量+4，3<-1 容量+4。
        *   `max_flow = 4`。
    *   路径 2: S(0) -> 2 -> 3
        *   S(0) -> 2 (剩余5)
        *   2 -> 3 (剩余8)
        *   增广量 `min_cap = min(5, 8) = 5`。
        *   更新流量：S->2 流量+5 (剩余0)，2->3 流量+5 (剩余3)。反向边 S<-2 容量+5，3<-2 容量+5。
        *   `max_flow = 4 + 5 = 9`。
    *   此时，S(0) 到 1 还有 3 容量，到 2 已经没有容量了。
    *   再尝试从 S(0) -> 1 (剩余3) 出发。
        *   1 -> 3 已经没有容量了。
        *   1 -> 2 (容量6)。`level[1]=1, level[2]=1`，不满足 `level[v] == level[u]+1`。
        *   **但是！** 此时残量网络中还有 2 -> 1 的反向边，其容量为 S->2 增广量 `5`。
        *   **注意这里**：第一次DFS只考虑了 `level[v] == level[u]+1` 的边。所以路径 `S->1->2->3` 在当前分层图上不可行。
    *   当前分层图上，S 已经无法到达 T。DFS 结束，第一次阻塞流找到，总流量 `max_flow = 9`。

4.  **第二次 BFS**：
    *   `level[0] = 0`
    *   `q = [0]`
    *   弹出 0：
        *   0 -> 1 (剩余容量3): `level[1] = 1`
        *   0 -> 2 (剩余容量0): 无法通过。
    *   弹出 1：
        *   1 -> 2 (容量6): `level[2]` 未访问 (因为上一轮 BFS 2 的层级是1，但这次重新开始，所有 level 数组被重置为-1)。所以 `level[2] = level[1]+1 = 2`。
        *   1 -> 3 (容量0): 无法通过。
    *   弹出 2：
        *   2 -> 3 (剩余容量3): `level[3] = level[2]+1 = 3`。
    *   BFS 结束。`level[0]=0, level[1]=1, level[2]=2, level[3]=3`。T 可达。
5.  **第二次 DFS (寻找阻塞流)**：
    *   从 S(0) 开始 DFS。
    *   路径 1: S(0) -> 1 -> 2 -> 3
        *   S(0) -> 1 (剩余容量3, level[0]->level[1])
        *   1 -> 2 (容量6, level[1]->level[2])
        *   2 -> 3 (剩余容量3, level[2]->level[3])
        *   增广量 `min_cap = min(3, 6, 3) = 3`。
        *   更新流量：S->1 流量+3 (剩余0)，1->2 流量+3 (剩余3)，2->3 流量+3 (剩余0)。
        *   反向边 S<-1 容量+3，2<-1 容量+3，3<-2 容量+3。
        *   `max_flow = 9 + 3 = 12`。
    *   此时，S(0) 到 1 已经没有容量了。DFS 结束。第二次阻塞流找到，总流量 `max_flow = 12`。

6.  **第三次 BFS**：
    *   S(0) -> 1 (剩余容量0): 无法通过。
    *   S(0) -> 2 (剩余容量0): 无法通过。
    *   BFS 无法从 S 到达 T。算法结束。

7.  **最终结果**：最大流为 12。

这个例子巧妙地展示了 Dinic 算法如何通过多次 BFS 构建分层图，从而找到最初被“忽略”的路径（如 S->1->2->3）。在第一次分层时，因为 S->2 也是一层，1->2 无法作为增广路。但当 S->2->3 被增广完后，新的分层图中 S->1->2->3 就可能满足层级递增条件了。

```cpp
#include <iostream>
#include <cstring> // For memset
#include <algorithm> // For std::min

// 最大节点数和最大边数 (注意：边数M是原图边数的两倍，因为每条边会添加正向和反向)
const int N = 1005; // 节点数 (根据题目 V 范围调整)
const int M = 200005; // 边数 (根据题目 E 范围调整，通常是 E * 2)
const int INF = 0x3f3f3f3f; // 表示无穷大，用于流量计算

// 存储图的邻接表 (使用数组模拟链表)
int head[N]; // head[u] 存储节点 u 的第一条边的索引
int nxt[M];  // nxt[i] 存储编号为 i 的边的下一条边的索引
int to[M];   // to[i] 存储编号为 i 的边的终点
int cap[M];  // cap[i] 存储编号为 i 的边的容量 (剩余容量)
int edge_cnt; // 当前边的总数 (索引从0开始)

// BFS 需要的辅助数组和队列
int level[N]; // level[u] 存储节点 u 在分层图中的层级
int q[N];     // 模拟队列
int q_head, q_tail; // 队列头尾指针

// DFS 需要的辅助数组 (当前弧优化)
int cur[N]; // cur[u] 存储 DFS 时节点 u 下一次从哪条边开始搜索

// 源点 S 和汇点 T
int S, T;

// 初始化图，清空邻接表
void init_graph() {
    memset(head, -1, sizeof(head)); // head 数组初始化为 -1，表示没有边
    edge_cnt = 0; // 边数从 0 开始
}

// 添加一条边 (u, v) 容量为 c
// 同时添加一条反向边 (v, u) 容量为 0 (用于残量网络的反悔机制)
void add_edge(int u, int v, int c) {
    // 正向边 u -> v
    to[edge_cnt] = v;
    cap[edge_cnt] = c;
    nxt[edge_cnt] = head[u]; // 将新边插入到 u 的邻接表头部
    head[u] = edge_cnt++;    // 更新 u 的邻接表头部，并增加边计数

    // 反向边 v -> u
    to[edge_cnt] = u;
    cap[edge_cnt] = 0; // 反向边初始容量为 0
    nxt[edge_cnt] = head[v]; // 将新边插入到 v 的邻接表头部
    head[v] = edge_cnt++;    // 更新 v 的邻接表头部，并增加边计数
}

// BFS 算法：构建分层图
// 返回 true 表示 S 到 T 仍有路径，可以继续增广；返回 false 表示 S 到 T 已无路径
bool bfs() {
    memset(level, -1, sizeof(level)); // 初始化 level 数组为 -1，表示未访问
    q_head = q_tail = 0; // 初始化队列

    q_push(S); // 源点 S 入队
    level[S] = 0; // 源点 S 的层级为 0

    while (!q_empty()) {
        int u = q_pop(); // 队头节点出队

        // 遍历 u 的所有出边
        for (int i = head[u]; i != -1; i = nxt[i]) {
            int v = to[i];
            // 如果 v 未访问 (level[v] == -1) 且边 (u, v) 还有剩余容量
            if (level[v] == -1 && cap[i] > 0) {
                level[v] = level[u] + 1; // 设置 v 的层级
                q_push(v); // v 入队
            }
        }
    }
    return level[T] != -1; // 如果汇点 T 可达 (level[T] != -1)，则返回 true
}

// DFS 算法：在分层图上寻找增广路并计算阻塞流
// u: 当前节点
// f: 当前路径上允许通过的最大流量
// 返回值：本次 DFS 找到的流量
int dfs(int u, int f) {
    if (u == T) { // 到达汇点 T，说明找到一条增广路
        return f;
    }

    int pushed_flow = 0; // 记录从当前节点 u 推送出去的总流量

    // 遍历 u 的所有出边 (使用 cur 数组进行当前弧优化)
    // cur[u] 记录了 u 上次 DFS 时尝试到的边，下次从这里继续，避免重复搜索
    for (int i = cur[u]; i != -1; i = nxt[i]) {
        cur[u] = i; // 更新 cur[u]，表示下次从这条边继续尝试

        int v = to[i];
        // 如果边 (u, v) 还有容量，且 v 的层级是 u 的下一层 (满足分层图条件)
        if (cap[i] > 0 && level[v] == level[u] + 1) {
            // 递归调用 DFS，尝试从 v 继续向下搜索
            // 可增广的流量是当前路径允许的最大流量 f 和当前边的剩余容量 cap[i] 的最小值
            int tr = dfs(v, std::min(f - pushed_flow, cap[i]));
            
            if (tr == 0) { // 如果从 v 无法再增广，跳过这条边
                continue;
            }

            // 更新边的容量：正向边容量减少，反向边容量增加
            cap[i] -= tr;
            cap[i ^ 1] += tr; // i^1 是边 i 的反向边索引 (因为成对添加，0和1，2和3...)

            pushed_flow += tr; // 累加从 u 推送出去的流量

            if (pushed_flow == f) { // 如果已经推送了当前路径允许的最大流量 f，则返回
                break;
            }
        }
    }
    return pushed_flow; // 返回从 u 推送出去的总流量
}

// Dinic 主函数
// n_nodes: 图中的节点总数 (包括 S 和 T)
// source: 源点 S 的编号
// sink: 汇点 T 的编号
// 返回值：最大流
int dinic(int n_nodes, int source, int sink) {
    S = source;
    T = sink;
    int max_flow = 0;

    // 只要 BFS 还能找到从 S 到 T 的路径，就继续循环
    while (bfs()) {
        // 每次 BFS 成功后，初始化 cur 数组 (将 cur[u] 指向 head[u])
        // 因为分层图已经改变，需要重新从每条边的起点开始尝试
        for (int i = 0; i < n_nodes; ++i) {
            cur[i] = head[i];
        }
        
        // 在当前分层图上，通过 DFS 寻找阻塞流
        // 每次 DFS 可能会找到多条增广路，直到从 S 无法再增广为止
        int flow_found;
        while ((flow_found = dfs(S, INF)) > 0) {
            max_flow += flow_found;
        }
    }
    return max_flow;
}

// ------------------- 队列函数 (为了符合 GESP 1-5 级不能用 STL 容器的要求) -------------------
// 实际竞赛中，直接使用 std::queue<int> q; q.push(x); q.front(); q.pop(); q.empty(); 更简洁高效。
void q_push(int x) {
    q[q_tail++] = x;
}

int q_pop() {
    return q[q_head++];
}

bool q_empty() {
    return q_head == q_tail;
}
// ---------------------------------------------------------------------------------------------

int main() {
    // 示例用法：
    // 假设有 4 个节点 (0, 1, 2, 3)，源点 S=0，汇点 T=3
    // 节点数 V，边数 E
    int n, m;
    // std::cin >> n >> m; // 实际比赛中会从输入读取节点数和边数

    // 假设例题1的图：
    // S=0, T=3
    // S(0) -> A(1): 容量 10
    // S(0) -> B(2): 容量 10
    // A(1) -> B(2): 容量 2
    // A(1) -> T(3): 容量 4
    // B(2) -> T(3): 容量 8

    n = 4; // 节点数 0, 1, 2, 3
    S = 0;
    T = 3;

    init_graph();
    add_edge(0, 1, 10);
    add_edge(0, 2, 10);
    add_edge(1, 2, 2);
    add_edge(1, 3, 4);
    add_edge(2, 3, 8);

    int result = dinic(n, S, T);
    std::cout << "Max Flow (Example 1): " << result << std::endl; // 预期输出 12

    // 假设例题2的图：
    // S=0, T=3
    // S(0) -> 1: 容量 7
    // S(0) -> 2: 容量 5
    // 1 -> 2: 容量 6
    // 1 -> 3: 容量 4
    // 2 -> 3: 容量 8

    init_graph(); // 重新初始化图
    add_edge(0, 1, 7);
    add_edge(0, 2, 5);
    add_edge(1, 2, 6);
    add_edge(1, 3, 4);
    add_edge(2, 3, 8);

    result = dinic(n, S, T);
    std::cout << "Max Flow (Example 2): " << result << std::endl; // 预期输出 12

    return 0;
}
```
