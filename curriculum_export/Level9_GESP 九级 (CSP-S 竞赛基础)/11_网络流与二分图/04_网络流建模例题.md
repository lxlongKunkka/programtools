## 深度知识点讲解

### 1. 网络流：资源调度的“秘密地图”

网络流，顾名思义，就是在“网络”中“流动”的东西。这个“网络”是一个有向图，而“流动”的东西可以是水、电、信息、货物，甚至是匹配关系。

*   **节点 (Node / Vertex):** 相当于快递网络中的仓库、分拣中心、顾客家。它们是资源的“中转站”。
*   **边 (Edge):** 相当于快递网络中的运输路线、卡车、传送带。它们是资源“流动”的路径。
*   **源点 (Source `s`):** 资源的起点，所有流量的“发源地”，比如我们的总仓库。
*   **汇点 (Sink `t`):** 资源的终点，所有流量的“目的地”，比如顾客的家。
*   **容量 (Capacity `c(u,v)`):** 一条边能承载的最大流量。比如一辆卡车最多能运100个包裹，这就是它的容量。
*   **流量 (Flow `f(u,v)`):** 实际上通过一条边的流量。比如卡车运了80个包裹，那么流量就是80。流量不能超过容量！

**网络流的核心思想：**
1.  **容量限制：** 任何一条边上的流量不能超过它的容量。`0 <= f(u,v) <= c(u,v)`
2.  **流量守恒：** 除了源点和汇点，任何一个中间节点，流入的流量必须等于流出的流量。想象一个分拣中心，收到的包裹数量，必须等于它发出去的包裹数量。`sum(f(v,u)) = sum(f(u,w))`
3.  **斜对称性：** 如果有流量从u到v，那么从v到u的流量就应该减少相同的值。这主要是为了方便算法处理“反悔”操作。`f(u,v) = -f(v,u)`

**目标：** 在满足上述条件的前提下，从源点到汇点的总流量最大化，这就是**最大流问题**。

### 2. 网络流建模：把问题翻译成“网络语言”

网络流建模，就是把一个实际问题，巧妙地转化成一个网络流图。这就像把中文翻译成英文一样，需要找到对应的词汇和语法。

**建模的关键步骤：**

1.  **识别“流动”的资源：** 问题中什么东西在“移动”？是人？是物品？是任务？这就是我们的“流量”。
2.  **确定“起点”和“终点”：** 资源的源头在哪里？最终要达到哪里？这通常就是我们的源点 `s` 和汇点 `t`。
3.  **设计“中间节点”：** 资源在流动过程中会经过哪些“站点”？这些站点有什么“属性”？它们就是图中的中间节点。
    *   **节点类型：** 有时需要将实际实体（如工人、机器、任务）分别创建为不同类型的节点。
    *   **层级关系：** 节点之间可能存在某种顺序或依赖关系，这决定了边的方向。
4.  **构建“连接关系”和“容量”：**
    *   **边：** 哪些节点之间可以互相连接？资源的流动路径是什么？
    *   **容量：** 连接路径有什么限制？节点的属性（如处理能力、可使用次数）如何体现在边的容量上？

    **特别注意：节点容量的处理！**
    有些问题会说“某个分拣中心最多能处理100个包裹”，这其实是**节点容量**。但网络流的容量是定义在**边**上的。怎么办呢？
    **技巧：拆点法 (Node Splitting)**
    把一个有容量限制的节点 `u` 拆分成两个节点：`u_in` 和 `u_out`。
    *   所有指向 `u` 的边，都改指向 `u_in`。
    *   所有从 `u` 出发的边，都改从 `u_out` 出发。
    *   在 `u_in` 和 `u_out` 之间添加一条有向边 `(u_in, u_out)`，容量设置为节点 `u` 的容量。
    这样，流经 `(u_in, u_out)` 的流量就代表了节点 `u` 的处理流量，且受到了容量限制！

### 3. 建模的“套路”与“灵感”

建模没有固定公式，但有一些常见的“套路”可以借鉴：

*   **二分图匹配：** 经典的匹配问题（如工人分配任务、学生选择课程），可以直接转化为网络流。
    *   源点 `s` 连接左部节点（容量1）。
    *   左部节点连接右部节点（容量1，如果可匹配）。
    *   右部节点连接汇点 `t`（容量1）。
    *   最大流就是最大匹配数。
*   **多源多汇：** 如果问题有多个源头和多个汇点，可以创建一个“超级源点”连接所有实际源点，创建一个“超级汇点”连接所有实际汇点。
*   **最小割与最大流：** Max-Flow Min-Cut Theorem 是网络流最强大的理论之一。它告诉我们，一个网络的最大流等于它的最小割。这意味着某些“最小代价切断”的问题，也可以转化为最大流。这通常用于一些“选择”或“删除”的问题，但对于初学建模来说，先掌握最大流的应用更为重要。

**建模就像玩乐高：** 你手头有各种形状的积木（节点、边、容量），你需要根据图纸（问题描述），巧妙地组合它们，搭建出你想要的模型。有时候需要一点想象力，把看似不相关的元素，用网络流的语言重新解释。

**关键在于多练习，多思考“为什么这样建图？”**

### 例题一：食堂打饭问题 (二分图多重匹配)

**题目描述：**
某个学校有 `n` 个学生和 `m` 个窗口。每个学生 `i` 都有自己偏爱的一些窗口 `j`。每个窗口 `j` 每天最多能为 `c_j` 个学生打饭。现在校长希望每天能有尽可能多的学生打到饭，请问最多能有多少学生打到饭？

**分析与建模：**

1.  **流动资源：** 学生。我们希望“学生”这个资源从“未打饭”状态流向“已打饭”状态。
2.  **源点 `s` 和汇点 `t`：**
    *   源点 `s`：代表所有未打饭的学生。
    *   汇点 `t`：代表所有已打饭的学生。
3.  **中间节点：**
    *   所有学生 `S_1, S_2, ..., S_n`。
    *   所有窗口 `W_1, W_2, ..., W_m`。
4.  **构建边和容量：**
    *   **`s` 到学生节点：** 从源点 `s` 向每个学生 `S_i` 连接一条边 `(s, S_i)`，容量为 `1`。这表示每个学生最多只能打一次饭。
    *   **学生节点到窗口节点：** 如果学生 `S_i` 偏爱窗口 `W_j`，则从 `S_i` 向 `W_j` 连接一条边 `(S_i, W_j)`，容量为 `1`。这表示一个学生只能通过一条路径（选择一个窗口）去打饭。
    *   **窗口节点到 `t`：** 从每个窗口 `W_j` 向汇点 `t` 连接一条边 `(W_j, t)`，容量为 `c_j`。这表示窗口 `W_j` 最多能为 `c_j` 个学生打饭。

**图示：**
```
      s
      | \
      |  \
      S1--W1--
      |   |  \
      S2--W2--- t
      |   |  /
      S3--W3--
      /   |
     /    |
    s ---- S_n
```

**最大流的意义：** 从 `s` 到 `t` 的最大流，就代表了最多能有多少学生打到饭。

**代码实现思路 (Dinic 算法框架):**
```cpp
#include <iostream>
#include <queue>
#include <cstring>
#include <algorithm> // For std::min

const int INF = 0x3f3f3f3f; // 表示无穷大
const int MAXN = 2005;     // 节点数上限 (n学生 + m窗口 + s + t)
const int MAXM = 200005;   // 边数上限 (s->S, S->W, W->t)

int head[MAXN];
int to[MAXM], nxt[MAXM], cap[MAXM];
int tot = 1; // 边的计数器，从1开始，方便异或操作找到反向边

int n_stu, m_win; // 学生数，窗口数
int s_node, t_node; // 源点和汇点

int level[MAXN]; // BFS分层
int cur[MAXN];   // 当前弧优化

void add_edge(int u, int v, int c) {
    to[++tot] = v; nxt[tot] = head[u]; cap[tot] = c; head[u] = tot;
    to[++tot] = u; nxt[tot] = head[v]; cap[tot] = 0; head[v] = tot; // 反向边初始容量为0
}

bool bfs() {
    memset(level, 0, sizeof(level));
    std::queue<int> q;
    q.push(s_node);
    level[s_node] = 1;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int i = head[u]; i; i = nxt[i]) {
            int v = to[i];
            if (cap[i] > 0 && level[v] == 0) {
                level[v] = level[u] + 1;
                q.push(v);
            }
        }
    }
    return level[t_node] != 0;
}

int dfs(int u, int flow) {
    if (u == t_node) {
        return flow;
    }
    int pushed = 0;
    for (int i = cur[u]; i; i = nxt[i]) { // 当前弧优化
        cur[u] = i; // 更新当前弧
        int v = to[i];
        if (cap[i] > 0 && level[v] == level[u] + 1) {
            int pushed_v = dfs(v, std::min(flow - pushed, cap[i]));
            if (pushed_v == 0) continue;
            cap[i] -= pushed_v;
            cap[i ^ 1] += pushed_v; // 反向边增加流量
            pushed += pushed_v;
            if (pushed == flow) break;
        }
    }
    return pushed;
}

int dinic() {
    int max_flow = 0;
    while (bfs()) {
        for (int i = 0; i <= m_win + n_stu + 1; ++i) { // 节点编号从0开始，最大节点为m_win+n_stu+1
            cur[i] = head[i]; // 每次BFS后重置当前弧
        }
        max_flow += dfs(s_node, INF);
    }
    return max_flow;
}
```

**复杂度分析：**
*   节点数 `V = n + m + 2` (学生数 + 窗口数 + 源点 + 汇点)。
*   边数 `E = n` (s到学生) `+ k` (学生到窗口，k是所有学生偏好窗口的总数) `+ m` (窗口到t)。
*   Dinic 算法在一般图上的时间复杂度是 `O(V^2 * E)`，但在特殊图（如二分图）上优化到 `O(E * sqrt(V))` 或 `O(E * V^(2/3))`。对于本题，节点数和边数相对较小，Dinic算法能够高效解决。

### 例题二：工厂生产计划 (带节点容量的资源分配)

**题目描述：**
某工厂有 `N` 条生产线，每条生产线每天最多能生产 `P_i` 个产品。这些产品需要运送到 `M` 个仓库，每个仓库 `j` 每天最多能接收 `Q_j` 个产品。从生产线 `i` 运送产品到仓库 `j` 的最大运输量是 `T_{ij}`。现在工厂希望每天能生产并运送尽可能多的产品到仓库。问最多能运送多少产品？

**分析与建模：**

1.  **流动资源：** 产品。
2.  **源点 `s` 和汇点 `t`：**
    *   源点 `s`：代表所有待生产的产品。
    *   汇点 `t`：代表所有已进入仓库的产品。
3.  **中间节点：**
    *   所有生产线 `L_1, L_2, ..., L_N`。
    *   所有仓库 `R_1, R_2, ..., R_M`。

4.  **构建边和容量：**
    *   **生产线容量处理 (拆点法)：** 每条生产线 `L_i` 有生产上限 `P_i`。这是节点容量。
        *   将 `L_i` 拆分成 `L_i_in` 和 `L_i_out` 两个节点。
        *   添加边 `(L_i_in, L_i_out)`，容量为 `P_i`。
    *   **仓库容量处理 (拆点法)：** 每个仓库 `R_j` 有接收上限 `Q_j`。这也是节点容量。
        *   将 `R_j` 拆分成 `R_j_in` 和 `R_j_out` 两个节点。
        *   添加边 `(R_j_in, R_j_out)`，容量为 `Q_j`。

    *   **`s` 到生产线：** 从源点 `s` 连接到每个生产线的 `L_i_in` 节点，容量为 `P_i`。实际上，这里可以直接连接到 `L_i_in` 节点，因为 `L_i_in` 和 `L_i_out` 之间的边已经限制了生产线的总容量。所以，为了简洁和避免重复限制，我们可以设 `s` 到 `L_i_in` 的容量为 `INF` (无穷大)，或者就是 `P_i`。为了模型的一致性，通常设为 `INF`，让 `L_i_in -> L_i_out` 的边来限制。但如果 `s` 到 `L_i_in` 的边容量设为 `P_i`，且 `L_i_in` 到 `L_i_out` 的边容量也设为 `P_i`，效果是一样的。这里我们选择直接让 `s` 连接到 `L_i_out` 的前一个点 `L_i_in`，并让 `s` 到 `L_i_in` 的容量为 `P_i`，这样就涵盖了生产线的总生产能力。
        *   **更常见的拆点用法：** 源点 `s` 到 `L_i_in`，容量 `INF`。然后 `L_i_in` 到 `L_i_out`，容量 `P_i`。然后 `L_i_out` 到 `R_j_in`，容量 `T_{ij}`。然后 `R_j_in` 到 `R_j_out`，容量 `Q_j`。最后 `R_j_out` 到 `t`，容量 `INF`。
        *   **简化：** 对于源点 `s` 和汇点 `t` 的直接连接，如果它们本身不是瓶颈，容量可以设为 `INF`。
            *   `s` 到每个生产线 `L_i_in`：边 `(s, L_i_in)`，容量 `P_i`。 (直接限制生产线总产出)
            *   每个仓库 `R_j_out` 到 `t`：边 `(R_j_out, t)`，容量 `Q_j`。 (直接限制仓库总接收)
    *   **生产线到仓库：** 从生产线 `L_i_out` 到仓库 `R_j_in` 连接边 `(L_i_out, R_j_in)`，容量为 `T_{ij}`。这表示从生产线 `i` 到仓库 `j` 的最大运输量。

**节点编号约定：**
*   `s = 0`
*   `t = N + M + N + M + 1` (如果每个节点都拆点，总节点数会翻倍)
*   生产线 `L_i_in`: `i` (1到N)
*   生产线 `L_i_out`: `N + i` (1到N)
*   仓库 `R_j_in`: `N + N + j` (1到M)
*   仓库 `R_j_out`: `N + N + M + j` (1到M)
    *   总节点数 `2N + 2M + 2`

**图示：**
```
      s
      | \
      |  \
   L1_in--L1_out----R1_in--R1_out--
      |   | \       / | \      /  |
   L2_in--L2_out----R2_in--R2_out--- t
      |   | /       \ | /      \  |
   LN_in--LN_out----RM_in--RM_out--
```

**最大流的意义：** 从 `s` 到 `t` 的最大流，就代表了最多能有多少产品被运送到仓库。

```cpp
// R_j_out:  2 * n_line + m_warehouse + j (1 to m_warehouse)
```

**复杂度分析：**
*   节点数 `V = 2 * N + 2 * M + 2`。
*   边数 `E = N` (s到生产线) `+ N` (生产线in-out) `+ M` (仓库in-out) `+ M` (仓库out到t) `+ N * M` (生产线到仓库)。大约 `O(N*M + N + M)`。
*   Dinic 算法的时间复杂度 `O(V^2 * E)`，在此类问题中通常表现更好。

### 总结建模技巧：

1.  **确定流量对象：** 什么东西在流动？
2.  **确定源汇点：** 流量的起点和终点。
3.  **确定中间节点：** 实体、阶段、状态等。
4.  **构建边：** 实体间的关系、流程。
5.  **设置容量：** 限制条件，包括对节点容量的拆点处理。
6.  **画图验证：** 绘制小规模的图，手动模拟一下流量，检查是否符合问题逻辑。

## 代码实现模板

这里提供一个基于 Dinic 算法的通用最大流模板，使用原生数组实现邻接表，并遵循 Allman 风格。

```cpp
#include <iostream>
#include <queue>
#include <cstring> // For memset
#include <algorithm> // For std::min

// 定义常量
const int INF = 0x3f3f3f3f; // 表示无穷大，用于初始化或表示无限容量
const int MAX_NODES = 2005; // 最大节点数，根据题目需求调整
const int MAX_EDGES = 200005; // 最大边数，注意每条边会创建正向和反向两条边

// 邻接表存储图
int head[MAX_NODES]; // 存储每个节点的第一条边的索引
int to[MAX_EDGES];   // 存储边的终点
int nxt[MAX_EDGES];  // 存储下一条边的索引
int cap[MAX_EDGES];  // 存储边的容量
int tot = 1;         // 边的计数器，从1开始，方便通过异或操作找到反向边 (i^1)

// Dinic 算法相关变量
int s_node, t_node; // 源点和汇点
int num_nodes;       // 实际使用的节点数量 (不包括s, t)
int level[MAX_NODES]; // BFS分层图的层级
int cur[MAX_NODES];   // 当前弧优化：记录每个节点当前DFS遍历到的边

// 添加一条从 u 到 v 容量为 c 的有向边
void add_edge(int u, int v, int c)
{
    to[++tot] = v;
    nxt[tot] = head[u];
    cap[tot] = c;
    head[u] = tot;

    to[++tot] = u; // 反向边
    nxt[tot] = head[v];
    cap[tot] = 0; // 反向边初始容量为0
    head[v] = tot;
}

// BFS 构建分层图
bool bfs()
{
    // 将level数组清零，表示所有节点都未访问
    memset(level, 0, sizeof(level));
    std::queue<int> q;

    q.push(s_node);
    level[s_node] = 1; // 源点层级为1

    while (!q.empty())
    {
        int u = q.front();
        q.pop();

        // 遍历所有从 u 出发的边
        for (int i = head[u]; i; i = nxt[i])
        {
            int v = to[i];
            // 如果边有剩余容量且终点 v 未被访问
            if (cap[i] > 0 && level[v] == 0)
            {
                level[v] = level[u] + 1; // 设置 v 的层级
                q.push(v);
            }
        }
    }
    // 如果汇点 t_node 可达，则返回 true
    return level[t_node] != 0;
}

// DFS 在分层图上寻找增广路径并增加流量
// u: 当前节点
// flow: 当前路径上允许的最大流量 (瓶颈)
int dfs(int u, int flow)
{
    // 到达汇点，返回当前流量
    if (u == t_node)
    {
        return flow;
    }

    int pushed = 0; // 记录从 u 推出去的总流量

    // 遍历从 u 出发的边，使用当前弧优化
    for (int i = cur[u]; i; i = nxt[i])
    {
        cur[u] = i; // 更新当前弧，避免重复遍历已饱和的边

        int v = to[i];
        // 如果边有剩余容量，且 v 在分层图中的层级是 u 的下一层
        if (cap[i] > 0 && level[v] == level[u] + 1)
        {
            // 递归调用 DFS 寻找从 v 到 t 的流量
            int pushed_v = dfs(v, std::min(flow - pushed, cap[i]));

            // 如果从 v 无法再推出流量，则跳过此边
            if (pushed_v == 0)
            {
                continue;
            }

            // 更新边的容量：正向边减少流量，反向边增加流量
            cap[i] -= pushed_v;
            cap[i ^ 1] += pushed_v;
            pushed += pushed_v;

            // 如果当前路径的流量已用尽，则提前退出
            if (pushed == flow)
            {
                break;
            }
        }
    }
    return pushed; // 返回从 u 推出去的总流量
}

// Dinic 算法主函数，计算最大流
int dinic()
{
    int max_flow = 0;
    // 只要能通过 BFS 找到一条从 s 到 t 的路径（即分层图存在），就继续增广
    while (bfs())
    {
        // 每次 BFS 后，重置所有节点的当前弧指针
        // 注意这里的循环范围需要覆盖所有可能的节点编号
        for (int i = 0; i <= num_nodes; ++i) // num_nodes 应该是图中最大的节点编号
        {
            cur[i] = head[i];
        }
        // 不断调用 DFS 寻找增广路径，直到无法找到更多流量
        max_flow += dfs(s_node, INF);
    }
    return max_flow;
}
```

1.  **题目：最小点覆盖与最大匹配**

    *   **提示：** Konig 定理指出，二分图的最小点覆盖数等于最大匹配数。因此，先求出最大匹配，然后利用最大流的残量网络性质，通过DFS找到一个最小点覆盖。这需要对最大流算法的内部机制有更深的理解。
