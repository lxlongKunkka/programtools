## 趣味引入

同学们，你们玩过“大富翁”这样的棋盘游戏吗？或者简单的掷骰子游戏？

想象一下这样一个场景：
小明和小红正在玩一个特别的“跳格子”游戏。棋盘上有 `N` 个格子，从 `0` 到 `N` 编号。他们从 `0` 号格子出发，目标是到达 `N` 号格子。每次掷一个神奇的骰子，它只有两面：一面写着“走一步”，一面写着“走两步”，并且每次掷出这两面的概率都是 `0.5`。

小明玩了一会儿，他突然很好奇：“我平均要掷多少次骰子才能走到 `N` 号格子呢？”小红听了也犯愁了，因为每次走几步是随机的，怎么能算出一个“平均”的次数呢？

这就是我们今天要面对的问题！“平均”这个词，在数学上就叫做“期望”。我们要求的是“期望步数”。

通常我们思考从 `0` 到 `N`，会觉得是“正向”的。但对于期望步数问题，尤其是目标明确（终点）的情况，我们换个角度思考会更巧妙：
如果我已经到达了 `N` 号格子，我还需要掷骰子吗？当然不用了，因为目标已经达成！所以，从 `N` 号格子到 `N` 号格子的期望步数是 `0`。
那如果我在 `N-1` 号格子呢？我只需要再掷一次骰子，就能到达 `N` 了。
如果我在 `N-2` 号格子呢？我可能掷一次走到 `N-1`，再从 `N-1` 走到 `N`；也可能掷一次直接走到 `N`。
咦，是不是感觉有点像“倒着推”？没错，这就是我们今天的主角——**逆推期望DP**！

它就像解一道谜题，从谜底（终点）开始，一步步逆向推理，最终找到谜题的答案（起点）。这种逆向思维，在很多期望问题中都显得特别强大和自然。

## 深度知识点讲解

### 1. 什么是期望？（快速回顾）

在概率论中，**期望**（Expectation）代表了随机变量所有可能取值的加权平均，权重就是每个取值对应的概率。
对于离散随机变量 `X`，其期望 `E[X]` 定义为：
`E[X] = Σ (x_i * P(X = x_i))`
其中 `x_i` 是 `X` 可能取到的值，`P(X = x_i)` 是 `X` 取到 `x_i` 的概率。

**生活类比：**
假设你买彩票，有 `0.01` 的概率中 `100` 元，`0.05` 的概率中 `10` 元，`0.94` 的概率中 `0` 元。
你的期望奖金就是：`100 * 0.01 + 10 * 0.05 + 0 * 0.94 = 1 + 0.5 + 0 = 1.5` 元。
这表示你平均每次买彩票能获得 `1.5` 元的奖金，即使你每次只能中 `0`、`10` 或 `100` 元。

### 2. 什么是期望DP？

当一个状态的期望值，依赖于其他一个或多个状态的期望值时，我们就可以用**动态规划（DP）**来求解。这就像我们之前学习的常规DP，只不过DP数组里存储的不再是最大值、最小值或者方案数，而是**期望值**。

### 3. 为什么要逆推？（核心思想）

在很多“求从状态 `S` 到达目标状态 `T` 的期望步数”的问题中，逆推DP有着天然的优势。

*   **状态定义：** 我们通常定义 `dp[i]` 表示**从状态 `i` 出发，到达目标状态 `T` 的期望步数**。
*   **目标状态：** 既然 `dp[i]` 是从 `i` 到 `T` 的期望步数，那么如果 `i` 就是目标状态 `T`，我们就不需要再走了，所以 `dp[T] = 0`。这是一个非常清晰的边界条件！
*   **转移方程：** 假设从状态 `i`，我们有 `k` 种可能的转移，分别以概率 `p_1, p_2, ..., p_k` 转移到状态 `j_1, j_2, ..., j_k`。那么从 `i` 到达 `T` 的期望步数 `dp[i]`，可以这样计算：
    `dp[i] = 1 + Σ (p_m * dp[j_m])`
    这里的 `1` 表示从状态 `i` 走到下一个状态 `j_m` 所花费的**当前这一步**。而 `dp[j_m]` 则是从 `j_m` 到达目标 `T` 的期望步数。我们将所有可能的下一步的期望步数加权平均，再加上当前这一步的消耗，就得到了 `dp[i]`。

**为什么是逆推？**
想象一下，我们想从 `0` 号格子走到 `N` 号格子。
如果我们正向定义 `dp[i]` 为“从 `0` 到达 `i` 的期望步数”，那么 `dp[i]` 会依赖于 `dp[i-1]` 或 `dp[i-2]`。但问题是，从 `i-1` 到 `i` 的概率是已知的，但从 `i-1` 到 `N` 的期望步数，和从 `i-2` 到 `N` 的期望步数，我们并不知道。这使得正向定义很难建立起清晰的递推关系。
然而，当我们逆向定义 `dp[i]` 为“从 `i` 到 `N` 的期望步数”时：
*   我们知道终点 `dp[N] = 0`。
*   我们知道从 `i` 走到 `j` 这一步的消耗是 `1`。
*   我们知道从 `j` 到 `N` 的期望步数是 `dp[j]`。
这样，我们就可以从终点开始，一步步向前（或者说，逆着路径方向）推导，直到计算出起点的期望值。

**生活类比：**
你玩一个迷宫游戏，目标是走到出口。如果你问：“从起点走到出口平均要走多少步？”这很难直接回答。
但如果你问：“如果我站在某个位置 `X`，我平均要走多少步才能走到出口？”
*   如果 `X` 就是出口，那么 `0` 步。
*   如果 `X` 旁边就是出口，那么 `1` 步。
*   如果 `X` 旁边有多个选择，每个选择通向不同的 `Y`、`Z` 位置，那么从 `X` 到出口的期望步数就是 `1 + (从X到Y的概率 * 从Y到出口的期望步数) + (从X到Z的概率 * 从Z到出口的期望步数)`。
你看，这种思考方式，天然就是从出口向内（逆向）推导的。

### 4. 关键点剖析

*   **状态定义必须清晰：** `dp[i]` 到底代表什么？是“从起点到 `i` 的期望”还是“从 `i` 到终点的期望”？一旦确定，就不能混淆。对于逆推，通常是“从 `i` 到终点”。
*   **目标状态的初始化：** 确保所有目标状态的期望值被正确设置为 `0`。
*   **转移方向和顺序：**
    *   对于线性DP（如格子游戏），通常从 `N-1` 倒推到 `0`。
    *   对于有向无环图（DAG），需要**反向拓扑排序**。也就是先处理那些只能转移到目标状态的节点，再处理那些能转移到已处理节点的节点。或者使用**记忆化搜索**，它能自动处理这种依赖关系。
    *   对于树形DP，如果目标是根节点或叶子节点，可能需要两次DFS（自底向上和自顶向下），或者通过解线性方程组来处理父子之间的相互依赖。
*   **概率计算：** 务必确保从状态 `i` 转移到状态 `j` 的概率 `p_j` 计算正确。
*   **别忘了加 `1`：** 每走一步，步数就增加 `1`。这个 `1` 经常被遗漏。

### 5. 常见误区

*   **混淆正推与逆推：** 期望DP既可以正推也可以逆推，但具体问题需要选择最合适的策略。逆推往往在“求到达某个终点的期望步数”时更自然。
*   **忘记当前步的消耗 `1`：** 这是最常见的错误之一。
*   **概率计算错误：** 尤其是在有多种选择且概率不均等时。
*   **状态定义不清晰：** 导致转移方程混乱或错误。
*   **循环依赖问题：** 在图中有环时，简单的DP顺序无法解决，可能需要高斯消元。但对于逆推期望步数，我们通常处理的是DAG或树，避免了直接的循环依赖。

## 典型例题精讲

### 例题1：线性格子游戏

**题目描述：**
一个 `N` 格的棋盘（编号 `0` 到 `N`）。你从 `0` 号格子出发，目标是到达 `N` 号格子。每次你可以选择向前走 `1` 格或者 `2` 格。
*   从当前格子 `i` 走 `1` 格到 `i+1` 的概率是 `p1`。
*   从当前格子 `i` 走 `2` 格到 `i+2` 的概率是 `p2`。
其中 `p1 + p2 = 1`。
求从 `0` 号格子到达 `N` 号格子的期望步数。

**思路分析：**
1.  **状态定义：** 设 `dp[i]` 表示从格子 `i` 出发，到达格子 `N` 的期望步数。
2.  **目标状态：** 当我们已经在格子 `N` 时，不需要再走了，所以 `dp[N] = 0`。
3.  **状态转移方程：** 对于任意格子 `i` (`0 <= i < N`)，我们有两种选择：
    *   以 `p1` 的概率走到 `i+1`。
    *   以 `p2` 的概率走到 `i+2`。
    从 `i` 走到下一个格子，这本身就算一步。所以：
    `dp[i] = 1 + p1 * dp[i+1] + p2 * dp[i+2]`
    **边界条件：**
    *   当 `i = N-1` 时，只能走一步到 `N`：`dp[N-1] = 1 + p1 * dp[N] + p2 * dp[N]` (如果 `N-1+2 > N`，那么 `dp[N-1+2]` 应该视为 `dp[N]`)。更严谨地，如果 `i+2 > N`，那么从 `i` 走 `2` 步实际上是直接到终点 `N`。
        `dp[N-1] = 1 + p1 * dp[N] + p2 * dp[N]` (因为 `dp[N]=0`)
        `dp[N-1] = 1`
    *   当 `i = N-2` 时：
        `dp[N-2] = 1 + p1 * dp[N-1] + p2 * dp[N]`
        `dp[N-2] = 1 + p1 * (1) + p2 * (0)`
        `dp[N-2] = 1 + p1`
    我们可以将 `dp[i+2]` 在 `i+2 > N` 时看作 `dp[N]`，这样可以统一公式。
4.  **计算顺序：** 从 `N-1` 倒推到 `0`。
5.  **最终答案：** `dp[0]`。

**C++ 代码实现：**

```cpp
#include <iostream>
#include <vector>
#include <iomanip> // 用于设置输出精度

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    double p1, p2;

    std::cin >> n >> p1 >> p2; // n: 终点格子编号, p1: 走1步概率, p2: 走2步概率

    // dp[i] 表示从格子 i 到达格子 n 的期望步数
    // 注意：GESP 9级允许使用vector，但对于DP状态，原生数组更符合基础教学要求，且避免一些潜在问题
    // 这里使用原生数组
    std::vector<double> dp(n + 1); // dp[0...n]

    // 目标状态：到达格子 n 的期望步数是 0
    dp[n] = 0.0;

    // 从 n-1 倒推到 0
    for (int i = n - 1; i >= 0; --i) {
        // 这一步的消耗是 1
        dp[i] = 1.0;
        
        // 加上走 1 步的期望：以 p1 概率走到 i+1
        dp[i] += p1 * dp[i + 1];

        // 加上走 2 步的期望：以 p2 概率走到 i+2
        // 如果 i+2 超过 n，则认为直接到达 n，其期望为 dp[n] (即 0)
        if (i + 2 <= n) {
            dp[i] += p2 * dp[i + 2];
        } else {
            // 如果 i+2 > n，那么走两步也直接到达终点 n，期望就是 dp[n] (0)
            // 所以这里可以不加，因为 p2 * dp[n] = p2 * 0 = 0
            dp[i] += p2 * dp[n]; // 明确写出，便于理解
        }
    }

    std::cout << std::fixed << std::setprecision(6) << dp[0] << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 循环从 `N-1` 到 `0`，共 `N` 次迭代，每次迭代常数时间操作。所以时间复杂度为 `O(N)`。
*   **空间复杂度：** 需要一个 `dp` 数组存储 `N+1` 个状态。所以空间复杂度为 `O(N)`。

### 例题2：DAG 上的期望步数

**题目描述：**
给定一个有向无环图（DAG），包含 `N` 个节点（编号 `1` 到 `N`）和 `M` 条有向边。节点 `1` 是起点，节点 `N` 是终点。
从一个节点 `u` 出发，每次等概率地选择一条从 `u` 出发的边走向一个相邻节点 `v`。
求从节点 `1` 走到节点 `N` 的期望步数。

**思路分析：**
1.  **状态定义：** 设 `dp[u]` 表示从节点 `u` 出发，到达节点 `N` 的期望步数。
2.  **目标状态：** `dp[N] = 0`。
3.  **状态转移方程：** 对于任意节点 `u` (`u != N`)：
    `dp[u] = 1 + Σ (P(u->v) * dp[v])`
    其中 `P(u->v)` 是从 `u` 走到 `v` 的概率。由于是等概率选择一条出边，如果节点 `u` 有 `out_degree[u]` 条出边，那么 `P(u->v) = 1 / out_degree[u]`。
    所以：`dp[u] = 1 + Σ_{v \text{ is successor of } u} (1.0 / out\_degree[u] * dp[v])`
    如果 `u` 没有出边（且 `u != N`），那么 `dp[u]` 应该视为无穷大（无法到达 `N`），但在竞赛中通常会保证存在路径。
4.  **计算顺序：** 由于是 DAG，且我们需要 `dp[v]` 来计算 `dp[u]`，而 `v` 是 `u` 的后继。这意味着 `v` 应该在 `u` 之前被计算。这正好是**反向拓扑排序**的顺序！我们从那些没有出边（除了终点N）的节点开始计算，逐步推回到起点。
    另一种方法是使用**记忆化搜索**：从 `dp[1]` 开始递归计算，遇到未计算过的状态就递归，已计算过的直接返回。

**C++ 代码实现（记忆化搜索）：**

```cpp
#include <iostream>
#include <vector>
#include <iomanip> // 用于设置输出精度
#include <numeric> // 用于 std::iota

const int MAXN = 100005; // 最大节点数
std::vector<int> adj[MAXN]; // 邻接表，存储正向边
double dp[MAXN];            // dp[u] 表示从 u 到达 N 的期望步数
bool visited[MAXN];         // 标记 dp[u] 是否已经计算过

int n_nodes, m_edges; // N: 节点数, M: 边数
int target_node;      // 目标节点 N

// 记忆化搜索函数
double solve(int u) {
    if (u == target_node) {
        return 0.0; // 到达目标节点，期望步数为 0
    }
    if (visited[u]) {
        return dp[u]; // 已经计算过，直接返回
    }

    visited[u] = true; // 标记为已访问，防止重复计算

    int out_degree = adj[u].size();
    if (out_degree == 0) {
        // 如果没有出边且不是目标节点，则无法到达目标，期望步数视为无穷大
        // 在实际问题中，通常会保证有路径可达，或者规定特殊处理
        // 这里为了简化，假设总能到达，或者是一个很大的数
        // 也可以返回一个特殊值表示不可达，例如 -1
        // 但根据题意，通常是可达的
        return dp[u] = 1e18; // 一个足够大的数表示不可达
    }

    double sum_next_dp = 0.0;
    for (int v : adj[u]) {
        sum_next_dp += solve(v); // 递归计算所有后继节点的期望
    }

    // dp[u] = 1 (当前这一步) + (所有后继节点的期望和 / 出度)
    dp[u] = 1.0 + sum_next_dp / out_degree;
    
    return dp[u];
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n_nodes >> m_edges;
    target_node = n_nodes; // 目标节点就是 N

    for (int i = 0; i < m_edges; ++i) {
        int u, v;
        std::cin >> u >> v;
        adj[u].push_back(v); // 添加有向边
    }

    // 初始化 visited 数组为 false
    // std::memset(visited, 0, sizeof(visited)); // C风格，对于bool数组可能不是最优
    // for (int i = 1; i <= n_nodes; ++i) {
    //     visited[i] = false;
    // } // 或者使用循环
    
    // 计算从节点 1 到节点 N 的期望步数
    double result = solve(1);

    // 如果结果是一个很大的数，说明可能无法到达
    if (result >= 1e17) {
        std::cout << "Target node " << target_node << " is unreachable from node 1." << std::endl;
    } else {
        std::cout << std::fixed << std::setprecision(6) << result << std::endl;
    }

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 每个节点和每条边最多被访问一次（通过记忆化搜索）。所以时间复杂度为 `O(N + M)`。
*   **空间复杂度：** 存储邻接表 `O(N + M)`，`dp` 数组 `O(N)`，`visited` 数组 `O(N)`。所以空间复杂度为 `O(N + M)`。

### 例题3：树上期望DP（两次DFS）

**题目描述：**
给定一棵包含 `N` 个节点的树，节点编号从 `1` 到 `N`。节点 `1` 是根节点。
你从节点 `u` 出发，每次等概率地选择一个相邻节点走过去。直到走到根节点 `1` 为止。
求从每个节点 `u` (`1 <= u <= N`) 到达根节点 `1` 的期望步数。

**思路分析：**
1.  **状态定义：** 设 `dp[u]` 表示从节点 `u` 出发，到达根节点 `1` 的期望步数。
2.  **目标状态：** `dp[1] = 0`。
3.  **状态转移方程：** 对于任意节点 `u` (`u != 1`)：
    `dp[u] = 1 + Σ_{v \text{ is neighbor of } u} (1.0 / degree[u] * dp[v])`
    这里 `degree[u]` 是节点 `u` 的度（邻居数量）。
    问题来了：`dp[u]` 的计算依赖于 `dp[parent(u)]` 和 `dp[child(v)]`。这意味着存在循环依赖，不能简单地一次DFS解决。
    例如：`dp[u] = 1 + (1/degree[u]) * dp[parent(u)] + sum_{v \text{ is child of } u} (1/degree[u] * dp[v])`
    为了解决这种循环依赖，我们通常使用**两次DFS**的技巧。

    **第一次DFS (自底向上，`dfs1(u, p)`):**
    我们尝试将 `dp[u]` 表示成 `dp[parent(u)]` 的线性关系：
    `dp[u] = A_u * dp[parent(u)] + B_u`
    *   对于叶子节点 `u` (除了根节点 `1` 之外的叶子)：
        `dp[u] = 1 + (1/degree[u]) * dp[parent(u)]`
        所以 `A_u = 1/degree[u]`，`B_u = 1`。
    *   对于非叶子节点 `u` (`u != 1`)：
        `dp[u] = 1 + (1/degree[u]) * dp[parent(u)] + sum_{v \text{ is child of } u} (1/degree[u] * dp[v])`
        将 `dp[v]` 替换为 `A_v * dp[u] + B_v` (注意这里 `parent(v)` 是 `u`，所以是 `A_v * dp[u]`)。
        `dp[u] = 1 + (1/degree[u]) * dp[parent(u)] + sum_{v \text{ is child of } u} (1/degree[u] * (A_v * dp[u] + B_v))`
        移项整理，将所有 `dp[u]` 项放到左边：
        `dp[u] * (1 - sum_{v \text{ is child of } u} (A_v / degree[u])) = 1 + (1/degree[u]) * dp[parent(u)] + sum_{v \text{ is child of } u} (B_v / degree[u])`
        设 `C_u = 1 - sum_{v \text{ is child of } u} (A_v / degree[u])`
        `D_u = 1 + sum_{v \text{ is child of } u} (B_v / degree[u])`
        则 `dp[u] * C_u = (1/degree[u]) * dp[parent(u)] + D_u`
        所以 `A_u = (1/degree[u]) / C_u`
        `B_u = D_u / C_u`
    通过自底向上（从叶子到根）的DFS，我们可以计算出每个节点的 `A_u` 和 `B_u`。

    **第二次DFS (自顶向下，`dfs2(u, p)`):**
    现在我们有了 `dp[1] = 0`，并且知道每个 `dp[u] = A_u * dp[parent(u)] + B_u`。
    从根节点 `1` 开始，自顶向下DFS。当计算到 `u` 时，其父节点 `parent(u)` 的 `dp` 值已经计算出来了。
    我们就可以直接计算 `dp[u] = A_u * dp[parent(u)] + B_u`。

**C++ 代码实现：**

#include <iostream>
#include <vector>
#include <iomanip> // 用于设置输出精度

const int MAXN = 100005; // 最大节点数
std::vector<int> adj[MAXN]; // 邻接表
int degree[MAXN];           // 存储每个节点的度
double dp[MAXN];            // dp[u] 表示从 u 到达根节点 1 的期望步数
double A[MAXN], B[MAXN];    // dp[u] = A[u] * dp[parent(u)] + B[u]

int n_nodes; // 节点数

// 第一次DFS：自底向上计算 A[u] 和 B[u]
void dfs1(int u, int p) {
    // 初始值：假设 u 是叶子节点（除了根节点1），那么它只有父节点 p 这一个邻居
    // dp[u] = 1 + (1/degree[u]) * dp[p]
    // 此时 A[u] = 1.0 / degree[u], B[u] = 1.0
    
    // 如果 u 是叶子节点 (除了根节点1)，其除了p以外没有其他邻居
    // 那么它的 degree[u] 实际上就是 1 (指向父节点)
    // 但在树中，degree[u]是所有邻居，所以需要考虑所有子节点。
    // 这里我们先计算子节点带来的影响
    
    double sum_A_child = 0.0;
    double sum_B_child = 0.0;
    
    for (int v : adj[u]) {
        if (v == p) continue; // 避免回到父节点
        dfs1(v, u); // 递归计算子节点 v 的 A[v] 和 B[v]
        sum_A_child += A[v];
        sum_B_child += B[v];
    }

    // 现在根据公式推导 A[u] 和 B[u]
    // dp[u] = 1 + (1/deg[u]) * dp[p] + sum_{v child of u} (1/deg[u] * dp[v])
    // dp[u] = 1 + (1/deg[u]) * dp[p] + sum_{v child of u} (1/deg[u] * (A[v]*dp[u] + B[v]))
    // dp[u] * (1 - sum_{v child of u} (A[v]/deg[u])) = 1 + (1/deg[u]) * dp[p] + sum_{v child of u} (B[v]/deg[u])
    
    // 如果 u 是根节点 1，或者 u 只有一个邻居 p (即 u 是叶子)
    if (u == 1) { // 根节点，dp[1]=0，不需要计算 A, B
        A[u] = 0.0; // 这里的 A[u] 和 B[u] 对根节点没有意义，因为dp[1]是已知的
        B[u] = 0.0;
        return;
    }
    
    // 对于非根节点，其度至少为1 (连接到父节点)
    // C_u = 1 - sum_{v child of u} (A[v] / degree[u])
    // D_u = 1 + sum_{v child of u} (B[v] / degree[u])
    
    // 确保分母不为0，degree[u] 至少为1
    double denom = 1.0 - sum_A_child / degree[u];
    
    if (denom == 0) { // 避免除以零，理论上这不应该发生对于期望值
        // 这表示存在循环依赖，或者某些路径期望无穷大
        // 对于树形结构，且根可达，通常不会出现
        A[u] = 1e18; // 设为极大值
        B[u] = 1e18;
        return;
    }

    A[u] = (1.0 / degree[u]) / denom;
    B[u] = (1.0 + sum_B_child / degree[u]) / denom;
}

// 第二次DFS：自顶向下计算 dp[u]
void dfs2(int u, int p) {
    if (u == 1) { // 根节点
        dp[u] = 0.0;
    } else {
        // dp[u] = A[u] * dp[parent(u)] + B[u]
        dp[u] = A[u] * dp[p] + B[u];
    }

    for (int v : adj[u]) {
        if (v == p) continue;
        dfs2(v, u); // 递归计算子节点
    }
}
