## 深度知识点讲解

### 一、暴力解法回顾：为什么我们需要Manacher？

在学习Manacher之前，我们先来看看“笨”办法是怎么做的。
如果我们要找最长回文子串，最直观的方法就是：
1.  枚举所有可能的子串（起点和终点）。
2.  对于每个子串，判断它是不是回文。
3.  记录最长的那个。

这个方法太慢了！一个长度为 `N` 的字符串，大约有 `N^2` 个子串。判断一个子串是否回文需要 `O(N)` 的时间。所以总时间复杂度是 `O(N^3)`。这简直是“龟速”！

稍微优化一下，我们可以从每个字符或者每两个字符的“缝隙”开始，向两边扩展，直到不是回文为止。
*   对于奇数长度回文：以 `s[i]` 为中心，向左 `s[i-1]`, `s[i-2]`... 和向右 `s[i+1]`, `s[i+2]`... 扩展。
*   对于偶数长度回文：以 `s[i]` 和 `s[i+1]` 之间的缝隙为中心，向左 `s[i-1]`, `s[i-2]`... 和向右 `s[i+2]`, `s[i+3]`... 扩展。

这样，枚举中心点是 `O(N)`，每次扩展最坏是 `O(N)`，总时间复杂度是 `O(N^2)`。
`N^2` 对于 `N=10^5` 来说就是 `10^{10}`，还是太慢了！我们的目标是 `O(N)`，也就是线性时间！

### 二、Manacher算法登场！—— 化繁为简，利用对称

Manacher算法之所以能达到 O(N) 的效率，主要靠两个“魔法”：
1.  **预处理：统一奇偶，让所有回文串都变成“奇数”长度！**
2.  **利用对称性：聪明地复用以前的计算结果，避免重复劳动！**

#### 魔法一：预处理——为回文串穿上“统一制服”

我们刚才提到，回文串有奇数长度（如 "aba"）和偶数长度（如 "abba"）两种。这两种情况在处理时有点麻烦，Manacher算法想了一个绝妙的办法来统一它们：在原字符串的每个字符之间、开头和结尾都插入一个特殊的、原字符串中不会出现的字符（比如 `#`）。

**例子：**
*   原字符串 `s = "aba"`
    *   预处理后 `t = "#a#b#a#"`
    *   这时，以 `t[3]` ('b') 为中心，半径为 3 的回文串是 `#a#b#a#`，长度 7。
    *   在原字符串中，`"aba"` 是一个长度为 3 的回文串。
*   原字符串 `s = "abba"`
    *   预处理后 `t = "#a#b#b#a#"`
    *   这时，以 `t[4]` ('b') 为中心，半径为 4 的回文串是 `#a#b#b#a#`，长度 9。
    *   在原字符串中，`"abba"` 是一个长度为 4 的回文串。

**思考一下：**
1.  为什么要在开头和结尾也加 `#`？
    *   为了防止越界，并让所有中心点都有一个对称的“外壳”，方便统一处理。比如 `s = "a"` 变成 `"#a#"`。
2.  预处理后，所有回文串的长度都是奇数吗？
    *   是的！因为每个字符和每个缝隙都被 `#` 包裹了。比如 `"#a#b#a#"`，中心是 `b`，两边都是 `#`。`"#a#b#b#a#"`，中心是 `b#b`，也是一个整体的“奇数”结构。
3.  预处理后的字符串 `t` 的长度是多少？
    *   如果原字符串 `s` 的长度是 `N`，那么 `t` 的长度是 `2*N + 1`。

**预处理函数实现：**
```cpp
// 假设 MAXN 是原字符串的最大长度
// 那么处理后的字符串最大长度是 2 * MAXN + 5
char t[2 * 1000000 + 5]; // 存储预处理后的字符串
int p[2 * 1000000 + 5];  // 存储半径数组

// 预处理函数
int preprocess(char s[], int n) {
    t[0] = '$'; // 哨兵字符，防止越界，且不与#冲突
    t[1] = '#';
    int k = 2;
    for (int i = 0; i < n; ++i) {
        t[k++] = s[i];
        t[k++] = '#';
    }
    t[k] = '\0'; // 字符串结束符
    return k;    // 返回处理后的字符串长度
}
```
**注意**：为了防止 `t[i-p[i]]` 越界，我们通常在 `t[0]` 放置一个特殊的字符，比如 `$`，它一定不会和 `#` 或字母相同。这样，`t[1]` 永远是 `#`，`t[0]` 是 `$`，`p[1]` 无论如何都不会导致 `t[0]` 的对称点越界。

#### 魔法二：记录半径——`p` 数组的秘密

现在我们有了一个统一格式的字符串 `t`。接下来，我们需要一个数组 `p` 来记录每个位置 `i` 的“回文半径”。
`p[i]` 的定义：以 `t[i]` 为中心的最长回文子串，其**半长**（包括中心 `t[i]` 本身）。
换句话说，如果 `p[i] = k`，那么以 `t[i]` 为中心，向左 `k-1` 个字符，向右 `k-1` 个字符，组成的子串 `t[i-(k-1) ... i+(k-1)]` 就是一个回文串，且是 `t[i]` 能形成的最长回文串。

**例子：**
如果 `t = "$#a#b#a#"` (长度 `m = 9`)
*   `t[0] = '$'`, `p[0] = 1` (最小半径)
*   `t[1] = '#'`, `p[1] = 1`
*   `t[2] = 'a'`, `p[2] = 2` (回文串 `#a#`，半长是 2)
*   `t[3] = '#'`, `p[3] = 1`
*   `t[4] = 'b'`, `p[4] = 4` (回文串 `#a#b#a#`，半长是 4)
*   `t[5] = '#'`, `p[5] = 1`
*   `t[6] = 'a'`, `p[6] = 2` (回文串 `#a#`，半长是 2)
*   `t[7] = '#'`, `p[7] = 1`
*   `t[8] = '\0'`, `p[8] = 1`

**`p[i]` 与原字符串回文长度的关系：**
如果 `p[i]` 是以 `t[i]` 为中心的回文串的半长，那么这个回文串在原字符串中的长度就是 `p[i] - 1`。
例如，`t[4] = 'b'`，`p[4] = 4`。对应的回文串是 `"#a#b#a#"`。去掉 `#`，就是原字符串中的 `"aba"`，长度是 `4 - 1 = 3`。
如果 `t[2] = 'a'`，`p[2] = 2`。对应的回文串是 `"#a#"`。去掉 `#`，就是原字符串中的 `"a"`，长度是 `2 - 1 = 1`。
这个关系非常重要！

#### 魔法三：加速秘籍——利用对称性！

这是Manacher算法最核心、最巧妙的部分。当我们在计算 `p[i]` 时，我们希望能够利用之前已经计算过的 `p` 值来加速。

我们维护两个变量：
*   `c` (center)：当前已经找到的、**右边界最远**的回文子串的中心。
*   `r` (right boundary)：以 `c` 为中心的回文子串的右边界，即 `r = c + p[c]`。

当我们遍历 `t` 数组，计算 `p[i]` 时，会遇到两种情况：

**情况1：当前 `i` 在 `r` 的外面 (`i >= r`)**
这意味着 `i` 超出了当前最右回文串的范围。我们没有可以利用的对称信息。
这时，我们只能老老实实地从 `p[i] = 1` 开始，向两边扩展，直到 `t[i - p[i]] != t[i + p[i]]` 或者越界。
```cpp
p[i] = 1; // 最小半径为1，即只包含中心字符本身
while (t[i - p[i]] == t[i + p[i]]) {
    p[i]++;
}
```

**情况2：当前 `i` 在 `r` 的里面 (`i < r`)**
这是Manacher算法发挥威力的地方！
我们知道 `c` 是一个回文串的中心，`i` 在这个回文串的右半部分。那么 `i` 在 `c` 左边有一个**对称点 `j`**。
`j` 的计算公式是：`j = 2 * c - i` (或者 `j = c - (i - c)`)。
因为 `c` 是中心，`t[c-k ... c+k]` 是回文。
`i` 和 `j` 关于 `c` 对称，所以以 `t[j]` 为中心的回文串，应该和以 `t[i]` 为中心的回文串有很强的关系！

我们知道 `p[j]` 的值，它告诉我们以 `t[j]` 为中心的最长回文串有多长。
根据对称性，以 `t[i]` 为中心的回文串，至少会和以 `t[j]` 为中心的回文串一样长，但是它不能超出以 `c` 为中心的回文串的左边界 `c - p[c]` (也就是 `c - (r - c)`)。

所以，`p[i]` 的初始值可以取 `min(p[j], r - i)`。
*   `p[j]`：来自对称点 `j` 的信息。
*   `r - i`：当前 `i` 到 `c` 为中心的回文串右边界 `r` 的距离。这表示 `t[i]` 能够向右扩展的最大距离。

具体来说，这里又分两种子情况：
*   **子情况 2a: `p[j]` 的回文串完全在以 `c` 为中心的回文串内部。**
    *   这意味着 `j - p[j]` ( `j` 回文串的左边界) 大于 `c - p[c]` ( `c` 回文串的左边界)。
    *   在这种情况下，由于对称性，`p[i]` 就等于 `p[j]`。
    *   `p[i] = p[j]`。
*   **子情况 2b: `p[j]` 的回文串超出了以 `c` 为中心的回文串的左边界。**
    *   这意味着 `j - p[j]` 小于或等于 `c - p[c]`。
    *   在这种情况下，以 `t[i]` 为中心的回文串，只能保证扩展到 `c` 的回文串的左边界。它的半径最多是 `r - i`。
    *   `p[i] = r - i`。

将这两种子情况合并，我们可以得到一个统一的初始值：
`p[i] = min(p[2*c - i], r - i);`

**无论哪种情况，这都只是一个“保底”的预测值。** 实际的 `p[i]` 可能比这个预测值更大。所以，在得到这个初始值后，我们仍然需要向两边扩展，去尝试找到更大的回文串：
```cpp
while (t[i - p[i]] == t[i + p[i]]) {
    p[i]++;
}
```

**更新 `c` 和 `r`：**
在每次计算完 `p[i]` 后，如果发现以 `t[i]` 为中心的回文串的右边界 `i + p[i]` 超过了当前的 `r`，那么我们就需要更新 `c` 和 `r`：
```cpp
if (i + p[i] > r) {
    c = i;
    r = i + p[i];
}
```

**总结Manacher算法的核心循环：**
```cpp
// 假设 t 是预处理后的字符串，m 是其长度
int c = 0, r = 0; // c: 最右回文串中心, r: 最右回文串右边界 (r = c + p[c])
int maxLen = 0;   // 记录最长回文子串的长度
int maxC = 0;     // 记录最长回文子串的中心

for (int i = 1; i < m; ++i) { // 从1开始，因为t[0]是哨兵'$'
    // 计算p[i]的初始值
    if (i < r) {
        // i在当前最右回文串内部，利用对称性
        p[i] = std::min(p[2 * c - i], r - i);
    } else {
        // i在当前最右回文串外部，无对称信息，从1开始扩展
        p[i] = 1;
    }

    // 尝试向两边扩展，找到真正的p[i]
    while (t[i - p[i]] == t[i + p[i]]) {
        p[i]++;
    }

    // 更新c和r，如果以i为中心的回文串的右边界更靠右
    if (i + p[i] > r) {
        c = i;
        r = i + p[i];
    }

    // 更新最长回文子串的长度和中心
    if (p[i] - 1 > maxLen) { // p[i]-1 才是原字符串中的实际长度
        maxLen = p[i] - 1;
        maxC = i;
    }
}
// maxLen 就是最长回文子串的长度
// maxC 就是最长回文子串在预处理字符串t中的中心
```

### 三、时间复杂度分析：为什么是 O(N)？

Manacher算法的时间复杂度是 O(N)，其中 N 是原字符串的长度。为什么这么快呢？
关键在于 `r` (最右回文串的右边界)。
*   在 `for` 循环中，`i` 从 `1` 遍历到 `m-1` ( `m` 是预处理字符串的长度，约 `2N`)，这部分是 O(N)。
*   `while` 循环 `t[i - p[i]] == t[i + p[i]]` 看起来会重复扩展，但仔细观察：每次 `p[i]` 增加 1，意味着 `i + p[i]` 这个右边界在向右移动。
*   而 `r` 变量，它总是记录当前所有回文串中能延伸到最右边的那个回文串的右边界。`r` 在整个算法过程中是**单调不减**的。它最大能增加到 `m`。
*   当 `i < r` 时，`p[i]` 会有一个初始值 `min(p[j], r - i)`。这意味着 `while` 循环的扩展是从一个相对较大的值开始的，而不是总是从 1 开始。
*   **核心观察**：每个字符 `t[k]` 作为 `i + p[i]` 或 `i - p[i]` 被访问的次数是常数级别的。因为 `r` 只向右移动，每次 `while` 循环的扩展都使得 `r` 更大（或者 `r` 不变，但 `i` 已经超过了旧的 `r`），所以总的扩展次数是 `O(N)`。

因此，Manacher算法的总时间复杂度是 O(N)。

## 典型例题精讲

### 例题1：最长回文子串的长度

**题目描述：**
给定一个字符串 `s`，找到 `s` 中最长的回文子串的长度。

**输入：**
一个字符串，长度不超过 1000000。

**输出：**
最长回文子串的长度。

**示例：**
输入: `babad`
输出: `3` (因为 "bab" 或 "aba" 都是最长回文子串，长度为3)

**思路分析：**
这正是Manacher算法的经典应用。
1.  首先，对输入的字符串 `s` 进行预处理，生成字符串 `t`。
2.  然后，使用Manacher算法的核心逻辑，遍历 `t` 字符串，计算每个中心 `i` 的回文半径 `p[i]`。
3.  在计算 `p[i]` 的过程中，同时维护一个 `maxLen` 变量，记录 `p[i]-1` 的最大值。
4.  最终 `maxLen` 就是答案。

**代码实现：**

```cpp
#include <iostream> // 用于输入输出
#include <cstring>  // 用于strlen
#include <algorithm> // 用于std::min

// 定义最大字符串长度，根据题目要求可能需要调整
// 原字符串最大100万，预处理后200万+5
const int MAX_LEN = 2000000 + 5;

char s[MAX_LEN / 2]; // 存储原始字符串，注意MAX_LEN是预处理后的
char t[MAX_LEN];     // 存储预处理后的字符串
int p[MAX_LEN];      // 存储Manacher半径数组

int main() {
    // 禁用同步流，加速I/O
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> s; // 读取原始字符串
    int n = strlen(s); // 获取原始字符串长度

    // 预处理字符串t
    t[0] = '$'; // 哨兵字符
    t[1] = '#';
    int k = 2;
    for (int i = 0; i < n; ++i) {
        t[k++] = s[i];
        t[k++] = '#';
    }
    t[k] = '\0'; // 结束符
    int m = k;   // 预处理后的字符串长度

    int c = 0, r = 0; // c: 最右回文中心, r: 最右回文右边界
    int maxLen = 0;   // 记录最长回文子串的长度

    for (int i = 1; i < m; ++i) { // 从1开始遍历
        // 计算p[i]的初始值
        if (i < r) {
            // i在当前最右回文串内部，利用对称性
            p[i] = std::min(p[2 * c - i], r - i);
        } else {
            // i在当前最右回文串外部，无对称信息，从1开始扩展
            p[i] = 1;
        }

        // 尝试向两边扩展，找到真正的p[i]
        // 注意：t[i - p[i]] 和 t[i + p[i]] 必须在有效范围内
        // t[0]是'$'，可以作为左边界的哨兵，保证t[i - p[i]]不会越界到负数
        // t[m]是'\0'，可以作为右边界的哨兵，保证t[i + p[i]]不会越界
        while (t[i - p[i]] == t[i + p[i]]) {
            p[i]++;
        }

        // 更新c和r，如果以i为中心的回文串的右边界更靠右
        if (i + p[i] > r) {
            c = i;
            r = i + p[i];
        }

        // 更新最长回文子串的长度
        // p[i]表示的是半长，实际长度为 p[i] - 1
        if (p[i] - 1 > maxLen) {
            maxLen = p[i] - 1;
        }
    }

    std::cout << maxLen << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** O(N)，其中 N 是原始字符串的长度。预处理字符串 `t` 的长度为 `2N+1`。Manacher算法的核心循环遍历 `t` 字符串一次，每个字符的访问和 `while` 循环的扩展总和为线性时间。
*   **空间复杂度：** O(N)，用于存储预处理后的字符串 `t` 和半径数组 `p`。

### 例题2：找出最长回文子串本身

**题目描述：**
给定一个字符串 `s`，找出并输出 `s` 中最长的回文子串。如果存在多个，输出任意一个即可。

**输入：**
一个字符串，长度不超过 1000000。

**输出：**
最长的回文子串。

**示例：**
输入: `babad`
输出: `bab` (或 `aba`)

**思路分析：**
在例题1的基础上，我们不仅要记录 `maxLen`，还需要记录这个 `maxLen` 对应的中心点 `maxC`。
1.  同样进行预处理，生成 `t`。
2.  执行Manacher算法，计算 `p` 数组。
3.  在更新 `maxLen` 时，同时更新 `maxC` 为当前 `i` 的值。
4.  Manacher算法结束后，我们得到 `maxLen` (最长回文子串的长度) 和 `maxC` (在 `t` 字符串中的中心索引)。
5.  根据 `maxC` 和 `maxLen`，我们可以推导出原字符串中最长回文子串的起始位置。
    *   在 `t` 中，以 `maxC` 为中心的回文串的范围是 `[maxC - maxLen, maxC + maxLen]`。
    *   这些字符中，`#` 是分隔符，字母才是原字符串的字符。
    *   `t[maxC - maxLen]` 对应的原字符串索引是 `(maxC - maxLen - 1) / 2`。
    *   `t[maxC + maxLen]` 对应的原字符串索引是 `(maxC + maxLen - 1) / 2`。
    *   所以，原字符串的起始索引 `start = (maxC - maxLen) / 2`。
    *   然后从 `s[start]` 开始，输出 `maxLen` 个字符。

**代码实现：**

```cpp
#include <iostream>
#include <cstring>
#include <algorithm>

const int MAX_LEN = 2000000 + 5;

char s[MAX_LEN / 2]; // 存储原始字符串
char t[MAX_LEN];     // 存储预处理后的字符串
int p[MAX_LEN];      // 存储Manacher半径数组

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> s;
    int n = strlen(s);

    // 预处理字符串t
    t[0] = '$';
    t[1] = '#';
    int k = 2;
    for (int i = 0; i < n; ++i) {
        t[k++] = s[i];
        t[k++] = '#';
    }
    t[k] = '\0';
    int m = k;

    int c = 0, r = 0;
    int maxLen = 0; // 最长回文子串的长度
    int maxC = 0;   // 最长回文子串在t中的中心索引

    for (int i = 1; i < m; ++i) {
        if (i < r) {
            p[i] = std::min(p[2 * c - i], r - i);
        } else {
            p[i] = 1;
        }

        while (t[i - p[i]] == t[i + p[i]]) {
            p[i]++;
        }

        if (i + p[i] > r) {
            c = i;
            r = i + p[i];
        }

        if (p[i] - 1 > maxLen) {
            maxLen = p[i] - 1;
            maxC = i; // 记录对应的中心
        }
    }

    // 根据maxC和maxLen还原原始字符串中的最长回文子串
    // 在t中，中心是maxC，回文半长是maxLen+1 (因为p[maxC] = maxLen+1)
    // t中回文串的起始索引是 maxC - (maxLen + 1 - 1) = maxC - maxLen
    // t中回文串的结束索引是 maxC + maxLen
    // 原字符串的起始索引：(maxC - maxLen) / 2
    int start = (maxC - maxLen) / 2;

    for (int i = 0; i < maxLen; ++i) {
        std::cout << s[start + i];
    }
    std::cout << std::endl;

    return 0;
}
```

**复杂度分析：**
与例题1相同，时间复杂度 O(N)，空间复杂度 O(N)。

## 代码实现模板

```cpp
#include <iostream> // 用于输入输出
#include <cstring>  // 用于strlen
#include <algorithm> // 用于std::min

// 定义最大字符串长度
// GESP 9级可能遇到10^6级别的数据
// 原字符串最大长度为 N，预处理后为 2N+1
const int MAXN_ORIG = 1000000;
const int MAXN_PROCESSED = 2 * MAXN_ORIG + 5; // 预处理后的字符串和p数组大小

// 原始字符串，注意大小
char s_original[MAXN_ORIG + 5];

// 预处理后的字符串
char s_processed[MAXN_PROCESSED];

// Manacher半径数组，p[i]表示以s_processed[i]为中心的最长回文串的半长（包含中心）
int p_radius[MAXN_PROCESSED];

// 预处理函数
// 将原始字符串 s 转换为 Manacher 算法所需的格式 s_processed
// 返回 s_processed 的有效长度
int preprocess(char s[], int n) {
    s_processed[0] = '$'; // 哨兵字符，防止越界
    s_processed[1] = '#';
    int k = 2; // s_processed 的当前写入位置
    for (int i = 0; i < n; ++i) {
        s_processed[k++] = s[i];
        s_processed[k++] = '#';
    }
    s_processed[k] = '\0'; // 字符串结束符
    return k;              // 返回预处理后的字符串长度
}

// Manacher 主算法函数
// s_processed: 预处理后的字符串
// m: s_processed 的长度
// 返回最长回文子串的长度
// 同时可以记录最长回文子串的中心索引（在s_processed中）和起始索引（在s_original中）
int manacher(int m, int& out_max_c, int& out_max_len) {
    int c = 0; // 当前已知最右回文子串的中心
    int r = 0; // 当前已知最右回文子串的右边界 (r = c + p_radius[c])

    out_max_len = 0; // 记录找到的最长回文子串的长度 (在s_original中)
    out_max_c = 0;   // 记录最长回文子串在s_processed中的中心索引

    for (int i = 1; i < m; ++i) { // 从索引1开始遍历s_processed
        // 1. 计算 p_radius[i] 的初始值
        if (i < r) {
            // 如果 i 在当前最右回文子串的范围内，利用对称性
            // j 是 i 关于 c 的对称点：j = 2 * c - i
            p_radius[i] = std::min(p_radius[2 * c - i], r - i);
        } else {
            // 如果 i 在当前最右回文子串的范围之外，无法利用对称性
            // 从最小半径1开始尝试扩展
            p_radius[i] = 1;
        }

        // 2. 尝试向两边扩展，找到 p_radius[i] 的真实值
        // t[i - p_radius[i]] 和 t[i + p_radius[i]] 必须相等才能扩展
        // s_processed[0]='$', s_processed[m]='\0' 作为哨兵，防止越界
        while (s_processed[i - p_radius[i]] == s_processed[i + p_radius[i]]) {
            p_radius[i]++;
        }

        // 3. 更新 c 和 r
        // 如果以 i 为中心的回文子串的右边界超过了当前的 r，则更新 c 和 r
        if (i + p_radius[i] > r) {
            c = i;
            r = i + p_radius[i];
        }

        // 4. 更新最长回文子串的长度和中心
        // p_radius[i] - 1 才是对应原始字符串中的实际回文长度
        if (p_radius[i] - 1 > out_max_len) {
            out_max_len = p_radius[i] - 1;
            out_max_c = i; // 记录最长回文子串在 s_processed 中的中心索引
        }
    }

    return out_max_len; // 返回最终的最长回文子串长度
}

int main() {
    // 优化I/O速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    // 读取原始字符串
    std::cin >> s_original;
    int n_original = strlen(s_original);

    // 预处理字符串
    int n_processed = preprocess(s_original, n_original);

    // 执行 Manacher 算法
    int final_max_c;    // 最长回文子串在s_processed中的中心索引
    int final_max_len;  // 最长回文子串在s_original中的长度
    manacher(n_processed, final_max_c, final_max_len);

    // 输出最长回文子串的长度
    std::cout << "最长回文子串的长度: " << final_max_len << std::endl;

    // 如果需要输出最长回文子串本身
    if (final_max_len > 0) {
        // 根据 final_max_c 和 final_max_len 还原原始字符串中的起始位置
        // 在 s_processed 中，中心是 final_max_c，半长是 final_max_len + 1
        // s_processed 中回文串的起始索引是 final_max_c - final_max_len
        // 原始字符串的起始索引：(final_max_c - final_max_len) / 2
        int start_original = (final_max_c - final_max_len) / 2;
        std::cout << "最长回文子串内容: ";
        for (int i = 0; i < final_max_len; ++i) {
            std::cout << s_original[start_original + i];
        }
        std::cout << std::endl;
    } else {
        std::cout << "没有找到回文子串 (或输入为空)." << std::endl;
    }

    return 0;
}
```

1.  **思考题1：** 为什么Manacher算法在预处理时，不仅在字符之间加 `#`，还在字符串的开头和结尾也加 `#` 呢？比如 "a" 变成 `"#a#"`，而不是只在中间加（如果字符串只有一个字符，就没有“中间”了）？
    *   **提示：** 考虑统一性，以及边界情况的处理。
    *   **答案要点：** 统一奇偶长度回文串的中心；防止越界，让所有中心点都有对称的“外壳”；例如 `s = "a"`，如果只加中间就没法加了，加成 `"#a#"` 后，中心是 `a`，`p[2]=2`，长度 `2-1=1`。

2.  **思考题2：** 在Manacher算法中，如果 `i >= r` (当前遍历到的位置 `i` 超过了 `c` 扩张到的最右边界 `r`)，我们为什么不能利用对称性来初始化 `p[i]`，而只能将 `p[i]` 初始化为 `1`？
    *   **提示：** 对称性依赖于 `i` 在回文范围 `[c-p[c], c+p[c]]` 内。
    *   **答案要点：** 当 `i >= r` 时，`i` 已经超出了以 `c` 为中心的最右回文串的范围。这意味着 `i` 的对称点 `j` 也会超出 `c` 回文串的左边界，或者 `c` 回文串的范围太小，无法提供有效的对称信息。此时，我们无法保证 `s[i]` 的回文性质与 `s[j]` 有任何关联，所以只能从最基础的半径 `1` 开始向外扩展。
