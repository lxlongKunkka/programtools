## 深度知识点讲解

### 1. 线段树基础回顾 (快速通道)

线段树（Segment Tree）是一种二叉树形数据结构，用于高效处理区间查询和区间修改问题。
*   **结构：** 每个节点代表一个区间 `[L, R]`。
    *   叶子节点 `[i, i]` 存储数组中第 `i` 个元素的值。
    *   非叶子节点 `[L, R]` 存储其子节点 `[L, M]` 和 `[M+1, R]` 的信息汇总（如区间和、区间最大值等）。
*   **建树 (build)：** 递归地将区间一分为二，直到叶子节点。
*   **单点修改 (update point)：** 从根节点出发，找到对应的叶子节点修改，然后逐层向上更新父节点信息。
*   **区间查询 (query range)：** 将查询区间 `[ql, qr]` 分解成线段树中若干个节点的区间，然后合并这些节点的信息。

以上操作的复杂度通常是 `O(logN)`。

### 2. 懒惰标记 (Lazy Propagation) 的诞生与原理

#### 2.1 为什么需要懒惰标记？

考虑这样的操作：给区间 `[L, R]` 内的每个元素都加上一个值 `X`。
*   **暴力修改：** 每次修改都遍历 `L` 到 `R` 的所有元素，时间复杂度 `O(N)`。如果 `M` 次操作，总复杂度 `O(M * N)`，太慢。
*   **线段树单点修改：** 每次修改都调用 `N` 次单点修改，时间复杂度 `O(N * logN)`。如果 `M` 次操作，总复杂度 `O(M * N * logN)`，还是太慢。

我们发现，当我们要修改一个大区间时，线段树中会有很多节点被这个修改区间完全覆盖。例如，修改 `[1, 8]`，根节点 `[1, 8]` 就被完全覆盖了。如果我们直接修改这个根节点的值，然后打上一个标记，表示“我这个节点代表的整个区间都被修改了 `X`，但我的子节点们还没来得及更新”，是不是可以省去很多不必要的递归？

这就是懒惰标记的核心思想：**“能拖就拖，非必要不更新”**。

#### 2.2 懒惰标记的核心机制

每个线段树节点除了存储其代表区间的信息（如 `sum` 或 `max`），还需要额外存储一个 `lazy` 标记。这个 `lazy` 标记的含义是：**“我这个节点所代表的区间，需要进行某个操作 `lazy`，但这个操作还没有下传给我的子节点们。”**

懒惰标记主要涉及两个关键操作：

1.  **下传标记 (pushDown)：**
    *   **时机：** 当我们访问一个节点 `u` 的子节点 `lc` 或 `rc` 之前，如果节点 `u` 带有 `lazy` 标记，就必须先把这个标记“下传”给它的子节点。
    *   **过程：**
        *   将节点 `u` 的 `lazy` 值应用到其子节点 `lc` 和 `rc` 的 `value` 上。
        *   将节点 `u` 的 `lazy` 值累加到子节点 `lc` 和 `rc` 的 `lazy` 上。
        *   清除节点 `u` 自身的 `lazy` 标记（因为它已经下传了）。
    *   **生活类比：** 想象一个公司经理（父节点）接到一个任务：“给所有员工加薪 500 元”。他不会立刻通知每一个员工，而是在自己的任务清单上写下“加薪 500”。当他的下属（子节点）来向他汇报工作，或者他需要了解下属部门的情况时，他才会把这个“加薪 500”的任务告诉下属部门经理，并清除自己任务清单上的这项。

2.  **上传信息 (pushUp)：**
    *   **时机：** 当一个节点的子节点信息发生变化后，需要更新父节点的信息。
    *   **过程：** 父节点根据其子节点（`lc` 和 `rc`）的最新信息，重新计算自己的 `value`。例如，如果 `value` 是区间和，`parent.sum = left_child.sum + right_child.sum`。
    *   **生活类比：** 部门经理（子节点）完成了任务，会把结果汇报给他的上级（父节点），上级经理会根据下属的汇报来更新自己的部门总览。

#### 2.3 懒惰标记的实现细节 (以区间加法为例)

假设我们希望实现区间加法和区间求和。
*   每个节点 `u` 存储 `sum` (区间和) 和 `lazy` (需要加上的值)。
*   `lazy` 初始为 0。

**`pushDown(u, L, R)` 函数：**
*   如果 `u` 的 `lazy` 为 0，说明没有待下传的标记，直接返回。
*   计算子节点区间长度：`len_left = M - L + 1`, `len_right = R - M`。
*   左子节点 `lc`：
    *   `tree[lc].sum += tree[u].lazy * len_left` (子节点的和增加 `lazy` 乘以区间长度)。
    *   `tree[lc].lazy += tree[u].lazy` (子节点的懒惰标记累加父节点的懒惰标记)。
*   右子节点 `rc`：
    *   `tree[rc].sum += tree[u].lazy * len_right`。
    *   `tree[rc].lazy += tree[u].lazy`。
*   **清除：** `tree[u].lazy = 0`。

**`update(u, L, R, ql, qr, x)` 函数 (区间 `[ql, qr]` 加上 `x`)：**
1.  **完全覆盖：** 如果当前节点 `u` 的区间 `[L, R]` 完全在修改区间 `[ql, qr]` 内部：
    *   `tree[u].sum += x * (R - L + 1)`。
    *   `tree[u].lazy += x`。
    *   返回。
2.  **非完全覆盖：**
    *   **下传标记：** `pushDown(u, L, R)`。
    *   **递归左右子树：**
        *   如果左子区间 `[L, M]` 与 `[ql, qr]` 有交集，递归 `update(lc, L, M, ql, qr, x)`。
        *   如果右子区间 `[M+1, R]` 与 `[ql, qr]` 有交集，递归 `update(rc, M+1, R, ql, qr, x)`。
    *   **上传信息：** `pushUp(u)` (更新 `tree[u].sum = tree[lc].sum + tree[rc].sum`)。

**`query(u, L, R, ql, qr)` 函数 (查询区间 `[ql, qr]` 的和)：**
1.  **完全覆盖：** 如果当前节点 `u` 的区间 `[L, R]` 完全在查询区间 `[ql, qr]` 内部：
    *   返回 `tree[u].sum`。
2.  **非完全覆盖：**
    *   **下传标记：** `pushDown(u, L, R)`。
    *   **递归左右子树，合并结果：**
        *   `res = 0`。
        *   如果左子区间 `[L, M]` 与 `[ql, qr]` 有交集，`res += query(lc, L, M, ql, qr)`。
        *   如果右子区间 `[M+1, R]` 与 `[ql, qr]` 有交集，`res += query(rc, M+1, R, ql, qr)`。
    *   返回 `res`。

#### 2.4 常见误区与注意事项

*   **忘记 `pushDown`：** 这是最常见的错误！在 `update` 和 `query` 函数中，只要需要访问当前节点的子节点，就必须先调用 `pushDown`。
*   **忘记 `pushUp`：** 在 `update` 函数中，递归修改子节点后，一定要 `pushUp` 来更新父节点的信息。
*   **`lazy` 标记的清除：** `pushDown` 之后，父节点的 `lazy` 标记必须清零，否则会导致重复计算。
*   **`lazy` 标记的类型：** 根据操作类型（加法、乘法、覆盖），`lazy` 标记的含义和下传方式会不同。例如，区间覆盖操作的 `lazy` 标记，会直接覆盖子节点的 `value` 和 `lazy`。
*   **`lazy` 标记的优先级：** 如果同时存在多种 `lazy` 标记（如加法和乘法），下传时需要考虑它们的优先级和顺序。

理解了这些，你就掌握了懒惰标记的精髓！

### 例题 1：区间加法，区间求和

**题目描述：**
给定一个长度为 `N` 的序列 `a`，初始值均为 0。现在有 `M` 次操作，每次操作分两种：
1.  `1 L R X`：表示将区间 `[L, R]` 内的每个数都加上 `X`。
2.  `2 L R`：表示查询区间 `[L, R]` 内所有数的和。

`N, M <= 10^5`

**思路分析：**
这正是懒惰标记的经典应用场景。
1.  **线段树节点存储：** 每个节点 `u` 存储 `sum` (其代表区间的和) 和 `lazy` (其代表区间内每个数需要加上的值)。
2.  **`build`：** 初始时所有数都是 0，所以 `sum` 和 `lazy` 都初始化为 0。
3.  **`pushUp`：** `tree[u].sum = tree[lc].sum + tree[rc].sum`。
4.  **`pushDown` (关键)：**
    *   如果 `tree[u].lazy` 不为 0，则说明有待下传的加法标记。
    *   左子节点 `lc` 的 `sum` 增加 `tree[u].lazy * (M - L + 1)`。
    *   左子节点 `lc` 的 `lazy` 累加 `tree[u].lazy`。
    *   右子节点 `rc` 的 `sum` 增加 `tree[u].lazy * (R - M)`。
    *   右子节点 `rc` 的 `lazy` 累加 `tree[u].lazy`。
    *   最后，`tree[u].lazy` 清零。
5.  **`update` (区间加法)：**
    *   如果当前节点 `u` 的区间 `[L, R]` 完全在操作区间 `[ql, qr]` 内，直接更新 `tree[u].sum` 和 `tree[u].lazy`。
    *   否则，先 `pushDown(u, L, R)`，然后递归左右子树，最后 `pushUp(u)`。
6.  **`query` (区间求和)：**
    *   如果当前节点 `u` 的区间 `[L, R]` 完全在查询区间 `[ql, qr]` 内，直接返回 `tree[u].sum`。
    *   否则，先 `pushDown(u, L, R)`，然后递归左右子树，合并结果。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;

struct Node {
    long long s; // sum
    long long lz; // lazy tag
} t[MAXN * 4]; // 线段树数组，通常开 MAXN * 4 大小

int a[MAXN]; // 原始数组，这里题目说初始为0，可以省略

int n_max_val; // 记录线段树维护的最大区间值，即 N

// 辅助函数：计算当前节点 u 的左右子节点索引
int lc(int u) { return u * 2; }
int rc(int u) { return u * 2 + 1; }

// 向上更新函数：根据子节点信息更新父节点信息
void pushUp(int u) {
    t[u].s = t[lc(u)].s + t[rc(u)].s;
}

// 向下传递懒惰标记函数
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
void pushDown(int u, int L, int R) {
    if (t[u].lz == 0) { // 如果没有懒惰标记，直接返回
        return;
    }

    int m = (L + R) / 2; // 中点

    // 左子节点更新
    // sum 增加 lazy * 区间长度
    t[lc(u)].s += t[u].lz * (m - L + 1);
    // lazy 累加
    t[lc(u)].lz += t[u].lz;

    // 右子节点更新
    t[rc(u)].s += t[u].lz * (R - m);
    t[rc(u)].lz += t[u].lz;

    t[u].lz = 0; // 清除当前节点的懒惰标记
}

// 建树函数
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
void build(int u, int L, int R) {
    if (L == R) { // 叶子节点
        t[u].s = a[L]; // 初始值，这里题目说都是0，所以a[L]可以是0
        t[u].lz = 0;
        return;
    }
    int m = (L + R) / 2;
    build(lc(u), L, m); // 递归建左子树
    build(rc(u), m + 1, R); // 递归建右子树
    pushUp(u); // 向上更新
    t[u].lz = 0; // 初始懒惰标记为0
}

// 区间修改函数：区间 [ql, qr] 加上 val
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 查询/修改区间 [ql, qr]
// val: 增加的值
void updateRange(int u, int L, int R, int ql, int qr, long long val) {
    if (ql <= L && R <= qr) { // 当前区间 [L, R] 完全在修改区间 [ql, qr] 内
        t[u].s += val * (R - L + 1); // 更新当前节点的和
        t[u].lz += val; // 更新当前节点的懒惰标记
        return;
    }

    // 如果不能完全覆盖，则需要下传懒惰标记
    pushDown(u, L, R);

    int m = (L + R) / 2;
    if (ql <= m) { // 左子区间有交集
        updateRange(lc(u), L, m, ql, qr, val);
    }
    if (qr > m) { // 右子区间有交集
        updateRange(rc(u), m + 1, R, ql, qr, val);
    }
    pushUp(u); // 递归修改后，向上更新
}

// 区间查询函数：查询区间 [ql, qr] 的和
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 查询区间 [ql, qr]
long long queryRange(int u, int L, int R, int ql, int qr) {
    if (ql <= L && R <= qr) { // 当前区间 [L, R] 完全在查询区间 [ql, qr] 内
        return t[u].s;
    }

    // 如果不能完全覆盖，则需要下传懒惰标记
    pushDown(u, L, R);

    long long res = 0;
    int m = (L + R) / 2;
    if (ql <= m) { // 左子区间有交集
        res += queryRange(lc(u), L, m, ql, qr);
    }
    if (qr > m) { // 右子区间有交集
        res += queryRange(rc(u), m + 1, R, ql, qr);
    }
    return res;
}

int main() {
    // 假设 N, M 从输入获取
    int n, m;
    scanf("%d %d", &n, &m);
    n_max_val = n; // 设置线段树维护的最大区间值

    // 题目说初始值均为0，这里可以不用读入a数组，直接build即可
    // for (int i = 1; i <= n; ++i) {
    //     scanf("%d", &a[i]);
    // }

    build(1, 1, n); // 从根节点1开始建树，区间 [1, n]

    for (int i = 0; i < m; ++i) {
        int op, l, r;
        scanf("%d %d %d", &op, &l, &r);
        if (op == 1) {
            long long x;
            scanf("%lld", &x);
            updateRange(1, 1, n, l, r, x);
        } else {
            printf("%lld\n", queryRange(1, 1, n, l, r));
        }
    }

    return 0;
}
```

**复杂度分析：**
*   **建树：** `O(N)`
*   **区间修改/查询：** 每次操作都会访问 `O(logN)` 个节点。每个节点的 `pushDown` 和 `pushUp` 都是 `O(1)`。所以每次操作的时间复杂度为 `O(logN)`。
*   **总复杂度：** `O(N + MlogN)`。

### 例题 2：区间修改为某个值，区间求最值

**题目描述：**
给定一个长度为 `N` 的序列 `a`，初始值均为 0。现在有 `M` 次操作，每次操作分两种：
1.  `1 L R X`：表示将区间 `[L, R]` 内的每个数都修改为 `X`。
2.  `2 L R`：表示查询区间 `[L, R]` 内所有数的最大值。

`N, M <= 10^5`

**思路分析：**
这个题目与例题 1 类似，只是操作从“加法”变成了“赋值/覆盖”，查询从“求和”变成了“求最大值”。懒惰标记的含义和下传方式需要相应调整。

1.  **线段树节点存储：** 每个节点 `u` 存储 `mx` (其代表区间的最大值) 和 `lz` (其代表区间内每个数需要被修改为的值)。
2.  **`build`：** 初始时所有数都是 0，所以 `mx` 初始化为 0。`lz` 需要一个特殊值表示“无标记”，例如 `0` (如果 `X` 不会为 0)，或者 `-1` (如果 `X` 可能为 0)。这里假设 `X` 都是非负数，我们用 `-1` 表示无标记。
3.  **`pushUp`：** `t[u].mx = std::max(t[lc(u)].mx, t[rc(u)].mx)`。
4.  **`pushDown` (关键)：**
    *   如果 `t[u].lz == -1` (无标记)，直接返回。
    *   左子节点 `lc` 的 `mx` 直接被 `t[u].lz` 覆盖。
    *   左子节点 `lc` 的 `lz` 也直接被 `t[u].lz` 覆盖。
    *   右子节点 `rc` 的 `mx` 直接被 `t[u].lz` 覆盖。
    *   右子节点 `rc` 的 `lz` 也直接被 `t[u].lz` 覆盖。
    *   最后，`t[u].lz` 清零 (或设为 `-1`)。
5.  **`update` (区间赋值)：**
    *   如果当前节点 `u` 的区间 `[L, R]` 完全在操作区间 `[ql, qr]` 内，直接更新 `t[u].mx = val`，`t[u].lz = val`。
    *   否则，先 `pushDown(u, L, R)`，然后递归左右子树，最后 `pushUp(u)`。
6.  **`query` (区间求最大值)：**
    *   如果当前节点 `u` 的区间 `[L, R]` 完全在查询区间 `[ql, qr]` 内，直接返回 `t[u].mx`。
    *   否则，先 `pushDown(u, L, R)`，然后递归左右子树，合并结果（取最大值）。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

const int MAXN = 100005;

struct Node {
    long long mx; // 区间最大值
    long long lz; // 懒惰标记：表示将区间所有值修改为 lz
                   // -1 表示无懒惰标记（假设 X >= 0）
} t[MAXN * 4];

int n_max_val; // 记录线段树维护的最大区间值，即 N

// 辅助函数：计算当前节点 u 的左右子节点索引
int lc(int u) { return u * 2; }
int rc(int u) { return u * 2 + 1; }

// 向上更新函数：根据子节点信息更新父节点信息
void pushUp(int u) {
    t[u].mx = std::max(t[lc(u)].mx, t[rc(u)].mx);
}

// 向下传递懒惰标记函数
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
void pushDown(int u, int L, int R) {
    if (t[u].lz == -1) { // 如果没有懒惰标记，直接返回
        return;
    }

    // 将父节点的懒惰标记下传给子节点
    t[lc(u)].mx = t[u].lz;
    t[lc(u)].lz = t[u].lz;

    t[rc(u)].mx = t[u].lz;
    t[rc(u)].lz = t[u].lz;

    t[u].lz = -1; // 清除当前节点的懒惰标记
}

// 建树函数
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
void build(int u, int L, int R) {
    if (L == R) { // 叶子节点
        t[u].mx = 0; // 初始值均为 0
        t[u].lz = -1; // 初始无懒惰标记
        return;
    }
    int m = (L + R) / 2;
    build(lc(u), L, m); // 递归建左子树
    build(rc(u), m + 1, R); // 递归建右子树
    pushUp(u); // 向上更新
    t[u].lz = -1; // 初始懒惰标记为 -1
}

// 区间修改函数：区间 [ql, qr] 的每个数修改为 val
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 查询/修改区间 [ql, qr]
// val: 修改的值
void updateRange(int u, int L, int R, int ql, int qr, long long val) {
    if (ql <= L && R <= qr) { // 当前区间 [L, R] 完全在修改区间 [ql, qr] 内
        t[u].mx = val; // 直接更新当前节点的最大值
        t[u].lz = val; // 更新当前节点的懒惰标记
        return;
    }

    // 如果不能完全覆盖，则需要下传懒惰标记
    pushDown(u, L, R);

    int m = (L + R) / 2;
    if (ql <= m) { // 左子区间有交集
        updateRange(lc(u), L, m, ql, qr, val);
    }
    if (qr > m) { // 右子区间有交集
        updateRange(rc(u), m + 1, R, ql, qr, val);
    }
    pushUp(u); // 递归修改后，向上更新
}

// 区间查询函数：查询区间 [ql, qr] 的最大值
// u: 当前节点索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 查询区间 [ql, qr]
long long queryRange(int u, int L, int R, int ql, int qr) {
    if (ql <= L && R <= qr) { // 当前区间 [L, R] 完全在查询区间 [ql, qr] 内
        return t[u].mx;
    }

    // 如果不能完全覆盖，则需要下传懒惰标记
    pushDown(u, L, R);

    long long res = 0; // 最大值初始化为0，因为题目说初始值0，修改值X也可能为0或正数
                       // 如果X可能为负数，这里需要初始化为负无穷
    int m = (L + R) / 2;
    if (ql <= m) { // 左子区间有交集
        res = std::max(res, queryRange(lc(u), L, m, ql, qr));
    }
    if (qr > m) { // 右子区间有交集
        res = std::max(res, queryRange(rc(u), m + 1, R, ql, qr));
    }
    return res;
}

int main() {
    int n, m;
    scanf("%d %d", &n, &m);
    n_max_val = n;

    build(1, 1, n); // 从根节点1开始建树，区间 [1, n]

    for (int i = 0; i < m; ++i) {
        int op, l, r;
        scanf("%d %d %d", &op, &l, &r);
        if (op == 1) {
            long long x;
            scanf("%lld", &x);
            updateRange(1, 1, n, l, r, x);
        } else {
            printf("%lld\n", queryRange(1, 1, n, l, r));
        }
    }

    return 0;
}
```

**复杂度分析：**
与例题 1 类似，每次操作的时间复杂度为 `O(logN)`。总复杂度 `O(N + MlogN)`。

## 代码实现模板

这里提供一个带有懒惰标记的线段树通用框架，以支持区间加法和区间求和为例。你可以根据具体问题修改 `Node` 结构体中的存储信息 (`s` 和 `lz`) 以及 `pushUp` 和 `pushDown` 的逻辑。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，如 iostream, algorithm, cstdio 等

// 定义线段树节点的最大数量。N是原始数组的最大长度，线段树通常需要 4*N 的空间。
const int MAXN = 100005;

// 定义线段树节点的结构体
struct Node {
    long long s;  // 当前节点代表区间的和 (sum)
    long long lz; // 懒惰标记 (lazy tag)，表示这个区间需要进行的操作。
                  // 对于区间加法，lz 表示需要给区间每个元素加上的值。
                  // 对于区间覆盖，lz 表示需要把区间每个元素修改成的值，
                  // 此时需要一个特殊值（如 -1 或 0）来表示没有懒惰标记。
} t[MAXN * 4]; // 线段树数组

// 原始数据数组，如果题目给定初始数据，会用到。本模板中作为示例。
// 如果题目初始值均为0，可以省略这个数组的读入和使用。
// long long a[MAXN]; 

// 全局变量，记录线段树维护的原始数组的实际长度 N。
// 在 main 函数中读取 N 后赋值。
int n_max_val; 

// 辅助函数：获取左子节点的索引
// 如果当前节点是 u，其左子节点通常是 u*2
int lc(int u) {
    return u * 2;
}

// 辅助函数：获取右子节点的索引
// 如果当前节点是 u，其右子节点通常是 u*2 + 1
int rc(int u) {
    return u * 2 + 1;
}

// 向上更新函数：根据左右子节点的信息更新父节点 u 的信息
// 通常在子节点修改后调用，用于维护父节点的信息汇总。
void pushUp(int u) {
    // 示例：区间求和，父节点的和等于左右子节点的和之和
    t[u].s = t[lc(u)].s + t[rc(u)].s;
    // 如果是区间求最大值，则是 t[u].mx = std::max(t[lc(u)].mx, t[rc(u)].mx);
}

// 向下传递懒惰标记函数：将当前节点 u 的懒惰标记下传给其子节点
// u: 当前节点的索引
// L, R: 当前节点代表的区间 [L, R]
// 注意：在访问子节点前（无论是修改还是查询），都必须先调用 pushDown
void pushDown(int u, int L, int R) {
    // 如果当前节点没有懒惰标记，则无需下传，直接返回
    if (t[u].lz == 0) { // 针对区间加法，0表示没有要加的值
        return;
    }

    int m = (L + R) / 2; // 计算当前区间的中间点

    // 1. 更新左子节点
    // 左子节点的和增加：当前节点的懒惰标记值 * 左子区间长度
    t[lc(u)].s += t[u].lz * (m - L + 1);
    // 左子节点的懒惰标记累加上当前节点的懒惰标记
    t[lc(u)].lz += t[u].lz;

    // 2. 更新右子节点
    // 右子节点的和增加：当前节点的懒惰标记值 * 右子区间长度
    t[rc(u)].s += t[u].lz * (R - m);
    // 右子节点的懒惰标记累加上当前节点的懒惰标记
    t[rc(u)].lz += t[u].lz;

    // 3. 清除当前节点的懒惰标记，因为它已经下传给子节点了
    t[u].lz = 0;
}

// 建树函数：递归构建线段树
// u: 当前节点的索引 (通常从 1 开始)
// L, R: 当前节点代表的区间 [L, R]
void build(int u, int L, int R) {
    t[u].lz = 0; // 初始时所有节点的懒惰标记都为 0

    if (L == R) { // 如果是叶子节点
        // t[u].s = a[L]; // 如果有初始数组 a，则赋值
        t[u].s = 0; // 题目若说初始值均为0，则直接赋值0
        return;
    }

    int m = (L + R) / 2; // 中点
    build(lc(u), L, m); // 递归构建左子树
    build(rc(u), m + 1, R); // 递归构建右子树
    pushUp(u); // 向上更新当前节点的信息
}

// 区间修改函数：将区间 [ql, qr] 内的每个元素都加上 val
// u: 当前节点的索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 需要修改的目标区间 [ql, qr]
// val: 需要加上的值
void updateRange(int u, int L, int R, int ql, int qr, long long val) {
    // 情况1：当前节点代表的区间 [L, R] 完全包含在修改目标区间 [ql, qr] 内
    if (ql <= L && R <= qr) {
        t[u].s += val * (R - L + 1); // 更新当前节点的和
        t[u].lz += val;             // 更新当前节点的懒惰标记
        return;
    }

    // 情况2：当前节点代表的区间与修改目标区间有交集，但不是完全包含
    // 需要先下传懒惰标记，确保子节点数据最新，再进行递归修改
    pushDown(u, L, R); 

    int m = (L + R) / 2; // 中点
    // 检查左子区间 [L, m] 是否与修改目标区间 [ql, qr] 有交集
    if (ql <= m) {
        updateRange(lc(u), L, m, ql, qr, val);
    }
    // 检查右子区间 [m+1, R] 是否与修改目标区间 [ql, qr] 有交集
    if (qr > m) {
        updateRange(rc(u), m + 1, R, ql, qr, val);
    }
    pushUp(u); // 递归修改子节点后，向上更新当前节点的信息
}

// 区间查询函数：查询区间 [ql, qr] 内所有元素的和
// u: 当前节点的索引
// L, R: 当前节点代表的区间 [L, R]
// ql, qr: 需要查询的目标区间 [ql, qr]
long long queryRange(int u, int L, int R, int ql, int qr) {
    // 情况1：当前节点代表的区间 [L, R] 完全包含在查询目标区间 [ql, qr] 内
    if (ql <= L && R <= qr) {
        return t[u].s; // 直接返回当前节点的和
    }

    // 情况2：当前节点代表的区间与查询目标区间有交集，但不是完全包含
    // 需要先下传懒惰标记，确保子节点数据最新，再进行递归查询
    pushDown(u, L, R); 

    long long res = 0; // 用于累计查询结果
    int m = (L + R) / 2; // 中点

    // 检查左子区间 [L, m] 是否与查询目标区间 [ql, qr] 有交集
    if (ql <= m) {
        res += queryRange(lc(u), L, m, ql, qr);
    }
    // 检查右子区间 [m+1, R] 是否与查询目标区间 [ql, qr] 有交集
    if (qr > m) {
        res += queryRange(rc(u), m + 1, R, ql, qr);
    }
    return res; // 返回合并后的查询结果
}

int main() {
    // 示例主函数
    int n, m; // n: 数组长度, m: 操作次数
    scanf("%d %d", &n, &m);
    n_max_val = n; // 赋值给全局变量

    // 如果题目需要从输入读取初始数组值，可以在这里读入
    // for (int i = 1; i <= n; ++i) {
    //     scanf("%lld", &a[i]);
    // }

    build(1, 1, n); // 构建线段树，从根节点 1 开始，覆盖区间 [1, n]

    for (int i = 0; i < m; ++i) {
        int op, l, r;
        scanf("%d %d %d", &op, &l, &r);
        if (op == 1) { // 操作类型 1: 区间加法
            long long x;
            scanf("%lld", &x);
            updateRange(1, 1, n, l, r, x);
        } else { // 操作类型 2: 区间查询
            printf("%lld\n", queryRange(1, 1, n, l, r));
        }
    }

    return 0;
}
```
