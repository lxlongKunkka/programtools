## 教学目标

### 知识目标
1.  理解“数论分块”（或称“整除分块”）的核心思想：对于正整数 `n`，表达式 `floor(n/i)` 在 `i` 取值 `1` 到 `n` 的过程中，其值最多只有 `2 * sqrt(n)` 种。
2.  掌握 `floor(n/i)` 取值相同的区间的特性：这些区间是连续的，并且能够快速计算出每个区间的右端点 `j = n / (n / i)`。
3.  了解数论分块算法的时间复杂度为 `O(sqrt(n))`。

### 能力目标
1.  能够识别出可以应用数论分块优化的求和问题，特别是涉及 `floor(n/i)` 形式的表达式。
2.  能够独立推导并实现数论分块算法，正确处理循环边界和区间计算。
3.  能够结合等差数列求和等基本数学知识，在分块区间内高效计算子表达式的和。

### 素养目标
1.  培养学生通过观察数学性质来优化算法的思维习惯，提升问题分析和抽象能力。
2.  激发学生对数学之美的兴趣，体会看似复杂的求和问题如何通过巧妙的分块思想变得简洁高效。
3.  培养严谨的编程习惯，特别是在处理整数除法和边界条件时的细致与准确。

第二个顾客说：“老板，我想要把这 `N` 块披萨平均分成 `2` 份，我拿走其中 `floor(N/2)` 块！”
第三个顾客说：“老板，我想要把这 `N` 块披萨平均分成 `3` 份，我拿走其中 `floor(N/3)` 块！”

第 `N` 个顾客说：“老板，我想要把这 `N` 块披萨平均分成 `N` 份，我拿走其中 `floor(N/N)` 块！”

每个顾客都拿走了 `floor(N/i)` 块披萨，我们想知道，总共被拿走了多少块披萨呢？也就是说，我们要计算 `sum(floor(N/i))`，从 `i=1` 到 `N`。

`floor(10/1) = 10`
`floor(10/2) = 5`
`floor(10/3) = 3`
`floor(10/4) = 2`
`floor(10/5) = 2`
`floor(10/6) = 1`
`floor(10/7) = 1`
`floor(10/8) = 1`
`floor(10/9) = 1`
`floor(10/10) = 1`

但是！如果 `N` 是一个天文数字，比如 `10^12` 甚至 `10^18`，你还能一个一个算吗？那可得算到地老天荒！
有没有一种“偷懒”又“聪明”的方法，能快速算出这个总和呢？这就是我们今天要学习的“数论分块”的秘密武器！它能让你在 `sqrt(N)` 的时间内，解决 `N` 这么大的问题！是不是很神奇？让我们一起来揭开它的面纱吧！

## 深度知识点讲解

### 1. 核心观察：`floor(n/i)` 的奇妙性质

我们先来仔细观察一下 `floor(n/i)` 的值是如何变化的。以 `n=20` 为例：

| `i` | `floor(20/i)` |
| :-- | :------------ |
| 1   | 20            |
| 2   | 10            |
| 3   | 6             |
| 4   | 5             |
| 5   | 4             |
| 6   | 3             |
| 7   | 2             |
| 8   | 2             |
| 9   | 2             |
| 10  | 2             |
| 11  | 1             |
| 12  | 1             |
| ... | ...           |
| 20  | 1             |

看到了吗？
*   当 `i` 从 `1` 变到 `n` 时，`floor(n/i)` 的值是**单调不增**的。
*   更重要的是，它不是每次都变！很多时候，连续的 `i` 值对应的 `floor(n/i)` 竟然是**一样的**！
    *   `floor(20/1)` 是 `20`。
    *   `floor(20/2)` 是 `10`。
    *   `floor(20/3)` 是 `6`。
    *   `floor(20/4)` 是 `5`。
    *   `floor(20/5)` 是 `4`。
    *   `floor(20/6)` 是 `3`。
    *   `floor(20/7), floor(20/8), floor(20/9), floor(20/10)` 都是 `2`。 (这是一个块！)
    *   `floor(20/11), ..., floor(20/20)` 都是 `1`。 (又是一个块！)

这个现象就是数论分块的基础！我们可以把 `floor(n/i)` 值相同的 `i` 归为一“块”，然后对这一整块进行快速处理。

### 2. 为什么 `floor(n/i)` 的值种类很少？

这个性质非常关键。对于一个给定的 `n`，`floor(n/i)` 的取值种类到底有多少呢？
我们可以把 `i` 的范围 `[1, n]` 分成两部分：
1.  当 `i <= sqrt(n)` 时：
    *   `i` 的取值有 `sqrt(n)` 种。
    *   `floor(n/i)` 的取值最多也有 `sqrt(n)` 种（因为 `i` 每次变，`floor(n/i)` 可能变）。
2.  当 `i > sqrt(n)` 时：
    *   这时 `floor(n/i)` 的值会怎样呢？
    *   因为 `i > sqrt(n)`，所以 `n/i < n/sqrt(n) = sqrt(n)`。
    *   这意味着 `floor(n/i)` 的值会小于 `sqrt(n)`。
    *   所以，`floor(n/i)` 在这个区间内的取值种类也最多只有 `sqrt(n)` 种（从 `1` 到 `floor(sqrt(n)-1)`）。

综合以上两点，`floor(n/i)` 的总取值种类最多只有 `sqrt(n) + sqrt(n) = 2 * sqrt(n)` 种！
`sqrt(10^12) = 10^6`，`2 * 10^6` 比 `10^12` 小太多了！这就是我们能够优化的根本原因。

### 3. 如何找到每个块的边界？

我们知道 `floor(n/i)` 的值是一块一块地变化的。现在的问题是，给定一个块的起始点 `s` (也就是当前的 `i`)，我们怎么快速找到这个块的结束点 `e` 呢？

假设当前 `i` 对应的 `floor(n/i)` 的值是 `k`。
那么，对于这个块内的所有 `x` (从 `s` 到 `e`)，都应该有 `floor(n/x) = k`。
我们可以推导出这个块的右端点 `e` 的计算公式：
`e = n / (n / s)`

**为什么是这个公式呢？**
*   `n / s` (整数除法) 就是 `floor(n/s)` 的值，我们把它记作 `k`。
*   那么，我们想找到最大的 `e`，使得 `floor(n/e)` 仍然等于 `k`。
*   `floor(n/x) = k` 等价于 `k <= n/x < k+1`。
*   这又等价于 `x <= n/k` 并且 `x > n/(k+1)`。
*   所以，满足 `floor(n/x) = k` 的最大的 `x` 就是 `floor(n/k)`。
*   把 `k` 替换回 `n/s`，就得到了 `e = floor(n / floor(n/s))`。
*   在 C++ 中，`n / (n / s)` 就是 `floor(n / floor(n/s))` 的精确实现（因为整数除法自动取底）。

**生活类比：**
想象你正在整理一堆扑克牌，按照点数分组。你拿到一张 `5`，你知道所有 `5` 都在一起。你找到第一张 `5` (起始点 `s`)，你想要知道这堆 `5` 什么时候结束 (结束点 `e`)。
`n / i` 就像是这张牌的“类型标签”。`n / (n / i)` 就像是“所有这种类型标签的牌，最远能放到哪里？”
这个公式就是告诉你，所有 `i'` 使得 `floor(n/i')` 和 `floor(n/i)` 值相同的 `i'`，它能取到的最大值就是 `n / (n / i)`。

### 4. 数论分块算法步骤

有了上面的观察和公式，数论分块的算法就呼之欲出了：

1.  初始化一个变量 `sum = 0`，用于存储最终结果。
2.  初始化一个循环变量 `s = 1`，表示当前块的起始位置。
3.  在一个 `while (s <= n)` 循环中，重复以下操作：
    *   **计算当前 `floor(n/s)` 的值：** 设 `k = n / s`。
    *   **计算当前块的结束位置 `e`：** `e = n / k`。
        *   注意：这里 `e` 的值不能超过 `n`。不过，因为 `k = n/s >= n/n = 1`，所以 `n/k <= n/1 = n`，所以 `e` 永远不会超过 `n`。
    *   **处理当前块 `[s, e]`：** 在这个区间内，所有的 `i` 都满足 `floor(n/i) = k`。根据具体问题，对这个区间进行求和或其他操作。
        *   例如，如果求 `sum(floor(n/i))`，那么这个块对总和的贡献就是 `k * (e - s + 1)`。
    *   **更新 `s`：** 将 `s` 更新为 `e + 1`，准备处理下一个块。
4.  循环结束后，`sum` 就是我们要求的结果。

### 5. 时间复杂度分析

*   我们知道 `floor(n/i)` 的取值种类最多只有 `2 * sqrt(n)` 种。
*   每次循环，我们都会处理一个完整的“块”，然后跳到下一个块的起始位置。
*   这意味着 `while` 循环的次数就等于 `floor(n/i)` 的取值种类数。
*   因此，整个算法的时间复杂度是 `O(sqrt(n))`。
*   对于 `N = 10^12`，`sqrt(N) = 10^6`，这个算法只需要进行大约 `2 * 10^6` 次循环，效率非常高！

### 6. 常见误区

*   **整数除法：** C++ 中的 `/` 运算符对整数执行的是向下取整的除法，这正好符合 `floor` 函数的要求。但要小心，如果 `n` 和 `i` 是负数，行为可能会不同，不过在数论分块中 `n` 和 `i` 都是正数，所以没问题。
*   **边界条件：** 循环的起始 `s=1` 和终止 `s<=n` 必须正确。`e` 的计算 `n / (n / s)` 也要确保逻辑正确。
*   **数据类型：** `n` 可能会很大（`10^12` 或 `10^18`），所以 `n` 和 `sum` 必须使用 `long long` 类型。`s` 和 `e` 也可能达到 `n` 的级别，所以也应使用 `long long`。

## 典型例题精讲

### 例题一：求 `sum(floor(n/i))`，其中 `i` 从 `1` 到 `n`。

**题目描述：**
给定一个正整数 `n`，计算表达式 `sum_{i=1}^{n} floor(n/i)` 的值。
例如，当 `n=10` 时，结果为 `27`。
输入：`n` ( `1 <= n <= 10^12` )
输出：`sum_{i=1}^{n} floor(n/i)` 的值。

**思路分析：**
这正是我们引入时提到的披萨问题。这是一个典型的数论分块应用场景。
1.  我们观察到 `floor(n/i)` 在 `i` 的某些连续区间内取值相同。
2.  我们可以利用这个特性，将 `[1, n]` 这个区间划分为若干个子区间 `[s, e]`。
3.  在每个子区间 `[s, e]` 内，`floor(n/i)` 的值都是一个常数 `k = floor(n/s)`。
4.  那么，这个子区间对总和的贡献就是 `k * (e - s + 1)`。
5.  通过循环迭代处理所有子区间，累加贡献即可。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含常用的头文件，如iostream

int main() {
    long long n; // n 可能很大，需要使用 long long
    scanf("%lld", &n); // 使用 scanf 读取大整数

    long long sum = 0; // 存储总和，也需要 long long
    
    // 数论分块的核心循环
    // s 代表当前块的起始位置
    // e 代表当前块的结束位置
    for (long long s = 1, e; s <= n; s = e + 1) {
        // 计算当前块的 floor(n/s) 值
        // 这个值在当前块 [s, e] 内都是相同的
        long long k = n / s; 

        // 计算当前块的结束位置 e
        // e = n / (n / s) 是一个重要的公式
        e = n / k; 

        // 如果 e 超过了 n，需要将其限制在 n 以内
        // 但实际上，e = n / k; 已经保证 e <= n 了，因为 k >= 1
        // e = std::min(e, n); // 这一行在这里是非必要的，但加上也无妨，更保险
        
        // 当前块 [s, e] 对总和的贡献
        // 在这个区间内，共有 (e - s + 1) 个数，每个数的 floor(n/i) 值都是 k
        sum += k * (e - s + 1);
    }

    printf("%lld\n", sum); // 输出结果
    
    return 0;
}
```

**时间复杂度：**
`O(sqrt(n))`。因为循环的次数取决于 `floor(n/i)` 的不同取值个数，而这个个数最多为 `2 * sqrt(n)`。

### 例题二：求 `sum(i * floor(n/i))`，其中 `i` 从 `1` 到 `n`。

**题目描述：**
给定一个正整数 `n`，计算表达式 `sum_{i=1}^{n} (i * floor(n/i))` 的值。
输入：`n` ( `1 <= n <= 10^12` )
输出：`sum_{i=1}^{n} (i * floor(n/i))` 的值。

**思路分析：**
这道题同样涉及 `floor(n/i)`，所以依然可以使用数论分块。
1.  和例题一一样，我们将 `[1, n]` 划分为若干个子区间 `[s, e]`。
2.  在每个子区间 `[s, e]` 内，`floor(n/i)` 的值是一个常数 `k = floor(n/s)`。
3.  那么，对于这个区间内的 `i`，每一项都是 `i * k`。
4.  所以，这个子区间对总和的贡献是 `sum_{i=s}^{e} (i * k)`。
5.  由于 `k` 是常数，我们可以提出来：`k * sum_{i=s}^{e} i`。
6.  `sum_{i=s}^{e} i` 是一个等差数列求和，其和为 `(s + e) * (e - s + 1) / 2`。
7.  因此，当前块的贡献是 `k * (s + e) * (e - s + 1) / 2`。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含常用的头文件，如iostream

int main() {
    long long n; // n 可能很大，需要使用 long long
    scanf("%lld", &n); // 使用 scanf 读取大整数

    long long sum = 0; // 存储总和，也需要 long long
    
    // 数论分块的核心循环
    for (long long s = 1, e; s <= n; s = e + 1) {
        // 计算当前块的 floor(n/s) 值
        long long k = n / s; 

        // 计算当前块的结束位置 e
        e = n / k; 
        
        // 计算当前块 [s, e] 对总和的贡献
        // 在这个区间内，floor(n/i) 的值都是 k
        // 我们需要计算 sum(i * k) = k * sum(i)
        // sum(i) 从 s 到 e 是一个等差数列求和：(s + e) * (e - s + 1) / 2
        sum += k * (s + e) * (e - s + 1) / 2;
    }

    printf("%lld\n", sum); // 输出结果
    
    return 0;
}
```

**时间复杂度：**
`O(sqrt(n))`。与例题一相同，算法的循环次数不变，只是每次循环内部的计算稍有变化，但仍是常数时间操作。

## 代码实现模板

下面是一个通用的数论分块模板，你可以根据具体问题修改 `// 在这里处理当前块 [s, e]` 部分。

```cpp
#include <bits/stdc++.h> // 包含常用的头文件

int main() {
    long long n; // n 的范围可能很大，需要使用 long long
    scanf("%lld", &n); // 使用 scanf 读取大整数，比 cin 稍快

    long long totalSum = 0; // 存储最终结果，需要 long long

    // 数论分块的核心循环
    // s: 当前块的起始位置 (start)
    // e: 当前块的结束位置 (end)
    for (long long s = 1, e; s <= n; s = e + 1) {
        // 1. 计算当前块的 floor(n/i) 值
        // 对于当前块 [s, e] 内的所有 i，floor(n/i) 的值都等于 k
        long long k = n / s; 

        // 2. 计算当前块的结束位置 e
        // 这是数论分块的关键公式：e = n / (n / s)
        // 它能找到使 floor(n/i) 值与 floor(n/s) 相同的所有 i 中最大的那个
        e = n / k; 
        
        // 【重要提醒】
        // 实际上，e = n / (n / s) 已经保证 e <= n。
        // 因为 n/s 至少为 1 (s <= n)，所以 n/(n/s) 不会大于 n。
        // 但为了代码的健壮性或在一些变体问题中，可能需要显式限制 e <= n。
        // e = std::min(e, n); // 通常情况下，这一行是不必要的，但可以保留以防万一。

        // 3. 在这里处理当前块 [s, e]
        // 根据具体题目要求，计算当前块对总和的贡献
        // 例如：
        //   - 如果要求 sum(floor(n/i))：
        //     totalSum += k * (e - s + 1);
        //   - 如果要求 sum(i * floor(n/i))：
        //     // 在当前块 [s, e] 中，floor(n/i) 都是 k
        //     // 需要计算 k * sum(i) for i from s to e
        //     // sum(i) = (s + e) * (e - s + 1) / 2 (等差数列求和)
        //     totalSum += k * (s + e) * (e - s + 1) / 2;
        //   - 其他更复杂的表达式...
        
        // 示例：计算 sum(floor(n/i))
        totalSum += k * (e - s + 1); 
    }

    printf("%lld\n", totalSum); // 输出最终结果
    
    return 0;
}
```

**Allman 风格大括号示例：**
上面的代码已经采用了 Allman 风格大括号。
特点是：
*   函数名、`for`、`while`、`if` 等关键字后的大括号另起一行。
*   代码块内容缩进。

```cpp
// 示例：Allman 风格
int main() 
{
    long long n;
    scanf("%lld", &n);

    for (long long s = 1, e; s <= n; s = e + 1) 
    {
        long long k = n / s;
        e = n / k;
        // ...
    }

    return 0;
}
```

## 分层练习题目

### 基础巩固

1.  **题目名称：** 简单整除和
    **题目描述：** 给定一个正整数 `n` ( `1 <= n <= 10^9` )，计算 `sum_{i=1}^{n} (floor(n/i) + 1)` 的值。
    **提示：** 在分块区间 `[s, e]` 内，`floor(n/i)` 是常数 `k`。那么 `floor(n/i) + 1` 也是常数 `k+1`。
2.  **题目名称：** 奇偶整除和
    **题目描述：** 给定一个正整数 `n` ( `1 <= n <= 10^9` )，计算 `sum_{i=1}^{n} (floor(n/i) % 2 == 0 ? floor(n/i) : 0)` 的值。 (如果 `floor(n/i)` 是偶数就加上它，否则加0)
    **提示：** 在分块区间 `[s, e]` 内，`floor(n/i)` 是常数 `k`。只需判断 `k` 的奇偶性。

### 能力提升

1.  **题目名称：** 平方整除和
    **题目描述：** 给定一个正整数 `n` ( `1 <= n <= 10^12` )，计算 `sum_{i=1}^{n} (floor(n/i) * floor(n/i))` 的值。
    **提示：** 在分块区间 `[s, e]` 内，`floor(n/i)` 是常数 `k`。该区间的贡献为 `k * k * (e - s + 1)`。
2.  **题目名称：** 等差数列整除和
    **题目描述：** 给定正整数 `n` ( `1 <= n <= 10^12` )，计算 `sum_{i=1}^{n} ( (2*i + 1) * floor(n/i) )` 的值。
    **提示：** 在分块区间 `[s, e]` 内，`floor(n/i)` 是常数 `k`。该区间的贡献为 `k * sum_{i=s}^{e} (2*i + 1)`。`sum_{i=s}^{e} (2*i + 1)` 是一个等差数列求和。

### 拓展挑战

1.  **题目名称：** 两个变量的整除和
    **题目描述：** 给定两个正整数 `n, m` ( `1 <= n, m <= 10^12` )，计算 `sum_{i=1}^{min(n, m)} (floor(n/i) * floor(m/i))` 的值。
    **提示：** 这需要对两个 `floor` 函数同时分块。一个区间的右端点 `e` 需要同时满足 `e <= n/(n/s)` 和 `e <= m/(m/s)`，所以 `e = min(n/(n/s), m/(m/s))`。
2.  **题目名称：** 莫比乌斯反演中的分块 (仅作了解，无需实现)
    **题目描述：** 在一些涉及莫比乌斯反演的问题中，最终的求和形式可能包含 `sum_{i=1}^{n} (F(i) * floor(n/i))` 的结构，其中 `F(i)` 是一个可以通过前缀和快速计算的函数。数论分块是解决这类问题的关键工具。

```cpp
#include <bits/stdc++.h>

int main() {
    long long n;
    scanf("%lld", &n);
    long long totalSum = 0;
    for (long long s = 1, e; s <= n; s = e + 1) {
        long long k = n / s;
        e = n / k;
        // 计算 sum_{i=s}^{e} (2*i + 1)
        // 这是一个等差数列，首项 a1 = 2*s + 1, 末项 an = 2*e + 1
        // 项数 count = e - s + 1
        // 和 = (a1 + an) * count / 2
        long long a1 = 2 * s + 1;
        long long an = 2 * e + 1;
        long long count = e - s + 1;
        long long sum_2i_plus_1 = (a1 + an) * count / 2;
        
        totalSum += k * sum_2i_plus_1;
    }
    printf("%lld\n", totalSum);
```

**拓展挑战 1：两个变量的整除和**

```cpp
#include <bits/stdc++.h>

int main() {
    long long n, m;
    scanf("%lld %lld", &n, &m);
    long long totalSum = 0;
    long long limit = std::min(n, m); // 循环到 min(n,m)

    for (long long s = 1, e; s <= limit; s = e + 1) {
        long long k1 = n / s; // floor(n/s)
        long long k2 = m / s; // floor(m/s)
        
        // 当前块的结束点 e，需要同时满足 n/i == k1 和 m/i == k2
        // 所以 e = min( n/k1, m/k2 )
        e = std::min(n / k1, m / k2);
        
        // 确保 e 不超过 limit
        e = std::min(e, limit); 
        
        totalSum += k1 * k2 * (e - s + 1);
    }
    printf("%lld\n", totalSum);
    return 0;
}
```
