## 深度知识点讲解

### 1. 莫队的本质：离线处理 + 分块优化 + 增量维护

莫队算法（Mo's Algorithm）是解决一类**区间查询问题**的利器。它的核心思想可以概括为三点：

*   **离线处理 (Offline Processing)**：
    *   **是什么？** 离线处理是指我们不立即回答每个查询，而是把所有的查询都收集起来，统一处理。就像你收到一大堆邮件，不会一封一封立即回复，而是先把所有邮件都看一遍，然后规划一个最佳的回复顺序，最后再集中回复。
    *   **为什么？** 这种“攒着一起干”的策略，是为了能够根据查询的特点进行优化排序，从而减少总体的计算量。如果必须立即回答（在线处理），我们就无法进行这种全局优化。
    *   **生活类比**：你面前有一堆待洗的衣服（查询），你可以选择一件一件洗（在线），也可以选择把所有衣服分类（排序），然后用洗衣机一次性洗完（离线批处理）。显然，后者效率更高。

*   **左右指针增量维护 (Sliding Window / Two Pointers with Incremental Updates)**：
    *   **是什么？** 假设我们已经知道了区间 `[currentL, currentR]` 的答案。如果下一个查询是 `[L, R]`，我们不需要重新计算整个 `[L, R]` 区间的答案。我们可以通过移动 `currentL` 和 `currentR` 这两个指针，一步步地将当前区间调整到目标区间 `[L, R]`。
    *   **怎么做？** 每次移动指针，我们只需要 `add` 一个新元素（指针向外扩展）或 `del` 一个旧元素（指针向内收缩），并根据这个元素的加入或移除来更新答案。
    *   **关键点**：`add(x)` 和 `del(x)` 这两个操作必须非常高效，最好是 `O(1)` 或 `O(logN)`。如果这两个操作很慢，莫队算法的优势就会大大削弱。

*   **分块优化 (Square Root Decomposition for Sorting Queries)**：
    *   **是什么？** 这是莫队算法最核心的优化策略。如果我们只是随机地移动 `currentL` 和 `currentR`，那么总的移动距离可能会非常大。为了减少指针的总移动距离，我们对所有的查询 `(L, R)` 进行特殊的排序。
    *   **怎么想到的？** 想象一下，我们有很多查询 `[L1, R1], [L2, R2], ..., [LQ, RQ]`。如果 `L` 相近的查询能排在一起处理，那么 `currentL` 就不用跳来跳去，而 `currentR` 也可以顺着一个方向移动。这不就是“分块”的思想吗？
    *   **具体做法**：
        1.  **分块**：将整个数组 `A` 划分为 `sqrt(N)` 个大小为 `sqrt(N)` 的块。每个元素 `A[i]` 属于第 `i / block_size` 个块。
        2.  **排序规则**：
            *   **主关键字**：按照查询的左端点 `L` 所属的块的编号进行升序排序。
            *   **次关键字**：如果两个查询的 `L` 属于同一个块，那么：
                *   如果这个块的编号是**偶数**，则按照 `R` 的**升序**排序。
                *   如果这个块的编号是**奇数**，则按照 `R` 的**降序**排序。
            *   **为什么要奇偶块交替排序？** 这是莫队算法的“神来之笔”！它能大大减少 `R` 指针的总移动距离。当 `L` 指针在一个块内移动时，`currentR` 指针在块内要么是单调递增，要么是单调递减，避免了来回跳跃。当 `L` 指针从一个块跳到下一个块时，`currentR` 指针会从当前块的末尾直接跳到下一个块的开头（或者末尾），节省了在块间来回移动的距离。

### 2. 时间复杂度分析

假设数组长度为 `N`，查询数量为 `Q`，块的大小为 `B`。

1.  **`L` 指针的移动**：
    *   **块间移动**：`L` 指针最多会跨越 `N/B` 个块。在处理 `Q` 个查询时，`L` 指针最多在块之间跳转 `Q` 次。每次跳转最多移动 `B` 的距离。所以这部分的总移动距离是 `O(Q * B)`。
    *   **块内移动**：对于每个块，当 `L` 指针在块内移动时，它最多移动 `B` 的距离。总共有 `N/B` 个块。所以这部分的总移动距离是 `O(N/B * B) = O(N)`。
    *   所以 `L` 指针的总移动距离为 `O(Q * B + N)`。

2.  **`R` 指针的移动**：
    *   **奇偶块优化**：由于奇偶块交替排序的策略，`R` 指针在同一个块内的移动几乎是单向的。对于一个块内的所有查询，`R` 指针最多从 `1` 移动到 `N`（或从 `N` 移动到 `1`），即移动距离为 `O(N)`。
    *   总共有 `N/B` 个块。所以 `R` 指针的总移动距离为 `O(N/B * N)`。

3.  **总时间复杂度**：
    *   假设 `add` 和 `del` 操作的复杂度为 `O(1)`。
    *   总复杂度 = `O(Q * B + N + N * N / B)`。
    *   为了使这个表达式最小，我们通常选择 `B = sqrt(N)`。
    *   代入 `B = sqrt(N)`：`O(Q * sqrt(N) + N + N * N / sqrt(N)) = O(Q * sqrt(N) + N * sqrt(N))`。
    *   因此，莫队算法的总时间复杂度是 `O((Q + N) * sqrt(N))`。

### 3. 适用场景

莫队算法适用于以下类型的区间查询问题：
*   **离线查询**：所有查询可以预先读取并排序。
*   **`add` 和 `del` 操作高效**：对区间进行单个元素的添加和删除操作的复杂度较低（最好是 `O(1)`）。

## 典型例题精讲

### 例题1：区间不同数的个数 (经典莫队入门题)

**题目描述**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中有多少个不同的数。

**思路分析**：

1.  **暴力想法**：对于每个查询 `[L, R]`，开一个 `bool` 数组或哈希表，遍历 `L` 到 `R`，统计不同数的个数。每次查询 `O(R-L+1)`，总复杂度 `O(Q*N)`，太慢。
2.  **莫队思想**：
    *   **离线化**：将所有查询 `(L, R, id)` 存起来。
    *   **`add(x)` 操作**：当我们将元素 `a[x]` 加入当前区间时，我们需要更新不同数的计数器 `ans`。
        *   `cnt[a[x]]` 表示 `a[x]` 在当前区间出现的次数。
        *   如果 `cnt[a[x]]` 从 `0` 变为 `1`，说明 `a[x]` 是第一次出现，`ans++`。
        *   然后 `cnt[a[x]]++`。
    *   **`del(x)` 操作**：当我们将元素 `a[x]` 从当前区间移除时，需要更新 `ans`。
        *   `cnt[a[x]]--`。
        *   如果 `cnt[a[x]]` 从 `1` 变为 `0`，说明 `a[x]` 已经全部移除，`ans--`。
    *   **分块排序**：按照莫队算法的规则对查询进行排序。

**代码实现**：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

// 使用原生数组代替 vector
const int MAXN = 100005; // 假设N最大10万
const int MAX_VAL = 1000005; // 假设数值范围最大100万

int a[MAXN]; // 原始数组
int cnt[MAX_VAL]; // 统计每个数值出现的次数
int ans[MAXN]; // 存储每个查询的结果
int currentL = 1, currentR = 0; // 当前莫队区间的左右指针
int current_distinct_count = 0; // 当前区间不同数的个数
int block_size; // 块大小

struct Query {
    int l, r, id;
};

// 莫队排序比较函数
bool cmp(Query x, Query y) {
    int block_x = x.l / block_size;
    int block_y = y.l / block_size;

    if (block_x != block_y) {
        return block_x < block_y;
    }
    // 奇偶块优化
    if (block_x % 2 == 0) { // 偶数块，R升序
        return x.r < y.r;
    } else { // 奇数块，R降序
        return x.r > y.r;
    }
}

// 添加元素到当前区间
void add(int x) {
    if (cnt[a[x]] == 0) {
        current_distinct_count++;
    }
    cnt[a[x]]++;
}

// 移除元素从当前区间
void del(int x) {
    cnt[a[x]]--;
    if (cnt[a[x]] == 0) {
        current_distinct_count--;
    }
}

void solve_example1() {
    int n, q;
    std::cin >> n >> q;

    block_size = static_cast<int>(sqrt(n)); // 计算块大小

    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i];
    }

    Query queries[MAXN]; // 使用原生数组存储查询
    for (int i = 0; i < q; ++i) {
        std::cin >> queries[i].l >> queries[i].r;
        queries[i].id = i;
    }

    std::sort(queries, queries + q, cmp); // 排序查询

    for (int i = 0; i < q; ++i) {
        int l = queries[i].l;
        int r = queries[i].r;

        // 移动 currentL
        while (currentL < l) {
            del(currentL);
            currentL++;
        }
        while (currentL > l) {
            currentL--;
            add(currentL);
        }

        // 移动 currentR
        while (currentR < r) {
            currentR++;
            add(currentR);
        }
        while (currentR > r) {
            del(currentR);
            currentR--;
        }

        ans[queries[i].id] = current_distinct_count; // 记录答案
    }

    for (int i = 0; i < q; ++i) {
        std::cout << ans[i] << std::endl;
    }
}

// int main() {
//     solve_example1();
//     return 0;
// }
```

**复杂度分析**：
*   **时间复杂度**：`O((N + Q) * sqrt(N))`。
*   **空间复杂度**：`O(N + MAX_VAL)`，用于存储原始数组、计数数组、查询和答案。

### 例题2：区间内每个数出现次数的平方和

**题目描述**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中所有数出现次数的平方和。例如，区间 `[1, 2, 2, 3]`，1出现1次，2出现2次，3出现1次，答案为 `1*1 + 2*2 + 1*1 = 6`。

**思路分析**：

1.  这个问题的 `add` 和 `del` 操作同样是 `O(1)`。我们维护一个 `current_sum_of_squares` 来存储答案。
2.  **`add(x)` 操作**：当我们将元素 `a[x]` 加入当前区间时：
    *   假设 `a[x]` 当前的出现次数是 `cnt[a[x]]`。
    *   在加入 `a[x]` 之前，它对总和的贡献是 `cnt[a[x]] * cnt[a[x]]`。
    *   我们先从 `current_sum_of_squares` 中减去这个旧贡献。
    *   然后 `cnt[a[x]]++`。
    *   `a[x]` 新的出现次数是 `cnt[a[x]]`。它对总和的新贡献是 `cnt[a[x]] * cnt[a[x]]`。
    *   将这个新贡献加到 `current_sum_of_squares` 中。
3.  **`del(x)` 操作**：当我们将元素 `a[x]` 从当前区间移除时：
    *   类似地，先从 `current_sum_of_squares` 中减去 `a[x]` 当前的贡献 `cnt[a[x]] * cnt[a[x]]`。
    *   然后 `cnt[a[x]]--`。
    *   将 `a[x]` 新的贡献 `cnt[a[x]] * cnt[a[x]]` 加到 `current_sum_of_squares` 中。

**代码实现**：

（只需要修改 `add` 和 `del` 函数以及 `current_distinct_count` 变量的名称和含义即可，其余框架与例题1相同）

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

const int MAXN_EX2 = 100005; 
const int MAX_VAL_EX2 = 1000005; 

int a_ex2[MAXN_EX2]; 
int cnt_ex2[MAX_VAL_EX2]; 
long long ans_ex2[MAXN_EX2]; // 答案可能很大，用 long long
long long current_sum_of_squares = 0; 
int currentL_ex2 = 1, currentR_ex2 = 0; 
int block_size_ex2; 

struct QueryEx2 {
    int l, r, id;
};

bool cmp_ex2(QueryEx2 x, QueryEx2 y) {
    int block_x = x.l / block_size_ex2;
    int block_y = y.l / block_size_ex2;

    if (block_x != block_y) {
        return block_x < block_y;
    }
    if (block_x % 2 == 0) {
        return x.r < y.r;
    } else {
        return x.r > y.r;
    }
}

void add_ex2(int x) {
    current_sum_of_squares -= (long long)cnt_ex2[a_ex2[x]] * cnt_ex2[a_ex2[x]]; // 减去旧贡献
    cnt_ex2[a_ex2[x]]++;
    current_sum_of_squares += (long long)cnt_ex2[a_ex2[x]] * cnt_ex2[a_ex2[x]]; // 加上新贡献
}

void del_ex2(int x) {
    current_sum_of_squares -= (long long)cnt_ex2[a_ex2[x]] * cnt_ex2[a_ex2[x]]; // 减去旧贡献
    cnt_ex2[a_ex2[x]]--;
    current_sum_of_squares += (long long)cnt_ex2[a_ex2[x]] * cnt_ex2[a_ex2[x]]; // 加上新贡献
}

void solve_example2() {
    int n, q;
    std::cin >> n >> q;

    block_size_ex2 = static_cast<int>(sqrt(n));

    for (int i = 1; i <= n; ++i) {
        std::cin >> a_ex2[i];
    }

    QueryEx2 queries[MAXN_EX2];
    for (int i = 0; i < q; ++i) {
        std::cin >> queries[i].l >> queries[i].r;
        queries[i].id = i;
    }

    std::sort(queries, queries + q, cmp_ex2);

    for (int i = 0; i < q; ++i) {
        int l = queries[i].l;
        int r = queries[i].r;

        while (currentL_ex2 < l) {
            del_ex2(currentL_ex2);
            currentL_ex2++;
        }
        while (currentL_ex2 > l) {
            currentL_ex2--;
            add_ex2(currentL_ex2);
        }

        while (currentR_ex2 < r) {
            currentR_ex2++;
            add_ex2(currentR_ex2);
        }
        while (currentR_ex2 > r) {
            del_ex2(currentR_ex2);
            currentR_ex2--;
        }

        ans_ex2[queries[i].id] = current_sum_of_squares;
    }

    for (int i = 0; i < q; ++i) {
        std::cout << ans_ex2[i] << std::endl;
    }
}
```

*   **时间复杂度**：`O((N + Q) * sqrt(N))`。
*   **空间复杂度**：`O(N + MAX_VAL)`。

## 代码实现模板

```cpp
#include <iostream>  // 输入输出流
#include <algorithm> // 用于 std::sort
#include <cmath>     // 用于 sqrt 函数

// 定义常量，避免魔术数字，提高代码可读性和可维护性
const int MAXN = 100005;    // 数组和查询的最大数量
const int MAX_VAL = 1000005; // 元素值的最大范围，根据题目实际情况调整

// 全局变量，方便在不同函数中访问和修改
int a[MAXN];          // 原始数据数组，下标从1开始
int cnt[MAX_VAL];     // 计数数组，用于存储当前区间内每个元素出现的次数
long long ans[MAXN];  // 存储每个查询的最终答案，使用long long防止溢出
int currentL = 1, currentR = 0; // 当前莫队算法维护的区间 [currentL, currentR]
long long current_answer = 0;   // 当前区间 [currentL, currentR] 的答案
int block_size;       // 块的大小，通常为 sqrt(N)

// 查询结构体，存储查询的左右端点和原始ID
struct Query {
    int l, r, id; // l: 左端点, r: 右端点, id: 原始查询序号
};

// 莫队算法的核心：比较函数，用于对查询进行排序
// 遵循分块+奇偶块优化的策略
bool cmp(Query x, Query y) {
    // 计算左右端点所在的块编号
    int block_x = x.l / block_size;
    int block_y = y.l / block_size;

    // 如果不在同一个块，则按块编号升序排序
    if (block_x != block_y) {
        return block_x < block_y;
    }
    // 如果在同一个块，则根据块编号的奇偶性进行R端点排序
    // 偶数块：R升序
    // 奇数块：R降序 (奇偶块优化，减少R指针来回移动)
    if (block_x % 2 == 0) {
        return x.r < y.r;
    } else {
        return x.r > y.r;
    }
}

    // if (cnt[a[x]] == 0) {

    // 示例：统计每个数出现次数的平方和
    current_answer -= (long long)cnt[a[x]] * cnt[a[x]]; // 减去旧贡献
    cnt[a[x]]++;                                       // 更新计数
    current_answer += (long long)cnt[a[x]] * cnt[a[x]]; // 加上新贡献
}

// 这个函数也需要根据具体问题进行实现
void del(int x) {

    // cnt[a[x]]--;
    // if (cnt[a[x]] == 0) {
    //     current_answer--;
    // }

    // 示例：统计每个数出现次数的平方和
    current_answer -= (long long)cnt[a[x]] * cnt[a[x]]; // 减去旧贡献
    cnt[a[x]]--;                                       // 更新计数
    current_answer += (long long)cnt[a[x]] * cnt[a[x]]; // 加上新贡献
}

int main() {

    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, q; // n: 数组长度, q: 查询数量
    std::cin >> n >> q;

    // 计算块大小，通常取 sqrt(N)。如果 N=0 避免除零错误。
    block_size = static_cast<int>(sqrt(n));
    if (block_size == 0) block_size = 1; // 至少为1，避免除零

    // 读取原始数组a的元素
    for (int i = 1; i <= n; ++i) {
        std::cin >> a[i];

    Query queries[MAXN]; // 使用原生数组存储查询
    for (int i = 0; i < q; ++i) {
        std::cin >> queries[i].l >> queries[i].r;
        queries[i].id = i; // 记录原始查询序号
    }

    // 对查询进行排序
    std::sort(queries, queries + q, cmp);

    // 逐个处理排序后的查询
    for (int i = 0; i < q; ++i) {
        int l = queries[i].l; // 当前查询的左端点
        int r = queries[i].r; // 当前查询的右端点

        // 移动 currentL 指针到目标 l
        while (currentL < l) {
            del(currentL);
            currentL++;
        }
        while (currentL > l) {
            currentL--;
            add(currentL);
        }

        // 移动 currentR 指针到目标 r
        while (currentR < r) {
            currentR++;
            add(currentR);
        }
        while (currentR > r) {
            del(currentR);
            currentR--;
        }

        // 记录当前查询的答案
        ans[queries[i].id] = current_answer;
    }

    // 按照原始查询顺序输出答案
    for (int i = 0; i < q; ++i) {
        std::cout << ans[i] << std::endl;
    }

    return 0;
}
```

## 课堂互动

### 提问与思考

1.  **莫队算法是离线算法还是在线算法？为什么？**
    *   **思考点**：回顾“离线处理”的概念，强调莫队算法需要预先知道所有查询才能进行排序优化。
    *   **预期回答**：莫队算法是离线算法。因为它需要将所有查询先收集起来，然后根据特定的规则（分块、奇偶块优化）进行排序，最后再统一处理。如果必须立即回答（在线），就无法进行这种全局的排序优化。

2.  **如果 `add` 和 `del` 操作的复杂度很高（例如 `O(N)`），莫队算法还会像现在这样高效吗？为什么？**
    *   **思考点**：强调 `add/del` 操作效率对莫队总复杂度的影响。
    *   **预期回答**：不会。如果 `add/del` 操作是 `O(N)`，那么总复杂度将变为 `O((N+Q)*sqrt(N)*N)`，这比暴力解法 `O(Q*N)` 还要慢。莫队算法高效的前提是 `add/del` 操作足够快，最好是 `O(1)`。

3.  **莫队算法中，查询的排序规则有什么讲究？为什么要分块？为什么要奇偶块交替排序？**
    *   **思考点**：这是莫队算法的核心优化，需要学生理解其背后的逻辑。
    *   **预期回答**：
        *   **分块**：是为了将 `L` 坐标相近的查询放在一起处理，减少 `L` 指针的跳跃距离。
        *   **奇偶块交替排序**：是为了进一步优化 `R` 指针的移动。当 `L` 在同一个块内移动时，`R` 指针可以近似单向移动（要么一直向右，要么一直向左），避免了来回“折返跑”，大大减少了 `R` 指针的总移动距离。

4.  **莫队算法中的 `block_size` 为什么通常选择 `sqrt(N)`？**
    *   **思考点**：引导学生回顾复杂度分析中的 `O(Q*B + N*N/B)` 表达式，尝试找到最优解。
    *   **预期回答**：当 `B = sqrt(N)` 时，`Q*B` 和 `N*N/B` 这两部分的时间复杂度达到平衡，都近似为 `O(Q*sqrt(N))` 和 `O(N*sqrt(N))`，使得总时间复杂度最小化，为 `O((N+Q)*sqrt(N))`。

**任务**：设计 `add(x)` 和 `del(x)` 函数，用于解决“查询区间内有多少个出现次数为偶数的数”的问题。

*   **提示**：你需要维护一个 `cnt` 数组（记录每个数出现次数）和一个 `even_count` 变量（记录当前区间内出现次数为偶数的数的个数）。
*   **讨论时间**：10分钟，鼓励小组内讨论并写出伪代码。

**参考实现（伪代码）**：
```cpp
int even_count = 0; // 全局变量，记录出现次数为偶数的数的个数

void add(int x) { // x 是数组下标
    // 在增加 a[x] 之前，检查 a[x] 的出现次数 cnt[a[x]]
    if (cnt[a[x]] % 2 == 0) { // 如果之前是偶数次（包括0次），现在要变奇数次
        even_count--;        // 偶数个数减1
    } else { // 如果之前是奇数次，现在要变偶数次
        even_count++;        // 偶数个数加1
    }
    cnt[a[x]]++; // 实际增加 a[x] 的计数
}

void del(int x) { // x 是数组下标
    // 在减少 a[x] 之后，检查 a[x] 的出现次数 cnt[a[x]]
    if (cnt[a[x]] % 2 == 0) { // 如果之前是偶数次，现在要变奇数次
        even_count--;        // 偶数个数减1
    } else { // 如果之前是奇数次（包括1次），现在要变偶数次（包括0次）
        even_count++;        // 偶数个数加1
    }
    cnt[a[x]]--; // 实际减少 a[x] 的计数
}
```

## 分层练习题目

### 基础巩固

1.  **区间异或和**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中所有数的异或和。
    *   **提示**：异或操作具有结合律和交换律。`add(x)` 和 `del(x)` 都只是将 `current_xor_sum` 与 `a[x]` 进行异或操作。

2.  **区间和**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中所有数的和。
    *   **提示**：`add(x)` 就是 `current_sum += a[x]`，`del(x)` 就是 `current_sum -= a[x]`。注意 `long long`。

### 能力提升

1.  **区间内出现次数为 `K` 的数的个数**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中有多少个数的出现次数恰好为 `K`。
    *   **提示**：需要维护 `cnt` 数组和 `count_of_k_freq` 变量。`add(x)` 和 `del(x)` 时，除了更新 `cnt[a[x]]`，还要判断 `cnt[a[x]]` 是否从 `K-1` 变为 `K` 或从 `K` 变为 `K+1`，以及从 `K+1` 变为 `K` 或从 `K` 变为 `K-1`，来更新 `count_of_k_freq`。

2.  **区间中位数查询**：给定一个长度为 `N` 的序列 `a`，有 `Q` 次查询，每次查询给定 `L, R`，求区间 `[L, R]` 中的中位数。
    *   **提示**：此题直接用普通莫队可能比较困难，`add/del` 无法 `O(1)` 维护中位数。需要结合其他数据结构，如树状数组或分块数组（值域分块），使 `add/del` 变为 `O(sqrt(MAX_VAL))` 或 `O(log MAX_VAL)`，从而莫队总复杂度变为 `O((N+Q)*sqrt(N)*sqrt(MAX_VAL))` 或 `O((N+Q)*sqrt(N)*log MAX_VAL)`。这已经超出了普通莫队的范畴，更倾向于根号分治的高级应用。对于GESP 9级，可以作为拓展思考题。

### 拓展挑战

1.  **带修改的莫队（回滚莫队/树上莫队）**：了解莫队算法的局限性，思考如何处理带修改的区间查询。
    *   **提示**：普通莫队无法直接处理修改操作。对于带修改的莫队（又称回滚莫队），需要增加一个时间维度，将查询和修改都按照时间戳排序。这需要更复杂的块划分和指针移动策略，通常会引入 `c` 指针（修改次数）并对块进行三维排序。这属于莫队的进阶内容。

### 答案（仅供参考，不包含在实际教学中）

**基础巩固1：区间异或和**
*   `current_xor_sum = 0;`
*   `add(x)`: `current_xor_sum ^= a[x];`
*   `del(x)`: `current_xor_sum ^= a[x];` (异或两次等于没变，所以移除也是异或一次)

**基础巩固2：区间和**
*   `current_sum = 0;`
*   `add(x)`: `current_sum += a[x];`
*   `del(x)`: `current_sum -= a[x];`

**能力提升1：区间内出现次数为 `K` 的数的个数**
*   `current_count_of_k_freq = 0;`
*   `add(x)`:
    *   `if (cnt[a[x]] == K) current_count_of_k_freq--;` // 原来是K次，现在要变K+1次，所以K次个数减1
    *   `cnt[a[x]]++;`
    *   `if (cnt[a[x]] == K) current_count_of_k_freq++;` // 现在是K次，所以K次个数加1
*   `del(x)`:
    *   `if (cnt[a[x]] == K) current_count_of_k_freq--;` // 原来是K次，现在要变K-1次，所以K次个数减1
    *   `cnt[a[x]]--;`
    *   `if (cnt[a[x]] == K) current_count_of_k_freq++;` // 现在是K次，所以K次个数加1
