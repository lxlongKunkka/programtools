## 深度知识点讲解

### 1. 为什么需要快速幂？

我们平时计算 $a^b$ 最直接的方法就是用一个循环，将 $a$ 乘以自身 $b-1$ 次。
例如，计算 $a^5$：
`ans = 1;`
`for (i = 0; i < 5; ++i) { ans = ans * a; }`
这样的时间复杂度是 $O(b)$。当 $b$ 达到 $10^{18}$ 级别时，这种方法会因为计算次数过多而超时 (Time Limit Exceeded)。我们需要一个更快的方法。

### 2. 快速幂的核心思想：分而治之 (Divide and Conquer)

快速幂（Binary Exponentiation）的核心思想是利用幂运算的性质，将大指数的幂运算分解为小指数的幂运算。

**核心性质：**
*   当指数 $b$ 是偶数时：$a^b = a^{b/2} \times a^{b/2}$
*   当指数 $b$ 是奇数时：$a^b = a^{b/2} \times a^{b/2} \times a$ (注意：这里的 $b/2$ 是整数除法，例如 $5/2=2$)

**生活类比：**
想象你急着要制作100份宣传单。
传统方法：你打印1份，再打印1份，重复100次。太慢了！
快速幂方法：
1.  你先打印50份。
2.  然后把这50份拿去复印，得到另外50份。这样你就有了100份！
如果需要99份呢？
1.  你先打印49份。
2.  把这49份复印，得到另外49份。现在有了98份。
3.  再单独打印1份。总共99份！
这个过程就像在做除法，每次把任务量减半，大大减少了操作次数。

**举个例子：计算 $3^{10}$**
1.  $3^{10} = (3^5) \times (3^5)$ (指数10是偶数)
2.  $3^5 = (3^2) \times (3^2) \times 3$ (指数5是奇数)
3.  $3^2 = (3^1) \times (3^1)$ (指数2是偶数)
4.  $3^1 = 3$ (指数1是奇数，无法再减半，基线情况)

从 $3^1$ 往回推：
*   $3^2 = 3^1 \times 3^1 = 3 \times 3 = 9$
*   $3^5 = 3^2 \times 3^2 \times 3 = 9 \times 9 \times 3 = 81 \times 3 = 243$
*   $3^{10} = 3^5 \times 3^5 = 243 \times 243 = 59049$

这个过程是不是比连续乘10次快多了？每次计算都把指数减半，所以计算次数大约是指数 $b$ 的 $\log_2 b$ 次。对于 $b=10^{18}$，$\log_2 10^{18} \approx 18 \log_2 10 \approx 18 \times 3.32 \approx 60$ 次乘法！这简直是质的飞跃！

### 3. 模幂运算 (Modular Exponentiation)

当 $a^b$ 的结果非常大，甚至超出 `long long` 的存储范围时，我们通常会要求结果对某个数 $m$ 取模，即计算 $a^b \pmod m$。

**模运算性质：**
一个非常重要的性质是：
$(A \times B) \pmod M = ((A \pmod M) \times (B \pmod M)) \pmod M$

**应用：**
这意味着在快速幂的每一步乘法操作中，我们都可以立即对结果取模 $m$，这样可以确保中间结果始终在 $m$ 的范围内，避免溢出。

**生活类比：**
你和你的朋友在玩一个游戏，每当你们的积分达到100分，就会自动清零并获得一个勋章。现在你要计算你连续赢了多少局，每局积分翻倍，最终你会有多少分（不考虑勋章，只看最终积分）。
如果你赢了5局，积分会是 $2^5 = 32$ 分。
如果你赢了8局，积分会是 $2^8 = 256$ 分。但是因为达到100分就清零，所以256分实际上是 56分 (256 % 100 = 56)。
如果你在每一步都取模，比如：
1局：2分
2局：$2 \times 2 = 4$分
3局：$4 \times 2 = 8$分
4局：$8 \times 2 = 16$分
5局：$16 \times 2 = 32$分
6局：$32 \times 2 = 64$分
7局：$64 \times 2 = 128 \pmod{100} = 28$分
8局：$28 \times 2 = 56$分
你看，结果是一样的，但中间的数字一直保持在100以内。

**常见误区：**
*   只在最终结果上取模：这会导致中间结果溢出，得到错误答案。
*   忘记对基数 $a$ 也取模：如果 $a$ 很大，在开始计算前就应该先 $a \pmod m$。因为 $(a \pmod m)^b \pmod m = a^b \pmod m$。

### 4. 快速幂的两种实现方式

#### (1) 递归实现

这是最直接的按照分治思想编写的方式。

```cpp
long long power_recursive(long long a, long long b, long long m) {
    if (b == 0) { // 任何数的0次方都是1
        return 1;
    }
    long long p = power_recursive(a, b / 2, m); // 先计算a的b/2次方
    if (b % 2 == 0) { // 如果b是偶数
        return (p * p) % m;
    } else { // 如果b是奇数
        return (((p * p) % m) * (a % m)) % m; // 注意对a也要取模
    }
}
```

#### (2) 迭代实现 (推荐，更高效，避免栈溢出)

迭代实现利用了指数 $b$ 的二进制表示。
任何一个整数 $b$ 都可以表示成二进制形式：
$b = c_k \cdot 2^k + c_{k-1} \cdot 2^{k-1} + \dots + c_1 \cdot 2^1 + c_0 \cdot 2^0$
其中 $c_i$ 要么是0，要么是1。
那么 $a^b = a^{(c_k \cdot 2^k + \dots + c_0 \cdot 2^0)} = a^{c_k \cdot 2^k} \times \dots \times a^{c_0 \cdot 2^0}$。
只有当 $c_i=1$ 时，对应的 $a^{2^i}$ 项才会被乘到结果中。
我们可以维护一个 `base` 变量，它从 $a$ 开始，每次自乘变成 $a^2, a^4, a^8, \dots$。
同时，我们检查指数 $b$ 的每一位。如果当前位是1，就把当前的 `base` 乘到结果 `ans` 中。

**举个例子：计算 $a^{13}$**
$13$ 的二进制表示是 $1101_2$，即 $13 = 1 \cdot 2^3 + 1 \cdot 2^2 + 0 \cdot 2^1 + 1 \cdot 2^0 = 8 + 4 + 1$。
所以 $a^{13} = a^8 \times a^4 \times a^1$。

**迭代过程：**
初始：`ans = 1`, `current_base = a`, `exponent = 13`
1.  **`exponent` 的最低位是 1 (13 & 1 == 1)？** 是。
    `ans = ans * current_base = 1 * a = a`
    `current_base = current_base * current_base = a * a = a^2`
    `exponent = exponent >> 1 = 13 / 2 = 6`
2.  **`exponent` 的最低位是 1 (6 & 1 == 1)？** 否。
    `ans` 不变。
    `current_base = current_base * current_base = a^2 * a^2 = a^4`
    `exponent = exponent >> 1 = 6 / 2 = 3`
3.  **`exponent` 的最低位是 1 (3 & 1 == 1)？** 是。
    `ans = ans * current_base = a * a^4 = a^5`
    `current_base = current_base * current_base = a^4 * a^4 = a^8`
    `exponent = exponent >> 1 = 3 / 2 = 1`
4.  **`exponent` 的最低位是 1 (1 & 1 == 1)？** 是。
    `ans = ans * current_base = a^5 * a^8 = a^{13}`
    `current_base = current_base * current_base = a^8 * a^8 = a^{16}`
    `exponent = exponent >> 1 = 1 / 2 = 0`
5.  **`exponent` 变为 0，循环结束。** 最终 `ans = a^{13}`。

**迭代实现代码：**

```cpp
long long power_iterative(long long a, long long b, long long m) {
    long long r = 1; // r (result) 存储最终结果
    a %= m;         // 先对基数a取模，防止a过大
    while (b > 0) {
        if (b % 2 == 1) { // 如果指数b的当前位是1 (b是奇数)
            r = (r * a) % m; // 将当前a的幂乘到结果中
        }
        a = (a * a) % m; // a自乘，更新为a^2, a^4, a^8...
        b /= 2;          // 指数b右移一位 (b = b / 2)
    }
    return r;
}
```

**对比两种实现：**
*   **递归实现**：代码直观，符合分治思想，但函数调用有开销，且当指数 $b$ 非常大时，可能导致栈溢出。
*   **迭代实现**：避免了函数调用的开销，更高效，不会有栈溢出问题，在实际应用中更常用。推荐大家掌握迭代实现。

### 5. 时间复杂度分析

无论是递归还是迭代，每次操作都将指数 $b$ 减半。从 $b$ 变为 $b/2$，再变为 $b/4$，直到 $1$。这个过程需要进行大约 $\log_2 b$ 次。每次操作中，我们都进行常数次（1或2次）乘法和取模运算。因此，快速幂算法的时间复杂度是 $O(\log b)$。

### 6. 边界条件与特殊情况

*   **$b = 0$：** 任何数的 $0$ 次方都是 $1$ (除了 $0^0$ 在数学上是未定义或通常约定为 $1$)。在编程中，我们通常处理 $a^0 = 1$ 的情况。
*   **$m = 1$：** 任何数对 $1$ 取模结果都是 $0$。
*   **$a = 0$：** 如果 $a=0$ 且 $b=0$，通常约定为 $1$。如果 $a=0$ 且 $b > 0$，结果是 $0$。在我们的 `power_iterative` 中，如果 `a` 为 `0`，`a %= m` 之后 `a` 依然是 `0`，所有乘法结果都是 `0`，最终 `r` 会是 `0` (除非 `b=0` 时 `r=1`)。这符合预期。

## 典型例题精讲

### 例题1: 计算 $A^B \pmod M$

**题目描述：**
给定三个正整数 $A, B, M$，求 $A^B \pmod M$ 的值。
其中 $1 \le A, M \le 10^9$， $0 \le B \le 10^{18}$。

**输入格式：**
一行，三个整数 $A, B, M$。

**输出格式：**
一个整数，表示 $A^B \pmod M$ 的结果。

**样例输入：**
`3 10 7`

**样例输出：**
`4`

**思路分析：**
1.  **问题特点：** 指数 $B$ 范围非常大 ($10^{18}$)，普通循环会超时。结果需要对 $M$ 取模。
2.  **核心算法：** 快速幂。因为 $B$ 很大，所以必须使用快速幂将时间复杂度从 $O(B)$ 降到 $O(\log B)$。
3.  **模运算：** 在每次乘法操作后，都对结果取模 $M$，以防止中间结果溢出 `long long` 范围。
4.  **数据类型：** $A, M$ 可以用 `long long` 存储。$B$ 必须用 `long long` 存储。中间结果 `r * a` 也可能超过 `long long` 最大值，所以每次乘法后都立即取模。

**代码实现 (迭代快速幂)：**

```cpp
#include <bits/stdc++.h>

// 快速幂函数，计算 (a^b) % m
long long fastPower(long long a, long long b, long long m) {
    long long r = 1; // r (result) 存储最终结果
    a %= m;         // 重要的第一步：先对基数a取模，防止a过大
    
    // 循环直到指数b变为0
    while (b > 0) {
        // 如果b的当前二进制位是1 (即b是奇数)
        // 那么需要将当前的a的幂乘到结果r中
        if (b % 2 == 1) { 
            r = (r * a) % m; // 每次乘法后都立即取模
        }
        
        // a自乘，更新为a^2, a^4, a^8, ...
        // 这里的a代表的是a^(2^k)
        a = (a * a) % m; // 每次乘法后都立即取模
        
        // 指数b右移一位 (相当于b = b / 2)
        b /= 2; 
    }
    return r; // 返回最终结果
}

int main() {
    long long a, b, m;
    std::cin >> a >> b >> m;
    
    long long result = fastPower(a, b, m);
    std::cout << result << std::endl;
    
    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** $O(\log b)$。因为每次循环 $b$ 都会减半，所以循环次数是 $\log_2 b$。每次循环内部是常数次乘法和取模操作。
*   **空间复杂度：** $O(1)$。只使用了常数个变量。

### 例题2: 求一个数 $A$ 的 $B$ 次方结果的后 $K$ 位数字

**题目描述：**
给定三个正整数 $A, B, K$，求 $A^B$ 的结果的后 $K$ 位数字。
其中 $1 \le A \le 10^9$， $0 \le B \le 10^{18}$， $1 \le K \le 9$。

**输入格式：**
一行，三个整数 $A, B, K$。

**输出格式：**
一个整数，表示 $A^B$ 的后 $K$ 位数字。如果结果不足 $K$ 位，则直接输出结果。

**样例输入：**
`123 4 2`

**样例输出：**
`81` (因为 $123^4 = 2286769281$，后两位是81)

**思路分析：**
1.  **问题转换：** “求一个数 $X$ 的后 $K$ 位数字” 实际上就是求 $X \pmod{10^K}$。
2.  **模数 $M$：** 根据题目要求，我们需要对 $10^K$ 取模。我们可以预先计算出 $M = 10^K$。例如，如果 $K=2$，则 $M=100$；如果 $K=3$，则 $M=1000$。
3.  **核心算法：** 依然是快速幂。指数 $B$ 很大，所以需要快速幂。
4.  **特殊情况：** 如果最终结果小于 $10^K$ (即结果本身不足 $K$ 位)，直接输出结果即可。但由于我们是取模操作，如果结果是 $081$ 这样的，取模后会得到 $81$，符合题目要求。

**代码实现：**

```cpp
#include <bits/stdc++.h>

// 快速幂函数，计算 (a^b) % m
long long fastPower(long long a, long long b, long long m) {
    long long r = 1; // r (result) 存储最终结果
    a %= m;         // 先对基数a取模
    
    while (b > 0) {
        if (b % 2 == 1) { 
            r = (r * a) % m; 
        }
        a = (a * a) % m; 
        b /= 2;          
    }
    return r; 
}

int main() {
    long long a, b, k;
    std::cin >> a >> b >> k;
    
    long long m = 1;
    for (int i = 0; i < k; ++i) { // 计算模数 10^K
        m *= 10;
    }
    
    long long result = fastPower(a, b, m);
    std::cout << result << std::endl;
    
    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 计算 $M$ 是 $O(K)$，快速幂是 $O(\log B)$。总时间复杂度为 $O(K + \log B)$。因为 $K$ 最大为9，可以看作常数。
*   **空间复杂度：** $O(1)$。

## 代码实现模板

这里提供一个常用的迭代快速幂模板，带有详细注释，方便大家直接使用。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，方便竞赛使用

// 定义一个函数，用于计算 (a^b) % m 的值
// a: 基数 (底数)
// b: 指数
// m: 模数
// 注意：a, b, m 都使用 long long 类型，以应对大数值
long long fastPower(long long a, long long b, long long m) {
    long long r = 1; // 初始化结果 r 为 1。任何数的0次方都是1。
                     // r 将在循环中累积乘法结果。
    
    // 关键一步：在计算之前，先将基数 a 对模数 m 取模。
    // 这样可以确保 a 始终在 [0, m-1] 范围内，避免 a 过大导致后续乘法溢出。
    // 根据模运算性质：(A * B) % M = ((A % M) * (B % M)) % M
    // 那么 (A^B) % M = ((A % M)^B) % M
    a %= m; 
    
    // 循环，直到指数 b 变为 0
    // 这个循环的次数大约是 log2(b)，效率非常高
    while (b > 0) {
        // 检查指数 b 的当前最低位是否为 1 (即 b 是否为奇数)
        // 如果 b 是奇数，说明当前这一项 a^(2^k) 需要被乘到结果 r 中
        if (b % 2 == 1) { 
            // 将当前的 a (它代表 a^(2^k)) 乘到结果 r 中
            // 并且每次乘法后都立即对 m 取模，防止 r 或 r*a 溢出
            r = (r * a) % m; 
        }
        
        // 将 a 自乘，更新为 a^2, a^4, a^8, ...
        // 这里的 a 实际上代表的是 a^(2^0), a^(2^1), a^(2^2), ...
        // 每次循环，a 的指数都翻倍
        a = (a * a) % m; 
        
        // 将指数 b 右移一位 (相当于 b = b / 2)
        // 这样可以检查 b 的下一位二进制位
        b /= 2;          
    }
    
    // 循环结束后，r 中存储的就是 (a^b) % m 的最终结果
    return r; 
}

int main() {
    // 示例用法
    long long base, exponent, modulus;
    
    std::cout << "请输入基数 A, 指数 B, 模数 M (例如: 3 10 7): ";
    std::cin >> base >> exponent >> modulus;
    
    long long result = fastPower(base, exponent, modulus);
    
    std::cout << base << "^" << exponent << " % " << modulus << " = " << result << std::endl;
    
    return 0;
}
```

**编码风格说明：**
*   使用 `long long` 确保处理大数值。
*   `#include <bits/stdc++.h>` 是竞赛中常用的万能头文件。
*   函数参数命名为 `a, b, m` 符合题目中的惯例。
*   变量命名简洁，如 `r` (result)。
*   代码采用 Allman 风格大括号。
*   注释详细，解释了每一步的“为什么”和“怎么做”。

1.  **思考题：为什么快速幂比普通循环快？它的时间复杂度具体是多少？**
    *   **引导：** 回想我们计算 $3^{10}$ 的例子，每次指数是怎么变化的？如果指数是 $B$，要变到 $1$，需要几次减半？
    *   **答案：** 快速幂每次将指数减半，所以计算次数是 $B$ 的二进制位数，大约是 $\log_2 B$ 次。而普通循环需要 $B$ 次。例如 $B=10^{18}$ 时，$\log_2 B \approx 60$，效率提升巨大。

2.  **讨论环节：模运算在快速幂中有什么作用？如果忘记在中间步骤取模会发生什么？**
    *   **引导：** 想象一下 $10^{18}$ 这么大的数，计算机能直接存下吗？
    *   **答案：** 模运算的主要作用是防止中间结果溢出 `long long` 的最大值，导致计算错误。如果忘记在中间步骤取模，当乘积太大时，会超出数据类型范围，变成一个错误的小负数或正数，最终结果就会是错的。

3.  **小组任务：请大家分别使用递归思路和迭代思路，手动模拟计算 $5^{12} \pmod{13}$ 的过程。**
    *   **要求：** 每个小组选派代表上台板书，展示计算步骤和中间结果。
    *   **参考答案：**
        *   **递归 $5^{12} \pmod{13}$：**
            *   `power(5, 12, 13)`
            *   `p = power(5, 6, 13)`
                *   `p_sub = power(5, 3, 13)`
                    *   `p_sub_sub = power(5, 1, 13)`
                        *   `p_sub_sub_sub = power(5, 0, 13) = 1`
                        *   `p_sub_sub = (((1*1)%13) * (5%13))%13 = 5`
                    *   `p_sub = (((5*5)%13) * (5%13))%13 = (25%13 * 5)%13 = (12 * 5)%13 = 60%13 = 8`
                *   `p = (8*8)%13 = 64%13 = 12`
            *   `return (12*12)%13 = 144%13 = 1`
        *   **迭代 $5^{12} \pmod{13}$：**
            *   初始：`r=1`, `a=5%13=5`, `b=12`
            *   循环1: `b=12`(偶数)。`r`不变。`a=(5*5)%13=12`。`b=6`。
            *   循环2: `b=6`(偶数)。`r`不变。`a=(12*12)%13=144%13=1`。`b=3`。
            *   循环3: `b=3`(奇数)。`r=(1*1)%13=1`。`a=(1*1)%13=1`。`b=1`。
            *   循环4: `b=1`(奇数)。`r=(1*1)%13=1`。`a=(1*1)%13=1`。`b=0`。
            *   循环结束。返回 `r=1`。

4.  **即时练习：用刚才学到的迭代快速幂方法，快速计算 $7^{100} \pmod{1000}$。**
    *   **要求：** 同学们在纸上写出关键步骤或在电脑上实现。
    *   **提示：** $m=1000$ 表示求结果的后三位。
    *   **答案：** $7^{100} \pmod{1000} = 1$。

## 分层练习题目

### 基础巩固

1.  **题目：** 计算 $2^{30} \pmod{1000000007}$。
    *   **输入：** 不需要输入，直接计算。
    *   **输出：** 一个整数。
    *   **提示：** $1000000007$ 是一个质数，常用于模数。
    *   **参考答案：** `1073741824 % 1000000007 = 73741817`

2.  **题目：** 给定 $A, B, M$，求 $(A^B) \pmod M$。$A=10, B=5, M=3$。
    *   **输入：** `10 5 3`
    *   **输出：** `1`
    *   **提示：** $10 \pmod 3 = 1$，所以 $(10^5) \pmod 3 = (1^5) \pmod 3 = 1$。快速幂会自然处理这个过程。

### 能力提升

1.  **题目：** **[GESP 七级真题改编]** 小明喜欢数字 $7$。他想知道 $7^N$ 的最后一位数字是什么。请你编写程序，帮助小明计算 $7^N$ 的最后一位数字。
    *   **输入格式：** 一行，一个整数 $N$ ($0 \le N \le 10^{18}$)。
    *   **输出格式：** 一个整数，表示 $7^N$ 的最后一位数字。
    *   **样例输入：** `10`
    *   **样例输出：** `9` (因为 $7^{10} = 282475249$，最后一位是9)
    *   **提示：** 求最后一位数字，相当于对 $10$ 取模。需要注意 $N=0$ 的情况。
    *   **参考答案：**
        *   对于 $N=0$，结果是 $1$。
        *   对于 $N>0$，使用快速幂计算 $7^N \pmod{10}$。
        *   $7^1 \pmod{10} = 7$
        *   $7^2 \pmod{10} = 49 \pmod{10} = 9$
        *   $7^3 \pmod{10} = 9 \times 7 \pmod{10} = 63 \pmod{10} = 3$
        *   $7^4 \pmod{10} = 3 \times 7 \pmod{10} = 21 \pmod{10} = 1$
        *   $7^5 \pmod{10} = 1 \times 7 \pmod{10} = 7$ (循环出现)
        *   周期是4。$N=10$，指数 $10 \pmod 4 = 2$。所以结果是 $7^2$ 的最后一位，即 $9$。
        *   使用快速幂直接计算 $fastPower(7, N, 10)$ 即可。

2.  **题目：** 计算 $(A^B + C^D) \pmod M$。
    *   **输入格式：** 一行，五个整数 $A, B, C, D, M$。
        $0 \le A, C \le 10^9$, $0 \le B, D \le 10^{18}$, $1 \le M \le 10^9$。
    *   **输出格式：** 一个整数，表示计算结果。
    *   **样例输入：** `2 3 3 2 5`
    *   **样例输出：** `2` (因为 $(2^3 + 3^2) \pmod 5 = (8+9) \pmod 5 = 17 \pmod 5 = 2$)
    *   **提示：** 分别计算 $A^B \pmod M$ 和 $C^D \pmod M$，然后将两个结果相加再对 $M$ 取模。注意相加后的结果也可能大于 $M$。
    *   **参考答案：** `(fastPower(A, B, M) + fastPower(C, D, M)) % M`。

### 拓展挑战

1.  **题目：** **[费马小定理与快速幂]** 给定一个质数 $P$ 和两个整数 $A, B$，计算 $A^B \pmod P$。
    *   **特殊要求：** 如果 $B \ge P-1$，根据费马小定理， $A^B \equiv A^{B \pmod{P-1}} \pmod P$ (当 $A$ 不是 $P$ 的倍数时)。如果 $A$ 是 $P$ 的倍数，则 $A^B \equiv 0 \pmod P$ (除非 $B=0$, 此时 $A^0=1$)。
    *   **输入格式：** 一行，三个整数 $A, B, P$。
        $0 \le A \le 10^9$, $0 \le B \le 10^{18}$, $2 \le P \le 10^9$ (且 $P$ 是质数)。
    *   **输出格式：** 一个整数，表示结果。
    *   **样例输入：** `2 10 3`
    *   **样例输出：** `1` (因为 $2^{10} \pmod 3 = (2^1)^10 \pmod 3 = (-1)^{10} \pmod 3 = 1$)
    *   **提示：** 费马小定理指出，如果 $P$ 是一个质数，那么对于任意整数 $A$，有 $A^P \equiv A \pmod P$。进一步推导，如果 $A$ 不是 $P$ 的倍数，那么 $A^{P-1} \equiv 1 \pmod P$。这意味着指数可以对 $P-1$ 取模。但要特别注意 $A$ 是 $P$ 的倍数的情况，以及 $B=0$ 的情况。
    *   **参考答案：**
        ```cpp
        long long A, B, P;
        std::cin >> A >> B >> P;
        if (A % P == 0) { // 如果A是P的倍数
            if (B == 0) std::cout << 1 << std::endl; // 0^0 通常为1
            else std::cout << 0 << std::endl; // 0^B (B>0) 为0
        } else {
            // 根据费马小定理，当P是质数且A不是P的倍数时，
            // A^B % P = A^(B % (P-1)) % P
            // 但如果 B % (P-1) 结果为0，实际上表示指数是 P-1 的倍数，
            // 此时 A^(P-1) % P = 1。
            // 为了处理 B % (P-1) = 0 的情况，我们通常让 B 的指数模 P-1，
            // 如果模出来是0，就用 P-1 作为指数。
            // 更简单的处理是 B %= (P-1)；如果B是0，则指数是P-1。
            // 但这里直接用 B %= (P-1) 可能会有问题，比如 B=0 导致 B %= (P-1) = 0，
            // 此时 A^0 = 1。
            // 正确做法是，如果B是0，直接返回1。否则，如果B不是0，对B取模P-1。
            // 甚至更稳妥的是，当 B >= P-1 时，让 B = B % (P-1) + (P-1) 
            // 确保指数不为0，或者 B %= (P-1)；若结果为0，则用 P-1。
            // 但对于竞赛，直接 B %= (P-1) 是常见的优化，只要处理好 B=0 和 B%(P-1)=0 的情况。
            // 最简单且安全的做法是：
            // 如果 B == 0，结果为 1。
            // 否则，如果 B % (P-1) == 0，那么指数实际上是 P-1 (或其倍数)，根据费马小定理结果是 1。
            // 否则，指数 B = B % (P-1)。
            // 但通常，直接 B %= (P-1) 在竞赛中是可行的，因为 B=0 的情况 fastPower 已经处理。
            // 唯一需要注意的是，如果 B % (P-1) == 0，且 B > 0，实际上指数应该是 P-1，而不是 0。
            // 所以，如果 B != 0 且 B % (P-1) == 0，那么修正 B 为 P-1。
            if (B != 0) {
                B %= (P - 1);
                if (B == 0) B = P - 1; // 修正指数，确保不是0，而是P-1
            }
            std::cout << fastPower(A, B, P) << std::endl;
        }
        ```

### 答案区

**基础巩固**
1.  `73741817`
2.  `1`

**能力提升**
1.  `9`
2.  `2`

**拓展挑战**
1.  对于 `2 10 3`：
    *   $A=2, B=10, P=3$
    *   $A \% P = 2 \% 3 = 2 \ne 0$
    *   $B \pmod{P-1} = 10 \pmod{2} = 0$
    *   由于 $B \ne 0$ 且 $B \pmod{P-1} = 0$，修正指数 $B$ 为 $P-1=2$。
    *   计算 $fastPower(2, 2, 3) = (2^2) \pmod 3 = 4 \pmod 3 = 1$。
    *   输出 `1`。

    对于 `5 0 7`：
    *   $A=5, B=0, P=7$
    *   $A \% P = 5 \% 7 = 5 \ne 0$
    *   $B=0$，直接返回 $1$。
    *   输出 `1`。

    对于 `7 123 7`：
    *   $A=7, B=123, P=7$
    *   $A \% P = 7 \% 7 = 0$
    *   $B=123 > 0$，输出 `0`。
