## 深度知识点讲解

### 1. 子集与组合：核心概念辨析

*   **集合 (Set)**：一堆互不相同的东西的合集。比如 `{1, 2, 3}` 是一个集合。
*   **子集 (Subset)**：从一个集合中取出部分或全部元素组成的新集合。顺序不重要。
    *   比如，集合 `{1, 2, 3}` 的子集有：
        *   空集 `{}` (什么都不选也是一种选择哦！)
        *   只选一个元素：`{1}`, `{2}`, `{3}`
        *   选两个元素：`{1, 2}`, `{1, 3}`, `{2, 3}`
        *   选三个元素：`{1, 2, 3}`
    *   总共有 8 个子集。是不是发现 2 的 3 次方等于 8？没错，一个有 N 个元素的集合，总共有 $2^N$ 个子集。
*   **组合 (Combination)**：从一个集合中取出**指定数量 K** 的元素组成的新集合。顺序不重要。
    *   比如，从集合 `{1, 2, 3, 4}` 中选择 2 个元素组成的组合有：
        *   `{1, 2}`, `{1, 3}`, `{1, 4}`
        *   `{2, 3}`, `{2, 4}`
        *   `{3, 4}`
    *   总共有 6 个组合。这就是数学上说的“组合数”，记作 $C(N, K)$ 或 $\binom{N}{K}$。

**核心区别**：子集是任意数量的元素（包括0个和全部），组合是固定数量 K 的元素。

### 2. DFS与回溯法：如何“做选择”和“撤销选择”

DFS（深度优先搜索）就像一个侦探，它沿着一条路一直走到底，发现走不通了，就退回来，换另一条路继续走。这个“退回来”就是**回溯 (Backtracking)**。

在子集/组合生成问题中，DFS的“路”就是我们做的“选择”。对于集合中的每一个元素，我们都面临一个“二选一”的决策：**选它**，还是**不选它**？

*   **生活类比：点菜系统**
    *   假设你面前有一份菜单，上面有 N 道菜。你现在要决定点哪些菜。
    *   你从第一道菜开始看：
        *   **决策1：点这道菜**。把它加入你的点菜清单。然后，你去看第二道菜，继续做决策。
        *   **决策2：不点这道菜**。不把它加入清单。然后，你去看第二道菜，继续做决策。
    *   当你把所有 N 道菜都看完了，你就形成了一个完整的点菜清单。这是一个可能的“子集”！
    *   然后，为了找到所有可能的清单，你得“回溯”：假设你点完所有菜，记录下清单后，你会“撤销”最后一道菜的选择，尝试另一种可能。

*   **DFS 的状态与决策树**
    1.  **状态 (State)**：在DFS的每一步，我们需要知道：
        *   我们正在考虑第几个元素（比如 `idx`）。
        *   目前已经做了哪些选择（比如 `当前选中的元素列表`）。
    2.  **决策树 (Decision Tree)**：想象一棵树，每个节点代表一个决策点。
        *   根节点：还没开始做任何选择。
        *   第一层节点：对第一个元素做选择（选或不选）。
        *   第二层节点：对第二个元素做选择（选或不选），以此类推。
        *   叶子节点：所有元素都考虑完毕，得到一个完整的子集/组合。

### 3. 子集生成：DFS实现详解

**核心思想**：对于集合中的每个元素 `a[idx]`，我们都有两种选择：
1.  **选择 `a[idx]`**：把它加入当前子集，然后递归考虑下一个元素 `a[idx+1]`。
2.  **不选择 `a[idx]`**：不把它加入当前子集，直接递归考虑下一个元素 `a[idx+1]`。

**DFS函数设计**：
`void dfs_subset(int idx, int current_len)`
*   `idx`: 当前正在考虑的元素在原始数组 `a` 中的下标。
*   `current_len`: 当前已经选入了多少个元素到 `b` 数组中。

**全局变量**：
*   `int n`: 原始集合的元素个数。
*   `int a[MAXN]`: 存储原始集合的元素。
*   `int b[MAXN]`: 存储当前正在构建的子集。

**伪代码逻辑**：

```
dfs_subset(idx, current_len):
    如果 idx == n (所有元素都考虑完了):
        打印 b 数组中从 0 到 current_len-1 的元素 (这就是一个子集)
        返回

    // 决策1: 不选择 a[idx]
    dfs_subset(idx + 1, current_len) // 直接考虑下一个元素，当前子集不变

    // 决策2: 选择 a[idx]
    b[current_len] = a[idx] // 把 a[idx] 加入当前子集
    dfs_subset(idx + 1, current_len + 1) // 考虑下一个元素，当前子集长度增加
```

**为什么这种写法能实现回溯？**
`b[current_len] = a[idx]` 这一行，我们是把 `a[idx]` 放到了 `b` 数组的 `current_len` 位置。当 `dfs_subset(idx + 1, current_len + 1)` 调用返回时，`current_len` 的值会恢复到调用前的值。下一次，如果 `current_len` 没变，`b[current_len]` 位置就会被新的元素覆盖，或者根本不会被使用。这种通过参数 `current_len` 来控制数组写入位置的方式，巧妙地实现了“撤销选择”的效果，因为每次递归调用都会拥有它自己独立的 `current_len` 视角。

### 4. 组合枚举：DFS实现详解 (固定大小 K)

组合枚举是子集生成的一个特殊情况：我们只关心那些长度恰好为 `k` 的子集。

**核心思想**：和子集生成类似，但需要增加两个条件：
1.  **计数**：我们必须记录当前已经选择了多少个元素。
2.  **剪枝 (Pruning)**：如果剩余的元素数量不足以凑够 `k` 个，或者已经选的元素数量超过 `k` 个，就没必要继续搜索了。

**DFS函数设计**：
`void dfs_combination(int idx, int current_count)`
*   `idx`: 当前正在考虑的元素在原始数组 `a` 中的下标。
*   `current_count`: 当前已经选入了多少个元素到 `b` 数组中。

**新增全局变量**：
*   `int k`: 目标组合的大小。

**伪代码逻辑**：

```
dfs_combination(idx, current_count):
    如果 current_count == k (已经选够了 k 个元素):
        打印 b 数组中从 0 到 k-1 的元素 (这就是一个组合)
        返回

    如果 idx == n (所有元素都考虑完了，但还没选够 k 个):
        返回 (这条路径无效)

    // 剪枝优化 (非常重要!)
    // 如果“剩下还没有考虑的元素数量” + “已经选了的元素数量” < k，
    // 那么即使把剩下的都选上，也凑不够 k 个了，直接返回。
    // 剩余元素数量: n - idx
    // 还需要选的元素数量: k - current_count
    如果 (n - idx) < (k - current_count):
        返回

    // 决策1: 不选择 a[idx]
    dfs_combination(idx + 1, current_count) // 直接考虑下一个元素

    // 决策2: 选择 a[idx]
    b[current_count] = a[idx] // 把 a[idx] 加入当前组合
    dfs_combination(idx + 1, current_count + 1) // 考虑下一个元素，已选数量增加
```
**注意**：在组合问题中，为了避免重复（例如 `{1,2}` 和 `{2,1}` 算同一种组合），我们通常会要求选择的元素是**递增**的。这意味着，如果我们选择了 `a[idx]`，那么下一个选择的元素必须是 `a[idx+1]` 或其之后。我们上面的 `dfs_combination(idx + 1, ...)` 结构天然保证了这一点，因为我们总是从 `idx` 向后考虑，不会回头。

### 5. 时间复杂度分析

*   **子集生成**:
    *   对于每个元素，都有“选”和“不选”两种决策。如果有 N 个元素，总共有 $2 \times 2 \times \dots \times 2$ (N 次) = $2^N$ 种不同的路径（即 $2^N$ 个子集）。
    *   每个子集生成后，需要遍历并打印，这通常需要 O(N) 的时间。
    *   因此，总时间复杂度约为 $O(N \cdot 2^N)$。
*   **组合枚举 (C(N, K))**:
    *   我们需要生成 $C(N, K)$ 个组合。
    *   每个组合生成后，需要遍历并打印，这通常需要 O(K) 的时间。
    *   因此，总时间复杂度约为 $O(K \cdot C(N, K))$。
    *   $C(N, K)$ 在 N 比较大时，是一个非常大的数字。例如 $C(20, 10)$ 已经超过 18 万。

**小结**：DFS和回溯法在处理排列组合问题时非常强大，但它们的效率往往是指数级的，所以通常适用于 N 比较小的情况（比如 N 小于 20-25）。

### 例题 1：生成所有子集

**题目描述**：给定一个包含 N 个不同整数的数组 `a`，请输出它的所有子集。

**输入样例**：
`3`
`1 2 3`

**输出样例**（顺序可能不同）：
```
{}
1
2
3
1 2
1 3
2 3
1 2 3
```

**思路分析**：
这正是我们前面讲解的子集生成问题的直接应用。对于数组 `a` 中的每一个元素 `a[i]`，我们都有两种选择：
1.  **选择 `a[i]`**：将其放入当前子集 `b` 中，然后继续处理 `a[i+1]`。
2.  **不选择 `a[i]`**：不将其放入当前子集 `b` 中，直接处理 `a[i+1]`。

当所有元素都处理完毕（即 `idx == n`）时，当前 `b` 中存储的就是一个完整的子集，将其打印出来。

**代码实现**：

```cpp
#include <iostream>

const int MAXN = 25; // N 最大为 20-25 左右

int n;          // 原始集合的元素个数
int a[MAXN];    // 存储原始集合的元素
int b[MAXN];    // 存储当前正在构建的子集
int len;        // 当前子集的长度

// idx: 当前正在考虑 a 数组中的第 idx 个元素
// len: b 数组中已经有多少个元素了
void dfs_subset(int idx) {
    // 递归终止条件：所有元素都考虑完了
    if (idx == n) {
        // 打印当前子集
        if (len == 0) {
            std::cout << "{}" << std::endl; // 空集
        } else {
            for (int i = 0; i < len; ++i) {
                std::cout << b[i] << (i == len - 1 ? "" : " ");
            }
            std::cout << std::endl;
        }
        return;
    }

    // 决策1: 不选择 a[idx]
    dfs_subset(idx + 1); // 直接考虑下一个元素，当前子集 b 不变

    // 决策2: 选择 a[idx]
    b[len] = a[idx];     // 将 a[idx] 加入当前子集
    len++;               // 子集长度增加
    dfs_subset(idx + 1); // 考虑下一个元素
    len--;               // 回溯：撤销选择，恢复子集长度，以便进行其他分支的搜索
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n;
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    len = 0; // 初始化当前子集长度为 0
    dfs_subset(0); // 从第一个元素 (下标 0) 开始搜索

    return 0;
}
```

**复杂度分析**：
*   时间复杂度：$O(N \cdot 2^N)$。因为有 $2^N$ 个子集，每个子集最多有 N 个元素，打印一个子集需要 O(N) 时间。
*   空间复杂度：$O(N)$。主要由递归栈的深度（N）和存储当前子集的数组 `b` (N) 决定。

### 例题 2：生成指定大小的组合

**题目描述**：给定一个包含 N 个不同整数的数组 `a`，和一个整数 `k`，请输出所有从 `a` 中选出 `k` 个元素组成的组合。

**输入样例**：
`4 2`
`1 2 3 4`

**输出样例**（顺序可能不同）：
```
1 2
1 3
1 4
2 3
2 4
3 4
```

**思路分析**：
这是组合枚举问题的直接应用。在子集生成的基础上，我们需要增加对已选元素数量 `count` 的判断。
1.  **基本决策**：同样对于 `a[idx]`，有“选”和“不选”两种决策。
2.  **终止条件**：
    *   如果 `count == k`，说明我们已经选够了 `k` 个元素，当前 `b` 中的就是一个合法的组合，打印并返回。
    *   如果 `idx == n` 并且 `count < k`，说明所有元素都考虑完了，但没选够 `k` 个，这条路径无效，返回。
3.  **剪枝优化**：
    *   如果当前已经选了 `count` 个元素，还需要 `k - count` 个元素。
    *   如果剩余未考虑的元素数量 `(n - idx)` 小于 `k - count`，那么即使把剩下的所有元素都选上，也无法凑够 `k` 个，此时可以直接返回，避免不必要的搜索。

**代码实现**：

```cpp
#include <iostream>

const int MAXN = 25; // N 最大为 20-25 左右

int n;          // 原始集合的元素个数
int k;          // 目标组合的大小
int a[MAXN];    // 存储原始集合的元素
int b[MAXN];    // 存储当前正在构建的组合

// idx: 当前正在考虑 a 数组中的第 idx 个元素
// current_count: b 数组中已经有多少个元素了
void dfs_combination(int idx, int current_count) {
    // 递归终止条件1：已经选够了 k 个元素
    if (current_count == k) {
        // 打印当前组合
        for (int i = 0; i < k; ++i) {
            std::cout << b[i] << (i == k - 1 ? "" : " ");
        }
        std::cout << std::endl;
        return;
    }

    // 递归终止条件2：所有元素都考虑完了，但还没选够 k 个
    if (idx == n) {
        return;
    }

    // 剪枝优化：如果“剩余未考虑的元素数量”不足以凑够“还需要选择的元素数量”，就直接返回
    // 剩余未考虑的元素数量: n - idx
    // 还需要选择的元素数量: k - current_count
    if ((n - idx) < (k - current_count)) {
        return;
    }

    // 决策1: 不选择 a[idx]
    dfs_combination(idx + 1, current_count); // 直接考虑下一个元素，当前组合 b 不变

    // 决策2: 选择 a[idx]
    b[current_count] = a[idx];     // 将 a[idx] 加入当前组合
    dfs_combination(idx + 1, current_count + 1); // 考虑下一个元素，已选数量增加
    // 注意：这里不需要显式地 '撤销' b[current_count] 的赋值，
    // 因为 current_count 是函数参数，在递归返回时会自动恢复到上层调用的值，
    // 下一次写入 b[current_count] 会覆盖掉当前的值。
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> k;
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    // 初始调用：从第一个元素 (下标 0) 开始，已选 0 个元素
    dfs_combination(0, 0); 

    return 0;
}
```

**复杂度分析**：
*   时间复杂度：$O(K \cdot C(N, K))$。有 $C(N, K)$ 个组合，每个组合打印需要 O(K) 时间。剪枝优化会减少实际搜索的路径，但最坏情况下复杂度依然是这个。
*   空间复杂度：$O(N)$。主要由递归栈的深度（N）和存储当前组合的数组 `b` (K) 决定。

## 代码实现模板

为了方便同学们快速上手，这里提供一个标准的组合枚举DFS模板。子集生成可以看作是 `k` 从 0 到 `n` 的组合枚举的总和。

```cpp
#include <iostream>
// #include <bits/stdc++.h> // GESP 7 允许使用，但为了教学清晰，我们只包含必要的头文件

const int MAXN = 25; // 定义数组的最大容量，通常根据题目 N 的范围来设定

int n; // 原始集合的元素数量
int k; // 目标组合的大小
int a[MAXN]; // 存储原始集合的元素
int b[MAXN]; // 存储当前正在构建的组合或子集

// DFS 函数用于生成组合
// @param idx: 当前我们正在考虑原始数组 a 中的第几个元素 (下标)
// @param current_count: 当前 b 数组中已经选择了多少个元素
void dfs_combination_template(int idx, int current_count) {
    // === 递归终止条件 ===
    // 1. 已经选够了 k 个元素，找到了一个合法的组合
    if (current_count == k) {
        // 打印或处理当前组合
        for (int i = 0; i < k; ++i) {
            std::cout << b[i] << (i == k - 1 ? "" : " ");
        }
        std::cout << std::endl;
        return; // 结束当前分支的搜索
    }

    // 2. 所有元素都考虑完了，但还没选够 k 个（或者 idx 已经超出了 n 的范围）
    if (idx == n) {
        return; // 结束当前分支的搜索，这条路径无法形成 k 个元素的组合
    }

    // === 剪枝优化 (可选但强烈推荐) ===
    // 如果“剩余未考虑的元素数量”不足以凑够“还需要选择的元素数量”，就直接返回
    // 剩余未考虑的元素数量: n - idx
    // 还需要选择的元素数量: k - current_count
    if ((n - idx) < (k - current_count)) {
        return; // 剪枝，提前结束不必要的搜索
    }

    // === 核心决策 ===

    // 决策1: 不选择当前元素 a[idx]
    // 直接跳过 a[idx]，考虑下一个元素 a[idx+1]，已选数量 current_count 不变
    dfs_combination_template(idx + 1, current_count);

    // 决策2: 选择当前元素 a[idx]
    // 将 a[idx] 加入当前组合 b
    b[current_count] = a[idx];
    // 考虑下一个元素 a[idx+1]，已选数量 current_count 增加 1
    dfs_combination_template(idx + 1, current_count + 1);

    // 注意：这里不需要显式地“回溯” b[current_count] 的值，
    // 因为 current_count 是函数参数，在递归返回时其值会自动恢复到上层调用时的状态。
    // 这种通过下标控制写入位置的方式，天然实现了回溯的效果。
}


// DFS 函数用于生成所有子集 (略有不同，不需要 k)
// @param idx: 当前我们正在考虑原始数组 a 中的第几个元素 (下标)
// @param current_len: 当前 b 数组中已经选择了多少个元素
void dfs_subset_template(int idx, int current_len) {
    // === 递归终止条件 ===
    // 所有元素都考虑完了
    if (idx == n) {
        // 打印或处理当前子集
        if (current_len == 0) {
            std::cout << "{}" << std::endl; // 空集
        } else {
            for (int i = 0; i < current_len; ++i) {
                std::cout << b[i] << (i == current_len - 1 ? "" : " ");
            }
            std::cout << std::endl;
        }
        return;
    }

    // === 核心决策 ===

    // 决策1: 不选择当前元素 a[idx]
    // 直接跳过 a[idx]，考虑下一个元素 a[idx+1]，已选长度 current_len 不变
    dfs_subset_template(idx + 1, current_len);

    // 决策2: 选择当前元素 a[idx]
    // 将 a[idx] 加入当前子集 b
    b[current_len] = a[idx];
    // 考虑下一个元素 a[idx+1]，已选长度 current_len 增加 1
    dfs_subset_template(idx + 1, current_len + 1);
    // 回溯：这里也不是显式地撤销 b[current_len] 的值，
    // 而是通过 current_len 参数在递归返回时自动恢复到上层状态。
    // 如果用全局变量 len，那么需要 `len--` 来撤销，就像例题1中那样。
    // 模板中，我们更倾向于使用参数传递来避免全局变量副作用，更清晰。
}


int main() {
    std::ios_base::sync_with_stdio(false); // 优化 C++ 标准输入输出速度
    std::cin.tie(NULL);                   // 解除 cin 和 cout 的绑定

    // 示例：生成组合
    std::cout << "--- 生成组合 ---" << std::endl;
    n = 4; // 集合大小
    k = 2; // 组合大小
    a[0] = 1; a[1] = 2; a[2] = 3; a[3] = 4; // 原始集合 {1, 2, 3, 4}
    dfs_combination_template(0, 0); // 从下标 0 开始，当前已选 0 个

    std::cout << "\n--- 生成子集 ---" << std::endl;
    n = 3; // 集合大小
    a[0] = 1; a[1] = 2; a[2] = 3; // 原始集合 {1, 2, 3}
    dfs_subset_template(0, 0); // 从下标 0 开始，当前已选 0 个

    return 0;
}
```

## 课堂互动

1.  **思考题：为什么叫“回溯”？**
    *   提问：同学们，我们今天讲的算法叫做“DFS回溯法”。“深度优先搜索”我们理解了，就是一条路走到黑。那“回溯”体现在哪里呢？它有什么作用？
    *   提示：想想我们刚才点菜的例子，当你发现一个菜品组合不好吃，你会怎么做？或者，当你完成了一个点菜清单，准备去尝试下一个清单时，你之前点的菜是会一直留在桌上，还是会“撤销”掉，重新开始？
    *   **预期回答**：回溯就是“撤销”之前的选择，回到上一步，尝试其他的选择。它让我们可以探索所有不同的路径，而不是只沿着一条路走到底。

2.  **讨论：子集和组合的区别在哪里？**
    *   提问：我们说子集和组合都是从一个集合里选元素，那它们到底有什么本质区别呢？举个例子。
    *   **预期回答**：子集是选任意数量的元素（包括0个和全部），而组合是必须选固定 K 个元素。例如，从 {1,2,3} 中选，{1,2} 是子集，也是大小为 2 的组合；而 {1} 是子集，但不是大小为 2 的组合。

3.  **小组任务：手动模拟 DFS 决策树**
    *   任务：请同学们以 2-3 人为一组，拿到纸和笔。
    *   **题目**：给定集合 `{A, B, C}`，请手动画出生成所有大小为 2 的组合的 DFS 决策树。在树上标出每一步的决策（选 A/不选 A），以及当前 `idx` 和 `current_count` 的值。
    *   **目的**：加深对 DFS 决策过程和剪枝的理解。

4.  **即时练习：口头实现剪枝**
    *   提问：如果我们想从 5 个同学中选 3 个去参加比赛。现在已经考虑了 3 个同学，选了 1 个。还剩下 2 个同学没考虑。请问，我们还需要继续搜索吗？为什么？
    *   **预期回答**：不需要。因为我们还需要选 2 个人 (`3 - 1 = 2`)，而剩下的同学也只有 2 个 (`5 - 3 = 2`)。如果剩下的同学不足 2 个，就不能选够。这个就是剪枝。

5.  **代码小改动：生成子集时，如果子集为空，输出 "Empty Set"**
    *   提问：在生成子集的代码中，如果 `len == 0`，我们输出的是 `{}`。如果我想让它输出 `"Empty Set"`，应该修改哪一行代码？
    *   **预期回答**：修改 `if (len == 0)` 块中的 `std::cout << "{}" << std::endl;` 为 `std::cout << "Empty Set" << std::endl;`。

    *   **输入**：

#### 基础巩固 1

```cpp
#include <iostream>

const int MAXN = 25;
int n;
int a[MAXN];
int b[MAXN];
int len;

void dfs(int idx) {
    if (idx == n) {
        if (len == 0) {
            std::cout << "{}" << std::endl;
        } else {
            for (int i = 0; i < len; ++i) {
                std::cout << b[i] << (i == len - 1 ? "" : " ");
            }
            std::cout << std::endl;
        }
        return;
    }

    // 不选 a[idx]
    dfs(idx + 1);

    // 选 a[idx]
    b[len] = a[idx];
    len++;
    dfs(idx + 1);
    len--; // 回溯
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n;
    for (int i = 0; i < n; ++i) {
        a[i] = i + 1; // 题目要求从 1 到 N
    }

    len = 0;
    dfs(0);

    return 0;
}
```

```cpp
const int MAXN = 25;

int a[MAXN];
int b[MAXN];
```

```cpp
void dfs(int idx, int current_sum) {
    // 剪枝1: 如果当前和已经超过目标和，这条路就不用走了
    if (current_sum > s_target) {
        return;
    }

    // 递归终止条件：所有元素都考虑完了
    if (idx == n) {
        // 如果当前和恰好等于目标和，则打印

    dfs(idx + 1, current_sum);

    b[len] = a[idx];
    len++;
```

```cpp
#include <iostream>

const int MAXN = 25;
int n, k_min; // k_min 是至少 K 个元素
int a[MAXN];
int b[MAXN];
int len;

void dfs(int idx) {
    // 递归终止条件：所有元素都考虑完了
    if (idx == n) {
        // 如果当前子集长度达到或超过 k_min，则打印
        if (len >= k_min) {
            for (int i = 0; i < len; ++i) {
                std::cout << b[i] << (i == len - 1 ? "" : " ");
            }
            std::cout << std::endl;
        }
        return;
    }

    // 剪枝：如果剩余元素全部选上，也达不到 k_min，则剪枝
    // 剩余元素数量: n - idx
    // 当前已选元素数量: len
    // 需要至少选 k_min 个
    // 如果 len + (n - idx) < k_min，即当前已选 + 剩余全部 < 目标，则剪枝
    // 实际上，更精确的剪枝是：如果当前已选 len 个，剩余 n-idx 个，
    // 那么最多能选 len + (n - idx) 个。
    // 如果 len + (n - idx) < k_min，则无法满足条件。
    // 也可以不做这个剪枝，因为题意是“至少”，所以只要最终 len >= k_min 就行，
    // 不像固定 k 那么严格。这里为了简单，可以不加这个剪枝。
    // 但如果题目是“恰好 K 个，且 N 小于某个值”，那么剪枝很重要。

    // 决策1: 不选择 a[idx]
    dfs(idx + 1);

    // 决策2: 选择 a[idx]
    b[len] = a[idx];
    len++;
    dfs(idx + 1);
    len--; // 回溯
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> k_min;
    for (int i = 0; i < n; ++i) {
        a[i] = i + 1; // 题目要求从 1 到 N
    }

    len = 0;
    dfs(0);

    return 0;
}
```

```cpp
#include <iostream>
#include <algorithm> // 用于 sort

const int MAXN = 25;
int n, k;
int a[MAXN];
int b[MAXN];

void dfs(int idx, int current_count) {
    if (current_count == k) {
        for (int i = 0; i < k; ++i) {
            std::cout << b[i] << (i == k - 1 ? "" : " ");
        }
        std::cout << std::endl;
        return;
    }

    if (idx == n) {
        return;
    }

    // 剪枝
    if ((n - idx) < (k - current_count)) {
        return;
    }

    // 决策1: 选择 a[idx]
    b[current_count] = a[idx];
    dfs(idx + 1, current_count + 1);

    // 决策2: 不选择 a[idx]
    // 为了避免重复组合，如果当前元素和下一个元素相同，跳过所有相同的元素
    // 这样可以保证对于连续相同的元素，我们只在第一次遇到它们时考虑“不选”的分支
    while (idx + 1 < n && a[idx] == a[idx+1]) {
        idx++;
    }
    dfs(idx + 1, current_count);
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> k;
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    std::sort(a, a + n); // 必须先排序，才能有效处理重复元素

    dfs(0, 0);

    return 0;
}
```
