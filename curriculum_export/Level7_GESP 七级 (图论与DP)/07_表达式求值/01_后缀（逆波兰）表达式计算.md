## 趣味引入

同学们，大家好！我们平时写数学算式都是像这样：`3 + 5 * 2`。这叫做“中缀表达式”，因为运算符在操作数“中间”。但是，这里有个小麻烦：我们知道要“先乘除后加减”，所以这个算式的结果是 `3 + (5 * 2) = 13`。如果换成 `(3 + 5) * 2`，结果就是 `8 * 2 = 16`。是不是有点让人头疼？为了避免歧义，我们常常需要加括号。

但是，计算机可没我们这么聪明，它可不会“先乘除后加减”。它需要一个明确无误的指令序列，告诉它“先算什么，后算什么”。想象一下，你是一位大厨，正在按照食谱做菜。如果食谱写着“把鸡蛋和面粉混合，然后加入牛奶”，你就知道要先做第一步。但如果食谱写着“鸡蛋 + 面粉 * 牛奶”，你会不会有点懵？是先“鸡蛋+面粉”再“*牛奶”，还是“鸡蛋+（面粉*牛奶）”呢？

为了让计算机这个“大厨”能毫无歧义地执行计算任务，科学家们发明了一种特殊的表达式，叫做“后缀表达式”，也叫“逆波兰表达式”。它就像一个超级清晰的食谱，把所有操作的顺序都安排得明明白白，根本不需要括号！比如 `3 5 2 * +`，计算机一看就知道：先把3和5和2准备好，然后把5和2拿出来做乘法，得到10，再把这个10和3拿出来做加法，得到13！一步一步，清清楚楚。

那么，这个神奇的后缀表达式到底是怎么计算的呢？它背后藏着一个非常重要的“工具”——栈。今天，我们就一起来揭开后缀表达式计算的神秘面纱，并亲手用代码实现它！

## 深度知识点讲解

### 1. 表达式的家族：中缀、前缀、后缀

*   **中缀表达式 (Infix Expression)**：我们日常使用的数学表达式，运算符位于操作数之间。
    *   例子：`a + b`, `(a + b) * c`
    *   特点：直观易懂，但需要括号来明确运算优先级，否则容易产生歧义。
*   **前缀表达式 (Prefix Expression)**：运算符位于操作数之前。
    *   例子：`+ a b`, `* + a b c`
    *   特点：无需括号，运算顺序明确，从右向左扫描。
*   **后缀表达式 (Postfix Expression)**：运算符位于操作数之后。也称为“逆波兰表达式”（Reverse Polish Notation, RPN），由波兰逻辑学家扬·武卡谢维奇发明。
    *   例子：`a b +`, `a b + c *`
    *   特点：**无需括号，运算顺序明确，从左向右扫描，特别适合计算机处理。**

**为什么后缀表达式适合计算机？**
因为它消除了运算符的优先级问题和括号问题。计算机在处理后缀表达式时，只需要按照固定的规则从左到右扫描，遇到操作数就“记住”，遇到操作符就“拿出来计算”，非常高效。

### 2. 核心工具：栈 (Stack)

后缀表达式的计算离不开一个非常重要的数据结构——**栈**。

#### 2.1 栈是什么？

想象一下，你有一叠盘子。你每次洗完一个盘子，就把它放到这叠盘子的最上面。当你需要用盘子时，你总是从最上面拿一个。你不能从中间或最下面拿，那样盘子会倒掉！

栈就是这样一种数据结构：
*   **后进先出 (LIFO: Last In, First Out)**：最后放进去的元素，第一个被拿出来。
*   **只允许在栈顶进行操作**：添加元素（`push`）和删除元素（`pop`）都只能在栈的“顶部”进行。

#### 2.2 栈的基本操作

1.  **`push(x)`**：将元素 `x` 放入栈顶。
2.  **`pop()`**：移除并返回栈顶元素。
3.  **`get_top()`**：查看（但不移除）栈顶元素。
4.  **`is_empty()`**：判断栈是否为空。

#### 2.3 如何用原生数组实现栈 (重点!)

在GSP等级考试中，我们通常不允许直接使用C++标准库中的 `std::stack` 容器。这意味着我们需要**手动**用一个普通数组来实现栈的功能。

我们可以用一个整数数组 `s` 来存储栈中的元素，再用一个整数变量 `top` 来表示栈顶的位置。

*   `s[MAXN]`：一个足够大的数组，用来存放栈中的元素。`MAXN` 是栈的最大容量。
*   `top`：一个整数变量，它表示当前栈中元素的个数，或者说，下一个元素应该存放的位置。初始时 `top = 0`，表示栈为空。

```cpp
const int MAXN = 1005; // 假设栈的最大容量是1000个元素
int s[MAXN];          // 存放栈元素的数组
int top = 0;          // 栈顶指针，初始为0，表示栈为空

// 压栈操作：将元素x放入栈顶
void push(int x) {
    if (top >= MAXN) {
        // 栈满溢出，实际比赛中通常不会发生，因为题目会保证数据合法
        // 或者需要根据实际情况进行错误处理
        // std::cerr << "Error: Stack overflow!" << std::endl;
        return;
    }
    s[top] = x; // 将元素放到当前top指向的位置
    top++;      // 栈顶指针向上移动一位
}

// 弹栈操作：移除并返回栈顶元素
int pop() {
    if (top == 0) {
        // 栈空溢出，尝试从空栈中弹出元素
        // std::cerr << "Error: Stack underflow!" << std::endl;
        return -1; // 返回一个特殊值表示错误，或者抛出异常
    }
    top--;      // 栈顶指针向下移动一位
    return s[top]; // 返回原栈顶位置的元素
}

// 查看栈顶元素（不移除）
int get_top() {
    if (top == 0) {
        // 栈空
        // std::cerr << "Error: Stack is empty!" << std::endl;
        return -1; // 返回一个特殊值表示错误
    }
    return s[top - 1]; // 返回当前栈顶位置的元素
}

// 判断栈是否为空
bool is_empty() {
    return top == 0;
}
```
**注意：** `top` 的具体含义可以有两种理解：一种是指向栈顶元素的下一个空位置（如上所示），另一种是指向栈顶元素本身。这里我们采用第一种，即 `top` 总是指向下一个可以 `push` 元素的位置，所以当 `top` 为 `0` 时栈为空，`s[top-1]` 才是栈顶元素。这种方式在实现 `push` 和 `pop` 时比较简洁。

### 3. 后缀表达式的计算原理

有了栈这个工具，计算后缀表达式就变得非常简单了：

1.  **从左到右扫描**：逐个读取后缀表达式中的每个“token”（可以是数字，也可以是运算符）。
2.  **遇到操作数（数字）**：将其直接压入栈中。
3.  **遇到运算符**：
    *   从栈中连续弹出**两个**操作数。注意顺序：**先弹出的是右操作数，后弹出的是左操作数**。
    *   对这两个操作数执行对应的运算。
    *   将运算结果再次压入栈中。
4.  **扫描结束**：当整个表达式扫描完毕后，栈中应该只剩下一个元素，这个元素就是整个表达式的计算结果。

**生活类比**：想象你是一个工厂的流水线工人。
*   如果看到一个“零件”（数字），你就把它放到旁边的“待处理区”（栈）里。
*   如果看到一个“指令”（运算符），你就从“待处理区”里取出最后放进去的两个零件，按照指令把它们组装起来，然后把组装好的“新零件”（结果）再放回“待处理区”。
*   最后，当所有指令都处理完，你的“待处理区”里就只剩下最终的产品了。

### 4. 处理多位数和负数 (GESP 7级要求)

后缀表达式通常以空格分隔各个操作数和操作符。在读取字符串时，我们需要：

1.  **识别数字**：当遇到一个数字字符时，它可能是一个多位数（例如 "123"）。我们需要持续读取后续字符，直到遇到非数字字符（例如空格或运算符），将这些数字字符组合起来，转换成一个整数。
2.  **识别负数**：如果数字前面有负号（例如 "-5"），在解析数字时也要一并处理。通常，负号会作为数字的一部分出现。
3.  **识别运算符**：如果不是数字，就检查它是否是 `+`, `-`, `*`, `/` 等运算符。

**字符串解析技巧**：
我们可以遍历输入字符串。
*   如果当前字符是数字，就进入一个循环，不断读取数字字符，构建完整的数字。
*   如果当前字符是运算符，就直接处理。
*   如果当前字符是空格，就跳过。

**示例解析 `"-12 3 +"`**
1.  扫描到 `'-'`。判断下一个字符是 `1`，所以这是一个负数 `-12`。将 `-12` 压栈。
2.  扫描到 `' '`，跳过。
3.  扫描到 `'3'`。将 `3` 压栈。
4.  扫描到 `' '`，跳过。
5.  扫描到 `'+'`。
    *   弹出 `3` (右操作数)。
    *   弹出 `-12` (左操作数)。
    *   计算 `-12 + 3 = -9`。
    *   将 `-9` 压栈。
6.  表达式结束，栈中剩下 `-9`，即为结果。

### 5. 常见误区

*   **操作数弹出顺序错误**：这是最常见的错误！对于非交换律的运算（减法和除法），`A - B` 和 `B - A` 结果不同。栈中先弹出的永远是右操作数，后弹出的是左操作数。
    *   例如：`A B -` 应该计算 `A - B`。
        *   弹出 `B` (右操作数)
        *   弹出 `A` (左操作数)
        *   计算 `A - B`
*   **除法中的零**：如果右操作数是0，会导致除零错误。虽然大部分竞赛题目会保证不会出现，但作为严谨的程序员，应有所了解。
*   **栈空判断**：在 `pop` 或 `get_top` 之前，最好先判断 `is_empty()`，防止对空栈进行操作导致程序崩溃。

## 典型例题精讲

### 例题1: 简单四则运算

**题目描述**：给定一个后缀表达式字符串，计算其结果。表达式中的数字为整数，运算符只有 `+`, `-`, `*`, `/`。数字和运算符之间用空格分隔。

**输入**：`3 4 + 5 *`
**输出**：`35`

**思路分析**：
我们按照后缀表达式的计算原理，一步步模拟栈的操作。

1.  **初始化**：空栈。
2.  **扫描 `3`**：数字，压栈。
    *   栈：`[3]`
3.  **扫描 `4`**：数字，压栈。
    *   栈：`[3, 4]`
4.  **扫描 `+`**：运算符。
    *   弹出 `4` (右操作数)。
    *   弹出 `3` (左操作数)。
    *   计算 `3 + 4 = 7`。
    *   将 `7` 压栈。
    *   栈：`[7]`
5.  **扫描 `5`**：数字，压栈。
    *   栈：`[7, 5]`
6.  **扫描 `*`**：运算符。
    *   弹出 `5` (右操作数)。
    *   弹出 `7` (左操作数)。
    *   计算 `7 * 5 = 35`。
    *   将 `35` 压栈。
    *   栈：`[35]`
7.  **扫描结束**：栈中只剩 `35`，即为结果。

### 例题2: 稍复杂运算，含减法和除法

**题目描述**：同例题1。

**输入**：`10 2 / 3 - 5 *`
**输出**：`10`

**思路分析**：
再次按照步骤模拟栈操作，特别注意减法和除法的操作数顺序。

1.  **初始化**：空栈。
2.  **扫描 `10`**：数字，压栈。
    *   栈：`[10]`
3.  **扫描 `2`**：数字，压栈。
    *   栈：`[10, 2]`
4.  **扫描 `/`**：运算符。
    *   弹出 `2` (右操作数)。
    *   弹出 `10` (左操作数)。
    *   计算 `10 / 2 = 5`。
    *   将 `5` 压栈。
    *   栈：`[5]`
5.  **扫描 `3`**：数字，压栈。
    *   栈：`[5, 3]`
6.  **扫描 `-`**：运算符。
    *   弹出 `3` (右操作数)。
    *   弹出 `5` (左操作数)。
    *   计算 `5 - 3 = 2`。
    *   将 `2` 压栈。
    *   栈：`[2]`
7.  **扫描 `5`**：数字，压栈。
    *   栈：`[2, 5]`
8.  **扫描 `*`**：运算符。
    *   弹出 `5` (右操作数)。
    *   弹出 `2` (左操作数)。
    *   计算 `2 * 5 = 10`。
    *   将 `10` 压栈。
    *   栈：`[10]`
9.  **扫描结束**：栈中只剩 `10`，即为结果。

### 复杂度分析

*   **时间复杂度**：O(N)，其中N是后缀表达式中操作数和运算符的总数（或者字符串的长度）。因为我们只需要从左到右扫描表达式一次，每个数字或运算符最多进行常数次栈操作。
*   **空间复杂度**：O(N)，最坏情况下（例如 `1 2 3 ... N + + + ...`），栈中可能需要存储所有操作数。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

// 使用Allman风格大括号，变量命名遵守a-z单字母原则

// 定义栈的最大容量，根据题目数据范围调整
const int M = 1005; 
int s[M]; // 声明一个整数数组作为栈的存储空间
int t = 0; // 栈顶指针，t=0表示栈空，t指向下一个可存放元素的位置

// 栈操作函数
void p_push(int x) { // p_push 防止与 std::push 冲突，虽然这里不用std::
    s[t] = x; // 将元素x存入栈顶位置
    t++;      // 栈顶指针上移
}

int p_pop() { // p_pop 防止与 std::pop 冲突
    t--;      // 栈顶指针下移
    return s[t]; // 返回原栈顶位置的元素
}

bool i_empty() { // i_empty 防止与 std::empty 冲突
    return t == 0; // 判断栈是否为空
}

// 主函数
int main() {
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string str;
    std::getline(std::cin, str); // 读取一整行字符串作为后缀表达式

    int i = 0; // 字符串遍历指针
    while (i < str.length()) {
        if (str[i] == ' ') { // 遇到空格，跳过
            i++;
            continue;
        }

        if (isdigit(str[i]) || (str[i] == '-' && isdigit(str[i+1]))) { // 识别数字，包括负数
            int n = 0; // 存储解析出的数字
            int f = 1; // 符号位，1为正，-1为负

            if (str[i] == '-') { // 如果是负数
                f = -1;
                i++; // 跳过负号
            }

            while (i < str.length() && isdigit(str[i])) { // 持续读取数字字符
                n = n * 10 + (str[i] - '0'); // 字符转数字
                i++;
            }
            p_push(n * f); // 将解析出的数字（带符号）压栈
        } else { // 遇到运算符
            char o = str[i]; // 当前字符是运算符
            i++; // 跳过运算符

            // 从栈中弹出两个操作数
            int b = p_pop(); // 先弹出的是右操作数
            int a = p_pop(); // 后弹出的是左操作数
            int r;           // 存储运算结果

            // 根据运算符进行计算
            if (o == '+') {
                r = a + b;
            } else if (o == '-') {
                r = a - b;
            } else if (o == '*') {
                r = a * b;
            } else if (o == '/') {
                r = a / b;
            }
            p_push(r); // 将结果压回栈中
        }
    }

    // 表达式计算完毕，栈中唯一的元素就是结果
    std::cout << p_pop() << std::endl;

    return 0;
}
```

**代码解析和注意事项**：

1.  **`#include <bits/stdc++.h>`**：这是一个竞赛常用的头文件，包含了几乎所有标准库。
2.  **原生数组栈**：我们手动实现了 `p_push`, `p_pop`, `i_empty` 等函数，避免使用 `std::stack`。`t` 变量作为栈顶指针，指向下一个空位。
3.  **输入读取**：`std::getline(std::cin, str);` 用于读取包含空格的整行表达式。
4.  **字符串遍历**：使用 `while (i < str.length())` 循环遍历字符串。
5.  **数字解析**：
    *   `isdigit(str[i])` 判断当前字符是否是数字。
    *   `str[i] == '-' && isdigit(str[i+1])` 用于判断是否是负数，例如 `-12`。注意 `i+1` 的边界检查，虽然这里简化了，实际应用中需要更严谨。
    *   `n = n * 10 + (str[i] - '0');` 是将字符数字转换为整数的经典方法。
    *   `f` 变量用于处理负号。
6.  **运算符处理**：
    *   **关键点**：`int b = p_pop();` 和 `int a = p_pop();` 确保了先弹出的是右操作数 `b`，后弹出的是左操作数 `a`。这样 `a - b` 和 `a / b` 才能得到正确结果。
7.  **结果输出**：最后栈中只会有一个元素，就是最终结果，直接 `p_pop()` 输出即可。

### 练习题目参考答案

#### 基础巩固答案：
1.  `"8 2 3 + *"`
    *   `8` 入栈
    *   `2` 入栈
    *   `3` 入栈
    *   `+`：弹出 `3, 2`，计算 `2+3=5`，`5` 入栈
    *   `*`：弹出 `5, 8`，计算 `8*5=40`，`40` 入栈
    *   结果：`40`
2.  `"15 3 2 + /"`
    *   `15` 入栈
    *   `3` 入栈
    *   `2` 入栈
    *   `+`：弹出 `2, 3`，计算 `3+2=5`，`5` 入栈
    *   `/`：弹出 `5, 15`，计算 `15/5=3`，`3` 入栈
    *   结果：`3`
3.  `"4 5 * 2 +"`
    *   `4` 入栈
    *   `5` 入栈
    *   `*`：弹出 `5, 4`，计算 `4*5=20`，`20` 入栈
    *   `2` 入栈
    *   `+`：弹出 `2, 20`，计算 `20+2=22`，`22` 入栈
    *   结果：`22`

#### 能力提升答案：
1.  `"12 3 4 + 2 * -"`
    *   `12` 入栈
    *   `3` 入栈
    *   `4` 入栈
    *   `+`：弹出 `4, 3`，计算 `3+4=7`，`7` 入栈
    *   `2` 入栈
    *   `*`：弹出 `2, 7`，计算 `7*2=14`，`14` 入栈
    *   `-`：弹出 `14, 12`，计算 `12-14=-2`，`-2` 入栈
    *   结果：`-2`
2.  `"20 5 / 3 2 * +"`
    *   `20` 入栈
    *   `5` 入栈
    *   `/`：弹出 `5, 20`，计算 `20/5=4`，`4` 入栈
    *   `3` 入栈
    *   `2` 入栈
    *   `*`：弹出 `2, 3`，计算 `3*2=6`，`6` 入栈
    *   `+`：弹出 `6, 4`，计算 `4+6=10`，`10` 入栈
    *   结果：`10`
3.  `"-10 2 3 * + 5 /"`
    *   `-10` 入栈
    *   `2` 入栈
    *   `3` 入栈
    *   `*`：弹出 `3, 2`，计算 `2*3=6`，`6` 入栈
    *   `+`：弹出 `6, -10`，计算 `-10+6=-4`，`-4` 入栈
    *   `5` 入栈
    *   `/`：弹出 `5, -4`，计算 `-4/5=0` (整数除法)，`0` 入栈
    *   结果：`0`

#### 拓展挑战答案：
1.  需要修改数字解析部分，增加对特殊负号字符的识别。例如，如果 `#` 表示负号，则当遇到 `#` 字符时，将其标记为负数，并跳过 `#` 继续解析后面的数字。
2.  **思考题提示**：将中缀表达式转换为后缀表达式也需要用到栈。基本思想是：从左到右扫描中缀表达式。遇到操作数直接输出；遇到左括号压栈；遇到右括号，将栈中运算符弹出并输出，直到遇到左括号并将其弹出（但不输出）；遇到运算符时，如果栈顶运算符优先级高于或等于当前运算符，则将栈顶运算符弹出并输出，直到栈为空或栈顶运算符优先级低于当前运算符，然后将当前运算符压栈。扫描结束后，将栈中所有剩余运算符弹出并输出。
