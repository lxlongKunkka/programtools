## 深度知识点讲解

### 1. 表达式的“语言”：中缀表达式、优先级和括号

我们日常使用的 `1 + 2 * 3` 这种表达式，叫做**中缀表达式**。因为运算符（`+`, `-`, `*`, `/`）在两个操作数（数字）的中间。

*   **运算符优先级**：
    *   乘法（`*`）和除法（`/`）的优先级最高。
    *   加法（`+`）和减法（`-`）的优先级次之。
    *   同级别的运算符，通常遵循从左到右的计算顺序。
*   **括号的作用**：
    *   括号 `()` 就像一个“特权通行证”，它能强制改变运算的优先级。括号里的内容永远优先计算。
    *   当遇到括号时，我们总是先处理括号内部的表达式，然后再将括号内部的计算结果作为一个整体参与外部的计算。

### 2. 秘密武器：栈 (Stack)

栈是一种非常重要的数据结构，它的特点是**“后进先出” (LIFO - Last In, First Out)**。

*   **生活类比**：
    *   **一叠盘子**：你把盘子一个一个叠起来，想拿最下面的盘子，必须先拿走上面的。最后放上去的盘子，最先被拿走。
    *   **子弹夹**：往弹夹里装子弹，最后装进去的子弹，最先被射出。
    *   **桶装薯片**：最后放进去的薯片，最先被你吃到。
*   **栈的操作**：
    *   **压入 (Push)**：把数据放到栈顶。
    *   **弹出 (Pop)**：从栈顶取走数据。
    *   **查看栈顶 (Peek/Top)**：看看栈顶的数据是什么，但不取走。
    *   **判断栈空 (IsEmpty)**：看看栈里还有没有数据。

### 3. “两栈法”求值原理：让电脑“思考”的流程

现在，我们有了栈这个工具，如何让它来处理表达式呢？我们可以模拟一个“计算流水线”，这条流水线需要两个栈：

1.  **数字栈 (Operand Stack)**：用来存放表达式中的数字（操作数）。
2.  **运算符栈 (Operator Stack)**：用来存放表达式中的运算符（`+`, `-`, `*`, `/`, `(`, `)`）。

想象一下，我们从左到右扫描表达式中的每一个字符，就像流水线上的产品一样。

#### 核心处理规则：

*   **遇到数字**：
    *   直接把这个数字压入**数字栈**。
    *   （思考：数字是操作数，它不需要优先级判断，直接等着被计算就行。）

*   **遇到左括号 `(`**：
    *   直接把左括号压入**运算符栈**。
    *   （思考：左括号表示一个“新的高优先级区域”开始了，它自己不参与运算，只是一个标记。）

*   **遇到右括号 `)`**：
    *   不断从**运算符栈**中弹出运算符，每弹出一个运算符，就从**数字栈**中弹出两个数字进行计算。
    *   计算结果再压回**数字栈**。
    *   直到遇到左括号 `(` 为止，将左括号从运算符栈中弹出（但不要压回）。
    *   （思考：右括号表示一个“高优先级区域”结束了，要赶紧把这个区域内的所有运算都完成掉。）

*   **遇到运算符 `+`, `-`, `*`, `/`**：
    *   **情况1：如果运算符栈为空，或者栈顶是左括号 `(`**：
        *   直接把当前运算符压入**运算符栈**。
        *   （思考：栈空或左括号，说明没有需要优先处理的运算符，或者当前运算符在一个新区域的开头，直接放进去。）
    *   **情况2：如果当前运算符的优先级** `高于` **运算符栈顶的运算符**：
        *   直接把当前运算符压入**运算符栈**。
        *   （思考：当前运算符优先级更高，它应该“后”计算，所以让它“后”压入栈，等着前面的低优先级运算符先被处理。）
    *   **情况3：如果当前运算符的优先级** `不高于` **运算符栈顶的运算符**：
        *   不断从**运算符栈**中弹出运算符，每弹出一个运算符，就从**数字栈**中弹出两个数字进行计算。
        *   计算结果再压回**数字栈**。
        *   这个过程一直持续，直到：
            *   运算符栈为空。
            *   运算符栈顶是左括号 `(`。
            *   当前运算符的优先级高于运算符栈顶的运算符。
        *   然后，再把当前运算符压入**运算符栈**。
        *   （思考：栈顶的运算符优先级更高或相同，它应该“先”计算。所以我们让它先弹出并完成运算，直到当前运算符可以安全地压入栈中。）

*   **表达式扫描完毕**：
    *   如果**运算符栈**中还有剩余的运算符，就继续不断地弹出运算符，从**数字栈**中弹出两个数字进行计算，结果压回**数字栈**。
    *   直到运算符栈为空。
    *   （思考：表达式都看完了，剩下的运算都是按照优先级从高到低排列的，依次完成即可。）

*   **最终结果**：
    *   **数字栈**中剩下的最后一个数字，就是整个表达式的计算结果。

#### 关键点剖析与常见误区：

1.  **为什么需要两个栈？**
    *   数字和运算符是两种不同类型的数据，它们在处理逻辑上是独立的。数字等待运算，运算符决定运算时机。
    *   如果只有一个栈，会混淆数据类型和操作逻辑。
2.  **为什么优先级判断是“不高于”才弹出？**
    *   因为栈的“后进先出”特性。如果当前运算符优先级低，那么栈顶那个优先级高或相同的运算符应该先计算，这样它才能先“出”栈。
3.  **括号处理的精髓？**
    *   左括号 `(` 像一个“封印”，它压入运算符栈后，会阻止任何优先级低于它的运算符被弹出，直到遇到对应的右括号 `)`。
    *   右括号 `)` 像一个“解封咒语”，它会强制把内部的运算全部执行完，直到遇到那个“封印”左括号。
4.  **数字的提取**：表达式中的数字可能不止一位（例如 `123`）。在扫描时，如果当前字符是数字，要继续向后看，直到遇到非数字字符，才能完整地提取出一个多位数字。
5.  **负数与一元运算符**：本节课为了简化，我们假设所有数字都是非负的，且 `-` 总是二元减法。处理一元负号（如 `-5`）需要额外的逻辑判断，通常是判断 `-` 前面是否是数字或右括号。

### 4. 手动实现栈 (Array-based Stack)

在 GESP 比赛中，通常要求使用原生数组来实现栈，而不是使用 `std::stack`。

```cpp
const int MAX_SIZE = 1005; // 栈的最大容量

int num_stack[MAX_SIZE]; // 数字栈
int num_top = -1;        // 数字栈栈顶指针，-1表示栈空

char op_stack[MAX_SIZE]; // 运算符栈
int op_top = -1;         // 运算符栈栈顶指针，-1表示栈空

// 数字栈操作
void num_push(int x) {
    num_stack[++num_top] = x;
}
int num_pop() {
    return num_stack[num_top--];
}
int num_peek() {
    return num_stack[num_top];
}
bool num_isEmpty() {
    return num_top == -1;
}

// 运算符栈操作
void op_push(char c) {
    op_stack[++op_top] = c;
}
char op_pop() {
    return op_stack[op_top--];
}
char op_peek() {
    return op_stack[op_top];
}
bool op_isEmpty() {
    return op_top == -1;
}
```

有了这些基础，我们就可以开始构建完整的表达式求值算法了！

**空栈处理**：当遇到当前运算符优先级不高于栈顶时，一直弹出计算，直到栈空。

```cpp
#include <iostream>
#include <string>
#include <map> // 用于存储运算符优先级

// 为了遵循GESP 1-5级严禁使用STL容器的限制精神，我们手动实现栈
// GESP 7级可能允许std::stack，但为教学深度和通用性，仍用数组实现

const int MAX_SIZE = 1005; // 栈的最大容量，根据表达式长度调整

// --- 手动实现数字栈 ---
int num_s[MAX_SIZE];
int num_t = -1; // 栈顶指针，-1表示空栈

void num_push(int x) {
    if (num_t >= MAX_SIZE - 1) {
        // 栈满错误处理，实际比赛中通常不考虑，因为题目保证数据范围
        return;
    }
    num_s[++num_t] = x;
}

int num_pop() {
    if (num_t == -1) {
        // 栈空错误处理
        return 0; // 或者抛出异常
    }
    return num_s[num_t--];
}

int num_peek() {
    if (num_t == -1) {
        return 0; // 栈空
    }
    return num_s[num_t];
}

bool num_isEmpty() {
    return num_t == -1;
}

// --- 手动实现运算符栈 ---
char op_s[MAX_SIZE];
int op_t = -1; // 栈顶指针，-1表示空栈

void op_push(char c) {
    if (op_t >= MAX_SIZE - 1) {
        // 栈满错误处理
        return;
    }
    op_s[++op_t] = c;
}

char op_pop() {
    if (op_t == -1) {
        // 栈空错误处理
        return '\0'; // 或者抛出异常
    }
    return op_s[op_t--];
}

char op_peek() {
    if (op_t == -1) {
        return '\0'; // 栈空
    }
    return op_s[op_t];
}

bool op_isEmpty() {
    return op_t == -1;
}

// --- 辅助函数 ---

// 获取运算符优先级
// 优先级数字越大表示优先级越高
// ( 为最低优先级，因为它只做分隔符，不参与立即计算
int get_priority(char c) {
    if (c == '*' || c == '/') return 2;
    if (c == '+' || c == '-') return 1;
    if (c == '(') return 0; // 左括号优先级最低，确保其能被压入栈
    return -1; // 非运算符
}

// 执行计算
int calculate(int a, int b, char op) {
    if (op == '+') return a + b;
    if (op == '-') return a - b;
    if (op == '*') return a * b;
    if (op == '/') {
        if (b == 0) {
            // 除零错误处理
            std::cerr << "Error: Division by zero!" << std::endl;
            exit(1);
        }
        return a / b;
    }
    return 0; // 错误
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s;
    std::cin >> s;

    int n = s.length();
    for (int i = 0; i < n; ++i) {
        char c = s[i];

        if (isdigit(c)) { // 如果是数字
            int num = 0;
            // 提取多位数字
            while (i < n && isdigit(s[i])) {
                num = num * 10 + (s[i] - '0');
                i++;
            }
            num_push(num);
            i--; // 回退一步，因为外层循环会再次i++
        } else if (c == '(') { // 如果是左括号
            op_push(c);
        } else if (c == ')') { // 如果是右括号
            while (op_peek() != '(') {
                int b = num_pop();
                int a = num_pop();
                char op = op_pop();
                num_push(calculate(a, b, op));
            }
            op_pop(); // 弹出左括号
        } else { // 如果是运算符 + - * /
            while (!op_isEmpty() && get_priority(c) <= get_priority(op_peek())) {
                // 当前运算符优先级不高于栈顶运算符，则先计算栈顶运算符
                int b = num_pop();
                int a = num_pop();
                char op = op_pop();
                num_push(calculate(a, b, op));
            }
            op_push(c); // 将当前运算符压入栈
        }
    }

    // 表达式扫描完毕，处理剩余的运算符
    while (!op_isEmpty()) {
        int b = num_pop();
        int a = num_pop();
        char op = op_pop();
        num_push(calculate(a, b, op));
    }

    std::cout << num_pop() << std::endl; // 最终结果在数字栈中

    return 0;
}
```

**5. 复杂度分析**：
*   **时间复杂度**：每个数字和运算符最多被压入栈一次，弹出一次。每次计算都是常数时间。因此，总时间复杂度为 O(N)，其中 N 是表达式的长度。
*   **空间复杂度**：在最坏情况下（例如 `((((1+1)+1)+1)...)`），两个栈的大小可能与表达式长度成正比。因此，空间复杂度为 O(N)。
