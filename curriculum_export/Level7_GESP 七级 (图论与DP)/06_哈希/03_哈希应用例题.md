## 深度知识点讲解

### 1. 哈希（Hash）的本质：信息指纹

**是什么？**
哈希，简单来说，就是一种函数，它把任意长度的输入（比如一个字符串、一个文件、一段数据）映射成一个固定长度的输出，这个输出就是哈希值（Hash Value），也叫散列值。哈希值就像是数据的“指纹”或“身份证号”。

**为什么？**
*   **高效比较**：比较两个长数据是否相同，只需要比较它们的哈希值即可。如果哈希值不同，数据一定不同；如果哈希值相同，数据很可能相同（但有极小概率不同，这就是哈希冲突）。
*   **快速查找**：在大量数据中查找某个特定数据时，可以先计算数据的哈希值，然后根据哈希值快速定位数据可能的位置。
*   **数据完整性校验**：通过比较数据传输前后的哈希值，可以判断数据在传输过程中是否被篡改。

**怎么想到的？**
人类在处理信息时，总是希望能用更简洁的方式来代表复杂的信息。比如，我们用学号代表学生，用身份证号代表公民。哈希就是这种思想在计算机科学中的体现。它将复杂、不规则的数据“标准化”为固定长度的数字，方便计算机进行处理。

### 2. 字符串哈希（String Hashing）：给字符串编号

**核心思想：多项式滚动哈希**
我们把一个字符串看作是一个P进制的数字！
比如字符串 "abc"：
如果把 'a' 看作数字 1，'b' 看作 2，'c' 看作 3 (通常用字符的ASCII码减去一个偏移量，比如 'a' - 'a' + 1 = 1)。
那么 "abc" 就可以看作是 `1 * P^2 + 2 * P^1 + 3 * P^0`。
这就是多项式哈希的基本形式。

**哈希函数：**
对于字符串 `s = s_0s_1...s_{n-1}`，它的哈希值通常计算为：
`H(s) = (s_0 * P^(n-1) + s_1 * P^(n-2) + ... + s_{n-1} * P^0) % M`
其中：
*   `s_i` 是字符串中第 `i` 个字符的数值表示（例如，`s[i] - 'a' + 1`）。
*   `P` 是一个选定的**基数**（Prime Base），通常选择一个大于字符集大小的素数，比如 131 或 13331。
*   `M` 是一个选定的**模数**（Modulus），通常选择一个很大的素数，比如 10^9 + 7 或 10^9 + 9。模运算的目的是防止哈希值过大溢出，并将其限制在一个可管理的范围内。

**关键点剖析：**

#### a) 基数 P 和模数 M 的选择
*   **P (基数)**：
    *   必须是一个素数。
    *   通常选择一个比字符集大小稍大的素数，以确保不同的字符有不同的贡献。例如，如果只包含小写字母，P可以选131或13331。
    *   P越大，冲突概率越小。
*   **M (模数)**：
    *   必须是一个很大的素数。
    *   常用的有 `10^9 + 7`，`10^9 + 9`，`998244353` 等。
    *   M越大，哈希值的范围越大，冲突概率越小。
    *   选择素数M是为了避免一些特殊的模运算性质导致的问题。
*   **双哈希 (Double Hashing)**：为了进一步降低哈希冲突的概率，尤其是在数据范围较大或对冲突容忍度极低的情况下，我们通常会使用两套不同的 (P, M) 组合来计算两个哈希值。只有当两个哈希值都相同时，才认为字符串相同。这就像给数据加了两道指纹锁。

#### b) 前缀哈希 (Prefix Hash) 的计算
为了快速获取任意子串的哈希值，我们通常会预处理出所有前缀的哈希值。
`h[i]` 表示字符串 `s` 的前缀 `s[0...i-1]` 的哈希值。
`h[0] = 0` (空字符串的哈希值)
`h[i] = (h[i-1] * P + s[i-1] 的数值) % M`
这个公式是递推的，可以在 O(N) 时间内计算出所有前缀哈希值。

**生活类比**：想象你有一个计算器，它只能做加法和乘法，并且只能显示1000以内的数字（模数M）。现在你要计算一个很长的数字（字符串）的值。你不会从头到尾一次性算完，而是从左边第一个数字开始，每读一个新数字，就把之前的计算结果乘以10（基数P），再加上新读到的数字，然后把结果除以1000取余数。这样，你每一步都能得到当前前缀的“魔法编号”，而且结果不会溢出。

#### c) 子串哈希 (Substring Hash) 的计算 (滚动哈希)
有了前缀哈希，我们就可以在 O(1) 时间内计算任意子串 `s[l...r-1]` 的哈希值。
子串 `s[l...r-1]` 的哈希值 `H(s[l...r-1])` 相当于 `s[0...r-1]` 的哈希值减去 `s[0...l-1]` 的哈希值，但是要注意位数和模运算。

`H(s[l...r-1]) = (h[r] - h[l] * P^(r-l) % M + M) % M`
其中：
*   `h[r]` 是前缀 `s[0...r-1]` 的哈希值。
*   `h[l]` 是前缀 `s[0...l-1]` 的哈希值。
*   `P^(r-l)` 是基数 P 的 `(r-l)` 次幂，表示 `s[0...l-1]` 这部分哈希值在整个 `s[0...r-1]` 中被左移了 `(r-l)` 位。
*   加 `M` 再模 `M` 是为了处理 `h[r] - h[l] * P^(r-l)` 结果为负数的情况。

为了计算 `P^(r-l)`，我们需要预处理出 `P` 的所有幂次。
`p_pow[i] = P^i % M`
`p_pow[0] = 1`
`p_pow[i] = (p_pow[i-1] * P) % M`

**怎么想到的？**
这就像在十进制中，数字 `12345` 的值是 `123 * 10^2 + 45`。如果我们想得到 `345` 的值，可以用 `12345 - 12000 = 12345 - 123 * 10^2`。
字符串哈希也是类似，`H(s[0...r-1])` 包含了 `H(s[0...l-1]) * P^(r-l)` 这一项。把这部分减掉，就是 `H(s[l...r-1])`。

#### d) 常见误区
*   **忘记取模**：在计算哈希值和P的幂次时，每一步都要进行模运算，否则结果可能溢出。
*   **模运算中出现负数**：在 `(A - B) % M` 的计算中，如果 `A < B`，结果可能为负。正确的做法是 `(A - B % M + M) % M`。
*   **基数和模数选择不当**：选择太小的P或M会大大增加哈希冲突的概率。
*   **哈希冲突的概率**：哈希值相同不代表字符串一定相同！在要求严谨的场景中，需要原始字符串进行二次校验，或者使用双哈希。在比赛中，双哈希通常足以应对绝大多数情况。

### 3. C++ 中的字符数值化
通常我们会将字符 `c` 转换为 `c - 'a' + 1` (对于小写字母) 或 `c - 'A' + 1` (对于大写字母) 或 `c - '0' + 1` (对于数字字符)，或者直接使用其 ASCII 码。加 `1` 的目的是避免字符值为 `0`，因为 `0` 乘以任何数都是 `0`，可能导致信息丢失。

### 例题一：统计不同子串的数量

**题目描述**
给定一个字符串 `s`，请你统计它有多少个不同的非空子串。

**输入格式**
一行，一个字符串 `s`。

**输出格式**
一个整数，表示不同子串的数量。

**样例输入**
`ababa`

**样例输出**
`7`
解释：子串有 "a", "b", "a", "b", "a", "ab", "ba", "ab", "ba", "aba", "bab", "aba", "abab", "baba", "ababa"。
不同的子串为 "a", "b", "ab", "ba", "aba", "bab", "ababa"，共 7 个。

**思路分析**
1.  **暴力解法**：枚举所有子串，然后把它们存到一个 `std::set<std::string>` 中，最后 `set` 的大小就是答案。这种方法会因为 `std::string` 的比较和存储开销过大而超时，时间复杂度为 O(N^3 * log(N^2)) 或 O(N^4) (字符串复制和比较)。
2.  **哈希优化**：
    *   我们可以用字符串哈希来代表每个子串。
    *   预处理出字符串 `s` 的前缀哈希值 `h` 和基数 `P` 的幂次 `p_pow`。
    *   使用双重循环枚举所有可能的子串 `s[i...j]` (其中 `0 <= i <= j < N`)。
    *   对于每个子串 `s[i...j]`，利用滚动哈希在 O(1) 时间内计算其哈希值 `H(s[i...j])`。
    *   将计算出的哈希值（为了防止冲突，最好是双哈希值构成的 `std::pair<long long, long long>`）存入 `std::set` 中。
    *   最终 `std::set` 的大小就是不同子串的数量。

**复杂度分析**
*   **预处理**：计算 `p_pow` 和 `h` 数组需要 O(N) 时间。
*   **枚举子串**：外层循环 `i` 从 `0` 到 `N-1`，内层循环 `j` 从 `i` 到 `N-1`，共 O(N^2) 对 `(i, j)`。
*   **计算子串哈希**：每对 `(i, j)` 计算哈希值需要 O(1) 时间。
*   **插入 `std::set`**：每次插入一个哈希值对（`std::pair`）到 `std::set` 中，时间复杂度是 O(log S)，其中 `S` 是 `set` 中元素的数量，最大为 O(N^2)。
*   **总时间复杂度**：O(N^2 log(N^2)) = O(N^2 log N)。对于 N=5000 左右的数据，这个复杂度是可接受的。

**C++ 代码实现**

```cpp
#include <bits/stdc++.h>

// GESP 7级通常允许使用STL容器，特别是std::set/map等，
// 因为它们在底层已经实现了高效的数据结构，使得我们能专注于算法逻辑。
// 如果题目明确要求不使用，则需手动实现哈希表。

const int N_MAX = 5005; // 字符串最大长度
const long long P1 = 131; // 第一个哈希函数的基数
const long long M1 = 1e9 + 7; // 第一个哈希函数的模数
const long long P2 = 13331; // 第二个哈希函数的基数
const long long M2 = 1e9 + 9; // 第二个哈希函数的模数

// 存储P的幂次
long long p_pow1[N_MAX];
long long p_pow2[N_MAX];

// 存储前缀哈希值
long long h1[N_MAX];
long long h2[N_MAX];

void precompute_hashes(const std::string& s) {
    int n = s.length();

    // 预处理P的幂次
    p_pow1[0] = 1;
    p_pow2[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow1[i] = (p_pow1[i - 1] * P1) % M1;
        p_pow2[i] = (p_pow2[i - 1] * P2) % M2;
    }

    // 预处理前缀哈希值
    h1[0] = 0;
    h2[0] = 0;
    for (int i = 0; i < n; ++i) {
        // 将字符s[i]转换为数值，这里使用 s[i] - 'a' + 1
        long long val_char = s[i] - 'a' + 1;
        h1[i + 1] = (h1[i] * P1 + val_char) % M1;
        h2[i + 1] = (h2[i] * P2 + val_char) % M2;
    }
}

// 获取子串 s[l...r-1] 的双哈希值
// l: 子串起始索引 (0-based)
// r: 子串结束索引+1 (0-based, 即子串长度为 r-l)
std::pair<long long, long long> get_substring_hash(int l, int r) {
    // 哈希值1
    long long hash_val1 = (h1[r] - (h1[l] * p_pow1[r - l]) % M1 + M1) % M1;
    // 哈希值2
    long long hash_val2 = (h2[r] - (h2[l] * p_pow2[r - l]) % M2 + M2) % M2;
    return {hash_val1, hash_val2};
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s;
    std::cin >> s;

    int n = s.length();

    precompute_hashes(s);

    // 使用std::set存储不同子串的哈希值，pair<long long, long long> 作为key
    std::set<std::pair<long long, long long>> distinct_hashes;

    // 枚举所有子串
    for (int i = 0; i < n; ++i) { // 子串起始位置
        for (int j = i; j < n; ++j) { // 子串结束位置
            // 子串为 s[i...j]
            // 对应的get_substring_hash参数为 (i, j+1)
            distinct_hashes.insert(get_substring_hash(i, j + 1));
        }
    }

    std::cout << distinct_hashes.size() << std::endl;

    return 0;
}
```

### 例题二：最长回文子串 (哈希优化)

**题目描述**
给定一个字符串 `s`，找到 `s` 中最长的回文子串。如果存在多个，输出任意一个即可。

**输入格式**
一行，一个字符串 `s`。

**输出格式**
一行，最长回文子串。

**样例输入**
`babad`

**样例输出**
`bab` (或 `aba`)

**思路分析**
1.  **回文子串的性质**：一个字符串是回文串，当且仅当它正读和反读都一样。
2.  **暴力解法**：枚举所有子串，对每个子串判断是否为回文串，然后记录最长的。判断回文串需要 O(L) 时间，总复杂度 O(N^3)。
3.  **动态规划 (DP) 解法**：`dp[i][j]` 表示 `s[i...j]` 是否为回文串。
    `dp[i][j] = (s[i] == s[j] && (j - i < 2 || dp[i+1][j-1]))`
    DP 状态转移 O(1)，总复杂度 O(N^2)。
4.  **哈希优化**：
    *   **方法一：正向哈希 + 反向哈希**
        *   预处理字符串 `s` 的前缀哈希值 `h_forward`。
        *   预处理字符串 `s` 的**反转串 `s_rev`** 的前缀哈希值 `h_reverse`。
        *   对于子串 `s[l...r-1]`，其正向哈希值为 `H_forward(s[l...r-1])`。
        *   其反向哈希值可以通过 `s_rev` 对应的子串 `s_rev[N-r...N-l-1]` 的哈希值 `H_reverse(s_rev[N-r...N-l-1])` 来获取。
        *   如果这两个哈希值（最好是双哈希）相等，则 `s[l...r-1]` 是回文串。
        *   通过二分查找回文子串的长度，再枚举中心点，可以达到 O(N log N) 的复杂度。
    *   **方法二：中心扩展法 + 哈希校验 (本例采用)**
        *   枚举所有可能的“回文中心”。回文中心可以是单个字符（奇数长度回文串），也可以是两个字符之间的空隙（偶数长度回文串）。共有 `2N-1` 个中心。
        *   对于每个中心，向两边扩展，直到不再是回文串。
        *   在扩展时，如果我们要检查 `s[i...j]` 是否为回文，我们可以通过哈希值快速判断 `s[i...k]` 和 `s[j-k...j]` 的哈希值是否相等（其中 `k` 是扩展的长度）。
        *   更直接的方法是，检查 `s[i...j]` 的正向哈希值是否等于 `s[i...j]` 反转后的哈希值。
        *   为了避免每次都反转子串，我们可以预处理出 `s` 的正向哈希和 `s` 反转后的前缀哈希。
            *   `s_forward_hash[i]` 存储 `s[0...i-1]` 的哈希。
            *   `s_backward_hash[i]` 存储 `s[N-i...N-1]` 的哈希 (即 `s` 的反转串的前缀哈希)。
        *   要检查 `s[l...r-1]` 是否为回文：
            *   计算 `s[l...r-1]` 的正向哈希 `H_f = get_substring_hash_forward(l, r)`。
            *   计算 `s[l...r-1]` 的反向哈希 `H_b = get_substring_hash_backward(N-r, N-l)`。
            *   如果 `H_f == H_b`，则它是回文。

**复杂度分析**
*   **预处理**：`s` 和 `s_rev` 的前缀哈希以及P的幂次，O(N) 时间。
*   **中心扩展**：共有 `2N-1` 个中心。
    *   对于每个中心，向两边扩展，最坏情况下扩展 O(N) 次。
    *   每次扩展，需要 O(1) 时间计算哈希并比较。
*   **总时间复杂度**：O(N^2)。尽管与DP相同，但哈希方法在常数因子上可能更快，并且代码实现有时更简洁。

**C++ 代码实现**

```cpp
#include <bits/stdc++.h>

const int N_MAX = 5005;
const long long P1 = 131;
const long long M1 = 1e9 + 7;
const long long P2 = 13331;
const long long M2 = 1e9 + 9;

long long p_pow1[N_MAX];
long long p_pow2[N_MAX];

// 正向哈希值 (s[0...i-1])
long long h_forward1[N_MAX];
long long h_forward2[N_MAX];

// 反向哈希值 (s_rev[0...i-1], 即 s[N-i...N-1])
long long h_backward1[N_MAX];
long long h_backward2[N_MAX];

void precompute_hashes(const std::string& s) {
    int n = s.length();

    p_pow1[0] = 1;
    p_pow2[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow1[i] = (p_pow1[i - 1] * P1) % M1;
        p_pow2[i] = (p_pow2[i - 1] * P2) % M2;
    }

    h_forward1[0] = 0;
    h_forward2[0] = 0;
    h_backward1[0] = 0;
    h_backward2[0] = 0;

    for (int i = 0; i < n; ++i) {
        long long val_char_f = s[i] - 'a' + 1; // 正向字符
        long long val_char_b = s[n - 1 - i] - 'a' + 1; // 反向字符 (s_rev[i])

        h_forward1[i + 1] = (h_forward1[i] * P1 + val_char_f) % M1;
        h_forward2[i + 1] = (h_forward2[i] * P2 + val_char_f) % M2;

        h_backward1[i + 1] = (h_backward1[i] * P1 + val_char_b) % M1;
        h_backward2[i + 1] = (h_backward2[i] * P2 + val_char_b) % M2;
    }
}

// 获取正向子串 s[l...r-1] 的哈希值
std::pair<long long, long long> get_forward_hash(int l, int r) {
    long long hash_val1 = (h_forward1[r] - (h_forward1[l] * p_pow1[r - l]) % M1 + M1) % M1;
    long long hash_val2 = (h_forward2[r] - (h_forward2[l] * p_pow2[r - l]) % M2 + M2) % M2;
    return {hash_val1, hash_val2};
}

// 获取反向子串 s_rev[l...r-1] 的哈希值
// 对应到原串 s 的子串为 s[N-r...N-l-1]
std::pair<long long, long long> get_backward_hash(int l, int r) {
    long long hash_val1 = (h_backward1[r] - (h_backward1[l] * p_pow1[r - l]) % M1 + M1) % M1;
    long long hash_val2 = (h_backward2[r] - (h_backward2[l] * p_pow2[r - l]) % M2 + M2) % M2;
    return {hash_val1, hash_val2};
}

// 检查 s[l...r-1] 是否为回文
bool is_palindrome(int l, int r, int n) {
    if (l >= r) return true; // 空串或单字符都是回文

    // 获取 s[l...r-1] 的正向哈希
    std::pair<long long, long long> forward_h = get_forward_hash(l, r);

    // 获取 s[l...r-1] 的反向哈希
    // 它对应 s_rev 中的子串 s_rev[N-r ... N-l-1]
    // s_rev 的索引是 0-based，长度是 N
    // 所以在 s_rev 中，子串的起始索引是 N-r，结束索引+1 是 N-l
    std::pair<long long, long long> backward_h = get_backward_hash(n - r, n - l);

    return forward_h == backward_h;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s;
    std::cin >> s;

    int n = s.length();
    if (n == 0) {
        std::cout << "" << std::endl;
        return 0;
    }

    precompute_hashes(s);

    int max_len = 1; // 最长回文子串的长度
    int start_idx = 0; // 最长回文子串的起始索引

    // 枚举回文中心
    for (int i = 0; i < n; ++i) {
        // 奇数长度回文串，中心是 s[i]
        // 左右扩展 j 步
        // 子串为 s[i-j ... i+j]
        for (int j = 0; i - j >= 0 && i + j < n; ++j) {
            if (is_palindrome(i - j, i + j + 1, n)) {
                if (2 * j + 1 > max_len) {
                    max_len = 2 * j + 1;
                    start_idx = i - j;
                }
            } else {
                break; // 不再是回文，停止扩展
            }
        }

        // 偶数长度回文串，中心是 s[i] 和 s[i+1] 之间
        // 左右扩展 j 步
        // 子串为 s[i-j ... i+1+j]
        for (int j = 0; i - j >= 0 && i + 1 + j < n; ++j) {
            if (is_palindrome(i - j, i + 1 + j + 1, n)) {
                if (2 * j + 2 > max_len) {
                    max_len = 2 * j + 2;
                    start_idx = i - j;
                }
            } else {
                break; // 不再是回文，停止扩展
            }
        }
    }

    std::cout << s.substr(start_idx, max_len) << std::endl;

    return 0;
}
```

    p_pow1[0] = 1;
    p_pow2[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow1[i] = (p_pow1[i - 1] * P1) % M1;
        p_pow2[i] = (p_pow2[i - 1] * P2) % M2;

        long long val_char = s[i] - 'a' + 1; 
        h1[i + 1] = (h1[i] * P1 + val_char) % M1;
        h2[i + 1] = (h2[i] * P2 + val_char) % M2;
