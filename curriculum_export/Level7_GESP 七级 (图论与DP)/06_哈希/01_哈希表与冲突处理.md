### 深度知识点讲解

#### 1. 什么是哈希表 (Hash Table)？

哈希表，也叫散列表，是一种神奇的数据结构，它能实现对数据的**快速查找、插入和删除**。它的核心思想是：将我们想要存储的“键值”（key，比如快递单号、学生姓名）通过一个特殊的“魔法函数”（哈希函数），转换成一个数字（数组下标），然后直接把数据存放到这个数字对应的位置上。

*   **键值 (Key):** 我们要查找的数据的标识，比如字符串、整数等。
*   **哈希函数 (Hash Function):** 一个函数，它接收键值作为输入，输出一个整数（通常是数组的下标）。这个过程就像是快递单号经过计算，直接告诉你“去X号货架！”。
*   **哈希表 (Hash Table / 散列表):** 实际上就是一个**数组**。哈希函数计算出来的下标，就是键值在数组中存放的位置。

**核心优势：** 在理想情况下，哈希表的查找、插入、删除操作的平均时间复杂度可以达到 O(1)！这意味着无论数据量有多大，这些操作都可能只需要一步！

#### 2. 哈希函数 (Hash Function) 的魔法

哈希函数是哈希表的“大脑”。它的好坏直接决定了哈希表的效率。

*   **作用：** 将任意长度的键值（Key）映射成一个固定范围内的整数（通常是数组的下标 `0` 到 `M-1`，其中 `M` 是哈希表的大小）。
*   **好的哈希函数特点：**
    1.  **计算快速：** 毕竟我们是为了快才用哈希表，如果哈希函数计算本身很慢，那就得不偿失了。
    2.  **均匀分布：** 尽量让不同的键值计算出来的下标能够均匀地分布在整个哈希表（数组）中，避免数据集中在少数几个位置。
    3.  **确定性：** 同一个键值，每次计算出来的哈希值必须是相同的。

**常见的哈希函数设计：**

*   **a) 除留余数法 (Division Method):**
    这是最简单也最常用的一种。
    `H(key) = key % M`
    其中 `M` 是哈希表（数组）的大小。
    **为什么用它？** 因为它能把任意大的整数 `key` 映射到 `0` 到 `M-1` 的范围内。
    **M 的选择：** 经验表明，选择一个**质数**（素数）作为 `M`，能够使得哈希值分布更均匀，减少冲突。

    *   **例子：** 假设哈希表大小 `M = 7`，我们要插入数字 `10, 23, 5, 17`。
        *   `H(10) = 10 % 7 = 3`
        *   `H(23) = 23 % 7 = 2`
        *   `H(5) = 5 % 7 = 5`
        *   `H(17) = 17 % 7 = 3` (哎呀！10 和 17 都映射到了 3 号位置，这就是冲突！)

*   **b) 字符串哈希 (Polynomial Hashing / P-进制哈希):**
    当我们处理字符串作为键值时，不能直接用除留余数法。我们需要先把字符串转换成一个大整数，然后再对这个大整数进行除留余数。
    **原理：** 我们可以把一个字符串看作是一个 P 进制的数字。
    例如，字符串 "abc" 可以看作是 `a * P^2 + b * P^1 + c * P^0`。
    `H("s1s2...sk") = (s1 * P^(k-1) + s2 * P^(k-2) + ... + sk * P^0) % M`
    其中 `s_i` 是字符 `s` 的 ASCII 码（或者自定义一个映射值），`P` 是一个质数（通常取 `31`, `131`, `13331` 等，避免与 `M` 有公约数），`M` 是一个大质数（通常取 `10^9 + 7`, `10^9 + 9` 等）。
    为了防止计算中间结果过大溢出，每一步计算都要取模 `M`。

    *   **例子：** 假设 `P = 31`, `M = 10^9 + 7`。
        字符串 "cat"
        `H('c') = 'c' - 'a' + 1 = 3`
        `H('a') = 'a' - 'a' + 1 = 1`
        `H('t') = 't' - 'a' + 1 = 20`
        `Hash("cat") = ( ( (3 * 31 + 1) % M * 31 ) % M + 20 ) % M`
        `= ( (93 + 1) % M * 31 + 20 ) % M`
        `= ( 94 % M * 31 + 20 ) % M`
        `= ( 2914 % M + 20 ) % M`
        `= ( 2914 + 20 ) % M`
        `= 2934 % M`
        最终得到一个哈希值。

#### 3. 哈希冲突 (Hash Collision) 怎么办？

无论哈希函数设计得多好，总会有不同的键值映射到同一个数组下标的情况，这就是**哈希冲突**。
**为什么会冲突？** 因为键值的集合是无限的（或者非常大），而哈希表的数组大小是有限的。根据**鸽巢原理**（如果你有10只鸽子，却只有9个鸽笼，那至少有一个鸽笼里会挤着两只或更多的鸽子），冲突是不可避免的。

**生活类比：** 快递分拣时，两个包裹的单号末尾都是“3”，都被分到3号货架。但3号货架只有一个位置，怎么办？

解决哈希冲突是哈希表设计的关键。主要有两种策略：

**a) 开放定址法 (Open Addressing):**
当发生冲突时，不把新的键值放在别处，而是在**哈希表（数组）中寻找下一个空闲的位置**。

*   **核心思想：** “这个位置被占了，那我就在附近找个空位！”
*   **查找方式：** 按照同样的探测序列查找，直到找到目标键值或空位。
*   **删除方式：** 不能直接删除，需要标记为“已删除”状态，否则会影响后续查找。

    *   **线性探测法 (Linear Probing):**
        这是最简单的开放定址法。
        当 `H(key)` 发生冲突时，就依次尝试 `(H(key) + 1) % M`, `(H(key) + 2) % M`, `(H(key) + 3) % M`... 直到找到一个空位。
        *   **优点：** 实现简单。
        *   **缺点：** 容易形成“聚集”现象（Primary Clustering），即连续的空闲位置被填满，导致后续插入和查找效率下降。

        *   **例子：** `M = 7`，插入 `10, 23, 5, 17`
            1.  `H(10) = 10 % 7 = 3`。 `h[3] = 10`。
            2.  `H(23) = 23 % 7 = 2`。 `h[2] = 23`。
            3.  `H(5) = 5 % 7 = 5`。 `h[5] = 5`。
            4.  `H(17) = 17 % 7 = 3`。 `h[3]` 已被 10 占用，发生冲突！
                *   尝试 `(3 + 1) % 7 = 4`。 `h[4]` 空闲。 `h[4] = 17`。
            哈希表状态：`[空, 空, 23, 10, 17, 5, 空]`

    *   **二次探测法 (Quadratic Probing):**
        为了缓解线性探测的聚集问题，它尝试 `(H(key) + 1^2) % M`, `(H(key) + 2^2) % M`, `(H(key) + 3^2) % M`... 或者 `(H(key) - 1^2) % M`, `(H(key) - 2^2) % M`...
        *   **优点：** 有效缓解聚集。
        *   **缺点：** 可能会出现某些位置永远无法被探测到。

**b) 拉链法 (Separate Chaining):**
当发生冲突时，不寻找下一个空位，而是在哈希表的每个“槽位”上，**挂一个链表（或动态数组）**，把所有哈希到这个槽位的键值都存储在这个链表中。

*   **核心思想：** “这个货架挤不下了，那就给它加一个副货架（链表），把冲突的包裹都放进去！”
*   **结构：** 哈希表是一个数组，数组的每个元素都是一个链表的头指针（或一个动态数组）。
*   **插入：** 计算 `H(key)`，将键值插入到对应槽位的链表末尾。
*   **查找：** 计算 `H(key)`，然后遍历对应槽位的链表，直到找到目标键值。
*   **删除：** 在对应链表中找到并删除节点。

    *   **例子：** `M = 7`，插入 `10, 23, 5, 17`
        1.  `H(10) = 3`。`h[3]` 链表：`[10]`。
        2.  `H(23) = 2`。`h[2]` 链表：`[23]`。
        3.  `H(5) = 5`。`h[5]` 链表：`[5]`。
        4.  `H(17) = 3`。与 10 冲突。将 17 加入 `h[3]` 的链表。
            `h[3]` 链表：`[10] -> [17]`。
        哈希表状态：
        `h[0]: 空`
        `h[1]: 空`
        `h[2]: [23]`
        `h[3]: [10] -> [17]`
        `h[4]: 空`
        `h[5]: [5]`
        `h[6]: 空`

*   **优点：**
    *   处理冲突简单，不会出现“死循环”找空位的问题。
    *   哈希表的装载因子（Load Factor，已存元素数 / 表大小）可以大于 1。
    *   删除操作相对简单。
*   **缺点：**
    *   需要额外的空间来存储链表节点（指针）。
    *   链表操作可能涉及更多内存访问，局部性不如开放定址法好。
    *   在最坏情况下（所有键值都哈希到同一个槽位），退化为链表，查找效率变为 O(N)。

#### 4. 总结与选择

| 特性       | 开放定址法           | 拉链法                     |
| :--------- | :------------------- | :------------------------- |
| 实现难度   | 相对简单（线性探测） | 需要模拟链表（稍复杂）     |
| 空间利用率 | 高（无额外指针开销） | 略低（需要额外指针）       |
| 聚集问题   | 存在（尤其线性探测） | 不存在                     |
| 装载因子   | 建议 < 0.7           | 可以 > 1                   |
| 删除       | 需特殊标记           | 链表节点删除               |
| 性能       | 冲突少时 O(1)        | 冲突少时 O(1)，冲突多时 O(链表长度) |

在实际编程竞赛中，如果你需要一个哈希表来存储键值对，并且键值是整数，开放定址法（特别是线性探测）配合一个足够大的数组（通常是数据量2-3倍的质数）是一个简单有效的选择。对于字符串键值，通常会结合多项式哈希和拉链法（或 C++ `std::unordered_map`）。

**由于 GESP 级别和“原生数组”的限制，我们本次课程的典型例题和模板将主要聚焦于使用开放定址法（线性探测）实现整数哈希表，以及字符串哈希函数的计算。**

### 典型例题精讲

#### 例题 1: 基于开放定址法（线性探测）的整数哈希表模拟

**题目描述：**
给定一个整数序列，请你实现一个哈希表，将这些整数插入到哈希表中。哈希表大小 `M` 固定为 `10007` (一个质数)。哈希函数使用除留余数法 `H(key) = key % M`。当发生冲突时，使用线性探测法解决冲突。最后，给定一些查询整数，判断它们是否在哈希表中。

**输入格式：**
第一行一个整数 `N`，表示要插入的整数个数。
第二行 `N` 个整数，表示要插入的序列。
第三行一个整数 `Q`，表示查询的整数个数。
第四行 `Q` 个整数，表示要查询的序列。

**输出格式：**
对于每个查询整数，如果存在于哈希表中输出 `YES`，否则输出 `NO`。

**数据范围：**
`1 <= N, Q <= 10000`
要插入和查询的整数范围 `0 <= key <= 10^9`

---

**思路分析：**

1.  **哈希表初始化：** 我们需要一个足够大的数组来作为哈希表。根据题目，`M = 10007`。我们可以定义一个 `int h[10007]`。由于键值都是非负数，我们可以用 `0` 来表示一个空位。但考虑到键值本身可能就是 `0`，所以更稳妥的做法是使用一个特殊值，比如 `-1`，来表示空位。
2.  **哈希函数：** `H(key) = key % 10007`。
3.  **插入操作：**
    *   计算 `key` 的初始哈希值 `pos = key % M`。
    *   从 `pos` 开始，循环检查 `h[pos]`：
        *   如果 `h[pos]` 是空位 (`-1`)，则将 `key` 存入 `h[pos]`，插入完成。
        *   如果 `h[pos]` 已经被 `key` 占用（即 `h[pos] == key`），说明 `key` 已经存在，无需重复插入，插入完成。
        *   如果 `h[pos]` 被其他键值占用（即 `h[pos] != key` 且 `h[pos] != -1`），则发生冲突。 `pos = (pos + 1) % M`，继续探测下一个位置。
    *   **注意：** 理论上线性探测可能一直找不到空位，但题目数据量 `N` 远小于 `M`，所以不会填满。
4.  **查找操作：**
    *   计算 `key` 的初始哈希值 `pos = key % M`。
    *   从 `pos` 开始，循环检查 `h[pos]`：
        *   如果 `h[pos]` 是空位 (`-1`)，说明从这个位置开始往后都没找到 `key`，查找失败。
        *   如果 `h[pos]` 是目标 `key`，查找成功。
        *   如果 `h[pos]` 是其他键值，则 `pos = (pos + 1) % M`，继续探测下一个位置。
    *   **注意：** 如果哈希表被填满，查找循环可能进入死循环。但本题 `N < M`，不会发生。

**时间复杂度：**
*   **最好情况 (无冲突)：** O(1)
*   **平均情况：** O(1) (前提是哈希函数均匀，装载因子低)
*   **最坏情况 (严重冲突)：** O(N) (所有元素都哈希到同一个位置，退化为线性查找)
本题中，由于 `M` 远大于 `N`，平均复杂度接近 O(1)。

---

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

const int M = 10007; // 哈希表的大小，选择一个质数
int h[M];            // 哈希表数组，存储键值

// 初始化哈希表
void initHashTable() {
    for (int i = 0; i < M; ++i) {
        h[i] = -1; // 用 -1 表示空位
    }
}

// 插入操作
void insert(int k) {
    int p = k % M; // 计算初始哈希地址

    // 线性探测解决冲突
    while (h[p] != -1 && h[p] != k) { // 如果当前位置非空且不是目标键值
        p = (p + 1) % M;               // 探测下一个位置
    }

    if (h[p] == -1) { // 找到空位，插入
        h[p] = k;
    }
    // else if (h[p] == k) { // 键值已存在，无需重复插入
    //     // do nothing
    // }
}

// 查找操作
bool query(int k) {
    int p = k % M; // 计算初始哈希地址

    // 线性探测查找
    while (h[p] != -1 && h[p] != k) { // 如果当前位置非空且不是目标键值
        p = (p + 1) % M;               // 探测下一个位置
    }

    return h[p] == k; // 如果找到目标键值，返回 true；否则（遇到空位或遍历完）返回 false
}

int main() {
    // 提高输入输出效率
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    initHashTable(); // 初始化哈希表

    int n;
    std::cin >> n; // 读取插入数量
    for (int i = 0; i < n; ++i) {
        int k;
        std::cin >> k;
        insert(k); // 插入整数
    }

    int q;
    std::cin >> q; // 读取查询数量
    for (int i = 0; i < q; ++i) {
        int k;
        std::cin >> k;
        if (query(k)) {
            std::cout << "YES\n";
        } else {
            std::cout << "NO\n";
        }
    }

    return 0;
}
```

#### 例题 2: 字符串集合去重 (使用字符串哈希)

**题目描述：**
给定 `N` 个字符串，请统计其中有多少个**不同的**字符串。

**输入格式：**
第一行一个整数 `N`。
接下来 `N` 行，每行一个字符串。

**输出格式：**
一个整数，表示不同字符串的数量。

**数据范围：**
`1 <= N <= 10000`
每个字符串长度 `1 <= len <= 100`
字符串只包含小写英文字母。

---

**思路分析：**

1.  **字符串哈希函数：** 我们需要一个函数将每个字符串转换成一个唯一的（或极大概率唯一的）整数哈希值。这里使用多项式哈希。
    *   选择一个基数 `P` (例如 `31`, `131`) 和一个大质数 `MOD` (例如 `10^9 + 7`, `10^9 + 9`)。
    *   哈希值的计算公式：`H(s) = (s[0]*P^(L-1) + s[1]*P^(L-2) + ... + s[L-1]*P^0) % MOD`。
    *   为了避免中间结果溢出，每一步乘法和加法后都要取模 `MOD`。
2.  **哈希表存储：** 我们将计算出的字符串哈希值存入一个哈希表。由于哈希值是整数，我们可以复用例题1中开放定址法（线性探测）的整数哈希表。
    *   哈希表大小 `M` 仍然可以取 `10007` 或更大，用来存储字符串哈希值。
    *   用 `-1` 表示空位，或用 `0` (如果哈希值不可能为 `0`)。
3.  **去重计数：**
    *   初始化一个计数器 `count = 0`。
    *   对于每个输入的字符串：
        *   计算其哈希值 `h_val`。
        *   在我们的哈希表 (`h[]`) 中查找 `h_val`。
        *   如果 `h_val` 不在哈希表中（即 `query(h_val)` 返回 `false`）：
            *   将 `h_val` 插入哈希表 (`insert(h_val)`)。
            *   `count` 加 1。
    *   最终 `count` 就是不同字符串的数量。

**注意：** 字符串哈希存在极小的概率发生哈希冲突（不同的字符串计算出相同的哈希值），但对于竞赛题目，通常在选择合适的 `P` 和 `MOD` 后，这种概率足够小，可以忽略。如果需要绝对避免，可以使用双哈希（计算两个不同的哈希值，只有两个都相同才认为是同一个字符串）。本题只用单哈希即可。

**时间复杂度：**
*   计算单个字符串哈希值：O(L)，其中 L 是字符串长度。
*   插入/查找哈希表：平均 O(1)。
*   总复杂度：N 个字符串，每个字符串平均长度 L。总时间复杂度约为 O(N * L)。

---

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

const int P = 131;        // 字符串哈希的基数，选择一个质数
const int MOD = 1e9 + 7;  // 字符串哈希的模数，选择一个大质数

const int HASH_TABLE_SIZE = 10007; // 存储哈希值的哈希表大小
long long h_table[HASH_TABLE_SIZE]; // 哈希表数组，存储字符串的哈希值

// 初始化哈希表
void initHashTable() {
    for (int i = 0; i < HASH_TABLE_SIZE; ++i) {
        h_table[i] = -1; // 用 -1 表示空位，因为字符串哈希值通常不会是 -1
    }
}

// 计算字符串的哈希值
long long getHash(const std::string& s) {
    long long currentHash = 0;
    for (char c : s) {
        currentHash = (currentHash * P % MOD + (c - 'a' + 1)) % MOD;
    }
    return currentHash;
}

// 插入字符串哈希值到哈希表 (开放定址法 - 线性探测)
void insertHashValue(long long k) {
    int p = k % HASH_TABLE_SIZE; // 计算初始哈希地址

    // 线性探测解决冲突
    while (h_table[p] != -1 && h_table[p] != k) { // 如果当前位置非空且不是目标键值
        p = (p + 1) % HASH_TABLE_SIZE;               // 探测下一个位置
    }

    if (h_table[p] == -1) { // 找到空位，插入
        h_table[p] = k;
    }
    // else if (h_table[p] == k) { // 键值已存在，无需重复插入
    //     // do nothing
    // }
}

// 查询字符串哈希值是否存在于哈希表
bool queryHashValue(long long k) {
    int p = k % HASH_TABLE_SIZE; // 计算初始哈希地址

    // 线性探测查找
    while (h_table[p] != -1 && h_table[p] != k) { // 如果当前位置非空且不是目标键值
        p = (p + 1) % HASH_TABLE_SIZE;               // 探测下一个位置
    }

    return h_table[p] == k; // 如果找到目标键值，返回 true；否则（遇到空位或遍历完）返回 false
}

int main() {
    // 提高输入输出效率
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    initHashTable(); // 初始化存储哈希值的哈希表

    int n;
    std::cin >> n; // 读取字符串数量
    int distinctCount = 0; // 记录不同字符串的数量

    for (int i = 0; i < n; ++i) {
        std::string s;
        std::cin >> s;
        long long currentHash = getHash(s); // 计算当前字符串的哈希值

        if (!queryHashValue(currentHash)) { // 如果这个哈希值不在哈希表中
            insertHashValue(currentHash);   // 插入
            distinctCount++;                // 不同字符串数量加一
        }
    }

    std::cout << distinctCount << std::endl; // 输出不同字符串的数量

    return 0;
}
```

### 代码实现模板

这里提供一个基于**开放定址法（线性探测）**的整数哈希表模板。你可以根据需要修改哈希函数和哈希表大小。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，方便竞赛使用

// 定义哈希表的最大容量，通常选择一个质数，且略大于预期存储的元素数量
// 经验值：如果预期存储 N 个元素，哈希表大小 M 可以是 2*N 到 3*N 之间的质数
const int MAX_HASH_TABLE_SIZE = 200003; // 例如，一个足够大的质数
int h[MAX_HASH_TABLE_SIZE];             // 哈希表数组，存储键值

// 定义一个特殊值表示空位。
// 如果键值可能包含 0，则不能用 0 表示空位，需要选择一个不可能出现的键值，
// 或者使用一个额外的布尔数组标记位置是否被占用。
const int EMPTY_SLOT = -1; // 假设键值都是非负数，用 -1 表示空位

// 哈希函数
// 这里使用除留余数法作为示例
// k: 要哈希的键值
// 返回值: 键值对应的哈希地址（数组下标）
int hashFunction(int k) {
    return k % MAX_HASH_TABLE_SIZE;
}

// 初始化哈希表
// 将所有槽位标记为空
void initHashTable() {
    for (int i = 0; i < MAX_HASH_TABLE_SIZE; ++i) {
        h[i] = EMPTY_SLOT;
    }
}

// 插入操作
// k: 要插入的键值
void insert(int k) {
    int p = hashFunction(k); // 计算初始哈希地址

    // 线性探测解决冲突：
    // 如果当前位置非空且不是目标键值，则探测下一个位置
    while (h[p] != EMPTY_SLOT && h[p] != k) {
        p = (p + 1) % MAX_HASH_TABLE_SIZE; // 循环到数组末尾后回到开头
    }

    // 找到空位或者键值已存在，进行插入
    if (h[p] == EMPTY_SLOT) {
        h[p] = k; // 插入键值
    }
    // else if (h[p] == k) {
    //     // 键值 k 已经存在，无需重复插入。
    //     // 如果需要计数，可以在这里增加 k 的计数。
    // }
}

// 查找操作
// k: 要查找的键值
// 返回值: 如果找到 k，返回 true；否则返回 false
bool query(int k) {
    int p = hashFunction(k); // 计算初始哈希地址

    // 线性探测查找：
    // 如果当前位置非空且不是目标键值，则探测下一个位置
    while (h[p] != EMPTY_SLOT && h[p] != k) {
        p = (p + 1) % MAX_HASH_TABLE_SIZE;
    }

    // 如果找到目标键值，返回 true；否则（遇到空位）返回 false
    return h[p] == k;
}

// --- 字符串哈希函数模板 ---

// 字符串哈希的基数，通常选择一个质数
const int STRING_HASH_P = 131;
// 字符串哈希的模数，通常选择一个大质数，避免溢出和减少冲突
const int STRING_HASH_MOD = 1e9 + 7;

// 计算字符串的哈希值
// s: 待计算哈希值的字符串
// 返回值: 字符串对应的哈希值（一个 long long 整数）
long long getStringHash(const std::string& s) {
    long long currentHash = 0;
    for (char c : s) {
        // 将字符映射为 1-26 的数字，并进行 P 进制计算
        // 每次计算后都取模，防止中间结果溢出
        currentHash = (currentHash * STRING_HASH_P % STRING_HASH_MOD + (c - 'a' + 1)) % STRING_HASH_MOD;
    }
    return currentHash;
}

// main 函数示例 (如何使用上述模板)
int main() {
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    initHashTable(); // 初始化整数哈希表

    // 示例：插入一些整数
    insert(10);
    insert(23);
    insert(5);
    insert(17); // 17 和 10 可能会发生冲突，取决于 MAX_HASH_TABLE_SIZE

    // 示例：查询整数
    if (query(23)) {
        std::cout << "23 is in the hash table.\n";
    } else {
        std::cout << "23 is not in the hash table.\n";
    }

    if (query(100)) {
        std::cout << "100 is in the hash table.\n";
    } else {
        std::cout << "100 is not in the hash table.\n";
    }

    // 示例：使用字符串哈希函数
    std::string s1 = "apple";
    std::string s2 = "banana";
    std::string s3 = "apple";

    long long hash1 = getStringHash(s1);
    long long hash2 = getStringHash(s2);
    long long hash3 = getStringHash(s3);

    std::cout << "Hash of " << s1 << ": " << hash1 << "\n";
    std::cout << "Hash of " << s2 << ": " << hash2 << "\n";
    std::cout << "Hash of " << s3 << ": " << hash3 << "\n";

    // 可以将字符串哈希值存入另一个哈希表进行去重或查找
    // 这里不再重复实现，原理与整数哈希表相同。

    return 0;
}
```
