## 深度知识点讲解

### 1. 什么是哈希？

“哈希”（Hash），从字面意思看，有点像“散列”或者“打乱”。在计算机科学里，哈希是一种把任意长度的输入（比如一个字符串、一个文件）通过一个函数，转换成一个固定长度输出（通常是一个数字）的过程。这个输出值就叫做**哈希值**（Hash Value），也叫哈希码或散列值。

生活中的例子：
*   **图书馆的图书编号**：每一本书都有一个唯一的编号，这个编号就是它的“哈希值”，通过编号就能快速找到书。
*   **身份证号**：每个人都有一个唯一的身份证号，通过身份证号就可以识别一个人。
*   **文件校验码**：下载软件时，经常会看到一个MD5或SHA1校验码。这个校验码就是文件的哈希值。如果下载的文件和官网给出的校验码一致，说明文件没有被篡改。

### 2. 字符串哈希的本质：把字符串看作一个P进制数

字符串哈希，顾名思义，就是把一个字符串映射成一个数字。最常用、最有效的方法就是**多项式哈希**（Polynomial Hashing）。

它的核心思想是：**把一个字符串看作是一个P进制的数字！**

我们知道，一个十进制数 `123` 可以表示为 `1 * 10^2 + 2 * 10^1 + 3 * 10^0`。
类似地，一个字符串 `S = "abc"`，我们可以把每个字符映射成一个数字（比如它的ASCII码或者给它一个自定义的小数字），然后把整个字符串看作一个P进制数。

假设字符 'a' 对应数字 `1`，'b' 对应 `2`，'c' 对应 `3`。
那么字符串 `"abc"` 的哈希值就可以表示为：
`H("abc") = a * P^2 + b * P^1 + c * P^0`
（这里为了方便，我们把字符直接当成了数字）

**公式化表示：**
对于一个长度为 `L` 的字符串 `S = S[0]S[1]...S[L-1]`，它的哈希值 `H(S)` 可以计算为：
`H(S) = (S[0] * P^(L-1) + S[1] * P^(L-2) + ... + S[L-1] * P^0)`

**思考：** 为什么我们需要一个 `P`（基数）？
因为字符串是字符序列，每个字符都有其位置。`P` 就像是进制的“底”，它确保了不同位置的字符对哈希值的贡献不同，从而让不同的字符串有不同的哈希值。

### 3. 模数 M 的作用：防止溢出与均匀分布

如果字符串很长，或者 `P` 很大，那么计算出来的哈希值 `H(S)` 可能会非常非常大，超出计算机整数类型的表示范围（溢出）。
为了解决这个问题，我们在每一步计算中都对一个大质数 `M` 取模。

**带模数的哈希公式：**
`H(S) = (S[0] * P^(L-1) + S[1] * P^(L-2) + ... + S[L-1] * P^0) % M`
更实际的计算方式是**递推**：
`h[0] = S[0]`
`h[1] = (S[0] * P + S[1]) % M`
`h[2] = (S[0] * P^2 + S[1] * P + S[2]) % M`
...
`h[i] = (h[i-1] * P + S[i]) % M`

这里 `S[i]` 表示字符串的第 `i` 个字符（从0开始）。在计算时，我们通常会将字符 `S[i]` 转换成一个数字，比如 `S[i] - 'a' + 1` 或者直接用 `S[i]` 的ASCII码。

**思考：** 为什么 `M` 要选择一个大质数？
1.  **防止溢出**：把结果控制在一个可接受的范围内。
2.  **减少冲突**：质数作为模数，能让哈希值分布更均匀，从而降低不同字符串产生相同哈希值的概率。

### 4. 基数 P 和模数 M 的选择

选择合适的 `P` 和 `M` 是字符串哈希成功的关键。

*   **基数 `P` 的选择：**
    *   `P` 通常选择一个**大于字符集大小的质数**。
    *   如果只包含小写字母，字符集大小是26，那么 `P` 可以选 31, 131, 13331 等。
    *   如果包含所有ASCII字符，字符集大小是128或256，那么 `P` 可以选 233, 2333, 23333 等。
    *   **为什么是质数？** 质数作为基数可以更好地“打散”字符串，使得不同的字符串产生相同哈希值的概率降低。
    *   **为什么大于字符集大小？** 避免不同字符 `c1, c2` 在 `P` 进制下被混淆。例如，如果 `P=10`，那么 `S[i]` 的值就不能超过9。

*   **模数 `M` 的选择：**
    *   `M` 通常选择一个**非常大的质数**，例如 `10^9 + 7` 或 `10^9 + 9`。
    *   **为什么是质数？** 同 `P` 的原因，为了更好地分布哈希值，减少冲突。
    *   **`unsigned long long` 自然溢出：** 有时也会选择 `2^64` 作为模数（即不取模，让 `unsigned long long` 类型自然溢出）。这种方式的好处是省去了取模操作，速度快，但冲突概率可能略高。在竞赛中，通常建议同时使用一个大质数取模和 `unsigned long long` 自然溢出，或者使用两个不同的 `P` 和 `M` 进行**双哈希**。

### 5. 哈希冲突：指纹相同，但人不同？

哈希冲突是指**两个不同的字符串，计算出来的哈希值却相同**。
这就像两个不同的人，身份证号的后几位碰巧一样。
哈希冲突是无法完全避免的，因为字符串的数量是无限的，而哈希值的范围是有限的。

**如何减少哈希冲突？**
1.  **选择合适的 `P` 和 `M`**：前面已经讲过，大质数能有效降低冲突。
2.  **双哈希（Double Hashing）**：这是最常用的方法。我们不只用一组 `(P, M)` 计算哈希值，而是用两组不同的 `(P1, M1)` 和 `(P2, M2)` 分别计算字符串的两个哈希值。
    *   如果两个字符串的 `(H1, H2)` 都相同，那么它们相同的概率就非常非常高了，几乎可以认为是相同的。
    *   例如，`(P1=131, M1=1e9+7)` 和 `(P2=233, M2=1e9+9)`。

### 6. 预处理与快速查询子串哈希值

字符串哈希最强大的地方在于，我们可以通过预处理，在 `O(1)` 时间内快速查询任意子串的哈希值。

**核心思想：前缀哈希**
我们预先计算出字符串所有前缀的哈希值。
假设字符串 `S` 的长度为 `N`。
`h[i]` 表示 `S[0...i-1]` 这个前缀的哈希值（即长度为 `i` 的前缀）。
`h[0] = 0`
`h[1] = S[0]`
`h[2] = (S[0] * P + S[1]) % M`
`h[i] = (h[i-1] * P + S[i-1]) % M` （注意这里 `S[i-1]` 是第 `i` 个字符）

**预处理 `P` 的幂次：**
为了计算子串哈希值，我们还需要预处理 `P` 的幂次。
`p_pow[i]` 表示 `P^i % M`。
`p_pow[0] = 1`
`p_pow[i] = (p_pow[i-1] * P) % M`

**如何计算子串 `S[L...R]` 的哈希值？**
假设我们要查询从下标 `L` 到 `R` 的子串 `S[L...R]` 的哈希值。这个子串的长度是 `len = R - L + 1`。
`H(S[L...R]) = (S[L] * P^(len-1) + S[L+1] * P^(len-2) + ... + S[R] * P^0) % M`

观察前缀哈希：
`h[R+1] = (S[0]*P^R + S[1]*P^(R-1) + ... + S[L-1]*P^(len) + S[L]*P^(len-1) + ... + S[R]*P^0) % M`
`h[L] = (S[0]*P^(L-1) + S[1]*P^(L-2) + ... + S[L-1]*P^0) % M`

如果我们用 `h[R+1]` 减去 `h[L]` 乘以 `P^len`，会发生什么？
`h[R+1] - h[L] * P^len`
`= (S[0]*P^R + ... + S[L-1]*P^(len) + S[L]*P^(len-1) + ... + S[R]*P^0)`
`- (S[0]*P^(L-1) + ... + S[L-1]*P^0) * P^len`
`= (S[0]*P^R + ... + S[L-1]*P^(len) + S[L]*P^(len-1) + ... + S[R]*P^0)`
`- (S[0]*P^(R) + ... + S[L-1]*P^(len))`
`= (S[L]*P^(len-1) + ... + S[R]*P^0)`

**最终公式：**
`H(S[L...R]) = (h[R+1] - (h[L] * p_pow[len]) % M + M) % M`
（`+ M` 再 `% M` 是为了处理负数结果，因为 `h[R+1]` 减去 `h[L] * p_pow[len]` 可能得到负数）

**步骤总结：**
1.  **预处理 `p_pow` 数组**：`p_pow[i] = P^i % M`。
2.  **预处理 `h` 数组**：`h[i]` 存储字符串 `S` 的前 `i` 个字符的哈希值。
3.  **查询子串 `S[L...R]`**：使用公式 `(h[R+1] - (h[L] * p_pow[R-L+1]) % M + M) % M`。

### 7. 常见误区与注意事项

*   **P 和 M 的选择**：不要随便选，要选大质数，并且 P 要大于字符集。
*   **计算顺序**：哈希值计算和幂次计算都要及时取模，避免中间结果溢出。
*   **负数取模**：C++中负数取模结果仍为负数，所以 `(a - b % M + M) % M` 是安全的写法。
*   **下标问题**：字符串下标从0开始，而前缀哈希数组 `h[i]` 通常表示前 `i` 个字符的哈希，即 `S[0...i-1]`。所以 `h[i]` 对应字符串的第 `i` 个字符 `S[i-1]`。查询 `S[L...R]` 子串时，对应的 `h` 数组下标是 `h[R+1]` 和 `h[L]`。

### 例题1：判断两个字符串是否相等

**题目描述：** 给定两个字符串 `s` 和 `t`，判断它们是否完全相同。

**思路分析：**
传统的做法是逐字符比较，时间复杂度是 `O(min(len(s), len(t)))`。
使用哈希，我们可以：
1.  计算字符串 `s` 的哈希值 `H(s)`。
2.  计算字符串 `t` 的哈希值 `H(t)`。
3.  如果 `H(s) == H(t)` 且 `len(s) == len(t)`，则我们认为它们相等。如果使用双哈希，则判断两个哈希值对是否都相等。
这种方法的时间复杂度是 `O(len(s) + len(t))`，比逐字符比较可能快不了多少，但这是哈希的基础。

**代码实现：**
```cpp
#include <iostream>
#include <string>
#include <vector> // GESP 7级允许使用sort，但vector/stack/queue不允许，这里用原生数组

// 定义基数和模数
const long long P = 131; // 基数
const long long M = 1e9 + 7; // 模数

// 计算一个字符串的哈希值
long long calc_string_hash(const std::string& s) {
    long long h = 0;
    for (char c : s) {
        h = (h * P + (c - 'a' + 1)) % M; // 将字符映射为1-26的数字
    }
    return h;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s1, s2;
    std::cin >> s1 >> s2;

    if (s1.length() != s2.length()) { // 长度不同肯定不相等
        std::cout << "No\n";
        return 0;
    }

    long long hash1 = calc_string_hash(s1);
    long long hash2 = calc_string_hash(s2);

    if (hash1 == hash2) {
        std::cout << "Yes\n";
    } else {
        std::cout << "No\n";
    }

    return 0;
}
```
**复杂度分析：**
*   时间复杂度：`O(len(s1) + len(s2))`，因为需要遍历两个字符串一次。
*   空间复杂度：`O(1)`，只存储了几个变量。

### 例题2：判断一个字符串中是否存在某个子串（模式匹配）

**题目描述：** 给定一个主字符串 `t` (文本串) 和一个模式字符串 `p` (模式串)，判断 `p` 是否是 `t` 的子串。

**思路分析：**
传统的KMP算法可以在 `O(len(t) + len(p))` 时间内解决。
使用哈希：
1.  计算模式串 `p` 的哈希值 `H(p)`。
2.  预处理文本串 `t` 的前缀哈希值 `h_t` 数组和 `P` 的幂次 `p_pow` 数组。
3.  遍历文本串 `t`，计算所有长度为 `len(p)` 的子串的哈希值。
4.  将这些子串的哈希值与 `H(p)` 进行比较。如果找到相等的，说明模式串存在。

**代码实现：**
```cpp
#include <iostream>
#include <string>
#include <vector> // 仅作为注释，实际用原生数组

// 定义基数和模数
const long long P = 131;
const long long M = 1e9 + 7;

// 定义全局数组，用于存储前缀哈希和P的幂次
const int MAXN = 100005; // 假设字符串最大长度
long long h[MAXN];       // h[i] 存储前i个字符的哈希值
long long p_pow[MAXN];   // p_pow[i] 存储 P^i % M

// 预处理函数
void init_hash_data(int n) {
    p_pow[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow[i] = (p_pow[i - 1] * P) % M;
    }
}

// 计算文本串的前缀哈希
void build_prefix_hash(const std::string& s) {
    h[0] = 0; // 0长度前缀哈希为0
    for (int i = 0; i < s.length(); ++i) {
        h[i + 1] = (h[i] * P + (s[i] - 'a' + 1)) % M;
    }
}

// 获取子串 s[l...r] 的哈希值 (l, r是0-based下标)
long long get_substring_hash(int l, int r) {
    long long len = r - l + 1;
    long long res = (h[r + 1] - (h[l] * p_pow[len]) % M + M) % M;
    return res;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string t, p; // t为文本串，p为模式串
    std::cin >> t >> p;

    int n = t.length();
    int m = p.length();

    if (m > n) { // 模式串比文本串长，不可能存在
        std::cout << "No\n";
        return 0;
    }

    init_hash_data(n); // 预处理P的幂次，最大到n
    build_prefix_hash(t); // 预处理文本串的前缀哈希

    long long pattern_hash = 0;
    for (char c : p) { // 计算模式串的哈希值
        pattern_hash = (pattern_hash * P + (c - 'a' + 1)) % M;
    }

    bool found = false;
    for (int i = 0; i <= n - m; ++i) { // 遍历文本串所有可能的起始位置
        // 获取 t[i...i+m-1] 这个子串的哈希值
        long long current_sub_hash = get_substring_hash(i, i + m - 1);
        if (current_sub_hash == pattern_hash) {
            found = true;
            break;
        }
    }

    if (found) {
        std::cout << "Yes\n";
    } else {
        std::cout << "No\n";
    }

    return 0;
}
```
**复杂度分析：**
*   时间复杂度：`O(n + m)`。预处理 `P` 幂次 `O(n)`，预处理文本串哈希 `O(n)`，计算模式串哈希 `O(m)`，遍历比较 `O(n)` 次，每次 `O(1)`。
*   空间复杂度：`O(n)`，用于存储 `h` 和 `p_pow` 数组。

### 例题3：求字符串中最长重复子串的长度

**题目描述：** 给定一个字符串 `s`，找出其中出现至少两次的最长子串的长度。

**思路分析：**
这个问题是经典的“最长重复子串”问题。如果直接枚举所有子串并比较，复杂度太高。
我们可以发现，如果存在长度为 `K` 的重复子串，那么也一定存在长度为 `K-1` 的重复子串。这具有**单调性**，提示我们可以使用**二分查找**来解决。

1.  **二分答案 `k`**：答案的范围是 `[0, n-1]` (n是字符串长度)。
2.  **`check(k)` 函数**：判断是否存在长度为 `k` 的重复子串。
    *   如果 `k=0`，认为存在。
    *   预处理字符串 `s` 的前缀哈希 `h` 数组和 `p_pow` 数组。
    *   遍历字符串 `s`，生成所有长度为 `k` 的子串的哈希值。
    *   将这些哈希值存储在一个数组中。
    *   对这个数组进行**排序**。
    *   遍历排序后的数组，如果发现相邻的哈希值相同，说明存在长度为 `k` 的重复子串，返回 `true`。
    *   如果遍历完都没有发现相同的哈希值，返回 `false`。

**代码实现：**
```cpp
#include <iostream>
#include <string>
#include <algorithm> // GESP 7级允许使用sort

// 定义基数和模数
const long long P = 131;
const long long M = 1e9 + 7; // 使用双哈希效果更好，这里只用一组

// 定义全局数组
const int MAXN = 100005;
long long h[MAXN];       // 前缀哈希
long long p_pow[MAXN];   // P的幂次
long long temp_hashes[MAXN]; // 临时存储子串哈希值

// 预处理P的幂次
void init_hash_data(int n) {
    p_pow[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow[i] = (p_pow[i - 1] * P) % M;
    }
}

// 建立前缀哈希
void build_prefix_hash(const std::string& s) {
    h[0] = 0;
    for (int i = 0; i < s.length(); ++i) {
        h[i + 1] = (h[i] * P + (s[i] - 'a' + 1)) % M;
    }
}

// 获取子串 s[l...r] 的哈希值 (l, r是0-based下标)
long long get_substring_hash(int l, int r) {
    long long len = r - l + 1;
    long long res = (h[r + 1] - (h[l] * p_pow[len]) % M + M) % M;
    return res;
}

// 检查是否存在长度为 len 的重复子串
bool check(const std::string& s, int len, int n) {
    if (len == 0) return true; // 长度为0的空串总是重复的

    // 提取所有长度为 len 的子串的哈希值
    int cnt = 0;
    for (int i = 0; i <= n - len; ++i) {
        temp_hashes[cnt++] = get_substring_hash(i, i + len - 1);
    }

    // 排序并查找是否有重复哈希值
    std::sort(temp_hashes, temp_hashes + cnt);

    for (int i = 0; i < cnt - 1; ++i) {
        if (temp_hashes[i] == temp_hashes[i + 1]) {
            return true; // 找到重复哈希值
        }
    }
    return false; // 没有找到重复哈希值
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s;
    std::cin >> s;
    int n = s.length();

    init_hash_data(n);
    build_prefix_hash(s);

    int l = 0, r = n - 1; // 二分查找长度的范围
    int ans = 0;

    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (mid == 0) { // 长度为0的子串永远重复
            ans = std::max(ans, mid);
            l = mid + 1;
            continue;
        }
        if (check(s, mid, n)) {
            ans = std::max(ans, mid); // 存在长度为mid的重复子串，尝试更长的
            l = mid + 1;
        } else {
            r = mid - 1; // 不存在，尝试更短的
        }
    }

    std::cout << ans << "\n";

    return 0;
}
```
**复杂度分析：**
*   时间复杂度：`O(N log N * log N)`。外层二分 `log N` 次，每次 `check` 函数中，生成 `N` 个哈希值 `O(N)`，排序 `O(N log N)`。总计 `O(N log N * log N)`。
*   空间复杂度：`O(N)`，存储 `h`，`p_pow` 和 `temp_hashes` 数组。

### 双哈希的简单示例

在上面的例题中，为了提高准确率，我们通常会使用双哈希。
只需要定义两组 `P` 和 `M`，然后计算两个哈希值，并把它们作为一个 `pair<long long, long long>` 来处理。

```cpp
// 定义两组基数和模数
const long long P1 = 131;
const long long M1 = 1e9 + 7;
const long long P2 = 233;
const long long M2 = 1e9 + 9;

// 全局数组 for P1, M1
long long h1[MAXN];
long long p_pow1[MAXN];

// 全局数组 for P2, M2
long long h2[MAXN];
long long p_pow2[MAXN];

// 预处理所有数据
void init_all_hash_data(int n) {
    p_pow1[0] = 1;
    p_pow2[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow1[i] = (p_pow1[i - 1] * P1) % M1;
        p_pow2[i] = (p_pow2[i - 1] * P2) % M2;
    }
}

// 建立前缀哈希
void build_all_prefix_hash(const std::string& s) {
    h1[0] = 0;
    h2[0] = 0;
    for (int i = 0; i < s.length(); ++i) {
        long long val = (s[i] - 'a' + 1);
        h1[i + 1] = (h1[i] * P1 + val) % M1;
        h2[i + 1] = (h2[i] * P2 + val) % M2;
    }
}

// 获取子串 s[l...r] 的双哈希值
std::pair<long long, long long> get_double_substring_hash(int l, int r) {
    long long len = r - l + 1;
    long long res1 = (h1[r + 1] - (h1[l] * p_pow1[len]) % M1 + M1) % M1;
    long long res2 = (h2[r + 1] - (h2[l] * p_pow2[len]) % M2 + M2) % M2;
    return {res1, res2};
}

// check 函数中比较时，就比较 std::pair<long long, long long>
// if (temp_hashes[i] == temp_hashes[i+1])
```
通过这种方式，哈希冲突的概率会大大降低，通常认为可以忽略不计。

## 代码实现模板

为了方便快速应用字符串哈希，我们可以将其封装成一个模板。

```cpp
#include <iostream>
#include <string>
#include <algorithm> // 允许使用 std::sort

// 使用 Allman 风格大括号

// 定义哈希参数
// 通常使用两组 (P, M) 进行双哈希，以降低冲突概率
const long long P1 = 131;        // 第一个基数
const long long M1 = 1e9 + 7;    // 第一个模数 (大质数)

const long long P2 = 233;        // 第二个基数
const long long M2 = 1e9 + 9;    // 第二个模数 (另一个大质数)

// 字符串最大长度限制，根据题目要求调整
const int MAX_LEN = 100005;

// === 存储预处理数据 (原生数组) ===
// p_pow1[i] 存储 P1^i % M1
long long p_pow1[MAX_LEN];
// h1[i] 存储字符串前 i 个字符的哈希值 (使用 P1, M1)
long long h1[MAX_LEN];

// p_pow2[i] 存储 P2^i % M2
long long p_pow2[MAX_LEN];
// h2[i] 存储字符串前 i 个字符的哈希值 (使用 P2, M2)
long long h2[MAX_LEN];

// === 预处理函数 ===
// 在 main 函数开始时调用一次，预计算所有 P 的幂次
void init_hash_powers(int n) {
    p_pow1[0] = 1;
    p_pow2[0] = 1;
    for (int i = 1; i <= n; ++i) {
        p_pow1[i] = (p_pow1[i - 1] * P1) % M1;
        p_pow2[i] = (p_pow2[i - 1] * P2) % M2;
    }
}

// === 构建前缀哈希函数 ===
// 根据给定的字符串 s 构建其前缀哈希数组
void build_prefix_hashes(const std::string& s) {
    h1[0] = 0; // 0长度前缀的哈希值为0
    h2[0] = 0;

    // 遍历字符串，计算每个前缀的哈希值
    for (int i = 0; i < s.length(); ++i) {
        // 将字符 s[i] 映射为一个正整数，例如 'a' -> 1, 'b' -> 2
        long long val = (s[i] - 'a' + 1);

        // 使用 P1, M1 更新第一个哈希值
        h1[i + 1] = (h1[i] * P1 + val) % M1;
        // 使用 P2, M2 更新第二个哈希值
        h2[i + 1] = (h2[i] * P2 + val) % M2;
    }
}

// === 查询子串哈希函数 ===
// 获取字符串 s 中从下标 l 到 r (0-based) 的子串的双哈希值
// 返回一个 pair，包含两个哈希值
std::pair<long long, long long> get_substring_double_hash(int l, int r) {
    // 子串的长度
    long long len = r - l + 1;

    // 计算第一个哈希值
    long long res1 = (h1[r + 1] - (h1[l] * p_pow1[len]) % M1 + M1) % M1;
    // 计算第二个哈希值
    long long res2 = (h2[r + 1] - (h2[l] * p_pow2[len]) % M2 + M2) % M2;

    return {res1, res2};
}

// === 主函数示例 ===
int main() {
    // 优化输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string s;
    std::cin >> s; // 输入字符串
    int n = s.length(); // 字符串长度

    // 1. 预处理 P 的幂次
    init_hash_powers(n);

    // 2. 构建字符串 s 的前缀哈希数组
    build_prefix_hashes(s);

    // 示例：查询子串 s[0...2] 和 s[3...5] 的哈希值，并判断是否相同
    // 假设 s = "abacaba"
    // s[0...2] 是 "aba"
    // s[3...5] 是 "cab"
    // s[4...6] 是 "aba"

    std::pair<long long, long long> hash_sub1 = get_substring_double_hash(0, 2); // "aba"
    std::pair<long long, long long> hash_sub2 = get_substring_double_hash(4, 6); // "aba"
    std::pair<long long, long long> hash_sub3 = get_substring_double_hash(3, 5); // "cab"

    std::cout << "Hash of s[0...2] ('aba'): (" << hash_sub1.first << ", " << hash_sub1.second << ")\n";
    std::cout << "Hash of s[4...6] ('aba'): (" << hash_sub2.first << ", " << hash_sub2.second << ")\n";
    std::cout << "Hash of s[3...5] ('cab'): (" << hash_sub3.first << ", " << hash_sub3.second << ")\n";

    if (hash_sub1 == hash_sub2) {
        std::cout << "Substrings s[0...2] and s[4...6] are considered identical.\n";
    } else {
        std::cout << "Substrings s[0...2] and s[4...6] are considered different.\n";
    }

    if (hash_sub1 == hash_sub3) {
        std::cout << "Substrings s[0...2] and s[3...5] are considered identical.\n";
    } else {
        std::cout << "Substrings s[0...2] and s[3...5] are considered different.\n";
    }

    return 0;
}
```

**注释说明：**
*   `MAX_LEN`：根据题目数据范围设置，保证数组不会越界。
*   `P1, M1, P2, M2`：两组基数和模数，用于双哈希。
*   `p_pow1, p_pow2`：分别存储 `P1` 和 `P2` 的幂次。
*   `h1, h2`：分别存储基于 `P1, M1` 和 `P2, M2` 计算的前缀哈希值。
*   `init_hash_powers(n)`：在程序开始时调用，预计算所有可能用到的 `P` 的幂次。
*   `build_prefix_hashes(s)`：在获取字符串 `s` 后调用，计算 `s` 的所有前缀哈希值。
*   `get_substring_double_hash(l, r)`：查询 `s[l...r]` 子串的哈希值，返回一个 `std::pair`。
*   **字符映射**：`s[i] - 'a' + 1` 将小写字母 'a' 映射为 1，'b' 映射为 2，以此类推。如果字符串包含大写字母、数字或其他字符，需要调整映射方式，例如直接使用 `s[i]` 的ASCII码。
