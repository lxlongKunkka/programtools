## 深度知识点讲解

### 1. 双向 BFS 是什么？

双向 BFS（Bidirectional Breadth-First Search），顾名思义，就是**从起点 (Source) 和终点 (Target) 同时开始进行广度优先搜索**。它不再是单枪匹马地从一个方向搜索，而是兵分两路，齐头并进，直到两支队伍在“战场”中央相遇。

### 2. 为什么要用双向 BFS？——“为什么”比“是什么”更重要！

同学们可能会问，单向 BFS 已经能找到最短路径了，为什么还要搞个双向 BFS 呢？难道只是为了好玩吗？当然不是！双向 BFS 的核心价值在于**效率**！

想象一下，你有一个非常大的迷宫，从起点到终点需要走 `D` 步。
*   **单向 BFS**：就像你从起点画一个半径为 `D` 的大圆（在图上是扩散的层），这个圆覆盖的节点非常多。
*   **双向 BFS**：你从起点画一个半径为 `D/2` 的圆，你的朋友从终点也画一个半径为 `D/2` 的圆。这两个“小圆”在中间相遇。

哪个覆盖的节点更少呢？当然是两个小圆！
假设每个节点平均有 `b` 条边（分支因子），那么：
*   单向 BFS 搜索 `D` 层，需要访问大约 `b^D` 个节点。
*   双向 BFS 搜索 `D/2` 层，两个方向各访问大约 `b^(D/2)` 个节点。总共访问 `2 * b^(D/2)` 个节点。

我们来算一笔账：
如果 `b = 10`，`D = 6`：
*   单向 BFS：`10^6 = 1,000,000` 个节点
*   双向 BFS：`2 * 10^(6/2) = 2 * 10^3 = 2 * 1,000 = 2,000` 个节点
看！`1,000,000` 和 `2,000`，差距是不是非常大？！这就像你从北京到上海，单向 BFS 是你一个人开车跑完全程。双向 BFS 是你从北京开车到南京，同时你的朋友从上海开车到南京，你们在南京碰面。这样你们每个人都只开了一半的路程，总的驾驶时间就大大缩短了！

**核心优势：时间复杂度从 O(b^D) 优化到 O(b^(D/2))。**

### 3. 双向 BFS 的工作原理——“怎么想到的”和“怎么做的”

双向 BFS 的基本思路是：同时维护两个 BFS 进程。
*   一个从起点 `S` 开始搜索，我们称之为**正向搜索**。
*   另一个从终点 `T` 开始搜索，我们称之为**反向搜索**。

当这两个搜索进程在某个节点 `M` 相遇时，就意味着我们找到了一条从 `S` 经过 `M` 到 `T` 的最短路径。这条路径的总长度就是 `S` 到 `M` 的距离加上 `T` 到 `M` 的距离。

**具体实现步骤：**

1.  **准备数据结构：**
    *   两个队列：`q1` 用于正向搜索，`q2` 用于反向搜索。
    *   两个访问标记数组：`vis1[u]` 记录 `u` 是否被正向搜索访问过，`vis2[u]` 记录 `u` 是否被反向搜索访问过。
    *   两个距离数组：`dist1[u]` 记录 `S` 到 `u` 的最短距离，`dist2[u]` 记录 `T` 到 `u` 的最短距离。
    *   **注意：** 初始时，所有节点的访问标记都为 `false`，距离都为 `-1` (或一个很大的值表示不可达)。

2.  **初始化：**
    *   将起点 `S` 加入 `q1`，设置 `vis1[S] = true`，`dist1[S] = 0`。
    *   将终点 `T` 加入 `q2`，设置 `vis2[T] = true`，`dist2[T] = 0`。
    *   如果 `S` 和 `T` 是同一个节点，那么最短路径就是 0，直接返回。

3.  **交替搜索：**
    *   在一个循环中，交替进行正向搜索和反向搜索。通常，我们每次从队列中取出一个节点进行扩展。为了效率，可以每次从节点数较少的队列中取出一个节点进行扩展。这里我们简化为每次从 `q1` 扩展一层，再从 `q2` 扩展一层。
    *   **正向搜索 (从 `q1` 取出 `u`):**
        *   遍历 `u` 的所有邻居 `v`。
        *   如果 `v` 没有被正向搜索访问过 (`!vis1[v]`)：
            *   标记 `vis1[v] = true`。
            *   设置 `dist1[v] = dist1[u] + 1`。
            *   将 `v` 加入 `q1`。
        *   **关键判断：** 如果 `v` 已经被反向搜索访问过 (`vis2[v]`)，说明两个搜索进程在 `v` 相遇了！此时，最短路径就是 `dist1[v] + dist2[v]`。立即返回这个结果。
    *   **反向搜索 (从 `q2` 取出 `u`):**
        *   与正向搜索类似，遍历 `u` 的所有邻居 `v`。
        *   如果 `v` 没有被反向搜索访问过 (`!vis2[v]`)：
            *   标记 `vis2[v] = true`。
            *   设置 `dist2[v] = dist2[u] + 1`。
            *   将 `v` 加入 `q2`。
        *   **关键判断：** 如果 `v` 已经被正向搜索访问过 (`vis1[v]`)，说明两个搜索进程在 `v` 相遇了！此时，最短路径就是 `dist1[v] + dist2[v]`。立即返回这个结果。

4.  **循环终止：**
    *   如果两个队列中的任何一个变空了，但还没有相遇，说明 `S` 和 `T` 不连通，无法到达。返回 `-1`。

### 4. 适用条件与常见误区

**适用条件：**
1.  **无权图或边权均为 1 的图**：BFS 只能求解无权图的最短路径。
2.  **需要寻找特定两点之间的最短路径**：如果只是从一个点到所有点的最短路径，单向 BFS 即可。
3.  **图的结构比较稀疏或分支因子 `b` 较大时效果显著**：这样 `b^D` 和 `b^(D/2)` 的差距才明显。

**常见误区：**
1.  **忘记初始化或错误初始化**：`dist` 数组一定要正确初始化为 `-1` 或无穷大，`vis` 数组为 `false`。
2.  **相遇时的距离计算错误**：最短路径是 `dist1[相遇点] + dist2[相遇点]`，而不是 `dist1[相遇点]` 或 `dist2[相遇点]`。
3.  **只判断 `vis1[v]` 或 `vis2[v]`**：在扩展 `u` 的邻居 `v` 时，需要判断 `v` 是否被**另一个方向**的搜索访问过。如果正向搜索扩展到 `v`，要检查 `vis2[v]`；如果反向搜索扩展到 `v`，要检查 `vis1[v]`。
4.  **不交替搜索**：如果只让一个方向搜索很多层，再让另一个方向搜索，就失去了双向 BFS 的优势。

### 生活类比：森林寻宝

想象你和你的队友在森林里寻宝。
*   **单向 BFS**：你从营地出发，拿着藏宝图，一步步地按照指示寻找宝藏。你每到一个地方，就标记一下“我来过这里了”，然后继续向周围探索。
*   **双向 BFS**：你从营地出发，你的队友从宝藏的已知位置出发。你们都拿着各自的藏宝图，同时向中间搜索。你每到一个地方，就放一个蓝色旗子；你的队友每到一个地方，就放一个红色旗子。
    *   如果你走到一个地方，发现那里已经有了一个红色旗子，那就说明你和队友在这里相遇了！你们加起来走过的步数，就是从营地到宝藏的最短路径。
    *   这种方法，就像两个水滴同时在纸上扩散，当它们第一次接触时，就是最短路径。效率大大提高！

## 典型例题精讲

### 例题 1：迷宫最短路径

**题目描述：**
给定一个 `N x M` 的迷宫，其中 `'S'` 表示起点，`'T'` 表示终点，`'.'` 表示可以通过的路径，`'#'` 表示障碍物。每次只能上下左右移动一步，求从 `'S'` 到 `'T'` 的最短路径长度。保证 `S` 和 `T` 只有一个。

**输入格式：**
第一行两个整数 `N, M` (1 <= N, M <= 50)。
接下来 `N` 行，每行 `M` 个字符表示迷宫。

**输出格式：**
一个整数，表示最短路径长度。如果无法到达，输出 `-1`。

**示例输入：**
```
5 5
S....
.###.
..#..
.###.
....T
```

**示例输出：**
```
8
```

**思路分析：**

1.  **问题识别**：求无权图（迷宫可看作图，每个格子是节点，相邻可通行的格子之间有边，边权为1）的最短路径。起点和终点明确。非常适合双向 BFS。
2.  **状态表示**：每个格子 `(x, y)` 是一个状态。
3.  **数据结构**：
    *   两个 `std::queue<std::pair<int, int>>`：`q1` 和 `q2`，分别存储正向和反向搜索待访问的格子坐标。
    *   两个 `int dist1[50][50]` 和 `int dist2[50][50]`：存储从起点 `S` 和终点 `T` 到 `(x, y)` 的最短距离。初始化为 `-1`。
    *   两个 `bool vis1[50][50]` 和 `bool vis2[50][50]`：标记 `(x, y)` 是否被正向或反向搜索访问过。初始化为 `false`。
    *   `char maze[50][50]` 存储迷宫地图。
    *   `dx[] = {-1, 1, 0, 0}`, `dy[] = {0, 0, -1, 1}` 辅助移动。
4.  **初始化**：
    *   找到 `S` 和 `T` 的坐标 `(sx, sy)` 和 `(tx, ty)`。
    *   将 `(sx, sy)` 加入 `q1`，`dist1[sx][sy] = 0`，`vis1[sx][sy] = true`。
    *   将 `(tx, ty)` 加入 `q2`，`dist2[tx][ty] = 0`，`vis2[tx][ty] = true`。
    *   如果 `S` 和 `T` 重合，返回 `0`。
5.  **搜索过程**：
    *   使用 `while (!q1.empty() && !q2.empty())` 循环。
    *   **每次循环，先扩展 `q1` 中的一个节点，再扩展 `q2` 中的一个节点** (或者每次扩展较小的队列)。
    *   **从 `q1` 扩展：**
        *   取出 `u = (ux, uy)`。
        *   遍历 `u` 的四个邻居 `v = (vx, vy)`：
            *   检查 `(vx, vy)` 是否越界、是否是障碍物。
            *   如果 `(vx, vy)` 未被 `vis1` 访问过：
                *   `dist1[vx][vy] = dist1[ux][uy] + 1`。
                *   `vis1[vx][vy] = true`。
                *   将 `(vx, vy)` 入队 `q1`。
                *   **关键：** 如果 `vis2[vx][vy]` 为 `true`，说明在 `(vx, vy)` 相遇，返回 `dist1[vx][vy] + dist2[vx][vy]`。
    *   **从 `q2` 扩展：** (与 `q1` 类似)
        *   取出 `u = (ux, uy)`。
        *   遍历 `u` 的四个邻居 `v = (vx, vy)`：
            *   检查 `(vx, vy)` 是否越界、是否是障碍物。
            *   如果 `(vx, vy)` 未被 `vis2` 访问过：
                *   `dist2[vx][vy] = dist2[ux][uy] + 1`。
                *   `vis2[vx][vy] = true`。
                *   将 `(vx, vy)` 入队 `q2`。
                *   **关键：** 如果 `vis1[vx][vy]` 为 `true`，说明在 `(vx, vy)` 相遇，返回 `dist1[vx][vy] + dist2[vx][vy]`。
6.  **结果**：如果循环结束仍未相遇，返回 `-1`。

**代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

using namespace std;

const int MAXN = 55; // 迷宫最大尺寸
char maze[MAXN][MAXN];
int dist1[MAXN][MAXN]; // S到(x,y)的距离
int dist2[MAXN][MAXN]; // T到(x,y)的距离
bool vis1[MAXN][MAXN]; // S方向是否访问过
bool vis2[MAXN][MAXN]; // T方向是否访问过

int N, M; // 迷宫的行和列

// 移动方向：上、下、左、右
int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

// 检查坐标是否在迷宫范围内且不是障碍物
bool isValid(int x, int y) {
    return x >= 0 && x < N && y >= 0 && y < M && maze[x][y] != '#';
}

int bidirectional_bfs(int sx, int sy, int tx, int ty) {
    // 如果起点和终点相同，路径长度为0
    if (sx == tx && sy == ty) {
        return 0;
    }

    // 初始化距离和访问数组
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < M; ++j) {
            dist1[i][j] = -1;
            dist2[i][j] = -1;
            vis1[i][j] = false;
            vis2[i][j] = false;
        }
    }

    // 两个队列，分别用于从起点和终点搜索
    queue<pair<int, int>> q1;
    queue<pair<int, int>> q2;

    // 初始化起点S的搜索
    q1.push({sx, sy});
    dist1[sx][sy] = 0;
    vis1[sx][sy] = true;

    // 初始化终点T的搜索
    q2.push({tx, ty});
    dist2[tx][ty] = 0;
    vis2[tx][ty] = true;

    // 当两个队列都不为空时，交替进行搜索
    while (!q1.empty() && !q2.empty()) {
        // 优先扩展节点较少的队列，可以提高效率，这里为了简化，每次各扩展一步
        
        // 扩展 q1 (从S出发的搜索)
        int ux = q1.front().first;
        int uy = q1.front().second;
        q1.pop();

        for (int i = 0; i < 4; ++i) {
            int vx = ux + dx[i];
            int vy = uy + dy[i];

            if (isValid(vx, vy) && !vis1[vx][vy]) {
                dist1[vx][vy] = dist1[ux][uy] + 1;
                vis1[vx][vy] = true;
                q1.push({vx, vy});

                // 如果这个点已经被反向搜索访问过，说明相遇了
                if (vis2[vx][vy]) {
                    return dist1[vx][vy] + dist2[vx][vy];
                }
            }
        }

        // 扩展 q2 (从T出发的搜索)
        // 注意：这里为了简化，每次只处理q1和q2各一个节点。
        // 更高效的做法是每次处理一层，或者总是处理节点数较少的队列。
        // 但对于GESP，这种简单的交替方式也足够理解原理。
        if (q2.empty()) continue; // 避免q2空了还去取

        ux = q2.front().first;
        uy = q2.front().second;
        q2.pop();

        for (int i = 0; i < 4; ++i) {
            int vx = ux + dx[i];
            int vy = uy + dy[i];

            if (isValid(vx, vy) && !vis2[vx][vy]) {
                dist2[vx][vy] = dist2[ux][uy] + 1;
                vis2[vx][vy] = true;
                q2.push({vx, vy});

                // 如果这个点已经被正向搜索访问过，说明相遇了
                if (vis1[vx][vy]) {
                    return dist1[vx][vy] + dist2[vx][vy];
                }
            }
        }
    }

    // 如果队列都空了还没相遇，说明无法到达
    return -1;
}

int main() {
    ios_base::sync_with_stdio(false); // 优化输入输出
    cin.tie(NULL);

    cin >> N >> M;

    int sx, sy, tx, ty;
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < M; ++j) {
            cin >> maze[i][j];
            if (maze[i][j] == 'S') {
                sx = i; sy = j;
            } else if (maze[i][j] == 'T') {
                tx = i; ty = j;
            }
        }
    }

    cout << bidirectional_bfs(sx, sy, tx, ty) << endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 在最坏情况下，双向 BFS 访问的节点数量大约是 `2 * b^(D/2)`，其中 `b` 是分支因子，`D` 是最短路径长度。对于 `N x M` 的迷宫，节点总数为 `N * M`。每个节点最多入队出队一次，并且检查其常数个邻居。因此，时间复杂度近似为 `O(N * M)`，但在实际搜索深度较小时，效率远高于单向 BFS。
*   **空间复杂度：** 需要存储两个距离数组和两个访问标记数组，以及两个队列。所以空间复杂度为 `O(N * M)`。

### 例题 2：数字转换

**题目描述：**
给定两个整数 `A` 和 `B`。你可以对 `A` 进行三种操作：
1.  `A = A + 1`
2.  `A = A - 1`
3.  `A = A * 2`
求将 `A` 转换为 `B` 的最少操作次数。
`A` 和 `B` 的范围在 `0` 到 `100000` 之间。

**输入格式：**
一行两个整数 `A, B`。

**输出格式：**
一个整数，表示最少操作次数。

**示例输入：**
```
5 17
```

**示例输出：**
```
4
```
**解释：** `5 -> 10 -> 9 -> 18 -> 17` (4步) 或 `5 -> 6 -> 12 -> 11 -> 22 -> 21 -> ...`

**思路分析：**

1.  **问题识别**：将一个数通过一系列操作转换为另一个数，求最少操作次数。这可以抽象为在一个隐式图上求最短路径的问题。每个数字是一个节点，每个操作是一个边权为 1 的边。适合双向 BFS。
2.  **状态表示**：每个整数 `x` 是一个状态。
3.  **数据结构**：
    *   `std::queue<int> q1, q2`：存储待访问的数字。
    *   `int dist1[100001], dist2[100001]`：存储从 `A` 和 `B` 到某个数字的最短距离。初始化为 `-1`。
    *   `bool vis1[100001], vis2[100001]`：标记数字是否被正向或反向搜索访问过。初始化为 `false`。
    *   最大数字范围 `100000`，所以数组大小为 `100001`。
4.  **初始化**：
    *   将 `A` 加入 `q1`，`dist1[A] = 0`，`vis1[A] = true`。
    *   将 `B` 加入 `q2`，`dist2[B] = 0`，`vis2[B] = true`。
    *   如果 `A == B`，返回 `0`。
5.  **搜索过程**：
    *   使用 `while (!q1.empty() && !q2.empty())` 循环。
    *   **从 `q1` 扩展：**
        *   取出 `u`。
        *   对 `u` 执行三种操作得到 `v1 = u+1, v2 = u-1, v3 = u*2`。
        *   对于每个 `v` (确保 `v` 在 `0` 到 `100000` 范围内)：
            *   如果 `v` 未被 `vis1` 访问过：
                *   `dist1[v] = dist1[u] + 1`。
                *   `vis1[v] = true`。
                *   将 `v` 入队 `q1`。
                *   **关键：** 如果 `vis2[v]` 为 `true`，说明在 `v` 相遇，返回 `dist1[v] + dist2[v]`。
    *   **从 `q2` 扩展：** (与 `q1` 类似)
        *   取出 `u`。
        *   对 `u` 执行三种**逆操作**得到 `v1 = u-1, v2 = u+1`。对于 `u*2` 的逆操作是 `u/2` (如果 `u` 是偶数)。
        *   **注意逆操作：**
            *   `u-1` 的逆操作是 `u+1`。
            *   `u+1` 的逆操作是 `u-1`。
            *   `u*2` 的逆操作是 `u/2` (如果 `u` 是偶数)。
        *   对于每个 `v` (确保 `v` 在 `0` 到 `100000` 范围内)：
            *   如果 `v` 未被 `vis2` 访问过：
                *   `dist2[v] = dist2[u] + 1`。
                *   `vis2[v] = true`。
                *   将 `v` 入队 `q2`。
                *   **关键：** 如果 `vis1[v]` 为 `true`，说明在 `v` 相遇，返回 `dist1[v] + dist2[v]`。
    *   **小优化：** 扩展时可以判断哪个队列更小，优先扩展小队列。这里为了教学简化，交替扩展。

**代码实现：**

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAX_VAL = 100000; // 数字的最大范围

int dist1[MAX_VAL + 1]; // 从A到x的距离
int dist2[MAX_VAL + 1]; // 从B到x的距离
bool vis1[MAX_VAL + 1]; // A方向是否访问过
bool vis2[MAX_VAL + 1]; // B方向是否访问过

int bidirectional_bfs_num_transform(int a, int b) {
    if (a == b) {
        return 0;
    }

    // 初始化距离和访问数组
    for (int i = 0; i <= MAX_VAL; ++i) {
        dist1[i] = -1;
        dist2[i] = -1;
        vis1[i] = false;
        vis2[i] = false;
    }

    queue<int> q1;
    queue<int> q2;

    // 初始化A的搜索
    q1.push(a);
    dist1[a] = 0;
    vis1[a] = true;

    // 初始化B的搜索
    q2.push(b);
    dist2[b] = 0;
    vis2[b] = true;

    while (!q1.empty() && !q2.empty()) {
        // 扩展 q1 (从A出发的搜索)
        int u = q1.front();
        q1.pop();

        // 尝试三种操作
        int next_nodes[3];
        next_nodes[0] = u + 1;
        next_nodes[1] = u - 1;
        next_nodes[2] = u * 2;

        for (int v : next_nodes) {
            // 检查数字范围
            if (v >= 0 && v <= MAX_VAL) {
                if (!vis1[v]) {
                    dist1[v] = dist1[u] + 1;
                    vis1[v] = true;
                    q1.push(v);

                    if (vis2[v]) {
                        return dist1[v] + dist2[v];
                    }
                }
            }
        }

        // 扩展 q2 (从B出发的搜索)
        if (q2.empty()) continue; // 避免q2空了还去取

        u = q2.front();
        q2.pop();

        // 尝试三种逆操作
        // 对于u+1，逆操作是u-1
        // 对于u-1，逆操作是u+1
        // 对于u*2，逆操作是u/2 (如果u是偶数)
        
        // 逆操作 1: u -> u-1 (对应正向的 u-1 -> u)
        int v_minus_1 = u - 1;
        if (v_minus_1 >= 0 && v_minus_1 <= MAX_VAL) {
            if (!vis2[v_minus_1]) {
                dist2[v_minus_1] = dist2[u] + 1;
                vis2[v_minus_1] = true;
                q2.push(v_minus_1);
                if (vis1[v_minus_1]) {
                    return dist1[v_minus_1] + dist2[v_minus_1];
                }
            }
        }

        // 逆操作 2: u -> u+1 (对应正向的 u+1 -> u)
        int v_plus_1 = u + 1;
        if (v_plus_1 >= 0 && v_plus_1 <= MAX_VAL) {
            if (!vis2[v_plus_1]) {
                dist2[v_plus_1] = dist2[u] + 1;
                vis2[v_plus_1] = true;
                q2.push(v_plus_1);
                if (vis1[v_plus_1]) {
                    return dist1[v_plus_1] + dist2[v_plus_1];
                }
            }
        }

        // 逆操作 3: u -> u/2 (对应正向的 u/2 -> u，只有当u是偶数时才考虑)
        if (u % 2 == 0) {
            int v_div_2 = u / 2;
            if (v_div_2 >= 0 && v_div_2 <= MAX_VAL) { // v_div_2 >= 0 总是成立，因为u>=0
                if (!vis2[v_div_2]) {
                    dist2[v_div_2] = dist2[u] + 1;
                    vis2[v_div_2] = true;
                    q2.push(v_div_2);
                    if (vis1[v_div_2]) {
                        return dist1[v_div_2] + dist2[v_div_2];
                    }
                }
            }
        }
    }

    return -1; // 理论上本题一定能到达，但作为通用模板保留
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int a, b;
    cin >> a >> b;

    cout << bidirectional_bfs_num_transform(a, b) << endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：** 状态空间大小为 `MAX_VAL`。每个状态最多入队出队一次，并进行常数次操作。因此，时间复杂度为 `O(MAX_VAL)`。双向 BFS 在这里可以显著减少实际访问的节点数，尤其当 `A` 和 `B` 相距较远时。
*   **空间复杂度：** 需要存储两个距离数组和两个访问标记数组，以及两个队列。所以空间复杂度为 `O(MAX_VAL)`。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

using namespace std;

// 定义图的节点数量上限，根据题目调整
const int MAX_NODES = 100005; 

// 邻接表表示图 (如果图是显式的)
// vector<int> adj[MAX_NODES]; // GESP 7允许使用vector，但若题目限定，可改用链式前向星或矩阵

// 存储从起点S到各节点的距离
int dist1[MAX_NODES]; 
// 存储从终点T到各节点的距离
int dist2[MAX_NODES]; 
// 标记节点是否被S方向访问过
bool vis1[MAX_NODES]; 
// 标记节点是否被T方向访问过
bool vis2[MAX_NODES]; 

// 定义队列，用于BFS搜索
queue<int> q1; // S方向的队列
queue<int> q2; // T方向的队列

// 函数：获取一个节点的所有邻居 (或通过操作可达的下一状态)
// 在实际问题中，这个函数体需要根据具体问题来填充
// 返回一个 vector<int> 包含所有邻居，或直接在函数内部处理邻居
vector<int> get_neighbors(int u) {
    vector<int> neighbors;
    // 示例：u的邻居是 u+1, u-1, u*2
    // 实际问题中可以是图的邻接点，或迷宫的上下左右
    // 注意：这里需要检查邻居的合法性（范围、是否是障碍等）
    
    // 示例：数字转换问题
    if (u + 1 < MAX_NODES) neighbors.push_back(u + 1);
    if (u - 1 >= 0) neighbors.push_back(u - 1);
    if (u * 2 < MAX_NODES) neighbors.push_back(u * 2);

    return neighbors;
}

// 主函数：双向BFS
// s: 起始节点
// t: 目标节点
int bidirectional_bfs(int s, int t) {
    // 1. 特殊情况处理：如果起点和终点相同，距离为0
    if (s == t) {
        return 0;
    }

    // 2. 初始化所有数据结构
    // C++11及以上可以用fill，或者for循环
    memset(dist1, -1, sizeof(dist1));
    memset(dist2, -1, sizeof(dist2));
    memset(vis1, false, sizeof(vis1));
    memset(vis2, false, sizeof(vis2));
    
    // 清空队列，防止上次运行遗留数据
    while (!q1.empty()) q1.pop();
    while (!q2.empty()) q2.pop();

    // 3. 初始化S方向的搜索
    q1.push(s);
    dist1[s] = 0;
    vis1[s] = true;

    // 4. 初始化T方向的搜索
    q2.push(t);
    dist2[t] = 0;
    vis2[t] = true;

    // 5. 核心搜索循环：当两个队列都不为空时，交替进行搜索
    while (!q1.empty() && !q2.empty()) {
        // --- 优先扩展节点较少的队列，以提高效率 ---
        // 这里为了简化，我们交替扩展，或者可以先扩展q1，再扩展q2
        // 实际应用中，可以判断 if (q1.size() <= q2.size()) { /* 扩展q1 */ } else { /* 扩展q2 */ }

        // 扩展S方向的队列 q1
        int u1 = q1.front();
        q1.pop();

        // 遍历u1的所有邻居v
        for (int v : get_neighbors(u1)) { // get_neighbors 需要根据具体问题实现
            // 如果v未被S方向访问过
            if (!vis1[v]) {
                dist1[v] = dist1[u1] + 1; // 更新距离
                vis1[v] = true;           // 标记已访问
                q1.push(v);               // 加入队列

                // 检查是否与T方向的搜索相遇
                if (vis2[v]) {
                    return dist1[v] + dist2[v]; // 相遇，返回总距离
                }
            }
        }

        // 扩展T方向的队列 q2
        // 注意：这里需要再次检查q2是否为空，因为q1的扩展可能已经导致相遇并返回
        if (q2.empty()) continue; 

        int u2 = q2.front();
        q2.pop();

        // 遍历u2的所有邻居v
        // 注意：这里的get_neighbors如果是针对逆操作，需要特别处理
        // 对于无向图，get_neighbors可以直接使用
        // 对于有向图且需要逆向搜索，get_neighbors需要返回前驱节点
        for (int v : get_neighbors(u2)) { // get_neighbors 需要根据具体问题实现
            // 如果v未被T方向访问过
            if (!vis2[v]) {
                dist2[v] = dist2[u2] + 1; // 更新距离
                vis2[v] = true;           // 标记已访问
                q2.push(v);               // 加入队列

                // 检查是否与S方向的搜索相遇
                if (vis1[v]) {
                    return dist1[v] + dist2[v]; // 相遇，返回总距离
                }
            }
        }
    }

    // 如果循环结束，队列都空了但仍未相遇，说明S和T不连通
    return -1;
}

int main() {
    ios_base::sync_with_stdio(false); // 优化输入输出
    cin.tie(NULL);                    // 解除cin与cout的绑定

    // 示例用法：
    // int start_node = ...;
    // int target_node = ...;
    // int shortest_path = bidirectional_bfs(start_node, target_node);
    // cout << shortest_path << endl;

    // 请根据具体题目在此处填充输入和调用 bidirectional_bfs 函数
    // 例如，使用例题2的get_neighbors和main函数
    int a, b;
    cin >> a >> b;
    cout << bidirectional_bfs(a, b) << endl;

    return 0;
}
```

1.  **思考题：**
    *   **问题一：** 如果我们在双向 BFS 中，不交替进行搜索，而是先让 `q1` 把所有能到的节点都搜完，然后再让 `q2` 开始搜，这还是双向 BFS 吗？它的效率会比正常的双向 BFS 更好还是更差？为什么？
        *   **参考答案：** 这就退化成了两个独立的单向 BFS，失去了双向 BFS 的效率优势。因为搜索空间是指数增长的，先搜完一边意味着搜索半径 `D`，而双向是 `D/2`，`b^D` 远大于 `2 * b^(D/2)`。
    *   **问题二：** 双向 BFS 适合在什么类型的图上寻找最短路径？如果图中有边的权重，还能直接使用双向 BFS 吗？
        *   **参考答案：** 适合无权图（或所有边权都为 1）寻找最短路径。如果图中有权重，BFS 就不能保证找到最短路径了，需要使用 Dijkstra 算法或 SPFA 算法。双向 BFS 的思想在带权图中也有变种（如双向 Dijkstra），但不是我们今天讨论的这种简单形式。
