## 知识点讲解

### 1. 什么是拓扑排序？

拓扑排序（Topological Sort）是对有向无环图（Directed Acyclic Graph, 简称 DAG）的顶点进行的一种线性排序。
它的核心要求是：如果图中存在一条从顶点 `u` 到顶点 `v` 的有向边 `(u, v)`，那么在排序结果中，`u` 必须出现在 `v` 的前面。

**关键点：**
*   **有向图（Directed Graph）**：边有方向，表示依赖关系（例如：`A` 依赖 `B`，则 `B -> A`）。
*   **无环图（Acyclic Graph）**：图中不能包含任何环。如果存在环，就意味着某些任务之间存在循环依赖（例如：`A` 依赖 `B`，`B` 依赖 `C`，`C` 又依赖 `A`），这种情况下无法确定一个明确的先后顺序，因此拓扑排序无法进行。
*   **线性排序**：最终结果是一个顶点的序列。

**应用场景：**
*   **任务调度**：确定一系列有依赖关系的任务的执行顺序。
*   **课程安排**：找出学习先修课程的合理顺序。
*   **编译依赖**：确定编译源代码文件的顺序。
*   **项目管理**：规划项目各个阶段的执行顺序。

### 2. 拓扑排序的两种主要算法

我们将学习两种常用的拓扑排序算法：Kahn 算法（基于广度优先搜索 BFS 的思想）和基于深度优先搜索 DFS 的算法。

#### 2.1 Kahn 算法 (入度法)

Kahn 算法的核心思想是：从图中选择那些没有前驱（即入度为0）的顶点开始排序。因为它们没有任何依赖，所以可以最先完成。当一个顶点被“完成”后，它指向的那些顶点的入度就会减少，如果某个顶点的入度也因此变为0，那它就可以成为下一个被“完成”的顶点。

**算法步骤：**

1.  **计算入度**：统计图中每个顶点的入度（有多少条边指向它）。
2.  **初始化队列**：将所有入度为0的顶点加入一个队列 `q` 中。
3.  **循环处理**：
    a.  从队列 `q` 中取出一个顶点 `u`。
    b.  将 `u` 添加到拓扑排序结果列表 `ans` 中。
    c.  对于 `u` 的每一个邻接顶点 `v`（即存在边 `u -> v`）：
        i.  将 `v` 的入度减 1。
        ii. 如果 `v` 的入度变为 0，则将 `v` 加入队列 `q`。
4.  **判断环**：如果最终拓扑排序结果列表中的顶点数量不等于图中的总顶点数，说明图中存在环，无法进行拓扑排序。

**时间复杂度**：O(V + E)，其中 `V` 是顶点数，`E` 是边数。因为每个顶点和每条边都只会被访问常数次。

#### 2.2 基于 DFS 的算法

基于 DFS 的拓扑排序算法利用了深度优先搜索的特性：当一个顶点 `u` 的所有后继顶点都被访问完（即 `u` 的所有依赖都已处理完毕）时，`u` 就可以被添加到拓扑排序的结果中。

**算法步骤：**

1.  **状态标记**：使用一个数组 `vis` 来标记顶点的访问状态：
    *   `0`：未访问
    *   `1`：正在访问（在当前的 DFS 路径中）
    *   `2`：已访问（所有后继都已处理）
2.  **遍历所有顶点**：对于图中的每一个顶点 `u`：
    *   如果 `u` 尚未访问 (`vis[u] == 0`)，则从 `u` 开始进行一次 DFS 遍历。
3.  **DFS 函数 `dfs(u)`**：
    a.  将 `u` 的状态标记为 `1`（正在访问）。
    b.  对于 `u` 的每一个邻接顶点 `v`（即存在边 `u -> v`）：
        i.  如果 `v` 的状态是 `1`（正在访问），则说明图中存在环，拓扑排序失败。
        ii. 如果 `v` 的状态是 `0`（未访问），则递归调用 `dfs(v)`。
    c.  将 `u` 的状态标记为 `2`（已访问）。
    d.  将 `u` 添加到拓扑排序结果列表 `ans` 的**最前端**（或者压入栈中，最后再逆序输出）。这是因为 `u` 的所有依赖都已处理完毕，它可以被输出了。

**时间复杂度**：O(V + E)，与 Kahn 算法相同。

**重要提示：**
*   一个有向无环图可能存在多个不同的拓扑排序结果。
*   如果图中存在环，两种算法都能检测到（Kahn 算法通过 `ans.size() != n`，DFS 算法通过 `vis[v] == 1`）。

## 代码示例

我们将用 C++ 来实现这两种拓扑排序算法。

首先，我们定义图的结构——使用邻接表来存储。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件
using namespace std;     // 使用标准命名空间

// 全局变量，方便在不同函数中使用
const int MAXN = 100005; // 最大顶点数
vector<int> adj[MAXN];   // 邻接表，adj[u] 存储 u 的所有邻居
int n_nodes, m_edges;    // 顶点数和边数

// Kahn 算法所需变量
int in_degree[MAXN];     // 存储每个顶点的入度
vector<int> kahn_result; // 存储 Kahn 算法的拓扑排序结果

// DFS 算法所需变量
int vis[MAXN];           // 访问状态：0-未访问, 1-正在访问, 2-已访问
vector<int> dfs_result;  // 存储 DFS 算法的拓扑排序结果
bool has_cycle = false;  // 标记是否检测到环

// Kahn 算法实现
void kahn_topological_sort() {
    queue<int> q; // 存储入度为0的顶点

    // 1. 初始化队列：将所有入度为0的顶点加入队列
    for (int i = 1; i <= n_nodes; ++i) {
        if (in_degree[i] == 0) {
            q.push(i);
        }
    }

    // 2. 循环处理队列
    while (!q.empty()) {
        int u = q.front(); // 取出队列头部顶点
        q.pop();

        kahn_result.push_back(u); // 将 u 添加到结果中

        // 遍历 u 的所有邻接顶点 v
        for (int v : adj[u]) {
            in_degree[v]--; // v 的入度减1

            // 如果 v 的入度变为0，则加入队列
            if (in_degree[v] == 0) {
                q.push(v);
            }
        }
    }

    // 3. 判断是否有环
    if (kahn_result.size() != n_nodes) {
        cout << "Kahn 算法检测到环，无法进行拓扑排序！" << endl;
    } else {
        cout << "Kahn 算法拓扑排序结果: ";
        for (int x : kahn_result) {
            cout << x << " ";
        }
        cout << endl;
    }
}

// DFS 算法实现
void dfs_topological_sort(int u) {
    vis[u] = 1; // 标记 u 正在访问

    // 遍历 u 的所有邻接顶点 v
    for (int v : adj[u]) {
        if (vis[v] == 1) { // 如果 v 正在访问，说明有环
            has_cycle = true;
            return;
        }
        if (vis[v] == 0) { // 如果 v 未访问，则递归访问 v
            dfs_topological_sort(v);
            if (has_cycle) { // 如果在递归调用中发现了环，则直接返回
                return;
            }
        }
    }

    vis[u] = 2; // 标记 u 已访问（所有后继都已处理）
    dfs_result.push_back(u); // 将 u 添加到结果的最前端（或压入栈）
                             // 注意：这里是 push_back，最后需要逆序输出
}

int main() {
    cout << "请输入顶点数 n 和边数 m: ";
    cin >> n_nodes >> m_edges;

    cout << "请依次输入 " << m_edges << " 条边的起点和终点 (u v):" << endl;
    for (int i = 0; i < m_edges; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v); // 添加有向边 u -> v
        in_degree[v]++;      // v 的入度加1
    }

    cout << "\n--- 运行 Kahn 算法 ---" << endl;
    kahn_topological_sort();

    cout << "\n--- 运行 DFS 算法 ---" << endl;
    // 重置访问状态和结果，因为可能运行多次
    memset(vis, 0, sizeof(vis));
    dfs_result.clear();
    has_cycle = false;

    // 遍历所有顶点，确保所有连通分量都被访问到
    for (int i = 1; i <= n_nodes; ++i) {
        if (vis[i] == 0 && !has_cycle) { // 如果未访问且没有发现环
            dfs_topological_sort(i);
        }
    }

    if (has_cycle) {
        cout << "DFS 算法检测到环，无法进行拓扑排序！" << endl;
    } else {
        cout << "DFS 算法拓扑排序结果 (逆序输出，因为是后进先出): ";
        // DFS 结果是逆序的，需要反转一下才是正确的拓扑序
        reverse(dfs_result.begin(), dfs_result.end());
        for (int x : dfs_result) {
            cout << x << " ";
        }
        cout << endl;
    }

    return 0;
}
```

**代码解释：**
*   `adj` 数组是一个 `vector<int>` 类型的数组，用于存储图的邻接表。`adj[u]` 存储所有从 `u` 出发的边所指向的顶点。
*   `in_degree` 数组用于 Kahn 算法，存储每个顶点的入度。
*   `vis` 数组用于 DFS 算法，标记顶点的访问状态，帮助我们检测环。
*   `kahn_topological_sort()` 函数实现了 Kahn 算法。它使用一个队列 `q` 来存储入度为0的顶点。
*   `dfs_topological_sort()` 函数实现了基于 DFS 的算法。它通过递归来遍历图，并在所有后继顶点都处理完后，将当前顶点加入结果列表。
*   `main()` 函数负责读取图的输入，并分别调用两种算法进行拓扑排序。
*   **注意**：DFS 算法将顶点加入 `dfs_result` 的顺序是其所有后继都处理完毕的顺序，这与拓扑排序要求的“前驱在前”是相反的。所以，`dfs_result` 存储的是逆拓扑序，我们需要使用 `reverse()` 函数将其反转。

**测试用例：**
输入：
5 6
1 2
1 3
2 4
3 4
4 5
1 5

输出示例 (Kahn 和 DFS 可能会给出不同但都正确的拓扑序):
Kahn 算法拓扑排序结果: 1 2 3 4 5
DFS 算法拓扑排序结果: 1 2 3 4 5 (或者 1 3 2 4 5 等)

如果输入有环：
3 3
1 2
2 3
3 1
输出：检测到环，无法进行拓扑排序！

## 练习题目

### 选择题/填空题

**请注意：以下题目是用于检测你对拓扑排序知识的掌握程度。请先独立思考并作答，答案在题目下方。**

**题目部分：**

1.  拓扑排序适用于哪种类型的图？
    A. 无向图
    B. 有向图
    C. 有向无环图
    D. 连通图

2.  Kahn 算法在每次迭代中，会从队列中取出什么类型的顶点？
    A. 出度为0的顶点
    B. 入度为0的顶点
    C. 度数为0的顶点
    D. 任意顶点

3.  如果一个有向图包含一个环，那么它的拓扑排序结果将如何？
    A. 结果唯一
    B. 结果不唯一，但总是存在
    C. 无法进行拓扑排序
    D. 结果和 DFS 遍历顺序相同

4.  基于 DFS 的拓扑排序算法，通常会将顶点添加到结果列表的哪个位置？
    A. 列表的开头
    B. 列表的末尾
    C. 列表的中间
    D. 随机位置

5.  一个有 N 个顶点和 M 条边的图，使用 Kahn 算法进行拓扑排序的时间复杂度是 ______。

---

**答案及解析部分：**

1.  **C. 有向无环图**
    *   **解析**：拓扑排序的核心是解决任务依赖的先后顺序，这需要有方向（有向图）。同时，如果存在循环依赖（环），则无法确定一个线性顺序，因此必须是无环图。

2.  **B. 入度为0的顶点**
    *   **解析**：Kahn 算法的思想是先处理那些没有前驱依赖的顶点。入度为0的顶点正是指没有其他顶点指向它的顶点，可以最先被处理。

3.  **C. 无法进行拓扑排序**
    *   **解析**：环表示循环依赖，例如 A 依赖 B，B 依赖 C，C 又依赖 A。在这种情况下，任何一个任务都无法在所有依赖之前完成，因此无法形成一个线性的拓扑顺序。

4.  **A. 列表的开头**
    *   **解析**：基于 DFS 的拓扑排序，当一个顶点 `u` 的所有后继顶点都处理完毕后，`u` 才被添加到结果中。这意味着 `u` 应该排在其所有后继之前。如果将 `u` 添加到列表末尾，那么最终得到的将是逆拓扑序，需要反转才能得到正确的拓扑序。或者直接压入栈，最后弹出时就是正确的拓扑序。

5.  **O(N + M)**
    *   **解析**：Kahn 算法中，每个顶点最多入队和出队一次，处理其邻接边时会将邻接顶点的入度减1。每条边最多被访问一次。因此，总的时间复杂度是线性的，与顶点数 `N` 和边数 `M` 之和成正比。
