## 深度知识点讲解

### 1. 什么是动态规划 (DP)？（快速回顾）

在深入区间 DP 之前，我们先快速回顾一下动态规划的核心思想。DP 就像搭桥，它有两个基石：
*   **重叠子问题 (Overlapping Subproblems)**：大问题可以拆分成很多小问题，而这些小问题中有很多是重复的。DP 会把这些小问题的答案存起来，避免重复计算。
*   **最优子结构 (Optimal Substructure)**：一个大问题的最优解，是由它的小问题的最优解组合而成的。

我们之前学过的线性 DP (比如求最长上升子序列) 或者背包 DP，都是 DP 的不同表现形式。今天，我们要学习的是一种特别擅长处理“序列”或“区间”问题的 DP——**区间 DP**。

### 2. 区间 DP 的核心思想：从小到大，分而治之

想象一下，你要解决一个关于序列 `a[0...n-1]` 的问题。如果这个问题的最优解可以由它的两个子序列 `a[i...k]` 和 `a[k+1...j]` 的最优解组合而来，那么恭喜你，你可能遇到了一个区间 DP 问题！

**核心特点：**
*   **处理对象：** 通常是序列、数组、字符串等，需要对一个“连续的区间”进行操作。
*   **目标：** 求解某个区间 `[i, j]` 的最优值（最大/最小）或方案数。
*   **依赖关系：** 区间 `[i, j]` 的解，依赖于比它更小的子区间 `[i, k]` 和 `[k+1, j]` 的解。

**生活类比：建造大桥**
假设我们要建造一座很长的桥。我们不能一下子建成整座桥，而是先建一小段一小段的桥墩和桥面。建好这些小段后，再把它们连接起来，形成更大的桥段，最终连接成整座大桥。
*   `dp[i][j]`：表示从第 `i` 个桥墩到第 `j` 个桥墩这段桥的最佳建造方案或最小成本。
*   `k`：就是中间的一个连接点，我们通过 `k` 把 `[i, j]` 这段桥分成 `[i, k]` 和 `[k+1, j]` 两段。
*   我们先算出所有短桥段的最佳方案，然后用这些短桥段的方案来推导更长桥段的方案，直到算出整座桥的方案。

### 3. 区间 DP 的标准框架与循环结构

区间 DP 的状态通常定义为 `dp[i][j]`，表示对原始序列或字符串的子区间 `[i, j]` 进行操作后得到的最佳结果（最大值、最小值、方案数等）。

它的计算顺序非常关键，必须保证在计算 `dp[i][j]` 时，它所依赖的子区间 `dp[i][k]` 和 `dp[k+1][j]` 的值都已经计算完毕。由于子区间的长度总是小于父区间，所以我们通常采用**枚举区间长度**的方式。

```cpp
// 假设原始序列长度为 n
// dp[i][j] 表示区间 [i, j] 的解

// 1. 初始化基本情况 (Base Cases)
// 通常是长度为 1 的区间，即 dp[i][i] 的值
// 有时也包括长度为 0 的区间，即 dp[i][i-1] 的值

// 2. 外层循环：枚举区间长度 len
// len 从 2 开始，因为长度为 1 的区间（dp[i][i]）通常是基本情况，或者在循环内部处理。
// len 最大到 n。
for (int len = 2; len <= n; ++len) { // len 代表当前处理的区间长度

    // 3. 中层循环：枚举区间起始点 i
    // i 从 0 开始，一直到 n - len (确保区间 [i, i+len-1] 不越界)
    for (int i = 0; i <= n - len; ++i) {
        // 计算区间结束点 j
        int j = i + len - 1;

        // 初始化 dp[i][j] 为一个不可能的值 (比如极大值或极小值)，或根据问题特点初始化
        // 例如，如果求最小值，就初始化为足够大的数。
        // dp[i][j] = ...

        // 4. 内层循环：枚举分割点 k
        // k 在区间 [i, j] 内部，将区间 [i, j] 分割成 [i, k] 和 [k+1, j]
        // k 可以从 i 到 j-1。
        for (int k = i; k < j; ++k) {
            // 状态转移方程：
            // dp[i][j] 可以由 dp[i][k] 和 dp[k+1][j] 组合得到
            // 同时可能需要加上合并 [i, k] 和 [k+1, j] 的额外“代价”或“收益”
            // dp[i][j] = min/max(dp[i][j], dp[i][k] + dp[k+1][j] + cost(i, j))
            // 注意：cost(i, j) 通常是与整个区间 [i, j] 相关的值。
        }
    }
}
```

**关键点剖析：**
*   **为什么 `len` 在外层？** 因为 `dp[i][j]` 的计算依赖于长度比 `len` 小的子区间 `dp[i][k]` 和 `dp[k+1][j]`。如果 `len` 在内层，我们可能在计算 `dp[i][j]` 时，它依赖的子区间还没有被计算出来。
*   **`k` 的作用？** `k` 是一个“尝试点”，我们尝试所有可能的分割点 `k`，找到一个能使 `dp[i][j]` 达到最优的分割方式。
*   **时间复杂度：** 三层循环，每层循环大约 `N` 次，所以通常是 `O(N^3)`。

**常见误区：**
1.  **循环顺序错误：** 最常见的错误就是 `len`、`i`、`k` 的循环顺序搞错，导致依赖的子问题未计算。
2.  **边界条件错误：** `i`, `j`, `k` 的循环范围，特别是 `k` 的范围，需要仔细推敲，避免数组越界或漏掉情况。
3.  **初始化错误：** `dp` 数组的初始值（0，极大值，极小值等）要根据问题目标（求和，求最大，求最小）正确设置。
4.  **忘记处理 `dp[i][i]` 或 `dp[i][i-1]` 等基本情况。**

## 典型例题精讲

### 例题 1：石子合并 (线性版本)

**问题描述：**
有 `n` 堆石子，排成一行。每次只能合并相邻的两堆石子。合并的代价是这两堆石子的总重量。求将这 `n` 堆石子合并成一堆的最小总代价。

**输入样例：**
`n = 4`
`a = [10, 20, 30, 40]`

**输出样例：**
`190`

**思路分析：**
1.  **定义状态：** `dp[i][j]` 表示将第 `i` 堆到第 `j` 堆石子合并成一堆的最小代价。
2.  **目标：** 求 `dp[0][n-1]`。
3.  **基本情况：**
    *   当 `len = 1` 时，即 `dp[i][i]`，表示只有一堆石子，不需要合并，代价为 0。
    *   当 `len = 0` 时，即 `dp[i][i-1]`，表示空区间，代价为 0。
4.  **状态转移：**
    *   要计算 `dp[i][j]`，我们可以枚举一个分割点 `k`，将 `[i, j]` 分割成 `[i, k]` 和 `[k+1, j]`。
    *   将 `[i, k]` 合并的最小代价是 `dp[i][k]`。
    *   将 `[k+1, j]` 合并的最小代价是 `dp[k+1][j]`。
    *   将 `[i, k]` 和 `[k+1, j]` 合并后的两堆再合并成一堆，其代价是这两堆石子的总重量，也就是原始 `[i, j]` 区间内所有石子的总重量。
    *   因此，`dp[i][j] = min(dp[i][k] + dp[k+1][j] + sum(i, j))`。
    *   这里的 `sum(i, j)` 表示从第 `i` 堆到第 `j` 堆石子的总重量。为了快速计算区间和，我们可以使用**前缀和**。
        *   设 `s[x]` 为前 `x` 堆石子的总重量 (即 `a[0] + ... + a[x-1]`)。
        *   那么 `sum(i, j) = s[j+1] - s[i]`。

**计算过程示例 (`a = [10, 20, 30, 40]`):**
*   前缀和 `s = [0, 10, 30, 60, 100]`
*   `dp` 数组初始化为足够大的数，`dp[i][i] = 0`。

*   **len = 2:**
    *   `dp[0][1]`: `k=0`. `dp[0][0] + dp[1][1] + sum(0,1)` = `0 + 0 + (10+20) = 30`.
    *   `dp[1][2]`: `k=1`. `dp[1][1] + dp[2][2] + sum(1,2)` = `0 + 0 + (20+30) = 50`.
    *   `dp[2][3]`: `k=2`. `dp[2][2] + dp[3][3] + sum(2,3)` = `0 + 0 + (30+40) = 70`.

*   **len = 3:**
    *   `dp[0][2]`: `sum(0,2) = 10+20+30 = 60`
        *   `k=0`: `dp[0][0] + dp[1][2] + sum(0,2)` = `0 + 50 + 60 = 110`
        *   `k=1`: `dp[0][1] + dp[2][2] + sum(0,2)` = `30 + 0 + 60 = 90`
        *   `dp[0][2] = min(110, 90) = 90`.
    *   `dp[1][3]`: `sum(1,3) = 20+30+40 = 90`
        *   `k=1`: `dp[1][1] + dp[2][3] + sum(1,3)` = `0 + 70 + 90 = 160`
        *   `k=2`: `dp[1][2] + dp[3][3] + sum(1,3)` = `50 + 0 + 90 = 140`
        *   `dp[1][3] = min(160, 140) = 140`.

*   **len = 4:**
    *   `dp[0][3]`: `sum(0,3) = 10+20+30+40 = 100`
        *   `k=0`: `dp[0][0] + dp[1][3] + sum(0,3)` = `0 + 140 + 100 = 240`
        *   `k=1`: `dp[0][1] + dp[2][3] + sum(0,3)` = `30 + 70 + 100 = 200`
        *   `k=2`: `dp[0][2] + dp[3][3] + sum(0,3)` = `90 + 0 + 100 = 190`
        *   `dp[0][3] = min(240, 200, 190) = 190`.

**代码实现：**

```cpp
#include <bits/stdc++.h>

const int INF = 0x3f3f3f3f; // 定义一个足够大的数作为无穷大

int a[105]; // 存储每堆石子的重量
int s[105]; // 存储前缀和，s[x] 表示 a[0]...a[x-1] 的和
int dp[105][105]; // dp[i][j] 表示合并区间 [i, j] 的最小代价

int main() {
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    // 读取石子重量并计算前缀和
    s[0] = 0; // s[0] 表示空区间和为0
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
        s[i + 1] = s[i] + a[i]; // s[i+1] 存储 a[0]...a[i] 的和
    }

    // 初始化 dp 数组
    // dp[i][i] 表示单个石子堆，不需要合并，代价为 0
    // dp[i][j] (i != j) 初始化为 INF
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < n; ++j) {
            if (i == j) {
                dp[i][j] = 0;
            } else {
                dp[i][j] = INF;
            }
        }
    }

    // 区间 DP 核心循环
    // len: 区间长度，从 2 到 n
    for (int len = 2; len <= n; ++len) {
        // i: 区间起始点，从 0 到 n - len
        for (int i = 0; i <= n - len; ++i) {
            // j: 区间结束点
            int j = i + len - 1;

            // k: 分割点，将区间 [i, j] 分割成 [i, k] 和 [k+1, j]
            for (int k = i; k < j; ++k) {
                // 当前区间 [i, j] 的合并代价是：
                // 合并 [i, k] 的最小代价 (dp[i][k])
                // + 合并 [k+1, j] 的最小代价 (dp[k+1][j])
                // + 将合并后的两堆再合并的代价 (即区间 [i, j] 所有石子的总重量)
                int current_cost = dp[i][k] + dp[k + 1][j] + (s[j + 1] - s[i]);
                dp[i][j] = std::min(dp[i][j], current_cost);
            }
        }
    }

    std::cout << dp[0][n - 1] << std::endl;

    return 0;
}
```

**时间复杂度：**
*   三层循环，每层循环的次数大约是 `N`。
*   总时间复杂度为 `O(N^3)`。
*   空间复杂度：`dp` 数组 `O(N^2)`，前缀和数组 `O(N)`。

### 例题 2：括号序列匹配 (最少添加字符)

**问题描述：**
给定一个只包含 `(`、`)`、`[`、`]` 四种字符的字符串，你需要通过添加最少数量的字符，使得整个字符串变成一个合法的括号序列。
合法的括号序列定义如下：
1.  空字符串是合法的。
2.  如果 `S` 是合法的，那么 `(S)` 和 `[S]` 也是合法的。
3.  如果 `S1` 和 `S2` 都是合法的，那么 `S1S2` 也是合法的。

**输入样例：**
`s = "([)]"`

**输出样例：**
`2` (例如，可以变为 `()[]` 或 `[()]`)

**思路分析：**
1.  **定义状态：** `dp[i][j]` 表示将字符串 `s` 的子串 `s[i...j]` 变为合法括号序列所需的最小添加字符数。
2.  **目标：** 求 `dp[0][n-1]`。
3.  **基本情况：**
    *   当 `len = 0` 时，即 `dp[i][i-1]`，表示空字符串，已经是合法的，需要添加 0 个字符。
    *   当 `len = 1` 时，即 `dp[i][i]`，表示单个字符 (例如 `(` 或 `)` 或 `[` 或 `]`)，需要添加 1 个字符才能变成合法序列（例如 `(` 变为 `()`）。
4.  **状态转移：**
    *   **情况一：分割合并**
        与石子合并类似，我们可以将 `s[i...j]` 分割成两部分 `s[i...k]` 和 `s[k+1...j]`，然后分别使这两部分合法，再拼接起来。
        `dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j])` (对于所有 `k` 从 `i` 到 `j-1`)。
        这是处理 `S1S2` 形式的合法序列。
    *   **情况二：首尾匹配**
        如果 `s[i]` 和 `s[j]` 是一对匹配的括号（例如 `(` 和 `)`，或者 `[` 和 `]`），那么 `s[i...j]` 可以看作是 `(S)` 或 `[S]` 的形式，其中 `S` 是 `s[i+1...j-1]`。
        这时，我们只需要让 `s[i+1...j-1]` 变为合法序列，`s[i]` 和 `s[j]` 就自然匹配了，不需要额外添加字符。
        `dp[i][j] = min(dp[i][j], dp[i+1][j-1])` (如果 `s[i]` 和 `s[j]` 匹配)。
        注意：`dp[i+1][j-1]` 可能表示一个空区间，即 `i+1 > j-1`，对应 `dp[i][i-1]`，其值为 0。

**计算过程示例 (`s = "([)]"`, `n=4`):**
*   `dp` 数组初始化：`dp[i][i-1] = 0`，`dp[i][i] = 1`，其他为极大值。

*   **len = 2:**
    *   `dp[0][1]` (`([`)：
        *   `k=0`: `dp[0][0] + dp[1][1]` = `1 + 1 = 2`
        *   `(s[0], s[1])` 不匹配。
        *   `dp[0][1] = 2`
    *   `dp[1][2]` (`[)`)：
        *   `k=1`: `dp[1][1] + dp[2][2]` = `1 + 1 = 2`
        *   `(s[1], s[2])` 不匹配。
        *   `dp[1][2] = 2`
    *   `dp[2][3]` (`)]`)：
        *   `k=2`: `dp[2][2] + dp[3][3]` = `1 + 1 = 2`
        *   `(s[2], s[3])` 不匹配。
        *   `dp[2][3] = 2`

*   **len = 3:**
    *   `dp[0][2]` (`([)`)：
        *   `k=0`: `dp[0][0] + dp[1][2]` = `1 + 2 = 3`
        *   `k=1`: `dp[0][1] + dp[2][2]` = `2 + 1 = 3`
        *   `(s[0], s[2])` 是 `(` 和 `)`，匹配！`dp[0][2] = min(dp[0][2], dp[1][1]) = min(3, 1) = 1` (例如 `[(` 变为 `([])`)
        *   `dp[0][2] = 1`
    *   `dp[1][3]` (`[)]`)：
        *   `k=1`: `dp[1][1] + dp[2][3]` = `1 + 2 = 3`
        *   `k=2`: `dp[1][2] + dp[3][3]` = `2 + 1 = 3`
        *   `(s[1], s[3])` 是 `[` 和 `]`，匹配！`dp[1][3] = min(dp[1][3], dp[2][2]) = min(3, 1) = 1`
        *   `dp[1][3] = 1`

*   **len = 4:**
    *   `dp[0][3]` (`([)]`)：
        *   `k=0`: `dp[0][0] + dp[1][3]` = `1 + 1 = 2`
        *   `k=1`: `dp[0][1] + dp[2][3]` = `2 + 2 = 4`
        *   `k=2`: `dp[0][2] + dp[3][3]` = `1 + 1 = 2`
        *   `(s[0], s[3])` 是 `(` 和 `]`，不匹配。
        *   `dp[0][3] = min(2, 4, 2) = 2`.

**代码实现：**

```cpp
#include <bits/stdc++.h>
#include <string>

const int INF = 0x3f3f3f3f; // 定义一个足够大的数作为无穷大

int dp[105][105]; // dp[i][j] 表示将 s[i...j] 变为合法括号序列所需的最小添加字符数
char c[105]; // 存储输入的字符串

// 辅助函数：判断两个字符是否是匹配的括号
bool is_match(char a, char b) {
    return (a == '(' && b == ')') || (a == '[' && b == ']');
}

int main() {
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL);

    std::string s_str;
    std::cin >> s_str;
    int n = s_str.length();

    // 将字符串转换为 char 数组，方便索引
    for (int i = 0; i < n; ++i) {
        c[i] = s_str[i];
    }

    // 初始化 dp 数组
    // dp[i][i-1] 表示空区间，已经合法，需要添加 0 个字符
    // dp[i][i] 表示单个字符，需要添加 1 个字符使其合法 (例如 '(' -> "()")
    for (int i = 0; i < n; ++i) {
        dp[i][i] = 1;
        // 注意：这里 i+1 <= n-1 是为了防止越界，但实际上 dp[i][i-1] 并不需要实际的循环。
        // 它只是一个概念上的基准值。
        // 可以直接在循环外面设置 dp[i][i-1] = 0。
    }

    // 区间 DP 核心循环
    // len: 区间长度，从 2 到 n
    for (int len = 2; len <= n; ++len) {
        // i: 区间起始点，从 0 到 n - len
        for (int i = 0; i <= n - len; ++i) {
            // j: 区间结束点
            int j = i + len - 1;

            // 初始化 dp[i][j] 为一个足够大的数
            dp[i][j] = INF;

            // 情况一：分割合并 (S1S2 形式)
            // 将 s[i...j] 分割成 s[i...k] 和 s[k+1...j]
            for (int k = i; k < j; ++k) {
                dp[i][j] = std::min(dp[i][j], dp[i][k] + dp[k + 1][j]);
            }

            // 情况二：首尾匹配 ((S) 或 [S] 形式)
            // 如果 s[i] 和 s[j] 是一对匹配的括号
            if (is_match(c[i], c[j])) {
                // 如果是空区间 s[i+1...j-1]，则 dp[i+1][j-1] 为 0
                // 例如 s="()"，i=0, j=1。s[i+1...j-1] 是空串，dp[1][0] = 0。
                if (i + 1 > j - 1) { // 对应空区间
                    dp[i][j] = std::min(dp[i][j], 0);
                } else { // 对应非空子区间
                    dp[i][j] = std::min(dp[i][j], dp[i + 1][j - 1]);
                }
            }
        }
    }

    std::cout << dp[0][n - 1] << std::endl;

    return 0;
}
```

**时间复杂度：**
*   三层循环，每层循环的次数大约是 `N`。
*   总时间复杂度为 `O(N^3)`。
*   空间复杂度：`dp` 数组 `O(N^2)`。
