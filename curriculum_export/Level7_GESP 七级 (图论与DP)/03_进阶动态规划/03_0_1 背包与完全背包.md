## 知识点讲解

### 1. 动态规划回顾

在深入背包问题之前，我们先快速回顾一下动态规划（DP）的基本思想：
*   **最优子结构：** 问题的最优解包含其子问题的最优解。
*   **无后效性：** 某个阶段的状态一旦确定，就不受之后决策的影响。
*   **重叠子问题：** 解决问题时，会重复计算相同的子问题。
动态规划通过存储子问题的解，避免重复计算，从而提高效率。

### 2. 0/1 背包问题 (0/1 Knapsack Problem)

#### 问题描述
有 `n` 件物品，每件物品有自己的 `重量 w[i]` 和 `价值 v[i]`。现在有一个背包，其最大承重是 `W`。每件物品只能选择 **一次** (0件或1件)。问：如何选择物品，使得装入背包的物品总价值最大？

#### 状态表示
我们用 `dp[j]` 来表示：**当背包容量为 `j` 时，所能获得的最大价值。**

#### 状态转移方程
考虑第 `i` 件物品：
*   **不选择第 `i` 件物品：** 此时背包的最大价值就是不考虑第 `i` 件物品时，容量为 `j` 的背包所能装载的最大价值，即 `dp[j]` (在当前迭代中，`dp[j]` 存储的是未考虑第 `i` 件物品时的最大价值)。
*   **选择第 `i` 件物品：** 前提是背包容量 `j` 必须大于或等于第 `i` 件物品的重量 `w[i]`。如果选择第 `i` 件物品，那么它会占用 `w[i]` 的容量，并带来 `v[i]` 的价值。此时，我们还需要在剩下的 `j - w[i]` 容量中，寻找之前物品能提供的最大价值，也就是 `dp[j - w[i]]`。所以选择第 `i` 件物品的总价值是 `dp[j - w[i]] + v[i]`。

综合这两种情况，我们取最大值：
`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`

#### 循环顺序的秘密（核心！）
这是0/1背包一维优化的关键点。
为了确保每件物品只被选择一次，我们在更新 `dp` 数组时，对于容量 `j` 的循环，必须采用 **倒序**（从 `W` 到 `w[i]`）的方式。

**为什么是倒序？**
假设我们正序遍历 `j`（从 `w[i]` 到 `W`）：
`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
当计算 `dp[j]` 时，`dp[j - w[i]]` 已经被更新过了，它可能已经包含了第 `i` 件物品的价值。这样，第 `i` 件物品就有可能被重复计算多次，这就不符合0/1背包“每件物品只能选择一次”的规则了。

而如果 **倒序** 遍历 `j`：
当计算 `dp[j]` 时，`dp[j - w[i]]` 还没有被当前第 `i` 件物品更新（它仍然是考虑前 `i-1` 件物品时的最优解）。这样，`dp[j - w[i]] + v[i]` 就保证了第 `i` 件物品只被选择了一次。

#### 初始化
*   `dp[0] = 0`：背包容量为0时，最大价值为0。
*   其他 `dp[j]` 初始化为0（表示在没有物品或无法装入物品时，价值为0）。如果题目要求装满背包，则需要初始化为负无穷，只有 `dp[0]` 为0。这里我们假设不要求装满。

### 3. 完全背包问题 (Unbounded Knapsack Problem)

#### 问题描述
有 `n` 件物品，每件物品有自己的 `重量 w[i]` 和 `价值 v[i]`。现在有一个背包，其最大承重是 `W`。每件物品可以被选择 **任意多次**。问：如何选择物品，使得装入背包的物品总价值最大？

#### 状态表示
同0/1背包，`dp[j]` 表示：**当背包容量为 `j` 时，所能获得的最大价值。**

#### 状态转移方程
同0/1背包，`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`。

#### 循环顺序的秘密（核心！）
与0/1背包不同，完全背包允许物品被选择多次。
为了允许物品被选择多次，我们在更新 `dp` 数组时，对于容量 `j` 的循环，必须采用 **正序**（从 `w[i]` 到 `W`）的方式。

**为什么是正序？**
假设我们正序遍历 `j`（从 `w[i]` 到 `W`）：
`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
当计算 `dp[j]` 时，`dp[j - w[i]]` 已经被当前第 `i` 件物品更新过了（或者说，`dp[j - w[i]]` 已经包含了选择第 `i` 件物品的可能）。这意味着，`dp[j - w[i]]` 可能已经包含了第 `i` 件物品，那么 `dp[j - w[i]] + v[i]` 实际上就代表了在 `j - w[i]` 的基础上再添加一件第 `i` 件物品。这正是完全背包所需要的——允许选择多件同种物品！

#### 初始化
同0/1背包，`dp[0] = 0`，其他 `dp[j]` 初始化为0。

### 4. 总结与对比

| 特性       | 0/1 背包问题                 | 完全背包问题                 |
| :--------- | :--------------------------- | :--------------------------- |
| **物品数量** | 每种物品只能选择 0 或 1 次 | 每种物品可以被选择任意多次   |
| **状态表示** | `dp[j]`：容量 `j` 的最大价值 | `dp[j]`：容量 `j` 的最大价值 |
| **状态转移** | `dp[j] = max(dp[j], dp[j - w[i]] + v[i])` | `dp[j] = max(dp[j], dp[j - w[i]] + v[i])` |
| **容量循环** | **倒序** (`W` 到 `w[i]`)      | **正序** (`w[i]` 到 `W`)      |
| **核心区别** | 倒序确保每件物品只用一次     | 正序允许每件物品用多次       |

理解了循环方向的这个关键区别，你就掌握了这两种背包问题的精髓！

## 代码示例

### 1. 0/1 背包问题代码示例

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    // n: 物品数量, W: 背包容量
    int n, W;
    cin >> n >> W;

    // w: 物品重量, v: 物品价值
    int w[n + 1];
    int v[n + 1];
    for (int i = 1; i <= n; ++i)
    {
        cin >> w[i] >> v[i];
    }

    // dp[j] 表示容量为 j 的背包能装的最大价值
    // 初始化为 0，因为不装任何物品价值就是 0
    int dp[W + 1];
    for (int j = 0; j <= W; ++j)
    {
        dp[j] = 0;
    }

    // 遍历每一个物品
    for (int i = 1; i <= n; ++i)
    {
        // 倒序遍历背包容量 j
        // 这样做是为了确保每个物品只被选择一次
        // 如果是正序，dp[j - w[i]] 可能已经包含了当前物品 i，导致重复选择
        for (int j = W; j >= w[i]; --j)
        {
            // 不选择物品 i: dp[j] (保持原值)
            // 选择物品 i: dp[j - w[i]] + v[i] (在剩余容量 j - w[i] 的基础上加上物品 i 的价值)
            // 取两者最大值
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    // 最终答案是容量为 W 的背包能装的最大价值
    cout << dp[W] << endl;

    return 0;
}
```

### 2. 完全背包问题代码示例

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    // n: 物品数量, W: 背包容量
    int n, W;
    cin >> n >> W;

    // w: 物品重量, v: 物品价值
    int w[n + 1];
    int v[n + 1];
    for (int i = 1; i <= n; ++i)
    {
        cin >> w[i] >> v[i];
    }

    // dp[j] 表示容量为 j 的背包能装的最大价值
    // 初始化为 0，因为不装任何物品价值就是 0
    int dp[W + 1];
    for (int j = 0; j <= W; ++j)
    {
        dp[j] = 0;
    }

    // 遍历每一个物品
    for (int i = 1; i <= n; ++i)
    {
        // 正序遍历背包容量 j
        // 这样做是为了允许每个物品被选择多次
        // 当计算 dp[j] 时，dp[j - w[i]] 已经包含了当前物品 i 的最优解
        // 这意味着可以在剩余容量 j - w[i] 中再次选择物品 i
        for (int j = w[i]; j <= W; ++j)
        {
            // 不选择物品 i: dp[j] (保持原值)
            // 选择物品 i: dp[j - w[i]] + v[i] (在剩余容量 j - w[i] 的基础上加上物品 i 的价值)
            // 取两者最大值
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    // 最终答案是容量为 W 的背包能装的最大价值
    cout << dp[W] << endl;

    return 0;
}
```

## 练习题目

### 选择题/填空题

1.  **选择题：** 在0/1背包问题的一维动态规划解法中，关于背包容量 `j` 的循环方向，以下哪项是正确的？
    A. 必须从小到大遍历 (`j` 从 `w[i]` 到 `W`)
    B. 必须从大到小遍历 (`j` 从 `W` 到 `w[i]`)
    C. 从小到大或从大到小都可以，没有区别
    D. 循环方向取决于物品的价值大小

2.  **选择题：** 下列哪种情况最适合使用完全背包问题来解决？
    A. 参加一场寻宝游戏，每个宝藏只能拿一次。
    B. 准备一个旅行背包，每件衣服、鞋子等物品都只有一件。
    C. 在超市购买零食，每种零食可以无限量购买，但总金额有限制。
    D. 规划一个项目，每个任务有固定的完成时间和收益，且任务之间有依赖关系。

3.  **填空题：** 0/1背包问题和完全背包问题，其状态转移方程 `dp[j] = max(dp[j], dp[j - w[i]] + v[i])` 的形式是相同的，但核心区别在于对容量 `j` 的循环方向：0/1背包是______序，而完全背包是______序。

4.  **选择题：** 假设我们有物品1（重量2，价值3）和物品2（重量3，价值4），背包容量为5。如果这是一个0/1背包问题，最终能获得的最大价值是多少？
    A. 3
    B. 4
    C. 7
    D. 5

5.  **填空题：** 假设我们有物品1（重量2，价值3）和物品2（重量3，价值4），背包容量为5。如果这是一个完全背包问题，最终能获得的最大价值是多少？（提示：物品可以重复选取）

---

### 答案与解析

1.  **答案：B**
    **解析：** 在0/1背包问题的一维动态规划中，为了确保每个物品只被选择一次，容量 `j` 必须从大到小（倒序）遍历。这样，当计算 `dp[j]` 时，`dp[j - w[i]]` 仍然是基于前 `i-1` 件物品的解，避免了当前物品 `i` 被重复计算。

2.  **答案：C**
    **解析：**
    A、B 描述的都是物品只能选一次的情况，属于0/1背包。
    C 描述的是每种零食可以无限量购买，符合完全背包的特点。
    D 描述的是带有依赖关系的任务规划，可能需要拓扑排序或更复杂的DP，不直接是完全背包。

3.  **答案：倒，正**
    **解析：** 0/1背包是倒序遍历容量，确保每件物品只用一次。完全背包是正序遍历容量，允许物品重复使用。

4.  **答案：C**
    **解析：**
    *   物品1: w=2, v=3
    *   物品2: w=3, v=4
    *   背包容量 W=5
    初始 `dp = [0, 0, 0, 0, 0, 0]`
    处理物品1 (w=2, v=3):
        `j=5`: `dp[5] = max(dp[5], dp[3]+3) = max(0, 0+3) = 3`
        `j=4`: `dp[4] = max(dp[4], dp[2]+3) = max(0, 0+3) = 3`
        `j=3`: `dp[3] = max(dp[3], dp[1]+3) = max(0, 0+3) = 3`
        `j=2`: `dp[2] = max(dp[2], dp[0]+3) = max(0, 0+3) = 3`
    此时 `dp = [0, 0, 3, 3, 3, 3]`
    处理物品2 (w=3, v=4):
        `j=5`: `dp[5] = max(dp[5], dp[2]+4) = max(3, 3+4) = 7`
        `j=4`: `dp[4] = max(dp[4], dp[1]+4) = max(3, 0+4) = 4`
        `j=3`: `dp[3] = max(dp[3], dp[0]+4) = max(3, 0+4) = 4`
    最终 `dp[5]` 为 7。

5.  **答案：7**
    **解析：**
    *   物品1: w=2, v=3
    *   物品2: w=3, v=4
    *   背包容量 W=5
    初始 `dp = [0, 0, 0, 0, 0, 0]`
    处理物品1 (w=2, v=3):
        `j=2`: `dp[2] = max(dp[2], dp[0]+3) = max(0, 0+3) = 3`
        `j=3`: `dp[3] = max(dp[3], dp[1]+3) = max(0, 0+3) = 3`
        `j=4`: `dp[4] = max(dp[4], dp[2]+3) = max(0, 3+3) = 6` (这里dp[2]已经包含了物品1)
        `j=5`: `dp[5] = max(dp[5], dp[3]+3) = max(0, 3+3) = 6`
    此时 `dp = [0, 0, 3, 3, 6, 6]`
    处理物品2 (w=3, v=4):
        `j=3`: `dp[3] = max(dp[3], dp[0]+4) = max(3, 0+4) = 4`
        `j=4`: `dp[4] = max(dp[4], dp[1]+4) = max(6, 0+4) = 6`
        `j=5`: `dp[5] = max(dp[5], dp[2]+4) = max(6, 3+4) = 7` (这里dp[2]包含了物品1，所以是物品1+物品2)
    最终 `dp[5]` 为 7。
