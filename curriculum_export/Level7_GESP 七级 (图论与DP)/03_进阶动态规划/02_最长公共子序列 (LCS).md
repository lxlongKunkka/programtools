## 知识点讲解

### 1. 什么是子序列？什么是公共子序列？

*   **子序列 (Subsequence)**：一个字符串的子序列是从原字符串中删除 0 个或多个字符后剩下的字符，并且这些字符的相对顺序保持不变。
    *   例如：字符串 "ABCDE" 的子序列有 "ACE", "ABD", "BCE", "A", "ABCDE" 等。
    *   "AED" 不是 "ABCDE" 的子序列，因为 'E' 在 'D' 后面，但在原字符串中 'E' 在 'D' 前面。
*   **公共子序列 (Common Subsequence)**：如果一个序列 S 既是字符串 A 的子序列，又是字符串 B 的子序列，那么 S 就是 A 和 B 的一个公共子序列。
    *   例如：`s1 = "ABCBDAB"`, `s2 = "BDCABA"`
    *   "B" 是公共子序列。
    *   "BA" 也是公共子序列。
*   **最长公共子序列 (Longest Common Subsequence, LCS)**：在所有公共子序列中，长度最长的那一个。
    *   例如：`s1 = "ABCBDAB"`, `s2 = "BDCABA"` 的 LCS 是 "BCBA" 或 "BDAB"，长度为 4。

### 2. 暴力枚举的困境

如果我们要暴力枚举 LCS，我们需要列出 `s1` 的所有子序列，再列出 `s2` 的所有子序列，然后找出它们共同的部分，最后取最长的。
一个长度为 `n` 的字符串，它的子序列有 `2^n` 个！这个数量增长得非常快，效率极低，无法应对稍长一点的字符串。所以，我们需要更高效的方法——动态规划！

### 3. 动态规划思路

动态规划的核心思想是将一个大问题分解成相互重叠的子问题，通过解决子问题来构建原问题的解，并存储子问题的解避免重复计算。

#### 3.1 状态定义

我们用一个二维数组 `dp` 来存储子问题的解。
`dp[i][j]` 表示：字符串 `s1` 的前 `i` 个字符 (`s1[0...i-1]`) 和字符串 `s2` 的前 `j` 个字符 (`s2[0...j-1]`) 的最长公共子序列的长度。

#### 3.2 边界条件

*   当 `i = 0` 时，`s1` 的前 0 个字符是空字符串。空字符串与任何字符串的 LCS 长度都是 0。
    所以，`dp[0][j] = 0` (对于所有 `0 <= j <= len2`)。
*   当 `j = 0` 时，`s2` 的前 0 个字符是空字符串。空字符串与任何字符串的 LCS 长度都是 0。
    所以，`dp[i][0] = 0` (对于所有 `0 <= i <= len1`)。

#### 3.3 状态转移方程

这是动态规划最核心的部分！我们考虑 `s1[i-1]` 和 `s2[j-1]` 这两个字符：

**情况一：当前字符匹配**
如果 `s1[i-1] == s2[j-1]` (即 `s1` 的第 `i` 个字符和 `s2` 的第 `j` 个字符相同)。
这意味着我们找到了一个新的公共字符！那么，`s1` 的前 `i` 个字符和 `s2` 的前 `j` 个字符的 LCS 长度，就等于 `s1` 的前 `i-1` 个字符和 `s2` 的前 `j-1` 个字符的 LCS 长度，再加上 1。
`dp[i][j] = dp[i-1][j-1] + 1`

**情况二：当前字符不匹配**
如果 `s1[i-1] != s2[j-1]`。
这时，这两个字符不能同时出现在 LCS 中。那么我们有两种选择：
1.  **不考虑 `s1[i-1]`**：我们去寻找 `s1` 的前 `i-1` 个字符和 `s2` 的前 `j` 个字符的 LCS。
    它的长度是 `dp[i-1][j]`。
2.  **不考虑 `s2[j-1]`**：我们去寻找 `s1` 的前 `i` 个字符和 `s2` 的前 `j-1` 个字符的 LCS。
    它的长度是 `dp[i][j-1]`。
我们应该取这两种情况中，LCS 长度更长的那个。
`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`

**总结状态转移方程：**
```
如果 s1[i-1] == s2[j-1]：
    dp[i][j] = dp[i-1][j-1] + 1
否则 (s1[i-1] != s2[j-1])：
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

#### 3.4 填充顺序

我们可以从 `i=1` 到 `len1`，从 `j=1` 到 `len2`，逐行或逐列地填充 `dp` 表。因为 `dp[i][j]` 依赖于 `dp[i-1][j-1]`, `dp[i-1][j]`, `dp[i][j-1]`，这些值都是在当前值左边或上边，所以从小到大遍历是正确的顺序。

#### 3.5 最终结果

`dp[len1][len2]` 就是字符串 `s1` 和 `s2` 的最长公共子序列的长度。

### 4. 举例说明

让我们用 `s1 = "AB"`，`s2 = "B"` 来走一遍 DP 过程。
`len1 = 2`, `len2 = 1`。

我们需要一个 `(len1+1) x (len2+1)` 的 `dp` 表，即 `3 x 2` 的表。

初始化 `dp` 表（第 0 行和第 0 列都为 0）：

| `dp` | "" | B (j=1) |
| :--- | :- | :------ |
| **""** | 0  | 0       |
| **A** (i=1) | 0  | ?       |
| **B** (i=2) | 0  | ?       |

---

**1. 填充 `dp[1][1]` (对应 `s1="A"`, `s2="B"`)**
`s1[0] = 'A'`, `s2[0] = 'B'`。
`s1[0] != s2[0]`。
`dp[1][1] = max(dp[0][1], dp[1][0]) = max(0, 0) = 0`

| `dp` | "" | B (j=1) |
| :--- | :- | :------ |
| **""** | 0  | 0       |
| **A** (i=1) | 0  | **0**   |
| **B** (i=2) | 0  | ?       |

---

**2. 填充 `dp[2][1]` (对应 `s1="AB"`, `s2="B"`)**
`s1[1] = 'B'`, `s2[0] = 'B'`。
`s1[1] == s2[0]`。
`dp[2][1] = dp[1][0] + 1 = 0 + 1 = 1`

| `dp` | "" | B (j=1) |
| :--- | :- | :------ |
| **""** | 0  | 0       |
| **A** (i=1) | 0  | 0       |
| **B** (i=2) | 0  | **1**   |

---

最终结果是 `dp[2][1] = 1`。确实，"AB" 和 "B" 的最长公共子序列是 "B"，长度为 1。

### 5. 时间复杂度和空间复杂度

*   **时间复杂度**：我们使用一个 `len1 x len2` 的二维数组，每个单元格的计算都是常数时间操作。因此，总的时间复杂度是 `O(len1 * len2)`。
*   **空间复杂度**：我们使用了一个 `(len1+1) x (len2+1)` 的二维数组来存储 `dp` 值。因此，总的空间复杂度是 `O(len1 * len2)`。
    *   **空间优化 (进阶)**：注意到 `dp[i][j]` 只依赖于 `dp[i-1]` 行和 `dp[i]` 行（的左边部分）。所以，我们可以将空间复杂度优化到 `O(min(len1, len2))`，只需要两个行（或列）的数组来存储。但这会增加代码的复杂性，在入门阶段我们先掌握 `O(len1 * len2)` 的实现。

## 代码示例

下面是计算最长公共子序列长度的 C++ 代码实现。

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    // 读取两个字符串
    string s1, s2;
    cin >> s1 >> s2;

    int n = s1.length(); // s1 的长度
    int m = s2.length(); // s2 的长度

    // 定义 dp 数组
    // dp[i][j] 表示 s1 的前 i 个字符和 s2 的前 j 个字符的最长公共子序列长度
    // 数组大小为 (n+1) x (m+1)
    int dp[n + 1][m + 1];

    // 初始化 dp 数组的边界条件
    // 当 i=0 或 j=0 时，LCS 长度为 0
    for (int i = 0; i <= n; ++i)
    {
        dp[i][0] = 0;
    }
    for (int j = 0; j <= m; ++j)
    {
        dp[0][j] = 0;
    }

    // 填充 dp 数组
    for (int i = 1; i <= n; ++i)
    {
        for (int j = 1; j <= m; ++j)
        {
            // 如果 s1 的第 i 个字符 (s1[i-1]) 和 s2 的第 j 个字符 (s2[j-1]) 相同
            if (s1[i - 1] == s2[j - 1])
            {
                // LCS 长度等于 s1 的前 i-1 个字符和 s2 的前 j-1 个字符的 LCS 长度加 1
                dp[i][j] = dp[i - 1][j - 1] + 1;
            }
            else
            {
                // 如果字符不相同，则取以下两种情况的最大值：
                // 1. 不考虑 s1 的第 i 个字符，LCS 长度为 dp[i-1][j]
                // 2. 不考虑 s2 的第 j 个字符，LCS 长度为 dp[i][j-1]
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }

    // 最终结果存储在 dp[n][m] 中
    cout << dp[n][m] << endl;

    return 0;
}
```

**代码运行示例：**

输入：
```
ABCBDAB
BDCABA
```

输出：
```
4
```

**【拓展思考：如何还原 LCS 路径？】**

如果我们不仅想知道 LCS 的长度，还想知道具体的 LCS 是什么，我们可以通过回溯 `dp` 表来找到它。
从 `dp[n][m]` 开始，根据状态转移方程逆向查找：
1.  如果 `s1[i-1] == s2[j-1]`，说明 `s1[i-1]` 是 LCS 的一个字符，将其加入结果，然后移动到 `dp[i-1][j-1]`。
2.  如果 `s1[i-1] != s2[j-1]`：
    *   如果 `dp[i-1][j] > dp[i][j-1]`，说明是 `s1` 的前 `i-1` 个字符和 `s2` 的前 `j` 个字符的 LCS 更长，所以我们忽略 `s1[i-1]`，移动到 `dp[i-1][j]`。
    *   否则 (`dp[i][j-1] >= dp[i-1][j]`)，说明是 `s1` 的前 `i` 个字符和 `s2` 的前 `j-1` 个字符的 LCS 更长，我们忽略 `s2[j-1]`，移动到 `dp[i][j-1]`。
    *   注意：如果 `dp[i-1][j] == dp[i][j-1]`，可能有多条路径可以得到最长 LCS，任意选择一个方向即可。

```cpp
#include <bits/stdc++.h>

using namespace std;

int main()
{
    string s1, s2;
    cin >> s1 >> s2;

    int n = s1.length();
    int m = s2.length();

    int dp[n + 1][m + 1];

    for (int i = 0; i <= n; ++i)
    {
        dp[i][0] = 0;
    }
    for (int j = 0; j <= m; ++j)
    {
        dp[0][j] = 0;
    }

    for (int i = 1; i <= n; ++i)
    {
        for (int j = 1; j <= m; ++j)
        {
            if (s1[i - 1] == s2[j - 1])
            {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            }
            else
            {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }

    cout << "LCS Length: " << dp[n][m] << endl;

    // 回溯构造 LCS
    string lcs_str = "";
    int i = n, j = m;
    while (i > 0 && j > 0)
    {
        // 如果当前字符匹配，说明它是 LCS 的一部分
        if (s1[i - 1] == s2[j - 1])
        {
            lcs_str += s1[i - 1]; // 将字符添加到结果字符串
            i--;
            j--;
        }
        else
        {
            // 如果不匹配，则看是来自 dp[i-1][j] 还是 dp[i][j-1]
            if (dp[i - 1][j] > dp[i][j - 1])
            {
                i--; // 向上移动，相当于S1的第i个字符没有被匹配
            }
            else
            {
                j--; // 向左移动，相当于S2的第j个字符没有被匹配
            }
        }
    }

    // 因为是逆序添加的，所以需要反转字符串
    reverse(lcs_str.begin(), lcs_str.end());
    cout << "LCS String: " << lcs_str << endl;

    return 0;
}
```

**二、填空题**

1.  对于字符串 `s1 = "HELLO"` 和 `s2 = "WORLD"`，它们的最长公共子序列的长度是 ______。

2.  计算两个长度分别为 `N` 和 `M` 的字符串的最长公共子序列，所需要的二维 `dp` 数组的大小通常是 `(______) x (______)`。

3.  最长公共子序列算法的时间复杂度是 `O(______)`。

---

### 答案与解析

**一、选择题答案**

1.  **B. 6**
    *   解析：`S1 = "ABACABA"`, `S2 = "BACABA"`。
        LCS 可以是 "BACABA"，长度为 6。
        或者 "ABACAB"，长度为 6。
        或者 "ACABA"，长度为 5。
        最长的长度为 6。

2.  **B. 最长公共子串要求字符连续，最长公共子序列不要求。**
    *   解析：这是 LCS 和 LCP (最长公共子串) 的最本质区别。子序列可以跳过字符，子串必须是连续的。

3.  **C. `s1` 的前 `i` 个字符和 `s2` 的前 `j` 个字符的最长公共子序列的长度。**
    *   解析：这是动态规划状态定义的核心，也是理解算法的关键。

4.  **B. `max(dp[1][2], dp[2][1])`**
    *   解析：当 `s1[i-1] != s2[j-1]` 时，状态转移方程是 `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。在这里 `i=2, j=2`，所以是 `max(dp[1][2], dp[2][1])`。

**二、填空题答案**

1.  **2**
    *   解析：`s1 = "HELLO"`, `s2 = "WORLD"`。公共字符只有 'L' 和 'O'。它们可以组成 "LO" 或 "OL" 这样的子序列。最长的公共子序列是 "LO" 或 "OL"，长度为 2。

2.  **(N+1) x (M+1)**
    *   解析：为了处理空字符串（即前 0 个字符的情况），`dp` 数组的索引通常从 0 开始，所以需要比实际长度多 1 的空间。

3.  **N*M** (或 `len1 * len2`)
    *   解析：双重循环遍历 `N x M` 的 `dp` 表，每个单元格计算是常数时间。
