## 知识点讲解

### 1. LIS 问题定义

给定一个长度为 `n` 的整数序列 `a = [a_0, a_1, ..., a_{n-1}]`，我们需要找到一个子序列 `[a_{i_1}, a_{i_2}, ..., a_{i_k}]`，使得 `0 <= i_1 < i_2 < ... < i_k < n`，并且 `a_{i_1} < a_{i_2} < ... < a_{i_k}`。我们的目标是找到这样子序列的最大长度 `k`。

**例子：**
序列 `a = [10, 9, 2, 5, 3, 7, 101, 18]`
可能的上升子序列：
*   `[2, 3, 7, 18]` (长度 4)
*   `[2, 5, 7, 18]` (长度 4)
*   `[2, 5, 7, 101]` (长度 4)
*   `[3, 7, 18]` (长度 3)
*   `[10, 101]` (长度 2)
最长上升子序列的长度是 4。

### 2. 暴力解法 (Brute Force)

最直观的方法是枚举所有可能的子序列，然后检查它们是否是上升的，并记录最长的。
一个长度为 `n` 的序列有 `2^n` 个子序列（每个元素选或不选）。对于每个子序列，我们还需要遍历一遍来检查其单调性。这种方法的复杂度是指数级的 `O(N * 2^N)`，对于 `N` 稍大一点（比如 `N > 20`）就无法接受了。所以，我们需要更高效的方法——动态规划！

### 3. 动态规划解法 (O(N^2))

这是解决 LIS 问题最基础的动态规划方法。

**状态定义：**
我们定义 `dp[i]` 表示：以 `a[i]` 这个元素**结尾**的最长上升子序列的长度。

**状态转移方程：**
要计算 `dp[i]`，我们需要考虑 `a[i]` 前面的所有元素 `a[j]`（其中 `j < i`）。
如果 `a[j] < a[i]`，那么 `a[i]` 就可以接在以 `a[j]` 结尾的某个上升子序列后面，形成一个新的更长的上升子序列。
因此，`dp[i]` 的值就等于所有满足 `a[j] < a[i]` 的 `dp[j]` 中的最大值，再加上 `1`（因为 `a[i]` 自己也算一个元素）。
`dp[i] = max(dp[j] + 1)`，其中 `0 <= j < i` 且 `a[j] < a[i]`。

**初始化：**
每个元素自身都可以构成一个长度为 1 的上升子序列。所以，`dp` 数组的所有元素初始值都为 `1`。

**最终结果：**
遍历完整个序列并计算出所有 `dp[i]` 后，整个序列的最长上升子序列的长度就是 `dp` 数组中的最大值。
`max_len = max(dp[0], dp[1], ..., dp[n-1])`。

**举例说明：**
序列 `a = [10, 9, 2, 5, 3, 7, 101, 18]`

1.  `i = 0`, `a[0] = 10`：
    *   `dp[0] = 1` (以 10 结尾，子序列是 `[10]`)
2.  `i = 1`, `a[1] = 9`：
    *   前面没有比 9 小的元素。
    *   `dp[1] = 1` (以 9 结尾，子序列是 `[9]`)
3.  `i = 2`, `a[2] = 2`：
    *   前面没有比 2 小的元素。
    *   `dp[2] = 1` (以 2 结尾，子序列是 `[2]`)
4.  `i = 3`, `a[3] = 5`：
    *   `j = 0, a[0] = 10` (10 不小于 5)
    *   `j = 1, a[1] = 9` (9 不小于 5)
    *   `j = 2, a[2] = 2` (2 小于 5) => `dp[2] + 1 = 1 + 1 = 2`
    *   `dp[3] = 2` (以 5 结尾，子序列是 `[2, 5]`)
5.  `i = 4`, `a[4] = 3`：
    *   `j = 2, a[2] = 2` (2 小于 3) => `dp[2] + 1 = 1 + 1 = 2`
    *   `dp[4] = 2` (以 3 结尾，子序列是 `[2, 3]`)
6.  `i = 5`, `a[5] = 7`：
    *   `j = 2, a[2] = 2` (2 小于 7) => `dp[2] + 1 = 2`
    *   `j = 3, a[3] = 5` (5 小于 7) => `dp[3] + 1 = 3`
    *   `j = 4, a[4] = 3` (3 小于 7) => `dp[4] + 1 = 3`
    *   取最大值 `max(2, 3, 3) = 3`
    *   `dp[5] = 3` (以 7 结尾，子序列是 `[2, 5, 7]` 或 `[2, 3, 7]`)
7.  `i = 6`, `a[6] = 101`：
    *   可以接在 `[10]`, `[9]`, `[2]`, `[5]`, `[3]`, `[7]` 后面。
    *   `dp[0]+1=2`, `dp[1]+1=2`, `dp[2]+1=2`, `dp[3]+1=3`, `dp[4]+1=3`, `dp[5]+1=4`
    *   取最大值 `max(2,2,2,3,3,4) = 4`
    *   `dp[6] = 4` (以 101 结尾，子序列是 `[2, 5, 7, 101]`)
8.  `i = 7`, `a[7] = 18`：
    *   可以接在 `[10]`, `[9]`, `[2]`, `[5]`, `[3]`, `[7]` 后面。
    *   `dp[0]+1=2` (10不小于18), `dp[1]+1=2` (9不小于18), `dp[2]+1=2`, `dp[3]+1=3`, `dp[4]+1=3`, `dp[5]+1=4`
    *   取最大值 `max(2,2,2,3,3,4) = 4`
    *   `dp[7] = 4` (以 18 结尾，子序列是 `[2, 5, 7, 18]`)

最终 `dp` 数组为 `[1, 1, 1, 2, 2, 3, 4, 4]`。最大值为 4。

**时间复杂度：**
外层循环 `i` 遍历 `n` 次，内层循环 `j` 遍历 `i` 次。总复杂度为 `O(N^2)`。
**空间复杂度：**
`dp` 数组需要 `O(N)` 空间。

### 4. 优化动态规划解法 (O(N log N))

`O(N^2)` 的解法在 `N` 达到几千时就会超时 (例如 `N=5000` 时 `N^2 = 2.5 * 10^7`，勉强通过；`N=10000` 时 `N^2 = 10^8`，可能超时)。当 `N` 达到 `10^5` 甚至 `10^6` 时，我们需要 `O(N log N)` 的解法。

这个优化方法的核心思想是：维护一个特殊的数组 `t`（我们称之为 `tails` 数组），它存储的是**所有长度为 `k+1` 的上升子序列中，最小的结尾元素**。

**`t` 数组的性质：**
`t` 数组是严格单调递增的。
为什么？假设存在 `t[k] >= t[k+1]`。这意味着长度为 `k+1` 的上升子序列的最小结尾元素 `t[k+1]`，比长度为 `k` 的上升子序列的最小结尾元素 `t[k]` 还要小或相等。
那么，我们可以取以 `t[k+1]` 结尾的长度为 `k+1` 的子序列的前 `k` 个元素，它们的最后一个元素一定小于 `t[k+1]`。如果这个元素小于 `t[k]`，那么我们就可以用它来更新 `t[k]`，使得 `t[k]` 更小，这与 `t[k]` 是最小结尾元素的定义矛盾。所以 `t` 数组必须是严格单调递增的。

**算法步骤：**
1.  初始化 `t` 数组为空。`len` 变量记录 `t` 数组的当前长度（也是当前找到的最长 LIS 的长度）。
2.  遍历输入序列 `a` 中的每个元素 `x`：
    *   **情况一：** 如果 `t` 数组为空，或者 `x` 大于 `t` 数组的最后一个元素 (`t[len-1]`)，这意味着 `x` 可以直接接在当前最长的 LIS 后面，使 LIS 长度增加 1。
        *   将 `x` 添加到 `t` 数组的末尾。
        *   `len` 加 1。
    *   **情况二：** 如果 `x` 不大于 `t` 数组的最后一个元素，这意味着 `x` 不能直接延长当前最长的 LIS。但是，`x` 有可能替代 `t` 数组中的某个元素，从而为未来的元素提供一个更小的“结尾”，使得后续的 LIS 能够更长。
        *   在 `t` 数组中，找到第一个大于等于 `x` 的元素。由于 `t` 数组是单调递增的，我们可以使用二分查找（C++ 中的 `std::lower_bound` 函数）来高效地找到这个位置。
        *   用 `x` 替换掉这个位置上的元素。
        *   **注意：** 替换操作并不会改变 `t` 数组的长度 `len`。但是，它确保了在相同长度的 LIS 中，我们总是维护一个最小的结尾元素，这是一种贪心策略，保证了后续能接上的元素更多。

3.  遍历结束后，`t` 数组的长度 `len` 就是最长上升子序列的长度。

**举例说明：**
序列 `a = [10, 9, 2, 5, 3, 7, 101, 18]`
`t` 数组最初为空，`len = 0`。

1.  `x = 10`：
    *   `t` 为空，将 `10` 加入 `t`。
    *   `t = [10]`, `len = 1`
2.  `x = 9`：
    *   `9` 不大于 `t` 的最后一个元素 `10`。
    *   在 `t` 中找到第一个 `>= 9` 的元素，是 `10`。用 `9` 替换 `10`。
    *   `t = [9]`, `len = 1`
    *   **思考：** 为什么替换是正确的？因为以 9 结尾的长度为 1 的 LIS 比以 10 结尾的 LIS 更“有潜力”接上后面的元素。
3.  `x = 2`：
    *   `2` 不大于 `t` 的最后一个元素 `9`。
    *   在 `t` 中找到第一个 `>= 2` 的元素，是 `9`。用 `2` 替换 `9`。
    *   `t = [2]`, `len = 1`
4.  `x = 5`：
    *   `5` 大于 `t` 的最后一个元素 `2`。
    *   将 `5` 加入 `t`。
    *   `t = [2, 5]`, `len = 2`
5.  `x = 3`：
    *   `3` 不大于 `t` 的最后一个元素 `5`。
    *   在 `t` 中找到第一个 `>= 3` 的元素，是 `5`。用 `3` 替换 `5`。
    *   `t = [2, 3]`, `len = 2`
    *   **思考：** 为什么替换？虽然 `[2,5]` 和 `[2,3]` 都是长度为 2 的 LIS，但 `[2,3]` 的结尾更小，更有机会在未来接上更多的元素。
6.  `x = 7`：
    *   `7` 大于 `t` 的最后一个元素 `3`。
    *   将 `7` 加入 `t`。
    *   `t = [2, 3, 7]`, `len = 3`
7.  `x = 101`：
    *   `101` 大于 `t` 的最后一个元素 `7`。
    *   将 `101` 加入 `t`。
    *   `t = [2, 3, 7, 101]`, `len = 4`
8.  `x = 18`：
    *   `18` 不大于 `t` 的最后一个元素 `101`。
    *   在 `t` 中找到第一个 `>= 18` 的元素，是 `101`。用 `18` 替换 `101`。
    *   `t = [2, 3, 7, 18]`, `len = 4`

遍历结束，`len = 4`，即最长上升子序列的长度为 4。

**时间复杂度：**
遍历 `n` 个元素，每个元素在 `t` 数组中进行一次二分查找。二分查找的复杂度是 `O(log L)`，其中 `L` 是 `t` 数组的当前长度，`L <= N`。
总复杂度为 `O(N log N)`。
**空间复杂度：**
`t` 数组需要 `O(N)` 空间。

**注意：** `t` 数组中存储的并不是一个真实的 LIS，它只是帮助我们计算 LIS 长度的一个辅助数组。如果需要输出 LIS 具体序列，需要做额外的记录。

## 代码示例

### 1. O(N^2) 动态规划解法

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve_n_squared() {
    int n;
    cin >> n;
    int a[n]; // 输入序列
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    int d[n]; // dp[i] 表示以 a[i] 结尾的最长上升子序列的长度

    // 初始化：每个元素自身构成一个长度为 1 的 LIS
    for (int i = 0; i < n; ++i) {
        d[i] = 1;
    }

    int max_len = 0; // 记录最终最长 LIS 的长度

    // 遍历序列中的每个元素作为 LIS 的结尾
    for (int i = 0; i < n; ++i) {
        // 查找在 a[i] 之前，哪个元素 a[j] 可以作为 a[i] 的前驱
        for (int j = 0; j < i; ++j) {
            // 如果 a[j] 小于 a[i]，则 a[i] 可以接在以 a[j] 结尾的 LIS 后面
            if (a[j] < a[i]) {
                // 更新 dp[i]，取所有可能的前驱中，能形成最长 LIS 的那个
                d[i] = max(d[i], d[j] + 1);
            }
        }
        // 更新全局最长 LIS 长度
        max_len = max(max_len, d[i]);
    }

    // 如果序列为空，max_len 默认为 0
    if (n == 0) {
        cout << 0 << endl;
    } else {
        cout << max_len << endl;
    }
}

int main() {
    cout << "--- O(N^2) LIS Solution ---" << endl;
    cout << "Enter number of elements (n): ";
    // Example Input:
    // 8
    // 10 9 2 5 3 7 101 18
    solve_n_squared();
    return 0;
}
```

### 2. O(N log N) 动态规划解法

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve_n_log_n() {
    int n;
    cin >> n;
    int a[n]; // 输入序列
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    // tails 数组，存储所有长度为 k+1 的上升子序列中，最小的结尾元素
    // 使用 std::vector 方便动态增删和二分查找
    vector<int> t;

    // 遍历输入序列中的每个元素
    for (int i = 0; i < n; ++i) {
        // 找到第一个大于等于当前元素 a[i] 的位置
        // lower_bound 返回一个迭代器，指向范围 [first, last) 中第一个不小于 val 的元素
        auto it = lower_bound(t.begin(), t.end(), a[i]);

        // 如果 it 指向 t 的末尾，说明 a[i] 大于 t 中所有元素
        // 此时 a[i] 可以接在当前最长的 LIS 后面，使其长度加 1
        if (it == t.end()) {
            t.push_back(a[i]);
        } else {
            // 否则，a[i] 可以替换掉 t 中找到的那个元素
            // 这样做是为了在相同长度的 LIS 中，保持结尾元素尽可能小
            // 从而为后续元素提供更多接续的机会，使 LIS 有潜力更长
            *it = a[i];
        }
    }

    // t 数组的长度就是最长上升子序列的长度
    cout << t.size() << endl;
}

int main() {
    cout << "--- O(N log N) LIS Solution ---" << endl;
    cout << "Enter number of elements (n): ";
    // Example Input:
    // 8
    // 10 9 2 5 3 7 101 18
    solve_n_log_n();
    return 0;
}
```

    *   `t` 数组中的元素组合起来，是不是一个真实的 LIS？为什么？（不一定，它只是记录了长度和结尾的“可能性”，例如 `[2, 3, 7, 18]` 是一个可能的 LIS，但 `t` 数组在过程中可能是 `[2, 5, 7, 101]` 然后变成 `[2, 5, 7, 18]`，但 `[2, 5, 7, 18]` 并不代表 `18` 后面跟着 `7` 后面跟着 `5` 后面跟着 `2`，它只是说存在一个以 `18` 结尾长度为 4 的 LIS。）

### 编程题答案

**题目一：最长上升子序列长度 (基础版)**
使用 O(N^2) 动态规划解法。

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false); // 提高输入输出效率
    cin.tie(NULL);

    int n;
    cin >> n;
    int a[n];
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }

    int d[n]; // d[i] 表示以 a[i] 结尾的最长上升子序列的长度

    for (int i = 0; i < n; ++i) {
        d[i] = 1; // 初始值，每个元素自身构成长度为 1 的 LIS
    }

    int max_len = 1; // 至少有一个元素时，LIS 长度至少为 1

    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < i; ++j) {
            if (a[j] < a[i]) {
                d[i] = max(d[i], d[j] + 1);
            }
        }
        max_len = max(max_len, d[i]);
    }

    cout << max_len << endl;

    return 0;
}
```

**题目二：最长上升子序列长度 (进阶版)**
使用 O(N log N) 动态规划解法。

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false); // 提高输入输出效率
    cin.tie(NULL);

    int n;
    cin >> n;
    int a[n];
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }

    vector<int> t; // tails 数组

    for (int i = 0; i < n; ++i) {
        // 找到第一个大于等于 a[i] 的元素的位置
        auto it = lower_bound(t.begin(), t.end(), a[i]);

        if (it == t.end()) {
            // 如果没找到（a[i] 比 t 中所有元素都大），则添加到末尾
            t.push_back(a[i]);
        } else {
            // 否则，替换掉找到的那个元素
            *it = a[i];
        }
    }

    cout << t.size() << endl;

    return 0;
}
```
