## 教学目标

### 知识目标
1.  **理解函数参数传递的两种基本方式**：学生能够区分“值传递”和“引用传递”在概念上的区别。
2.  **掌握引用参数的语法和作用**：学生能够识别并在函数定义中使用 `&` 符号来声明引用参数。
3.  **理解值传递和引用传递对变量操作的影响**：学生能够解释为什么值传递不能修改外部变量，而引用传递可以。

### 能力目标
1.  **能够分析代码中参数传递的方式**：学生在阅读代码时，能够判断函数参数是值传递还是引用传递。
2.  **能够根据需求选择合适的参数传递方式**：当需要函数修改外部变量时，学生能够主动选择引用传递。
3.  **能够正确编写使用引用参数的函数**：学生能够独立设计并实现带有引用参数的函数，以解决特定问题（如交换两个变量的值）。

### 素养目标
1.  **培养严谨的编程思维**：通过深入理解参数传递机制，学生能够更细致地思考程序执行的每一步。
2.  **提升解决问题的能力**：学生能够根据问题的具体要求（是否需要修改外部变量），灵活运用不同的参数传递方法。
3.  **初步建立“效率”和“数据安全”的意识**：理解值传递保护原始数据，引用传递可能更高效（避免复制），但需谨慎使用。

你写完了一道数学题，想让小明帮你检查。你怕小明不小心把你的原版作业弄脏或改错，所以你悄悄地把这道题**复印了一份**，把复印件给了小明。小明在复印件上圈圈画画，做了很多修改。
问题：小明在复印件上的修改，会影响到你的**原版作业**吗？（不会！）

现在，你和小明要一起布置教室。教室里有一个很大的黑板报，你希望小明能直接帮你修改黑板报上的内容。你不能给小明一个黑板报的“复印件”（因为黑板报没法复印啊！），你直接给了小明**教室的钥匙**，告诉他黑板报在哪里。小明拿着钥匙进了教室，直接在黑板报上写写画画。
问题：小明在黑板报上的修改，会影响到**真正的黑板报**吗？（会！）
这就是编程里的“引用传递”：函数拿到的是变量的“钥匙”（或者说“地址”），可以通过这把钥匙直接找到并修改原版变量。

今天，我们就来学习编程世界里这两种有趣的“传递作业”和“共享钥匙”的方式——值传递和引用传递！

## 深度知识点讲解

### 1. 函数参数传递：函数与外部世界的沟通桥梁

在C++中，函数就像一个小小的“加工厂”，我们给它原材料（参数），它加工后可能返回一个产品（返回值）。参数传递就是我们把“原材料”送进“加工厂”的方式。主要有两种方式：值传递和引用传递。

### 2. 值传递 (Value Passing) —— “复印件模式”

**核心思想**：函数在被调用时，会为每个参数创建一个**全新的、独立的副本**。函数内部对这些参数的任何操作，都只作用于这些副本，不会影响到函数外部的原始变量。

**生活类比**：就像你把一张纸条上的信息（比如“100”）告诉你的朋友。你的朋友可以在他自己的纸条上写下“100”，然后他可以把自己的纸条撕掉、涂改，但你手里的那张写着“100”的纸条是完全不受影响的。

**工作原理**：
1.  当函数被调用时，系统会在内存中为函数内部的参数开辟新的空间。
2.  然后，将外部实际参数的值**复制**一份到这些新的空间中。
3.  函数内部的操作，都是针对这些**副本**进行的。
4.  函数执行完毕后，这些副本的内存空间会被释放，对外部变量没有任何影响。

**语法特点**：
参数类型后面**没有** `&` 符号。
例如：`void printNumber(int n)`，这里的 `n` 就是值传递。

**优点**：
*   **安全**：原始数据得到保护，不会被函数意外修改。
*   **简单**：易于理解和使用，是函数参数传递的默认方式。

**缺点**：
*   **效率**：如果传递的数据量很大（比如一个大型结构体，虽然GESP 4不涉及），复制操作会消耗时间和内存。
*   **局限**：无法通过函数修改外部变量。

**代码示例**：
```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

// 这是一个值传递的函数
void addOne(int n)
{
    n = n + 1; // 在函数内部，修改的是 n 的副本
    std::cout << "在函数内部，n 变成了：" << n << std::endl;
}

int main()
{
    int a = 5;
    std::cout << "调用函数前，a 的值是：" << a << std::endl;
    addOne(a); // 调用 addOne 函数，把 a 的值 5 复制一份给 n
    std::cout << "调用函数后，a 的值是：" << a << std::endl; // a 的值仍然是 5
    return 0;
}
```
**输出结果**：
```
调用函数前，a 的值是：5
在函数内部，n 变成了：6
调用函数后，a 的值是：5
```
**思考**：为什么 `a` 的值没有变？因为 `addOne` 函数接收的是 `a` 的一个“复印件” `n`，它只在复印件上加了1，原件 `a` 并没有被动到。

### 3. 引用传递 (Reference Passing) —— “共享钥匙模式”

**核心思想**：函数在被调用时，参数不再是原始变量的副本，而是原始变量的一个**别名（alias）**或者说是一个**直接的链接**。函数内部对这个别名的任何操作，都等同于直接操作函数外部的原始变量。

**生活类比**：你有一个外号叫“小可爱”，你的真名叫“张三”。当别人叫“小可爱”的时候，指的就是你张三本人。对“小可爱”说的话，就是对张三本人说的话。引用参数就是变量的“外号”。

**工作原理**：
1.  当函数被调用时，系统不会为引用参数开辟新的内存空间来存储值。
2.  引用参数会**直接指向**外部实际参数所在的内存地址。
3.  函数内部对引用参数的任何读写操作，都会直接作用于外部的原始变量。

**语法特点**：
参数类型后面跟着一个 `&` 符号。
例如：`void modifyNumber(int &n)`，这里的 `n` 就是引用传递。

**优点**：
*   **修改外部变量**：这是引用传递最主要的功能，允许函数通过参数改变外部变量的值。
*   **效率高**：避免了复制大型数据时的开销，因为只是传递了一个“链接”或“地址”，而不是整个数据。

**缺点**：
*   **不安全**：由于可以直接修改外部变量，如果使用不当，可能会导致意想不到的错误，降低代码的安全性。
*   **理解难度**：对于初学者来说，理解其工作原理比值传递稍微复杂一些。

**代码示例**：
```cpp
#include <bits/stdc++.h> // 包含所有常用头文件

// 这是一个引用传递的函数
void addOneRef(int &n) // 注意这里的 & 符号
{
    n = n + 1; // 在函数内部，修改的是外部变量 a 本身
    std::cout << "在函数内部，n 变成了：" << n << std::endl;
}

int main()
{
    int a = 5;
    std::cout << "调用函数前，a 的值是：" << a << std::endl;
    addOneRef(a); // 调用 addOneRef 函数，n 变成了 a 的别名
    std::cout << "调用函数后，a 的值是：" << a << std::endl; // a 的值变成了 6
    return 0;
}
```
**输出结果**：
```
调用函数前，a 的值是：5
在函数内部，n 变成了：6
调用函数后，a 的值是：6
```
**思考**：这次 `a` 的值为什么变了？因为 `addOneRef` 函数接收的是 `a` 的“钥匙” `n`，通过 `n` 直接找到了 `a` 所在的内存位置，并对 `a` 的值进行了修改。

### 4. 值传递与引用传递的对比

| 特性         | 值传递 (Value Passing)                       | 引用传递 (Reference Passing)                 |
| :----------- | :------------------------------------------- | :------------------------------------------- |
| **语法**     | `void func(int x)`                           | `void func(int &x)`                          |
| **本质**     | 传递的是变量的**副本**                       | 传递的是变量的**别名**或**直接链接**         |
| **内存**     | 为参数开辟新的内存空间，存储副本             | 参数直接指向原变量的内存空间                 |
| **对原变量** | 不会影响原变量                               | 可以直接修改原变量                           |
| **安全性**   | 高，原变量受保护                             | 低，需谨慎，可能意外修改原变量               |
| **效率**     | 复制操作有开销 (尤其对大对象)                | 没有复制开销，效率高                         |
| **使用场景** | 不希望函数修改外部变量时；传递小数据类型     | 需要函数修改外部变量时；传递大数据类型 (GESP 4 初步了解) |

**常见误区**：
1.  **忘记 `&`**：想要通过函数修改外部变量，却忘记在参数类型前加上 `&`，导致修改无效。
2.  **混淆 `&` 的作用**：在函数参数中 `int &n` 表示引用；在表达式中 `&a` 通常表示取变量 `a` 的地址（指针相关，GESP 4 暂时不深究）。这里我们只关注函数参数中的 `&`。
3.  **误以为值传递可以修改外部变量**：认为只要函数内部对参数进行了操作，外部变量也会跟着变。

记住，值传递是“复印件”，引用传递是“钥匙”！

## 典型例题精讲

### 例题一：交换两个数（值传递的失败尝试）

**题目描述**：
编写一个函数 `swapVal`，尝试交换两个整数 `a` 和 `b` 的值。在 `main` 函数中调用此函数，并观察 `a` 和 `b` 的值是否真的被交换。

**思路分析**：
1.  我们定义一个函数 `swapVal(int x, int y)`，它接收两个 `int` 类型的参数。由于没有 `&`，这会是值传递。
2.  在函数内部，我们使用一个临时变量 `t` 来实现 `x` 和 `y` 的交换逻辑。
3.  因为是值传递，函数内部操作的是 `x` 和 `y` 的副本，而不是 `main` 函数中 `a` 和 `b` 的原始变量。
4.  所以，即使函数内部 `x` 和 `y` 成功交换了，函数结束后，`a` 和 `b` 的值也不会改变。

**代码**：
```cpp
#include <bits/stdc++.h>

// 尝试使用值传递交换两个数
void swapVal(int x, int y) // x 和 y 是 a 和 b 的副本
{
    std::cout << "--- 进入 swapVal 函数 ---" << std::endl;
    std::cout << "函数内部原始值：x = " << x << ", y = " << y << std::endl;

    int t = x; // t 存储 x 的副本值
    x = y;     // x 的副本被修改为 y 的副本值
    y = t;     // y 的副本被修改为 t (原 x 的副本值)

    std::cout << "函数内部交换后：x = " << x << ", y = " << y << std::endl;
    std::cout << "--- 退出 swapVal 函数 ---" << std::endl;
}

int main()
{
    int a = 10;
    int b = 20;

    std::cout << "调用 swapVal 前：a = " << a << ", b = " << b << std::endl;
    swapVal(a, b); // 将 a 和 b 的值复制给 swapVal 函数的参数 x 和 y
    std::cout << "调用 swapVal 后：a = " << a << ", b = " << b << std::endl;

    return 0;
}
```

**运行结果**：
```
调用 swapVal 前：a = 10, b = 20
--- 进入 swapVal 函数 ---
函数内部原始值：x = 10, y = 20
函数内部交换后：x = 20, y = 10
--- 退出 swapVal 函数 ---
调用 swapVal 后：a = 10, b = 20
```
**复杂度**：时间复杂度 O(1)，空间复杂度 O(1)。

### 例题二：交换两个数（引用传递的成功实现）

**题目描述**：
编写一个函数 `swapRef`，利用引用传递来真正交换两个整数 `a` 和 `b` 的值。在 `main` 函数中调用此函数，并观察 `a` 和 `b` 的值是否被交换。

**思路分析**：
1.  为了让函数能够修改外部变量，我们需要使用引用传递。所以，函数定义为 `swapRef(int &x, int &y)`。
2.  在函数内部，`x` 和 `y` 不再是副本，而是 `main` 函数中 `a` 和 `b` 的别名。
3.  对 `x` 和 `y` 的任何操作，都将直接作用于 `a` 和 `b`。
4.  使用临时变量 `t` 来完成交换逻辑，这次交换是直接作用在 `a` 和 `b` 上的。

**代码**：
```cpp
#include <bits/stdc++.h>

// 使用引用传递交换两个数
void swapRef(int &x, int &y) // x 是 a 的别名，y 是 b 的别名
{
    std::cout << "--- 进入 swapRef 函数 ---" << std::endl;
    std::cout << "函数内部原始值：x = " << x << ", y = " << y << std::endl;

    int t = x; // t 存储 x (也就是 a) 的值
    x = y;     // x (也就是 a) 被修改为 y (也就是 b) 的值
    y = t;     // y (也就是 b) 被修改为 t (原 a 的值)

    std::cout << "函数内部交换后：x = " << x << ", y = " << y << std::endl;
    std::cout << "--- 退出 swapRef 函数 ---" << std::endl;
}

int main()
{
    int a = 10;
    int b = 20;

    std::cout << "调用 swapRef 前：a = " << a << ", b = " << b << std::endl;
    swapRef(a, b); // 调用 swapRef 函数，a 和 b 的引用被传递
    std::cout << "调用 swapRef 后：a = " << a << ", b = " << b << std::endl;

    return 0;
}
```

**运行结果**：
```
调用 swapRef 前：a = 10, b = 20
--- 进入 swapRef 函数 ---
函数内部原始值：x = 10, y = 20
函数内部交换后：x = 20, y = 10
--- 退出 swapRef 函数 ---
调用 swapRef 后：a = 20, b = 10
```
**复杂度**：时间复杂度 O(1)，空间复杂度 O(1)。

### 例题三：计算并修改结果

**题目描述**：
编写一个函数 `calculate`，它接收两个整数 `num1` 和 `num2`，计算它们的和以及它们的乘积。要求函数返回它们的和，并通过引用参数修改外部变量来存储它们的乘积。

**思路分析**：
1.  函数需要返回和，所以函数的返回值类型是 `int`。
2.  函数需要修改外部变量来存储乘积，所以乘积的参数需要使用引用传递。
3.  函数的输入 `num1` 和 `num2` 不需要被修改，所以可以使用值传递。

**代码**：
```cpp
#include <bits/stdc++.h>

// 计算和并修改乘积
// a 和 b 是值传递，不会被函数修改
// p 是引用传递，函数内部修改 p 会影响外部变量
int calculate(int a, int b, int &p)
{
    p = a * b; // 计算乘积，并通过引用修改外部变量
    return a + b; // 返回和
}

int main()
{
    int n1 = 3;
    int n2 = 4;
    int productResult = 0; // 用于存储乘积的外部变量

    std::cout << "调用 calculate 前：productResult = " << productResult << std::endl;

    int sumResult = calculate(n1, n2, productResult); // 调用函数

    std::cout << "调用 calculate 后：" << std::endl;
    std::cout << "和 (sumResult) = " << sumResult << std::endl;
    std::cout << "积 (productResult) = " << productResult << std::endl;

    return 0;
}
```

**运行结果**：
```
调用 calculate 前：productResult = 0
调用 calculate 后：
和 (sumResult) = 7
积 (productResult) = 12
```
**复杂度**：时间复杂度 O(1)，空间复杂度 O(1)。

## 代码实现模板

这里提供一个使用引用参数的标准函数框架，帮助同学们快速上手。

```cpp
#include <bits/stdc++.h> // 包含所有常用的C++标准库头文件

// 函数定义模板
// 类型1 参数1, 类型2 &参数2, ...
// 参数1 是值传递：函数内部对它的一切操作，都不会影响到函数外部的原始变量。
// 参数2 是引用传递：函数内部对它的一切操作，都会直接影响到函数外部的原始变量。
void myUsefulFunction(int valParam, int &refParam)
{
    // ----------------------------------------------------------------
    // 1. 访问参数的值
    // valParam 和 refParam 都可以像普通变量一样被读取
    std::cout << "函数内部：valParam 的当前值是 " << valParam << std::endl;
    std::cout << "函数内部：refParam 的当前值是 " << refParam << std::endl;

    // ----------------------------------------------------------------
    // 2. 修改参数的值
    // 对 valParam 的修改只影响函数内部的副本
    valParam = valParam * 2;
    std::cout << "函数内部：valParam 修改后变成 " << valParam << std::endl;

    // 对 refParam 的修改会直接影响到外部的变量！
    refParam = refParam + 10;
    std::cout << "函数内部：refParam 修改后变成 " << refParam << std::endl;

    // ----------------------------------------------------------------
    // 3. 其他逻辑...
    // 可以在这里进行任何你需要的计算或操作
}

int main()
{
    // ----------------------------------------------------------------
    // 定义外部变量
    int a = 5;  // 对应函数中的 valParam
    int b = 10; // 对应函数中的 refParam

    std::cout << "--- 调用函数前 ---" << std::endl;
    std::cout << "外部变量 a = " << a << std::endl;
    std::cout << "外部变量 b = " << b << std::endl;

    // ----------------------------------------------------------------
    // 调用函数
    // 将 a 的值传递给 valParam (值传递)
    // 将 b 的引用传递给 refParam (引用传递)
    myUsefulFunction(a, b);

    std::cout << "--- 调用函数后 ---" << std::endl;
    std::cout << "外部变量 a = " << a << std::endl; // a 的值保持不变 (因为是值传递)
    std::cout << "外部变量 b = " << b << std::endl; // b 的值会改变 (因为是引用传递)

    return 0;
}
```

**运行结果示例**：
```
--- 调用函数前 ---
外部变量 a = 5
外部变量 b = 10
函数内部：valParam 的当前值是 5
函数内部：refParam 的当前值是 10
函数内部：valParam 修改后变成 10
函数内部：refParam 修改后变成 20
--- 调用函数后 ---
外部变量 a = 5
外部变量 b = 20
```

**总结**：
*   当函数参数前面**没有 `&`** 时，是**值传递**，函数操作的是变量的“复印件”，不影响外部原变量。
*   当函数参数前面**有 `&`** 时，是**引用传递**，函数操作的是变量的“钥匙”，会直接修改外部原变量。

## 课堂互动

**1. 思考与讨论：**
*   **提问 1**：如果我想写一个函数 `printInfo(int age, std::string name)`，只用来打印一个人的年龄和名字，我需要使用引用传递吗？为什么？
    *   **预期回答**：不需要。因为 `printInfo` 只是读取年龄和名字，不需要修改它们。值传递已经足够，并且更安全。
*   **提问 2**：如果我写了一个函数 `resetScore(int s)`，希望把一个同学的分数 `s` 重置为 0。但是运行后发现 `main` 函数里同学的真实分数并没有变成 0。你觉得是哪里出错了？应该怎么修改？
    *   **预期回答**：`s` 是值传递，函数内部修改的是 `s` 的副本，没有影响到外部变量。应该把 `s` 改成引用传递：`void resetScore(int &s)`。
*   **提问 3**：为什么在实现 `swap` 函数时，值传递会失败，而引用传递会成功？请用“复印件”和“钥匙”的比喻来解释。
    *   **预期回答**：值传递时，函数拿到的是两个数的复印件，交换的只是复印件，原件没动。引用传递时，函数拿到的是两个数的钥匙，通过钥匙直接进入内存，交换的是原件。
*   **提问 4**：在 C++ 中，`&` 符号除了在函数参数中表示引用之外，你还在哪里见过它？（提示：整数按位与运算符，或者取地址符，但 GESP 4 暂时不深究取地址）
    *   **预期回答**：可能在一些位运算中见过（例如 `a & b`），那是按位与操作符。强调在函数参数中 `&` 有特殊含义。

**2. 小组任务：**

    *   函数内部实现将 `x` 和 `y` 各自乘以 2。
    *   `main` 函数中初始化两个整数，调用函数，并打印结果。

    ```cpp
    #include <bits/stdc++.h>

    void sortTwo(int &a, int &b) {
        if (a > b) {
            int t = a;
            a = b;
            b = t;
        }
    }

    int main() {
        int x = 5, y = 2;
        std::cout << "Before sort: x = " << x << ", y = " << y << std::endl;
        sortTwo(x, y);
        std::cout << "After sort: x = " << x << ", y = " << y << std::endl; // x=2, y=5

        int p = 10, q = 20;
        std::cout << "Before sort: p = " << p << ", q = " << q << std::endl;
        sortTwo(p, q);
        std::cout << "After sort: p = " << p << ", q = " << q << std::endl; // p=10, q=20
        return 0;
    }
    ```
