## 趣味引入

同学们，大家有没有玩过乐高积木或者拼装模型呢？当你拿到一盒乐高，里面有很多小零件，还有一本厚厚的说明书。说明书里可能有很多步骤，比如“搭建一个轮子”、“组装一个机翼”等等。

想象一下，如果每次你需要一个轮子，你都要从头开始描述“拿起这个圆片，再拿起那个轴，把它们拼起来……”，是不是很麻烦？

而我们通常的做法是，说明书上会有一个小章节叫做“**如何搭建一个轮子**”，里面详细说明了所有步骤。以后，无论哪里需要轮子，我们只要说一声“**搭建一个轮子**”就可以了，而不用重复所有的细节。

在编程世界里，函数就扮演着这个“如何搭建一个轮子”的角色！它是一段预先写好的代码，用来完成一个特定的任务。当我们需要这个任务时，只需要“叫”一声它的名字，它就会自动去完成，是不是很酷？

让我们来看一个例子。假设我们想在程序中打印一些分隔线来让输出更美观：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    cout << "-------------" << endl; // 分隔线1
    cout << "欢迎来到我的程序！" << endl;
    cout << "-------------" << endl; // 分隔线2
    cout << "这是一个很棒的功能！" << endl;
    cout << "-------------" << endl; // 分隔线3

    return 0;
}
```

这段代码中，`cout << "-------------" << endl;` 这行代码重复出现了好几次。如果分隔线样式需要改变呢？比如从 `-------------` 变成 `***********`，那我们是不是要修改好几处？有没有更“偷懒”又“聪明”的办法呢？

答案就是——**函数**！今天，我们就来学习如何使用这个强大的“小助手”！

## 知识点讲解

### 1. 什么是函数？

函数就像是程序中的一个“小工厂”或者“子程序”，它负责完成一个特定的、独立的功能。我们可以给这个“小工厂”一个名字，当需要它工作时，只要“喊”它的名字就行了。

**为什么我们要用函数呢？**

*   **代码复用：** 一段代码写好后，可以在程序的任何地方多次使用，避免重复编写。
*   **程序模块化：** 把一个大程序分成一个个小功能模块，每个模块由一个函数完成，让程序结构更清晰，更容易理解和管理。
*   **提高可读性：** 通过有意义的函数名，我们可以一眼看出这个函数是做什么的，让代码更容易阅读。
*   **便于维护：** 如果某个功能需要修改，只需要修改对应的函数，而不会影响程序的其他部分。

### 2. 函数的定义

定义函数，就是告诉计算机这个“小工厂”叫什么名字、它需要什么材料（参数）、它会生产出什么产品（返回值），以及它具体怎么工作（函数体）。

函数的定义格式如下：

```cpp
返回类型 函数名(参数类型 参数1, 参数类型 参数2, ...)
{
    // 函数体：执行特定任务的代码
    // 如果函数有返回值，需要使用 return 语句
    return 返回值; // 如果返回类型不是 void
}
```

让我们来逐一解释这些部分：

*   **`返回类型` (Return Type)：**
    *   表示函数执行完毕后，会“带回来”什么类型的结果。
    *   比如，如果函数是计算两个整数的和，那么它会返回一个 `int` 类型的结果，所以返回类型就是 `int`。
    *   如果函数只是完成一个动作，不“带回来”任何结果（比如只负责打印一条消息），那么它的返回类型就是 `void`。`void` 就像是“空”的意思，表示没有返回值。

*   **`函数名` (Function Name)：**
    *   这是函数的名称，就像我们的名字一样，用来唯一标识这个函数。
    *   函数名应该有意义，能反映函数的功能（比如 `add`、`printMessage`）。
    *   命名规则和变量名类似，通常使用小驼峰命名法（第一个单词小写，后续单词首字母大写，如 `printLine`）。

*   **`参数列表` (Parameter List)：**
    *   括号 `()` 里面是函数的“输入材料”，也叫形式参数。
    *   它告诉函数在执行任务时需要哪些数据。
    *   每个参数都需要指定它的类型和名称，参数之间用逗号 `,` 分隔。
    *   如果函数不需要任何输入，参数列表可以为空，写成 `()` 或者 `(void)`。
    *   例如：`int add(int x, int y)` 表示 `add` 函数需要两个 `int` 类型的参数，分别叫做 `x` 和 `y`。

*   **`函数体` (Function Body)：**
    *   花括号 `{}` 里面是函数具体执行的代码，也就是这个“小工厂”的工作流程。
    *   所有的任务指令都写在这里面。

*   **`return` 语句：**
    *   如果函数的返回类型不是 `void`，那么在函数体中必须使用 `return` 语句来返回一个值。
    *   `return` 语句会结束函数的执行，并将指定的值返回给调用它的地方。
    *   例如：`return sum;`

### 3. 函数的调用

定义好函数后，我们就可以在程序中“使用”它了，这个过程叫做函数调用。

调用函数的格式很简单：

```cpp
函数名(实际参数1, 实际参数2, ...);
```

*   **`实际参数` (Actual Arguments)：**
    *   调用函数时，我们传递给函数的值，它们会对应地传递给函数定义中的形式参数。
    *   实际参数的类型和顺序必须与形式参数列表中的定义一致。
    *   例如：`int z = add(3, 5);` 这里 `3` 和 `5` 就是实际参数。

**重要概念：`main` 函数**

我们之前所有的代码都写在 `int main() { ... }` 里面。其实，`main` 函数也是一个特殊的函数！它是我们程序的“入口点”，程序总是从 `main` 函数开始执行。它返回一个 `int` 类型的值（通常是 `0` 表示程序正常结束），没有参数。

### 4. 函数的声明 (前向声明)

有时候，我们可能会把函数的定义写在 `main` 函数的后面，但 `main` 函数又需要调用它。这时候，编译器在编译 `main` 函数时，会发现它要调用的函数还没有被定义过，就会报错。

为了解决这个问题，我们需要在 `main` 函数之前先“声明”一下这个函数。函数声明就像是给编译器一个“预告”，告诉它“嘿，我后面会定义一个叫这个名字的函数，你先知道一下！”。

函数声明的格式：

```cpp
返回类型 函数名(参数类型, 参数类型, ...); // 注意末尾有分号
```

*   声明时不需要写函数体，只需要写返回类型、函数名和参数类型列表。参数名可以省略，但最好写上，增加可读性。

**总结一下定义和声明：**
*   **定义：** 告诉编译器函数具体做什么。
*   **声明：** 告诉编译器函数长什么样（返回类型、函数名、参数），但不说明具体实现。

## 代码示例

让我们通过几个具体的例子来理解函数的定义与调用。

### 示例 1：无参无返回值的函数 (`void`)

我们来解决引入部分的问题，用函数来打印分隔线。

```cpp
#include <bits/stdc++.h>
using namespace std;

// 定义一个函数，用来打印一条分隔线
// 返回类型是 void，表示这个函数不返回任何值
// 参数列表为空，表示这个函数不需要任何输入
void printLine()
{
    cout << "-------------" << endl;
}

int main()
{
    printLine(); // 调用 printLine 函数
    cout << "欢迎来到我的程序！" << endl;
    printLine(); // 再次调用 printLine 函数
    cout << "这是一个很棒的功能！" << endl;
    printLine(); // 又一次调用 printLine 函数

    return 0;
}
```

### 示例 2：带参无返回值的函数 (`void`)

如果我想打印不同长度或不同字符的分隔线怎么办？我们可以给函数传递参数！

```cpp
#include <bits/stdc++.h>
using namespace std;

// 定义一个函数，用来打印指定字符和长度的分隔线
// s: 要打印的字符
// l: 分隔线的长度
void printCustomLine(char s, int l)
{
    for (int i = 0; i < l; i++)
    {
        cout << s;
    }
    cout << endl;
}

int main()
{
    printCustomLine('-', 15); // 调用函数，打印15个'-'
    cout << "欢迎来到我的程序！" << endl;
    printCustomLine('*', 20); // 调用函数，打印20个'*'
    cout << "这是一个很棒的功能！" << endl;
    printCustomLine('=', 10); // 调用函数，打印10个'='

    return 0;
}
```

### 示例 3：带参有返回值的函数 (`int`)

现在我们来写一个计算两个整数之和的函数。

```cpp
#include <bits/stdc++.h>
using namespace std;

// 定义一个函数，计算两个整数的和
// x, y: 两个要相加的整数
// 返回类型是 int，因为和是整数
int add(int x, int y)
{
    int sum = x + y;
    return sum; // 返回计算出的和
}

// 定义一个函数，比较两个整数并返回较大的那个
int getMax(int x, int y)
{
    if (x > y)
    {
        return x;
    }
    else
    {
        return y;
    }
}

int main()
{
    int a = 10, b = 25;
    int c = add(a, b); // 调用 add 函数，将返回值赋给 c
    cout << "10 + 25 = " << c << endl;

    cout << "50 + 70 = " << add(50, 70) << endl; // 直接在输出语句中调用函数

    int m = getMax(a, b); // 调用 getMax 函数
    cout << "10 和 25 中较大的数是: " << m << endl;

    int p = 100, q = 50;
    cout << "100 和 50 中较大的数是: " << getMax(p, q) << endl;

    return 0;
}
```

### 示例 4：函数的声明与定义分离

当函数定义在 `main` 函数之后时，我们需要先声明它。

```cpp
#include <bits/stdc++.h>
using namespace std;

// 函数声明 (Forward Declaration)
// 告诉编译器，有一个叫做 multiply 的函数，它接收两个 int 参数，并返回一个 int 值。
int multiply(int x, int y);

// 另一个函数声明
void sayHello(int n);

int main()
{
    int a = 6, b = 7;
    int res = multiply(a, b); // 在这里调用 multiply，编译器已经知道它的存在
    cout << a << " * " << b << " = " << res << endl;

    sayHello(3); // 调用 sayHello

    return 0;
}

// 函数定义 (Function Definition)
// 具体的实现写在 main 函数之后
int multiply(int x, int y)
{
    return x * y;
}

void sayHello(int n)
{
    for (int i = 0; i < n; i++)
    {
        cout << "Hello, World!" << endl;
    }
}
```

**小提示：** 在信奥比赛中，为了方便和习惯，我们通常会把所有函数定义都写在 `main` 函数之前，这样就不用额外写声明了。但了解声明的作用也是很重要的哦！

### 练习题目答案与解析

1.  **C.** `int myFunc(int a, int b) { return a + b; }`
    *   A 选项错误，参数列表中的 `b` 缺少类型声明。
    *   B 选项是函数声明的格式，缺少函数体，不能算作完整的定义。
    *   D 选项缺少返回类型。

2.  **B.** `void`
    *   `void` 关键字表示“空”，用于声明函数不返回任何值。

3.  **C.** `main()`
    *   `main()` 函数是 C++ 程序的固定入口点。

4.  **13**
    *   `calculate(2, 3)` 意味着 `x = 2`, `y = 3`。
    *   函数体计算 `2 * 2 + 3 * 3 = 4 + 9 = 13`。

5.  **声明**
    *   当函数定义在 `main` 函数之后时，为了让编译器知道这个函数的存在和它的签名（返回类型、函数名、参数），需要在 `main` 函数之前进行函数声明（或前向声明）。
