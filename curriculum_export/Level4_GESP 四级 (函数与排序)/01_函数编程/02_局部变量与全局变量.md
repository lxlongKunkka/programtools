## 知识点讲解

### 1. 变量的小复习

还记得变量是什么吗？它就像一个“盒子”，可以用来存放数据。比如 `int a = 10;` 就是定义了一个叫做 `a` 的盒子，里面放着整数 `10`。

### 2. 局部变量 (Local Variables)

*   **定义**：在函数内部（比如 `main` 函数里面，或者我们自己定义的其他函数里面）、循环（`for`, `while`）或条件判断（`if`）的代码块 `{}` 内部定义的变量，就是局部变量。
*   **作用域**：它们只在定义它们的代码块内部有效，出了这个块，它们就“隐身”了，别人访问不到。
*   **生命周期**：当程序执行到定义局部变量的那一行时，它就诞生了；当程序执行完它所在的代码块时，它就“消失”了。
*   **特点**：
    *   不同的函数可以有同名的局部变量，它们之间互不影响，就像两个同学都有自己的笔，虽然都叫“笔”，但不是同一支。
    *   局部变量在没有初始化时，它的值是不确定的（一个随机的垃圾值）。

### 3. 全局变量 (Global Variables)

*   **定义**：在所有函数外部定义的变量，也就是在程序的“最外面”定义的变量。
*   **作用域**：从它被定义的那一行开始，一直到整个程序文件的末尾，都有效。程序里的所有函数都可以访问它，甚至修改它。
*   **生命周期**：当程序启动时，全局变量就被创建了；当程序运行结束时，全局变量才会被销毁。它贯穿了程序的整个生命周期。
*   **特点**：
    *   如果全局变量在定义时没有初始化，它会被自动初始化为0（对于数值类型）、空字符（对于字符类型）或 `nullptr`（对于指针类型）。
    *   **重要提示**：虽然全局变量很方便，但过度使用它会让程序变得复杂，难以理解和维护，还容易引发意想不到的错误。我们应该尽量少用全局变量，除非确实需要多个函数共享同一个数据。

### 4. 局部变量与全局变量同名时的“遮蔽”现象

当一个局部变量与一个全局变量同名时，在局部变量的作用域内，局部变量会“遮蔽”全局变量。也就是说，当你在函数内部使用这个变量名时，访问到的是局部变量，而不是全局变量。这就像你在教室里，即使黑板上写着“苹果”，但你面前的桌子上放着一个写着“苹果”的纸条，你首先看到并使用的是桌子上的纸条。

### 5. 总结对比

| 特性     | 局部变量                                  | 全局变量                                  |
| :------- | :---------------------------------------- | :---------------------------------------- |
| **定义位置** | 函数内部、代码块内部                      | 所有函数外部                              |
| **作用域** | 只在定义它的函数或代码块内部有效          | 从定义处到整个程序文件结束都有效          |
| **生命周期** | 随代码块的进入而创建，随代码块的退出而销毁 | 随程序启动而创建，随程序结束而销毁        |
| **初始值** | 未初始化时值不确定（随机值）              | 未初始化时自动初始化为0（或对应类型的默认值） |
| **同名情况** | 与全局变量同名时会“遮蔽”全局变量          | 可能被局部变量“遮蔽”                      |
| **使用建议** | 推荐优先使用，提高程序模块化和安全性      | 尽量少用，仅在必要时使用，提高程序可维护性 |

## 代码示例

让我们通过一个C++程序来实际看看局部变量和全局变量是如何工作的吧！

```cpp
#include <bits/stdc++.h>
using namespace std;

// 这是一个全局变量，所有函数都可以访问和修改它
int g = 100; 

// 一个简单的函数，用来演示局部变量和全局变量
void f() 
{
    // 这是一个局部变量，只在函数 f 内部有效
    int l = 20; 
    cout << "在函数 f 内部：" << endl;
    cout << "局部变量 l 的值是: " << l << endl; // 访问函数 f 自己的局部变量 l
    cout << "全局变量 g 的值是: " << g << endl; // 访问全局变量 g
    
    // 尝试修改全局变量 g
    g = 200; 
    cout << "在函数 f 内部，修改全局变量 g 后，g 的值是: " << g << endl;
}

// 另一个函数，演示局部变量和全局变量同名的情况
void h()
{
    // 这个 'g' 是函数 h 的局部变量，它和全局变量 g 同名
    int g = 50; 
    cout << "\n在函数 h 内部：" << endl;
    cout << "局部变量 g 的值是: " << g << endl; // 访问函数 h 自己的局部变量 g
    // 注意：在这里直接使用 g 访问的是局部变量 g，而不是全局变量 g
}


int main() 
{
    // 这是一个局部变量，只在 main 函数内部有效
    int m = 10; 

    cout << "在 main 函数开始时：" << endl;
    cout << "局部变量 m 的值是: " << m << endl;
    cout << "全局变量 g 的值是: " << g << endl;

    // 调用函数 f
    f(); 

    cout << "\n回到 main 函数后：" << endl;
    // 尝试访问函数 f 中的局部变量 l，会出错！因为它已经“消失”了。
    // cout << "函数 f 内部的局部变量 l 的值是: " << l << endl; // 这一行如果取消注释会报错！

    cout << "局部变量 m 的值是: " << m << endl; // main 函数的局部变量 m 依然存在
    cout << "全局变量 g 的值是: " << g << endl; // 全局变量 g 的值已经被函数 f 修改了！

    // 调用函数 h
    h();

    cout << "\n再次回到 main 函数后：" << endl;
    cout << "全局变量 g 的值是: " << g << endl; // 注意，函数 h 内部的局部变量 g 并没有影响到全局变量 g
    
    return 0;
}
```

**运行结果会是什么样呢？**

```
在 main 函数开始时：
局部变量 m 的值是: 10
全局变量 g 的值是: 100

在函数 f 内部：
局部变量 l 的值是: 20
全局变量 g 的值是: 100
在函数 f 内部，修改全局变量 g 后，g 的值是: 200

回到 main 函数后：
局部变量 m 的值是: 10
全局变量 g 的值是: 200

在函数 h 内部：
局部变量 g 的值是: 50

再次回到 main 函数后：
全局变量 g 的值是: 200
```

通过这个例子，我们可以清楚地看到：
*   `l` 和 `m` 是局部变量，只能在各自的函数内部使用。
*   `g` 是全局变量，它的值可以在不同函数中被访问和修改。
*   在 `h` 函数中，局部变量 `g` 遮蔽了全局变量 `g`，所以 `h` 函数内部的 `cout << g;` 打印的是局部 `g` 的值 `50`，并没有改变全局 `g` 的值。

## 练习题目

### 选择题/填空题

1.  以下关于局部变量的描述，哪一项是正确的？
    A. 可以在程序的任何地方被访问。
    B. 在定义时如果没有初始化，会自动被赋值为0。
    C. 它的生命周期从定义时开始，到其所在的代码块结束时销毁。
    D. 不同的函数不能有同名的局部变量。

2.  如果一个全局变量 `a` 和一个局部变量 `a` 同时存在于一个函数中，那么在这个函数内部，当程序使用 `a` 时，访问到的是_______变量。

3.  全局变量的特点之一是：如果它在定义时没有被显式初始化，它会被自动初始化为______。

4.  请判断以下代码片段中，变量 `k` 的作用域是哪里？
    ```cpp
    #include <bits/stdc++.h>
    using namespace std;

    int g = 1;

    void process()
    {
        int k = 10;
        if (k > g)
        {
            int p = 5;
            // 在这里 k 和 p 都可以访问
        }
        // 在这里 k 可以访问，但 p 不能访问
    }

    int main()
    {
        // 在这里 g 可以访问，k 和 p 都不能访问
        process();
        return 0;
    }
    ```
    A. 整个程序文件
    B. `main` 函数内部
    C. `process` 函数内部
    D. `if` 语句的代码块内部

5.  在C++中，为了提高程序的模块化和减少错误，我们通常推荐优先使用_______变量。

### 答案与解析

1.  **答案**：C
    **解析**：
    A. 错误，局部变量只在其定义的作用域内可访问。
    B. 错误，局部变量未初始化时值不确定。全局变量才会自动初始化为0。
    C. 正确，这是局部变量的生命周期定义。
    D. 错误，不同函数可以有同名的局部变量，它们是独立的。

2.  **答案**：局部
    **解析**：当局部变量和全局变量同名时，局部变量会“遮蔽”全局变量，程序会优先访问局部变量。

3.  **答案**：0
    **解析**：全局变量在未初始化时，数值类型会被自动初始化为0。

4.  **答案**：C
    **解析**：变量 `k` 在 `process` 函数内部定义，所以它的作用域是整个 `process` 函数。变量 `p` 在 `if` 语句块内部定义，作用域更小。

5.  **答案**：局部
    **解析**：优先使用局部变量可以限制变量的作用范围，避免不必要的修改，使程序更容易理解、测试和维护。
