## 知识点讲解

### 什么是选择排序？

选择排序（Selection Sort）是一种简单直观的排序算法。它的核心思想是：

1.  **第一轮：** 从待排序的数据元素中选出最小（或最大）的一个元素，存放到序列的起始位置，也就是已排序部分的第一个位置。
2.  **第二轮：** 再从剩余未排序的元素中继续寻找最小（或最大）的元素，然后放到已排序部分的第二个位置。
3.  **重复：** 重复这个过程，直到所有元素都排序完毕。

就像我们刚才找同学排队一样，每次都“选择”出当前最小的，然后“放置”到它应该在的位置。

### 选择排序的详细步骤（以升序为例）

我们以数组 `a = [5, 2, 8, 1, 9]` 为例，来一步步看看它是如何进行排序的：

**原始数组：`[5, 2, 8, 1, 9]`**

**第一轮（找到最小的元素，放到第 0 个位置）：**
*   在 `[5, 2, 8, 1, 9]` 中，最小的元素是 `1`。
*   `1` 应该和当前第一个元素 `5` 交换位置。
*   **交换后：`[1, 2, 8, 5, 9]`**
*   此时，`[1]` 是已排序部分。

**第二轮（从剩余元素中找到最小的，放到第 1 个位置）：**
*   在剩余未排序部分 `[2, 8, 5, 9]` 中，最小的元素是 `2`。
*   `2` 应该和当前未排序部分的第一个元素 `2` 交换位置（这里 `2` 已经在正确位置，不发生实际数据移动）。
*   **交换后：`[1, 2, 8, 5, 9]`**
*   此时，`[1, 2]` 是已排序部分。

**第三轮（从剩余元素中找到最小的，放到第 2 个位置）：**
*   在剩余未排序部分 `[8, 5, 9]` 中，最小的元素是 `5`。
*   `5` 应该和当前未排序部分的第一个元素 `8` 交换位置。
*   **交换后：`[1, 2, 5, 8, 9]`**
*   此时，`[1, 2, 5]` 是已排序部分。

**第四轮（从剩余元素中找到最小的，放到第 3 个位置）：**
*   在剩余未排序部分 `[8, 9]` 中，最小的元素是 `8`。
*   `8` 应该和当前未排序部分的第一个元素 `8` 交换位置（不发生实际数据移动）。
*   **交换后：`[1, 2, 5, 8, 9]`**
*   此时，`[1, 2, 5, 8]` 是已排序部分。

**排序完成！** 当只剩下一个元素时，它自然就是最大的，也就在它正确的位置上了，无需再进行比较。所以，对于 N 个元素的数组，我们只需要进行 N-1 轮选择。

### 算法特点

1.  **时间复杂度：** O(N^2)
    *   在每一轮中，我们都需要遍历未排序的部分来找到最小（或最大）的元素。
    *   第一轮需要比较 N-1 次，第二轮需要比较 N-2 次，以此类推，直到最后一轮比较 1 次。
    *   总的比较次数大约是 `(N-1) + (N-2) + ... + 1 = N * (N-1) / 2`，所以是 O(N^2)。
    *   交换次数最多是 N-1 次。
    *   无论原始数组的顺序如何（有序、逆序、随机），比较次数都是固定的 O(N^2)。
2.  **空间复杂度：** O(1)
    *   选择排序是一种“原地排序”算法，它只需要常数级别的额外空间来存储临时变量（比如交换时使用的临时变量 `t`）。
3.  **稳定性：** 不稳定
    *   选择排序是**不稳定**的排序算法。这意味着如果有两个相同的元素，排序后它们的相对位置可能会发生改变。例如，对于数组 `[5a, 8, 5b]`，如果 `5a` 在 `5b` 前面。当选择 `5a` 放到第一个位置时，如果 `5b` 在后续的交换中被移动到 `5a` 之前，那么就失去了稳定性。

## 代码示例

下面是使用 C++ 语言实现选择排序的代码。请大家仔细阅读，并注意代码中的注释，它们会帮助你理解每一步的作用。

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，方便竞赛使用
using namespace std;       // 使用标准命名空间

// selectionSort 函数用于对整数数组进行选择排序（升序）
// a: 待排序的数组
// n: 数组的元素个数
void selectionSort(int a[], int n)
{
    // 外层循环：控制已排序部分的边界
    // i 从 0 到 n-2，因为当 i 达到 n-1 时，只剩最后一个元素，它自然就是最大的，无需再比较
    for (int i = 0; i < n - 1; i++)
    {
        // 假设当前未排序部分的第一个元素 a[i] 是最小的
        int min_idx = i; // min_idx 存储当前最小元素的索引

        // 内层循环：从 i+1 开始，寻找未排序部分中的最小元素
        // j 从 i+1 到 n-1，遍历所有未排序的元素
        for (int j = i + 1; j < n; j++)
        {
            // 如果找到一个比当前 min_idx 指向的元素更小的元素
            if (a[j] < a[min_idx])
            {
                min_idx = j; // 更新 min_idx 为新找到的最小元素的索引
            }
        }

        // 经过内层循环，min_idx 已经指向了未排序部分中的最小元素
        // 如果最小元素不在当前位置 i，则进行交换
        if (min_idx != i)
        {
            // 经典的交换操作：使用一个临时变量 t
            int t = a[i];
            a[i] = a[min_idx];
            a[min_idx] = t;
        }
    }
}

int main()
{
    int a[100]; // 定义一个足够大的数组，最多可以存储 100 个整数
    int n;      // 存储数组的实际元素个数

    cout << "请输入数组的元素个数 (1-100): ";
    cin >> n; // 读取元素个数

    // 检查输入是否合法
    if (n <= 0 || n > 100)
    {
        cout << "元素个数不合法，请输入 1 到 100 之间的整数。" << endl;
        return 1; // 异常退出
    }

    cout << "请输入 " << n << " 个整数，用空格分隔: ";
    // 循环读取 n 个整数到数组 a 中
    for (int i = 0; i < n; i++)
    {
        cin >> a[i];
    }

    cout << "原始数组: ";
    // 打印原始数组
    for (int i = 0; i < n; i++)
    {
        cout << a[i] << " ";
    }
    cout << endl;

    selectionSort(a, n); // 调用选择排序函数对数组进行排序

    cout << "排序后的数组: ";
    // 打印排序后的数组
    for (int i = 0; i < n; i++)
    {
        cout << a[i] << " ";
    }
    cout << endl;

    return 0; // 程序正常结束
}
```

## 练习题目

### 选择题或填空题

1.  选择排序算法在每一轮迭代中，都会从当前**未排序**的部分中找到一个特定的元素（通常是最小或最大），并将其放置到**已排序**部分的末尾。
2.  对数组 `[4, 1, 3, 2]` 进行升序选择排序。第一轮排序结束后，数组变为 `[___, ___, ___, ___]`。
3.  对含有 `N` 个元素的数组进行选择排序，其时间复杂度为 `O(____)`。
4.  下列关于选择排序的说法中，正确的是：
    A. 它的空间复杂度是 O(N)。
    B. 它是稳定的排序算法。
    C. 它的比较次数与原始数组的顺序无关。
    D. 它的交换次数总是 N*(N-1)/2。
5.  对数组 `[9, 8, 7, 6]` 进行升序选择排序，在第二轮循环结束后，数组会变为 `[___, ___, ___, ___]`。

### 编程题

**基础题：**
请修改课堂上的 `selectionSort` 函数，使其能够对一个给定数组进行**降序**（从大到小）选择排序。

**进阶题：**
假设有一个包含 N 个学生的数组，每个学生有一个姓名和一个分数。请你使用选择排序的思想，实现一个函数，能够根据学生的分数进行**升序**排序。如果分数相同，则保持原始相对顺序（此题略微超出选择排序的“不稳定”特性，但你可以尝试在交换时考虑如何处理）。

---

### 练习题目答案与解析

1.  **未排序，已排序**
    *   **解析：** 选择排序的核心就是每次从未排序区域选出合适元素，放到已排序区域的末尾。
2.  `[1, 4, 3, 2]`
    *   **解析：** 原始数组 `[4, 1, 3, 2]`。第一轮，找到最小元素 `1`，它在索引 `1`。与索引 `0` 的 `4` 交换。数组变为 `[1, 4, 3, 2]`。
3.  `N^2` (或 `n^2`)
    *   **解析：** 选择排序的比较次数是 `N*(N-1)/2`，所以时间复杂度是 O(N^2)。
4.  **C**
    *   **解析：**
        *   A. 空间复杂度是 O(1)，不是 O(N)。
        *   B. 选择排序是**不稳定**的。
        *   C. 无论数组是否有序，选择排序都需要通过内层循环遍历未排序部分来寻找最小（或最大）元素，所以比较次数是固定的 O(N^2)。
        *   D. 交换次数最多是 N-1 次，不是 N*(N-1)/2。
5.  `[6, 7, 9, 8]`
    *   **解析：**
        *   原始数组 `[9, 8, 7, 6]`。
        *   **第一轮：** 最小元素 `6` (索引 `3`) 与 `9` (索引 `0`) 交换。数组变为 `[6, 8, 7, 9]`。
        *   **第二轮：** 从 `[8, 7, 9]` 中找最小，是 `7` (索引 `2`)。与 `8` (索引 `1`) 交换。数组变为 `[6, 7, 8, 9]`。
        *   第二轮结束后，数组是 `[6, 7, 8, 9]`。
