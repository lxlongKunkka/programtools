## 深度知识点讲解

### 1. 什么是枚举？

**概念本质**：
“枚举”（Enumeration），也叫“穷举”或“暴力破解”，是一种最直接、最朴素的解决问题的方法。它的核心思想是：**把问题的所有可能情况都尝试一遍，然后从这些尝试中找出符合条件的解。**

-   **为什么叫“暴力”？** 因为它不讲究技巧，不拐弯抹角，直接“硬碰硬”，把所有可能性都试一遍。
-   **为什么叫“穷举”？** 因为它把所有可能性都“穷尽”了，一个不漏。

**生活类比**：
-   **找钥匙**：你有很多把长得很像的钥匙，但不知道哪一把能开门。最笨但最有效的方法就是一把一把地试，直到门被打开。每一把钥匙就是一种“可能性”，试钥匙就是“枚举”。
-   **猜密码**：你忘了手机的四位数字密码，但你知道它是由 0-9 组成的。如果你有无限次尝试机会，你可以从 0000 开始，一直试到 9999。这就是枚举。
-   **找特定颜色的糖果**：在一大堆混合颜色的糖果中，你要找出所有红色的糖果。你会把每一颗糖果都拿起来看看，是不是红色的。

### 2. 什么时候用枚举？

枚举虽然“暴力”，但非常有用。它主要适用于以下情况：
1.  **可能性数量有限且较小**：如果所有可能性非常多，比如几十亿种，那么枚举就会花费太长时间。但如果只有几百、几千、几十万种可能性，枚举通常是一个不错的选择。
2.  **没有更优的算法时**：有些问题，我们暂时想不到特别巧妙的解法，或者根本就没有比枚举更快的解法，这时候枚举就是我们的“救星”。

对于我们 GESP 二级的同学来说，遇到的问题通常数据范围不会太大，单层枚举、双层枚举都能轻松应对！

### 3. 单层枚举的核心工具：`for` 循环

在 C++ 编程中，我们要让计算机“一个一个地试”，最常用的工具就是**循环语句**，特别是 `for` 循环。

`for` 循环可以让我们指定一个变量从某个值开始，每次增加或减少一个量，直到达到某个条件为止。这就像我们给计算机下命令：“从 1 号盒子开始，一个一个地检查，直到 10 号盒子！”

一个基本的 `for` 循环长这样：

```cpp
for (int i = /* 起始值 */; i <= /* 终止值 */; i = i + /* 步长 */)
{
    // 在这里写上对当前 i 值进行的操作或判断
}
```

**关键点剖析**：

-   **`int i = /* 起始值 */`**：这是循环的“起点”。我们定义一个整数变量 `i`（通常我们用 `i` 作为循环变量，因为它代表 index，索引），并给它一个初始值。这个 `i` 就是我们当前正在“尝试”的那个可能性。
    -   **思考**：如果我们要从 1 开始找偶数，`i` 应该从几开始？（答：1）
-   **`i <= /* 终止值 */`**：这是循环的“终点”和“条件”。只要 `i` 满足这个条件，循环就会继续。当 `i` 不满足这个条件时，循环就停止。
    -   **思考**：如果我们要找到 100 以内的偶数，`i` 应该到几结束？（答：100）
-   **`i = i + /* 步长 */`**：这是循环变量每次“前进”的步伐。通常，我们是“一个一个地试”，所以步长是 1。`i = i + 1` 也可以简写为 `++i` 或 `i++`。
    -   **思考**：如果我们要找 1 到 100 之间的偶数，`i` 每次增加 1，还是增加 2 呢？（答：增加 1，然后用 `if` 判断是不是偶数）
-   **循环体 `{ ... }`**：这是每次循环时，计算机要执行的具体任务。比如，检查当前的 `i` 是不是偶数，或者是不是我们要找的宝藏。

### 4. 常见误区

1.  **循环范围错误**：起始值、终止值设置不对，导致漏掉一些可能性或者多尝试了不需要的可能性。
    -   例如，题目要求 1 到 N，你写成了 `i < N`，结果 N 就被漏掉了。
    -   **小技巧**：仔细看题目是“到 N”还是“小于 N”，包含不包含 N。
2.  **判断条件错误**：在循环体内部的 `if` 语句写错了，导致找出来的不是我们想要的结果。
    -   例如，要求偶数，你写成了 `i % 2 != 0`（判断奇数）。
    -   **小技巧**：多读几遍题目要求，确认你的 `if` 条件和题目完全匹配。
3.  **变量命名不规范**：虽然 GESP 要求单字母变量，但在实际编程中，良好的变量命名能大大提高代码可读性。但在 GESP 考试中，请严格遵守单字母变量命名规则。

理解了这些，我们就可以开始用单层枚举解决实际问题啦！

## 典型例题精讲

### 例题 1：寻找偶数

**题目描述**：
输入一个正整数 `n` (1 <= `n` <= 100)，请你输出从 1 到 `n` 之间所有的偶数，每个偶数之间用一个空格隔开。

**思路分析**：
1.  **目标**：找出 1 到 `n` 之间的所有偶数。
2.  **枚举对象**：从 1 到 `n` 的每一个整数。
3.  **枚举范围**：
    -   起始值：1
    -   终止值：`n`
    -   步长：1 (因为我们要检查每一个数)
4.  **判断条件**：对于当前枚举到的数 `i`，判断它是否为偶数。偶数的特点是能被 2 整除，即 `i % 2 == 0`。
5.  **操作**：如果 `i` 是偶数，就把它输出。

**C++ 代码实现**：

```cpp
#include <iostream> // 引入输入输出库

int main() // 主函数入口
{
    int n; // 定义一个整数变量 n，用于存储输入的上限
    std::cin >> n; // 从键盘读取 n 的值

    // 开始单层枚举
    // 循环变量 i 从 1 开始
    // 只要 i 小于或等于 n，循环就继续
    // 每次循环结束后，i 增加 1
    for (int i = 1; i <= n; ++i)
    {
        // 在循环体内部，判断当前的 i 是否为偶数
        // 如果 i 除以 2 的余数为 0，说明 i 是偶数
        if (i % 2 == 0)
        {
            std::cout << i << " "; // 输出这个偶数，并在后面加一个空格
        }
    }
    std::cout << std::endl; // 所有偶数输出完毕后，输出一个换行符，让输出更整洁

    return 0; // 程序正常结束
}
```

**复杂度分析**：
-   时间复杂度：`O(n)`。因为 `for` 循环从 1 运行到 `n`，循环体内的操作是常数时间（判断和输出），所以总共执行了 `n` 次操作。当 `n` 比较小时（例如 `n` <= 100），这个速度非常快。
-   空间复杂度：`O(1)`。我们只使用了几个变量来存储数据，没有使用额外的大型数据结构，所以空间占用是常数级别的。

### 例题 2：统计特定倍数

**题目描述**：
输入一个正整数 `n` (1 <= `n` <= 100)，统计从 1 到 `n` 之间有多少个数字是 3 的倍数。

**思路分析**：
1.  **目标**：统计 1 到 `n` 之间 3 的倍数的数量。
2.  **枚举对象**：从 1 到 `n` 的每一个整数。
3.  **枚举范围**：
    -   起始值：1
    -   终止值：`n`
    -   步长：1
4.  **判断条件**：对于当前枚举到的数 `i`，判断它是否为 3 的倍数。3 的倍数的特点是能被 3 整除，即 `i % 3 == 0`。
5.  **操作**：如果 `i` 是 3 的倍数，我们就让一个计数器加 1。
6.  **额外变量**：我们需要一个变量来记录 3 的倍数的数量，初始化为 0。

**C++ 代码实现**：

```cpp
#include <iostream> // 引入输入输出库

int main() // 主函数入口
{
    int n; // 定义一个整数变量 n，用于存储输入的上限
    std::cin >> n; // 从键盘读取 n 的值

    int c = 0; // 定义一个计数器变量 c，并初始化为 0，用于统计 3 的倍数的数量

    // 开始单层枚举
    // 循环变量 i 从 1 开始
    // 只要 i 小于或等于 n，循环就继续
    // 每次循环结束后，i 增加 1
    for (int i = 1; i <= n; ++i)
    {
        // 在循环体内部，判断当前的 i 是否为 3 的倍数
        // 如果 i 除以 3 的余数为 0，说明 i 是 3 的倍数
        if (i % 3 == 0)
        {
            c = c + 1; // 如果是 3 的倍数，计数器 c 增加 1
        }
    }

    std::cout << c << std::endl; // 循环结束后，输出计数器 c 的值，即 3 的倍数的总数量

    return 0; // 程序正常结束
}
```

**复杂度分析**：
-   时间复杂度：`O(n)`。与例题 1 类似，循环执行 `n` 次，每次操作为常数时间。
-   空间复杂度：`O(1)`。只使用了少量变量。

### 基础巩固

1.  **输出奇数**：
    输入一个整数 `n` (1 <= `n` <= 100)，输出从 1 到 `n` 之间所有的奇数，每个奇数之间用一个空格隔开。
    *   **输入样例**：`10`
    *   **输出样例**：`1 3 5 7 9`

2.  **统计范围内的数**：
    输入两个整数 `l` 和 `r` (1 <= `l` <= `r` <= 100)，统计从 `l` 到 `r` 之间（包含 `l` 和 `r`）有多少个整数。
    *   **输入样例**：`5 10`
    *   **输出样例**：`6`

### 能力提升

1.  **找公倍数**：
    输入一个整数 `n` (1 <= `n` <= 100)，输出从 1 到 `n` 之间所有既能被 3 整除又能被 5 整除的数，每个数之间用一个空格隔开。
    *   **输入样例**：`30`
    *   **输出样例**：`15 30`

2.  **计算偶数和**：
    输入一个整数 `n` (1 <= `n` <= 100)，计算 1 到 `n` 之间所有偶数的和。
    *   **输入样例**：`10`
    *   **输出样例**：`30` (2+4+6+8+10 = 30)

### 拓展挑战

1.  **判断素数**：

    *   **输出样例 1**：`Yes`
    *   **输入样例 2**：`9`
    *   **输出样例 2**：`No`
    *   **提示**：需要枚举 2 到 `n-1` 之间的所有数，看 `n` 是否能被它们整除。特别注意 `n` 为 1 或 2 的情况。

2.  **包含特定数字的数**：
    输入一个整数 `n` (1 <= `n` <= 100)，输出从 1 到 `n` 之间所有十位或个位包含数字 '7' 的数，每个数之间用一个空格隔开。
    *   **输入样例**：`20`
    *   **输出样例**：`7 17`
    *   **提示**：对于一个数 `i`，个位是 `i % 10`，十位是 `i / 10 % 10`。
