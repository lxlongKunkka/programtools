## 趣味引入

同学们，大家好！今天我们要学习一种非常厉害的“笨办法”来解决问题，叫做“暴力枚举”！

想象一下，你和你的好朋友小明、小红，三个人一起玩一个寻宝游戏。你们来到一个大大的院子，院子里有好多好多小盒子，每个盒子里可能藏着宝贝，也可能什么都没有。

**场景一：寻找一对宝物**
游戏规则是：你需要在院子里找到**两个不同**的小盒子，它们里面**正好**藏着一对“友谊之戒”。你不知道哪两个盒子是，怎么办呢？
你可能会说：“我先打开第一个盒子，看看里面有没有戒指。如果没有，就再打开第二个盒子，看看它和第一个盒子能不能凑成一对。如果不行，就打开第三个盒子，看看它和第一个盒子能不能凑成一对……直到把第一个盒子和所有其他盒子都试一遍。如果还没找到，我就换第二个盒子，再和它后面的所有盒子都试一遍……”
哇，这听起来有点累，但是不是一定能找到答案呢？没错！只要院子里真的有这对戒指，你这样找就一定能找到！

**场景二：寻找三件宝物**
现在游戏升级了！你需要找到**三件不同**的宝物——“友谊之戒”、“智慧之冠”和“勇气之剑”——它们分别藏在三个不同的盒子里。你又该怎么找呢？

是不是感觉盒子越来越多，要试的组合也越来越多，但是原理是一样的——把所有可能的三个盒子的组合都试一遍！

这种“把所有可能的情况都试一遍”的方法，在编程里就叫做“暴力枚举”！今天我们就要学习怎么用电脑来帮我们做这种“笨办法”里的“聪明事”——特别是当我们需要选择两个或三个东西的时候，也就是“双重枚举”和“三重枚举”！

## 深度知识点讲解

### 1. 暴力枚举的本质：地毯式搜索

**什么是暴力枚举？**
暴力枚举（Brute Force Enumeration），顾名思义，就是像“暴力”一样，不讲技巧，把所有可能的答案都尝试一遍，直到找到符合条件的那个。它就像我们玩游戏时，如果不知道密码，就从0000、0001、0002……一直试到9999，总会试出来一样。

**为什么暴力枚举很重要？**
1.  **简单直接：** 它是最容易理解和实现的方法。
2.  **保证正确：** 只要问题有解，暴力枚举一定能找到。
3.  **学习基础：** 很多高级算法都是在暴力枚举的基础上进行优化的。

**生活类比：**
*   **找钥匙：** 你有一串钥匙，不知道哪把能开门。最笨但最有效的方法就是一把一把地试，直到门开为止。
*   **点菜：** 餐厅的菜单上有很多菜，你想点一个你最喜欢，并且价格不超过预算的菜。你可以把菜单上的菜一个一个看过去，直到找到符合条件的。

### 2. 单重枚举（复习）

在学习双重、三重枚举之前，我们先快速回顾一下单重枚举。
如果你要从一堆数字中找到一个特定的数字，你会怎么做？
```cpp
#include <iostream>

int main() {
    int a[5] = {10, 25, 7, 30, 15}; // 假设有一组数字
    int target = 7; // 我们要找的目标数字
    bool found = false; // 用来标记是否找到

    // 从第一个数字开始，一个一个地检查
    for (int i = 0; i < 5; ++i) { // i 从 0 到 4
        if (a[i] == target) { // 如果当前数字是我们要找的
            found = true;    // 就标记为找到了
            break;           // 找到了就可以停止了
        }
    }

    if (found) {
        std::cout << "找到了目标数字！" << std::endl;
    } else {
        std::cout << "没有找到目标数字。" << std::endl;
    }

    return 0;
}
```
这里我们只用了一个 `for` 循环，因为它只需要检查“一个”数字。

### 3. 双重枚举（嵌套两层 `for` 循环）

**概念：** 当我们需要从一组数据中选择“两个”元素，或者需要同时考虑“两种”情况的组合时，通常会用到双重枚举。它由两个 `for` 循环嵌套而成，外层循环控制第一个选择，内层循环控制第二个选择。

**结构：**
```cpp
for (第一个选择的所有可能性) {
    for (第二个选择的所有可能性) {
        // 在这里检查这对组合是否符合条件
    }
}
```

**生活类比：**
*   **搭配衣服：** 你有3件上衣（红、黄、蓝）和2条裤子（黑、白）。你想知道所有可能的搭配组合。
    *   外层循环：选择上衣 (i)
    *   内层循环：选择裤子 (j)
    *   组合：(红,黑), (红,白), (黄,黑), (黄,白), (蓝,黑), (蓝,白)
*   **找情侣：** 一个班级里有N个男生和N个女生，你要为他们牵线搭桥，找出所有可能的“一对一”情侣组合。
    *   外层循环：选择一个男生 (i)
    *   内层循环：选择一个女生 (j)

**关键点剖析：**

1.  **循环范围：** 确定 `i` 和 `j` 的起始值和结束值非常重要。
    *   **独立选择：** 如果两个选择是独立的，比如从两个不同的数组中各选一个，那么 `i` 和 `j` 的循环范围通常是独立的。
        ```cpp
        // 假设a数组有n个元素，b数组有m个元素
        for (int i = 0; i < n; ++i) { // 遍历a数组的每个元素
            for (int j = 0; j < m; ++j) { // 遍历b数组的每个元素
                // 组合是 a[i] 和 b[j]
            }
        }
        ```
    *   **从同一组中选择两个不同的元素：** 这是最常见的情况。例如，从一个数组 `a` 中选择两个不同的元素 `a[i]` 和 `a[j]`。
        *   **方法一：`j` 从 `0` 到 `n-1`，并跳过 `i`：**
            ```cpp
            for (int i = 0; i < n; ++i) {
                for (int j = 0; j < n; ++j) {
                    if (i == j) { // 确保选的是两个不同的元素
                        continue; // 跳过自己和自己组合的情况
                    }
                    // 组合是 a[i] 和 a[j]
                }
            }
            ```
            这种方法会产生重复的组合，比如 `(a[0], a[1])` 和 `(a[1], a[0])`。
        *   **方法二（更常用，更高效）：`j` 从 `i+1` 开始：**
            ```cpp
            for (int i = 0; i < n; ++i) {
                for (int j = i + 1; j < n; ++j) { // j 从 i 后面一个位置开始
                    // 这样就能保证 i != j，并且每个组合 (a[i], a[j]) 只会出现一次
                    // 例如，当 i=0, j=1 时，会检查 (a[0], a[1])
                    // 后面就不会再出现 i=1, j=0 的情况了
                }
            }
            ```
            这种方法既保证了 `i != j`，又避免了重复的组合 `(a[i], a[j])` 和 `(a[j], a[i])`。

2.  **条件判断：** 在内层循环中，你需要编写 `if` 语句来检查当前这对组合是否符合题目要求。

**常见误区：**
*   **循环范围错误：** 忘记 `j` 从 `i+1` 开始，导致选到同一个元素或者重复计算。
*   **条件判断逻辑错误：** `if` 语句写错了，导致判断结果不正确。
*   **遗漏 `break`：** 如果题目只要求找到一个符合条件的组合就停止，记得使用 `break` 提前退出循环，提高效率。

### 4. 三重枚举（嵌套三层 `for` 循环）

**概念：** 当我们需要从一组数据中选择“三个”元素，或者需要同时考虑“三种”情况的组合时，通常会用到三重枚举。它由三个 `for` 循环嵌套而成，最外层控制第一个选择，中间层控制第二个选择，最内层控制第三个选择。

**结构：**
```cpp
for (第一个选择的所有可能性) {
    for (第二个选择的所有可能性) {
        for (第三个选择的所有可能性) {
            // 在这里检查这三个组合是否符合条件
        }
    }
}
```

**生活类比：**
*   **组合套餐：** 餐厅提供主食、配菜、饮料各多种选择。你想知道所有可能的“主食+配菜+饮料”套餐组合。
    *   最外层循环：选择主食 (i)
    *   中间层循环：选择配菜 (j)
    *   最内层循环：选择饮料 (k)
*   **找三件宝物：** 就像我们引入时提到的，寻找三件不同的宝物。

**关键点剖析：**

1.  **循环范围：** 与双重枚举类似，需要仔细确定 `i`, `j`, `k` 的循环范围。
    *   **独立选择：** 如果三个选择是独立的，比如从三个不同的数组中各选一个，那么 `i`, `j`, `k` 的循环范围通常是独立的。
    *   **从同一组中选择三个不同的元素：** 这是常见情况。例如，从一个数组 `a` 中选择三个不同的元素 `a[i]`, `a[j]`, `a[k]`。
        *   通常使用 `i` 从 `0` 到 `n-1`，`j` 从 `i+1` 到 `n-1`，`k` 从 `j+1` 到 `n-1` 的结构。
        ```cpp
        for (int i = 0; i < n; ++i) {
            for (int j = i + 1; j < n; ++j) {
                for (int k = j + 1; k < n; ++k) {
                    // 组合是 a[i], a[j], a[k]
                    // 这样保证 i < j < k，即三个元素不同，且不重复计算组合
                }
            }
        }
        ```

2.  **条件判断：** 在最内层循环中，编写 `if` 语句检查当前这三个组合是否符合题目要求。

**常见误区：**
*   **循环嵌套层数不对：** 少一层或多一层循环。
*   **循环变量混淆：** 比如三个循环都用 `i` 作为循环变量，导致错误。
*   **效率问题：** 三层循环的计算量会比两层循环大很多，对于 `N` 很大的题目可能不适用。

### 5. 算法效率（简单理解）

虽然暴力枚举简单直接，但它有一个缺点：当数据量很大时，它可能会非常慢！
*   **单重循环：** 如果有 `N` 个元素，大约需要检查 `N` 次。
*   **双重循环：** 如果有 `N` 个元素，需要选择两个，大约需要检查 `N * N` 次（写成 `N^2`）。
*   **三重循环：** 如果有 `N` 个元素，需要选择三个，大约需要检查 `N * N * N` 次（写成 `N^3`）。

想想看，如果 `N` 是 100：
*   单重循环：100 次
*   双重循环：100 * 100 = 10000 次
*   三重循环：100 * 100 * 100 = 1000000 次

所以，暴力枚举很适合解决数据量不大的问题（比如 `N` 在 100 左右，三重循环还能接受；`N` 在 1000 左右，双重循环还能接受）。当 `N` 变得很大时，我们就要考虑更聪明的算法了。不过，对于 GESP 二级来说，暴力枚举已经是非常实用的利器了！

### 例题一：寻找和为目标值的两个数 (双重枚举)

**题目描述：**
给定 `n` 个整数（存储在一个数组 `a` 中）和一个目标值 `target`。请你找出数组中**两个不同**的数，它们的和等于 `target`。如果找到了，输出 `YES`；否则，输出 `NO`。

**输入格式：**
第一行一个整数 `n` (2 <= n <= 100)。
第二行 `n` 个整数，表示数组 `a` 中的元素 (0 <= a[i] <= 1000)。
第三行一个整数 `target` (0 <= target <= 2000)。

**输出格式：**
如果找到，输出 `YES`；否则，输出 `NO`。

**思路分析：**
1.  **目标：** 找到两个不同的数，它们的和是 `target`。
2.  **暴力枚举思想：** 既然要找两个数，那我们就把所有可能的“两个数”的组合都试一遍！
3.  **如何选择两个数？**
    *   我们可以用一个外层 `for` 循环来选择第一个数 `a[i]`。
    *   再用一个内层 `for` 循环来选择第二个数 `a[j]`。
4.  **“两个不同”的限制：** 这很重要！我们不能选择 `a[i]` 和 `a[i]` 自己相加。
    *   为了避免 `i == j` 的情况，并且避免 `(a[0], a[1])` 和 `(a[1], a[0])` 这种重复的组合，我们可以让内层循环的 `j` 从 `i + 1` 开始。这样，`j` 总是比 `i` 大，就保证了选出的两个数是不同的，而且每对组合只会被检查一次。
5.  **检查条件：** 对于每一对 `a[i]` 和 `a[j]`，我们检查 `a[i] + a[j]` 是否等于 `target`。
6.  **输出结果：**
    *   如果找到了符合条件的组合，就立刻输出 `YES`，然后可以停止程序了（因为题目只要求找到一个即可）。
    *   如果所有的组合都试完了，还没有找到，就输出 `NO`。

**代码实现：**

```cpp
#include <iostream> // 输入输出库
// #include <bits/stdc++.h> // GESP 比赛常用，包含了很多常用库

int main() {
    // 加快输入输出速度，C++比赛常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 整数的数量
    std::cin >> n;

    int a[100]; // 定义一个数组来存储 n 个整数，题目说 n <= 100
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i]; // 读取每个整数
    }

    int target; // 目标值
    std::cin >> target;

    bool found = false; // 用一个布尔变量来标记是否找到了

    // 外层循环：选择第一个数 a[i]
    for (int i = 0; i < n; ++i) {
        // 内层循环：选择第二个数 a[j]
        // j 从 i + 1 开始，确保 a[j] 是 a[i] 后面的一个数，这样 a[i] 和 a[j] 肯定是不同的
        // 并且避免了重复检查 (a[i], a[j]) 和 (a[j], a[i])
        for (int j = i + 1; j < n; ++j) {
            // 检查这两个数的和是否等于目标值
            if (a[i] + a[j] == target) {
                found = true; // 找到了！
                break;        // 找到一个就够了，内层循环可以停止
            }
        }
        if (found) { // 如果内层循环找到了，外层循环也可以停止了
            break;
        }
    }

    if (found) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0; // 程序结束
}
```

**复杂度分析：**
*   外层循环 `i` 运行了 `n` 次。
*   内层循环 `j` 在最坏情况下（当 `i=0` 时）运行了 `n-1` 次，当 `i` 增大时，运行次数逐渐减少。
*   总的循环次数大约是 `n * (n-1) / 2` 次。
*   我们通常用 `O(N^2)` 来表示它的时间复杂度，意思是当 `N` 变大时，运行时间大约与 `N` 的平方成正比。对于 `N=100`，`N^2 = 10000`，这个计算量非常小，电脑可以在瞬间完成。

### 例题二：寻找和为目标值的三个数 (三重枚举)

**题目描述：**
给定 `n` 个整数（存储在一个数组 `a` 中）和一个目标值 `target`。请你找出数组中**三个**数，它们的和等于 `target`。这三个数可以是数组中的相同元素（即 `a[i]`, `a[j]`, `a[k]` 可以是同一个值，但它们是数组中的不同位置的元素）。如果找到了，输出 `YES`；否则，输出 `NO`。

**输入格式：**
第一行一个整数 `n` (3 <= n <= 50)。
第二行 `n` 个整数，表示数组 `a` 中的元素 (0 <= a[i] <= 1000)。
第三行一个整数 `target` (0 <= target <= 3000)。

**输出格式：**
如果找到，输出 `YES`；否则，输出 `NO`。

**思路分析：**
1.  **目标：** 找到三个数，它们的和是 `target`。
2.  **暴力枚举思想：** 既然要找三个数，那就把所有可能的“三个数”的组合都试一遍！
3.  **如何选择三个数？**
    *   用一个最外层 `for` 循环选择第一个数 `a[i]`。
    *   用一个中间层 `for` 循环选择第二个数 `a[j]`。
    *   再用一个最内层 `for` 循环选择第三个数 `a[k]`。
4.  **“三个不同”的限制（或不限制）：** 题目说“可以是数组中的相同元素”，这指的是值可以相同，但位置必须是不同的。
    *   为了确保选出的三个数是不同的位置，并且避免重复的组合，我们可以让 `j` 从 `i + 1` 开始，`k` 从 `j + 1` 开始。这样，`i < j < k`，就保证了 `a[i]`, `a[j]`, `a[k]` 是数组中三个不同位置的元素。
5.  **检查条件：** 对于每一组 `a[i]`, `a[j]`, `a[k]`，我们检查 `a[i] + a[j] + a[k]` 是否等于 `target`。
6.  **输出结果：**
    *   如果找到了符合条件的组合，就立刻输出 `YES`，然后停止程序。
    *   如果所有的组合都试完了，还没有找到，就输出 `NO`。

**代码实现：**

```cpp
#include <iostream> // 输入输出库
// #include <bits/stdc++.h> // GESP 比赛常用，包含了很多常用库

int main() {
    // 加快输入输出速度，C++比赛常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 整数的数量
    std::cin >> n;

    int a[50]; // 定义一个数组来存储 n 个整数，题目说 n <= 50
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i]; // 读取每个整数
    }

    int target; // 目标值
    std::cin >> target;

    bool found = false; // 用一个布尔变量来标记是否找到了

    // 最外层循环：选择第一个数 a[i]
    for (int i = 0; i < n; ++i) {
        // 中间层循环：选择第二个数 a[j]
        // j 从 i + 1 开始，确保 a[j] 是 a[i] 后面的一个数
        for (int j = i + 1; j < n; ++j) {
            // 最内层循环：选择第三个数 a[k]
            // k 从 j + 1 开始，确保 a[k] 是 a[j] 后面的一个数
            // 这样就保证了 i < j < k，选出的三个数是数组中不同位置的，且组合不重复
            for (int k = j + 1; k < n; ++k) {
                // 检查这三个数的和是否等于目标值
                if (a[i] + a[j] + a[k] == target) {
                    found = true; // 找到了！
                    break;        // 找到一个就够了，最内层循环可以停止
                }
            }
            if (found) { // 如果内层循环找到了，中间层循环也可以停止了
                break;
            }
        }
        if (found) { // 如果中间层循环找到了，最外层循环也可以停止了
            break;
        }
    }

    if (found) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0; // 程序结束
}
```

**复杂度分析：**
*   最外层循环 `i` 运行了 `n` 次。
*   中间层循环 `j` 大约运行了 `n` 次。
*   最内层循环 `k` 大约运行了 `n` 次。
*   总的循环次数大约是 `n * (n-1) * (n-2) / 6` 次。
*   我们通常用 `O(N^3)` 来表示它的时间复杂度。对于 `N=50`，`N^3 = 50 * 50 * 50 = 125000`，这个计算量虽然比双重循环大，但电脑仍然可以在瞬间完成。

    #include <limits> // 用于 numeric_limits

        int a[100];
        for (int i = 0; i < n; ++i) {

        }

        int minDiff = std::numeric_limits<int>::max(); // 初始化为最大整数值

        for (int i = 0; i < n; ++i) {
            for (int j = i + 1; j < n; ++j) {

                if (currentDiff < minDiff) {
                    minDiff = currentDiff; // 更新最小差值
                }
            }
        }
        std::cout << minDiff << std::endl;
        return 0;
    }
