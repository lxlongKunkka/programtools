## 趣味引入

同学们，你们听过兔子繁殖的故事吗？据说有一对刚出生的兔子，它们在第一个月还不能生小兔子，但从第二个月开始，每对兔子每个月都能生一对小兔子。假设兔子都不会死亡，那么每个月会有多少对兔子呢？

*   **第1个月**：1 对兔子（刚出生）
*   **第2个月**：1 对兔子（这对兔子成年了，但还没生）
*   **第3个月**：1 对成年兔子生了 1 对小兔子，所以是 1 + 1 = 2 对
*   **第4个月**：第2个月的 1 对成年兔子生了 1 对小兔子，第3个月的 1 对成年兔子也生了 1 对小兔子，所以是 2 + 1 = 3 对
*   **第5个月**：...

大家有没有发现什么规律？第 `n` 个月的兔子数量，好像和前两个月的兔子数量有关！

如果我们用 `F(n)` 来表示第 `n` 个月的兔子对数，那么：
`F(1) = 1`
`F(2) = 1`
`F(n) = F(n-1) + F(n-2)` (当 `n > 2` 时)

这个数列就是著名的**斐波那契数列**！

如果我们用递归的方法来计算 `F(n)`，比如计算 `F(5)`：
`F(5) = F(4) + F(3)`
`F(4) = F(3) + F(2)`
`F(3) = F(2) + F(1)`
你会发现 `F(3)` 被计算了两次，`F(2)` 被计算了三次！如果 `n` 很大，会有大量的重复计算，效率非常低。

**问题来了：** 我们能不能有一种更聪明的方法，避免这些重复计算，从而更快地得到答案呢？
答案就是——**动态规划 (Dynamic Programming, DP)**！它就像一个聪明的记账员，把已经算过的结果都记下来，下次需要的时候直接查阅，再也不用重复劳动啦！

## 知识点讲解

### 什么是动态规划 (DP)？

动态规划是一种解决复杂问题的算法思想。它的核心思想是：**将一个大问题分解成若干个相互重叠的子问题，通过解决这些子问题，并把子问题的解存储起来，避免重复计算，最终组合成原问题的解。**

动态规划通常适用于具有以下两个特征的问题：
1.  **最优子结构 (Optimal Substructure)**：一个问题的最优解包含其子问题的最优解。
2.  **重叠子问题 (Overlapping Subproblems)**：在解决问题的过程中，会多次遇到相同的子问题。

### 动态规划三要素

解决一个动态规划问题，通常需要明确以下三个要素：

1.  **状态定义 (State Definition)**：
    `dp[i]` 代表什么？这是最关键也是最难的一步。它通常表示原问题在规模为 `i` 时的解，或者某种特定情况下的最优值、方案数等。
    *   例如在斐波那契数列中，`dp[i]` 可以定义为第 `i` 个斐波那契数。

2.  **状态转移方程 (State Transition Equation)**：
    `dp[i]` 如何从之前已经计算好的 `dp[j]` (其中 `j < i`) 推导出来？这就像一个递推公式，它描述了子问题之间的关系。
    *   例如在斐波那契数列中，`dp[i] = dp[i-1] + dp[i-2]`。

3.  **初始化 (Initialization)**：
    `dp` 数组的初始值是什么？这通常是最小规模的子问题的解，或者说是递推的边界条件，是后续所有计算的基础。
    *   例如在斐波那契数列中，`dp[0]=0` (或者 `dp[1]=1`) 和 `dp[1]=1` (或者 `dp[2]=1`) 是初始值。

### 动态规划的实现方式：递推（自底向上）

动态规划通常采用**自底向上（Bottom-up）**的递推方式实现。也就是说，我们从最小的子问题开始解决，逐步推导出更大规模问题的解，直到最终解决原问题。

**步骤：**
1.  创建一个 `dp` 数组（或 `a` 数组），大小足以存储所有子问题的解。
2.  根据初始化条件，填充 `dp` 数组的初始值。
3.  通过循环，按照状态转移方程的顺序，依次计算并填充 `dp` 数组的其余部分。

### 案例一：斐波那契数列 (Fibonacci Sequence)

让我们用DP的思想来解决斐波那契数列问题。

**问题：** 计算第 `n` 个斐波那契数 `F(n)`。

1.  **状态定义：**
    `dp[i]` 表示第 `i` 个斐波那契数。

2.  **状态转移方程：**
    `dp[i] = dp[i-1] + dp[i-2]` (当 `i >= 2` 时)

3.  **初始化：**
    `dp[0] = 0` (或者 `dp[1] = 1`，取决于你的数列从 `F(0)` 还是 `F(1)` 开始)
    `dp[1] = 1` (或者 `dp[2] = 1`)
    为了方便，我们假设 `F(0)=0, F(1)=1`。
    那么 `dp[0] = 0`
    `dp[1] = 1`

现在，我们可以按照这个思路来计算 `F(n)`。

### 案例二：爬楼梯问题 (Climbing Stairs)

假设你正在爬楼梯。总共有 `n` 级台阶。每次你可以爬 `1` 级或者 `2` 级。那么，你有多少种不同的方法可以爬到楼顶呢？

**问题：** 爬 `n` 级楼梯，每次可爬 1 级或 2 级，求到达楼顶的不同方法总数。

1.  **状态定义：**
    `dp[i]` 表示爬到第 `i` 级台阶的不同方法总数。

2.  **状态转移方程：**
    要爬到第 `i` 级台阶，你可能从第 `i-1` 级台阶一步爬上来（1 级），也可能从第 `i-2` 级台阶两步爬上来（2 级）。
    所以，爬到第 `i` 级台阶的方法总数，等于爬到第 `i-1` 级的方法总数加上爬到第 `i-2` 级的方法总数。
    `dp[i] = dp[i-1] + dp[i-2]` (当 `i >= 2` 时)
    这和斐波那契数列的转移方程一模一样！

3.  **初始化：**
    *   `dp[0]`：爬到第 0 级台阶（也就是还没开始爬），我们认为这有 1 种方法（不爬）。
        `dp[0] = 1`
    *   `dp[1]`：爬到第 1 级台阶，只有 1 种方法（从第 0 级爬 1 步）。
        `dp[1] = 1`
    *   注意：这里 `dp[0]=1` 是为了让 `dp[2]` 的计算 `dp[2] = dp[1] + dp[0] = 1 + 1 = 2` 成立 (爬到第2级有两种方法: 1+1 或 2)。

通过这两个例子，大家应该对DP的核心思想和三要素有了初步的了解。记住，关键在于找到问题的子结构，并把它们用一个数组（DP表）记录下来！

## 代码示例

### 案例一：斐波那契数列的DP实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    // 定义要计算的斐波那契数的项数 n
    // 例如，计算第 10 个斐波那契数
    int n;
    cin >> n;

    // 创建一个数组 dp 来存储斐波那契数
    // dp[i] 表示第 i 个斐波那契数
    // 数组大小为 n+1，因为我们可能需要计算到 dp[n]
    int a[105]; // 假设 n 不会超过 100，所以开大一点的数组

    // 初始化：设置最小子问题的解
    // 根据定义 F(0)=0, F(1)=1
    a[0] = 0;
    a[1] = 1;

    // 状态转移：从 i=2 开始，计算后续的斐波那契数
    for (int i = 2; i <= n; i++)
    {
        a[i] = a[i - 1] + a[i - 2]; // 状态转移方程
    }

    // 输出结果，即第 n 个斐波那契数
    cout << a[n] << endl;

    return 0;
}
```

### 案例二：爬楼梯问题的DP实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    // 定义楼梯的级数 n
    // 例如，爬 5 级楼梯
    int n;
    cin >> n;

    // 创建一个数组 dp 来存储爬到第 i 级台阶的方法总数
    // dp[i] 表示爬到第 i 级台阶的方法数
    // 数组大小为 n+1，因为我们可能需要计算到 dp[n]
    int a[105]; // 假设 n 不会超过 100

    // 初始化：设置最小子问题的解
    // 爬到第 0 级台阶（不爬），有 1 种方法
    a[0] = 1;
    // 爬到第 1 级台阶，有 1 种方法（1步）
    a[1] = 1;

    // 状态转移：从 i=2 开始，计算爬到后续台阶的方法总数
    for (int i = 2; i <= n; i++)
    {
        // 爬到第 i 级，可以从 i-1 级走 1 步，也可以从 i-2 级走 2 步
        a[i] = a[i - 1] + a[i - 2]; // 状态转移方程
    }

    // 输出结果，即爬到第 n 级台阶的方法总数
    cout << a[n] << endl;

    return 0;
}
```

### 空间优化（了解）

同学们有没有注意到，计算 `dp[i]` 时，我们只需要 `dp[i-1]` 和 `dp[i-2]` 的值。这意味着我们并不需要整个 `dp` 数组！我们只需要用几个变量来保存前两个值就可以啦。

例如斐波那契数列的优化：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    int n;
    cin >> n;

    if (n == 0)
    {
        cout << 0 << endl;
        return 0;
    }
    if (n == 1)
    {
        cout << 1 << endl;
        return 0;
    }

    int p = 0; // 对应 dp[i-2]
    int q = 1; // 对应 dp[i-1]
    int r = 0; // 对应 dp[i]

    for (int i = 2; i <= n; i++)
    {
        r = p + q; // 计算当前值
        p = q;     // 更新 p 为上一个 q
        q = r;     // 更新 q 为当前计算出的 r
    }

    cout << r << endl;

    return 0;
}
```
这种优化可以将空间复杂度从 O(N) 降低到 O(1)，在某些对内存要求严格的题目中非常有用。

        *   **结论：** 题目中的状态转移方程 `dp[i] = dp[i-1] + dp[i-3]` 和给出的初始值 `dp[0]=1, dp[1]=1, dp[2]=2` 算出来的 `dp[4]` 是 4。
        *   为了避免混淆，我将修改问题，使其与斐波那契或爬楼梯的常见模式一致，这样答案是5。
        *   **修改后的第5题：**
            一个动态规划问题，其状态转移方程为 `dp[i] = dp[i-1] + dp[i-2]`。如果已知 `dp[0]=1, dp[1]=1`，那么 `dp[4]` 的值是 ______。
            *   **解析（修改后）：**
                *   已知 `dp[0]=1, dp[1]=1`
                *   `dp[2] = dp[1] + dp[0] = 1 + 1 = 2`
                *   `dp[3] = dp[2] + dp[1] = 2 + 1 = 3`
                *   `dp[4] = dp[3] + dp[2] = 3 + 2 = 5`
                *   所以 `dp[4]` 的值是 **5**。

    *   **最终确定第5题及其解析如下：**
        5.  一个动态规划问题，其状态转移方程为 `dp[i] = dp[i-1] + dp[i-2]`。如果已知 `dp[0]=1, dp[1]=1`，那么 `dp[4]` 的值是 ______。
        *   **解析：**
            *   已知 `dp[0]=1, dp[1]=1`
            *   根据状态转移方程 `dp[i] = dp[i-1] + dp[i-2]`：
            *   `dp[2] = dp[1] + dp[0] = 1 + 1 = 2`
            *   `dp[3] = dp[2] + dp[1] = 2 + 1 = 3`
            *   `dp[4] = dp[3] + dp[2] = 3 + 2 = 5`
            *   所以 `dp[4]` 的值是 **5**。

### 课后作业

1.  **基础巩固：青蛙跳台阶**
    *   **问题：** 有 `n` 级台阶，一只青蛙每次可以跳上 1 级台阶，也可以跳上 2 级台阶。请你编写一个C++程序，计算青蛙跳到第 `n` 级台阶总共有多少种不同的方法。
    *   **要求：** 使用动态规划的思想解决，并用 `int a[]` 数组实现。
    *   **输入：** 一个整数 `n` (1 <= n <= 50)
    *   **输出：** 一个整数，表示方法总数。

2.  **进阶思考：斐波那契数列的优化**
    *   请你尝试修改斐波那契数列的代码示例，实现空间优化（即只用少量变量存储前几个值，而不是整个 `dp` 数组）。
    *   思考：这种空间优化在什么情况下特别有用？

3.  **挑战拓展：带障碍的爬楼梯 (选做)**
    *   **问题：** 假设你正在爬楼梯。总共有 `n` 级台阶。每次你可以爬 `1` 级或者 `2` 级。但是，有些台阶是“坏的”（不能踩）。给定 `n` 和一个布尔数组 `is_broken[]`（`is_broken[i]` 为 `true` 表示第 `i` 级台阶是坏的），请你计算有多少种不同的方法可以爬到楼顶。
    *   **提示：** 思考 `dp[i]` 的状态定义如何变化？遇到坏的台阶时，状态转移方程应该如何处理？
