## 知识点讲解

### 1. 什么是深度优先搜索 (DFS)？

DFS 是一种用于遍历或搜索树或图的算法。它的核心思想是：**从起点出发，尽可能深地探索每个分支，直到不能再深入为止，然后回溯到上一个节点，探索其他分支。**

用我们迷宫的例子来说，就是：
1.  选择一个方向走。
2.  如果能继续走，就一直往前走。
3.  如果遇到死胡同或者已经走过的路，就“回头”（回溯）。
4.  回到上一个路口，选择一个还没走过的方向继续走。
5.  重复以上步骤，直到找到目标或者所有路径都探索完毕。

### 2. 图的表示方法

在计算机中，我们要处理迷宫或网络时，通常会把它们抽象成“图”。图由节点（或顶点）和边组成。
*   **节点 (Node/Vertex)**：迷宫中的一个格子，或者网络中的一个城市。
*   **边 (Edge)**：连接两个节点的路径，表示可以从一个格子走到另一个格子，或者从一个城市到达另一个城市。

图有两种常见的表示方法：

#### (1) 邻接矩阵 (Adjacency Matrix)
一个二维数组 `a[N][N]`。如果 `a[i][j] = 1`，表示节点 `i` 和节点 `j` 之间有边；如果 `a[i][j] = 0`，表示没有边。
*   **优点**：判断两个节点之间是否有边非常快（O(1)）。
*   **缺点**：对于节点很多但边很少的图（稀疏图），会浪费大量存储空间。

#### (2) 邻接表 (Adjacency List)
使用一个 `vector` 数组 `adj[N]`，其中 `adj[i]` 存储一个列表，包含所有与节点 `i` 相连的节点。
*   **优点**：对于稀疏图，节省存储空间。遍历一个节点的所有邻居很方便。
*   **缺点**：判断两个节点是否有边需要遍历 `adj[i]`，时间复杂度较高。

在 DFS 和 BFS 中，**邻接表**通常是更常用且高效的表示方法，尤其是在处理大型稀疏图时。

### 3. DFS 的实现：递归是核心！

DFS 通常使用**递归**来实现。递归函数 `dfs(u)` 表示从节点 `u` 开始进行深度优先搜索。

**基本步骤：**
1.  **标记当前节点 `u` 为已访问**：避免重复访问和死循环。
2.  **处理当前节点 `u`**：根据问题需求，可以在这里进行一些操作（比如打印节点、计数等）。
3.  **遍历所有与 `u` 相邻的节点 `v`**：
    *   如果 `v` **未被访问**：
        *   递归调用 `dfs(v)`，继续深入探索。

**关键点：**
*   **`visited` 数组**：一个布尔数组 `bool v[N];` 用来记录每个节点是否已经被访问过。
*   **递归回溯**：当 `dfs(v)` 返回时，意味着以 `v` 为起点的所有路径都探索完毕了，或者找到了目标。此时程序会回到 `dfs(u)` 中，继续探索 `u` 的下一个未访问邻居。

### 4. 迷宫问题中的 DFS

迷宫问题是 DFS 的经典应用。我们可以把迷宫看作一个二维网格，每个格子是一个节点。相邻的格子之间如果有路，就相当于有边。

**迷宫 DFS 的核心思路：**
1.  **定义方向数组**：通常用 `dx[]` 和 `dy[]` 来表示上下左右四个方向的坐标变化。
    *   `dx = {-1, 1, 0, 0}` (上, 下, 不变, 不变)
    *   `dy = {0, 0, -1, 1}` (不变, 不变, 左, 右)
2.  **`dfs(x, y)` 函数**：表示从 `(x, y)` 这个格子开始搜索。
3.  **边界检查**：确保 `(x, y)` 在迷宫范围内。
4.  **障碍物检查**：确保 `(x, y)` 不是墙。
5.  **访问标记**：用一个二维布尔数组 `bool v[R][C];` 标记 `(x, y)` 是否已访问。
6.  **目标检查**：如果 `(x, y)` 是出口，就成功了！
7.  **遍历邻居**：尝试向 `(x, y)` 的上下左右四个方向移动，对于每个可以移动的新位置 `(nx, ny)`：
    *   如果 `(nx, ny)` 有效且未访问：
        *   递归调用 `dfs(nx, ny)`。

## 代码示例

### 示例一：图的深度优先遍历 (使用邻接表)

这个例子展示了如何遍历一个无向图的所有节点。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 1005; // 最大节点数
vector<int> g[N];   // 邻接表，g[i] 存储与节点 i 相连的节点
bool v[N];          // 访问标记数组，v[i] 为 true 表示节点 i 已被访问
int n, m;           // n 为节点数，m 为边数

// DFS 函数，从节点 u 开始进行深度优先搜索
void dfs(int u)
{
    v[u] = true; // 标记当前节点 u 已被访问
    cout << u << " "; // 访问节点 u (这里是打印节点编号)

    // 遍历节点 u 的所有邻居节点
    for (int i = 0; i < g[u].size(); ++i)
    {
        int x = g[u][i]; // x 是 u 的一个邻居节点
        if (!v[x])       // 如果邻居节点 x 没有被访问过
        {
            dfs(x);      // 递归调用 dfs，继续深入探索 x
        }
    }
}

int main()
{
    // 输入节点数 n 和边数 m
    // 例如： 5 4
    // 1 2
    // 1 3
    // 2 4
    // 3 5
    cin >> n >> m;

    // 构建图的邻接表
    for (int i = 0; i < m; ++i)
    {
        int a, b;
        cin >> a >> b;
        g[a].push_back(b); // a 到 b 有边
        g[b].push_back(a); // b 到 a 也有边 (无向图)
    }

    cout << "DFS 遍历结果 (从节点 1 开始):" << endl;
    dfs(1); // 从节点 1 开始进行 DFS
    cout << endl;

    return 0;
}
```

**代码解释：**
*   `g[N]` 是一个 `vector<int>` 类型的数组，`g[i]` 存储了所有与节点 `i` 相连的节点。
*   `v[N]` 是一个布尔数组，用于记录节点是否已被访问过。初始化时所有元素都为 `false`。
*   `dfs(u)` 函数：
    *   首先将 `v[u]` 设为 `true`，表示 `u` 已经被访问。
    *   然后打印 `u`（这是我们对节点的“处理”）。
    *   接着遍历 `g[u]` 中的所有邻居 `x`。
    *   如果 `x` 还没有被访问过 (`!v[x]`)，就递归调用 `dfs(x)`，继续向下探索。
*   `main` 函数中，我们先读取图的结构，然后从节点 1 开始调用 `dfs(1)` 进行遍历。

### 示例二：迷宫问题 (判断从起点到终点是否有路径)

假设有一个 `R x C` 的迷宫，`0` 表示通路，`1` 表示墙壁。判断从 `(sx, sy)` 到 `(ex, ey)` 是否有通路。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 105; // 迷宫的最大行数和列数
char maze[MAXN][MAXN]; // 存储迷宫地图
bool v[MAXN][MAXN];    // 访问标记数组
int r, c;              // 迷宫的行数和列数
int sx, sy, ex, ey;    // 起点 (sx, sy) 和终点 (ex, ey)

// 定义四个方向：上、下、左、右
int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

bool found_path = false; // 标记是否找到路径

// DFS 函数，从 (x, y) 位置开始搜索
void dfs(int x, int y)
{
    // 如果已经找到路径，直接返回，避免不必要的搜索
    if (found_path)
    {
        return;
    }

    // 标记当前位置 (x, y) 已被访问
    v[x][y] = true;

    // 如果当前位置就是终点，说明找到路径了
    if (x == ex && y == ey)
    {
        found_path = true;
        return;
    }

    // 尝试向四个方向移动
    for (int i = 0; i < 4; ++i)
    {
        int nx = x + dx[i]; // 新的行坐标
        int ny = y + dy[i]; // 新的列坐标

        // 检查新位置是否合法：
        // 1. 在迷宫范围内
        // 2. 不是墙壁 ('1')
        // 3. 没有被访问过
        if (nx >= 0 && nx < r && ny >= 0 && ny < c &&
            maze[nx][ny] == '0' && !v[nx][ny])
        {
            dfs(nx, ny); // 递归调用 dfs，继续深入探索
        }
    }
}

int main()
{
    // 输入迷宫的行数和列数
    // 例如：
    // 5 5
    // 00100
    // 00000
    // 01010
    // 00000
    // 00010
    // 0 0 4 4 (起点 0,0 终点 4,4)
    cin >> r >> c;

    for (int i = 0; i < r; ++i)
    {
        for (int j = 0; j < c; ++j)
        {
            cin >> maze[i][j];
        }
    }

    cin >> sx >> sy >> ex >> ey; // 输入起点和终点坐标

    // 检查起点和终点是否是通路
    if (maze[sx][sy] == '1' || maze[ex][ey] == '1')
    {
        cout << "起点或终点是墙壁，无法通行！" << endl;
        return 0;
    }
    
    // 从起点开始 DFS
    dfs(sx, sy);

    if (found_path)
    {
        cout << "恭喜你，找到了从 (" << sx << "," << sy << ") 到 (" << ex << "," << ey << ") 的通路！" << endl;
    }
    else
    {
        cout << "很遗憾，没有找到从 (" << sx << "," << sy << ") 到 (" << ex << "," << ey << ") 的通路。" << endl;
    }

    return 0;
}
```

**代码解释：**
*   `maze[MAXN][MAXN]` 存储迷宫地图，`'0'` 是通路，`'1'` 是墙。
*   `v[MAXN][MAXN]` 是二维布尔数组，标记迷宫中的格子是否被访问过。
*   `dx[]` 和 `dy[]` 数组用来方便地计算四个方向的邻居坐标。
*   `found_path` 是一个全局变量，一旦找到路径就设为 `true`，可以提前终止搜索。
*   `dfs(x, y)` 函数：
    *   首先检查 `found_path`，如果已经找到，则立即返回。
    *   标记当前格子 `(x, y)` 已访问。
    *   如果 `(x, y)` 是终点，则设置 `found_path = true` 并返回。
    *   遍历四个方向，计算新的坐标 `(nx, ny)`。
    *   **合法性检查**：`nx, ny` 必须在迷宫范围内，不能是墙 (`'1'`)，并且不能是已访问过的格子。
    *   如果合法，则递归调用 `dfs(nx, ny)`。
*   `main` 函数读取迷宫和起点终点，然后调用 `dfs(sx, sy)`。

*   **第7步**：2 没有未访问的邻居了，**回溯到 1**。
*   **第8步**：1 还有邻居 3，但 3 已访问。1 没有未访问的邻居了，搜索结束。

最终遍历顺序可能是：1 -> 2 -> 4 -> 3。

1.  **答案：B**

3.  **答案：C**
    **解析**：
    图结构：
    1 -- 2 -- 4
    |    |
    3 -- 5

    DFS 是深度优先，它会尽可能深入。
    A. 1 -> 2 (深) -> 4 (深) -> 回溯到 2 -> 回溯到 1 -> 3 (深) -> 5 (深)。这个序列是可能的。
    B. 1 -> 3 (深) -> 5 (深) -> 回溯到 3 -> 回溯到 1 -> 2 (深) -> 4 (深)。这个序列是可能的。
    C. 1 -> 2 -> 3 -> 4 -> 5。这个序列是先访问 1 的邻居 2，然后访问 1 的另一个邻居 3。**这意味着在访问 3 之前，2 的所有子节点（4）都必须被访问完或者 2 没有其他未访问的邻居。** 但在这个序列中，在访问 2 之后，并没有深入到 4，而是回溯到 1 访问了 3。这更像是广度优先或某种特殊遍历，而不是纯粹的深度优先。DFS 会先 1->2->4，或者 1->3->5，不会在 1->2 之后，不访问 4 而直接跳到 3。
    D. 1 -> 3 -> 2 -> 4 -> 5。这个序列是可能的。

    因此，C 不符合 DFS 的深度优先特性。

4.  **答案：B**
    **解析**：在迷宫问题中，我们通常使用二维数组来直接表示迷宫的地图（通路和墙壁），并通过一个方向数组（`dx`, `dy`）来方便地计算上下左右四个方向的邻居坐标，然后递归调用 DFS。

5.  **答案：1, 2**
    **解析**：
    *   **每个节点被访问 1 次**：在 DFS 遍历中，我们使用 `visited` 数组来确保每个节点只在第一次被发现时真正“处理”和深入探索。一旦一个节点被标记为已访问，之后再遇到它就不会再次深入。
    *   **每条边被访问 2 次**：对于无向图中的一条边 (u, v)，当从 u 探索到 v 时，这条边被“遍历”一次；当从 v 回溯到 u（或者从 v 探索其他邻居时，检查到 u 已经访问过）时，这条边从 v 的角度又被“检查”了一次。所以实际上是，从 u 访问 v 算一次，从 v 访问 u 算一次，总共两次。
