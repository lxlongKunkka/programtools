## 趣味引入

同学们，你们有没有想过，我们平时发送的短信、聊天的文字、传输的文件，为什么能占用那么小的空间，传输速度还很快呢？这背后有一个非常重要的技术，叫做“数据压缩”！

想象一下，我们想给朋友发一条消息：“你好呀，你好呀，你好呀！”。如果我们把每个字都用相同的长度编码，比如“你”是00，“好”是01，“呀”是10。那么这条消息的编码就是：`000110000110000110`，总共18位。

但是，如果有些字出现的频率很高，有些字出现的频率很低，我们能不能给出现频率高的字一个短一点的编码，给出现频率低的字一个长一点的编码，这样总体的编码长度是不是就能更短呢？

比如，在我们的消息“你好呀，你好呀，你好呀！”中，“你”出现了3次，“好”出现了3次，“呀”出现了3次。假设我们现在有这些字符和它们的出现频率：

*   A: 8次
*   B: 3次
*   C: 1次
*   D: 1次

如果我们用普通的固定长度编码（比如A=00, B=01, C=10, D=11），总长度就是 (8+3+1+1) * 2 = 13 * 2 = 26 位。

但如果我们给 A 一个短编码，比如 `0`；给 B 一个稍微长一点的，比如 `10`；给 C 和 D 更长的，比如 `110` 和 `111`。那么总长度就变成了：
8 * 1 (A) + 3 * 2 (B) + 1 * 3 (C) + 1 * 3 (D) = 8 + 6 + 3 + 3 = 20 位。
是不是比固定长度编码短了很多？

这种根据字符出现频率，给高频字符短编码、低频字符长编码，并且保证编码唯一性的方法，就叫做**哈夫曼编码 (Huffman Coding)**。而实现这种编码的秘密武器，就是我们今天的主角——**哈夫曼树 (Huffman Tree)**！它是一种特殊的二叉树，能够帮助我们找到最优的编码方案，让总的编码长度最短。是不是很神奇呢？让我们一起揭开它的面纱吧！

## 知识点讲解

### 1. 树和带权路径长度 (WPL) 回顾

在学习哈夫曼树之前，我们先来回顾一下树的一些基本概念。
*   **节点 (Node)**：树的基本组成单元。
*   **根节点 (Root Node)**：树的最顶端节点，没有父节点。
*   **叶子节点 (Leaf Node)**：没有子节点的节点。
*   **路径 (Path)**：从一个节点到另一个节点所经过的边。
*   **路径长度 (Path Length)**：路径上边的数量。
*   **节点的权值 (Weight)**：通常表示该节点的重要性或出现的频率。
*   **带权路径长度 (Weighted Path Length, WPL)**：从根节点到某个叶子节点的路径长度与该叶子节点的权值的乘积。
*   **树的带权路径长度 (Total WPL)**：树中所有叶子节点的带权路径长度之和。

**举例说明 WPL：**
假设我们有一棵二叉树，叶子节点分别为 A(权值5)、B(权值2)、C(权值3)。
*   如果 A 在深度2 (路径长度2)，B 在深度2，C 在深度1。
*   那么 WPL = 5 * 2 + 2 * 2 + 3 * 1 = 10 + 4 + 3 = 17。

我们的目标就是找到一棵树，使得它的总 WPL 最小。

### 2. 哈夫曼树 (Huffman Tree) 的定义与特点

**定义：**
哈夫曼树，又称最优二叉树，是指给定 N 个带权值的叶子节点，构造一棵二叉树，使得所有叶子节点的带权路径长度之和最小。

**特点：**
1.  **没有度为1的节点**：哈夫曼树中，除了根节点，每个非叶子节点都有两个子节点（度为2），或者它是叶子节点（度为0）。
2.  **权值越大的节点离根越近**：频率高的字符编码短，频率低的字符编码长。
3.  **最优性**：在所有可能的二叉树中，哈夫曼树的带权路径长度是最小的。

### 3. 哈夫曼树的构建过程 (贪心策略)

哈夫曼树的构建过程是一个经典的贪心算法应用。贪心策略是指在每一步都选择当前看起来最优的方案，希望最终能得到全局最优解。

**构建步骤：**
假设我们有 n 个字符及其对应的频率（权值）。

1.  **初始化**：将每个字符看作一个独立的节点，其权值就是它的频率。将所有这些节点放入一个集合中。
2.  **选择最小**：从集合中选出两个当前权值最小的节点。
3.  **合并**：创建一个新的父节点，其权值为这两个最小节点的权值之和。将这两个最小节点作为新父节点的左、右子节点（左右顺序不影响 WPL）。
4.  **替换**：将这两个最小节点从集合中删除，并将新创建的父节点加入集合。
5.  **重复**：重复步骤 2-4，直到集合中只剩下一个节点。这个唯一的节点就是哈夫曼树的根节点。

**图解示例：**
假设我们有字符及其频率：A(5), B(2), C(3), D(1), E(1)。

*   **步骤1：** 将所有节点放入集合：{A(5), B(2), C(3), D(1), E(1)}

*   **步骤2：** 选出最小的两个节点 D(1) 和 E(1)。
    合并它们，生成新节点 F(1+1=2)。
    集合变为：{A(5), B(2), C(3), F(2)}
    ```
        F(2)
       /    \
      D(1)  E(1)
    ```

*   **步骤3：** 选出最小的两个节点 B(2) 和 F(2)。
    合并它们，生成新节点 G(2+2=4)。
    集合变为：{A(5), C(3), G(4)}
    ```
            G(4)
           /    \
          B(2)   F(2)
                 /    \
                D(1)  E(1)
    ```

*   **步骤4：** 选出最小的两个节点 C(3) 和 G(4)。
    合并它们，生成新节点 H(3+4=7)。
    集合变为：{A(5), H(7)}
    ```
                H(7)
               /    \
              C(3)   G(4)
                     /    \
                    B(2)   F(2)
                           /    \
                          D(1)  E(1)
    ```

*   **步骤5：** 选出最小的两个节点 A(5) 和 H(7)。
    合并它们，生成新节点 I(5+7=12)。
    集合变为：{I(12)}。只剩一个节点，构建完成！
    ```
                    I(12) (Root)
                   /     \
                  A(5)    H(7)
                         /    \
                        C(3)   G(4)
                               /    \
                              B(2)   F(2)
                                     /    \
                                    D(1)  E(1)
    ```

### 4. 哈夫曼编码 (Huffman Coding)

哈夫曼树构建完成后，就可以生成哈夫曼编码了。
1.  从根节点开始，向左子树的边标记为 `0`，向右子树的边标记为 `1`。
2.  从根节点到每个叶子节点的路径上的 `0` 和 `1` 序列，就是该叶子节点（字符）的哈夫曼编码。

**以上面构建的哈夫曼树为例：**

*   A: 路径 `根 -> 左`，编码 `0`
*   C: 路径 `根 -> 右 -> 左`，编码 `10`
*   B: 路径 `根 -> 右 -> 右 -> 左`，编码 `110`
*   D: 路径 `根 -> 右 -> 右 -> 右 -> 左`，编码 `1110`
*   E: 路径 `根 -> 右 -> 右 -> 右 -> 右`，编码 `1111` (Oops, my previous diagram for D, E was `D(1), E(1)` resulting `F(2)`. Then `B(2), F(2)` resulting `G(4)`. Then `C(3), G(4)` resulting `H(7)`. Then `A(5), H(7)` resulting `I(12)`. Let's re-trace the D and E path carefully using the final tree structure.)

    Let's re-trace based on the final tree:
    ```
                        I(12) (Root)
                       /     \
                      A(5)    H(7)
                     (0)     (1)
                             /    \
                            C(3)   G(4)
                           (0)    (1)
                                  /    \
                                 B(2)   F(2)
                                (0)    (1)
                                       /    \
                                      D(1)  E(1)
                                     (0)   (1)
    ```
    *   A: `0`
    *   C: `10`
    *   B: `110`
    *   D: `1110`
    *   E: `1111`

**特点：**
*   **前缀码 (Prefix Code)**：任何一个字符的哈夫曼编码都不是其他任何字符编码的前缀。这保证了解码的唯一性，我们不需要额外的分隔符就能正确解码。比如，如果你收到 `010`，你知道 `0` 是 A，剩下的 `10` 是 C，所以是 `AC`。如果 `0` 是 A，`01` 是 B，那么 `010` 就会有歧义。哈夫曼编码避免了这种情况。

**计算总 WPL：**
WPL = 5 * 1 (A) + 3 * 2 (C) + 2 * 3 (B) + 1 * 4 (D) + 1 * 4 (E)
WPL = 5 + 6 + 6 + 4 + 4 = 25。

这就是哈夫曼树和哈夫曼编码的核心思想！

## 代码示例

我们将使用 C++ 来实现哈夫曼树的构建和编码生成。为了方便地取出最小的两个节点，我们会用到优先队列 (`priority_queue`)，它能自动维护一个最小堆。

首先，我们需要定义树的节点结构体，以及一个比较器，让优先队列按照节点的权值从小到大排序。

```cpp
#include <bits/stdc++.h> // 包含所有常用头文件，方便竞赛使用

using namespace std;

// 哈夫曼树节点结构体
struct Node {
    int w; // 节点的权值（频率）
    char c; // 如果是叶子节点，存储对应的字符
    Node *l, *r; // 左子节点和右子节点

    // 构造函数：用于叶子节点
    Node(char ch, int weight) : c(ch), w(weight), l(nullptr), r(nullptr) {}

    // 构造函数：用于合并产生的非叶子节点
    Node(int weight, Node* left, Node* right) : c('\0'), w(weight), l(left), r(right) {}

    // 析构函数：释放动态分配的内存，防止内存泄漏
    ~Node() {
        delete l;
        delete r;
    }
};

// 优先队列的比较器：实现最小堆
// priority_queue 默认是大顶堆，通过重载 operator() 可以实现小顶堆
struct CompareNode {
    bool operator()(Node* a, Node* b) {
        return a->w > b->w; // 权值小的优先级高 (Min-heap)
    }
};

// 存储哈夫曼编码的映射
map<char, string> huffman_codes;

// 递归生成哈夫曼编码的函数
// 参数：当前节点，当前路径字符串
void generate_codes(Node* root, string s) {
    // 如果当前节点为空，直接返回
    if (root == nullptr) {
        return;
    }

    // 如果是叶子节点，说明找到了一个字符的编码
    if (root->l == nullptr && root->r == nullptr) {
        huffman_codes[root->c] = s;
        return;
    }

    // 递归遍历左子树，路径加 '0'
    generate_codes(root->l, s + "0");
    // 递归遍历右子树，路径加 '1'
    generate_codes(root->r, s + "1");
}

int main() {
    // 示例输入：字符及其频率
    // 可以替换为你的输入方式，比如从文件读取或标准输入
    map<char, int> frequencies;
    frequencies['A'] = 5;
    frequencies['B'] = 2;
    frequencies['C'] = 3;
    frequencies['D'] = 1;
    frequencies['E'] = 1;

    // 创建优先队列，存储节点指针
    priority_queue<Node*, vector<Node*>, CompareNode> pq;

    // 将所有字符及其频率作为叶子节点加入优先队列
    for (auto p : frequencies) {
        pq.push(new Node(p.first, p.second));
    }

    // 构建哈夫曼树
    while (pq.size() > 1) {
        // 取出权值最小的两个节点
        Node* l = pq.top();
        pq.pop();
        Node* r = pq.top();
        pq.pop();

        // 创建新的父节点，权值为两者之和
        Node* parent = new Node(l->w + r->w, l, r);
        // 将新父节点放回优先队列
        pq.push(parent);
    }

    // 此时优先队列中只剩一个节点，即哈夫曼树的根节点
    Node* root = pq.top();
    pq.pop(); // 取出根节点，队列为空

    // 生成哈夫曼编码
    generate_codes(root, "");

    cout << "哈夫曼编码表：" << endl;
    for (auto p : huffman_codes) {
        cout << p.first << ": " << p.second << endl;
    }

    // 计算总的带权路径长度 (Total WPL)
    long long total_wpl = 0;
    for (auto p : frequencies) {
        char ch = p.first;
        int freq = p.second;
        string code = huffman_codes[ch];
        total_wpl += (long long)freq * code.length();
    }
    cout << "总带权路径长度 (Total WPL): " << total_wpl << endl;

    // 释放根节点及其所有子节点内存
    // Node 的析构函数会递归删除子节点
    delete root;

    return 0;
}
```

**代码解析：**
1.  **`struct Node`**：定义了哈夫曼树的节点结构。每个节点包含权值 `w`、字符 `c`（只有叶子节点有）、以及指向左右子节点的指针 `l` 和 `r`。为了防止内存泄漏，我们还添加了一个析构函数 `~Node()`，它会递归地删除左右子节点。
2.  **`struct CompareNode`**：这是一个自定义的比较器，用于 `priority_queue`。`operator()` 被重载，使得当 `a->w > b->w` 时返回 `true`，这意味着 `a` 的优先级低于 `b`，从而实现了小顶堆（权值小的节点优先被取出）。
3.  **`huffman_codes`**：一个 `map`，用于存储字符到其哈夫曼编码（字符串）的映射。
4.  **`generate_codes(Node* root, string s)`**：这是一个递归函数，用于遍历哈夫曼树并生成编码。
    *   当遍历到叶子节点时，说明 `s` 就是该叶子节点对应字符的编码，将其存入 `huffman_codes`。
    *   否则，递归调用左子树（路径加 `0`）和右子树（路径加 `1`）。
5.  **`main` 函数**：
    *   首先定义了字符及其频率，并将其作为叶子节点创建 `Node` 对象，放入 `priority_queue`。
    *   进入 `while (pq.size() > 1)` 循环，这是哈夫曼树构建的核心。每次循环取出权值最小的两个节点，合并成一个新父节点，再将父节点放回优先队列。
    *   循环结束后，优先队列中只剩下一个节点，即哈夫曼树的根节点。
    *   调用 `generate_codes` 函数生成所有字符的哈夫曼编码。
    *   打印编码表和总带权路径长度。
    *   最后，调用 `delete root` 释放所有动态分配的内存。

2.  **填空题：** 给定四个字符及其频率：A(1), B(2), C(3), D(4)。构建哈夫曼树后，字符A的哈夫曼编码长度是 \_\_\_\_\_ 位。

3.  **单选题：** 在哈夫曼树的构建过程中，如果当前集合中有若干个节点权值相同，且都是最小的。我们应该如何选择进行合并？
    A.  随机选择两个进行合并。
    B.  选择其中任意两个进行合并，不影响最终的 WPL。
    C.  必须选择字符 ASCII 码最小的两个进行合并。
    D.  必须选择深度最深的两个进行合并。

4.  **填空题：** 如果一个哈夫曼树中有 5 个叶子节点，那么该哈夫曼树的总节点数是 \_\_\_\_\_ 个。

5.  **单选题：** 假设有字符集 {a, b, c, d}，对应的频率分别为 {5, 9, 12, 13}。构建哈夫曼树后，总带权路径长度 (WPL) 是多少？
    A.  78
    B.  80
    C.  82
    D.  84

---

**答案与解析部分：**

1.  **答案：** D
    **解析：** 哈夫曼树的构建过程不一定是唯一的。当存在多个权值相同的最小节点时，选择哪两个合并，或者左右子节点的顺序，都会导致树的形态不同，但最终的带权路径长度 (WPL) 是相同的。例如，D(1), E(1) 合并，也可以是 E(1), D(1) 合并，左右子树顺序不同。

2.  **答案：** 3
    **解析：**
    *   初始：{A(1), B(2), C(3), D(4)}
    *   1. 合并 A(1), B(2) -> 新节点 P1(3)
        集合：{P1(3), C(3), D(4)}
    *   2. 合并 P1(3), C(3) -> 新节点 P2(6) (这里也可以是 C(3), D(4) 先合并)
        集合：{P2(6), D(4)}
    *   3. 合并 P2(6), D(4) -> 新节点 Root(10)
        树的结构：
             Root(10)
            /        \
           P2(6)      D(4)
          /    \
         P1(3)  C(3)
        /    \
       A(1)   B(2)
        字符 A 的路径长度是 3。

3.  **答案：** B
    **解析：** 哈夫曼树的构建核心是保证每次合并的都是当前权值最小的两个节点。如果存在多个权值相同的最小节点，选择其中任意两个进行合并，都不会影响最终的带权路径长度 (WPL)，因为它们的权值都是一样的，只是树的形态可能略有不同。

4.  **答案：** 9
    **解析：** 在哈夫曼树中，如果叶子节点有 n 个，那么非叶子节点有 n-1 个。所以总节点数 = 叶子节点数 + 非叶子节点数 = n + (n-1) = 2n - 1。
    对于 5 个叶子节点，总节点数 = 2 * 5 - 1 = 9 个。

5.  **答案：** 78
    **解析：**
    *   初始：{W(5), X(9), Y(12), Z(13)} （按从小到大排列）
    *   1. 合并 W(5), X(9) -> P1(14)
        集合：{P1(14), Y(12), Z(13)}
    *   2. 合并 Y(12), Z(13) -> P2(25)
        集合：{P1(14), P2(25)}
    *   3. 合并 P1(14), P2(25) -> Root(39)
        树结构：
                Root(39)
               /        \
              P1(14)    P2(25)
             /    \    /    \
            W(5)  X(9) Y(12) Z(13)

        编码及路径长度：
        W: 00 (长度2)
        X: 01 (长度2)
        Y: 10 (长度2)
        Z: 11 (长度2)

        WPL = 5*2 + 9*2 + 12*2 + 13*2
            = 10 + 18 + 24 + 26
            = 78
