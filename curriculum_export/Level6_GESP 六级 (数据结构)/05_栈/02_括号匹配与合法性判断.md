## 深度知识点讲解

### 1. 栈（Stack）的本质：魔法收纳筒

栈是一种非常特别的数据结构，它的核心特点是“**先进后出**”（**LIFO**: Last In, First Out）。
想象一下，你有一个很深的魔法收纳筒，它只有一个口。你往里面放东西（压入，`push`），每次都从筒口放进去。当你需要拿东西的时候（弹出，`pop`），也只能从筒口拿。这意味着，你最后放进去的东西，会最先被拿出来。

*   **生活类比**：
    *   **叠盘子**：你洗完盘子，一个一个叠起来。当你需要用盘子时，总是从最上面（最后叠上去的）拿走。
    *   **Pringles薯片筒**：吃薯片时，总是从最上面那片开始吃。
    *   **子弹夹**：往弹夹里装子弹，最后装的子弹会最先被射出。
    *   **浏览器历史记录**：你访问网页时，每访问一个新页面，它就被“压入”历史记录的顶部。当你点击“后退”按钮时，总是回到你“最后访问”的那个页面。

*   **栈的基本操作**：
    *   `push(element)`：将一个元素放入栈的顶部（压入）。
    *   `pop()`：从栈的顶部取出一个元素（弹出）。
    *   `top()` / `peek()`：查看栈顶元素，但不取出。
    *   `isEmpty()`：判断栈是否为空。
    *   `isFull()`：判断栈是否已满（当我们用固定大小的数组模拟栈时需要考虑）。

### 2. 为什么栈能解决括号匹配问题？

括号匹配问题的核心规则是“**后打开的括号，必须先关闭**”。这和栈的“先进后出”特性简直是天作之合！

*   当遇到一个**开括号**（例如 `(`, `[`, `{`）时，它表示我们“打开”了一个新的作用域或结构，需要记住它，因为它将来需要被关闭。我们把它**压入栈**中，就像把它放进我们的魔法收纳筒。
*   当遇到一个**闭括号**（例如 `)`, `]`, `}`）时，它表示我们试图“关闭”一个作用域。根据规则，它应该关闭的是**最近一次打开且尚未关闭**的那个括号。而栈顶的元素，恰好就是那个“最近一次打开”的括号！
    *   我们检查栈顶元素是否与当前的闭括号匹配。
    *   如果匹配，说明这对括号成功地手拉手了，我们就可以把栈顶的开括号**弹出栈**。
    *   如果不匹配，或者栈已经空了（根本没有开括号可以关闭），那么这个表达式就是不合法的。

*   **关键点剖析**：
    1.  **开括号入栈**：遇到 `(`, `[`, `{` 时，将它们 `push` 到栈中。这代表我们“记住了”一个待关闭的开括号。
    2.  **闭括号出栈并匹配**：
        *   遇到 `)`, `]`, `}` 时，首先检查栈是否为空。如果栈为空，却遇到了闭括号，说明没有对应的开括号，直接判定为不合法。
        *   如果栈不为空，则取出栈顶元素（`pop`）。
        *   判断取出的栈顶元素是否与当前闭括号类型匹配。例如，如果当前是 `)`，栈顶应该是 `(`；如果是 `]`，栈顶应该是 `[`；如果是 `}`，栈顶应该是 `{`。如果不匹配，判定为不合法。
    3.  **最终检查**：遍历完整个表达式后，如果栈中还有元素（栈不为空），说明有一些开括号没有被关闭，表达式不合法。只有当栈最终为空时，所有括号都成功匹配，表达式才合法。

*   **常见误区**：
    *   **栈空遇到闭括号**：`)(` 这种，第一个 `)` 就直接判断不合法。
    *   **类型不匹配**：`([)]` 这种，`]` 遇到栈顶的 `(`，类型不匹配，不合法。
    *   **遍历结束栈不空**：`((` 这种，遍历完两个 `(` 都入栈了，但没有 `)` 弹出，最后栈不为空，不合法。

### 3. 用数组模拟栈

在C++中，GESP 考试通常要求我们使用原生数组来模拟栈，而不是直接使用 `std::stack`。这能让我们更深入理解栈的内部工作原理。

我们需要一个字符数组 `s` 来存储栈的元素，以及一个整数 `t`（通常表示 `top`）来记录栈顶元素的位置。

*   `s[0], s[1], ..., s[t-1]` 存放栈中的元素。
*   `t` 表示栈中元素的个数，同时也是下一个元素应该存放的下标。
*   当 `t == 0` 时，栈为空。

**操作实现**：
*   **push(c)**: `s[t++] = c;` （先赋值，再 `t` 自增）
*   **pop()**: `return s[--t];` （先 `t` 自减，再返回 `s[t]`）
*   **isEmpty()**: `return t == 0;`
*   **top()**: `return s[t - 1];` （注意，如果栈为空，调用 `top()` 会出错，需要先判断 `isEmpty()`）

## 典型例题精讲

**例题：判断括号字符串是否合法**

给定一个只包含 `'('`, `')'`, `'[`, `']'`, `'{'`, `'}'` 这六种字符的字符串，判断该字符串是否合法。

**输入格式**：
一行字符串 `str`。

**输出格式**：
如果合法，输出 `YES`；否则输出 `NO`。

**示例**：
*   输入: `([])`
*   输出: `YES`

*   输入: `([)]`
*   输出: `NO`

*   输入: `({[]})`
*   输出: `YES`

### 思路分析

1.  **初始化**：创建一个空栈（用数组 `s` 和栈顶指针 `t` 模拟）。
2.  **遍历字符串**：从左到右逐个字符处理输入字符串 `str`。
3.  **处理开括号**：如果当前字符是 `(`, `[`, `{` 中的任意一个，将其 `push` 到栈中。
4.  **处理闭括号**：如果当前字符是 `)`, `]`, `}` 中的任意一个：
    *   **检查栈是否为空**：如果此时栈为空（`isEmpty()` 为真），说明没有对应的开括号可以匹配，直接判定为不合法。
    *   **弹出栈顶元素并匹配**：如果栈不为空，则 `pop` 出栈顶元素。
        *   如果当前字符是 `)`，但弹出的不是 `(`，则不匹配，判定为不合法。
        *   如果当前字符是 `]`，但弹出的不是 `[`，则不匹配，判定为不合法。
        *   如果当前字符是 `}`，但弹出的不是 `{`，则不匹配，判定为不合法。
5.  **最终检查**：字符串遍历结束后，如果栈仍然不为空（`isEmpty()` 为假），说明有未匹配的开括号，判定为不合法。
6.  **合法判断**：如果以上所有检查都通过，则字符串是合法的。

### 逐步演示 (`([)]`)

假设输入字符串 `str = "([)]"`

1.  **初始化**：`s` 数组空，`t = 0`。
2.  **字符 `(`**：是开括号，`push('(')`。
    *   栈: `(`
    *   `t = 1`
3.  **字符 `[`**：是开括号，`push('[')`。
    *   栈: `(`, `[`
    *   `t = 2`
4.  **字符 `)`**：是闭括号。
    *   栈不为空。
    *   `pop()`，弹出 `[`。
    *   当前是 `)`，弹出的却是 `[`。**不匹配！** 判定为不合法。输出 `NO`。

### 逐步演示 (`({[]})`)

假设输入字符串 `str = "({[]})"`

1.  **初始化**：`s` 数组空，`t = 0`。
2.  **字符 `(`**：`push('(')`。
    *   栈: `(`
    *   `t = 1`
3.  **字符 `{`**：`push('{')`。
    *   栈: `(`, `{`
    *   `t = 2`
4.  **字符 `[`**：`push('[')`。
    *   栈: `(`, `{`, `[`
    *   `t = 3`
5.  **字符 `]`**：是闭括号。
    *   栈不为空。
    *   `pop()`，弹出 `[`。
    *   当前是 `]`，弹出的是 `[`。**匹配！**
    *   栈: `(`, `{`
    *   `t = 2`
6.  **字符 `}`**：是闭括号。
    *   栈不为空。
    *   `pop()`，弹出 `{`。
    *   当前是 `}`，弹出的是 `{`。**匹配！**
    *   栈: `(`
    *   `t = 1`
7.  **字符 `)`**：是闭括号。
    *   栈不为空。
    *   `pop()`，弹出 `(`。
    *   当前是 `)`，弹出的是 `(`。**匹配！**
    *   栈: 空
    *   `t = 0`
8.  **遍历结束**。
    *   栈为空。
    *   **合法！** 输出 `YES`。

### 代码实现

```cpp
#include <iostream> // 用于输入输出
#include <string>   // 用于处理字符串

// 为了方便，这里使用bits/stdc++.h，实际比赛中也可以用
// #include <bits/stdc++.h> // 包含大部分常用头文件，比赛常用

// 定义栈的最大容量，通常根据题目字符串长度限制来设置
const int MAXN = 1005; 

// 模拟栈的数组
char s[MAXN]; 
// 栈顶指针，初始为0表示栈为空，指向下一个可以放入元素的位置
int t = 0; 

// 栈的压入操作
void push(char c) {
    if (t < MAXN) { // 防止栈溢出
        s[t++] = c; // 将元素放入栈顶，并移动栈顶指针
    }
}

// 栈的弹出操作
char pop() {
    if (t > 0) { // 栈不为空才能弹出
        return s[--t]; // 移动栈顶指针，并返回原栈顶元素
    }
    return '\0'; // 栈为空时返回一个特殊值表示错误或空
}

// 判断栈是否为空
bool isEmpty() {
    return t == 0;
}

// 判断给定括号字符串是否合法
bool checkParentheses(const std::string& str) {
    t = 0; // 每次检查前重置栈，确保是空栈状态

    for (char c : str) { // 遍历字符串中的每一个字符
        if (c == '(' || c == '[' || c == '{') {
            // 如果是开括号，压入栈中
            push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            // 如果是闭括号
            if (isEmpty()) {
                // 如果栈为空，却遇到了闭括号，说明没有对应的开括号，不合法
                return false; 
            }
            char p = pop(); // 弹出栈顶的开括号

            // 判断弹出的开括号是否与当前闭括号类型匹配
            if ((c == ')' && p != '(') ||
                (c == ']' && p != '[') ||
                (c == '}' && p != '{')) {
                // 类型不匹配，不合法
                return false; 
            }
        }
        // 如果字符串中包含其他字符，我们这里选择忽略它们
        // 实际题目可能要求处理或直接视为不合法，需根据题目具体要求
    }

    // 遍历完所有字符后，如果栈为空，说明所有开括号都成功匹配并关闭
    // 否则，栈中还有未关闭的开括号，不合法
    return isEmpty(); 
}

int main() {
    std::string str;
    std::cin >> str; // 读取输入的字符串

    if (checkParentheses(str)) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0;
}

```

### 复杂度分析

*   **时间复杂度**：
    *   我们遍历输入字符串 `str` 一次，对每个字符进行常数次操作（`push` 或 `pop` 或比较）。
    *   因此，时间复杂度与字符串的长度 `N` 成正比，记为 **O(N)**。

*   **空间复杂度**：
    *   在最坏情况下（例如 `((((...))))` 这样的字符串），栈中可能存储所有开括号。栈的最大深度与字符串的长度 `N` 成正比。
    *   因此，空间复杂度也与字符串的长度 `N` 成正比，记为 **O(N)**。

## 代码实现模板

```cpp
#include <iostream> // 标准输入输出流
#include <string>   // 字符串处理
// 注意：GESP 考试通常禁止使用 STL 容器如 std::stack，
// 必须手动实现栈。这里为了方便演示，如果允许，也可以用 #include <bits/stdc++.h>

// ========== 栈的实现部分 ==========
// 定义栈的最大容量，根据题目字符串长度上限调整
const int MAX_STACK_SIZE = 1005; 
char stack_data[MAX_STACK_SIZE]; // 存储栈元素的数组
int stack_top = 0;               // 栈顶指针，指示下一个要压入元素的位置

// 栈操作：压入元素
void stack_push(char c) {
    if (stack_top < MAX_STACK_SIZE) { // 检查是否栈满
        stack_data[stack_top++] = c;   // 存入元素，栈顶指针后移
    }
    // else { // 可选：处理栈满错误，例如输出错误信息
    //     std::cerr << "Error: Stack overflow!" << std::endl;
    // }
}

// 栈操作：弹出元素
char stack_pop() {
    if (stack_top > 0) { // 检查是否栈空
        return stack_data[--stack_top]; // 栈顶指针前移，返回原栈顶元素
    }
    // else { // 可选：处理栈空错误，例如输出错误信息
    //     std::cerr << "Error: Stack underflow!" << std::endl;
    // }
    return '\0'; // 返回一个特殊值表示栈空或错误
}

// 栈操作：判断栈是否为空
bool stack_is_empty() {
    return stack_top == 0;
}

// 栈操作：获取栈顶元素（不弹出）
char stack_peek() {
    if (stack_top > 0) { // 检查是否栈空
        return stack_data[stack_top - 1];
    }
    return '\0'; // 返回特殊值表示栈空或错误
}

// ========== 括号匹配逻辑部分 ==========

// 判断字符 c1 和 c2 是否是匹配的括号对
bool is_matching_pair(char c1, char c2) {
    return (c1 == '(' && c2 == ')') ||
           (c1 == '[' && c2 == ']') ||
           (c1 == '{' && c2 == '}');
}

// 判断字符串中的括号是否合法
bool check_parentheses_validity(const std::string& s_input) {
    stack_top = 0; // 每次检查前，重置栈为空

    for (char c : s_input) { // 遍历输入字符串中的每个字符
        if (c == '(' || c == '[' || c == '{') {
            // 如果是开括号，压入栈中
            stack_push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            // 如果是闭括号
            if (stack_is_empty()) {
                // 栈为空却遇到闭括号，说明没有对应的开括号，不合法
                return false; 
            }
            // 弹出栈顶的开括号，并检查是否匹配
            char top_char = stack_pop();
            if (!is_matching_pair(top_char, c)) {
                // 如果不匹配，则不合法
                return false; 
            }
        }
        // 对于其他非括号字符，根据题目要求可以忽略或视为不合法
        // 在本例中，我们假设字符串只包含括号，或者忽略其他字符。
    }

    // 遍历结束后，如果栈为空，说明所有开括号都被正确关闭了，合法
    // 否则，栈中还有未关闭的开括号，不合法
    return stack_is_empty();
}

int main() {
    std::string input_string;
    std::cin >> input_string; // 从标准输入读取字符串

    if (check_parentheses_validity(input_string)) {
        std::cout << "YES" << std::endl; // 合法输出 YES
    } else {
        std::cout << "NO" << std::endl;  // 不合法输出 NO
    }

    return 0; // 程序正常结束
}

```

    *   输出: `()` (删除了 `[` 和 `]`)

### 答案

#### 基础巩固答案

1.  **简单判断**
    *   A. `()`: `YES`
    *   B. `][`: `NO` (第一个 `]` 栈空，不合法)
    *   C. `{}`: `YES`
    *   D. `(()`: `NO` (最后栈不为空，`(` 未匹配)
    *   E. `[]}`: `NO` (最后一个 `}` 遇到栈顶 `[`，不匹配)

2.  **完整模拟**
    *   `(`: `push('(')`。栈：`(`
    *   `{`: `push('{')`。栈：`(`, `{`
    *   `[`: `push('[')`。栈：`(`, `{`, `[`
    *   `]` `pop()` 弹出 `[`。`]` 与 `[` 匹配。栈：`(`, `{`
    *   `}` `pop()` 弹出 `{`。`}` 与 `{` 匹配。栈：`(`
    *   `)` `pop()` 弹出 `(`。`)` 与 `(` 匹配。栈：空
    *   遍历结束，栈为空。所以 `({[]})` 是 `YES`。

3.  **填空题**
    当程序处理完一个括号字符串后，如果栈中还有元素，说明这个字符串**不合法**。

#### 能力提升答案

1.  **包含其他字符**

    ```cpp
                return false;
            }

        // else { // 遇到其他字符，直接跳过，不影响栈操作
        //     continue; 
        // }
    }
    ```
    （默认行为就是跳过，因为 `if-else if` 结构只处理括号字符，其他字符会被自动忽略。）

2.  **空字符串**
    空字符串 `""` 应该被认为是**合法**的括号表达式。因为它没有不匹配的括号。
    修改 `checkParentheses` 函数：
    *   当前代码已经可以正确处理空字符串：`for` 循环不会执行，`stack_top` 保持为 `0`，`isEmpty()` 返回 `true`，所以输出 `YES`。无需修改。

3.  **找出第一个错误**
    这需要修改函数返回类型和记录位置。
    ```cpp
    // 可以返回一个表示错误类型的枚举或int，或者直接在函数内部输出并退出
    // 这里以输出错误信息为例
    bool checkParenthesesWithDetail(const std::string& str) {
        t = 0;
        for (int i = 0; i < str.length(); ++i) { // 使用索引i来记录位置
            char c = str[i];
            if (c == '(' || c == '[' || c == '{') {
                push(c);
            } else if (c == ')' || c == ']' || c == '}') {
                if (isEmpty()) {
                    std::cout << "Error: Unexpected closing bracket '" << c << "' at position " << i + 1 << std::endl;
                    return false;
                }
                char p = pop();
                if (!((c == ')' && p == '(') || (c == ']' && p == '[') || (c == '}' && p == '{'))) {
                    std::cout << "Error: Mismatched bracket. Expected '" << getMatchingOpen(c) << "' but found '" << p << "' for closing '" << c << "' at position " << i + 1 << std::endl;
                    return false;
                }
            }
        }

        if (!isEmpty()) {
            std::cout << "Error: Unclosed opening bracket(s) remaining." << std::endl;
            // 还可以遍历栈，指出具体是哪些没关
            // for (int i = 0; i < t; ++i) {
            //     std::cout << "  - Unclosed: " << s[i] << std::endl;
            // }
            return false;
        }
        return true;
    }

    // 辅助函数，获取闭括号对应的开括号类型
    char getMatchingOpen(char c) {
        if (c == ')') return '(';
        if (c == ']') return '[';
        if (c == '}') return '{';
        return '\0';
    }
    ```

#### 拓展挑战答案

1.  **最长合法括号子串** (提示：使用栈存储开括号的索引，或不匹配的括号的索引)
    ```cpp
    // 这是一个常见的解法，核心思想是栈中存储不匹配括号的索引。
    // 初始时，-1 入栈作为哨兵，表示右边界。
    int longestValidParentheses(const std::string& s_input) {
        int max_len = 0;
        // 使用一个数组模拟栈，存储索引
        int idx_stack[MAX_STACK_SIZE];
        int idx_top = 0;

        idx_stack[idx_top++] = -1; // 哨兵值，表示合法子串的起始前一个位置

        for (int i = 0; i < s_input.length(); ++i) {
            char c = s_input[i];
            if (c == '(') {
                idx_stack[idx_top++] = i; // 压入开括号的索引
            } else { // 遇到 ')'
                if (idx_top > 0) { // 栈不为空
                    idx_top--; // 弹出栈顶元素
                    if (idx_top > 0) { // 如果栈不为空，说明匹配成功
                        // 当前合法子串的长度 = 当前字符索引 - 栈顶索引（即合法子串的起始前一个位置）
                        max_len = std::max(max_len, i - idx_stack[idx_top - 1]);
                    } else { // 栈为空，说明当前 ')' 无法匹配，或者匹配了哨兵
                        idx_stack[idx_top++] = i; // 将当前 ')' 的索引压入栈，作为新的合法子串的起始前一个位置
                    }
                } else { // 栈为空，直接将当前 ')' 的索引压入栈
                    idx_stack[idx_top++] = i;
                }
            }
        }
        return max_len;
    }
    ```

2.  **删除最少字符使括号合法** (提示：这通常涉及两次遍历或更复杂的动态规划)
    *   第一次遍历：从左到右，用栈找出所有多余的 `)`。
    *   第二次遍历：从右到左，用栈找出所有多余的 `(`。
    *   两次遍历中被标记为多余的字符，都删除。
