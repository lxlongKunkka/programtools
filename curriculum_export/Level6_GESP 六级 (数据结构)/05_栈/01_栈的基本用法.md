## 深度知识点讲解

### 1. 栈的本质：后进先出 (LIFO)

栈（Stack）是一种线性数据结构，它只允许在表的一端进行插入和删除操作。我们把这一端称为“栈顶”（Top），另一端称为“栈底”（Bottom）。

*   **LIFO (Last In, First Out)**：这是栈最核心的特性。就像我们引入的盘子或Pez糖果的例子一样，最后进入栈的元素，会最先被取出。最先进入栈的元素，反而会最后被取出。

**为什么是LIFO？**
因为栈这种结构，就好像一个只有一个开口的容器。你只能从这个开口放东西进去，也只能从这个开口拿东西出来。当你放了第一个、第二个、第三个东西进去后，第三个东西在最上面，你当然只能先拿到它。只有把第三个拿走，才能拿到第二个，以此类推。这种物理上的限制，自然就形成了LIFO的规则。

### 2. 栈的基本操作

虽然栈的结构很简单，但它有几个非常重要的基本操作：

1.  **入栈 (Push)**：将一个新元素添加到栈顶。
    *   想象：把一个新盘子放到盘子堆的最上面。
2.  **出栈 (Pop)**：从栈顶移除一个元素。
    *   想象：从盘子堆的最上面拿走一个盘子。
    *   **注意**：出栈操作通常只移除元素，不返回元素的值。如果需要获取栈顶元素的值，应该先使用 `Top` 操作。
3.  **查看栈顶元素 (Top / Peek)**：返回栈顶元素的值，但不移除它。
    *   想象：看看盘子堆最上面的是什么盘子，但不拿走它。
4.  **判断栈是否为空 (Empty)**：检查栈中是否没有任何元素。
    *   想象：看看盘子堆上还有没有盘子。
5.  **获取栈中元素数量 (Size)**：返回栈中当前元素的个数。
    *   想象：数一数盘子堆上一共有多少个盘子。

### 3. 栈的数组实现（C++）

在C++中，我们通常使用一个原生数组（或者动态数组，但GESP 6级要求原生数组）来模拟栈的存储空间，并用一个整型变量来指示栈顶的位置。

*   **存储空间**：一个固定大小的数组 `s[]`，用来存放栈中的元素。
*   **栈顶指针/索引**：一个整型变量 `t`（通常表示 `top`），它指向栈顶元素的下标。

**如何管理 `t`？**
*   **空栈状态**：当栈为空时，`t` 通常初始化为 `-1`。这意味着数组中没有任何元素被认为是栈的一部分。
*   **入栈 `push(x)`**：
    1.  `t` 先加 1 (`++t`)。
    2.  将元素 `x` 存放到 `s[t]`。
*   **出栈 `pop()`**：
    1.  `t` 减 1 (`t--`)。
    2.  （可选：如果需要获取出栈元素，可以在 `t--` 之前返回 `s[t]` 的值）。
*   **查看栈顶 `top()`**：
    1.  返回 `s[t]` 的值。
*   **判断栈空 `empty()`**：
    1.  如果 `t == -1`，说明栈为空。
*   **获取大小 `size()`**：
    1.  栈中元素的数量就是 `t + 1`。

**生活类比：停车场**
想象一个狭窄的停车场，只有一个出入口，并且车辆只能一辆接一辆地停进去，也只能一辆接一辆地开出来。最先进去的车被堵在最里面，最后进去的车停在最外面，最先开走。
*   **入栈**：车开进停车场。
*   **出栈**：车开出停车场。
*   **栈顶**：最外面那辆车。
*   **栈底**：最里面那辆车。
*   **栈空**：停车场里没车了。
*   **栈满**：停车场满了，不能再进车了。

### 4. 常见误区与边界情况

1.  **栈下溢 (Stack Underflow)**：
    *   **问题**：尝试从一个空栈中执行 `pop` 或 `top` 操作。
    *   **后果**：程序可能崩溃（访问了无效内存），或者返回一个无意义的值。
    *   **解决方法**：在执行 `pop` 或 `top` 之前，务必先用 `empty()` 检查栈是否为空。
2.  **栈溢出 (Stack Overflow)**：
    *   **问题**：尝试向一个已经满了的栈中执行 `push` 操作。
    *   **后果**：程序可能崩溃，或者覆盖了不该覆盖的内存区域。
    *   **解决方法**：在使用原生数组实现栈时，栈的大小是固定的。在 `push` 之前，需要检查栈是否已满。我们可以定义一个 `MAXN` 常量表示数组的最大容量，并添加一个 `isFull()` 函数来检查 `t == MAXN - 1`。

### 5. 栈的应用场景（启发）

*   **函数调用栈**：程序运行时，函数之间的调用关系就是通过一个栈来管理的。
*   **表达式求值**：将中缀表达式转换为后缀表达式，或者直接计算表达式的值。
*   **括号匹配**：检查代码中的括号、大括号、方括号是否正确匹配。
*   **撤销/重做功能 (Undo/Redo)**：在文本编辑器或绘图软件中，每一步操作都可以被“压入”一个栈，点击撤销就是从栈中“弹出”最近的操作。

### 例题一：模拟栈操作

**题目描述**
小明正在学习栈这种数据结构。现在他想用一个数组来模拟栈的操作。请你实现以下功能：
1.  `push x`：将整数 `x` 入栈。
2.  `pop`：将栈顶元素出栈。
3.  `top`：输出栈顶元素。
4.  `empty`：判断栈是否为空，是则输出 `YES`，否则输出 `NO`。
5.  `size`：输出栈中元素个数。
假设栈的最大容量为 100。当尝试对空栈进行 `pop` 或 `top` 操作时，输出 `Error`。当尝试对满栈进行 `push` 操作时，输出 `Error`。

**输入格式**
第一行一个整数 `N`，表示操作次数。
接下来 `N` 行，每行一个操作指令。

**输出格式**
根据指令输出相应结果。

**输入样例**
```
7
push 10
push 20
top
pop
empty
size
pop
```

**输出样例**
```
20
NO
1
Error
```

**思路分析**
1.  我们需要一个数组 `s` 来存储栈的元素，以及一个变量 `t` 来表示栈顶的索引。
2.  `t` 初始化为 `-1`，表示栈为空。
3.  定义一个常量 `MAXN` 为 100，表示栈的最大容量。
4.  根据不同的指令执行相应的操作，并注意处理栈空和栈满的边界情况。

    *   **`push x`**:
        *   先判断 `t` 是否等于 `MAXN - 1` (栈是否已满)。如果满了，输出 `Error`。
        *   否则，`t` 加 1 (`++t`)，然后 `s[t] = x`。
    *   **`pop`**:
        *   先判断 `t` 是否等于 `-1` (栈是否为空)。如果空了，输出 `Error`。
        *   否则，`t` 减 1 (`t--`)。
    *   **`top`**:
        *   先判断 `t` 是否等于 `-1` (栈是否为空)。如果空了，输出 `Error`。
        *   否则，输出 `s[t]`。
    *   **`empty`**:
        *   如果 `t == -1`，输出 `YES`。
        *   否则，输出 `NO`。
    *   **`size`**:
        *   输出 `t + 1`。

**代码实现**

```cpp
#include <iostream>
#include <string> // 用于读取字符串指令

const int MAXN = 100; // 栈的最大容量
int s[MAXN];          // 存储栈元素的数组
int t = -1;           // 栈顶指针，初始化为-1表示空栈

void push(int x) {
    if (t == MAXN - 1) { // 检查栈是否已满
        std::cout << "Error" << std::endl;
        return;
    }
    s[++t] = x; // 栈顶指针先加1，再存入元素
}

void pop() {
    if (t == -1) { // 检查栈是否为空
        std::cout << "Error" << std::endl;
        return;
    }
    t--; // 栈顶指针减1，移除栈顶元素
}

int top() {
    if (t == -1) { // 检查栈是否为空
        // 这里为了简化，直接输出Error并返回一个特殊值，
        // 实际应用中可以抛出异常或通过返回值指示错误
        std::cout << "Error" << std::endl;
        return -1; // 返回一个不可能的栈顶值
    }
    return s[t]; // 返回栈顶元素
}

bool empty() {
    return t == -1; // 判断栈是否为空
}

int size() {
    return t + 1; // 栈中元素个数
}

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出速度
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    std::string cmd;
    int x;

    for (int i = 0; i < n; ++i) {
        std::cin >> cmd;
        if (cmd == "push") {
            std::cin >> x;
            push(x);
        } else if (cmd == "pop") {
            pop();
        } else if (cmd == "top") {
            if (!empty()) { // 只有非空栈才能输出栈顶
                std::cout << top() << std::endl;
            } else {
                // top函数内部已经处理了Error输出，这里无需重复
                top(); // 再次调用top，让它输出Error
            }
        } else if (cmd == "empty") {
            if (empty()) {
                std::cout << "YES" << std::endl;
            } else {
                std::cout << "NO" << std::endl;
            }
        } else if (cmd == "size") {
            std::cout << size() << std::endl;
        }
    }

    return 0;
}
```

**时间复杂度**
每个栈操作（push, pop, top, empty, size）都只涉及对栈顶指针 `t` 和数组 `s` 的常数次访问和修改，因此它们的时间复杂度都是 **O(1)**。
总的时间复杂度取决于操作次数 `N`，为 **O(N)**。

### 例题二：括号匹配 (简化版)

**题目描述**
给定一个只包含小括号 `(` 和 `)` 的字符串，请判断字符串中的括号是否匹配。如果匹配，输出 `YES`；否则，输出 `NO`。
例如：`(())` 匹配，`()(` 不匹配，`)(` 不匹配。

**输入格式**
一行字符串 `s`。

**输出格式**
`YES` 或 `NO`。

**输入样例**
```
(()(()))
```

**输出样例**
```
YES
```

**思路分析**
这是一个经典的栈应用问题。
1.  遍历输入的字符串。
2.  遇到左括号 `(`，就将其入栈。
3.  遇到右括号 `)`：
    *   如果此时栈为空，说明没有对应的左括号与当前右括号匹配，直接判定为不匹配，输出 `NO`。
    *   如果栈不为空，则将栈顶的左括号出栈，表示这对括号匹配成功。
4.  字符串遍历结束后：
    *   如果栈为空，说明所有左括号都找到了对应的右括号，匹配成功，输出 `YES`。
    *   如果栈不为空，说明还有多余的左括号没有匹配，判定为不匹配，输出 `NO`。

**代码实现**

```cpp
#include <iostream>
#include <string>
#include <bits/stdc++.h> // 包含常用的头文件

const int MAXN = 1005; // 假设字符串长度不会超过1000，栈容量也足够
char s[MAXN];          // 存储栈元素的数组
int t = -1;            // 栈顶指针

void push(char c) {
    if (t == MAXN - 1) { // 简单处理栈满，虽然本题输入不会导致栈满
        // std::cout << "Stack Overflow!" << std::endl;
        return;
    }
    s[++t] = c;
}

void pop() {
    if (t == -1) { // 简单处理栈空
        // std::cout << "Stack Underflow!" << std::endl;
        return;
    }
    t--;
}

bool empty() {
    return t == -1;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::string str;
    std::cin >> str;

    bool balanced = true; // 标志是否匹配

    for (char c : str) {
        if (c == '(') {
            push(c); // 遇到左括号，入栈
        } else if (c == ')') {
            if (empty()) { // 遇到右括号，但栈为空，说明没有匹配的左括号
                balanced = false;
                break; // 提前结束
            } else {
                pop(); // 栈不为空，出栈一个左括号进行匹配
            }
        }
    }

    if (!empty()) { // 循环结束后，如果栈不为空，说明还有多余的左括号
        balanced = false;
    }

    if (balanced) {
        std::cout << "YES" << std::endl;
    } else {
        std::cout << "NO" << std::endl;
    }

    return 0;
}
```

**时间复杂度**
程序需要遍历输入字符串一次。在遍历过程中，每个字符最多进行一次 `push` 或 `pop` 操作。由于 `push` 和 `pop` 操作的时间复杂度都是 O(1)，因此总的时间复杂度为 **O(L)**，其中 `L` 是字符串的长度。

## 代码实现模板

下面是一个使用原生数组实现栈的通用模板，包含了所有基本操作，并考虑了栈空和栈满的边界情况。

```cpp
#include <iostream> // 输入输出流
#include <string>   // 字符串（如果需要处理字符串指令）
// #include <bits/stdc++.h> // GESP常用的万能头文件，包含大部分标准库

// 定义栈的最大容量。根据题目要求设置，例如1000、10000等。
// 建议定义为常量，方便修改和管理。
const int MAX_STACK_SIZE = 1005; 

// 存储栈元素的数组。
// 这里的元素类型是int，可以根据实际需求修改为char, double, string等。
int s[MAX_STACK_SIZE]; 

// 栈顶指针。
// 初始化为-1表示栈为空，因为数组下标从0开始。
// 如果初始化为0，则表示栈顶在s[0]，且s[0]是第一个元素。
// 不同的初始化方式会影响push/pop中t的自增自减顺序。
// 推荐t=-1，先++t再赋值，这样t始终指向栈顶元素。
int t = -1; 

// 1. 判断栈是否为空
// 返回true表示空，false表示非空。
bool empty() {
    return t == -1;
}

// 2. 判断栈是否已满
// 返回true表示满，false表示未满。
bool isFull() {
    return t == MAX_STACK_SIZE - 1;
}

// 3. 入栈操作 (Push)
// 参数x：要入栈的元素。
void push(int x) {
    if (isFull()) { // 检查栈是否已满，防止栈溢出
        std::cout << "Error: Stack is full. Cannot push " << x << std::endl;
        return;
    }
    // 栈顶指针先自增，然后将元素存入新的栈顶位置
    s[++t] = x; 
    // std::cout << "Pushed: " << x << std::endl; // 可选：打印调试信息
}

// 4. 出栈操作 (Pop)
// 注意：pop通常只移除元素，不返回元素值。
// 如果需要获取出栈元素，应先调用top()。
void pop() {
    if (empty()) { // 检查栈是否为空，防止栈下溢
        std::cout << "Error: Stack is empty. Cannot pop." << std::endl;
        return;
    }
    // 栈顶指针自减，表示移除当前栈顶元素
    t--; 
    // std::cout << "Popped an element." << std::endl; // 可选：打印调试信息
}

// 5. 获取栈顶元素 (Top / Peek)
// 返回栈顶元素的值，但不移除。
// 注意：该函数返回int，所以需要处理空栈返回什么。
int top() {
    if (empty()) { // 检查栈是否为空，防止栈下溢
        std::cout << "Error: Stack is empty. No top element." << std::endl;
        // 实际应用中，这里可能抛出异常或返回一个特定错误码
        // 这里返回一个不可能的值（例如-1或INT_MIN）作为错误指示
        return -1; 
    }
    // 返回栈顶指针当前指向的元素
    return s[t]; 
}

// 6. 获取栈中元素数量 (Size)
// 返回栈中元素的个数。
int size() {
    return t + 1; // 因为t从-1开始计数，所以实际元素个数是t+1
}

// 主函数示例：演示如何使用这些栈操作
int main() {
    // 优化C++标准输入输出流，使其与C风格I/O同步，通常用于提高速度。
    std::ios_base::sync_with_stdio(false); 
    // 解除cin和cout的绑定，进一步提高输入输出效率。
    std::cin.tie(NULL); 

    std::cout << "Stack Operations Demo:" << std::endl;

    // 初始状态
    std::cout << "Is stack empty? " << (empty() ? "YES" : "NO") << std::endl; // YES
    std::cout << "Stack size: " << size() << std::endl; // 0

    push(10); // Push 10
    push(20); // Push 20
    push(30); // Push 30

    std::cout << "After pushing 10, 20, 30:" << std::endl;
    std::cout << "Is stack empty? " << (empty() ? "YES" : "NO") << std::endl; // NO
    std::cout << "Stack size: " << size() << std::endl; // 3
    std::cout << "Top element: " << top() << std::endl; // 30

    pop(); // Pop
    std::cout << "After popping one element:" << std::endl;
    std::cout << "Stack size: " << size() << std::endl; // 2
    std::cout << "Top element: " << top() << std::endl; // 20

    pop(); // Pop
    pop(); // Pop
    std::cout << "After popping two more elements:" << std::endl;
    std::cout << "Is stack empty? " << (empty() ? "YES" : "NO") << std::endl; // YES
    std::cout << "Stack size: " << size() << std::endl; // 0

    pop(); // Try to pop from an empty stack
    top(); // Try to get top from an empty stack

    return 0;
}
```

**代码风格说明**
*   `#include <bits/stdc++.h>`：这是一个GCC编译器特有的万能头文件，包含了C++标准库中的大部分头文件。在GSP竞赛中常用来方便快捷地引入所需库。
*   **Allman风格大括号**：函数体、`if/else`、`for/while` 等语句块的开括号 `{` 另起一行，与语句头对齐。
*   **变量命名**：按照GESP要求，使用 `a-z` 单字母变量名（如 `n, m, i, j, k, x, y, c, s, t` 等）。对于全局变量和常量，可以使用更具描述性的名称（如 `MAX_STACK_SIZE`）。
*   **STL限制**：严格遵守GESP 1-5级（乃至6级大部分基础题）不使用 `std::vector`, `std::stack`, `std::queue` 等STL容器的规定，而是使用原生数组 `s[]` 和手动管理栈顶指针 `t`。`std::sort` 函数是允许的，但本专题暂时用不到。

    1.  `push(1)`: `[1]`, `t=0`
    2.  `push(2)`: `[1, 2]`, `t=1`
    3.  `pop()`: `[1]`, `t=0`
    4.  `push(3)`: `[1, 3]`, `t=1`
    5.  `push(4)`: `[1, 3, 4]`, `t=2`
    6.  `top()`: 输出 `4`
    7.  `push(5)`: `[1, 3, 4, 5]`, `t=3`
    8.  `push(6)`: `[1, 3, 4, 5, 6]`, `t=4` (栈满，但仍可入栈，因为容量是5)

        *   `push(6)` 之后栈为 `[1, 3, 4, 5, 6]`，`t=4`。
        *   如果再 `push`，则会触发 `isFull()` 报错。
        *   最终栈中元素（从底到顶）：`[1, 3, 4, 5, 6]`，`t=4`。

#### 能力提升 (代码略，只提供思路)

1.  **括号匹配（多类型）**
    *   在 `main` 函数的循环中：
        *   如果 `c` 是 `(`, `[`, `{`，则 `push(c)`。
        *   如果 `c` 是 `)`:
            *   若 `empty()` 或 `top() != '('`，则 `balanced = false; break;`
            *   否则 `pop()`。
        *   如果 `c` 是 `]`:
            *   若 `empty()` 或 `top() != '['`，则 `balanced = false; break;`
            *   否则 `pop()`。
        *   如果 `c` 是 `}`:
            *   若 `empty()` 或 `top() != '{'`，则 `balanced = false; break;`
            *   否则 `pop()`。
    *   最后判断 `balanced` 和 `empty()`。

        *   `std::cout << top();`
        *   `pop();`
