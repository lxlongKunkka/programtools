## 趣味引入

同学们，想象一下，你是一名勇敢的探险家，终于找到了一座传说中的宝藏洞穴！洞穴里有各种各样的宝物，金币、宝石、魔法卷轴……但是，你的背包是有限的！它只能装下一定重量的物品。而且，每个宝物都是独一无二的，你只能选择拿走或者不拿走，不能拿走一半，也不能拿走多个相同的宝物。

这就是我们今天要学习的经典问题——“0-1背包问题”！为什么叫“0-1”呢？因为对于每个宝物，你只有两种选择：0代表不拿，1代表拿走。是不是很有趣？今天我们就来一起解开这个“宝藏之谜”！

## 深度知识点讲解

### 1. 什么是0-1背包问题？

**核心概念：**
我们有 `n` 件物品和一个容量为 `m` 的背包。
每件物品 `i` 都有一个重量 `w[i]` 和一个价值 `v[i]`。
每个物品只能选择一次（拿走或不拿走）。
目标是：在不超过背包总容量的前提下，使得背包中物品的总价值最大。

**生活类比：**
*   **超市购物：** 你有一个购物袋（背包），容量有限。货架上有很多商品（物品），每个商品有重量和价格。你只能买一个商品，不能买半个，也不能买多个相同的。如何在容量限制内，买到总价值最高的商品？
*   **考试复习：** 你有有限的复习时间（背包容量）。有很多知识点（物品），每个知识点有掌握难度（重量）和考试分值（价值）。每个知识点你只能选择复习一次或不复习。如何在有限时间内，复习到总分最高的知识点？

**为什么不能用贪心？**
有些同学可能会想：“这还不简单？每次都选单位重量价值最高的物品不就行了吗？” 这就是“贪心算法”的思想。我们来举个例子：

假设背包容量 `m = 10`。
物品A：重量 `w=6`，价值 `v=10` （单位价值约1.67）
物品B：重量 `w=4`，价值 `v=7` （单位价值约1.75）
物品C：重量 `w=4`，价值 `v=6` （单位价值约1.5）

如果用贪心：
1.  计算单位价值：B(1.75) > A(1.67) > C(1.5)。
2.  优先选B：背包剩余容量 `10 - 4 = 6`，总价值 `7`。
3.  接下来选A：背包剩余容量 `6 - 6 = 0`，总价值 `7 + 10 = 17`。
4.  不能再选C了。
贪心结果：总价值 `17`。

但是，如果我们不选B，而是选择A和C呢？
物品A(w=6, v=10) + 物品C(w=4, v=6)。
总重量 `6 + 4 = 10`，刚好装满。
总价值 `10 + 6 = 16`。

再换一种：物品B(w=4, v=7) + 物品C(w=4, v=6). 总重量 8, 总价值 13.

咦，在这个例子中，贪心算法得到的 `17` 似乎是正确的。
那我们换一个例子：
背包容量 `m = 10`。
物品A：重量 `w=3`，价值 `v=6` （单位价值 2）
物品B：重量 `w=4`，价值 `v=7` （单位价值 1.75）
物品C：重量 `w=5`，价值 `v=8` （单位价值 1.6）

如果用贪心：
1.  单位价值：A(2) > B(1.75) > C(1.6)。
2.  优先选A：背包剩余容量 `10 - 3 = 7`，总价值 `6`。
3.  接下来选B：背包剩余容量 `7 - 4 = 3`，总价值 `6 + 7 = 13`。
4.  不能再选C了。
贪心结果：总价值 `13`。

但是，最优解是选择B和C：
物品B(w=4, v=7) + 物品C(w=5, v=8)。
总重量 `4 + 5 = 9`，不超过容量10。
总价值 `7 + 8 = 15`。

发现了吗？贪心算法得到的 `13` 并不是最优解！这说明贪心算法在0-1背包问题中是行不通的。因为局部最优不一定能导致全局最优。我们需要一种更强大的方法——动态规划！

### 2. 动态规划思想

动态规划的核心思想是：把一个大问题拆分成若干个小问题，通过解决小问题来解决大问题。并且，这些小问题之间存在重叠子问题，我们可以通过存储小问题的解来避免重复计算。

**什么是“重叠子问题”和“最优子结构”？**
*   **最优子结构：** 如果问题的最优解包含其子问题的最优解，就称该问题具有最优子结构。在0-1背包中，如果容量为 `j` 的背包，装前 `i` 个物品的最优方案是X，那么方案X中装前 `i-1` 个物品的子方案，也一定是容量为 `j` 或 `j - w[i]` 的背包，装前 `i-1` 个物品的最优方案。
*   **重叠子问题：** 解决问题的过程中，会反复计算相同的子问题。通过DP表（数组）存储结果，可以避免重复计算。

### 3. DP状态定义 (二维数组)

我们用一个二维数组 `dp[i][j]` 来表示：
`dp[i][j]` 表示：**考虑前 `i` 件物品，当前背包容量为 `j` 时，所能获得的最大价值。**

*   `i` 的范围：`0` 到 `n` (物品数量)
*   `j` 的范围：`0` 到 `m` (背包容量)

**为什么是“前 `i` 件物品”？**
这是一种思考的顺序。我们可以一件一件地考虑物品，每考虑一件物品，都更新所有可能的背包容量下的最大价值。

### 4. 状态转移方程 (二维数组)

对于第 `i` 件物品，我们有两种选择：

1.  **不选择第 `i` 件物品：**
    如果我不拿第 `i` 件物品，那么我能得到的最大价值，就和“考虑前 `i-1` 件物品，背包容量仍然是 `j`”时的最大价值一样。
    所以，这种情况下，价值是 `dp[i-1][j]`。

2.  **选择第 `i` 件物品：**
    前提是：当前背包容量 `j` 必须大于或等于第 `i` 件物品的重量 `w[i]`。
    如果我拿了第 `i` 件物品，那么：
    *   背包容量会减少 `w[i]`，变成 `j - w[i]`。
    *   我得到的总价值会增加 `v[i]`。
    *   那么，我还需要在“考虑前 `i-1` 件物品，背包容量为 `j - w[i]`”时，获得的最大价值。
    所以，这种情况下，价值是 `dp[i-1][j - w[i]] + v[i]`。

综合这两种情况，`dp[i][j]` 应该是这两种选择中能获得的最大价值。

**状态转移方程：**

*   **如果当前背包容量 `j < w[i]`：**
    背包容量不够装第 `i` 件物品，所以只能不选择它。
    `dp[i][j] = dp[i-1][j]`

*   **如果当前背包容量 `j >= w[i]`：**
    背包容量足够装第 `i` 件物品，我们可以选择拿或不拿。
    `dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])`

**图解：**
想象一个表格，行代表物品，列代表容量。
`dp[i][j]` 的值依赖于 `dp[i-1][j]` (上一行同列) 和 `dp[i-1][j - w[i]]` (上一行左边某列)。
当前格子的值，只依赖于它上面一行的值。

### 5. 初始化

*   `dp[0][j] = 0` (当没有物品可以选择时，无论背包容量多大，最大价值都是0)。
*   `dp[i][0] = 0` (当背包容量为0时，无论有多少物品，都装不下，最大价值都是0)。

通常，我们会把整个 `dp` 数组初始化为0，这样 `dp[0][j]` 和 `dp[i][0]` 就自动满足条件了。

### 6. 空间优化 (一维数组)

观察状态转移方程 `dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])`。
我们发现，计算 `dp[i][j]` 的值，只依赖于 `dp[i-1]` 这一行的值。
这意味着，我们不需要存储所有的 `i` 行，只需要存储上一行的值即可。

我们可以将二维数组 `dp[i][j]` 优化为一维数组 `dp[j]`。
`dp[j]` 现在表示：**当前背包容量为 `j` 时，所能获得的最大价值。**

**核心问题：如何确保 `dp[j - w[i]]` 引用的是“上一行”的值？**

如果我们像这样写：
`for i from 1 to n:`
  `for j from 0 to m:`
    `dp[j] = max(dp[j], dp[j - w[i]] + v[i])` (这是错的！)

问题出在哪里？当 `j` 从小到大遍历时，`dp[j - w[i]]` 可能已经被当前第 `i` 件物品更新过了。
例如，计算 `dp[5]` 时，如果 `w[i]=2`，`dp[3]` 已经被更新成了包含第 `i` 件物品的价值。但我们希望 `dp[3]` 是不包含第 `i` 件物品的价值（即 `dp[i-1][3]`）。这就相当于把第 `i` 件物品放了两次！这是0-1背包不允许的。

**解决方案：倒序遍历容量 `j`！**

`for i from 1 to n:`
  `for j from m down to w[i]:` (注意：`j` 必须从 `m` 倒序到 `w[i]`)
    `dp[j] = max(dp[j], dp[j - w[i]] + v[i])`

**原理：**
当 `j` 从 `m` 倒序遍历到 `w[i]` 时：
*   `dp[j]` 存储的是 `dp[i-1][j]` 的值（因为还没有被第 `i` 件物品更新）。
*   `dp[j - w[i]]` 存储的也是 `dp[i-1][j - w[i]]` 的值（因为 `j - w[i]` 比 `j` 小，它在当前循环中还没被更新到）。
这样，`dp[j]` 在更新时，引用的都是“上一轮”（即 `i-1` 物品）的状态值，从而保证了每件物品只被选择一次。

**初始化 (一维数组)：**
*   `dp` 数组全部初始化为 `0`。
*   `dp[0]` 自动为 `0` (背包容量为0时，价值为0)。

**最终答案：**
无论使用二维DP还是优化后的一维DP，最终的答案都是 `dp[n][m]` (二维) 或 `dp[m]` (一维)。

### 7. 复杂度分析

*   **时间复杂度：**
    *   外层循环 `i` 遍历 `n` 件物品。
    *   内层循环 `j` 遍历 `m` 种容量。
    *   所以，时间复杂度为 `O(n * m)`。

*   **空间复杂度：**
    *   二维DP：需要一个 `(n+1) * (m+1)` 大小的数组，空间复杂度为 `O(n * m)`。
    *   一维DP优化：需要一个 `(m+1)` 大小的数组，空间复杂度为 `O(m)`。
    *   由于 GESP 6 级别的题目中，`m` 通常不会特别大（例如 `m` 在 10000 到 100000 之间），`O(m)` 的空间复杂度是完全可以接受的。

**关键点总结：**
1.  **0-1背包：** 每个物品只能选一次。
2.  **DP状态：** `dp[i][j]` (前 `i` 物品，容量 `j` 的最大价值)。
3.  **转移方程：** `max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])`。
4.  **初始化：** 全0。
5.  **空间优化：** 一维 `dp[j]`，容量 `j` **倒序**遍历。

## 典型例题精讲

### 例题1：基础0-1背包 (二维DP)

**题目描述：**
给定 `n` 件物品，每件物品有重量 `w[i]` 和价值 `v[i]`。有一个背包，最大容量为 `m`。每件物品只能选择一次。求在不超过背包容量的前提下，能获得的最大总价值。

**输入格式：**
第一行两个整数 `n` 和 `m`，分别表示物品数量和背包容量。
接下来 `n` 行，每行两个整数 `w[i]` 和 `v[i]`，分别表示第 `i` 件物品的重量和价值。

**输出格式：**
一个整数，表示最大总价值。

**样例输入：**
```
3 10
3 6
4 7
5 8
```

**样例输出：**
```
15
```

**思路分析：**

1.  **定义DP状态：** `dp[i][j]` 表示考虑前 `i` 个物品，背包容量为 `j` 时能获得的最大价值。
2.  **初始化：** 将 `dp` 数组所有元素初始化为 `0`。这自动处理了 `dp[0][j]` (没有物品可选) 和 `dp[i][0]` (背包容量为0) 的情况。
3.  **循环填充DP表：**
    *   外层循环 `i` 从 `1` 到 `n` (遍历物品)。
    *   内层循环 `j` 从 `1` 到 `m` (遍历容量)。
    *   在循环体内部，根据当前物品 `i` 的重量 `w[i]` 和价值 `v[i]`，以及当前背包容量 `j`，应用状态转移方程。
        *   如果 `j < w[i]` (容量不够装第 `i` 件物品)：`dp[i][j] = dp[i-1][j]`。
        *   如果 `j >= w[i]` (容量足够装第 `i` 件物品)：`dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])`。
4.  **最终结果：** `dp[n][m]` 就是问题的答案。

**代码实现 (2D DP)：**

```cpp
#include <iostream> // 包含输入输出流库
#include <algorithm> // 包含max函数

int w[105]; // 存储物品重量，假设最多100个物品
int v[105]; // 存储物品价值
int dp[105][1005]; // dp[i][j] 表示前i个物品，背包容量j的最大价值
                   // 物品数量N最大100，容量M最大1000

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出速度
    std::cin.tie(NULL); // 解除cin与cout的绑定

    int n, m; // n: 物品数量, m: 背包容量
    std::cin >> n >> m;

    for (int i = 1; i <= n; ++i) { // 物品编号从1开始
        std::cin >> w[i] >> v[i];
    }

    // 初始化dp数组为0，这里可以省略，因为全局数组默认初始化为0
    // for (int i = 0; i <= n; ++i) {
    //     for (int j = 0; j <= m; ++j) {
    //         dp[i][j] = 0;
    //     }
    // }

    // 动态规划填充dp表
    for (int i = 1; i <= n; ++i) { // 遍历每一件物品
        for (int j = 0; j <= m; ++j) { // 遍历每一种背包容量
            // 情况1: 不选择第i件物品
            dp[i][j] = dp[i - 1][j];

            // 情况2: 选择第i件物品 (如果容量足够)
            if (j >= w[i]) {
                dp[i][j] = std::max(dp[i][j], dp[i - 1][j - w[i]] + v[i]);
            }
        }
    }

    std::cout << dp[n][m] << std::endl; // 输出最终结果

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：两层循环，`n` 乘以 `m`，所以是 `O(n * m)`。
*   空间复杂度：一个 `(n+1) * (m+1)` 的二维数组，所以是 `O(n * m)`。

### 例题2：0-1背包 (一维DP优化)

**题目描述：**
同例题1，但要求使用空间优化，将二维DP优化为一维DP。

**输入格式：**
同例题1。

**输出格式：**
同例题1。

**样例输入：**
```
3 10
3 6
4 7
5 8
```

**样例输出：**
```
15
```

**思路分析：**

1.  **定义DP状态：** `dp[j]` 表示背包容量为 `j` 时能获得的最大价值。
2.  **初始化：** 将 `dp` 数组所有元素初始化为 `0`。
3.  **循环填充DP表：**
    *   外层循环 `i` 从 `1` 到 `n` (遍历物品)。
    *   内层循环 `j` 从 `m` **倒序**到 `w[i]` (遍历容量，注意倒序和起始点)。
    *   在循环体内部，应用状态转移方程：
        `dp[j] = std::max(dp[j], dp[j - w[i]] + v[i]);`
        这里 `dp[j]` (未更新前) 相当于二维DP中的 `dp[i-1][j]`。
        `dp[j - w[i]]` (未更新前) 相当于二维DP中的 `dp[i-1][j - w[i]]`。
        由于 `j` 是倒序遍历的，`dp[j - w[i]]` 一定是上一次外层循环 (即 `i-1` 物品) 的状态，所以不会出现重复使用第 `i` 件物品的问题。
4.  **最终结果：** `dp[m]` 就是问题的答案。

**代码实现 (1D DP):**

```cpp
#include <iostream> // 包含输入输出流库
#include <algorithm> // 包含max函数

int w[105]; // 存储物品重量，假设最多100个物品
int v[105]; // 存储物品价值
int dp[1005]; // dp[j] 表示背包容量j的最大价值
              // 容量M最大1000

int main() {
    std::ios_base::sync_with_stdio(false); // 优化输入输出速度
    std::cin.tie(NULL); // 解除cin与cout的绑定

    int n, m; // n: 物品数量, m: 背包容量
    std::cin >> n >> m;

    for (int i = 1; i <= n; ++i) { // 物品编号从1开始
        std::cin >> w[i] >> v[i];
    }

    // dp数组默认初始化为0，这里可以省略显式初始化
    // for (int j = 0; j <= m; ++j) {
    //     dp[j] = 0;
    // }

    // 动态规划填充dp数组
    for (int i = 1; i <= n; ++i) { // 遍历每一件物品
        // 关键点：容量j从m倒序遍历到w[i]
        // 确保dp[j - w[i]]是上一轮(i-1物品)的状态
        for (int j = m; j >= w[i]; --j) { // 遍历背包容量，从大到小
            // 状态转移方程：
            // dp[j] (不选当前物品i)
            // dp[j - w[i]] + v[i] (选当前物品i)
            dp[j] = std::max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    std::cout << dp[m] << std::endl; // 输出最终结果

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：两层循环，`n` 乘以 `m`，所以是 `O(n * m)`。
*   空间复杂度：一个 `(m+1)` 的一维数组，所以是 `O(m)`。

## 代码实现模板

下面提供一个标准的0-1背包问题（一维DP优化版）的C++代码模板，方便大家理解和快速应用。

```cpp
#include <iostream>  // 用于输入输出，如 std::cin, std::cout
#include <algorithm> // 用于 std::max 函数

// 定义全局数组来存储物品的重量和价值，以及DP数组
// 题目通常会给出N和M的范围，这里假设N最大100，M最大1000
const int MAX_N = 105;   // 物品数量的最大值 + 5，留一些余量
const int MAX_M = 1005;  // 背包容量的最大值 + 5，留一些余量

int w[MAX_N]; // 存储每个物品的重量 (weight)
int v[MAX_N]; // 存储每个物品的价值 (value)

// dp[j] 表示当背包容量为 j 时，所能获得的最大价值
// 使用一维数组进行空间优化
int dp[MAX_M];

int main() {
    // 优化C++标准库的输入输出流，使其更快
    // 对于大规模输入输出的竞赛题，这是一个好习惯
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 物品的数量
    int m; // 背包的最大容量

    // 读取物品数量 N 和背包容量 M
    std::cin >> n >> m;

    // 读取每个物品的重量和价值
    // 注意：物品的索引通常从1开始，这样可以和DP数组的i对应
    for (int i = 1; i <= n; ++i) {
        std::cin >> w[i] >> v[i];
    }

    // DP数组的初始化：
    // dp[j] 默认初始化为0，表示背包容量为 j 时，
    // 如果不放任何物品，总价值为0。
    // 对于0-1背包问题，通常将dp数组全部初始化为0即可。
    // 这里因为是全局数组，默认就是0，所以可以省略显式初始化。
    // for (int j = 0; j <= m; ++j) {
    //     dp[j] = 0;
    // }

    // 动态规划的核心循环
    // 外层循环：遍历每一件物品
    for (int i = 1; i <= n; ++i) {
        // 内层循环：遍历背包的容量
        // 关键点：j 必须从 m 倒序遍历到 w[i]
        // 这样做是为了确保在计算 dp[j] 时，dp[j - w[i]] 仍然是
        // 上一轮（即考虑前 i-1 件物品）的状态值，避免重复计算当前物品。
        for (int j = m; j >= w[i]; --j) {
            // 状态转移方程：
            // dp[j] 的当前值代表“不选择第 i 件物品”时的最大价值 (dp[i-1][j])
            // dp[j - w[i]] + v[i] 代表“选择第 i 件物品”时的最大价值 (dp[i-1][j - w[i]] + v[i])
            // 我们取这两种情况的最大值
            dp[j] = std::max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    // 最终结果：
    // dp[m] 存储的就是考虑了所有 N 件物品，背包容量为 M 时的最大价值
    std::cout << dp[m] << std::endl;

    return 0; // 程序成功执行
}
```

**1. 思考题 (个人思考，小组讨论)：**
*   **问题1：** 如果我们想求的不是最大价值，而是恰好装满背包时的最大价值，并且如果无法恰好装满则输出-1。那么我们的 `dp` 数组应该如何初始化？（提示：思考“不可能达到”的状态）

    小明要去参加一个野营活动，他有一个容量为 `20` 的背包。现在有以下物品可供选择：

    *   急救箱：重量 `3`，价值 `7`
    *   水壶：重量 `2`，价值 `4`
    请你编写程序，计算小明最多能带走多少价值的物品？

*   **要求：**

    2.  完成后举手示意老师检查。

---

**互动答案提示：**

*   **思考题1答案：**
    *   `dp[0]` 初始化为 `0` (背包容量为0时，价值为0)。
    *   `dp[1]` 到 `dp[m]` 都初始化为 `-1` (或一个足够小的负数，表示无法达到)。
    *   状态转移方程不变。
    *   最终结果如果为 `-1`，则表示无法恰好装满。

*   **思考题2答案：**
    *   **问题：** 如果 `j` 正序遍历，当计算 `dp[j]` 时，`dp[j - w[i]]` 已经可能被第 `i` 件物品更新过了。
    *   **例子：** 背包容量 `m=5`，物品 `i`：`w[i]=2, v[i]=3`。
        *   假设当前 `dp` 数组是考虑前 `i-1` 件物品后的状态，例如 `dp[0]=0, dp[1]=0, dp[2]=0, dp[3]=0, dp[4]=0, dp[5]=0`。
        *   `for j from 0 to m:`
            *   `j=2`: `dp[2] = max(dp[2], dp[2-2] + v[i]) = max(0, dp[0] + 3) = 3`。此时 `dp[2]` 包含了物品 `i`。
            *   `j=4`: `dp[4] = max(dp[4], dp[4-2] + v[i]) = max(0, dp[2] + 3)`。这里的 `dp[2]` 已经包含了物品 `i`。所以 `dp[4]` 的计算变成了 `max(0, 3 + 3) = 6`。
            *   这相当于在容量为4的背包中，我们用了一次物品 `i` (得到容量2的 `dp[2]` )，然后又在容量为2的剩余空间中，再次用了物品 `i` (得到 `dp[2-2]` )，总共用了两次物品 `i`，这是0-1背包不允许的。
    *   **结论：** 倒序遍历 `j` 保证了 `dp[j - w[i]]` 始终是“上一轮” (即 `i-1` 物品) 的状态。

*   **思考题3答案：**
    *   **共同点：** 都是通过已知的小问题（或前一步的状态）来推导出当前问题（或下一步的状态）。都关注状态的定义和状态之间的关系。
    *   **不同点：** 递推通常是线性的，或者简单多维的（比如斐波那契数列，阶乘）。动态规划更强调“选择”和“最优”，通常会涉及到 `max/min` 操作，并且子问题之间可能存在更复杂的依赖关系（例如背包问题中，是否选择当前物品取决于容量是否足够）。动态规划是递推的一种更高级、更强大的应用。

#include <iostream>
#include <algorithm>

const int MAX_N = 105;
const int MAX_M = 10005;

int coin[MAX_N];
int dp[MAX_M];

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);
