## 深度知识点讲解

### 1. 多重背包问题 (Multiple Knapsack Problem)

#### 1.1 问题定义

多重背包问题是0/1背包问题和完全背包问题的“混血儿”。
*   **物品种类:** `n` 种。
*   **背包容量:** `m`。
*   **每种物品的属性:**
    *   `w[i]`: 第 `i` 种物品的重量。
    *   `v[i]`: 第 `i` 种物品的价值。
    *   `s[i]`: 第 `i` 种物品的数量（这是关键，它不是1，也不是无限，而是有限的正整数）。

**目标:** 在不超过背包容量的前提下，选择物品使得总价值最大。

#### 1.2 朴素解法：暴力枚举 (Naive Approach)

最直观的想法是：对于第 `i` 种物品，我可以拿 `0` 个、`1` 个、`2` 个... 直到 `s[i]` 个。

我们可以把这个选择过程融入到我们熟悉的0/1背包的DP方程中。
设 `f[j]` 表示背包容量为 `j` 时能获得的最大价值。

**DP状态转移方程:**
`f[j] = max(f[j], f[j - k * w[i]] + k * v[i])`
其中，`k` 是我们选择第 `i` 种物品的数量，`k` 的范围是 `0 <= k <= s[i]`，并且 `k * w[i] <= j`。

**代码实现思路:**

```cpp
// 假设 f 数组已经初始化为0
// n 种物品，m 是背包容量
// w[i], v[i], s[i] 分别是第 i 种物品的重量、价值和数量

for (int i = 0; i < n; ++i) { // 遍历每种物品
    for (int j = m; j >= 0; --j) { // 遍历背包容量，倒序确保每个物品只用一次（针对当前层物品）
        for (int k = 1; k <= s[i]; ++k) { // 尝试拿 k 个第 i 种物品
            if (j >= k * w[i]) { // 如果当前容量足够装 k 个物品
                f[j] = max(f[j], f[j - k * w[i]] + k * v[i]);
            } else { // 容量不够，再多拿就装不下了
                break;
            }
        }
    }
}
```

**为什么 `j` 循环是倒序？**
这里 `f[j - k * w[i]]` 应该使用**上一轮（即第 `i-1` 种物品处理完后）**的 `f` 值。如果 `j` 是正序，`f[j - k * w[i]]` 可能会用到**本轮（即第 `i` 种物品处理中）**更新过的 `f` 值，这相当于允许我们重复选择第 `i` 种物品，变成了完全背包。而我们这里的 `k` 循环已经表示了拿 `k` 个物品，每个物品种类 `i` 只能处理一次，所以 `j` 必须倒序，以确保 `f[j - k * w[i]]` 对应的是未选择第 `i` 种物品之前的状态。

**时间复杂度分析:**
外层 `i` 循环 `n` 次。
中层 `j` 循环 `m` 次。
内层 `k` 循环最坏情况下 `s[i]` 次。
总时间复杂度：`O(n * m * max(s[i]))`。
如果 `s[i]` 很大（比如 `m / w[i]`），这个复杂度可能达到 `O(n * m^2)`，这对于 `m=10^5` 的数据范围来说是无法接受的。我们需要优化！

#### 1.3 优化：二进制优化 (Binary Optimization)

**核心思想:**
多重背包问题之所以慢，是因为我们用 `k` 循环一个一个地尝试拿物品。有没有办法把这 `s[i]` 个物品打包成更少的“等价”物品，然后用0/1背包的方法来解决呢？

想象一下，你有 `s` 张面值相同的优惠券。你可以选择拿 1 张、2 张、3 张...直到 `s` 张。
我们知道，任何一个整数都可以表示成若干个2的幂次之和（二进制表示）。例如：
*   拿 1 张：`1`
*   拿 2 张：`2`
*   拿 3 张：`1 + 2`
*   拿 4 张：`4`
*   拿 5 张：`1 + 4`
*   拿 6 张：`2 + 4`
*   拿 7 张：`1 + 2 + 4`

所以，我们可以把 `s` 个物品拆分成若干个新的“物品组”，每个物品组的数量是 `2^0, 2^1, 2^2, ..., 2^k`，直到这些 `2^p` 的和接近或等于 `s`。然后，把剩下的数量打包成最后一个物品组。

**拆分过程举例:**
假设有 `s = 13` 个物品，每个重 `w`，价值 `v`。
1.  **第一个包:** 数量 `2^0 = 1`。这个包的重量是 `1 * w`，价值是 `1 * v`。
    剩余数量：`13 - 1 = 12`。
2.  **第二个包:** 数量 `2^1 = 2`。这个包的重量是 `2 * w`，价值是 `2 * v`。
    剩余数量：`12 - 2 = 10`。
3.  **第三个包:** 数量 `2^2 = 4`。这个包的重量是 `4 * w`，价值是 `4 * v`。
    剩余数量：`10 - 4 = 6`。
4.  **第四个包:** 数量 `2^3 = 8`。哦，`8 > 6` 了，不能再用 `2^k` 了。
    所以，剩下的 `6` 个物品打包成最后一个包。
    这个包的重量是 `6 * w`，价值是 `6 * v`。
    剩余数量：`6 - 6 = 0`。

现在，我们把 `13` 个物品拆成了 `1, 2, 4, 6` 这四个“新物品”。
这四个新物品，每个都只能选或不选（0/1背包）。
那么，我们能否用这四个新物品来表示拿 `1` 到 `13` 之间的任意数量的原物品呢？
*   拿 1 个：选 `1`
*   拿 2 个：选 `2`
*   拿 3 个：选 `1, 2`
*   拿 4 个：选 `4`
*   拿 5 个：选 `1, 4`
*   ...
*   拿 7 个：选 `1, 2, 4`
*   拿 8 个：选 `2, 6` (或者 `1, ?`)
*   拿 9 个：选 `1, 2, 6`
*   拿 13 个：选 `1, 2, 4, 6`

是的，通过组合这些 `1, 2, 4, ...` 以及最后一个“余数”的包，我们可以凑出 `1` 到 `s` 之间的任意数量！
**证明:** `1, 2, 4, ..., 2^(k-1)` 这些数的和是 `2^k - 1`。如果我们再有一个 `2^k` 的包，就可以凑出 `1` 到 `2^(k+1)-1` 之间的所有数。我们最后一个包是 `s - (2^k - 1)`，只要这个余数包的重量不超过 `2^k - 1 + 1 = 2^k`，就可以凑出 `1` 到 `s` 之间的所有数。实际上，只要 `s` 大于等于 `2^k - 1`，这个方法就能表示 `1` 到 `s` 之间的所有数。

**拆分后的物品数量:**
对于 `s` 个物品，拆分出的新物品数量大约是 `log2(s)` 个。
例如，`s = 10^5`，`log2(10^5)` 大约是 `17`。
所以，原本 `n` 种物品，现在会变成 `n * log2(max(s[i]))` 种新物品。
新的总物品数量 `N'` 大约是 `n * log2(s)`。
这样，多重背包问题就转化成了 `N'` 个物品的0/1背包问题。

**时间复杂度分析:**
总物品数量 `N'` 约为 `n * log(max(s[i]))`。
转化后，使用0/1背包的DP，时间复杂度为 `O(N' * m)`，即 `O(n * m * log(max(s[i])))`。
这比朴素解法的 `O(n * m * max(s[i]))` 效率大大提高！

**代码实现思路 (二进制优化):**

1.  创建一个新的物品列表 `new_w`, `new_v`。
2.  遍历原始的每种物品 `i` (重量 `w[i]`, 价值 `v[i]`, 数量 `s[i]`)。
3.  对于每种物品 `i`，进行二进制拆分：
    *   设 `k = 1`。
    *   当 `s[i] > 0` 时：
        *   `cnt = min(k, s[i])` (当前包的数量)。
        *   将一个重量为 `cnt * w[i]`，价值为 `cnt * v[i]` 的新物品添加到 `new_w`, `new_v` 列表中。
        *   `s[i] -= cnt`。
        *   `k *= 2`。
4.  所有原始物品拆分完毕后，`new_w`, `new_v` 中包含了所有可以用于0/1背包的“新物品”。
5.  对 `new_w`, `new_v` 中的所有新物品执行标准的0/1背包DP。

```cpp
// 假设 MAX_ITEMS 是所有拆分后物品的最大数量， MAX_M 是背包容量
int W_prime[MAX_ITEMS], V_prime[MAX_ITEMS]; // 存储拆分后的物品的重量和价值
int item_count_prime = 0; // 拆分后物品的总数

// 原始物品数据 (w_orig, v_orig, s_orig)
// n_orig 是原始物品种类数

for (int i = 0; i < n_orig; ++i) {
    int current_w = w_orig[i];
    int current_v = v_orig[i];
    int current_s = s_orig[i];

    for (int k = 1; current_s > 0; k *= 2) {
        int cnt = min(k, current_s);
        W_prime[item_count_prime] = cnt * current_w;
        V_prime[item_count_prime] = cnt * current_v;
        item_count_prime++;
        current_s -= cnt;
    }
}

// 现在 W_prime 和 V_prime 包含了所有可以进行0/1背包处理的物品
// f[j] 表示容量为 j 时的最大价值
int f[MAX_M + 1]; // 注意数组大小
for (int j = 0; j <= m; ++j) {
    f[j] = 0;
}

for (int i = 0; i < item_count_prime; ++i) { // 遍历所有新的0/1物品
    int cw = W_prime[i]; // 当前物品重量
    int cv = V_prime[i]; // 当前物品价值
    for (int j = m; j >= cw; --j) { // 0/1背包，j 倒序
        f[j] = max(f[j], f[j - cw] + cv);
    }
}
// 最终答案是 f[m]
```

### 2. 混合背包问题 (Mixed Knapsack Problem)

#### 2.1 问题定义

混合背包问题是指在同一个背包问题中，同时存在三种类型的物品：
1.  **0/1背包物品:** 每种物品只能选择 0 个或 1 个。
2.  **完全背包物品:** 每种物品可以选择 0 个、1 个、2 个... 直到装不下。
3.  **多重背包物品:** 每种物品可以选择 0 个、1 个、2 个... 直到 `s[i]` 个。

**目标:** 在不超过背包容量的前提下，选择物品使得总价值最大。

#### 2.2 处理策略

最简洁的处理方式是：**将所有类型的物品都转化为0/1背包物品，然后统一使用0/1背包的DP算法。**

1.  **0/1背包物品:** 保持不变，直接加入到新的物品列表中。
2.  **完全背包物品:**
    *   理论上可以拿无限个，但实际上受限于背包容量 `m`。
    *   对于重量为 `w` 的完全背包物品，最多能拿 `m / w` 个。
    *   所以，我们可以把它看作是一个数量为 `m / w` 的多重背包物品。
    *   然后，对它进行二进制优化拆分，再加入到新的物品列表中。
    *   *简化处理：* 也可以直接使用完全背包的DP循环（`j` 正序），但为了统一，二进制优化是更通用的方法。

3.  **多重背包物品:**
    *   直接使用上面讲过的二进制优化方法，将其拆分成若干个新的0/1背包物品，加入到新的物品列表中。

**统一转化后的步骤:**

1.  **预处理:** 遍历所有原始物品。
    *   如果是0/1物品，直接将其 `(w[i], v[i])` 加入到新的物品列表。
    *   如果是完全物品，将其视为数量为 `m / w[i]` 的多重物品，然后进行二进制拆分，将拆分后的 `(w', v')` 加入到新的物品列表。
    *   如果是多重物品，直接进行二进制拆分，将拆分后的 `(w', v')` 加入到新的物品列表。
2.  **DP计算:** 对新的物品列表中的所有物品执行标准的0/1背包DP。

**优点:** 这种方法将三种不同类型的物品统一处理成了一种类型，简化了DP主循环的逻辑，避免了在同一个 `j` 循环中混合正序和倒序迭代的复杂性。

**代码实现思路 (混合背包):**

```cpp
// 假设 MAX_ITEMS_TOTAL 是所有拆分后物品的最大总数
// MAX_M 是背包容量
int final_W[MAX_ITEMS_TOTAL], final_V[MAX_ITEMS_TOTAL]; // 存储所有转化为0/1的物品
int final_item_count = 0; // 转化后物品的总数

// 原始物品数据 (w_orig, v_orig, s_orig, type_orig)
// type_orig: 0表示0/1背包，1表示完全背包，2表示多重背包
// n_orig 是原始物品种类数
// m 是背包容量

for (int i = 0; i < n_orig; ++i) {
    int current_w = w_orig[i];
    int current_v = v_orig[i];
    int current_s = s_orig[i]; // 对于0/1和完全背包，这个值需要特殊处理

    if (type_orig[i] == 0) { // 0/1背包物品
        final_W[final_item_count] = current_w;
        final_V[final_item_count] = current_v;
        final_item_count++;
    } else if (type_orig[i] == 1) { // 完全背包物品
        // 转化为多重背包，数量上限为 m / current_w
        int count_limit = m / current_w;
        for (int k = 1; count_limit > 0; k *= 2) {
            int cnt = min(k, count_limit);
            final_W[final_item_count] = cnt * current_w;
            final_V[final_item_count] = cnt * current_v;
            final_item_count++;
            count_limit -= cnt;
        }
    } else { // 多重背包物品 (type_orig[i] == 2)
        // 直接进行二进制优化
        for (int k = 1; current_s > 0; k *= 2) {
            int cnt = min(k, current_s);
            final_W[final_item_count] = cnt * current_w;
            final_V[final_item_count] = cnt * current_v;
            final_item_count++;
            current_s -= cnt;
        }
    }
}

// 现在 final_W 和 final_V 包含了所有可以进行0/1背包处理的物品
// f[j] 表示容量为 j 时的最大价值
int f[MAX_M + 1];
for (int j = 0; j <= m; ++j) {
    f[j] = 0;
}

for (int i = 0; i < final_item_count; ++i) { // 遍历所有最终的0/1物品
    int cw = final_W[i]; // 当前物品重量
    int cv = final_V[i]; // 当前物品价值
    for (int j = m; j >= cw; --j) { // 标准0/1背包，j 倒序
        f[j] = max(f[j], f[j - cw] + cv);
    }
}
// 最终答案是 f[m]
```

**常见误区:**
*   **多重背包朴素解法中 `j` 循环使用正序:** 会导致物品被重复选择，变成完全背包。
*   **二进制优化时 `k` 递增过快导致 `s` 剩余不足:** 记得 `cnt = min(k, current_s)`，确保打包数量不超过剩余数量。
*   **混合背包中完全背包物品的 `s` 怎么定:** 可以设为 `m / w[i]`，这样二进制拆分是安全的。

## 典型例题精讲

### 例题1：多重背包问题（二进制优化）

**题目描述:**
你有 `n` 种物品，每种物品有自己的重量 `w[i]`、价值 `v[i]` 和数量 `s[i]`。你的背包容量为 `m`。请问在不超过背包容量的前提下，你能获得的最大总价值是多少？

**输入格式:**
第一行两个整数 `n, m` (物品种类数，背包容量)。
接下来 `n` 行，每行三个整数 `w[i], v[i], s[i]` (物品重量，物品价值，物品数量)。

**输出格式:**
一个整数，表示最大总价值。

**数据范围:**
`1 <= n <= 1000`
`1 <= m <= 2000`
`1 <= w[i], v[i], s[i] <= 2000`

**思路分析:**
这道题是典型的多重背包问题。由于 `s[i]` 的范围是 `2000`，如果使用朴素的三重循环 `O(n * m * max(s[i]))`，时间复杂度将达到 `1000 * 2000 * 2000 = 4 * 10^9`，会超时。因此，我们必须使用二进制优化。

1.  **物品转化:**
    *   创建一个临时的物品列表 `temp_w`, `temp_v`。
    *   遍历输入的 `n` 种物品。对于每种物品 `(w, v, s)`：
        *   用 `k = 1` 开始拆分。
        *   当 `s > 0` 时，计算当前包的数量 `cnt = min(k, s)`。
        *   将 `(cnt * w, cnt * v)` 加入 `temp_w`, `temp_v` 列表。
        *   `s -= cnt`。
        *   `k *= 2`。
    *   这样，所有的多重物品都被转化成了若干个0/1物品。

2.  **0/1背包DP:**
    *   创建一个 `dp` 数组，`dp[j]` 表示容量为 `j` 时的最大价值，初始化为0。
    *   遍历 `temp_w`, `temp_v` 中的每一个转化后的0/1物品 `(cw, cv)`。
    *   对于每个物品，从 `j = m` 倒序遍历到 `cw`：
        *   `dp[j] = max(dp[j], dp[j - cw] + cv)`。

3.  **结果:** `dp[m]` 就是最大总价值。

**C++代码实现:**

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// GESP 6级要求，不能用vector，用原生数组
const int MAX_N_ORIG = 1005; // 原始物品种类数最大值
const int MAX_M = 2005;      // 背包容量最大值
// 二进制优化后，物品总数会增加。
// 假设 s[i] 最大 2000，log2(2000) 约等于 11。
// 所以总物品数大约是 1000 * 11 = 11000。
const int MAX_ITEMS_OPTIMIZED = MAX_N_ORIG * 12; 

int f[MAX_M]; // dp数组，f[j] 表示容量为 j 时的最大价值

int W_prime[MAX_ITEMS_OPTIMIZED]; // 存储拆分后的物品的重量
int V_prime[MAX_ITEMS_OPTIMIZED]; // 存储拆分后的物品的价值
int item_count_prime = 0;          // 拆分后物品的总数

int main() {
    // 快速IO
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 原始物品种类数, m: 背包容量
    std::cin >> n >> m;

    for (int i = 0; i < n; ++i) {
        int w_orig, v_orig, s_orig; // 原始物品的重量、价值、数量
        std::cin >> w_orig >> v_orig >> s_orig;

        // 对当前物品进行二进制优化拆分
        for (int k = 1; s_orig > 0; k *= 2) {
            int cnt = std::min(k, s_orig); // 当前包的数量
            W_prime[item_count_prime] = cnt * w_orig;
            V_prime[item_count_prime] = cnt * v_orig;
            item_count_prime++;
            s_orig -= cnt;
        }
    }

    // 现在 W_prime 和 V_prime 包含了所有可以进行0/1背包处理的物品
    // f 数组已经全局初始化为0，符合要求

    // 执行0/1背包DP
    for (int i = 0; i < item_count_prime; ++i) { // 遍历所有新的0/1物品
        int cw = W_prime[i]; // 当前物品重量
        int cv = V_prime[i]; // 当前物品价值
        for (int j = m; j >= cw; --j) { // 0/1背包，j 倒序
            f[j] = std::max(f[j], f[j - cw] + cv);
        }
    }

    std::cout << f[m] << std::endl; // 输出最大价值

    return 0;
}
```

**时间复杂度:**
*   物品转化阶段：对于每种原始物品，`log(s[i])` 次循环。总共 `O(n * log(max_s))`。
*   0/1背包DP阶段：`O(item_count_prime * m)`，其中 `item_count_prime` 最大为 `n * log(max_s)`。
*   总时间复杂度：`O(n * m * log(max_s))`。
    *   对于 `n=1000, m=2000, max_s=2000`，`log2(2000)` 约为 `11`。
    *   `1000 * 2000 * 11 = 2.2 * 10^7`，这个复杂度是完全可以接受的。

### 例题2：混合背包问题

**题目描述:**
你有 `n` 种物品，背包容量为 `m`。
每种物品有三种类型之一：
1.  `type = 0`: 0/1背包物品，只能拿 0 或 1 个。
2.  `type = 1`: 完全背包物品，可以拿任意多个。
3.  `type = 2`: 多重背包物品，有数量限制 `s[i]`。

请问在不超过背包容量的前提下，你能获得的最大总价值是多少？

**输入格式:**
第一行两个整数 `n, m` (物品种类数，背包容量)。
接下来 `n` 行，每行四个整数 `w[i], v[i], s[i], type[i]`。
其中 `s[i]` 对于 `type=0` 和 `type=1` 的物品可以忽略（或者规定 `s[i]=1` 表示0/1，`s[i]=0` 表示完全）。为了统一，这里我们假设 `s[i]` 字段对于 `type=0` 的物品是 `1`，对于 `type=1` 的物品是 `0` (表示无限)，对于 `type=2` 的物品是实际数量。

**输出格式:**
一个整数，表示最大总价值。

**数据范围:**
`1 <= n <= 1000`
`1 <= m <= 2000`
`1 <= w[i], v[i] <= 2000`
`s[i]` (当 `type=2` 时) `1 <= s[i] <= 2000`
`type[i] ∈ {0, 1, 2}`

**思路分析:**
这是一道混合背包问题，需要根据物品类型进行不同的处理。最通用且简洁的方法是将所有物品都转化为0/1背包物品。

1.  **物品转化:**
    *   创建一个新的物品列表 `final_W`, `final_V`。
    *   遍历输入的 `n` 种物品：
        *   **如果 `type = 0` (0/1物品):** 直接将 `(w[i], v[i])` 加入 `final_W`, `final_V`。
        *   **如果 `type = 1` (完全物品):**
            *   将其视为数量为 `m / w[i]` 的多重物品（因为背包容量限制，最多只能拿 `m/w[i]` 个）。
            *   对这个虚拟的多重物品进行二进制优化拆分，将拆分后的 `(w', v')` 加入 `final_W`, `final_V`。
        *   **如果 `type = 2` (多重物品):**
            *   直接对 `(w[i], v[i], s[i])` 进行二进制优化拆分，将拆分后的 `(w', v')` 加入 `final_W`, `final_V`。

2.  **0/1背包DP:**
    *   创建一个 `dp` 数组，`dp[j]` 表示容量为 `j` 时的最大价值，初始化为0。
    *   遍历 `final_W`, `final_V` 中的每一个转化后的0/1物品 `(cw, cv)`。
    *   对于每个物品，从 `j = m` 倒序遍历到 `cw`：
        *   `dp[j] = max(dp[j], dp[j - cw] + cv)`。

3.  **结果:** `dp[m]` 就是最大总价值。

**C++代码实现:**

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// GESP 6级要求，不能用vector，用原生数组
const int MAX_N_ORIG = 1005; // 原始物品种类数最大值
const int MAX_M = 2005;      // 背包容量最大值
// 假设最坏情况：所有物品都是完全背包，且 w[i]=1。
// 那么每个物品会拆分出 log2(M) 个。
// 总物品数大约 n * log2(M) = 1000 * log2(2000) 约 1000 * 11 = 11000。
const int MAX_ITEMS_TOTAL_OPTIMIZED = MAX_N_ORIG * 12;

int f[MAX_M]; // dp数组，f[j] 表示容量为 j 时的最大价值

int final_W[MAX_ITEMS_TOTAL_OPTIMIZED]; // 存储所有转化为0/1的物品的重量
int final_V[MAX_ITEMS_TOTAL_OPTIMIZED]; // 存储所有转化为0/1的物品的价值
int final_item_count = 0;                // 转化后物品的总数

int main() {
    // 快速IO
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 原始物品种类数, m: 背包容量
    std::cin >> n >> m;

    for (int i = 0; i < n; ++i) {
        int w_orig, v_orig, s_orig, type_orig; // 原始物品的重量、价值、数量、类型
        // 题目描述中 s_orig 的含义：
        // type=0: s_orig 可以是1 (表示0/1)，或者任意值，我们会忽略
        // type=1: s_orig 可以是0 (表示无限)，或者任意值，我们会忽略
        // type=2: s_orig 是实际数量
        std::cin >> w_orig >> v_orig >> s_orig >> type_orig;

        if (type_orig == 0) { // 0/1背包物品
            final_W[final_item_count] = w_orig;
            final_V[final_item_count] = v_orig;
            final_item_count++;
        } else if (type_orig == 1) { // 完全背包物品
            // 转化为多重背包，数量上限为 m / current_w。
            // 确保 current_w 不为0，虽然题目一般会保证 w >= 1
            int count_limit = m / w_orig;
            if (w_orig == 0) { // 如果重量为0，可以无限拿，价值无限大，特判
                // 实际竞赛中，通常不会出现重量为0的物品，如果出现，需要根据题目要求处理
                // 比如直接输出一个极大值，或跳过
                continue; 
            }
            
            for (int k = 1; count_limit > 0; k *= 2) {
                int cnt = std::min(k, count_limit); // 当前包的数量
                final_W[final_item_count] = cnt * w_orig;
                final_V[final_item_count] = cnt * v_orig;
                final_item_count++;
                count_limit -= cnt;
            }
        } else { // 多重背包物品 (type_orig == 2)
            // 直接进行二进制优化
            for (int k = 1; s_orig > 0; k *= 2) {
                int cnt = std::min(k, s_orig); // 当前包的数量
                final_W[final_item_count] = cnt * w_orig;
                final_V[final_item_count] = cnt * v_orig;
                final_item_count++;
                s_orig -= cnt;
            }
        }
    }

    // 现在 final_W 和 final_V 包含了所有可以进行0/1背包处理的物品
    // f 数组已经全局初始化为0

    // 执行0/1背包DP
    for (int i = 0; i < final_item_count; ++i) { // 遍历所有最终的0/1物品
        int cw = final_W[i]; // 当前物品重量
        int cv = final_V[i]; // 当前物品价值
        for (int j = m; j >= cw; --j) { // 标准0/1背包，j 倒序
            f[j] = std::max(f[j], f[j - cw] + cv);
        }
    }

    std::cout << f[m] << std::endl; // 输出最大价值

    return 0;
}
```

**时间复杂度:**
*   物品转化阶段：对于每种原始物品，0/1物品是 `O(1)`，完全背包和多重背包是 `O(log(max_s))` 或 `O(log(m/min_w))`。总共 `O(n * log(max(s, m/min_w)))`。
*   0/1背包DP阶段：`O(final_item_count * m)`，其中 `final_item_count` 最大为 `n * log(max(s, m/min_w))`。
*   总时间复杂度：`O(n * m * log(max(s, m/min_w)))`。与多重背包类似，在数据范围内可接受。

### 多重背包 (二进制优化) 模板

```cpp
#include <bits/stdc++.h> // 包含常用头文件

// 定义常量，方便管理数组大小
// GESP 6级要求使用原生数组，所以需要提前定义最大尺寸
const int MAX_N_ORIG = 1005; // 原始物品种类数最大值
const int MAX_M = 2005;      // 背包容量最大值
// 经过二进制优化后，原始一个物品可能会拆分成 log(s) 个新的0/1物品
// 假设 s[i] 最大 2000，log2(2000) 约等于 11。
// 所以总物品数大约是 原始物品种类数 * log2(最大数量)
const int MAX_ITEMS_OPTIMIZED = MAX_N_ORIG * 12; // 1000 * 11 左右，取个余量

// DP数组：f[j] 表示背包容量为 j 时能获得的最大价值
// 数组大小为 MAX_M + 1，因为容量从 0 到 M
int f[MAX_M];

// 存储二进制优化后生成的0/1物品的重量和价值
int W_prime[MAX_ITEMS_OPTIMIZED];
int V_prime[MAX_ITEMS_OPTIMIZED];
int item_count_prime = 0; // 记录优化后物品的实际数量

int main() {
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 原始物品种类数, m: 背包容量
    std::cin >> n >> m;

    // 1. 物品预处理：将多重背包物品转化为0/1背包物品
    for (int i = 0; i < n; ++i) {
        int w_orig, v_orig, s_orig; // 当前原始物品的重量、价值、数量
        std::cin >> w_orig >> v_orig >> s_orig;

        // 对当前物品进行二进制优化拆分
        // k 从 1, 2, 4, 8... 递增，表示打包的数量
        for (int k = 1; s_orig > 0; k *= 2) {
            // cnt 是当前这个包实际包含的原始物品数量
            // 它不能超过 k (2的幂次)，也不能超过剩余的 s_orig
            int cnt = std::min(k, s_orig); 
            
            // 将这个“打包”物品的重量和价值存入新的列表
            W_prime[item_count_prime] = cnt * w_orig;
            V_prime[item_count_prime] = cnt * v_orig;
            item_count_prime++; // 新物品数量增加

            s_orig -= cnt; // 原始物品剩余数量减少
        }
    }

    // 2. DP计算：执行标准的0/1背包DP
    // f 数组已全局初始化为0，无需额外 memset
    
    // 遍历所有转化后的0/1物品
    for (int i = 0; i < item_count_prime; ++i) {
        int cw = W_prime[i]; // 当前0/1物品的重量
        int cv = V_prime[i]; // 当前0/1物品的价值

        // 0/1背包的内层循环，容量从大到小遍历
        // 确保每个物品只被选择一次
        for (int j = m; j >= cw; --j) {
            // 状态转移方程：
            // f[j] = max(不选当前物品, 选当前物品)
            // 不选：f[j] 保持不变
            // 选：f[j - cw] + cv (从容量 j-cw 的最大价值基础上加上当前物品价值)
            f[j] = std::max(f[j], f[j - cw] + cv);
        }
    }

    // 3. 输出结果
    std::cout << f[m] << std::endl; // 最终答案是背包容量为 m 时的最大价值

    return 0;
}
```
