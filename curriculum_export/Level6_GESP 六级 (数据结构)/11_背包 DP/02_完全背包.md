## 趣味引入

同学们，还记得我们上次一起去森林探险，为了减轻负担，我们只能从每种道具中选择一个，装进我们容量有限的背包吗？那个就是“0/1背包”问题，每种道具“要么选0个，要么选1个”。

今天，我们换个场景！想象你来到了一个神奇的“魔法糖果屋”！这里有 `N` 种不同口味的魔法糖果，每种糖果都有独特的“甜度值”和“体积”。最棒的是，**每种糖果你可以拿任意多个！** 只要你的小肚子（背包）容量装得下就行。

现在，你的小肚子容量是 `C`，你希望装满糖果后，获得的“总甜度”最大！你会怎么选择呢？

举个例子：
*   你的肚子容量 `C` 是 10。
*   有2种糖果：
    *   糖果A：体积 `w_A` = 2，甜度 `v_A` = 3
    *   糖果B：体积 `w_B` = 3，甜度 `v_B` = 4

如果你只选一次：
*   选5个糖果A：总体积 10，总甜度 15。
*   选3个糖果B：总体积 9，总甜度 12。
*   选2个糖果A和2个糖果B：总体积 4+6=10，总甜度 6+8=14。

是不是感觉有点复杂？这就是我们今天要学习的“完全背包”问题！它和0/1背包只有一点点不同，但这一点点不同，却让我们的解法也发生了一个非常关键的改变！

## 深度知识点讲解

### 1. 回顾0/1背包问题 (快速复习)

在0/1背包中，我们有 `N` 种物品，每种物品有重量 `w[i]` 和价值 `v[i]`。背包容量 `C`。每种物品只能选择一次（0个或1个）。目标是最大化总价值。

我们用 `f[j]` 表示容量为 `j` 时能获得的最大价值。
当考虑第 `i` 种物品时，状态转移方程是：
`f[j] = max(f[j], f[j - w[i]] + v[i])`

**核心点：** 为了确保每种物品只选一次，我们在更新 `f[j]` 时，内层循环 `j` 是**从大到小**（从 `C` 遍历到 `w[i]`）！
为什么？因为当 `j` 从大到小遍历时，计算 `f[j]` 所依赖的 `f[j - w[i]]` 还是**上一个物品**（或者说，不包含当前物品 `i`）时的状态值。这样就保证了物品 `i` 不会被重复计算。

### 2. 完全背包问题的核心变化：无限次选择

现在，来到完全背包。唯一的区别是：**每种物品可以选取任意多次！**

假设我们依然用 `f[j]` 表示容量为 `j` 时能获得的最大价值。
当考虑第 `i` 种物品时，我们仍然可以选择“不放第 `i` 种物品”或者“放第 `i` 种物品”。
*   如果“不放第 `i` 种物品”，那么最大价值就是 `f[j]`（不考虑当前物品 `i` 时的值）。
*   如果“放第 `i` 种物品”，那么我们需要消耗 `w[i]` 的容量，并获得 `v[i]` 的价值。剩下的容量是 `j - w[i]`。此时，我们去查找 `f[j - w[i]]`。

**关键来了！**
在0/1背包中，`f[j - w[i]]` 是指“在不放物品 `i` 的前提下，容量 `j - w[i]` 的最大价值”。
但在完全背包中，既然物品 `i` 可以无限次选择，那么当我们决定放一个物品 `i` 之后，剩下的容量 `j - w[i]` **仍然可以继续放物品 `i`**！

所以，`f[j - w[i]]` 应该代表“在**允许放物品 `i`** 的前提下，容量 `j - w[i]` 的最大价值”。
这意味着，当我们在计算 `f[j]` 的时候，`f[j - w[i]]` 应该是一个**已经考虑过物品 `i`** 的状态值。

### 3. 状态转移方程与内层循环方向

为了实现这个“允许继续放物品 `i`”的特性，我们只需要将内层循环 `j` 的方向改变一下：**从小到大**（从 `w[i]` 遍历到 `C`）！

**状态转移方程：**
`f[j] = max(f[j], f[j - w[i]] + v[i])`
这个方程看起来和0/1背包一样，但它的**含义**和**实现方式**因为循环方向而不同了。

**为什么 `j` 要从小到大？**
当 `j` 从 `w[i]` 遍历到 `C` 时：
*   当我们计算 `f[j]` 时，如果 `j - w[i]` >= 0，那么 `f[j - w[i]]` 这个值，**已经是在当前物品 `i` 的循环中被更新过的**（因为它比 `j` 小，所以它已经被计算过了）。
*   这意味着 `f[j - w[i]]` 可能已经包含了物品 `i` 的价值！
*   所以，`f[j - w[i]] + v[i]` 就表示：在容量 `j - w[i]` 已经放了若干个物品 `i` 的基础上，再加一个物品 `i`。这正是完全背包的精髓——物品 `i` 可以重复选择。

**生活类比：**
想象你有一个魔法冰箱，里面有无限多的同款蛋糕。
*   **0/1背包（反向循环 `j`）**：你每放一个蛋糕，就给冰箱贴个标签说“这个蛋糕已经放过了，下次不能再放了”。所以当你考虑放第 `i` 种蛋糕时，你只能依赖一个“没放过第 `i` 种蛋糕”的状态。
*   **完全背包（正向循环 `j`）**：你每放一个蛋糕，冰箱会自动补充一个一模一样的。所以当你考虑放第 `i` 种蛋糕时，你放了一个之后，剩下的空间你仍然可以放第 `i` 种蛋糕，因为冰箱里总是有无限个。正向循环 `j` 使得 `f[j - w[i]]` 已经“知道”可以放第 `i` 种蛋糕了。

### 4. 初始化

*   `f[0] = 0`：容量为0时，最大价值是0。
*   其他 `f[j]` ( `j > 0` )：通常初始化为0。这表示“如果不用任何物品，容量 `j` 的最大价值是0”。
    *   如果题目要求“背包必须被装满”，并且物品价值都是正数，那么除了 `f[0]=0` 之外，所有 `f[j]` ( `j > 0` ) 都应该初始化为负无穷大 (`-INF`)。这样，只有当 `f[j]` 能够通过有效路径（即装满）更新时，它才会有非负值。但对于GESP 6，一般默认初始化为0即可。

### 5. 复杂度分析

*   **时间复杂度：** 外层循环 `N` 次 (物品种类)，内层循环 `C` 次 (容量)。所以是 `O(N * C)`。
*   **空间复杂度：** 我们使用了一维数组 `f`，大小为 `C+1`。所以是 `O(C)`。

这与0/1背包的复杂度是相同的。

### 6. 常见误区

1.  **循环方向搞混：** 这是最常见的错误！0/1背包 `j` 倒序，完全背包 `j` 正序。切记！
2.  **初始化错误：** 如果题目要求“恰好装满”，但却初始化为0，可能导致错误结果（例如，`f[j]` 即使没有被任何物品组合装满，也显示为0）。

```cpp
// 伪代码框架
// N: 物品种类数, C: 背包容量
// w[i]: 第i种物品的重量, v[i]: 第i种物品的价值

// f[j] 表示容量为 j 时能获得的最大价值
int f[C + 1];

// 初始化：容量为0时，价值为0。其他容量初始为0。
// 如果要求“必须装满”，则 f[j > 0] 需初始化为负无穷。
for (int j = 0; j <= C; ++j) {
    f[j] = 0;
}

// 遍历每种物品
for (int i = 1; i <= N; ++i) { // 物品编号从1到N
    // 遍历背包容量，从小到大
    for (int j = w[i]; j <= C; ++j) { // 注意：j 从 w[i] 开始
        f[j] = max(f[j], f[j - w[i]] + v[i]);
    }
}

// 最终答案在 f[C]
```

## 典型例题精讲

### 例题一：魔法糖果屋

**题目描述：**
你来到了一个充满魔法的糖果屋。这里有 `N` 种不同类型的魔法糖果，每种糖果都有一个体积 `w_i` 和一个甜度 `v_i`。你的背包容量是 `C`。每种糖果你可以拿任意多个，只要你的背包能装下。请问，你最多能获得多少总甜度？

**输入格式：**
第一行包含两个整数 `N` 和 `C`，分别表示糖果种类数和背包容量。
接下来 `N` 行，每行包含两个整数 `w_i` 和 `v_i`，分别表示第 `i` 种糖果的体积和甜度。

**输出格式：**
一个整数，表示能获得的最大总甜度。

**样例输入：**
```
2 10
2 3
3 4
```

**样例输出：**
```
15
```

**思路分析：**
1.  **识别问题类型：** 题目明确指出“每种糖果你可以拿任意多个”，这正是完全背包问题的特征。
2.  **确定状态：** 设 `f[j]` 表示当背包容量为 `j` 时，能够获得的最大甜度。
3.  **确定初始状态：** `f[0] = 0`。其他 `f[j]` 初始化为0。因为即使不放任何糖果，甜度也是0。
4.  **确定状态转移方程：** 对于每种糖果 `i` (体积 `w_i`，甜度 `v_i`)，我们考虑两种情况：
    *   不放当前糖果 `i`。
    *   放当前糖果 `i`。如果放了，背包容量减少 `w_i`，甜度增加 `v_i`。因为可以放多个，所以剩下的 `j - w_i` 容量所能获得的最大甜度 `f[j - w_i]` 应该是在考虑了糖果 `i` 之后的状态。
    所以，`f[j] = max(f[j], f[j - w_i] + v_i)`。
5.  **确定循环顺序：**
    *   外层循环：遍历每种糖果 `i` (从1到 `N`)。
    *   内层循环：遍历背包容量 `j` (从 `w_i` 到 `C`)。**这里 `j` 必须从小到大，以保证 `f[j - w_i]` 已经包含了糖果 `i` 的可能选择。**
6.  **结果：** 最终的答案就是 `f[C]`。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

using namespace std; // 使用标准命名空间

const int MAXC = 1005; // 最大背包容量
const int MAXN = 105; // 最大物品种类数

int w[MAXN]; // 存储每种糖果的体积
int v[MAXN]; // 存储每种糖果的甜度
int f[MAXC]; // f[j] 表示容量为 j 时能获得的最大甜度

int main() {
    // 关闭同步流，加速I/O
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, c; // n: 糖果种类数, c: 背包容量
    cin >> n >> c;

    // 读取每种糖果的体积和甜度
    for (int i = 1; i <= n; ++i) {
        cin >> w[i] >> v[i];
    }

    // 初始化 f 数组
    // f[0] = 0; 其他 f[j] 默认初始化为0，表示不放任何糖果时的甜度
    // C++ 全局数组或静态数组默认初始化为0，局部数组需要手动初始化
    // 这里 f 是全局数组，所以不需要显式循环初始化为0
    // 如果 f 是局部数组，需要 for (int j = 0; j <= c; ++j) { f[j] = 0; }

    // 动态规划核心部分
    // 遍历每种糖果
    for (int i = 1; i <= n; ++i) {
        // 遍历背包容量，注意这里是正向循环
        for (int j = w[i]; j <= c; ++j) {
            // f[j] = max(不放当前糖果i, 放一个当前糖果i)
            // f[j - w[i]] 已经包含了当前糖果i的多次选择可能
            f[j] = max(f[j], f[j - w[i]] + v[i]);
        }
    }

    // 输出最终结果
    cout << f[c] << endl;

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：外层 `N` 次循环，内层 `C` 次循环，共 `O(N * C)`。
*   空间复杂度：`f` 数组大小为 `C+1`，`w` 和 `v` 数组大小为 `N+1`。共 `O(N + C)`。

### 例题二：神奇的钱币

**题目描述：**
你是一个收藏家，拥有 `N` 种不同面值的神奇钱币。每种钱币都有一个面值 `w_i` 和一个独特的收藏价值 `v_i`。你现在需要凑出一个总金额 `S`。你可以使用任意数量的同种钱币。请问，在凑出总金额 `S` 的前提下，你能够获得的最大总收藏价值是多少？

**输入格式：**
第一行包含两个整数 `N` 和 `S`，分别表示钱币种类数和目标总金额。
接下来 `N` 行，每行包含两个整数 `w_i` 和 `v_i`，分别表示第 `i` 种钱币的面值和收藏价值。

**输出格式：**
一个整数，表示能获得的最大总收藏价值。如果无法凑出总金额 `S`，则输出0。

**样例输入：**
```
3 11
2 3
5 8
3 4
```

**样例输出：**
```
18
```

**思路分析：**
这个问题本质上仍然是完全背包。
1.  **识别问题类型：** “可以使任意数量的同种钱币” -> 完全背包。
2.  **映射到背包模型：**
    *   “钱币面值 `w_i`” 对应背包问题的“物品重量”。
    *   “收藏价值 `v_i`” 对应背包问题的“物品价值”。
    *   “目标总金额 `S`” 对应背包问题的“背包容量 `C`”。
3.  **状态定义：** `f[j]` 表示凑出金额 `j` 时，能够获得的最大总收藏价值。
4.  **初始化：** `f[0] = 0`。其他 `f[j]` 初始化为0。因为题目没有要求必须凑出 `S` 才能获得价值，如果无法凑出，默认是0。
5.  **状态转移方程：** `f[j] = max(f[j], f[j - w[i]] + v[i])`。
6.  **循环顺序：** 外层遍历钱币种类 `i`，内层遍历目标金额 `j` (从 `w_i` 到 `S`)，**正向循环**。
7.  **结果：** `f[S]`。

**C++ 代码实现：**

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXS = 10005; // 最大目标金额
const int MAXN = 105;  // 最大钱币种类数

int w[MAXN]; // 存储每种钱币的面值 (对应背包重量)
int v[MAXN]; // 存储每种钱币的收藏价值 (对应背包价值)
int f[MAXS]; // f[j] 表示凑出金额 j 时能获得的最大收藏价值

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, s; // n: 钱币种类数, s: 目标总金额
    cin >> n >> s;

    // 读取每种钱币的面值和收藏价值
    for (int i = 1; i <= n; ++i) {
        cin >> w[i] >> v[i];
    }

    // 初始化 f 数组
    // f[0] = 0; 其他 f[j] 默认初始化为0
    // 如果 f 是局部数组，需要手动 for (int j = 0; j <= s; ++j) { f[j] = 0; }

    // 动态规划核心部分
    // 遍历每种钱币
    for (int i = 1; i <= n; ++i) {
        // 遍历目标金额，注意这里是正向循环
        for (int j = w[i]; j <= s; ++j) {
            // f[j] = max(不使用当前钱币凑出j, 使用当前钱币凑出j)
            // f[j - w[i]] 已经包含了当前钱币i的多次选择可能
            f[j] = max(f[j], f[j - w[i]] + v[i]);
        }
    }

    // 输出最终结果
    cout << f[s] << endl;

    return 0;
}
```

**复杂度分析：**
*   时间复杂度：`O(N * S)`。
*   空间复杂度：`O(N + S)`。

## 代码实现模板

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件，如iostream, algorithm等

using namespace std; // 使用标准命名空间

// 定义常量，用于数组大小，防止越界
// MAXC 或 MAXS 表示背包容量或目标金额的最大值
// MAXN 表示物品种类或钱币种类的最大值
const int MAXC = 1005; // 根据题目数据范围调整
const int MAXN = 105;  // 根据题目数据范围调整

// 定义存储物品信息和DP状态的数组
int w[MAXN]; // w[i] 存储第 i 种物品的重量/面值
int v[MAXN]; // v[i] 存储第 i 种物品的价值/甜度/收藏价值
int f[MAXC]; // f[j] 存储容量 j (或金额 j) 时能获得的最大价值

int main() {
    // 优化C++标准库的I/O流，使其与C语言的I/O流不同步，并解除cin/cout的绑定
    // 这通常能显著提高输入输出速度，在竞赛编程中非常常用
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, c; // n: 物品种类数，c: 背包容量 (或目标总金额)
    cin >> n >> c;

    // 读取每种物品的重量和价值
    // 注意：数组通常从索引1开始存储，0可以空着或用于特殊目的
    for (int i = 1; i <= n; ++i) {
        cin >> w[i] >> v[i];
    }

    // 初始化 f 数组
    // f[0] = 0 表示容量为0时，最大价值为0。
    // 其他 f[j] 默认为0，表示在不放任何物品的情况下，价值为0。
    // 如果题目要求“必须装满”，且物品价值为正，则 f[j > 0] 需要初始化为负无穷大
    // (例如：memset(f, 0xcf, sizeof(f)); f[0] = 0;)
    // 但对于大部分 GESP 题目，初始化为0即可。
    // 注意：如果是局部数组，需要手动循环初始化：
    // for (int j = 0; j <= c; ++j) { f[j] = 0; }
    // 全局数组或静态数组在C++中默认初始化为0。

    // 动态规划核心：两层循环
    // 外层循环：遍历每种物品 (从第1种到第n种)
    for (int i = 1; i <= n; ++i) {
        // 内层循环：遍历背包容量 (从当前物品重量 w[i] 到总容量 c)
        // **关键点：这里必须是正向循环 (j 从小到大)**
        // 这样 f[j - w[i]] 在计算时已经包含了当前物品 i 的选择可能
        for (int j = w[i]; j <= c; ++j) {
            // 状态转移方程：
            // f[j] = max(
            //     f[j],               // 方案一：不放第 i 种物品 (f[j] 保持原值，即不考虑物品 i 的最大价值)
            //     f[j - w[i]] + v[i]  // 方案二：放第 i 种物品 (从容量 j-w[i] 的最大价值基础上，加上物品 i 的价值)
            // );
            f[j] = max(f[j], f[j - w[i]] + v[i]);
        }
    }

    // 输出最终结果，即容量为 c 时的最大价值
    cout << f[c] << endl;

    return 0; // 程序成功结束
}
```

## 课堂互动

### 提问环节 (3-5个)

1.  **思考题1：** 0/1背包和完全背包，它们的**问题描述**有什么核心区别？（提示：只差一个词！）
    *   **预期答案：** 0/1背包每种物品“只能选择一次”，完全背包每种物品“可以选取任意多次”。
2.  **思考题2：** 为什么在完全背包问题中，我们计算 `f[j]` 时，内层循环 `j` 必须是**从小到大**的？如果用0/1背包的倒序循环，会发生什么？
    *   **预期答案：** 正向循环 `j` 使得 `f[j - w[i]]` 已经包含了当前物品 `i` 的选择，从而允许物品 `i` 被重复选择。如果用倒序循环，`f[j - w[i]]` 还没有被当前物品 `i` 更新过，就退化成了0/1背包，无法实现重复选择。
3.  **思考题3：** 如果题目要求“背包必须恰好装满，且所有物品价值都是正数”，那么 `f` 数组应该如何初始化，才能保证结果的正确性？
    *   **预期答案：** `f[0]` 初始化为0，表示容量0时恰好装满价值为0。所有 `f[j]` ( `j > 0` ) 初始化为负无穷大（一个很大的负数，比如 `0x3f3f3f3f` 的负值，或者 `INT_MIN`），这样只有当 `j` 能够被物品组合“恰好装满”时，`f[j]` 才会被更新为有效值。

    **题目描述：** 商店里有 `N` 种面值的钱币 `v_1, v_2, ..., v_N`。每种钱币都有无限多个。现在你需要凑出一个总金额 `M`。请问，最少需要多少枚钱币才能凑出总金额 `M`？如果无法凑出，输出 `-1`。

### 参考答案

**1. 零食采购**
```cpp
#include <bits/stdc++.h>
using namespace std;
const int MAXM = 1005;
const int MAXN = 105;
int p[MAXN]; // 价格 (重量)
int t[MAXN]; // 美味值 (价值)
int f[MAXM]; // dp数组
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= n; ++i) {
        cin >> p[i] >> t[i];
    }
    // f数组默认初始化为0
    for (int i = 1; i <= n; ++i) {
        for (int j = p[i]; j <= m; ++j) { // 正向循环
            f[j] = max(f[j], f[j - p[i]] + t[i]);
        }
    }
    cout << f[m] << endl;
    return 0;
}
```

**2. 凑数字**
```cpp
#include <bits/stdc++.h>
using namespace std;
const int MAXT = 10005;
const int MAXN = 105;
int a[MAXN]; // 数字值 (重量)
bool f[MAXT]; // f[j] 表示是否能凑出和为 j
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int n, t;
    cin >> n >> t;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
    }
    f[0] = true; // 凑出和为0是可能的 (不选任何数字)
    for (int i = 1; i <= n; ++i) {
        for (int j = a[i]; j <= t; ++j) { // 正向循环
            f[j] = f[j] || f[j - a[i]]; // f[j] 为真，如果原来就能凑出 j，或者 f[j-a[i]] 能凑出
        }
    }
    if (f[t]) {
        cout << "YES" << endl;
    } else {
        cout << "NO" << endl;
    }
    return 0;
}
```

**3. 最少钱币凑和**
```cpp
#include <bits/stdc++.h>
using namespace std;
const int MAXM = 10005;
const int MAXN = 105;
const int INF = 0x3f3f3f3f; // 表示一个很大的数，无穷大
int v[MAXN]; // 钱币面值
int f[MAXM]; // f[j] 表示凑出金额 j 所需的最少钱币数
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= n; ++i) {
        cin >> v[i];
    }
    // 初始化 f 数组
    // f[0] = 0 (凑出金额0需要0枚钱币)
    // f[j > 0] = INF (其他金额初始为无法凑出，需要无穷多枚钱币)
    for (int j = 1; j <= m; ++j) {
        f[j] = INF;
    }
    f[0] = 0;

    for (int i = 1; i <= n; ++i) {
        for (int j = v[i]; j <= m; ++j) { // 正向循环
            if (f[j - v[i]] != INF) { // 只有当 f[j-v[i]] 是可达的时候才更新
                f[j] = min(f[j], f[j - v[i]] + 1);
            }
        }
    }
    if (f[m] == INF) {
        cout << -1 << endl;
    } else {
        cout << f[m] << endl;
    }
    return 0;
}
```
