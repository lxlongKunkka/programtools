# 土拨鼠炒果子2

# 股票买卖最大利润 (多笔交易) 深入浅出解题教案

## 1. 题目深度解析
土拨鼠的炒果子问题，本质上是经典的“买卖股票的最佳时机 II”问题。它允许我们进行多次买卖操作，以获取最大利润。

-   **核心考点**：
    *   **贪心算法**：本题最核心的思想是贪心策略。在允许进行多次交易的情况下，我们不需要考虑复杂的全局最优解，只需关注每一步的局部最优。
    *   **数组遍历**：需要对给定的价格数组进行一次或多次遍历。
    *   **简单逻辑判断**：根据相邻日期的价格关系进行决策。

-   **题目难点**：
    *   **与单次交易问题的混淆**：很多初学者会首先想到“只能买卖一次”的股票问题，试图寻找一个最低买入点和一个最高卖出点。但本题允许多次交易，这大大简化了问题，也容易让学生陷入过度思考。
    *   **“今天买入今天卖出”的理解**：虽然题目允许今天买入今天卖出，但如果价格不变，利润为 $0$；如果价格上涨，则利润为正。这实际上可以融入到相邻日期的价格比较中。
    *   **贪心策略的证明**：虽然贪心策略直观，但理解其正确性背后的数学原理对初学者来说可能是一个小挑战。

-   **易错点警示**：
    *   **数据溢出**：虽然本题的利润总和在 $10^5 \times 10^4 = 10^9$ 级别，对于 `int` 类型来说可能刚刚好，但如果价格范围或天数更大，就需要考虑使用 `long long`。在本题中，`int` 足够。
    *   **边界条件**：当 $n=1$ 时，无法进行交易，利润应为 $0$。当价格数组是单调递减时，也无法获取利润，应为 $0$。这些情况需要算法能够正确处理。
    *   **负利润**：题目要求“如果你不能获取任何利润，返回 $0$”，这意味着我们只累加正利润，或者说，如果最终计算出负利润（这在我们的贪心策略下不会发生），也应该返回 $0$。

## 2. 教学目标
通过本题的学习，学生应该能够：
*   识别并应用贪心算法解决允许进行多次交易的股票买卖问题。
*   理解“局部最优解如何构成全局最优解”的贪心思想。
*   掌握对数组进行单次遍历并进行简单决策的技巧。
*   培养从复杂问题中提取简化模型的能力。

## 3. 循序渐进的解题思路

### 阶段一：直观想法（暴力/朴素解法）

很多同学在第一次遇到这类问题时，可能会试图枚举所有可能的买卖组合。

-   **描述最容易想到的思路**：
    1.  **单次交易的暴力解法**：如果只能买卖一次，我们可以遍历所有可能的买入日期 $i$ 和卖出日期 $j$（其中 $j > i$），计算 $a[j] - a[i]$，然后取所有正利润中的最大值。
    2.  **多次交易的复杂化**：但本题允许多次交易。如果我们要用暴力法来解决多次交易，问题会变得异常复杂。我们需要考虑在哪些天买入，哪些天卖出，每次交易之间不能重叠，且不能在没有持股时卖出。这可能涉及到对数组进行各种分割，然后对每个子数组应用单次交易的逻辑，再尝试组合这些利润。这会形成一个递归结构，或者动态规划的状态转移，但状态定义和转移方程会非常复杂。

-   **分析其时间复杂度**：
    *   对于单次交易的暴力解法，需要两层循环，时间复杂度为 $O(N^2)$。
    *   对于多次交易的暴力解法，其时间复杂度会远超 $O(N^2)$，可能达到指数级别。

-   **指出为什么它可能无法通过（TLE/MLE）**：
    *   题目给定的数据范围 $N \le 10^5$。
    *   $O(N^2)$ 的算法在 $N=10^5$ 时，操作次数将达到 $(10^5)^2 = 10^{10}$ 级别，这远远超出了现代计算机在 $1$ 秒内能处理的 $10^8$ 级别操作，因此会导致 **TLE (Time Limit Exceeded)**。
    *   更复杂的暴力解法，如果需要存储大量中间状态，还可能导致 **MLE (Memory Limit Exceeded)**。

### 阶段二：优化突破口

既然暴力法行不通，我们需要寻找更高效的方法。问题的关键在于“可以进行多次交易”。这意味着我们不需要像单次交易那样，必须找到一个全局的最低点买入和全局的最高点卖出。

-   **引导学生思考如何优化**：
    *   既然可以多次交易，那么我们是不是可以把一个大的利润分解成多个小的利润？
    *   例如，如果价格从 $1$ 涨到 $5$，再涨到 $6$，我们可以在 $1$ 买入，在 $6$ 卖出，利润是 $6-1=5$。
    *   我们也可以在 $1$ 买入， $5$ 卖出（利润 $5-1=4$），然后在 $5$ 再买入， $6$ 卖出（利润 $6-5=1$）。总利润是 $4+1=5$。
    *   你会发现，这两种方式得到的总利润是相同的！ $ (6-1) = (5-1) + (6-5) $。
    *   这意味着，只要价格上涨，我们就可以“假装”进行一次买卖，把这部分利润收入囊中。

-   **使用类比、图解（文字描述）或生活实例来解释核心算法思想**：
    想象一下果子价格的走势图，就像一座座山峰和山谷。
    *   如果只能买卖一次，我们就要找到最低的山谷买入，然后找到最高的山峰卖出。
    *   但是，现在我们可以多次交易！这意味着我们可以把每一段“上坡路”的利润都赚到。
    *   例如，价格从 $P_1 \to P_2 \to P_3 \to P_4 \to P_5$。
        *   如果 $P_2 > P_1$，我们就赚取 $P_2 - P_1$。
        *   如果 $P_3 > P_2$，我们就赚取 $P_3 - P_2$。
        *   如果 $P_4 < P_3$，我们就不操作，因为是亏损。
        *   如果 $P_5 > P_4$，我们就赚取 $P_5 - P_4$。
    *   最终的总利润就是所有这些“上坡”利润的总和。

-   **关键逻辑推导**：
    我们来严谨地推导一下这个贪心策略的正确性。
    假设我们有价格序列 $a_0, a_1, a_2, \dots, a_{n-1}$。
    我们的贪心策略是：遍历数组，如果发现 $a_{i+1} > a_i$，就将差值 $a_{i+1} - a_i$ 加入总利润。

    考虑一个连续上涨的区间，例如 $a_i < a_{i+1} < a_{i+2} < \dots < a_j$。
    如果按照贪心策略，我们获得的利润是：
    $(a_{i+1} - a_i) + (a_{i+2} - a_{i+1}) + \dots + (a_j - a_{j-1})$
    这是一个经典的“裂项求和”：
    $= a_{i+1} - a_i + a_{i+2} - a_{i+1} + \dots + a_j - a_{j-1}$
    $= a_j - a_i$

    这恰好等于在这个上涨区间内，在最低点 $a_i$ 买入，在最高点 $a_j$ 卖出所获得的利润。
    这意味着，我们的贪心策略，通过累加所有相邻日期的正利润，能够等价地捕获到所有“山谷买入，山峰卖出”的利润。
    由于我们可以进行无数次交易，我们可以在任何时候卖出，然后在第二天再次买入（如果有利可图）。所以，只要价格上涨，我们就可以立即“兑现”这部分利润。如果价格下跌，我们就不操作，等待下一次上涨机会。这种策略保证了我们不会错过任何一个可以赚钱的机会，并且不会进行亏损的交易。因此，这种贪心策略能够得到最大总利润。

    **总结**：只要 $a_{i+1} > a_i$，我们就认为在第 $i$ 天买入，第 $i+1$ 天卖出，可以获得 $a_{i+1} - a_i$ 的利润。将所有这样的正利润累加起来，就是最大总利润。

### 阶段三：最终算法流程

1.  初始化一个变量 `total_profit` 为 $0$，用于存储总利润。
2.  从数组的第一个元素开始，遍历到倒数第二个元素（即从 $i=0$ 到 $n-2$）。
3.  在每次遍历中，比较当前元素 $a[i]$ 和下一个元素 $a[i+1]$ 的价格。
4.  如果 $a[i+1] > a[i]$，说明从第 $i$ 天到第 $i+1$ 天价格上涨了，我们可以获得 $a[i+1] - a[i]$ 的利润。将这个利润累加到 `total_profit` 中。
5.  遍历结束后，`total_profit` 中存储的就是最大利润。

## 4. 核心代码讲解

```cpp
<!-- AC_CODE -->
#include <iostream> // 引入输入输出流库
#include <vector>   // 引入动态数组vector库
#include <numeric>  // 引入numeric库，虽然本题未使用，但常用

int main()
{
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 声明整数变量n，表示天数
    std::cin >> n; // 读取天数n

    // 声明一个vector<int>类型的动态数组prices，用于存储每天的果子价格
    // 数组大小为n
    std::vector<int> prices(n); 
    for (int i = 0; i < n; ++i)
    {
        std::cin >> prices[i]; // 逐个读取每天的果子价格
    }

    long long total_profit = 0; // 声明长整型变量total_profit，初始化为0
                                // 使用long long以防利润总和超出int范围，虽然本题int也够用，但这是好习惯

    // 遍历价格数组，从第一天（索引0）到倒数第二天（索引n-2）
    // 每次比较prices[i]和prices[i+1]
    for (int i = 0; i < n - 1; ++i)
    {
        // 如果下一天的价格高于今天的价格，说明有利可图
        if (prices[i+1] > prices[i])
        {
            // 将差价（利润）累加到total_profit中
            // 相当于在prices[i]买入，prices[i+1]卖出
            total_profit += (prices[i+1] - prices[i]);
        }
    }

    std::cout << total_profit << std::endl; // 输出计算出的最大总利润

    return 0; // 程序正常结束
}

```

## 5. 复杂度分析

-   **时间复杂度**：
    *   读取输入价格数组需要遍历 $N$ 个元素，耗时 $O(N)$。
    *   计算最大利润的核心循环从 $i=0$ 到 $n-2$，也即遍历了 $N-1$ 次。在每次循环中，我们只进行了一次比较和一次加法操作，这些都是常数时间操作 $O(1)$。
    *   因此，总的时间复杂度为 $O(N)$。

-   **空间复杂度**：
    *   我们使用了一个 `std::vector<int> prices` 来存储 $N$ 天的价格，这占用了 $O(N)$ 的空间。
    *   除了存储价格数组外，我们只使用了几个额外的变量（`n`, `total_profit`, `i`），它们占用的空间是常数级别的 $O(1)$。
    *   因此，总的空间复杂度为 $O(N)$。
    *   **优化思考**：如果我们不需要存储整个价格数组，而是在读取价格的同时进行计算，那么空间复杂度可以优化到 $O(1)$。例如，可以每次只读取当前价格和前一天的价格进行比较。但通常情况下，为了代码的清晰和通用性，存储整个数组是可接受的。
