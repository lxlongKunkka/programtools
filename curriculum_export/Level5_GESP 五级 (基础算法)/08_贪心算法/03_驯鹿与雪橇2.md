# 驯鹿与雪橇2 深入浅出解题教案

## 1. 题目深度解析

- **核心考点**  
  1. 贪心算法  
  2. 排序  
  3. 大数求和与 64 位整型溢出处理

- **题目难点**  
  如何将“拉雪橇的总力量 $\sum P_{\text{拉}}$ 不小于 骑雪橇的总重量 $\sum W_{\text{骑}}$”的约束，转换成一个可以通过贪心排序解决的问题。

- **易错点警示**  
  1. 没注意到 $\sum P_i$ 与 $\sum W_i$ 都可能达到 $10^9 \times 3\times10^5\approx3\times10^{14}$，必须使用 `long long`。  
  2. 在计算“每个驯鹿成为骑手的“额外负担””时容易搞错公式。  
  3. 边界条件：当所有驯鹿都不能骑时（答案 $0$）要正确输出。

---

## 2. 教学目标

- 理解如何在有“全局能量”或“全局容量”约束下，把「选取最多项」转化为「贪心选择单位负担最小的项」。
- 掌握将题目约束转换成「背包容量」+「项目权重」的思路，从而用排序+前缀和快速解决。
- 学会警惕大数溢出，并使用 64 位整型保存累加和。

## 3. 循序渐进的解题思路

### 阶段一：直观想法（暴力/朴素解法）

提出问题  
- 试着枚举骑雪橇的驯鹿集合 $S$，判断能否满足  
  $$\sum_{i\notin S}P_i \;\ge\;\sum_{i\in S}W_i\,. $$  
- 枚举所有子集显然 $O(2^N)$，爆炸。

尝试解决  
- 或者先固定骑的数量 $k$，再判断选 $k$ 只怎样骑最多？  
- 仍需从 $N$ 只中挑 $k$ 只，枚举组合 $O\bigl(\binom Nk\bigr)$，依旧不可行。

发现问题  
- 暴力和组合枚举都不可行。必须找到结构化的贪心或动态规划思路。

### 阶段二：优化突破口

引导思考  
- 总的拉力是所有没骑的驯鹿的 $\sum P_{\text{拉}} = P_{\text{total}} - \sum_{i\in S}P_i$。  
- 骑的总重是 $\sum_{i\in S}W_i$。  
- 约束变为  
  $$P_{\text{total}} - \sum_{i\in S}P_i \;\ge\;\sum_{i\in S}W_i$$  
  等价于  
  $$\sum_{i\in S}(P_i + W_i)\;\le\;P_{\text{total}}\,. $$

生活类比  
- 想象 $P_{\text{total}}$ 是你的背包最大承重，每个驯鹿如果被当作“骑手”，它同时占用两种资源：  
  - 占用拉力资源 $P_i$（因为它不再拉雪橇）  
  - 增加重量 $W_i$（因为它骑在雪橇上）  
- 合并成一个“负担”$$c_i = P_i + W_i$$  
- 问题就变成：在总负担不超过 $P_{\text{total}}$ 的前提下，选出最多个体。

**关键逻辑推导**  
- 既然每个“项”都只有一个「负担值」$c_i$，而且「收益」都相同（每多骑一只算 +1），最优策略：  
  **贪心选择** $c_i$ 从小到大尽可能多地装进容量 $P_{\text{total}}$ 的“背包”中。  

### 阶段三：最终算法流程

1. 读取 $N$ 只驯鹿的参数 $(W_i,P_i)$，同时累加  
   $$P_{\text{total}}=\sum_{i=1}^N P_i\,. $$
2. 对每只驯鹿计算负担  
   $$c_i = W_i + P_i\,. $$
3. 将所有 $c_i$ 按升序排序。
4. 从头开始做前缀和  
   ```
   cum = 0
   ans = 0
   for each c in sorted c_i:
       if cum + c <= P_total:
           cum += c
           ans += 1
       else:
           break
   ```
5. 输出 `ans` 即为最多可以骑雪橇的驯鹿数量。

## 4. 核心代码讲解

```cpp
<!-- AC_CODE -->
#include <bits/stdc++.h>
using namespace std;

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--)
    {
        int N;
        cin >> N;

        vector<long long> c(N);
        long long P_total = 0;  // 拉雪橇的总力量上限

        for (int i = 0; i < N; i++)
        {
            long long W, P;
            cin >> W >> P;
            P_total += P;    // 累加所有驯鹿的力量
            c[i] = W + P;    // 计算该驯鹿成为骑手的“负担”
        }

        // 将负担从小到大排序
        sort(c.begin(), c.end());

        long long cum = 0;   // 当前已使用的拉力资源
        int ans = 0;         // 当前可骑的人数

        // 贪心装载：尽可能多地选最小负担
        for (auto &x : c)
        {
            if (cum + x <= P_total)
            {
                cum += x;
                ans++;
            }
            else
            {
                break;
            }
        }

        cout << ans << "\n";
    }

    return 0;
}
```

- 关键注释已在代码中说明了每一步的目的。
- **Allman 风格**：每个大括号单独占一行，便于阅读和维护。
- 注意所有累加和都使用 `long long` 避免溢出。

## 5. 复杂度分析

- **时间复杂度**  
  - 对每个测试用例，读取 $N$ 对数据：$O(N)$  
  - 计算并排序负担数组：$O(N\log N)$  
  - 遍历前缀和求解：$O(N)$  
  合计：$\displaystyle O\bigl(N\log N\bigr)$。  
  由于 $\sum N\le3\times10^5$，整体可接受。

- **空间复杂度**  
  - 主要开销在于存储负担数组 $\text{vector}<long\ long>$，大小 $O(N)$。  
  - 额外常数级空间：若干整型变量。  
  总体：$\displaystyle O(N)$。
