## 知识点讲解

### 1. 什么是贪心算法？

贪心算法（Greedy Algorithm）是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是全局最好或最优的算法策略。它不考虑后续步骤的影响，只顾眼前的最优。

**核心思想：**
- **局部最优选择：** 每一步都做出当前看起来最好的选择。
- **期望全局最优：** 希望通过一系列局部最优选择，最终能达到整个问题的最优解。

**举个例子：** 如果你想从山脚走到山顶，贪心策略就是每一步都向上走，并且选择最陡峭的方向。但是，这可能让你走到一个“假山顶”（局部最高点），而不是真正的山顶（全局最高点）。所以，贪心算法并不总是正确的，但如果它正确，通常会非常高效。

### 2. 贪心算法的适用条件

贪心算法之所以能够奏效，通常需要满足两个重要的性质：

1.  **贪心选择性质 (Greedy Choice Property)：**
    这意味着一个全局最优解可以通过一系列局部最优（贪心）选择来达到。换句话说，在做出一个贪心选择后，原问题就变为一个相似的、更小的子问题。

2.  **最优子结构性质 (Optimal Substructure Property)：**
    一个问题的最优解包含其子问题的最优解。这是许多优化算法（包括动态规划）共同的性质。

如果一个问题同时满足这两个性质，那么它很可能可以用贪心算法来解决。

### 3. 经典案例一：找零钱问题

**问题描述：** 假设我们有面值为 100, 50, 10, 5, 1 的人民币。给定一个金额 `n`，如何用最少的钱币数量来凑出这个金额？

**贪心策略：**
每次都选择当前能用的最大面额的钱币，直到找零金额为零。

**例如，找零 68 元：**
1.  当前金额 68 元，最大面额 100 元用不了。
2.  用 50 元：68 - 50 = 18 元。使用 1 张 50 元。
3.  当前金额 18 元，最大面额 50 元用不了。
4.  用 10 元：18 - 10 = 8 元。使用 1 张 10 元。
5.  当前金额 8 元，最大面额 10 元用不了。
6.  用 5 元：8 - 5 = 3 元。使用 1 个 5 元。
7.  当前金额 3 元，最大面额 5 元用不了。
8.  用 1 元：3 - 1 = 2 元。使用 1 个 1 元。
9.  用 1 元：2 - 1 = 1 元。使用 1 个 1 元。
10. 用 1 元：1 - 1 = 0 元。使用 1 个 1 元。
总共：1 张 50 元，1 张 10 元，1 个 5 元，3 个 1 元。

**为什么这个策略在这里有效？**
对于我们日常使用的标准货币面额系统（1, 5, 10, 50, 100），这种贪心策略是正确的。因为任何小于最大面额的金额，都可以通过更小的面额组合出来，并且不会因为先用了大面额而导致后续需要更多小面额来弥补。
**但是，请注意：** 如果面额系统不是这样设计的，贪心策略可能就不行了。例如，如果只有 1, 3, 4 元的硬币，要找 6 元。
- 贪心策略：先用 4 元，剩下 2 元；再用两个 1 元。总共 3 枚硬币 (4, 1, 1)。
- 最优策略：用两个 3 元。总共 2 枚硬币 (3, 3)。
所以，贪心算法不是万能的！

### 4. 经典案例二：活动安排问题

**问题描述：** 假设有 `n` 个活动，每个活动 `i` 都有一个开始时间 `s_i` 和结束时间 `e_i`。你只能参加不重叠的活动（即一个活动结束前，另一个活动不能开始）。目标是选择尽可能多的活动。

**贪心策略：**
1.  **排序：** 将所有活动按照它们的**结束时间**从小到大进行排序。
2.  **选择第一个活动：** 选择结束时间最早的活动。
3.  **迭代选择：** 从剩下的活动中，选择第一个开始时间**晚于或等于**上一个已选活动结束时间的活动。
4.  **重复：** 重复步骤 3，直到没有可选活动为止。

**例如：** 假设有以下活动 (开始时间, 结束时间)：
A: (1, 4)
B: (3, 5)
C: (0, 6)
D: (5, 7)
E: (3, 9)
F: (5, 9)
G: (6, 10)
H: (8, 11)
I: (2, 12)
J: (12, 14)

**按照结束时间排序后：**
A: (1, 4)
B: (3, 5)
D: (5, 7)
C: (0, 6) (注意：C的结束时间是6，D是7，所以D在C后面)
G: (6, 10)
H: (8, 11)
I: (2, 12)
J: (12, 14)
E: (3, 9)
F: (5, 9)

让我们重新按照结束时间排序：
A: (1, 4)
B: (3, 5)
C: (0, 6)
D: (5, 7)
E: (3, 9)
F: (5, 9)
G: (6, 10)
H: (8, 11)
I: (2, 12)
J: (12, 14)

正确的排序应该是：
活动 | 开始时间 | 结束时间
---|---|---
A | 1 | 4
B | 3 | 5
C | 0 | 6
D | 5 | 7
G | 6 | 10
E | 3 | 9
F | 5 | 9
H | 8 | 11
I | 2 | 12
J | 12 | 14

**执行贪心策略：**
1.  **选择 A (1, 4)**。`last_end_time = 4`。已选活动数 = 1。
2.  下一个活动 B (3, 5)。`3 < 4`，重叠，跳过。
3.  下一个活动 C (0, 6)。`0 < 4`，重叠，跳过。
4.  下一个活动 D (5, 7)。`5 >= 4`，不重叠，**选择 D (5, 7)**。`last_end_time = 7`。已选活动数 = 2。
5.  下一个活动 G (6, 10)。`6 < 7`，重叠，跳过。
6.  下一个活动 E (3, 9)。`3 < 7`，重叠，跳过。
7.  下一个活动 F (5, 9)。`5 < 7`，重叠，跳过。
8.  下一个活动 H (8, 11)。`8 >= 7`，不重叠，**选择 H (8, 11)**。`last_end_time = 11`。已选活动数 = 3。
9.  下一个活动 I (2, 12)。`2 < 11`，重叠，跳过。
10. 下一个活动 J (12, 14)。`12 >= 11`，不重叠，**选择 J (12, 14)**。`last_end_time = 14`。已选活动数 = 4。

最终选择了 4 个活动：A, D, H, J。

**为什么这个策略有效？**
选择最早结束的活动，能为后续活动留下最大的空闲时间。这是一种非常经典的贪心策略，并且被证明是正确的。

## 代码示例

### 1. 找零钱问题

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n; // 需要找零的金额
    cout << "请输入需要找零的金额: ";
    cin >> n;

    int c100 = 0, c50 = 0, c10 = 0, c5 = 0, c1 = 0; // 各面额钱币的数量

    // 贪心策略：优先使用大面额
    // 计算100元纸币数量
    c100 = n / 100; // 整除得到100元纸币的数量
    n = n % 100;    // 取模得到剩余金额

    // 计算50元纸币数量
    c50 = n / 50;
    n = n % 50;

    // 计算10元纸币数量
    c10 = n / 10;
    n = n % 10;

    // 计算5元硬币数量
    c5 = n / 5;
    n = n % 5;

    // 计算1元硬币数量
    c1 = n / 1; // n % 1 结果总是0，所以可以直接赋值
    // n = n % 1; 

    cout << "找零方案如下：" << endl;
    if (c100 > 0) {
        cout << "100元: " << c100 << " 张" << endl;
    }
    if (c50 > 0) {
        cout << "50元: " << c50 << " 张" << endl;
    }
    if (c10 > 0) {
        cout << "10元: " << c10 << " 张" << endl;
    }
    if (c5 > 0) {
        cout << "5元: " << c5 << " 个" << endl;
    }
    if (c1 > 0) {
        cout << "1元: " << c1 << " 个" << endl;
    }
    cout << "总共使用了 " << c100 + c50 + c10 + c5 + c1 << " 枚/张钱币。" << endl;

    return 0;
}
```

### 2. 活动安排问题

```cpp
#include <bits/stdc++.h>
using namespace std;

// 定义活动结构体
struct A {
    int s; // 开始时间
    int e; // 结束时间
};

// 比较函数，用于按结束时间升序排序
bool cmp(A x, A y) {
    return x.e < y.e; // 如果x的结束时间比y早，则x排在y前面
}

int main() {
    int n; // 活动数量
    cout << "请输入活动数量: ";
    cin >> n;

    A a[n]; // 定义一个活动数组，存储所有活动

    cout << "请依次输入每个活动的开始时间和结束时间：" << endl;
    for (int i = 0; i < n; i++) {
        cout << "活动 " << i + 1 << " (开始 结束): ";
        cin >> a[i].s >> a[i].e;
    }

    // 使用sort函数对活动数组进行排序
    // sort函数的第三个参数是比较函数
    sort(a, a + n, cmp);

    cout << "\n排序后的活动列表 (按结束时间):" << endl;
    for (int i = 0; i < n; i++) {
        cout << "活动 " << i + 1 << ": (" << a[i].s << ", " << a[i].e << ")" << endl;
    }

    int cnt = 0;         // 记录选择的活动数量
    int last_e = -1;     // 记录上一个选择活动的结束时间，初始化为-1或一个足够小的值

    cout << "\n选择的活动是: " << endl;
    for (int i = 0; i < n; i++) {
        // 如果当前活动的开始时间晚于或等于上一个选择活动的结束时间
        // 说明不重叠，可以选择这个活动
        if (a[i].s >= last_e) {
            cnt++; // 活动数量加1
            last_e = a[i].e; // 更新上一个选择活动的结束时间
            cout << "  (" << a[i].s << ", " << a[i].e << ")" << endl;
        }
    }

    cout << "\n最多可以参加的活动数量是: " << cnt << endl;

    return 0;
}
```

        *   总共 1 + 1 + 3 = 5 枚。
        *   哎呀，题目给的是 1, 5, 10，答案选项没有 5。检查一下我的计算。
        *   18 元：10元 * 1 = 10，剩余 8元。
        *   8元：5元 * 1 = 5，剩余 3元。
        *   3元：1元 * 3 = 3，剩余 0元。
        *   总共 1 + 1 + 3 = 5 枚。
        *   重新检查选项，如果选项是 5，那就选 5。如果选项是 4，可能是 10, 5, 1, 1, 1，那总共是 5 枚。

        *   我的计算结果是5枚，所以应该选C。

        *   **最终答案：C**

    *   **解析：**
        *   商品 A：单位重量价值 = 10元 / 2kg = 5元/kg
        *   商品 B：单位重量价值 = 15元 / 3kg = 5元/kg
        *   商品 C：单位重量价值 = 6元 / 1kg = 6元/kg
        *   贪心策略：优先选择单位重量价值最高的商品。
        *   1. 选择商品 C (6元/kg)。背包剩余承重 5 - 1 = 4kg。总价值 6元。
        *   2. 商品 A 和 B 单位重量价值都是 5元/kg，可以任意选择一个。假设选择商品 B。背包剩余承重 4 - 3 = 1kg。总价值 6 + 15 = 21元。
        *   3. 剩余 1kg，选择商品 A 的 1kg（因为可以分割）。价值 5元/kg * 1kg = 5元。
        *   最终选择：商品 C (1kg), 商品 B (3kg), 商品 A (1kg)。总价值 6 + 15 + 5 = 26元。
        *   所以填 C, B, A。

#### 1. 找零钱优化问题

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n; // 需要找零的金额
    cout << "请输入需要找零的金额: ";
    cin >> n;

    int c10 = 0, c7 = 0, c1 = 0; // 各面额硬币的数量
    int total_coins = 0;

    // 贪心策略：优先使用大面额
    // 计算10元硬币数量
    c10 = n / 10;
    n = n % 10;
    total_coins += c10;

    // 计算7元硬币数量
    c7 = n / 7;
    n = n % 7;
    total_coins += c7;

    // 计算1元硬币数量
    c1 = n / 1;
    total_coins += c1;

    cout << "贪心策略下使用的硬币数量: " << total_coins << endl;

    // 思考这个结果是否最优：
    // 对于 15 元：
    // 贪心策略： 1 个 10 元 (剩 5 元) -> 0 个 7 元 (剩 5 元) -> 5 个 1 元。总共 1 + 0 + 5 = 6 枚。
    // 但是，如果你用 2 个 7 元 (14 元)，剩下 1 元，再用 1 个 1 元。总共 2 + 1 = 3 枚。
    // 所以，贪心策略在这里不是最优的。
    // 题目示例输出是 7，是因为它可能把 15 拆成了 10 + 1 + 1 + 1 + 1 + 1 + 1 = 7枚。
    // 我的代码输出 6，因为 15 / 10 = 1 (剩 5)， 5 / 7 = 0 (剩 5)， 5 / 1 = 5。 1+0+5 = 6。
    // 如果题目示例 7 是对的，那说明 10 元面额的硬币并没有被优先使用。
    // 让我们按照题目示例的逻辑来思考：15 -> 10 + 5*1 (7枚)
    // 这道题的目的是让大家理解贪心不一定最优。
    // 实际上，15元的最优解是 7元 * 2 + 1元 * 1 = 3枚。
    // 如果是严格按照“优先使用面额最大的硬币”的贪心策略，我的代码是正确的，会输出 6。
    // 题目示例输出 7，可能是一种“错误的贪心”，或者我理解错题目意思了。
    // 按照“总是优先使用面额最大的硬币”的原则，15元：
    // 1个10元，剩余5元。
    // 5元无法用7元找，只能用5个1元。
    // 总共1+5=6枚。
    // 那么，示例输出 7 是怎么来的呢？ 15 = 10 + 1 + 1 + 1 + 1 + 1 + 1。这确实是 7 枚。
    // 可能是 10元 -> 5个1元。
    // 也许题目想表达的是，如果优先用10元，但剩下的钱用1元凑，是不是会比用7元更多？
    // 10 + 5*1 = 6枚
    // 7 + 7 + 1 = 3枚
    // 15*1 = 15枚
    // 这里我严格按照“优先最大面额”的贪心策略，代码输出 6 是符合逻辑的。
    // 示例输出 7 可能是一个误导，或者暗示了一种“更差的贪心”策略。
    // 无论如何，这个例子都说明了贪心算法的局限性。
```

#### 2. 会议室安排问题

```cpp
#include <bits/stdc++.h>
using namespace std;

// 定义会议结构体
struct M {
    int s; // 开始时间
    int e; // 结束时间
};

// 比较函数，用于按结束时间升序排序
bool cmp(M x, M y) {
    return x.e < y.e; // 如果x的结束时间比y早，则x排在y前面
}

int main() {
    int n; // 会议数量
    // cout << "请输入会议数量: ";
    cin >> n;

    M m[n]; // 定义一个会议数组，存储所有会议

    // cout << "请依次输入每个会议的开始时间和结束时间：" << endl;
    for (int i = 0; i < n; i++) {
        // cout << "会议 " << i + 1 << " (开始 结束): ";
        cin >> m[i].s >> m[i].e;
    }

    // 使用sort函数对会议数组进行排序
    // sort函数的第三个参数是比较函数
    sort(m, m + n, cmp);

    int cnt = 0;         // 记录选择的会议数量
    int last_e = -1;     // 记录上一个选择会议的结束时间，初始化为-1或一个足够小的值

    for (int i = 0; i < n; i++) {
        // 如果当前会议的开始时间晚于或等于上一个选择会议的结束时间
        // 说明不重叠，可以选择这个会议
        if (m[i].s >= last_e) {
            cnt++; // 会议数量加1
            last_e = m[i].e; // 更新上一个选择会议的结束时间
        }
    }

    cout << cnt << endl;

    return 0;
}
```
