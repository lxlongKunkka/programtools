## 趣味引入

同学们，想象一下这样一个场景：

小明老师今天心情特别好，带来了一大包美味的糖果要分给大家。班里有 `N` 个小朋友，每个小朋友都有自己对糖果的“胃口值”——比如，小红的胃口值是 1，意思是她至少要吃到大小为 1 的糖果才满意；小刚的胃口值是 3，他至少要吃到大小为 3 的糖果才满意。

同时，老师带来的糖果也大小不一。有大小为 1 的小糖果，有大小为 2 的中糖果，还有大小为 5 的大糖果。

现在问题来了：小明老师想让尽可能多的小朋友感到满意。如果一个小朋友的胃口值是 `g`，他得到了一颗大小为 `s` 的糖果，只要 `s >= g`，这个小朋友就满意了。每颗糖果只能分给一个小朋友。

小明老师挠了挠头：“这可怎么办呢？我应该先把小糖果分出去，还是把大糖果分出去？是先满足胃口大的小朋友，还是先满足胃口小的小朋友？”

这就是我们今天要学习的“分发糖果”问题，也是一种非常重要的算法思想——**贪心算法**！

## 深度知识点讲解

### 1. 什么是贪心算法？（Greedy Algorithm）

想象你面前有一堆苹果，每个苹果大小不一。你的目标是：在最短的时间内，尽可能多地装满你的篮子。你会怎么做？

你可能会想：“管他三七二十一，我看到最大的苹果就拿，这样肯定能很快装满！”这就是一种**贪心**的策略。

**贪心算法**，顾名思义，就是在解决问题时，不从整体最优上加以考虑，而是做出在当前看来是最好的选择。它希望通过一系列局部最优的选择，最终能够得到一个全局最优解。就像一步一步地走，每一步都选择当前看起来最好的方向，希望最后能走到终点。

**核心思想：**
*   **局部最优选择：** 每一步都选择当前状态下最好的方案。
*   **无后效性：** 之前的选择不会影响后续的选择，或者说，一旦做出了选择，就不能再更改。
*   **期望全局最优：** 希望局部最优的选择能够累积成全局最优解。

### 2. 为什么贪心算法有时候能成功，有时候会失败？

不是所有问题都能用贪心算法解决。比如，如果你想从起点走到迷宫的出口，每一步都选择离出口最近的方向，你可能很快就会撞墙，陷入死胡同！这说明局部最优不一定能带来全局最优。

贪心算法能成功，通常需要满足两个重要的性质：

1.  **最优子结构（Optimal Substructure）**：一个问题的最优解包含其子问题的最优解。
2.  **贪心选择性质（Greedy Choice Property）**：通过做一系列局部最优（贪心）的选择，可以得到问题的全局最优解。

对于我们今天的“分发糖果”问题，我们来分析一下，为什么贪心策略在这里会成功。

### 3. “分发糖果”问题的贪心策略解析

我们回到小明老师分糖果的场景。我们想让尽可能多的小朋友满意。

**小朋友的胃口值 `g`：** `g[0], g[1], ..., g[N-1]`
**糖果的大小 `s`：** `s[0], s[1], ..., s[M-1]`

假设我们有以下几种策略：

**策略一：先满足胃口大的小朋友？**
如果先满足胃口最大的小朋友，比如他需要大小为 5 的糖果。我们给他一颗大小为 5 的糖果。这颗大糖果本来可以满足很多胃口小的孩子。这样会不会“浪费”了它？

**策略二：先用大糖果去满足小朋友？**
如果我们有一颗大小为 10 的大糖果，给了一个胃口值只有 1 的小朋友。这颗大糖果就用掉了。如果后面来了一个胃口值是 8 的小朋友，他就没糖果吃了。这显然也不太好。

**策略三：先满足胃口最小的小朋友，用最小能满足他的糖果？**
假设我们把所有小朋友的胃口值从小到大排序，把所有糖果的大小也从小到大排序。
现在，我们从胃口最小的小朋友开始看。他需要 `g_min` 大小的糖果。
我们从最小的糖果开始找，找到第一颗能满足他的糖果 `s_k`。
如果 `s_k` 能够满足 `g_min` (即 `s_k >= g_min`)，我们就把这颗 `s_k` 糖果给他。然后把 `g_min` 小朋友和 `s_k` 糖果都从列表中移除，继续处理下一个胃口最小的小朋友和下一个最小的糖果。
如果 `s_k` 不能满足 `g_min` (即 `s_k < g_min`)，那么这颗 `s_k` 糖果肯定也满足不了胃口更大的小朋友们。所以这颗 `s_k` 糖果就没用了，我们直接跳过它，找下一颗更大的糖果。

**为什么这个策略是有效的呢？**
我们来思考一下：
1.  **小朋友排序（从小到大）**：先满足胃口小的小朋友，是因为他们的需求更容易被满足。如果一个胃口小的孩子都无法被满足，那胃口更大的孩子就更不可能了。
2.  **糖果排序（从小到大）**：当我们尝试满足一个小朋友时，我们总是用能满足他的**最小**的那颗糖果。
    *   **生活类比：** 想象你有一堆不同面额的钱（1元、5元、10元），你要去买东西（需要2元、7元、12元）。如果你想尽可能多地买东西，你会用最小的钱去买能买到的东西。比如，买2元的东西，你肯定用1元+1元，而不是用5元或10元。因为用大钱会“浪费”它的购买力，大钱可能以后能买更贵的东西。
    *   **回到糖果：** 如果一个小朋友的胃口值是 `g`，有两颗糖果 `s1` 和 `s2` 都能满足他，且 `s1 < s2`。如果我们把 `s2` 给了他，那么 `s1` 就被“剩下”了。`s1` 只能满足胃口小于等于 `s1` 的小朋友。而 `s2` 则能满足胃口小于等于 `s2` 的小朋友。显然，留下 `s2` 会比留下 `s1` 更有可能满足后面的小朋友。所以，我们应该用最小的 `s1` 来满足当前小朋友，这样 `s2` 就可以留给胃口更大的小朋友。

所以，**“先满足胃口最小的小朋友，并给他能满足他的最小的糖果”** 这个贪心策略，是正确的！它保证了我们不会“浪费”大糖果的满足能力，从而最大化了满意小朋友的数量。

### 4. 常见误区

*   **没有排序：** 如果不先对小朋友的胃口值和糖果大小进行排序，我们的贪心策略就无法生效，因为我们无法确定“最小的胃口”和“最小能满足的糖果”。
*   **错误的匹配逻辑：** 比如，总是尝试用最大的糖果去满足小朋友，或者没有考虑糖果和胃口值的大小关系。
*   **认为所有优化问题都能用贪心解决：** 贪心算法很强大，但它不是万能的。很多问题（比如背包问题、旅行商问题）需要更复杂的动态规划或其他算法来解决。

## 典型例题精讲

**题目：分发糖果 (Assign Cookies)**

假设你是一位仁慈的老师，你有一群孩子和一些饼干。每个孩子 `i` 有一个胃口值 `g[i]`，这是孩子 `i` 能够满足的最小饼干尺寸。每个饼干 `j` 有一个尺寸 `s[j]`。如果 `s[j] >= g[i]`，你可以把饼干 `j` 分给孩子 `i`。你的目标是最大化能够满足的孩子数量。每个饼干只能分发给一个孩子。

**输入格式：**
第一行一个整数 `N`，表示孩子数量。
第二行 `N` 个整数，表示每个孩子的胃口值 `g[0], g[1], ..., g[N-1]`。
第三行一个整数 `M`，表示饼干数量。
第四行 `M` 个整数，表示每个饼干的尺寸 `s[0], s[1], ..., s[M-1]`。

**输出格式：**
一个整数，表示能够满足的最大孩子数量。

**示例：**
**输入:**
3
1 2 3
2
1 1
**输出:**
1

**解释：**
孩子们的胃口值是 `[1, 2, 3]`。
饼干的尺寸是 `[1, 1]`。
你只能给胃口值为 1 的孩子分发一个尺寸为 1 的饼干。其他孩子无法满足。

**输入:**
3
1 2
3
1 2 3
**输出:**
2

**解释：**
孩子们胃口值是 `[1, 2]`。
饼干尺寸是 `[1, 2, 3]`。
你可以给胃口值为 1 的孩子分发尺寸为 1 的饼干。
你可以给胃口值为 2 的孩子分发尺寸为 2 的饼干。
总共可以满足 2 个孩子。

---

**思路分析：**

1.  **排序是关键：** 根据我们之前的分析，为了实现“先满足胃口最小的小朋友，并给他能满足他的最小的糖果”这个贪心策略，我们首先需要对孩子们的胃口值和饼干的尺寸都进行**升序排序**。
    *   排序 `g` 数组：`g[0]` 是最小胃口，`g[N-1]` 是最大胃口。
    *   排序 `s` 数组：`s[0]` 是最小饼干，`s[M-1]` 是最大饼干。

2.  **双指针遍历：** 排序后，我们可以使用两个指针，一个指向当前要考虑的孩子（`i`），另一个指向当前要考虑的饼干（`j`）。
    *   初始时，`i = 0` (指向胃口最小的孩子)，`j = 0` (指向最小尺寸的饼干)。
    *   我们维护一个计数器 `ans` 来记录已经满足的孩子数量，初始为 0。

3.  **贪心匹配：**
    *   当 `i < N` (还有孩子未被考虑) 并且 `j < M` (还有饼干未被使用) 时，进行循环：
        *   **如果当前饼干 `s[j]` 能够满足当前孩子 `g[i]` (即 `s[j] >= g[i]`)：**
            *   太好了！我们成功满足了一个孩子。
            *   `ans` 加 1。
            *   这个孩子已经被满足了，所以 `i` 指针向后移动一位，考虑下一个胃口更大的孩子。
            *   这颗饼干已经被用掉了，所以 `j` 指针也向后移动一位，考虑下一颗更大的饼干。
        *   **如果当前饼干 `s[j]` 不能满足当前孩子 `g[i]` (即 `s[j] < g[i]`)：**
            *   这颗小饼干对当前孩子来说太小了。
            *   而且，由于 `g` 数组是升序的，这颗饼干 `s[j]` 也肯定满足不了 `g[i]` 后面胃口更大的孩子们。
            *   所以，这颗 `s[j]` 饼干就没用了，我们直接跳过它，`j` 指针向后移动一位，尝试用下一颗更大的饼干来满足当前孩子 `g[i]`。
            *   注意：此时 `i` 指针不移动，因为当前孩子 `g[i]` 还没被满足。

4.  **循环结束：** 当 `i` 达到 `N` (所有孩子都被考虑过) 或者 `j` 达到 `M` (所有饼干都被用完或跳过) 时，循环结束。此时 `ans` 就是我们能满足的最大孩子数量。

**代码实现：**

```cpp
#include <iostream>  // 用于输入输出
#include <algorithm> // 用于排序函数 sort

// 定义数组的最大尺寸，防止越界
const int MAXN = 100005; // 最大孩子数量
const int MAXM = 100005; // 最大饼干数量

int g[MAXN]; // 存储孩子们的胃口值
int s[MAXM]; // 存储饼干的尺寸

int main() {
    // 优化输入输出，使程序运行更快
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 孩子数量
    std::cin >> n;
    for (int i = 0; i < n; ++i) {
        std::cin >> g[i]; // 读取每个孩子的胃口值
    }

    int m; // 饼干数量
    std::cin >> m;
    for (int i = 0; i < m; ++i) {
        std::cin >> s[i]; // 读取每个饼干的尺寸
    }

    // 第一步：对孩子们的胃口值和饼干尺寸进行升序排序
    std::sort(g, g + n); // 排序胃口值数组
    std::sort(s, s + m); // 排序饼干尺寸数组

    int ans = 0; // 记录能满足的孩子数量
    int i = 0;   // 指向当前孩子的指针（胃口值数组 g 的索引）
    int j = 0;   // 指向当前饼干的指针（尺寸数组 s 的索引）

    // 第二步：使用双指针进行贪心匹配
    // 当还有孩子未被满足，且还有饼干可供选择时，继续循环
    while (i < n && j < m) {
        // 如果当前饼干 s[j] 的尺寸足够满足当前孩子 g[i] 的胃口
        if (s[j] >= g[i]) {
            ans++; // 满足一个孩子，计数加一
            i++;   // 孩子指针后移，考虑下一个孩子
            j++;   // 饼干指针后移，这颗饼干已经用掉了
        } else {
            // 如果当前饼干 s[j] 太小，无法满足当前孩子 g[i]
            // 那么这颗饼干也无法满足胃口更大的孩子
            // 所以直接跳过这颗饼干，尝试下一颗更大的饼干
            j++;   // 饼干指针后移，当前孩子 g[i] 保持不变，等待更大的饼干
        }
    }

    std::cout << ans << std::endl; // 输出最终满足的孩子数量

    return 0;
}

```

**复杂度分析：**

*   **时间复杂度：**
    *   `std::sort(g, g + n)` 对 `N` 个元素进行排序，时间复杂度为 `O(N log N)`。
    *   `std::sort(s, s + m)` 对 `M` 个元素进行排序，时间复杂度为 `O(M log M)`。
    *   双指针 `while` 循环：两个指针 `i` 和 `j` 最多都只会遍历一次数组。所以这一部分的复杂度是 `O(N + M)`。
    *   **总时间复杂度：`O(N log N + M log M)`**。
        *   当 `N` 和 `M` 都很大时，排序是主要的时间消耗。

*   **空间复杂度：**
    *   我们使用了两个额外的数组 `g` 和 `s` 来存储输入数据，这部分空间是 `O(N + M)`。
    *   `std::sort` 函数在大多数实现中，如果对原生数组进行排序，通常是原地排序或使用少量辅助空间，例如 `O(log N)` 或 `O(N)` (取决于具体实现，例如 Introsort)。在竞赛中，我们通常认为其辅助空间是 `O(1)` 或 `O(log N)`。
    *   **总空间复杂度：`O(N + M)`** (主要用于存储输入数据)。

## 代码实现模板

这个模板展示了贪心算法中常用的“排序 + 双指针”模式。

```cpp
#include <iostream>  // 用于输入输出
#include <algorithm> // 用于 std::sort 排序函数
// #include <bits/stdc++.h> // 竞赛中常用，包含大部分标准库

// 定义常量，表示数组的最大可能大小，防止越界
const int MAX_SIZE_A = 100005; // 假设第一个数组的最大大小
const int MAX_SIZE_B = 100005; // 假设第二个数组的最大大小

// 全局数组，方便在函数中使用，避免传递大数组
int a[MAX_SIZE_A]; // 存储第一个集合的元素
int b[MAX_SIZE_B]; // 存储第二个集合的元素

int main() {
    // 提升C++标准输入输出流的效率
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n 为第一个集合的大小，m 为第二个集合的大小

    // 1. 读取输入数据到数组
    std::cin >> n;
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i]; // 读取第一个数组的元素
    }

    std::cin >> m;
    for (int i = 0; i < m; ++i) {
        std::cin >> b[i]; // 读取第二个数组的元素
    }

    // 2. 对两个数组进行排序
    // 贪心算法通常需要数据有序才能做出正确的局部最优选择
    std::sort(a, a + n); // 将数组 a 升序排序
    std::sort(b, b + m); // 将数组 b 升序排序

    // 3. 使用双指针（或其他遍历方式）实现贪心匹配逻辑
    int p1 = 0; // 指针1，用于遍历数组 a
    int p2 = 0; // 指针2，用于遍历数组 b
    int result = 0; // 存储贪心算法的结果，例如满足的数量、最大值等

    // 循环条件通常是两个指针都没有越界
    while (p1 < n && p2 < m) {
        // 贪心选择的核心逻辑：
        // 根据具体问题，判断当前 a[p1] 和 b[p2] 是否可以匹配
        // 或者哪个元素应该先被处理
        if (/* 匹配条件，例如 b[p2] >= a[p1] */ true) { // 替换为实际条件
            // 如果匹配成功：
            result++; // 更新结果
            p1++;     // 移动指针1，表示 a[p1] 已被处理
            p2++;     // 移动指针2，表示 b[p2] 已被处理
        } else {
            // 如果不匹配，根据贪心策略决定移动哪个指针：
            // 例如，如果 b[p2] 太小，无法满足 a[p1]，则移动 p2 寻找更大的 b
            p2++;
            // 或者，如果 a[p1] 需求太高，暂时无法满足，则移动 p1 寻找更容易满足的 a
            // p1++;
        }
    }

    // 4. 输出最终结果
    std::cout << result << std::endl;

    return 0;
}
```

**如何套用这个模板到“分发糖果”问题：**

*   `a` 数组对应 `g` (孩子胃口值)，`n` 对应孩子数量。
*   `b` 数组对应 `s` (饼干尺寸)，`m` 对应饼干数量。
*   `p1` 对应 `i` (孩子指针)，`p2` 对应 `j` (饼干指针)。
*   `result` 对应 `ans` (满足的孩子数量)。
*   匹配条件：`b[p2] >= a[p1]` (饼干尺寸足以满足孩子胃口)。
*   如果匹配成功：`result++`, `p1++`, `p2++`。
*   如果不匹配 (`b[p2] < a[p1]`)：`p2++` (饼干太小，换下一颗更大的饼干)。

通过这个模板，你会发现很多类似的贪心问题，都可以用“排序 + 双指针”的思路来解决！

**能力提升答案：**

1.  **数据范围与效率：**
    *   如果 `N` 和 `M` 都达到 `10^5`，代码可以在 1 秒内运行完成。
    *   时间复杂度是 `O(N log N + M log M)`。
    *   当 `N, M = 10^5` 时，`N log N` 大约是 `10^5 * log_2(10^5)`。`log_2(10^5)` 大约是 17。
    *   所以 `10^5 * 17 = 1.7 * 10^6` 次操作。
    *   `O(N + M)` 部分是 `10^5 + 10^5 = 2 * 10^5` 次操作。
    *   总操作次数约为 `1.7 * 10^6 + 1.7 * 10^6 + 2 * 10^5`，大约是 `3.6 * 10^6` 次。
    *   现代计算机每秒可以执行大约 `10^8` 次操作，所以 `3.6 * 10^6` 次操作远小于 `10^8`，可以在 1 秒内完成。

2.  **反向贪心：**

    ```cpp
    #include <iostream>
    #include <algorithm> // 需要 sort

    const int MAX_SIZE = 100005;
    int g[MAX_SIZE];
    int s[MAX_SIZE];

    int main() {
        std::ios_base::sync_with_stdio(false);
        std::cin.tie(NULL);

        int n;
        std::cin >> n;
        for (int i = 0; i < n; ++i) {
            std::cin >> g[i];
        }

        int m;
        std::cin >> m;
        for (int i = 0; i < m; ++i) {
            std::cin >> s[i];
        }

        // 对 g 数组进行降序排序 (从大到小)
        std::sort(g, g + n, std::greater<int>());
        // 对 s 数组进行降序排序 (从大到小)
        std::sort(s, s + m, std::greater<int>());

        int ans = 0;
        int i = 0; // 指向胃口最大的孩子
        int j = 0; // 指向尺寸最大的饼干

        while (i < n && j < m) {
            // 如果当前最大饼干 s[j] 能够满足当前胃口最大的孩子 g[i]
            if (s[j] >= g[i]) {
                ans++; // 满足一个孩子
                i++;   // 孩子指针后移，考虑下一个胃口较小的孩子
                j++;   // 饼干指针后移，这颗饼干已用
            } else {
                // 如果当前最大饼干 s[j] 都不能满足胃口最大的孩子 g[i]
                // 那么 g[i] 肯定无法被满足了（因为其他饼干都比 s[j] 小）
                // 所以我们只能放弃这个孩子，考虑下一个胃口较小的孩子
                i++;   // 孩子指针后移，当前饼干 s[j] 保持不变，留给胃口更小的孩子
            }
        }

        std::cout << ans << std::endl;
        return 0;
    }
    ```

**拓展挑战答案：**

1.  **最多能买多少件商品？**

    **思路：**
    为了购买尽可能多的商品，我们应该每次都选择当前最便宜的商品。这样可以最大限度地保留我们的预算，以便购买更多的商品。这是一个经典的贪心策略。

    1.  将所有商品的价格进行升序排序。
    2.  从最便宜的商品开始遍历，如果当前预算足够购买该商品，则购买它，并从预算中减去其价格，同时增加已购买商品数量。
    3.  继续遍历下一件商品，直到预算不足或所有商品都已遍历。

    ```cpp
    #include <iostream>
    #include <algorithm> // 需要 sort

    const int MAX_N = 100005; // 最大商品数量
    int c[MAX_N];             // 存储商品价格

    int main() {
        std::ios_base::sync_with_stdio(false);
        std::cin.tie(NULL);

        int b; // 预算
        std::cin >> b;

        int n; // 商品数量
        std::cin >> n;
        for (int i = 0; i < n; ++i) {
            std::cin >> c[i]; // 读取每个商品的价格
        }

        // 贪心策略：为了买到尽可能多的商品，应该优先购买便宜的商品
        // 所以首先对商品价格进行升序排序
        std::sort(c, c + n);

        int count = 0; // 记录购买的商品数量
        // 遍历排序后的商品价格
        for (int i = 0; i < n; ++i) {
            // 如果当前预算足够购买这件商品
            if (b >= c[i]) {
                b -= c[i]; // 从预算中减去商品价格
                count++;   // 购买数量加一
            } else {
                // 如果预算不足以购买当前商品，
                // 那么也无法购买更贵的商品了，直接退出循环
                break;
            }
        }

        std::cout << count << std::endl; // 输出最终购买的商品数量

        return 0;
    }
    ```
