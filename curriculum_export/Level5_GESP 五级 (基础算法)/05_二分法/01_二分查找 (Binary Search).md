**方法一：线性查找**
从字典的第一页开始，一页一页地往后翻，直到找到“algorithm”为止。
*   优点：简单粗暴，不管字典有没有顺序都能用。
*   缺点：如果“algorithm”在最后一页，那你就得翻完整本字典，太慢了！

**方法二：二分查找 (Binary Search)**
你肯定不会真的从头翻到尾，对不对？你会怎么做？
1.  你会先大致翻到字典的中间。
2.  看看中间的单词是“M”开头的，还是“A”开头的。
3.  如果中间是“M”，而“algorithm”是“A”开头的，你就会知道“algorithm”肯定在前半部分，于是你把字典的后半部分合上，只看前半部分。
4.  然后，你又在前半部分的中间再翻开，重复上面的步骤。
5.  这样，你每次都能排除掉一半的单词，是不是很快就能找到啦？

这就是我们今天要学习的“二分查找”的核心思想！它就像一个聪明的侦探，每次都能把搜索范围缩小一半，效率非常高。

那么问题来了，这种查找方法有什么前提条件吗？对了，字典里的单词必须是**按字母顺序排列**的，如果字典里的单词是乱七八糟的，这种方法就没法用了。所以，二分查找的第一个也是最重要的条件就是：**数据必须是有序的**！

## 知识点讲解

### 1. 二分查找的核心思想

二分查找，顾名思义，就是“二分”——每次把查找区间分成两半。它的核心思想是“分治”，通过不断缩小查找范围来定位目标元素。

**基本步骤：**
1.  **确定查找范围**：我们用两个指针 `l` (left) 和 `r` (right) 来表示当前查找的区间。最初，`l` 指向数组的第一个元素，`r` 指向数组的最后一个元素。
2.  **计算中间位置**：在 `l` 和 `r` 之间找到一个中间位置 `mid`。常用的计算方式是 `mid = l + (r - l) / 2`。
3.  **比较**：将 `a[mid]`（中间位置的元素）与我们要找的目标值 `x` 进行比较。
    *   如果 `a[mid] == x`：恭喜你，找到了！返回 `mid` 或者 `true`。
    *   如果 `a[mid] < x`：说明目标值 `x` 在 `mid` 的右边（因为数组是递增的），所以我们把查找范围缩小到 `mid` 的右半部分，即 `l = mid + 1`。
    *   如果 `a[mid] > x`：说明目标值 `x` 在 `mid` 的左边（因为数组是递增的），所以我们把查找范围缩小到 `mid` 的左半部分，即 `r = mid - 1`。
4.  **循环**：重复步骤 2 和 3，直到找到目标值，或者查找范围 `l > r`（表示查找范围为空，目标值不存在）。

### 2. 适用条件

二分查找**只能**用于**有序序列**（递增或递减）。如果数据是无序的，你需要先对数据进行排序，才能使用二分查找。

### 3. 时间复杂度

每次比较都会将查找范围缩小一半。假设初始有 N 个元素，第一次剩下 N/2，第二次剩下 N/4，第三次剩下 N/8，以此类推。直到剩下 1 个元素为止。这个过程大约需要 log₂N 次操作。
所以，二分查找的时间复杂度是 **O(log N)**。
与线性查找的 O(N) 相比，O(log N) 效率高得多！比如 N=1,000,000 时，线性查找可能需要 1,000,000 次操作，而二分查找只需要大约 log₂1,000,000 ≈ 20 次操作！是不是非常惊人？

### 4. 二分查找的常见变体

除了查找一个元素是否存在，二分查找还可以有很多灵活的应用：
*   **查找第一个等于 `x` 的元素的位置**：当数组中有重复元素时，我们可能需要找到第一个出现 `x` 的位置。
*   **查找最后一个等于 `x` 的元素的位置**：同理，找到最后一个出现 `x` 的位置。
*   **查找第一个大于等于 `x` 的元素的位置 (Lower Bound)**：在数组中找到第一个不小于 `x` 的元素。
*   **查找第一个大于 `x` 的元素的位置 (Upper Bound)**：在数组中找到第一个严格大于 `x` 的元素。

这些变体在实现时，主要是调整 `l` 和 `r` 的更新策略，以及循环结束后的返回结果。

## 代码示例

我们将从最基本的二分查找开始，然后看一个查找第一个等于目标值的变体。

### 示例一：基本二分查找 - 查找目标元素是否存在

这个函数会返回 `true` 如果找到目标值，否则返回 `false`。

```cpp
#include <bits/stdc++.h>
using namespace std;

// 功能：在有序数组 a 中查找目标值 x 是否存在
// 参数：
//   a[]: 一个已排序的整数数组
//   n: 数组的元素个数
//   x: 要查找的目标值
// 返回值：如果找到 x，返回 true；否则返回 false
bool binary_search_basic(int a[], int n, int x) {
    int l = 0;             // 查找范围的左边界，初始为数组第一个元素的下标
    int r = n - 1;         // 查找范围的右边界，初始为数组最后一个元素的下标

    while (l <= r) {       // 当左边界不大于右边界时，查找范围有效
        int mid = l + (r - l) / 2; // 计算中间位置，防止 l+r 溢出

        if (a[mid] == x) { // 如果中间元素就是目标值
            return true;   // 找到了，返回 true
        } else if (a[mid] < x) { // 如果中间元素小于目标值，说明目标在 mid 的右边
            l = mid + 1;   // 更新左边界为 mid + 1
        } else {           // 如果中间元素大于目标值，说明目标在 mid 的左边
            r = mid - 1;   // 更新右边界为 mid - 1
        }
    }
    return false;          // 循环结束，l > r，说明没有找到目标值
}

int main() {
    int a[] = {1, 3, 5, 7, 9, 11, 13};
    int n = sizeof(a) / sizeof(a[0]); // 计算数组元素个数

    int target1 = 7;
    int target2 = 6;

    if (binary_search_basic(a, n, target1)) {
        cout << "目标 " << target1 << " 存在于数组中。" << endl;
    } else {
        cout << "目标 " << target1 << " 不存在于数组中。" << endl;
    }

    if (binary_search_basic(a, n, target2)) {
        cout << "目标 " << target2 << " 存在于数组中。" << endl;
    } else {
        cout << "目标 " << target2 << " 不存在于数组中。" << endl;
    }

    return 0;
}
```

### 示例二：二分查找变体 - 查找第一个等于目标值的元素位置

如果数组中可能有重复元素，我们想找到目标值第一次出现的位置。

```cpp
#include <bits/stdc++.h>
using namespace std;

// 功能：在有序数组 a 中查找目标值 x 第一次出现的位置
// 参数：
//   a[]: 一个已排序的整数数组
//   n: 数组的元素个数
//   x: 要查找的目标值
// 返回值：如果找到 x，返回其第一个出现的位置的下标；否则返回 -1
int binary_search_first_occurrence(int a[], int n, int x) {
    int l = 0;             // 左边界
    int r = n - 1;         // 右边界
    int ans = -1;          // 记录找到的第一个位置，初始为 -1 (未找到)

    while (l <= r) {
        int mid = l + (r - l) / 2;

        if (a[mid] == x) { // 如果中间元素是目标值
            ans = mid;     // 记录当前位置，这可能是一个答案
            r = mid - 1;   // 但我们还要继续往左找，看有没有更靠前的 x
        } else if (a[mid] < x) { // 如果中间元素小于目标值，说明目标在 mid 的右边
            l = mid + 1;   // 更新左边界
        } else {           // 如果中间元素大于目标值，说明目标在 mid 的左边
            r = mid - 1;   // 更新右边界
        }
    }
    return ans;            // 返回找到的第一个位置
}

int main() {
    int a[] = {1, 3, 5, 5, 5, 7, 9, 11, 13}; // 数组中有重复元素 5
    int n = sizeof(a) / sizeof(a[0]);

    int target1 = 5;
    int target2 = 6;

    int pos1 = binary_search_first_occurrence(a, n, target1);
    if (pos1 != -1) {
        cout << "目标 " << target1 << " 第一次出现在下标 " << pos1 << " (值为 " << a[pos1] << ")。" << endl;
    } else {
        cout << "目标 " << target1 << " 不存在于数组中。" << endl;
    }

    int pos2 = binary_search_first_occurrence(a, n, target2);
    if (pos2 != -1) {
        cout << "目标 " << target2 << " 第一次出现在下标 " << pos2 << " (值为 " << a[pos2] << ")。" << endl;
    } else {
        cout << "目标 " << target2 << " 不存在于数组中。" << endl;
    }

    return 0;
}
```

**【小贴士】**
*   `l + (r - l) / 2` 这种计算 `mid` 的方式可以有效避免 `l + r` 过大导致整数溢出的问题，尤其是在 `l` 和 `r` 都非常大的时候。
*   循环条件 `l <= r` 是为了确保当 `l` 和 `r` 相等时（查找范围只剩一个元素），`mid` 仍然能指向这个元素并进行比较。
*   当 `a[mid] == x` 时，我们找到了一个可能的答案，但为了找到“第一个”，我们仍然需要向左边（`r = mid - 1`）继续搜索。

### 练习题目答案与解析

1.  **答案**：D. 有序排列的数组
    **解析**：二分查找的核心是每次将搜索范围缩小一半，这要求数据必须是按顺序排列的，这样才能根据中间元素的大小判断目标在左半边还是右半边。链表无法通过下标 O(1) 访问中间元素，所以不适合。

2.  **答案**：B. O(log N)
    **解析**：二分查找每次都将搜索范围减半，所以其时间复杂度是对数级的。

3.  **答案**：
    第一次循环：`mid = 3`，`a[mid] = 40`。
    第二次循环：`l = 0`, `r = 2` (这里是针对查找第一个等于 40 的情况，如果是基础查找，第一次就找到了)。
    最终找到 `40` 的下标是 `3`。
    **解析**：
    *   初始：`l=0, r=6`
    *   `mid = 0 + (6-0)/2 = 3`，`a[3] = 40`。找到了！下标为 `3`。

4.  **答案**：B. 前者可以避免 `l + r` 导致整数溢出
    **解析**：当 `l` 和 `r` 都非常大时（接近 `int` 类型的最大值），`l + r` 可能会超过 `int` 的最大表示范围，导致溢出。而 `r - l` 即使 `l` 和 `r` 都很大，它们的差值也不会超过 `int` 范围，因此 `l + (r - l) / 2` 更安全。

5.  **答案**：`l = mid + 1`
    **解析**：为了找到最后一个等于 `x` 的元素，当 `a[mid] == x` 时，我们记录下当前 `mid`，然后需要继续向右边搜索，看是否存在更靠后的 `x`，所以将 `l` 更新为 `mid + 1`。
