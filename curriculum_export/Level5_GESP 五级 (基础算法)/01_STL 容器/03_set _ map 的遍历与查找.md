## 深度知识点讲解

### 1. `set`：独一无二的集合

#### 1.1 `set`是什么？——“去重排序机”

想象你有一个超级智能的垃圾回收站，你把各种各样的垃圾（数据）扔进去，它会自动帮你把重复的垃圾（相同的数据）只保留一份，并且还能把剩下的垃圾按照大小（从小到大）排得整整齐齐。这个“垃圾回收站”就是`set`。

**本质**：`set`是C++标准库中的一种关联容器，它存储的是**互不相同的元素**，并且这些元素通常是**自动排序**的（默认为升序）。

**生活类比**：
*   **电话本里的联系人列表**：每个人的名字都是独一无二的，你不会把同一个人的名字存两次。而且通常电话本是按字母顺序排列的。
*   **班级花名册**：每个同学的名字只出现一次。
*   **扑克牌中的牌面点数**：一副牌中，每种点数（A, 2, ..., K）都是独一无二的，你不会有两张完全一样的牌（比如两张红桃A）。

#### 1.2 为什么需要`set`？——“查找快，去重省心”

`set`最主要的优点是：
1.  **自动去重**：你不需要手动检查元素是否已经存在，`set`会帮你搞定。
2.  **快速查找**：判断一个元素是否在`set`中非常快。就像你在一个已经排好序的电话本里找一个名字，可以很快翻到。它的查找效率通常是O(log N)，比遍历整个数组快很多。
3.  **自动排序**：元素总是保持有序状态，方便遍历。

#### 1.3 `set`怎么工作的（简单原理）？

`set`内部通常是基于一种叫做“**平衡二叉搜索树**”（Balanced Binary Search Tree，例如红黑树）的数据结构实现的。
*   **二叉搜索树**：左子树的元素都比根节点小，右子树的元素都比根节点大。这样，查找时可以每次排除一半的数据，非常快。
*   **平衡**：为了防止树变得“一边倒”（退化成链表），影响查找效率，它会进行自动调整，保持树的平衡，确保查找效率始终是O(log N)。

#### 1.4 GSP 5级限制与替代方案：用数组模拟`set`

**重要提示：** `set`是STL容器，根据GSP 1-5级的规定，**严禁使用**！在GSP 5级考试中，我们需要用原生数组和允许的算法来模拟`set`的功能。

**如何用数组模拟`set`？**
1.  **存储元素**：用一个普通数组`a[]`来存储所有输入数据。
2.  **去重**：
    *   首先，对数组进行**排序**。`sort(a, a + n)`。
    *   然后，**遍历排序后的数组**。从第二个元素开始，如果当前元素和前一个元素不同，就把它加入到“去重后的新数组”中。
    *   或者，更简洁地，直接在原数组上进行“**原地去重**”：
        ```cpp
        // 假设数组a已经排序，n是元素个数
        int k = 0; // k用来记录不重复元素的个数，也是新数组的下标
        if (n > 0) {
            a[k++] = a[0]; // 第一个元素肯定不重复
            for (int i = 1; i < n; ++i) {
                if (a[i] != a[i-1]) { // 如果当前元素和前一个不同
                    a[k++] = a[i];    // 就把它加入到不重复的序列中
                }
            }
        }
        // 此时，a[0]到a[k-1]就是不重复且有序的元素
        // 更新n = k;
        ```
3.  **查找元素**：由于数组已经排序，我们可以使用**二分查找（Binary Search）**来快速判断一个元素是否存在。
    *   二分查找的效率也是O(log N)。

#### 1.5 `set`常见误区
*   误以为`set`可以存储重复元素。
*   误以为`set`内部是无序的（实际上它是有序的）。

### 2. `map`：智能字典（键值对映射）

#### 2.1 `map`是什么？——“智能配对器”

想象你有一个超级智能的配对器，你给它一个“钥匙”（Key），它就能立刻给你一个“答案”（Value）。比如，你给它“西游记”，它就给你“吴承恩”；你给它“红楼梦”，它就给你“曹雪芹”。这个“配对器”就是`map`。

**本质**：`map`是C++标准库中的一种关联容器，它存储的是**键值对（key-value pair）**。每个键（Key）都是**独一无二**的，并且通过键可以快速地查找或修改其对应的值（Value）。键通常是**自动排序**的。

**生活类比**：
*   **英汉词典**：英文单词是键，中文解释是值。通过英文单词可以快速查到中文解释。
*   **身份证号与姓名**：身份证号是键（独一无二），姓名是值。通过身份证号快速找到对应的姓名。
*   **电话号码与联系人姓名**：电话号码是键，联系人姓名是值。
*   **物品价格表**：商品名称是键，价格是值。

#### 2.2 为什么需要`map`？——“快速映射，方便关联”

`map`最主要的优点是：
1.  **快速查找**：通过键可以非常快地找到对应的值，效率通常是O(log N)。
2.  **键值关联**：将两个相关联的数据（如书名和作者）绑定在一起，方便管理。
3.  **键的唯一性**：保证每个键都是独一无二的，避免混淆。

#### 2.3 `map`怎么工作的（简单原理）？

`map`的内部实现和`set`非常相似，也是基于“**平衡二叉搜索树**”。区别在于，`set`只存储元素本身，而`map`存储的是`pair<Key, Value>`这样的键值对。树的排序和查找是根据键（Key）来进行的。

#### 2.4 GSP 5级限制与替代方案：用数组模拟`map`

**重要提示：** `map`也是STL容器，根据GSP 1-5级的规定，**严禁使用**！在GSP 5级考试中，我们需要用原生数组来模拟`map`的功能。

**如何用数组模拟`map`？**

这取决于键（Key）的类型和范围：

1.  **如果键是小范围的整数（例如0到100000）**：
    *   我们可以直接使用一个普通数组作为“**哈希表**”或“**计数器**”。数组的**下标**就是键（Key），数组中存储的**值**就是对应的值（Value）。
    *   例如，统计每个数字出现的次数：`int count[100001] = {0};`
        *   `count[x]++` 表示数字`x`出现了一次。
        *   `count[x]`就是数字`x`出现的总次数。
    *   这种方法查找和更新效率是O(1)（常数时间），非常快。

2.  **如果键是较大范围的整数，或者不是整数（例如字符串），或者键的数量很少但值域很大**：
    *   **模拟键值对数组**：我们可以使用两个（或多个）并行的数组，一个数组存储键，另一个数组存储对应的值。
        *   `int keys[N];`
        *   `int values[N];`
        *   当需要查找某个键对应的值时：
            *   首先，将`keys`数组进行**排序**。
            *   然后，使用**二分查找**在`keys`数组中找到目标键的下标。
            *   找到下标后，`values[下标]`就是对应的值。
        *   如果键不是整数，例如字符串，那么`keys`数组需要是`string`类型（但GSP 5级通常不涉及`string`作为键，更倾向于整数键）。

#### 2.5 `map`常见误区
*   误以为`map`的键可以重复（键是唯一的）。
*   误以为`map`的查找效率是O(N)（实际上是O(log N)）。
*   误以为`map`的键和值可以是任意类型（实际上键需要支持比较操作，值可以是任意类型）。

**总结：** 虽然`set`和`map`是强大的“魔法盒子”，能大大提高编程效率，但在GSP 5级，我们要学会用“手工工具”——数组、排序、二分查找——来模拟它们的功能。这不仅能帮助我们通过考试，更能让我们深入理解这些高级工具的底层原理，为将来学习更复杂的算法和数据结构打下坚实的基础！

### 例题一：统计不重复数字的个数

**题目描述**：给定N个整数，请你统计其中有多少个不重复的数字。

**输入**：
第一行一个整数N (1 <= N <= 100000)。
第二行N个整数，每个整数的范围是1到1000000000。

**输出**：
一个整数，表示不重复数字的个数。

---

**思路分析 (GESP 5级允许的数组方法)**：

1.  **接收输入**：首先，我们需要一个数组来存储这N个输入的整数。
2.  **排序**：为了方便去重，我们应该对这个数组进行排序。排序后，所有相同的数字都会紧挨在一起。
3.  **去重计数**：
    *   初始化一个计数器`c`为0。
    *   如果数组不为空，那么第一个元素肯定是不重复的，`c`增加1。
    *   从第二个元素开始遍历数组。如果当前元素与前一个元素不同，说明这是一个新的不重复数字，`c`增加1。
    *   如果当前元素与前一个元素相同，则跳过，因为它是一个重复的数字。

**代码实现 (GESP 5级)**：

```cpp
#include <iostream>
#include <algorithm> // 包含sort函数

int a[100005]; // 定义一个足够大的全局数组

int main() {
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    if (n == 0) { // 特殊情况：没有数字
        std::cout << 0 << std::endl;
        return 0;
    }

    // 步骤1: 排序
    std::sort(a, a + n);

    // 步骤2: 去重计数
    int c = 1; // 至少有一个不重复数字（第一个）
    for (int i = 1; i < n; ++i) {
        if (a[i] != a[i-1]) { // 如果当前数字与前一个不同
            c++;              // 计数器增加
        }
    }

    std::cout << c << std::endl;

    return 0;
}
```

**复杂度分析 (GESP 5级)**：
*   **时间复杂度**：
    *   读取输入：O(N)
    *   排序：O(N log N) (使用`std::sort`，基于快排或归并排序)
    *   去重计数：O(N)
    *   总时间复杂度：**O(N log N)**
*   **空间复杂度**：
    *   存储数组：O(N)
    *   总空间复杂度：**O(N)**

---

**`set`解决方案 (高级知识点，GSP 5级**严禁使用**)**：

```cpp
#include <iostream>
#include <set> // 包含set容器

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    std::set<int> s; // 定义一个int类型的set

    for (int i = 0; i < n; ++i) {
        int x;
        std::cin >> x;
        s.insert(x); // 将元素插入set，set会自动处理去重
    }

    std::cout << s.size() << std::endl; // set的size()方法返回不重复元素的个数

    return 0;
}
```

**复杂度分析 (`set`方案)**：
*   **时间复杂度**：
    *   每次插入元素：O(log K)，其中K是`set`中当前元素的数量。总共N次插入，所以是O(N log N)。
    *   获取大小：O(1)。
    *   总时间复杂度：**O(N log N)**
*   **空间复杂度**：
    *   `set`存储元素：O(N) (最坏情况下所有元素都不同)
    *   总空间复杂度：**O(N)**

**对比与总结**：
可以看到，虽然`set`代码更简洁，但其内部实现（平衡二叉搜索树）的效率与我们手动排序+去重数组的效率在时间复杂度上是相同的（都是O(N log N)）。这说明了掌握基础算法的重要性！

### 例题二：统计数字出现频率

**题目描述**：给定N个整数，请你统计每个数字出现的次数。最后按照数字从小到大的顺序输出每个数字及其出现次数。
**输入**：
第一行一个整数N (1 <= N <= 100000)。
第二行N个整数，每个整数`x`的范围是1到100000。

**输出**：
若干行，每行两个整数，表示数字`x`和它出现的次数`c`，按`x`从小到大排序。

---

**思路分析 (GESP 5级允许的数组方法)**：

由于题目中数字的范围是1到100000，这是一个相对较小的正整数范围。我们可以利用数组的下标来直接表示数字，数组的值来存储该数字的出现次数。

1.  **定义计数数组**：创建一个足够大的整数数组，例如`int count[100001]`，并将其所有元素初始化为0。数组的下标`i`就代表数字`i`。
2.  **遍历输入并计数**：读取每一个输入的数字`x`，然后将`count[x]`的值加1。
3.  **遍历计数数组并输出**：从下标1到100000遍历`count`数组。如果`count[i]`的值大于0，说明数字`i`出现过，就输出`i`和`count[i]`。

**代码实现 (GESP 5级)**：

```cpp
#include <iostream>

int count[100001]; // 定义一个足够大的全局数组，用于计数，下标i代表数字i

int main() {
    std::ios_base::sync_with_stdio(false); // 提高输入输出效率
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    int max_val = 0; // 记录出现的最大数字，方便后续遍历
    for (int i = 0; i < n; ++i) {
        int x;
        std::cin >> x;
        count[x]++; // 对应数字的计数器加1
        if (x > max_val) {
            max_val = x;
        }
    }

    // 遍历计数数组，按照数字从小到大的顺序输出
    for (int i = 1; i <= max_val; ++i) { // 从1遍历到出现过的最大数字
        if (count[i] > 0) { // 如果这个数字出现过
            std::cout << i << " " << count[i] << std::endl;
        }
    }

    return 0;
}
```

**复杂度分析 (GESP 5级)**：
*   **时间复杂度**：
    *   读取输入并计数：O(N)
    *   遍历计数数组并输出：O(MaxVal)，其中MaxVal是数字的最大可能值（这里是100000）。
    *   总时间复杂度：**O(N + MaxVal)**
*   **空间复杂度**：
    *   存储计数数组：O(MaxVal)
    *   总空间复杂度：**O(MaxVal)**

---

**`map`解决方案 (高级知识点，GSP 5级**严禁使用**)**：

```cpp
#include <iostream>
#include <map> // 包含map容器

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n;
    std::cin >> n;

    std::map<int, int> m; // 定义一个键为int，值为int的map

    for (int i = 0; i < n; ++i) {
        int x;
        std::cin >> x;
        m[x]++; // 如果x不存在，则创建并初始化为0再加1；如果x存在，则直接加1
    }

    // map会自动按照键（数字）从小到大排序
    // 遍历map并输出
    for (auto const& pair : m) { // C++11 range-based for loop
        std::cout << pair.first << " " << pair.second << std::endl;
    }
    // 也可以使用迭代器遍历：
    // for (std::map<int, int>::iterator it = m.begin(); it != m.end(); ++it) {
    //     std::cout << it->first << " " << it->second << std::endl;
    // }

    return 0;
}
```

**复杂度分析 (`map`方案)**：
*   **时间复杂度**：
    *   每次插入/更新元素：O(log K)，其中K是`map`中当前键值对的数量。总共N次操作，所以是O(N log N)。
    *   遍历`map`：O(M)，其中M是`map`中键值对的数量（即不重复数字的个数）。
    *   总时间复杂度：**O(N log N)**
*   **空间复杂度**：
    *   `map`存储键值对：O(M) (M是不重复数字的个数)
    *   总空间复杂度：**O(M)**

**对比与总结**：
对于本题，当数字范围较小时（如100000），使用数组作为计数器是最高效的，时间复杂度为O(N + MaxVal)。而`map`的O(N log N)在N很大时可能会慢于数组计数。但如果数字范围非常大（比如1到10亿），或者键不是整数（比如字符串），那么`map`的优势就体现出来了，因为它不需要预先分配一个巨大甚至无法分配的数组，只存储实际存在的键值对。

在GSP 5级，我们应该优先考虑数组计数这种简单高效的方法，因为它完美符合了题目的限制和要求。

## 代码实现模板

由于GSP 5级严格限制STL容器，我们主要提供基于原生数组的模板。这里会给出`set`和`map`概念对应的数组模拟模板。

### 1. 模拟`set`功能的模板（去重与查找）

**核心思想**：使用排序数组 + 二分查找。

```cpp
#include <iostream>
#include <algorithm> // 包含 std::sort 和 std::binary_search

// 定义一个全局数组，用于存储数据。根据题目要求设置大小。
// GESP 题目通常会给出 N 的范围，例如 N <= 100000
int a[100005]; 

// 二分查找函数：判断 val 是否在数组 a 的 [left, right] 区间内
// 注意：该函数要求数组 a 在 [left, right] 区间内是有序的
bool binarySearch(int left, int right, int val) {
    while (left <= right) {
        int mid = left + (right - left) / 2; // 防止溢出
        if (a[mid] == val) {
            return true; // 找到
        } else if (a[mid] < val) {
            left = mid + 1; // 在右半部分查找
        } else {
            right = mid - 1; // 在左半部分查找
        }
    }
    return false; // 未找到
}

int main() {
    // 优化输入输出，GSP 常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 元素个数
    std::cin >> n;

    // 读取所有元素到数组
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    // 步骤1: 对数组进行排序
    std::sort(a, a + n); // 将数组 a 的前 n 个元素排序

    // 步骤2: 去重
    // k 用来记录去重后数组的有效元素个数（也是下一个不重复元素的存放位置）
    int k = 0; 
    if (n > 0) { // 如果有元素，至少第一个元素是不重复的
        a[k++] = a[0]; // 将第一个元素放入去重后的序列
        for (int i = 1; i < n; ++i) {
            if (a[i] != a[i-1]) { // 如果当前元素与前一个不同
                a[k++] = a[i];    // 则将其加入去重后的序列
            }
        }
    }
    // 此时，数组 a 的前 k 个元素就是所有不重复且有序的元素
    // 我们可以将 n 更新为 k，表示现在有效元素有 k 个
    n = k; 

    // 示例：输出不重复元素的个数
    std::cout << "不重复元素的个数: " << n << std::endl;

    // 示例：查找某个元素是否存在 (使用自定义的二分查找函数)
    int target_val = 100; // 假设要查找的元素
    if (binarySearch(0, n - 1, target_val)) {
        std::cout << target_val << " 存在于集合中。" << std::endl;
    } else {
        std::cout << target_val << " 不存在于集合中。" << std::endl;
    }

    // 示例：查找某个元素是否存在 (使用STL的二分查找函数，GSP 5级允许)
    // std::binary_search 返回 bool 值，判断元素是否存在
    // 需要注意的是，std::binary_search 要求区间 [first, last) 是已排序的
    int another_target = 50;
    if (std::binary_search(a, a + n, another_target)) {
        std::cout << another_target << " 存在于集合中 (STL binary_search)。" << std::endl;
    } else {
        std::cout << another_target << " 不存在于集合中 (STL binary_search)。" << std::endl;
    }

    return 0;
}
```

### 2. 模拟`map`功能的模板（计数与键值映射）

**核心思想**：根据键的范围，使用数组作为计数器或并行数组。

#### 2.1 键是小范围整数时的计数器模板

**适用场景**：键（数字）的范围在0到某个不太大的值（例如1000000以内）。

```cpp
#include <iostream>

// 定义一个全局数组作为计数器。
// 数组下标代表键（数字），数组值代表该键出现的次数。
// 数组大小根据题目中数字的最大范围来设定，例如最大数字为 100000，则大小为 100001
int cnt[100001]; 

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 元素个数
    std::cin >> n;

    int max_key_seen = 0; // 记录出现过的最大键，方便后续遍历
    // 步骤1: 遍历输入，进行计数
    for (int i = 0; i < n; ++i) {
        int x; // 当前输入的数字（键）
        std::cin >> x;
        cnt[x]++; // 对应数字的计数器加1
        if (x > max_key_seen) {
            max_key_seen = x;
        }
    }

    // 步骤2: 遍历计数数组，输出键值对
    // 从最小可能的键（通常是1或0）遍历到出现过的最大键
    for (int i = 1; i <= max_key_seen; ++i) { 
        if (cnt[i] > 0) { // 如果这个键（数字i）出现过
            // 输出键 i 及其对应的值 cnt[i]
            std::cout << i << " 出现了 " << cnt[i] << " 次。" << std::endl;
        }
    }

    // 示例：查找某个键的值
    int query_key = 50;
    std::cout << "数字 " << query_key << " 出现了 " << cnt[query_key] << " 次。" << std::endl;

    return 0;
}
```

#### 2.2 键是较大范围整数或需要存储更多信息时的并行数组模板

**适用场景**：键的范围太大不适合直接用数组下标，或者每个键需要对应存储多个值。

```cpp
#include <iostream>
#include <algorithm> // 包含 std::sort

// 定义两个并行数组，一个存储键，一个存储对应的值
// 注意：这里的键-值对数量可能比总输入 N 少（如果键有重复）
// 为了简化，我们假设键值对的数量最多和 N 相同
int keys[100005];   // 存储所有不重复的键
int values[100005]; // 存储对应键的值（例如计数）

// 结构体用于存储键值对，方便排序（GSP 5级通常不直接用结构体排序，但概念上理解）
// struct Pair {
//     int key;
//     int value;
// };
// Pair data[100005];

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n; // 输入的总元素个数
    std::cin >> n;

    // 为了使用并行数组模拟 map，我们需要先将所有键收集起来，并进行去重和计数
    // 这里我们先用一个临时数组存储所有输入，再进行处理
    int raw_input[100005];
    for (int i = 0; i < n; ++i) {
        std::cin >> raw_input[i];
    }

    // 步骤1: 对原始输入进行排序
    std::sort(raw_input, raw_input + n);

    // 步骤2: 遍历排序后的输入，构建 keys 和 values 数组
    int unique_count = 0; // 记录不重复键值对的数量
    if (n > 0) {
        keys[unique_count] = raw_input[0]; // 第一个键
        values[unique_count] = 1;          // 出现1次
        unique_count++;

        for (int i = 1; i < n; ++i) {
            if (raw_input[i] == raw_input[i-1]) {
                // 如果当前键和前一个相同，则将前一个键的计数加1
                values[unique_count-1]++;
            } else {
                // 如果是新的键
                keys[unique_count] = raw_input[i];
                values[unique_count] = 1;
                unique_count++;
            }
        }
    }

    // 此时，keys[0...unique_count-1] 存储了所有不重复的键（已排序）
    // values[0...unique_count-1] 存储了对应键的计数

    // 步骤3: 遍历键值对并输出
    for (int i = 0; i < unique_count; ++i) {
        std::cout << "键: " << keys[i] << ", 值: " << values[i] << std::endl;
    }

    // 示例：查找某个键的值
    // 由于 keys 数组已经排序，我们可以使用二分查找来找到键的下标
    int target_key = 50;
    int found_idx = -1;
    int l = 0, r = unique_count - 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (keys[mid] == target_key) {
            found_idx = mid;
            break;
        } else if (keys[mid] < target_key) {
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }

    if (found_idx != -1) {
        std::cout << "键 " << target_key << " 的值为: " << values[found_idx] << std::endl;
    } else {
        std::cout << "键 " << target_key << " 不存在。" << std::endl;
    }

    return 0;
}
```

**Allman风格大括号说明**：
在上述代码模板中，所有的开括号`{`都单独占一行，与`if`, `for`, `while`等关键字垂直对齐，这种风格被称为Allman风格，有助于代码阅读和结构清晰。
