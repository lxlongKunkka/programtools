## 深度知识点讲解

### 什么是“滑动窗口”？

想象你手上有一个可以移动的“窗户”，这个窗户固定在一个长长的“序列”（比如一个数组）上。这个窗户可以向右“滑动”，窗户里面框住的就是我们当前关注的“连续子序列”。

*   **窗户的“左边框”**：我们用一个指针 `l` (left) 来表示。
*   **窗户的“右边框”**：我们用一个指针 `r` (right) 来表示。

这两个指针都从序列的左边开始，并且都向右移动，所以我们叫它“同向双指针”。当它们框住一段区域时，就像一个窗户在滑动，所以也叫“滑动窗口”。

### 为什么需要“滑动窗口”？

在很多问题中，我们需要找到一个**连续的子数组**，它满足某种条件（比如最大和、最小长度、包含特定元素等）。

**暴力解法**通常是这样的：
1.  用一个外层循环 `i` 遍历所有可能的起始点。
2.  用一个内层循环 `j` 遍历所有可能的结束点。
3.  对于每个 `[i, j]` 区间，再用一个循环计算它的性质（比如求和）。
这样下来，时间复杂度通常是 O(N^3)，如果优化一下求和，也至少是 O(N^2)。当 N 很大时（比如 N=100000），O(N^2) 就会超时（10^10 次操作）。

**滑动窗口**的优势在于：当窗户从 `[l, r]` 滑动到 `[l+1, r+1]` 或者 `[l, r+1]`、`[l+1, r]` 时，我们不是重新计算整个新窗口的性质，而是**基于旧窗口的性质进行增量更新**。
比如，从 `[l, r]` 变成 `[l, r+1]`，只需要加上 `a[r+1]` 的值；从 `[l, r]` 变成 `[l+1, r]`，只需要减去 `a[l]` 的值。这样，每个元素最多被 `l` 和 `r` 访问两次（`r` 经过一次，`l` 经过一次），所以总的时间复杂度可以降到 **O(N)**，效率大大提升！

### 滑动窗口的通用步骤（核心思想）

1.  **初始化**：
    *   `l = 0` (左指针指向数组起始位置)
    *   `r = 0` (右指针指向数组起始位置)
    *   `current_sum = 0` (或 `current_count` 等，用来维护当前窗口的性质)
    *   `ans = 初始值` (用来记录我们最终要找的答案，比如最大长度、最小和等)

2.  **右指针 `r` 扩张窗口**：
    *   `r` 从 `0` 开始，每次向右移动一步 (`r++`)，将 `a[r]` 加入到 `current_sum` 中，表示窗口向右扩张。
    *   `current_sum += a[r]`

3.  **左指针 `l` 收缩窗口（满足条件时）**：
    *   在 `r` 扩张窗口后，我们需要检查当前窗口 `[l, r]` 是否满足题目要求。
    *   如果**不满足**（比如 `current_sum` 太大了，超过了限制），那么 `l` 就需要向右移动 (`l++`)，同时将 `a[l]` 从 `current_sum` 中减去，直到窗口再次满足条件。
    *   `while (条件不满足)`:
        *   `current_sum -= a[l]`
        *   `l++`

4.  **更新答案 `ans`**：
    *   每次 `r` 扩张窗口，并且（或者在 `l` 调整后）窗口满足条件时，我们就可以用当前窗口 `[l, r]` 的信息来更新我们的 `ans`。
    *   比如，如果找最大长度，`ans = max(ans, r - l + 1)`。

5.  **循环结束**：
    *   当 `r` 遍历完整个数组（`r` 达到数组末尾）时，循环结束。

### 生活类比：吃自助餐

想象你去吃自助餐，你有一个盘子（这就是你的**窗口**）。
*   `l` 是你盘子左边第一道菜的位置。
*   `r` 是你盘子右边最后一道菜的位置。

你想要吃一盘菜，它的总“美味指数”要达到某个值 `S`。
1.  你从第一道菜开始拿 (`r` 往前走)，每拿一道菜，盘子里的“美味指数”就增加了。
2.  如果盘子里的“美味指数”还没达到 `S`，你就继续往右拿菜 (`r++`)。
3.  如果盘子里的“美味指数”已经**超过了** `S`（或者达到了 `S` 但你想找更短的盘子），你就得从盘子左边丢掉一些菜 (`l` 往前走)，直到盘子里的“美味指数”又变得合适。
4.  每次盘子里的菜合适的时候，你就记录下这盘菜的长度或者其他信息。

整个过程，你只是在盘子的两端进行加减，而不用每次都把盘子里的菜全部倒掉再重新盛一盘。这就是滑动窗口的精髓！

### 关键点剖析与常见误区

*   **`while` 循环还是 `if` 判断？**
    *   `r` 扩张窗口通常是 `for` 循环，因为它要遍历所有元素。
    *   `l` 收缩窗口通常是 `while` 循环，因为可能需要连续收缩多次才能满足条件。
*   **边界条件**：
    *   `l` 和 `r` 的初始值。
    *   `r - l + 1` 是窗口的长度。
    *   当 `l` 移动时，确保 `l <= r`。
*   **更新答案的位置**：
    *   通常在 `l` 调整完毕，窗口满足条件后，更新一次 `ans`。
    *   有些问题在 `r` 扩张后立刻更新 `ans` 也可以（比如固定长度窗口）。
*   **数据类型**：注意 `current_sum` 可能很大，需要使用 `long long`。
*   **空窗口的处理**：根据题目要求，有些问题允许空窗口或单元素窗口，有些不允许。

## 典型例题精讲

### 例题一：固定长度的滑动窗口 - 查找和最大的子数组

**题目描述**：给定一个整数数组 `a` 和一个整数 `k`，找出数组中长度为 `k` 的连续子数组的最大总和。

**示例**：
输入: `a = [1, 3, -1, -3, 5, 3, 6, 7]`, `k = 3`
输出: `16`
解释:
长度为 3 的子数组及其和：
`[1, 3, -1]` -> 3
`[3, -1, -3]` -> -1
`[-1, -3, 5]` -> 1
`[-3, 5, 3]` -> 5
`[5, 3, 6]` -> 14
`[3, 6, 7]` -> 16 (最大)

**思路分析**：

1.  **暴力解法**：
    *   外层循环 `i` 从 `0` 到 `n - k`，表示子数组的起始位置。
    *   内层循环 `j` 从 `i` 到 `i + k - 1`，计算当前子数组的和。
    *   记录最大和。
    *   时间复杂度：O(N * K)，因为有 N-K 个子数组，每个计算和需要 K 次操作。

2.  **滑动窗口解法**：
    *   我们想要一个长度固定为 `k` 的窗口。
    *   **初始化**：
        *   `current_sum = 0`，用来记录当前窗口的和。
        *   `max_sum = 负无穷大` (或者先计算第一个窗口的和)。
        *   `l = 0`，`r = 0`。
    *   **构建第一个窗口**：
        *   先让 `r` 移动 `k` 次，把前 `k` 个元素加到 `current_sum` 中。此时 `r` 指向 `k-1`。
        *   `max_sum` 更新为 `current_sum`。
    *   **滑动窗口**：
        *   现在，`l` 指向 `0`，`r` 指向 `k-1`。
        *   `r` 继续向右移动 (`r++`)，每次移动一步。
        *   当 `r` 移动到新位置 `r'` 时，新的窗口是 `[l+1, r']`。
        *   为了得到新窗口的和，我们只需要：
            *   减去旧窗口最左边的元素 `a[l]`。
            *   加上新窗口最右边的元素 `a[r']`。
            *   `current_sum = current_sum - a[l] + a[r']`
            *   同时，`l` 也向右移动一步 (`l++`)。
        *   每次 `current_sum` 更新后，都和 `max_sum` 比较，更新 `max_sum`。
    *   **循环结束**：当 `r` 到达数组末尾时，所有长度为 `k` 的子数组都已检查完毕。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含常用头文件，如 iostream, algorithm 等

int main() {
    std::ios::sync_with_stdio(false); // 加速输入输出
    std::cin.tie(0); // 解除cin和cout的绑定

    int n; // 数组的长度
    std::cin >> n;

    int a[n]; // 定义数组a
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    int k; // 固定窗口的长度
    std::cin >> k;

    if (n < k) { // 如果数组长度小于k，则无法形成长度为k的子数组
        std::cout << "Invalid input: array length < k" << std::endl;
        return 0;
    }

    long long currentSum = 0; // 当前窗口的和，可能很大，用 long long
    // 初始化第一个窗口的和 (a[0] 到 a[k-1])
    for (int i = 0; i < k; ++i) {
        currentSum += a[i];
    }

    long long maxSum = currentSum; // 第一个窗口的和就是当前的最小最大和

    // 滑动窗口
    // l 指针代表窗口的起始位置
    // r 指针代表窗口的结束位置 (当 r 移动时，它是新加入窗口的元素)
    // 初始时，窗口是 [0, k-1]，所以 l=0, r=k-1
    // 下一次滑动，r 会走到 k，l 会走到 1，窗口变为 [1, k]
    for (int r = k; r < n; ++r) {
        // 窗口向右移动一步
        // 减去窗口最左边的元素 a[r-k] (即旧的 l 位置的元素)
        // 加上新进入窗口的元素 a[r]
        currentSum = currentSum - a[r - k] + a[r];
        
        // 更新最大和
        if (currentSum > maxSum) {
            maxSum = currentSum;
        }
    }

    std::cout << maxSum << std::endl;

    return 0;
}
```

**复杂度分析**：
*   **时间复杂度**：O(N)。初始化第一个窗口需要 O(K) 次操作，之后滑动窗口的循环执行 N-K 次，每次循环是常数操作 O(1)。所以总共是 O(K + (N-K)*1) = O(N)。
*   **空间复杂度**：O(1) (不计算输入数组的空间)。

### 例题二：可变长度的滑动窗口 - 查找最短的满足条件的子数组

**题目描述**：给定一个正整数数组 `a` 和一个正整数 `S`，找出数组中和**大于或等于** `S` 的最短连续子数组的长度。如果不存在，则返回 `0`。

**示例**：
输入: `a = [2, 3, 1, 2, 4, 3]`, `S = 7`
输出: `2`
解释: 子数组 `[4, 3]` 的和为 7，是符合条件的长度最短的子数组。

**思路分析**：

1.  **暴力解法**：
    *   外层循环 `i` 遍历所有起始点。
    *   内层循环 `j` 遍历所有结束点。
    *   计算 `[i, j]` 的和，如果和 `>= S`，则更新最短长度。
    *   时间复杂度：O(N^2)。

2.  **滑动窗口解法**：
    *   窗口的长度是可变的，目标是找到**最短**的。
    *   **初始化**：
        *   `l = 0` (左指针)
        *   `current_sum = 0` (当前窗口的和)
        *   `min_len = N + 1` (一个比任何可能长度都大的值，表示目前还没找到)
    *   **右指针 `r` 扩张窗口**：
        *   `r` 从 `0` 开始遍历整个数组。
        *   每次 `r` 移动，将 `a[r]` 加入 `current_sum`。
    *   **左指针 `l` 收缩窗口（当 `current_sum >= S` 时）**：
        *   当 `current_sum` 达到或超过 `S` 时，说明当前窗口 `[l, r]` 是一个符合条件的子数组。
        *   我们尝试让窗口更短：记录当前窗口的长度 `r - l + 1`，并与 `min_len` 比较，更新 `min_len`。
        *   然后，为了寻找更短的子数组，我们尝试将窗口左边界 `l` 向右移动，即 `current_sum -= a[l]`，`l++`。
        *   这个收缩过程会一直进行，只要 `current_sum` 仍然 `>= S`，就继续收缩 `l`，并更新 `min_len`。
    *   **循环结束**：当 `r` 遍历完整个数组，`min_len` 就是最终答案。如果 `min_len` 仍然是 `N + 1`，说明没有找到符合条件的子数组，返回 `0`。

**代码实现**：

```cpp
#include <bits/stdc++.h>

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);

    int n; // 数组的长度
    std::cin >> n;

    int a[n]; // 定义数组a
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    int S; // 目标和
    std::cin >> S;

    int l = 0; // 窗口的左指针
    long long currentSum = 0; // 当前窗口的和
    int minLen = n + 1; // 记录最短长度，初始化为一个不可能达到的值 (比 n 大)

    // r 是窗口的右指针，负责扩张窗口
    for (int r = 0; r < n; ++r) {
        currentSum += a[r]; // 将 a[r] 加入窗口，扩张窗口

        // 当窗口的和满足条件时，尝试收缩窗口，寻找最短长度
        while (currentSum >= S) {
            // 更新最短长度
            if (r - l + 1 < minLen) {
                minLen = r - l + 1;
            }
            
            // 尝试将窗口左边界向右移动，收缩窗口
            currentSum -= a[l];
            l++;
        }
    }

    // 如果 minLen 仍然是初始值，说明没有找到符合条件的子数组
    if (minLen == n + 1) {
        std::cout << 0 << std::endl;
    } else {
        std::cout << minLen << std::endl;
    }

    return 0;
}
```

**复杂度分析**：
*   **时间复杂度**：O(N)。`r` 指针从 `0` 到 `N-1` 移动一次，`l` 指针也从 `0` 到 `N-1` 移动一次。每个元素最多被 `l` 和 `r` 各访问一次。所以是 O(N)。
*   **空间复杂度**：O(1)。

## 代码实现模板

这里提供一个通用的滑动窗口代码模板，你可以根据具体问题进行修改。

```cpp
#include <bits/stdc++.h> // 包含常用的标准库，例如 iostream, algorithm 等

int main() {
    // 优化输入输出速度
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);

    // 1. 读取输入数据
    int n; // 数组的长度
    std::cin >> n;

    int a[n]; // 定义数组 a
    for (int i = 0; i < n; ++i) {
        std::cin >> a[i];
    }

    // 根据题目可能需要读取其他参数，例如目标值 S, 固定长度 k 等
    // int S;
    // std::cin >> S;

    // 2. 初始化滑动窗口相关变量
    int l = 0; // 窗口的左指针
    long long currentProperty = 0; // 维护当前窗口的某个性质（例如：和、计数等），注意数据类型
    
    // 初始化最终答案，根据问题求最大/最小/计数等选择合适的初始值
    // 例如：求最大长度，min_len = N + 1; 求最大和，max_sum = 负无穷; 求计数，count = 0;
    int ans = 初始值; 

    // 3. 右指针 r 遍历数组，扩张窗口
    for (int r = 0; r < n; ++r) {
        // 将 a[r] 加入当前窗口，更新 currentProperty
        // 例如：currentProperty += a[r];
        // 例如：处理其他元素的加入逻辑...
        currentProperty += a[r]; // 示例：计算和

        // 4. 左指针 l 根据条件收缩窗口
        // 当 currentProperty 不满足题目条件时 (例如：和太大，超过限制)
        // 或者需要优化窗口时 (例如：窗口长度过长，为了找最短)
        while (条件判断 (currentProperty, l, r, a)) { // 这里的条件判断是关键，根据题目具体要求
            // 更新答案 (如果需要在收缩前更新)
            // 例如：ans = std::min(ans, r - l + 1); // 如果找最短
            
            // 将 a[l] 从窗口中移除，更新 currentProperty
            // 例如：currentProperty -= a[l];
            // 例如：处理其他元素的移除逻辑...
            currentProperty -= a[l]; // 示例：减去和

            // 左指针向右移动
            l++;
        }

        // 5. 在窗口满足条件（或者每次扩张后）时，更新最终答案
        // 例如：ans = std::max(ans, r - l + 1); // 如果找最长
        // 例如：如果问题是统计满足条件的窗口数量，这里可能需要 ans += (r - l + 1);
        // 具体更新逻辑取决于题目
        // 示例：这里假设我们找的是最长的满足条件的子数组
        if (满足条件的判断 (currentProperty, l, r, a)) { // 再次检查条件，确保更新的是有效窗口
             ans = std::max(ans, r - l + 1); // 假设找最长
        }
    }

    // 6. 输出结果
    std::cout << ans << std::endl;

    return 0;
}

// 辅助函数，用于示例中的“条件判断”和“满足条件的判断”
// 实际使用时，这些逻辑会直接写在 main 函数的 while 或 if 中
// bool 条件判断(long long currentProp, int l, int r, int arr[]) {
//     // 假设我们要找和 <= 某个值 MAX_SUM 的最长子数组
//     // 那么当 currentProp > MAX_SUM 时，需要收缩窗口
//     // return currentProp > MAX_SUM;
//     // 假设我们要找和 >= 某个值 MIN_SUM 的最短子数组
//     // 那么当 currentProp >= MIN_SUM 时，可以收缩窗口
//     // return currentProp >= MIN_SUM;
//     return false; // 占位符，实际请根据题目替换
// }

// bool 满足条件的判断(long long currentProp, int l, int r, int arr[]) {
//     // 假设我们要找和 <= 某个值 MAX_SUM 的最长子数组
//     // return currentProp <= MAX_SUM;
//     // 假设我们要找和 >= 某个值 MIN_SUM 的最短子数组
//     // return currentProp >= MIN_SUM;
//     return true; // 占位符，实际请根据题目替换
// }
```

**注释说明**：
*   `l` 和 `r` 是窗口的左右边界，`l` 是闭区间，`r` 也是闭区间。
*   `currentProperty` 是窗口内元素的某个聚合值（如和、积、计数等），需要根据题目来定义。
*   `ans` 是最终的答案，它的初始化值取决于你要求的是最大值、最小值还是计数等。
*   `while` 循环中的“条件判断”是滑动窗口的核心，它决定了何时收缩窗口，以及收缩到什么程度。
*   `for` 循环中的“更新答案”发生在 `r` 扩张窗口后，或者 `l` 收缩窗口后，窗口状态稳定且满足条件时。

## 课堂互动

### 提问环节 (3-5个)

1.  **思考题**：滑动窗口算法中，`r` 指针和 `l` 指针的移动方向有什么共同点？有什么不同点？它们分别代表了什么操作？
    *   **预期答案**：共同点是都向右移动。不同点是 `r` 是主动扩张窗口，每次移动一步；`l` 是被动收缩窗口，当窗口不满足条件时才移动，可能会移动多步。`r` 代表“加入新元素”，`l` 代表“移除旧元素”。
2.  **思考题**：为什么说滑动窗口算法比暴力解法更高效？它避免了哪些重复计算？
    *   **预期答案**：因为它避免了每次都重新计算子数组的性质。当窗口移动时，它只是在旧窗口的基础上进行增量更新（加一个元素，减一个元素），而不是重新遍历整个子数组。
3.  **思考题**：如果我们要找的是“和小于等于 S 的最长连续子数组”，那么 `l` 指针的 `while` 循环的条件应该是什么？`current_sum` 太大还是太小的时候需要收缩 `l`？
    *   **预期答案**：当 `current_sum > S` 的时候，`l` 指针需要收缩。因为我们希望和小于等于 S，如果当前和已经超出了，就需要从左边移除元素来减小和。
4.  **思考题**：在滑动窗口算法中，`min_len` (最短长度) 通常初始化为 `N+1`，而 `max_sum` (最大和) 通常初始化为负无穷。这是为什么？
    *   **预期答案**：`N+1` 是一个比任何可能长度都大的值，这样第一次找到符合条件的长度时，`min_len` 就能被正确更新。`负无穷` 是一个比任何可能和都小的值，这样第一次计算出和时，`max_sum` 就能被正确更新。这是一种确保第一次有效更新成功的技巧。

### 小组任务

**任务名称**：模拟滑动窗口
**任务描述**：
小组为单位，每组发一个数组 `a = [4, 1, 2, 5, 3]` 和一个目标和 `S = 8`。
请你们模拟寻找“和大于等于 S 的最短连续子数组”的过程。
每组需要：
1.  用卡片或纸条代表数组元素。
2.  用两只笔或手指代表 `l` 和 `r` 指针。
3.  一步步地移动 `r`，计算 `current_sum`。
4.  当 `current_sum >= S` 时，记录当前窗口长度，并移动 `l`，更新 `min_len`。
5.  最终说出你们找到的最短长度和对应的子数组。

### 即时练习

**题目**：
给定数组 `a = [10, 5, 2, 7, 1, 9]` 和目标和 `S = 15`。请你口头说出，使用滑动窗口算法找到和**大于或等于** `S` 的最短连续子数组的长度。

**步骤提示**：
*   `l = 0`, `current_sum = 0`, `min_len = N + 1` (这里 `N = 6`)
*   `r = 0`: `a[0]=10`, `current_sum = 10`
*   `r = 1`: `a[1]=5`, `current_sum = 10+5=15`。
    *   此时 `current_sum >= S`。`min_len = min(7, 1-0+1) = 2`。
    *   `current_sum -= a[l]` (`current_sum = 15-10=5`)，`l++` (`l=1`)。
    *   现在 `current_sum = 5`，不再 `>= S`，`while` 循环结束。
*   `r = 2`: `a[2]=2`, `current_sum = 5+2=7`
*   ... 持续模拟下去。

**预期答案**：
最短长度是 `2`，对应的子数组是 `[10, 5]` 和 `[7, 9]`。
在 `r=1` 时，`[10, 5]` 长度为2，`min_len=2`。
继续模拟：
`r=2`: `current_sum = 7`
`r=3`: `a[3]=7`, `current_sum = 7+7=14`
`r=4`: `a[4]=1`, `current_sum = 14+1=15`。
    *   `current_sum >= S`。`min_len = min(2, 4-1+1) = min(2, 4) = 2`。
    *   `current_sum -= a[l]` (`current_sum = 15-5=10`)，`l++` (`l=2`)。
    *   `current_sum = 10`，仍然 `>= S`。`min_len = min(2, 4-2+1) = min(2, 3) = 2`。
    *   `current_sum -= a[l]` (`current_sum = 10-2=8`)，`l++` (`l=3`)。
    *   `current_sum = 8`，不再 `>= S`。`while` 循环结束。
`r=5`: `a[5]=9`, `current_sum = 8+9=17`。
    *   `current_sum >= S`。`min_len = min(2, 5-3+1) = min(2, 3) = 2`。
    *   `current_sum -= a[l]` (`current_sum = 17-7=10`)，`l++` (`l=4`)。
    *   `current_sum = 10`，不再 `>= S`。`while` 循环结束。
最终 `min_len` 为 `2`。

## 分层练习题目

### 基础巩固

1.  **题目**：**最大平均值子数组**
    给定一个整数数组 `a`，找到一个长度为 `k` 的连续子数组，使其平均值最大。输出这个最大平均值。
    （提示：平均值最大等价于总和最大）
    **输入**：
    `n = 8, k = 4`
    `a = [1, 12, -5, -6, 50, 3, 2, 1]`
    **输出**：
    `15.75` (对应子数组 `[12, -5, -6, 50]`)

2.  **题目**：**最长连续子数组**
    给定一个只包含 `0` 和 `1` 的数组 `a`，找到包含最多 `1` 的连续子数组的长度。
    **输入**：
    `n = 7`
    `a = [1, 1, 0, 1, 1, 1, 0]`
    **输出**：
    `3` (对应 `[1, 1, 1]`)

### 能力提升

1.  **题目**：**长度为 K 且和小于 S 的子数组数量**
    给定一个整数数组 `a` 和两个整数 `k` 和 `S`。找出所有长度为 `k` 且其元素总和严格小于 `S` 的连续子数组的数量。
    **输入**：
    `n = 5, k = 3, S = 10`
    `a = [1, 2, 3, 4, 5]`
    **输出**：
    `2` (对应 `[1, 2, 3]` 和 `[2, 3, 4]`)

2.  **题目**：**最长和不超过 S 的子数组**
    给定一个正整数数组 `a` 和一个正整数 `S`，找出数组中和**小于或等于** `S` 的最长连续子数组的长度。
    **输入**：
    `n = 6, S = 10`
    `a = [3, 1, 2, 1, 4, 2]`
    **输出**：
    `4` (对应 `[1, 2, 1, 4]`)

### 拓展挑战

1.  **题目**：**子数组乘积小于 K**
    给定一个正整数数组 `a` 和一个正整数 `K`。统计乘积小于 `K` 的连续子数组的数量。
    **输入**：
    `n = 4, K = 100`
    `a = [10, 5, 2, 6]`
    **输出**：
    `8` (子数组分别是 `[10]`, `[5]`, `[2]`, `[6]`, `[10, 5]` (乘积50), `[5, 2]` (乘积10), `[2, 6]` (乘积12), `[5, 2, 6]` (乘积60))

---

### 练习题目答案

**基础巩固**
1.  **最大平均值子数组**：`15.75`
2.  **最长连续子数组**：`3`

**能力提升**
1.  **长度为 K 且和小于 S 的子数组数量**：`2`
    *   `[1, 2, 3]` Sum=6 < 10
    *   `[2, 3, 4]` Sum=9 < 10
    *   `[3, 4, 5]` Sum=12 >= 10 (不满足)
2.  **最长和不超过 S 的子数组**：`4`
    *   当 `r=4` 时，窗口 `[1, 2, 1, 4]` (即 `a[1]` 到 `a[4]`)，和为 `1+2+1+4 = 8 <= 10`，长度为 `4`。这是最长的。

**拓展挑战**
1.  **子数组乘积小于 K**：`8`
    *   `r=0, a[0]=10`. `prod=10`. `ans+=1` (`[10]`). `l=0`.
    *   `r=1, a[1]=5`. `prod=50`. `ans+=2` (`[5]`, `[10,5]`). `l=0`.
    *   `r=2, a[2]=2`. `prod=100`. `prod>=K`, `prod/=a[l](10)`, `l=1`. `prod=10`. `ans+=2` (`[2]`, `[5,2]`). `l=1`.
    *   `r=3, a[3]=6`. `prod=60`. `ans+=3` (`[6]`, `[2,6]`, `[5,2,6]`). `l=1`.
    *   总计 `1+2+2+3 = 8`。
