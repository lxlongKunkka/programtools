## 趣味引入

同学们，想象一下，你是一个小小的侦探，面前有一堆数字，你的任务是找出那些“独来独往”的数字。这些数字非常特别，它们除了能被自己和1整除之外，就不能被其他任何比它小的正整数整除了。我们把这些“独来独往”的数字叫做“质数”！

比如，数字 7：
- 7 ÷ 1 = 7 (可以)
- 7 ÷ 2 = 3 余 1 (不行)
- 7 ÷ 3 = 2 余 1 (不行)
- 7 ÷ 4 = 1 余 3 (不行)
- 7 ÷ 5 = 1 余 2 (不行)
- 7 ÷ 6 = 1 余 1 (不行)
看，7 除了被1和7自己整除，就不能被2、3、4、5、6这些数整除了，所以 7 就是一个“独来独往”的质数！

那数字 12 呢？
- 12 ÷ 1 = 12 (可以)
- 12 ÷ 2 = 6 (可以！)
- 12 ÷ 3 = 4 (可以！)
- 12 ÷ 4 = 3 (可以！)
- 12 ÷ 6 = 2 (可以！)
哎呀，12 不仅能被1和12整除，还能被2、3、4、6整除，它可不是“独来独往”的，它有很多朋友，所以 12 就不是质数，我们叫它“合数”。

现在问题来了，如果给你一个很大的数字，比如 997，你该怎么快速判断它是不是质数呢？难道我们要一个一个地去除到 996 吗？这可太累了！别担心，今天我们就来学习一种聪明的方法，叫做“试除法”，帮助我们快速找出这些“独来独往”的质数！

## 知识点讲解

### 1. 质数与合数的定义

首先，让我们精确地定义一下这些特殊的数字：

*   **质数 (Prime Number)**：
    *   **定义**：一个大于1的自然数，如果除了1和它本身以外不再有其他因数（也叫约数），那么这个数就是质数。
    *   **例子**：2, 3, 5, 7, 11, 13, 17, ...
    *   **特别注意**：2是最小的质数，也是唯一一个偶数质数。

*   **合数 (Composite Number)**：
    *   **定义**：一个大于1的自然数，如果除了1和它本身以外还有其他因数，那么这个数就是合数。
    *   **例子**：4, 6, 8, 9, 10, 12, 14, ...

*   **特殊数字**：
    *   **1**：既不是质数也不是合数。
    *   **0和负数**：我们通常只在自然数（正整数）范围内讨论质数和合数。

### 2. 试除法：判断质数的基本思路

判断一个数 `n` 是否为质数，最直接的方法就是从 2 开始，一直到 `n-1`，用这些数去除 `n`。

*   **如果 `n` 能被其中任何一个数整除**（即 `n % i == 0`），那么 `n` 就不是质数。
*   **如果 `n` 不能被所有这些数整除**，那么 `n` 就是质数。

**举个例子**：判断 10 是不是质数。
1.  `10 % 2 == 0`。发现 10 能被 2 整除，所以 10 不是质数。
**举个例子**：判断 13 是不是质数。
1.  `13 % 2 != 0`
2.  `13 % 3 != 0`
3.  `13 % 4 != 0`
4.  ...
5.  `13 % 12 != 0`
直到检查到 12，都没有发现能整除 13 的数，所以 13 是质数。

### 3. 试除法的优化：让程序更聪明！

同学们，如果我们要判断一个很大的数字，比如 1000000007（这是一个质数哦），用上面的方法要除多少次呢？要除将近 10 亿次！太慢了！计算机虽然快，但也不能这么浪费时间。有没有更聪明的方法呢？当然有！

**优化一：检查到 `n` 的平方根就够了！**

这是试除法最重要的优化。
假设 `n` 是一个合数，那么它一定有一个因数 `d`，使得 `1 < d < n`。
如果 `d` 是 `n` 的一个因数，那么 `n / d` 也是 `n` 的一个因数。
这两个因数 `d` 和 `n / d` 中，至少有一个会小于或等于 `sqrt(n)`（`sqrt(n)` 表示 `n` 的平方根）。

**为什么呢？**
假设所有的因数 `d` 都大于 `sqrt(n)`。
那么 `d > sqrt(n)` 并且 `n / d > sqrt(n)`。
如果我们将这两个不等式相乘：`d * (n / d) > sqrt(n) * sqrt(n)`。
结果就是 `n > n`，这显然是矛盾的！
所以，这个假设是错误的，至少有一个因数会小于或等于 `sqrt(n)`。

**这意味着什么？**
我们只需要从 2 开始，检查到 `sqrt(n)` 就行了！如果在这个范围内找不到 `n` 的因数，那么 `n` 就一定是质数！

**举个例子**：判断 100 是不是质数。
`sqrt(100)` 是 10。我们只需要检查 2 到 10 之间的数。
1.  `100 % 2 == 0`。发现能被 2 整除，所以 100 不是质数。我们甚至不需要检查到 10！

**举个例子**：判断 101 是不是质数。
`sqrt(101)` 大约是 10.04。我们只需要检查 2 到 10 之间的数。
1.  `101 % 2 != 0`
2.  `101 % 3 != 0`
3.  `101 % 4 != 0`
4.  `101 % 5 != 0`
5.  `101 % 6 != 0`
6.  `101 % 7 != 0`
7.  `101 % 8 != 0`
8.  `101 % 9 != 0`
9.  `101 % 10 != 0`
检查完发现都没有能整除 101 的，所以 101 是质数。是不是比检查到 100 快多了？

**优化二：只检查奇数（除了2之外）！**

我们知道，除了 2 以外，所有的偶数都不是质数。因为它们都能被 2 整除。
所以，我们可以：
1.  先特判一下 `n` 是否为 2。如果是 2，就是质数。
2.  如果 `n` 是偶数（且 `n` 不等于 2），那么它就不是质数。
3.  如果 `n` 是奇数，我们就从 3 开始，只检查奇数（3, 5, 7, ...），直到 `sqrt(n)`。这样检查的次数又少了一半！

**总结一下优化后的试除法步骤：**
1.  如果 `n <= 1`，则不是质数。
2.  如果 `n == 2`，则是质数。
3.  如果 `n` 是偶数（`n % 2 == 0`），则不是质数。
4.  从 `i = 3` 开始，每次 `i` 增加 2（只检查奇数），直到 `i * i <= n`。
    *   如果在循环中发现 `n % i == 0`，则 `n` 不是质数。
5.  如果循环结束都没有找到因数，则 `n` 是质数。

这个优化的试除法已经非常高效了，对于单个数字的质数判断，它是一个非常棒的方法！

## 代码示例

现在，我们把这些聪明的方法变成 C++ 代码！

```cpp
#include <bits/stdc++.h> // 包含大部分常用头文件

using namespace std;

/**
 * @brief 判断一个数是否为质数（基础版试除法）
 * @param n 待判断的整数
 * @return 如果 n 是质数，返回 true；否则返回 false。
 */
bool is_prime_basic(int n)
{
    // 1. 小于等于1的数不是质数
    if (n <= 1)
    {
        return false;
    }
    // 2. 从2开始，尝试除到 n-1
    // 如果 n 能被 i 整除，说明 n 有除了1和n以外的因数 i，所以 n 不是质数
    for (int i = 2; i < n; ++i)
    {
        if (n % i == 0)
        {
            return false;
        }
    }
    // 如果循环结束都没有找到因数，说明 n 是质数
    return true;
}

/**
 * @brief 判断一个数是否为质数（优化版试除法：检查到平方根，并跳过偶数）
 * @param n 待判断的整数
 * @return 如果 n 是质数，返回 true；否则返回 false。
 */
bool is_prime_optimized(int n)
{
    // 1. 处理特殊情况：小于等于1的数不是质数
    if (n <= 1)
    {
        return false;
    }
    // 2. 特判2：2是最小的质数，也是唯一的偶数质数
    if (n == 2)
    {
        return true;
    }
    // 3. 特判所有其他偶数：除了2之外的偶数都不是质数
    if (n % 2 == 0)
    {
        return false;
    }
    // 4. 从3开始，只检查奇数，直到 sqrt(n)
    // 注意：i * i <= n 比 i <= sqrt(n) 更推荐，可以避免浮点数运算可能带来的精度问题
    for (int i = 3; i * i <= n; i = i + 2) // i = i + 2 等价于 i += 2，每次跳过偶数
    {
        if (n % i == 0) // 如果 n 能被当前奇数 i 整除，则 n 不是质数
        {
            return false;
        }
    }
    // 如果循环结束都没有找到因数，说明 n 是质数
    return true;
}

int main()
{
    int a;
    cout << "请输入一个整数，我来判断它是不是质数：";
    cin >> a;

    // 使用基础版判断
    if (is_prime_basic(a))
    {
        cout << a << " 是一个质数 (使用基础版判断)." << endl;
    }
    else
    {
        cout << a << " 不是一个质数 (使用基础版判断)." << endl;
    }

    // 使用优化版判断
    if (is_prime_optimized(a))
    {
        cout << a << " 是一个质数 (使用优化版判断)." << endl;
    }
    else
    {
        cout << a << " 不是一个质数 (使用优化版判断)." << endl;
    }

    cout << endl;

    // 更多测试用例
    cout << "测试 7: " << (is_prime_optimized(7) ? "是质数" : "不是质数") << endl;    // 质数
    cout << "测试 1: " << (is_prime_optimized(1) ? "是质数" : "不是质数") << endl;    // 不是质数
    cout << "测试 4: " << (is_prime_optimized(4) ? "是质数" : "不是质数") << endl;    // 不是质数
    cout << "测试 2: " << (is_prime_optimized(2) ? "是质数" : "不是质数") << endl;    // 质数
    cout << "测试 97: " << (is_prime_optimized(97) ? "是质数" : "不是质数") << endl;  // 质数
    cout << "测试 100: " << (is_prime_optimized(100) ? "是质数" : "不是质数") << endl; // 不是质数
    cout << "测试 997: " << (is_prime_optimized(997) ? "是质数" : "不是质数") << endl; // 质数

    return 0;
}
```

### 选择题

1.  下列哪个数字既不是质数也不是合数？
    A. 0
    B. 1
    C. 2
    D. 3

2.  判断一个正整数 `n` 是否为质数，使用优化后的试除法时，除了对 2 和偶数进行特殊处理外，循环检查的除数 `i` 的范围通常是：
    A. 从 3 到 `n-1`
    B. 从 3 到 `n/2`
    C. 从 3 到 `sqrt(n)`
    D. 从 1 到 `n`

3.  关于质数，以下说法错误的是：
    A. 质数大于 1。
    B. 2 是唯一的偶数质数。
    C. 任何一个大于 1 的自然数，不是质数就是合数。
    D. 质数只有两个因数：1 和它本身。

4.  如果一个数 `N` 是合数，那么它至少有一个因数 `d` 满足：
    A. `d = N`
    B. `d < N` 且 `d` 是奇数
    C. `d <= sqrt(N)`
    D. `d > sqrt(N)`

### 填空题

5.  最小的质数是 ______。
6.  10 以内（不包括 10）最大的质数是 ______。
7.  数字 51 是否为质数？______（请填“是”或“否”）。

---

### 答案与解析

1.  **B. 1**
    *   解析：质数和合数都定义为“大于1的自然数”。0不是自然数，2和3是质数。

2.  **C. 从 3 到 `sqrt(n)`**
    *   解析：优化后的试除法利用了质数因数的对称性，只需要检查到 `sqrt(n)` 即可。同时，由于已经特判了偶数，循环中只需检查奇数。

3.  **C. 任何一个大于 1 的自然数，不是质数就是合数。**
    *   解析：这个说法错误。因为 1 既不是质数也不是合数，而它并不大于 1。然而，根据我们对质数和合数的定义，它们都要求是“大于1的自然数”。所以，如果问题是“任何一个**大于1**的自然数，不是质数就是合数”，那这个说法就正确了。但原题是“任何一个大于1的自然数”，这个表述本身就包含了质数和合数的定义域，而1不在此定义域中。因此，选项C是错误的，因为它没有考虑到1这个特殊情况。

4.  **C. `d <= sqrt(N)`**
    *   解析：这是试除法优化的核心原理，如果一个数 `N` 是合数，它至少有一个小于或等于 `sqrt(N)` 的因数。

5.  **2**
    *   解析：2是唯一的偶数质数，也是所有质数中最小的一个。

6.  **7**
    *   解析：10以内的质数有 2, 3, 5, 7。其中最大的是 7。

7.  **否**
    *   解析：51 可以被 3 整除 (`51 = 3 * 17`)，所以 51 是一个合数，不是质数。
