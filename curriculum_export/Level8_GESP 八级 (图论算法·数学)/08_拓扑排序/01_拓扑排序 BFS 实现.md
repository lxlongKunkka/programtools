## 深度知识点讲解

### 1. 什么是拓扑排序？（What is Topological Sort?）

拓扑排序是针对**有向无环图（Directed Acyclic Graph, DAG）**的一种排序方法。它的目标是得到一个图中所有顶点的线性序列，使得对于图中的任意一条有向边 `(u, v)`（从 `u` 指向 `v`），在这个序列中，`u` 都出现在 `v` 的前面。

**生活类比**：
*   **课程学习**：要学习“数据结构”，你得先学“C++编程基础”；要学“C++编程基础”，你得先学“编程入门”。拓扑排序就是帮你规划出一条合理的学习路径。
*   **穿衣顺序**：你不能先穿鞋再穿袜子（通常情况下），也不能先穿外套再穿内衣。拓扑排序能给你一个正确的穿衣顺序。

**核心思想**：找到一个没有前置条件的任务（或节点），先完成它，然后告诉所有依赖于它的任务：“我已经完成了，你们可以开始了！” 接着，再找下一个没有前置条件的任务，如此循环，直到所有任务完成。

### 2. 有向无环图（DAG）—— 拓扑排序的舞台

拓扑排序只能应用于**有向无环图（DAG）**。
*   **有向图**：图中的边有方向，表示依赖关系或流程。
*   **无环图**：图中不存在任何环路。如果存在环，比如 A -> B -> C -> A，那么 A 依赖 B，B 依赖 C，C 又依赖 A，形成一个死循环，谁也无法真正“开始”或“完成”，也就无法进行拓扑排序。

**生活类比**：
想象你正在一个城市里开车，所有的街道都是单行道（有向）。如果这个城市的路网是“无环”的，那么你总能找到一条路，从起点走到终点，而不会陷入一个永远绕不出去的圈子。如果城市里有环路，你可能就会被困住。

### 3. 核心概念：入度（In-degree）

**入度**：对于图中的一个顶点 `v`，指向它的边的数量就是它的入度。
*   如果 `u -> v` 是一条边，那么 `v` 的入度就增加了 1。

**为什么入度很重要？**
*   一个顶点的入度为 0，意味着没有其他顶点指向它，它没有任何前置依赖。这就像一个可以立即开始的任务，因为它不需要等待任何其他任务完成。
*   在拓扑排序中，我们总是从入度为 0 的节点开始。当我们“完成”一个节点后，它所指向的那些节点的入度就会减少，因为它们的其中一个前置依赖被满足了。

**生活类比**：
你准备做一道菜，比如“番茄炒蛋”。
*   “切番茄”和“打鸡蛋”的入度都是 0（不需要等待其他步骤）。
*   “炒番茄鸡蛋”的入度是 2（需要等待“切番茄”和“打鸡蛋”都完成）。
当你完成了“切番茄”这个步骤，那么“炒番茄鸡蛋”的“等待清单”上就少了一项。

### 4. BFS 实现拓扑排序（Kahn's Algorithm）

基于BFS的拓扑排序算法，也称为Kahn's Algorithm，步骤如下：

**步骤详解**：

1.  **计算所有顶点的入度**：
    *   遍历图中的所有边 `(u, v)`，每遇到一条边，就将 `v` 的入度加 1。
    *   我们需要一个数组 `d[N]` 来存储每个节点的入度。
2.  **初始化队列**：
    *   创建一个队列 `q`。
    *   遍历所有顶点，将所有入度为 0 的顶点加入队列 `q`。这些是我们可以立即“开始”的任务。
    *   同时，用一个计数器 `cnt` 记录已经加入拓扑序列的顶点数量。
3.  **广度优先遍历**：
    *   当队列 `q` 不为空时，重复以下操作：
        *   **出队**：从队列 `q` 中取出一个顶点 `u`。
        *   **加入序列**：将 `u` 加入到拓扑序列中，并将 `cnt` 加 1。
        *   **更新邻居**：遍历 `u` 的所有邻居 `v`（即 `u` 指向的所有顶点 `v`）：
            *   将 `v` 的入度减 1 (`d[v]--`)。这表示 `u` 这个前置任务已经完成了。
            *   如果 `v` 的入度变为 0 (`d[v] == 0`)，说明 `v` 的所有前置任务都已完成，现在 `v` 可以“开始”了。将 `v` 加入队列 `q`。
4.  **判断是否有环**：
    *   当队列为空时，算法结束。
    *   检查计数器 `cnt`。如果 `cnt` 等于图中顶点的总数 `N`，说明所有顶点都被成功地加入了拓扑序列，图是DAG。
    *   如果 `cnt` 小于 `N`，说明图中存在环，有部分顶点无法被处理（它们的入度永远不会变为 0），因此无法形成完整的拓扑序列。

**为什么这样是正确的？（怎么想到的？）**
这个算法的巧妙之处在于，它模拟了任务的执行过程。我们总是选择那些“没有前置条件”的任务来执行。当我们执行完一个任务 `u` 后，它就为所有依赖于它的任务 `v` 扫清了一个障碍，使得 `v` 的“待办事项”减少了一项。当 `v` 的所有待办事项都清空（入度变为0）时，它就成为了新的“没有前置条件”的任务，可以被执行了。BFS的特性保证了我们总是按“层”处理任务，即先处理完所有第一批可执行的任务，再处理第二批，依此类推。

### 5. 数据结构实现（无STL限制）

由于GESP的特殊要求，我们不能使用 `std::vector`、`std::queue` 等STL容器，需要用原生数组来实现：

*   **邻接表存储图**：
    *   `int h[N]`：头数组，`h[i]` 存储顶点 `i` 的第一条出边在 `e` 数组中的索引。
    *   `int e[M]`：边数组，存储边的终点。
    *   `int ne[M]`：next数组，`ne[i]` 存储与边 `i` 同起点的下一条边在 `e` 数组中的索引。
    *   `int idx`：当前边的索引计数器。
    *   `void add(int a, int b)` 函数：添加一条从 `a` 到 `b` 的有向边。
*   **入度数组**：
    *   `int d[N]`：`d[i]` 存储顶点 `i` 的入度。
*   **队列**：
    *   `int q[N]`：队列数组。
    *   `int hh, tt`：`hh` 是队头指针，`tt` 是队尾指针。
    *   `q[tt++] = x`：入队操作。
    *   `u = q[hh++]`：出队操作。
*   **拓扑序列结果数组**：
    *   `int ans[N]`：存储最终的拓扑序列。
    *   `int k`：`ans` 数组的当前填充位置。

### 6. 常见误区与注意点

*   **图的类型**：拓扑排序只适用于有向图。如果图是无向的，拓扑排序没有意义。
*   **图的连通性**：拓扑排序算法可以处理非连通的DAG。BFS会自然地找到所有入度为0的节点并处理它们。
*   **多重拓扑序列**：一个DAG可能存在多个合法的拓扑序列。BFS得到的序列是其中一种，具体取决于入队顺序（通常是小的节点ID优先）。
*   **循环检测**：如果最终拓扑序列中的节点数量小于图的总节点数，则图中一定存在环。这是判断图是否为DAG的关键方法。

## 典型例题精讲

### 例题一：基本拓扑排序

**题目描述**：
给定一个 `N` 个顶点 `M` 条边的有向图，求其任意一个拓扑序列。如果图中有环，则输出“Graph has a cycle.”。

**输入格式**：
第一行包含两个整数 `N` 和 `M`，表示顶点数和边数。
接下来 `M` 行，每行两个整数 `u, v`，表示一条从 `u` 到 `v` 的有向边。顶点编号从 1 到 `N`。

**输出格式**：
如果存在拓扑序列，输出序列中的 `N` 个顶点，用空格隔开。否则输出“Graph has a cycle.”。

**示例输入**：
```
5 6
1 2
1 3
2 4
3 4
4 5
```

**示例输出**：
```
1 2 3 4 5
```
（注意：`1 3 2 4 5` 也是一个合法的拓扑序列）

---

**思路分析**：

1.  **建图与入度计算**：
    *   根据输入的 `N` 和 `M`，初始化邻接表和入度数组 `d`。
    *   遍历 `M` 条边，每读取一条边 `u -> v`，就用 `add(u, v)` 将边加入邻接表，并同时将 `v` 的入度 `d[v]` 加 1。
2.  **初始化队列**：
    *   创建一个队列 `q`，初始化队头 `hh = 0`，队尾 `tt = 0`。
    *   遍历所有顶点 `i` 从 1 到 `N`。如果 `d[i]` 为 0，说明它是初始的“无依赖任务”，将其加入队列 `q[tt++] = i`。
3.  **BFS 遍历**：
    *   创建一个数组 `ans` 来存储拓扑序列，用 `k = 0` 记录当前序列长度。
    *   当 `hh < tt`（队列不为空）时循环：
        *   从队头取出一个顶点 `u = q[hh++]`。
        *   将 `u` 加入拓扑序列 `ans[k++] = u`。
        *   遍历 `u` 的所有邻居 `v`（通过邻接表）：
            *   将 `v` 的入度减 1：`d[v]--`。
            *   如果 `d[v]` 变为 0，说明 `v` 现在也“无依赖”了，将其加入队列 `q[tt++] = v`。
4.  **结果判断**：
    *   循环结束后，如果 `k`（拓扑序列中的顶点数量）等于 `N`，说明成功找到了一个拓扑序列，输出 `ans` 数组。
    *   否则，说明图中存在环，输出“Graph has a cycle.”。

---

**代码实现**：

```cpp
#include <iostream> // 使用iostream进行输入输出

// 定义常量，N表示最大顶点数，M表示最大边数（这里M可以设为N的平方级别，或者根据题目数据范围）
const int N = 100005; // 顶点数最大10万
const int M = 200005; // 边数最大20万 (通常是N的两倍左右，确保空间足够)

// 邻接表存储图
int h[N];   // h[u] 存储顶点 u 的第一条边的索引
int e[M];   // e[idx] 存储当前边的终点
int ne[M];  // ne[idx] 存储与当前边同起点的下一条边的索引
int idx;    // 边的计数器，每次添加新边时递增

// 入度数组
int d[N];   // d[u] 存储顶点 u 的入度

// 队列
int q[N];   // 队列数组
int hh, tt; // hh 队头指针，tt 队尾指针

// 拓扑序列结果数组
int ans[N]; // 存储拓扑排序结果
int k;      // ans数组的当前填充位置

// 辅助函数：添加一条从 a 到 b 的有向边
void add(int a, int b)
{
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

// 拓扑排序函数
bool topsort(int n)
{
    // 初始化队列
    hh = 0;
    tt = 0;
    k = 0; // 拓扑序列长度计数器

    // 将所有入度为0的顶点加入队列
    for (int i = 1; i <= n; ++i)
    {
        if (d[i] == 0)
        {
            q[tt++] = i;
        }
    }

    // BFS 遍历
    while (hh < tt)
    {
        // 从队头取出一个顶点 u
        int u = q[hh++];
        // 将 u 加入拓扑序列
        ans[k++] = u;

        // 遍历 u 的所有邻居 v
        for (int i = h[u]; i != -1; i = ne[i])
        {
            int v = e[i];
            // 邻居 v 的入度减1
            d[v]--;
            // 如果 v 的入度变为0，则将其加入队列
            if (d[v] == 0)
            {
                q[tt++] = v;
            }
        }
    }

    // 判断是否所有顶点都加入了拓扑序列
    return k == n; // 如果 k 等于 n，说明无环，成功排序
}

int main()
{
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    // 初始化邻接表头数组，-1 表示链表末尾
    for (int i = 1; i <= n; ++i)
    {
        h[i] = -1;
    }

    // 读取边并建图，同时计算入度
    for (int i = 0; i < m; ++i)
    {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        d[v]++; // 顶点 v 的入度加 1
    }

    // 执行拓扑排序
    if (topsort(n))
    {
        // 成功排序，输出拓扑序列
        for (int i = 0; i < n; ++i)
        {
            std::cout << ans[i] << (i == n - 1 ? "" : " ");
        }
        std::cout << std::endl;
    }
    else
    {
        // 图中存在环
        std::cout << "Graph has a cycle." << std::endl;
    }

    return 0;
}

```

---

**复杂度分析**：

*   **时间复杂度**：
    *   初始化 `h` 数组：O(N)
    *   读取 `M` 条边并建图，计算入度：O(M)
    *   初始化队列：遍历 `N` 个顶点，O(N)
    *   BFS 遍历：每个顶点最多入队出队一次，每条边最多被访问一次（当它的起点出队时），所以BFS过程是 O(N + M)。
    *   总时间复杂度：**O(N + M)**。这与图的顶点数和边数呈线性关系，非常高效。
*   **空间复杂度**：
    *   邻接表 `h, e, ne`：O(N + M)
    *   入度数组 `d`：O(N)
    *   队列 `q`：O(N)
    *   结果数组 `ans`：O(N)
    *   总空间复杂度：**O(N + M)**。

### 例题二：检测图中的环

**题目描述**：
给定一个 `N` 个顶点 `M` 条边的有向图，判断图中是否存在环。

**输入格式**：
第一行包含两个整数 `N` 和 `M`，表示顶点数和边数。
接下来 `M` 行，每行两个整数 `u, v`，表示一条从 `u` 到 `v` 的有向边。顶点编号从 1 到 `N`。

**输出格式**：
如果图中存在环，输出“Yes”。否则输出“No”。

**示例输入1**：
```
3 3
1 2
2 3
3 1
```
**示例输出1**：
```
Yes
```

**示例输入2**：
```
3 2
1 2
2 3
```
**示例输出2**：
```
No
```

---

**思路分析**：
这正是拓扑排序的一个重要应用。一个有向图如果能进行完整的拓扑排序（即所有顶点都能被纳入拓扑序列），那么它就是有向无环图（DAG），不存在环。反之，如果拓扑排序无法将所有顶点都纳入序列，就说明图中存在环。

我们直接使用例题一的 `topsort` 函数，根据其返回值 `k == n` 来判断。

---

**代码实现**：

（与例题一的代码几乎完全相同，只需修改 `main` 函数的输出逻辑）

```cpp
#include <iostream>

const int N = 100005;
const int M = 200005;

int h[N], e[M], ne[M], idx;
int d[N];
int q[N], hh, tt;
int ans[N]; // ans数组在这里主要用于计数，实际内容不输出
int k;

void add(int a, int b)
{
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

// 拓扑排序函数，返回是否成功排序（即是否无环）
bool topsort(int n)
{
    hh = 0;
    tt = 0;
    k = 0; // 记录拓扑序列的长度

    for (int i = 1; i <= n; ++i)
    {
        if (d[i] == 0)
        {
            q[tt++] = i;
        }
    }

    while (hh < tt)
    {
        int u = q[hh++];
        ans[k++] = u; // 只是计数，不一定需要存储具体序列

        for (int i = h[u]; i != -1; i = ne[i])
        {
            int v = e[i];
            d[v]--;
            if (d[v] == 0)
            {
                q[tt++] = v;
            }
        }
    }

    return k == n; // 如果 k 等于 n，说明所有顶点都被处理，无环
}

int main()
{
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m;
    std::cin >> n >> m;

    for (int i = 1; i <= n; ++i)
    {
        h[i] = -1;
        d[i] = 0; // 确保入度初始化为0
    }

    for (int i = 0; i < m; ++i)
    {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        d[v]++;
    }

    if (topsort(n))
    {
        std::cout << "No" << std::endl; // 无环
    }
    else
    {
        std::cout << "Yes" << std::endl; // 有环
    }

    return 0;
}
```

## 代码实现模板

```cpp
#include <iostream> // 用于输入输出

// 定义常量，N表示最大顶点数，M表示最大边数
const int N = 100005; // 通常 N 设为题目最大顶点数 + 5
const int M = 200005; // 通常 M 设为题目最大边数 + 5 (M >= N)

// 邻接表：用于存储图的结构
int h[N];   // h[u] 存储顶点 u 的第一条出边在 e 数组中的索引
int e[M];   // e[idx] 存储当前边的终点
int ne[M];  // ne[idx] 存储与当前边同起点的下一条边在 e 数组中的索引
int idx;    // 边的计数器，每次添加新边时递增

// 入度数组：存储每个顶点的入度
int d[N];   // d[u] 存储顶点 u 的入度

// 队列：用于 BFS 遍历
int q[N];   // 队列数组
int hh, tt; // hh 队头指针，tt 队尾指针

// 拓扑序列结果数组：存储最终的拓扑序列
int ans[N]; // 存储拓扑排序结果
int k;      // ans 数组的当前填充位置，也表示已排序的顶点数量

// 辅助函数：添加一条从 a 到 b 的有向边
void add(int a, int b)
{
    e[idx] = b;         // 边的终点是 b
    ne[idx] = h[a];     // 将当前边的 next 指向原来 a 的第一条边
    h[a] = idx++;       // 更新 a 的第一条边为当前边，并递增 idx
}

// 拓扑排序函数
// 参数 n: 图中的顶点总数
// 返回值: bool 类型，如果成功完成拓扑排序（即图是DAG），则返回 true；否则（图中存在环）返回 false
bool topsort(int n)
{
    // 1. 初始化队列
    hh = 0; // 队头指针归零
    tt = 0; // 队尾指针归零
    k = 0;  // 拓扑序列长度计数器归零

    // 2. 将所有入度为 0 的顶点加入队列
    for (int i = 1; i <= n; ++i) // 假设顶点编号从 1 到 n
    {
        if (d[i] == 0)
        {
            q[tt++] = i; // 将入度为 0 的顶点入队
        }
    }

    // 3. BFS 遍历
    while (hh < tt) // 当队列不为空时
    {
        int u = q[hh++]; // 从队头取出一个顶点 u (并更新队头指针)
        ans[k++] = u;    // 将 u 加入拓扑序列，并更新序列长度

        // 遍历顶点 u 的所有邻居 v
        // 从 h[u] 开始，沿着 ne 链表遍历
        for (int i = h[u]; i != -1; i = ne[i])
        {
            int v = e[i]; // 获取邻居顶点 v

            d[v]--;       // 邻居 v 的入度减 1 (因为 u 已经处理完毕)
            if (d[v] == 0) // 如果 v 的入度变为 0
            {
                q[tt++] = v; // 将 v 加入队列，因为它现在可以被处理了
            }
        }
    }

    // 4. 判断是否所有顶点都被成功地加入了拓扑序列
    return k == n; // 如果 k 等于 n，表示所有顶点都已排序，图无环
                   // 否则，图中存在环，部分顶点无法被处理
}

int main()
{
    // 优化输入输出速度
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, m; // n: 顶点数, m: 边数
    std::cin >> n >> m;

    // 初始化邻接表头数组，-1 表示链表末尾
    for (int i = 1; i <= n; ++i)
    {
        h[i] = -1;
        d[i] = 0; // 确保入度数组初始化为0
    }

    // 读取 m 条边，建图并计算入度
    for (int i = 0; i < m; ++i)
    {
        int u, v;
        std::cin >> u >> v;
        add(u, v); // 添加从 u 到 v 的有向边
        d[v]++;    // 顶点 v 的入度加 1
    }

    // 执行拓扑排序
    if (topsort(n))
    {
        // 如果成功排序（图是DAG），则输出拓扑序列
        for (int i = 0; i < n; ++i)
        {
            std::cout << ans[i] << (i == n - 1 ? "" : " ");
        }
        std::cout << std::endl;
    }
    else
    {
        // 如果图中存在环
        std::cout << "Graph has a cycle." << std::endl;
    }

    return 0;
}

```

### 拓展挑战

1.  **题目**：在DAG中，找到最长路径。
    *   **描述**：在一个加权有向无环图（DAG）中，每条边都有一个权重（表示完成某个任务所需的时间）。请你找出从源点 `S` 到汇点 `T` 的最长路径（最大权重和）。
    *   **提示**：拓扑排序可以帮助你确定动态规划的计算顺序。DP状态 `dp[u]` 表示从源点到 `u` 的最长路径长度。
    *   **输入**：`N, M`，边 `u, v, w` (权重)。
    *   **输出**：最长路径长度。

---

### 练习题目答案

#### 基础巩固答案
1.  **拓扑序列**：1 2 3 4 5 6 (或 1 3 2 5 4 6 等，答案不唯一)
2.  **判断环**：Yes (1->2->3->1 形成一个环)

#### 能力提升答案
1.  **课程学习顺序**：
    *   注意：`u` 依赖 `v` 意味着边是 `v -> u`。所以读取 `u, v` 后，应 `add(v, u)` 并 `d[u]++`。
    *   如果最终拓扑序列长度等于 `N`，输出序列。
    *   否则，输出“无法完成”。
2.  **影响力排序**：
    *   如果用户 A 关注 B，则 A 在排序中必须在 B 之前，这表示边是 `A -> B`。
    *   如果拓扑排序成功，输出序列。
    *   否则，输出“存在相互关注的循环”。

#### 拓展挑战答案
1.  **最长路径**：
    *   **思路**：
        1.  先进行拓扑排序，得到一个合法的顶点访问顺序。
        2.  初始化 `dp` 数组，`dp[S] = 0`，其他 `dp[i] = -infinity`。
        3.  按照拓扑排序的顺序遍历每个顶点 `u`：
            *   对于 `u` 的每个邻居 `v`，如果 `dp[u]` 不是负无穷，则 `dp[v] = max(dp[v], dp[u] + weight(u, v))`。
        4.  最终 `dp[T]` 就是从 `S` 到 `T` 的最长路径。
    *   **注意**：对于无权图的最长路径问题，如果边权为1，则可以通过BFS层数来计算。但这里是加权图，且求最长路径，需要DP。
