## 知识点讲解

### 1. 最短路径问题回顾

在图论中，**最短路径问题**是指在一个赋权图G=(V, E)中，给定一个源点s和一个目标点t，找到一条从s到t的路径，使得路径上所有边的权值之和最小。我们今天要学习的是**单源最短路径**问题，即从一个起点到图中所有其他顶点的最短路径。

Dijkstra算法是解决单源最短路径问题的一个经典算法，但它有一个非常重要的前提：**图中的所有边的权值都必须是非负数！** 如果有负权边，Dijkstra算法就会失效。

### 2. Dijkstra算法的基本思想（贪心策略）

Dijkstra算法的核心思想是**贪心**。它一步一步地“生长”出最短路径，每次都选择当前看来最优的下一步。

想象一下，你站在起点，想去所有其他城市。Dijkstra算法的工作方式就像这样：

1.  **从起点出发：** 你知道你到自己的距离是0，到其他所有城市的距离都是“无穷远”（因为你还没去过）。
2.  **选择最近的邻居：** 在所有你还没确定最短路径的城市中，选择一个离你当前已知最短距离最近的城市。
3.  **确定最短路径：** 一旦你选择了这个城市，你就确定了从起点到它的最短路径。因为如果你能通过其他城市到达它更近，那么那个城市早就被你选了。
4.  **更新邻居：** 确定了这个城市的最短路径后，你就可以通过它去“看看”它的邻居城市。如果通过这个城市到达某个邻居城市比你之前知道的路径更短，那就更新那个邻居城市的最短距离。这个过程叫做**松弛（Relaxation）**。
5.  **重复：** 不断重复步骤2、3、4，直到所有城市的最短路径都被你确定下来。

### 3. 算法步骤详解

我们用以下数据结构来帮助Dijkstra算法运行：

*   `d[i]`: 存储从起点到顶点 `i` 的当前最短距离。初始时 `d[起点] = 0`，其他 `d[i] = 无穷大`。
*   `vis[i]`: 标记顶点 `i` 是否已经确定了最短路径。`vis[i] = true` 表示已确定，`vis[i] = false` 表示未确定。初始时所有 `vis[i] = false`。
*   图的表示：我们使用邻接表来存储图，`adj[u]` 存储与 `u` 相连的所有边，每条边表示为 `(v, w)`，其中 `v` 是邻居顶点，`w` 是边权。

**Dijkstra算法（优先队列优化版）详细步骤：**

1.  **初始化：**
    *   将所有顶点的 `d` 值设为一个很大的数（表示无穷大），除了起点 `s`，`d[s] = 0`。
    *   将所有顶点的 `vis` 值设为 `false`。
    *   创建一个**优先队列** `pq`。优先队列中存储 `(距离, 顶点)` 对，并按照距离从小到大排序。我们将 `(0, s)` 放入 `pq`。
        *   **注意：** C++的 `priority_queue` 默认是大根堆（最大值优先），为了实现最小值优先，我们可以存储 `(-距离, 顶点)`，或者使用 `std::greater<pair<int, int>>`。这里我们选择存储 `(距离, 顶点)`，并在比较时将距离取负。

2.  **循环：** 当优先队列不为空时，重复以下步骤：
    *   从 `pq` 中取出距离最小的 `(距离u, 顶点u)`。
    *   如果 `vis[u]` 已经是 `true`（说明 `u` 的最短路径已经被确定过了），则跳过这个顶点，继续下一个。这是因为优先队列中可能存在同一个顶点的多条路径，我们只关心最短的那条。
    *   将 `vis[u]` 设为 `true`，表示顶点 `u` 的最短路径已确定。
    *   **松弛操作：** 遍历顶点 `u` 的所有邻居 `v`：
        *   设 `u` 到 `v` 的边权为 `w`。
        *   如果 `d[u] + w < d[v]`（即通过 `u` 到 `v` 的路径比之前已知的路径更短），则：
            *   更新 `d[v] = d[u] + w`。
            *   将新的 `(d[v], v)` 放入优先队列 `pq`。

3.  **结束：** 当优先队列为空时，算法结束。此时 `d[i]` 中存储的就是从起点 `s` 到所有顶点 `i` 的最短距离。

### 4. 数据结构选择与优化

*   **朴素Dijkstra (O(N^2))**: 每次循环都需要遍历所有未访问的顶点来找到距离最小的那个。当顶点数量 `N` 很大时，效率较低。
*   **优先队列优化版 (O((N+M) log N))**: 使用优先队列 `priority_queue` 来自动维护当前距离最小的未访问顶点，每次取出 `O(log N)`，松弛操作 `O(M log N)`。这是在竞赛中常用的高效版本。
    *   `N` 是顶点数，`M` 是边数。

### 5. 适用范围与局限性

*   **优点:** 算法思想直观，实现相对简单，效率高（优先队列优化后），广泛应用于导航、网络路由等领域。
*   **缺点:** **不能处理含有负权边的图**。如果图中存在负权边，Dijkstra算法可能会给出错误的结果。遇到负权边图，需要使用Bellman-Ford或SPFA算法。

## 代码示例

下面是使用优先队列优化的Dijkstra算法的C++实现。

```cpp
#include <bits/stdc++.h>

using namespace std;

const int INF = 0x3f3f3f3f; // 定义一个很大的数表示无穷大

// N: 最大顶点数
const int N = 100005; 
// M: 最大边数 (通常为 N 的倍数)
const int M = 200005; 

// 邻接表存储图
// adj[u] 存储 pair<int, int>，表示 (邻居顶点v, 边权w)
vector<pair<int, int>> adj[N]; 

int d[N];       // 存储从起点到各个顶点的最短距离
bool vis[N];    // 标记顶点是否已经确定了最短路径

int n, m, s;    // n: 顶点数, m: 边数, s: 起点

void dijkstra() {
    // 初始化距离数组为无穷大，起点距离为0
    for (int i = 1; i <= n; ++i) {
        d[i] = INF;
        vis[i] = false;
    }
    d[s] = 0;

    // 优先队列，存储 pair<int, int>，表示 (距离, 顶点)
    // C++的priority_queue默认是大根堆，我们希望距离小的优先出队
    // 所以存储 (-距离, 顶点)，或者使用 greater<pair<int, int>>
    // 这里我们使用 pair<int, int>，第一个元素是距离，第二个是顶点
    // 为了实现小根堆，我们存储负的距离
    priority_queue<pair<int, int>> pq; 
    pq.push({0, s}); // 将起点s及其距离0放入优先队列

    while (!pq.empty()) {
        // 取出当前距离最小的顶点
        int u = pq.top().second;
        int dist_u = -pq.top().first; // 注意：取出的距离是负数，要转正
        pq.pop();

        // 如果u已经确定了最短路径，则跳过
        if (vis[u]) {
            continue;
        }

        // 标记u的最短路径已确定
        vis[u] = true;

        // 松弛操作：遍历u的所有邻居v
        for (auto& edge : adj[u]) {
            int v = edge.first;  // 邻居顶点
            int w = edge.second; // 边权

            // 如果通过u到达v的路径更短，则更新d[v]
            if (d[u] + w < d[v]) {
                d[v] = d[u] + w;
                // 将更新后的 (距离v, 顶点v) 放入优先队列
                pq.push({-d[v], v}); // 再次存储负距离
            }
        }
    }
}

int main() {
    // 读入顶点数n，边数m，起点s
    cin >> n >> m >> s;

    // 读入m条边
    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w}); // 添加从u到v，权值为w的边
    }

    dijkstra(); // 调用Dijkstra算法

    // 输出从起点s到每个顶点的最短距离
    for (int i = 1; i <= n; ++i) {
        cout << d[i] << " ";
    }
    cout << endl;

    return 0;
}
```

**代码解释：**

*   `const int INF = 0x3f3f3f3f;`: 定义一个非常大的整数作为“无穷大”，通常用于初始化距离。
*   `adj[N]`: 这是一个邻接表，`adj[u]` 存储所有从 `u` 出发的边。每条边是一个 `pair<int, int>`，第一个元素是目标顶点 `v`，第二个元素是边权 `w`。
*   `d[N]`: 数组 `d[i]` 存放从起点 `s` 到顶点 `i` 的最短距离。
*   `vis[N]`: 布尔数组 `vis[i]` 标记顶点 `i` 是否已经从优先队列中取出并处理过（即其最短路径已确定）。
*   `priority_queue<pair<int, int>> pq;`: 优先队列，用于存储 `(距离, 顶点)` 对。为了让距离小的优先出队（小根堆），我们将距离取负后存入队列。这样，距离越小，负距离的绝对值越大，在默认大根堆中就会优先出队。
*   `pq.push({-d[v], v});`: 将更新后的距离 `d[v]` 和顶点 `v` 压入优先队列。
*   `dist_u = -pq.top().first;`: 取出时，将负距离转回正距离。
*   `if (vis[u]) continue;`: 这是一个重要的优化。优先队列中可能会有同一个顶点的多个 `(距离, 顶点)` 对。如果一个顶点 `u` 已经被标记 `vis[u]=true`，说明它已经被处理过，并且找到了最短路径，那么队列中后续出现的关于 `u` 的路径（必然是更长的或等长的）就可以直接跳过。

## 课堂互动

### 1. 提问环节

*   同学们，Dijkstra算法的核心思想是什么？它每次都做出了什么“看起来最好”的选择？
    *   **提示：** 贪心策略，每次选择当前离起点最近的未确定最短路径的顶点。
*   Dijkstra算法能处理负权边的图吗？为什么？
    *   **提示：** 不能。因为一旦一个点的最短路径被确定，Dijkstra就认为它是最终的，不会再改变。如果存在负权边，后续可能通过负权边发现更短的路径，但Dijkstra无法回溯。
*   在我们的代码中，优先队列里存放的是 `(-距离, 顶点)`，这是为什么呢？
    *   **提示：** C++的 `priority_queue` 默认是大根堆，它会把最大的元素放在顶部。为了让距离最小的元素优先出队，我们把距离取负，这样距离越小，负值越大（例如 -2 > -10），就能实现小根堆的效果。

### 2. 小组任务：手动模拟Dijkstra

现在，请大家分成小组，我们来一起模拟Dijkstra算法的执行过程。

**图示：**

```
    (A) --5-- (B)
     |       / |
     2     3   6
     |   /     |
    (C) --1-- (D)
             \
              4
               \
                (E)
```

**任务：**
请以顶点 `A` 为起点，使用Dijkstra算法计算出从 `A` 到所有其他顶点的最短距离。
请记录每一步 `d` 数组和 `vis` 数组的变化，以及优先队列 `pq` 的内容。

**步骤提示：**

1.  初始化 `d` 和 `vis` 数组，将 `(0, A)` 加入 `pq`。
2.  循环：取出 `pq` 顶部元素，标记访问，更新邻居。

**答案解析（老师在学生完成后来讲解）：**

*   **初始化:**
    *   `d = {A:0, B:INF, C:INF, D:INF, E:INF}`
    *   `vis = {A:F, B:F, C:F, D:F, E:F}`
    *   `pq = {(-0, A)}`

*   **第1步:**
    *   取出 `(0, A)`。 `vis[A]=T`。
    *   邻居B: `d[A]+5 < d[B]` (`0+5 < INF`) -> `d[B]=5`, `pq.push({-5, B})`
    *   邻居C: `d[A]+2 < d[C]` (`0+2 < INF`) -> `d[C]=2`, `pq.push({-2, C})`
    *   `d = {A:0, B:5, C:2, D:INF, E:INF}`
    *   `vis = {A:T, B:F, C:F, D:F, E:F}`
    *   `pq = {(-2, C), (-5, B)}` (顺序可能不同，取决于实现)

*   **第2步:**
    *   取出 `(2, C)`。 `vis[C]=T`。
    *   邻居B: `d[C]+3 < d[B]` (`2+3 < 5`) -> `d[B]=5` (不更新，因为不严格小于)
        *   或者 `d[C]+3 < d[B]` (`5 < 5`) -> 这一步取决于你是否允许 `d[u]+w == d[v]` 时更新，通常是严格小于才更新。
    *   邻居D: `d[C]+1 < d[D]` (`2+1 < INF`) -> `d[D]=3`, `pq.push({-3, D})`
    *   `d = {A:0, B:5, C:2, D:3, E:INF}`
    *   `vis = {A:T, B:F, C:T, D:F, E:F}`
    *   `pq = {(-3, D), (-5, B)}`

*   **第3步:**
    *   取出 `(3, D)`。 `vis[D]=T`。
    *   邻居B: `d[D]+6 < d[B]` (`3+6 < 5`) -> `9 < 5` (不成立)
    *   邻居E: `d[D]+4 < d[E]` (`3+4 < INF`) -> `d[E]=7`, `pq.push({-7, E})`
    *   `d = {A:0, B:5, C:2, D:3, E:7}`
    *   `vis = {A:T, B:F, C:T, D:T, E:F}`
    *   `pq = {(-5, B), (-7, E)}`

*   **第4步:**
    *   取出 `(5, B)`。 `vis[B]=T`。
    *   邻居D: `d[B]+6 < d[D]` (`5+6 < 3`) -> `11 < 3` (不成立)
    *   `d = {A:0, B:5, C:2, D:3, E:7}`
    *   `vis = {A:T, B:T, C:T, D:T, E:F}`
    *   `pq = {(-7, E)}`

*   **第5步:**
    *   取出 `(7, E)`。 `vis[E]=T`。
    *   E没有邻居。
    *   `d = {A:0, B:5, C:2, D:3, E:7}`
    *   `vis = {A:T, B:T, C:T, D:T, E:T}`
    *   `pq = {}`

*   **最终结果：**
    *   A到A：0
    *   A到B：5
    *   A到C：2
    *   A到D：3
    *   A到E：7

    **解析：** Dijkstra算法不能处理负权边，因此A、C、D选项都不正确。含有环的图只要边权非负，Dijkstra算法是完全可以处理的。

3.  **答案：C**
    **解析：** 使用邻接表存储图，并结合优先队列进行优化的Dijkstra算法，其时间复杂度为 `O((N+M) log N)`。其中 `N` 是顶点数，`M` 是边数。每次优先队列操作是 `O(log N)`，每个顶点最多入队出队一次，每条边最多松弛一次。

4.  **答案：C**
    **解析：** 松弛操作（Relaxation）是指当我们确定了一个顶点 `u` 的最短路径后，尝试通过 `u` 来更新其所有邻居 `v` 的最短距离 `d[v]`。如果 `d[u] + w(u,v) < d[v]`，则更新 `d[v]`。

5.  **答案：C**
    **解析：** 如果Dijkstra算法结束后，从起点到某个顶点的距离仍然是 `INF`，这意味着在算法执行过程中，该顶点从未被松弛过，即从起点到该顶点不存在任何路径。

### 编程题参考答案

（此部分代码与“代码示例”中的代码完全一致，因为编程题就是Dijkstra的模板题，旨在让学生实践）

```cpp
#include <bits/stdc++.h>

using namespace std;

const int INF = 0x3f3f3f3f; 

const int N = 100005; 
const int M = 200005; 

vector<pair<int, int>> adj[N]; 

int d[N];       
bool vis[N];    

int n, m, s;    

void dijkstra() {
    for (int i = 1; i <= n; ++i) {
        d[i] = INF;
        vis[i] = false;
    }
    d[s] = 0;

    priority_queue<pair<int, int>> pq; 
    pq.push({0, s}); 

    while (!pq.empty()) {
        int u = pq.top().second;
        int dist_u = -pq.top().first; 
        pq.pop();

        if (vis[u]) {
            continue;
        }

        vis[u] = true;

        for (auto& edge : adj[u]) {
            int v = edge.first;  
            int w = edge.second; 

            if (d[u] + w < d[v]) {
                d[v] = d[u] + w;
                pq.push({-d[v], v}); 
            }
        }
    }
}

int main() {
    cin >> n >> m >> s;

    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w}); 
    }

    dijkstra(); 

    for (int i = 1; i <= n; ++i) {
        cout << d[i] << " ";
    }
    cout << endl;

    return 0;
}
```
