# 趣味引入

同学们，想象一下我们有一个长长的灯带，上面有N个灯泡，编号从1到N。一开始，所有的灯泡亮度都是0。

现在，我们想玩一个游戏：
1.  **调整亮度**：有时，我们会选择灯带上的一段区间 `[L, R]`，然后把这一段内所有灯泡的亮度都增加 `X`。
2.  **查看亮度**：有时，我们会想知道某个特定灯泡 `P` 的当前亮度是多少。
3.  **查看总亮度**：有时，我们还会想知道某一段区间 `[L, R]` 内所有灯泡的总亮度是多少。

如果N和操作M都很大（比如10万），我们用一个普通数组 `a[N]` 来存储亮度，每次调整亮度都循环 `L` 到 `R`，这样效率会非常低，可能需要 `O(N)` 的时间。M次操作就是 `O(M*N)`，这在竞赛中肯定会超时！

我们之前学过树状数组（Fenwick Tree），它能高效地处理单点更新和区间求和（前缀和）。但是，它能直接处理区间更新吗？
比如，`a[L...R] += X`，如果用树状数组，我们得对 `L` 到 `R` 之间的每个点都进行 `update` 操作，这不还是 `O(N)` 吗？

有没有一种“魔法”，能把区间更新变成树状数组擅长的单点更新呢？今天，我们就来学习这种“魔法”——**差分树状数组**！它能让我们在 `O(logN)` 的时间完成区间更新和区间查询！

# 深度知识点讲解

## 1. 差分数组的奥秘

我们先来复习一下差分数组（Difference Array）的概念。

### 1.1 什么是差分数组？
假设我们有一个原始数组 `a`，它的差分数组 `d` 定义为：
`d[i] = a[i] - a[i-1]` （约定 `a[0] = 0`）

那么，原始数组 `a[i]` 就可以通过差分数组的前缀和得到：
`a[i] = d[1] + d[2] + ... + d[i]`
是不是很神奇？ `a[i]` 就是 `d` 数组的“前缀和”！

**生活类比**：
想象你有一张存折，`a[i]` 是你第 `i` 天结束时的存款总额。那么 `d[i]` 就是你第 `i` 天的收入（或支出，如果是负数）。
`a[i]` 自然就是从第1天到第 `i` 天所有收入的总和。

### 1.2 差分数组如何处理区间更新？
这是差分数组最核心的优点！
假设我们要对原始数组 `a` 的区间 `[L, R]` 加上一个值 `X`，即 `a[i] += X`，对于 `L <= i <= R`。

来看看这会如何影响 `d` 数组：
*   **对于 `i < L`**：`a[i]` 和 `a[i-1]` 都没有变化，所以 `d[i] = a[i] - a[i-1]` 也不变。
*   **对于 `i = L`**：
    `d[L]` 变为 `(a[L]+X) - a[L-1]`。
    新的 `d[L]` = `(a[L] - a[L-1]) + X` = 原来的 `d[L] + X`。
    所以，我们只需要让 `d[L] += X`。
*   **对于 `L < i <= R`**：
    `a[i]` 和 `a[i-1]` 都增加了 `X`。
    新的 `d[i]` = `(a[i]+X) - (a[i-1]+X)` = `a[i] - a[i-1]` = 原来的 `d[i]`。
    所以， `d[i]` 在这个区间内是**不变**的！
*   **对于 `i = R+1`**：
    `a[R+1]` 没有变化，`a[R]` 增加了 `X`。
    新的 `d[R+1]` = `a[R+1] - (a[R]+X)` = `(a[R+1] - a[R]) - X` = 原来的 `d[R+1] - X`。
    所以，我们只需要让 `d[R+1] -= X`。
*   **对于 `i > R+1`**：`a[i]` 和 `a[i-1]` 都没变化，`d[i]` 也不变。

**结论**：对原始数组 `a` 的区间 `[L, R]` 加上 `X`，只需要对差分数组 `d` 进行两次单点更新：
1.  `d[L] += X`
2.  `d[R+1] -= X`

**形象比喻**：
想象你有一个长长的水管，代表数组 `a`。
`d[i]` 就像在 `i` 位置有一个阀门，它控制着 `i` 之后的水位相对 `i-1` 位置的变化。
当你在 `[L, R]` 区间内所有位置都增加 `X` 的水位时：
*   在 `L` 位置，你打开一个水龙头，让水位增加 `X` (`d[L]+=X`)。
*   这段水流会一直流到 `R`。为了不影响 `R+1` 之后的水位，你在 `R+1` 位置关掉这个水龙头（或者说，打开一个排水口，让水位减少 `X`）(`d[R+1]-=X`)。
这样，只有 `[L, R]` 区间内的水位增加了 `X`，而其他地方不受影响。

## 2. 差分树状数组（Range Update, Point Query）

现在我们结合树状数组。
我们知道 `a[i] = sum(d[1...i])`。
如果 `d` 数组的单点更新和前缀和查询都能高效进行，那么 `a` 数组的区间更新和单点查询就能高效进行。

树状数组（Fenwick Tree）正是擅长：
*   单点更新 `d[k] += val`：`O(logN)`
*   前缀和查询 `sum(d[1...k])`：`O(logN)`

所以，我们可以用一个树状数组来维护差分数组 `d`。

**操作方法**：
*   **区间更新 `a[L...R] += X`**：
    转换为对差分数组 `d` 的单点更新：
    `BIT.update(L, X)`
    `BIT.update(R+1, -X)`
    时间复杂度：`O(logN)`
*   **单点查询 `a[P]`**：
    `a[P]` 等于 `d` 数组的前缀和 `sum(d[1...P])`：
    `BIT.query(P)`
    时间复杂度：`O(logN)`

**初始化**：
如果原始数组 `a` 初始不全为0，我们需要先构建 `d` 数组。
`d[1] = a[1]`
`d[i] = a[i] - a[i-1]` for `i > 1`
然后用这些 `d[i]` 值构建树状数组。或者更简单地，将 `a[i]` 视为对 `a[i]` 的区间 `[i, i]` 加上 `a[i]`。
所以，`BIT.update(i, a[i] - a[i-1])` for `i=1...N` (where `a[0]=0`).

## 3. 差分树状数组（Range Update, Range Query）

这是更高级的应用，我们不仅要查询单点，还要查询区间和 `sum(a[L...R])`。
我们知道 `sum(a[L...R]) = sum(a[1...R]) - sum(a[1...L-1])`。
所以，问题转化为如何高效地计算 `sum(a[1...k])`。

我们已经知道 `a[i] = sum(d[1...i])`。
那么 `sum(a[1...k])` 就是：
`sum(a[1...k]) = a[1] + a[2] + ... + a[k]`
`= (d[1])`
`+ (d[1] + d[2])`
`+ (d[1] + d[2] + d[3])`
`+ ...`
`+ (d[1] + d[2] + ... + d[k])`

我们可以把上面这个表达式重新整理一下，计算每个 `d[j]` 出现了多少次：
*   `d[1]` 出现了 `k` 次
*   `d[2]` 出现了 `k-1` 次
*   ...
*   `d[j]` 出现了 `k-j+1` 次
*   ...
*   `d[k]` 出现了 `1` 次

所以，`sum(a[1...k]) = sum_{j=1 to k} (k-j+1) * d[j]`
`= sum_{j=1 to k} (k+1)*d[j] - sum_{j=1 to k} j*d[j]`

这个式子非常重要！
它把一个复杂的区间和问题，拆分成了两个相对简单的求和问题：
1.  `sum_{j=1 to k} d[j]`：这是 `d` 数组的前缀和，可以用一个树状数组 `BIT1` 来维护。
2.  `sum_{j=1 to k} j*d[j]`：这是 `d[j]` 乘上其下标 `j` 后的前缀和，可以用另一个树状数组 `BIT2` 来维护。

**操作方法**：
我们需要两个树状数组：
*   `BIT1`：维护 `d[i]` 的值。
*   `BIT2`：维护 `i * d[i]` 的值。

*   **区间更新 `a[L...R] += X`**：
    根据差分数组的性质，这会导致：
    `d[L] += X`
    `d[R+1] -= X`

    所以，我们需要更新 `BIT1` 和 `BIT2`：
    1.  更新 `BIT1` (维护 `d[i]`)：
        `BIT1.update(L, X)`
        `BIT1.update(R+1, -X)`
    2.  更新 `BIT2` (维护 `i * d[i]`)：
        `BIT2.update(L, L * X)`
        `BIT2.update(R+1, (R+1) * (-X))`
    时间复杂度：`O(logN)`

*   **前缀和查询 `sum(a[1...k])`**：
    根据推导出的公式：
    `sum(a[1...k]) = (k+1) * BIT1.query(k) - BIT2.query(k)`
    时间复杂度：`O(logN)`

*   **区间和查询 `sum(a[L...R])`**：
    `sum(a[L...R]) = query_prefix_sum(R) - query_prefix_sum(L-1)`
    其中 `query_prefix_sum(k)` 就是上面的 `(k+1) * BIT1.query(k) - BIT2.query(k)`。
    时间复杂度：`O(logN)`

**初始化**：
同样，如果原始数组 `a` 初始不全为0，需要先构建 `d` 数组。
`d[1] = a[1]`
`d[i] = a[i] - a[i-1]` for `i > 1`
然后对 `i=1...N`，执行：
`BIT1.update(i, d[i])`
`BIT2.update(i, i * d[i])`
或者更巧妙地，将原始数组的每个 `a[i]` 视为对区间 `[i, i]` 加上 `a[i]`。那么，我们可以通过 `update(i, a[i])` 和 `update(i+1, -a[i])` 来初始化。这个方法有点绕，直接计算 `d[i]` 并更新两个BIT会更直观。

**常见误区**：
1.  **`R+1` 边界问题**：在 `update` 时，`d[R+1]` 是关键，很容易写成 `R`。
2.  **`long long` 类型**：在计算前缀和时，尤其是有 `N*X` 这种乘法，结果可能非常大，要记得使用 `long long`。
3.  **两个BIT的同步更新**：区间更新时，必须同时更新 `BIT1` 和 `BIT2`。

# 典型例题精讲

## 例题1：区间更新，单点查询

**题目描述**：
有一个长度为 `N` 的数组 `a`，初始值均为0。现在有 `M` 次操作，每次操作有两种类型：
1.  `1 L R X`：表示将区间 `[L, R]` 内的每个元素都加上 `X`。
2.  `2 P`：表示查询 `a[P]` 的值。

**输入格式**：
第一行两个整数 `N, M`。
接下来 `M` 行，每行表示一个操作。

**输出格式**：
对于每个类型2操作，输出查询结果。

**数据范围**：`N, M <= 10^5`，`|X| <= 1000`。

**思路分析**：
这正是我们差分树状数组（单点查询）的经典应用场景。
1.  维护一个树状数组 `BIT` 来存储差分数组 `d` 的信息。
2.  对于类型1操作 `1 L R X`：
    *   将 `a[L...R] += X` 转化为 `d[L] += X` 和 `d[R+1] -= X`。
    *   对应 `BIT.update(L, X)` 和 `BIT.update(R+1, -X)`。
3.  对于类型2操作 `2 P`：
    *   `a[P]` 等于 `d` 数组的前缀和 `sum(d[1...P])`。
    *   对应 `BIT.query(P)`。

**代码实现**：

```cpp
#include <bits/stdc++.h>

// 最大数组长度
const int MAXN = 100005;

// 树状数组，使用 long long 避免溢出
long long bit[MAXN];
int n; // 数组长度

// lowbit 函数
int lowbit(int x) {
    return x & (-x);
}

// 单点更新：将下标 i 处的值增加 val
void update(int i, long long val) {
    for (; i <= n; i += lowbit(i)) {
        bit[i] += val;
    }
}

// 前缀和查询：查询下标 1 到 i 的和
long long query(int i) {
    long long s = 0;
    for (; i > 0; i -= lowbit(i)) {
        s += bit[i];
    }
    return s;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int m;
    std::cin >> n >> m;

    // 初始数组 a 均为0，所以差分数组 d 也均为0
    // 不需要额外的初始化操作

    for (int k = 0; k < m; ++k) {
        int op, L, R, P;
        long long X; // X 可能为负数
        std::cin >> op;
        if (op == 1) {
            std::cin >> L >> R >> X;
            update(L, X);
            // 注意 R+1 可能越界，但由于我们只查询到 N，
            // 且 R+1 处的更新只影响 R+1 及之后，所以即使越界到 N+1 也是可以的。
            // 只要保证 update 内部 i <= n 即可
            // 如果 R+1 > N，则这个 update 不会影响 1..N 的查询结果
            if (R + 1 <= n) { // 优化：只在R+1在有效范围内才更新，否则无意义
                update(R + 1, -X);
            }
        } else {
            std::cin >> P;
            std::cout << query(P) << "\n";
        }
    }

    return 0;
}
```

**时间复杂度分析**：
*   初始化：`O(1)` (因为初始全0，不需要构建 `d` 数组)
*   区间更新：`update` 操作执行两次，每次 `O(logN)`。总计 `O(logN)`。
*   单点查询：`query` 操作执行一次，`O(logN)`。
*   总时间复杂度：`M` 次操作，每次 `O(logN)`，所以总计 `O(M logN)`。
    对于 `N, M <= 10^5`，`10^5 * log(10^5) ≈ 10^5 * 17`，完全可以接受。

## 例题2：区间更新，区间查询

**题目描述**：
有一个长度为 `N` 的数组 `a`，初始值均为0。现在有 `M` 次操作，每次操作有两种类型：
1.  `1 L R X`：表示将区间 `[L, R]` 内的每个元素都加上 `X`。
2.  `2 L R`：表示查询区间 `[L, R]` 内所有元素的和。

**输入格式**：
第一行两个整数 `N, M`。
接下来 `M` 行，每行表示一个操作。

**输出格式**：
对于每个类型2操作，输出查询结果。

**数据范围**：`N, M <= 10^5`，`|X| <= 1000`。

**思路分析**：
这正是我们差分树状数组（区间查询）的经典应用场景，需要两个树状数组。
1.  维护两个树状数组：
    *   `BIT1` 维护 `d[i]` 的信息。
    *   `BIT2` 维护 `i * d[i]` 的信息。
2.  对于类型1操作 `1 L R X`：
    *   将 `a[L...R] += X` 转化为 `d[L] += X` 和 `d[R+1] -= X`。
    *   对应 `BIT1.update(L, X)` 和 `BIT1.update(R+1, -X)`。
    *   同时，对于 `i * d[i]` 的影响是：
        `d[L]` 增加 `X`，导致 `L * d[L]` 增加 `L * X`。
        `d[R+1]` 减少 `X`，导致 `(R+1) * d[R+1]` 减少 `(R+1) * X`。
    *   对应 `BIT2.update(L, L * X)` 和 `BIT2.update(R+1, (R+1) * (-X))`。
3.  对于类型2操作 `2 L R`：
    *   查询 `sum(a[L...R])` 等于 `query_prefix_sum(R) - query_prefix_sum(L-1)`。
    *   其中 `query_prefix_sum(k) = (k+1) * BIT1.query(k) - BIT2.query(k)`。

**代码实现**：

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005;

// 两个树状数组
long long bit1[MAXN]; // 维护 d[i]
long long bit2[MAXN]; // 维护 i * d[i]
int n; // 数组长度

// lowbit 函数
int lowbit(int x) {
    return x & (-x);
}

// 树状数组更新函数 (通用，根据传入的 bit 数组更新)
void update_bit(long long bit_arr[], int i, long long val) {
    for (; i <= n; i += lowbit(i)) {
        bit_arr[i] += val;
    }
}

// 树状数组查询函数 (通用，根据传入的 bit 数组查询)
long long query_bit(long long bit_arr[], int i) {
    long long s = 0;
    for (; i > 0; i -= lowbit(i)) {
        s += bit_arr[i];
    }
    return s;
}

// 区间更新操作
void range_update(int L, int R, long long X) {
    // 更新 BIT1 (维护 d[i])
    update_bit(bit1, L, X);
    if (R + 1 <= n) {
        update_bit(bit1, R + 1, -X);
    }

    // 更新 BIT2 (维护 i * d[i])
    update_bit(bit2, L, L * X);
    if (R + 1 <= n) {
        update_bit(bit2, R + 1, (R + 1) * (-X));
    }
}

// 查询前缀和 sum(a[1...k])
long long query_prefix_sum(int k) {
    if (k <= 0) return 0; // 边界情况处理
    return (k + 1) * query_bit(bit1, k) - query_bit(bit2, k);
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int m;
    std::cin >> n >> m;

    // 初始数组 a 均为0，所以差分数组 d 也均为0
    // 不需要额外的初始化操作

    for (int k = 0; k < m; ++k) {
        int op, L, R;
        long long X;
        std::cin >> op;
        if (op == 1) {
            std::cin >> L >> R >> X;
            range_update(L, R, X);
        } else {
            std::cin >> L >> R;
            // sum(a[L...R]) = sum(a[1...R]) - sum(a[1...L-1])
            std::cout << query_prefix_sum(R) - query_prefix_sum(L - 1) << "\n";
        }
    }

    return 0;
}
```

**时间复杂度分析**：
*   初始化：`O(1)`。
*   区间更新：`update_bit` 操作执行四次，每次 `O(logN)`。总计 `O(logN)`。
*   区间查询：`query_prefix_sum` 操作执行两次，每次 `O(logN)`。总计 `O(logN)`。
*   总时间复杂度：`M` 次操作，每次 `O(logN)`，所以总计 `O(M logN)`。
    对于 `N, M <= 10^5`，同样可以高效处理。

# 代码实现模板

这里提供一个通用的树状数组 `BIT` 结构，方便同学们在实际编程中套用。
对于差分树状数组，你只需要创建两个 `BIT` 实例即可。

```cpp
#include <bits/stdc++.h> // 包含常用头文件，如 iostream, algorithm 等

const int MAXN = 100005; // 定义最大数组长度，根据题目要求调整

// 树状数组结构体
struct FenwickTree {
    long long tree[MAXN]; // 存储树状数组的数据，使用 long long 防止溢出
    int size;             // 树状数组维护的原始数组的有效长度 (1到size)

    // 构造函数，初始化树状数组
    FenwickTree(int n_size) : size(n_size) {
        // 将所有元素清零
        for (int i = 0; i <= size; ++i) {
            tree[i] = 0;
        }
    }

    // lowbit 函数：计算 x 的最低位1所代表的数值
    // 例如：lowbit(4) = 4, lowbit(6) = 2, lowbit(7) = 1
    int lowbit(int x) {
        return x & (-x);
    }

    // 单点更新：将原始数组中下标 i 处的值增加 val
    // 对应到差分数组 d[i] += val
    void update(int i, long long val) {
        // 从 i 开始，向上遍历所有包含 i 的区间，并更新它们的值
        for (; i <= size; i += lowbit(i)) {
            tree[i] += val;
        }
    }

    // 前缀和查询：查询原始数组中下标 1 到 i 的和
    // 对应到差分数组 sum(d[1...i])
    long long query(int i) {
        long long s = 0; // 存储前缀和
        // 从 i 开始，向下遍历所有包含 i 的区间，并累加它们的值
        for (; i > 0; i -= lowbit(i)) {
            s += tree[i];
        }
        return s;
    }
};
```

**使用说明**：
1.  根据题目 `N` 的最大值，调整 `MAXN`。
2.  在 `main` 函数中创建 `FenwickTree` 的实例。
    *   对于区间更新、单点查询，只需要一个 `FenwickTree bit_d(n);`

# 课堂互动

### 1. 思考题

1.  **为什么在对 `a[L...R]` 加上 `X` 时，`d[R+1]` 要减去 `X`？如果只 `d[L] += X` 会发生什么？**
    *   **提示**：回顾 `a[i]` 是 `d` 数组的前缀和这个性质。如果只 `d[L] += X`，那么 `a[L], a[L+1], ..., a[N]` 都会增加 `X`。为了让 `a[R+1]` 及之后的元素不受影响，我们需要在 `R+1` 处“抵消”掉这个 `X` 的增加。
    *   **答案**：因为 `a[i] = sum(d[1...i])`。如果只 `d[L] += X`，那么 `a[L]` 会增加 `X`，`a[L+1]` 也会增加 `X`，一直到 `a[N]` 都会增加 `X`。为了保证 `a[R+1]` 及之后的值不受影响，我们需要在 `d[R+1]` 处减去 `X`，这样 `d[L]` 增加的 `X` 的影响，从 `R+1` 开始就被抵消了。

2.  **在区间更新、区间查询的场景中，为什么需要两个树状数组 `BIT1` 和 `BIT2`？它们分别维护了什么信息？**
    *   **提示**：回顾 `sum(a[1...k])` 的推导公式。
    *   **答案**：因为 `sum(a[1...k]) = (k+1) * sum(d[1...k]) - sum(j * d[j] for j=1 to k)`。
        *   `BIT1` 维护 `sum(d[1...k])`，即差分数组 `d` 的前缀和。
        *   `BIT2` 维护 `sum(j * d[j] for j=1 to k)`，即差分数组元素乘以下标后的前缀和。
        这两个部分是计算 `a` 数组前缀和所必需的。

3.  **如果题目要求对区间 `[L, R]` 进行“赋值”操作（例如 `a[i] = X`），而不是“增加”操作，差分树状数组还能直接使用吗？为什么？**
    *   **提示**：差分数组的本质是记录“变化量”。赋值操作是绝对值，不是变化量。
    *   **答案**：不能直接使用。差分树状数组基于差分数组的性质，它擅长处理对区间进行“增加/减少”某个值的操作，因为这种操作可以转化为差分数组上的两个单点更新。而“赋值”操作会改变 `a[L-1]` 和 `a[L]` 之间的差值，以及 `a[R]` 和 `a[R+1]` 之间的差值，同时还会影响 `a[L]` 到 `a[R]` 之间所有相邻元素的差值，变得非常复杂。这种情况下，可能需要其他数据结构，如线段树（Segment Tree）。
