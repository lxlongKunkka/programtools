# 扩展并查集与应用

## 教学目标

### 知识目标
1.  **理解**：掌握普通并查集的局限性，理解“带权并查集”（Extended DSU）的核心思想，即在并查集结构中维护节点与其父节点之间的相对关系。
2.  **掌握**：理解 `d[i]` 数组的含义（表示节点 `i` 与其直接父节点 `parent[i]` 之间的某种关系），并掌握这种关系在 `find` 操作（路径压缩时更新）和 `union` 操作（合并时计算）中的维护方式。
3.  **熟悉**：了解带权并查集在处理具有相对关系（如相等、相反、距离差等）的连通性问题中的应用场景。

### 能力目标
1.  **建模能力**：能够将实际问题中涉及相对关系的连通性问题抽象为带权并查集模型。
2.  **实现能力**：能够独立编写带权并查集的 `find` 和 `union` 函数，并正确处理路径压缩中的权重更新及合并操作中的权重计算。
3.  **分析能力**：能够分析带权并查集的时间复杂度，理解路径压缩和按秩/大小合并对效率的提升作用。

### 素养目标
1.  **计算思维**：培养学生将复杂关系分解为基本相对关系，并通过迭代和累加（或异或）处理整体关系的能力。
2.  **抽象思维**：提升学生对数据结构进行改造和扩展，以适应更复杂问题的抽象能力。
3.  **严谨性**：在推导和实现关系更新逻辑时，培养学生逻辑推理的严谨性和代码实现的准确性。

## 深度知识点讲解

### 1. 普通并查集的局限性回顾

我们先来回忆一下普通的并查集：
*   **核心功能**：判断两个元素是否属于同一个集合，以及合并两个集合。
*   **数据结构**：一个 `parent` 数组，`parent[i]` 存储元素 `i` 的父节点。如果 `parent[i] == i`，则 `i` 是集合的根。
*   **关键操作**：
    *   `find(i)`：查找 `i` 所在集合的根节点，并进行路径压缩（让 `i` 直接指向根）。
    *   `union_sets(a, b)`：合并 `a` 和 `b` 所在的集合（通常通过将一个根的 `parent` 指向另一个根）。
*   **局限**：它只能回答“是/否”的问题（是否连通），无法存储和查询集合内元素之间的**相对关系**。比如，A和B是连通的，但A比B大5，或者A和B阵营相反，这些信息普通并查集都无法处理。

### 2. 什么是带权并查集？（Extended DSU）

带权并查集，顾名思义，就是在并查集的边上“带上权重”或者说“带上关系”。
这里的“权”不是指路径长度，而是指**当前节点与其父节点之间的某种相对关系**。

我们新增一个数组 `d[]`：
*   `d[i]`：表示节点 `i` 相对于其**直接父节点 `parent[i]`** 的某种关系值。
*   这个关系值可以是：
    *   **异或值**：例如，0表示同类，1表示异类。
    *   **差值**：例如，`i` 比 `parent[i]` 大多少。
    *   **模数**：例如，`i` 的值与 `parent[i]` 的值在模某个数意义下的关系。

**核心思想：**
当 `find` 操作进行路径压缩时，不仅要更新 `parent[i]` 指向根节点，还要同时更新 `d[i]`，使其表示 `i` 相对于**根节点**的关系。
当 `union` 操作合并两个集合时，需要根据两个集合根节点之间的关系，计算并更新被合并集合的根节点与其新父节点之间的关系。

### 3. `find` 操作的深度剖析：关系累加

这是带权并查集最精妙的部分！

想象一下，你正在追踪一个秘密任务：
*   你离你的直接上司有 `d[你]` 这么远。
*   你的直接上司离他的直接上司有 `d[你的上司]` 这么远。
*   以此类推，直到你们的老大（根节点）。

当你执行 `find(i)` 向上查找根节点时，`i` 会经过一系列中间节点 `p1, p2, ..., root`。
*   `i` 相对于 `p1` 的关系是 `d[i]`。
*   `p1` 相对于 `p2` 的关系是 `d[p1]`。
*   ...
*   `p_k` 相对于 `root` 的关系是 `d[p_k]`。

**路径压缩时，我们希望 `i` 直接指向 `root`。那么 `d[i]` 应该更新为 `i` 相对于 `root` 的关系。**
这个新 `d[i]` 应该等于 `d[i]` 加上 `d[p1]` 加上 `d[p2]` ... 加上 `d[root]`（当然 `d[root]` 默认为0，因为根相对于自己）。
这就像一个“关系链”的累加。

**具体实现：**
在 `find(i)` 递归向上查找时，如果 `parent[i]` 不是根节点：
1.  递归调用 `find(parent[i])`，这会返回 `parent[i]` 所在集合的根 `r`，并且 `parent[i]` 的 `d` 值会更新为 `parent[i]` 相对于 `r` 的关系。
2.  当 `find(parent[i])` 返回后，`parent[i]` 已经被压缩到 `r`。
3.  此时，`i` 相对于 `parent[i]` 的关系是旧的 `d[i]`。
4.  `parent[i]` 相对于 `r` 的关系是新的 `d[parent[i]]`。
5.  所以，`i` 相对于 `r` 的关系就是 `d[i]` 加上 `d[parent[i]]`。
6.  我们将 `parent[i]` 设置为 `r`，并将 `d[i]` 更新为这个累加后的值。

**伪代码（以加法为例）：**
```cpp
int find(int i) {
    if (parent[i] == i) {
        return i;
    }
    // 递归查找父节点的根
    int root = find(parent[i]);
    // 路径压缩时，更新 i 相对于 root 的关系
    // d[i] = (i 相对于 parent[i]) + (parent[i] 相对于 root)
    d[i] = d[i] + d[parent[i]];
    // 将 i 的父节点直接指向 root
    parent[i] = root;
    return root;
}
```
**注意：** 如果关系是异或，则用 `^` 代替 `+`。

### 4. `union` 操作的深度剖析：关系计算

当我们要合并 `a` 所在的集合和 `b` 所在的集合，并已知 `a` 和 `b` 之间存在某种关系 `T` 时：
1.  首先，调用 `find(a)` 和 `find(b)`，得到它们各自的根节点 `ra` 和 `rb`，以及 `a` 相对于 `ra` 的关系 `d_a_ra` (即 `d[a]` 的最终值) 和 `b` 相对于 `rb` 的关系 `d_b_rb` (即 `d[b]` 的最终值)。
2.  如果 `ra == rb`，说明 `a` 和 `b` 已经在同一个集合里了。此时可以检查关系是否冲突（比如，A和B已经是朋友了，你又说他们是敌人，这就有矛盾）。
3.  如果 `ra != rb`，我们需要将 `ra` 和 `rb` 合并。假设我们让 `rb` 的父节点指向 `ra` (即 `parent[rb] = ra`)。
4.  那么，新的 `d[rb]` 应该是什么呢？它应该表示 `rb` 相对于 `ra` 的关系。

我们来画个图理清关系链：
*   `ra` <-- `d_a_ra` <-- `a`
*   `rb` <-- `d_b_rb` <-- `b`
*   已知 `a` -- `T` -- `b` (表示 `a` 相对于 `b` 的关系是 `T`)

现在我们想知道 `rb` 相对于 `ra` 的关系。
从 `ra` 出发，经过 `a`，再到 `b`，再到 `rb`：
`ra` -- `(-d_a_ra)` -- `a` -- `T` -- `b` -- `(d_b_rb)` -- `rb`

所以，`ra` 相对于 `rb` 的关系是 `(-d_a_ra) + T + (d_b_rb)` (以加法为例，负号表示逆向关系)。
如果我们要设置 `parent[rb] = ra`，那么 `d[rb]` 存储的是 `rb` 相对于 `ra` 的关系。
所以 `d[rb] = (d_a_ra) + T - (d_b_rb)`。

**推导过程（以加法为例，`d[x]` 表示 `x - parent[x]`）：**
我们已知：
1.  `a - ra = d_a_ra` (即 `a` 相对于 `ra` 的关系)
2.  `b - rb = d_b_rb` (即 `b` 相对于 `rb` 的关系)
3.  `a - b = T` (即 `a` 相对于 `b` 的关系)

我们想计算 `rb - ra`。
`rb - ra = (rb - b) + (b - a) + (a - ra)`
`rb - ra = -(b - rb) + -(a - b) + (a - ra)`
`rb - ra = -d_b_rb - T + d_a_ra`
所以，如果 `parent[rb] = ra`，那么 `d[rb]` 就应该设置为 `d_a_ra - T - d_b_rb`。
**等等，这里有一个坑！** `T` 是 `a` 相对于 `b` 的关系。如果 `d[i]` 存储的是 `i` 相对于 `parent[i]` 的关系，那么 `a` 相对于 `ra` 的关系是 `d_a_ra`，`b` 相对于 `rb` 的关系是 `d_b_rb`。
我们想要让 `parent[rb] = ra`，也就是让 `rb` 的父节点是 `ra`。那么 `d[rb]` 应该存储 `rb` 相对于 `ra` 的关系。

**更直观的推导：**
假设 `d[i]` 表示 `i` 相对于 `parent[i]` 的某种“偏移量”或“状态差”。
`find(a)` 之后，`d[a]` 变成了 `a` 相对于 `ra` 的偏移量。
`find(b)` 之后, `d[b]` 变成了 `b` 相对于 `rb` 的偏移量。
我们现在知道 `a` 相对于 `b` 的关系是 `T`。
我们希望 `rb` 指向 `ra`，所以需要知道 `rb` 相对于 `ra` 的关系。

`ra` -- `d[a]` -- `a` -- `T` -- `b` -- `d[b]` -- `rb`
(这里的 `d[a]` 和 `d[b]` 已经是 `find` 后的值，表示 `a` 到 `ra` 和 `b` 到 `rb` 的关系)
从 `ra` 到 `rb` 的关系可以表示为：`ra -> a -> b -> rb`。
如果我们定义 `d[x]` 为 `x` 相对于 `parent[x]` 的关系，那么：
`关系(ra, a)` 是 `d[a]` (或者说 `d[a]` 是 `a` 相对于 `ra` 的关系，那么 `ra` 相对于 `a` 是 `-d[a]`)
`关系(a, b)` 是 `T`
`关系(b, rb)` 是 `d[b]` (或者说 `d[b]` 是 `b` 相对于 `rb` 的关系，那么 `rb` 相对于 `b` 是 `-d[b]`)

所以，`ra` 相对于 `rb` 的关系是 `(-d[a]) + T + d[b]` (注意正负号和方向)。
如果我们设置 `parent[rb] = ra`，那么 `d[rb]` 应该存储 `rb` 相对于 `ra` 的关系。
`d[rb] = - ((-d[a]) + T + d[b])`
`d[rb] = d[a] - T - d[b]` (这仍然是加法情况)

**以“朋友/敌人”为例（异或运算）：**
*   `d[i] = 0` 表示 `i` 和 `parent[i]` 是朋友。
*   `d[i] = 1` 表示 `i` 和 `parent[i]` 是敌人。
*   `find(x)` 之后，`d[x]` 表示 `x` 和 `root_x` 的关系（0为朋友，1为敌人）。
    *   `d[i] = d[i] ^ d[parent[i]]`
*   `union_sets(a, b, type)`：`type=0` 表示 `a, b` 是朋友，`type=1` 表示 `a, b` 是敌人。
    *   `ra = find(a)`, `rb = find(b)`
    *   `d[a]` 现在是 `a` 和 `ra` 的关系。
    *   `d[b]` 现在是 `b` 和 `rb` 的关系。
    *   我们想把 `rb` 挂到 `ra` 下面：`parent[rb] = ra`。
    *   我们需要计算 `rb` 和 `ra` 的关系，即新的 `d[rb]`。
    *   `ra` -- `d[a]` (关系) -- `a` -- `type` (关系) -- `b` -- `d[b]` (关系) -- `rb`
    *   从 `ra` 到 `rb` 的关系链是 `d[a] ^ type ^ d[b]`。
    *   但 `d[rb]` 存储的是 `rb` 相对于 `ra` 的关系，所以是 `d[a] ^ type ^ d[b]`。 (异或运算的逆运算是它本身)
    *   所以 `d[rb] = d[a] ^ type ^ d[b]`。

**总结：** 关键在于理解 `d[i]` 的确切含义，并根据这个含义进行**一致的**关系推导。
无论是加法还是异或，核心都是：`新关系(i, root) = 旧关系(i, parent[i]) + 新关系(parent[i], root)` (这里的 `+` 是广义的运算)。

### 5. 时间复杂度

与普通并查集相同，带权并查集结合路径压缩和按秩/大小合并，其平均时间复杂度接近于常数 `O(α(N))`，其中 `α` 是反阿克曼函数，增长极其缓慢，可以认为是常数时间。

### 6. 常见误区

1.  **`d[i]` 的含义不明确**：一定要明确 `d[i]` 到底表示 `i` 相对于 `parent[i]` 的什么关系，以及这种关系的正反方向。
2.  **`find` 中 `d` 值更新错误**：忘记在路径压缩时累加 `d[parent[i]]` 到 `d[i]`，或者累加顺序错误。
3.  **`union` 中 `d` 值计算错误**：推导 `d[新父]` 与 `d[旧父]` 之间的关系时，逻辑不清晰导致计算公式出错。
4.  **忘记初始化 `d` 数组**：通常 `d[i]` 初始为 0，表示每个节点初始时相对于自己（作为父节点）没有“偏移”。

## 典型例题精讲

### 例题一：食物链 (POJ 1182 简化版)

**题目描述：**
有 `N` 种动物，分别编号为 `1` 到 `N`。动物之间存在捕食关系：A 吃 B，B 吃 C，C 吃 A。这构成一个循环捕食链。
现在给出 `K` 条信息，每条信息包含三个整数 `T, X, Y`：
1.  如果 `T=1`：表示 `X` 和 `Y` 是同类动物。
2.  如果 `T=2`：表示 `X` 吃 `Y`。
你的任务是判断有多少条信息是假的（与之前的信息矛盾或动物编号超出范围）。

**分析：**
这是一个经典的带权并查集问题。
*   **元素**：`N` 只动物。
*   **关系**：同类 (0)，捕食 (1)，被捕食 (2)。
*   **`d[i]` 的含义**：我们让 `d[i]` 表示 `i` 相对于 `parent[i]` 的关系。
    *   `d[i] = 0`：`i` 和 `parent[i]` 是同类。
    *   `d[i] = 1`：`i` 吃 `parent[i]`。
    *   `d[i] = 2`：`parent[i]` 吃 `i` (即 `i` 被 `parent[i]` 吃)。
    *   这种关系是模3的：如果 `A` 吃 `B` (`d=1`)，`B` 吃 `C` (`d=1`)，那么 `A` 吃 `C` (`d=2`) 吗？不，`A` 和 `C` 是同类 (`d=0`)。
        *   `A` 吃 `B` (1), `B` 吃 `C` (1) => `A` 和 `C` 是同类 (0)
        *   `A` 吃 `B` (1), `B` 和 `C` 同类 (0) => `A` 吃 `C` (1)
        *   `A` 和 `B` 同类 (0), `B` 吃 `C` (1) => `A` 吃 `C` (1)
        *   这表明关系是 `(d_1 + d_2) % 3`。
        *   所以 `d[i]` 存储的是 `i` 相对于 `parent[i]` 的“捕食等级差”，模3。
            *   `d[i] = 0`：`i` 和 `parent[i]` 同类。
            *   `d[i] = 1`：`i` 比 `parent[i]` 高一级 (即 `i` 吃 `parent[i]`)。
            *   `d[i] = 2`：`i` 比 `parent[i]` 低一级 (即 `parent[i]` 吃 `i`)。

**`find(i)` 函数：**
当 `find(i)` 递归向上查找时：
`root = find(parent[i])`
`d[i] = (d[i] + d[parent[i]]) % 3` (累加关系，模3)
`parent[i] = root`
返回 `root`。

**`union_sets(x, y, T)` 函数：**
1.  **初步判断**：
    *   如果 `X` 或 `Y` 超出范围 `[1, N]`，则这条信息是假的。
    *   如果 `T=2` (X 吃 Y)，但 `X == Y`，则这条信息是假的（自己不能吃自己）。
2.  **获取根和相对关系**：
    `rx = find(x)` 得到 `x` 相对于 `rx` 的关系 `d[x]`。
    `ry = find(y)` 得到 `y` 相对于 `ry` 的关系 `d[y]`。
3.  **判断是否同集合**：
    *   如果 `rx == ry`：`x` 和 `y` 已经在同一个集合里。需要检查当前信息是否与已知信息矛盾。
        *   我们知道 `x` 相对于 `rx` 的关系是 `d[x]`。
        *   我们知道 `y` 相对于 `ry` 的关系是 `d[y]`。
        *   由于 `rx == ry`，所以 `x` 相对于 `y` 的已知关系是 `(d[x] - d[y] + 3) % 3`。
        *   如果 `T=1` (X, Y 同类)，则期望 `(d[x] - d[y] + 3) % 3 == 0`。
        *   如果 `T=2` (X 吃 Y)，则期望 `(d[x] - d[y] + 3) % 3 == 1`。
        *   如果期望关系不符，则这条信息是假的。
4.  **合并集合**：
    *   如果 `rx != ry`：将 `ry` 挂到 `rx` 下面 (`parent[ry] = rx`)。
    *   计算新的 `d[ry]`：它应该表示 `ry` 相对于 `rx` 的关系。
    *   已知：
        *   `x` 相对于 `rx` 的关系是 `d[x]`。
        *   `y` 相对于 `ry` 的关系是 `d[y]`。
        *   `x` 相对于 `y` 的关系是 `T-1` (0为同类，1为捕食)。
    *   我们希望 `ry` 相对于 `rx` 的关系。
    *   链条：`rx` -- `d[x]` -- `x` -- `(T-1)` -- `y` -- `d[y]` -- `ry`
    *   `ry` 相对于 `rx` 的关系是 `(d[x] - (T-1) - d[y] + 3) % 3`。
    *   所以 `d[ry] = (d[x] - (T-1) - d[y] + 3) % 3`。
        *   (这里 `T-1` 是 `x` 相对于 `y` 的关系，`d[x]` 是 `x` 相对于 `rx` 的关系，`d[y]` 是 `y` 相对于 `ry` 的关系。注意加减顺序和模3操作)
        *   更直观的推导：`rx` 到 `ry` 经过 `x` 和 `y`。
            `rel(rx, ry) = rel(rx, x) + rel(x, y) + rel(y, ry)` (这里的 `+` 是模3加法)
            `rel(rx, x)` 是 `(3 - d[x]) % 3` (因为 `d[x]` 是 `x` 相对于 `rx` 的关系)
            `rel(x, y)` 是 `(T-1)`
            `rel(y, ry)` 是 `(3 - d[y]) % 3`
            所以 `d[ry] = ( (3 - d[x]) % 3 + (T-1) + (3 - d[y]) % 3 ) % 3`
            化简：`d[ry] = ( d[x] + (T-1) - d[y] + 3 ) % 3` (注意这里 `d[x]` 是 `x` 相对于 `rx` 的关系，`d[y]` 是 `y` 相对于 `ry` 的关系。如果 `parent[ry] = rx`，`d[ry]` 应该表示 `ry` 相对于 `rx` 的关系，所以是 `d[x] - (T-1) - d[y]` 模3)
            最终的公式是 `d[ry] = (d[x] - (T-1) - d[y] + 3) % 3`
            因为 `(T-1)` 是 `x` 相对于 `y` 的关系，我们希望 `ry` 相对于 `rx` 的关系。
            `ry` 相对于 `rx` 的关系 = (`ry` 相对于 `y`) + (`y` 相对于 `x`) + (`x` 相对于 `rx`)
            `ry` 相对于 `y` 是 `(3 - d[y]) % 3`
            `y` 相对于 `x` 是 `(3 - (T-1)) % 3`
            `x` 相对于 `rx` 是 `d[x]`
            所以 `d[ry] = ((3 - d[y]) % 3 + (3 - (T-1)) % 3 + d[x]) % 3`
            `d[ry] = (d[x] - (T-1) - d[y] + 300) % 3` (为了避免负数，加一个足够大的3的倍数)
            这个公式是正确的。

**代码实现：**

```cpp
#include <iostream>

const int MAXN = 50005;
int parent[MAXN];
int d[MAXN]; // d[i] 表示 i 相对于 parent[i] 的关系
             // 0: 同类, 1: i 吃 parent[i], 2: parent[i] 吃 i

// 初始化并查集
void init(int n) {
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        d[i] = 0; // 初始时，每个节点相对于自己是同类
    }
}

// 查找根节点并路径压缩，同时更新 d 值
int find(int i) {
    if (parent[i] == i) {
        return i;
    }
    int root = find(parent[i]);
    // 关键步骤：更新 d[i]
    // d[i] 存储的是 i 相对于 parent[i] 的关系
    // d[parent[i]] 存储的是 parent[i] 相对于 root 的关系
    // i 相对于 root 的关系 = (i 相对于 parent[i]) + (parent[i] 相对于 root) (模3运算)
    d[i] = (d[i] + d[parent[i]]) % 3;
    parent[i] = root;
    return root;
}

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n, k;
    std::cin >> n >> k;

    init(n);

    int false_statements = 0;
    for (int q = 0; q < k; ++q) {
        int t, x, y;
        std::cin >> t >> x >> y;

        // 1. 检查编号是否合法
        if (x < 1 || x > n || y < 1 || y > n) {
            false_statements++;
            continue;
        }
        // 2. 特殊情况：自己吃自己是不允许的
        if (t == 2 && x == y) {
            false_statements++;
            continue;
        }

        int rx = find(x);
        int ry = find(y);

        if (rx == ry) {
            // X 和 Y 已经在同一个集合中，检查是否与现有关系矛盾
            // x 相对于 y 的关系 = (x 相对于 rx) - (y 相对于 ry) (模3运算)
            // (d[x] - d[y] + 3) % 3 表示 x 相对于 y 的关系
            if (t == 1) { // X 和 Y 是同类
                if ((d[x] - d[y] + 3) % 3 != 0) {
                    false_statements++;
                }
            } else { // t == 2, X 吃 Y
                if ((d[x] - d[y] + 3) % 3 != 1) {
                    false_statements++;
                }
            }
        } else {
            // X 和 Y 不在同一个集合中，合并它们
            parent[ry] = rx; // 将 ry 的根指向 rx
            // 计算新的 d[ry]：ry 相对于 rx 的关系
            // 关系链：rx --d[x]--> x --(t-1)--> y --d[y]--> ry
            // d[ry] = (ry 相对于 y) + (y 相对于 x) + (x 相对于 rx) (模3运算)
            // ry 相对于 y 是 (3 - d[y]) % 3
            // y 相对于 x 是 (3 - (t-1)) % 3
            // x 相对于 rx 是 d[x]
            d[ry] = ((3 - d[y]) % 3 + (3 - (t - 1)) % 3 + d[x]) % 3;
        }
    }

    std::cout << false_statements << std::endl;

    return 0;
}
```
**复杂度分析：**
*   **时间复杂度**：初始化 `O(N)`。每次查询或合并操作，`find` 函数平均时间复杂度为 `O(α(N))`。总共 `K` 次操作，所以总时间复杂度为 `O(N + Kα(N))`。
*   **空间复杂度**：`O(N)` 用于 `parent` 和 `d` 数组。

## 代码实现模板

这里提供一个通用的带权并查集模板，以处理**加法或异或**类型的关系为例。

```cpp
#include <iostream>
// #include <vector> // GESP 8 严禁使用 STL 容器 vector, stack, queue
// #include <numeric> // GESP 8 严禁使用，但有时可以用 <algorithm> 中的 sort

const int MAXN = 100005; // 根据题目 N 的范围调整
int parent[MAXN];
int d[MAXN]; // d[i] 存储 i 相对于 parent[i] 的关系

// 初始化并查集
// n: 元素的数量
void init_dsu(int n) {
    for (int i = 1; i <= n; ++i) { // 元素通常从 1 开始编号
        parent[i] = i;
        d[i] = 0; // 初始时，每个节点相对于自己（作为父节点）关系值为 0
                  // (例如，差值为 0，异或为 0 (同类))
    }
}

// 查找元素 i 的根节点，并进行路径压缩，同时更新 d 值
// 返回值：i 所在集合的根节点
// 副作用：d[i] 会更新为 i 相对于根节点的关系
int find_root(int i) {
    if (parent[i] == i) {
        return i; // i 就是根节点
    }
    // 递归查找父节点的根
    int root = find_root(parent[i]);
    
    // 关键步骤：路径压缩时更新 d[i]
    // d[i] 原来是 i 相对于 parent[i] 的关系
    // d[parent[i]] 现在是 parent[i] 相对于 root 的关系 (因为 find_root(parent[i]) 已经更新了它)
    // 所以，i 相对于 root 的关系 = (i 相对于 parent[i]) 关系运算 (parent[i] 相对于 root)
    // 具体运算取决于 d 的含义，可以是加法、异或等
    d[i] = (d[i] + d[parent[i]]); // 示例：加法类型关系
    // d[i] = (d[i] ^ d[parent[i]]); // 示例：异或类型关系
    // d[i] = (d[i] + d[parent[i]]) % MOD; // 示例：模数加法关系

    parent[i] = root; // 将 i 的父节点直接指向根
    return root;
}

// 合并元素 a 和 b 所在的集合，并设置它们之间的关系
// a, b: 要合并的两个元素
// relation_type: a 相对于 b 的关系值 (例如，1表示a比b大1，0表示a和b同类)
// 返回值：true 如果成功合并或关系一致，false 如果关系冲突
bool union_sets(int a, int b, int relation_type) {
    int ra = find_root(a); // a 的根，d[a] 此时是 a 相对于 ra 的关系
    int rb = find_root(b); // b 的根，d[b] 此时是 b 相对于 rb 的关系

    if (ra == rb) {
        // a 和 b 已经在同一个集合中，检查关系是否一致
        // 计算 a 相对于 b 的当前已知关系: (a 相对于 ra) - (b 相对于 rb)
        // (d[a] - d[b]) (加法类型) 或 (d[a] ^ d[b]) (异或类型)
        // 注意处理负数或模运算
        int current_relation_a_to_b = (d[a] - d[b]); // 示例：加法类型
        // int current_relation_a_to_b = (d[a] ^ d[b]); // 示例：异或类型

        // 如果关系不一致，说明有矛盾
        if (current_relation_a_to_b != relation_type) { // (注意模运算时要用 (current_relation_a_to_b % MOD + MOD) % MOD)
            return false; // 关系冲突
        }
        return true; // 关系一致
    } else {
        // a 和 b 不在同一个集合中，进行合并
        parent[rb] = ra; // 将 rb 所在的集合合并到 ra 所在的集合
                         // 此时 rb 的父节点变为 ra

        // 计算新的 d[rb]，它表示 rb 相对于其新父节点 ra 的关系
        // 关系链：ra --d[a]--> a --relation_type--> b --d[b]--> rb
        // 我们需要计算 rb 相对于 ra 的关系
        // rb 相对于 ra = (rb 相对于 b) + (b 相对于 a) + (a 相对于 ra)
        // (rb 相对于 b) 是 -(b 相对于 rb) = -d[b]
        // (b 相对于 a) 是 -(a 相对于 b) = -relation_type
        // (a 相对于 ra) 是 d[a]
        // 所以 d[rb] = d[a] - relation_type - d[b]
        // 注意这里的减法在模运算时要加 MOD
        d[rb] = (d[a] - relation_type - d[b]); // 示例：加法类型
        // d[rb] = (d[a] ^ relation_type ^ d[b]); // 示例：异或类型

        return true; // 成功合并
    }
}

// 示例主函数 (可以替换为具体问题的逻辑)
int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n = 5; // 5个元素
    init_dsu(n);

    // 假设关系是加法：d[i] = i - parent[i]
    // relation_type: a 相对于 b 的差值

    // 1 和 2，1比2大1 (1-2=1)
    union_sets(1, 2, 1);
    // 2 和 3，2比3大2 (2-3=2)
    union_sets(2, 3, 2);

    // 检查 1 和 3 的关系
    int r1 = find_root(1);
    int r3 = find_root(3);
    if (r1 == r3) {
        // 1 相对于 3 的差值 = (1 相对于 r1) - (3 相对于 r3)
        // = d[1] - d[3]
        std::cout << "1 和 3 的差值: " << (d[1] - d[3]) << std::endl; // 应该是 1 + 2 = 3
    }

    // 验证：
    // init: parent={0,1,2,3,4,5}, d={0,0,0,0,0,0}
    // union(1,2,1):
    //   ra=1, rb=2, d[1]=0, d[2]=0
    //   parent[2]=1, d[2]=(d[1]-1-d[2]) = (0-1-0) = -1
    //   parent={0,1,1,3,4,5}, d={0,0,-1,0,0,0}
    // union(2,3,2):
    //   find(2): root=1, d[2]更新为d[2]+d[1]=-1+0=-1. parent[2]=1.
    //            d={0,0,-1,0,0,0}, parent={0,1,1,3,4,5}
    //   find(3): root=3, d[3]=0.
    //   ra=1 (from 2), rb=3. d[2]=-1, d[3]=0.
    //   parent[3]=1, d[3]=(d[2]-2-d[3]) = (-1-2-0) = -3
    //   parent={0,1,1,1,4,5}, d={0,0,-1,-3,0,0}
    //
    // find_root(1): root=1, d[1]=0
    // find_root(3): root=1, d[3]更新为d[3]+d[1]=-3+0=-3. parent[3]=1.
    //               d={0,0,-1,-3,0,0}, parent={0,1,1,1,4,5}
    // r1=1, r3=1.
    // (d[1]-d[3]) = (0 - (-3)) = 3. 正确！

    return 0;
}
```
