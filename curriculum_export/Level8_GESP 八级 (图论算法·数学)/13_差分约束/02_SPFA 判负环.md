## 趣味引入

同学们，想象一下，你是一个小小的商人，手里拿着100块钱。你听说有三个国家 A、B、C 之间可以进行货币兑换。

*   从 A 国到 B 国，1块 A 币可以换 1.1 块 B 币。
*   从 B 国到 C 国，1块 B 币可以换 0.9 块 C 币。
*   从 C 国到 A 国，1块 C 币可以换 1.2 块 A 币。

你兴奋地拿起计算器算了一下：
1.  用 100 A 币换 B 币：100 * 1.1 = 110 B 币。
2.  再用 110 B 币换 C 币：110 * 0.9 = 99 C 币。
3.  最后用 99 C 币换回 A 币：99 * 1.2 = 118.8 A 币。

哇！你从 100 A 币出发，经过一圈兑换，竟然变成了 118.8 A 币！你赚了 18.8 A 币！如果你可以无限次地重复这个兑换过程，是不是就能无限地赚钱了呢？

这个“无限赚钱”的循环，在图论中，就叫做**负环**！
我们把“兑换比例”看作“边权”，但这里是乘法，不好直接用最短路。如果我们把兑换比例取个对数，再取个负号，那么“乘积最大化”就变成了“和最小化”了！
例如，从 A 到 B 兑换比例是 `rate_AB`，我们设边权为 `-log(rate_AB)`。那么 `log(rate_AB * rate_BC * rate_CA)` = `log(rate_AB) + log(rate_BC) + log(rate_CA)`. 如果这个和是正数，那么 `rate_AB * rate_BC * rate_CA` > 1，表示赚钱了！
而我们的边权是 `-log(rate)`，那么一个循环的路径和就是 `(-log(rate_AB)) + (-log(rate_BC)) + (-log(rate_CA))`。如果这个和是负数，就说明 `log(rate_AB * rate_BC * rate_CA)` 是正数，也就是 `rate_AB * rate_BC * rate_CA > 1`，我们找到了一个“赚钱”的负环！

所以，**负环（Negative Cycle）**，就是指图中一个环路的边权之和为负数。它在最短路问题中是个“麻烦精”，因为一旦有负环，最短路就可以无限地短下去（一直绕着负环转圈），导致最短路没有定义。但同时，它也是解决像货币套利这类问题的“金钥匙”！

今天，我们就来学习如何用 SPFA 算法，这个“最短路加速器”，来找出图中的负环！

## 深度知识点讲解

### 1. 什么是负环？为什么它很重要？

**负环**：在一个有向图中，如果存在一个环路，其所有边的权值之和为负数，那么这个环路就称为负环。

**为什么重要？**
*   **最短路问题中的“敌人”**：对于一般的最短路算法（如 Dijkstra、Bellman-Ford、SPFA），如果图中存在负环并且从源点可以到达这个负环，那么从源点到负环上任意一点的最短路将是负无穷大。因为你可以无限次地绕着负环走，每次都能让路径总长度变得更小。这使得最短路失去了意义。
*   **实际问题中的“财富”**：就像我们引入的货币套利问题，负环可能代表着可以无限获利的机会（在理想模型下），或者某种资源可以无限循环利用并产生净收益。因此，检测负环本身就是一个有用的应用。

### 2. 回顾 SPFA 算法（简要）

SPFA (Shortest Path Faster Algorithm) 是 Bellman-Ford 算法的一种队列优化版本。它用于解决带有负权边的单源最短路径问题。

**SPFA 的核心思想：**
1.  **初始化**：所有节点的距离 `d[u]` 设为无穷大，源点 `d[s]` 设为 0。
2.  **队列**：维护一个队列 `q`，存储所有可能需要进行松弛操作的节点。初始时，将源点 `s` 加入队列。
3.  **松弛操作**：
    *   每次从队列中取出一个节点 `u`。
    *   遍历 `u` 的所有出边 `(u, v)`，权值为 `w`。
    *   如果 `d[u] + w < d[v]`，说明经过 `u` 到 `v` 可以得到更短的路径，则更新 `d[v] = d[u] + w`。
    *   如果 `v` 更新了距离，并且 `v` 当前不在队列中，则将 `v` 加入队列。
4.  **重复**：直到队列为空。

**SPFA 的特点：**
*   **处理负权边**：能够正确处理负权边。
*   **平均效率高**：在随机图上表现优秀，平均时间复杂度接近 O(E)。
*   **最坏情况**：最坏情况下时间复杂度为 O(VE)，与 Bellman-Ford 相同。

### 3. SPFA 如何检测负环？—— 核心原理

Bellman-Ford 算法有一个重要性质：在一个含有 `n` 个节点的图中，如果不存在负环，那么从源点到任意节点的最短路径最多包含 `n-1` 条边。这意味着，在 Bellman-Ford 算法执行 `n-1` 轮松弛操作后，所有最短路径都会被找到。如果在第 `n` 轮（或之后）仍然有节点的距离可以被更新，那么图中一定存在负环。

SPFA 是 Bellman-Ford 的优化版本，它也隐含着这个性质。我们通过引入一个额外的数组 `count` 来记录每个节点被松弛（即其距离 `d[u]` 被更新）的次数。

**SPFA 检测负环的逻辑：**
1.  **`count` 数组**：对于每个节点 `u`，我们维护一个 `count[u]`，表示从源点到 `u` 的最短路径中，边的数量。更准确地说，它记录的是 `d[u]` 被成功更新的次数。
2.  **松弛与计数**：
    *   当 `d[u] + w < d[v]` 时，`d[v]` 被更新，说明我们找到了一条从源点到 `v` 的更短路径。
    *   这条新路径是通过 `u` 来的，所以它的边的数量应该是 `count[u] + 1`。因此，我们更新 `count[v] = count[u] + 1`。
    *   如果 `v` 之前不在队列中，且 `d[v]` 被更新了，就把它加入队列。
3.  **负环判定**：
    *   如果在松弛过程中，某个节点 `v` 的 `count[v]` 达到了 `n`（图中总节点数，通常用 `N` 表示，这里我们用 `n` 来表示节点数），这意味着从源点到 `v` 的一条路径，经过了 `n` 条边，并且这条路径还在不断变短（因为 `d[v]` 还在被更新）。
    *   在一个 `n` 个节点的图中，如果一条简单路径（不重复经过节点）包含 `n` 条边，这是不可能的（最多 `n-1` 条边）。因此，如果 `count[v]` 达到了 `n`，就说明这条路径必然包含了一个环。
    *   由于 `d[v]` 还在不断减小，这个环必然是**负环**。
    *   一旦发现 `count[v] >= n`，我们就可以立即判断图中存在负环，并终止算法。

**为什么是 `n` 次而不是 `n-1` 次？**
`count[u]` 记录的是 `d[u]` 被更新的次数。当 `d[u]` 第一次被更新时，`count[u]` 变为 1。当 `d[u]` 被第 `k` 次更新时，`count[u]` 变为 `k`。如果 `count[u]` 达到 `n`，意味着 `d[u]` 已经被更新了 `n` 次。这表示存在一条从源点到 `u` 的路径，其长度为 `n` 条边，并且这条路径仍然可以被松弛，从而使得 `d[u]` 变小。这只有在存在负环的情况下才会发生。

**处理图不连通的情况：**
SPFA 只能检测从源点可达的负环。如果图中存在一个负环，但它不在源点可达的范围内，SPFA 将无法检测到。为了检测图中**所有可能存在的负环**，我们通常会引入一个“超级源点”或“虚拟源点”`s`。
*   将这个虚拟源点 `s` 连接到图中所有其他节点 `i`，边权为 0 (`(s, i)`，权重 0)。
*   然后从这个虚拟源点 `s` 开始运行 SPFA。
*   这样，虚拟源点 `s` 就可以到达图中的所有节点，从而能够检测到任何一个可达的负环。通常虚拟源点会是节点 `0` 或节点 `n` (如果节点编号从 1 到 n-1)。

### 4. 关键点与常见误区

*   **`INF` 的设置**：`INF` 必须足够大，防止 `d[u] + w` 溢出，同时也要保证 `INF` 不会小于任何可能的路径和。通常设置为 `0x3f3f3f3f` 或 `LLONG_MAX / 2` (对于长整型)。
*   **`inQueue` 数组**：用于标记节点是否在队列中。这很重要，可以避免同一个节点被重复多次加入队列，从而提高效率。当节点从队列中取出时，要将 `inQueue` 标记设为 `false`。
*   **`count` 数组的清零**：对于多组测试数据，`d`, `count`, `inQueue` 数组以及图的邻接表都需要在每组测试前清零或重新初始化。
*   **虚拟源点**：检测图中任意负环时，务必使用虚拟源点连接所有节点。
*   **SPFA 的局限性**：虽然 SPFA 在平均情况下效率高，但在一些特定的“网格图”或“菊花图”上，其性能会退化到 O(VE)，甚至比 Bellman-Ford 还慢。因此，在边权非负时，Dijkstra 算法（配合优先队列）是更好的选择。

## 典型例题精讲

**题目：** 判断负环

给定一个包含 `n` 个节点和 `m` 条边的有向图。每条边都有一个权值。请判断图中是否存在负环。

**输入格式：**
第一行包含两个整数 `n` 和 `m`，表示节点数和边数。
接下来 `m` 行，每行包含三个整数 `u, v, w`，表示存在一条从节点 `u` 到节点 `v` 的有向边，权值为 `w`。节点编号从 `1` 到 `n`。

**输出格式：**
如果图中存在负环，输出 "YES"；否则输出 "NO"。

**示例 1：**
输入：
3 3
1 2 -1
2 3 -1
3 1 -1

输出：
YES

**思路分析 1：**
这是一个经典的负环检测问题。我们可以直接使用 SPFA 算法。
1.  **图的表示**：由于节点数 `n` 和边数 `m` 可能较大，使用邻接表（数组模拟链表）来存储图的边。
2.  **初始化**：
    *   `d[i]` 数组初始化为 `INF` (一个很大的数)，表示从源点到 `i` 的距离是无穷大。
    *   `count[i]` 数组初始化为 `0`，表示每个节点被松弛的次数。
    *   `inQueue[i]` 数组初始化为 `false`，表示节点不在队列中。
    *   使用一个虚拟源点 `0`，连接到所有节点 `1` 到 `n`，边权为 `0`。将 `d[0]` 设为 `0`，`count[0]` 设为 `0`，并将 `0` 加入队列。
3.  **SPFA 过程**：
    *   当队列不为空时，取出队头节点 `u`。
    *   将 `inQueue[u]` 设为 `false`。
    *   遍历 `u` 的所有邻接边 `(u, v)`，权值为 `w`。
    *   如果 `d[u] + w < d[v]`：
        *   更新 `d[v] = d[u] + w`。
        *   更新 `count[v] = count[u] + 1`。
        *   **负环判断**：如果 `count[v] >= n + 1` (因为我们加了一个虚拟源点0，所以总节点数是 `n+1`，如果从虚拟源点0开始，那路径长度达到 `n+1` 就说明有负环了，或者更简单的，如果节点编号是1到n，那总共是n个节点，如果count[v] >= n，就说明有负环)，则立即返回 `true` (存在负环)。
        *   如果 `v` 不在队列中，则将 `v` 加入队列，并设 `inQueue[v] = true`。
4.  **结果**：如果 SPFA 算法正常结束（队列为空，没有触发负环判断），则返回 `false` (不存在负环)。

**复杂度分析：**
*   **时间复杂度**：最坏情况下为 O(V * E)，其中 V 是节点数，E 是边数。在稀疏图上，平均表现会更好。
*   **空间复杂度**：O(V + E)，用于存储图、距离数组、计数数组和队列。

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <cstring> // For memset

// GESP 8 级，根据要求使用数组模拟邻接表和队列，禁止STL容器
// 变量命名使用a-z单字母，Allman风格大括号

const int MAXN = 2505; // 节点数最大2500，加一个虚拟源点0，所以MAXN需要大一点
const int MAXM = 6005; // 边数最大6000
const long long INF = 0x3f3f3f3f3f3f3f3fLL; // 使用long long防止溢出

struct Edge {
    int v; // 目标节点
    int w; // 权值
    int next; // 下一条边在edges数组中的索引
};

Edge e[MAXM]; // 存储所有边
int head[MAXN]; // head[u] 存储节点u发出的第一条边的索引
int edgeCount; // 当前边的数量

long long d[MAXN]; // d[i] 存储从源点到i的最短距离
int cnt[MAXN]; // cnt[i] 存储节点i被松弛的次数
bool inQ[MAXN]; // inQ[i] 标记节点i是否在队列中

int q[MAXN * 2]; // 模拟队列，队列大小最大为N，但为防止意外，可以稍大一点
int front, rear; // 队列头尾指针

int n, m; // 节点数和边数

// 添加边函数
void addEdge(int u, int v, int w) {
    e[edgeCount].v = v;
    e[edgeCount].w = w;
    e[edgeCount].next = head[u];
    head[u] = edgeCount++;
}

// SPFA判断负环函数
bool spfa() {
    // 初始化
    for (int i = 0; i <= n; ++i) { // 节点编号1到n，虚拟源点0
        d[i] = INF;
        cnt[i] = 0;
        inQ[i] = false;
    }

    front = 0;
    rear = 0;

    // 虚拟源点0，连接到所有1到n的节点，边权为0
    // d[0] = 0;
    // q[rear++] = 0;
    // inQ[0] = true;
    // cnt[0] = 0;

    // 为了检测图中所有负环，通常将所有节点都视为可能的“源点”
    // 或者更标准地，加入一个虚拟源点0，连接到所有节点1~n，边权为0
    // 然后从0开始跑SPFA。
    // 但是，如果直接把所有节点都加入队列，并设置d[i]=0，cnt[i]=0，效果是一样的。
    // 这里我们直接将所有节点1到n都加入队列，并设置d[i]=0，这样可以检测所有连通分量中的负环。
    // 当然，更严谨的做法是增加一个虚拟源点 N+1，连接到 1~N，边权为 0。
    // 为了符合题目节点1~n的编号，我们让所有节点1~n都作为“源点”开始。
    for (int i = 1; i <= n; ++i) {
        d[i] = 0; // 假设从每个点出发的初始距离为0
        q[rear++] = i;
        inQ[i] = true;
        cnt[i] = 0; // 每个点初始被松弛0次
    }


    while (front != rear) {
        int u = q[front++]; // 取出队头
        // 模拟循环队列
        if (front >= MAXN * 2) front = 0; 
        
        inQ[u] = false; // 标记u已出队

        // 遍历u的所有邻接边
        for (int i = head[u]; i != -1; i = e[i].next) {
            int v = e[i].v;
            int w = e[i].w;

            if (d[u] != INF && d[u] + w < d[v]) { // 松弛操作
                d[v] = d[u] + w;
                cnt[v] = cnt[u] + 1; // 经过u到达v，松弛次数+1

                if (cnt[v] >= n) { // 如果松弛次数达到n，说明存在负环
                    return true;
                }

                if (!inQ[v]) { // 如果v不在队列中，则加入队列
                    q[rear++] = v;
                    // 模拟循环队列
                    if (rear >= MAXN * 2) rear = 0;
                    inQ[v] = true;
                }
            }
        }
    }
    
    return false; // 队列为空，没有发现负环
}


int main() {
    // 优化输入输出
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m;

    // 初始化邻接表头
    memset(head, -1, sizeof(head));
    edgeCount = 0;

    for (int i = 0; i < m; ++i) {
        int u, v, w;
        std::cin >> u >> v >> w;
        addEdge(u, v, w);
    }

    if (spfa()) {
        std::cout << "YES\n";
    } else {
        std::cout << "NO\n";
    }

    return 0;
}
```

**代码解释：**
1.  **邻接表**：`Edge e[MAXM]` 存储所有边，`head[MAXN]` 存储每个节点的第一条边的索引，`addEdge` 函数用于添加边。这种方式避免了 `std::vector`。
2.  **SPFA 初始化**：`d` 数组初始化为 `INF`，`cnt` 数组初始化为 0。`inQ` 数组标记节点是否在队列中。
3.  **队列模拟**：`q` 数组和 `front`, `rear` 指针模拟队列。为了避免 `front` 或 `rear` 溢出（虽然对于SPFA，队列大小通常不会超过N，但为了通用性，简单处理了循环队列），这里用了 `MAXN * 2` 的大小，并进行了简单的循环处理。
4.  **虚拟源点处理**：为了检测图中所有负环，我们直接将所有节点 `1` 到 `n` 都初始化 `d[i]=0` 并加入队列。这等价于有一个虚拟源点，到所有点距离为0。
5.  **松弛与计数**：核心的 `d[u] + w < d[v]` 松弛操作，同时更新 `cnt[v] = cnt[u] + 1`。
6.  **负环判断**：`if (cnt[v] >= n)` 是判断负环的关键。如果某个节点被松弛的次数达到或超过 `n`，则说明存在负环。
7.  **`main` 函数**：读取输入，构建图，调用 `spfa` 函数并输出结果。

## 代码实现模板

```cpp
#include <iostream> // 输入输出
#include <cstring>  // 用于memset初始化数组

// GESP 8 级要求，禁止使用STL容器 (vector, queue, stack等)
// 变量命名使用a-z单字母，Allman风格大括号

const int MAX_NODES = 2505;   // 最大节点数 (N)，根据题目范围调整，留有余地
const int MAX_EDGES = 6005;   // 最大边数 (M)，根据题目范围调整

// 定义一个足够大的值表示无穷大，防止加法溢出，使用long long
const long long L_INF = 0x3f3f3f3f3f3f3f3fLL; 

// ====== 图的存储：使用数组模拟邻接表 ======
struct Edge {
    int to;       // 这条边的目标节点
    int weight;   // 这条边的权值
    int next;     // 指向从同一个起点出发的下一条边的索引
};

Edge e[MAX_EDGES];       // 存储所有边的数组
int head[MAX_NODES];     // head[u] 存储从节点u出发的第一条边的索引
int edgeCount;           // 当前已经添加的边的数量

// 添加一条从u到v，权值为w的有向边
void addEdge(int u, int v, int w) {
    e[edgeCount].to = v;
    e[edgeCount].weight = w;
    e[edgeCount].next = head[u]; // 将新边插入到u的邻接链表头部
    head[u] = edgeCount;         // 更新u的头部指针
    edgeCount++;                 // 边计数器加一
}

// ====== SPFA 算法所需数组 ======
long long d[MAX_NODES];  // d[i] 存储从源点到节点i的最短距离
int c[MAX_NODES];        // c[i] 存储节点i被松弛（距离更新）的次数
bool inQ[MAX_NODES];     // inQ[i] 标记节点i是否在队列中

// ====== 队列的模拟 ======
int q[MAX_NODES * 2];    // 模拟队列的数组，队列的最大元素个数为N，这里给大一点以防万一
int front, rear;         // 队列的头指针和尾指针

// SPFA 算法主函数，判断图中是否存在负环
// n: 节点总数
// 返回值: true 表示存在负环，false 表示不存在
bool spfa_detect_negative_cycle(int n_nodes) {
    // 1. 初始化所有数组
    for (int i = 0; i <= n_nodes; ++i) { // 节点编号从1到n_nodes，如果有虚拟源点0，也需要初始化
        d[i] = L_INF;    // 所有距离初始化为无穷大
        c[i] = 0;        // 所有节点的松弛次数初始化为0
        inQ[i] = false;  // 所有节点都不在队列中
    }

    // 2. 初始化队列和源点
    front = 0; // 队列头指针
    rear = 0;  // 队列尾指针

    // 为了检测图中所有连通分量中的负环，通常将所有节点都加入队列作为“可能的起点”
    // 并将它们的距离设为0，松弛次数设为0。
    // 这等价于添加一个虚拟源点，连接到所有实际节点，边权为0。
    for (int i = 1; i <= n_nodes; ++i) { // 假设节点编号从1到n_nodes
        d[i] = 0;       // 初始距离设为0 (因为我们想检查所有可能的环，所以从每个点开始都设为0)
        q[rear++] = i;  // 将节点i加入队列
        inQ[i] = true;  // 标记节点i在队列中
        // c[i] 保持为0，因为这是第一次“松弛” (从0到0)
    }

    // 3. SPFA 核心循环
    while (front != rear) {
        int u = q[front++]; // 从队列头部取出一个节点u
        // 简单的循环队列处理，防止front/rear超出数组范围
        // 对于SPFA，通常队列大小不会超过N，所以不严格的循环处理也可能够用
        if (front >= MAX_NODES * 2) front = 0; 

        inQ[u] = false; // 节点u已出队，标记为不在队列中

        // 遍历节点u的所有出边
        for (int i = head[u]; i != -1; i = e[i].next) {
            int v = e[i].to;      // 目标节点v
            int w = e[i].weight;  // 边权w

            // 进行松弛操作：如果通过u到达v的路径更短
            // d[u] != L_INF 确保u是可达的，避免INF+w的溢出问题
            if (d[u] != L_INF && d[u] + w < d[v]) {
                d[v] = d[u] + w;          // 更新d[v]
                c[v] = c[u] + 1;          // 增加v的松弛次数

                // 负环检测：如果某个节点被松弛的次数达到或超过n_nodes
                // 说明图中存在负环
                if (c[v] >= n_nodes) {
                    return true; // 发现负环，立即返回true
                }

                // 如果v不在队列中，则将其加入队列，以便后续对其邻接点进行松弛
                if (!inQ[v]) {
                    q[rear++] = v; // 将v加入队列
                    // 简单的循环队列处理
                    if (rear >= MAX_NODES * 2) rear = 0;
                    inQ[v] = true; // 标记v在队列中
                }
            }
        }
    }

    // 4. 队列为空，且没有检测到负环，说明图中不存在负环
    return false;
}

int main() {
    // 优化输入输出，在竞赛编程中常用
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);

    int n_nodes, m_edges; // 节点数和边数
    std::cin >> n_nodes >> m_edges;

    // 初始化邻接表头，所有head[i]都设为-1表示没有边
    memset(head, -1, sizeof(head));
    edgeCount = 0; // 边的计数器清零

    // 读取m_edges条边并添加到图中
    for (int i = 0; i < m_edges; ++i) {
        int u, v, w;
        std::cin >> u >> v >> w;
        addEdge(u, v, w);
    }

    // 调用SPFA函数判断是否存在负环
    if (spfa_detect_negative_cycle(n_nodes)) {
        std::cout << "YES\n"; // 存在负环
    } else {
        std::cout << "NO\n";  // 不存在负环
    }

    return 0; // 程序正常结束
}
```

## 分层练习题目答案

**基础巩固：**

1.  **题目名称：** 简单负环检测
    *   **输出：** YES
    *   **解释：** 路径 1 -> 2 -> 3 -> 1，总权值 -3 + -2 + 4 = -1，是一个负环。

2.  **题目名称：** 无负环图
    *   **输出：** NO
    *   **解释：** 尽管存在负权边 (2->4, -1) 和 (5->2, -2)，但不存在一个总和为负数的环。

**能力提升：**

1.  **题目名称：** 货币兑换
    *   **思路：** 将兑换问题转化为最短路问题。
        *   如果从货币 `A` 兑换到货币 `B`，汇率为 `R`，手续费为 `C`，那么 1 单位 `A` 变为 `(1-C) * R` 单位 `B`。
        *   我们希望找到一个环，使得最终货币数量 `P_final / P_initial > 1`。
        *   取对数：`log(P_final / P_initial) > 0`。
        *   设 `d[i]` 为从源货币到 `i` 货币的 `log` 值。
        *   边权 `w(u, v)` 设为 `-log((1-C) * R)`。
        *   那么 `d[v] = d[u] + w(u, v)`。我们希望找到一个环路，其边的权值和为负数。
        *   如果 `sum(w)` 为负数，则 `sum(-log((1-C)*R))` 为负数，即 `log(product((1-C)*R))` 为正数，即 `product((1-C)*R) > 1`。
        *   因此，问题转化为检测图中是否存在负环。
    *   **输出：** 根据具体输入而定。

2.  **题目名称：** 限制步数的负环
    *   **思路：** 在 SPFA 的 `count[v]` 更新时，除了判断 `c[v] >= n_nodes`，还需要同时判断 `c[v] <= K`。如果 `c[v] >= n_nodes` 且 `c[v] <= K`，则说明存在满足条件的负环。

**拓展挑战：**

1.  **题目名称：** 负环路径输出
    *   **思路：**
        1.  在 SPFA 中，维护一个 `parent[v]` 数组，记录 `d[v]` 是由哪个节点 `u` 松弛而来的。即 `parent[v] = u`。
        2.  当检测到 `c[v] >= n_nodes` 时，说明 `v` 处于一个负环中。
        3.  为了找到环，从 `v` 开始，沿着 `parent` 指针回溯，直到找到一个已经访问过的节点。
        4.  为了避免在回溯时陷入无限循环（因为 `parent` 链可能很长），可以先沿着 `parent` 指针回溯 `n_nodes` 次，确保我们进入了环。
        5.  从这个节点开始，再次沿着 `parent` 指针回溯，并将经过的节点存入一个列表，直到再次回到起点。逆序输出列表中的节点即可得到负环路径。
