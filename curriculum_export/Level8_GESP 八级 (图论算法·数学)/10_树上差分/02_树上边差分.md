## 趣味引入

同学们，想象一下，我们生活在一个由N个城市组成的国家，城市之间由N-1条道路连接，形成一个巨大的“交通网络”，而且这个网络没有环路，就像一棵大树一样！每条道路都有它自己的“重要性”或者“交通流量”。

有一天，国王发布了一道命令：
**任务一：** 有M队探险家要沿着不同的路径从一个城市走到另一个城市。国王想知道，在所有探险家走完之后，**每一条道路总共被经过了多少次？** 哪些路最繁忙，哪些路最冷清？

**任务二：** 还有一些特别任务，比如，每次对某条路径上的所有道路进行“维护”，让它们的“重要性”增加一个值 K。经过一系列维护后，国王又想知道，**某条特定道路的最终“重要性”是多少？**

如果每次有探险家走过一条路，我们就把这条路的计数器加1；如果每次维护一条路径，我们就真的把路径上的每一条路都遍历一遍加K。这样效率是不是太低了？尤其当探险家队伍和维护任务特别多，或者路径特别长的时候，我们岂不是要累死啦？

有没有一种更“聪明”的办法，可以快速地完成这些统计和修改呢？就像我们平时做数学题，遇到一堆连续的数字相加，我们不会一个一个加，而是会找规律，用公式计算。在编程的世界里，也有这样的“魔法”，它能把对“一整条路径”的操作，巧妙地转化成对“路径两端点”的几个简单操作，然后通过一次遍历就能得到所有结果！

这个“魔法”就叫做——**树上边差分**！今天，我们就来揭开它的神秘面纱！

## 深度知识点讲解

### 1. 差分思想的复习与升华

还记得我们学过的一维数组差分吗？
如果有一个数组 `a = [a_1, a_2, ..., a_n]`。
它的差分数组 `d = [d_1, d_2, ..., d_n]` 定义为 `d_i = a_i - a_{i-1}` (设 `a_0 = 0`)。
那么 `a_i = d_1 + d_2 + ... + d_i` (前缀和)。

**核心思想：**
*   **区间修改变单点修改**：对 `a[L...R]` 加上 `K`，我们只需要 `d[L] += K` 和 `d[R+1] -= K`。
*   **单点查询变区间查询**：查询 `a[i]` 只需要计算 `d[1...i]` 的和。
*   **区间查询变单点查询**：如果想查询 `a[L...R]` 的和，我们通常需要 `a` 数组的二次前缀和。

树上差分就是把这种“化繁为简”的思想，从线性的数组推广到复杂的树结构上。

### 2. 为什么需要树上差分？

在树上，一条路径可能很长，包含很多条边。如果每次对一条路径上的边进行操作（比如修改边权、统计经过次数），都老老实实地遍历一遍路径上的所有边，那么效率会非常低下。特别是在数据规模 `N` 和操作次数 `M` 都很大的时候，会直接超时。

树上差分的目标是：将对**树上路径**的复杂操作，转化为对**路径端点以及它们的LCA（最近公共祖先）**的几个简单操作，然后通过一次深度优先搜索（DFS）就能高效地完成最终的统计或修改。

### 3. 树上边差分的核心原理

我们来思考，一条从节点 `u` 到节点 `v` 的路径，我们可以把它拆分成两部分：从 `u` 到 `LCA(u,v)` 的路径，和从 `v` 到 `LCA(u,v)` 的路径。这两条路径在 `LCA(u,v)` 处汇合。

假设我们要对路径 `(u, v)` 上的所有边进行一个操作，比如让它们的权值都增加 `K`。我们如何用“差分”的思想来标记呢？

我们引入一个 `diff` 数组，`diff[x]` 记录的是在节点 `x` 处对边权做出的“标记”。这个标记的含义是：**从 `x` 到其父节点 `fa[x]` 这条边的权值，会受到 `diff[x]` 的影响。**

**核心操作步骤：**

1.  **预处理**：首先，我们需要对树进行一次DFS，计算每个节点的深度 `dep[x]`、父节点 `fa[x][0]`（直接父节点），以及用于LCA查询的倍增数组 `fa[x][k]`（`x` 的 `2^k` 次祖先）。
2.  **LCA查询**：对于每一条路径 `(u, v)`，我们需要快速找到它们的最近公共祖先 `l = LCA(u, v)`。
3.  **差分标记**：现在，我们想对路径 `(u, v)` 上的所有边都进行 `+K` 操作。
    *   我们在 `u` 处标记 `diff[u] += K`。
    *   我们在 `v` 处标记 `diff[v] += K`。
    *   思考一下：从 `u` 到根节点的路径上的所有边，以及从 `v` 到根节点的路径上的所有边，都会受到这个 `+K` 的影响。
    *   但是，路径 `(u, v)` 上的边，其实只包括 `u` 到 `l` 的路径和 `v` 到 `l` 的路径。从 `l` 往上到根节点的那些边，不属于 `(u, v)` 路径。
    *   因为 `diff[u]` 和 `diff[v]` 会沿着各自的祖先链向上累加，导致 `LCA(u,v)` 到根节点的路径上的边被多加了两次 `K`。而实际上，这些边根本不属于 `(u,v)` 路径。
    *   更准确地说， `u` 到 `l` 的路径上的边被加了 `K`，`v` 到 `l` 的路径上的边被加了 `K`。公共部分的边是 `l` 到 `fa[l]` 这条边（如果 `l` 不是根）。这条边被加了两次，但它不属于 `u` 到 `v` 路径。
    *   所以，我们需要在 `LCA(u, v)` 处进行抵消，`diff[l] -= K`。
    *   同时，`LCA(u,v)` 的父节点 `fa[LCA(u,v)]` 处的边，也被多加了两次，所以也要抵消。`diff[fa[l]] -= K`。

    **总结一下对路径 `(u, v)` 上的所有边进行 `+K` 操作的标记方式：**
    ```cpp
    int l = lca(u, v); // 找到LCA
    diff[u] += k;
    diff[v] += k;
    diff[l] -= k;
    // 如果l不是根节点，它的父节点也需要抵消
    if (l != 1) { // 假设根节点是1
        diff[fa[l][0]] -= k;
    }
    ```
    这里的 `fa[l][0]` 是 `l` 的直接父节点。

4.  **结果还原（DFS累加）**：
    *   完成所有路径的差分标记后，我们需要进行第二次DFS来计算每条边的最终值。
    *   我们从根节点开始DFS。对于当前节点 `u`，先递归处理它的所有子节点 `v`。
    *   当 `dfs_calc(v, u)` 返回后，`diff[u]` 就要加上 `diff[v]`。
    *   `dfs_calc(u, p)` 函数：
        *   对于 `u` 的每个子节点 `v` (且 `v != p`):
            *   `dfs_calc(v, u)` // 递归处理子节点
            *   `diff[u] += diff[v]` // 将子节点的差分值累加到父节点
        *   此时，`diff[u]` 的最终值，就是**以 `u` 为下端点，`fa[u]` 为上端点的这条边 `(u, fa[u])` 被操作的总次数或最终权值**。
        *   注意：根节点到其父节点（不存在）的边，其 `diff` 值通常不用于表示真实边权，或者设为0。

    **为什么这样累加？**
    考虑一个节点 `x`，它对 `diff[x]` 做了 `+K` 标记。这个 `+K` 意味着从 `x` 向上到根节点的路径上的所有边都会受到影响。而我们第二次DFS从下往上累加时，`diff[u] += diff[v]` 实际上是在说：子树 `v` 中所有节点对 `diff[v]` 的标记，都会向上“传递”到 `u`，并最终影响到边 `(u, fa[u])`。只有当一条路径经过了边 `(u, fa[u])`，并且 `u` 是这条路径上更靠近叶子的那个节点时，`diff[u]` 才会包含这个路径的贡献。通过巧妙的加减抵消，最终 `diff[u]` 恰好表示了边 `(u, fa[u])` 的实际值。

### 4. LCA（最近公共祖先）的实现

LCA是树上差分的基础工具，通常使用倍增法实现。

*   **预处理 (`dfs_lca`)**:
    *   `dep[u]`：节点 `u` 的深度。
    *   `fa[u][k]`：节点 `u` 的 `2^k` 次祖先。
    *   DFS遍历树，计算 `dep[u]` 和 `fa[u][0]`。
    *   然后通过动态规划计算 `fa[u][k] = fa[fa[u][k-1]][k-1]`。
*   **查询 (`lca`)**:
    *   先将两个节点 `u, v` 调整到同一深度。
    *   如果 `u == v`，则 `u` 就是LCA。
    *   从最大的 `k` 开始，向上跳，直到 `fa[u][k] == fa[v][k]`。
    *   此时 `fa[u][0]` 就是LCA。

### 5. 关键点与常见误区

*   **根节点处理**：根节点（通常是1号节点）没有父节点。在 `diff[fa[l][0]] -= k` 这步，如果 `l` 就是根节点，`fa[l][0]` 不存在，需要特殊处理（例如，判断 `l != 1` 再进行操作）。
*   **边的表示**：在树上差分中，我们通常用节点 `u` 来代表它到父节点 `fa[u]` 的这条边。所以，`ans[u]` 存储的就是边 `(u, fa[u])` 的最终值。根节点的 `ans[1]` 通常没有实际意义。
*   **数组大小**：注意 `MAXN` 和 `MAXLOG` 的设置，防止数组越界。
*   **初始化**：`diff` 数组需要初始化为0。邻接表需要清空。

理解了这些，我们就可以开始用树上边差分解决实际问题了！

## 典型例题精讲

### 例题1：统计每条边被经过的次数

**题目描述**
给定一棵N个节点的树，节点编号从1到N。接下来有M条查询，每条查询给定两个节点 `u` 和 `v`，表示一条从 `u` 到 `v` 的路径。请统计树中每条边被这M条路径覆盖了多少次。

**输入格式**
第一行两个整数 `N, M`。
接下来 `N-1` 行，每行两个整数 `x, y`，表示 `x` 和 `y` 之间有一条边。
接下来 `M` 行，每行两个整数 `u, v`，表示一条查询路径。

**输出格式**
输出 `N-1` 行，每行一个整数，表示对应边被覆盖的次数。输出顺序为：按输入的边的顺序。

**样例输入**
```
5 3
1 2
1 3
2 4
2 5
4 5
1 3
2 3
```

**思路分析**

1.  **存储边信息**：由于输出要求按输入的边顺序，我们可以在读入边时记录下每条边对应的两个端点，以便后续输出。但更简洁的做法是，最终 `diff[u]` 存储的是边 `(u, fa[u])` 的答案。所以我们只需要输出 `diff[2]` 到 `diff[N]` 的值即可（假设1是根节点）。
2.  **LCA预处理**：首先，我们需要进行一次DFS来构建树的结构（邻接表），并计算每个节点的深度 `dep` 和直接父节点 `fa[u][0]`。然后利用倍增法预处理 `fa[u][k]` 数组，用于快速查询LCA。
3.  **差分标记**：对于每条查询路径 `(u, v)`，我们找到它们的LCA `l = LCA(u, v)`。
    *   在 `diff[u]` 上加1。
    *   在 `diff[v]` 上加1。
    *   在 `diff[l]` 上减1。
    *   如果 `l` 不是根节点（假设根节点是1），在 `diff[fa[l][0]]` 上减1。
4.  **DFS累加**：完成所有 `M` 条路径的差分标记后，进行第二次DFS。这个DFS从叶子节点向上回溯，将子节点的 `diff` 值累加到父节点。
    *   `dfs_calc(u, p)` 函数：
        *   对于 `u` 的每个子节点 `v` (且 `v != p`):
            *   `dfs_calc(v, u)`
            *   `diff[u] += diff[v]`
        *   此时 `diff[u]` 就表示边 `(u, fa[u])` 被经过的次数。
5.  **输出结果**：遍历节点 `2` 到 `N`，输出 `diff[i]` 即可。

**代码实现**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005; // 最大节点数
const int MAXLOG = 18;   // log2(MAXN) 向上取整

// 邻接表存储树
int h[MAXN], nxt[MAXN * 2], to[MAXN * 2], idx;

// LCA相关数组
int dep[MAXN];       // 节点深度
int fa[MAXN][MAXLOG]; // fa[u][k] 表示u的2^k次祖先

// 差分数组
int d[MAXN]; // d[u] 存储节点u的差分标记，最终表示边(u, fa[u])的权值

int n, m;

void add(int a, int b) {
    to[idx] = b;
    nxt[idx] = h[a];
    h[a] = idx++;
}

// 第一次DFS，预处理深度和LCA的fa[u][0]
void dfs_lca_pre(int u, int p) {
    dep[u] = dep[p] + 1;
    fa[u][0] = p;

    for (int i = 1; i < MAXLOG; ++i) {
        fa[u][i] = fa[fa[u][i - 1]][i - 1];
    }

    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue;
        dfs_lca_pre(v, u);
    }
}

// 查询LCA
int lca(int u, int v) {
    if (dep[u] < dep[v]) std::swap(u, v); // 保证u更深

    // 将u跳到与v相同深度
    for (int i = MAXLOG - 1; i >= 0; --i) {
        if (dep[fa[u][i]] >= dep[v]) {
            u = fa[u][i];
        }
    }

    if (u == v) return u;

    // 同时向上跳，直到父节点相同
    for (int i = MAXLOG - 1; i >= 0; --i) {
        if (fa[u][i] != fa[v][i]) {
            u = fa[u][i];
            v = fa[v][i];
        }
    }
    return fa[u][0]; // 此时u和v的父节点就是LCA
}

// 第二次DFS，累加差分值
void dfs_calc_diff(int u, int p) {
    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue;
        dfs_calc_diff(v, u);
        d[u] += d[v]; // 将子节点的差分值累加到父节点
    }
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m;

    std::memset(h, -1, sizeof(h)); // 初始化邻接表头

    for (int i = 0; i < n - 1; ++i) {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        add(v, u);
    }

    // 预处理LCA相关信息，根节点为1，其父节点为0
    dfs_lca_pre(1, 0);

    for (int i = 0; i < m; ++i) {
        int u, v;
        std::cin >> u >> v;
        int l = lca(u, v);
        d[u]++;
        d[v]++;
        d[l]--;
        if (fa[l][0] != 0) { // 如果LCA不是根节点，则其父节点也需要减去
            d[fa[l][0]]--;
        }
    }

    // 累加差分值
    // 注意：这里的d[u]最终表示的是边(u, fa[u])被经过的次数
    // 根节点1到其父节点0的边不存在，所以d[1]的值没有实际意义，
    // 或者说d[1]累加的是所有子节点向上汇总的差分值，它代表的是根节点到其“虚拟父节点”的边。
    // 我们只需要计算并输出d[2]到d[N]的值。
    // 但是，为了确保dfs_calc_diff正确计算，它需要从根节点开始遍历所有子树。
    // 最终d[u]是其子树所有路径经过(u,fa[u])的贡献。
    dfs_calc_diff(1, 0); 

    // 输出每条边的覆盖次数
    // 注意：题目要求按输入的边顺序输出，
    // 但是我们现在d[u]代表的是(u, fa[u])这条边。
    // 如果题目没有特殊要求，通常输出d[2]...d[N]即可。
    // 若要严格按输入顺序，则需要在读入边时保存下来，然后根据保存的信息输出。
    // 这里我们直接输出d[i] for i from 2 to N，表示边(i, fa[i][0])的覆盖次数。
    // 实际竞赛中，通常是问某条边被经过的次数，或者问所有边的最大值/最小值。
    // 这里为了简化输出，我们假设输出d[i] for i=2..N，表示i与其父节点之间的边。
    // 实际题目中会更明确。
    for (int i = 2; i <= n; ++i) {
        std::cout << d[i] << "\n";
    }

    return 0;
}
```

**复杂度分析**
*   **LCA预处理**：`dfs_lca_pre` 遍历所有节点一次 `O(N)`，倍增数组填充 `N * log N` 次，总计 `O(N log N)`。
*   **差分标记**：`M` 条路径，每条路径查询LCA `O(log N)`，差分标记 `O(1)`。总计 `O(M log N)`。
*   **DFS累加**：`dfs_calc_diff` 遍历所有节点一次 `O(N)`。
*   **总时间复杂度**：`O(N log N + M log N)`。
*   **空间复杂度**：`O(N log N)` (主要用于LCA的倍增数组)。

### 例题2：路径边权修改，查询单条边权值 (简化版)

**题目描述**
给定一棵N个节点的树，初始所有边的权值为0。有M次操作，每次操作给定一条路径 `(u, v)` 和一个整数 `k`，表示将这条路径上的所有边的权值增加 `k`。在所有操作完成后，有 `Q` 次查询，每次查询给定一个节点 `x`，要求输出边 `(x, fa[x])` 的最终权值。（假设1为根节点，查询不会是节点1）

**输入格式**
第一行三个整数 `N, M, Q`。
接下来 `N-1` 行，每行两个整数 `x, y`，表示 `x` 和 `y` 之间有一条边。
接下来 `M` 行，每行三个整数 `u, v, k`，表示一条修改操作。
接下来 `Q` 行，每行一个整数 `x`，表示一个查询操作。

**输出格式**
对于每个查询，输出一行一个整数，表示边 `(x, fa[x])` 的最终权值。

**样例输入**
```
5 2 2
1 2
1 3
2 4
2 5
4 5 10
1 3 5
4
2
```

**思路分析**

这个问题的核心与例题1非常相似，只是将“被经过的次数”变成了“边权增加值 `k`”。所有修改都是在所有查询之前完成的，这是一种“离线处理”的模式。

1.  **LCA预处理**：同例题1，进行DFS构建树、计算深度、预处理LCA倍增数组。
2.  **差分标记**：对于每条修改操作 `(u, v, k)`，找到它们的LCA `l = LCA(u, v)`。
    *   在 `diff[u]` 上加 `k`。
    *   在 `diff[v]` 上加 `k`。
    *   在 `diff[l]` 上减 `k`。
    *   如果 `l` 不是根节点（假设根节点是1），在 `diff[fa[l][0]]` 上减 `k`。
3.  **DFS累加**：完成所有 `M` 次修改的差分标记后，进行第二次DFS。这个DFS从叶子节点向上回溯，将子节点的 `diff` 值累加到父节点。
    *   `dfs_calc_diff(u, p)` 函数：
        *   对于 `u` 的每个子节点 `v` (且 `v != p`):
            *   `dfs_calc_diff(v, u)`
            *   `diff[u] += diff[v]`
        *   此时 `diff[u]` 就表示边 `(u, fa[u])` 的最终权值。
4.  **查询结果**：对于每个查询 `x`，直接输出 `diff[x]` 即可。

**代码实现**

```cpp
#include <bits/stdc++.h>

const int MAXN = 100005; // 最大节点数
const int MAXLOG = 18;   // log2(MAXN) 向上取整

// 邻接表存储树
int h[MAXN], nxt[MAXN * 2], to[MAXN * 2], idx;

// LCA相关数组
int dep[MAXN];       // 节点深度
int fa[MAXN][MAXLOG]; // fa[u][k] 表示u的2^k次祖先

// 差分数组
long long d[MAXN]; // d[u] 存储节点u的差分标记，最终表示边(u, fa[u])的权值，可能很大用long long

int n, m, q;

void add(int a, int b) {
    to[idx] = b;
    nxt[idx] = h[a];
    h[a] = idx++;
}

// 第一次DFS，预处理深度和LCA的fa[u][0]
void dfs_lca_pre(int u, int p) {
    dep[u] = dep[p] + 1;
    fa[u][0] = p;

    for (int i = 1; i < MAXLOG; ++i) {
        fa[u][i] = fa[fa[u][i - 1]][i - 1];
    }

    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue;
        dfs_lca_pre(v, u);
    }
}

// 查询LCA
int lca(int u, int v) {
    if (dep[u] < dep[v]) std::swap(u, v); // 保证u更深

    // 将u跳到与v相同深度
    for (int i = MAXLOG - 1; i >= 0; --i) {
        if (dep[fa[u][i]] >= dep[v]) {
            u = fa[u][i];
        }
    }

    if (u == v) return u;

    // 同时向上跳，直到父节点相同
    for (int i = MAXLOG - 1; i >= 0; --i) {
        if (fa[u][i] != fa[v][i]) {
            u = fa[u][i];
            v = fa[v][i];
        }
    }
    return fa[u][0]; // 此时u和v的父节点就是LCA
}

// 第二次DFS，累加差分值
void dfs_calc_diff(int u, int p) {
    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue;
        dfs_calc_diff(v, u);
        d[u] += d[v]; // 将子节点的差分值累加到父节点
    }
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(NULL);

    std::cin >> n >> m >> q;

    std::memset(h, -1, sizeof(h)); // 初始化邻接表头

    for (int i = 0; i < n - 1; ++i) {
        int u, v;
        std::cin >> u >> v;
        add(u, v);
        add(v, u);
    }

    // 预处理LCA相关信息，根节点为1，其父节点为0
    dfs_lca_pre(1, 0);

    for (int i = 0; i < m; ++i) {
        int u, v, k;
        std::cin >> u >> v >> k;
        int l = lca(u, v);
        d[u] += k;
        d[v] += k;
        d[l] -= k;
        if (fa[l][0] != 0) { // 如果LCA不是根节点，则其父节点也需要减去
            d[fa[l][0]] -= k;
        }
    }

    // 累加差分值
    dfs_calc_diff(1, 0); 

    for (int i = 0; i < q; ++i) {
        int x;
        std::cin >> x;
        // d[x] 存储的就是边(x, fa[x][0])的最终权值
        std::cout << d[x] << "\n";
    }

    return 0;
}
```

**复杂度分析**
*   **LCA预处理**：`O(N log N)`。
*   **差分标记**：`M` 次操作，每次 `O(log N)`。总计 `O(M log N)`。
*   **DFS累加**：`O(N)`。
*   **查询**：`Q` 次查询，每次 `O(1)`。总计 `O(Q)`。
*   **总时间复杂度**：`O(N log N + M log N + Q)`。
*   **空间复杂度**：`O(N log N)`。

## 代码实现模板

下面是一个通用的树上边差分框架，包含了LCA的预处理和查询，以及差分标记和累加。

```cpp
#include <bits/stdc++.h>

// 常量定义
const int MAXN = 100005; // 最大节点数，根据题目N的范围调整
const int MAXLOG = 18;   // log2(MAXN) 向上取整，例如10^5大约是2^17，所以18足够

// 邻接表表示树
int h[MAXN], nxt[MAXN * 2], to[MAXN * 2], idx; // h: 头节点数组，nxt: 下一个边，to: 目标节点

// LCA相关数组
int dep[MAXN];       // dep[u]: 节点u的深度 (根节点深度为1，其子节点深度为2)
int fa[MAXN][MAXLOG]; // fa[u][k]: 节点u的2^k次祖先

// 差分数组
long long d[MAXN]; // d[u]: 节点u的差分标记，最终用于表示边(u, fa[u])的权值/计数

int n; // 节点数量

// 邻接表添加边函数
void add(int a, int b) {
    to[idx] = b;
    nxt[idx] = h[a];
    h[a] = idx++;
}

// 第一次DFS：预处理LCA相关信息 (深度、直接父节点、倍增祖先)
// u: 当前节点，p: u的父节点
void dfs_lca_pre(int u, int p) {
    dep[u] = dep[p] + 1; // 深度比父节点多1
    fa[u][0] = p;        // 直接父节点

    // 倍增数组预处理：fa[u][i] = fa[fa[u][i-1]][i-1]
    // 意思是u的2^i次祖先等于u的2^(i-1)次祖先的2^(i-1)次祖先
    for (int i = 1; i < MAXLOG; ++i) {
        fa[u][i] = fa[fa[u][i - 1]][i - 1];
    }

    // 遍历u的所有邻居
    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue; // 避免访问父节点
        dfs_lca_pre(v, u);    // 递归处理子节点
    }
}

// 查询两个节点的最近公共祖先 (LCA)
// u, v: 待查询LCA的两个节点
int lca(int u, int v) {
    // 保证u的深度大于等于v，方便后续统一操作
    if (dep[u] < dep[v]) std::swap(u, v);

    // 将u向上跳，直到与v处于同一深度
    for (int i = MAXLOG - 1; i >= 0; --i) {
        // 如果u的2^i次祖先的深度大于等于v的深度，则u向上跳
        if (dep[fa[u][i]] >= dep[v]) {
            u = fa[u][i];
        }
    }

    // 如果此时u和v相同，说明v就是LCA
    if (u == v) return u;

    // u和v同时向上跳，直到它们的父节点相同（即LCA的直接子节点）
    for (int i = MAXLOG - 1; i >= 0; --i) {
        // 如果u和v的2^i次祖先不同，则u和v都向上跳
        if (fa[u][i] != fa[v][i]) {
            u = fa[u][i];
            v = fa[v][i];
        }
    }
    // 此时u和v的父节点就是LCA
    return fa[u][0];
}

// 第二次DFS：从叶子节点向上累加差分值
// u: 当前节点，p: u的父节点
void dfs_calc_diff(int u, int p) {
    // 遍历u的所有邻居
    for (int i = h[u]; ~i; i = nxt[i]) {
        int v = to[i];
        if (v == p) continue; // 避免访问父节点
        dfs_calc_diff(v, u);    // 递归处理子节点
        d[u] += d[v];         // 将子节点的差分值累加到当前节点
    }
    // 当dfs_calc_diff(u, p)返回时，d[u]就包含了其子树所有节点对其的贡献，
    // 此时d[u]的值就代表了边(u, fa[u])的最终权值或计数。
    // 注意：d[1] (根节点) 的值通常不代表任何边，因为它没有父节点。
}
