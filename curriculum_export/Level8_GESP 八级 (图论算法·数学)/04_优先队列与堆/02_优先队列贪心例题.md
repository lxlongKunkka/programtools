## 深度知识点讲解

### 1. 什么是优先队列？

**本质：** 优先队列是一种抽象数据类型，它允许我们随时插入元素，并随时取出当前优先级最高的元素。它不像普通队列那样遵循“先进先出”（FIFO）原则，也不像栈那样遵循“后进先出”（LIFO）原则，而是遵循“优先级最高者优先出队”的原则。

**生活类比：**
*   **医院急诊室：** 病人进入急诊室（入队），但医生总是先看最危急的病人（优先级最高的出队）。
*   **机场VIP通道：** 旅客进入通道（入队），但VIP旅客可以优先安检（优先级最高的出队）。

**核心操作：**
*   `push(element)`：将一个元素加入队列。
*   `top()`：查看队列中优先级最高的元素（不删除）。
*   `pop()`：删除并返回队列中优先级最高的元素。
*   `empty()`：判断队列是否为空。
*   `size()`：返回队列中元素的数量。

### 2. 底层实现：堆（Heap）

优先队列通常不是凭空变出来的，它的背后有一个强大的“管家”——**堆（Heap）**。堆是一种特殊的完全二叉树，它保证了父节点和子节点之间存在特定的顺序关系。

**堆的分类：**
*   **大顶堆（Max-Heap）：** 任何一个父节点的值都大于或等于其所有子节点的值。这意味着，根节点（也就是堆顶）一定是整个堆中最大的元素。
*   **小顶堆（Min-Heap）：** 任何一个父节点的值都小于或等于其所有子节点的值。这意味着，根节点（也就是堆顶）一定是整个堆中最小的元素。

**堆的性质：**
1.  **完全二叉树：** 树的每一层（除最后一层外）都被完全填充，并且最后一层的所有节点都尽可能地靠左。这个性质使得堆可以用数组来高效存储。
2.  **堆序性：** 满足大顶堆或小顶堆的性质。

**堆的数组表示：**
由于堆是完全二叉树，我们可以用一个数组来存储它，而不需要显式地存储指针。
*   根节点存储在 `a[1]` (或 `a[0]`)。
*   对于节点 `a[i]`：
    *   它的左孩子是 `a[2*i]`。
    *   它的右孩子是 `a[2*i + 1]`。
    *   它的父节点是 `a[i / 2]`。
这种表示方法非常简洁高效。

**插入与删除操作的原理：**
*   **插入 (push)：** 将新元素放到数组末尾（也就是堆的最后一个叶子节点），然后通过“上浮”（Heapify-up 或 Sift-up）操作，不断与父节点比较并交换，直到满足堆序性。
*   **删除 (pop)：** 通常是删除堆顶元素（最大或最小）。我们将堆顶元素与数组末尾元素交换，然后删除末尾元素，再通过“下沉”（Heapify-down 或 Sift-down）操作，不断与子节点比较并交换，直到满足堆序性。

**时间复杂度：**
无论是插入还是删除操作，因为每次操作都只涉及一条从根到叶子（或叶子到根）的路径，所以它们的时间复杂度都是 `O(log n)`，其中 `n` 是堆中元素的数量。查看堆顶元素是 `O(1)`。

### 3. C++ 中的 `std::priority_queue`

C++ 标准库为我们提供了 `std::priority_queue`，它默认实现了一个大顶堆。

**语法：**
```cpp
#include <queue> // 包含头文件

// 默认是大顶堆 (Max-Heap)
// 存储整数，优先级高的（值大的）先出队
std::priority_queue<int> q;

// 创建小顶堆 (Min-Heap)
// 存储整数，优先级高的（值小的）先出队
// 需要指定底层容器 (std::vector<int>) 和比较器 (std::greater<int>)
std::priority_queue<int, std::vector<int>, std::greater<int>> min_q;

// 常用操作
q.push(10);    // 插入元素10
q.push(5);     // 插入元素5
q.push(20);    // 插入元素20

int t = q.top(); // 查看队首元素 (大顶堆是20, 小顶堆是5)
q.pop();         // 删除队首元素
bool e = q.empty(); // 判断是否为空
int s = q.size();   // 获取元素数量
```

**注意事项：**
*   `std::priority_queue` 默认是一个大顶堆，即 `top()` 返回的是最大元素。
*   如果需要小顶堆，必须在模板参数中指定底层容器（通常是 `std::vector<T>`）和自定义比较器（`std::greater<T>`）。
*   对于自定义数据类型，需要重载 `<` 运算符，或者提供自定义比较器，才能正确地进行优先级排序。

### 4. 优先队列与贪心算法

**贪心算法回顾：** 贪心算法是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是全局最好或最优的算法。

**优先队列如何帮助实现贪心：**
贪心算法的关键在于每一步如何做出“最优”选择。在很多问题中，“最优”选择并不是固定不变的，它会随着问题的进展而动态变化。优先队列正是解决这一问题的利器：
1.  **动态维护候选集合：** 优先队列可以把所有当前的“候选”方案都存储起来。
2.  **快速找到最优选择：** 它的 `top()` 操作总是能以 `O(1)` 的时间复杂度告诉我们当前“最佳”的那个元素。
3.  **高效移除和更新：** `pop()` 和 `push()` 操作以 `O(log n)` 的时间复杂度完成，使得我们可以在做出一个选择后，快速更新候选集合，并继续寻找下一个最优选择。

**何时考虑使用优先队列贪心？**
当问题满足以下特点时，可以考虑使用优先队列结合贪心策略：
*   问题需要你**反复地从一个集合中选择“最优”的元素**（最大/最小，最紧急/最不紧急等）。
*   选择一个元素后，集合会发生变化（元素被移除，或者新元素被加入），你需要**动态地维护这个“最优”的特性**。
*   传统的排序方法可能效率不够，因为每次选择后都需要重新排序或者遍历查找。

例如，合并果子问题中，我们每次都想合并最小的两堆，优先队列可以帮我们快速找到最小的两堆。任务调度问题中，我们可能需要动态地决定完成哪些任务，或者替换掉哪些任务以获得最大收益，优先队列也能发挥作用。

**关键点剖析：**
*   **思考“优先级”是什么：** 在具体问题中，什么决定了一个元素的“优先”程度？是大小、时间、成本、收益，还是其他复合指标？
*   **选择大顶堆还是小顶堆：** 如果需要总是取出最大的元素，用大顶堆；如果需要总是取出最小的元素，用小顶堆。
*   **贪心策略的正确性：** 优先队列只是一个工具，核心还是你设计的贪心策略是否正确。需要证明每一步的局部最优选择能否导向全局最优。

**常见误区：**
*   **滥用优先队列：** 并非所有需要“最大/最小”的问题都需要优先队列，如果集合是静态的，一次排序可能更简单。
*   **错误选择堆类型：** 大顶堆和小顶堆搞混，导致结果错误。
*   **自定义类型比较器错误：** 对于结构体或类，忘记重载 `<` 运算符或提供 `cmp` 函数，导致优先级判断出错。

### 例题 1: 合并果子

**题目描述：**
在一个果园里，有 `n` 堆果子，每堆果子的数量已知。现在要把所有的果子都合并成一堆。
每一次合并，只能选择其中任意两堆果子进行合并，合并的代价是这两堆果子的总数。
我们的目标是，经过 `n-1` 次合并，将所有果子合并成一堆，并使总的合并代价最小。
例如，有 3 堆果子，数量分别为 1, 2, 3。
*   第一次合并 1 和 2，得到新堆 3，代价为 1+2=3。现在有堆 3, 3。
*   第二次合并 3 和 3，得到新堆 6，代价为 3+3=6。总代价为 3+6=9。
*   如果第一次合并 1 和 3，得到新堆 4，代价为 1+3=4。现在有堆 2, 4。
*   第二次合并 2 和 4，得到新堆 6，代价为 2+4=6。总代价为 4+6=10。
显然，第一种方案代价更小。

**思路分析：**
这个问题是经典的哈夫曼编码的简化版。我们希望总代价最小。合并的代价是两堆果子的总和。这意味着，越早参与合并的果子，它们的数量就会被累加的次数越多。为了让总代价最小，我们应该让那些数量较小的果子尽可能多地参与早期合并，从而减少它们对总代价的贡献。
这启发我们：每次都选择当前数量最少的两堆果子进行合并。

1.  **初始状态：** 将所有果子堆的数量放入一个数据结构中。
2.  **贪心选择：** 每次从数据结构中取出数量最小的两堆果子 `a` 和 `b`。
3.  **执行操作：** 将 `a` 和 `b` 合并，代价为 `a + b`。将这个代价累加到总代价中。
4.  **更新状态：** 将合并后的新堆 `a + b` 重新放回数据结构中。
5.  **重复：** 重复步骤 2-4，直到数据结构中只剩下一堆果子。

什么数据结构可以帮助我们高效地完成“取出最小的两个”和“放入新的”操作呢？没错，就是**小顶优先队列**！

*   **入队：** 将所有果子堆的数量 `n` 次 `push` 到小顶优先队列中。
*   **循环：** 当队列中的元素数量大于 1 时，重复以下操作：
    *   取出队首元素 `a` (最小)。
    *   取出队首元素 `b` (次小)。
    *   计算代价 `a + b`，累加到总代价 `s` 中。
    *   将新堆 `a + b` 重新 `push` 回队列。
*   **结束：** 队列中只剩一个元素时，循环结束，`s` 就是最小总代价。

**代码实现：**

```cpp
#include <iostream>
#include <queue>      // 包含优先队列头文件
#include <vector>     // 优先队列需要底层容器，通常是vector
#include <functional> // 小顶堆需要std::greater

int main() {
    // 使用Allman风格大括号
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int n; // 果子堆的数量
    std::cin >> n;

    // 创建一个小顶优先队列
    // <int, std::vector<int>, std::greater<int>> 表示存储int类型，
    // 底层容器是std::vector<int>，使用std::greater<int>进行比较（即小的优先级高）
    std::priority_queue<int, std::vector<int>, std::greater<int>> q;

    for (int i = 0; i < n; ++i) {
        int x; // 每堆果子的数量
        std::cin >> x;
        q.push(x); // 将每堆果子数量放入优先队列
    }

    long long s = 0; // 总合并代价，可能很大，用long long

    // 当队列中不止一堆果子时，继续合并
    while (q.size() > 1) {
        int a = q.top(); // 取出最小的一堆
        q.pop();

        int b = q.top(); // 取出次小的一堆
        q.pop();

        s += (a + b); // 累加合并代价
        q.push(a + b); // 将合并后的新堆放回队列
    }

    std::cout << s << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：**
    *   初始化：`n` 次 `push` 操作，每次 `O(log n)`，总计 `O(n log n)`。
    *   循环：进行 `n-1` 次合并。每次合并包括两次 `pop` 和一次 `push`，每次操作都是 `O(log n)`。总计 `O(n log n)`。
    *   因此，总时间复杂度为 `O(n log n)`。
*   **空间复杂度：**
    *   优先队列最多存储 `n` 个元素，所以空间复杂度为 `O(n)`。

### 例题 2: 任务调度 (带有截止时间和收益)

**题目描述：**
有 `n` 个任务，每个任务 `i` 都有一个截止时间 `d[i]` 和一个完成它能获得的收益 `p[i]`。
你有一台机器，可以在每个单位时间内完成一个任务。在任何时刻，机器只能处理一个任务。
任务的完成时间必须在它的截止时间之前（含截止时间）。
你的目标是选择一些任务来完成，使得总收益最大。

**思路分析：**
这是一个经典的贪心问题，但直接贪心可能会遇到困难。
如果按收益从大到小排序，先完成收益高的任务，但可能因为截止时间太晚而无法完成其他高收益任务。
如果按截止时间从早到晚排序，先完成截止时间早的任务，但可能浪费时间去完成收益很低的任务。

**正确的贪心策略：**
1.  **排序：** 将所有任务按照它们的**截止时间 `d` 从小到大排序**。
2.  **遍历任务：** 依次考虑每个任务。
3.  **决策：**
    *   假设当前任务是第 `i` 个任务，它的截止时间是 `d[i]`，收益是 `p[i]`。
    *   我们可以在 `1` 到 `d[i]` 的任何一个单位时间完成这个任务。
    *   我们希望在不影响其他任务的前提下，尽可能完成高收益的任务。
    *   用一个**大顶优先队列**来维护当前已经完成的任务的**收益**。
    *   **关键思想：** 假设我们已经处理了截止时间到 `d[i]` 的所有任务。如果当前时间槽位 `d[i]` 已经被占用了（即我们已经完成了 `d[i]` 个任务），并且我们现在想完成当前任务 `i`：
        *   我们可以把当前任务 `i` 的收益 `p[i]` 加入优先队列。
        *   如果优先队列中的任务数量超过了当前可以完成的时间槽位（即 `d[i]`），那么说明我们完成的任务太多了。我们必须放弃一个任务。
        *   应该放弃哪个任务呢？当然是当前已完成任务中**收益最小**的那个！因为放弃它对总收益的影响最小。
        *   所以，如果 `q.size() > d[i]`，我们就 `q.pop()` 掉收益最小的那个（注意，这里需要的是小顶堆）。

**修正后的贪心策略（使用小顶优先队列）：**
1.  **排序：** 将所有任务按照它们的**截止时间 `d` 从小到大排序**。
2.  **数据结构：** 使用一个**小顶优先队列 `q`**，用来存储我们“决定完成”的任务的收益。
3.  **遍历：** 遍历排序后的每个任务 `(d, p)`：
    *   将当前任务的收益 `p` 加入优先队列 `q.push(p)`。
    *   **检查：** 如果当前优先队列中的任务数量 `q.size()` **超过了当前任务的截止时间 `d`**，这意味着我们目前为止已经选择了超过 `d` 个任务，但我们只有 `d` 个时间槽位来完成它们。为了保证总收益最大，我们必须放弃一个任务。
    *   **决策：** 放弃当前已选择任务中收益最小的那个。所以，执行 `q.pop()`。
4.  **计算总收益：** 遍历结束后，优先队列中剩下的所有任务就是我们最终决定的任务集。将这些任务的收益累加起来，就是最大总收益。

**为什么这个贪心策略是正确的？**
*   **按截止时间排序：** 确保我们总是优先考虑那些必须在更早时间完成的任务。
*   **小顶优先队列：** 确保了当时间槽位不足时，我们总是放弃收益最小的任务，从而最大化整体收益。
*   **核心思想：** 对于截止时间为 `D` 的任务，我们最多可以完成 `D` 个任务。优先队列动态地维护了这些任务，确保在截止时间 `D` 之前，我们选择的 `D` 个任务的收益是最大的。

**代码实现：**

```cpp
#include <iostream>
#include <vector>
#include <algorithm> // 用于排序
#include <queue>     // 包含优先队列头文件
#include <functional> // 小顶堆需要std::greater

// 定义任务结构体
struct Task {
    int d; // 截止时间 (deadline)
    int p; // 收益 (profit)
};

// 自定义比较函数，用于按截止时间排序
bool compareTasks(const Task& a, const Task& b) {
    return a.d < b.d; // 按截止时间升序排序
}

int main() {
    // 使用Allman风格大括号
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int n; // 任务数量
    std::cin >> n;

    std::vector<Task> tasks(n);
    for (int i = 0; i < n; ++i) {
        std::cin >> tasks[i].d >> tasks[i].p;
    }

    // 1. 按照截止时间d升序排序
    std::sort(tasks.begin(), tasks.end(), compareTasks);

    // 2. 创建一个小顶优先队列，存储已选择任务的收益
    std::priority_queue<int, std::vector<int>, std::greater<int>> q;

    // 3. 遍历排序后的任务
    for (int i = 0; i < n; ++i) {
        // 将当前任务的收益加入优先队列
        q.push(tasks[i].p);

        // 如果当前已选择的任务数量超过了当前任务的截止时间
        // 说明我们没有足够的时间槽位完成所有这些任务
        // 必须放弃一个任务，为了最大化总收益，放弃收益最小的那个
        if (q.size() > tasks[i].d) {
            q.pop(); // 弹出收益最小的任务
        }
    }

    // 4. 计算最终的总收益
    long long s = 0; // 总收益，可能很大，用long long
    while (!q.empty()) {
        s += q.top();
        q.pop();
    }

    std::cout << s << std::endl;

    return 0;
}
```

**复杂度分析：**
*   **时间复杂度：**
    *   读取输入并存储：`O(n)`。
    *   任务排序：`O(n log n)`。
    *   遍历任务并使用优先队列：`n` 次循环，每次循环包括一次 `push` 和一次可能的 `pop`，每次操作都是 `O(log k)`，其中 `k` 是优先队列中的元素数量，`k <= n`。所以这部分是 `O(n log n)`。
    *   计算最终总收益：最多 `n` 次 `pop`，`O(n log n)`。
    *   因此，总时间复杂度为 `O(n log n)`。
*   **空间复杂度：**
    *   存储任务：`O(n)`。
    *   优先队列：最多存储 `n` 个元素，`O(n)`。
    *   因此，总空间复杂度为 `O(n)`。

// 模板三：自定义类型优先队列 (结构体作为元素)

// =========================================================
struct Item {
    int v; // 值
    int p; // 优先级 (假设优先级越高，p值越大)

    // 重载 < 运算符，定义“小于”的含义
    // 对于大顶堆，a < b 为 true 表示 b 的优先级高于 a
    // 所以，如果 p 值越大优先级越高，那么 p 小的应该被认为是“小于”p 大的
    bool operator<(const Item& o) const {
        return p < o.p; // 按照p值降序排列 (p值大的优先级高)
    }
    // 如果想要小顶堆，可以写 return p > o.p; 或者使用自定义比较器
};

// 自定义比较器 for 小顶堆 (P值越小优先级越高)
struct CompareItemMinP {
    bool operator()(const Item& a, const Item& b) const {
        return a.p > b.p; // 如果a的p值大于b的p值，则a的优先级更低 (b优先)
    }
};

void custom_type_priority_queue_example() {
    std::cout << "--- 自定义类型优先队列示例 ---" << std::endl;

    // 使用重载 < 运算符的大顶堆 (按p值从大到小排列)
    std::priority_queue<Item> max_q_item;
    max_q_item.push({100, 5});
    max_q_item.push({200, 10});
    max_q_item.push({50, 2});
    std::cout << "大顶堆 (按p值): 队首元素值=" << max_q_item.top().v
              << ", 优先级=" << max_q_item.top().p << std::endl; // 200, 10

    // 使用自定义比较器的小顶堆 (按p值从小到大排列)
    std::priority_queue<Item, std::vector<Item>, CompareItemMinP> min_q_item;
    min_q_item.push({100, 5});
    min_q_item.push({200, 10});
    min_q_item.push({50, 2});
    std::cout << "小顶堆 (按p值): 队首元素值=" << min_q_item.top().v
              << ", 优先级=" << min_q_item.top().p << std::endl; // 50, 2
    std::cout << std::endl;
}

**详细注释：**
*   `#include <queue>`：必须包含此头文件才能使用 `std::priority_queue`。
*   `#include <vector>`：`std::priority_queue` 默认底层容器是 `std::vector`，虽然可以直接写，但明确包含更好。
*   `#include <functional>`：如果需要小顶堆，会用到 `std::greater`，它定义在 `functional` 头文件中。
*   `std::priority_queue<int> q;`：这是最简单的声明，创建一个存储 `int` 类型的大顶优先队列。
*   `std::priority_queue<int, std::vector<int>, std::greater<int>> q;`：这是小顶优先队列的声明。
    *   `int`：队列中存储的元素类型。
    *   `std::vector<int>`：指定底层容器为 `std::vector<int>`。
    *   `std::greater<int>`：指定比较器。`std::greater<int>()` 意味着当 `a > b` 时，`a` 的优先级更高（即小的元素排在前面），从而实现小顶堆。
*   `q.push(x);`：将元素 `x` 插入优先队列。
*   `q.top();`：获取队首元素（优先级最高的元素），不删除。
*   `q.pop();`：删除队首元素。
*   `q.empty();`：判断队列是否为空。
*   `q.size();`：返回队列中元素的数量。
*   对于自定义类型 `struct Item`：
    *   重载 `operator<`：这是 C++ STL 容器（包括 `priority_queue`）默认使用的比较方式。对于 `priority_queue`，如果 `a < b` 返回 `true`，则 `b` 的优先级高于 `a`（即 `b` 会排在 `a` 前面）。所以，如果我们要实现大顶堆（`p` 值越大优先级越高），那么当 `a.p < b.p` 时，我们认为 `a` “小于” `b`，这样 `b` 就会被排在前面。
    *   自定义比较器 `struct CompareItemMinP`：这是一个函数对象（functor）。它的 `operator()` 定义了两个 `Item` 对象的比较规则。如果 `operator()(a, b)` 返回 `true`，则 `a` 被认为“优先级低于” `b`。对于小顶堆（`p` 值越小优先级越高），如果 `a.p > b.p`，那么 `a` 的优先级就低于 `b`，`b` 应该排在前面。

**1. 思考题：优先队列和普通队列有什么本质区别？**
*   **学生回答引导：** 普通队列是“先来先服务”，就像排队买票。优先队列是“VIP优先服务”，就像医院急诊室。它们的出队顺序不同。

**2. 提问：为什么优先队列的底层通常是堆（Heap）而不是其他数据结构，比如链表或者平衡二叉搜索树？**
*   **学生回答引导：**
    *   链表：虽然插入 `O(1)`，但每次取最大/最小需要遍历 `O(n)`。
    *   平衡二叉搜索树（如 `std::set`）：插入、删除、查找最大/最小都是 `O(log n)`，但实现更复杂，常数因子更大。
    *   堆：插入和删除都是 `O(log n)`，查找最大/最小 `O(1)`。而且堆可以用数组实现，存储效率高，缓存友好，实现相对简单。综合来看，堆在效率和实现复杂度上是优先队列的最佳选择。

---

### 答案

#### 基础巩固

**1. 找出第 K 大的元素：**
使用一个小顶优先队列 `q`。
遍历所有 `n` 个数：
*   将当前数 `x` 插入 `q.push(x)`。
*   如果 `q.size() > k`，则 `q.pop()` (移除最小的元素)。
遍历结束后，`q.top()` 就是第 `k` 大的元素。
**理由：** 队列中始终维护着当前看到的 `k` 个最大的元素。当新元素加入且队列大小超过 `k` 时，说明当前队列中最小的那个（也就是第 `k` 大的）不再是全局的第 `k` 大，因为它被更大的元素挤出去了。

**2. 数据流中的中位数：**
维护两个优先队列：
*   `max_q` (大顶堆)：存储较小的一半元素。
*   `min_q` (小顶堆)：存储较大的一半元素。
始终保持 `max_q.size() >= min_q.size()` 且 `max_q.size() - min_q.size() <= 1`。
并且，`max_q.top() <= min_q.top()` (如果 `min_q` 不为空)。

插入元素 `x`：
*   如果 `x <= max_q.top()` (或 `max_q` 为空)，`push x` 到 `max_q`。
*   否则，`push x` 到 `min_q`。
调整平衡：
*   如果 `max_q.size() < min_q.size()`，将 `min_q.top()` 移到 `max_q`。
*   如果 `max_q.size() > min_q.size() + 1`，将 `max_q.top()` 移到 `min_q`。
中位数：`max_q.top()`。

#### 能力提升

**3. K 个最接近的点：**
使用一个**大顶优先队列** `q`。优先队列中存储 `pair<long long, pair<int, int>>`，其中 `long long` 是距离平方 `x^2 + y^2`，`pair<int, int>` 是点的坐标。
遍历所有 `n` 个点 `(x, y)`：
*   计算距离平方 `d = x*x + y*y`。
*   将 `(d, {x, y})` `push` 到 `q` 中。
*   如果 `q.size() > k`，则 `q.pop()` (移除距离最大的点)。
遍历结束后，`q` 中剩下的 `k` 个点就是距离最近的 `k` 个点。

**4. 最小区间覆盖：**
1.  **预处理：** 移除所有 `R_i < S` 或 `L_i > T` 的区间。将剩余区间按 `L_i` 升序排序。
2.  **贪心：**
    *   `curr = S` (当前已覆盖到的最右端点)。
    *   `ans = 0` (选择的区间数量)。
    *   循环直到 `curr >= T`：
        *   `max_r = curr` (从 `curr` 出发能到达的最远右端点)。
        *   使用一个**大顶优先队列** `q` 来存储所有满足 `L_i <= curr` 的区间的 `R_i` 值。
        *   遍历排序后的区间，将所有 `L_i <= curr` 的区间的 `R_i` `push` 到 `q` 中。
        *   如果 `q` 为空，且 `max_r == curr` (没有任何区间能覆盖 `curr`
