## 趣味引入

同学们，你们知道吗？我们平时生活中使用的数字，比如10、-5、200，叫做十进制数。但计算机这个“小脑袋瓜”可不认识十进制数哦！它只认识0和1，也就是二进制数。

我们都知道，正数比如10，可以表示成二进制的 `1010`。那么，负数比如 `-5`，该怎么用 `0` 和 `1` 来表示呢？如果直接在前面加一个负号，计算机可读不懂！

想象一下，我们想让计算机做 `5 + (-5)` 这样的计算，结果应该是 `0`。如果计算机不能正确地表示负数，那这个简单的加法它也算不出来。这就好像我们去买东西，如果商店的收款机连负数都处理不了，那退货找零就麻烦了！

为了解决这个问题，聪明的计算机科学家们发明了几种给数字“穿衣服”的方法，让计算机也能理解正数和负数。今天，我们就来学习这三种神奇的“衣服”：**原码、反码、补码**！它们就像是数字在计算机世界里的不同身份证，各有各的特点和用途。

## 知识点讲解

### 1. 二进制基础回顾

在深入学习原码、反码、补码之前，我们先快速回顾一下二进制数。
*   二进制只有 `0` 和 `1` 两个数字。
*   每一位（Bit）的权重是 `2` 的幂次。
*   例如：`1011_2` = `1 * 2^3 + 0 * 2^2 + 1 * 2^1 + 1 * 2^0` = `8 + 0 + 2 + 1` = `11_10`。

### 2. 有符号数的表示问题

计算机中存储数据通常有固定的位数，比如8位、16位、32位或64位。
假设我们用一个8位的二进制数来表示整数。
如果只表示正数和0，那么8位二进制数可以表示的范围是 `00000000` (`0`) 到 `11111111` (`255`)。
但是，我们还需要表示负数！怎么办呢？

### 3. 原码 (Sign-Magnitude Code)

原码是最直观的表示方法。
*   **最高位（最左边一位）** 用作**符号位**：
    *   `0` 表示正数
    *   `1` 表示负数
*   **其余位** 表示数的**绝对值**。

**例子 (以8位为例)：**

*   **正数 +5**：
    *   符号位为 `0` (正数)
    *   绝对值 `5` 的二进制是 `00000101`
    *   所以，`+5` 的原码是 `00000101`

*   **负数 -5**：
    *   符号位为 `1` (负数)
    *   绝对值 `5` 的二进制是 `00000101`
    *   所以，`-5` 的原码是 `10000101`

**原码的优点：** 直观，容易理解。
**原码的缺点：**
1.  **0 的表示不唯一**：`+0` 的原码是 `00000000`，`-0` 的原码是 `10000000`。这会给计算机带来麻烦，因为 `0` 和 `-0` 应该是同一个值。
2.  **加减法运算复杂**：例如 `5 + (-5)`，如果直接相加 `00000101 + 10000101` 得到 `10001010` (`-10`)，结果就不对了。计算机需要额外判断是加法还是减法，这增加了硬件设计的复杂性。

### 4. 反码 (One's Complement)

为了解决原码中加减法运算复杂的问题，人们发明了反码。

*   **正数的反码**：与原码**相同**。
*   **负数的反码**：在原码的基础上，符号位不变，其余位**按位取反**（`0` 变 `1`，`1` 变 `0`）。

**例子 (以8位为例)：**

*   **正数 +5**：
    *   原码是 `00000101`
    *   反码也是 `00000101`

*   **负数 -5**：
    *   原码是 `10000101`
    *   符号位 `1` 不变，其余位 `0000101` 取反得到 `1111010`
    *   所以，`-5` 的反码是 `11111010`

**反码的优点：** 加减法运算比原码稍微简化了一点点，例如 `5 + (-5)`：
    `00000101` (`+5` 的反码)
  `+ 11111010` (`-5` 的反码)
  `------------`
  `11111111`
这个结果 `11111111` 看起来像 `-0` 的反码。如果进行“循环进位”（末位有进位要加到最低位），可以得到正确结果。

**反码的缺点：**
1.  **0 的表示仍不唯一**：`+0` 的反码是 `00000000`，`-0` 的反码是 `11111111`。仍然存在两个零。
2.  **“循环进位”** 增加了运算的复杂性。

### 5. 补码 (Two's Complement)

补码是计算机中表示有符号整数的**标准方法**，它完美解决了原码和反码的所有问题！

*   **正数的补码**：与原码、反码**相同**。
*   **负数的补码**：在反码的基础上，**末位加 `1`**。

**转换步骤总结 (负数)：**
1.  写出其绝对值的原码。
2.  将原码的符号位变为 `1`，其余位取反，得到反码。
3.  将反码的末位加 `1`，得到补码。

**例子 (以8位为例)：**

*   **正数 +5**：
    *   原码是 `00000101`
    *   反码是 `00000101`
    *   补码也是 `00000101`

*   **负数 -5**：
    1.  绝对值 `5` 的原码（8位）：`00000101`
    2.  转换为 `-5` 的原码：`10000101`
    3.  转换为 `-5` 的反码：符号位不变，其余位取反 `11111010`
    4.  反码末位加 `1`：`11111010 + 1 = 11111011`
    *   所以，`-5` 的补码是 `11111011`

**我们来验证一下 `5 + (-5)` 在补码下的运算：**
    `00000101` (`+5` 的补码)
  `+ 11111011` (`-5` 的补码)
  `------------`
  `100000000` (最高位溢出，被截断)
结果是 `00000000`，这正是 `0` 的补码！完美！

**补码的优点：**
1.  **0 的表示唯一**：`0` 的补码只有 `00000000` 一种表示。
2.  **加减法运算统一**：无论是正数加正数、正数加负数，还是负数加负数，都可以统一用加法电路进行运算，大大简化了计算机硬件的设计。
3.  **表示范围更广**：对于 N 位二进制数，补码可以表示 `-(2^(N-1))` 到 `(2^(N-1) - 1)`。例如，8位补码可以表示 `-(2^7)` 到 `(2^7 - 1)`，即 `-128` 到 `127`。

**为什么是 `-128` 到 `127`？**
*   最大的正数 `127` 是 `01111111`。
*   最小的负数 `-128` 是 `10000000`。
    *   如果按照原码来看 `10000000` 应该是 `-0`。
    *   但是补码中 `0` 只有 `00000000`。所以 `10000000` 这个编码就被用来表示比 `-(2^(N-1)-1)` 还要小一个的负数，即 `-(2^(N-1))`。

### 6. 补码转回十进制

如何将一个补码转换回十进制数呢？
*   如果补码的符号位是 `0` (正数)，直接按无符号二进制转十进制。
*   如果补码的符号位是 `1` (负数)，有两种方法：
    1.  **逆向转换法**：对补码减 `1`，然后对除符号位外的其余位取反，最后加上负号。
    2.  **加权法**：最高位（符号位）的权重是 `-(2^(N-1))`，其余位的权重不变。
        例如：`11111011` (`-5` 的补码，8位)
        `1 * (-2^7) + 1 * 2^6 + 1 * 2^5 + 1 * 2^4 + 1 * 2^3 + 0 * 2^2 + 1 * 2^1 + 1 * 2^0`
        `= -128 + 64 + 32 + 16 + 8 + 0 + 2 + 1`
        `= -128 + 123`
        `= -5`

这两种方法都能得到正确结果，第二种方法更直接，但需要记住最高位的特殊权重。

### 小结：

| 类型   | 正数                                | 负数                                |
| :----- | :---------------------------------- | :---------------------------------- |
| **原码** | 符号位 `0`，其余位表示绝对值        | 符号位 `1`，其余位表示绝对值        |
| **反码** | 与原码相同                          | 符号位 `1`，其余位按位取反          |
| **补码** | 与原码、反码相同                    | 反码末位加 `1`                      |

记住：计算机内部**只存储补码**！

## 代码示例

我们来用 C++ 模拟一下十进制数转换为8位原码、反码、补码的过程。

```cpp
#include <bits/stdc++.h>

using namespace std;

// 函数：将十进制数转换为指定位数的二进制字符串（用于表示绝对值）
// num: 十进制数，bits: 位数
string toBinaryString(int num, int bits) {
    string s = "";
    for (int i = bits - 1; i >= 0; --i) {
        if ((num >> i) & 1) { // 检查第i位是否为1
            s += '1';
        } else {
            s += '0';
        }
    }
    return s;
}

// 函数：将二进制字符串按位取反
string flipBits(string s) {
    for (int i = 0; i < s.length(); ++i) {
        if (s[i] == '0') {
            s[i] = '1';
        } else {
            s[i] = '0';
        }
    }
    return s;
}

// 函数：将二进制字符串末位加1
string addOne(string s) {
    int n = s.length();
    for (int i = n - 1; i >= 0; --i) {
        if (s[i] == '0') { // 遇到0，变成1，加法结束
            s[i] = '1';
            return s;
        } else { // 遇到1，变成0，继续进位
            s[i] = '0';
        }
    }
    // 如果所有位都是1，加1后会溢出，这里简化处理，实际8位不会发生
    // 比如 11111111 + 1 = 100000000 (9位)，但我们只取8位
    return s;
}


int main() {
    int n;
    int bits = 8; // 设定为8位

    cout << "请输入一个十进制整数 (范围: -128 到 127): ";
    cin >> n;

    if (n > 127 || n < -128) {
        cout << "输入的数字超出了8位有符号数的表示范围！" << endl;
        return 1;
    }

    string s_original, s_one_complement, s_two_complement;

    // 1. 计算原码
    if (n >= 0) {
        // 正数或0，符号位为0，后面是绝对值
        s_original = '0' + toBinaryString(n, bits - 1);
    } else {
        // 负数，符号位为1，后面是绝对值
        s_original = '1' + toBinaryString(abs(n), bits - 1);
    }
    cout << "原码: " << s_original << endl;

    // 2. 计算反码
    if (n >= 0) {
        // 正数或0，反码和原码相同
        s_one_complement = s_original;
    } else {
        // 负数，符号位不变，其余位取反
        s_one_complement = '1' + flipBits(s_original.substr(1)); // substr(1) 获取除符号位外的部分
    }
    cout << "反码: " << s_one_complement << endl;

    // 3. 计算补码
    if (n >= 0) {
        // 正数或0，补码和原码、反码相同
        s_two_complement = s_original;
    } else {
        // 负数，反码末位加1
        s_two_complement = addOne(s_one_complement);
    }
    cout << "补码: " << s_two_complement << endl;

    return 0;
}
```
**代码讲解：**
*   `toBinaryString(int num, int bits)`: 这个函数负责把一个十进制正整数转换成指定位数的二进制字符串。我们用位运算 `(num >> i) & 1` 来检查每一位是0还是1。
*   `flipBits(string s)`: 这个函数用来对一个二进制字符串进行按位取反操作，把 `0` 变成 `1`，把 `1` 变成 `0`。
*   `addOne(string s)`: 这个函数实现二进制字符串的末位加 `1` 操作，模拟补码的最后一步。
*   `main` 函数中，我们根据输入的 `n` 是正数还是负数，分别计算它的原码、反码和补码。注意，对于负数，我们先取出它的绝对值，然后根据规则一步步转换。

**运行示例：**
输入 `5`
```
原码: 00000101
反码: 00000101
补码: 00000101
```

输入 `-5`
```
原码: 10000101
反码: 11111010
补码: 11111011
```

输入 `-128`
```
原码: 110000000 // 注意：-128的原码在8位下无法直接表示，因为绝对值128需要8位，加上符号位就9位了。
               // 我们的代码这里处理的是：符号位 + 7位绝对值，所以128的绝对值是10000000，加上符号位1，原码会变成110000000，超出了8位。
               // 这个例子恰好说明了原码的表示限制。
               // 正确的8位有符号原码范围是 -127 到 127。
               // 所以，在8位下，-128是不能用原码表示的。
               // 但补码可以表示-128。
               // 为了代码的通用性，我们先假设可以表示，但要记住这个局限。
               // 在代码中，我加入了范围检查，避免了-128的原码直接计算问题。
               // 如果输入-128，代码会提示超出范围，因为toBinaryString(128, 7)会得到10000000，但我们只需要7位。
               // 实际上，-128的补码是10000000。
               // 让我们修改一下代码，使其能正确处理-128的补码情况，并更清晰地解释原码的局限。
```
**修正后的代码思考：** 8位表示 `-(2^(N-1))` 到 `(2^(N-1) - 1)`。
对于 `n = -128` (8位)：
*   原码：`10000000` (如果允许8位表示绝对值，最高位再作为符号位，则需要9位。8位原码只能表示到-127)。
    *   为了兼容8位原码的限制，我们的 `toBinaryString(abs(n), bits - 1)` 传入 `abs(-128)` 即 `128` 和 `bits-1` 即 `7`。`toBinaryString(128, 7)` 会得到 `0000000` (因为128需要8位才能表示，7位无法表示)。
    *   这会使得 `s_original` 变为 `10000000`，这在原码中代表 `-0` 或 `-128` (如果 `10000000` 是特殊规定)。
    *   为了教学清晰，我们应该强调8位原码无法表示-128。
    *   **更正：** 在 `main` 函数的范围检查中，如果 `n < -127` (对于原码)，就应该提示原码无法表示。但补码可以。

**为了简化和符合 Level 3，我们暂时将范围限制在 `-127` 到 `127`，这样原码也能正确显示。** 否则，对于 `-128` 的补码，它的原码和反码是无法通过常规方法得到的（或者说，8位原码无法直接表示 `-128`）。

让我们将范围检查调整为更符合原码、反码、补码通用转换的 `[-127, 127]`，或者明确指出 `-128` 的特殊性。
考虑到 Level 3，我们先限制在 `[-127, 127]`，这样所有代码转换都是通用且正确的。
如果需要处理 `-128`，则需要特殊说明和处理。

**修正 `main` 函数的范围判断：**
```cpp
#include <bits/stdc++.h>

using namespace std;

// 函数：将十进制数转换为指定位数的二进制字符串（用于表示绝对值）
// num: 十进制数，bits: 位数
string toBinaryString(int num, int bits) {
    string s = "";
    // 确保 num 是正数
    num = abs(num);
    for (int i = bits - 1; i >= 0; --i) {
        if ((num >> i) & 1) { // 检查第i位是否为1
            s += '1';
        } else {
            s += '0';
        }
    }
    return s;
}

// 函数：将二进制字符串按位取反
string flipBits(string s) {
    for (int i = 0; i < s.length(); ++i) {
        if (s[i] == '0') {
            s[i] = '1';
        } else {
            s[i] = '0';
        }
    }
    return s;
}

// 函数：将二进制字符串末位加1
string addOne(string s) {
    int n_len = s.length();
    for (int i = n_len - 1; i >= 0; --i) {
        if (s[i] == '0') { // 遇到0，变成1，加法结束
            s[i] = '1';
            return s;
        } else { // 遇到1，变成0，继续进位
            s[i] = '0';
        }
    }
    // 如果所有位都是1，例如11111111，加1后会溢出到9位 100000000
    // 对于8位补码，11111111是-1，加1后是00000000
    // 这种情况在我们的上下文（负数反码加1）中，不会导致所有位都是1的情况，因为反码的符号位是1
    // 例如 -1 的反码是 11111110，加1是 11111111
    // 只有 -0 的反码是 11111111，加1后是 00000000（溢出一位，取8位）
    // 为了教学清晰，我们假设输入在 [-127, 127] 范围内，避免 -128 和 -0 的特殊情况。
    return s; // 理论上这里不会被执行到，除非输入是-0的反码11111111
}


int main() {
    int n;
    int bits = 8; // 设定为8位

    cout << "请输入一个十进制整数 (范围: -127 到 127): ";
    cin >> n;

    // 8位有符号数的原码和反码表示范围是 -127 到 127
    // 8位有符号数的补码表示范围是 -128 到 127
    // 为了统一处理，我们目前限制在 [-127, 127] 这个所有编码都能直接表示的范围。
    if (n > 127 || n < -127) {
        cout << "输入的数字超出了8位原码/反码的表示范围！" << endl;
        return 1;
    }

    string s_original, s_one_complement, s_two_complement;

    // 1. 计算原码
    if (n >= 0) {
        // 正数或0，符号位为0，后面是绝对值
        s_original = '0' + toBinaryString(n, bits - 1); // bits-1 是因为一位是符号位
    } else {
        // 负数，符号位为1，后面是绝对值
        s_original = '1' + toBinaryString(abs(n), bits - 1);
    }
    cout << "十进制数 " << n << " 的8位原码: " << s_original << endl;

    // 2. 计算反码
    if (n >= 0) {
        // 正数或0，反码和原码相同
        s_one_complement = s_original;
    } else {
        // 负数，符号位不变，其余位取反
        s_one_complement = '1' + flipBits(s_original.substr(1)); // substr(1) 获取除符号位外的部分
    }
    cout << "十进制数 " << n << " 的8位反码: " << s_one_complement << endl;

    // 3. 计算补码
    if (n >= 0) {
        // 正数或0，补码和原码、反码相同
        s_two_complement = s_original;
    } else {
        // 负数，反码末位加1
        s_two_complement = addOne(s_one_complement);
    }
    cout << "十进制数 " << n << " 的8位补码: " << s_two_complement << endl;

    return 0;
}
```

## 练习题目

### 选择题

1.  在8位二进制中，十进制数 `-10` 的原码是？
    A. `00001010`
    B. `10001010`
    C. `11110101`
    D. `11110110`

2.  在8位二进制中，十进制数 `+7` 的补码是？
    A. `00000111`
    B. `10000111`
    C. `01111000`
    D. `11111001`

3.  一个8位二进制数 `11111110`，如果它表示的是补码，那么它对应的十进制数是？
    A. `-1`
    B. `-2`
    C. `-126`
    D. `-0`

4.  计算机内部为什么普遍使用补码来表示有符号数？
    A. 补码的表示范围比原码和反码更大。
    B. 补码的转换最简单。
    C. 补码能够将加法和减法运算统一处理，且0的表示唯一。
    D. 补码占用的存储空间最少。

### 填空题

1.  十进制数 `-3` 的8位原码是 `________`，反码是 `________`，补码是 `________`。
2.  8位二进制补码 `10000000` 对应的十进制数是 `________`。
3.  在8位二进制中，能够表示的最大正整数（补码形式）是 `________`，对应的十进制数是 `________`。

---

### 答案与解析

**选择题答案：**

1.  **B**
    *   解析：`-10` 的绝对值 `10` 的二进制是 `00001010` (7位)。
    *   原码的符号位是 `1` (负数)，所以是 `10001010`。

2.  **A**
    *   解析：正数的原码、反码、补码都相同。
    *   `+7` 的二进制是 `00000111` (7位)。
    *   加上符号位 `0`，所以是 `00000111`。

3.  **B**
    *   解析：补码 `11111110`。符号位是 `1`，说明是负数。
    *   逆向转换：补码减 `1` 得到 `11111101` (这是反码)。
    *   反码除符号位外取反得到 `00000010` (这是绝对值的二进制)。
    *   `00000010` 转换为十进制是 `2`。
    *   所以原数是 `-2`。
    *   或者用加权法：`1*(-128) + 1*64 + 1*32 + 1*16 + 1*8 + 1*4 + 1*2 + 0*1`
        `= -128 + 64 + 32 + 16 + 8 + 4 + 2`
        `= -128 + 126`
        `= -2`。

4.  **C**
    *   解析：补码最重要的优点是解决了原码和反码中 `0` 表示不唯一的问题，并且能够将加法和减法运算统一为加法，简化了计算机硬件设计。

**填空题答案：**

1.  原码：`10000011`，反码：`11111100`，补码：`11111101`。
    *   解析：
        *   `+3` 的二进制是 `00000011` (7位)。
        *   原码：符号位 `1` + `0000011` = `10000011`。
        *   反码：原码符号位不变 `1`，其余位取反 `1111100` = `11111100`。
        *   补码：反码 `11111100` 末位加 `1` = `11111101`。

2.  `-128`
    *   解析：`10000000` 是8位补码能表示的最小负数。最高位 `1` 表示负数，其余位全 `0`。用加权法计算为 `1 * (-2^7) + 0 * 2^6 + ... + 0 * 2^0 = -128`。

3.  补码：`01111111`，十进制数：`127`。
    *   解析：最大的正数，符号位为 `0`，其余位全 `1`。`01111111` 对应的十进制数是 `64+32+16+8+4+2+1 = 127`。
