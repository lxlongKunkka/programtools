## 深度知识点讲解

### 什么是数据结构？

在开始学习栈和队列之前，我们先简单聊聊什么是“数据结构”。
想象一下，你有很多很多玩具，如果它们乱七八糟地堆在一起，你找起来是不是很麻烦？但如果你把它们分门别类，比如乐高放一个箱子，小汽车放一个箱子，是不是就好找多了？
**数据结构**，就是计算机科学家们想出来的，用来**组织和存储数据**的各种“箱子”或者“货架”。不同的数据结构，有不同的组织方式，适合解决不同的问题。栈和队列就是其中两种非常基础且重要的“货架”。

### 1. 栈 (Stack)：后进先出 (LIFO)

栈，顾名思义，就像一叠东西。它的特点是“后进先出”（Last In, First Out），简称 LIFO。

*   **生活类比**：
    *   **叠盘子**：新放的盘子总是在最上面，要拿走也得先拿最上面的。
    *   **自动铅笔芯**：你把铅笔芯一根根塞进去，最后塞进去的，会最先被按出来使用。
    *   **弹夹装子弹**：子弹从上面压进去，最后压进去的子弹，最先被射击出去。
    *   **浏览器的“后退”按钮**：你访问的网页A，然后B，再C。当你点“后退”时，会从C退到B，再退到A。最后访问的C，最先被“退”掉。

*   **栈的核心操作**：
    *   `push(x)`：**入栈**。把元素 `x` 放到栈的顶部。
    *   `pop()`：**出栈**。把栈顶的元素拿走。
    *   `top()`：**查看栈顶**。看看栈顶的元素是谁，但不把它拿走。
    *   `empty()`：**判断栈空**。问问栈里还有没有东西。
    *   `size()`：**栈的大小**。问问栈里有多少东西。

*   **如何用数组模拟栈？**
    *   我们用一个普通的数组 `s[MAXN]` 来存放栈里的元素。
    *   我们还需要一个“小助手”，一个整数变量 `t` (代表 `top` 栈顶)，来记录栈顶在哪里。
        *   最开始，栈是空的，我们可以让 `t = 0`。这表示栈里一个元素都没有。
        *   当 `t` 等于某个值时，表示数组中 `s[0]` 到 `s[t-1]` 存放着栈里的元素。`t` 的值，也正好是栈里元素的个数。
    *   **`push(x)` 入栈**：
        *   把 `x` 放到数组的 `s[t]` 位置。
        *   然后让 `t` 增加 1，指向下一个空位置。
        *   就像：`s[t] = x; t = t + 1;` 简写为 `s[t++] = x;`
        *   **注意**：在 `push` 之前，要检查栈是不是满了（`t == MAXN`）。如果满了，就不能再放了，这叫“栈溢出”。
    *   **`pop()` 出栈**：
        *   我们不需要真的“删除”数组里的元素，只要把 `t` 减少 1。
        *   这样，`t` 就不再指向原来的栈顶元素了，那个元素也就“不在栈里”了。
        *   就像：`t = t - 1;` 简写为 `t--;`
        *   **注意**：在 `pop` 之前，要检查栈是不是空的（`t == 0`）。如果空了，就不能再拿了，这叫“栈空”。
    *   **`top()` 查看栈顶**：
        *   栈顶元素就是 `s[t-1]`。
        *   **注意**：在 `top` 之前，也要检查栈是不是空的。
    *   **`empty()` 判断栈空**：
        *   如果 `t` 等于 0，说明栈里没有元素，就是空的。
        *   `return t == 0;`
    *   **`size()` 栈的大小**：
        *   `t` 的值就代表栈里元素的个数。
        *   `return t;`

*   **常见误区**：
    *   搞混 `push` 和 `pop` 的顺序，导致 LIFO 特性不符。
    *   忘记检查栈满和栈空的情况，直接操作可能导致程序崩溃。

### 2. 队列 (Queue)：先进先出 (FIFO)

队列，就像排队一样。它的特点是“先进先出”（First In, First Out），简称 FIFO。

*   **生活类比**：
    *   **排队买票/打饭**：最先排队的人，最先获得服务。
    *   **打印机任务**：你发了一个打印任务，小明发了一个，小红发了一个。打印机总是先打印你的，然后小明的，最后小红的。
    *   **公交车站**：公交车来了，先上车的人，先到达目的地（当然，这是理想情况）。

*   **队列的核心操作**：
    *   `push(x)`：**入队**。把元素 `x` 放到队列的尾部。
    *   `pop()`：**出队**。把队列头部的元素拿走。
    *   `front()`：**查看队头**。看看队头元素是谁，但不把它拿走。
    *   `empty()`：**判断队空**。问问队列里还有没有人排队。
    *   `size()`：**队列的大小**。问问队列里有多少人。

*   **如何用数组模拟队列？**
    *   我们用一个普通的数组 `q[MAXN]` 来存放队列里的元素。
    *   我们这次需要两个“小助手”：
        *   `h` (代表 `head` 队头)，记录队头元素的下标。
        *   `t` (代表 `tail` 队尾)，记录队尾元素的下一个空位置的下标。
        *   最开始，队列是空的，我们可以让 `h = 0`，`t = 0`。
    *   **`push(x)` 入队**：
        *   把 `x` 放到数组的 `q[t]` 位置。
        *   然后让 `t` 增加 1，指向下一个空位置。
        *   就像：`q[t] = x; t = t + 1;` 简写为 `q[t++] = x;`
        *   **注意**：在 `push` 之前，要检查队列是不是满了（`t == MAXN`）。如果满了，就不能再放了，这叫“队列溢出”。
    *   **`pop()` 出队**：
        *   我们不需要真的“删除”数组里的元素，只要让 `h` 增加 1。
        *   这样，`h` 就指向了下一个元素，原来的队头元素就“出队”了。
        *   就像：`h = h + 1;` 简写为 `h++;`
        *   **注意**：在 `pop` 之前，要检查队列是不是空的（`h == t`）。如果空了，就不能再拿了，这叫“队列空”。
    *   **`front()` 查看队头**：
        *   队头元素就是 `q[h]`。
        *   **注意**：在 `front` 之前，也要检查队列是不是空的。
    *   **`empty()` 判断队空**：
        *   如果 `h` 等于 `t`，说明队头和队尾指针重合，队列里没有元素，就是空的。
        *   `return h == t;`
    *   **`size()` 队列的大小**：
        *   队列里元素的个数是 `t - h`。
        *   `return t - h;`

*   **常见误区**：
    *   搞混 `h` 和 `t` 的作用，导致 FIFO 特性不符。
    *   忘记检查队列满和队列空的情况。
    *   **“假溢出”问题**：当 `h` 不断增加，即使前面有很多空位置，`t` 还是会一直往后走，最终可能达到 `MAXN`，导致队列“看起来满了”，但前面还有很多空间没用。对于 GESP 三级，我们暂时不深入循环队列解决这个问题，只需理解这是“线性队列”的一个小缺点。

### 栈与队列的对比

| 特性     | 栈 (Stack)       | 队列 (Queue)     |
| :------- | :--------------- | :--------------- |
| **原则** | LIFO (后进先出)  | FIFO (先进先出)  |
| **操作端** | 只有一个端（栈顶） | 两个端（队头、队尾） |
| **入操作** | `push` (入栈)    | `push` (入队)    |
| **出操作** | `pop` (出栈)     | `pop` (出队)     |
| **查看操作** | `top` (栈顶)     | `front` (队头)   |
| **生活例子** | 叠盘子、弹夹     | 排队买票、打印任务 |

## 典型例题精讲

### 例题 1：模拟栈操作

**题目描述**：
假设我们有一个最多能放 5 个整数的栈。请模拟以下操作序列，并输出每次 `pop` 操作弹出的元素。
操作序列：
1.  `push 10`
2.  `push 20`
3.  `pop`
4.  `push 30`
5.  `pop`
6.  `push 40`
7.  `push 50`
8.  `pop`
9.  `push 60`
10. `pop`
11. `pop`
12. `empty` (判断栈是否为空)
13. `size` (获取栈中元素个数)

**思路分析**：
我们用一个数组 `s[5]` 和一个栈顶指针 `t` 来模拟。`t` 初始为 0。

1.  `push 10`: `s[0] = 10; t = 1;` 栈: `[10]`
2.  `push 20`: `s[1] = 20; t = 2;` 栈: `[10, 20]`
3.  `pop`: `t = 1;` 弹出 `s[1]` (即 20)。输出：`20`。 栈: `[10]`
4.  `push 30`: `s[1] = 30; t = 2;` 栈: `[10, 30]`
5.  `pop`: `t = 1;` 弹出 `s[1]` (即 30)。输出：`30`。 栈: `[10]`
6.  `push 40`: `s[1] = 40; t = 2;` 栈: `[10, 40]`
7.  `push 50`: `s[2] = 50; t = 3;` 栈: `[10, 40, 50]`
8.  `pop`: `t = 2;` 弹出 `s[2]` (即 50)。输出：`50`。 栈: `[10, 40]`
9.  `push 60`: `s[2] = 60; t = 3;` 栈: `[10, 40, 60]`
10. `pop`: `t = 2;` 弹出 `s[2]` (即 60)。输出：`60`。 栈: `[10, 40]`
11. `pop`: `t = 1;` 弹出 `s[1]` (即 40)。输出：`40`。 栈: `[10]`
12. `empty`: `t` 是 1，不等于 0，所以栈不为空。输出：`false`。
13. `size`: `t` 是 1，栈里有 1 个元素。输出：`1`。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含常用的库

const int MAXN = 5; // 栈的最大容量

int s[MAXN]; // 存放栈元素的数组
int t = 0;   // 栈顶指针，指示下一个空位置，也代表栈中元素个数

// 入栈操作
void push(int x) {
    if (t < MAXN) { // 检查栈是否已满
        s[t] = x;   // 将元素x放入栈顶
        t++;        // 栈顶指针上移
        std::cout << "Push " << x << " successful." << std::endl;
    } else {
        std::cout << "Error: Stack is full, cannot push " << x << std::endl;
    }
}

// 出栈操作
int pop() {
    if (t > 0) { // 检查栈是否为空
        t--;     // 栈顶指针下移
        int x = s[t]; // 获取栈顶元素
        std::cout << "Pop " << x << std::endl;
        return x;
    } else {
        std::cout << "Error: Stack is empty, cannot pop." << std::endl;
        return -1; // 返回一个特殊值表示错误
    }
}

// 获取栈顶元素
int top() {
    if (t > 0) { // 检查栈是否为空
        return s[t - 1]; // 返回栈顶元素
    } else {
        std::cout << "Error: Stack is empty, no top element." << std::endl;
        return -1; // 返回特殊值表示错误
    }
}

// 判断栈是否为空
bool empty() {
    return t == 0;
}

// 获取栈中元素个数
int size() {
    return t;
}

int main() {
    std::cout << "--- Simulating Stack Operations ---" << std::endl;

    push(10);
    push(20);
    pop();
    push(30);
    pop();
    push(40);
    push(50);
    pop();
    push(60);
    pop();
    pop();

    std::cout << "Is stack empty? " << (empty() ? "Yes" : "No") << std::endl;
    std::cout << "Stack size: " << size() << std::endl;
    if (!empty()) {
        std::cout << "Top element: " << top() << std::endl;
    }

    // 尝试栈满和栈空操作
    push(70);
    push(80);
    push(90); // 应该会显示栈满错误
    pop();
    pop();
    pop();
    pop(); // 应该会显示栈空错误

    return 0;
}
```

**复杂度分析**：
栈的 `push`, `pop`, `top`, `empty`, `size` 这些操作都只需要常数时间，也就是 O(1)。无论栈里有多少元素，这些操作都只需要固定的几步，非常高效！

### 例题 2：模拟队列操作

**题目描述**：
假设我们有一个最多能放 5 个整数的队列。请模拟以下操作序列，并输出每次 `pop` 操作出队的元素。
操作序列：
1.  `push 100`
2.  `push 200`
3.  `pop`
4.  `push 300`
5.  `pop`
6.  `push 400`
7.  `push 500`
8.  `pop`
9.  `push 600`
10. `pop`
11. `pop`
12. `empty` (判断队列是否为空)
13. `size` (获取队列中元素个数)

**思路分析**：
我们用一个数组 `q[5]` 和两个指针 `h` (队头) 和 `t` (队尾) 来模拟。`h` 和 `t` 初始都为 0。

1.  `push 100`: `q[0] = 100; t = 1;` 队列: `[100]`
2.  `push 200`: `q[1] = 200; t = 2;` 队列: `[100, 200]`
3.  `pop`: `h = 1;` 弹出 `q[0]` (即 100)。输出：`100`。 队列: `[200]` (逻辑上，`q[0]`的空间被浪费)
4.  `push 300`: `q[2] = 300; t = 3;` 队列: `[200, 300]`
5.  `pop`: `h = 2;` 弹出 `q[1]` (即 200)。输出：`200`。 队列: `[300]`
6.  `push 400`: `q[3] = 400; t = 4;` 队列: `[300, 400]`
7.  `push 500`: `q[4] = 500; t = 5;` 队列: `[300, 400, 500]`
8.  `pop`: `h = 3;` 弹出 `q[2]` (即 300)。输出：`300`。 队列: `[400, 500]`
9.  `push 600`: **注意！** `t` 已经等于 `MAXN` (5) 了，队列已满！不能再 `push`。
10. `pop`: `h = 4;` 弹出 `q[3]` (即 400)。输出：`400`。 队列: `[500]`
11. `pop`: `h = 5;` 弹出 `q[4]` (即 500)。输出：`500`。 队列: `[]`
12. `empty`: `h` 是 5，`t` 也是 5，`h == t`，所以队列为空。输出：`true`。
13. `size`: `t - h` 是 `5 - 5 = 0`，队列里有 0 个元素。输出：`0`。

**代码实现**：

```cpp
#include <bits/stdc++.h> // 包含常用的库

const int MAXN = 5; // 队列的最大容量

int q[MAXN]; // 存放队列元素的数组
int h = 0;   // 队头指针，指示队头元素
int t = 0;   // 队尾指针，指示下一个空位置

// 入队操作
void push(int x) {
    if (t < MAXN) { // 检查队列是否已满
        q[t] = x;   // 将元素x放入队尾
        t++;        // 队尾指针后移
        std::cout << "Push " << x << " successful." << std::endl;
    } else {
        std::cout << "Error: Queue is full, cannot push " << x << std::endl;
    }
}

// 出队操作
int pop() {
    if (h < t) { // 检查队列是否为空 (h == t 表示空)
        int x = q[h]; // 获取队头元素
        h++;          // 队头指针后移
        std::cout << "Pop " << x << std::endl;
        return x;
    } else {
        std::cout << "Error: Queue is empty, cannot pop." << std::endl;
        return -1; // 返回一个特殊值表示错误
    }
}

// 获取队头元素
int front() {
    if (h < t) { // 检查队列是否为空
        return q[h]; // 返回队头元素
    } else {
        std::cout << "Error: Queue is empty, no front element." << std::endl;
        return -1; // 返回特殊值表示错误
    }
}

// 判断队列是否为空
bool empty() {
    return h == t;
}

// 获取队列中元素个数
int size() {
    return t - h;
}

int main() {
    std::cout << "--- Simulating Queue Operations ---" << std::endl;

    push(100);
    push(200);
    pop();
    push(300);
    pop();
    push(400);
    push(500);
    pop();
    push(600); // 此时队列已满，会报错
    pop();
    pop();

    std::cout << "Is queue empty? " << (empty() ? "Yes" : "No") << std::endl;
    std::cout << "Queue size: " << size() << std::endl;
    if (!empty()) {
        std::cout << "Front element: " << front() << std::endl;
    }

    // 尝试队列满和队列空操作
    push(700);
    push(800);
    push(900);
    push(1000); // 应该会显示队列满错误
    pop();
    pop();
    pop();
    pop(); // 应该会显示队列空错误

    return 0;
}
```

**复杂度分析**：
队列的 `push`, `pop`, `front`, `empty`, `size` 这些操作也都只需要常数时间，也就是 O(1)。和栈一样，无论队列里有多少元素，这些操作都只需要固定的几步，非常高效！

```cpp
#include <bits/stdc++.h> // 包含常用的库，如iostream用于输入输出

const int MAXN = 105; // 定义队列的最大容量，根据题目要求调整

int q[MAXN]; // 声明一个整数数组q，用于存储队列中的元素
int h = 0;   // 声明一个整数h，作为队头指针 (head)。
             // h指向队列的第一个元素。初始值为0。
int t = 0;   // 声明一个整数t，作为队尾指针 (tail)。
             // t指向下一个可以放入元素的位置。初始值为0。
             // 当h == t时，队列为空。

// ======================= 队列的基本操作函数 =======================

// 1. 入队操作 (Push)
// 将元素x放入队列尾部
void push(int x) {
    if (t < MAXN) { // 检查队列是否已满。如果t等于MAXN，说明数组已满。
        q[t] = x;   // 将元素x放入当前t指向的位置
        t++;        // 队尾指针向后移动一位，指向下一个空位置
        // std::cout << "Element " << x << " pushed into queue." << std::endl;
    } else {
        // std::cout << "Error: Queue is full, cannot push " << x << "." << std::endl;
        // 实际比赛中，通常不会有输出，而是根据题目要求处理错误或直接忽略
    }
}

// 2. 出队操作 (Pop)
// 移除队头元素，并返回该元素的值
int pop() {
    if (h < t) { // 检查队列是否为空。如果h等于t，说明队列是空的。
        // std::cout << "Element " << q[h] << " popped from queue." << std::endl;
        int x = q[h]; // 获取队头元素的值
        h++;          // 队头指针向后移动一位，表示移除当前队头元素
        return x;     // 返回被移除的队头元素的值
    } else {
        // std::cout << "Error: Queue is empty, cannot pop." << std::endl;
        return -1; // 返回一个特殊值（如-1），表示出错或队列空
    }
}

// 3. 获取队头元素 (Front)
// 返回队头元素的值，但不移除它
int front() {
    if (h < t) { // 检查队列是否为空
        return q[h]; // 队头元素就是q[h]
    } else {
        // std::cout << "Error: Queue is empty, no front element." << std::endl;
        return -1; // 返回特殊值表示出错或队列空
    }
}

// 4. 判断队列是否为空 (Empty)
// 如果队列为空，返回true；否则返回false
bool empty() {
    return h == t; // 如果h等于t，说明队列中没有元素
}

// 5. 获取队列中元素个数 (Size)
// 返回队列中当前元素的数量
int size() {
    return t - h; // 队列中元素的个数是队尾指针减去队头指针
}
```

1.  **括号匹配检查器**：
    ```cpp
    #include <bits/stdc++.h>

    const int MAXN = 105;
    char s[MAXN]; // 字符栈
    int t = 0;

    void push(char c) {
        if (t < MAXN) s[t++] = c;
    }
    char pop() {
        if (t > 0) return s[--t];
        return '\0'; // 表示错误或空
    }
    bool empty() {
        return t == 0;
    }

    int main() {
        std::string str;
        std::cin >> str;
        bool ok = true;
        for (char c : str) {
            if (c == '(') {
                push(c);
            } else if (c == ')') {
                if (empty()) { // 栈空但遇到右括号，不匹配
                    ok = false;
                    break;
                }
                pop(); // 匹配成功，出栈
            }
        }
        if (!empty()) { // 循环结束后栈不空，说明左括号多了
            ok = false;
        }

        if (ok) {
            std::cout << "匹配" << std::endl;
        } else {
            std::cout << "不匹配" << std::endl;
        }
    ```

**拓展挑战**

1.  **思考题**：
    *   **问题**：假溢出。当队头 `h` 不断向前移动，即使数组前面有很多空闲空间，如果队尾 `t` 达到了数组的最大下标 `MAXN-1`，队列就会被判定为“满”，无法再入队，但实际上还有很多空间未被利用。
    *   **解决办法**：**循环队列**。可以将数组看作一个环形结构。当 `t` 到达 `MAXN-1` 后，如果 `q[0]` 还是空的，`t` 可以回到 `0` 位置继续入队。`h` 和 `t` 都需要对 `MAXN` 取模 (`% MAXN`) 来实现循环。判断满和空的条件会稍微复杂一点。
2.  **双栈模拟队列**：
    *   思路：
        *   `push(x)`: 直接将 `x` 压入 `stack_in`。
        *   `pop()`:
            *   如果 `stack_out` 不为空，直接从 `stack_out` 弹出。
            *   如果 `stack_out` 为空，将 `stack_in` 中的所有元素依次弹出并压入 `stack_out`。然后从 `stack_out` 弹出。
            *   如果 `stack_in` 和 `stack_out` 都为空，则队列为空。
        *   `front()`: 类似 `pop()` 的逻辑，只是不弹出。
        *   `empty
