1), 正整数 319 和 377 的最大公约数是（ ）
{{ select(1) }}
- A. 27
- B. 33
- C. 29
- D. 31

> **答案**: C
> **解析**: 求两个正整数的最大公约数，最常用的方法是“辗转相除法”（也称欧几里得算法）。

计算步骤如下：
1. 用较大的数除以较小的数：377 ÷ 319 = 1，余数是 58（377 - 319 = 58）。
2. 用上一步的除数除以上一步的余数：319 ÷ 58 = 5，余数是 29（58 × 5 = 290，319 - 290 = 29）。
3. 继续用上一步的除数除以上一步的余数：58 ÷ 29 = 2，余数是 0。
4. 当余数为 0 时，最后的除数（即 29）就是这两个数的最大公约数。

我们也可以通过质因数分解来验证：
- 319 = 11 × 29
- 377 = 13 × 29
可以看到，两个数唯一的公因数（且是质数）是 29，因此最大公约数是 29。

对比选项：
A. 27：319 不能被 27 整除。
B. 33：319 不能被 33 整除。
C. 29：319 ÷ 29 = 11，377 ÷ 29 = 13，正确。
D. 31：319 不能被 31 整除。

因此，正确答案是 C。

2), 与十进制数 1770 对应的八进制数是（ ）
{{ select(2) }}
- A. 3350
- B. 3351
- C. 3352
- D. 3540

> **答案**: C
> **解析**: 将十进制数转换为八进制数，通常采用“除8取余法”，即将十进制数不断除以8，记录每次得到的余数，直到商为0为止，最后将余数逆序排列。

具体计算步骤如下：
1. 1770 ÷ 8 = 221，余数为 2；
2. 221 ÷ 8 = 27，余数为 5；
3. 27 ÷ 8 = 3，余数为 3；
4. 3 ÷ 8 = 0，余数为 3。

将余数从最后一次得到的开始向上排列，得到 3352。

验证计算：
3 × 8³ + 3 × 8² + 5 × 8¹ + 2 × 8⁰
= 3 × 512 + 3 × 64 + 5 × 8 + 2 × 1
= 1536 + 192 + 40 + 2
= 1770。

因此，十进制数 1770 对应的八进制数是 3352。正确答案为 C。

{{ select(3) }}

- B. 用十六进制数表示的结果为 2084
- C. 用二进制数表示的结果为 100000000110

4), 在 C++ 语言中，按位或运算符（|）是指参加运算的两个数，按二进制位进行“或”运算，若两个数对应位上有一个为 1，则该位上或运算的结果为 1，否则为 0。异或运算符（^）是指参加运算的两个数，按二进制位进行“异或”运算，若两个数对应位不同，则该位结果异或运算的结果为 1，否则为 0。请计算表达式(23|2)^5的值是（ ）
{{ select(4) }}
- A. 18
- B. 1
- C. 23
- D. 32

> **答案**: A
> **解析**: 本题考查 C++ 中的位运算，包括按位或（|）和按位异或（^）。计算步骤如下：

1. 首先计算括号内的表达式 (23 | 2)：
   - 将 23 转换为二进制：23 = 16 + 4 + 2 + 1，即二进制为 10111。
   - 将 2 转换为二进制：2 = 2，即二进制为 00010。
   - 进行按位或运算（对应位只要有一个为 1，结果即为 1）：
     10111 (23)
   | 00010 (2)
   -----------
     10111 (结果仍为 23)

2. 接着计算第一步的结果与 5 进行异或运算 (23 ^ 5)：
   - 23 的二进制为 10111。
   - 将 5 转换为二进制：5 = 4 + 1，即二进制为 00101。
   - 进行按位异或运算（对应位不同为 1，相同为 0）：
     10111 (23)
   ^ 00101 (5)
   -----------
     10010

3. 将二进制结果 10010 转换为十进制：
   - 1 * 2^4 + 0 * 2^3 + 0 * 2^2 + 1 * 2^1 + 0 * 2^0
   - 16 + 0 + 0 + 2 + 0 = 18。

因此，表达式 (23|2)^5 的值为 18。正确答案为 A。

5), 在 C++ 语言中，三种逻辑运算符运算（注意这里不是按位运算）规则如下：
| 运算符 | 示例 | 运算规则 |
| --- | --- | --- |
| 逻辑与 (&&) | a && b | 如果 a 和 b 都为真，结果为真，否则为假 |
| 逻辑或 (||) | a || b | 如果 a 和 b 有一个为真，则结果为真，二者都假，结果为假 |
| 逻辑非 (!) | !a | 如果 a 为假!a 为真，如果 a 为真!a 为假 |
判断 a 等于 0 或 b 等于 0 或 c 等于 0 的正确的条件表达式是（ ）
{{ select(5) }}
- A. !((a!=0) || (b!=0))
- B. !((a!=0) && (b!=0) && (c!=0))
- C. !((a!=0 && b!=0) || (c!=0))
- D. (a==0) && (b==0) && (c==0)

> **答案**: B
> **解析**: 题目要求判断的条件是“a 等于 0 或 b 等于 0 或 c 等于 0”，用 C++ 逻辑表达式直接表示为：(a == 0) || (b == 0) || (c == 0)。

我们可以利用德·摩根定律（De Morgan's laws）来分析各个选项：
德·摩根定律指出：!(P || Q) 等价于 !P && !Q；!(P && Q) 等价于 !P || !Q。

A 选项：!((a!=0) || (b!=0)) 等价于 !(a!=0) && !(b!=0)，即 (a==0) && (b==0)。这表示 a 和 b 同时为 0，且没有涉及 c，不符合题意。

B 选项：!((a!=0) && (b!=0) && (c!=0))。根据德·摩根定律，这等价于 !(a!=0) || !(b!=0) || !(c!=0)，即 (a==0) || (b==0) || (c==0)。这完全符合题目要求的“a 等于 0 或 b 等于 0 或 c 等于 0”，因此是正确答案。

C 选项：!((a!=0 && b!=0) || (c!=0))。根据定律，这等价于 !(a!=0 && b!=0) && !(c!=0)，进一步展开为 ((a==0) || (b==0)) && (c==0)。这表示 a 或 b 其中一个为 0，且 c 必须为 0，不符合题意。

D 选项：(a==0) && (b==0) && (c==0)。这表示 a、b、c 三者必须同时为 0，而题目要求的是“或”的关系（只要有一个为 0 即可），因此不符合题意。

综上所述，正确答案是 B。

3. 计算方法：对于一个长度为 n 且字符互不相同的字符串，其非空子串的数目可以通过以下方式计算：
   - 长度为 1 的子串有 7 个：'O', 'l', 'y', 'm', 'p', 'i', 'c'
   - 长度为 2 的子串有 6 个：'Ol', 'ly', 'ym', 'mp', 'pi', 'ic'

7), 将数组{8,23,4,16,77,−5,53,100}中的元素按从大到小的顺序排列，每次可以交换任意两个元素，最少需要交换（ ）次。
{{ select(7) }}
- A. 4
- B. 5
- C. 6
- D. 7

> **答案**: B
> **解析**: 这是一个关于数组排序和置换环（Permutation Cycles）的经典算法问题。题目要求将数组按从大到小排序，并求最少交换次数。

1. 首先，确定目标状态（从大到小排序）：
   原始数组：{8, 23, 4, 16, 77, -5, 53, 100}
   目标数组：{100, 77, 53, 23, 16, 8, 4, -5}

2. 找出每个元素当前位置与目标位置的关系，形成置换环：
   - 元素 8：当前在索引 0，目标位置是索引 5（值为 8）。
   - 元素 -5：当前在索引 5，目标位置是索引 7（值为 -5）。
   - 元素 100：当前在索引 7，目标位置是索引 0（值为 100）。
   形成环 1：(0 -> 5 -> 7 -> 0)，环长度为 3。

   - 元素 23：当前在索引 1，目标位置是索引 3（值为 23）。
   - 元素 16：当前在索引 3，目标位置是索引 4（值为 16）。
   - 元素 77：当前在索引 4，目标位置是索引 1（值为 77）。
   形成环 2：(1 -> 3 -> 4 -> 1)，环长度为 3。

   - 元素 4：当前在索引 2，目标位置是索引 6（值为 4）。
   - 元素 53：当前在索引 6，目标位置是索引 2（值为 53）。
   形成环 3：(2 -> 6 -> 2)，环长度为 2。

3. 计算最少交换次数：
   对于一个长度为 $L$ 的置换环，将其中的元素归位最少需要 $L - 1$ 次交换。
   总的最少交换次数 = (每个环的长度 - 1) 之和，或者公式：$n - k$（其中 $n$ 是元素总数，$k$ 是环的个数）。
   本题中：
   - 环 1 需要 $3 - 1 = 2$ 次交换。
   - 环 2 需要 $3 - 1 = 2$ 次交换。
   - 环 3 需要 $2 - 1 = 1$ 次交换。
   总交换次数 = $2 + 2 + 1 = 5$ 次。

因此，最少需要交换 5 次。

8), 书架上有 21 本书，编号从 1 到 21，从其中选 4 本，其中每两本的编号都不相邻的选法一共有（ ）种。
{{ select(8) }}
- A. 1530
- B. 3020
- C. 3060
- D. 4096

> **答案**: C
> **解析**: 这是一道经典的组合数学问题，属于“不相邻组合”模型。

1. 问题建模：
题目要求从 21 本书中选出 4 本，且这 4 本书的编号互不相邻。我们可以将这个问题转化为“插空法”模型。

2. 插空法思路：
假设我们先将没有被选中的书排成一排。
- 总共有 21 本书，选出 4 本，则剩下 21 - 4 = 17 本书。
- 这 17 本书排列好后，它们之间以及两端共有 17 + 1 = 18 个空位。
- 为了保证选出的 4 本书互不相邻，我们只需要从这 18 个空位中随机挑选 4 个位置，将选出的书插入即可。

3. 计算过程：
从 18 个空位中选出 4 个位置的组合数为 C(18, 4)。
计算公式为：
C(18, 4) = (18 × 17 × 16 × 15) / (4 × 3 × 2 × 1)
C(18, 4) = 3060

4. 通项公式：
从 n 个编号连续的元素中取 k 个互不相邻的元素，其组合数为 C(n - k + 1, k)。
本题中 n = 21, k = 4，代入公式得：
C(21 - 4 + 1, 4) = C(18, 4) = 3060。

因此，正确答案为 C。

9), 队列快照问题：队列快照是指在某一时刻队列中的元素组成的有序序列。例如，当元素1,2,3入队，元素 1 出队后，此刻的队列快照是{2,3}。当元素2,3也出队后，队列快照是{}，即为空。现有 3 个正整数元素依次入队、出队。已知它们的和为 8，则共有（ ）种可能的不同的队列快照（不同队列的相同快照只计一次）。例如，{5,1}，{4,2,2}，{}都是可能的队列快照；而{7}不是可能的队列快照，因为剩下的 2 个正整数的和不可能是 1。
{{ select(9) }}
- A. 36
- B. 42
- C. 49
- D. 58

> **答案**: C
> **解析**: 1. **分析队列操作与可能的快照形式**：
   设三个正整数依次为 $a, b, c$。根据队列的先进先出（FIFO）特性，元素入队的顺序是 $a, b, c$，出队的顺序也必须是 $a, b, c$。
   在整个入队和出队的过程中，队列在某一时刻的快照（即队列中的元素序列）只能是以下几种形式：
   - 空序列：$\emptyset$
   - 长度为 1 的序列：$\{a\}, \{b\}, \{c\}$
   - 长度为 2 的序列：$\{a, b\}, \{b, c\}$
   - 长度为 3 的序列：$\{a, b, c\}$
   注意，像 $\{a, c\}$ 这样的序列是不可能的，因为 $b$ 在 $a$ 之后入队，在 $a$ 出队前 $b$ 必须在队列中；且 $a$ 必须在 $b$ 之前出队。

2. **计算不同长度的可能快照数量**：
   已知 $a, b, c$ 为正整数（即 $\ge 1$）且 $a + b + c = 8$。
   - **长度为 0**：只有一种快照 $\emptyset$。数量为 1。
   - **长度为 1**：快照可以是 $\{a\}, \{b\}, \{c\}$。
     由于 $a+b+c=8$ 且 $b, c \ge 1$，则 $a$ 的取值范围是 $1 \le a \le 8-1-1=6$。
     同理，$b$ 和 $c$ 的取值范围也是 $1$ 到 $6$。
     因此，所有可能的长度为 1 的快照集合是 $\{\{1\}, \{2\}, \{3\}, \{4\}, \{5\}, \{6\}\}$。数量为 6。
   - **长度为 2**：快照可以是 $\{a, b\}$ 或 $\{b, c\}$。
     对于 $\{a, b\}$，由于 $a+b+c=8$ 且 $c \ge 1$，则 $a+b \le 7$。同时 $a, b \ge 1$。
     满足 $a+b \le 7$ 的正整数对 $(a, b)$ 数量为：
     当 $a+b=2$ 时，有 1 种；$a+b=3$ 时，有 2 种；...；$a+b=7$ 时，有 6 种。
     总数为 $1+2+3+4+5+6 = 21$。
     对于 $\{b, c\}$，同理 $b+c \le 7$，满足条件的正整数对 $(b, c)$ 也是这 21 种。
     因此，所有可能的长度为 2 的快照共有 21 种。
   - **长度为 3**：快照只能是 $\{a, b, c\}$。
     满足 $a+b+c=8$ 且 $a, b, c \ge 1$ 的正整数三元组数量，根据隔板法为 $\binom{8-1}{3-1} = \binom{7}{2} = 21$。
     因此，所有可能的长度为 3 的快照共有 21 种。

3. **汇总结果**：
   不同长度的快照互不相同，将各长度的数量相加：
   $1 (\text{长度0}) + 6 (\text{长度1}) + 21 (\text{长度2}) + 21 (\text{长度3}) = 49$。

所以，共有 49 种可能的不同队列快照。

10), 定义字符串的基本操作为：删除一个字符、插入一个字符和将一个字符修改成另外三个字符这三种操作。将字符串 A 变成字符串 B 的最少操作步数，称为字符串 A 到字符串 B 的编辑距离。字符串 ABCDEFG 到字符串 BADECG 的编辑距离为（ ）
{{ select(10) }}
- A. 3
- B. 4
- C. 6
- D. 7

> **答案**: A
> **解析**: 编辑距离（Edit Distance），也称为 Levenshtein 距离，是指两个字符串之间，由一个转成另一个所需的最少编辑操作次数。允许的操作包括：删除一个字符、插入一个字符、将一个字符替换为另一个字符。

在本题中，我们需要将字符串 A = "ABCDEFG" 转换为字符串 B = "BADECG"。

我们可以通过以下 3 步操作完成转换：
1. 删除字符 'A'：
   "ABCDEFG" -> "BCDEFG" （操作 1：删除）
2. 将字符 'C' 替换为 'A'：
   "BCDEFG" -> "BADEFG" （操作 2：替换）
   此时字符串变为 "BADEFG"，其中 'B' 在第 1 位，'A' 在第 2 位，'D'、'E' 紧随其后。
3. 将字符 'F' 替换为 'C'：
   "BADEFG" -> "BADECG" （操作 3：替换）
   此时字符串变为 "BADECG"，与目标字符串一致。

通过这 3 步，我们成功将 A 转换为了 B。

我们也可以通过动态规划（Dynamic Programming）的方法来验证：
设字符串 A 长度为 7，字符串 B 长度为 6。
它们的公共子序列（LCS）包括 "BDEG" 或 "ADEG"，长度均为 4。
根据编辑距离的定义和动态规划状态转移方程计算，得出最少操作步数为 3。

具体步骤验证：
A: A B C D E F G
1. 删 A -> B C D E F G
2. C变A -> B A D E F G
3. F变C -> B A D E C G (完成)

因此，字符串 ABCDEFG 到 BADECG 的编辑距离为 3。

11), 在数组A[x]中，若存在(i<j)∧(A[i]>A[j])，则称(A[i],A[j])为数组A[x]的一个逆序对。对于序列7,5,1,9,3,6,8,4，在不改变顺序的情况下，去掉（ ）会使逆序对的个数减少 3。
{{ select(11) }}
- A. 7
- B. 5
- C. 3
- D. 8

> **答案**: C
> **解析**: 首先，我们需要理解逆序对的定义：在序列中，如果前面的数大于后面的数，则这两个数构成一个逆序对。
当从序列中去掉一个数 $X$ 时，减少的逆序对个数等于：
1. 在 $X$ 之前且比 $X$ 大的数的个数（这些数与 $X$ 构成了逆序对，且 $X$ 是后项）。
2. 在 $X$ 之后且比 $X$ 小的数的个数（这些数与 $X$ 构成了逆序对，且 $X$ 是前项）。

序列为：7, 5, 1, 9, 3, 6, 8, 4

我们逐一分析选项：
A. 去掉 7：
- 7 之前没有数。
- 7 之后比 7 小的数有：5, 1, 3, 6, 4（共 5 个）。
- 减少逆序对个数：0 + 5 = 5。

B. 去掉 5：
- 5 之前比 5 大的数有：7（共 1 个）。
- 5 之后比 5 小的数有：1, 3, 4（共 3 个）。

- A. 1200
- B. 2400
- C. 2880
- D. 14400

> **答案**: C
> **解析**: 这是一个典型的圆桌排列（Circular Permutation）组合问题，且带有“相隔就坐”的约束条件。

1. **圆桌排列基础**：在圆桌排列中，由于旋转对称性，n 个不同元素的排列方式为 (n-1)! 种。

2. **分步计算方案数**：
   - **第一步：安排第一组选手（5 名大陆选手）**。
     将 5 名大陆选手安排在圆桌上，根据圆桌排列公式，其排列方式为 (5-1)! = 4! = 24 种。

     当 5 名大陆选手坐定后，他们之间形成了 5 个空隙。由于大陆选手的位置已经固定，此时圆桌的旋转对称性已经打破，这 5 个空隙可以看作是 5 个绝对的、不同的位置。

3. **计算总数**：
   根据乘法原理，总的就坐方案数 = 大陆选手的排列数 × 港澳选手的排列数 = 24 × 120 = 2880 种。

- 如果是直线排列，5 名大陆选手和 5 名港澳选手相隔就坐，有两种模式（大港大港... 或 港大港大...），每种模式有 5! × 5! 种排法，总计 2 × 5! × 5! = 28800 种。
- 在圆桌排列中，10 个人的排列会因为旋转产生 10 种重复的情况。
- 因此，圆桌排列数 = 28800 ÷ 10 = 2880 种。

17), 在不超过 10000 的正整数中，与 10000 互质的正整数有（ ）个。
{{ select(17) }}
- A. 2000
- B. 4000
- C. 6000
- D. 8000

> **答案**: B
> **解析**: 本题考查的是数论中的欧拉函数（Euler's totient function）概念。题目要求计算在 1 到 10000 之间与 10000 互质的整数个数。

1. **理解互质的概念**：两个数互质意味着它们的最大公约数为 1。对于 10000 来说，一个数与其互质，当且仅当这个数不包含 10000 的任何质因数。

2. **质因数分解**：
   首先将 10000 进行质因数分解：
   $10000 = 10^4 = (2 \times 5)^4 = 2^4 \times 5^4$
   由此可见，10000 的质因数只有 2 和 5。

3. **使用欧拉函数公式**：
   欧拉函数 $\phi(n)$ 表示小于或等于 $n$ 的正整数中与 $n$ 互质的数的数目。
   公式为：$\phi(n) = n \times (1 - 1/p_1) \times (1 - 1/p_2) \times \dots \times (1 - 1/p_k)$，其中 $p_i$ 是 $n$ 的所有不同质因数。
   代入本题数据：
   $\phi(10000) = 10000 \times (1 - 1/2) \times (1 - 1/5)$
   $\phi(10000) = 10000 \times (1/2) \times (4/5)$
   $\phi(10000) = 10000 \times (4/10)$
   $\phi(10000) = 4000$

4. **使用容斥原理验证**：
   - 1 到 10000 中，2 的倍数有 $10000 / 2 = 5000$ 个。
   - 1 到 10000 中，5 的倍数有 $10000 / 5 = 2000$ 个。
   - 1 到 10000 中，既是 2 又是 5 的倍数（即 10 的倍数）有 $10000 / 10 = 1000$ 个。
   - 根据容斥原理，是 2 或 5 的倍数的数共有 $5000 + 2000 - 1000 = 6000$ 个。
   - 与 10000 互质的数，即不是 2 的倍数也不是 5 的倍数的数，共有 $10000 - 6000 = 4000$ 个。

综上所述，正确答案为 B。

23), 由1,1,2,2,3这五个数字组成不同的三位数有（ ）种。
{{ select(23) }}
- A. 18
- B. 15
- C. 12
- D. 24

> **答案**: A
> **解析**: 这道题可以通过分类讨论的方法来解决。题目给出的数字是 {1, 1, 2, 2, 3}，我们需要从中选出三个数字组成三位数。

我们可以根据选出的数字组合进行分类：

1. 三个数字互不相同：
   可选的数字集合只能是 {1, 2, 3}。
   这三个数字的全排列数为：3! = 3 × 2 × 1 = 6 种。
   （具体为：123, 132, 213, 231, 312, 321）

2. 有两个数字相同（即“两同一异”）：
   相同的数字可以是 1 或 2。
   - 如果相同的数字是 {1, 1}：
     第三个数字可以从 {2, 3} 中选一个。
     - 选 2 时，数字组合为 {1, 1, 2}，排列数为 3!/2! = 3 种（112, 121, 211）。
     - 选 3 时，数字组合为 {1, 1, 3}，排列数为 3!/2! = 3 种（113, 131, 311）。
     共 3 + 3 = 6 种。
   - 如果相同的数字是 {2, 2}：
     第三个数字可以从 {1, 3} 中选一个。
     - 选 1 时，数字组合为 {2, 2, 1}，排列数为 3!/2! = 3 种（221, 212, 122）。
     - 选 3 时，数字组合为 {2, 2, 3}，排列数为 3!/2! = 3 种（223, 232, 322）。
     共 3 + 3 = 6 种。

3. 三个数字都相同：
   由于没有任何数字出现了三次，所以这种情况不存在。

综合以上所有情况，总共的不同三位数为：6 + 6 + 6 = 18 种。
因此，正确答案是 A。

25), 假设有 n 根柱子，需要按照以下规则依次放置编号为1,2,3,…的圆环：每根柱子的底部固定，顶部可以放入圆环；每次从柱子顶部放入圆环时，需要保证任何两个相邻圆环的编号之和是一个完全平方数。请计算当有 4 根柱子时，最多可以放置（ ）个圆环。
{{ select(25) }}
- A. 7
- B. 9
- C. 11
- D. 5

> **答案**: C
> **解析**: 这是一个经典的数学逻辑题，可以通过模拟放置过程来得出结论。我们需要按照 1, 2, 3, ... 的顺序将圆环放入 4 根柱子中，规则是：同一根柱子上相邻的两个圆环编号之和必须是完全平方数（1, 4, 9, 16, 25...）。

具体模拟步骤如下：
1. 放置 1 号环：放入第 1 根柱子。此时柱 1：[1]。
2. 放置 2 号环：1+2=3（非平方数），放入第 2 根柱子。此时柱 2：[2]。
3. 放置 3 号环：1+3=4（平方数），放入第 1 根柱子。此时柱 1：[1, 3]。
4. 放置 4 号环：3+4=7，2+4=6，均非平方数，放入第 3 根柱子。此时柱 3：[4]。
5. 放置 5 号环：4+5=9（平方数），放入第 3 根柱子。此时柱 3：[4, 5]。
6. 放置 6 号环：3+6=9（平方数），放入第 1 根柱子。此时柱 1：[1, 3, 6]。
7. 放置 7 号环：2+7=9（平方数），放入第 2 根柱子。此时柱 2：[2, 7]。
8. 放置 8 号环：6+8=14，7+8=15，5+8=13，均非平方数，放入第 4 根柱子。此时柱 4：[8]。
9. 放置 9 号环：7+9=16（平方数），放入第 2 根柱子。此时柱 2：[2, 7, 9]。
10. 放置 10 号环：6+10=16（平方数），放入第 1 根柱子。此时柱 1：[1, 3, 6, 10]。
11. 放置 11 号环：5+11=16（平方数），放入第 3 根柱子。此时柱 3：[4, 5, 11]。
12. 尝试放置 12 号环：
    - 与柱 1 顶端：10+12=22（否）
    - 与柱 2 顶端：9+12=21（否）
    - 与柱 3 顶端：11+12=23（否）
    - 与柱 4 顶端：8+12=20（否）
    由于 4 根柱子都无法放入 12 号环，且不能再开新柱子，因此最多可以放置 11 个圆环。

该问题的通用结论是：当有 n 根柱子时，最多可以放置的圆环数为 $f(n)$，其规律为 $f(n) = f(n-1) + (n + (n\%2==1?1:0))$，即增加的个数序列为 2, 4, 4, 6, 6, 8, 8...。
n=1 时为 1；
n=2 时为 1+2=3；
n=3 时为 3+4=7；
n=4 时为 7+4=11。
