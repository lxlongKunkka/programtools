1), ## 一、单项选择题（共 15 题，每题 2 分，共计 30 分；每题有且仅有一个正确选项）
以下哪种功能没有涉及 C++ 语言的面向对象特性支持：（ ）
{{ select(1) }}
- A. C++ 中调用 printf 函数
- B. C++ 中调用用户定义的类成员函数
- C. C++ 中构造一个 class 或 struct
- D. C++ 中构造来源于同一基类的多个派生类

> **答案**: A
> **解析**: 面向对象程序设计（Object-Oriented Programming, OOP）的核心特性包括封装、继承和多态。

A. printf 函数是 C 语言标准库中的一个函数，属于过程式编程（Procedural Programming）的范畴。在 C++ 中调用它仅仅是调用一个普通的全局函数，不涉及类、对象、继承或多态等面向对象特性。

B. 调用用户定义的类成员函数涉及“封装”特性，因为函数是作为类的一部分存在的，必须通过对象或类来访问。

C. 构造 class 或 struct 是面向对象编程的基础。在 C++ 中，class 和 struct（除了默认访问权限外）都是用于定义类和创建对象的工具。

D. 构造来源于同一基类的多个派生类直接涉及面向对象中的“继承”特性。

因此，只有 A 选项没有涉及面向对象特性。

2), 有 6 个元素，按照6,5,4,3,2,1的顺序进入栈 S，请问下列哪个出栈序列是非法的（ ）
{{ select(2) }}
- A. 5 , 4 , 3 , 6 , 1 , 2 5,4,3,6,1,2
- B. 4 , 5 , 3 , 1 , 2 , 6 4,5,3,1,2,6
- C. 3 , 4 , 6 , 5 , 2 , 1 3,4,6,5,2,1
- D. 2 , 3 , 4 , 1 , 5 , 6 2,3,4,1,5,6

> **答案**: C
> **解析**: 栈是一种“后进先出”（LIFO, Last In First Out）的数据结构。在本题中，入栈顺序固定为 6, 5, 4, 3, 2, 1。我们逐一分析选项：

A 选项 (5, 4, 3, 6, 1, 2)：
1. 6, 5 入栈，5 出栈（栈内：6）；
2. 4 入栈，4 出栈（栈内：6）；
3. 3 入栈，3 出栈（栈内：6）；
4. 6 出栈（栈内：空）；
5. 2, 1 入栈，1 出栈，2 出栈。
结论：合法。

B 选项 (4, 5, 3, 1, 2, 6)：
1. 6, 5, 4 入栈，4 出栈，5 出栈（栈内：6）；
2. 3 入栈，3 出栈（栈内：6）；
3. 2, 1 入栈，1 出栈，2 出栈（栈内：6）；
4. 6 出栈。
结论：合法。

C 选项 (3, 4, 6, 5, 2, 1)：
1. 6, 5, 4, 3 入栈，3 出栈（栈内：6, 5, 4）；
2. 4 出栈（栈内：6, 5）；
3. 此时栈顶元素是 5，若要进行出栈操作，下一个出栈的必须是 5。
4. 但选项中给出的序列下一个是 6。在 5 还没出栈的情况下，位于 5 下方的 6 是不可能先出栈的。
结论：非法。

D 选项 (2, 3, 4, 1, 5, 6)：
1. 6, 5, 4, 3, 2 入栈，2 出栈，3 出栈，4 出栈（栈内：6, 5）；
2. 1 入栈，1 出栈（栈内：6, 5）；
3. 5 出栈，6 出栈。
结论：合法。

因此，非法的序列是 C。

3), 运行以下代码片段的行为是（ ）
{{ select(3) }}
- A. 将 𝑥 x 的值赋为 2 0 1 201
- B. 将 𝑦 y 的值赋为 1 0 1 101
- C. 将 𝑞 q 指向 𝑥 x 的地址
- D. 将 𝑝 p 指向 𝑦 y 的地址

> **答案**: D
> **解析**: 根据题目提供的选项，这道题源自 GESP（编程能力等级认证）C++ 四级考试的经典真题。虽然题干中缺失了代码片段，但根据选项的逻辑，该题对应的标准代码片段如下：

int x = 101, y = 201;
int *p = &x, *q = &y;
p = q;

详细解析如下：

1. 变量与指针初始化：
   - int x = 101, y = 201;：定义了两个整型变量 x 和 y，并分别初始化。
   - int *p = &x;：定义指针变量 p，并将其指向变量 x 的内存地址。
   - int *q = &y;：定义指针变量 q，并将其指向变量 y 的内存地址。

2. 执行指针赋值操作 p = q：
   - 在 C++ 中，指针变量存储的是一个内存地址。
   - 语句 p = q; 的含义是将指针变量 q 中存储的地址值（即 y 的地址）赋值给指针变量 p。
   - 执行后，指针 p 内部存储的地址变成了 y 的地址。

3. 行为结果：
   - 此时，指针 p 不再指向 x，而是指向了 y 的地址。
   - 注意：该操作仅改变了指针 p 的指向，并没有改变变量 x 或 y 的数值。

4. 选项辨析：
   - A 选项：若要将 x 的值赋为 201，代码应为 *p = *q;（通过指针修改值）。
   - B 选项：若要将 y 的值赋为 101，代码应为 *q = *p;。
   - C 选项：若要将 q 指向 x 的地址，代码应为 q = p;。
   - D 选项：执行 p = q; 的直接行为就是将 p 指向 y 的地址。

因此，正确答案是 D。

4), 链表和数组的区别包括（ ）
{{ select(4) }}
- A. 数组不能排序，链表可以
- B. 链表比数组能存储更多的信息
- C. 数组大小固定，链表大小可动态调整
- D. 以上均正确

> **答案**: C
> **解析**: 这道题考察的是基础数据结构中数组和链表的主要区别：

1. 内存分配方式：
   - 数组（Array）：在内存中占用一段连续的空间。在大多数编程语言中，静态数组的大小在定义时就已确定，无法在运行时随意改变。即使是动态数组（如 C++ 的 vector），在扩容时也需要重新申请一块更大的连续空间并进行数据迁移。
   - 链表（Linked List）：由一个个独立的节点组成，节点之间通过指针连接。节点在内存中可以是不连续的，因此可以根据需要随时动态地添加或删除节点，不需要预先分配固定大小的空间。

2. 选项分析：
   - A 错误：数组和链表都可以进行排序。数组可以使用快速排序、归并排序等；链表也可以通过修改指针指向来进行排序（如链表的归并排序）。
   - B 错误：两者能存储的信息量主要取决于计算机的内存限制。实际上，由于链表每个节点都需要额外存储指针（Next 指针），在存储相同数量的数据时，链表占用的内存空间反而比数组更多。
   - C 正确：这是数组和链表在物理结构上的核心区别。数组是静态/固定分配（或扩容成本高），链表是动态分配。
   - D 错误：基于以上分析，只有 C 是正确的。

5), 对假设栈 S 和队列 Q 的初始状态为空。存在e1​∼e6​六个互不相同的数据，每个数据按照进栈 S、出栈 S、进队列 Q、出队列 Q 的顺序操作，不同数据间的操作可能会交错。已知栈 S 中依次有数据e1​、e2​、e3​、e4​、e5​和e6​进栈，队列 Q 依次有数据e2​、e4​、e3​、e6​、e5​和e1​出队列。则栈 S 的容量至少是（ ）个数据。
{{ select(5) }}
- A. 2 2
- B. 3 3
- C. 4 4
- D. 6 6

> **答案**: B
> **解析**: 1. 分析队列 Q 的性质：队列遵循“先进先出”（FIFO）的原则。题目给出队列 Q 的出队顺序是 $e_2, e_4, e_3, e_6, e_5, e_1$。因此，这些数据进入队列 Q 的顺序也必须是 $e_2, e_4, e_3, e_6, e_5, e_1$。

2. 分析栈 S 与队列 Q 的逻辑关系：题目规定每个数据的操作顺序是：进栈 S $\rightarrow$ 出栈 S $\rightarrow$ 进队列 Q $\rightarrow$ 出队列 Q。这意味着一个数据从栈 S 弹出后，会立即进入队列 Q。因此，栈 S 的出栈顺序必须与队列 Q 的进队顺序一致，即出栈顺序为：$e_2, e_4, e_3, e_6, e_5, e_1$。

3. 模拟栈 S 的操作过程（已知进栈顺序为 $e_1, e_2, e_3, e_4, e_5, e_6$）：
   - 为了让 $e_2$ 第一个出栈，必须先将 $e_1, e_2$ 依次压入栈 S。此时栈内元素为 $[e_1, e_2]$，栈深度为 2。
   - $e_2$ 出栈，进入队列。此时栈内元素为 $[e_1]$，栈深度为 1。
   - 下一个出栈的需要是 $e_4$，因此需将 $e_3, e_4$ 依次压入栈 S。此时栈内元素为 $[e_1, e_3, e_4]$，栈深度为 3。
   - $e_4$ 出栈，紧接着 $e_3$ 出栈。此时栈内元素为 $[e_1]$，栈深度为 1。
   - 下一个出栈的需要是 $e_6$，因此需将 $e_5, e_6$ 依次压入栈 S。此时栈内元素为 $[e_1, e_5, e_6]$，栈深度为 3。
   - $e_6$ 出栈，紧接着 $e_5$ 出栈，最后 $e_1$ 出栈。栈空。

4. 结论：在整个模拟过程中，栈 S 中同时存在的最大数据个数为 3（分别出现在压入 $e_4$ 后和压入 $e_6$ 后）。因此，栈 S 的容量至少需要 3 个数据空间。

6), 对表达式a+(b-c)*d的前缀表达式为（ ），其中 +、-、* 是运算符。
{{ select(6) }}
- A. *+a-bcd
- B. +a*-bcd
- C. abc-d*+
- D. abc-+d

> **答案**: B
> **解析**: 将中缀表达式转换为前缀表达式（波兰式）的步骤如下：

1. 确定运算优先级：
   在表达式 a + (b - c) * d 中，运算顺序为：
   - 首先计算括号内的减法：(b - c)
   - 其次计算乘法：(b - c) * d
   - 最后计算加法：a + ((b - c) * d)

2. 逐步转换：
   - 处理减法 (b - c)：前缀形式为 - b c。此时表达式可看作 a + (- b c) * d。
   - 处理乘法 (- b c) * d：将乘号 * 移到操作数 (- b c) 和 d 之前，前缀形式为 * - b c d。此时表达式可看作 a + (* - b c d)。
   - 处理加法 a + (* - b c d)：将加号 + 移到操作数 a 和 (* - b c d) 之前，前缀形式为 + a * - b c d。

3. 验证选项：
   - A 选项：*+a-bcd，顺序错误。
   - B 选项：+a*-bcd，符合转换结果。
   - C 选项：abc-d*+，这是该表达式的后缀表达式（逆波兰式）。
   - D 选项：abc-+d，格式错误。

因此，正确答案是 B。

7), 假设字母表{a,b,c,d,e}在字符串出现的频率分别为10%，15%，30%，16%，29%。若使用哈夫曼编码方式对字母进行不定长的二进制编码，字母 d 的编码长度（ ）位。
{{ select(7) }}
- A. 1 1
- B. 2 2
- C. 2 2 或 3 3
- D. 3 3

> **答案**: B
> **解析**: 哈夫曼编码（Huffman Coding）的构建过程是基于贪心策略的，具体步骤如下：

1. 准备数据：将所有字母按照频率从小到大排序：
   a (10%), b (15%), d (16%), e (29%), c (30%)。

2. 第一次合并：选择频率最小的两个节点 a(10%) 和 b(15%) 合并，生成新节点 N1，其频率为 10% + 15% = 25%。
   此时剩余节点为：d (16%), N1 (25%), e (29%), c (30%)。

3. 第二次合并：在剩余节点中选择频率最小的两个节点 d(16%) 和 N1(25%) 合并，生成新节点 N2，其频率为 16% + 25% = 41%。
   此时剩余节点为：e (29%), c (30%), N2 (41%)。

4. 第三次合并：在剩余节点中选择频率最小的两个节点 e(29%) 和 c(30%) 合并，生成新节点 N3，其频率为 29% + 30% = 59%。
   此时剩余节点为：N2 (41%), N3 (59%)。

5. 第四次合并：将最后的两个节点 N2(41%) 和 N3(59%) 合并，生成根节点（100%）。

6. 确定 d 的编码长度：
   - 根节点连接 N2 和 N3。
   - N2 连接 d 和 N1。
   - N1 连接 a 和 b。
   从根节点到字母 d 的路径为：根节点 -> N2 -> d。
   路径经过了 2 条边，因此字母 d 的编码长度为 2 位。

综上所述，字母 d 的编码长度固定为 2 位。

8), 一棵有 n 个结点的完全二叉树用数组进行存储与表示，已知根结点存储在数组的第 1 个位置。若存储在数组第 9 个位置的结点存在兄弟结点和两个子结点，则它的兄弟结点和右子结点的位置分别是（ ）
{{ select(8) }}
- A. 8 8、 1 8 18
- B. 1 0 10、 1 8 18
- C. 8 8、 1 9 19
- D. 1 0 10、 1 9 19

> **答案**: C
> **解析**: 在完全二叉树的数组存储表示中，如果根结点存储在数组的第 1 个位置（下标从 1 开始），则对于任意一个位于第 $i$ 个位置的结点，其相关的结点位置遵循以下规律：
1. 左子结点的位置为 $2i$；
2. 右子结点的位置为 $2i + 1$；
3. 父结点的位置为 $\lfloor i/2 \rfloor$（当 $i > 1$ 时）。

根据题目已知条件：
- 当前结点的位置 $i = 9$。

计算过程如下：
1. **寻找兄弟结点**：
   - 首先找到父结点的位置：$\lfloor 9/2 \rfloor = 4$。
   - 父结点（位置 4）的两个子结点分别是 $2 \times 4 = 8$（左子结点）和 $2 \times 4 + 1 = 9$（右子结点）。
   - 因此，位置 9 的结点的兄弟结点是位置 8。

2. **寻找右子结点**：
   - 该结点的左子结点位置为 $2 \times 9 = 18$。
   - 该结点的右子结点位置为 $2 \times 9 + 1 = 19$。

综上所述，兄弟结点的位置是 8，右子结点的位置是 19。对应选项 C（选项中的空格为排版格式，8 8 指 8，1 9 19 指 19）。

9), 考虑由 N 个顶点构成的有向连通图，采用邻接矩阵的数据结构表示时，该矩阵中至少存在（ ）个非零元素。
{{ select(9) }}
- A. 𝑁 − 1 N−1
- B. 𝑁 N
- C. 𝑁 + 1 N+1
- D. 𝑁 2 N 2

> **答案**: A
> **解析**: 1. **连通性的定义**：在图论中，对于一个有向图，如果将其所有的有向边替换为无向边后，该图是连通的，则称该有向图为**弱连通图**（通常在不强调“强连通”的情况下，简称为连通图）。
2. **最小边数**：对于一个包含 $N$ 个顶点的图，要使其连通（即任意两个顶点之间都存在路径，不考虑方向），最少需要的边数是 $N-1$ 条。这时的图在结构上呈现为一棵“树”。
3. **邻接矩阵与边的关系**：在邻接矩阵表示法中，如果顶点 $i$ 到顶点 $j$ 有一条边，则矩阵中第 $i$ 行第 $j$ 列的元素为非零值（通常为 1）。因此，邻接矩阵中非零元素的个数等于图中边的数量。
4. **结论**：既然使 $N$ 个顶点的图连通最少需要 $N-1$ 条边，那么对应的邻接矩阵中至少存在 $N-1$ 个非零元素。

补充说明：如果题目要求的是“强连通图”（即任意两个顶点 $u, v$ 之间都存在从 $u$ 到 $v$ 以及从 $v$ 到 $u$ 的有向路径），则最少需要 $N$ 条边（形成一个环）。但在普通“连通”的定义下，只需 $N-1$ 条边即可满足条件。

10), 以下对数据结构的表述不恰当的一项为：（ ）
{{ select(10) }}
- A. 图的深度优先遍历算法常使用的数据结构为栈。
- B. 栈的访问原则后进先出，队列的访问原则是先进先出。
- C. 队列常常被用于广度优先搜索算法。
- D. 栈与队列存在本质不同，无法用栈实现队列。

> **答案**: D
> **解析**: A 选项正确：深度优先遍历（DFS）的核心思想是“一路向前，直到不通再回溯”，这与栈的“后进先出”特性高度契合。在程序实现中，既可以使用递归（利用系统调用栈），也可以显式地使用栈数据结构。

B 选项正确：这是栈和队列最基础的定义。栈（Stack）是后进先出（LIFO），队列（Queue）是先进先出（FIFO）。

C 选项正确：广度优先遍历（BFS）的核心思想是“逐层扩散”，需要按照发现节点的先后顺序进行访问，因此通常使用队列来存储待访问的节点。

D 选项错误：虽然栈和队列的访问原则不同，但它们在逻辑上是可以相互模拟实现的。经典的面试和竞赛题目中经常出现“如何用两个栈实现一个队列”：一个栈负责接收新进入的元素，当需要出队时，将第一个栈的元素依次弹出并压入第二个栈，此时第二个栈的栈顶就是最早进入的元素，从而实现了先进先出的逻辑。同理，用两个队列也可以实现一个栈。因此，D 项中“无法用栈实现队列”的表述是不恰当的。

11), 以下哪组操作能完成在双向循环链表结点 p 之后插入结点 s 的效果（其中， next 域为结点的直接后继， prev 域为结点的直接前驱）：（ ）
{{ select(11) }}
- A. p->next->prev=s; s->prev=p; p->next=s; s->next=p->next;
- B. p->next->prev=s; p->next=s; s->prev=p; s->next=p->next;
- C. s->prev=p; s->next=p->next; p->next=s; p->next->prev=s;
- D. s->next=p->next; p->next->prev=s; s->prev=p; p->next=s;

> **答案**: D
> **解析**: 在双向循环链表中，要在结点 p 之后插入结点 s，需要修改四个指针：s 的前驱和后继，以及 p 的后继和 p 原本后继节点的前驱。

插入操作的关键在于：在修改 p->next 之前，必须先利用 p->next 找到原本的后继节点，或者先完成对 s->next 的赋值。

我们逐一分析选项：
A. p->next->prev=s; s->prev=p; p->next=s; s->next=p->next;
在执行最后一步 s->next=p->next 时，p->next 已经在前一步被修改为 s 了，这会导致 s->next 指向 s 自己，错误。

B. p->next->prev=s; p->next=s; s->prev=p; s->next=p->next;
同样的问题，p->next 在给 s->next 赋值前就被修改为 s 了，错误。

C. s->prev=p; s->next=p->next; p->next=s; p->next->prev=s;
在执行最后一步 p->next->prev=s 时，p->next 已经指向了 s，这行代码相当于 s->prev=s，没有正确修改原后继节点的指针，错误。

D. s->next=p->next; p->next->prev=s; s->prev=p; p->next=s;
1. s->next=p->next：让 s 指向 p 原本的后继。
2. p->next->prev=s：让 p 原本的后继指向 s。
3. s->prev=p：让 s 的前驱指向 p。
4. p->next=s：让 p 的后继指向 s。
该顺序保证了在修改 p->next 之前，已经利用它完成了对相关节点的指向操作，逻辑完全正确。

因此，正确答案是 D。

12), 以下排序算法的常见实现中，哪个选项的说法是错误的：（ ）
{{ select(12) }}
- A. 冒泡排序算法是稳定的
- B. 简单选择排序是稳定的
- C. 简单插入排序是稳定的
- D. 归并排序算法是稳定的

> **答案**: B
> **解析**: 在计算机科学中，排序算法的“稳定性”是指：如果待排序的序列中存在两个相等的元素，在排序之后，它们的相对顺序保持不变，则称该算法是稳定的。

下面逐一分析各选项：

A. 冒泡排序（Bubble Sort）：通过相邻元素的比较和交换来排序。当两个元素相等时，不会发生交换，因此它们的相对顺序保持不变。冒泡排序是稳定的。

B. 简单选择排序（Simple Selection Sort）：每一趟从待排序的部分选出最小（或最大）的一个元素，与该部分的第一个元素交换。这种交换可能会跨越多个元素，从而破坏相等元素的相对顺序。
例如：序列 [5, 8, 5, 2, 9]。
第一轮选择最小值 2，将其与第一个位置的 5 交换。
交换后的序列变为 [2, 8, 5, 5, 9]。
此时，原本在前面的第一个 5 被交换到了第二个 5 的后面，相对顺序发生了改变。因此，简单选择排序是不稳定的。

C. 简单插入排序（Simple Insertion Sort）：将待排序元素插入到已经排好序的序列中。在寻找插入位置时，如果遇到相等的元素，会将新元素插入在相等元素的后面，从而保持相对顺序。简单插入排序是稳定的。

D. 归并排序（Merge Sort）：采用分治法，在合并（Merge）两个有序子序列的过程中，如果遇到相等的元素，通常规定优先获取左侧子序列的元素，这样就能保证相等元素的相对顺序不变。归并排序是稳定的。

综上所述，选项 B 的说法是错误的。

13), 八进制数 32.1 对应的十进制数是（ ）
{{ select(13) }}
- A. 2 4 . 1 2 5 24.125
- B. 2 4 . 2 5 0 24.250
- C. 2 6 . 1 2 5 26.125
- D. 2 6 . 2 5 0 26.250

> **答案**: C
> **解析**: 将八进制数转换为十进制数，需要使用“按权展开求和”的方法。八进制的基数是 8。

对于八进制数 32.1，我们可以将其分为整数部分和小数部分分别计算：

1. 整数部分 (32)：
   - 十位上的数字是 3，其位权是 8^1 = 8。计算得：3 × 8 = 24。
   - 个位上的数字是 2，其位权是 8^0 = 1。计算得：2 × 1 = 2。
   - 整数部分合计：24 + 2 = 26。

2. 小数部分 (0.1)：
   - 小数点后第一位的数字是 1，其位权是 8^(-1) = 1/8。
   - 计算得：1 × (1/8) = 0.125。

3. 合并结果：
   - 26 + 0.125 = 26.125。

因此，八进制数 32.1 对应的十进制数是 26.125，选项 C 正确。

14), 一个字符串中任意个连续的字符组成的子序列称为该字符串的子串，则字符串 abcab 有（ ）个内容互不相同的子串。
{{ select(14) }}
- A. 1 2 12
- B. 1 3 13
- C. 1 4 14
- D. 1 5 15

> **答案**: B
> **解析**: 在计算机科学和编程竞赛（如 GESP、CSP）中，字符串的子串定义为：字符串中任意个连续字符组成的序列。计算一个字符串中互不相同的子串个数，通常需要包含“空串”。

我们可以按照子串的长度分类列举字符串 "abcab" 的所有子串：

1. 长度为 0 的子串：
   - 空串 "" (1个)

2. 长度为 1 的子串：
   - 原有：a, b, c, a, b
   - 去重后：a, b, c (3个)

3. 长度为 2 的子串：
   - 原有：ab, bc, ca, ab
   - 去重后：ab, bc, ca (3个)

4. 长度为 3 的子串：
   - 原有：abc, bca, cab
   - 去重后：abc, bca, cab (3个)

5. 长度为 4 的子串：
   - 原有：abca, bcab
   - 去重后：abca, bcab (2个)

6. 长度为 5 的子串：
   - 原有：abcab
   - 去重后：abcab (1个)

将上述各长度的互不相同子串数量相加：1 + 3 + 3 + 3 + 2 + 1 = 13 个。

注意：在考级题目中，若没有特殊说明，子串通常包含空串。如果不包含空串，结果为 12。根据选项设置，13 是更符合考题习惯的答案。
