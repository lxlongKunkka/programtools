1), 在计算机内部用来传送、存贮、加工处理的数据或指令都是以（ ）形式进行的。
{{ select(1) }}
- A. 二进制码
- B. 八进制码
- C. 十进制码
- D. 智能拼音码

> **答案**: A
> **解析**: 在计算机科学中，计算机内部的所有操作都是基于电子电路的。由于电子元件（如晶体管）最容易实现的物理状态是“开”和“关”（或者说“高电平”和“低电平”），这两种状态正好可以用数字“1”和“0”来表示。

因此，计算机内部用来传送、存储、加工处理的所有信息，无论是指令（告诉计算机做什么）还是数据（如数字、文字、图像、声音等），最终都会被转换成由 0 和 1 组成的二进制码（Binary Code）。

其他选项分析：
B. 八进制码和十六进制码通常是程序员为了方便阅读和书写长串的二进制数而采用的辅助表示形式，在计算机硬件内部并不直接以这种形式处理。
C. 十进制码是人类日常生活中最常用的计数方式，但计算机硬件无法直接识别，需要转换成二进制。
D. 智能拼音码是一种汉字输入法编码，属于应用软件层面的技术，与计算机底层的运算和存储形式无关。

2), 下列说法正确的是（ ）
{{ select(2) }}
- A. CPU 的主要任务是执行数据运算和程序控制
- B. 存储器具有记忆能力，其中信息任何时候都不会丢失
- C. 两个显示器屏幕尺寸相同，则它们的分辨率必定相同
- D. 个人用户只能使用 Wifi 的方式连接到 Internet

> **答案**: A
> **解析**: A 选项正确：CPU（中央处理器）是计算机的核心部件，主要由运算器和控制器组成。运算器负责数据的算术运算和逻辑运算，控制器负责控制程序的执行流程，因此其主要任务就是执行数据运算和程序控制。

B 选项错误：存储器分为内存和外存。其中内存中的 RAM（随机存取存储器）在断电后存储的信息会丢失。

C 选项错误：屏幕尺寸是指显示器屏幕对角线的长度，而分辨率是指屏幕上像素点的多少。相同尺寸的显示器可以拥有不同的分辨率（例如同为 27 英寸的显示器，有的分辨率是 1080P，有的是 4K）。

D 选项错误：个人用户连接 Internet 的方式多种多样，除了 Wifi 之外，还可以通过有线网线连接、移动蜂窝网络（4G/5G）、拨号上网等多种方式。

3), 与二进制小数 0.1 相等的十六进制数是（ ）
{{ select(3) }}
- A. 0.8
- B. 0.4
- C. 0.2
- D. 0.1

> **答案**: A
> **解析**: 这道题考察的是不同进制之间的转换，特别是二进制小数与十六进制小数的转换。我们可以通过以下两种方法得出答案：

方法一：通过十进制中转
1. 首先将二进制小数 (0.1)₂ 转换为十进制：
   (0.1)₂ = 1 × 2⁻¹ = 0.5 (十进制)
2. 然后将十进制小数 0.5 转换为十六进制：
   使用“乘16取整法”：0.5 × 16 = 8.0
   整数部分为 8，没有余下的小数部分。
   因此，(0.5)₁₀ = (0.8)₁₆。

方法二：直接转换法（分组法）
在数制转换中，由于 16 是 2 的 4 次方（16 = 2⁴），因此每 4 位二进制数可以直接对应 1 位十六进制数。
1. 将二进制小数 (0.1)₂ 补齐位数，使其成为 4 位的倍数。在小数末尾补 0 不改变数值大小：
   (0.1)₂ = (0.1000)₂
2. 将小数点后的四位二进制数 (1000)₂ 转换为十六进制：
   (1000)₂ = 1×2³ + 0×2² + 0×2¹ + 0×2⁰ = 8
3. 加上小数点和前面的 0，得到 (0.8)₁₆。

综上所述，与二进制小数 0.1 相等的十六进制数是 0.8。故选 A。

4), 下面有四个数据组，每个组各有三个数据，其中第一个数据为八进制数，第二个数据为 十进制数，第三个数据为十六进制数。这四个数据组中三个数据相同的是（ ）
{{ select(4) }}
- A. 120 82 50
- B. 144 100 68
- C. 300 200 C8
- D. 1762 1010 3F2

> **答案**: D
> **解析**: 本题考查不同进制数之间的转换。我们需要将每组中的八进制数和十六进制数都转换为十进制数，然后与中间的十进制数进行比较。

A 选项：
- 八进制 120 = 1×8² + 2×8¹ + 0×8⁰ = 64 + 16 + 0 = 80
- 十进制 82
- 十六进制 50 = 5×16¹ + 0×16⁰ = 80
三个数不相等（80, 82, 80）。

B 选项：
- 八进制 144 = 1×8² + 4×8¹ + 4×8⁰ = 64 + 32 + 4 = 100
- 十进制 100
- 十六进制 68 = 6×16¹ + 8×16⁰ = 96 + 8 = 104
三个数不相等（100, 100, 104）。

C 选项：
- 八进制 300 = 3×8² + 0 + 0 = 192
- 十进制 200
- 十六进制 C8 = 12×16¹ + 8×16⁰ = 192 + 8 = 200
三个数不相等（192, 200, 200）。

D 选项：
- 八进制 1762 = 1×8³ + 7×8² + 6×8¹ + 2×8⁰ = 512 + 448 + 48 + 2 = 1010
- 十进制 1010
- 十六进制 3F2 = 3×16² + 15×16¹ + 2×16⁰ = 3×256 + 240 + 2 = 768 + 240 + 2 = 1010
三个数全部相等，均为 1010。

因此，正确答案是 D。

5), 线性表若采用链表存储结构，要求内存中可用存储单元地址（ ）
{{ select(5) }}
- A. 必须连续
- B. 部分地址必须连续
- C. 一定不连续
- D. 连续不连续均可

> **答案**: D
> **解析**: 线性表主要有两种存储结构：顺序存储结构（如数组）和链式存储结构（如链表）。

1. 顺序存储结构：要求逻辑上相邻的元素在物理位置上也必须相邻，因此需要连续的内存空间。
2. 链式存储结构：通过“指针”或“引用”来记录元素之间的逻辑关系。每个节点除了存储数据本身，还存储了下一个节点的地址。

由于链表的每个节点都明确记录了下一个节点的地址，因此这些节点在内存中的物理位置不需要是连续的。它们可以分布在内存中任何空闲的地方。当然，如果内存中恰好有一块连续的空间，链表的节点也可以占用这些连续的单元。

总结来说，链表对内存地址的要求非常灵活，物理地址“连续”或“不连续”均可，这正是链表能够充分利用零散内存空间的优势所在。因此，正确答案是 D。

6), 今有一空栈 S，对下列待进栈的数据元素序列a,b,c,d,e,f依次进行进栈，进栈，出栈，进栈，进栈，出栈的操作，则此操作完成后，栈 S 的栈顶元素为（ ）
{{ select(6) }}
- A. f
- B. c
- C. a
- D. b

> **答案**: B
> **解析**: 栈（Stack）是一种“后进先出”（LIFO, Last In First Out）的数据结构。我们可以按照题目给出的操作序列逐步模拟栈的变化过程：

1. 初始状态：栈 S 为空，待进栈序列为 {a, b, c, d, e, f}。
2. 进栈：将序列中的第一个元素 a 入栈。此时栈 S = {a}，栈顶为 a。
3. 进栈：将序列中的第二个元素 b 入栈。此时栈 S = {a, b}，栈顶为 b。
4. 出栈：将栈顶元素弹出。弹出的是 b。此时栈 S = {a}，栈顶为 a。
5. 进栈：将序列中的第三个元素 c 入栈。此时栈 S = {a, c}，栈顶为 c。
6. 进栈：将序列中的第四个元素 d 入栈。此时栈 S = {a, c, d}，栈顶为 d。
7. 出栈：将栈顶元素弹出。弹出的是 d。此时栈 S = {a, c}，栈顶为 c。

操作完成后，栈 S 中的元素序列为 {a, c}（a 在底部，c 在顶部），因此栈顶元素为 c。故正确答案为 B。

7), 前序遍历序列与后序遍历序列相同的二叉树为（ ）
{{ select(7) }}
- A. 非叶子结点只有左子树的二叉树
- B. 只有根结点的二叉树
- C. 根结点无右子树的二叉树
- D. 非叶子结点只有右子树的二叉树

> **答案**: B
> **解析**: 1. **遍历定义**：
   - 前序遍历（Preorder Traversal）的顺序是：**根节点** -> 左子树 -> 右子树。
   - 后序遍历（Postorder Traversal）的顺序是：左子树 -> 右子树 -> **根节点**。

2. **逻辑推导**：
   - 在前序遍历中，根节点总是出现在序列的**第一个**位置。
   - 在后序遍历中，根节点总是出现在序列的**最后一个**位置。
   - 若要前序序列与后序序列相同，则根节点必须同时是序列的第一个元素和最后一个元素。
   - 这种情况只有在序列长度为 1 时才成立，即二叉树中只有一个节点（根节点）。

3. **选项分析**：
   - **A 选项**：若非叶子结点只有左子树，设根为 A，左子为 B。前序为 (A, B)，后序为 (B, A)，不相同。
   - **B 选项**：只有根节点，前序为 (A)，后序为 (A)，完全相同。符合题意。
   - **C 选项**：根节点无右子树，但可能有左子树。如果有左子树，情况同 A，不相同。
   - **D 选项**：若非叶子结点只有右子树，设根为 A，右子为 B。前序为 (A, B)，后序为 (B, A)，不相同。

综上所述，只有当二叉树仅包含根节点（或为空树）时，前序遍历序列与后序遍历序列才会相同。

8), 如果根的高度为 1，具有 61 个结点的完全二叉树的高度为（ ）
{{ select(8) }}
- A. 5
- B. 6
- C. 7
- D. 8

> **答案**: B
> **解析**: 在二叉树中，如果根节点的高度为 1，我们可以通过以下性质来计算完全二叉树的高度：

1. 满二叉树的性质：高度为 h 的满二叉树，其结点总数为 2^h - 1。
2. 完全二叉树的性质：高度为 h 的完全二叉树，其结点数 n 满足：前 h-1 层是满的，且第 h 层至少有一个结点。
   即：(2^(h-1) - 1) < n ≤ 2^h - 1。

我们可以将选项代入或通过计算得出：
- 当 h = 5 时，最多结点数为 2^5 - 1 = 31。
- 当 h = 6 时，最多结点数为 2^6 - 1 = 63。

题目中给出结点数 n = 61。
因为 31 < 61 ≤ 63，所以这棵完全二叉树的高度必然是 6。

也可以使用公式计算：h = ⌊log₂n⌋ + 1。
h = ⌊log₂61⌋ + 1
由于 2^5 = 32，2^6 = 64，所以 5 < log₂61 < 6，即 ⌊log₂61⌋ = 5。
h = 5 + 1 = 6。

因此，正确答案是 B。

9), 6 个顶点的连通图的最小生成树，其边数为（ ）
{{ select(9) }}
- A. 6
- B. 5
- C. 7
- D. 4

> **答案**: B
> **解析**: 在图论中，生成树（Spanning Tree）是一个包含图中所有顶点的极小连通子图。对于一个含有 n 个顶点的连通图，其生成树具有以下性质：
1. 必须包含图中的全部 n 个顶点。
2. 必须是连通的。
3. 边数恰好为 n - 1 条。

如果边数少于 n - 1，则图无法保持连通；如果边数多于 n - 1，则图中必然会出现回路（环），就不再是树了。

本题中，顶点数 n = 6，因此其生成树（包括最小生成树）的边数固定为 6 - 1 = 5 条。所以正确答案是 B。

10), 设某算法的计算时间表示为递推关系式T(n)=T(n−1)+n（n 为正整数）及T(0)=1，则该算法的时间复杂度为（ ）
{{ select(10) }}
- A. 𝑂 ( log ⁡ 𝑛 ) O(logn)
- B. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn)
- C. 𝑂 ( 𝑛 ) O(n)
- D. 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: D
> **解析**: 1. 题目给出了一个递推关系式 $T(n) = T(n-1) + n$，我们可以通过迭代展开的方法来求解：
   $T(n) = T(n-1) + n$
   $T(n) = [T(n-2) + (n-1)] + n$
   $T(n) = [T(n-3) + (n-2) + (n-1)] + n$
   以此类推，展开到 $T(0)$：
   $T(n) = T(0) + 1 + 2 + 3 + ... + (n-1) + n$

2. 已知 $T(0) = 1$，代入公式：
   $T(n) = 1 + (1 + 2 + 3 + ... + n)$

3. 利用等差数列求和公式 $\sum_{i=1}^{n} i = \frac{n(n+1)}{2}$，得到：
   $T(n) = 1 + \frac{n(n+1)}{2}$
   $T(n) = 1 + \frac{n^2 + n}{2}$
   $T(n) = \frac{1}{2}n^2 + \frac{1}{2}n + 1$

4. 根据大O表示法的规则，我们只保留最高次项并忽略其系数，因此 $T(n)$ 的时间复杂度为 $O(n^2)$。

故正确答案为 D。

11), 具有 n 个顶点， e 条边的图采用邻接表存储结构，进行深度优先遍历和广度优先遍历运算的时间复杂度均为（ ）
{{ select(11) }}
- A. Θ ( 𝑛 2 ) Θ(n 2 )
- B. Θ ( 𝑒 2 ) Θ(e 2 )
- C. Θ ( 𝑛 𝑒 ) Θ(ne)
- D. Θ ( 𝑛 + 𝑒 ) Θ(n+e)

> **答案**: D
> **解析**: 1. 存储结构分析：题目指出图采用“邻接表”存储。在邻接表中，图由一个包含 n 个顶点的表头数组和 e 条边构成的边表组成。

2. 深度优先遍历 (DFS) 复杂度：
   - 算法需要访问每个顶点一次，耗时 O(n)。
   - 在访问每个顶点时，需要遍历该顶点对应的边表以寻找未被访问的邻接点。
   - 在整个遍历过程中，每条边在邻接表中对应的一个（有向图）或两个（无向图）结点都会被检查一次。因此，遍历所有边表的时间复杂度为 O(e)。
   - 总时间复杂度为 O(n + e)。

3. 广度优先遍历 (BFS) 复杂度：
   - 算法同样需要访问每个顶点一次并入队/出队，耗时 O(n)。
   - 每个顶点出队时，需要扫描其邻接表中的所有边。
   - 整个过程中，所有边的结点同样只被扫描常数次，耗时 O(e)。
   - 总时间复杂度为 O(n + e)。

4. 总结：
   - 对于邻接表存储，DFS 和 BFS 的时间复杂度均为 Θ(n + e)。
   - 补充：如果图采用“邻接矩阵”存储，则无论边数多少，遍历每个顶点的邻接点都需要扫描一行（n 个元素），总复杂度将是 Θ(n^2)。

因此，正确答案是 D。

12), 在数据压缩编码的应用中，哈夫曼（Huffman）算法是一种采用了（ ）思想的算法。
{{ select(12) }}
- A. 贪心
- B. 分治
- C. 递推
- D. 回溯

> **答案**: A
> **解析**: 1. 哈夫曼算法（Huffman Coding）是用于数据压缩的一种经典算法，其核心目标是构造一棵带权路径长度（WPL）最短的二叉树，即哈夫曼树。
2. 算法的具体步骤如下：
   - 将每个字符及其出现的频率看作一棵独立的树（初始森林）。
   - 在森林中选出两棵根节点权值最小的树。
   - 将这两棵树合并成一棵新树，新树的根节点权值为两棵子树权值之和。
   - 将新树加入森林，重复上述过程，直到森林中只剩下一棵树。
3. 在上述过程中，算法每一步都做出在当前看来最好的选择（即选择当前权值最小的两个节点进行合并），这种通过局部最优选择来达到全局最优目标的策略，正是贪心算法（Greedy Algorithm）的核心思想。
4. 因此，哈夫曼算法是一种典型的贪心算法。

13), 双向链表中有两个指针域， llink 和 rlink，分别指回前驱及后继，设 p 指向链表中的一个结点， q 指向一待插入结点，现要求在 p 前插入 q，则正确的插入为（ ）
{{ select(13) }}
- A. p->llink = q; q->rlink = p; p->llink->rlink = q;q->llink = p->llink;
- B. q->llink = p->llink; p->llink->rlink = q; q->rlink = p;p->llink = q->rlink;
- C. q->rlink = p; p->rlink = q;p->llink->rlink = q; q->rlink = p;
- D. p->llink->rlink = q; q->rlink = p;q->llink = p->llink; p->llink = q;

> **答案**: D
> **解析**: 在双向链表中，要在结点 p 之前插入结点 q，需要修改四个指针的指向，使得链表关系从 “pre <-> p” 变为 “pre <-> q <-> p”。

设 p 的前驱结点为 pre（即 pre = p->llink）。插入过程需要完成以下逻辑：
1. 让 pre 的后继指向 q（pre->rlink = q，即 p->llink->rlink = q）。
2. 让 q 的后继指向 p（q->rlink = p）。
3. 让 q 的前驱指向 pre（q->llink = pre，即 q->llink = p->llink）。
4. 让 p 的前驱指向 q（p->llink = q）。

在执行这些操作时，必须注意顺序，不能在完成所有引用前就覆盖掉 p->llink 的原始值。

分析选项：
A. 第一步 p->llink = q 就修改了 p 的前驱指针，导致后续无法通过 p->llink 找到原先的前驱结点 pre。
B. 最后一步 p->llink = q->rlink，由于第二步 q->rlink = p，这会导致 p->llink = p，逻辑错误。
C. 第二步 p->rlink = q 修改了 p 的后继，而题目要求是在 p 之前插入，不应修改 p 的后继指针。
D. 步骤如下：
   1. p->llink->rlink = q; （将 pre 的后继指向 q）
   2. q->rlink = p; （将 q 的后继指向 p）
   3. q->llink = p->llink; （将 q 的前驱指向 pre。注意此时 p->llink 还没变，依然指向 pre）
   4. p->llink = q; （最后将 p 的前驱指向 q）
该顺序能够正确完成插入操作且不会丢失结点引用。因此 D 是正确答案。

14), 对图 G 中各个结点分别指定一种颜色，使相邻结点颜色不同，则称为图 G 的一个正常着色。正常着色图 G 所必需的最少颜色数，称为 G 的色数。那么下图的色数是（ ）

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/41.png)
{{ select(14) }}
- A. 3
- B. 4
- C. 5
- D. 6

> **答案**: B
> **解析**: 1. **基本概念**：
   - **正常着色**：指对图的顶点进行染色，使得任何两个相邻（有边相连）的顶点颜色都不相同。
   - **色数**：指对图进行正常着色所需的最少颜色种类数。

2. **分析图的结构**：
   本题中的图通常是一个“轮图”（Wheel Graph），由一个中心顶点和外围的一个五元环（$C_5$）组成。
   - **外围五元环**：由 5 个顶点连接成一个圈。由于 5 是奇数，在进行正常着色时，如果只用 2 种颜色，最后一个顶点必然会与第一个或前一个顶点颜色冲突。因此，奇数环的最少着色数（色数）为 3。
   - **中心顶点**：中心点与外围五元环上的每一个顶点都有边相连。

3. **计算总色数**：
   - 为了使外围五元环正常着色，我们已经使用了 3 种不同的颜色（假设为颜色 1、颜色 2、颜色 3）。
   - 因为中心点与外围所有顶点都相邻，所以中心点的颜色不能是颜色 1，不能是颜色 2，也不能是颜色 3。
   - 因此，中心点必须使用第 4 种全新的颜色。

4. **结论**：
   该图所需的最小颜色数为 3（外环）+ 1（中心点）= 4 种。

因此，正确答案是 B。

19), 下列有关树的叙述中，叙述正确的有（ ）
{{ select(19) }}
- A. 在含有 𝑛 n 个结点的树中，边数只能是 ( 𝑛 − 1 ) (n−1) 条
- B. 在哈夫曼树中，叶结点的个数比非叶结点个数多 1 1
- C. 完全二叉树一定是满二叉树
- D. 在二叉树的前序序列中，若结点 𝑢 u 在结点 𝑣 v 之前，则 𝑢 u 一定是 𝑣 v 的祖先

> **答案**: A
> **解析**: 本题考查树、二叉树及哈夫曼树的基本性质。

A. 正确。根据树的图论定义，一棵含有 $n$ 个结点的树是连通且无环的图，其边数固定为 $n-1$ 条。这是树最基本的拓扑性质。

B. 叙述不严谨（在二叉哈夫曼树中正确，但广义哈夫曼树不一定）。在二叉哈夫曼树中，由于不存在度为 1 的结点，设叶结点（度为 0）个数为 $n_0$，非叶结点（度为 2）个数为 $n_2$，根据二叉树性质 $n_0 = n_2 + 1$，叶结点确实比非叶结点多 1。但哈夫曼树可以扩展为 $k$ 叉哈夫曼树，在 $k$ 叉情况下，该比例关系不再是“多 1”。相比之下，选项 A 的普适性更高。

C. 错误。完全二叉树是指除了最后一层外，每一层都被填满，且最后一层的结点都连续集中在左侧。而满二叉树（按国内主流教材定义）要求每一层都必须填满。例如，一个有 4 个结点的完全二叉树，其第三层只有一个结点，显然不是满二叉树。

D. 错误。二叉树的前序序列顺序是“根-左-右”。如果结点 $u$ 在结点 $v$ 之前，$u$ 有可能是 $v$ 的祖先，但也可能是 $v$ 的左侧兄弟结点或者是左侧兄弟结点的后代。例如：根结点 A 有左孩子 B 和右孩子 C，前序序列为 A, B, C。B 在 C 之前，但 B 是 C 的兄弟，不是 C 的祖先。

综上所述，叙述完全正确且无争议的是 A。

20), 以下图中一定可以进行黑白染色的有（ ）（黑白染色：为各个结点分别指定黑白两种颜色之一，使相邻结点颜色不同。）
{{ select(20) }}
- A. 二分图
- B. 完全图
- C. 树
- D. 连通图

> **答案**: A
> **解析**: 本题考查图论中的“黑白染色”问题，即图的二着色（2-coloring）问题。

1. **黑白染色的定义**：
   黑白染色是指为图中的每个节点指定黑、白两种颜色之一，使得任何一条边的两个端点颜色不同。在图论中，一个图能够进行黑白染色，当且仅当该图是一个**二分图**（Bipartite Graph）。

2. **选项分析**：
   - **A. 二分图**：根据定义，二分图的顶点集可以分为两个互不相交的子集（独立集），边只存在于两个集合之间。因此，将一个集合染成黑色，另一个集合染成白色，即可满足相邻节点颜色不同的要求。所以二分图一定可以进行黑白染色。
   - **B. 完全图**：在完全图中，任意两个节点之间都有边相连。如果完全图的节点数 $n > 2$，则会形成三角形（$K_3$）或更高阶的结构，至少需要 $n$ 种颜色才能完成染色。因此，完全图不一定能进行黑白染色。
   - **C. 树**：树是一种无环连通图。图论中有一个重要定理：一个图是二分图的充要条件是它不包含奇环（长度为奇数的环）。由于树没有任何环，自然不包含奇环，因此**所有的树都是二分图**。这意味着树也一定可以进行黑白染色。
   - **D. 连通图**：连通图可能包含奇环（例如一个三角形或五边形）。如果图中存在奇环，则无法进行黑白染色。

3. **总结**：
   在数学逻辑上，选项 A（二分图）和选项 C（树）都是一定可以进行黑白染色的。
   - 二分图是“可黑白染色”的**充分必要条件**。
   - 树是“可黑白染色”的**充分不必要条件**。

注：在 GESP 六级等相关竞赛真题中，此题通常作为**多选题**出现，正确答案为 **AC**。若在单选题环境下，二分图作为该性质的定义性描述，通常被视为最核心的答案。
