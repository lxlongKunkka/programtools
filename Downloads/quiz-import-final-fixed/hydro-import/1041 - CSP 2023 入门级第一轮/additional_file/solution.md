在 C++ 中，下面哪个关键字用于声明一个变量， 其值不能被修改?
{{ select(1) }}
- A. unsigned
- B. const
- C. static
- D. mutable

> **答案**: B
> **解析**: 在 C++ 编程语言中，各个选项关键字的作用如下：

1. const：是 constant（常量）的缩写。使用 const 关键字声明的变量被称为常量，必须在声明时进行初始化，且在其生命周期内其值不能被修改。如果程序尝试修改 const 变量的值，编译器将会报错。
2. unsigned：用于修饰整型变量，表示该变量为无符号类型，即只能存储非负整数，它并不限制变量是否可以被修改。
3. static：用于声明静态变量。静态变量在程序的整个运行期间都存在，但它的值是可以被修改的（除非同时使用了 const）。
4. mutable：主要用于类的成员变量，允许在 const 成员函数中修改该成员变量的值，这与题目要求的“不能修改”含义相反。

因此，正确答案是 B。

> **答案**: D
> **解析**: 这道题要求计算两个八进制数 $12345670_8$ 和 $07654321_8$ 的和。

八进制加法的规则是“逢八进一”。我们可以列竖式进行逐位相加：

```
    1 2 3 4 5 6 7 0 (8)
+   0 7 6 5 4 3 2 1 (8)
-----------------------
```

从右向左逐位相加：
1.  第1位（个位）：$0 + 1 = 1$。
2.  第2位：$7 + 2 = 9$。在八进制中，$9 = 1 \times 8 + 1$，所以写 $1$ 进 $1$。
3.  第3位：$6 + 3 + 1$（进位）$= 10$。在八进制中，$10 = 1 \times 8 + 2$，所以写 $2$ 进 $1$。
4.  第4位：$5 + 4 + 1$（进位）$= 10$。在八进制中，$10 = 1 \times 8 + 2$，所以写 $2$ 进 $1$。
5.  第5位：$4 + 5 + 1$（进位）$= 10$。在八进制中，$10 = 1 \times 8 + 2$，所以写 $2$ 进 $1$。
6.  第6位：$3 + 6 + 1$（进位）$= 10$。在八进制中，$10 = 1 \times 8 + 2$，所以写 $2$ 进 $1$。
7.  第7位：$2 + 7 + 1$（进位）$= 10$。在八进制中，$10 = 1 \times 8 + 2$，所以写 $2$ 进 $1$。
8.  第8位：$1 + 0 + 1$（进位）$= 2$。

将各数位组合起来，得到的结果为 $22222211_8$。

对比选项：
A. $22222221_8$
B. $21111111_8$
C. $22111111_8$
D. $22222211_8$

3), 阅读下述代码，请问修改 data 的 value 成员以存储 3.14，正确的方式是
{{ select(3) }}
- A. data.value = 3.14;
- B. value.data = 3.14;
- C. data -> value = 3.14;
- D. value->data = 3.14;

> **答案**: A
> **解析**: 在 C++ 编程语言中，访问结构体（struct）、类（class）或联合体（union）实例的成员变量，主要有两种方式，取决于该变量是对象本身还是指向对象的指针：

1. 点运算符（.）：如果变量是一个对象实例（如题目中的 data），则使用“对象名.成员名”的格式来访问其成员。因此，修改 data 的 value 成员的正确语法是 data.value = 3.14;。
2. 箭头运算符（->）：如果变量是一个指向对象的指针（例如 data 是一个指针类型），则使用“指针名->成员名”的格式。

根据题目描述“修改 data 的 value 成员”，在没有明确说明 data 是指针的情况下，通常默认 data 是一个对象实例。

选项分析：
A. data.value = 3.14; —— 正确，使用点运算符访问对象的成员。
B. value.data = 3.14; —— 错误，这表示访问 value 对象的 data 成员，逻辑颠倒了。
C. data -> value = 3.14; —— 错误，只有当 data 是指针时才使用此方式。在基础考题中，若未提及指针，优先选择点运算符。
D. value->data = 3.14; —— 错误，逻辑颠倒且使用了指针运算符。

因此，正确答案是 A。

4), 假设有一个链表的节点定义如下： 现在有一个指向链表头部的指针：Node* head。如果想要在链表中插入一个新节点，其成员 data 的值为 42，并使新节点成为链表的第一个节点，下面哪个操作是正确的？
{{ select(4) }}
- A. Node* newNode = new Node; newNode->data = 42; newNode->next = head; head = newNode;
- B. Node* newNode = new Node; head->data = 42; newNode->next = head; head = newNode;
- C. Node* newNode = new Node; newNode->data = 42; head->next = newNode;
- D. Node* newNode = new Node; newNode->data = 42; newNode->next = head;

> **答案**: A
> **解析**: 在单链表中，要在头部插入一个新节点，通常需要执行以下四个步骤：

1. 创建新节点：使用 `new Node` 分配内存。
2. 设置数据：将新节点的 `data` 成员赋值为题目要求的 42。
3. 建立连接：将新节点的 `next` 指针指向当前的头节点 `head`。这样新节点就排在了原链表所有节点的前面。
4. 更新头指针：将 `head` 指针指向新创建的节点，确保 `head` 始终指向链表的起始位置。

对比选项：
- A 选项：完整且顺序正确地执行了上述四个步骤。
- B 选项：`head->data = 42` 错误地修改了原头节点的数据，而不是新节点的数据，且如果原链表为空（head 为 nullptr），会导致程序崩溃。
- C 选项：`head->next = newNode` 是将新节点插入到头节点之后，而不是作为第一个节点。
- D 选项：虽然创建并连接了新节点，但没有更新 `head` 指针。如果不更新 `head`，外部程序仍然认为原来的节点是链表头部，新节点会被“丢失”。

因此，正确答案是 A。

5), 根节点的高度为 1，一棵拥有 2023 个节点的三叉树高度至少为（ ）
{{ select(5) }}
- A. 6
- B. 7
- C. 8
- D. 9

> **答案**: C
> **解析**: 1. 题目要求计算拥有 2023 个节点的三叉树的最小高度。为了使高度最小，这棵三叉树必须尽可能地填满每一层，即除了最后一层外，每一层都是满的。

2. 在三叉树中，第 $i$ 层（根节点为第 1 层）最多拥有的节点数为 $3^{i-1}$。

3. 高度为 $h$ 的满三叉树的总节点数 $S$ 的计算公式为等比数列求和：
   $S = 1 + 3 + 9 + ... + 3^{h-1} = \frac{3^h - 1}{3 - 1} = \frac{3^h - 1}{2}$

4. 我们需要找到最小的整数 $h$，使得高度为 $h$ 的满三叉树能容纳至少 2023 个节点，即：
   $\frac{3^h - 1}{2} \ge 2023$
   $3^h - 1 \ge 4046$
   $3^h \ge 4047$

5. 计算 3 的幂次方：
   - 当 $h = 7$ 时，$3^7 = 2187$。此时最大节点数为 $(2187 - 1) / 2 = 1093$。1093 小于 2023，说明高度为 7 不够。
   - 当 $h = 8$ 时，$3^8 = 6561$。此时最大节点数为 $(6561 - 1) / 2 = 3280$。3280 大于 2023，说明高度为 8 足以容纳 2023 个节点。

因此，这棵三叉树的高度至少为 8。正确答案为 C。

6), 小明在某一天中依次有七个空闲时间段，他想要选出至少一个空闲时间段来练习唱歌，但他希望任意两个练习的时间段之间都有至少两个空闲的时间段让他休息。则小明一共有（ ）种选择时间段的方案。
{{ select(6) }}
- A. 31
- B. 18
- C. 21
- D. 33

> **答案**: B
> **解析**: 这是一道关于组合数学的计数问题。我们可以根据小明选择的时间段个数进行分类讨论。

设这七个时间段分别为 1, 2, 3, 4, 5, 6, 7。题目要求任意两个练习的时间段之间至少有两段空闲时间，这意味着如果选择了时间段 $a$ 和 $b$（假设 $a < b$），则必须满足 $b - a \ge 3$。

1. 选出 1 个时间段：
从 7 个时间段中任选 1 个，共有 $C(7, 1) = 7$ 种方案。

2. 选出 2 个时间段：
设选出的时间段为 $x$ 和 $y$（$x < y$），根据要求 $y - x \ge 3$。
我们可以使用“转换法”：令 $y' = y - 2$，则原问题转化为从 $\{1, 2, 3, 4, 5\}$ 中选出两个不同的数 $x$ 和 $y'$，使得 $1 \le x < y' \le 5$。
方案数为 $C(5, 2) = \frac{5 \times 4}{2 \times 1} = 10$ 种。
具体方案为：(1,4), (1,5), (1,6), (1,7), (2,5), (2,6), (2,7), (3,6), (3,7), (4,7)。

3. 选出 3 个时间段：
设选出的时间段为 $x, y, z$（$x < y < z$），根据要求 $y - x \ge 3$ 且 $z - y \ge 3$。
同样使用“转换法”：令 $y' = y - 2, z' = z - 4$，则原问题转化为从 $\{1, 2, 3\}$ 中选出三个不同的数。
方案数为 $C(3, 3) = 1$ 种。
具体方案为：(1, 4, 7)。

4. 选出 4 个及以上时间段：
若选 4 个时间段，最少需要的时间段总数为 $1 + 3 + 3 + 3 = 10$ 个，而现在只有 7 个，因此不可能选出 4 个或更多的时间段。

综上所述，总方案数为 $7 + 10 + 1 = 18$ 种。故选 B。

7), 以下关于高精度运算的说法错误的是（ ）
{{ select(7) }}
- A. 高精度计算主要是用来处理大整数或需要保留多位小数的运算
- B. 大整数除以小整数的处理的步骤可以是，将被除数和除数对齐，从左到右逐位尝试将除数乘以某个数，通过减法得到新的被除数，并累加商
- C. 高精度乘法的运算时间只与参与运算的两个整数中长度较长者的位数有关
- D. 高精度加法运算的关键在于逐位相加并处理进位

> **答案**: C
> **解析**: A选项正确：高精度计算（Arbitrary-precision arithmetic）的主要目的就是处理超出标准数据类型（如 int, long long, double）表示范围的大整数或需要极高精度的小数运算。

B选项正确：这描述的是模拟人工竖式除法的过程。在大整数除以小整数时，我们通常从被除数的高位开始，逐位下落并与当前的余数结合，计算当前位的商和新的余数。

C选项错误：在高精度乘法的普通算法（模拟竖式乘法）中，如果两个整数的长度分别为 n 和 m，那么需要进行 n * m 次单位数字的乘法。因此，运算时间与两个参与运算的整数的位数都有关，时间复杂度为 O(n * m)。即使使用更高级的算法（如 FFT），其时间复杂度也与两个数的位数均相关，而不仅仅取决于较长者的位数。

D选项正确：高精度加法的核心逻辑就是模拟竖式加法，通过数组存储每一位数字，从低位向高位逐位相加，并判断是否需要向高位进位。

8), 后缀表达式6 2 3 + - 3 8 2 / + * 2 ^ 3 +对应的中缀表达式是
{{ select(8) }}
- A. ((6-(2+3))*(3+8/2))^2+3
- B. 6-2+3*3+8/2^2+3
- C. (6-(2+3))*((3+8/2)^2)+3
- D. 6-((2+3)*(3+8/2))^2+3

> **答案**: A
> **解析**: 后缀表达式（逆波兰表达式）转换为中缀表达式的通用方法是使用栈。从左到右扫描后缀表达式，遇到数字则入栈，遇到运算符则从栈中弹出两个操作数进行运算（先弹出的作为右操作数，后弹出的作为左操作数），并将运算结果（带括号以保持优先级）重新压入栈中。

具体步骤如下：
1. 遇到 6, 2, 3：依次入栈。栈内：[6, 2, 3]
2. 遇到 +：弹出 3 和 2，计算 (2 + 3)，结果入栈。栈内：[6, (2 + 3)]
3. 遇到 -：弹出 (2 + 3) 和 6，计算 (6 - (2 + 3))，结果入栈。栈内：[(6 - (2 + 3))]
4. 遇到 3, 8, 2：依次入栈。栈内：[(6 - (2 + 3)), 3, 8, 2]
5. 遇到 /：弹出 2 和 8，计算 (8 / 2)，结果入栈。栈内：[(6 - (2 + 3)), 3, (8 / 2)]
6. 遇到 +：弹出 (8 / 2) 和 3，计算 (3 + 8 / 2)，结果入栈。栈内：[(6 - (2 + 3)), (3 + 8 / 2)]
7. 遇到 *：弹出 (3 + 8 / 2) 和 (6 - (2 + 3))，计算 ((6 - (2 + 3)) * (3 + 8 / 2))，结果入栈。栈内：[((6 - (2 + 3)) * (3 + 8 / 2))]
8. 遇到 2：入栈。栈内：[((6 - (2 + 3)) * (3 + 8 / 2)), 2]
9. 遇到 ^：弹出 2 和前面的表达式，计算 ((6 - (2 + 3)) * (3 + 8 / 2)) ^ 2，结果入栈。
10. 遇到 3：入栈。
11. 遇到 +：弹出 3 和前面的表达式，得到最终结果：((6 - (2 + 3)) * (3 + 8 / 2)) ^ 2 + 3。

对比选项，A 选项完全符合上述推导过程。

10), 假设有一组字符{a,b,c,d,e,f}, 对应的频率分别为5%,9%,12%,13%,16%,45%。请问以下哪个选项是字符 abcdef 分别对应的一组哈夫曼编码?
{{ select(10) }}
- A. 1111,1110,101,100,110,0
- B. 1010,1001,1000,011,010,00
- C. 000,001,010,011,10,11
- D. 1010,1011,110,111,00,01

> **答案**: A
> **解析**: 哈夫曼编码的构造过程是基于字符出现的频率，通过构建哈夫曼树来实现的。具体步骤如下：

1. 排序：将字符按频率从小到大排列：a(5%), b(9%), c(12%), d(13%), e(16%), f(45%)。
2. 合并：
   - 取最小的两个频率 a(5) 和 b(9) 合并，得到新节点 n1(14)。此时序列为：c(12), d(13), n1(14), e(16), f(45)。
   - 取当前最小的两个频率 c(12) 和 d(13) 合并，得到新节点 n2(25)。此时序列为：n1(14), e(16), n2(25), f(45)。
   - 取当前最小的两个频率 n1(14) 和 e(16) 合并，得到新节点 n3(30)。此时序列为：n2(25), n3(30), f(45)。
   - 取当前最小的两个频率 n2(25) 和 n3(30) 合并，得到新节点 n4(55)。此时序列为：f(45), n4(55)。
   - 最后合并 f(45) 和 n4(55)，得到根节点。

3. 确定编码长度：
   - f 直接连接在根节点的一侧，长度为 1。
   - n4 在另一侧，其下分为 n2 和 n3。
   - n2 下分为 c 和 d，因此 c 和 d 的长度为 3。
   - n3 下分为 n1 和 e，因此 e 的长度为 3。
   - n1 下分为 a 和 b，因此 a 和 b 的长度为 4。

   - A 选项：a(1111, 4位), b(1110, 4位), c(101, 3位), d(100, 3位), e(110, 3位), f(0, 1位)。长度完全符合，且满足前缀码定义（没有任何编码是另一个编码的前缀）。
   - B、C、D 选项的长度分布均不符合哈夫曼树的构造结果（特别是 f 的频率高达 45%，在哈夫曼编码中必然是 1 位）。

因此，正确答案是 A。

11), 给定一棵二叉树，其前序遍历结果为： ABDECFG ,中序遍历结果为： DEBACFG。请问这棵树的正确后序遍历结果是什么?
{{ select(11) }}
- A. EDBGFCA
- B. EDGBFCA
- C. DEBGFCA
- D. DBEGFCA

> **答案**: A
> **解析**: 通过二叉树的前序遍历和中序遍历，我们可以唯一确定一棵二叉树。具体推导步骤如下：

1. 确定根节点：前序遍历的第一个节点是 A，因此 A 是整棵树的根节点。
2. 划分左右子树：在中序遍历中找到 A，A 的左侧是 DEB，右侧是 CFG。
   - 左子树的中序遍历为 DEB，前序遍历为 BDE（对应前序中 A 之后的三个节点）。
   - 右子树的中序遍历为 CFG，前序遍历为 CFG（对应前序中最后的三个节点）。
3. 构建左子树：
   - 在左子树的前序 BDE 中，第一个节点 B 是左子树的根。
   - 在左子树的中序 DEB 中，B 的左侧是 DE，右侧为空。
   - 继续看 DE：前序为 DE，中序为 DE。D 是根，E 是 D 的右孩子（因为中序中 E 在 D 后面）。
   - 因此左子树结构为：B 是根，B 的左孩子是 D，D 的右孩子是 E。
4. 构建右子树：
   - 在右子树的前序 CFG 中，第一个节点 C 是右子树的根。
   - 在右子树的中序 CFG 中，C 的左侧为空，右侧是 FG。
   - 继续看 FG：前序为 FG，中序为 FG。F 是根，G 是 F 的右孩子。
   - 因此右子树结构为：C 是根，C 的右孩子是 F，F 的右孩子是 G。
5. 确定后序遍历：
   - 后序遍历的顺序是“左子树 -> 右子树 -> 根节点”。
   - 左子树 (B, D, E) 的后序遍历：E -> D -> B。
   - 右子树 (C, F, G) 的后序遍历：G -> F -> C。

1.  **①处解析**：当 `i == 0` 时，表示第一个字符串 `str1` 为空。要将空字符串转换为长度为 `j` 的字符串 `str2[0...j-1]`，需要进行 `j` 次插入操作。因此 `dp[0][j] = j`。对应选项 **A**。
2.  **②处解析**：当 `j == 0` 时，表示第二个字符串 `str2` 为空。要将长度为 `i` 的字符串 `str1[0...i-1]` 转换为空字符串，需要进行 `i` 次删除操作。因此 `dp[i][0] = i`。对应选项 **F**。
3.  **③处解析**：在动态规划转移时，如果当前两个字符相等，则不需要任何操作。字符串是 0 索引的，`dp[i][j]` 对应的是 `str1[i-1]` 和 `str2[j-1]`。因此判断条件是 `str1[i-1] == str2[j-1]`。对应选项 **I**。
4.  **④处解析**：如果 `str1[i-1] == str2[j-1]`，则 `dp[i][j]` 的值等于前一个状态的值，即 `dp[i-1][j-1]`，不需要增加操作次数。对应选项 **N**。
5.  **⑤处解析**：如果字符不相等，我们需要在三种操作中选择代价最小的：
    *   删除（Delete）：`dp[i-1][j] + 1`
    *   插入（Insert）：`dp[i][j-1] + 1`
    *   替换（Replace）：`dp[i-1][j-1] + 1`
    代码中已经写了 `1 + min({dp[i - 1][j], dp[i][j - 1], ⑤})`，因此 ⑤ 处应填入替换操作对应的状态 `dp[i-1][j-1]`。对应选项 **S**。
