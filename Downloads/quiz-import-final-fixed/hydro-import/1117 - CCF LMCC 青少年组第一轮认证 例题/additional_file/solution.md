# 1117 - CCF LMCC 青少年组第一轮认证 例题 - 题目与解析

1), 在掩码语言模型预训练中，核心任务“掩码预测”的主要目的是什么？
{{ select(1) }}
- A. 学习语言的深层双向上下文表征
- B. 评估模型最终的分类准确率
- C. 专门优化模型的文本生成流畅度
- D. 减少模型训练所需的计算资源

> **答案**: A
> **解析**: 掩码语言模型（Masked Language Model, MLM）是深度学习中自然语言处理（NLP）领域的一种重要预训练技术，最著名的代表是 BERT 模型。

1. 核心机制：在预训练过程中，系统会随机将输入句子中的一部分单词（通常为15%）替换为特殊的 [MASK] 标记。模型的目标是根据该单词左右两边的上下文信息来预测被掩盖的原始单词。

2. 目的解析：
   - 选项 A 正确：传统的语言模型（如 GPT）通常是单向的（从左到右），而掩码预测迫使模型同时利用被掩盖词左侧和右侧的信息。这种“双向”观察能力使得模型能够学习到更深层、更丰富的语言上下文表征，从而更好地理解词义和语法结构。
   - 选项 B 错误：分类准确率通常是下游任务（如情感分析）的评估指标，而不是预训练任务本身的目的。
   - 选项 C 错误：掩码预测主要用于理解任务。对于文本生成任务，通常使用自回归语言模型（Causal Language Model），因为 MLM 在生成时由于缺乏自回归特性，效果并不理想。
   - 选项 D 错误：掩码预测实际上增加了训练的复杂度和对计算资源的需求，其目的在于提升模型性能而非节省资源。

因此，掩码预测的核心目的是为了让模型学习到高质量的双向上下文语义表示。

2), 小明和小华在学习大语言模型时，尝试让它写太空探险的故事。小明的提示词是：“写一个关于太空探险的故事。”小华的提示词是：“请你扮演一位经验丰富的科幻小说家，为中学生写一个大约 500 字的短篇故事。故事的主角是一位名叫‘星尘’的年轻宇航员，他在探索一颗未知行星时，意外发现了一个古老外星文明的遗迹。故事风格要紧张刺激，结局要留下悬念。”结果小华生成的故事比小明更好。从提示词工程的角度分析，为什么小华能获得远比小明更好的结果？
{{ select(2) }}
- A. 因为小华提问时，大语言模型的知识库恰好更新了与“外星文明”相关的数据，所以能写出更具体的内容。
- B. 因为小华的提示词更长、更复杂，根据提示工程原理，投入的文字量与生成质量成正比，只要提示词足够长，效果就会更好。
- C. 因为小华的提示词为模型设定了明确的角色、目标受众和具体约束，有效地引导了模型的思维链，使其在庞大的可能性空间中聚焦于一个高质量的输出方向。
- D. 因为大语言模型具有真正的创造力和情感理解能力，小华的提示词中“紧张刺激”和“悬念”等词语激发了它的创作灵感。

> **答案**: C
> **解析**: 这道题考察的是“提示词工程（Prompt Engineering）”的基本原理。

1. 提示词工程的核心：大语言模型（LLM）本质上是基于概率预测的统计模型。当提示词过于笼统（如小明的提示词）时，模型会在极大的可能性空间中随机选择一个方向，导致生成的内容平庸、泛化。
2. 小华提示词的优势：
   - 角色设定（Role）：要求模型扮演“经验丰富的科幻小说家”，这能引导模型调用与专业写作相关的语料和风格。
   - 目标受众（Audience）：明确为“中学生”，使模型能自动调整语言的难度和趣味性。
   - 具体约束（Constraints）：规定了字数（500字）、主角名字、具体情节（发现遗迹）和风格（紧张刺激、悬念）。
   - 减少歧义：这些具体的指令有效地缩小了模型的输出范围，使其能够精准地在特定的“高质量路径”上生成内容。

3. 选项分析：
   - A 错误：模型生成质量的提升主要源于提示词的优化，而非知识库的实时更新。
   - B 错误：提示词并非越长越好，关键在于信息的有效性和结构化。无意义的堆砌文字反而可能干扰模型。
   - D 错误：目前的大语言模型并不具备人类意义上的“真实情感”或“灵感”，它们是通过复杂的数学计算来模拟人类的表达。

因此，小华之所以获得更好的结果，是因为他通过结构化、具体的提示词为模型提供了清晰的引导。

3), Transformer 架构中，使大语言模型能够有效处理长距离依赖关系的关键组件是：
{{ select(3) }}
- A. 前馈神经网络
- B. 层归一化
- C. 自注意力机制
- D. 残差连接

> **答案**: C
> **解析**: 1. 自注意力机制（Self-Attention Mechanism）是 Transformer 架构的核心创新。它允许模型在处理序列中的每一个元素时，都能直接“关注”到序列中其他所有的元素，无论它们之间的距离有多远。
2. 在传统的循环神经网络（RNN）中，信息必须按顺序通过中间隐藏状态进行传递，随着序列长度的增加，容易出现梯度消失或信息丢失的问题，难以处理长距离依赖。
3. 在自注意力机制中，任意两个位置之间的交互路径长度都是常数 O(1)，这使得模型能够直接捕捉到长文本中相距甚远的词汇之间的语义关联（例如代词与其指代的远端名词之间的关系）。
4. 其他选项的作用：
   - A. 前馈神经网络：主要负责对注意力机制提取的特征进行非线性变换和映射。
   - B. 层归一化：用于稳定神经元的输入分布，加速模型收敛。
   - D. 残差连接：通过将输入直接加到输出上，解决深层网络训练中的梯度消失问题。
因此，处理长距离依赖的关键组件是自注意力机制。

4), 指令微调的主要目的是什么？
{{ select(4) }}
- A. 让模型从零开始学习海量无标注数据中的通用知识。
- B. 增强模型理解和遵循人类指令或意图的能力，并改善其输出风格。
- C. 大幅降低模型的基础参数量，以提高推理速度。
- D. 专门用于修复模型在预训练阶段产生的事实性错误。

> **答案**: B
> **解析**: 指令微调（Instruction Fine-tuning）是大型语言模型（LLM）训练过程中的关键步骤，其核心目的是将预训练模型与人类的意图进行“对齐”。

1. 增强指令遵循能力：预训练阶段的模型主要是在海量文本上学习“预测下一个词”，它虽然掌握了大量知识，但并不一定能理解用户的具体指令（如“请帮我写一首诗”）。指令微调通过在“指令-回答”对的数据集上进行训练，让模型学会如何理解并执行人类的特定意图。
2. 改善输出风格：通过指令微调，可以规范模型的回答方式，使其输出更加自然、有用且符合人类的对话习惯。
3. 任务泛化：经过指令微调的模型在面对从未见过的任务指令时，通常能表现出更好的泛化能力。

选项分析：
A 选项描述的是“预训练（Pre-training）”阶段的目标。
C 选项描述的是“模型压缩（Model Compression）”或量化技术的目标，微调通常不会减少参数量。
D 选项虽然微调可能在一定程度上修正知识，但其主要目的并非修复事实错误，修复事实错误通常需要检索增强生成（RAG）或知识编辑技术。

5), 大语言模型中存在的社会偏见，其主要根源最可能是：
{{ select(5) }}
- A. 模型训练时使用的 GPU 硬件存在设计缺陷。
- B. 模型 Transformer 架构中的注意力机制算法存在固有偏差。
- C. 模型从预训练数据中学习了现实世界中存在的社会偏见。
- D. 模型在指令微调阶段，工程师有意注入了有偏见的指令。

> **答案**: C
> **解析**: 大语言模型（LLM）产生社会偏见的主要根源在于其训练数据。以下是详细解析：

1. 数据来源：大语言模型通常在海量的互联网文本、书籍和文章上进行预训练。这些数据是人类社会活动的产物，不可避免地包含了现实世界中存在的各种偏见（如性别歧视、种族偏见、地域歧视等）。
2. 学习机制：模型的核心任务是学习文本中的统计规律和关联。如果训练数据中某些群体经常与负面词汇或特定职业关联，模型就会捕捉并内化这些模式，从而在生成内容时表现出偏见。
3. 选项分析：
   - A 选项：GPU 硬件负责数值计算，与语义层面的偏见无关。
   - B 选项：Transformer 架构和注意力机制是处理序列信息的数学框架，本身是中性的工具，不具备产生社会偏见的逻辑。
   - D 选项：在指令微调阶段，工程师的目标通常是让模型更安全、更符合人类价值观（即“对齐”），通常会努力消除偏见而非有意注入。

因此，预训练数据中蕴含的现实世界偏见是模型产生偏见的最主要根源。

6), 目前主流的生成式大语言模型（如 GPT 系列）的核心架构通常基于：
{{ select(6) }}
- A. 编码器模型
- B. 编码器-解码器模型
- C. 解码器模型
- D. 卷积神经网络模型

> **答案**: C
> **解析**: 1. GPT 的全称是 Generative Pre-trained Transformer（生成式预训练 Transformer）。
2. Transformer 架构在最初提出时包含编码器（Encoder）和解码器（Decoder）两个部分。
3. 不同的预训练模型根据任务需求选择了不同的部分：
   - 编码器模型（Encoder-only）：如 BERT，主要用于文本理解、分类等任务。
   - 编码器-解码器模型（Encoder-Decoder）：如 T5、BART，常用于翻译、摘要等任务。
   - 解码器模型（Decoder-only）：如 GPT 系列、Llama、Claude 等。这类模型采用自回归（Autoregressive）的方式，通过上文预测下一个字符，是目前主流生成式大语言模型的核心架构。
4. 卷积神经网络（CNN）主要应用于计算机视觉领域（如图像识别），并非目前主流大语言模型的核心架构。

因此，正确答案是 C。

7), 用于指令微调的数据通常具有什么典型特征？
{{ select(7) }}
- A. 大规模、无标注的原始网页文本。
- B. 由（指令/问题，期望输出）配对组成的监督数据集。
- C. 纯粹的代码仓库和编程语言片段。
- D. 未经处理的原始音频和图像数据。

> **答案**: B
> **解析**: 指令微调（Instruction Fine-tuning）是大语言模型训练过程中的一个关键步骤，其目的是让模型能够理解并遵循人类的指令。

1. 指令微调的定义：在预训练阶段，模型通过海量的无标注文本（如选项A所述）学习语言规律和知识。而在指令微调阶段，需要通过有监督学习（Supervised Fine-Tuning, SFT）来规范模型的输出行为。
2. 数据特征：指令微调所使用的数据必须是“高质量”且“有监督”的。它通常由三部分或两部分组成：指令（Instruction）、输入（Input，可选）和期望的输出（Output/Response）。这种（问题/指令，答案）的配对形式是其最典型的特征。
3. 选项分析：
   - A选项：描述的是“预训练（Pre-training）”阶段的数据特征。
   - B选项：准确描述了指令微调所需的数据格式，即监督数据集。
   - C选项：描述的是针对编程能力的专项训练数据。
   - D选项：描述的是多模态模型（如视觉、音频模型）的原始输入数据。

因此，正确答案是 B。

8), 关于“贪心搜索”和“束搜索”这两种解码策略，以下哪种说法是最准确的？
{{ select(8) }}
- A. 贪心搜索因为每次都选最好的，所以总能生成最完美、最富有创造力的句子。
- B. 束搜索需要同时考虑多条路径，计算起来更复杂，但它更有可能找到一个整体上更通顺、更合理的句子。
- C. 束搜索的速度通常比贪心搜索更快，因为它是并行的。
- D. 这两种策略没有本质区别，无论用哪一种，AI 生成的句子质量都完全一样。

> **答案**: B
> **解析**: 这道题考察的是自然语言处理（NLP）中大模型生成文本时的两种主要解码策略：

1. 贪心搜索 (Greedy Search)：
在生成每一个词时，都只选择当前概率最大的那一个。它的优点是计算速度极快，但缺点是“短视”。因为当前最好的选择并不一定能导致最终整体序列的最优，往往会导致生成的句子重复、逻辑不通或陷入死循环。因此，选项 A 说它能生成“最完美、最富有创造力”的句子是不正确的。

2. 束搜索 (Beam Search)：
它在每一步生成时，会保留概率最大的前 k 个候选序列（k 被称为束宽）。在下一步时，它会基于这 k 个序列继续向下探索，最后从所有路径中选出整体概率最高的一条。
- 优点：它考虑了更多的可能性，能够有效避免贪心搜索带来的局部最优问题，生成的句子通常更通顺、质量更高。
- 缺点：因为要同时维护和计算多条路径，它的计算量比贪心搜索大，速度也相对较慢。

选项分析：
- A 错误：贪心搜索容易陷入局部最优，质量通常不如束搜索。
- B 正确：准确描述了束搜索的原理（考虑多条路径）、代价（计算更复杂）和优势（整体更合理）。
- C 错误：束搜索需要处理更多数据，速度比贪心搜索慢。
- D 错误：两者在算法逻辑和生成效果上有显著区别。

9), 大模型“人类对齐”的核心目标是使得模型的行为：
{{ select(9) }}
- A. 在所有的数学计算上达到零误差。
- B. 无限接近地在所有任务上超越人类专家水平。
- C. 符合人类的价值观、意图，并做到安全、有帮助。
- D. 其内部神经网络的计算过程对人类完全透明可解释。

> **答案**: C
> **解析**: 大模型“人类对齐”（Human Alignment）是人工智能安全和伦理领域的一个核心概念。其详细解析如下：

1. 核心定义：人类对齐是指确保人工智能系统的目标、行为和决策与人类的价值观、意图以及伦理准则保持一致。简单来说，就是让模型“听话”且“懂事”，做人类想让它做的事，不做人类不想让它做的事。

2. 三大原则（HHH原则）：在业界（如OpenAI、Anthropic等公司），对齐的目标通常被总结为：
   - 有用性（Helpfulness）：模型应尽力帮助用户完成任务。
   - 诚实性（Honesty）：模型应提供准确的信息，不误导用户，并在不知道答案时承认。
   - 无害性（Harmlessness）：模型不应产生歧视、暴力、违法或有害的建议。

3. 选项分析：
   - A选项：数学零误差属于模型的“能力（Capability）”范畴，而非“对齐”范畴。
   - B选项：超越人类专家水平属于“通用人工智能（AGI）”或“超智能”的目标，能力强并不代表它与人类价值观对齐（甚至可能因为能力太强而带来更大的失控风险）。
   - C选项：准确描述了对齐的核心——价值观一致、符合意图、安全且有帮助。
   - D选项：内部过程透明属于“可解释性（Interpretability）”研究，它是实现和验证对齐的一种手段，但不是对齐定义本身的核心目标。

因此，正确答案是 C。

10), 大语言模型在解决一道复杂的数学应用题时，生成了一段非常长的思考过程，其中包含了将问题分解为多个子步骤、对每个步骤进行详细解释、并逐步推导出中间结果。这种推理模式的主要优势是什么？
{{ select(10) }}
- A. 能够确保最终答案的正确性，避免计算错误。
- B. 通过展示详尽的思维过程，提高了解题逻辑的可解释性和可靠性。
- C. 显著减少了模型处理问题所需的总时间和计算资源。
- D. 主要目的是为了生成更多的文本内容，使回答看起来更丰富。

> **答案**: B
> **解析**: 题目描述的这种推理模式在人工智能领域被称为“思维链”（Chain of Thought, CoT）。以下是针对各选项的详细分析：

1. 为什么选择 B：
大语言模型通过将复杂问题拆解为中间步骤，不仅能够模拟人类“思考”的过程，更重要的是将原本黑盒化的推理过程透明化。这种详尽的思维过程让用户能够追踪模型的逻辑路径，验证每一步的合理性。如果结果出错，用户也能发现是哪一步出了问题，从而显著提高了解题逻辑的可解释性和可靠性。

2. 为什么 A 是错误的：
虽然思维链推理能显著提高模型处理复杂问题的准确率，但它并不能“确保”最终答案一定正确。模型在推理过程中仍然可能出现逻辑谬误或计算错误。

3. 为什么 C 是错误的：
生成长篇的思考过程实际上会消耗更多的计算资源（Token 数量增加）和推理时间。这是一种“以计算资源换取推理质量”的策略，而不是为了节省资源。

4. 为什么 D 是错误的：
生成更多文本只是思维链的表现形式，其核心目的是为了通过逐步推理来解决逻辑难题，而非单纯为了让回答看起来内容丰富。

总结：这种推理模式的核心价值在于增强了模型处理复杂逻辑任务的能力，并为用户提供了可追溯的逻辑依据。

11), 关于 AI 智能体的角色配置文件设置，下列哪个说法是最准确的？
{{ select(11) }}

14), 在 Transformer 的自注意力机制中，关于 Query、Key 和 Value 的来源，以下哪一项描述是最准确的？
{{ select(14) }}
- A. Query、Key 和 Value 是由三个完全不同的、随机初始化的参数矩阵生成的，与输入序列无关。
- B. Key 和 Value 是输入序列的嵌入向量本身，而 Query 是目标序列的嵌入向量本身。
- C. Query 来自编码器的最终隐藏状态，而 Key 和 Value 来自解码器的输入序列。
- D. Query、Key 和 Value 是由同一个输入向量通过三个不同的线性变换层（权重矩阵）投影得到的。

> **答案**: D
> **解析**: 在 Transformer 的自注意力（Self-Attention）机制中，Query (Q)、Key (K) 和 Value (V) 的生成过程如下：

1. 来源一致性：之所以称为“自”注意力，是因为这三个向量都源自同一个输入序列（或者是上一层注意力机制的输出）。
2. 线性变换：对于输入序列中的每一个向量（或嵌入向量），系统会分别乘以三个不同的可学习权重矩阵 $W^Q$、$W^K$ 和 $W^V$。
3. 投影过程：通过这三个不同的线性变换层，原始输入被投影到了三个不同的特征空间，分别代表了查询（Query）、键（Key）和值（Value）。

选项分析：
A 错误：Q、K、V 虽然由参数矩阵生成，但它们是输入序列与这些矩阵相乘的结果，与输入序列密切相关。
B 错误：这描述的不是自注意力。在自注意力中，三者来源相同；在交叉注意力中，Query 来自解码器，Key 和 Value 来自编码器。
C 错误：描述完全颠倒。在 Encoder-Decoder Attention 中，Query 来自解码器，而 Key 和 Value 来自编码器的输出。
D 正确：准确描述了自注意力机制中 Q、K、V 的产生方式，即同一个输入通过三个不同的线性投影得到。

16), 为了保障社会群体的“公平安全”，避免模型输出产生歧视性内容，最关键的缓解方法是在哪个阶段介入？
{{ select(16) }}
- A. 仅在模型推理部署后，通过实时过滤敏感词来屏蔽有害输出。
- B. 主要依靠扩大模型规模，期望其自动学习并消除偏见。
- C. 在模型预训练和微调阶段，系统性地清洗数据和引入公平性约束。
- D. 完全依赖第三方评测机构在发布前进行一次性的偏见评估。

> **答案**: C
> **解析**: 在人工智能和大型语言模型的开发中，保障“公平安全”和消除歧视性内容是一个系统性工程，需要从源头和核心环节进行干预。

A 选项（推理后过滤）：这属于“治标不治本”的方法。虽然实时过滤可以拦截部分显性违规词汇，但无法处理隐性歧视、刻板印象或复杂的逻辑偏见，且容易被绕过。

B 选项（扩大规模）：单纯增加模型参数和数据量并不能自动消除偏见。相反，如果训练数据中本身含有社会偏见，模型规模越大，往往会越高效地学习并放大这些偏见。

C 选项（预训练和微调阶段）：这是最关键、最核心的缓解方法。
1. 预训练阶段：通过对海量数据进行去偏处理、平衡不同群体的代表性，可以从底层减少模型吸收歧视性信息的概率。
2. 微调阶段（如指令微调和 RLHF 人类反馈强化学习）：通过引入公平性约束和人类价值观对齐，明确告诉模型哪些输出是不被允许的，从而在模型逻辑层面建立起安全防线。

D 选项（第三方评测）：评测是发现问题的手段，而不是解决问题的“缓解方法”。仅靠发布前的一次性评估无法从根本上改变模型的输出特性。

因此，系统性地在数据准备和模型训练（预训练+微调）阶段介入，是目前公认最有效、最关键的手段。

18), 基于强化学习的推理能力优化方法面临的主要挑战是什么？
{{ select(18) }}
- A. 模型会因此完全丧失其原有的语言生成能力。
- B. 推理路径的搜索空间巨大，导致训练不稳定和效率低下。
- C. 强化学习会使模型的词汇量急剧减少。
- D. 该方法只能应用于数学推理，无法泛化到其他领域。

> **答案**: B
> **解析**: 在基于强化学习（Reinforcement Learning, RL）优化大语言模型的推理能力时，面临的核心挑战主要集中在以下几个方面：

1. 搜索空间巨大：推理任务通常涉及多个步骤（即“思维链”）。在每一步，模型都有极多的词汇或逻辑路径可以选择。随着推理步骤的增加，可能的路径组合呈指数级增长，这使得模型很难在浩如烟海的选择中找到通往正确答案的最优路径。

2. 奖励稀疏性与信用分配问题：在推理任务中，通常只有最终答案正确时才能给出明确的奖励。然而，一个错误的答案可能是由于中间某一步逻辑错误导致的，也可能是步步都错。强化学习很难准确地将奖励（或惩罚）分配给推理路径中的具体某一步骤，这导致了训练的不稳定。

3. 训练效率低下：由于搜索空间巨大且奖励稀疏，模型需要大量的采样和试错才能学习到有效的推理策略，这消耗了极大的计算资源。

其他选项分析：
A 选项：虽然强化学习可能会带来“对齐税”（即在某些通用任务上性能微降），但绝不会导致模型“完全丧失”原有的语言生成能力。
C 选项：强化学习优化的是模型选择词汇的概率分布（策略），而不是改变模型预定义的词汇表，不会导致词汇量减少。
D 选项：强化学习推理优化方法具有通用性，除了数学推理，还广泛应用于代码生成、逻辑推理、常识问答等多个领域。

19), 关于语言模型的困惑度，以下描述正确的是：
{{ select(19) }}
- A. 困惑度越低，说明模型对测试数据的预测越不确定，性能越差。
- B. 困惑度是模型在训练集上的准确率直接转换而来的指标。
- C. 困惑度反映模型对一组文本的平均预测不确定性，值越低表示模型性能越好。
- D. 困惑度与模型的训练步数无关，只与模型架构有关。

> **答案**: C
> **解析**: 困惑度（Perplexity, PPL）是衡量语言模型优劣的重要指标，以下是针对各选项的详细解析：

1. 困惑度的定义：在语言模型中，困惑度衡量的是模型预测一个样本的确定性。从数学上讲，它是测试集中每个词概率几何平均值的倒数。
2. 性能关系：困惑度越低，表示模型在预测下一个词时越确定（即分配给真实词汇的概率越高），说明模型的语言建模能力越强，性能越好。因此，选项 A 描述错误，选项 C 描述正确。
3. 与准确率的区别：困惑度是基于概率分布（通常与交叉熵损失函数直接相关）计算的，而准确率（Accuracy）通常用于分类任务。虽然它们都反映模型性能，但困惑度并不是由训练集准确率直接转换而来的。因此，选项 B 错误。
4. 影响因素：困惑度不仅与模型架构（如 Transformer、RNN 等）有关，还与训练数据的质量、训练步数（模型是否充分收敛）、超参数设置等密切相关。随着训练步数的增加，模型通常能更好地拟合数据，困惑度会逐渐降低。因此，选项 D 错误。

综上所述，正确答案是 C。

20), 关于监督学习和无监督学习的区别，以下说法正确的是：
{{ select(20) }}

- B. 监督学习使用标注数据学习，无监督学习从无标签数据中发现模式
- C. 无监督学习只能处理数值型数据，监督学习可以处理任何类型数据
