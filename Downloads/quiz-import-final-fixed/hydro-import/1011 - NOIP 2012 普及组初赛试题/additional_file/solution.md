# 1011 - NOIP 2012 普及组初赛试题 - 题目与解析

1), 计算机如果缺少（ ），将无法正常启动。
{{ select(1) }}
- A. 内存
- B. 鼠标
- C. U 盘
- D. 摄像头

> **答案**: A
> **解析**: 计算机的正常启动需要核心硬件的支持，主要包括中央处理器（CPU）、内存（RAM）、主板、电源以及存储有操作系统的硬盘。

1. 内存（A选项）：内存是计算机中重要的部件之一，它是与 CPU 进行沟通的桥梁。计算机中所有程序的运行都是在内存中进行的。在启动过程中，计算机需要将 BIOS/UEFI 指令以及操作系统的核心数据加载到内存中才能运行。如果没有内存，计算机在开机自检（POST）阶段就会报错（通常伴随蜂鸣器报警），无法进入系统。
2. 鼠标（B选项）：属于输入设备，缺少鼠标不影响计算机的启动和基本运行。
3. U 盘（C选项）：属于外部存储设备，除非需要从 U 盘启动系统，否则平时缺少 U 盘不会影响计算机正常启动。
4. 摄像头（D选项）：属于多媒体输入设备，不是计算机启动的必要条件。

因此，正确答案是 A。

2), （ ）是一种先进先出的线性表。
{{ select(2) }}
- A. 栈
- B. 队列
- C. 哈希表（散列表）
- D. 二叉树

> **答案**: B
> **解析**: 1. 队列（Queue）是一种特殊的线性表，它只允许在表的前端（front）进行删除操作，而在表的后端（rear）进行插入操作。这种特性被称为“先进先出”（First-In-First-Out，简称 FIFO），即最先进入队列的元素最先被移出。
2. 栈（Stack）也是一种线性表，但它的特性是“后进先出”（Last-In-First-Out，简称 LIFO），即最后进入的元素最先出来。
3. 哈希表（Hash Table）是一种通过键（Key）直接访问内存存储位置的数据结构，主要用于快速查找，不属于典型的先进先出线性表。
4. 二叉树（Binary Tree）是一种非线性数据结构，属于树形结构。

综上所述，符合“先进先出”且是“线性表”定义的只有队列。

4), 十六进制数 9A 在（ ）进制下是 232。
{{ select(4) }}
- A. 四
- B. 八
- C. 十
- D. 十二

> **答案**: B
> **解析**: 这是一道关于进制转换的题目。我们可以通过将两个数都转换为十进制来求解。

1. 首先，将十六进制数 9A 转换为十进制：
   9A₁₆ = 9 × 16¹ + A × 16⁰
   由于在十六进制中 A 代表 10，所以：
   9A₁₆ = 9 × 16 + 10 × 1 = 144 + 10 = 154₁₀

2. 设目标进制为 x，根据题意有：
   232ₓ = 154₁₀
   展开多项式：
   2 × x² + 3 × x¹ + 2 × x⁰ = 154
   2x² + 3x + 2 = 154
   2x² + 3x - 152 = 0

3. 解这个一元二次方程：
   使用求根公式 x = [-b ± sqrt(b² - 4ac)] / 2a
   x = [-3 ± sqrt(3² - 4 × 2 × (-152))] / (2 × 2)
   x = [-3 ± sqrt(9 + 1216)] / 4
   x = [-3 ± sqrt(1225)] / 4
   x = [-3 ± 35] / 4
   取正值：x = 32 / 4 = 8

4. 验证：
   如果进制是八进制（B选项）：
   232₈ = 2 × 8² + 3 × 8¹ + 2 × 8⁰ = 2 × 64 + 3 × 8 + 2 = 128 + 24 + 2 = 154₁₀
   计算结果与 9A₁₆ 的十进制值一致。

因此，正确答案是 B。

6), 如果一棵二叉树的中序遍历是 BAC，那么它的先序遍历不可能是（ ）
{{ select(6) }}
- A. ABC ABC
- B. CBA CBA
- C. ACB ACB
- D. BAC BAC

> **答案**: C
> **解析**: 在二叉树遍历中，已知中序遍历和先序遍历的对应关系如下：
1. 先序遍历（Pre-order）：根节点 -> 左子树 -> 右子树
2. 中序遍历（In-order）：左子树 -> 根节点 -> 右子树

已知中序遍历为 BAC，我们可以根据先序遍历中可能的根节点（第一个字符）进行分类讨论：

1. 如果先序遍历是 ABC（选项 A）：
   - 根节点是 A。
   - 在中序遍历 BAC 中，A 的左边是 B，右边是 C。
   - 这表示 B 是 A 的左子树，C 是 A 的右子树。
   - 对应的先序遍历为：根(A) -> 左(B) -> 右(C)，即 ABC。这是可能的。

2. 如果先序遍历是 CBA（选项 B）：
   - 根节点是 C。
   - 在中序遍历 BAC 中，C 的左边是 BA，右边为空。
   - 这表示 BA 构成了 C 的左子树。
   - 对于左子树 BA，其中序遍历是 BA。如果 B 是 A 的父节点且 A 是 B 的右孩子，则其先序遍历也是 BA。
   - 对应的整棵树先序遍历为：根(C) -> 左(BA)，即 CBA。这是可能的。

3. 如果先序遍历是 BAC（选项 D）：
   - 根节点是 B。
   - 在中序遍历 BAC 中，B 的左边为空，右边是 AC。
   - 这表示 AC 构成了 B 的右子树。
   - 对于右子树 AC，其中序遍历是 AC。如果 A 是 C 的父节点且 C 是 A 的右孩子，则其先序遍历也是 AC。
   - 对应的整棵树先序遍历为：根(B) -> 右(AC)，即 BAC。这是可能的。

4. 如果先序遍历是 ACB（选项 C）：
   - 根节点是 A。
   - 如前所述，如果 A 是根节点，根据中序遍历 BAC，B 必须在左子树，C 必须在右子树。
   - 那么先序遍历必须是 A(根) B(左) C(右)，即 ABC。
   - 因此，先序遍历不可能是 ACB。

注：选项中的重复字符串（如 ABC ABC）通常是由于题目排版或录入错误导致的，不影响逻辑判断。

8), 使用冒泡排序对序列进行升序排列，每执行一次交换操作系统将会减少 1 个逆序对，因此序列5,4,3,2,1需要执行（ ）次操作，才能完成冒泡排序。
{{ select(8) }}
- A. 0
- B. 5
- C. 10
- D. 15

> **答案**: C
> **解析**: 1. 逆序对的概念：在一个序列中，如果位置在前的数字大于位置在后的数字（即 $i < j$ 且 $a[i] > a[j]$），则称这两个数构成一个逆序对。
2. 冒泡排序与逆序对的关系：冒泡排序的核心操作是比较相邻的两个元素，如果前一个比后一个大则交换。每次交换相邻的两个逆序元素，序列的逆序对数量会精确地减少 1。当序列变为升序排列时，逆序对数量为 0。因此，冒泡排序所需的交换次数等于初始序列中逆序对的总数。
3. 计算序列 5, 4, 3, 2, 1 的逆序对：
   - 数字 5 产生的逆序对有：(5,4), (5,3), (5,2), (5,1)，共 4 个；
   - 数字 4 产生的逆序对有：(4,3), (4,2), (4,1)，共 3 个；
   - 数字 3 产生的逆序对有：(3,2), (3,1)，共 2 个；
   - 数字 2 产生的逆序对有：(2,1)，共 1 个；
   - 数字 1 后面没有数字，产生 0 个。
   总逆序对数 = 4 + 3 + 2 + 1 = 10。
4. 也可以使用公式计算：对于一个长度为 $n$ 的完全逆序序列，其逆序对个数为 $n(n-1)/2$。本题中 $n=5$，计算得 $5 \times (5-1) / 2 = 10$。

因此，完成排序需要执行 10 次交换操作。正确答案为 C。

9), 1946 年诞生于美国宾夕法尼亚大学的 ENIAC 属于（ ）计算机。
{{ select(9) }}
- A. 电子管
- B. 晶体管
- C. 集成电路
- D. 超大规模集成电路

> **答案**: A
> **解析**: 计算机的发展历程通常根据其主要逻辑元件划分为四个阶段：

1. 第一代计算机（1946年-1950年代后期）：主要逻辑元件是电子管（Vacuum Tubes）。1946年在美国宾夕法尼亚大学诞生的 ENIAC（电子数值积分计算机）是世界上第一台通用电子计算机，它使用了约18,000个电子管，体积庞大、功耗高且运算速度相对较慢。
2. 第二代计算机（1950年代后期-1960年代中期）：主要逻辑元件是晶体管（Transistors）。
3. 第三代计算机（1960年代中期-1970年代初期）：主要逻辑元件是中小规模集成电路（Integrated Circuits）。
4. 第四代计算机（1970年代初期至今）：主要逻辑元件是大规模和超大规模集成电路（LSI/VLSI）。

因此，ENIAC 属于电子管计算机，正确答案为 A。

10), 无论是 TCP/IP 模型还是 OSI 模型，都可以视为网络的分层模型，每个网络协议都会被归入某一层中。如果用现实生活中的例子来比喻这些“层”，以下最恰当的是（ ）
{{ select(10) }}
- A. 中国公司的经理与波兰公司的经理交互商业文件
- B. 军队发布命令
- C. 国际会议中，每个人都与他国地位对等的人直接进行会谈
- D. 体育比赛中，每一级比赛的优胜者晋级上一级比赛

> **答案**: A
> **解析**: 网络协议的分层模型（如 OSI 七层模型或 TCP/IP 四层模型）的核心思想是将复杂的网络通信过程分解为多个功能独立的层。每一层都为上层提供服务，并利用下层的服务。

选项 A 最为恰当，原因如下：
1. 对等层通信：中国公司的经理与波兰公司的经理在逻辑上是直接交互商业文件的（相当于应用层之间的对等通信）。
2. 逐层处理与封装：虽然经理们感觉是在直接对话，但实际上文件需要经过秘书翻译（表示层）、办公室打包（传输层/网络层）、快递员运输（数据链路层/物理层）等步骤。
3. 透明性：经理不需要知道快递员走哪条路，快递员也不需要知道文件里的具体商业内容。这完美契合了网络模型中“层间独立、下层对上层透明”的原则。

其他选项分析：
B 选项：军队发布命令更多体现的是等级制度和单向指令传递，没有体现出对等层协议交互和复杂的协议栈处理。
C 选项：每个人直接会谈忽略了通信过程中底层技术的支撑和数据的逐层封装过程。
D 选项：体育比赛的晋级是一种筛选机制，而网络分层是每一层同时存在并协同工作完成一次通信，并非从低级进化到高级。

12), 如果一个栈初始时为空，且当前栈中的元素从栈底到栈顶依次为a,b,c，另有元素 d 已经出栈，则可能的入栈顺序是（ ）
{{ select(12) }}
- A. 𝑎 , 𝑑 , 𝑐 , 𝑏 a,d,c,b
- B. 𝑏 , 𝑎 , 𝑐 , 𝑑 b,a,c,d
- C. 𝑎 , 𝑐 , 𝑏 , 𝑑 a,c,b,d
- D. 𝑑 , 𝑎 , 𝑏 , 𝑐 d,a,b,c

> **答案**: D
> **解析**: 栈是一种后进先出（LIFO, Last In First Out）的数据结构。根据题目描述，我们需要满足两个条件：
1. 当前栈内元素从栈底到栈顶依次为 a, b, c。这意味着在当前状态下，a 比 b 先入栈，b 比 c 先入栈，且它们都还没有出栈。
2. 元素 d 已经出栈。这意味着 d 必须在某个时刻进入过栈，并且在当前状态之前已经执行了出栈操作。

我们逐一分析选项：
A 选项：入栈顺序 a, d, c, b。如果 a 入栈，d 入栈后出栈，接着 c 入栈，b 入栈。此时栈内从底到顶的顺序是 a, c, b。与题目要求的 a, b, c 不符。
B 选项：入栈顺序 b, a, c, d。如果 b 先入栈，那么 b 就会在栈底。与题目要求的 a 在栈底不符。
C 选项：入栈顺序 a, c, b, d。如果 a 入栈，c 入栈，b 入栈。此时栈内从底到顶的顺序是 a, c, b。与题目要求的 a, b, c 不符。
D 选项：入栈顺序 d, a, b, c。
   - 第一步：d 入栈，栈内为 [d]。
   - 第二步：d 出栈，栈为空，d 已出栈。
   - 第三步：a 入栈，栈内为 [a]。
   - 第四步：b 入栈，栈内为 [a, b]。
   - 第五步：c 入栈，栈内为 [a, b, c]。
此时，栈内从底到顶依次为 a, b, c，且 d 已经出栈。完全符合题目要求。

因此，正确答案是 D。

13), （ ）是主要用于显示网页服务器或者文件系统的 HTML 文件的内容，并让用户与这些文件交互的一种软件。
{{ select(13) }}
- A. 资源管理器
- B. 浏览器
- C. 电子邮件
- D. 编译器

14), （ ）是目前互联网上常用的 E-mail 服务协议。
{{ select(14) }}
- A. HTTP
- B. FTP
- C. POP3
- D. Telnet

> **答案**: C
> **解析**: 在互联网协议中，不同的协议负责不同的功能：

A. HTTP (HyperText Transfer Protocol，超文本传输协议)：主要用于在万维网（WWW）上传输网页数据。
B. FTP (File Transfer Protocol，文件传输协议)：主要用于在网络上进行文件的上传和下载。
C. POP3 (Post Office Protocol - Version 3，邮局协议版本3)：是目前互联网上常用的电子邮件接收协议，允许用户从邮件服务器将邮件下载到本地计算机。常见的电子邮件协议还包括用于发送邮件的 SMTP 和用于在线管理邮件的 IMAP。
D. Telnet (远程登录协议)：主要用于在本地计算机上远程登录和操作远程服务器。

综上所述，只有 POP3 是专门用于电子邮件服务的协议。因此正确答案是 C。

15), （ ）就是把一个复杂的问题分成两个或更多的相同类似的子问题，再把子问题分解成更小的子问题……直到最后的子问题可以简单地直接求解。而原问题的解就是子问题解的并。
{{ select(15) }}
- A. 动态规划
- B. 贪心
- C. 分治
- D. 搜索

> **答案**: C
> **解析**: 分治法（Divide and Conquer）是计算机科学中一种重要的算法思想，其核心逻辑正如题目所述：

1. 分解（Divide）：将原问题分解为若干个规模较小、相互独立、与原问题形式相同的子问题。
2. 解决（Conquer）：若子问题规模较小而容易被解决则直接解，否则递归地解各个子问题。
3. 合并（Combine）：将各个子问题的解合并为原问题的解。

其他选项分析：
A. 动态规划（Dynamic Programming）：虽然也将问题分解为子问题，但其核心在于处理“重叠子问题”和“最优子结构”，通常通过填表的方式保存已解决的子问题答案，避免重复计算。
B. 贪心（Greedy）：在每一步选择中都采取在当前状态下最好或最优的选择，从而希望导致结果是全局最好或最优的算法。它不一定将问题分解为类似的子问题。
D. 搜索（Search）：通常指在问题的解空间树中进行遍历（如深度优先搜索 DFS 或广度优先搜索 BFS），以寻找符合条件的解。

综上所述，题目中的定义完全符合“分治”算法的特征。

16), 地址总线的位数决定了 CPU 可直接寻址的内存空间大小，例如地址总线为 16 位，其最大的可寻址空间为 64KB。如果地址总线是 32 位，则理论上最大可寻址的内存空间为（ ）
{{ select(16) }}
- A. 1 2 8 KB 128 KB
- B. 1 MB 1 MB
- C. 1 GB 1 GB
- D. 4 GB 4 GB

> **答案**: D
> **解析**: 1. 地址总线的位数决定了 CPU 能够发出的不同地址编码的数量。如果有 n 位地址线，则可以表示 2^n 个不同的地址。
2. 在计算机体系结构中，内存通常是按字节（Byte）寻址的，即每一个唯一的地址对应内存中的一个字节。
3. 题目中提到 16 位地址总线的寻址空间为 64KB，计算方法为：2^16 字节 = 65536 字节 = 64 × 1024 字节 = 64 KB。
4. 对于 32 位地址总线，其理论最大寻址空间为 2^32 字节。
5. 具体的换算过程如下：
   - 2^10 字节 = 1024 字节 = 1 KB
   - 2^20 字节 = 1024 KB = 1 MB
   - 2^30 字节 = 1024 MB = 1 GB
   - 2^32 字节 = 2^2 × 2^30 字节 = 4 × 1 GB = 4 GB
因此，32 位地址总线理论上最大可寻址的内存空间为 4 GB。

17), 蓝牙和 Wi-Fi 都是（ ）设备。
{{ select(17) }}
- A. 无线广域网
- B. 无线城域网
- C. 无线局域网
- D. 无线路由器

> **答案**: C
> **解析**: 1. 网络按地理覆盖范围分类：
   - 无线广域网 (WWAN)：覆盖范围极大，如跨城市、跨国家，典型的例子是 4G/5G 蜂窝网络。
   - 无线城域网 (WMAN)：覆盖范围通常为一个城市，如 WiMAX 技术。
   - 无线局域网 (WLAN)：覆盖范围较小，通常在家庭、办公室或校园内（几十米到几百米）。
   - 无线个域网 (WPAN)：覆盖范围更小，通常在个人身边（几米到十几米）。

2. 蓝牙 (Bluetooth) 与 Wi-Fi 的特点：
   - Wi-Fi 是无线局域网 (WLAN) 的标准技术，用于设备连接到局域网或互联网。
   - 蓝牙虽然在严格定义上属于无线个域网 (WPAN)，但在计算机基础知识的分类题目中，它与 Wi-Fi 一样，都属于短距离、局部范围内的无线通信技术。

3. 选项分析：
   - A 和 B 覆盖范围过广，不符合蓝牙和 Wi-Fi 的特性。
   - D 选项“无线路由器”是一个具体的硬件设备，而题目问的是它们属于哪种类型的网络设备/技术。
   - C 选项“无线局域网”最符合这两者在短距离无线通信中的定位。因此，正确答案为 C。

18), 在程序运行过程中，如果递归调用的层数过多，会因为（ ）引发错误。
{{ select(18) }}
- A. 系统分配的栈空间溢出
- B. 系统分配的堆空间溢出
- C. 系统分配的队列空间溢出
- D. 系统分配的链表空间溢出

> **答案**: A
> **解析**: 在计算机程序运行过程中，函数调用是通过“栈”（Stack）这种数据结构来实现的。

1. 栈的作用：每当一个函数被调用时，系统都会在内存的栈区中为该函数分配一个“栈帧”（Stack Frame），用于存储该函数的局部变量、参数、返回地址以及调用状态等信息。
2. 递归的特点：递归调用是指函数直接或间接地调用自身。每一次递归调用都会产生一个新的栈帧并压入栈中。只有当递归达到终止条件并开始返回时，这些栈帧才会逐一被弹出并释放。
3. 错误原因：系统分配给每个程序的栈空间是有限的。如果递归调用的层数过多（例如没有设置正确的终止条件，或者递归深度超过了系统限制），栈空间就会被耗尽，从而导致“栈溢出”（Stack Overflow）错误。

相比之下：
- 堆（Heap）空间主要用于动态内存分配（如使用 new 或 malloc 申请的空间），虽然也有限制，但不是递归调用的直接管理区域。
- 队列和链表是逻辑数据结构，系统并不会专门分配所谓的“队列空间”或“链表空间”来管理函数调用。

因此，正确答案是 A。

19), 原字符串中任意一段连续的字符所组成的新字符串称为子串。则字符 AAABBBCCC 共有（ ）个不同的非空子串。
{{ select(19) }}
- A. 3
- B. 12
- C. 36
- D. 45

> **答案**: C
> **解析**: 1. 题目定义：子串是原字符串中任意一段连续的字符。题目要求计算“AAABBBCCC”中“不同”的“非空”子串个数。
2. 总子串数计算：对于长度为 n 的字符串，如果不考虑重复，其子串总数为 n(n+1)/2。
   本题中字符串长度 n = 9，所以总子串数为 9 × (9 + 1) / 2 = 45 个。
3. 排除重复子串：
   原字符串为 AAABBBCCC，其中存在重复的字符序列。
   - 长度为 1 的子串：A, A, A, B, B, B, C, C, C。其中 A 重复出现 3 次（多计 2 次），B 重复出现 3 次（多计 2 次），C 重复出现 3 次（多计 2 次）。重复计入：2 + 2 + 2 = 6 个。
   - 长度为 2 的子串：AA, AA, AB, BB, BB, BC, CC, CC。其中 AA 重复出现 2 次（多计 1 次），BB 重复出现 2 次（多计 1 次），CC 重复出现 2 次（多计 1 次）。重复计入：1 + 1 + 1 = 3 个。
   - 长度为 3 及以上的子串：观察发现，长度大于等于 3 的子串在原字符串中都是唯一的（例如 AAA 只出现一次，AAB 只出现一次等）。
4. 最终计算：
   不同的非空子串个数 = 总子串数 - 重复的次数
   不同的非空子串个数 = 45 - (6 + 3) = 45 - 9 = 36 个。

也可以按长度枚举：
- 长度1：A, B, C (3个)
- 长度2：AA, AB, BB, BC, CC (5个)
- 长度3：AAA, AAB, ABB, BBB, BBC, BCC, CCC (7个)
- 长度4：AAAB, AABB, ABBB, BBBC, BBCC, BCCC (6个)
- 长度5：AAABB, AABBB, ABBBC, BBBCC, BBCCC (5个)
- 长度6：AAABBB, AABBBC, ABBBCC, BBBCCC (4个)
- 长度7：AAABBBC, AABBBCC, ABBBCCC (3个)
- 长度8：AAABBBCC, AABBBCCC (2个)
- 长度9：AAABBBCCC (1个)
总计：3+5+7+6+5+4+3+2+1 = 36 个。
