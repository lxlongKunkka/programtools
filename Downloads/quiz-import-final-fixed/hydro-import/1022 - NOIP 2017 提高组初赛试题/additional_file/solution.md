2), 在 8 位二进制补码中， 10101011 表示的数是十进制下的（ ）
{{ select(2) }}
- A. 43
- B. 85
- C. 43
- D. 84

> **答案**: C
> **解析**: 1. **确定符号位**：在 8 位二进制补码中，最高位（最左边的一位）是符号位。如果最高位是 0，表示正数；如果最高位是 1，表示负数。本题中二进制数为 `10101011`，最高位是 1，因此它表示一个负数。

2. **补码转十进制的计算方法**：
   对于负数补码，我们可以通过以下两种常用方法将其转换为十进制：
   
   *   **方法一：公式法（位权法）**
       在 8 位补码中，最高位的权值是 $-2^7 = -128$，其余位的权值与普通二进制相同。
       计算过程：
       $1 \times (-128) + 0 \times 64 + 1 \times 32 + 0 \times 16 + 1 \times 8 + 0 \times 4 + 1 \times 2 + 1 \times 1$
       $= -128 + 32 + 8 + 2 + 1$
       $= -128 + 43$
       $= -85$

   *   **方法二：逆向转换法（补码 -> 反码 -> 原码）**
       1. 补码：`10101011`
       2. 反码（补码减 1）：`10101010`
       3. 原码（反码符号位不变，数值位取反）：`11010101`
       4. 将原码转换为十进制：
          符号位为 1，表示负号 `-`。
          数值位 `1010101` = $1 \times 2^6 + 0 \times 2^5 + 1 \times 2^4 + 0 \times 2^3 + 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 = 64 + 16 + 4 + 1 = 85$。
          所以结果为 `-85`。

3. **结论**：
   该二进制补码表示的十进制数是 -85。在给出的选项中，选项 C 出现了重复（通常是题目排版错误，原题 C 选项应为 -85），根据计算结果，正确答案应为 -85。

3), 分辨率为1600×900、 16 位色的位图，存储图像信息所需的空间为（ ）
{{ select(3) }}
- A. 2 8 1 2 . 5 KB 2812.5 KB
- B. 4 2 1 8 . 7 5 KB 4218.75 KB
- C. 4 3 2 0 KB 4320 KB
- D. 2 8 8 0 KB 2880 KB

> **答案**: A
> **解析**: 计算位图存储空间的步骤如下：

1. **计算总像素数**：
   分辨率为 1600 × 900，意味着图像共有 1600 × 900 = 1,440,000 个像素点。

2. **计算总位数（bits）**：
   题目给出是 16 位色，即每个像素占用 16 位（bit）。
   总位数 = 1,440,000 × 16 = 23,040,000 bits。

3. **转换为字节（Bytes）**：
   因为 1 字节（Byte）= 8 位（bits），
   总字节数 = 23,040,000 / 8 = 2,880,000 Bytes。

4. **转换为千字节（KB）**：
   在计算机存储单位换算中，1 KB = 1024 Bytes。
   存储空间（KB）= 2,880,000 / 1024 = 2812.5 KB。

对比选项，A 选项 2812.5 KB 符合计算结果。注意，如果按照 1 KB = 1000 Bytes 计算则为 2880 KB（选项 D），但在 GESP/CSP 等计算机专业考试中，通常严格按照 1024 进行换算。

4), 2017 年 10 月 1 日是星期日，1949 年 10 月 1 日是（ ）
{{ select(4) }}
- A. 星期三
- B. 星期日
- C. 星期六
- D. 星期二

> **答案**: C
> **解析**: 这是一道关于日期推算的逻辑题。我们可以通过计算两个日期之间相差的天数来确定星期几。

1. 计算年份差：
从 1949 年 10 月 1 日到 2017 年 10 月 1 日，总共经过了 2017 - 1949 = 68 年。

2. 计算闰年个数：
在这 68 年中，我们需要找出有多少个闰年。根据闰年的判定规则（四年一闰，百年不闰，四百年再闰）：
- 1949 年到 2017 年之间的第一个闰年是 1952 年，最后一个闰年是 2016 年。
- 闰年个数 = (2016 - 1952) / 4 + 1 = 16 + 1 = 17 个。
- 注意：2000 年是世纪年且能被 400 整除，所以也是闰年，已经包含在上述计算中。

3. 计算总天数对 7 的余数（位移量）：
- 平年有 365 天，365 ÷ 7 = 52 周……余 1 天。
- 闰年有 366 天，366 ÷ 7 = 52 周……余 2 天。
- 总位移天数 = (总年数 × 1 + 闰年数 × 1) = 68 + 17 = 85 天。
- 85 ÷ 7 = 12 周……余 1 天。

4. 推算星期：
已知 2017 年 10 月 1 日是星期日。因为我们要推算的是过去的时间，所以要从星期日往前推 1 天。
星期日的前一天是星期六。

因此，1949 年 10 月 1 日是星期六。正确答案为 C。

5), 设 G 是有 n 个结点、 m 条边(n≤m)的连通图，必须删去 G 的（ ）条边， 才能使得 G 变成一棵树。
{{ select(5) }}
- A. 𝑚 − 𝑛 + 1 m−n+1
- B. 𝑚 − 𝑛 m−n
- C. 𝑚 + 𝑛 + 1 m+n+1
- D. 𝑛 − 𝑚 + 1 n−m+1

> **答案**: A
> **解析**: 1. 树的定义与性质：在图论中，一棵包含 n 个结点的树必须满足两个条件：它是连通的，且没有环。对于包含 n 个结点的树，其边的数量固定为 n - 1 条。
2. 题目条件：给定的图 G 是一个包含 n 个结点和 m 条边的连通图。
3. 目标：将图 G 变成一棵树。由于图 G 已经是连通的，我们只需要通过删边来消除图中所有的环，直到边的数量减少到 n - 1 条为止。
4. 计算过程：
   - 初始边数：m
   - 目标边数（树的边数）：n - 1
   - 需要删去的边数 = 初始边数 - 目标边数
   - 需要删去的边数 = m - (n - 1) = m - n + 1

因此，正确答案是 A。

6), 若某算法的计算时间表示为递推关系式：
- T(N)=2T(2N​)+NlogN
- T(1)=1 则该算法的时间复杂度为（ ）
{{ select(6) }}
- A. 𝑂 ( 𝑁 ) O(N)
- B. 𝑂 ( 𝑁 log ⁡ 𝑁 ) O(NlogN)
- C. 𝑂 ( 𝑁 log ⁡ 2 𝑁 ) O(Nlog 2 N)
- D. 𝑂 ( 𝑁 2 ) O(N 2 )

> **答案**: C
> **解析**: 我们可以通过主定理（Master Theorem）或者递归树法来求解该递推关系式的时间复杂度。

1. **主定理分析**：
   递推式为 $T(N) = aT(N/b) + f(N)$，其中 $a=2, b=2, f(N) = N \log N$。
   计算 $N^{\log_b a} = N^{\log_2 2} = N^1 = N$。
   将 $f(N)$ 与 $N^{\log_b a}$ 进行比较：
   $f(N) = N \log N$，这属于主定理的扩展情况（Case 2 的扩展）：
   如果 $f(N) = \Theta(N^{\log_b a} \log^k N)$，其中 $k \ge 0$，则 $T(N) = \Theta(N^{\log_b a} \log^{k+1} N)$。
   在本题中，$k=1$，因此 $T(N) = \Theta(N \log^{1+1} N) = \Theta(N \log^2 N)$。

2. **递归树法分析**：
   - 第 0 层：$N \log N$
   - 第 1 层：$2 \times (\frac{N}{2} \log \frac{N}{2}) = N \log \frac{N}{2} = N(\log N - 1)$
   - 第 2 层：$4 \times (\frac{N}{4} \log \frac{N}{4}) = N \log \frac{N}{4} = N(\log N - 2)$
   - ...
   - 第 $i$ 层：$N(\log N - i)$
   - 递归深度为 $\log N$ 层。
   
   总和 $T(N) = \sum_{i=0}^{\log N - 1} N(\log N - i) + N \cdot T(1)$
   $T(N) = N \times [\log N + (\log N - 1) + (\log N - 2) + \dots + 1] + N$
   括号内是一个等差数列，和为 $\frac{\log N(1 + \log N)}{2}$。
   所以 $T(N) = N \cdot \frac{\log^2 N + \log N}{2} + N = \frac{1}{2} N \log^2 N + \frac{1}{2} N \log N + N$。
   忽略常数项和低次项，其时间复杂度为 $O(N \log^2 N)$。

选项 C 中的 $O(N \log 2 N)$ 通常在排版中表示 $O(N \log^2 N)$，即 $N$ 乘以 $\log N$ 的平方。因此正确答案为 C。

7), 表达式a*(b+c)*d的后缀形式是（ ）
{{ select(7) }}
- A. a b c d * + * a b c d * + *
- B. a b c + * d * a b c + * d *
- C. a * b c + * d a * b c + * d
- D. b + c * a * d b + c * a * d

> **答案**: B
> **解析**: 1. 理解中缀表达式：题目给出的表达式 a(b+c)d 在数学和编程逻辑中隐含了乘法运算，完整的中缀表达式应记为 a * (b + c) * d。

2. 转换步骤（遵循优先级）：
   - 第一步：处理括号内的运算 (b + c)。后缀表达式要求运算符在操作数之后，因此变为 b c +。
   - 第二步：处理左侧的乘法 a * (b + c)。将 (b + c) 的结果看作一个整体，则 a * [b c +] 转换为后缀形式为 a b c + *。
   - 第三步：处理右侧的乘法 [a b c + *] * d。将前面的结果看作一个整体，则 [a b c + *] * d 转换为后缀形式为 a b c + * d *。

3. 分析选项：
   - 选项 B 的内容 a b c + * d * 正好符合上述推导过程。
   - 选项 A 的逻辑错误，操作符顺序不匹配。
   - 选项 C 和 D 中出现了运算符夹在操作数中间的情况，这不符合后缀表达式（逆波兰表示法）的定义。

注意：选项中出现的重复字符串（如 a b c + * d * a b c + * d *）通常是由于题目排版或复制产生的冗余，核心逻辑应以其前半部分为准。

8), 由四个不同的点构成的简单无向连通图的个数是（ ）
{{ select(8) }}
- A. 32
- B. 35
- C. 38
- D. 41

> **答案**: C
> **解析**: 这是一个关于图论的计数问题。我们需要计算由 4 个有标记（不同）的顶点构成的简单无向连通图的个数。

1. **基本概念：**
   - **简单图**：没有自环和重边。
   - **无向图**：边没有方向。
   - **连通图**：图中任意两个顶点之间都存在路径。
   - **四个不同的点**：意味着顶点是有标签的（例如标记为 A, B, C, D），顶点的顺序和组合会影响图的计数。

2. **计算总图数：**
   四个顶点之间最多可以有 $C(4, 2) = \frac{4 \times 3}{2} = 6$ 条边。
   每个边都有“存在”或“不存在”两种状态，因此 4 个顶点的简单无向图总数为 $2^6 = 64$ 种。

3. **计算连通图的个数（使用补集法）：**
   连通图个数 = 总图数 - 不连通图数。
   为了不重不漏地计算不连通图，我们可以根据“顶点 A 所在的连通分量的大小”进行分类：
   - **A 所在分量大小为 1**：
     从剩下的 3 个点中选 0 个与 A 组合：$C(3, 0) = 1$ 种。
     剩下的 3 个点之间可以任意连边：$2^{C(3, 2)} = 2^3 = 8$ 种。
     情况数：$1 \times 8 = 8$ 种。
   - **A 所在分量大小为 2**：
     从剩下的 3 个点中选 1 个与 A 组合：$C(3, 1) = 3$ 种。
     这两个点必须连通：只有 1 种情况（即 A 与该点连边）。
     剩下的 2 个点之间可以任意连边：$2^{C(2, 2)} = 2^1 = 2$ 种。
     情况数：$3 \times 1 \times 2 = 6$ 种。
   - **A 所在分量大小为 3**：
     从剩下的 3 个点中选 2 个与 A 组合：$C(3, 2) = 3$ 种。
     这 3 个点必须连通：3 个点的连通图有 4 种（1个三角形和3个V字形路径）。
     剩下的 1 个点之间可以任意连边：$2^{C(1, 2)} = 2^0 = 1$ 种。
     情况数：$3 \times 4 \times 1 = 12$ 种。

4. **汇总：**
   不连通图的总数 = $8 + 6 + 12 = 26$ 种。
   连通图的总数 = $64 - 26 = 38$ 种。

也可以按边数 $m$ 分类讨论：
- $m=6$：$C(6, 6) = 1$ 种（全连通）。
- $m=5$：$C(6, 5) = 6$ 种（全连通）。
- $m=4$：$C(6, 4) = 15$ 种（全连通）。
- $m=3$：$C(6, 3) = 20$ 种，其中不连通的是“一个三角形+一个孤立点”，有 $C(4, 1) = 4$ 种，所以连通的有 $20 - 4 = 16$ 种。
总计：$1 + 6 + 15 + 16 = 38$ 种。

9), 将 7 个名额分给 4 个不同的班级，允许有的班级没有名额，有（ ）种不 同的分配方案。
{{ select(9) }}
- A. 60
- B. 84
- C. 96
- D. 120

> **答案**: D
> **解析**: 这是一个典型的组合数学问题，可以使用“隔板法”的变形来解决。

1. 问题模型：
将 n 个相同的元素（名额）分配给 m 个不同的对象（班级），且允许有的对象分配到的数量为 0。

2. 隔板法原理：
标准的隔板法要求每个对象至少分到一个元素。如果允许为空，我们可以采用“借名额”的思想：
假设我们先向这 4 个班级每个班级“借”1个名额，那么总名额就变成了 7 + 4 = 11 个。
现在问题转化为：将 11 个名额分给 4 个班级，且每个班级至少分到 1 个名额。
这样，在分完之后，每个班级再还回 1 个名额，就对应了原题中“允许有的班级没有名额”的情况。

3. 计算过程：
将 11 个名额排成一排，中间有 10 个空隙。我们需要在 10 个空隙中插入 3 块隔板，将名额分成 4 份。
方案数 = C(11-1, 4-1) = C(10, 3)

4. 具体计算：
C(10, 3) = (10 × 9 × 8) / (3 × 2 × 1)
C(10, 3) = 720 / 6
C(10, 3) = 120

因此，共有 120 种不同的分配方案。正确答案为 D。

10), 若f0​=0,f1​=1,fn+1​=2fn​+fn−1​​，则随着 i 的增大，fi​将接近于（ ）
{{ select(10) }}
- A. 1 / 2 1/2
- B. 2 / 3 2/3
- C. 5 − 1 2 2 5 ​ −1 ​
- D. 1 1

> **答案**: B
> **解析**: 1. **分析递推关系**：题目给出的递推公式为 $f_{n+1} = \frac{2f_n + f_{n-1}}{3}$，初始值为 $f_0 = 0, f_1 = 1$。

2. **计算数列项**：
   - $f_0 = 0$
   - $f_1 = 1$
   - $f_2 = \frac{2 \times 1 + 0}{3} = \frac{2}{3} \approx 0.667$
   - $f_3 = \frac{2 \times \frac{2}{3} + 1}{3} = \frac{7}{9} \approx 0.778$
   - $f_4 = \frac{2 \times \frac{7}{9} + \frac{2}{3}}{3} = \frac{20}{27} \approx 0.741$
   - $f_5 = \frac{2 \times \frac{20}{27} + \frac{7}{9}}{3} = \frac{61}{81} \approx 0.753$

3. **求极限值**：
   该数列是一个线性递推数列。我们可以通过特征方程 $3x^2 - 2x - 1 = 0$ 来求解。
   解得特征根为 $x_1 = 1, x_2 = -1/3$。
   通项公式形式为 $f_n = A \cdot (1)^n + B \cdot (-1/3)^n$。
   代入 $f_0=0, f_1=1$：
   - $A + B = 0$
   - $A - \frac{1}{3}B = 1$
   解得 $A = 3/4, B = -3/4$。
   因此 $f_n = \frac{3}{4} - \frac{3}{4}(-\frac{1}{3})^n$。当 $n \to \infty$ 时，$f_n \to 3/4$。

4. **选项分析与修正**：
   计算结果为 $3/4$（0.75），但观察选项：A(1/2), B(2/3), C(黄金分割), D(1)。选项中没有 $3/4$。
   在 GESP/CSP 等编程竞赛的数学题库中，这道题是一个非常经典的题目原型，其原始公式通常为 $f_{n+1} = \frac{f_n + f_{n-1}}{2}$。
   对于公式 $f_{n+1} = \frac{f_n + f_{n-1}}{2}$：
   - $f_n = \frac{2}{3}(1 - (-\frac{1}{2})^n)$
   - 当 $n \to \infty$ 时，$f_n \to 2/3$。
   
   由于选项 B 为 $2/3$ 且它是此类“加权平均递推”问题中最具代表性的答案，可以推断题目中的系数 $2$ 和分母 $3$ 可能是排版错误，原题意图考察的是极限为 $2/3$ 的模型。在考场上遇到此类选项不匹配的情况，应优先选择最接近经典模型且在选项中的答案。

11), 设 A 和 B 是两个长为 n 的有序数组，现在需要将 A 和 B 合并成一个排好序的 数组，请问任何以元素比较作为基本运算的归并算法最坏情况下至少要做（ ）次比较。
{{ select(11) }}
- A. 𝑛 2 n 2
- B. 𝑛 log ⁡ 𝑛 nlogn
- C. 2 𝑛 2n
- D. 2 𝑛 − 1 2n−1

> **答案**: D
> **解析**: 这道题考察的是归并排序中“合并（Merge）”操作的时间复杂度分析。

1. 归并过程原理：
在合并两个长度为 n 的有序数组 A 和 B 时，我们通常使用两个指针分别指向两个数组的起始位置。每次比较两个指针所指的元素，将较小的元素放入结果数组中，并将该指针后移一位。

2. 比较次数分析：
- 每进行一次比较，就会有一个元素被确定最终位置并放入结果数组。
- 两个数组共有 2n 个元素。
- 在最坏的情况下（即两个数组的元素交替大小，例如 A={1, 3, 5}, B={2, 4, 6}），我们需要不断比较直到其中一个数组只剩下一个元素。
- 当其中一个数组的所有元素都已放入结果数组，或者其中一个数组只剩最后一个元素且该元素比另一个数组剩余的所有元素都大时，比较停止。
- 具体来说，当已经放好了 2n - 1 个元素时，最后一个元素不需要再进行比较，直接放入结果数组即可。

3. 结论：
因此，在最坏情况下，合并两个长度为 n 的有序数组所需的比较次数为 (n + n - 1) = 2n - 1 次。

故正确答案为 D。

12), 在n(n≥3)枚硬币中有一枚质量不合格的硬币（质量过轻或质量过重）,如果只有一架天平可以用来称重且称重的硬币数没有限制，下面是找出这枚不合格的硬币的算法。请把 a-c 三行代码补全到算法中。a. A ← X ∪ Yb. A ← Zc. n ← |A|算法Coin(A, n) 正确的填空顺序是（ ）
{{ select(12) }}
- A. b, c, a
- B. c, b, a
- C. c, a, b
- D. a, b, c

> **答案**: D
> **解析**: 这道题目考查的是分治算法在“找假币问题”中的应用。

1. **算法逻辑分析**：
   在 $n$ 枚硬币中寻找一枚不合格（轻或重）的硬币，通常采用分治策略。将硬币集合 $A$ 分成三部分：$X$、$Y$ 和 $Z$，其中 $X$ 和 $Y$ 的硬币数量相等（$|X| = |Y|$）。

2. **称重过程**：
   使用天平称量 $X$ 和 $Y$ 两组硬币：
   - **情况 1**：如果天平不平衡（$X \neq Y$），说明不合格的硬币一定在 $X$ 或 $Y$ 组中。此时，新的待查集合应该是 $X$ 和 $Y$ 的并集。对应代码 **a. $A \leftarrow X \cup Y$**。
   - **情况 2**：如果天平平衡（$X = Y$），说明 $X$ 和 $Y$ 中的硬币都是合格的，不合格的硬币一定在剩余的 $Z$ 组中。此时，新的待查集合应该是 $Z$。对应代码 **b. $A \leftarrow Z$**。

3. **更新规模**：
   在确定了新的待查集合 $A$ 之后，需要更新集合中硬币的数量 $n$，以便进行下一次递归或循环。对应代码 **c. $n \leftarrow |A|$**。

4. **填空顺序**：
   在算法的伪代码结构中，通常先通过条件判断（if-else）来决定如何缩小搜索范围（即更新 $A$），然后再统一更新规模参数 $n$。
   结构如下：
   ```
   if weight(X) != weight(Y):
       A ← X ∪ Y  (代码 a)
   else:
       A ← Z      (代码 b)
   n ← |A|        (代码 c)
   ```
   因此，代码补全的正确顺序是 a, b, c。

故正确答案为 D。

14), 小明要去南美洲旅游，一共乘坐三趟航班才能到达目的地，其中第 1 个航班准点的概率是 0.9，第 2 个航班准点的概率为 0.8，第 3 个航班准点的概率为 0.9。如果存在第 i 个（i=1,2）航班晚点，第 i+1 个航班准点，则小明将赶不上第 i+1 个航班，旅行失败；除了这种情况，其他情况下旅行都能成功。请问小明此次旅行成功的概率是（ ）
{{ select(14) }}
- A. 0.5
- B. 0.648
- C. 0.72
- D. 0.74

> **答案**: D
> **解析**: 这是一道关于概率计算的题目。我们需要计算小明旅行成功的概率。

1. **分析题目条件**：
   - 航班 1 准点概率 $P(T_1) = 0.9$，晚点概率 $P(L_1) = 0.1$。
   - 航班 2 准点概率 $P(T_2) = 0.8$，晚点概率 $P(L_2) = 0.2$。
   - 航班 3 准点概率 $P(T_3) = 0.9$，晚点概率 $P(L_3) = 0.1$。
   - **失败条件**：存在第 $i$ 个航班晚点且第 $i+1$ 个航班准点（$i=1, 2$）。
     即：(航班 1 晚点且航班 2 准点) 或 (航班 2 晚点且航班 3 准点) 时，旅行失败。

2. **计算失败的概率**：
   - 事件 A（$i=1$ 失败）：航班 1 晚点且航班 2 准点。
     $P(A) = P(L_1) \times P(T_2) = 0.1 \times 0.8 = 0.08$。
   - 事件 B（$i=2$ 失败）：航班 2 晚点且航班 3 准点。
     $P(B) = P(L_2) \times P(T_3) = 0.2 \times 0.9 = 0.18$。
   - 注意：事件 A 发生时要求航班 2 准点（$T_2$），事件 B 发生时要求航班 2 晚点（$L_2$）。由于一个航班不能既准点又晚点，因此事件 A 和事件 B 是互斥事件（不能同时发生）。
   - 总失败概率 $P(\text{Failure}) = P(A) + P(B) = 0.08 + 0.18 = 0.26$。

3. **计算成功的概率**：
   - 成功概率 $P(\text{Success}) = 1 - P(\text{Failure}) = 1 - 0.26 = 0.74$。

也可以通过列举所有 8 种可能的情况来验证：
- $T_1, T_2, T_3$: $0.9 \times 0.8 \times 0.9 = 0.648$ (成功)
- $T_1, T_2, L_3$: $0.9 \times 0.8 \times 0.1 = 0.072$ (成功)
- $T_1, L_2, T_3$: $0.9 \times 0.2 \times 0.9 = 0.162$ (失败)
- $T_1, L_2, L_3$: $0.9 \times 0.2 \times 0.1 = 0.018$ (成功)
- $L_1, T_2, T_3$: $0.1 \times 0.8 \times 0.9 = 0.072$ (失败)
- $L_1, T_2, L_3$: $0.1 \times 0.8 \times 0.1 = 0.008$ (失败)
- $L_1, L_2, T_3$: $0.1 \times 0.2 \times 0.9 = 0.018$ (失败)
- $L_1, L_2, L_3$: $0.1 \times 0.2 \times 0.1 = 0.002$ (成功)
成功概率总和 = $0.648 + 0.072 + 0.018 + 0.002 = 0.74$。

因此，正确答案是 D。

{{ select(17) }}
- A. 𝑎 , 𝑏 , 𝑐 , 𝑑 , 𝑒 , 𝑓 , 𝑔 a,b,c,d,e,f,g
- B. 𝑎 , 𝑑 , 𝑐 , 𝑏 , 𝑒 , 𝑔 , 𝑓 a,d,c,b,e,g,f
- C. 𝑎 , 𝑑 , 𝑏 , 𝑐 , 𝑔 , 𝑓 , 𝑒 a,d,b,c,g,f,e
- D. 𝑔 , 𝑓 , 𝑒 , 𝑑 , 𝑐 , 𝑏 , 𝑎 g,f,e,d,c,b,a

> **答案**: C
> **解析**: 栈（Stack）是一种后进先出（LIFO, Last-In-First-Out）的数据结构。在处理出栈序列问题时，一个核心规律是：如果元素 x 在元素 y 之前入栈，且 y 在 x 之前出栈，那么在 y 出栈时，x 必然已经在栈中。此时，任何在 x 之后、y 之前入栈的元素，如果还没出栈，就必须按照它们入栈顺序的逆序依次出栈。

我们逐一分析选项：

A. a, b, c, d, e, f, g：每次入栈一个元素就立即出栈一个元素。这是合法的。
B. a, d, c, b, e, g, f：
   - 入 a，出 a。
   - 入 b, c, d，出 d，此时栈内从底向上为 [b, c]。
   - 出 c，出 b。
   - 入 e，出 e。
   - 入 f, g，出 g，出 f。
   这是合法的。
C. a, d, b, c, g, f, e：
   - 入 a，出 a。
   - 入 b, c, d，出 d。此时栈内从底向上依次是 [b, c]。
   - 根据后进先出原则，下一个出栈的必须是栈顶元素 c，而选项中给出的下一个出栈元素是 b。
   - 在 c 还没出栈的情况下，位于 c 下方的 b 是不可能先于 c 出栈的。
   因此，该序列不合法。
D. g, f, e, d, c, b, a：将 a, b, c, d, e, f, g 全部入栈后，再依次出栈。这是合法的。

综上所述，选项 C 是不可能的出栈序列。

18), 下列算法中，（ ）是稳定的排序算法。
{{ select(18) }}
- A. 快速排序
- B. 堆排序
- C. 希尔排序
- D. 插入排序

> **答案**: D
> **解析**: 1. 排序算法稳定性的定义：
稳定性是指在待排序的记录序列中，存在多个具有相同关键字的记录。若经过排序后，这些记录的相对次序保持不变，则称这种排序算法是稳定的；否则称为不稳定的。

2. 选项分析：
- A. 快速排序：在进行分区（Partition）操作时，通过交换元素来将序列分为两部分。这种跨距离的交换往往会破坏相同元素的原始相对顺序，因此是不稳定的。
- B. 堆排序：在建立大顶堆以及将堆顶元素与堆尾元素交换的过程中，相同大小的元素可能会因为位置调整而改变相对顺序，因此是不稳定的。
- C. 希尔排序：希尔排序是插入排序的改进版，它通过设置增量（Gap）将序列分成若干子序列分别进行插入排序。虽然单次插入排序是稳定的，但在不同的增量分组中，相同元素可能会被分配到不同的组中进行处理，从而导致相对顺序改变，因此是不稳定的。
- D. 插入排序：在插入过程中，算法将当前元素与前面已排序的元素逐一比较。只有当待插入元素“严格小于”前面的元素时才会进行移动；如果遇到“相等”的元素，待插入元素会直接放在该相等元素的后面，从而保证了相对顺序不变，因此是稳定的。

3. 总结：
常见的稳定排序算法包括：冒泡排序、插入排序、归并排序、基数排序等。
常见的不稳定排序算法包括：选择排序、快速排序、堆排序、希尔排序等。

因此，本题正确答案为 D。
