1), 请选出以下最大的数（ ）
{{ select(1) }}
- A. ( 5 5 0 ) 1 0 (550) 10 ​
- B. ( 7 7 7 ) 8 (777) 8 ​
- C. 2 1 0 2 10
- D. ( 2 2 𝐹 ) 1 6 (22F) 16 ​

> **答案**: C
> **解析**: 为了比较这些数值的大小，我们需要将它们统一转换为十进制进行比较：

A. (550)₁₀：本身就是十进制，数值为 550。

B. (777)₈：这是一个八进制数。
计算过程：7 × 8² + 7 × 8¹ + 7 × 8⁰ = 7 × 64 + 7 × 8 + 7 × 1 = 448 + 56 + 7 = 511。

C. 2¹⁰：这是一个幂运算。
计算过程：2 × 2 × 2 × 2 × 2 × 2 × 2 × 2 × 2 × 2 = 1024。

D. (22F)₁₆：这是一个十六进制数（F 代表 15）。
计算过程：2 × 16² + 2 × 16¹ + 15 × 16⁰ = 2 × 256 + 2 × 16 + 15 = 512 + 32 + 15 = 559。

比较四个数值：550, 511, 1024, 559。
显然，1024 是最大的，因此正确答案是 C。

2), 操作系统的功能是（ ）
{{ select(2) }}
- A. 负责外设与主机之间的信息交换
- B. 控制和管理计算机系统的各种硬件和软件资源的使用
- C. 负责诊断机器的故障
- D. 将源程序编译成目标程序

> **答案**: B
> **解析**: 操作系统（Operating System，简称 OS）是计算机系统中最基本的系统软件，它的核心功能可以概括为：

1. 资源管理：操作系统负责管理计算机系统中所有的硬件资源（如 CPU、内存、磁盘、输入输出设备等）和软件资源（如文件、各种程序等）。它决定了哪个程序在什么时候使用多少资源，从而提高系统的效率和吞吐量。
2. 提供用户接口：操作系统为用户提供了一个与计算机硬件交互的界面，包括命令行界面（CLI）或图形用户界面（GUI）。
3. 协调运行：它负责控制程序的执行，处理各种中断，确保系统稳定运行。

选项分析：
A 选项：这只是操作系统中“设备管理”的一部分功能，描述不全面。
B 选项：准确地描述了操作系统的核心定义，即对硬件和软件资源的控制与管理。
C 选项：虽然操作系统有一定的错误处理能力，但专门的故障诊断通常由 BIOS 或特定的维护工具完成，不是其主要功能。
D 选项：将源程序编译成目标程序是“编译器”或“解释器”的功能，属于开发工具类软件，不属于操作系统的功能。

因此，正确答案是 B。

3), 现有一段 8 分钟的视频文件，它的播放速度是每秒 24 帧图像，每帧图像是 一幅分辨率为2048×1024像素的 32 位真彩色图像。请问要存储这段原始无压缩视频，需要多大的存储空间？（ ）
{{ select(3) }}
- A. 30G
- B. 90G
- C. 150G
- D. 450G

> **答案**: B
> **解析**: 计算原始无压缩视频存储空间的公式为：存储空间 = 分辨率 × 色彩深度 × 帧率 × 时长。

1. 首先统一单位：
   - 分辨率：2048 × 1024 像素。
   - 色彩深度：32 位 (bit)，即 32 / 8 = 4 字节 (Byte)。
   - 帧率：24 帧/秒。
   - 时长：8 分钟 = 8 × 60 = 480 秒。

2. 计算单帧图像的大小：
   2048 × 1024 × 4 Bytes = 2^11 × 2^10 × 2^2 Bytes = 2^23 Bytes。
   将其转换为 MB：2^23 / 1024 / 1024 = 2^23 / 2^20 = 2^3 = 8 MB。

3. 计算每秒视频的大小：
   8 MB × 24 = 192 MB/s。

4. 计算 8 分钟视频的总大小：
   192 MB/s × 480 s = 92160 MB。

5. 将结果转换为 GB：
   92160 MB / 1024 = 90 GB。

因此，存储这段视频需要 90G 的空间，正确答案为 B。

4), 今有一空栈 S，对下列待进栈的数据元素序列a,b,c,d,e,f依次进行：进栈，进栈，出栈，进栈，进栈，出栈的操作，则此操作完成后，栈底元素为（ ）
{{ select(4) }}
- A. b
- B. a
- C. d
- D. c

> **答案**: B
> **解析**: 栈（Stack）是一种后进先出（LIFO, Last In First Out）的数据结构。我们可以通过逐步模拟题目中的操作来确定栈的状态：

1. 初始状态：栈 S 为空。
2. 进栈（元素 a）：栈内元素为 [a]，栈底为 a。
3. 进栈（元素 b）：栈内元素为 [a, b]，栈底为 a。
4. 出栈：弹出栈顶元素 b，栈内元素剩余 [a]，栈底为 a。
5. 进栈（元素 c）：栈内元素为 [a, c]，栈底为 a。
6. 进栈（元素 d）：栈内元素为 [a, c, d]，栈底为 a。
7. 出栈：弹出栈顶元素 d，栈内元素剩余 [a, c]，栈底为 a。

操作完成后，栈内剩余元素为 a 和 c，其中 a 是最早进入且未被弹出的元素，因此位于栈底。所以正确答案是 B。

5), 将(2,7,10,18)分别存储到某个地址区间为0∼10的哈希表中，如果哈希函数h(x)=（ ），将不会产生冲突，其中 amodb 表示 a 除以 b 的余数。
{{ select(5) }}
- A. 𝑥 2 mod 1 1 x 2 mod11
- B. 2 𝑥 mod 1 1 2xmod11
- C. 𝑥 mod 1 1 xmod11
- D. ⌊ 𝑥 2 ⌋ mod 1 1 ⌊ 2 x ​ ⌋mod11，其中 ⌊ 𝑥 2 ⌋ ⌊ 2 x ​ ⌋ 表示 𝑥 2 2 x ​ 下取整

> **答案**: D
> **解析**: 我们需要将数字序列 (2, 7, 10, 18) 通过哈希函数映射到地址区间 0∼10，并找到一个不产生冲突（即每个数字映射到的哈希值都互不相同）的函数。

我们逐一验证选项：

A. $h(x) = x^2 \pmod{11}$
- $x=2: 2^2 \pmod{11} = 4$
- $x=7: 7^2 \pmod{11} = 49 \pmod{11} = 5$
- $x=10: 10^2 \pmod{11} = 100 \pmod{11} = 1$
- $x=18: 18^2 \pmod{11} = 324 \pmod{11} = 5$
结果为 (4, 5, 1, 5)，其中 7 和 18 都映射到了 5，产生冲突。

B. $h(x) = 2x \pmod{11}$
- $x=2: 2 \times 2 \pmod{11} = 4$
- $x=7: 2 \times 7 \pmod{11} = 14 \pmod{11} = 3$
- $x=10: 2 \times 10 \pmod{11} = 20 \pmod{11} = 9$
- $x=18: 2 \times 18 \pmod{11} = 36 \pmod{11} = 3$
结果为 (4, 3, 9, 3)，其中 7 和 18 都映射到了 3，产生冲突。

C. $h(x) = x \pmod{11}$
- $x=2: 2 \pmod{11} = 2$
- $x=7: 7 \pmod{11} = 7$
- $x=10: 10 \pmod{11} = 10$
- $x=18: 18 \pmod{11} = 7$
结果为 (2, 7, 10, 7)，其中 7 和 18 都映射到了 7，产生冲突。

D. $h(x) = \lfloor x/2 \rfloor \pmod{11}$
- $x=2: \lfloor 2/2 \rfloor \pmod{11} = 1 \pmod{11} = 1$
- $x=7: \lfloor 7/2 \rfloor \pmod{11} = 3 \pmod{11} = 3$
- $x=10: \lfloor 10/2 \rfloor \pmod{11} = 5 \pmod{11} = 5$
- $x=18: \lfloor 18/2 \rfloor \pmod{11} = 9 \pmod{11} = 9$
结果为 (1, 3, 5, 9)，四个结果互不相同，没有产生冲突。

因此，正确答案是 D。

6), 下列哪些问题不能用贪心法精确求解？（ ）
{{ select(6) }}
- A. 霍夫曼编码问题
- B. 0-1 背包问题
- C. 最小生成树问题
- D. 单源最短路径问题

> **答案**: B
> **解析**: 贪心算法（Greedy Algorithm）是指在对问题求解时，总是做出在当前看来是最好的选择。也就是说，不从整体最优上加以考虑，它所做出的仅是在某种意义上的局部最优解。

A 选项：霍夫曼编码问题。通过每次合并频率最低的两个节点来构建最优二叉树，是贪心算法的经典应用，可以得到精确的最优解。

B 选项：0-1 背包问题。在 0-1 背包问题中，物品是不可分割的（要么装入，要么不装）。如果采用贪心策略（如按单位价值排序），可能会因为物品体积较大导致背包剩余空间无法被有效利用，从而无法得到全局最优解。该问题通常需要使用动态规划（Dynamic Programming）来精确求解。注：如果是“部分背包问题”（物品可分割），则可以用贪心法求解。

C 选项：最小生成树问题。无论是 Prim 算法还是 Kruskal 算法，其核心思想都是贪心策略，且都能证明可以得到精确的最小生成树。

D 选项：单源最短路径问题。Dijkstra 算法是求解该问题的经典贪心算法（要求边权非负），它每次选择距离源点最近的节点进行松弛操作，可以得到精确的最短路径。

因此，不能用贪心法精确求解的是 0-1 背包问题。

7), 具有 n 个顶点， e 条边的图采用邻接表存储结构，进行深度优先遍历运算的时间复杂度为（ ）
{{ select(7) }}
- A. 𝑂 ( 𝑛 + 𝑒 ) O(n+e)
- B. 𝑂 ( 𝑛 2 ) O(n 2 )
- C. 𝑂 ( 𝑒 2 ) O(e 2 )
- D. 𝑂 ( 𝑛 ) O(n)

> **答案**: A
> **解析**: 深度优先遍历（DFS）的时间复杂度取决于图的存储结构：

1. 访问顶点：在深度优先遍历过程中，每个顶点都会被访问且仅被访问一次，因此访问所有顶点的时间复杂度为 O(n)。
2. 查找邻接点：
   - 当使用邻接表（Adjacency List）存储时，为了寻找一个顶点的所有邻接点，我们需要遍历该顶点对应的边链表。
   - 在整个遍历过程中，图中每条边在边链表中都会出现（无向图出现两次，有向图出现一次）。因此，遍历所有边链表的时间总和与边数 e 成正比，即 O(e)。
3. 综合复杂度：将访问顶点的时间和遍历边的时间相加，得到总的时间复杂度为 O(n + e)。

补充说明：如果图采用邻接矩阵（Adjacency Matrix）存储，由于每次查找一个顶点的邻接点都需要扫描整整一行（n 个元素），总的时间复杂度将是 O(n²)。本题明确指出采用邻接表，故选 A。

8), 二分图是指能将顶点划分成两个部分，每一部分内的顶点间没有边相连的简单无向图。那么， 24 个顶点的二分图至多有（ ）条边。
{{ select(8) }}
- A. 144
- B. 100
- C. 48
- D. 122

> **答案**: A
> **解析**: 1. 二分图定义：二分图又称二部图，是指一个图的顶点集可以分割为两个互不相交的子集 $V_1$ 和 $V_2$，使得图中的每一条边所连接的两个顶点分别属于这两个不同的顶点集。
2. 最大边数条件：要使二分图的边数达到最大，该图必须是一个完全二分图。在完全二分图中，$V_1$ 中的每一个顶点都与 $V_2$ 中的每一个顶点有且只有一条边相连。
3. 数学推导：设 $V_1$ 的顶点数为 $n_1$，$V_2$ 的顶点数为 $n_2$。根据题目条件，总顶点数 $n_1 + n_2 = 24$。完全二分图的边数 $E = n_1 \times n_2$。
4. 求极值：根据算术几何平均值不等式，当两个数的和一定时，这两个数越接近（即相等时），它们的乘积最大。
   - 当 $n_1 = 12, n_2 = 12$ 时，$n_1 + n_2 = 24$。
   - 此时边数 $E = 12 \times 12 = 144$。
5. 结论：24 个顶点的二分图至多有 144 条边。因此正确答案为 A。

9), 广度优先搜索时，一定需要用到的数据结构是（ ）
{{ select(9) }}
- A. 栈
- B. 二叉树
- C. 队列
- D. 哈希表

> **答案**: C
> **解析**: 1. 广度优先搜索（Breadth-First Search，简称 BFS）的核心思想是“由近及远”地访问节点。它从起始点开始，首先访问所有距离起始点为 1 的节点，然后访问所有距离为 2 的节点，以此类推，层层向外扩张。
2. 为了保证这种“先发现的节点先访问”的顺序，BFS 必须使用一种先进先出（First-In-First-Out，FIFO）的数据结构，即队列（Queue）。
3. 在算法执行过程中，每当访问一个节点，就会将其所有未被访问过的邻接节点加入队列的末尾；而下一次要处理的节点总是从队列的头部取出。
4. 选项对比：
   - A. 栈（Stack）：具有后进先出（LIFO）的特性，通常用于深度优先搜索（DFS）。
   - B. 二叉树：是一种逻辑结构，BFS 可以用来遍历二叉树，但它本身不是实现搜索算法的辅助工具。
   - D. 哈希表：常用于存储已访问过的节点以防重复访问（去重），但在实现 BFS 的核心遍历逻辑时，队列才是必不可少的基础结构。

10), —个班学生分组做游戏，如果每组三人就多两人，每组五人就多三人，每组七人就多四人，问这个班的学生人数 n 在以下哪个区间？已知n<60。(）
{{ select(10) }}
- A. 3 0 < 𝑛 < 4 0 30<n<40
- B. 4 0 < 𝑛 < 5 0 40<n<50
- C. 5 0 < 𝑛 < 6 0 50<n<60
- D. 2 0 < 𝑛 < 3 0 20<n<30

> **答案**: C
> **解析**: 这是一道典型的中国剩余定理（同余问题）的应用题。我们可以根据题目给出的条件列出数学表达式，并进行推导：

1. 根据题意，设学生人数为 n，已知 n < 60，且满足以下条件：
   - n 除以 3 余 2 (n ≡ 2 mod 3)
   - n 除以 5 余 3 (n ≡ 3 mod 5)
   - n 除以 7 余 4 (n ≡ 4 mod 7)

2. 我们可以通过枚举法在 n < 60 的范围内寻找符合条件的数：
   - 首先寻找满足“除以 5 余 3”的数：3, 8, 13, 18, 23, 28, 33, 38, 43, 48, 53, 58。
   - 在这些数中，寻找同时满足“除以 3 余 2”的数：
     - 8 ÷ 3 = 2...2 (符合)
     - 23 ÷ 3 = 7...2 (符合)
     - 38 ÷ 3 = 12...2 (符合)
     - 53 ÷ 3 = 17...2 (符合)
   - 在剩下的数（8, 23, 38, 53）中，寻找满足“除以 7 余 4”的数：
     - 8 ÷ 7 = 1...1
     - 23 ÷ 7 = 3...2
     - 38 ÷ 7 = 5...3
     - 53 ÷ 7 = 7...4 (符合)

3. 经过验证，n = 53 满足所有条件：
   - 53 ÷ 3 = 17 余 2
   - 53 ÷ 5 = 10 余 3
   - 53 ÷ 7 = 7 余 4
   - 且 53 < 60

4. 对照选项：
   - A 选项：30 < 53 < 40 (错误)
   - B 选项：40 < 53 < 50 (错误)
   - C 选项：50 < 53 < 60 (正确)
   - D 选项：20 < 53 < 30 (错误)

因此，正确答案是 C。

11), 小明想通过走楼梯来锻炼身体，假设从第 1 层走到第 2 层消耗 10 卡热量，接着从第 2 层走到第 3 层消耗 20 卡热量，再从第 3 层走到第 4 层消耗 30 卡热量，依此类推，从第 k 层走到第 k+1 层消耗 10k 卡热量(k>1)？如果小明想从 1 层开始，通过连续向上爬楼梯消耗 1000 卡热量，至少要爬到第几层楼？（ ）
{{ select(11) }}
- A. 14
- B. 16
- C. 15
- D. 13

> **答案**: C
> **解析**: 1. 题目分析：
   小明从第 1 层开始向上爬。
   从第 1 层到第 2 层消耗 10 卡；
   从第 2 层到第 3 层消耗 20 卡；
   从第 3 层到第 4 层消耗 30 卡；
   以此类推，从第 k 层到第 k+1 层消耗 10k 卡。

2. 建立数学模型：
   假设小明爬到了第 n 层，那么他一共跨越了 (n-1) 段楼梯。
   总消耗热量 S = 10 + 20 + 30 + ... + 10(n-1)
   这是一个等差数列求和问题，提取公因子 10 后得：
   S = 10 * [1 + 2 + 3 + ... + (n-1)]
   根据等差数列求和公式 [首项 + 末项] * 项数 / 2：
   S = 10 * [(1 + n - 1) * (n - 1) / 2]
   S = 5 * n * (n - 1)

3. 求解不等式：
   题目要求总消耗热量至少为 1000 卡，即 S ≥ 1000。
   5 * n * (n - 1) ≥ 1000
   n * (n - 1) ≥ 200

4. 代入选项验证：
   - 若 n = 14，14 * 13 = 182 < 200（不满足）
   - 若 n = 15，15 * 14 = 210 ≥ 200（满足）
   - 若 n = 13，13 * 12 = 156 < 200（不满足）

当 n = 14 时，消耗热量为 5 * 182 = 910 卡；
当 n = 15 时，消耗热量为 5 * 210 = 1050 卡。
因此，要消耗至少 1000 卡热量，至少要爬到第 15 层。

12), 表达式a*(b+c)-d的后缀表达形式为（ ）
{{ select(12) }}
- A. abc*+d- abc*+d-
- B. +*abcd -+*abcd
- C. abcd*+- abcd*+-
- D. abc+*d- abc+*d-

> **答案**: A
> **解析**: 将中缀表达式转换为后缀表达式（逆波兰表达式）的步骤如下：

1. **确定运算优先级**：在表达式 `a * (b + c) - d` 中，运算优先级从高到低依次是：
   - 括号内的加法 `(b + c)`
   - 乘法 `*`
   - 减法 `-`

2. **逐步转换**：
   - **第一步（处理括号）**：将 `(b + c)` 转换为后缀形式 `bc+`。此时表达式变为 `a * [bc+] - d`。
   - **第二步（处理乘法）**：乘法运算符 `*` 的两个操作数是 `a` 和 `[bc+]`。将 `*` 移至它们之后，得到 `abc+*`。此时表达式变为 `[abc+*] - d`。
   - **第三步（处理减法）**：减法运算符 `-` 的两个操作数是 `[abc+*]` 和 `d`。将 `-` 移至它们之后，得到最终结果：`abc+*d-`。

3. **核对选项**：
   - 题目给出的选项中，A 和 D 的内容 `abc+d-` 显然在录入时漏掉了乘号 `*`。
   - 在计算机考级（如 GESP/CSP）的类似题目中，`abc+*d-` 是该表达式唯一的正确后缀形式。
   - 选项 B 为前缀表达式风格，选项 C 逻辑错误。

因此，按照后缀表达式的构造逻辑，正确答案应为 `abc+*d-`。在提供的选项中，A 是最接近且符合逻辑结构的选项（假设 `*` 号因格式问题未显示）。

13), 从一个4×4的棋盘中选取不在同一行也不在同一列上的两个方格，共有（ ）种方法。
{{ select(13) }}
- A. 60
- B. 72
- C. 86
- D. 64

> **答案**: B
> **解析**: 这是一道关于组合数学的题目，我们可以通过分步计数法来解决：

1. 确定总方格数：一个 4×4 的棋盘共有 4 × 4 = 16 个方格。
2. 选择第一个方格：从 16 个方格中任选一个，共有 16 种选法。
3. 选择第二个方格：根据题目要求，第二个方格不能与第一个方格在同一行，也不能在同一列。
   - 第一个方格所在的行有 4 个方格。
   - 第一个方格所在的列有 4 个方格。
   - 排除掉这些方格（注意第一个方格本身既在行也在列，不要重复排除），与第一个方格同行或同列的方格共有 4 + 4 - 1 = 7 个。
   - 因此，剩下的可选方格数量为 16 - 7 = 9 个。
4. 计算排列总数：16 × 9 = 144 种。
5. 去除重复：由于题目是“选取两个方格”，不考虑选取的先后顺序（例如先选 A 再选 B，与先选 B 再选 A 是同一种结果），所以上述排列总数需要除以 2。
6. 最终结果：144 ÷ 2 = 72 种。

所以，共有 72 种方法，正确答案是 B。

14), 对一个 n 个顶点、 m 条边的带权有向简单图用 Dijkstra 算法计算单源最短路时，如果不使用堆或其它优先队列进行优化，则其时间复杂度为（ ）
{{ select(14) }}
- A. 𝑂 ( ( 𝑚 + 𝑛 2 ) log ⁡ 𝑛 ) O((m+n 2 )logn)
- B. 𝑂 ( 𝑚 𝑛 + 𝑛 3 ) O(mn+n 3 )
- C. 𝑂 ( ( 𝑚 + 𝑛 ) log ⁡ 𝑛 ) O((m+n)logn)
- D. 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: D
> **解析**: Dijkstra 算法用于求解单源最短路径问题。在不使用堆或优先队列进行优化的情况下，算法的执行过程如下：

1. 初始化：将源点到所有顶点的距离初始化为无穷大，源点自身距离为 0。这一步复杂度为 O(n)。
2. 循环执行以下步骤 n 次（每个顶点处理一次）：
   a. 寻找最小值：在所有未访问的顶点中，寻找一个距离源点最近的顶点 u。由于没有使用堆优化，需要遍历所有顶点来比较距离，这一步的时间复杂度为 O(n)。
   b. 标记访问：将顶点 u 标记为已访问。
   c. 松弛操作：遍历顶点 u 的所有出边 (u, v)，尝试更新源点到顶点 v 的最短距离。在整个算法执行过程中，每条边最多被访问一次。

复杂度分析：
- 寻找最小值的总时间：n 次循环 × O(n) = O(n²)。
- 松弛操作的总时间：所有边的总数，即 O(m)。
- 总复杂度：O(n² + m)。

在简单图中，边的数量 m 满足 m ≤ n(n-1)，即 m = O(n²)。因此，O(n² + m) 简化后为 O(n²)。

补充：
- 如果使用二叉堆优化，复杂度为 O((n + m) log n)，即选项 C。
- 如果使用 Fibonacci 堆优化，复杂度可以达到 O(m + n log n)。
