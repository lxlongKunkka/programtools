1), 以补码存储的 8 位有符号整数 10110111 的十进制表示为
{{ select(1) }}
- A. − 7 3 −73
- B. 1 8 3 183
- C. − 7 2 −72
- D. 7 2 72

> **答案**: A
> **解析**: 在计算机系统中，有符号整数通常以补码形式存储。对于 8 位有符号整数 10110111，我们可以通过以下步骤将其转换为十进制：

1. 确定符号位：最高位（左起第一位）是 1，表示这是一个负数。
2. 转换方法（补码求原码）：
   - 方法一：逆向操作。补码减 1 得到反码，反码符号位不变、数值位取反得到原码。
     补码：10110111
     反码：10110110（补码减 1）
     原码：11001001（反码数值位取反）
     计算原码的数值部分：2^6 + 2^3 + 2^0 = 64 + 8 + 1 = 73。
     加上负号，结果为 -73。

   - 方法二：快捷公式。对补码（除符号位外）取反再加 1 得到其绝对值。
     补码数值位 0110111 取反得到 1001000。
     加 1 得到 1001001。
     计算 1001001 的十进制：64 + 8 + 1 = 73。
     因为符号位为 1，所以结果为 -73。

   - 方法三：权重法。最高位的权重为 -2^7 = -128。
     计算：-128 * 1 + 64 * 0 + 32 * 1 + 16 * 1 + 8 * 0 + 4 * 1 + 2 * 1 + 1 * 1
     = -128 + 32 + 16 + 4 + 2 + 1
     = -128 + 55
     = -73。

综上所述，该补码对应的十进制数为 -73，故选 A。

2), 现有一段 24 分钟的视频文件，它的帧率是 30Hz，分辨率是1920×1080，每帧图像都是 32 位真彩色图像，使用的视频编码算法达到了25%的压缩率。则这个视频文件占用的存储空间大小约是
{{ select(2) }}
- A. 668GiB
- B. 334GiB
- C. 85GiB
- D. 500GiB

> **答案**: C
> **解析**: 计算视频文件存储大小的步骤如下：

1. 计算未压缩视频的总像素点：
分辨率为 1920 × 1080，即每帧有 1920 × 1080 = 2,073,600 个像素。

2. 计算每帧图像的大小：
每帧是 32 位真彩色，32 bits = 4 Bytes（字节）。
每帧大小 = 2,073,600 × 4 = 8,294,400 字节。

3. 计算视频的总帧数：
视频时长为 24 分钟，帧率为 30Hz（每秒 30 帧）。
总秒数 = 24 × 60 = 1440 秒。
总帧数 = 1440 × 30 = 43,200 帧。

4. 计算未压缩视频的总大小：
总大小 = 43,200 帧 × 8,294,400 字节/帧 = 358,318,080,000 字节。

5. 计算压缩后的视频大小：
压缩率为 25%（即压缩后的大小为原大小的 25%）。
压缩后大小 = 358,318,080,000 × 25% = 89,579,520,000 字节。

6. 单位换算为 GiB：
1 GiB = 1024 × 1024 × 1024 字节 = 1,073,741,824 字节。
存储空间 = 89,579,520,000 / 1,073,741,824 ≈ 83.43 GiB。

在给出的选项中，85GiB 是最接近的估算值。因此，正确答案为 C。

4), 对一个 n 个顶点， m 条边的带正权有向简单图使用 Dijkstra 算法计算单源最短路时，如果再使用一个可以在Θ(logn)时间复杂度内查询堆内最小值、在Θ(n​)时间复杂度内合并两个堆、在Θ(1)时间复杂度内将堆内一个元素变小、在Θ(logn)时间复杂度内弹出堆内最小值的堆优化 Dijkstra 算法，则整个 Dijkstra 算法的时间复杂度为
{{ select(4) }}
- A. Θ ( 𝑛 𝑛 + 𝑚 log ⁡ 𝑛 ) Θ(n n ​ +mlogn)
- B. Θ ( ( 𝑛 + 𝑚 ) log ⁡ 𝑛 ) Θ((n+m)logn)
- C. Θ ( 𝑚 + 𝑛 log ⁡ 𝑛 ) Θ(m+nlogn)
- D. Θ ( 𝑚 𝑛 + 𝑛 log ⁡ 𝑛 ) Θ(m n ​ +nlogn)

> **答案**: C
> **解析**: 在堆优化的 Dijkstra 算法中，算法的整体时间复杂度由以下几个部分组成：

1.  **初始化**：对 $n$ 个顶点进行初始化（如设置距离为无穷大），时间复杂度为 $\Theta(n)$。
2.  **弹出最小值 (Extract-Min)**：在算法的执行过程中，每个顶点都会从堆中被弹出一次。题目给定弹出最小值的操作时间复杂度为 $\Theta(\log n)$，因此 $n$ 次弹出操作的总时间复杂度为 $\Theta(n \log n)$。
3.  **查询最小值 (Query-Min)**：虽然题目给出查询最小值的时间复杂度为 $\Theta(\log n)$，但在 Dijkstra 算法中，查询最小值通常是与弹出最小值结合在一起的。即使单独计算，每个顶点弹出前查询一次，总时间复杂度也是 $\Theta(n \log n)$，不会改变整体复杂度的阶。
4.  **更新距离 (Decrease-Key)**：每当通过一条边进行松弛操作（Relaxation）并成功更新了目标顶点的最短距离时，就需要执行一次“将堆内一个元素变小”的操作。在最坏情况下，图中的 $m$ 条边都会触发一次该操作。题目给定该操作的时间复杂度为 $\Theta(1)$，因此 $m$ 次更新操作的总时间复杂度为 $\Theta(m \times 1) = \Theta(m)$。
5.  **合并堆 (Merge)**：标准的 Dijkstra 算法在执行过程中只维护一个堆，并不涉及合并两个堆的操作。因此，题目中给出的合并堆的时间复杂度 $\Theta(\sqrt{n})$ 是一个干扰项。

综上所述，整个算法的时间复杂度为：
$$ \text{初始化} + n \times \text{弹出最小值} + m \times \text{更新距离} $$
$$ = \Theta(n) + \Theta(n \log n) + \Theta(m) $$
$$ = \Theta(m + n \log n) $$

因此，正确选项为 C。

5), 具有 n 个顶点， m 条边的连通图采用邻接矩阵存储结构，进行深度优先遍历运算的时间复杂度为
{{ select(5) }}
- A. Θ ( 𝑛 3 ) Θ(n 3 )
- B. Θ ( 𝑛 2 ) Θ(n 2 )
- C. Θ ( 𝑛 + 𝑚 ) Θ(n+m)
- D. Θ ( 𝑚 2 ) Θ(m 2 )

> **答案**: B
> **解析**: 在图的遍历算法（包括深度优先遍历 DFS 和广度优先遍历 BFS）中，时间复杂度主要取决于所采用的存储结构：

1. 邻接矩阵存储结构：
   - 邻接矩阵是一个 n × n 的二维数组。
   - 在进行深度优先遍历时，对于每一个被访问的顶点，都需要在矩阵中扫描与其对应的整行元素，以确定哪些顶点是它的邻接点。
   - 每一行有 n 个元素，查找一个顶点的邻接点需要 O(n) 的时间。
   - 图中共有 n 个顶点，每个顶点都会被访问一次，因此总的时间复杂度为 O(n * n) = O(n²)。
   - 无论图中边的数量 m 是多少，扫描矩阵行的开销是固定的。

2. 邻接表存储结构：
   - 如果采用邻接表，访问所有顶点需要 O(n) 时间，检查所有边需要 O(m) 时间，总复杂度才是 O(n + m)。

本题明确指出采用“邻接矩阵”存储结构，因此正确答案为 B。

6), 下列算法中，没有运用分治思想的一项是
{{ select(6) }}
- A. 归并排序算法
- B. 求二叉树的前序遍历
- C. 快速排序算法
- D. 求二叉树的层次遍历

> **答案**: D
> **解析**: 分治思想（Divide and Conquer）的核心是将一个复杂的问题分解成两个或多个相同或相似的子问题，再将子问题的解合并以得到原问题的解。

A. 归并排序算法：典型的分治法。它将序列分成两半，分别对每一半进行排序，最后将两个有序序列合并。
B. 求二叉树的前序遍历：具有分治思想。遍历整棵树可以分解为：访问根节点、递归遍历左子树、递归遍历右子树。这符合将大问题分解为子问题的特征。
C. 快速排序算法：典型的分治法。它通过一个基准值将数组分为两部分，左边比基准小，右边比基准大，然后递归地对这两部分进行排序。
D. 求二叉树的层次遍历：通常使用广度优先搜索（BFS）实现，借助“队列”数据结构按照层级顺序逐个访问节点。这是一种迭代搜索的过程，并不涉及将问题划分为子问题并合并结果的分治逻辑。

因此，没有运用分治思想的是 D 选项。

7), 前缀表达式*+ab+cd的中缀形式是
{{ select(7) }}
- A. (a + b) * (c + d) (a + b) * (c + d)
- B. a + b * c + d a + b * c + d
- C. a * b + c * d a * b + c * d
- D. (d + c) * (b + a) (d + c) * (b + a)

> **答案**: A
> **解析**: 前缀表达式（又称波兰式）的转换可以通过从右向左扫描的方法来实现。

1. **转换步骤解析：**
   - 我们从前缀表达式 `* + a b + c d` 的右端开始向左扫描。
   - 遇到操作数 `d` 和 `c`，将它们依次放入栈中。
   - 遇到运算符 `+`，从栈中弹出两个操作数 `c` 和 `d`（先弹出的在左，后弹出的在右，或者根据前缀定义结合），组成中缀表达式 `(c + d)`，并将其作为一个整体放回栈中。
   - 继续向左扫描，遇到操作数 `b` 和 `a`，将它们放入栈中。
   - 遇到运算符 `+`，从栈中弹出 `a` 和 `b`，组成中缀表达式 `(a + b)`，放回栈中。
   - 最后遇到运算符 `*`，从栈中弹出 `(a + b)` 和 `(c + d)`，组成最终的中缀表达式 `(a + b) * (c + d)`。

2. **分层拆解法：**
   - 观察表达式 `* [+ a b] [+ c d]`。
   - 第一部分 `+ a b` 对应中缀的 `(a + b)`。
   - 第二部分 `+ c d` 对应中缀的 `(c + d)`。
   - 最外层的 `*` 将这两部分连接起来，即 `(a + b) * (c + d)`。

因此，正确答案是 A。选项中的重复文字应为格式排版问题，核心表达式一致。

8), 有 5 个从 1 到 5 标号的小球和 5 个同样标号的盒子，现将小球随机放入盒子，每个盒子仅放 1 个小球，问每个盒子中的小球都与盒子标号不同的概率是
{{ select(8) }}
- A. 2 4 6 2 5 625 24 ​
- B. 7 2 0 20 7 ​
- C. 4 3 1 2 0 120 43 ​
- D. 1 1 3 0 30 11 ​

> **答案**: D
> **解析**: 这是一道典型的组合数学中的“错排问题”（Derangement）。

1. 总情况数：
将 5 个不同标号的小球放入 5 个不同标号的盒子，每个盒子放一个球，总的排列方式为 5 的阶乘，即：
n! = 5! = 5 × 4 × 3 × 2 × 1 = 120 种。

2. 符合条件的情况数（错排数）：
错排问题是指将 n 个元素重新排列，使得每个元素都不在它原来的位置上。错排数通常用 D_n 表示。
错排公式为：D_n = (n-1)(D_{n-1} + D_{n-2})，其中 D_1 = 0, D_2 = 1。
根据公式计算：
D_1 = 0
D_2 = 1
D_3 = (3-1) × (1 + 0) = 2
D_4 = (4-1) × (2 + 1) = 9
D_5 = (5-1) × (9 + 2) = 4 × 11 = 44

或者使用通项公式计算：
D_5 = 5! × (1/0! - 1/1! + 1/2! - 1/3! + 1/4! - 1/5!)
D_5 = 120 × (1 - 1 + 1/2 - 1/6 + 1/24 - 1/120)
D_5 = 60 - 20 + 5 - 1 = 44 种。

3. 计算概率：
概率 P = 符合条件的情况数 / 总情况数 = 44 / 120。
对分数进行约分：44 / 120 = 11 / 30。

选项 D 中的数字排列“1 1 3 0 30 11”对应的是 11/30。因此正确答案为 D。

9), 设 x=true,y=false,z=true。以下逻辑运算表达式值为 true 的是
{{ select(9) }}
- A. ( ¬ 𝑥 ∨ 𝑦 ) ∧ 𝑧 (¬x∨y)∧z
- B. ( 𝑦 ∧ 𝑧 ) ∨ ( 𝑥 ∧ 𝑦 ) (y∧z)∨(x∧y)
- C. ( 𝑥 ∧ 𝑦 ) ∨ 𝑧 (x∧y)∨z
- D. ( 𝑥 ∧ 𝑧 ) ∧ 𝑦 (x∧z)∧y

> **答案**: C
> **解析**: 首先，我们需要明确题目中给出的变量值：x = true, y = false, z = true。
逻辑运算符的含义如下：
- ¬ (非)：取反，true 变 false，false 变 true。
- ∧ (与)：两者都为 true 时结果才为 true。
- ∨ (或)：只要有一个为 true，结果就为 true。

接下来逐一分析选项：

A. ( ¬ x ∨ y ) ∧ z
- ¬ x = ¬ true = false
- ( ¬ x ∨ y ) = ( false ∨ false ) = false
- ( ¬ x ∨ y ) ∧ z = false ∧ true = false

B. ( y ∧ z ) ∨ ( x ∧ y )
- ( y ∧ z ) = ( false ∧ true ) = false
- ( x ∧ y ) = ( true ∧ false ) = false
- ( y ∧ z ) ∨ ( x ∧ y ) = false ∨ false = false

C. ( x ∧ y ) ∨ z
- ( x ∧ y ) = ( true ∧ false ) = false
- ( x ∧ y ) ∨ z = false ∨ true = true
此项结果为 true。

D. ( x ∧ z ) ∧ y
- ( x ∧ z ) = ( true ∧ true ) = true
- ( x ∧ z ) ∧ y = true ∧ false = false

综上所述，只有选项 C 的逻辑运算表达式值为 true。

10), 假设某算法的计算时间表示为递推关系式T(n)=3T(2n​)+Θ(n)，T(1)=Θ(1)，则算法的时间复杂度为
{{ select(10) }}
- A. Θ ( 𝑛 ) Θ(n)
- B. Θ ( 𝑛 log ⁡ 2 3 ) Θ(n log 2 ​ 3 )
- C. Θ ( 𝑛 log ⁡ 𝑛 ) Θ(nlogn)
- D. Θ ( 𝑛 log ⁡ 2 3 log ⁡ 𝑛 ) Θ(n log 2 ​ 3 logn)

> **答案**: B
> **解析**: 本题考查算法的时间复杂度分析，通常使用主定理（Master Theorem）来解决递推关系式。

1. 主定理公式：
对于递推关系式 $T(n) = aT(n/b) + f(n)$，其中 $a \ge 1, b > 1$。
我们将 $f(n)$ 与 $n^{\log_b a}$ 进行比较：
- 情况1：如果 $f(n) = O(n^{\log_b a - \epsilon})$（其中 $\epsilon > 0$），则 $T(n) = \Theta(n^{\log_b a})$。
- 情况2：如果 $f(n) = \Theta(n^{\log_b a})$，则 $T(n) = \Theta(n^{\log_b a} \log n)$。
- 情况3：如果 $f(n) = \Omega(n^{\log_b a + \epsilon})$（其中 $\epsilon > 0$），且满足正则条件，则 $T(n) = \Theta(f(n))$。

2. 参数分析：
在本题中，$T(n) = 3T(n/2) + \Theta(n)$。
- $a = 3$
- $b = 2$
- $f(n) = \Theta(n)$

3. 计算比较项：
计算 $n^{\log_b a} = n^{\log_2 3}$。
因为 $\log_2 3 \approx 1.585$，所以 $n^{\log_2 3} \approx n^{1.585}$。

4. 比较与结论：
将 $f(n) = n^1$ 与 $n^{\log_2 3}$ 进行比较。
显然 $n^1 = O(n^{\log_2 3 - \epsilon})$（例如取 $\epsilon \approx 0.585$）。
这符合主定理的情况1。
因此，算法的时间复杂度为 $T(n) = \Theta(n^{\log_b a}) = \Theta(n^{\log_2 3})$。

故正确答案为 B。

11), 在一条长度为 1 的线段上随机取一个点，再在以原线段的左端点和取的该点为端点的线段上随机取一个点，则以取的两个点为端点的线段的期望长度是
{{ select(11) }}
- A. 1 2 2 1 ​
- B. 1 3 3 1 ​
- C. 1 4 4 1 ​
- D. 2 3 3 2 ​

> **答案**: C
> **解析**: 这是一道关于概率论中连续型随机变量期望值的题目。我们可以通过以下步骤得出答案：

1. 设定随机变量：
   - 设第一个点的位置为随机变量 $X$。根据题意，$X$ 在线段 $[0, 1]$ 上随机选取，因此 $X$ 服从均匀分布 $U(0, 1)$。
   - 设第二个点的位置为随机变量 $Y$。根据题意，$Y$ 是在左端点 $0$ 到第一个点 $X$ 之间随机选取的，因此在已知 $X=x$ 的条件下，$Y$ 服从均匀分布 $U(0, x)$。

2. 确定目标线段长度：
   - 题目要求的是以这两个点为端点的线段长度的期望值。由于 $Y$ 是在 $[0, X]$ 之间选取的，所以 $X \ge Y$，线段长度 $L = X - Y$。

3. 利用全期望公式计算：
   - 首先计算在 $X$ 确定的情况下，$L$ 的条件期望 $E[L|X]$。
   - 因为 $Y$ 在 $[0, X]$ 上均匀分布，所以 $Y$ 的条件期望 $E[Y|X] = \frac{X}{2}$。
   - 那么，条件期望 $E[L|X] = E[X - Y | X] = X - E[Y|X] = X - \frac{X}{2} = \frac{X}{2}$。

4. 计算最终期望：
   - 根据全期望公式 $E[L] = E[E[L|X]]$，我们有：
     $E[L] = E[\frac{X}{2}] = \frac{1}{2} E[X]$
   - 因为 $X$ 在 $[0, 1]$ 上均匀分布，其期望值 $E[X] = \frac{1}{2}$。
   - 所以，$E[L] = \frac{1}{2} \times \frac{1}{2} = \frac{1}{4}$。

题目选项中的 "1 4 4 1" 对应分数 $\frac{1}{4}$，因此正确答案为 C。

12), 以下排序算法中最好情况下时间复杂度与最坏情况下时间复杂度相同的是
{{ select(12) }}
- A. 选择排序
- B. 冒泡排序
- C. 插入排序
- D. 快速排序

> **答案**: A
> **解析**: 1. 选择排序（Selection Sort）：其基本思想是每一趟从待排序的数据元素中选出最小（或最大）的一个元素，存放在序列的起始位置。无论数组初始状态是有序、逆序还是随机分布，选择排序都需要进行相同次数的比较。对于长度为 n 的序列，它总是需要进行 n-1 轮选择，总的比较次数固定为 (n-1) + (n-2) + ... + 1 = n(n-1)/2。因此，它的最好、最坏和平均时间复杂度均为 O(n²)。

2. 冒泡排序（Bubble Sort）：在标准优化（增加交换标志位）的情况下，如果输入数组已经是有序的，只需进行一轮比较（n-1 次）即可结束，最好时间复杂度为 O(n)；而最坏情况下（逆序）需要 O(n²) 的时间。

3. 插入排序（Insertion Sort）：如果输入数组已经是有序的，每个元素只需与前一个元素比较一次，不需要移动，最好时间复杂度为 O(n)；而最坏情况下（逆序）需要 O(n²) 的时间。

4. 快速排序（Quick Sort）：其性能取决于基准值（pivot）的选择。在最好情况下（每次划分都能将数组平分），时间复杂度为 O(n log n)；在最坏情况下（例如数组已经有序且每次选择第一个元素作为基准），会退化为 O(n²)。

综上所述，只有选择排序在最好和最坏情况下的时间复杂度保持一致。

13), 有 4 个结点和 4 条边的有标号简单无向图的数量是
{{ select(13) }}
- A. 15
- B. 16
- C. 6
- D. 4

> **答案**: A
> **解析**: 1. **理解基本概念**：
   - **有标号图**：指图的结点是不同的（例如编号为 1, 2, 3, 4）。在这种情况下，不同的边连接方式产生的图被视为不同的图。
   - **简单无向图**：指图中没有自环（结点连接到自身）且没有重边（两个结点之间最多只有一条边）。

2. **计算最大边数**：
   - 对于一个有 $n$ 个结点的简单无向图，任意两个结点之间最多可以有一条边。
   - 4 个结点中任取 2 个结点的组合数为 $C(4, 2) = \frac{4 \times 3}{2} = 6$。
   - 这意味着 4 个结点的完全图（所有可能的边都存在）共有 6 条边。

3. **计算符合条件的图的数量**：
   - 题目要求图中有 4 条边。
   - 我们需要从这 6 条可能的边中，任选出 4 条来构成一个图。
   - 这是一个组合问题，计算公式为 $C(6, 4)$。
   - 根据组合数的性质，$C(6, 4) = C(6, 6-4) = C(6, 2)$。
   - $C(6, 2) = \frac{6 \times 5}{2 \times 1} = 15$。

4. **验证结果**：
   - 这 15 种图可以分为两类同构类：
     - **环形图 (C4)**：4 个结点连成一个圈。这种结构的图有 $\frac{3!}{2} = 3$ 种标号方式。
     - **三角形加一条边**：3 个结点构成三角形，第 4 个结点连接到三角形的其中一个顶点。这种结构的图有 $4 \times 3 = 12$ 种标号方式。
   - 总计 $3 + 12 = 15$ 种。

因此，正确答案是 A。
