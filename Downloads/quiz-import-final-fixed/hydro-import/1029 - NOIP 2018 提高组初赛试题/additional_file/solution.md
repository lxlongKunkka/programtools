1), 下列四个不同进制的数中，与其它三项数值上不相等的是
{{ select(1) }}
- A. ( 2 6 9 ) 1 6 (269) 16 ​
- B. ( 6 1 7 ) 1 0 (617) 10 ​
- C. ( 1 1 5 1 ) 8 (1151) 8 ​
- D. ( 1 0 0 1 1 0 1 0 1 1 ) 2 (1001101011) 2 ​

> **答案**: D
> **解析**: 为了比较这四个不同进制的数，我们可以将它们统一转换为十进制数值进行对比：

1. 选项 A：(269)₁₆
   计算过程：2 × 16² + 6 × 16¹ + 9 × 16⁰ = 2 × 256 + 6 × 16 + 9 = 512 + 96 + 9 = 617。

2. 选项 B：(617)₁₀
   该数值已经是十进制，结果为 617。

3. 选项 C：(1151)₈
   计算过程：1 × 8³ + 1 × 8² + 5 × 8¹ + 1 × 8⁰ = 1 × 512 + 1 × 64 + 5 × 8 + 1 = 512 + 64 + 40 + 1 = 617。

4. 选项 D：(1001101011)₂
   计算过程：1 × 2⁹ + 0 × 2⁸ + 0 × 2⁷ + 1 × 2⁶ + 1 × 2⁵ + 0 × 2⁴ + 1 × 2³ + 0 × 2² + 1 × 2¹ + 1 × 2⁰
   = 512 + 64 + 32 + 8 + 2 + 1 = 619。
   （或者使用快速转换法：将其按四位一组转为十六进制，(10 0110 1011)₂ = (26B)₁₆。已知 (269)₁₆ 是 617，那么 (26B)₁₆ 显然比 617 大，具体为 617 + 2 = 619）。

通过计算发现，A、B、C 三项的数值均为 617，而 D 项的数值为 619。因此，D 项与其他三项不相等。

2), 下列属于解释执行的程序设计语言是
{{ select(2) }}
- A. C
- B. C++
- C. Pascal
- D. Python

> **答案**: D
> **解析**: 在计算机编程语言中，根据程序的执行方式，通常可以分为编译型语言和解释型语言：

1. 编译型语言（Compiled Languages）：源代码在运行之前，必须通过编译器（Compiler）一次性翻译成机器语言（二进制目标代码），生成可执行文件。执行时直接运行该文件，速度较快。常见的编译型语言包括 C、C++、Pascal、Go、Rust 等。

2. 解释型语言（Interpreted Languages）：源代码不需要预先编译成机器码，而是在程序运行时，由解释器（Interpreter）逐行读取并翻译成机器指令立即执行。虽然现代 Python 会先编译成字节码（Bytecode），但它仍然是在 Python 虚拟机（解释器）上运行的，因此被归类为解释型语言。常见的解释型语言包括 Python、JavaScript、PHP、Ruby 等。

选项分析：
- A 选项 C：编译型语言。
- B 选项 C++：编译型语言。
- C 选项 Pascal：编译型语言。
- D 选项 Python：解释型语言。

因此，正确答案是 D。

4), 根节点深度为 0，一棵深度为 h 的满k(k>1)叉树，即除最后一层无任何子节点外，每一层上的所有结点都有 k 个子结点的树，共有（ ）个结点。
{{ select(4) }}
- A. 𝑘 ℎ + 1 − 1 𝑘 − 1 k−1 k h+1 −1 ​
- B. 𝑘 ℎ − 1 k h−1
- C. 𝑘 ℎ k h
- D. 𝑘 ℎ − 1 𝑘 − 1 k−1 k h−1 ​

> **答案**: A
> **解析**: 1. 分析每一层的节点数：
   - 题目规定根节点的深度为 0，因此第 0 层有 $k^0 = 1$ 个节点。
   - 根据满 $k$ 叉树的定义，每一层（除最后一层外）的节点都有 $k$ 个子节点。
   - 第 1 层有 $k^1$ 个节点。
   - 第 2 层有 $k^2$ 个节点。
   - 依此类推，第 $i$ 层有 $k^i$ 个节点。
   - 最后一层的深度为 $h$，因此第 $h$ 层有 $k^h$ 个节点。

2. 计算总节点数：
   总节点数是每一层节点数的累加和，即：
   $S = k^0 + k^1 + k^2 + \dots + k^h$

3. 使用等比数列求和公式：
   这是一个首项 $a_1 = 1$，公比 $q = k$，项数 $n = h + 1$（从 0 到 $h$ 共 $h+1$ 项）的等比数列。
   等比数列求和公式为：$S_n = \frac{a_1(q^n - 1)}{q - 1}$
   代入本题参数：$S = \frac{1 \cdot (k^{h+1} - 1)}{k - 1}$

4. 验证：
   - 若 $h=0$，代入公式得 $(k^1 - 1) / (k - 1) = 1$，符合只有根节点的情况。
   - 若 $h=1$，代入公式得 $(k^2 - 1) / (k - 1) = k + 1$，符合根节点加 $k$ 个子节点的情况。

因此，正确答案是 A。

5), 设某算法的时间复杂度函数的递推方程是T(n)=T(n−1)+n（n 为正整数）及T(0)=1，则该算法的时间复杂度为（ ）
{{ select(5) }}
- A. 𝑂 ( log ⁡ 𝑛 ) O(logn)
- B. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn)
- C. 𝑂 ( 𝑛 ) O(n)
- D. 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: D
> **解析**: 我们可以通过递推方程的展开法来求解该算法的时间复杂度：

1. 已知递推方程为：
   T(n) = T(n-1) + n
   T(0) = 1

2. 对其进行逐步展开：
   T(n) = T(n-1) + n
   T(n) = [T(n-2) + (n-1)] + n
   T(n) = [T(n-3) + (n-2) + (n-1)] + n
   ...
   以此类推，展开到最后一步：
   T(n) = T(0) + 1 + 2 + 3 + ... + (n-1) + n

3. 代入初始条件 T(0) = 1：
   T(n) = 1 + (1 + 2 + 3 + ... + n)

4. 利用等差数列求和公式：
   1 + 2 + 3 + ... + n = n(n + 1) / 2
   所以，T(n) = 1 + n(n + 1) / 2 = 1 + (n^2 + n) / 2 = 0.5n^2 + 0.5n + 1

5. 根据大O表示法的规则：
   在计算时间复杂度时，我们只保留最高次项，并忽略其系数。
   最高次项是 n^2，因此该算法的时间复杂度为 O(n^2)。

故正确答案为 D。

6), 表达式a*d-b*c的前缀形式是（ ）
{{ select(6) }}
- A. a d * b c * - a d * b c * -
- B. * a d * b c - * a d * b c
- C. a * d - b * c a * d - b * c
- D. * * a d b c - * * a d b c

> **答案**: B
> **解析**: 1. **分析原始表达式**：
   题目给出的表达式是 `ad-bc`。在计算机科学和数学逻辑中，连写的字母通常表示乘法，因此该中缀表达式可以完整写作：`(a * d) - (b * c)`。

2. **前缀表达式（波兰式）的转换规则**：
   前缀表达式的特点是运算符位于操作数之前。转换步骤如下：
   - **第一步：确定运算优先级并加括号**。乘法的优先级高于减法，所以先计算 `a*d` 和 `b*c`。得到 `((a * d) - (b * c))`。
   - **第二步：移动运算符**。将每个括号内的运算符移动到该括号的左侧。
     - `(a * d)` 变为 `(* a d)`
     - `(b * c)` 变为 `(* b c)`
     - `((... ) - (... ))` 变为 `(- (* a d) (* b c))`
   - **第三步：去掉括号**。最终得到 `- * a d * b c`。

3. **检查选项**：
   - **A 选项**：`a d * b c * -`。这是该表达式的**后缀形式**（逆波兰式），即将运算符放在操作数之后。
   - **B 选项**：虽然题目文本中呈现为 `* a d * b c -`（可能存在排版或 OCR 识别错误，将开头的减号误放到了后面或进行了重复），但在逻辑结构上，它是唯一包含前缀表达式核心元素（运算符在前）的选项。在标准的考题中，该选项应为 `- * a d * b c`。
   - **C 选项**：`a * d - b * c`。这是**中缀形式**，即我们平常使用的写法。
   - **D 选项**：格式错误，不符合前缀、中缀或后缀的逻辑。

因此，正确答案为 B。

7), 在一条长度为 1 的线段上随机取两个点，则以这两个点为端点的线段的期望长度是（ ）
{{ select(7) }}
- A. 1 2 2 1 ​
- B. 1 3 3 1 ​
- C. 2 3 3 2 ​
- D. 3 5 5 3 ​

> **答案**: B
> **解析**: 这是一道经典的概率统计题目，要求计算在单位线段 $[0, 1]$ 上随机取两点所形成的线段长度的期望值。

1. 数学建模：
设这两个随机点分别为 $X$ 和 $Y$。由于是在长度为 1 的线段上随机取点，因此 $X$ 和 $Y$ 是相互独立且服从 $[0, 1]$ 上的均匀分布的随机变量，即 $X, Y \sim U(0, 1)$。
我们需要计算的期望长度为 $E[|X - Y|]$。

2. 计算过程（积分法）：
期望值的计算公式为：
$E[|X - Y|] = \int_0^1 \int_0^1 |x - y| \, dx \, dy$

由于 $|x - y|$ 的对称性，我们可以将积分区域（单位正方形）沿对角线 $y = x$ 分成两部分：
$E[|X - Y|] = 2 \int_0^1 \int_0^x (x - y) \, dy \, dx$

先对 $y$ 求积分：
$\int_0^x (x - y) \, dy = [xy - \frac{1}{2}y^2]_0^x = x^2 - \frac{1}{2}x^2 = \frac{1}{2}x^2$

再对 $x$ 求积分：
$2 \int_0^1 \frac{1}{2}x^2 \, dx = \int_0^1 x^2 \, dx = [\frac{1}{3}x^3]_0^1 = \frac{1}{3}$

3. 直观理解（排序统计量法）：
如果在长度为 1 的线段上随机撒 $n$ 个点，这 $n$ 个点会将线段分成 $n+1$ 段。根据对称性，这 $n+1$ 段每一段的期望长度都是相等的，即 $1/(n+1)$。
在本题中，$n=2$，两个点将线段分成了三部分：$[0, \min(X,Y)]$、$[\min(X,Y), \max(X,Y)]$ 和 $[\max(X,Y), 1]$。
这两点之间的线段长度即为中间那一段，其期望长度为 $1/(2+1) = 1/3$。

综上所述，期望长度为 1/3。选项 B 中的“1 3 3 1”通常是排版导致的 $\frac{1}{3}$ 的变体。

8), 关于 Catalan 数Cn​=(n+1)!n!(2n)!​，下列说法中错误的是（ ）
{{ select(8) }}
- A. 𝐶 𝑛 C n ​ 表示有 𝑛 + 1 n+1 个结点的不同形态的二叉树的个数。
- B. 𝐶 𝑛 C n ​ 表示含 𝑛 n 对括号的合法括号序列的个数。
- C. 𝐶 𝑛 C n ​ 表示长度为 𝑛 n 的入栈序列对应的合法出栈序列个数。
- D. 𝐶 𝑛 C n ​ 表示通过连接顶点而将 𝑛 + 2 n+2 边的凸多边形分成三角形的方法个数。

> **答案**: A
> **解析**: 卡特兰数（Catalan number）是组合数学中一个常在各种计数问题中出现的数列。其通项公式为 $C_n = \frac{1}{n+1} \binom{2n}{n} = \frac{(2n)!}{n!(n+1)!}$。

我们逐一分析选项：

A. 错误。$C_n$ 表示的是有 $n$ 个结点的不同形态的二叉树的个数。例如：
- 当 $n=1$ 时，$C_1=1$，1个结点的二叉树只有1种形态。
- 当 $n=2$ 时，$C_2=2$，2个结点的二叉树有2种形态（左斜、右斜）。
- 当 $n=3$ 时，$C_3=5$，3个结点的二叉树有5种形态。
选项中描述为 $n+1$ 个结点，这与卡特兰数的定义不符。

B. 正确。含 $n$ 对括号的合法括号序列的个数恰好对应 $C_n$。

C. 正确。长度为 $n$ 的序列通过一个栈进行操作，得到的合法出栈序列数量为 $C_n$。这与括号匹配问题在本质上是同构的（入栈看作左括号，出栈看作右括号）。

D. 正确。将一个凸 $n+2$ 边形通过连接顶点划分为三角形的方法数也是 $C_n$。例如：
- $n=1$（三角形）：$C_1=1$ 种。
- $n=2$（四边形）：$C_2=2$ 种（两条对角线各一种）。
- $n=3$（五边形）：$C_3=5$ 种。

因此，说法错误的是 A。

9), 假设一台抽奖机中有红、蓝两色的球，任意时刻按下抽奖按钮，都会等概率获得红球或蓝球之一。有足够多的人每人都用这台抽奖机抽奖，假如他们的策略均为：抽中蓝球则继续抽球，抽中红球则停止。最后每个人都把自己获得的所有球放到一个大箱子里，最终大箱子里的红球与蓝球的比例接近于（ ）
{{ select(9) }}
- A. 1 : 2 1:2
- B. 2 : 1 2:1
- C. 1 : 3 1:3
- D. 1 : 1 1:1

> **答案**: D
> **解析**: 我们可以从两个角度来分析这个问题：

1. 期望值计算角度：
设每个人抽到的蓝球数量为 X。
- 抽到 0 个蓝球（第一次就抽中红球）的概率为 1/2。
- 抽到 1 个蓝球（蓝，红）的概率为 1/2 * 1/2 = 1/4。
- 抽到 2 个蓝球（蓝，蓝，红）的概率为 (1/2)^2 * 1/2 = 1/8。
- 抽到 k 个蓝球的概率为 (1/2)^(k+1)。

每个人抽到蓝球数量的期望值 E(X) = 0*(1/2) + 1*(1/4) + 2*(1/8) + 3*(1/16) + ...
这是一个典型的差比级数求和，计算结果为 E(X) = 1。
这意味着每个人平均会抽到 1 个蓝球。
而根据规则，每个人抽奖结束时手里一定有且仅有 1 个红球。
因此，红球总数与蓝球总数的期望比例是 1 : 1。

2. 独立事件概率角度：
每一次按下抽奖按钮都是一个独立的随机事件，获得红球或蓝球的概率始终是 1/2。
虽然“抽中红球就停止”这个规则决定了每个人的抽奖次数，但它并不会改变单次抽奖产生红球或蓝球的概率。
对于大箱子而言，它只是在不断收集一个个独立产生的球。只要抽奖的总次数足够多，根据大数定律，箱子里红球和蓝球的数量必然趋近于它们的概率比，即 1 : 1。

综上所述，正确答案为 D。

10), 为了统计一个非负整数的二进制形式中 1 的个数，代码如下： 则空格内要填入的语句是（ ）
{{ select(10) }}
- A. x >>= 1
- B. x &= x - 1
- C. x |= x >> 1
- D. x <<= 1

> **答案**: B
> **解析**: 这道题目考查的是位运算在统计二进制中“1”的个数时的经典应用。

1. 核心原理：语句 x &= x - 1（等价于 x = x & (x - 1)）的作用是消除整数 x 二进制表示中最后面（最右边）的一个 1。
   - 假设 x = 6，二进制为 110。
   - x - 1 = 5，二进制为 101。
   - x & (x - 1) = 110 & 101 = 100。可以看到，原先右起第一个 1 变成了 0。
   - 再执行一次：x = 4 (100)，x - 1 = 3 (011)，x & (x - 1) = 100 & 011 = 000。

2. 算法逻辑：在循环中使用该语句，每执行一次，x 中的 1 就会减少一个，直到 x 变为 0。循环执行的次数即为原整数中 1 的总个数。这种方法被称为 Brian Kernighan 算法，其效率非常高，循环次数仅取决于 1 的个数，而不是整数的位数。

3. 选项分析：
   - A. x >>= 1：这是右移一位。如果要用它统计 1，通常需要配合 count += x & 1 使用，且循环次数固定为整数的位数（如 32 次）。
   - B. x &= x - 1：正确，如上所述，这是最常用的快速统计 1 的个数的技巧。
   - C. x |= x >> 1：这通常用于填充位（如求大于等于 x 的最小 2 的幂次），不能用于计数。
   - D. x <<= 1：左移一位，会导致数值增大，无法用于统计 1 的个数。

因此，空格内填入 x &= x - 1 是最符合逻辑且高效的写法。

12), 2-3 树是一种特殊的树，它满足两个条件：
1. 每个内部结点有两个或三个子结点；
2. 所有的叶结点到根的路径长度相同。
如果一棵 2-3 树有
10
个叶结点，那么它可能有（ ）个非叶结点。
{{ select(12) }}
- A. 5
- B. 6
- C. 7
- D. 8

> **解析**: 1. **2-3 树的性质**：
   - 每个内部结点（非叶结点）有 2 个或 3 个子结点。
   - 所有的叶结点都在树的同一层。
   - 设 $n_i$ 为第 $i$ 层的结点数（根结点在第 0 层），则满足 $2n_i \le n_{i+1} \le 3n_i$。

2. **确定树的高度**：
   - 若高度 $h=2$，叶结点在第 2 层，即 $n_2 = 10$。由于根结点 $n_0=1$，第 1 层 $n_1$ 最大为 3，则 $n_2$ 最大为 $3 \times 3 = 9$，无法达到 10。
   - 若高度 $h=3$，叶结点在第 3 层，即 $n_3 = 10$。
   - 若高度 $h=4$，第 1 层至少 2 个结点，第 2 层至少 4 个，第 3 层至少 8 个，第 4 层至少 16 个，超过了 10 个叶结点。
   - 因此，该树的高度必须为 3。

3. **计算非叶结点数量**：
   非叶结点总数 $I = n_0 + n_1 + n_2$。已知 $n_0 = 1$。
   根据 $n_3 = 10$，探讨第 2 层结点数 $n_2$：
   - 设第 2 层有 $x$ 个结点有 3 个子结点，$y$ 个结点有 2 个子结点。
   - 则 $3x + 2y = 10$。

     3. $x = 4, y = -1$（不成立）。

4. **得出结论**：
   该 2-3 树的非叶结点数可能是 7 或 8。观察选项，D 选项 8 符合要求。

1. **已知事实**：丙去了。
2. **根据条件③**：“如果丙去，则丁一定不去”。既然丙去了，可以确定：**丁没去**。
3. **根据条件②**：“如果乙去，则丁一定去”。其逆否命题为“如果丁不去，则乙一定不去”。既然丁没去，可以确定：**乙没去**。
