1), 浮点数 2E+03 表示（ ）
{{ select(1) }}
- A. 2.03
- B. 5
- C. 8
- D. 2000

> **答案**: D
> **解析**: 在计算机编程和数学表示中，E（或 e）表示科学计数法。其通用格式为 aEb，表示 a 乘以 10 的 b 次方（即 a × 10^b）。

针对本题：
1. a = 2
2. b = +03（即 3）
3. 计算过程为：2 × 10^3 = 2 × 1000 = 2000。

因此，浮点数 2E+03 表示的数值是 2000。选项 D 正确。

2), 一个字节（byte）由（ ）个二进制位组成。
{{ select(2) }}
- A. 8
- B. 16
- C. 32
- D. 以上都有可能

> **答案**: A
> **解析**: 在计算机科学中，字节（Byte）是衡量存储容量的基本单位。根据计算机体系结构的通用标准：
1. 位（bit）：是计算机中最小的信息单位，只能表示 0 或 1。
2. 字节（Byte）：由 8 个二进制位（bit）组成。

这是计算机基础知识中的固定定义。1 个字节可以表示 $2^8 = 256$ 种不同的数值（通常为 0 到 255）。虽然在早期的计算机历史中曾出现过非 8 位的字节，但在现代计算机标准（如 ISO/IEC 80000-13）中，1 Byte 严格等于 8 bits。因此，选项 A 是正确答案。

3), 以下逻辑表达式的值恒为真的是（ ）
{{ select(3) }}
- A. P∨(¬P∧Q)∨(¬P∧¬Q)
- B. Q∨(¬P∧Q)∨(P∧¬Q)
- C. P∨Q∨(P∧¬Q)∨(¬P∧Q)
- D. P∨¬Q∨(P∧¬Q)∨(¬P∧¬Q)

> **答案**: A
> **解析**: 本题考查逻辑表达式的化简与恒真式（重言式）的判断。我们可以通过逻辑代数运算法则对各个选项进行化简：

选项 A：P ∨ (¬P ∧ Q) ∨ (¬P ∧ ¬Q)
1. 观察后两项：(¬P ∧ Q) ∨ (¬P ∧ ¬Q)。根据分配律，可以提取 ¬P，得到 ¬P ∧ (Q ∨ ¬Q)。
2. 根据排中律，Q ∨ ¬Q 恒为真（1），所以该部分简化为 ¬P ∧ 1 = ¬P。
3. 整个表达式简化为 P ∨ ¬P。
4. 根据排中律，P ∨ ¬P 恒为真。因此 A 选项是恒真式。

选项 B：Q ∨ (¬P ∧ Q) ∨ (P ∧ ¬Q)
1. 根据吸收律，Q ∨ (¬P ∧ Q) = Q。
2. 表达式简化为 Q ∨ (P ∧ ¬Q)。
3. 根据分配律，Q ∨ (P ∧ ¬Q) = (Q ∨ P) ∧ (Q ∨ ¬Q) = (Q ∨ P) ∧ 1 = Q ∨ P。
4. 当 P 和 Q 都为假时，结果为假，不是恒真式。

选项 C：P ∨ Q ∨ (P ∧ ¬Q) ∨ (¬P ∧ Q)
1. 根据吸收律，P ∨ (P ∧ ¬Q) = P。
2. 根据吸收律，Q ∨ (¬P ∧ Q) = Q。
3. 表达式简化为 P ∨ Q。
4. 当 P 和 Q 都为假时，结果为假，不是恒真式。

选项 D：P ∨ ¬Q ∨ (P ∧ ¬Q) ∨ (¬P ∧ ¬Q)
1. 根据吸收律，¬Q ∨ (P ∧ ¬Q) = ¬Q。
2. 根据吸收律，¬Q ∨ (¬P ∧ ¬Q) = ¬Q。
3. 表达式简化为 P ∨ ¬Q。
4. 当 P 为假且 Q 为真时，结果为假，不是恒真式。

综上所述，只有 A 选项的值恒为真。

4), Linux 下可执行文件的默认扩展名为（ ）
{{ select(4) }}
- A. exe
- B. com
- C. dll
- D. 以上都不是

> **答案**: D
> **解析**: 在 Linux 操作系统中，文件是否可执行并不取决于其扩展名（后缀名），而是取决于文件的权限属性（即是否具有可执行权限 x）。

1. 选项 A (exe)、B (com) 是 Windows 操作系统下常见的可执行文件扩展名。
2. 选项 C (dll) 是 Windows 下的动态链接库文件扩展名（在 Linux 中对应的通常是 .so 文件）。
3. 在 Linux 中，二进制可执行文件（如系统命令 ls, cd 等）通常没有扩展名。虽然脚本文件可能会使用 .sh, .py 等扩展名以方便用户识别，但这并非系统强制要求的“默认扩展名”。

因此，Linux 下没有所谓的可执行文件默认扩展名，正确答案为 D。

5), 如果树根算第 1 层，那么一棵 n 层的二叉树最多有（ ）个结点。
{{ select(5) }}
- A. 2 𝑛 − 1 2 n −1
- B. 2 𝑛 2 n
- C. 2 𝑛 + 1 2 n +1
- D. 2 𝑛 + 1 2 n+1

> **答案**: A
> **解析**: 这是一道关于二叉树性质的基础题目。我们可以通过以下步骤得出结论：

1. 确定每一层的最大结点数：
   在二叉树中，规定根结点为第 1 层。
   - 第 1 层最多有 $2^0 = 1$ 个结点；
   - 第 2 层最多有 $2^1 = 2$ 个结点；
   - 第 3 层最多有 $2^2 = 4$ 个结点；
   - 依此类推，第 $i$ 层最多有 $2^{i-1}$ 个结点。

2. 计算 $n$ 层总共的最大结点数：
   一棵 $n$ 层的二叉树，当它是“满二叉树”时结点数最多。总结点数 $S$ 是每一层最大结点数的累加：
   $S = 2^0 + 2^1 + 2^2 + ... + 2^{n-1}$

3. 使用等比数列求和公式：
   这是一个首项 $a_1 = 1$，公比 $q = 2$，项数为 $n$ 的等比数列。
   求和公式为：$S = \frac{a_1(1-q^n)}{1-q}$
   代入数值：$S = \frac{1(1-2^n)}{1-2} = \frac{1-2^n}{-1} = 2^n - 1$

4. 举例验证：
   - 如果 $n=1$，最多有 $2^1 - 1 = 1$ 个结点（只有根结点）。
   - 如果 $n=2$，最多有 $2^2 - 1 = 3$ 个结点（1个根结点 + 2个子结点）。
   - 如果 $n=3$，最多有 $2^3 - 1 = 7$ 个结点（1+2+4=7）。

因此，正确答案是 A。

6), 提出“存储程序”的计算机工作原理的是（ ）
{{ select(6) }}
- A. 克劳德·香农
- B. 戈登·摩尔
- C. 查尔斯·巴比奇
- D. 冯·诺依曼

> **答案**: D
> **解析**: 本题考查计算机发展史中的重要人物及其贡献。

1. 冯·诺依曼（John von Neumann）：美籍匈牙利数学家，被称为“计算机之父”。他在1945年发表的《关于EDVAC的报告草案》中提出了“存储程序”原理。该原理的核心思想是：将程序指令和数据一起存储在计算机的存储器中，计算机按照程序指定的顺序自动执行指令。这一架构被称为“冯·诺依曼架构”，至今仍是绝大多数现代计算机的基础。

2. 其他选项解析：
   - A. 克劳德·香农（Claude Shannon）：信息论的创始人，提出了信息熵的概念，为现代通信理论奠定了基础。
   - B. 戈登·摩尔（Gordon Moore）：英特尔公司的创始人之一，提出了著名的“摩尔定律”，即集成电路上可容纳的晶体管数目约每隔18-24个月便会增加一倍。
   - C. 查尔斯·巴比奇（Charles Babbage）：19世纪的英国数学家，设计了差分机和分析机，被认为是通用计算机的先驱，但他的设计是机械式的，并非现代意义上的电子存储程序计算机。

因此，正确答案是 D。

7), 设X,Y,Z分别代表三进制下的一位数字，若等式XY+ZX=XYX在三进制下成立，那么同样在三进制下，等式XY×ZX=（ ）也成立。
{{ select(7) }}
- A. 𝑌 𝑋 𝑍 ‾ YXZ
- B. 𝑍 𝑋 𝑌 ‾ ZXY
- C. 𝑋 𝑌 𝑍 ‾ XYZ
- D. 𝑋 𝑍 𝑌 ‾ XZY

> **答案**: B
> **解析**: 1. 首先，根据三进制的定义，X, Y, Z 是三进制下的数字，其取值范围为 {0, 1, 2}。
2. 将已知等式 $XY + ZX = XYX$ 转换为十进制表达式：
   - $XY = 3X + Y$
   - $ZX = 3Z + X$
   - $XYX = 3^2 \cdot X + 3Y + X = 9X + 3Y + X = 10X + 3Y$
3. 代入等式得：$(3X + Y) + (3Z + X) = 10X + 3Y$
   整理得：$4X + Y + 3Z = 10X + 3Y$
   移项得：$3Z = 6X + 2Y$
4. 根据 X, Y, Z 的取值范围 {0, 1, 2} 进行讨论：
   - 若 $X = 0$，则 $3Z = 2Y$。若 $Y=0, Z=0$，则所有数字为0，不符合三位数 $XYX$ 的常规意义（首位不为0）。
   - 若 $X = 1$，则 $3Z = 6 + 2Y$。
     - 当 $Y = 0$ 时，$3Z = 6 \implies Z = 2$。符合范围。
     - 当 $Y = 1$ 时，$3Z = 8$（无整数解）。
     - 当 $Y = 2$ 时，$3Z = 10$（无整数解）。
   - 若 $X = 2$，则 $3Z = 12 + 2Y$。即使 $Y=0$，$Z=4$ 也超出了三进制范围。
   因此，唯一解为：$X = 1, Y = 0, Z = 2$。
5. 计算目标等式 $XY \times ZX$：
   - $XY = 10_3$（十进制为 3）
   - $ZX = 21_3$（十进制为 $2 \times 3 + 1 = 7$）
   - $XY \times ZX = 3 \times 7 = 21$（十进制）
6. 将十进制的 21 转换为三进制：
   - $21 \div 3 = 7 \dots 0$
   - $7 \div 3 = 2 \dots 1$
   - $2 \div 3 = 0 \dots 2$
   所以 $21_{10} = 210_3$。
7. 对应字母：
   - $2 \to Z$
   - $1 \to X$
   - $0 \to Y$
   结果为 $ZXY$。故选 B。

8), Pascal 语言、C 语言和 C++ 语言都属于（ ）
{{ select(8) }}
- A. 面向对象语言
- B. 脚本语言
- C. 解释性语言
- D. 编译性语言

> **答案**: D
> **解析**: 1. 编译性语言（Compiled Language）：是指在程序执行之前，需要使用专门的编译器将源代码一次性翻译成机器语言（二进制目标代码），然后才能运行。Pascal、C 语言和 C++ 语言都是典型的编译性语言，它们在运行前都需要经过编译、链接等过程生成可执行文件。

2. 选项分析：
   - A 选项（面向对象语言）：C++ 是面向对象的，但 C 语言是典型的面向过程语言，Pascal 最初也是面向过程的（虽然后来有了 Object Pascal），因此 A 选项不完全符合。
   - B 选项（脚本语言）：脚本语言如 Python、JavaScript、Shell 等，通常不需要编译成二进制文件，而是由解释器执行。Pascal、C、C++ 不属于此类。
   - C 选项（解释性语言）：解释性语言在运行时由解释器逐行读取并翻译执行。而 Pascal、C、C++ 必须先编译成机器码才能运行，不属于解释性语言。
   - D 选项（编译性语言）：这三种语言都需要通过编译器将源代码转换为机器指令，符合它们的共同特征。

因此，正确答案是 D。

9), 前缀表达式+ 3 * 2 + 5 12的值是（ ）
{{ select(9) }}
- A. 23
- B. 25
- C. 37
- D. 65

> **答案**: C
> **解析**: 前缀表达式（波兰表达式）的求值方法通常是从右向左扫描。具体步骤如下：

1. 从右向左扫描表达式：`+ 3 * 2 + 5 12`。
2. 遇到数字 12，入栈。栈内：[12]
3. 遇到数字 5，入栈。栈内：[12, 5]
4. 遇到运算符 +，弹出栈顶的两个数字 5 和 12，计算 5 + 12 = 17，将结果 17 入栈。栈内：[17]
5. 遇到数字 2，入栈。栈内：[17, 2]
6. 遇到运算符 *，弹出栈顶的两个数字 2 和 17，计算 2 * 17 = 34，将结果 34 入栈。栈内：[34]
7. 遇到数字 3，入栈。栈内：[34, 3]
8. 遇到运算符 +，弹出栈顶的两个数字 3 和 34，计算 3 + 34 = 37，将结果 37 入栈。栈内：[37]

最终栈顶的结果为 37。

也可以通过将前缀表达式转换为中缀表达式来理解：
+ 3 * 2 (+ 5 12)
= + 3 (* 2 17)
= + 3 34
= 37

因此，正确答案是 C。

10), 主存储器的存取速度比中央处理器（CPU）的工作速度慢得多，从而使得后者的效率受到影响。而根据局部性原理，CPU 所访问的存储单元通常都趋于聚集在一个较小的连续区域中。于是，为了提高系统整体的执行效率，在 CPU 中引入了（ ）
{{ select(10) }}
- A. 寄存器
- B. 高速缓存
- C. 闪存
- D. 外存

> **答案**: B
> **解析**: 本题考查计算机体系结构中的存储层次结构。

1. 速度矛盾：中央处理器（CPU）的运算速度极快，而主存储器（内存）的存取速度相对较慢。如果 CPU 每次都直接从内存中读取数据，会因为等待内存响应而导致效率大幅下降。
2. 局部性原理：研究发现，程序在执行时具有“局部性原理”，即在一段时间内，CPU 所访问的指令和数据往往集中在一段较小的连续内存区域内（空间局部性），或者某些数据会被反复访问（时间局部性）。
3. 高速缓存（Cache）的作用：为了解决 CPU 与内存之间的速度匹配问题，计算机设计者在 CPU 和内存之间引入了高速缓存。Cache 的速度接近 CPU，容量较小。它利用局部性原理，将 CPU 近期可能用到的数据从内存预先调入 Cache 中。当 CPU 需要数据时，先在 Cache 中查找，如果命中则直接读取，从而大大提高了系统的整体执行效率。

其他选项分析：
A. 寄存器：位于 CPU 内部，速度最快，但数量极少，主要用于存放当前正在执行的指令和操作数，不是为了解决内存速度瓶颈而基于局部性原理引入的中间层。
C. 闪存（Flash Memory）：属于非易失性存储器，常用于 U 盘、固态硬盘等，速度远慢于内存。
D. 外存：如硬盘等，用于长期保存数据，速度比内存更慢。

11), 一个字长为 8 位的整数的补码是 11111001，则它的原码是（ ）
{{ select(11) }}
- A. 0 0 0 0 0 1 1 1 00000111
- B. 0 1 1 1 1 0 0 1 01111001
- C. 1 1 1 1 1 0 0 1 11111001
- D. 1 0 0 0 0 1 1 1 10000111

> **答案**: D
> **解析**: 在计算机系统中，数值通常以补码的形式存储。根据补码求原码的步骤如下：

1. **判断正负**：观察补码的最高位（符号位）。该补码为 11111001，最高位是 1，表示这是一个负数。
2. **负数补码转原码的规则**：
   - 方法一：符号位不变，其余各位取反，最后在末位加 1。
     - 补码：11111001
     - 取反（符号位除外）：10000110
     - 加 1：10000111
   - 方法二：符号位不变，先减 1，再取反。
     - 补码：11111001
     - 减 1：11111000
     - 取反（符号位除外）：10000111
3. **验证**：
   - 原码 10000111 对应的十进制数是 -7。
   - -7 的反码是 11111000。
   - -7 的补码（反码+1）是 11111001，与题干相符。

因此，该整数的原码是 10000111。选项 D 符合。

12), 基于比较的排序时间复杂度的下限是（ ），其中 n 表示待排序的元素个数。
{{ select(12) }}
- A. Θ ( 𝑛 ) Θ(n)
- B. Θ ( 𝑛 log ⁡ 𝑛 ) Θ(nlogn)
- C. Θ ( log ⁡ 𝑛 ) Θ(logn)
- D. Θ ( 𝑛 2 ) Θ(n 2 )

> **答案**: B
> **解析**: 1. 排序算法主要分为两大类：基于比较的排序（如冒泡排序、快速排序、归并排序等）和非基于比较的排序（如计数排序、基数排序、桶排序）。本题考察的是基于比较的排序。
2. 在基于比较的排序中，算法通过比较两个元素的大小来决定它们的相对顺序。这种过程可以用“决策树”模型来表示。
3. 对于 n 个不同的元素，它们可能的排列方式共有 n!（n 的阶乘）种。决策树中的每一个叶子节点都代表一种可能的排列结果。
4. 根据二叉树的性质，一棵拥有 L 个叶子节点的二叉树，其深度（即从根到叶子的最长路径，代表最坏情况下的比较次数）至少为 log2(L)。
5. 将 L = n! 代入，根据斯特林公式（Stirling's approximation），log2(n!) 的渐进复杂度为 Θ(n log n)。
6. 因此，任何基于比较的排序算法，在最坏情况下至少需要进行 Ω(n log n) 次比较。
7. 选项 A (Θ(n)) 是非基于比较排序在特定条件下的复杂度；选项 C (Θ(log n)) 甚至不足以遍历完所有数据；选项 D (Θ(n^2)) 是简单排序算法（如冒泡、插入排序）的复杂度，但不是所有比较排序的下限。故正确答案为 B。

13), 一个自然数在十进制下有 n 位，则它在二进制下的位数与（ ）最接近。
{{ select(13) }}
- A. 5 𝑛 5n
- B. 𝑛 log ⁡ 2 1 0 nlog 2 ​ 10
- C. 1 0 log ⁡ 2 𝑛 10log 2 ​ n
- D. 1 0 𝑛 log ⁡ 2 𝑛 10 n log 2 ​ n

> **答案**: B
> **解析**: 1. 位数计算原理：对于一个正整数 $x$，它在 $b$ 进制下的位数可以用公式 $\lfloor \log_b x \rfloor + 1$ 来表示。
2. 十进制位数：如果一个自然数在十进制下有 $n$ 位，那么这个数 $x$ 的范围是 $10^{n-1} \le x < 10^n$。
3. 二进制位数：设该数在二进制下的位数为 $m$，则 $m \approx \log_2 x$。
4. 换算关系：由于 $x$ 的最大值接近 $10^n$，我们可以通过对数换算得到：
   $\log_2 x \approx \log_2 (10^n)$
   根据对数的运算性质 $\log_a (b^c) = c \cdot \log_a b$，可得：
   $\log_2 (10^n) = n \cdot \log_2 10$
5. 结论：因此，该数在二进制下的位数与 $n \log_2 10$ 最接近。实际上，$\log_2 10 \approx 3.32$，这意味着十进制每增加 1 位，其对应的二进制位数大约增加 3.32 位。

14), 在下列 HTML 语句中，可以正确产生一个指向 NOI 官方网站的超链接的是（ ）
{{ select(14) }}
- A. <a url="http://www.noi.cn">欢迎访问 NOI 网站</a>
- B. <a href="http://www.noi.cn">欢迎访问 NOI 网站</a>
- C. <a>http://www.noi.cn</a>
- D. <a name="http://www.noi.cn">欢迎访问 NOI 网站</a>

> **答案**: B

15), 元素R1​,R2​,R3​,R4​,R5​入栈的顺序为R1​,R2​,R3​,R4​,R5​。如果第 1 个出栈的是R3​，那么第 5 个出栈的不可能是（ ）
{{ select(15) }}
- A. 𝑅 1 R 1 ​
- B. 𝑅 2 R 2 ​
- C. 𝑅 4 R 4 ​
- D. 𝑅 5 R 5 ​

> **答案**: B
> **解析**: 本题考查栈（Stack）的基本性质。栈是一种“后进先出”（LIFO, Last In First Out）的数据结构。

1. **分析入栈过程**：题目给出入栈顺序为 $R_1, R_2, R_3, R_4, R_5$。
2. **分析已知条件**：第 1 个出栈的是 $R_3$。
   - 为了让 $R_3$ 出栈，必须先将 $R_1, R_2, R_3$ 依次压入栈中。
   - 当 $R_3$ 出栈后，栈内的剩余元素从底到顶依次为：$R_1, R_2$。
3. **分析后续可能性**：
   - 此时，待入栈的元素还有 $R_4, R_5$。
   - 栈底元素是 $R_1$，这意味着只要 $R_1$ 还在栈内，它下面的元素（无）或它上面的元素（如 $R_2$）必须遵循栈的规则。
   - 关键点在于：**$R_1$ 在 $R_2$ 的下方**。根据后进先出的原则，只要 $R_1$ 和 $R_2$ 都在栈中，$R_2$ 必然比 $R_1$ 先出栈。
4. **判断第 5 个出栈的元素**：
   - 第 5 个出栈即为最后一个出栈的元素。
   - 由于 $R_1$ 在 $R_2$ 的下方，在 $R_1$ 出栈之前，$R_2$ 必须已经出栈。因此，$R_2$ 不可能是最后一个出栈的。
   - 我们可以验证其他选项：
     - 若序列为 $R_3, R_4, R_5, R_2, R_1$，则第 5 个是 $R_1$（A可能）。
     - 若序列为 $R_3, R_2, R_1, R_5, R_4$，则第 5 个是 $R_4$（C可能）。
     - 若序列为 $R_3, R_2, R_1, R_4, R_5$，则第 5 个是 $R_5$（D可能）。
   - 无论如何操作，$R_2$ 都会在 $R_1$ 之前出栈，所以 $R_2$ 不可能是第 5 个。

故正确答案为 B。

16), 双向链表中有两个指针域 llink 和 rlink，分别指向该结点的前驱及后继。设 p 指向链表中的一个结点，它的左右结点均非空。现要求删除结点 p，则下面语句序列中错误的是（ ）
{{ select(16) }}
- A. p->rlink->llink = p->rlink;p->llink->rlink = p->llink; delete p;
- B. p->llink->rlink = p->rlink; p->rlink->llink = p->llink; delete p;
- C. p->rlink->llink = p->llink;p->rlink->llink->rlink = p->rlink; delete p;
- D. p->llink->rlink = p->rlink;p->llink->rlink->llink = p->llink; delete p;

> **答案**: A
> **解析**: 在双向链表中删除结点 p，需要完成两个核心步骤：
1. 让 p 的前驱结点的 rlink 指针指向 p 的后继结点。
2. 让 p 的后继结点的 llink 指针指向 p 的前驱结点。

下面逐一分析选项：

A. 错误。
- `p->rlink->llink = p->rlink;` 这行代码将 p 的后继结点的前驱指针指向了后继结点自己。
- `p->llink->rlink = p->llink;` 这行代码将 p 的前驱结点的后继指针指向了前驱结点自己。
这不仅没有正确连接 p 的前后结点，反而造成了错误的自循环，逻辑完全错误。

B. 正确。
- `p->llink->rlink = p->rlink;` 将 p 的前驱的后继指向 p 的后继。
- `p->rlink->llink = p->llink;` 将 p 的后继的前驱指向 p 的前驱。
这是最标准的删除操作。

C. 正确。
- `p->rlink->llink = p->llink;` 先将 p 的后继的前驱指向 p 的前驱。
- `p->rlink->llink->rlink = p->rlink;` 此时 `p->rlink->llink` 已经指向了 p 的前驱，所以这句等价于 `p->llink->rlink = p->rlink`。

D. 正确。
- `p->llink->rlink = p->rlink;` 先将 p 的前驱的后继指向 p 的后继。
- `p->llink->rlink->llink = p->llink;` 此时 `p->llink->rlink` 已经指向了 p 的后继，所以这句等价于 `p->rlink->llink = p->llink`。

因此，选项 A 是错误的语句序列。

17), 一棵二叉树的前序遍历序列是 ABCDEFG，后序遍历序列是 CBFEGDA，则根结点的左子树的结点个数可能是（ ）
{{ select(17) }}
- A. 2
- B. 3
- C. 4
- D. 5

> **答案**: A
> **解析**: 通过二叉树的前序遍历和后序遍历序列，我们可以确定树的结构：

1. 确定根结点：前序遍历的第一个字符是 A，后序遍历的最后一个字符也是 A，因此 A 是整棵树的根结点。
2. 确定左右子树范围：
   - 在前序序列中，根结点 A 之后紧跟着的是 B。
   - 在后序序列中，根结点 A 之前紧跟着的是 D。
   - 如果 A 有两个子结点，那么 B 必然是左子树的根，D 必然是右子树的根。
   - 如果 A 只有一个子结点，那么前序的第二个结点和后序的倒数第二个结点应该是同一个，但这里 B ≠ D，说明 A 确实有两个子树。
3. 划分序列：
   - 在后序序列 `C B F E G D A` 中，左子树的所有结点必须出现在右子树结点之前，且左子树序列必须以其根结点 B 结尾。
   - 观察后序序列，B 出现的位置非常靠前，其前面的序列是 `C B`。
   - 这意味着左子树的后序遍历序列只能是 `C B`，对应的结点是 {B, C}。
   - 相应地，右子树的后序遍历序列是 `F E G D`，对应的结点是 {D, E, F, G}。
4. 验证：
   - 左子树：前序为 `B C`，后序为 `C B`。这表示 B 是父结点，C 是其左子结点或右子结点。
   - 右子树：前序为 `D E F G`，后序为 `F E G D`。根为 D，D 的左子树前序为 `E F` 后序为 `F E`，右子树前序为 `G` 后序为 `G`。
   - 结构完全匹配。

综上所述，根结点 A 的左子树结点为 {B, C}，个数为 2。因此正确答案为 A。

18), 关于拓扑排序，下面说法正确的是（ ）
{{ select(18) }}
- A. 所有连通的有向图都可以实现拓扑排序
- B. 对同一个图而言，拓扑排序的结果是唯一的
- C. 拓扑排序中入度为 0 0 的结点总会排在入度大于 0 0 的结点的前面
- D. 拓扑排序结果序列中的第一个结点一定是入度为 0 0 的点

> **答案**: D
> **解析**: 拓扑排序（Topological Sort）是对有向无环图（DAG, Directed Acyclic Graph）的一种排序方式。下面是对各选项的详细解析：

A. **错误**。并不是所有连通的有向图都可以实现拓扑排序。如果图中存在环路（Cycle），则无法进行拓扑排序。只有有向无环图（DAG）才能进行拓扑排序。

B. **错误**。拓扑排序的结果不一定是唯一的。在算法执行过程中，如果在某一时刻有多个结点的入度均为 0，那么选择其中任意一个结点作为序列的下一个元素都是合法的，这会导致产生不同的拓扑序列。

C. **错误**。该说法不准确。虽然拓扑排序的起始点一定是入度为 0 的点，但在排序过程中，一个初始入度大于 0 的结点，在其所有前驱结点都被排好序并移除后，它的入度会变为 0。此时，它可能会排在其他初始入度为 0 的结点之前。
   *例如：图中有两条路径 A -> B 和 C（C 是孤立点，入度为 0）。合法的拓扑序列可以是 A, B, C。在这个序列中，初始入度为 1 的结点 B 排在了初始入度为 0 的结点 C 之前。*

D. **正确**。在拓扑排序的定义中，对于每一条从结点 $u$ 到结点 $v$ 的有向边 $(u, v)$，$u$ 在排序序列中必须出现在 $v$ 之前。如果序列中的第一个结点入度大于 0，说明至少存在一个结点指向它，那么那个指向它的结点就必须排在它前面，这与它是“第一个结点”相矛盾。因此，拓扑排序结果序列中的第一个结点其入度必然为 0。

19), 完全二叉树的顺序存储方案，是指将完全二叉树的结点从上至下、从左至右依次存放到一个顺序结构的数组中。假定根结点存放在数组的 1 号位置，则第 k 号结点的父结点如果存在的话，应当存放在数组的（ ）号位置。
{{ select(19) }}
- A. 2 𝑘 2k
- B. 2 𝑘 + 1 2k+1
- C. ⌊ 𝑘 2 ⌋ ⌊ 2 k ​ ⌋
- D. ⌊ 𝑘 + 1 2 ⌋ ⌊ 2 k+1 ​ ⌋

> **答案**: C
> **解析**: 在完全二叉树的顺序存储结构中，如果根结点存放在数组的 1 号位置，根据二叉树的性质，结点之间的索引关系如下：

1. 对于位于 $i$ 号位置的结点：
   - 其左孩子（如果存在）存放在 $2i$ 号位置。
   - 其右孩子（如果存在）存放在 $2i + 1$ 号位置。

2. 反之，对于位于 $k$ 号位置的结点（$k > 1$）：
   - 如果 $k$ 是偶数，它是其父结点的左孩子，父结点的编号为 $k / 2$。
   - 如果 $k$ 是奇数，它是其父结点的右孩子，父结点的编号为 $(k - 1) / 2$。

3. 统一规律：
   无论是左孩子还是右孩子，其父结点的编号都可以统一表示为 $\lfloor k / 2 \rfloor$（即 $k$ 除以 2 并向下取整）。

举例验证：
- 若 $k = 2$（根的左孩子），父结点为 $\lfloor 2 / 2 \rfloor = 1$。
- 若 $k = 3$（根的右孩子），父结点为 $\lfloor 3 / 2 \rfloor = 1$。
- 若 $k = 4$（2号结点的左孩子），父结点为 $\lfloor 4 / 2 \rfloor = 2$。
- 若 $k = 5$（2号结点的右孩子），父结点为 $\lfloor 5 / 2 \rfloor = 2$。

因此，正确答案是 C。
