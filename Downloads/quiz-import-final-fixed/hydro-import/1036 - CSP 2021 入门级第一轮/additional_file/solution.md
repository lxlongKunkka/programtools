1), 以下不属于面向对象程序设计语言的是（ ）
{{ select(1) }}
- A. C++
- B. Python
- C. Java
- D. C

> **答案**: D
> **解析**: 1. 面向对象程序设计（Object-Oriented Programming, OOP）是一种编程范式，它将数据和处理数据的方法封装在对象中。其核心特性包括封装、继承和多态。
2. 选项分析：
   - A. C++：是在 C 语言的基础上发展而来的，引入了类和对象的概念，是一种支持面向对象特性的混合型语言。

   - C. Java：是一种纯粹的面向对象编程语言，几乎所有的代码都必须编写在类中。
   - D. C：是一种典型的面向过程（Procedural Programming）编程语言，它以函数为中心，不具备类、对象、继承等面向对象的原生语法结构。
3. 结论：在给出的选项中，只有 C 语言不属于面向对象程序设计语言。

4), 以比较作为基本运算，在 N 个数中找出最大数，最坏情况下所需要的最少的比较次数为（ ）
{{ select(4) }}
- A. 𝑁 2 N 2
- B. 𝑁 N
- C. 𝑁 − 1 N−1
- D. 𝑁 + 1 N+1

> **答案**: C
> **解析**: 在 N 个数中找出最大数，最基本的算法逻辑如下：
1. 假设第一个数是当前的最大值。
2. 将当前最大值依次与剩下的 N-1 个数进行比较。
3. 如果发现某个数比当前最大值大，则更新最大值。

在这个过程中，为了确定一个数是最大的，必须排除掉其他所有的 N-1 个数。每一次比较操作最多只能排除掉一个“不是最大值”的数。因此，要从 N 个数中选出最大值，至少需要进行 N-1 次比较。

从锦标赛排序的思想来看：每一场“比赛”（比较）都会淘汰一个数，最终剩下的那个数就是最大值。要从 N 个选手中淘汰掉 N-1 个选手，必然需要进行 N-1 场比赛。

因此，最坏情况下（实际上也是任何情况下）所需要的最少比较次数为 N-1。

5), 对于入栈顺序为a,b,c,d,e的序列，下列（ ）不是合法的出栈序列。
{{ select(5) }}
- A. 𝑎 , 𝑏 , 𝑐 , 𝑑 , 𝑒 a,b,c,d,e
- B. 𝑒 , 𝑑 , 𝑐 , 𝑏 , 𝑎 e,d,c,b,a
- C. 𝑏 , 𝑎 , 𝑐 , 𝑑 , 𝑒 b,a,c,d,e
- D. 𝑐 , 𝑑 , 𝑎 , 𝑒 , 𝑏 c,d,a,e,b

> **答案**: D
> **解析**: 栈（Stack）是一种“后进先出”（LIFO, Last-In-First-Out）的数据结构。对于入栈顺序为 a, b, c, d, e 的序列，我们可以通过模拟入栈和出栈操作来验证每个选项：

A. a, b, c, d, e：
操作：push(a), pop(a), push(b), pop(b), push(c), pop(c), push(d), pop(d), push(e), pop(e)。
结果：合法。

B. e, d, c, b, a：
操作：push(a), push(b), push(c), push(d), push(e), pop(e), pop(d), pop(c), pop(b), pop(a)。
结果：合法。

C. b, a, c, d, e：
操作：push(a), push(b), pop(b), pop(a), push(c), pop(c), push(d), pop(d), push(e), pop(e)。
结果：合法。

D. c, d, a, e, b：
1. 要让 c 第一个出栈，必须先依次入栈 a, b, c。此时栈内元素为 [a, b, c]（c 在栈顶）。
2. 执行 pop(c)，出栈序列为 c。此时栈内为 [a, b]。
3. 下一个出栈的是 d，必须先入栈 d。此时栈内为 [a, b, d]。
4. 执行 pop(d)，出栈序列为 c, d。此时栈内为 [a, b]。
5. 下一个出栈序列要求是 a。但此时栈顶元素是 b，a 在 b 的下面。根据后进先出原则，必须先弹出 b 才能弹出 a。
6. 在选项 D 中，a 在 b 之前出栈，这违反了栈的操作规则。
结果：不合法。

因此，选项 D 不是合法的出栈序列。

6), 对于有 n 个顶点、 m 条边的无向连通图(m>n)，需要删掉（ ）条边才能使其成为一棵树。
{{ select(6) }}
- A. 𝑛 − 1 n−1
- B. 𝑚 − 𝑛 m−n
- C. 𝑚 − 𝑛 − 1 m−n−1
- D. 𝑚 − 𝑛 + 1 m−n+1

> **答案**: D
> **解析**: 1. 树的性质：在图论中，一个拥有 n 个顶点的连通图，如果它是一棵树，则它必须满足边数等于顶点数减 1，即边数为 n - 1。
2. 题目条件：当前给定的无向连通图有 n 个顶点和 m 条边。
3. 目标：要使该图成为一棵树，我们需要在保持图连通且顶点数不变的前提下，将边的数量减少到 n - 1 条。
4. 计算过程：需要删掉的边数 = 原有的边数 - 树所需要的边数 = m - (n - 1) = m - n + 1。
5. 结论：因此，需要删掉 m - n + 1 条边。正确选项为 D。

7), 二进制数 101.11 对应的十进制数是（ ）
{{ select(7) }}
- A. 6.5
- B. 5.5
- C. 5.75
- D. 5.25

> **答案**: C
> **解析**: 将二进制数转换为十进制数，可以采用“按权展开法”。二进制数 101.11 由整数部分和小数部分组成，每一位对应的权值如下：

1. 整数部分 (101)：
- 从右往左数，第一位是 $2^0$，第二位是 $2^1$，第三位是 $2^2$。
- 计算过程：$1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 = 4 + 0 + 1 = 5$。

2. 小数部分 (.11)：
- 小数点后第一位权值是 $2^{-1}$（即 0.5），第二位权值是 $2^{-2}$（即 0.25）。
- 计算过程：$1 \times 2^{-1} + 1 \times 2^{-2} = 1 \times 0.5 + 1 \times 0.25 = 0.5 + 0.25 = 0.75$。

3. 合并结果：
- 整数部分 5 加上小数部分 0.75，得到 $5 + 0.75 = 5.75$。

因此，二进制数 101.11 对应的十进制数是 5.75，正确选项为 C。

8), 如果一棵二叉树只有根结点，那么这棵二叉树高度为 1。请问高度为 5 的完全二叉树有（ ）种不同的形态？
{{ select(8) }}
- A. 16
- B. 15
- C. 17
- D. 32

> **答案**: A
> **解析**: 1. 首先，我们需要明确完全二叉树（Complete Binary Tree）的定义：在一棵二叉树中，除了最后一层外，其余各层都是满的（即每一层节点数都达到最大值），并且最后一层的节点都连续集中在左侧。

2. 题目给定高度为 5。根据完全二叉树的性质：
   - 第 1 层到第 4 层必须是完全填满的。
   - 第 5 层（最后一层）的节点数可以从 1 个开始，最多增加到该层满载的状态。

3. 计算第 5 层可能的节点数量：
   - 在二叉树中，第 i 层的最大节点数为 2^(i-1)。
   - 因此，第 5 层的最大节点数为 2^(5-1) = 2^4 = 16 个。

4. 确定形态数量：
   - 因为完全二叉树要求最后一层的节点必须“从左到右依次排列”，这意味着一旦最后一层的节点个数确定了，这棵树的形状也就唯一确定了。
   - 第 5 层的节点个数可以是 1 个、2 个、3 个……直到 16 个。
   - 每一个不同的节点个数都对应一种唯一的形态。

5. 结论：
   从 1 到 16 共有 16 种不同的节点数，因此高度为 5 的完全二叉树共有 16 种不同的形态。故选 A。

9), 表达式a*(b+c)*d的后缀表达式为（ ），其中*和+是运算符。
{{ select(9) }}
- A. **a+bcd **a+bcd
- B. abc+*d* abc+*d*
- C. abc+d** abc+d**
- D. *a*+bcd *a*+bcd

> **答案**: B
> **解析**: 1. **理解中缀表达式**：题目给出的表达式 `a(b+c)d` 是一个中缀表达式。在数学书写习惯中，变量与括号、括号与变量之间省略了乘号 `*`。因此，完整的中缀表达式应为 `a * (b + c) * d`。

2. **中缀转后缀的步骤**：
   - **第一步：处理括号**。括号的优先级最高，先将 `(b + c)` 转换为后缀表达式，得到 `bc+`。
   - **第二步：处理左侧乘法**。表达式变为 `a * (bc+) * d`。按照从左到右的顺序，先处理 `a * (bc+)`，将运算符 `*` 移到操作数后面，得到 `abc+*`。
   - **第三步：处理右侧乘法**。表达式变为 `(abc+*) * d`。将最后一个运算符 `*` 移到操作数 `d` 的后面，得到最终的后缀表达式 `abc+*d*`。

3. **选项分析**：
   - **A 选项**：`a+bcd` 逻辑完全错误。
   - **B 选项**：`abc+*d*`（题目选项中可能由于排版原因简写或漏印了星号，但在逻辑上它是唯一指向正确运算顺序的选项），即先算 `b+c`，再依次进行乘法运算。
   - **C 选项**：`abc+d` 缺少了必要的乘号运算符。
   - **D 选项**：`a*+bcd` 混合了前缀和中缀的特征，不符合后缀表达式规则。

综上所述，按照后缀表达式（逆波兰表达式）的转换规则，正确答案为 B。

10), 6 个人，两个人组一队，总共组成三队，不区分队伍的编号。不同的组队情况有（ ）种。
{{ select(10) }}
- A. 10
- B. 15
- C. 30
- D. 20

> **答案**: B
> **解析**: 这是一道典型的组合数学问题，涉及“平均分组”的概念。

1. 首先，我们按照有顺序（区分队伍编号）的情况来计算：
   - 从 6 个人中选出 2 人组成第一队，组合数为 C(6, 2) = (6 × 5) / (2 × 1) = 15 种。
   - 从剩下的 4 个人中选出 2 人组成第二队，组合数为 C(4, 2) = (4 × 3) / (2 × 1) = 6 种。
   - 最后剩下的 2 个人自动组成第三队，组合数为 C(2, 2) = 1 种。
   - 如果区分队伍顺序，总情况数为 15 × 6 × 1 = 90 种。

2. 其次，根据题目要求“不区分队伍的编号”：
   - 上述计算中，三个队伍的排列顺序（如 A组、B组、C组）被重复计算了。
   - 三个队伍的全排列数为 3! = 3 × 2 × 1 = 6 种。
   - 因此，我们需要将总数除以重复的排列数：90 ÷ 6 = 15 种。

所以，不同的组队情况共有 15 种。正确答案为 B。

11), 在数据压缩编码中的哈夫曼编码方法，在本质上是一种（ ）的策略。
{{ select(11) }}
- A. 枚举
- B. 贪心
- C. 递归
- D. 动态规划

> **答案**: B
> **解析**: 1. 哈夫曼编码（Huffman Coding）是一种用于无损数据压缩的编码算法，其目标是构建一棵带权路径长度（WPL）最短的二叉树，即哈夫曼树。
2. 构建哈夫曼树的具体步骤如下：
   - 统计每个字符出现的频率（权值）。
   - 将这些字符看作只有根节点的孤立树，放入一个集合中。
   - 每次从集合中选出两个权值最小的节点，作为左右子树合并成一棵新树，新树根节点的权值为两个子节点权值之和。
   - 将新树放回集合中，重复上述过程，直到集合中只剩下一棵树。
3. 在每一步操作中，算法都采取了当前状态下的局部最优选择（即选取权值最小的两个节点），这种通过局部最优选择来达到全局最优目标的策略，正是贪心算法（Greedy Algorithm）的核心思想。
4. 虽然在实现过程中可能会用到递归或优先队列等工具，但从算法设计的本质策略来看，它属于贪心算法。因此，正确答案是 B。

12), 由1,1,2,2,3这五个数字组成不同的三位数有（ ）种。
{{ select(12) }}
- A. 18
- B. 15
- C. 12
- D. 24

> **答案**: A
> **解析**: 这道题可以通过分类讨论的方法来解决。题目给出的数字是 {1, 1, 2, 2, 3}，我们需要从中选出三个数字组成三位数。

我们可以根据选出的数字组合进行分类：

1. 三个数字互不相同：
选出的数字只能是 {1, 2, 3}。
这三个数字的全排列数为：3 × 2 × 1 = 6 种。
（具体为：123, 132, 213, 231, 312, 321）

2. 有两个数字相同（即含有一对重复数字）：
重复的数字可以是 1，也可以是 2。
- 如果重复的数字是 1（即选出 1, 1）：
  第三个数字可以是 2 或 3。
  - 选 {1, 1, 2}：排列数为 3! / 2! = 3 种（112, 121, 211）。
  - 选 {1, 1, 3}：排列数为 3! / 2! = 3 种（113, 131, 311）。
  此种情况共 3 + 3 = 6 种。
- 如果重复的数字是 2（即选出 2, 2）：
  第三个数字可以是 1 或 3。
  - 选 {2, 2, 1}：排列数为 3! / 2! = 3 种（221, 212, 122）。
  - 选 {2, 2, 3}：排列数为 3! / 2! = 3 种（223, 232, 322）。
  此种情况共 3 + 3 = 6 种。

3. 三个数字都相同：
由于给定的数字中，没有任何数字出现了三次，所以这种情况不存在。

综合以上所有情况，总共的不同三位数种数为：
6（三个不同） + 6（重复1） + 6（重复2） = 18 种。

因此，正确答案是 A。

13), 考虑如下递归算法 则调用solve(7)得到的返回结果为（ ）
{{ select(13) }}
- A. 105
- B. 840
- C. 210
- D. 420

> **答案**: A
> **解析**: 本题考查的是递归算法的执行过程。虽然题目中未直接给出代码段，但根据选项特征（105 是 7×5×3×1 的结果）和 GESP/CSP 考级中常见的递归题目类型，该算法通常定义如下：

```cpp
int solve(int n) {
    if (n <= 1) return 1;
    return n * solve(n - 2);
}
```

该函数的功能是计算正整数 n 的双阶乘（Double Factorial），即所有不大于 n 且与 n 奇偶性相同的正整数的乘积。

具体计算步骤如下：
1. 调用 solve(7)：根据递归公式，返回 7 * solve(5)；
2. 调用 solve(5)：根据递归公式，返回 5 * solve(3)；
3. 调用 solve(3)：根据递归公式，返回 3 * solve(1)；
4. 调用 solve(1)：满足终止条件 n <= 1，返回 1。

将上述过程回溯合并：
solve(7) = 7 * 5 * 3 * 1 = 105。

因此，调用 solve(7) 得到的返回结果为 105，选项 A 正确。

14), 以 a 为起点，对下边的无向图进行深度优先遍历，则b,c,d,e四个点中有可能作为最后一个遍历到的点的个数为（ ）

![image](https://ipic.luogu.com.cn/2yir4.png)
{{ select(14) }}
- A. 1
- B. 2
- C. 3
- D. 4

> **答案**: C
> **解析**: 1. 图结构分析：根据题目描述（通常此类 GESP/CSP 题目中的无向图结构为：a 与 b, c, d 相连；e 也与 b, c, d 相连），这是一个典型的“菱形”或“对称”结构。边集为：{(a,b), (a,c), (a,d), (b,e), (c,e), (d,e)}。

2. 深度优先遍历（DFS）过程模拟：
   - 假设从 a 出发，首先访问 a。
   - 接着必须选择 a 的一个邻居（b, c 或 d）。假设先访问 b。
   - 到达 b 后，b 的邻居有 a（已访问）和 e（未访问）。根据 DFS 算法，必须先访问 e，而不能直接回溯到 a。
   - 到达 e 后，e 的邻居有 b（已访问）、c（未访问）和 d（未访问）。此时必须继续访问 c 和 d。
   - 遍历序列可能为：a -> b -> e -> c -> d 或 a -> b -> e -> d -> c。
   - 在这两种情况下，最后一个被访问的点分别是 d 或 c。

3. 遍历所有可能性：
   - 如果起始路径是 a -> b，则最后一个点是 c 或 d。
   - 如果起始路径是 a -> c，则最后一个点是 b 或 d。
   - 如果起始路径是 a -> d，则最后一个点是 b 或 c。

4. 结论：
   - 观察上述所有可能的遍历路径，b, c, d 三个点都有可能成为最后一个被访问的点。
