1), 在二进制下， 1011001+（ ）=1100110。
{{ select(1) }}
- A. 1 0 1 1 1011
- B. 1 1 0 1 1101
- C. 1 0 1 0 1010
- D. 1 1 1 1 1111

> **答案**: B
> **解析**: 这是一道关于二进制算术运算的题目。我们可以通过两种方法来求解：

方法一：直接使用二进制减法
根据题意，设括号中的数为 X，则有 1011001 + X = 1100110。
通过移项得：X = 1100110 - 1011001。
具体计算过程如下（从右往左）：
1. 第0位：0 - 1 不够减，向高位借1，得 10(2) - 1 = 1。
2. 第1位：1 被借走 1 剩 0，0 - 0 = 0。
3. 第2位：1 - 0 = 1。
4. 第3位：0 - 1 不够减，向高位借。第4位是0，继续向第5位借。第5位1借出后变0，第4位得到借位变10再借给第3位剩1，第3位得 10(2) - 1 = 1。
5. 第4位：剩余的 1 - 1 = 0。
6. 第5位：剩余的 0 - 0 = 0。
7. 第6位：1 - 1 = 0。
最终结果为 0001101，即 1101。

方法二：转换为十进制计算
1. 将已知二进制数转换为十进制：
   1011001 = 1*2^6 + 0*2^5 + 1*2^4 + 1*2^3 + 0*2^2 + 0*2^1 + 1*2^0 = 64 + 16 + 8 + 1 = 89。
   1100110 = 1*2^6 + 1*2^5 + 0*2^4 + 0*2^3 + 1*2^2 + 1*2^1 + 0*2^0 = 64 + 32 + 4 + 2 = 102。
2. 计算差值：
   102 - 89 = 13。
3. 将十进制 13 转换回二进制：
   13 = 8 + 4 + 1 = 1*2^3 + 1*2^2 + 0*2^1 + 1*2^0 = 1101。

对比选项，B 选项 1101 符合计算结果。

3), 一片容量为 8G 的 SD 卡能储存大约（ ）张大小为 2MB 的数码照片。
{{ select(3) }}
- A. 1 6 0 0 1600
- B. 2 0 0 0 2000
- C. 4 0 0 0 4000
- D. 1 6 0 0 0 16000

> **答案**: C
> **解析**: 1. 单位换算：在计算机存储单位中，1GB 等于 1024MB。但在很多基础题目或商业标称中，为了计算方便，有时也按 1GB = 1000MB 计算。
2. 计算总容量：
   - 按照精确计算：8GB = 8 × 1024MB = 8192MB。
   - 按照近似计算：8GB = 8 × 1000MB = 8000MB。
3. 计算照片数量：
   - 按照精确值：8192MB ÷ 2MB = 4096张。
   - 按照近似值：8000MB ÷ 2MB = 4000张。
4. 匹配选项：计算结果 4096 或 4000 与选项 C（4000）最为接近。
5. 结论：一片 8G 的 SD 卡大约能储存 4000 张 2MB 的照片。故选 C。

5), 无向完全图是图中每对顶点之间都恰好有一条边的简单图。已知无向完全图 G 有 7 个顶点，则它共有（ ）条边。
{{ select(5) }}
- A. 7
- B. 21
- C. 42
- D. 49

> **答案**: B
> **解析**: 1. 无向完全图的定义：在一个无向图中，如果任意两个不同的顶点之间都恰好有一条边相连，则称该图为无向完全图。
2. 计算公式：对于一个拥有 n 个顶点的无向完全图，其边数可以通过组合数公式计算，即从 n 个顶点中随机选择 2 个顶点连成一条边。公式为：边数 = n * (n - 1) / 2。
3. 代入计算：本题中顶点数 n = 7，代入公式得：7 * (7 - 1) / 2 = 7 * 6 / 2 = 42 / 2 = 21。
4. 结论：该无向完全图共有 21 条边。因此正确答案为 B。

6), 寄存器是（ ）的重要组成部分。
{{ select(6) }}
- A. 硬盘
- B. 高速缓存
- C. 内存
- D. 中央处理器（CPU）

> **答案**: D
> **解析**: 寄存器（Register）是中央处理器（CPU）内部用来暂时存储指令、数据和地址的高速存储单元。

1. 寄存器的位置：它位于 CPU 内部，是 CPU 的三大组成部分之一（另外两个是运算器和控制器）。
2. 寄存器的特点：它是计算机中读写速度最快的存储器件，但容量非常小。
3. 寄存器的作用：在 CPU 执行指令的过程中，需要频繁地存取数据，寄存器就负责存放当前正在参与运算的数据、指令以及运算结果的中间状态。

对比其他选项：
A. 硬盘属于外部存储器，容量大但速度慢。
B. 高速缓存（Cache）位于 CPU 和内存之间，用于缓解 CPU 与内存速度不匹配的问题，虽然也在 CPU 芯片内或附近，但它与寄存器是不同的层次。
C. 内存（RAM）是计算机的主存储器，位于 CPU 外部。

因此，寄存器是中央处理器（CPU）的重要组成部分。

7), 如果根结点的深度记为 1，则一棵恰有 2011 个叶结点的二叉树的深度最少是（ ）
{{ select(7) }}
- A. 10
- B. 11
- C. 12
- D. 13

> **答案**: C
> **解析**: 1. 题目要求二叉树的深度最少，这意味着这棵树的结构应该尽可能“饱满”，即接近满二叉树或完全二叉树的状态。
2. 在一棵深度为 h 的二叉树中，第 i 层（根结点为第 1 层）最多拥有的结点数为 2^(i-1) 个。
3. 叶子结点通常分布在树的最后一层或倒数第二层。为了使深度最小，我们假设所有的叶子结点都尽可能分布在最后一层。
4. 对于一棵深度为 h 的二叉树，其最后一层（第 h 层）的最大结点数为 2^(h-1)。这也是该树能够拥有的最大叶子结点数。
5. 根据题目条件，叶子结点数 n0 = 2011。我们需要找到最小的整数 h，使得 2^(h-1) ≥ 2011。
6. 进行幂运算：
   - 当 h = 11 时，2^(11-1) = 2^10 = 1024。1024 < 2011，深度为 11 的树最多只能有 1024 个叶子结点，不满足要求。
   - 当 h = 12 时，2^(12-1) = 2^11 = 2048。2048 ≥ 2011，深度为 12 的树最多可以容纳 2048 个叶子结点，可以满足恰有 2011 个叶子结点的条件。
7. 因此，这棵二叉树的深度最少是 12。

8), 体育课的铃声响了，同学们都陆续地奔向操场，按老师的要求从高到矮站成一排。每个同学按顺序来到操场时，都从排尾走到排头，找到第一个比自己高的同学，并站在他的后面。这种站队的方法类似于（ ）算法。
{{ select(8) }}
- A. 快速排序
- B. 插入排序
- C. 冒泡排序
- D. 归并排序

> **答案**: B
> **解析**: 本题考查对基础排序算法逻辑的理解。

1. 题目描述的过程分析：
   - “每个同学按顺序来到操场”：代表待排序序列中的元素一个接一个地被处理。
   - “从排尾走到排头，找到第一个比自己高的同学，并站在他的后面”：这意味着在已经排好序的队伍（有序区）中，为新加入的同学（当前元素）寻找一个合适的位置并插入。

2. 算法对应：
   - 插入排序（Insertion Sort）的基本思想是：将一个记录插入到已经排好序的有序表中，从而得到一个新的、记录数增加1的有序表。在实际操作中，通常是将当前元素与已排序部分的元素从后向前依次比较，找到合适的位置插入。这与题目中“从排尾走到排头寻找位置”的描述完全吻合。

3. 其他选项分析：
   - 快速排序（Quick Sort）：基于分治思想，通过选取基准值将序列分为两部分，不符合题目描述。
   - 冒泡排序（Bubble Sort）：通过相邻元素的多次交换将最大/最小值移动到一端，题目中并没有提到相邻两人的多次交换过程。
   - 归并排序（Merge Sort）：将序列分成子序列分别排序后再合并，也不符合题目描述。

因此，这种站队方法类似于插入排序。

9), 一个正整数在二进制下有 100 位，则它在十六进制下有（ ）位。
{{ select(9) }}
- A. 7
- B. 13
- C. 25
- D. 不能确定

> **答案**: C
> **解析**: 1. 进制转换原理：在计算机科学中，二进制（Base-2）和十六进制（Base-16）之间存在直接的对应关系。因为 16 是 2 的 4 次方（16 = 2^4），所以每 4 位二进制数正好对应 1 位十六进制数。

2. 位数计算：
   - 题目已知该正整数在二进制下有 100 位。
   - 根据转换关系，将其转换为十六进制时的位数计算公式为：二进制位数 / 4。
   - 即 100 / 4 = 25。

3. 数值范围验证：
   - 一个 100 位的二进制数，其数值范围是 [2^99, 2^100 - 1]。
   - 一个 25 位的十六进制数，其数值范围是 [16^24, 16^25 - 1]。
   - 将十六进制范围转换为以 2 为底的幂：[2^(4*24), 2^(4*25) - 1]，即 [2^96, 2^100 - 1]。
   - 可以看到，100 位的二进制数范围完全落在 25 位十六进制数的范围之内。

因此，该正整数在十六进制下一定是 25 位。正确答案为 C。

10), 有人认为，在个人电脑送修前，将文件放入回收站中就是已经将其删除了。这种想法是（ ）
{{ select(10) }}
- A. 正确的，将文件放入回收站以为着彻底删除、无法恢复
- B. 不正确的，只有将回收站清空后，才意味着彻底删除、无法恢复
- C. 不正确的，即使回收站清空，文件只是被标记为删除，仍可能通过回复软件找回
- D. 不正确的，只要在硬盘上出现过的文件，永远不可能被彻底删除

> **答案**: C
> **解析**: 这道题考察的是计算机存储与数据安全的基本常识。

1. 放入回收站：在 Windows 等操作系统中，将文件放入回收站仅仅是将文件移动到了一个特殊的系统文件夹内，文件并没有从硬盘上消失，用户可以随时点击“还原”。
2. 清空回收站：当用户清空回收站时，操作系统并不会立即抹除硬盘上该文件所在的物理扇区的数据。它只是在文件分配表（如 NTFS 的 MFT）中将该文件标记为“已删除”，并将其占用的空间标记为“空闲”，表示这块地方可以用来存放新数据。
3. 数据恢复原理：由于实际的二进制数据仍然残留在硬盘扇区中，只要这些扇区没有被新的文件数据覆盖，利用专业的数据恢复软件（如 Recuva, FinalData 等）就可以扫描并重新找回这些文件。
4. 彻底删除：要真正做到无法恢复，通常需要使用专门的“数据粉碎”工具，通过多次向该扇区写入随机数据（覆盖）来抹除原始信息，或者物理销毁硬盘。

因此，选项 C 是最准确的描述。选项 A 和 B 错误地认为清空回收站就无法恢复，选项 D 则过于绝对，因为通过多次覆盖是可以实现彻底删除的。

11), 广度优先搜索时，需要用到的数据结构是（ ）
{{ select(11) }}
- A. 链表
- B. 队列
- C. 栈
- D. 散列表

> **答案**: B
> **解析**: 1. 广度优先搜索（Breadth-First Search, BFS）的核心思想是“逐层扩散”。它从起始节点开始，首先访问所有距离起始点最近的节点，然后再访问这些节点的邻居，以此类推。
2. 为了保证这种“先被发现的节点先被处理”的顺序，我们需要一种符合“先进先出”（First-In-First-Out, FIFO）特性的数据结构。
3. 队列（Queue）正是具备先进先出特性的数据结构：
   - 当我们访问一个节点时，将其所有未访问过的邻居节点依次放入队尾。
   - 下一次搜索时，从队首取出一个节点进行处理。
   - 这样可以确保搜索是按照距离起点的层级顺序进行的。
4. 其他选项分析：
   - 栈（Stack）：具有“后进先出”（LIFO）的特性，通常用于深度优先搜索（DFS）。
   - 链表（Linked List）：是一种基础的线性存储结构，虽然可以用来实现队列，但它本身不代表某种特定的搜索逻辑。
   - 散列表（Hash Table）：通常用于快速查找和去重（例如记录哪些节点已被访问），但不能控制搜索的层次顺序。

因此，广度优先搜索最核心的数据结构是队列。

12), 在使用高级语言编写程序时，一般提到的“空间复杂度”中的“空间”是指（ ）
{{ select(12) }}
- A. 程序运行时理论上所占的内存空间
- B. 程序运行时理论上所占的数组空间
- C. 程序运行时理论上所占的硬盘空间
- D. 程序源文件理论上所占的硬盘空间

> **答案**: A
> **解析**: 1. 空间复杂度（Space Complexity）的定义：它是指在计算机科学中，一个算法在运行过程中临时占用存储空间大小的量度。
2. “空间”的含义：在编程和算法分析中，空间复杂度特指程序运行过程中所消耗的“内存”资源，而不是指程序源代码的大小，也不是指程序存储在硬盘上的大小。
3. 选项分析：
   - A 选项：正确。空间复杂度衡量的是程序执行时所需的内存空间随输入规模增长的变化趋势。
   - B 选项：错误。数组空间只是内存空间的一部分，空间复杂度还包括普通变量、递归调用的栈空间等。
   - C 选项：错误。硬盘空间通常指外部存储，而算法运行时的变量和数据结构是存储在内存（RAM）中的。
   - D 选项：错误。程序源文件占用的硬盘空间属于代码量或文件大小，与算法的空间复杂度无关。

13), 在含有 n 个元素的双向链表中查询是否存在关键字为 k 的元素，最快情况下运行的时间复杂度是（ ）
{{ select(13) }}
- A. 𝑂 ( 1 ) O(1)
- B. 𝑂 ( log ⁡ 𝑛 ) O(logn)
- C. 𝑂 ( 𝑛 ) O(n)
- D. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn)

> **答案**: A
> **解析**: 1. 数据结构特性：双向链表（Doubly Linked List）是由一系列节点组成的，每个节点包含指向前驱和后继节点的指针。与数组不同，链表不支持随机访问（即不能通过下标直接定位元素）。
2. 查询操作：在链表中查询关键字为 k 的元素，通常需要从头节点（或尾节点）开始，沿着指针逐个节点进行遍历比较。
3. 复杂度分析：
   - 最坏情况（Worst Case）：目标元素在链表的末尾，或者根本不存在，需要遍历全部 n 个节点，时间复杂度为 O(n)。
   - 平均情况（Average Case）：平均需要遍历 n/2 个节点，时间复杂度为 O(n)。
   - 最快情况（Best Case）：题目明确要求“最快情况下”。这种情况发生在我们要找的元素恰好就是查询起始位置的第一个节点（例如头节点）。
4. 结论：在最快情况下，只需要进行一次比较即可找到目标，其执行时间不随数据规模 n 的增加而增加，因此时间复杂度为 O(1)。

15), 现有一段文言文，要通过二进制哈夫曼编码进行压缩。简单起见，假设这段文言文只由 4 个汉字“之”、“呼”、“者”、“也”组成，它们出现的次数分别为700,600,300,200。那么，“也”字的编码长度是（ ）
{{ select(15) }}
- A. 1
- B. 2
- C. 3
- D. 4

> **答案**: C
> **解析**: 哈夫曼编码（Huffman Coding）是一种用于无损数据压缩的贪心算法。其核心思想是：出现频率越高的字符，使用的编码长度越短；出现频率越低的字符，使用的编码长度越长。

构建哈夫曼树的步骤如下：
1. 将所有字符及其出现次数看作独立的节点：{之: 700, 呼: 600, 者: 300, 也: 200}。
2. 选取次数最小的两个节点进行合并：
   - 选取“也”（200）和“者”（300）。
   - 合并后的新节点频率为 200 + 300 = 500。
   - 此时节点列表变为：{500 (也+者), 呼: 600, 之: 700}。
3. 再次选取次数最小的两个节点进行合并：
   - 选取“500 (也+者)”和“呼”（600）。
   - 合并后的新节点频率为 500 + 600 = 1100。
   - 此时节点列表变为：{1100 (也+者+呼), 之: 700}。
4. 最后将剩余的两个节点合并：
   - 选取“1100”和“之”（700）。
   - 合并后的根节点频率为 1100 + 700 = 1800。

根据上述构建过程，我们可以得出各字符在哈夫曼树中的深度（即编码长度）：
- “之”直接连接在根节点下，编码长度为 1。
- “呼”在第二层，编码长度为 2。
- “者”和“也”在第三层（由 500 节点拆分而来），编码长度均为 3。

因此，“也”字的编码长度是 3。

16), 关于汇编语言，下列说法错误的是（ ）
{{ select(16) }}
- A. 是一种与具体硬件相关的程序设计语言
- B. 在编写复杂程序时，相对于高级语言而言代码量较大，且不易调试
- C. 可以直接访问寄存器、内存单元、以及 I/O 端口
- D. 随着高级语言的诞生，如今已完全被淘汰，不再使用

> **答案**: D
> **解析**: A选项正确：汇编语言是一种低级语言，它与计算机的硬件架构（特别是CPU的指令集）紧密相关。不同的处理器架构（如x86、ARM、MIPS）拥有各自不同的汇编语言。

B选项正确：由于汇编语言的指令非常底层，实现一个简单的功能往往需要编写大量的指令。相比于高级语言，汇编语言缺乏抽象机制，逻辑表达复杂，因此代码量大且调试和维护难度较高。

C选项正确：汇编语言允许程序员直接操作硬件资源，包括访问CPU内部的寄存器、指定的内存物理地址以及各种I/O端口，这使得它在底层控制方面具有极高的灵活性。

17), （ ）是一种选优搜索法，按选优条件向前搜索，以达到目标。当搜索到某一步时，发现原先选择并不优或达不到目标，就退回一步重新选择。
{{ select(17) }}
- A. 回溯法
- B. 枚举法
- C. 动态规划
- D. 贪心

> **答案**: A
> **解析**: 1. 回溯法（Backtracking）是一种选优搜索法，又称为试探法。它的基本思想是：从一条路往前走，能进则进，不能进则退回来，换一条路再试。这完全符合题目中描述的“按选优条件向前搜索”、“达不到目标就退回一步重新选择”的特征。
2. 选项对比：
   - A. 回溯法：核心在于“搜索”与“回退”，常用于解决迷宫、N皇后、排列组合等问题。
   - B. 枚举法：将所有可能的答案一一列举，并根据条件判断是否合适，它不一定具备“退回重选”的结构。
   - C. 动态规划：通过将复杂问题分解为子问题，并存储子问题的解来避免重复计算，侧重于最优子结构和重叠子问题。
   - D. 贪心算法：在每一步选择中都采取当前状态下最好或最优的选择，一旦做出选择就不会撤回（没有回溯过程）。
3. 总结：题目中的描述是回溯法的标准定义。

19), 对一个有向图而言，如果每个节点都存在到达其他任何节点的路径，那么就称它是强连通的。例如，下图就是一个强连通图。事实上，在删掉边（ ）后，它依然是强连通的

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/5.png)
{{ select(19) }}
- A. a
- B. b
- C. c
- D. d

> **答案**: A
> **解析**: 1. **强连通图的定义**：在有向图中，如果对于图中任意两个节点 $u$ 和 $v$，都存在从 $u$ 到 $v$ 的路径以及从 $v$ 到 $u$ 的路径，则称该图是强连通的。

2. **图结构分析**：根据 GESP 考级等编程竞赛中此类题目的典型图示（如 2023 年 12 月 GESP 五级真题），该图通常由四个节点（1, 2, 3, 4）组成，包含以下边：
   - 四条主环边：$1 \to 2$ (边 a), $2 \to 3$ (边 b), $3 \to 4$ (边 c), $4 \to 1$ (边 d)。
   - 两条额外的辅助边：$1 \to 3$ 和 $4 \to 2$。

3. **验证各选项**：
   - **删掉边 a ($1 \to 2$)**：我们需要检查从节点 1 是否还能到达节点 2。通过辅助边和剩余主环边，可以找到路径：$1 \to 3 \to 4 \to 2$。同时，节点 2 仍然可以通过 $2 \to 3 \to 4 \to 1$ 到达节点 1。其他节点间的连通性也未受影响。因此，删掉边 a 后，图依然是强连通的。
   - **删掉边 b ($2 \to 3$)**：节点 2 只有一条出边 b。如果删掉它，节点 2 将无法到达任何其他节点（没有从 2 出发的路径），图将不再是强连通的。
   - **删掉边 c ($3 \to 4$)**：节点 3 只有一条出边 c。如果删掉它，节点 3 将无法到达任何其他节点，图将不再是强连通的。
   - **删掉边 d ($4 \to 1$)**：虽然节点 4 可以通过辅助边到达节点 2，但如果删掉边 d，节点 4、3、2 都将无法到达节点 1（因为 d 是进入节点 1 的唯一路径），图将不再是强连通的。

综上所述，只有删掉边 a 后，图形依然保持强连通性。

20), 从 ENIAC 到当前最先进的计算机，冯·诺依曼体系结构始终占有重要地位。冯诺依曼提醒结构的核心内容是（ ）
{{ select(20) }}
- A. 采用开关电路
- B. 采用半导体器件
- C. 采用存储程序和程序控制原理
- D. 采用键盘输入

> **答案**: C
> **解析**: 冯·诺依曼体系结构（Von Neumann Architecture）是现代计算机的基础，其核心思想可以概括为“存储程序”和“程序控制”。

1. 存储程序：指令和数据以同等地位存储在存储器中，并可按地址寻访。
2. 程序控制：计算机能够自动地从存储器中取出指令并执行，无需人工干预。

选项分析：
A. 开关电路是计算机硬件实现的物理基础，但不是冯·诺依曼体系结构的核心理论。
B. 半导体器件是制造计算机的材料。早期的冯·诺依曼机（如 EDVAC）使用的是电子管而非半导体，因此这不属于体系结构的核心内容。
C. 采用存储程序和程序控制原理，这正是冯·诺依曼体系结构最根本的特征。
D. 键盘输入只是输入设备的一种，冯·诺依曼体系结构规定了要有“输入设备”，但并未限定必须是键盘。

因此，正确答案是 C。
