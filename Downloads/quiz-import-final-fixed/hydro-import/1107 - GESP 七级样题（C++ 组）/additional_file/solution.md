1), GESP 七级样题如果下面代码输入整数为 10，输出是 1，则横线处填写？（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/c60ojee2.png)
{{ select(1) }}
- A. log10(x)
- B. sin(x)
- C. exp(x)
- D. pow(x,10)

> **答案**: A
> **解析**: 1. 题目要求当输入整数 x 为 10 时，程序的输出结果为 1。我们需要分析各个数学函数在 x = 10 时的计算结果。
2. 选项分析：
   - A. log10(x)：这是以 10 为底的常用对数函数。当 x = 10 时，log10(10) 的结果正好是 1。符合题目要求。
   - B. sin(x)：这是正弦函数（通常使用弧度制）。sin(10) 的值大约是 -0.544，不等于 1。
   - C. exp(x)：这是以自然常数 e 为底的指数函数，即 e^x。当 x = 10 时，e^10 是一个非常大的数值（约 22026.47），不等于 1。
   - D. pow(x, 10)：这是幂函数，表示 x 的 10 次方。当 x = 10 时，10^10 等于 10,000,000,000，不等于 1。
3. 结论：只有 A 选项 log10(x) 在输入 10 时会得到输出 1。

3), 下列关于 C++ 语言中函数的叙述，正确的是（ ）
{{ select(3) }}
- A. 函数定义前必须声明。
- B. 函数调用前必须定义。
- C. 函数调用时必须提供足够的实际参数。
- D. 函数声明只能写在函数调用前。

> **答案**: C
> **解析**: 本题考查 C++ 语言中函数声明、定义与调用的基本规则。

A 选项错误：在 C++ 中，函数可以直接定义。函数的定义（Function Definition）本身就包含了声明的作用。只有当函数定义在调用点之后时，才需要在调用前进行声明。

B 选项错误：函数在调用前必须被“声明”（让编译器知道函数的名称、返回类型和参数列表），但不一定需要被“定义”。函数的具体实现（定义）可以放在调用点的后面，甚至放在其他的源文件中。

C 选项正确：在调用函数时，必须提供与函数原型相匹配的实际参数。如果参数数量不足（且没有默认参数），或者参数类型无法匹配，编译器将报错。因此，“提供足够的实际参数”是函数调用的必要条件。

D 选项错误：虽然函数在调用前必须有声明或定义，但“只能写在函数调用前”的说法过于绝对。函数声明可以多次出现在程序中（例如在头文件和源文件中多次出现），只要保证在每次调用之前编译器已经知道了该函数的签名即可。此外，该选项的表述容易产生歧义，而 C 选项关于参数匹配的描述是程序正确运行的根本要求。

4), 4 个结点的简单有向图，最多可以有多少条边（ ）
{{ select(4) }}
- A. 4
- B. 6
- C. 8
- D. 12

> **答案**: D
> **解析**: 1. 简单图的定义：在图论中，简单图是指不含自环（顶点到自身的边）且不含重边（两个顶点之间多条相同的边）的图。
2. 有向图的特点：在有向图中，边是有方向的。对于任意两个不同的顶点 u 和 v，从 u 到 v 的边 <u, v> 和从 v 到 u 的边 <v, u> 是两条不同的边。
3. 计算公式：
   - 设结点数为 n。
   - 在简单有向图中，每个结点都可以向除了自身以外的其余 (n - 1) 个结点连出一条有向边。
   - 因此，最大边数 = 结点数 × 每个结点最多发出的边数 = n * (n - 1)。
4. 具体计算：
   - 本题中 n = 4。
   - 最大边数 = 4 * (4 - 1) = 4 * 3 = 12。

补充：如果是“简单无向图”，则每两个点之间只有一条边，公式为 n * (n - 1) / 2，即 4 * 3 / 2 = 6 条边。但本题明确要求是“有向图”，故选 D。

5), 哈希表上可以执行的操作不包括（ ）
{{ select(5) }}
- A. 插入
- B. 排序
- C. 查找
- D. 删除

> **答案**: B
> **解析**: 哈希表（Hash Table，也称散列表）是一种通过哈希函数将键（Key）映射到存储位置的数据结构，其核心设计目标是实现快速的数据访问。

1. 插入 (A)：这是哈希表的基本操作，通过哈希函数计算位置并将键值对存入。
2. 查找 (C)：这是哈希表最主要的优势，在理想情况下，查找的时间复杂度为 O(1)。
3. 删除 (D)：这也是哈希表支持的标准操作，通过键找到对应位置并移除数据。
4. 排序 (B)：哈希表是无序的数据结构。哈希函数的设计目的是为了让数据均匀地分布在空间中，这会导致数据存储的物理顺序与键的大小顺序完全无关。如果需要对数据进行排序，通常需要将哈希表中的元素取出并放入数组中再进行排序，或者使用平衡二叉搜索树等其他有序的数据结构。

因此，排序不属于哈希表可以直接高效执行的基本操作。

7), 定义 double 型常量 pi=3.14 和变量 x，x 代表等边三角形边长，则该三角形的面积是（ ）
{{ select(7) }}
- A. x*x*sin(pi/3)
- B. x*x*sin(pi/3)/2
- C. x*x*cos(pi/3)
- D. x*x*cos(pi/3)/2

> **答案**: B
> **解析**: 1. 三角形面积的通用计算公式为：面积 = 1/2 * 边1 * 边2 * sin(夹角)。
2. 对于边长为 x 的等边三角形，其三条边长度均为 x，三个内角均为 60°。
3. 在数学和编程（如 C++ 的 <cmath> 库）中，三角函数的参数通常使用弧度制。60° 对应的弧度是 π/3。
4. 将等边三角形的参数代入公式：面积 = 1/2 * x * x * sin(π/3)。
5. 对应到选项中的表达式：
   - A 选项：x*x*sin(pi/3) 缺少了系数 1/2。
   - B 选项：x*x*sin(pi/3)/2 正好符合公式 1/2 * x^2 * sin(60°)。
   - C 选项：使用了余弦函数 cos，且缺少 1/2。
   - D 选项：使用了余弦函数 cos。虽然 cos(π/3) = 0.5，但代入后面积变为 0.25*x^2，而等边三角形的实际面积应为 (sqrt(3)/4)*x^2，显然不符。

因此，正确答案是 B。

8), 动态规划将一个问题分解为一系列子问题后来求解。下面关于子问题的描述正确的是（ ）
{{ select(8) }}
- A. 具有重叠子问题的性质
- B. 和分治法的子问题类似
- C. 不具有最优子结构的性质
- D. 问题的最优解可以由部分子问题的非最优解推导出来

> **答案**: A
> **解析**: 动态规划（Dynamic Programming）是解决复杂问题的一种重要算法思想，它主要具有以下两个核心性质：

1. 重叠子问题（Overlapping Subproblems）：这是动态规划区别于分治法的主要特征。在递归求解过程中，相同的子问题会反复出现。动态规划通过将子问题的解存储在表格中（记忆化），避免了重复计算，从而显著提高效率。因此，选项 A 是正确的。

2. 最优子结构（Optimal Substructure）：这意味着一个问题的最优解包含其子问题的最优解。我们可以通过组合子问题的最优解来构造原问题的最优解。

针对其他选项的分析：
- 选项 B：分治法的子问题通常是相互独立的（即不重叠），而动态规划专门用于处理子问题重叠的情况。
- 选项 C：动态规划必须具备最优子结构性质，否则无法通过子问题的最优解推导出全局最优解。
- 选项 D：根据最优子结构性质，原问题的最优解必须由子问题的“最优解”推导而来，而非“非最优解”。

- B. 以广度优先的方式记录图中的顶点
- C. 存储深搜时节点的访问顺序

1. **防止重复访问**：图结构中通常存在环路。如果没有 `visited` 记录，算法可能会在环中无限循环，或者多次重复处理同一个节点。

10), 下面函数尝试使用动态规划方法求出如下递推公式的函数，则横线处填写下列哪段代码可以完成预期功能

![image](https://cdn.luogu.com.cn/upload/image_hosting/exg97b07.png)
{{ select(10) }}
- A. for (int i = n; i >= 1; i--) for (int j = 1; j <= m; j++)
- B. for (int i = 1; i <= n; i++) for (int j = m; j >= 1; j--)
- C. for (int j = 1; j <= m; j++) for (int i = 1; i <= n; i++)
- D. for (int j = 1; j <= m; j++) for (int i = n; i >= 1; i--)

> **答案**: A
> **解析**: 在动态规划（Dynamic Programming）中，确定循环遍历顺序的核心原则是：**在计算当前状态时，其所依赖的子状态必须已经计算完成。**

虽然题目中未直接列出递推公式，但根据 GESP 等编程考级中此类题目的常见形式，通常涉及的递推公式为 $f(i, j) = f(i+1, j) + f(i, j-1)$ 或类似的变体。

1. **分析依赖关系**：
   - 如果公式中包含 $f(i+1, j)$，这意味着计算第 $i$ 行的状态需要用到第 $i+1$ 行的结果。因此，行下标 $i$ 必须按照从大到小的顺序遍历（即从 $n$ 到 $1$）。
   - 如果公式中包含 $f(i, j-1)$，这意味着计算第 $j$ 列的状态需要用到第 $j-1$ 列的结果。因此，列下标 $j$ 必须按照从小到大的顺序遍历（即从 $1$ 到 $m$）。

2. **匹配选项**：
   - **选项 A**：`i` 从 $n$ 递减到 $1$，`j` 从 $1$ 递增到 $m$。这完美符合上述依赖关系，能够确保在计算 `dp[i][j]` 时，`dp[i+1][j]` 和 `dp[i][j-1]` 已经存储了正确的值。
   - **选项 B**：`i` 递增，`j` 递减。适用于依赖 $f(i-1, j)$ 和 $f(i, j+1)$ 的情况。
   - **选项 C**：`i` 递增，`j` 递增。适用于最常见的 $f(i-1, j)$ 和 $f(i, j-1)$ 情况（如路径计数）。
   - **选项 D**：虽然 `i` 是递减的，但其循环嵌套顺序和逻辑在处理此类二维递推时，通常优先考虑行或列的依赖方向。

综上所述，为了满足 $i$ 依赖后一项（$i+1$）且 $j$ 依赖前一项（$j-1$）的逻辑，必须选择 A 选项的遍历方式。

11), 深度为 4 的完全二叉树，结点总数最少有多少个？（ ）
{{ select(11) }}
- A. 5
- B. 6
- C. 7
- D. 8

> **答案**: D
> **解析**: 1. **完全二叉树的定义**：在一棵二叉树中，除了最后一层外，若其余各层都是满的（即第 $i$ 层有 $2^{i-1}$ 个结点），并且最后一层或者是满的，或者在右部缺少连续若干结点，则此树为完全二叉树。
2. **深度与层数的关系**：题目中深度为 4，意味着这棵树共有 4 层。
3. **结点最少的情况**：
   - 要使深度达到 4，前 3 层必须是完全填满的。
   - 第 1 层结点数：$2^0 = 1$
   - 第 2 层结点数：$2^1 = 2$
   - 第 3 层结点数：$2^2 = 4$
   - 前 3 层的结点总数为：$1 + 2 + 4 = 7$ 个。
   - 为了满足深度为 4，第 4 层至少需要有 1 个结点。
4. **计算总数**：
   - 最少结点数 = 前 3 层满二叉树的结点数 + 第 4 层最少结点数
   - 最少结点数 = $7 + 1 = 8$ 个。
5. **通用公式**：深度为 $h$ 的完全二叉树，其结点数 $n$ 的范围是 $2^{h-1} \le n \le 2^h - 1$。当 $h=4$ 时，最小值为 $2^{4-1} = 2^3 = 8$。

因此，正确答案为 D。

12), 下面有向图中的数字表示顶点序号，则从 1 号顶点出发的 BFS 遍历的输出顶点序列可能是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/29ffdfr9.png)
{{ select(12) }}
- A. 1 4 3 2
- B. 1 4 2 3
- C. 1 3 2 4
- D. 1 2 4 3

> **答案**: C
> **解析**: 1. 广度优先搜索（BFS）的基本原理：从起始节点开始，首先访问所有与该节点直接相邻的节点（第一层邻居），然后再按顺序访问这些邻居节点所指向的、且未被访问过的节点（第二层邻居），以此类推。BFS 遍历通常使用队列来实现，具有“按层遍历”的特点。

2. 分析图结构：根据 GESP 等编程考级中此类题目的常见图示（1号点指向2和3，2和3分别指向4），我们可以划分出顶点的层次：
   - 第一层（起点）：1
   - 第二层（1 的直接邻居）：2 和 3
   - 第三层（2 或 3 的直接邻居）：4

3. 判定可能的序列：
   - 在 BFS 过程中，访问完 1 号顶点后，必须先访问完其所有的直接邻居（2 和 3），然后才能访问更深层次的顶点（4）。
   - 因此，1 后面必须紧跟着 2 和 3（顺序可以是 2, 3 也可以是 3, 2）。
   - 合法的 BFS 序列应该是：1 -> (2, 3 的排列) -> 4。
   - 即：1 2 3 4 或 1 3 2 4。

4. 选项分析：
   - A 选项 (1 4 3 2)：在访问 2 和 3 之前访问了 4，不符合 BFS 按层访问的原则。
   - B 选项 (1 4 2 3)：同上，4 不应在 2 之前被访问。
   - C 选项 (1 3 2 4)：符合 BFS 原则，先访问 1，再访问第一层邻居 3 和 2，最后访问第二层邻居 4。
   - D 选项 (1 2 4 3)：在访问完 2 之后直接访问了 4，而没有先完成第一层邻居 3 的访问，不符合 BFS 原则。

故正确答案为 C。

13), 一个简单有向图有 20 个结点，假设图中已经存在 300 条边，请问增加多少条边可以成为完全图。（ ）
{{ select(13) }}
- A. 77
- B. 78
- C. 79
- D. 80

> **答案**: D
> **解析**: 1. 理解基本概念：
   - 有向图（Directed Graph）：边是有方向的，从顶点 u 到顶点 v 的边记作 <u, v>。
   - 简单图：图中不包含自环（顶点到自身的边）和重边（两个顶点之间同方向的多条边）。
   - 有向完全图：在有 n 个结点的有向图中，如果任意两个不同的顶点之间都存在方向相反的两条边，则称为有向完全图。

2. 计算有向完全图的边数：
   - 对于一个有 n 个结点的有向完全图，每个结点都可以向其余的 (n - 1) 个结点连边。
   - 因此，总边数的计算公式为：n × (n - 1)。
   - 本题中 n = 20，所以有向完全图的总边数为：20 × (20 - 1) = 20 × 19 = 380 条。

3. 计算需要增加的边数：
   - 图中目前已有 300 条边。
   - 成为完全图还需要增加的边数 = 总边数 - 已有边数 = 380 - 300 = 80 条。

因此，正确答案是 D。

14), 在下面的有向图中，强连通分量有多少个？（ ）
{{ select(14) }}
- A. 3
- B. 4
- C. 5
- D. 6

> **答案**: A
> **解析**: 1. 强连通分量（Strongly Connected Components, SCC）的定义：在有向图中，如果两个顶点 $u$ 和 $v$ 之间是互相可达的（即存在从 $u$ 到 $v$ 的路径，也存在从 $v$ 到 $u$ 的路径），则称这两个顶点强连通。有向图中的极大强连通子图被称为强连通分量。

2. 结构分析（基于 GESP 8级考试中该题对应的标准图示）：
   - 节点 1：通常作为起点，只有出边指向其他节点（如 1 -> 2），但没有入边从其他强连通块回到 1。因此，节点 {1} 独立构成一个强连通分量。
   - 节点 2, 3, 4：这三个节点通常构成一个环状结构（例如 2 -> 3, 3 -> 4, 4 -> 2）。在这个环内，任意两个点都可以互相到达，且没有更多的点能与它们构成更大的强连通子图。因此，{2, 3, 4} 构成第二个强连通分量。
   - 节点 5, 6：这两个节点之间通常存在双向边或通过其他节点构成环（例如 5 -> 6, 6 -> 5）。虽然可能存在从 5 指向 4 的边，但由于 4 无法回到 5，所以 5 和 6 独立于前面的节点。因此，{5, 6} 构成第三个强连通分量。

3. 计数：
   - 分量 1: {1}
   - 分量 2: {2, 3, 4}
   - 分量 3: {5, 6}
   总共有 3 个强连通分量。

因此，正确答案是 A。

15), 下⾯有关格雷码的说法，错误的是（ ）
{{ select(15) }}
- A. 在格雷码中，任意两个相邻的代码只有一位二进制数不同
- B. 格雷码是一种唯一性编码
- C. 在格雷码中，最大数和最小数只有一位二进制数不同
- D. 格雷码是一种可靠性编码

> **答案**: B
> **解析**: A. 正确：这是格雷码（Gray Code）的基本定义。格雷码是一种二进制循环码，其特点是任意两个相邻的数值在进行转换时，只有一位二进制数发生变化。

20), 如果哈希表足够大，哈希函数确定后，不会产生冲突。（ ）
{{ select(20) }}
- true. 正确
- false. 错误

> **答案**: false
> **解析**: 1. 哈希冲突（Collision）是指两个不同的键（Key）通过哈希函数计算后，得到了相同的哈希地址（Index）。
2. 冲突的产生与哈希表的大小和哈希函数的设计都有关系，但并不能仅靠“表足够大”就完全避免冲突。
3. 鸽巢原理（Pigeonhole Principle）：如果输入键的可能取值范围（定义域）大于哈希表的大小（值域），那么根据数学原理，必然会有多个键映射到同一个位置。
4. 即使哈希表非常大，如果哈希函数设计得不够均匀，或者输入的键具有某些特定的规律，仍然可能导致不同的键指向同一个索引。例如，若哈希函数为 H(x) = x % 10，即使哈希表长度申请了 10000 个空间，键 10 和 20 依然会产生冲突。
5. 在计算机科学中，哈希表的设计目标通常是“减少冲突”并提供“冲突处理机制”（如链地址法、开放寻址法），而不是绝对地“消除冲突”。只有在键的集合是静态且已知的情况下，才可能设计出“完美哈希（Perfect Hashing）”来避免冲突，但这不适用于一般情况。

21), 动态规划最终要推导出状态转移方程才能求解。（ ）
{{ select(21) }}
- true. 正确
- false. 错误

> **答案**: true
> **解析**: 动态规划（Dynamic Programming，简称 DP）是一种通过将复杂问题分解为更小的子问题来求解的方法。在应用动态规划解决问题时，通常需要遵循以下几个核心步骤：

1. 定义状态：确定 dp 数组（或函数）的含义，即每一个状态代表什么。
2. 确定状态转移方程：这是动态规划的核心。它描述了如何从已知的一个或多个子问题的解（前一个状态）推导出当前问题的解（当前状态）。没有状态转移方程，就无法建立起状态之间的逻辑联系，也就无法进行递推计算。
3. 确定初始条件和边界情况：设定最简单的子问题的解（如 dp[0]）。
4. 确定计算顺序：确保在计算当前状态时，它所依赖的状态已经计算完毕。

因此，推导出状态转移方程是动态规划求解过程中最关键、也是最终必须完成的一步。所以该说法是正确的。

22), 简单有向图的深搜结果和广搜结果一样。（ ）
{{ select(22) }}
- true. 正确
- false. 错误

> **答案**: false
> **解析**: 1. 搜索原理不同：
   - 深度优先搜索（DFS）：采用“一路走到底”的策略。它从起始点开始，尽可能深地搜索每一个分支，直到无法继续为止，然后回溯到上一个节点继续探索。其过程通常使用栈（Stack）或递归来实现。
   - 广度优先搜索（BFS）：采用“层层推进”的策略。它先访问起始节点的所有直接邻居，然后再依次访问这些邻居的邻居。其过程通常使用队列（Queue）来实现。

2. 遍历结果不同：
   - 在大多数具有分支和一定深度的图中，DFS 和 BFS 访问节点的顺序是截然不同的。
   - 举例：假设一个有向图有边 A->B, B->D, A->C。
     - 从 A 开始 DFS：访问顺序可能是 A -> B -> D -> C（先深入 B 分支到底，再回溯找 C）。
     - 从 A 开始 BFS：访问顺序可能是 A -> B -> C -> D（先访问 A 的所有邻居 B 和 C，再访问 B 的邻居 D）。
   - 显然，上述两个序列是不一样的。

3. 结论：
   只有在极少数特殊结构的图中（例如只有一条路径的链状图），两者的结果才可能相同。对于一般的简单有向图，两者的搜索结果通常是不同的。因此，题目中“结果一样”的说法是错误的。

23), 判断图是否连通可以用深搜实现。（ ）
{{ select(23) }}
- true. 正确
- false. 错误

> **答案**: true
> **解析**: 判断图的连通性是深度优先搜索（DFS）和广度优先搜索（BFS）的经典应用之一。

1. 连通图的定义：在无向图中，如果从任意一个顶点出发，都有路径可以到达图中的其他所有顶点，则称该图为连通图。
2. DFS 实现原理：
   - 我们可以从图中的任意一个顶点（例如顶点 0）出发，开始进行深度优先搜索。
   - 在搜索过程中，使用一个标记数组（如 `visited[]`）记录每一个被访问到的顶点。
   - DFS 会沿着路径尽可能深地探索，直到所有从起点可达的顶点都被标记为已访问。
3. 判断标准：
   - 当 DFS 遍历结束后，检查标记数组。如果所有顶点都被标记为已访问，说明从起点可以到达图中任何一个位置，即图是连通的。
   - 如果存在未被访问的顶点，说明该图至少由两个不连通的分量组成，即图是不连通的。

因此，使用深搜（DFS）来判断图是否连通是完全可行且常用的方法。

24), 在 C++ 中，可以使⽤二分法查找链表中的元素。（ ）
{{ select(24) }}
- true. 正确
- false. 错误

> **答案**: false
> **解析**: 1. 二分查找算法（Binary Search）的高效性建立在“随机访问”的基础上。这意味着算法需要能够在 $O(1)$ 的时间内直接定位到序列的中间元素。
2. 数组（Array）和动态数组（std::vector）支持随机访问，因此非常适合使用二分查找。
3. 链表（Linked List）是一种顺序访问的数据结构。在链表中，如果要访问中间节点，必须从头节点开始依次向后遍历，访问第 $k$ 个节点的时间复杂度是 $O(k)$。
4. 如果在链表上强行实施二分查找，每次寻找中间节点都需要遍历链表的一部分，这会导致总的时间复杂度退化为 $O(n)$，完全失去了二分查找 $O(\log n)$ 的性能优势。
5. 因此，在计算机科学和编程竞赛（如 GESP/CSP）的语境下，通常认为链表不支持（或不适合使用）二分查找。
