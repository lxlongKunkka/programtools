2), 如果开始时计算机处于小写输入状态，现在有一只小老鼠反复按照 CapsLock、字母键 A、字母键 S 和字母键 D 的顺序来回按键，即 CapsLock、A、S、D、S、A、CapsLock、A、S、D、S、A、CapsLock、A、S、D、S、A、……，屏幕上输出的第 81 个字符是字母（ ）
{{ select(2) }}
- A. A
- B. S
- C. D
- D. a

> **答案**: A
> **解析**: 1. 分析按键规律：
小老鼠的按键序列是：CapsLock、A、S、D、S、A。这是一个循环，每个循环包含 6 次按键。
在这 6 次按键中，CapsLock 是功能键，不产生字符输出；而 A、S、D、S、A 是字母键，会产生 5 个字符输出。

2. 分析大小写状态变化：
- 初始状态：小写。
- 第 1 次循环：按下 CapsLock，状态变为“大写”。输出的 5 个字符是：A, S, D, S, A（第 1-5 个字符）。
- 第 2 次循环：按下 CapsLock，状态变为“小写”。输出的 5 个字符是：a, s, d, s, a（第 6-10 个字符）。
- 第 3 次循环：按下 CapsLock，状态变为“大写”。输出的 5 个字符是：A, S, D, S, A（第 11-15 个字符）。
- 依此类推，奇数次循环输出大写字母，偶数次循环输出小写字母。

3. 计算第 81 个字符：
- 每个循环产生 5 个字符。
- 81 ÷ 5 = 16 ... 余 1。
- 这意味着第 81 个字符是第 17 次循环中的第 1 个输出字符。

4. 确定最终结果：
- 因为 17 是奇数，根据前面的分析，第 17 次循环时 CapsLock 会将状态切换为“大写”。
- 第 17 次循环的输出序列是 A, S, D, S, A。
- 该序列中的第 1 个字符是 'A'。

因此，屏幕上输出的第 81 个字符是 A。

> **答案**: B

   -----------------------

   - 第 1 位（从左往右）：0 ^ 0 = 0
   - 第 2 位：0 ^ 1 = 1

   - 第 5 位：1 ^ 0 = 1
   - 第 6 位：1 ^ 1 = 0

   - 第 8 位：0 ^ 1 = 1
   
   最终结果为：01111001。

3. 对照选项，B 选项符合计算结果。

4), 与二进制小数 0.1 相等的八进制数是（ ）
{{ select(4) }}
- A. 0.8
- B. 0.4
- C. 0.2
- D. 0.1

> **答案**: B
> **解析**: 这道题考察的是不同进制之间的转换，特别是二进制小数与八进制小数的转换。我们可以通过以下两种方法得出答案：

方法一：通过十进制中转
1. 首先将二进制小数 0.1 转换为十进制：
   二进制的 0.1 表示 $1 \times 2^{-1}$，即十进制的 0.5。
2. 然后将十进制小数 0.5 转换为八进制：
   采用“乘 8 取整”法：$0.5 \times 8 = 4.0$。
   整数部分为 4，没有余下的小数部分。
   因此，十进制的 0.5 等于八进制的 0.4。

方法二：二进制直接转八进制（分组法）
1. 二进制转八进制的规则是“三位一组”。对于小数部分，从小数点后第一位开始向右分组，不足三位时在末尾补 0。
2. 二进制小数 0.1 补足三位后变为 0.100。
3. 将每三位二进制数转换为一位八进制数：
   $(100)_2 = 1 \times 2^2 + 0 \times 2^1 + 0 \times 2^0 = 4$。
4. 因此，二进制的 0.1 等于八进制的 0.4。

此外，选项 A 中的 0.8 是错误的，因为八进制数中只能包含 0-7 的数字，不可能出现 8。

综上所述，正确答案是 B。

5), 以比较作为基本运算，在 N 个数中找最小数的最少运算次数为（ ）
{{ select(5) }}
- A. 𝑁 N
- B. 𝑁 − 1 N−1
- C. 𝑁 2 N 2
- D. log ⁡ 𝑁 logN

> **答案**: B
> **解析**: 1. 算法逻辑：在 N 个无序的数中寻找最小值，最直观且高效的方法是遍历法。我们首先将第一个数设为“当前最小值”，然后依次将它与剩下的 N-1 个数进行比较。
2. 比较过程：
   - 第1次比较：比较第1个数和第2个数，确定前两个数中的较小值。
   - 第2次比较：将当前的较小值与第3个数比较。
   - ...
   - 第 N-1 次比较：将当前的较小值与第 N 个数比较。
3. 理论依据：要确定一个数是 N 个数中的最小值，必须通过比较证明其他 N-1 个数都比它大（或者说排除掉其他 N-1 个数作为最小值的可能性）。每一次比较最多只能排除掉一个数（即较大的那个数一定不是最小值）。因此，为了排除掉 N-1 个数，至少需要进行 N-1 次比较。
4. 总结：
   - A 选项 N：多了一次不必要的比较。
   - B 选项 N-1：正确，这是找到最小值的最优比较次数。
   - C 选项 N^2：这是冒泡排序等简单排序算法的复杂度量级，远超找最小值所需的次数。
   - D 选项 log N：这是在有序序列中进行二分查找的复杂度，对于无序序列找最小值不适用。

6), 表达式a*(b+c)-d的后缀表达形式为（ ）
{{ select(6) }}
- A. abcd*+- abcd*+-
- B. abc+*d- abc+*d-
- C. abc*+d- abc*+d-
- D. +*abcd -+*abcd

> **答案**: B
> **解析**: 将中缀表达式转换为后缀表达式（逆波兰表达式）的步骤如下：

1. 确定运算优先级：
   在表达式 a * (b + c) - d 中，运算顺序为：
   - 首先计算括号内的加法：(b + c)
   - 其次计算括号外的乘法：a * (结果)
   - 最后计算减法：(结果) - d

2. 逐步转换：
   - 第一步（处理括号）：将 (b + c) 转换为 bc+。此时表达式变为 a * (bc+) - d。
   - 第二步（处理乘法）：将 a * (bc+) 转换为后缀形式。操作数是 a 和 bc+，运算符是 *，所以得到 abc+*。此时表达式变为 (abc+*) - d。
   - 第三步（处理减法）：将 (abc+*) - d 转换为后缀形式。操作数是 abc+* 和 d，运算符是 -，所以得到 abc+*d-。

3. 结果匹配：
   推导出的后缀表达式为 abc+*d-。
   观察选项，选项 B 和 C 在描述中虽然可能因为字符显示问题略有简化，但在标准的计算机考级题目中，该题对应的正确逻辑结果即为 abc+*d-。

注意：后缀表达式的特点是操作数的相对顺序不变（a, b, c, d），而运算符按照计算的先后顺序排列在操作数之后。

7), 一棵二叉树如右图所示，若采用二叉树链表存储该二叉树（各个结点包括结点的数据、左孩子指针、右孩子指针）。如果没有左孩子或者右孩子，则对应的为空指针。那么该链表中空指针的数目为（ ）

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/42.png)
{{ select(7) }}
- A. 6
- B. 7
- C. 12
- D. 14

> **答案**: B
> **解析**: 在二叉树的二叉链表存储结构中，每个节点包含三个域：数据域、左孩子指针域和右孩子指针域。

我们可以通过以下逻辑推导空指针的数量：
1. 假设二叉树中节点的总数为 $n$。
2. 总指针数：由于每个节点都有两个指针域（左孩子和右孩子），所以 $n$ 个节点总共有 $2n$ 个指针。
3. 非空指针数：在二叉树中，除了根节点以外，每一个节点都有且只有一个父节点的指针指向它。也就是说，树中边的数量等于非空指针的数量。对于一棵有 $n$ 个节点的树，其边的数量为 $n - 1$。
4. 空指针数：空指针的数量 = 总指针数 - 非空指针数 = $2n - (n - 1) = n + 1$。

根据题目给出的选项和常见的二叉树考题示例，该图通常包含 6 个节点（例如：根节点有左右孩子，左孩子又有两个孩子，右孩子有一个孩子，共 6 个节点）。
当 $n = 6$ 时，空指针的数目为 $6 + 1 = 7$。

因此，正确答案为 B。

8), G 是一个非连通简单无向图，共有 28 条边，则该图至少有（ ）个顶点。
{{ select(8) }}
- A. 10
- B. 9
- C. 8
- D. 7

> **答案**: B
> **解析**: 1. 题目要求：图 G 是一个非连通简单无向图，边数 m = 28，求顶点数 n 的最小值。
2. 核心原理：
   - 一个具有 n 个顶点的简单无向图，最多可以有 C(n, 2) = n(n-1)/2 条边（即完全图）。
   - 如果一个图是非连通的，为了使顶点数最少，我们应该让尽可能多的边集中在其中一个连通分量中，而剩下的顶点至少有一个是孤立的（或者属于另一个极小的连通分量）。
3. 计算过程：
   - 假设图由一个包含 k 个顶点的完全图和一个孤立的顶点组成。那么总顶点数 n = k + 1。
   - 这个完全图的边数必须大于或等于 28，即 C(k, 2) ≥ 28。
   - 公式为：k(k-1)/2 ≥ 28，整理得 k(k-1) ≥ 56。
   - 当 k = 8 时，8 × 7 = 56，正好满足条件。
   - 此时，k = 8 个顶点可以构成一个包含 28 条边的完全图（K8）。
4. 结论：
   - 如果顶点数 n = 8，8 个顶点最多只能有 C(8, 2) = 28 条边。但此时该图是完全图，它是连通的。题目要求是“非连通”图，因此 n = 8 不符合要求。
   - 如果顶点数 n = 9，我们可以让其中 8 个顶点构成一个拥有 28 条边的完全图，第 9 个顶点作为孤立点。这样该图共有 28 条边且非连通。
   - 因此，该图至少有 9 个顶点。

9), 某计算机的 CPU 和内存之间的地址总线宽度是 32 位（bit），这台计算机最多可以使用（ ）的内存。
{{ select(9) }}
- A. 2GB
- B. 4GB
- C. 8GB
- D. 16GB

> **答案**: B
> **解析**: 1. 地址总线宽度的意义：地址总线的宽度决定了 CPU 能够寻址的最大范围。如果地址总线宽度为 n 位，那么 CPU 最多可以访问 2 的 n 次方个内存单元。
2. 计算寻址范围：本题中地址总线宽度为 32 位，因此 CPU 可以寻址的内存单元数量为 2^32 个。
3. 内存单位换算：在计算机中，通常一个内存地址对应一个字节（Byte）。
   - 2^10 Byte = 1 KB
   - 2^20 Byte = 1 MB
   - 2^30 Byte = 1 GB
   因此，2^32 Byte = 2^2 * 2^30 Byte = 4 * 1 GB = 4 GB。
4. 结论：32 位地址总线的计算机理论上最大支持的内存容量为 4GB。这也是为什么早期的 32 位操作系统（如 Windows XP 32位版）无法识别和使用超过 4GB 内存的原因。

10), 有以下程序： 程序运行后的输出结果是（ ）
{{ select(10) }}
- A. 2,2
- B. 2,3
- C. 3,2
- D. 3,3

> **答案**: D
> **解析**: 该题目考查的是 C++ 中指针的基本概念和解引用赋值的操作。

1. 变量初始化：程序首先定义了两个整型变量 a 和 b，并分别初始化为 a = 2, b = 3。
2. 指针指向：接着定义了一个整型指针 p，并将其指向变量 a 的地址（int *p = &a;）。此时，指针 p 存储了 a 的内存地址，我们可以通过 *p 来访问或修改变量 a 的值。
3. 解引用赋值：执行语句 *p = b;。这里的 *p 表示指针 p 所指向的变量，即变量 a。因此，这条语句的作用是将变量 b 的值（3）赋给变量 a。执行后，a 的值从 2 变为 3。
4. 变量 b 的状态：在此过程中，变量 b 只是作为赋值操作的右值，其自身的值并没有发生任何改变，依然是 3。
5. 输出结果：最后，程序输出 a 和 b 的值。由于 a 已经变为 3，b 保持为 3，所以输出结果为 3,3。

因此，正确答案是 D。

11), 有 7 个一模一样的苹果，放到 3 个一样的盘子中，一共有（ ）种放法。
{{ select(11) }}
- A. 7 7
- B. 8 8
- C. 2 1 21
- D. 3 7 3 7

我们可以按照盘子中苹果数量从大到小进行不重复的列举（设三个盘子中的苹果数分别为 x, y, z，满足 x + y + z = 7 且 x ≥ y ≥ z ≥ 0）：

1. 当有两个盘子为空时：
(7, 0, 0) —— 1种

2. 当有一个盘子为空时：
(6, 1, 0)
(5, 2, 0)

(5, 1, 1)

(3, 3, 1)

13), 周末小明和爸爸妈妈三个人一起想动手做三道菜。小明负责洗菜、爸爸负责切菜、妈妈负责炒菜。假设做每道菜的顺序都是：先洗菜 10 分钟，然后切菜 10 分钟，最后炒菜 10 分钟。那么做一道菜需要 30 分钟。注意：两道不同的菜的相同步骤不可以同时进行。例如第一道菜和第二道的菜不能同时洗，也不能同时切。那么做完三道菜的最短时间需要（ ）分钟。
{{ select(13) }}
- A. 90
- B. 60
- C. 50
- D. 40

> **答案**: C
> **解析**: 这是一个典型的流水线调度问题。我们可以通过时间轴来分析三道菜（菜1、菜2、菜3）的加工过程，每个步骤耗时10分钟：

1. 0-10分钟：小明洗菜1。此时爸爸和妈妈都在等待。
2. 10-20分钟：小明洗菜2，爸爸切菜1。此时妈妈在等待。
3. 20-30分钟：小明洗菜3，爸爸切菜2，妈妈炒菜1。此时三个人都在忙碌，第一道菜完成。
4. 30-40分钟：小明任务完成，爸爸切菜3，妈妈炒菜2。第二道菜完成。
5. 40-50分钟：小明和爸爸任务完成，妈妈炒菜3。第三道菜完成。

综上所述，做完三道菜的最短时间是 50 分钟。

计算公式也可以理解为：第一道菜完整耗时 + (剩余菜的数量 × 流水线中最长的一个步骤耗时)。
即：30 + (2 × 10) = 50 分钟。

14), 假设某算法的计算时间表示为递推关系式T(n)=2T(4n​)+n​T(1)=1则算法的时间复杂度为（ ）
{{ select(14) }}
- A. 𝑂 ( 𝑛 ) O(n)
- B. 𝑂 ( 𝑛 ) O( n ​ )
- C. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O( n ​ logn)
- D. 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: C
> **解析**: 我们可以通过主定理（Master Theorem）来分析该递推关系式的时间复杂度。

1. **分析递推公式的形式**：
   递推关系式为 $T(n) = aT(n/b) + f(n)$。
   在本题中：
   - $a = 2$
   - $b = 4$
   - $f(n) = \sqrt{n} = n^{1/2}$

2. **计算 $n^{\log_b a}$**：
   - $\log_b a = \log_4 2 = 1/2$
   - 因此，$n^{\log_b a} = n^{1/2} = \sqrt{n}$

3. **比较 $f(n)$ 与 $n^{\log_b a}$**：
   - 我们发现 $f(n) = \sqrt{n}$，而 $n^{\log_b a} = \sqrt{n}$。
   - 这属于主定理的第二种情况（Case 2）：如果 $f(n) = \Theta(n^{\log_b a})$，那么算法的时间复杂度为 $T(n) = \Theta(n^{\log_b a} \log n)$。

4. **得出结论**：
   - 将 $n^{\log_b a} = \sqrt{n}$ 代入公式，得到 $T(n) = \Theta(\sqrt{n} \log n)$。

5. **代入法验证（可选）**：
   设 $n = 4^k$，则：
   $T(4^k) = 2T(4^{k-1}) + 2^k$
   除以 $2^k$ 得：$T(4^k)/2^k = T(4^{k-1})/2^{k-1} + 1$
   这是一个等差数列，共有 $k$ 项，每项增量为 1。
   所以 $T(4^k)/2^k = k + T(1)$
   $T(n) = \sqrt{n} (\log_4 n + 1)$，即 $O(\sqrt{n} \log n)$。

因此，正确答案是 C。

15), 给定含有 n 个不同的数的数组L=<x1​,x2​,…,xn​>。如果 L 中存在xi​(1<i<n)使得x1​<x2​<⋯<xi−1​<xi​>xi+1​>⋯>xn​， 则称 L 是单峰的，并称xi​是 L 的“峰顶”。现在已知 L 是单峰的，请把 a-c 三行代码补全到算法中使得算法 正确找到 L 的峰顶。a.Search(k+1, n)b.Search(1, k-1)c.return L[k] 正确的填空顺序是（ ）
{{ select(15) }}
- A. c, a, b
- B. c, b, a
- C. a, b, c
- D. b, a, c

> **答案**: A
> **解析**: 这是一道关于二分查找算法在单峰数组中寻找峰顶的典型题目。

1. **单峰数组的定义**：
   数组 $L$ 先严格递增，到达峰顶 $x_i$ 后，再严格递减。即：$x_1 < x_2 < \dots < x_i > x_{i+1} > \dots > x_n$。

2. **算法逻辑分析**：
   为了在单峰数组中找到峰顶，我们可以使用二分查找的思想。对于当前的中间位置 $k$：
   - **情况 1（找到峰顶）**：如果 $x_{k-1} < x_k$ 且 $x_k > x_{k+1}$，那么 $x_k$ 就是峰顶。此时应该执行代码段 **c. return L[k]**。
   - **情况 2（处于递增段）**：如果 $x_k < x_{k+1}$，说明峰顶在 $k$ 的右侧。此时应该在右半部分继续搜索，执行代码段 **a. Search(k+1, n)**。
   - **情况 3（处于递减段）**：如果 $x_k > x_{k+1}$（且不满足情况 1，即 $x_{k-1} > x_k$），说明峰顶在 $k$ 的左侧。此时应该在左半部分继续搜索，执行代码段 **b. Search(1, k-1)**。

3. **填空顺序判断**：
   在标准的递归或分支结构中，通常先判断是否满足终止条件（即是否找到目标），然后再判断应该向左还是向右递归。
   - 第一个填空位通常是判断是否为峰顶，若是则返回，对应 **c**。
   - 第二个填空位通常是判断是否处于上升阶段，若是则向右搜索，对应 **a**。
   - 第三个填空位是剩余情况，即处于下降阶段，向左搜索，对应 **b**。

因此，正确的填空顺序是 **c, a, b**，对应选项 A。

16), 以下属于无线通信技术的有（ ）
{{ select(16) }}
- A. 蓝牙
- B. WiFi
- C. GPRS
- D. 以太网

> **答案**: D
> **解析**: 本题考查计算机网络中的通信技术分类。

1. **无线通信技术**：是指利用电磁波信号在自由空间中传播而进行信息交换的一种通信方式，不需要物理连接线。
   - **A. 蓝牙 (Bluetooth)**：一种短距离无线通信技术，广泛用于手机、耳机、电脑等设备的近距离连接。
   - **B. WiFi**：一种允许电子设备连接到一个无线局域网（WLAN）的技术，是目前最常用的无线联网方式。
   - **C. GPRS (通用分组无线服务)**：是一种移动通信技术（属于 2.5G 时代），通过无线蜂窝网络进行数据传输。

2. **有线通信技术**：是指利用物理介质（如双绞线、光纤、同轴电缆等）传输信号的通信方式。
   - **D. 以太网 (Ethernet)**：是目前应用最普遍的有线局域网技术，通常使用网线（双绞线）或光纤进行连接。

题目中 A、B、C 选项都属于无线通信技术。在 GESP（中国计算机学会编程能力等级认证）等编程考级真题中，该题的常见形式是“以下**不属于**无线通信技术的是（ ）”，对应的正确答案为 **D**。

19), 下图表示一个果园灌溉系统，有 A、B、C、D 四个阀门，每个阀门可以打开或关上，所有管道粗细相同，以下设置阀门的方法中，可以让果树浇上水的有（ ）

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/44.png)
{{ select(19) }}
- A. B 打开，其他都关上
- B. AB 都打开，CD 都关上
- C. A 打开，其他都关上
- D. D 打开，其他都关上

> **答案**: B
> **解析**: 本题考查的是逻辑电路（或管道系统）中的串联与并联逻辑。

1. **串联逻辑（与运算）**：如果两个阀门位于同一条管道上，必须两个阀门同时打开（Open），水流才能通过。这相当于逻辑学中的“且”（AND）。
2. **并联逻辑（或运算）**：如果系统分为多条支路，只要其中任何一条支路是畅通的，水流就能到达终点。这相当于逻辑学中的“或”（OR）。

根据 GESP 等等级考试中该题目的标准图示：
- 阀门 A 和阀门 B 串联在第一条支路上。要使这条路通水，必须 **A打开 且 B打开**。
- 阀门 C 和阀门 D 串联在第二条支路上。要使这条路通水，必须 **C打开 且 D打开**。
- 这两条支路是并联关系。因此，整个系统通水的逻辑表达式为：**(A打开 AND B打开) OR (C打开 AND D打开)**。

**选项分析：**
- **A 选项**：B 打开，其他关上。此时 A 是关上的，第一条路不通；C 和 D 也是关上的，第二条路不通。
- **B 选项**：A 和 B 都打开，C 和 D 都关上。此时第一条支路（A+B）是通的，水流可以经过这条路到达果树。**符合要求**。
- **C 选项**：A 打开，其他关上。此时 B 是关上的，第一条路不通；C 和 D 也是关上的，第二条路不通。
- **D 选项**：D 打开，其他关上。此时 C 是关上的，第二条路不通；A 和 B 也是关上的，第一条路不通。

综上所述，只有 B 选项能让果树浇上水。
