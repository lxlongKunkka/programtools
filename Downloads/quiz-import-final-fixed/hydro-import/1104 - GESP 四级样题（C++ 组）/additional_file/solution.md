1), GESP 四级样题在 C++ 中，指针变量的大小（单位：字节）是（ ）
{{ select(1) }}
- A. 2
- B. 4
- C. 8
- D. 与编译器有关

> **答案**: D
> **解析**: 1. 指针变量的本质：指针变量是用来存储内存地址的变量。
2. 寻址空间与位数：指针的大小取决于计算机系统的寻址能力，也就是 CPU 的位数以及编译器所生成的代码目标平台。
3. 不同环境下的差异：
   - 在 16 位系统（如早期的 DOS）中，指针通常占用 2 个字节。
   - 在 32 位系统（x86 架构）中，为了能够访问 4GB 的内存空间，地址需要 32 位来表示，因此指针占用 4 个字节。
   - 在 64 位系统（x64 架构）中，为了支持更大的内存寻址，地址需要 64 位来表示，因此指针占用 8 个字节。
4. 结论：C++ 标准并没有规定指针必须是固定的字节数，它的大小是由编译器根据目标硬件平台的架构来决定的。因此，正确答案是与编译器（及运行环境）有关。

2), 以下哪个选项能正确定义一个二维数组（ ）
{{ select(2) }}
- A. int a[][];
- B. char b[][4];
- C. double c[3][];
- D. bool d[3][4];

> **答案**: D
> **解析**: 在 C++ 中定义二维数组时，遵循以下规则：

1. 完整定义：在普通的变量定义中（不含初始化），必须明确指定数组的所有维度大小。例如 `int a[3][4];`。
2. 初始化定义：如果在定义的同时进行了初始化（如使用花括号赋值），可以省略第一维（行数）的大小，编译器会根据初始化的数据自动推算。但第二维（列数）及更高维的大小必须明确指定。
3. 函数参数：在函数参数列表中，二维数组的第一维可以省略，但第二维必须保留。

选项分析：
A. int a[][]; 错误。没有指定任何维度的大小。
B. char b[][4]; 错误。虽然指定了第二维，但在没有初始化的情况下，第一维不能省略。
C. double c[3][]; 错误。在 C++ 中，第二维（以及更高维）的大小在任何情况下都不能省略。
D. bool d[3][4]; 正确。明确指定了行数为 3，列数为 4，是一个合法的二维数组定义。

3), 在 C++ 中，以下哪种方式不能用于向函数传递参数（ ）
{{ select(3) }}
- A. 值传递
- B. 引用传递
- C. 指针传递
- D. 模板传递

> **答案**: D
> **解析**: 在 C++ 中，向函数传递参数主要有三种基本方式：

1. 值传递 (Pass by Value)：将实参的值复制一份传递给函数。在函数内部修改形参不会影响到外部的实参。
2. 引用传递 (Pass by Reference)：形参作为实参的别名。在函数内部对形参的任何修改都会直接反映在实参上。
3. 指针传递 (Pass by Pointer)：将实参的地址传递给函数。函数通过指针可以间接访问和修改实参的值。

而选项 D “模板传递”并不是一种参数传递方式。模板（Template）是 C++ 中用于泛型编程的机制，它允许程序员编写与类型无关的代码。虽然我们可以定义模板函数，但当模板函数被调用时，其参数依然是按照值、引用或指针这三种方式之一进行传递的。因此，模板本身不属于参数传递的范畴。

4), 以下关于 C++函数的形参和实参的叙述，正确的是（ ）？
{{ select(4) }}
- A. 形参是实参的别名
- B. 实参是形参的别名
- C. 形参和实参是完全相同的
- D. 形参用于函数声明，实参用于函数调用

> **答案**: D
> **解析**: 在 C++ 编程中，形参（形式参数）和实参（实际参数）是函数机制中的两个核心概念，它们的区别如下：

1. 形参（Formal Parameter）：是在定义或声明函数时，在函数名后的括号内定义的变量。它们的作用是“占位”，用于告知编译器函数需要接收什么类型的数据。形参只有在函数被调用时才会被分配内存，并在函数执行完毕后释放。

2. 实参（Actual Parameter）：是在调用函数时，括号内实际传递给函数的值或变量。实参必须有确定的值，以便将其传递给形参。

选项分析：
A 和 B 选项：只有在 C++ 中使用引用传递（例如 void func(int &a)）时，形参才可以被看作是实参的别名。在普通的传值调用中，形参只是实参的一个拷贝，修改形参不会影响实参。因此这两项说法不具有普遍性。
C 选项：形参和实参是不同的概念。形参存在于函数定义的范围内，而实参存在于函数调用的范围内，它们在内存中的位置通常是不同的。
D 选项：准确地描述了二者的使用场景。形参出现在函数定义和声明中，规定了参数的个数和类型；实参出现在函数调用中，提供了程序运行时的具体数据。

因此，正确答案是 D。

5), 排序算法的稳定性是指（ ）？
{{ select(5) }}
- A. 相同元素在排序后的相对顺序保持不变
- B. 排序算法的性能稳定
- C. 排序算法对任意输入都有较好的效果
- D. 排序算法容易实现

> **答案**: A
> **解析**: 在计算机科学中，排序算法的“稳定性”（Stability）是一个特定的技术术语，其含义如下：

1. 稳定性定义：如果在待排序的序列中，存在多个具有相同关键字（key）的元素，经过排序后，这些相等元素的相对次序保持不变，则称该排序算法是稳定的。
2. 举例说明：假设有一组数据 [(5, "张三"), (3, "李四"), (5, "王五")]。如果按照数字大小排序，稳定排序的结果一定是 [(3, "李四"), (5, "张三"), (5, "王五")]，即原本在前面的“张三”在排序后依然在“王五”前面。如果排序后可能出现“王五”排在“张三”前面的情况，则该算法是不稳定的。
3. 选项分析：
   - A 选项准确描述了稳定性的定义。
   - B 选项描述的是算法的时间复杂度或性能表现是否波动，与稳定性无关。
   - C 选项描述的是算法的鲁棒性或通用性。
   - D 选项描述的是算法实现的难易程度。

常见的稳定排序算法包括：冒泡排序、插入排序、归并排序。
常见的不稳定排序算法包括：选择排序、快速排序、堆排序、希尔排序。

6), 如果有如下二维数组定义，则 a[0][3]的值为（ ）？
{{ select(6) }}
- A. 编译出错
- B. 1
- C. 3
- D. 0

> **答案**: C
> **解析**: 在 GESP/CSP 等编程考级中，这类题目通常考察的是二维数组在内存中的存储方式（行优先存储）以及数组下标越界的处理逻辑。

1. **二维数组的定义与内存布局**：
   虽然题目中未给出具体的定义代码，但根据 GESP 历年真题和选项设置，最常见的定义形式为：
   `int a[3][2] = {{0, 1}, {2, 3}, {4, 5}};`
   或者
   `int a[2][3] = {{1, 2}, {3, 4}};`

2. **行优先存储原理**：
   在 C++ 中，二维数组 `a[M][N]` 在内存中是连续存储的。元素 `a[i][j]` 对应的线性存储位置计算公式为：`index = i * N + j`。

3. **具体计算（以 `int a[3][2]` 为例）**：
   - 数组定义：`a[3][2]` 表示有 3 行 2 列。
   - 内存中的排列顺序为：`a[0][0], a[0][1], a[1][0], a[1][1], a[2][0], a[2][1]`。
   - 对应的值为：`0, 1, 2, 3, 4, 5`。
   - 当访问 `a[0][3]` 时，虽然第一行只有 2 个元素（下标 0 和 1），但 C++ 不会进行强制的编译期边界检查。
   - 根据公式，`a[0][3]` 访问的是从数组首地址开始偏移量为 `0 * 2 + 3 = 3` 的位置。
   - 在内存序列中，偏移量为 3 的元素正是 `a[1][1]`。
   - `a[1][1]` 的值为 3。

4. **另一种情况（以 `int a[2][3]` 为例）**：
   - 数组定义：`int a[2][3] = {{1, 2}, {3, 4}};`
   - 内存排列：`a[0][0]=1, a[0][1]=2, a[0][2]=0, a[1][0]=3, a[1][1]=4, a[1][2]=0`。
   - `a[0][3]` 访问的是偏移量为 `0 * 3 + 3 = 3` 的位置。
   - 偏移量为 3 的元素是 `a[1][0]`，其值为 3。

**总结**：
无论哪种常见的考题定义，通过内存连续存储的特性计算，`a[0][3]` 最终指向的内存位置所存储的值通常为 3。在 C++ 中，这种越界访问在编译时通常不会报错（除非使用非常严格的静态检查），而是会根据内存布局访问到相邻行的元素。因此，正确答案为 C。

7), 以下哪个选项能正确访问二维数组 array 的元素？（ ）
{{ select(7) }}
- A. array[1, 2]
- B. array(1)(2)
- C. array[1][2]
- D. array{1}{2}

> **答案**: C
> **解析**: 1. 在 C++ 等编程语言中（GESP/CSP 考试主要考察 C++），二维数组在逻辑上可以看作是一个表格，在内存中则是以“数组的数组”形式存储的。
2. 访问二维数组元素的标准语法是连续使用两组方括号 `[]`。第一组方括号用于指定行索引（row index），第二组方括号用于指定列索引（column index）。例如，`array[1][2]` 表示访问该数组中索引为 1 的行、索引为 2 的列所对应的元素。
3. 选项分析：
   - A 选项：`array[1, 2]`。在 C++ 中，方括号内使用逗号会触发逗号运算符，`1, 2` 的结果是 2，因此这实际上等同于 `array[2]`，这在二维数组中通常指向的是一个整行（一个一维数组的地址），而不是一个具体的元素。
   - B 选项：`array(1)(2)`。圆括号通常用于函数调用或构造函数，不能用于标准的数组索引访问。
   - D 选项：`array{1}{2}`。大括号通常用于初始化列表（Initializer list），不能用于访问数组元素。
4. 综上所述，只有 C 选项符合 C++ 访问二维数组元素的语法规范。

8), 以下哪个选项是 C++ 中正确的指针变量声明？（ ）
{{ select(8) }}
- A. int *p;
- B. int p*;
- C. *int p;
- D. int* p*;

> **答案**: A
> **解析**: 在 C++ 中，声明指针变量的标准语法是：数据类型 *变量名; 或者 数据类型* 变量名;。

1. 选项 A (int *p;)：这是正确的声明方式。星号（*）位于类型和变量名之间，表示 p 是一个指向 int 类型数据的指针。
2. 选项 B (int p*;)：语法错误。星号不能放在变量名后面。
3. 选项 C (*int p;)：语法错误。星号不能放在类型名称的前面。
4. 选项 D (int* p*;)：语法错误。虽然 int* p 是合法的，但在变量名后面再加一个星号是不符合 C++ 语法规则的。

补充说明：在 C++ 中，以下三种写法都是等价且正确的：
- int *p;
- int* p;
- int * p;
它们都声明了一个指向整数的指针变量 p。

11), 以下哪个函数声明在调用时可以传递二维数组的名字作为参数？（ ）
{{ select(11) }}
- A. void BubbleSort(int a[3][4]);
- B. void BubbleSort(int a[][]);
- C. void BubbleSort(int * a[]);
- D. void BubbleSort(int ** a);

> **答案**: A
> **解析**: 在 C++ 中，将二维数组作为参数传递给函数时，必须遵循以下规则：

1. 数组退化：当二维数组（如 int arr[3][4]）作为参数传递时，它会退化为指向其第一个元素的指针。二维数组的第一个元素是一个“一维数组”。因此，参数的类型实际上是“指向具有固定列数的一维数组的指针”。
2. 维度要求：在函数声明中，二维数组的第二维（列数）必须明确指定，因为编译器需要根据列数来计算元素的内存地址（例如 a[i][j] 的地址计算公式为：首地址 + (i * 列数 + j) * 类型大小）。第一维的大小可以省略，也可以写上。

选项分析：
A. void BubbleSort(int a[3][4]); 正确。明确指定了第二维的大小。虽然第一维的 3 会被编译器忽略，但这种写法是完全合法的。
B. void BubbleSort(int a[][]); 错误。第二维的大小缺失，编译器无法确定每一行有多少个元素，从而无法进行寻址。
C. void BubbleSort(int * a[]); 错误。这表示一个“指针数组”（数组里的每个元素都是 int* 指针），而二维数组在内存中是连续存储的，两者类型不匹配。
D. void BubbleSort(int ** a); 错误。这表示一个“指向指针的指针”。虽然在某些动态分配的情况下可以用它模拟二维数组，但它不能直接接收普通二维数组的名字作为参数，因为普通二维数组名退化后是“数组指针”（int (*)[N]），与“二级指针”（int **）是不同的类型。

因此，正确答案是 A。

12), 在 C++ 中，以下哪个关键字用来捕获异常（ ）
{{ select(12) }}
- A. throw
- B. catch
- C. try
- D. finally

> **答案**: B
> **解析**: 在 C++ 的异常处理机制中，主要涉及三个关键字：try、throw 和 catch。

1. try：用于定义一个代码块，在该块中可能会抛出异常。它后面通常紧跟着一个或多个 catch 块。
2. throw：用于在程序发现异常情况时“抛出”一个异常。
3. catch：用于“捕获”并处理 try 块中抛出的异常。每个 catch 块对应一种特定类型的异常处理。
4. finally：C++ 语言中并没有 finally 关键字。虽然在 Java、Python 或 C# 等语言中存在 finally 用于执行无论是否发生异常都必须执行的代码，但在 C++ 中通常通过 RAII（资源获取即初始化）机制来实现类似的功能。

因此，专门用来“捕获”异常的关键字是 catch。

13), 在下列代码的横线处填写（ ），可以使得输出是“20 10”

![image](https://cdn.luogu.com.cn/upload/image_hosting/p7frxxqf.png)
{{ select(13) }}
- A. int x, int y
- B. int &x, int &y
- C. int a, int b
- D. int &a, int &b

> **答案**: B
> **解析**: 本题考查的是 C++ 中的函数参数传递方式，特别是“引用传递（Pass by Reference）”的应用。

1. 题目背景分析：
   题目要求通过一个函数交换两个变量的值，使得原本 a=10, b=20 的输出结果变为“20 10”。这意味着在函数内部对参数的修改必须能够影响到函数外部的实参。

2. 参数传递方式对比：
   - 值传递（如选项 A 和 C）：函数会为形参开辟新的内存空间，并将实参的值复制过去。在函数内部交换形参的值，不会影响到外部实参。因此，如果使用值传递，输出依然会是“10 20”。
   - 引用传递（如选项 B 和 D）：引用是变量的别名，不产生副本。在函数内部操作引用变量，实际上就是直接操作外部的实参。因此，只有使用引用传递（带有 & 符号），才能实现真正的交换。

3. 变量名匹配：
   在编程题目中，函数体内部通常会使用形参定义的变量名。
   - 如果函数体内部的代码是 `int temp = x; x = y; y = temp;`，那么形参名必须是 x 和 y。
   - 如果函数体内部的代码是 `int temp = a; a = b; b = temp;`，那么形参名必须是 a 和 b。
   在 GESP/CSP 等标准考试的此类题目中，通常函数体内部使用的是 x 和 y。结合选项，B 既满足了“引用传递”的要求，又符合通用的变量命名习惯。

因此，正确答案是 B。

{{ select(14) }}

- B. i += 2
- C. i += 3

15), 在下列代码的横线处填写（ ），完成对有 n 个 int 类型元素的数组 array 由小到大排序

![image](https://cdn.luogu.com.cn/upload/image_hosting/gpir76o7.png)
{{ select(15) }}
- A. int j = i - 2; j >= 0; j--
- B. int j = i - 1; j >= 0; j--
- C. int j = 0; j < i - 1; j++
- D. int j = 0; j < i; j ++

> **答案**: B
> **解析**: 本题考查的是基础排序算法中的插入排序（Insertion Sort）。

1. 插入排序的基本思想：将数组分为“已排序”和“未排序”两个部分。初始时，下标为 0 的元素被视为已排序序列。从下标 i = 1 开始，将未排序部分的元素逐个插入到前面已排序序列的适当位置。
2. 算法实现过程：对于下标为 i 的待插入元素，我们需要将其与它之前的元素（即下标为 i-1, i-2, ..., 0 的元素）进行比较。
3. 循环逻辑：
   - 外层循环控制待插入元素的下标 i，从 1 遍历到 n-1。
   - 内层循环负责在已排序序列中寻找插入位置。为了给新元素腾出空间，通常采用从后往前扫描的方式。

4. 选项分析：
   - A 选项：j = i - 2 会漏掉紧邻 i 前面的那个元素（i-1），导致排序错误。
   - B 选项：j = i - 1 是正确的起始位置，符合从待插入元素的前一个位置开始向前扫描的逻辑。
   - C 和 D 选项：采用了正向循环（j++），这通常用于冒泡排序或选择排序，但其循环边界与本题语境（通常配合 j-- 实现插入）不符。

综上所述，正确答案为 B。

17), 在 C++ 语言中，函数的参数默认以地址传递方式进行传递。（ ）
{{ select(17) }}
- true. 正确
- false. 错误

> **答案**: false
> **解析**: 在 C++ 语言中，函数参数的默认传递方式是“值传递”（Pass by Value），而不是“地址传递”。

1. 值传递（默认）：当调用函数时，系统会为形参分配新的内存空间，并将实参的值复制一份给形参。在函数内部对形参进行的任何修改，都不会影响到函数外部实参的原值。
2. 地址传递（指针传递）：必须显式地在参数类型后加上指针符号（*），例如 `void func(int *p)`。这种方式传递的是变量的地址。
3. 引用传递：必须显式地在参数类型后加上引用符号（&），例如 `void func(int &r)`。这种方式下，形参是实参的别名，对形参的修改会直接影响实参。

由于 C++ 默认采用值传递，只有在程序员明确指定使用指针或引用时才会改变传递方式，所以该题目的说法是错误的。

20), 冒泡排序是一种稳定的排序算法。（ ）
{{ select(20) }}
- true. 正确
- false. 错误

> **答案**: true
> **解析**: 冒泡排序（Bubble Sort）是一种简单的排序算法。它的基本原理是通过重复地遍历待排序的数列，一次比较两个相邻的元素，如果它们的顺序错误（如在升序排序中前一个比后一个大）就交换它们。

在冒泡排序的过程中，如果遇到两个相等的元素，算法通常不会执行交换操作。这意味着相等元素在排序前后的相对位置保持不变。根据算法稳定性的定义：如果待排序的序列中存在多个具有相同关键字的记录，经过排序后，这些记录的相对次序保持不变，则称该算法是稳定的。

因此，冒泡排序是一种稳定的排序算法。
