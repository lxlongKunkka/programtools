# 1108 - GESP 八级样题（C++ 组） - 题目与解析

1), GESP 八级样题从 A 城到 C 城需要经过 B 城，其中 A 到 B 可选高铁和飞机，B 到 C 可以自驾或打的，请问 A 到 C 有几种交通选择（ ）
{{ select(1) }}
- A. 2
- B. 4
- C. 8
- D. 不知道

> **答案**: B
> **解析**: 这是一个典型的排列组合问题，可以通过数学中的“乘法原理”来解决。

1. 分析步骤：
   从 A 城到 C 城必须经过 B 城，因此整个行程可以分为两个阶段：
   第一阶段：从 A 城到 B 城。
   第二阶段：从 B 城到 C 城。

2. 计算各阶段的选择数：
   第一阶段（A 到 B）：题目给出可选高铁和飞机，共 2 种选择。
   第二阶段（B 到 C）：题目给出可选自驾或打的，共 2 种选择。

3. 应用乘法原理：
   乘法原理是指：如果完成一件事需要分成 n 个步骤，第一步有 m1 种方法，第二步有 m2 种方法……那么完成这件事共有 m1 × m2 × ... 种方法。
   在本题中，总的选择数 = 第一阶段的选择数 × 第二阶段的选择数 = 2 × 2 = 4 种。

具体的四种方案分别是：
1. 高铁 + 自驾
2. 高铁 + 打的
3. 飞机 + 自驾
4. 飞机 + 打的

因此，正确答案是 B。

2), 下面程序输出的 n 的值是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/3yudqrb7.png)
{{ select(2) }}
- A. 4
- B. 12
- C. 18
- D. 24

> **答案**: D
> **解析**: 这道题目考查的是 C++ 中嵌套循环（Nested Loops）的执行逻辑以及变量累加的过程。虽然题目中没有直接给出代码段，但根据 GESP 二级考试的常见题型和给出的选项（4, 12, 18, 24），该程序最可能的代码结构如下：

```cpp
int n = 0;
for (int i = 1; i <= 3; i++)
    for (int j = 1; j <= 4; j++)
        n += i;
cout << n << endl;
```

我们来详细分析这个程序的执行过程：

1.  **初始化**：变量 `n` 被初始化为 0。
2.  **外层循环**：变量 `i` 从 1 循环到 3，总共执行 3 次。
3.  **内层循环**：对于每一个 `i` 的值，变量 `j` 都会从 1 循环到 4，总共执行 4 次。
4.  **累加过程**：
    *   **当 i = 1 时**：内层循环执行 4 次，每次执行 `n += 1`。
        `n` 的变化为：0 + 1 + 1 + 1 + 1 = 4。
    *   **当 i = 2 时**：内层循环执行 4 次，每次执行 `n += 2`。
        `n` 的变化为：4 + 2 + 2 + 2 + 2 = 4 + 8 = 12。
    *   **当 i = 3 时**：内层循环执行 4 次，每次执行 `n += 3`。
        `n` 的变化为：12 + 3 + 3 + 3 + 3 = 12 + 12 = 24。
5.  **最终结果**：循环结束后，`n` 的最终值为 24。

**选项分析**：
*   **A. 4**：这是外层循环执行完第一次（i=1）后的 `n` 值。
*   **B. 12**：这是如果程序执行的是 `n++`（即只计算总循环次数 3 * 4 = 12），或者是外层循环执行完第二次（i=2）后的 `n` 值。
*   **C. 18**：这是如果内层循环次数为 3 时的计算结果（4 * (1+2+3) = 24，若为 3 * (1+2+3) = 18）。
*   **D. 24**：这是程序完整执行后的正确结果。

因此，正确答案是 D。

3), 对(a+b)5，想求出 a3b2 的系数可以使用（ ）
{{ select(3) }}
- A. 杨氏三角
- B. 祖冲之角
- C. 勾股三角
- D. 杨辉三角

> **答案**: D
> **解析**: 1. 题目背景：本题考查的是二项式定理及其系数的求法。
2. 二项式定理：根据二项式定理，$(a+b)^n$ 的展开式中，各项的系数对应于组合数 $C(n, k)$。对于 $(a+b)^5$，其展开式中 $a^3b^2$ 的系数是 $C(5, 2)$ 或 $C(5, 3)$。
3. 杨辉三角的性质：杨辉三角（在西方称为帕斯卡三角形）是一个由数字排列成的三角形阵列，它的每一行数字正好对应了二项式 $(a+b)^n$ 展开式的各项系数。
   - 第 0 行：1 （对应 $(a+b)^0$）
   - 第 1 行：1, 1 （对应 $(a+b)^1$）
   - 第 2 行：1, 2, 1 （对应 $(a+b)^2$）
   - 第 3 行：1, 3, 3, 1 （对应 $(a+b)^3$）
   - 第 4 行：1, 4, 6, 4, 1 （对应 $(a+b)^4$）
   - 第 5 行：1, 5, 10, 10, 5, 1 （对应 $(a+b)^5$）
4. 结果匹配：在第 5 行中，系数依次对应 $a^5, a^4b, a^3b^2, a^2b^3, ab^4, b^5$。可以看到 $a^3b^2$ 的系数是 10。
5. 选项分析：
   - A 选项“杨氏三角”通常指杨氏矩阵（Young Tableau），用于组合表示论。
   - B 选项“祖冲之角”和 C 选项“勾股三角”均不是求解二项式系数的标准数学工具。
   - D 选项“杨辉三角”是求二项式展开系数最直接、最常用的方法。

4), 对于 4 个结点的简单有向图，最少（ ）条边可以形成一条覆盖所有顶点的环。
{{ select(4) }}
- A. 5
- B. 4
- C. 3
- D. 2

> **答案**: B
> **解析**: 1. 题目要求在 4 个结点的简单有向图中，找到形成一个“覆盖所有顶点的环”所需的最少边数。
2. 覆盖所有顶点的环在图论中被称为哈密顿回路（Hamiltonian Cycle）。
3. 对于一个包含 n 个顶点的环，其结构是：从第一个顶点出发，经过第二个、第三个……直到第 n 个顶点，最后再从第 n 个顶点回到第一个顶点。
4. 具体的路径可以表示为：V1 → V2 → V3 → V4 → V1。
5. 统计边数：
   - 第一条边：V1 → V2
   - 第二条边：V2 → V3
   - 第三条边：V3 → V4
   - 第四条边：V4 → V1
6. 因此，对于 4 个结点的图，形成覆盖所有顶点的环最少需要 4 条边。
7. 如果边数少于 4 条（如 3 条），则最多只能形成一条连接所有顶点的路径（链），而无法构成闭合的环。

5), 对正整数 a 和 n （n 为 2 的正整数次幂），下面求 an 值的方法是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/42cji5on.png)
{{ select(5) }}
- A. 折半
- B. 二分
- C. 倍增
- D. 迭代

> **答案**: C
> **解析**: 1. 题目分析：题目要求计算 $a^n$ 的值，且给定条件是 $n$ 为 2 的正整数次幂（即 $n = 2^k$）。
2. 计算逻辑：由于 $n$ 是 2 的幂，我们可以通过连续平方的方式来快速求解。
   - 第一步：$a^2 = a \times a$
   - 第二步：$a^4 = a^2 \times a^2$
   - 第三步：$a^8 = a^4 \times a^4$
   - ...
   - 第 k 步：$a^{2^k} = a^{2^{k-1}} \times a^{2^{k-1}}$
3. 算法定义：这种利用已有的结果，通过翻倍（在指数上体现为翻倍，在数值上体现为平方）来快速达到目标的算法思想被称为“倍增法”（Doubling）。
4. 选项辨析：
   - A. 折半：通常指将问题规模缩小一半（如归并排序），虽然快速幂（Binary Exponentiation）有折半的思想，但在描述从 $a^1$ 到 $a^{2^k}$ 的构造过程时，倍增更准确。
   - B. 二分：通常指二分查找（Binary Search），侧重于在有序序列中通过比较缩小范围。
   - C. 倍增：核心特征是每次步长或规模翻倍（1, 2, 4, 8...），完全符合本题计算 $a^{2^k}$ 的过程。
   - D. 迭代：是计算机执行程序的一种基本方式（循环），不是针对该数学特性的特定算法名称。

因此，正确答案是 C。

6), 平面内，通过一点可以作（ ）条平行于给定直线的直线？
{{ select(6) }}
- A. 0
- B. 1
- C. 2
- D. 无限多

> **答案**: B
> **解析**: 本题考查的是平面几何中的平行公理（Parallel Postulate）。

1. **平行公理的基本内容**：在同一平面内，经过直线外一点，有且只有一条直线与这条直线平行。
2. **分类讨论**：
   - **点在直线外**：根据平行公理，可以作且只能作出 **1条** 直线与已知直线平行。
   - **点在直线上**：经过该点的任何直线要么与已知直线重合，要么与已知直线相交，在欧几里得几何的定义下，不认为它们平行。因此，在这种情况下可以作 0 条平行线。
3. **题目解析**：在编程考级（如 GESP/CSP）或数学基础测试中，此类题目通常考察的是平行公理的核心性质，即平行线的**唯一性**。虽然题目中没有明确说明“点在直线外”，但在给出的选项中，B 选项（1条）是符合平行公理唯一性特征的标准答案。在平面几何的常规语境下，我们默认讨论的是能够产生平行线的情况。

因此，正确答案为 B。

7), 定义常量const double pi=3.14159。如果一个等边三角形的边长为 4，下列（ ）表达式可以求其面积。
{{ select(7) }}
- A. 16*sin(pi/3)
- B. 16*cos(pi/3)
- C. 8*sin(pi/3)
- D. 8*cos(pi/3)

> **答案**: C
> **解析**: 1. 等边三角形的性质：等边三角形的三条边长度相等，三个内角也相等，均为 60°。
2. 弧度制转换：在编程语言（如 C++）中，数学函数库 `<cmath>` 里的 `sin()` 函数接收的参数是弧度。因为 180° 对应 pi，所以 60° 对应的弧度是 pi / 3。
3. 三角形面积公式：根据三角形面积的通用公式 $S = \frac{1}{2} \times a \times b \times \sin(C)$，其中 $a$ 和 $b$ 为三角形的两条边，$C$ 为这两条边的夹角。
4. 代入计算：
   - 边长 $a = 4$
   - 边长 $b = 4$
   - 夹角 $C = 60^\circ = \pi / 3$
   - 面积 $S = \frac{1}{2} \times 4 \times 4 \times \sin(\pi / 3)$
   - 简化后得：$S = 8 \times \sin(\pi / 3)$
5. 选项分析：
   - A 选项系数为 16，错误。
   - B 选项使用了余弦函数 cos 且系数错误。
   - C 选项符合推导结果，正确。
   - D 选项使用了余弦函数 cos，错误。

8), 下面程序使用 BFS 遍历一个有 n 个顶点、边权都为 1 的无向图 G，下面说法正确的是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/wtizz9mb.png)
{{ select(8) }}
- A. tl 记录遍历的结点数
- B. dis 按照贪心法变化
- C. dis 存储 st 到其他顶点的距离
- D. 算法复杂度是 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: C
> **解析**: 本题考查的是广度优先搜索（BFS）算法的基本原理及其在无向图中的应用。

1. **BFS 的核心特性**：在边权均为 1 的图中，BFS 按照距离起点的远近，由近及远地访问节点。即先访问距离起点为 1 的节点，再访问距离为 2 的节点，以此类推。
2. **关于选项 C**：在 BFS 的实现过程中，通常会使用一个 `dis` 数组来记录从起点 `st` 到各个顶点的最短距离。每当通过节点 `u` 发现一个尚未访问的邻接点 `v` 时，就会更新 `dis[v] = dis[u] + 1`。因此，`dis` 存储的是起点到其他顶点的距离，这是 BFS 在等权图中最重要的功能之一。
3. **关于选项 A**：在手写队列的 BFS 代码中，`tl`（tail）通常表示队列的尾指针。虽然在算法结束时，如果所有点都连通，`tl` 的值可能与节点数相关，但它的直接作用是维护队列的入队操作，而非专门用于“记录遍历的节点数”。
4. **关于选项 B**：贪心算法通常指在每一步选择中都采取当前状态下最好或最优的选择。虽然 BFS 找到的是最短路径，但它属于一种盲目搜索（按层遍历），通常 Dijkstra 算法才被明确描述为基于贪心策略。
5. **关于选项 D**：BFS 的时间复杂度取决于图的存储方式。如果使用邻接表存储，复杂度为 $O(n + m)$（$n$ 为顶点数，$m$ 为边数）；只有在使用邻接矩阵时，复杂度才是 $O(n^2)$。在没有明确存储结构的情况下，直接说复杂度是 $O(n^2)$ 是不准确的。

综上所述，选项 C 是对 BFS 算法逻辑最准确的描述。

9), 下面的冒泡排序中尝试提前结束比较过程，横线处应该填写的代码是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/f0nl7bzg.png)
{{ select(9) }}
- A. i = lastExchangeIndex + 1
- B. i = lastExchangeIndex - 1
- C. i = lastExchangeIndex
- D. i = lastExchangeIndex - j

> **答案**: C
> **解析**: 这道题目考查的是冒泡排序的一种经典优化算法——“最后一次交换位置优化”（Last Exchange Index Optimization）。

1. **冒泡排序基本原理**：传统的冒泡排序通过相邻元素的比较和交换，每一轮将当前未排序部分的最大值移动到末尾。通常需要进行 $n-1$ 轮。

2. **优化思路**：在某一轮排序中，如果在位置 `j` 发生了最后一次交换，这意味着 `j` 之后的所有元素都已经处于它们最终的有序位置上了（因为在这一轮中，`j` 之后的元素都没有发生过交换）。

3. **代码实现逻辑**：
   - 我们使用一个变量 `lastExchangeIndex` 来记录每一轮中最后一次发生交换的下标。
   - 在内层循环结束后，外层循环的边界 `i` 不再是简单地减 1，而是直接更新为 `lastExchangeIndex`。
   - 这样，下一轮排序只需处理到 `lastExchangeIndex` 为止，大大减少了不必要的比较次数。
   - 如果某一轮完全没有发生交换，`lastExchangeIndex` 将保持初始值（通常为 0），循环会直接结束，从而实现提前终止。

4. **选项分析**：
   - **A, B, D**：这些选项的逻辑不符合最后一次交换位置优化的标准做法，可能会导致漏掉必要的比较或者逻辑错误。
   - **C**：`i = lastExchangeIndex` 正确地将下一轮的比较范围缩小到了上一次最后交换的位置，是标准的优化写法。

因此，正确答案是 C。

10), 对数列3,4,7,12,19,28,39求和，除简单累加外，还可以用下面（ ）来直接计算。
{{ select(10) }}
- A. 等差数列求和
- B. 等比数列求和
- C. 斐波拉契数列
- D. 其他某种有规律序列

> **答案**: D
> **解析**: 1. **分析数列规律**：
   观察数列：3, 4, 7, 12, 19, 28, 39。
   计算相邻两项的差值：
   - 4 - 3 = 1
   - 7 - 4 = 3
   - 12 - 7 = 5
   - 19 - 12 = 7
   - 28 - 19 = 9
   - 39 - 28 = 11
   可以看到，差值序列为 1, 3, 5, 7, 9, 11，这是一个公差为 2 的等差数列。

2. **判定选项**：
   - **A. 等差数列**：原数列的差值不是常数，所以原数列不是等差数列。
   - **B. 等比数列**：4/3 ≠ 7/4，比值不固定，所以不是等比数列。
   - **C. 斐波拉契数列**：斐波拉契数列的规律是 $a_n = a_{n-1} + a_{n-2}$。虽然 3+4=7，但 4+7=11 ≠ 12，所以不是斐波拉契数列。
   - **D. 其他某种有规律序列**：这种原数列的差值构成等差数列的序列被称为“二阶等差数列”。其通项公式可以表示为关于 $n$ 的二次函数。在本题中，通项公式为 $a_n = (n-1)^2 + 3$。

3. **结论**：
   由于该数列具有明确的数学规律（二阶等差数列），我们可以利用求和公式（如平方和公式）来直接计算其总和，而不需要逐一累加。因此，它属于“其他某种有规律序列”。

11), 约定杨辉三角形第 0 行只有 1 个元素是 1，第 1 行有 2 个元素都是 1，第四行的所有元素之和是？（ ）
{{ select(11) }}
- A. 8
- B. 16
- C. 24
- D. 32

> **答案**: B
> **解析**: 杨辉三角形（Pascal's Triangle）具有以下重要性质：
1. 第 n 行（从第 0 行开始计数）的元素个数为 n + 1 个。
2. 第 n 行的第 k 个数字（从 0 开始计数）对应组合数 C(n, k)。
3. 第 n 行的所有元素之和等于 2 的 n 次方（即 2^n）。

根据题目描述：
- 第 0 行：1，元素之和为 2^0 = 1。
- 第 1 行：1, 1，元素之和为 2^1 = 2。
- 第 2 行：1, 2, 1，元素之和为 2^2 = 4。
- 第 3 行：1, 3, 3, 1，元素之和为 2^3 = 8。
- 第 4 行：1, 4, 6, 4, 1，元素之和为 2^4 = 16。

计算第 4 行元素之和：1 + 4 + 6 + 4 + 1 = 16。
因此，正确答案是 B。

8. 第 7 次循环：sum = 21 + 7 = 28
9. 第 8 次循环：sum = 28 + 8 = 36
10. 第 9 次循环：sum = 36 + 9 = 45
11. 第 10 次循环：sum = 45 + 10 = 55

14), Dijkstra 的算法在实现时一般可以选用（ ）来提高效率？
{{ select(14) }}
- A. 数组
- B. 链表
- C. 堆
- D. 栈

> **答案**: C
> **解析**: Dijkstra 算法是一种用于求解单源最短路径问题的经典算法。其核心操作步骤如下：
1. 初始化：将起点的距离设为 0，其余点的距离设为无穷大。
2. 迭代：在所有未访问的节点中，寻找当前距离起点最近的节点 $u$。
3. 松弛：更新节点 $u$ 的所有邻接点 $v$ 的距离。

在上述第 2 步中，如果使用普通的数组来存储距离，每次寻找最小值需要遍历整个数组，时间复杂度为 $O(V)$（$V$ 为顶点数），整个算法的时间复杂度为 $O(V^2)$。

为了提高效率，我们可以使用“堆”（Heap）这种数据结构（在 C++ STL 中通常体现为 `priority_queue` 优先队列）。堆可以动态维护一组数据的最小值：
1. 查找并取出最小值：时间复杂度降低为 $O(\log V)$。
2. 更新距离（插入新值）：时间复杂度为 $O(\log V)$。

通过使用堆优化，Dijkstra 算法的总时间复杂度可以优化到 $O((V+E) \log V)$，其中 $E$ 是边数。在处理稀疏图时，这种优化能显著提升程序的运行效率。因此，堆是实现 Dijkstra 算法时提高效率的首选数据结构。

20), 在 C++ 语言中，所有 int 类型的值，经过若干次左移操作（<<）后，它们的值总会变为 0。（ ）
{{ select(20) }}
- true. 正确
- false. 错误

> **答案**: false
> **解析**: 这道题的答案是“错误”，主要原因可以从 C++ 语言标准和底层实现两个方面来分析：

1. **有符号整数的未定义行为 (Undefined Behavior)**：
   在 C++ 中，`int` 是有符号整数类型。根据 C++ 标准（特别是 C++20 之前），如果左移的操作数是负数，或者左移后的结果超出了该类型的表示范围（即溢出，例如将 1 移入了符号位），这被定义为“未定义行为”。这意味着 C++ 标准并不保证此时程序会发生什么，结果可能是任何值，甚至导致程序崩溃。因此，不能说它“总会”变为 0。

2. **移位位数的限制**：
   C++ 标准规定，如果移位的位数大于或等于该类型的总位数（例如，在一个 32 位的系统上左移 32 位或更多），这也是未定义行为。在实际的硬件实现中，许多 CPU（如 x86 架构）会对移位次数进行取模运算。例如，对于 32 位的 `int`，执行 `x << 32` 实际上相当于执行 `x << (32 % 32)`，即 `x << 0`，结果仍然是原值 `x`，并不会变为 0。

3. **逻辑位移与算术位移**：
   虽然从位运算的直观逻辑上看，左移会在右侧补 0，经过足够多次移位后所有原始位都会被移出，但在 C++ 语言的规范下，由于上述的未定义行为和硬件处理方式，这一结论并不成立。

综上所述，在 C++ 语言中，不能保证所有 `int` 类型的值经过若干次左移后一定会变为 0。

2. **第 1 次探索**：从所有未访问的节点中选择距离最小的点，即源点 A。通过 A 来更新其所有直接邻居（如 B、C 等）的距离。此时，只有 A 的直接邻居的距离被更新。
