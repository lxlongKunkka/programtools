1), GESP 三级样题下列关于负数的原码、反码、补码的描述中，正确的是（ ）
{{ select(1) }}
- A. 原码和反码互为按位取反（符号位除外），补码为反码加 1
- B. 原码和反码互为按位取反（符号位除外），补码为原码加 1
- C. 反码和补码互为按位取反（符号位除外），原码为反码加 1
- D. 补码和原码互为按位取反（符号位除外），反码为补码加 1

> **答案**: A
> **解析**: 在计算机系统中，数值的表示通常有原码、反码和补码三种形式。对于负数，它们的转换规则如下：

1. 原码：最高位为符号位（负数为 1），其余位表示该数的绝对值。
2. 反码：在原码的基础上，符号位保持不变，其余各位按位取反（0 变 1，1 变 0）。
3. 补码：在反码的基础上加 1。

选项分析：
A 选项：准确描述了负数原码到反码（符号位除外按位取反）以及反码到补码（加 1）的转换过程，描述正确。
B 选项：补码应该是反码加 1，而不是原码加 1，描述错误。
C 选项：反码和补码并不是互为按位取反的关系，描述错误。
D 选项：补码和原码也不是互为按位取反的关系，描述错误。

因此，正确答案是 A。

2), 一个 int 类型的值乘以 8，等价于以下哪个位运算？（ ）
{{ select(2) }}
- A. 左移 3 位
- B. 右移 3 位
- C. 左移 8 位
- D. 右移 8 位

> **答案**: A
> **解析**: 1. 在计算机二进制运算中，位移运算与乘除法有直接的对应关系。
2. 左移运算（<<）：将二进制位向左移动 n 位，相当于在右侧补 n 个 0。在不发生溢出的情况下，每左移 1 位相当于原数乘以 2，左移 n 位则相当于原数乘以 2 的 n 次方（2^n）。
3. 本题要求乘以 8。因为 8 = 2^3，所以乘以 8 等价于将该数值的二进制位向左移动 3 位。
4. 选项分析：
   - A. 左移 3 位：相当于乘以 2^3 = 8，正确。
   - B. 右移 3 位：相当于除以 2^3 = 8（取整），错误。
   - C. 左移 8 位：相当于乘以 2^8 = 256，错误。
   - D. 右移 8 位：相当于除以 2^8 = 256（取整），错误。

3), 以下哪个属于 C++ 语言中的位运算符？（ ）
{{ select(3) }}
- A. +
- C. *
- D. &

> **答案**: C
> **解析**: 在 C++ 编程语言中，运算符根据其功能可以分为多种类型。以下是对选项中各运算符的详细分析：

1. 选项 A (+)：这是一个算术运算符，用于执行加法运算。
2. 选项 B (*)：这通常是一个算术运算符，用于执行乘法运算；在指针操作中，它也可用作解引用运算符。
3. 选项 C (&)：这是一个位运算符，称为“按位与”（Bitwise AND）。它会对操作数的每一位二进制位进行逻辑与运算（只有当两个对应的二进制位都为 1 时，结果位才为 1）。此外，& 在 C++ 中还可以作为单目运算符表示“取地址”，或用于声明引用类型。

C++ 中的常用位运算符包括：
- & （按位与）
- | （按位或）
- ^ （按位异或）
- ~ （按位取反）
- << （左移）
- >> （右移）

因此，正确答案是 C。

4), 若有以下代码，则数组 arr 的长度是（ ）？
{{ select(4) }}
- A. 3
- B. 4
- C. 5
- D. 6

> **答案**: C
> **解析**: 在 C++ 编程中，数组的长度（即它能存储的元素个数）通常由其定义时的方括号 `[]` 中的数值决定。

1. **自动推导长度**：如果定义数组时没有在方括号内指定具体的数值，而是直接通过大括号 `{}` 进行初始化（例如 `int arr[] = {1, 2, 3, 4, 5};`），编译器会根据初始化列表中的元素个数自动确定数组的长度。
2. **计数**：在常见的 GESP 考题中，该代码通常为 `int arr[] = {1, 2, 3, 4, 5};`。可以看到大括号内共有 5 个元素（1, 2, 3, 4, 5）。
3. **结论**：因此，编译器会为数组 `arr` 分配 5 个整数的空间，数组的长度就是 5。

注意：如果代码是 `int arr[10] = {1, 2, 3, 4, 5};`，虽然只初始化了 5 个值，但数组的长度依然是定义的 10。但在本题给出的选项和常见的考题背景下，最符合逻辑的初始化形式是 `arr[] = {1, 2, 3, 4, 5}`，故选 C。

5), 在 C++ 语言中，可以定义一个一维整型数组的是（ ）？
{{ select(5) }}
- A. int array[5];
- B. int array[];
- C. int[5] array;
- D. int[] array;

> **答案**: A
> **解析**: 在 C++ 语言中，定义一维数组的标准语法格式为：`数据类型 数组名[数组长度];`。

1. 选项 A：`int array[5];` 完全符合 C++ 语法，定义了一个名为 array 的整型数组，其长度为 5。
2. 选项 B：`int array[];` 是错误的。在定义数组时，必须指定数组的长度，除非在定义的同时进行了初始化（例如 `int array[] = {1, 2, 3};`），否则编译器无法确定需要分配多少内存空间。
3. 选项 C：`int[5] array;` 语法错误。C++ 要求方括号 `[]` 必须放在变量名之后，而不是类型之后。
4. 选项 D：`int[] array;` 语法错误。这属于 Java 或 C# 等语言的语法风格，在 C++ 中是不合法的。

因此，正确答案是 A。

6), 枚举算法的主要特点是（ ）？
{{ select(6) }}
- A. 以空间换时间
- B. 逐个尝试所有可能的解
- C. 动态规划
- D. 贪心策略

> **答案**: B
> **解析**: 枚举算法（也称为穷举法或暴力搜索法）是一种最基础且直观的算法设计策略。

1. 枚举算法的核心思想：根据问题本身的性质，将所有可能的候选解按照某种顺序逐一列举出来，并对每一个候选解进行判断，检查它是否符合题目给定的条件。如果符合，则该候选解就是问题的一个解。
2. 选项分析：
   - A 选项：以空间换时间通常是指通过增加内存消耗（如使用哈希表、预处理数组等）来降低时间复杂度，这并非枚举算法的主要特点，枚举算法往往时间复杂度较高。
   - B 选项：准确描述了枚举算法的工作方式，即“不重不漏”地尝试所有可能性。
   - C 选项：动态规划是一种将复杂问题分解为子问题，并通过存储子问题的解来避免重复计算的算法，与枚举有本质区别。
   - D 选项：贪心策略是指在每一步选择中都采取当前状态下最好或最优的选择，从而希望导致结果是全局最优的算法，它并不尝试所有可能的解。

因此，枚举算法的主要特点是逐个尝试所有可能的解。

7), 对于一个十进制数 37，以下哪个是它的二进制表示？（ ）
{{ select(7) }}
- A. 10101
- B. 100101
- C. 101001
- D. 1000101

> **答案**: B
> **解析**: 将十进制数转换为二进制数，最常用的方法是“除2取余法”：

1. 37 ÷ 2 = 18，余数为 1
2. 18 ÷ 2 = 9，余数为 0
3. 9 ÷ 2 = 4，余数为 1
4. 4 ÷ 2 = 2，余数为 0
5. 2 ÷ 2 = 1，余数为 0
6. 1 ÷ 2 = 0，余数为 1

将得到的余数从下往上依次排列，得到 100101。

也可以使用“按权展开法”进行验证：
100101 = 1 × 2^5 + 0 × 2^4 + 0 × 2^3 + 1 × 2^2 + 0 × 2^1 + 1 × 2^0
       = 32 + 0 + 0 + 4 + 0 + 1
       = 37

因此，十进制数 37 的二进制表示是 100101。正确答案为 B。

9), 下列哪个是 C++ 语言中用于获取字符串长度的函数（ ）
{{ select(9) }}
- A. length（ ）
- B. len（ ）
- C. getLength（ ）
- D. strlen（ ）

> **答案**: D
> **解析**: 在 C++ 编程语言中，获取字符串长度的方法主要取决于字符串的类型（是 C 风格字符串还是 C++ 标准库中的 `std::string` 对象）：

1.  **strlen()**：这是定义在 `<cstring>`（或 C 语言中的 `<string.h>`）头文件中的一个**标准库函数**。它用于计算 C 风格字符串（以空字符 `\0` 结尾的字符数组）的长度。由于它是一个独立的全局函数，且在 C++ 中被广泛继承和使用，因此在许多编程考试中，当题目问及“函数”时，`strlen()` 通常被视为标准答案。

2.  **length()**：这是 C++ 标准库 `std::string` 类的一个**成员函数**（也称为方法）。要使用它，必须先创建一个 `string` 对象，例如 `str.length()`。虽然它也能获取长度，但在严格的术语区分中，它被归类为“成员函数”而非“独立函数”。

3.  **len()**：这是 Python 等语言中获取长度的函数，在 C++ 中并不存在。

4.  **getLength()**：这通常是某些自定义类中定义的函数名，并不是 C++ 标准库中的标准函数。

**总结**：
虽然在现代 C++ 开发中 `std::string::length()` 更为常用，但在考级题目（如 GESP/CSP）的语境下，为了区分“函数”与“成员函数”，或者考查对 C 风格字符串处理的理解，通常将 **strlen()** 视为获取字符串长度的标准函数。如果选项中没有“以上都是”，`strlen()` 是最符合“函数”这一技术定义的选项。

11), 如果 a 和 b 均为 int 类型的变量，下列表达式能够正确判断 “a 等于 0 且 b 等于 0” 的是（ ）？
{{ select(11) }}
- A. ((~a) && (~b))
- B. ((a & b) == 0)
- C. ((a | b) == 0)
- D. ((a ^ b) == 0)

> **答案**: C
> **解析**: 本题考查 C++ 中的位运算符和逻辑判断。我们需要找到一个表达式，当且仅当 a = 0 且 b = 0 时，表达式的值为真（true）。

A 选项：((~a) && (~b))。~ 是按位取反运算符。如果 a = 0，~a 的结果是 -1（在补码表示中所有位均为 1）。在逻辑判断中，非 0 即为真，所以 ~0 为真。但是，如果 a = 1，~a 的结果是 -2，同样是非 0，也为真。因此，这个表达式不能唯一确定 a 和 b 是否为 0。

B 选项：((a & b) == 0)。& 是按位与运算符。当 a 和 b 的二进制位没有共同的 1 时，结果就为 0。例如 a = 1 (01) 和 b = 2 (10)，a & b 的结果是 0，但 a 和 b 都不等于 0。

C 选项：((a | b) == 0)。| 是按位或运算符。按位或的规则是：只要对应的二进制位中有一个为 1，结果位就为 1。只有当 a 和 b 的所有二进制位都为 0 时（即 a = 0 且 b = 0），a | b 的结果才会是 0。因此，(a | b) == 0 能够准确判断 a 和 b 是否同时为 0。

D 选项：((a ^ b) == 0)。^ 是按位异或运算符。按位异或的规则是：对应位相同为 0，不同为 1。因此，a ^ b == 0 等价于判断 a == b。如果 a = 5, b = 5，结果也为 0，但它们不等于 0。

综上所述，正确答案是 C。

12), 如果 a 为 int 类型的变量，下列哪个表达式可以正确求出满足 “大于等于 a 且是 4 的倍数” 的整数中最小的？（ ）
{{ select(12) }}
- A. (a & (~3))
- B. (a / 4 * 4)
- C. ((a - 1) | 3) + 1
- D. (a << 2)

> **答案**: C
> **解析**: 本题考查位运算的应用以及对“向上取整到 4 的倍数”逻辑的理解。

1. 目标分析：我们需要找到一个整数 $x$，满足 $x \ge a$ 且 $x$ 是 4 的倍数，且 $x$ 是所有满足条件中最小的一个。

2. 选项逐一分析：
   - A 选项 `(a & (~3))`：`~3` 的二进制低两位是 `00`，其余位是 `1`。进行按位与运算会将 `a` 的低两位清零。这实际上是求“小于等于 a 且是 4 的倍数”的最大整数。例如当 $a=5$ 时，结果为 4，不符合“大于等于 a”的条件。
   - B 选项 `(a / 4 * 4)`：在 C++ 整数运算中，`a / 4` 会向下取整。这同样是求“小于等于 a 且是 4 的倍数”的最大整数。
   - C 选项 `((a - 1) | 3) + 1`：
     - `| 3` 的作用是将二进制的低两位强制设为 `11`。
     - 如果 $a$ 是 4 的倍数（如 $a=4$），则 $a-1=3$（二进制 `...011`），`3 | 3` 仍为 3，加 1 后得 4。结果正确。
     - 如果 $a$ 不是 4 的倍数（如 $a=5, 6, 7$），则 $a-1$ 的范围是 $[4, 6]$，它们的二进制低两位可能是 `00, 01, 10`。执行 `| 3` 后，低两位都变成 `11`，即结果统一变成了 7（二进制 `...111`），再加 1 得到 8。结果正确。
     - 该表达式完美实现了向上取整到 4 的倍数的逻辑。
   - D 选项 `(a << 2)`：这相当于 $a \times 4$，显然不符合题意。

综上所述，正确答案是 C。

14), 在下列代码的横线处填写（ ），可以使得输出是“1357”，不会有多余字符

![image](https://cdn.luogu.com.cn/upload/image_hosting/7npq1bov.png)
{{ select(14) }}
- A. int i = 0; i < strlen(str); i++
- B. int i = 0; str[i] != '\0'; i++
- C. int i = 1; i <= 7; i += 2
- D. int i = 0; i <= 6; i += 2

> **答案**: D
> **解析**: 这道题目考查的是循环控制语句以及数组/字符串的索引访问。

1. **题目背景推断**：
   虽然题干没有给出完整的代码上下文，但从选项 A 和 B 中出现的 `str` 和 `strlen(str)` 可以推断，代码中定义了一个字符串（例如 `char str[] = "1234567";`），并且循环体内部的操作是 `cout << str[i];`。

2. **选项分析**：
   - **选项 A (`int i = 0; i < strlen(str); i++`)**：这是最常见的字符串遍历方式，会依次访问字符串中的每一个字符。如果字符串是 "1234567"，它会输出 "1234567"。
   - **选项 B (`int i = 0; str[i] != '\0'; i++`)**：这与选项 A 逻辑相同，也是逐个遍历字符直到遇到结束符。输出结果同样是 "1234567"。
   - **选项 C (`int i = 1; i <= 7; i += 2`)**：如果代码是 `cout << str[i]`，且 `str` 是 "1234567"，那么 `i` 的取值将是 1, 3, 5, 7。对应的字符分别是 `str[1]`('2')、`str[3]`('4')、`str[5]`('6')，而 `str[7]` 是字符串结束符 `\0`。输出将是 "246"。
   - **选项 D (`int i = 0; i <= 6; i += 2`)**：如果代码是 `cout << str[i]`，且 `str` 是 "1234567"，那么 `i` 的取值将是 0, 2, 4, 6。
     - 当 `i = 0` 时，输出 `str[0]` 即 '1'；
     - 当 `i = 2` 时，输出 `str[2]` 即 '3'；
     - 当 `i = 4` 时，输出 `str[4]` 即 '5'；
     - 当 `i = 6` 时，输出 `str[6]` 即 '7'；
     循环结束，最终输出为 "1357"。

3. **结论**：

15), 在下列代码的横线处填写（ ），可以使得输出是“17 11”

![image](https://cdn.luogu.com.cn/upload/image_hosting/z9kh4clx.png)
{{ select(15) }}
- A. a + b
- B. a - b
- C. a ^ b
- D. a & b

> **答案**: C
> **解析**: 这道题目考察的是位运算中的异或（XOR）运算在变量交换中的应用。

在计算机编程中，如果不使用临时变量来交换两个变量 `a` 和 `b` 的值，常用的方法之一是利用异或运算的特性。异或运算有以下几个重要性质：
1. 任意数与 0 异或等于它本身：`x ^ 0 = x`
2. 任意数与自身异或等于 0：`x ^ x = 0`
3. 异或运算满足交换律和结合律：`a ^ b ^ c = a ^ (b ^ c)`

典型的异或交换代码如下：
```cpp
int a = 11; // 二进制: 01011
int b = 17; // 二进制: 10001

a = a ^ b;  // 第一步：a 存储了 a 和 b 的混合信息
b = a ^ b;  // 第二步：b = (a ^ b) ^ b = a ^ (b ^ b) = a ^ 0 = 11 (此时 b 变成了原来的 a)
a = a ^ b;  // 第三步：a = (a ^ b) ^ a = (a ^ a) ^ b = 0 ^ b = 17 (此时 a 变成了原来的 b)
```

根据题目描述，输出结果是 "17 11"，即 `a` 和 `b` 的值发生了互换（假设初始值 `a=11, b=17`）。在异或交换法的逻辑中，第三步的操作依然是 `a ^ b`。

解析选项：
- A. `a + b`：加法运算，不符合异或交换逻辑。
- B. `a - b`：减法运算，虽然加减法也能实现交换（`a=a+b; b=a-b; a=a-b;`），但如果前两步使用的是异或，最后一步必须是异或。
- C. `a ^ b`：正确，完成异或交换的最后一步。
- D. `a & b`：按位与运算，无法实现变量交换。

因此，正确答案是 C。
