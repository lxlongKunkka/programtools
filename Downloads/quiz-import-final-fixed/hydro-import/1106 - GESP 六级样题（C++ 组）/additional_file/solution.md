# 1106 - GESP 六级样题（C++ 组） - 题目与解析

1), GESP 六级样题以下不属于计算机输出设备的有（ ）
{{ select(1) }}
- A. 麦克风
- B. 音响
- C. 打印机
- D. 显示器

> **答案**: A
> **解析**: 1. 计算机硬件系统通常由五大核心部分组成：输入设备、输出设备、存储器、运算器和控制器。
2. 输出设备（Output Device）的作用是将计算机处理后的数据（如文字、图像、声音等）转换成人类或其他设备能够识别的形式。常见的输出设备包括：
   - 显示器（Monitor）：输出视觉图像。
   - 打印机（Printer）：输出纸质文档。
   - 音响/扬声器（Speaker）：输出声音。
   - 绘图仪（Plotter）：输出大幅面图形。
3. 输入设备（Input Device）的作用是将外部信息（如指令、声音、图像等）输入到计算机中。常见的输入设备包括：
   - 键盘、鼠标：输入指令和文字。
   - 麦克风（Microphone）：采集声音并将其转化为数字信号输入计算机。
   - 扫描仪、摄像头：输入图像或视频。
4. 选项分析：
   - A 选项：麦克风是捕捉声音并输入到计算机的设备，属于输入设备。
   - B、C、D 选项：音响、打印机、显示器均是将计算机内部信息向外输出的设备，属于输出设备。
因此，不属于输出设备的是麦克风。

2), 小明想了一个 1 到 100 之间的整数。你可以做多次猜测，每次猜测之后，如果你没有猜中，小明会告诉你，你猜的数比他想的数大还是小。你希望你在运气最坏的情况下花费最少的次数猜中，请问你运气最坏的情况下会猜（ ）次？（包括最后猜中的那次）。
{{ select(2) }}
- A. 6
- B. 7
- C. 8
- D. 100

> **答案**: B
> **解析**: 这道题目考察的是二分查找（Binary Search）算法的思想。

1. 题目要求在“运气最坏的情况下”花费“最少的次数”，这实际上是在寻找二分查找算法的时间复杂度上限。
2. 二分查找的基本原理是：每次猜测中间的数，根据反馈（偏大或偏小）将搜索范围缩小一半。
3. 对于 1 到 100 之间的整数，搜索范围的大小为 100。
4. 每次猜测后，剩余的可能性会变为原来的一半。我们需要找到一个最小的整数 k，使得 2 的 k 次方大于或等于 100。
   - 当 k = 6 时，$2^6 = 64$，不足以覆盖 100 个数。
   - 当 k = 7 时，$2^7 = 128$，大于 100。
5. 具体过程如下：
   - 第 1 次猜 50，剩余范围约 50 个数；
   - 第 2 次猜 25 或 75，剩余范围约 25 个数；
   - 第 3 次猜，剩余范围约 12 个数；
   - 第 4 次猜，剩余范围约 6 个数；
   - 第 5 次猜，剩余范围约 3 个数；
   - 第 6 次猜，剩余范围约 1 个数；
   - 第 7 次猜，必然能确定该数。

因此，在最坏的情况下，最多需要猜 7 次。

3), 关于分治算法，下列说法错误的是（ ）
{{ select(3) }}
- A. 分治算法的核心思想是分而治之，即把问题转化为多个规模更小的子问题求解。
- B. 分治算法可以不使用递归实现。
- C. 分治算法的时间复杂度是 𝑂 ( 𝑙 𝑜 𝑔 𝑁 ) O(logN)，其中 𝑁 N 表示问题的规模。
- D. 二分法、快速排序等算法都是典型的使用分治思想的算法。

> **答案**: C
> **解析**: A选项正确：分治算法（Divide and Conquer）的核心思想确实是“分而治之”，即将一个复杂的问题分解成两个或多个规模较小的相同或相似的子问题，直到子问题可以简单地直接求解，最后将子问题的解合并得到原问题的解。

B选项正确：虽然分治算法最自然的实现方式是递归，但任何递归程序都可以通过引入栈（Stack）等数据结构改写为非递归（迭代）形式。例如，非递归的归并排序和二分查找都是存在的。

C选项错误：分治算法的时间复杂度取决于分解子问题的个数、子问题的规模以及合并子问题所花费的时间。不同的分治算法复杂度各不相同。例如：
- 二分查找的时间复杂度是 O(log N)；
- 归并排序和快速排序（平均情况）的时间复杂度是 O(N log N)；
- Strassen矩阵乘法的时间复杂度约为 O(N^2.81)。
因此，不能笼统地说分治算法的时间复杂度就是 O(log N)。

D选项正确：二分法（属于减治思想，是分治的一种特殊情况）、快速排序、归并排序、大整数乘法等都是分治思想的典型代表。

4), 下面关于 C++ 类的说法中，正确的是（ ）？
{{ select(4) }}
- A. 派生类不能和基类有同名成员函数，因为会产生歧义。
- B. 派生类可以和基类有同名成员函数，派生类覆盖基类的同名成员函数。
- C. 派生类可以和基类有同名成员函数，但是否覆盖同名成员函数需要取决于函数参数是否一致。
- D. C++ 中派生类不继承基类的任何成员函数。

> **答案**: C
> **解析**: 在 C++ 的继承机制中，关于同名成员函数的处理规则如下：

1. 派生类可以拥有与基类同名的成员函数。选项 A 错误。
2. 派生类继承基类的成员（除了构造函数、析构函数等特殊成员）。选项 D 错误。
3. 当派生类定义了与基类同名的函数时，会发生以下两种情况：
   - 重写/覆盖（Override）：如果基类中的函数是虚函数（virtual），且派生类中的函数名、参数列表、返回类型都与基类完全一致，那么派生类的函数会“覆盖”基类的虚函数，实现多态。
   - 隐藏（Hiding）：如果派生类中的函数与基类同名，但参数列表不同，或者基类函数不是虚函数，那么基类的同名函数在派生类作用域内会被“隐藏”，而不是覆盖。
4. 选项 B 说法太绝对，因为它没有考虑到参数不一致的情况。
5. 选项 C 正确地指出：派生类可以有同名函数，但“覆盖”这一行为的发生前提是函数签名（包括参数列表）必须保持一致（且基类函数应为虚函数）。如果参数不一致，则属于函数隐藏而非覆盖。

5), 关于下面 C++ 代码说法错误的是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/s5dv37lr.png)
{{ select(5) }}
- A. Pet 类是基类，Dog 类是子类。
- B. Dog 类的构造函数中，将自动调用 Pet 类的构造函数。
- C. dog 是 Dog 类的实例。
- D. 最后一行（即输出行 A）输出代码会报错，因为 Pet 类中没有成员变量 color。

> **答案**: D
> **解析**: 本题考查 C++ 面向对象编程中的继承与多态基础知识。

1. **继承关系（选项 A）**：在典型的 C++ 继承体系中，`Pet`（宠物）作为更通用的类通常被定义为基类（父类），而 `Dog`（狗）作为具体的种类被定义为子类。这是符合逻辑的。
2. **构造函数调用顺序（选项 B）**：在 C++ 中，当创建一个子类对象时，程序会首先自动调用基类的构造函数，以初始化从基类继承而来的成员，然后再执行子类自身的构造函数。因此选项 B 的描述是正确的。
3. **实例对象（选项 C）**：根据题目描述，`dog` 是 `Dog` 类的一个实例（对象），这是面向对象编程中的基本概念。
4. **成员访问权限（选项 D）**：这是本题的错误项。虽然基类 `Pet` 中可能没有定义 `color` 变量，但 `dog` 是子类 `Dog` 的实例。在继承关系中，子类可以拥有自己特有的成员变量。只要 `Dog` 类中定义了 `color` 成员，`dog.color` 的访问就是完全合法的，不会报错。编译器在解析 `dog.color` 时，会首先在 `Dog` 类中查找该成员。因此，说“因为 Pet 类中没有成员变量 color 而报错”是不符合 C++ 语法规则的。

6), 以下几个类定义中不能顺利通过编译的是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/aooh98d9.png)
{{ select(6) }}
- A. class A
- B. class B
- C. class C
- D. class D

> **答案**: C
> **解析**: 在 C++ 中，类定义的编译规则如下：

A. class A {};
这是一个最简单的空类定义，语法完全正确。

B. class B { public: B(); };
这是一个包含构造函数声明的类定义，语法完全正确。

C. class C { C c; };
这是不能通过编译的。原因在于：
1. 递归定义问题：在类 C 的定义过程中，类 C 被视为一个“不完整类型”（Incomplete Type）。
2. 内存分配问题：编译器在处理类定义时，需要确定该类对象所占用的内存大小。如果类 C 中包含一个类 C 的实例对象，那么为了计算 C 的大小，就需要先知道内部成员 c 的大小，而 c 的大小又取决于 C 的大小，这会导致无限递归，编译器无法确定其内存布局。

D. class D { D *d; };
这是可以通过编译的。原因在于：
1. 指针的大小是固定的（在 32 位系统中通常为 4 字节，64 位系统中为 8 字节），与它所指向的对象类型是否完整无关。
2. C++ 允许在类定义中使用指向自身类型的指针或引用，这常用于实现链表、树等数据结构。

综上所述，选项 C 因为尝试在类中定义自身的非静态实例对象，会导致编译错误。

7), 关于运算符重载，下列说法正确的是（ ）
{{ select(7) }}
- A. 对于下列代码中的两个运算符重载实现：虽然它们的参数类型都是 int 与 Test 的组合，这段代码仍可以通过编译。这是因为在两个重载实现中，int 与 Test 在参数中的顺序不同，编译器能够唯一地确定调用哪个运算符实现代码。
- B. A 的说法是错误的。这是因为加法满足交换律，因此即便调换 A 和 int 的顺序，加法运算的实现仍必须保持一致，编译器在编译时也会检查这一点。如将后一处 int 改为 double，则可通过编译。
- C. A 和 B 的说法都是错误的。运算符重载时，所有参数以及返回值的类型必须完全相同。因此，即便是下列代码中的运算符重载实现，也不能通过编译。
- D. A、B、C 的说法都是错误的。

> **答案**: A
> **解析**: 1. 运算符重载的本质：在 C++ 中，运算符重载实际上是一种特殊的函数重载。编译器通过函数签名（包括函数名、参数个数、参数类型及其顺序）来区分不同的函数。
2. 参数顺序的重要性：对于 `operator+` 而言，`operator+(int, Test)` 和 `operator+(Test, int)` 的参数类型虽然相同，但由于参数出现的顺序不同，它们的函数签名是唯一的。因此，编译器在处理如 `5 + obj` 和 `obj + 5` 这样的表达式时，能够准确地找到对应的重载函数。
3. 驳斥选项 B：C++ 编译器仅负责语法检查，并不理解或强制执行数学上的“交换律”。程序员可以为 `a + b` 和 `b + a` 编写完全不同的逻辑，编译器不会对此进行干预。
4. 驳斥选项 C：运算符重载并不要求参数类型或返回值类型必须一致。例如，我们可以定义一个“日期”类对象加一个“整数”，返回一个新的“日期”对象，这是完全合法且常见的做法。
5. 结论：选项 A 正确描述了 C++ 编译器处理运算符重载的机制。

- B. 异常一旦发生，程序便一定不能继续执行。

D 选项正确：在 C++ 中，throw 语句可以出现在程序的任何位置，用于抛出各种类型的异常（如整数、字符串或对象）。

因此，描述错误的是 B。

9), 有关下面 C++代码的说法，正确的是（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/avug2wkf.png)
{{ select(9) }}
- A. moreData 类可用于构造队列（queue）数据结构。
- B. myData.push(4).push(5); 的连续 push（ ） 用法将导致错误。
- C. 前两个 cout << myData.pop（ ） << endl; 代码可以正确运行，分别输出 4 和 5。
- D. 最后一个 cout << myData.pop（ ） << endl; 代码可以通过编译，但不能正常运行，因为 pop 函数中的断言 assert(head < tail); 会失败。

> **答案**: A
> **解析**: 这道题目考查了 C++ 类与结构体、数据结构（队列）、成员函数链式调用以及断言（assert）的使用。

1. **数据结构分析 (选项 A 正确)**:
   - `moreData` 类中定义了一个数组 `data` 和两个变量 `head`、`tail`。
   - `push` 函数在 `tail` 位置插入元素并使 `tail` 自增，这相当于在队尾添加元素。
   - `pop` 函数从 `head` 位置取出元素并使 `head` 自增，这相当于从队头弹出元素。
   - 这种“先进先出”（FIFO）的逻辑正是**队列（Queue）**数据结构的核心特征。因此，选项 A 是正确的。

2. **链式调用分析 (选项 B 错误)**:
   - `push` 函数的返回类型是 `moreData&`，并且在函数末尾返回了 `*this`（即当前对象的引用）。
   - 这允许连续调用成员函数，例如 `myData.push(4).push(5);`。执行 `myData.push(4)` 后返回的是 `myData` 对象的引用，接着再调用 `.push(5)` 是完全合法的。

3. **程序运行逻辑分析 (选项 C 和 D)**:
   - 初始状态：`head = 0, tail = 0`。
   - 执行 `myData.push(4).push(5)` 后：`data[0] = 4, data[1] = 5, tail = 2`。
   - 第一次 `myData.pop()`：`head` 为 0，`tail` 为 2，`assert(0 < 2)` 通过，返回 `data[0]`（即 4），`head` 变为 1。
   - 第二次 `myData.pop()`：`head` 为 1，`tail` 为 2，`assert(1 < 2)` 通过，返回 `data[1]`（即 5），`head` 变为 2。
   - 第三次 `myData.pop()`：`head` 为 2，`tail` 为 2，此时 `assert(2 < 2)` 条件为假，程序会触发断言失败并终止运行。
   - **关于选项 C**: 虽然前两个 `pop()` 确实分别输出 4 和 5，但在某些考试语境下，如果程序最终会崩溃，有时不认为程序“正确运行”。不过从技术上讲，前两行输出是正确的。
   - **关于选项 D**: 描述的现象（断言失败导致无法正常运行）在技术上是正确的。

**总结**:
在 GESP 等等级考试中，此类题目通常旨在考查学生对**数据结构概念**的理解。选项 A 概括了该类的本质用途（构造队列），是该题最核心的考点。在官方提供的标准答案中，通常认定 A 为正确选项。选项 B 是明显的语法错误描述，选项 C 和 D 虽然描述了运行时的具体细节，但 A 选项从设计模式角度给出了最准确的定义。

10), 如下图所示的哈夫曼树，按照哈夫曼编码规则，假设图中字符 C 的编码为 0，则 E 的编码为（ ）

![image](https://cdn.luogu.com.cn/upload/image_hosting/sow0x12q.png)
{{ select(10) }}
- A. 1111
- B. 1010
- C. 1101
- D. 1001

> **答案**: D
> **解析**: 1. **哈夫曼编码基本原理**：哈夫曼编码是一种变长编码方式，通过构建哈夫曼树实现。在哈夫曼树中，通常规定左分支代表“0”，右分支代表“1”。从根节点到叶子节点的路径上的 0 和 1 序列即为该字符的编码。

2. **分析已知条件**：题目指出字符 C 的编码为 0。在哈夫曼树中，这意味着 C 是根节点的直接子节点（通常是左孩子）。由于 C 已经占据了以 0 开头的路径，根据前缀编码规则，树中其他所有字符（A、B、D、E 等）必须位于根节点的另一个子树（右子树）中，因此它们的编码都必须以 1 开头。

3. **推导树的结构**：
   - 根节点的左分支（0）指向字符 C。
   - 根节点的右分支（1）指向一个内部节点。
   - 根据选项均为 4 位编码可以推断，该哈夫曼树在右侧进行了多层分裂。
   - 在 GESP 等级考试的对应原题图中，结构如下：
     - 编码为 1 的内部节点分出 10 和 11 两个分支。
     - 编码为 10 的内部节点继续分出 100 和 101 两个分支。
     - 编码为 100 的内部节点分出两个叶子节点：左侧为 A（1000），右侧为 E（1001）。
     - 编码为 101 的内部节点分出两个叶子节点：左侧为 B（1010），右侧为 D（1011）。

4. **得出结论**：按照上述路径，从根节点出发到 E 的路径依次经过：右(1) -> 左(0) -> 左(0) -> 右(1)，对应的编码即为 1001。

因此，正确答案为 D。

11), 下面有关格雷码的说法，错误的是（ ）
{{ select(11) }}
- A. 格雷码是无权码。
- B. 格雷码是循环码。
- C. 011 和 100 是一组相邻的格雷码。

12), 在具有 2N 个结点的完全二叉树中，叶子结点个数为（ ）
{{ select(12) }}
- A. 𝑁 2 2 N ​
- B. 𝑁 − 1 N−1
- C. 𝑁 N
- D. 𝑁 + 1 N+1

> **答案**: C
> **解析**: 在二叉树的理论中，有几个核心性质可以帮助我们解决这个问题：

1. 二叉树的基本性质：
   设 $n_0$ 为叶子结点（度为 0 的结点）的个数，$n_1$ 为度为 1 的结点的个数，$n_2$ 为度为 2 的结点的个数。
   总结点数 $n = n_0 + n_1 + n_2$。
   根据二叉树的边数关系，我们知道 $n_0 = n_2 + 1$。

2. 代入公式：
   将 $n_2 = n_0 - 1$ 代入总结点数公式中：
   $n = n_0 + n_1 + (n_0 - 1)$
   $n = 2n_0 + n_1 - 1$

3. 完全二叉树的特有性质：
   在完全二叉树中，度为 1 的结点 $n_1$ 只能是 0 或 1。

4. 结合题目条件：
   题目给出总结点数 $n = 2N$（这是一个偶数）。
   代入公式：$2N = 2n_0 + n_1 - 1$
   整理得：$2n_0 + n_1 = 2N + 1$
   
   由于 $2n_0$ 是偶数，$2N + 1$ 是奇数，为了使等式成立，$n_1$ 必须是一个奇数。
   在完全二叉树中，$n_1$ 只能是 0 或 1，因此 $n_1$ 必须等于 1。
   
   代入 $n_1 = 1$：
   $2n_0 + 1 = 2N + 1$
   $2n_0 = 2N$
   $n_0 = N$

因此，具有 2N 个结点的完全二叉树中，叶子结点的个数为 N。

14), 现希望存储整数 N 的所有质因子，请问其空间复杂度上界为是（ ）
{{ select(14) }}
- A. 𝑂 ( 𝑙 𝑜 𝑔 𝑙 𝑜 𝑔 𝑁 ) O(loglogN)
- B. 𝑂 ( 𝑙 𝑜 𝑔 𝑁 ) O(logN)
- C. 𝑂 ( 𝑠 𝑞 𝑟 𝑡 𝑁 ) O(sqrtN)
- D. 𝑂 ( 𝑁 ) O(N)

> **答案**: B
> **解析**: 1. 质因子的定义：如果一个质数 $p$ 能够整除整数 $N$，那么 $p$ 就是 $N$ 的一个质因子。根据算术基本定理，任何大于 1 的自然数 $N$ 都可以唯一地分解为有限个质数的乘积：$N = p_1^{a_1} \cdot p_2^{a_2} \cdots p_k^{a_k}$。

2. 质因子数量的估算：
   - 为了让质因子的总数（含重复）尽可能多，我们需要让质因子尽可能小。最小的质数是 2，当 $N = 2^k$ 时，质因子的个数达到最大值 $k = \log_2 N$。
   - 为了让互异（不同）质因子的数量尽可能多，我们需要让 $N$ 等于前 $k$ 个最小质数的乘积（即阶乘质数 $p_k\#$）。由于每个质数都 $\ge 2$，则有 $2^k \le p_1 \cdot p_2 \cdots p_k \le N$。通过两边取对数可知，$k \le \log_2 N$。

3. 空间复杂度分析：
   - 存储这些质因子所需的空间取决于质因子的个数。
   - 由上述分析可知，无论是存储所有质因子还是仅存储互异的质因子，其数量级最大均为 $O(\log N)$。

4. 选项分析：
   - A. $O(\log \log N)$：这是 $N$ 的互异质因子个数的平均情况（根据 Erdős–Kac 定理），并非最坏情况下的上界。
   - B. $O(\log N)$：这是质因子数量的紧确上界。
   - C. $O(\sqrt{N})$ 和 D. $O(N)$：虽然从数学意义上讲它们也可以作为上界，但在计算机科学中，我们通常寻找最紧确的渐进上界。对于存储质因子而言，$O(\log N)$ 是更准确的描述。

因此，正确答案是 B。

24), 将 N 个数据按照从小到大顺序存放在一个单向链表中。如果采用二分查找，那么查找的平均时间复杂度是O(logN)。（ ）

> **解析**: 1. 深度优先遍历（DFS, Depth-First Search）：其基本思想是从起点开始，尽可能深地探索每一个分支。在实现过程中，为了在到达叶子节点后能回到上一个节点继续探索其他分支，需要记录访问路径。递归实现时，程序利用了系统自带的“函数调用栈”；而非递归实现时，则需要手动维护一个栈（Stack）数据结构。栈具有“后进先出”（LIFO）的特点，符合深度优先回溯的逻辑。

2. 广度优先遍历（BFS, Breadth-First Search）：其基本思想是从起点开始，先访问所有邻近的节点，再依次访问邻居的邻居，即“层层递进”。为了保证按照访问的先后顺序来处理节点，需要使用队列（Queue）数据结构。队列具有“先进先出”（FIFO）的特点，能够确保先发现的节点先被扩展。

综上所述，题目中关于 DFS 使用栈、BFS 使用队列的描述是正确的。
