1), 一个 32 位整型变量占用（ ）个字节。
{{ select(1) }}
- A. 4
- B. 8
- C. 32
- D. 128

> **答案**: A
> **解析**: 在计算机科学中，存储单位的换算关系是：1 字节（Byte）等于 8 位（bit）。
题目中提到的“32 位整型变量”指的是该变量在内存中占据 32 个二进制位（bits）。
计算其占用的字节数，公式为：位数 ÷ 8 = 字节数。
即：32 ÷ 8 = 4 字节。

因此，一个 32 位整型变量占用 4 个字节。常见的 C++ 类型 `int` 在大多数现代编译器中就是 32 位的，占用 4 字节。

2), 二进制数 11.01 在十进制下是（ ）
{{ select(2) }}
- A. 3.25
- B. 4.125
- C. 6.25
- D. 11.125

> **答案**: A
> **解析**: 将二进制数转换为十进制数，需要使用“按权展开法”。二进制数的每一位都有对应的权值，以小数点为界：
1. 整数部分：从右向左（从小数点往左），权值依次为 2^0, 2^1, 2^2...
2. 小数部分：从左向右（从小数点往右），权值依次为 2^-1, 2^-2, 2^-3...

对于二进制数 11.01：
- 整数部分 (11)：
  第1位是 1，对应权值为 2^1 = 2
  第0位是 1，对应权值为 2^0 = 1
  整数部分之和 = 1 × 2 + 1 × 1 = 3

- 小数部分 (.01)：
  小数点后第1位是 0，对应权值为 2^-1 = 0.5
  小数点后第2位是 1，对应权值为 2^-2 = 0.25
  小数部分之和 = 0 × 0.5 + 1 × 0.25 = 0.25

将整数部分和小数部分相加：3 + 0.25 = 3.25。

因此，二进制数 11.01 转换为十进制数是 3.25。正确答案为 A。

3), 下面的故事与（ ）算法有着异曲同工之妙。从前有座山，山里有座庙，庙里有个老和尚在给小和尚讲故事：“从前有座山，山里有座庙，庙里有个老和尚在给小和尚讲故事：‘从前有座山，山里有座庙，庙里有个老和尚给小和尚讲故事……’”
{{ select(3) }}
- A. 枚举
- B. 递归
- C. 贪心
- D. 分治

> **答案**: B
> **解析**: 递归（Recursion）是计算机科学中的一种重要算法思想，其核心特征是一个函数或过程在其定义中直接或间接地调用自身。

题目中描述的故事：“从前有座山，山里有座庙，庙里有个老和尚在给小和尚讲故事：‘从前有座山……’”，这个故事的内容又包含了讲这个故事本身。这种“故事里套着相同故事”的结构，完美契合了递归定义中“自我调用”的特点。

对比其他选项：
A. 枚举：指按顺序逐一列举所有可能的情况，与自我调用无关。
C. 贪心：指在每一步选择中都采取当前状态下的最优决策，不涉及自我嵌套。
D. 分治：指将大问题分解为小问题解决，虽然分治常通过递归实现，但题目描述的这种纯粹的自我重复结构最直接对应的术语是递归。

因此，正确答案是 B。

5), 已知一棵二叉树有 2013 个节点，则其中至多有（ ）个节点有 2 个子节点。
{{ select(5) }}
- A. 1006
- B. 1007
- C. 1023
- D. 1024

> **答案**: A
> **解析**: 本题考查二叉树的基本性质。

1. 在任何一棵二叉树中，设 $n_0$ 为叶子节点（度为 0 的节点）的数量，$n_1$ 为度为 1 的节点的数量，$n_2$ 为度为 2 的节点的数量。
2. 二叉树有一个重要的性质：$n_0 = n_2 + 1$。即叶子节点的数量总是比度为 2 的节点数量多 1。
3. 树的总节点数 $n$ 可以表示为：$n = n_0 + n_1 + n_2$。
4. 将性质 $n_0 = n_2 + 1$ 代入总节点数公式中，得到：
   $n = (n_2 + 1) + n_1 + n_2$
   $n = 2n_2 + n_1 + 1$
5. 题目已知总节点数 $n = 2013$，代入公式：
   $2013 = 2n_2 + n_1 + 1$
   $2012 = 2n_2 + n_1$
6. 题目要求 $n_2$（度为 2 的节点）的最大值。为了让 $n_2$ 最大，我们需要让 $n_1$ 尽可能小。在二叉树中，$n_1$ 的最小值是 0。
7. 当 $n_1 = 0$ 时：
   $2n_2 = 2012$
   $n_2 = 1006$

此时，$n_0 = 1007$，$n_1 = 0$，$n_2 = 1006$，总节点数为 $1007 + 0 + 1006 = 2013$，符合题意。因此，度为 2 的节点至多有 1006 个。

6), 在一个无向图中，如果任意两点之间都存在路径相连，则称其为连通图。右图是一个有 5 个顶点、 8 条边的连通图。若要使它不再是连通图，至少要删去其中的（ ）条边

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/34.png)
{{ select(6) }}
- A. 2
- B. 3
- C. 4
- D. 5

> **答案**: B
> **解析**: 1. **连通图的概念**：在无向图中，如果任意两个顶点之间都存在路径相连，则称该图为连通图。要使图不再连通，只需通过删边操作，使得图中至少存在两个顶点之间没有路径即可（例如孤立出一个顶点）。

2. **图的属性分析**：
   - 顶点数 $V = 5$
   - 边数 $E = 8$
   - 在一个有 5 个顶点的图中，完全图（每两个点之间都有边）的边数为 $C(5, 2) = (5 \times 4) / 2 = 10$ 条。
   - 当前图有 8 条边，说明它只比完全图少了 2 条边。

3. **度数与连通性**：
   - 根据握手定理，所有顶点的度数之和等于边数的两倍：$\sum deg(v) = 2 \times E = 2 \times 8 = 16$。
   - 平均度数为 $16 \div 5 = 3.2$。
   - 这意味着图中一定存在度数（连接该点的边数）小于或等于 3 的顶点。因为如果所有顶点的度数都 $\ge 4$，则总边数至少需要 $(5 \times 4) / 2 = 10$ 条。

4. **最小删边数（边连通度）**：
   - 要使一个图不再连通，最直接的方法是删去与某个顶点相连的所有边，使其成为孤立点。
   - 在 5 个顶点、8 条边的连通图中，最小度数 $\delta(G)$ 最大可能为 3（例如度数分布为 4, 3, 3, 3, 3）。
   - 只要删去与该顶点相连的 3 条边，该顶点就会变成孤立点，图便不再连通。
   - 经观察或推导，对于 5 点 8 边的结构，不存在只删 2 条边就能使图不连通的情况（除非图中原本就存在度数为 2 的点，但通常此类题目给出的标准连通图结构较为稳固，最小度为 3）。

因此，至少要删去 3 条边才能使该图不再连通。

7), 斐波那契数列的定义如下：F1​=1,F2​=1,Fn​=Fn−1​+Fn−2​(n≥3)。如果用下面的函数计算斐波那契数列的第 n 项，则其时间复杂度为（ ）
{{ select(7) }}
- A. 𝑂 ( 1 ) O(1)
- B. 𝑂 ( 𝑛 ) O(n)
- C. 𝑂 ( 𝑛 2 ) O(n 2 )
- D. 𝑂 ( 𝐹 𝑛 ) O(F n ​ )

> **答案**: D
> **解析**: 1. 题目描述的是斐波那契数列的定义。在编程考级（如 GESP/CSP）中，如果没有给出具体代码但要求分析复杂度，通常是指代最基础的递归实现方式：
   ```cpp
   int fib(int n) {
       if (n <= 2) return 1;
       return fib(n - 1) + fib(n - 2);
   }
   ```
2. 分析递归调用的次数：设计算第 n 项的时间复杂度为 T(n)，则有 $T(n) = T(n-1) + T(n-2) + O(1)$。这个递推公式的形式与斐波那契数列本身的定义 $F_n = F_{n-1} + F_{n-2}$ 非常相似。
3. 实际上，计算 $F_n$ 所需的递归调用总次数恰好为 $2 \times F_n - 1$ 次。
4. 因此，该函数的时间复杂度与斐波那契数列的数值 $F_n$ 成正比，即 $O(F_n)$。
5. 知识点补充：
   - 斐波那契数列的数值 $F_n$ 随 n 的增加呈指数级增长（约等于 $1.618^n$），因此递归实现效率极低。
   - 如果使用循环（迭代）法计算，时间复杂度为 $O(n)$。
   - 如果使用带有记忆化的递归（动态规划），时间复杂度为 $O(n)$。
   - 如果使用矩阵快速幂，时间复杂度可以达到 $O(\log n)$。
   - 本题选项中 D 最符合递归实现的复杂度特征。

8), 二叉查找树具有如下性质：每个节点的值都大于其左子树上所有节点的值、小于其右子 树上所有节点的值。那么，二叉查找树的（ ）是一个有序序列。
{{ select(8) }}
- A. 先序遍历
- B. 中序遍历
- C. 后序遍历
- D. 宽度优先遍历

> **答案**: B
> **解析**: 1. 二叉查找树（Binary Search Tree，简称 BST）的定义性质是：对于树中的任意节点，其左子树中所有节点的值都小于该节点的值，而其右子树中所有节点的值都大于该节点的值。
2. 中序遍历（In-order Traversal）的访问顺序是：左子树 -> 根节点 -> 右子树。
3. 结合 BST 的性质：
   - 先访问左子树（所有值都比根节点小）；
   - 再访问根节点；
   - 最后访问右子树（所有值都比根节点大）。
   这种访问顺序在递归过程中会确保所有节点按从小到大的顺序被访问。
4. 其他遍历方式：
   - 先序遍历（根-左-右）：根节点最先访问，不保证有序。
   - 后序遍历（左-右-根）：根节点最后访问，不保证有序。
   - 宽度优先遍历（层序遍历）：按树的深度逐层访问，不保证有序。

因此，二叉查找树的中序遍历序列一定是一个升序的有序序列。

9), 将(2,6,10,17)分别存储到某个地址区间为0∼10的哈希表中，如果哈希函数h(x)=（ ），将不会产生冲突，其中 amodb 表示 a 除以 b 的余数。
{{ select(9) }}
- A. 𝑥 mod 1 1 xmod11
- B. 𝑥 2 mod 1 1 x 2 mod11
- C. ( 2 𝑥 ) mod 1 1 (2x)mod11
- D. ⌊ 𝑥 ⌋ mod 1 1 ⌊ x ​ ⌋mod11，其中 ⌊ 𝑥 ⌋ ⌊ x ​ ⌋ 表示 𝑥 x ​ 下取整

> **答案**: D
> **解析**: 本题考查哈希表（Hash Table）中哈希函数的设计以及冲突（Collision）的概念。哈希冲突是指不同的键值通过哈希函数计算得到了相同的哈希地址。我们需要将 (2, 6, 10, 17) 这四个数映射到 0~10 的地址空间中，且不产生冲突。

我们逐一验证各选项：

A. $h(x) = x \pmod{11}$
- $h(2) = 2 \pmod{11} = 2$
- $h(6) = 6 \pmod{11} = 6$
- $h(10) = 10 \pmod{11} = 10$
- $h(17) = 17 \pmod{11} = 6$
由于 $h(6) = h(17) = 6$，产生冲突。

B. $h(x) = x^2 \pmod{11}$
- $h(2) = 2^2 \pmod{11} = 4$
- $h(6) = 6^2 \pmod{11} = 36 \pmod{11} = 3$
- $h(10) = 10^2 \pmod{11} = 100 \pmod{11} = 1$
- $h(17) = 17^2 \pmod{11} = 289 \pmod{11} = 3$ （因为 $289 = 11 \times 26 + 3$）
由于 $h(6) = h(17) = 3$，产生冲突。

C. $h(x) = (2x) \pmod{11}$
- $h(2) = (2 \times 2) \pmod{11} = 4$
- $h(6) = (2 \times 6) \pmod{11} = 12 \pmod{11} = 1$
- $h(10) = (2 \times 10) \pmod{11} = 20 \pmod{11} = 9$
- $h(17) = (2 \times 17) \pmod{11} = 34 \pmod{11} = 1$ （因为 $34 = 11 \times 3 + 1$）
由于 $h(6) = h(17) = 1$，产生冲突。

D. $h(x) = \lfloor \sqrt{x} \rfloor \pmod{11}$ （根据题目 LaTeX 描述，此处应为 $x$ 的平方根下取整）
- $h(2) = \lfloor \sqrt{2} \rfloor \pmod{11} = \lfloor 1.414 \rfloor = 1$
- $h(6) = \lfloor \sqrt{6} \rfloor \pmod{11} = \lfloor 2.449 \rfloor = 2$
- $h(10) = \lfloor \sqrt{10} \rfloor \pmod{11} = \lfloor 3.162 \rfloor = 3$
- $h(17) = \lfloor \sqrt{17} \rfloor \pmod{11} = \lfloor 4.123 \rfloor = 4$
四个结果 $\{1, 2, 3, 4\}$ 互不相同，且都在 0~10 的范围内，因此不会产生冲突。

综上所述，正确答案为 D。

10), IPv4 协议使用 32 位地址，随着其不断被分配，地址资源日趋枯竭。因此，它正逐渐被 使用（ ）位地址的 IPv6 协议所取代。

因此，正确答案是 D。

11), 二分图是指能将顶点划分成两个部分，每一部分内的顶点间没有边相连的简单无向图。 那么， 12 个顶点的二分图至多有（ ）条边。
{{ select(11) }}
- A. 18
- B. 24
- C. 36
- D. 66

> **答案**: C
> **解析**: 1. 二分图定义：二分图（Bipartite Graph）是指图中的顶点集可以分割为两个互不相交的子集 $U$ 和 $V$，使得图中的每一条边所连接的两个顶点分别属于这两个不同的顶点集。

2. 最大边数分析：
   - 设二分图的两个顶点集的大小分别为 $n_1$ 和 $n_2$。
   - 根据题目条件，总顶点数 $n = n_1 + n_2 = 12$。
   - 在二分图中，边只能存在于 $U$ 集合和 $V$ 集合的顶点之间。为了使边数最多，我们应该构建一个“完全二分图”，即 $U$ 中的每一个顶点都与 $V$ 中的每一个顶点相连。
   - 此时，总边数 $E = n_1 \times n_2$。

3. 数学计算：
   - 我们需要求在 $n_1 + n_2 = 12$ 的条件下，$n_1 \times n_2$ 的最大值。
   - 根据算术几何平均值不等式（或二次函数的性质），当两个数越接近时，它们的乘积越大。
   - 当 $n_1 = 6, n_2 = 6$ 时，$n_1 \times n_2 = 6 \times 6 = 36$。
   - 如果取其他组合，如 $5 \times 7 = 35$，$4 \times 8 = 32$ 等，结果都小于 36。

因此，12 个顶点的二分图至多有 36 条边。正确答案为 C。

13), 把 64 位非零浮点数强制转换成 32 位浮点数后，不可能（ ）
{{ select(13) }}
- A. 大于原数
- B. 小于原数
- C. 等于原数
- D. 与原数符号相反

> **答案**: D
> **解析**: 1. 浮点数结构：根据 IEEE 754 标准，无论是 64 位浮点数（double）还是 32 位浮点数（float），其二进制表示都由三部分组成：符号位（Sign）、指数位（Exponent）和尾数位（Mantissa）。
2. 符号位保留：在进行强制类型转换时，最高位的符号位会被直接保留。如果原数是正数，转换后仍为正数（或正无穷、正零）；如果原数是负数，转换后仍为负数（或负无穷、负零）。因此，转换后的数值符号绝对不可能与原数相反。
3. 精度与舍入（A、B 选项）：由于 32 位浮点数的尾数精度（23 位）远低于 64 位浮点数（52 位），在转换过程中通常会发生舍入。根据舍入规则（如“就近舍入”），转换后的数值可能会比原数稍微大一点，也可能稍微小一点。
4. 精确转换（C 选项）：如果 64 位浮点数所表示的数值恰好在 32 位浮点数的表示范围内，且其尾数部分在 23 位以内就能精确表示（例如整数 1.0 或 0.5 等），那么转换后的数值将与原数完全相等。
5. 极端情况：如果 64 位浮点数数值过大，超过了 32 位浮点数的最大表示范围，会转换成无穷大（Infinity）；如果数值过小，会转换成 0。但无论哪种情况，符号位都不会改变。

综上所述，不可能出现的情况是与原数符号相反。

14), 对一个 n 个顶点、 m 条边的带权有向简单图用 Dijkstra 算法计算单源最短路时，如果不使用堆或其它优先队列进行优化，则其时间复杂度为（ ）
{{ select(14) }}
- A. 𝑂 ( 𝑚 𝑛 + 𝑛 3 ) O(mn+n 3 )
- B. 𝑂 ( 𝑛 2 ) O(n 2 )
- C. 𝑂 ( ( 𝑚 + 𝑛 ) log ⁡ 𝑛 ) O((m+n)logn)
- D. 𝑂 ( ( 𝑚 + 𝑛 2 ) log ⁡ 𝑛 ) O((m+n 2 )logn)

> **答案**: B
> **解析**: Dijkstra 算法用于求解单源最短路径问题。在不使用堆或优先队列进行优化的情况下，算法的基本步骤如下：

1. 初始化：将源点到所有顶点的距离初始化为无穷大，源点自身距离为 0。这一步复杂度为 O(n)。
2. 循环执行以下操作 n 次（每个顶点处理一次）：
   a. 在所有未访问的顶点中，寻找距离源点最近的顶点 u。由于没有使用堆优化，需要遍历所有顶点来寻找最小值，这一步的时间复杂度为 O(n)。
   b. 将顶点 u 标记为已访问。
   c. 遍历从顶点 u 出发的所有边 (u, v)，尝试更新顶点 v 的最短距离（松弛操作）。在整个算法执行过程中，每条边最多被访问一次。

复杂度计算：
- 寻找最小距离顶点的总时间：n 次操作 × O(n) = O(n²)。
- 松弛操作的总时间：所有边的总数，即 O(m)。
- 总复杂度：O(n² + m)。

由于在简单图中，边的数量 m 最大为 n(n-1)，即 m = O(n²)，因此总的时间复杂度可以简化为 O(n²)。

补充：
- 如果使用二叉堆优化，复杂度为 O((n + m) log n)。
- 如果使用 Fibonacci 堆优化，复杂度可以达到 O(m + n log n)。
- 本题明确要求“不使用堆或其它优先队列”，故选 B。

15), T(n)表示某个算法输入规模为 n 时的运算次数。如果T(1)为常数，且有递归式T(n)=2×T(2n​)+2n，那么T(n)=（ ）
{{ select(15) }}
- A. Θ ( 𝑛 ) Θ(n)
- B. Θ ( 𝑛 log ⁡ 𝑛 ) Θ(nlogn)
- C. Θ ( 𝑛 2 ) Θ(n 2 )
- D. Θ ( 𝑛 2 log ⁡ 𝑛 ) Θ(n 2 logn)

> **答案**: B
> **解析**: 这道题可以通过主定理（Master Theorem）或者代入递归展开法来求解。

1. 使用主定理求解：
递归式的一般形式为：T(n) = aT(n/b) + f(n)
在本题中：
- a = 2（子问题的个数）
- b = 2（子问题的规模缩小的倍数）
- f(n) = 2n（合并子问题所需的额外运算次数）

首先计算 n^(log_b a)：
n^(log_2 2) = n^1 = n

接下来比较 f(n) 与 n^(log_b a)：
因为 f(n) = 2n，且 n^(log_b a) = n，所以 f(n) = Θ(n)。
这符合主定理的第二种情况：当 f(n) = Θ(n^(log_b a)) 时，T(n) = Θ(n^(log_b a) * log n)。

因此，T(n) = Θ(n * log n)。

2. 使用递归展开法验证：
T(n) = 2 * T(n/2) + 2n
T(n) = 2 * [2 * T(n/4) + 2 * (n/2)] + 2n = 4 * T(n/4) + 2n + 2n = 4 * T(n/4) + 4n
T(n) = 4 * [2 * T(n/8) + 2 * (n/4)] + 4n = 8 * T(n/8) + 2n + 4n = 8 * T(n/8) + 6n
以此类推，经过 k 次展开后：
T(n) = 2^k * T(n / 2^k) + 2kn

令 n / 2^k = 1，则 k = log_2 n。
代入上式：
T(n) = n * T(1) + 2n * log_2 n
由于 T(1) 是常数，上式简化为 T(n) = C*n + 2n log n。
在渐进时间复杂度中，忽略常数项和低阶项，得到 T(n) = Θ(n log n)。

综上所述，正确选项为 B。

16), 下列程序中，正确计算1,2,…,100这 100 个自然数之和 sum （初始值为 0）的是（ ）
{{ select(16) }}
- A. for (i = 1; i <= 100; i++) sum += i;
- B. i = 1; while (i > 100) { sum += i; i++;}
- C. i = 1; do { sum += i; i++; } while (i <= 100);
- D. i = 1; do { sum += i; i++; } while (i > 100);

> **答案**: A
> **解析**: 本题考查的是循环结构（for、while、do-while）在实现累加求和逻辑时的应用。我们需要计算 1 到 100 的自然数之和。

1. **选项 A 分析**：
   `for (i = 1; i <= 100; i++) sum += i;`
   - **初始化**：`i` 从 1 开始。
   - **循环条件**：`i <= 100`，意味着循环会一直执行到 `i` 等于 100 为止。
   - **累加过程**：每次循环将当前的 `i` 加到 `sum` 中。
   - **更新**：每次循环结束后 `i` 自增 1。
   - **结果**：当 `i` 变为 101 时退出循环，此时 `sum` 已经累加了 1, 2, ..., 100。逻辑完全正确。

2. **选项 B 分析**：
   `i = 1; while (i > 100) { sum += i; i++;}`
   - **初始化**：`i = 1`。
   - **循环条件**：`i > 100`。由于初始值 `i = 1` 并不大于 100，条件从一开始就不成立，循环体一次也不会执行。
   - **结果**：`sum` 保持初始值 0，计算错误。

3. **选项 C 分析**：
   `i = 1; do { sum += i; i++; } while (i <= 100);`
   - **初始化**：`i = 1`。
   - **执行过程**：`do...while` 循环会先执行一次循环体。
   - **累加过程**：先加 1，`i` 变 2，判断 `2 <= 100` 为真，继续循环……直到 `i` 为 100 时，执行 `sum += 100`，`i` 变为 101，判断 `101 <= 100` 为假，退出。
   - **结果**：逻辑上也是正确的。但在单选题中，对于已知次数的计数循环，`for` 循环（选项 A）是最标准、最简洁的写法。

4. **选项 D 分析**：
   `i = 1; do { sum += i; i++; } while (i > 100);`
   - **执行过程**：先执行一次循环体，`sum` 变为 1，`i` 变为 2。
   - **循环条件**：判断 `i > 100`（即 `2 > 100`），结果为假，循环立即停止。
   - **结果**：只加了数字 1，计算错误。

**总结**：
虽然选项 A 和 C 在逻辑上都能完成任务，但在编程规范和考题逻辑中，`for` 循环是处理固定范围累加的首选结构。在绝大多数此类编程题目中，A 选项被视为标准答案。

17), （ ）的平均时间复杂度为O(nlogn)，其中 n 是待排序的元素个数。
{{ select(17) }}
- A. 快速排序
- B. 插入排序
- C. 冒泡排序
- D. 归并排序

> **答案**: A
> **解析**: 在计算机科学中，排序算法的时间复杂度是衡量算法效率的重要指标。针对题目中的选项，我们逐一分析其平均时间复杂度：

1.  **A. 快速排序 (Quick Sort)**：采用分治法的思想。其平均时间复杂度为 **O(n log n)**。虽然在最坏情况下（如待排序序列已基本有序或基本逆序时）会退化到 O(n²)，但在随机情况下的表现非常优秀，是实际应用中最常用的排序算法之一。
2.  **B. 插入排序 (Insertion Sort)**：通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。其平均时间复杂度为 **O(n²)**，最好情况下（序列已有序）为 O(n)。

**总结：**

*   若选项确实如题所示，则 A 和 D 均符合题意。在单选环境下，快速排序（A）通常是此类题目中最具代表性的答案，因为它强调了“平均”这一概念（与其最坏情况 O(n²) 形成对比）。

19), （ ）属于 NP 类问题。
{{ select(19) }}
- A. 存在一个 P 类问题
- B. 任何一个 P 类问题
- C. 任何一个不属于 P 类的问题
- D. 任何一个在（输入规模的）指数时间内能够解决的问题

> **答案**: B
> **解析**: 在计算复杂性理论中，P 类问题和 NP 类问题的定义及关系如下：

1. **P 类问题 (Polynomial time)**：指可以在多项式时间内“解决”的问题。
2. **NP 类问题 (Nondeterministic Polynomial time)**：指可以在多项式时间内“验证”一个解的正确性的问题。
3. **二者的关系**：如果一个问题可以在多项式时间内被解决（属于 P 类），那么我们一定可以在多项式时间内验证它的解（只需亲自算一遍并对比结果）。因此，所有的 P 类问题都属于 NP 类问题，即 $P \subseteq NP$。

**选项分析：**
- **A 选项**：虽然存在 P 类问题属于 NP，但表述不够完整。
- **B 选项**：正确。因为 $P$ 是 $NP$ 的子集，所以“任何一个” P 类问题都必然属于 NP 类。
- **C 选项**：错误。不属于 P 类的问题可能属于 NP（如著名的 NP 完全问题），但也可能完全超出 NP 的范围（如不可判定问题或需要指数级时间才能验证的问题）。
- **D 选项**：错误。指数时间内解决的问题属于 EXP 类。虽然 NP 问题都可以在指数时间内解决（$NP \subseteq EXP$），但反过来并不成立，有些指数级问题是无法在多项式时间内验证的。

因此，正确答案是 B。
