1), 一个 32 位整型变量占用（ ）个字节。
{{ select(1) }}
- A. 4
- B. 8
- C. 32
- D. 128

> **答案**: A
> **解析**: 在计算机科学中，存储单位的换算关系如下：
1. 位（bit）：是计算机中最小的数据单位，每一位只能是 0 或 1。
2. 字节（Byte）：是计算机存储容量的基本单位。
3. 换算关系：1 字节 (Byte) = 8 位 (bit)。

题目中指出该整型变量是“32 位”，要求计算它占用的“字节”数。
计算过程为：32 ÷ 8 = 4。

因此，一个 32 位整型变量占用 4 个字节。常见的 C++ 类型 `int` 在大多数现代编译器中就是 32 位的，占用 4 字节。

2), 二进制数 11.01 在十进制下是（ ）
{{ select(2) }}
- A. 3.25
- B. 4.125
- C. 6.25
- D. 11.125

> **答案**: A
> **解析**: 将二进制数转换为十进制数，需要使用“按权展开法”。二进制数的每一位都有对应的权值，以小数点为界：
1. 小数点左侧（整数部分）从右往左的权值依次为 2^0, 2^1, 2^2...
2. 小数点右侧（小数部分）从左往右的权值依次为 2^-1, 2^-2, 2^-3...

对于二进制数 11.01：
- 整数部分：
  第1位是 1，权值为 2^1 = 2
  第0位是 1，权值为 2^0 = 1
  整数部分之和 = 1 × 2 + 1 × 1 = 3

- 小数部分：
  小数点后第1位是 0，权值为 2^-1 = 0.5
  小数点后第2位是 1，权值为 2^-2 = 0.25
  小数部分之和 = 0 × 0.5 + 1 × 0.25 = 0.25

将两部分相加：3 + 0.25 = 3.25。
因此，二进制数 11.01 对应的十进制数是 3.25。正确答案为 A。

4), 逻辑表达式（ ）的值与变量 A 的真假无关。
{{ select(4) }}
- A. (A ∨ B) ∧﹃A
- B. (A ∨ B) ∧﹃B
- C. (A ∧ B) ∨ (﹃ A ∧ B)
- D. (A ∨ B) ∧﹃A ∧ B

> **答案**: C
> **解析**: 要判断逻辑表达式的值是否与变量 A 的真假无关，我们需要对每个选项进行逻辑化简，看化简后的结果是否还包含变量 A。

我们使用逻辑代数中的分配律和化简公式：
1. 分配律：(A ∧ B) ∨ (A ∧ C) = A ∧ (B ∨ C)；(A ∨ B) ∧ (A ∨ C) = A ∨ (B ∧ C)
2. 互补律：A ∨ ﹃A = 真 (1)；A ∧ ﹃A = 假 (0)
3. 同一律：A ∧ 1 = A；A ∨ 0 = A

逐一分析选项：

A. (A ∨ B) ∧ ﹃A
利用分配律展开：(A ∧ ﹃A) ∨ (B ∧ ﹃A)
根据互补律：0 ∨ (B ∧ ﹃A) = B ∧ ﹃A
该结果与 A 的取值有关（当 B 为真时，结果取决于 ﹃A）。

B. (A ∨ B) ∧ ﹃B
利用分配律展开：(A ∧ ﹃B) ∨ (B ∧ ﹃B)
根据互补律：(A ∧ ﹃B) ∨ 0 = A ∧ ﹃B
该结果与 A 的取值有关（当 B 为假时，结果取决于 A）。

C. (A ∧ B) ∨ (﹃A ∧ B)
利用分配律提取公因子 B：(A ∨ ﹃A) ∧ B
根据互补律：1 ∧ B = B
化简结果为 B。可以看到，无论 A 是真还是假，表达式的值始终等于 B 的值。因此，该表达式的值与 A 的真假无关。

D. (A ∨ B) ∧ ﹃A ∧ B
先化简前两项（同选项 A）：(B ∧ ﹃A) ∧ B
利用交换律和结合律：(B ∧ B) ∧ ﹃A = B ∧ ﹃A
该结果与 A 的取值有关。

综上所述，只有选项 C 的值与 A 无关。

5), 将{2,6,10,17}分别存储到某个地址区间为0∼10的哈希表中，如果哈希函数h(x)=（ ），将不会产生冲突，其中 amodb 表示 a 除以 b 的余数。
{{ select(5) }}
- A. 𝑥 mod 1 1 xmod11
- B. 𝑥 2 mod 1 1 x 2 mod11
- C. ( 2 𝑥 ) mod 1 1 (2x)mod11
- D. ⌊ 𝑥 ⌋ mod 1 1 ⌊ x ​ ⌋mod11，其中 ⌊ 𝑥 ⌋ ⌊ x ​ ⌋ 表示 𝑥 x ​ 下取整

> **答案**: D
> **解析**: 1. **分析题目要求**：
   我们需要将集合 {2, 6, 10, 17} 存储到地址区间为 0∼10 的哈希表中。这意味着哈希函数的结果必须在 0 到 10 之间，且对于这四个不同的输入值，哈希函数的结果必须互不相同（即不产生冲突）。

2. **关键观察**：
   注意到在集合中，$17 \equiv 6 \pmod{11}$。
   对于选项 A、B、C，它们的哈希函数形式都是先对 $x$ 进行某种运算再对 11 取模。由于 $17$ 和 $6$ 对 $11$ 同余，那么任何只依赖于 $x \pmod{11}$ 的函数 $f(x \pmod{11})$ 都会使得 $f(6)$ 和 $f(17)$ 相等。

3. **逐项验证**：
   - **A 选项**：$h(x) = x \pmod{11}$
     $h(2) = 2$
     $h(6) = 6$
     $h(10) = 10$
     $h(17) = 17 \pmod{11} = 6$
     结果中 $h(6) = h(17) = 6$，产生冲突。

   - **B 选项**：$h(x) = x^2 \pmod{11}$
     $h(2) = 4$
     $h(6) = 36 \pmod{11} = 3$
     $h(10) = 100 \pmod{11} = 1$
     $h(17) = 289 \pmod{11} = 3$ （或者利用 $17 \equiv 6$，则 $17^2 \equiv 6^2 \equiv 3$）
     结果中 $h(6) = h(17) = 3$，产生冲突。

   - **C 选项**：$h(x) = 2x \pmod{11}$
     $h(2) = 4$
     $h(6) = 12 \pmod{11} = 1$
     $h(10) = 20 \pmod{11} = 9$
     $h(17) = 34 \pmod{11} = 1$ （或者利用 $17 \equiv 6$，则 $2 \times 17 \equiv 2 \times 6 \equiv 1$）
     结果中 $h(6) = h(17) = 1$，产生冲突。

   - **D 选项**：$h(x) = \lfloor \sqrt{x} \rfloor \pmod{11}$
     $h(2) = \lfloor \sqrt{2} \rfloor = \lfloor 1.414 \rfloor = 1$
     $h(6) = \lfloor \sqrt{6} \rfloor = \lfloor 2.449 \rfloor = 2$
     $h(10) = \lfloor \sqrt{10} \rfloor = \lfloor 3.162 \rfloor = 3$
     $h(17) = \lfloor \sqrt{17} \rfloor = \lfloor 4.123 \rfloor = 4$
     结果为 {1, 2, 3, 4}，四个地址各不相同，没有冲突。

4. **结论**：
   只有选项 D 的哈希函数能够将这四个数映射到不同的地址。

6), 在十六进制表示法中，字母 A 相当于十进制中的（ ）
{{ select(6) }}
- A. 9
- B. 10
- C. 15
- D. 16

> **答案**: B
> **解析**: 十六进制（Hexadecimal）是一种逢 16 进 1 的进位计数制。在十六进制中，为了表示 10 到 15 这六个数字，使用了字母 A 到 F 来代替：
- 0-9 保持不变，分别对应十进制的 0-9；
- 字母 A 对应十进制的 10；
- 字母 B 对应十进制的 11；
- 字母 C 对应十进制的 12；
- 字母 D 对应十进制的 13；
- 字母 E 对应十进制的 14；
- 字母 F 对应十进制的 15。

因此，字母 A 在十进制中相当于 10。正确答案为 B。

7), 下图中所使用的数据结构是（ ）

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/10.png)
{{ select(7) }}
- A. 哈希表
- B. 栈
- C. 队列
- D. 二叉树

> **答案**: B
> **解析**: 在计算机科学和 GESP/CSP 等等级考试中，这类题目通常会给出一个数据结构的示意图。根据选项和常见的考试题目，解析如下：

1. **栈 (Stack)**：
   - **图形特征**：通常表现为一个一端封闭、一端开口的容器（像一个桶或垂直的管子）。
   - **操作特性**：数据只能从开口的一端进入（入栈 Push）和弹出（出栈 Pop）。
   - **核心原则**：遵循“后进先出”（LIFO, Last In First Out）的原则。即最后存入的数据最先被取出。

2. **其他选项对比**：
   - **队列 (Queue)**：通常表现为两端开口的管道，数据从一端进（入队 Rear），从另一端出（出队 Front），遵循“先进先出”（FIFO）原则。
   - **二叉树 (Binary Tree)**：表现为层级结构，有一个根节点，每个节点最多有两个子节点（左孩子和右孩子），形状像倒挂的树。
   - **哈希表 (Hash Table)**：表现为键（Key）与值（Value）的映射关系，通常通过一个哈希函数将数据映射到数组的特定索引位置。

**总结**：如果题目中的图形显示数据只能从一端进出，或者明确标注了“栈顶”（Top）等字样，那么该数据结构就是**栈**。在 GESP 等级考试的经典题库中，此题图示多为垂直容器结构，故选 B。

9), 已知一棵二叉树有 10 个节点，则其中至多有（ ）个节点有 2 个子节点。
{{ select(9) }}
- A. 4
- B. 5
- C. 6
- D. 7

> **答案**: A
> **解析**: 这道题考察的是二叉树的基本性质。

1. 二叉树的节点分类：
   设 $n_0$ 为度为 0 的节点（叶子节点）数量。
   设 $n_1$ 为度为 1 的节点数量。
   设 $n_2$ 为度为 2 的节点数量。
   总节点数 $n = n_0 + n_1 + n_2$。

2. 二叉树的核心性质：
   在任何一棵二叉树中，度为 0 的节点数总是比度为 2 的节点数多 1，即：$n_0 = n_2 + 1$。

3. 建立方程：
   将性质代入总节点数公式：
   $n = (n_2 + 1) + n_1 + n_2$
   $n = 2n_2 + n_1 + 1$

4. 代入已知条件：
   已知 $n = 10$，则有：
   $10 = 2n_2 + n_1 + 1$
   $9 = 2n_2 + n_1$

5. 求解最大值：
   题目要求 $n_2$ 的最大值。为了让 $n_2$ 最大，我们需要让 $n_1$ 尽可能小。
   - 如果 $n_1 = 0$，则 $2n_2 = 9$，$n_2 = 4.5$（节点数必须为整数，舍去）。
   - 如果 $n_1 = 1$，则 $2n_2 = 8$，$n_2 = 4$。此时 $n_0 = n_2 + 1 = 5$。
   验证：$n_0 + n_1 + n_2 = 5 + 1 + 4 = 10$，符合题意。

因此，度为 2 的节点至多有 4 个。正确答案为 A。

10), 在一个无向图中，如果任意两点之间都存在路径相连，则称其为连通图。下图是一个有 4 个顶点、 6 条边的连通图。若要使它不再是连通图，至少要删去其中的（ ）条边

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/11.png)
{{ select(10) }}
- A. 1
- B. 2
- C. 3
- D. 4

> **答案**: C
> **解析**: 1. 题目分析：题目给出的图有 4 个顶点和 6 条边。根据完全图的公式，n 个顶点的完全图边数为 n(n-1)/2。对于 4 个顶点，4 × 3 / 2 = 6 条边。这说明该图是一个完全图（即任意两个顶点之间都有一条边直接相连）。

2. 连通图的概念：在无向图中，如果从任意顶点到任意其他顶点都有路径，则称该图为连通图。要使它不再连通，至少要让其中一个顶点与其他部分断开，或者将图分成两个互不相通的子图。

3. 最小边数计算：
   - 在这个 4 个顶点的完全图中，每个顶点的度数（连接的边数）都是 3。
   - 如果我们删去 1 条或 2 条边，剩下的顶点依然可以通过其他路径保持连通。例如，删去一个顶点的 2 条边，它依然还有 1 条边连接到图中，整个图仍是连通的。
   - 如果我们删去 3 条边，可以刚好切断一个顶点与剩下 3 个顶点的所有联系（因为每个点的度数是 3），此时该顶点成为孤立点，图不再连通。
   - 另一种情况是将图分为两个包含 2 个顶点的子图，这需要删去连接这两个集合的所有边，即 2 × 2 = 4 条边，这比 3 条边更多。

因此，要使该图不再连通，至少需要删去 3 条边。正确答案为 C。

11), 二叉树的（ ）第一个访问的节点是根节点。
{{ select(11) }}
- A. 先序遍历
- B. 中序遍历
- C. 后序遍历
- D. 以上都是

> **答案**: A
> **解析**: 二叉树的常用遍历方式有三种，它们的访问顺序如下：

1. 先序遍历（Pre-order Traversal）：访问顺序是“根节点 -> 左子树 -> 右子树”。因为根节点排在最前面，所以先序遍历第一个访问的节点一定是根节点。
2. 中序遍历（In-order Traversal）：访问顺序是“左子树 -> 根节点 -> 右子树”。在这种遍历中，根节点在左子树访问完成后才被访问。
3. 后序遍历（Post-order Traversal）：访问顺序是“左子树 -> 右子树 -> 根节点”。在这种遍历中，根节点是最后才被访问的。

因此，题目中要求的“第一个访问的节点是根节点”对应的是先序遍历。正确答案为 A。

12), 以A0​作为起点，对下面的无向图进行深度优先遍历时，遍历顺序不可能是（ ）

![image](http://luogu-ipic.oss-cn-shanghai.aliyuncs.com/youti/12.png)
{{ select(12) }}
- A. 𝐴 0 , 𝐴 1 , 𝐴 2 , 𝐴 3 A 0 ​ ,A 1 ​ ,A 2 ​ ,A 3 ​
- B. 𝐴 0 , 𝐴 1 , 𝐴 3 , 𝐴 2 A 0 ​ ,A 1 ​ ,A 3 ​ ,A 2 ​
- C. 𝐴 0 , 𝐴 2 , 𝐴 1 , 𝐴 3 A 0 ​ ,A 2 ​ ,A 1 ​ ,A 3 ​
- D. 𝐴 0 , 𝐴 3 , 𝐴 1 , 𝐴 2 A 0 ​ ,A 3 ​ ,A 1 ​ ,A 2 ​

> **答案**: B
> **解析**: 深度优先遍历（DFS）的基本原理是：从起点开始，尽可能深地探索每一个分支。在访问一个节点时，如果它还有未被访问的邻接节点，DFS 必须选择其中一个进行访问，直到当前节点的所有邻接节点都被访问过，才会回溯到上一个节点。

根据 GESP 等考级中常见的此类题目，该无向图的结构通常为：$A_0$ 与 $A_1, A_2, A_3$ 均有边相连，且 $A_1$ 与 $A_2$ 之间有边相连。即边集为：$(A_0, A_1), (A_0, A_2), (A_0, A_3), (A_1, A_2)$。

我们逐一分析选项：

A. $A_0, A_1, A_2, A_3$：从 $A_0$ 到 $A_1$，再从 $A_1$ 到其邻居 $A_2$，此时 $A_2$ 没有未访问的邻居（$A_0, A_1$ 已访问），回溯到 $A_0$，再访问 $A_3$。这是可能的。

B. $A_0, A_1, A_3, A_2$：从 $A_0$ 到 $A_1$。此时 $A_1$ 的邻居中，$A_0$ 已访问，$A_2$ 未访问。根据 DFS 原则，必须先访问 $A_2$ 才能回溯。由于 $A_1$ 与 $A_3$ 之间没有直接的边，要访问 $A_3$ 必须回溯到 $A_0$。但在 $A_2$ 未被访问前，DFS 不会从 $A_1$ 回溯到 $A_0$。因此，在 $A_1$ 之后直接出现 $A_3$ 是不可能的。

C. $A_0, A_2, A_1, A_3$：从 $A_0$ 到 $A_2$，再从 $A_2$ 到其邻居 $A_1$，回溯到 $A_0$，再访问 $A_3$。这是可能的。

D. $A_0, A_3, A_1, A_2$：从 $A_0$ 到 $A_3$，由于 $A_3$ 没有其他未访问邻居，回溯到 $A_0$，再访问 $A_1$，最后从 $A_1$ 访问 $A_2$。这是可能的。

综上所述，遍历顺序不可能是 B。

13), IPv4 协议使用 32 位地址，随着其不断被分配，地址资源日趋枯竭。因此，它正逐渐被使用（ ）位地址的 IPv6 协议所取代。
{{ select(13) }}
- A. 40
- B. 48
- C. 64
- D. 128

> **答案**: D
> **解析**: IPv4（Internet Protocol version 4）使用 32 位地址，通常以点分十进制表示（如 192.168.1.1），其地址空间约为 43 亿个，随着互联网设备的爆发式增长，这些地址已基本分配殆尽。

为了解决地址枯竭问题，IPv6（Internet Protocol version 6）被设计出来。IPv6 采用 128 位地址长度，通常以冒号十六进制表示。128 位地址提供了极其庞大的地址空间（约 3.4×10^38 个），足以支撑未来物联网等技术对海量 IP 地址的需求。

因此，正确答案是 D。

14), （ ）的平均时间复杂度为O(nlogn)，其中 n 是待排序的元素个数。
{{ select(14) }}
- A. 快速排序
- B. 插入排序
- C. 冒泡排序
- D. 基数排序

> **答案**: A
> **解析**: 本题考查常见排序算法的时间复杂度。

1. 快速排序（Quick Sort）：采用分治策略。在平均情况下，每次分区操作都能将数组分为两个规模大致相等的子部分，递归树的深度为 log n，每一层的工作量为 n，因此平均时间复杂度为 O(n log n)。虽然其最坏情况为 O(n²)，但题目要求的是平均时间复杂度。
2. 插入排序（Insertion Sort）：在平均情况下，每个元素需要移动一半的已排序序列，总的操作次数与 n 的平方成正比，因此平均时间复杂度为 O(n²)。
3. 冒泡排序（Bubble Sort）：通过相邻元素的比较和交换来排序，平均情况下需要进行约 n²/2 次比较，因此平均时间复杂度为 O(n²)。
4. 基数排序（Radix Sort）：这是一种非比较排序算法，其时间复杂度取决于元素的位数 d 和基数 r，通常表示为 O(d(n+r))。在 n 较大的情况下，它通常被认为是线性阶的复杂度，而不是 O(n log n)。

综上所述，平均时间复杂度为 O(n log n) 的选项是快速排序。

15), 下面是根据欧几里得算法编写的函数，它所计算的是 a 和 b 的（ ）
{{ select(15) }}
- A. 最大公共质因子
- B. 最小公共质因子
- C. 最大公约数
- D. 最小公倍数

> **答案**: C
> **解析**: 欧几里得算法（Euclidean algorithm），在中国通常被称为“辗转相除法”，是用于计算两个非负整数 a 和 b 的最大公约数（Greatest Common Divisor, GCD）的一种高效算法。

该算法的核心原理是：两个整数的最大公约数等于其中较小的数和两数相除余数的最大公约数。用数学公式表示为：gcd(a, b) = gcd(b, a % b)。

算法步骤如下：
1. 用 a 除以 b，得到余数 r。
2. 若 r 为 0，则 b 即为最大公约数。
3. 若 r 不为 0，则令 a = b，b = r，继续执行步骤 1。

因此，该算法计算的是最大公约数，正确选项为 C。

16), 通常在搜索引擎中，对某个关键词加上双引号表示（ ）
{{ select(16) }}
- A. 排除关键词，不显示任何包含该关键词的结果
- B. 将关键词分解，在搜索结果中必须包含其中的一部分
- C. 精确搜索，只显示包含整个关键词的结果
- D. 站内搜索，只显示关键词所指向网站的内容

> **答案**: C
> **解析**: 在搜索引擎（如百度、Google等）中，使用双引号（""）是一种常用的高级搜索技巧，其具体含义和作用如下：

1. 精确匹配：给关键词加上双引号，告诉搜索引擎搜索结果必须包含与引号内完全一致的短语，包括字符的顺序和空格。这被称为“精确搜索”或“完全匹配搜索”。
2. 过滤干扰：如果不加双引号，搜索引擎通常会进行“分词搜索”，即只要网页中包含了关键词中的部分词汇，或者词汇分散在页面不同位置，都会被列入结果。加上双引号可以有效过滤掉那些虽然包含关键词但语序不对或含义不相关的页面。

A 选项：排除关键词通常使用减号（-），例如“苹果 -手机”表示搜索苹果但排除手机相关内容。
B 选项：这是搜索引擎默认的模糊匹配或分词匹配行为，不需要加双引号。
D 选项：站内搜索通常使用“site:”指令，例如“site:edu.cn 编程”表示只在教育网域名下搜索。

18), 把 64 位非零浮点数强制转换成 32 位浮点数后，不可能（ ）
{{ select(18) }}
- A. 大于原数
- B. 小于原数
- C. 等于原数
- D. 与原数符号相反

> **答案**: D
> **解析**: 本题考查浮点数的存储格式及数据类型转换的原理。

1. 浮点数结构：根据 IEEE 754 标准，无论是 64 位浮点数（double）还是 32 位浮点数（float），其二进制表示都由三部分组成：符号位（Sign）、指数位（Exponent）和尾数位（Mantissa）。其中符号位决定了数值的正负，0 表示正，1 表示负。

2. 转换机制：当 64 位浮点数强制转换为 32 位浮点数时：
   - 精度损失：由于 32 位浮点数的尾数位（23位）少于 64 位浮点数（52位），转换时会发生舍入。如果舍入模式是向正无穷方向舍入，结果可能大于原数（选项 A 可能）；如果向负无穷方向舍入，结果可能小于原数（选项 B 可能）。
   - 精确匹配：如果原 64 位浮点数的值恰好在 32 位浮点数的表示范围内，且尾数部分不需要那么多位数（例如整数 1.0），则转换后的值等于原数（选项 C 可能）。
   - 溢出处理：如果数值超过了 32 位浮点数能表示的最大范围，会变成无穷大（Infinity），但符号依然保持不变。

3. 符号位保留：在强制转换过程中，最高位的符号位会被直接保留或映射到新格式的符号位上。正数转换后依然为正（或正无穷、正零），负数转换后依然为负（或负无穷、负零）。

因此，转换后的数值绝对不可能出现与原数符号相反的情况。故正确答案为 D。

19), 下列程序中，正确计算1,2,…,100这 100 个自然数之和 sum （初始值为 0）的是（ ）
{{ select(19) }}
- A. i = 1 do{ sum +=i; i++; }while(i<=100);
- B. i = 1; do{ sum +=i; i++; }while(i > 100);
- C. i = 1; while(i < 100){ sum+=i; i++; }
- D. i = 1; while(i >= 100){ sum+=i; i++; }

> **答案**: A
> **解析**: 本题考查循环结构（while 和 do-while）的逻辑控制。我们需要计算 1+2+...+100 的和。

A 选项：使用 do-while 循环。
1. 初始 i = 1。
2. 执行循环体：sum += 1, i 变为 2。
3. 判断条件 i <= 100 成立，继续循环。
4. 当 i 增加到 100 时，执行 sum += 100, i 变为 101。
5. 判断条件 101 <= 100 不成立，循环结束。
最终 sum 累加了 1 到 100 的所有自然数，逻辑正确。

B 选项：使用 do-while 循环，但条件为 i > 100。
第一次执行完循环体后 i = 2，判断 2 > 100 为假，循环立即结束，sum 只等于 1。

C 选项：使用 while 循环，条件为 i < 100。
当 i 增加到 100 时，判断 100 < 100 为假，循环结束。这意味着 sum 只累加到了 99，漏掉了 100。

D 选项：使用 while 循环，条件为 i >= 100。
初始 i = 1，判断 1 >= 100 为假，循环体一次都不会执行，sum 保持初始值 0。

综上所述，只有 A 选项能正确计算 1 到 100 的和。
