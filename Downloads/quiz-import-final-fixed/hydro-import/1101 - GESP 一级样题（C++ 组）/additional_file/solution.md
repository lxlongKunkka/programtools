3), 下列关于 C++ 语言的叙述，不正确的是（ ）
{{ select(3) }}
- A. 变量使用前必须先定义
- B. if 语句中的判断条件必须写在（ ） 中
- C. for 语句的循环体必须写在 {} 中
- D. 程序必须先编译才能运行

> **答案**: C
> **解析**: A. 变量使用前必须先定义：这是 C++ 的基本语法要求。在 C++ 中，任何变量在被使用（赋值、运算、输出等）之前，都必须先声明其类型和名称，以便编译器分配内存空间。

B. if 语句中的判断条件必须写在（ ） 中：这是 C++ 的语法规范。`if` 关键字后面必须紧跟一对圆括号，括号内是布尔表达式或可以转换为布尔值的表达式。

C. for 语句的循环体必须写在 {} 中：这是不正确的。在 C++ 中，如果 `for` 循环、`while` 循环或 `if` 语句的执行体只包含一条语句，大括号 `{}` 是可以省略的。只有当循环体包含多条语句时，才必须使用大括号将它们组成一个复合语句（代码块）。

D. 程序必须先编译才能运行：C++ 是一种典型的编译型语言。程序员编写的源代码（.cpp 文件）不能直接被计算机硬件执行，必须经过编译器翻译成机器指令（目标代码/可执行文件）后才能运行。

4), 不可以作为 C++ 标识符的是（ ）
{{ select(4) }}
- A. a_plus_b
- B. a_b
- C. a+b
- D. ab

> **答案**: C
> **解析**: 在 C++ 编程语言中，标识符（用于给变量、函数、类等命名的名称）必须遵循以下命名规则：

1. 组成字符：只能由字母（A-Z，a-z）、数字（0-9）和下划线（_）组成。
2. 开头字符：必须以字母或下划线开头，不能以数字开头。
3. 关键字限制：不能使用 C++ 的保留关键字（如 int, if, while 等）作为标识符。
4. 特殊符号：不能包含空格、标点符号或特殊运算符（如 +、-、*、/、@、# 等）。

逐一分析选项：
A. aplusb：全部由字母组成，符合规则。
B. a_b：由字母和下划线组成，符合规则。
C. a+b：包含了特殊运算符“+”，这在 C++ 中被视为加法操作，而不是标识符的一部分。因此，它不能作为标识符。
D. ab：全部由字母组成，符合规则。

因此，正确答案是 C。

6), 按照 C++ 语言的语法，以下不是正确的变量定义语句是（ ）
{{ select(6) }}

- B. int a = 10;
- C. int a (10);
- D. a = 10;

> **答案**: D
> **解析**: 在 C++ 编程语言中，定义一个变量必须包含“数据类型”和“变量名”两个基本要素。

B 选项：int a = 10; 是在定义变量的同时进行初始化，使用等号（=）赋值，符合语法。

因此，D 选项不是正确的变量定义语句。

7), 下列符号不是 C++语言的运算符的是（ ）
{{ select(7) }}
- A. $
- B. %
- C. =
- D. *

> **答案**: A
> **解析**: 在 C++ 编程语言中，运算符是用于执行特定数学或逻辑操作的符号。我们来逐一分析选项：

A. $：在标准 C++ 语言中，美元符号 $ 既不是运算符，也不是合法的标识符组成部分（尽管某些编译器可能允许在标识符中使用它作为扩展，但它绝不是运算符）。

B. %：这是取模运算符（Modulo Operator），用于计算两个整数相除后的余数。例如：5 % 2 的结果是 1。

C. =：这是赋值运算符（Assignment Operator），用于将右侧表达式的值赋给左侧的变量。例如：a = 10。

D. *：这是乘法运算符（Multiplication Operator），用于计算两个数的乘积；同时它也可以作为指针运算符（解引用）。

综上所述，只有 A 选项中的符号不属于 C++ 的运算符。

8), 如果用两个 int 类型的变量 a 和 b 分别表达直角三角形两条直角边的长度，则下列哪个表达式可以用来计算三角形的面积？（ ）
{{ select(8) }}
- A. a * b / 2
- B. a / 2 * b
- C. 1 / 2 * a * b
- D. a * b * 0.5

> **答案**: D
> **解析**: 在 C++ 等编程语言中，计算三角形面积需要注意数据类型和运算顺序：

1. 选项 A (a * b / 2)：a 和 b 都是 int 类型，它们的乘积也是 int。当乘积为奇数时，除以 2 会执行整数除法，导致小数部分被舍弃（例如 3 * 1 / 2 的结果是 1 而不是 1.5），存在精度丢失。
2. 选项 B (a / 2 * b)：先执行 a / 2。如果 a 是奇数，a / 2 会立即丢失精度（例如 3 / 2 结果为 1），再乘以 b 得到的结果偏差更大。
3. 选项 C (1 / 2 * a * b)：在 C++ 中，1 和 2 都是整数常量，1 / 2 执行整数除法的结果是 0。因此，无论 a 和 b 是多少，整个表达式的结果永远是 0。
4. 选项 D (a * b * 0.5)：0.5 是一个浮点数（double 类型）。当整数与浮点数进行运算时，程序会自动将整数转换为浮点数进行计算。a * b 的结果会与 0.5 进行浮点数乘法，从而保留正确的小数部分（例如 3 * 1 * 0.5 的结果是 1.5）。

综上所述，选项 D 能够最准确地表达三角形面积的计算公式。

9), 表达式 (3 + 12 / 3 * 2) 的计算结果为（ ）
{{ select(9) }}
- A. 10
- B. 5
- C. 11
- D. 2

> **答案**: C
> **解析**: 这是一个关于算术表达式运算优先级的题目。在计算表达式 (3 + 12 / 3 * 2) 时，我们需要遵循以下运算规则：

1. 优先级顺序：在没有括号嵌套的情况下，先进行乘除运算，后进行加减运算。
2. 同级运算方向：乘法和除法属于同一优先级，按照从左到右的顺序依次计算。

具体计算步骤如下：
第一步：计算除法，12 / 3 = 4。此时表达式变为 (3 + 4 * 2)。
第二步：计算乘法，4 * 2 = 8。此时表达式变为 (3 + 8)。
第三步：计算加法，3 + 8 = 11。

因此，表达式的最终计算结果为 11，正确选项为 C。

11), 如果 a 和 b 均为 int 类型的变量，下列表达式能够正确判断 “a 不等于 0 或 b 不等于 0” 的是（ ）
{{ select(11) }}
- A. !a == 0 && !b == 0
- B. !(a == 0 && b == 0)
- C. (a != 0) && (b != 0)
- D. a && b

> **答案**: B
> **解析**: 本题考查的是逻辑表达式的等价转换。

题目要求判断的逻辑是：“a 不等于 0 或 b 不等于 0”，用逻辑运算符表示为：`(a != 0) || (b != 0)`。

我们逐一分析选项：
A. !a == 0 && !b == 0：在 C++ 中，!a 如果 a 为非 0 则结果为 0，如果 a 为 0 则结果为 1。因此 !a == 0 等价于 a != 0。整个表达式等价于 (a != 0) && (b != 0)，表示“且”的关系，不符合题意。
B. !(a == 0 && b == 0)：根据德·摩根定律（De Morgan's laws），!(P && Q) 等价于 !P || !Q。在这里，P 是 (a == 0)，Q 是 (b == 0)。因此，!(a == 0 && b == 0) 等价于 !(a == 0) || !(b == 0)，即 (a != 0) || (b != 0)。这完全符合题目要求的“或”关系。
C. (a != 0) && (b != 0)：这表示 a 不等于 0 且 b 不等于 0，不符合题意。
D. a && b：在 C++ 中，非 0 整数被视为真，0 被视为假。因此 a && b 等价于 (a != 0) && (b != 0)，同样表示“且”的关系，不符合题意。

综上所述，正确答案是 B。

12), 如果 a 为 int 类型的变量，下列表达式不能正确表达 “a 是奇数时结果为 0，否则结果非 0” 的是（ ）
{{ select(12) }}
- A. a %= 2
- B. a / 2 * 2 == a
- C. a % 2 == 0
- D. (a + 1) % 2

> **答案**: A
> **解析**: 本题考查 C++ 表达式的逻辑判断。题目要求：当 a 是奇数时，表达式的结果为 0；当 a 是偶数时，表达式的结果非 0。

我们逐一分析选项：

A. a %= 2：这是一个赋值表达式，其结果是 a 对 2 取模后的值。
- 如果 a 是奇数（如 3），a % 2 的结果是 1（或 -1），赋值后表达式的结果为 1（或 -1），是非 0 值。
- 如果 a 是偶数（如 4），a % 2 的结果是 0，赋值后表达式的结果为 0。
这与题目要求的“奇数时为 0，偶数时非 0”正好相反，因此 A 错误。

B. a / 2 * 2 == a：这是一个关系表达式。
- 如果 a 是奇数（如 3），3 / 2 等于 1（整数除法丢弃小数），1 * 2 等于 2，2 == 3 为假，结果为 0。
- 如果 a 是偶数（如 4），4 / 2 等于 2，2 * 2 等于 4，4 == 4 为真，结果为 1（非 0）。
符合题目要求。

C. a % 2 == 0：这是一个关系表达式。
- 如果 a 是奇数（如 3），3 % 2 等于 1，1 == 0 为假，结果为 0。
- 如果 a 是偶数（如 4），4 % 2 等于 0，0 == 0 为真，结果为 1（非 0）。
符合题目要求。

D. (a + 1) % 2：这是一个算术表达式。
- 如果 a 是奇数（如 3），a + 1 是偶数，偶数对 2 取模的结果是 0。
- 如果 a 是偶数（如 4），a + 1 是奇数，奇数对 2 取模的结果是 1 或 -1（非 0）。
符合题目要求。

综上所述，只有 A 选项不能正确表达题目要求的逻辑。

13), 在下列代码的横线处填写（ ），可以使得输出是“20 10”

![image](https://cdn.luogu.com.cn/upload/image_hosting/e0lhy5bj.png)
{{ select(13) }}
- A. a = b; b = a;
- B. a = max(a, b); b = min(a, b);
- C. a = a + b; a = a - b; b = a - b;
- D. int tmp = a; a = b; b = tmp;

> **答案**: D
> **解析**: 本题考查的是 C++ 中交换两个变量值的基本逻辑。根据题目要求，输出结果为“20 10”，而通常这类题目的初始设定为 a = 10, b = 20，因此目标是实现 a 和 b 的值交换。

下面逐一分析选项：

A. a = b; b = a;
执行 a = b 后，a 的值变为 20。接着执行 b = a，此时 a 已经是 20，所以 b 也变为 20。最终 a=20, b=20。无法实现交换。

B. a = max(a, b); b = min(a, b);
执行 a = max(10, 20) 后，a 变为 20。接着执行 b = min(a, b)，即 min(20, 20)，b 变为 20。最终 a=20, b=20。无法实现交换。

C. a = a + b; a = a - b; b = a - b;
这是一个错误的数值交换变体。
第一步：a = 10 + 20 = 30；
第二步：a = 30 - 20 = 10；
第三步：b = 10 - 20 = -10。
最终 a=10, b=-10。正确的数值交换算法应该是：a = a + b; b = a - b; a = a - b;

D. int tmp = a; a = b; b = tmp;
这是最经典的使用临时变量交换两个变量值的方法：
1. 将 a 的原始值（10）存入临时变量 tmp；
2. 将 b 的值（20）赋给 a，此时 a = 20；
3. 将 tmp 中保存的 a 的原始值（10）赋给 b，此时 b = 10。
最终 a=20, b=10，符合题目输出要求。

14), 在下列代码的横线处填写（ ），可以使得输出是“111111”

![image](https://cdn.luogu.com.cn/upload/image_hosting/50bigpsf.png)
{{ select(14) }}
- A. i++
- B. i += 3
- C. i *= 2
- D. i = i * 3 - 1

> **答案**: C
> **解析**: 本题考查的是 for 循环的执行次数以及循环变量的更新规律。

虽然题目中没有给出完整的代码段，但根据 GESP/CSP 等等级考试的常见题型，此类题目通常的完整代码结构如下：
for (int i = 1; i <= 32; i = ______)
    cout << 1;

题目要求输出“111111”，即要求循环体执行 6 次。我们逐一分析选项：

A. i++：循环变量 i 每次加 1。从 1 到 32 会执行 32 次，输出 32 个“1”。
B. i += 3：循环变量 i 每次加 3。执行序列为 i = 1, 4, 7, 10, 13, 16, 19, 22, 25, 28, 31。共执行 11 次，输出 11 个“1”。
C. i *= 2：循环变量 i 每次翻倍（按 2 的幂次增长）。
   - 第 1 次：i = 1，输出第 1 个“1”，i 变为 2；
   - 第 2 次：i = 2，输出第 2 个“1”，i 变为 4；
   - 第 3 次：i = 4，输出第 3 个“1”，i 变为 8；
   - 第 4 次：i = 8，输出第 4 个“1”，i 变为 16；
   - 第 5 次：i = 16，输出第 5 个“1”，i 变为 32；
   - 第 6 次：i = 32，输出第 6 个“1”，i 变为 64；
   - 此时 i = 64，不满足 i <= 32 的条件，循环结束。
   刚好输出 6 个“1”，符合题意。
D. i = i * 3 - 1：执行序列为 i = 1, 2, 5, 14, 41。当 i 变为 41 时超过 32，循环结束。共执行 4 次，输出 4 个“1”。

因此，正确答案是 C。

16), 操作系统是让用户可以操纵计算机完成各种功能的软件。（ ）
{{ select(16) }}
- true. 正确

21), 能用 while 语句编写的循环，就可以使用 for 语句编写出具有同样功能的循环。（ ）
{{ select(21) }}
- true. 正确
- false. 错误

> **答案**: true
> **解析**: 在大多数编程语言（如 C++、Python、Java 等）中，while 循环和 for 循环在逻辑上是等价的，它们可以互相转换。

1. while 循环的标准形式是：
   while (条件) {
       循环体;
   }

2. for 循环的标准形式是：
   for (初始化; 条件; 更新) {
       循环体;
   }

如果我们想把一个 while 循环改写为 for 循环，只需要将 for 语句的初始化部分和更新部分留空即可。例如：
for (; 条件; ) {
    循环体;
}
这种形式的 for 循环与 while (条件) { 循环体; } 的执行逻辑完全一致。

反之，任何 for 循环也可以通过在循环外初始化变量、在循环体内更新变量的方式改写为 while 循环。

因此，凡是能用 while 语句实现的循环逻辑，一定可以使用 for 语句来实现。题目表述正确。

- false. 错误

> **解析**: 1. 运算符含义：在 C++ 等编程语言中，`%` 是取模（取余）运算符。表达式 `a % 4` 的计算结果是变量 `a` 除以 4 后的余数。
2. 逻辑判断：表达式 `(a % 4 == 0)` 的结果为真（True），意味着 `a` 除以 4 的余数为 0，即 `a` 能够被 4 整除。
3. 倍数定义：根据数学定义，如果一个整数 $a$ 除以一个非零整数 $b$ 的余数为 0，则称 $a$ 是 $b$ 的倍数。

   - 当 `a` 为负整数（如 -4, -8...）时，在 C++ 语言标准中，这些负数取模 4 的结果也是 0，且它们在数学上同样被视为 4 的倍数（如 $-4 = -1 \times 4$）。
因此，该表达式为真时，说明 `a` 的值确实是 4 的倍数。
