1), 以下哪个是面向对象的高级语言（ ）
{{ select(1) }}
- A. 汇编语言
- B. C++
- C. Fortran
- D. Basic

> **答案**: B
> **解析**: 1. 汇编语言（Assembly Language）：属于低级语言，是机器指令的符号化表示，直接操作硬件寄存器和内存，不具备面向对象的特性。
2. C++：是在 C 语言的基础上发展而来的，它引入了类（Class）、对象（Object）、继承、多态和封装等核心概念，是典型的面向对象（Object-Oriented Programming, OOP）的高级编程语言。
3. Fortran：是世界上最早的高级编程语言之一，主要用于科学计算和工程计算，属于过程式（面向过程）语言。
4. Basic：是一种设计给初学者使用的编程语言，虽然现代版本（如 VB.NET）支持面向对象，但在计算机基础知识的分类中，它通常被视为早期的过程式高级语言。

综上所述，选项中明确属于面向对象的高级语言是 C++。

2), 1TB 代表的字节数是（ ）
{{ select(2) }}
- A. 2 2 的 1 0 10 次方
- B. 2 2 的 2 0 20 次方
- C. 2 2 的 3 0 30 次方
- D. 2 2 的 4 0 40 次方

> **答案**: D
> **解析**: 在计算机科学中，存储单位的换算通常以 1024（即 2 的 10 次方）为进位单位。具体的换算关系如下：

1. 1 KB (Kilobyte) = 1024 Bytes = 2^10 Bytes
2. 1 MB (Megabyte) = 1024 KB = 2^10 * 2^10 Bytes = 2^20 Bytes
3. 1 GB (Gigabyte) = 1024 MB = 2^10 * 2^20 Bytes = 2^30 Bytes
4. 1 TB (Terabyte) = 1024 GB = 2^10 * 2^30 Bytes = 2^40 Bytes

因此，1TB 代表的字节数是 2 的 40 次方。选项 D 正确。

3), 二进制数 00100100 和 00010101 的和是（ ）
{{ select(3) }}
- A. 00101000
- B. 001010100
- C. 01000101
- D. 00111001

> **答案**: D
> **解析**: 这道题考察的是二进制数的加法运算。我们可以通过两种方法来得出答案：

方法一：直接进行二进制竖式加法
按照二进制加法规则（0+0=0, 0+1=1, 1+1=10即写0进1）：
  00100100
+ 00010101
-----------
  00111001

具体步骤如下（从右往左）：
1. 第1位：0 + 1 = 1
2. 第2位：0 + 0 = 0
3. 第3位：1 + 1 = 10，写0向高位进1
4. 第4位：0 + 0 + 1(进位) = 1
5. 第5位：0 + 1 = 1
6. 第6位：1 + 0 = 1
7. 第7位：0 + 0 = 0
8. 第8位：0 + 0 = 0
最终结果为 00111001。

方法二：转换为十进制计算后再转回二进制
1. 将 00100100 转换为十进制：2^5 + 2^2 = 32 + 4 = 36。
2. 将 00010101 转换为十进制：2^4 + 2^2 + 2^0 = 16 + 4 + 1 = 21。
3. 计算和：36 + 21 = 57。
4. 将 57 转换为二进制：
   57 = 32 + 16 + 8 + 1 = 2^5 + 2^4 + 2^3 + 2^0，即 111001。
   补齐 8 位后为 00111001。

对比选项，正确答案为 D。

6), CPU、存储器、I/O 设备是通过（ ）连接起来的。
{{ select(6) }}
- A. 接口
- B. 总线
- C. 控制线
- D. 系统文件

> **答案**: B
> **解析**: 在计算机体系结构中，连接 CPU、存储器（内存）和 I/O 设备（输入/输出设备）的公共通信通道被称为“总线”（Bus）。

详细解析如下：
1. 总线（Bus）：是计算机各种功能部件之间传送信息的公共通信干线。它是由导线组成的传输路径，按照所传输的信息种类，通常分为数据总线（Data Bus）、地址总线（Address Bus）和控制总线（Control Bus）。
2. 接口（Interface）：通常指不同部件之间连接的物理界面或逻辑协议，虽然设备通过接口接入，但它们之间的数据流动是依靠总线完成的。
3. 控制线（Control Line）：只是总线的一部分（即控制总线中的单根信号线），不能代表整个连接系统。
4. 系统文件（System File）：属于软件范畴，是存储在磁盘上的程序或数据，不是硬件连接媒介。

因此，正确答案是 B。

7), 断电后会丢失数据的存储器是（ ）
{{ select(7) }}
- A. RAM
- B. ROM
- C. 硬盘
- D. 光盘

> **答案**: A
> **解析**: 在计算机存储器中，根据断电后数据是否丢失，可以分为易失性存储器和非易失性存储器：

1. RAM（Random Access Memory，随机存取存储器）：它是计算机的主内存，用于暂时存放 CPU 正在运行的程序和数据。RAM 的特点是读写速度极快，但它是易失性的，一旦断电，其中存储的所有数据都会立即消失。
2. ROM（Read-Only Memory，只读存储器）：它通常用于存储计算机最基本的启动指令（如 BIOS）。ROM 是非易失性的，断电后数据依然保留。
3. 硬盘（Hard Disk）：属于外部存储设备，利用磁性材料（HDD）或闪存芯片（SSD）来持久化存储数据，断电后数据不会丢失。
4. 光盘（Optical Disc）：利用激光在盘片上刻录物理痕迹来存储数据，属于永久性存储介质，断电后数据不会丢失。

综上所述，断电后会丢失数据的存储器是 RAM。

8), 以下哪一种是属于电子邮件收发的协议（ ）
{{ select(8) }}
- A. SMTP
- B. UDP
- C. P2P
- D. FTP

> **答案**: A
> **解析**: 本题考查的是计算机网络中常见的应用层协议及其功能。

A. SMTP (Simple Mail Transfer Protocol)：简单邮件传输协议。它是互联网上用于发送电子邮件的标准协议。此外，常见的邮件接收协议还包括 POP3 和 IMAP。

B. UDP (User Datagram Protocol)：用户数据报协议。它属于传输层协议，提供的是无连接、不可靠的数据传输服务，常用于对实时性要求高但对丢包不敏感的场景（如视频会议、在线游戏），并非专门用于邮件收发。

C. P2P (Peer-to-Peer)：点对点技术。它是一种网络架构，在这种架构中，网络参与者既是资源提供者也是资源消费者，常用于文件共享（如 BitTorrent），不属于专门的邮件收发协议。

D. FTP (File Transfer Protocol)：文件传输协议。它主要用于在网络上进行文件的上传和下载，与电子邮件的收发机制不同。

因此，正确答案是 A。

10), 链表不具有的特点是（ ）
{{ select(10) }}
- A. 不必事先估计存储空间
- B. 可随机访问任一元素
- C. 插入删除不需要移动元素
- D. 所需空间与线性表长度成正比

> **答案**: B
> **解析**: 本题考查链表（Linked List）的基本性质及其与顺序表（数组）的区别。

A 选项正确：链表是一种动态数据结构，其节点是在程序运行过程中根据需要动态申请的，因此不需要像静态数组那样事先估计并声明存储空间的大小。

B 选项错误：这是链表不具有的特点。链表在内存中是非连续存储的，每个节点只知道下一个节点的地址。要访问链表中的第 i 个元素，必须从头节点（Head）开始顺着指针逐个查找，这种访问方式称为“顺序访问”，其时间复杂度为 O(n)。而“随机访问”是指可以通过下标在 O(1) 时间内直接定位元素，这是数组（顺序表）的特点。

C 选项正确：在链表中进行插入或删除操作时，只需要修改相关节点的指针指向即可，不需要像数组那样为了保持逻辑顺序而移动插入/删除点之后的所有元素。

D 选项正确：链表的每个节点包含数据域和指针域，存储一个长度为 n 的线性表需要 n 个节点，因此所需的存储空间与线性表的长度成正比，空间复杂度为 O(n)。

因此，正确答案是 B。

11), 下列各无符号十进制整数中，能用八位二进制表示的数中最大的是（ ）
{{ select(11) }}
- A. 296
- B. 133
- C. 256
- D. 199

> **答案**: D
> **解析**: 1. 理解八位二进制数的表示范围：
   在计算机中，一个 n 位的无符号二进制数可以表示的数值范围是 0 到 2^n - 1。
   对于八位（8-bit）二进制数，其最小值为 (00000000)₂ = 0，最大值为 (11111111)₂。

2. 计算最大值：
   (11111111)₂ = 2^7 + 2^6 + 2^5 + 2^4 + 2^3 + 2^2 + 2^1 + 2^0 = 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255。
   或者使用公式：2^8 - 1 = 256 - 1 = 255。
   因此，八位二进制能表示的最大无符号十进制整数是 255。

3. 逐一分析选项：
   A. 296：大于 255，无法用八位二进制表示（需要 9 位）。
   B. 133：小于 255，可以用八位二进制表示。
   C. 256：大于 255，无法用八位二进制表示（256 的二进制是 100000000，共 9 位）。
   D. 199：小于 255，可以用八位二进制表示。

4. 结论：
   在给出的选项中，能用八位二进制表示的数有 133 和 199。其中最大的是 199。故选 D。

13), 要求以下程序的功能是计算：s=1+21​+31​+⋯+101​。 程序运行后输出结果错误，导致错误结果的程序行是（ ）
{{ select(13) }}

- B. for(n = 10; n > 1; n--)
- C. s = s + 1 / n;
- D. cout << s << endl;

> **答案**: C
> **解析**: 1. 程序的目标是计算 $s = 1 + \frac{1}{2} + \frac{1}{3} + \dots + \frac{1}{10}$。

3. 在选项 C 的表达式 `s = s + 1 / n;` 中，数字 `1` 是整数常量，变量 `n` 是循环变量（通常定义为 int）。当 `n` 取值为 2, 3, ..., 10 时，`1 / n` 的计算结果始终为 0。

5. 要修正这个错误，需要将 `1` 改为 `1.0`（使其变为浮点数），或者使用强制类型转换，例如 `s = s + 1.0 / n;`，这样程序才会执行浮点数除法，保留小数部分。

14), 设变量 x 为 float 型且已赋值，则以下语句中能将 x 中的数值保留到小数点后两位，并将第三位四舍五入的是（ ）
{{ select(14) }}
- A. x = (x * 100) + 0.5 / 100.0;
- B. x = (x * 100 + 0.5) / 100.0;
- C. x = (int)(x * 100 + 0.5)/100.0;
- D. x = (x / 100 + 0.5) * 100.0;

> **答案**: C
> **解析**: 在计算机编程中，对一个浮点数进行“保留 n 位小数并四舍五入”的标准处理流程如下：

1. 扩大倍数：将小数点向右移动 n 位。如果要保留两位小数，则乘以 100。
2. 四舍五入：加上 0.5。这样如果原来的第三位小数大于等于 5，加 0.5 后整数部分就会进 1；如果小于 5，则整数部分保持不变。
3. 取整截断：使用强制类型转换 (int) 丢弃所有小数部分。这是实现“四舍五入”逻辑的关键步骤，如果不取整，小数位依然存在。
4. 还原倍数：将结果除以 100.0，将小数点移回原位。注意这里要除以 100.0（浮点数）而不是 100（整数），以确保结果重新变回 float/double 类型。

选项分析：
A. x = (x * 100) + 0.5 / 100.0; 运算逻辑错误，相当于 x * 100 + 0.005。
B. x = (x * 100 + 0.5) / 100.0; 虽然加了 0.5，但没有进行强制类型转换取整，结果只是 x + 0.005，没有截断多余的小数位。
C. x = (int)(x * 100 + 0.5) / 100.0; 正确执行了上述四个步骤。
D. x = (x / 100 + 0.5) * 100.0; 运算方向错误，会导致数值变大。

因此，正确答案是 C。

15), 有以下程序: 若要使程序的输出值为 2，则应该从键盘给 n 输入的值是（ ）
{{ select(15) }}
- A. 1
- B. 3
- C. 5
- D. 0

> **答案**: C
> **解析**: 虽然题目中未直接给出具体的程序代码，但根据 GESP/CSP 等编程考级中此类题目的常见逻辑，该程序通常是统计从 1 到 n 之间满足某个条件的整数个数。最常见的逻辑是统计 1 到 n 之间的偶数个数。

其代码逻辑通常如下：
```cpp
int n, s = 0;
cin >> n;
for (int i = 1; i <= n; i++) {
    if (i % 2 == 0) s++; // 如果 i 是偶数，计数器 s 加 1
}
cout << s;
```

我们代入各选项进行验证：

3. C 选项，n = 5：i 可以取 1, 2, 3, 4, 5，其中 2 和 4 是偶数，s = 2，输出为 2。符合题意。
4. D 选项，n = 0：循环条件不成立，s = 0，输出为 0。

另外，如果程序逻辑是简单的整数除法 `cout << n / 2;`，代入 n = 5 同样会得到 5 / 2 = 2（向下取整）。

综上所述，要使输出值为 2，输入的 n 应为 5。

16), 一棵具有 5 层的满二叉树中结点数为（ ）
{{ select(16) }}
- A. 31
- B. 32
- C. 33
- D. 16

> **答案**: A
> **解析**: 1. 满二叉树的定义：在二叉树中，除了叶子结点外，每一个结点都有两个子结点，且所有的叶子结点都在同一层。
2. 结点总数公式：对于一棵深度（层数）为 h 的满二叉树，其结点总数 N 的计算公式为 N = 2^h - 1。
3. 代入计算：本题中层数 h = 5，则结点总数 N = 2^5 - 1 = 32 - 1 = 31。
4. 逐层验证：
   - 第 1 层：1 个结点
   - 第 2 层：2 个结点
   - 第 3 层：4 个结点
   - 第 4 层：8 个结点
   - 第 5 层：16 个结点
   - 总和：1 + 2 + 4 + 8 + 16 = 31。

因此，正确答案是 A。

17), 有向图中每个顶点的度等于该顶点的（ ）
{{ select(17) }}
- A. 入度
- B. 出度
- C. 入度和出度之和
- D. 入度和出度之差

> **答案**: C
> **解析**: 在图论中，顶点的“度”（Degree）是指与该顶点相关联的边的数目。

1. 对于无向图（Undirected Graph）：一个顶点的度就是连接到该顶点的边的数量。
2. 对于有向图（Directed Graph）：由于边具有方向性，顶点的度被进一步细分为“入度”和“出度”。
   - 入度（In-degree）：是以该顶点为终点的有向边的条数。
   - 出度（Out-degree）：是以该顶点为起点的有向边的条数。
   - 顶点的度（Total Degree）：在有向图中，一个顶点的总度数定义为该顶点的入度与出度之和。

因此，正确选项为 C。

18), 设有 100 个数据元素，采用折半搜索时，最大比较次数为（ ）
{{ select(18) }}
- A. 6
- B. 7
- C. 8
- D. 10

> **答案**: B
> **解析**: 折半搜索（也称为二分查找）的基本原理是每次将当前的搜索范围缩小一半。对于具有 n 个元素的有序序列，其最大比较次数（即最坏情况下的查找次数）与该搜索过程对应的判定树的高度一致。

计算最大比较次数的公式为：⌊log₂n⌋ + 1。

在本题中，数据元素个数 n = 100：
1. 计算 log₂100：因为 2⁶ = 64，2⁷ = 128，所以 6 < log₂100 < 7。
2. 取整：⌊log₂100⌋ = 6。
3. 加 1：6 + 1 = 7。

我们也可以通过手动模拟搜索范围的变化来验证：
- 第 1 次比较后，剩余元素约 50 个；
- 第 2 次比较后，剩余元素约 25 个；
- 第 3 次比较后，剩余元素约 12 个；
- 第 4 次比较后，剩余元素约 6 个；
- 第 5 次比较后，剩余元素约 3 个；
- 第 6 次比较后，剩余元素约 1 个；
- 第 7 次比较后，确定找到目标或查找失败。

因此，最大比较次数为 7 次。正确选项为 B。

19), 若有如下程序段，其中s,a,b,c均已定义为整型变量，且a,c均已赋值，c>0。 则与上述程序段功能等价的赋值语句是（ ）
{{ select(19) }}
- A. s = a + b
- B. s = a + c
- C. s = s + c
- D. s = b + c

> **答案**: B
> **解析**: 虽然题目描述中缺失了具体的程序段代码，但根据 GESP/CSP 等编程考级中此类题目的典型原型，该程序段通常如下：
s = a;
for (b = 1; b <= c; b++)
    s = s + 1;

对该程序段进行逻辑分析：
1. 初始状态：变量 s 被赋值为 a。
2. 循环过程：循环变量 b 从 1 开始，每次递增 1，直到达到 c。因为题目已知 c > 0，所以循环体会执行 c 次。
3. 累加操作：在每次循环中，s 的值都会增加 1。
4. 最终结果：s 的初始值是 a，经过 c 次“加 1”的操作后，s 的最终值等于 a + c。

因此，整个程序段实现的功能就是将 a 与 c 的和计算出来并赋值给 s，等价的赋值语句是 s = a + c。选项 B 正确。

20), 计算机界的最高奖是（ ）
{{ select(20) }}

- B. 诺贝尔奖
- C. 图灵奖
- D. 普利策奖

A. 菲尔兹奖：被誉为“数学界的诺贝尔奖”，是数学领域的最高奖项。
B. 诺贝尔奖：涵盖物理、化学、生理学或医学、文学、和平及经济学，但并不包含计算机科学。
D. 普利策奖：是美国新闻界的最高荣誉，涵盖新闻、文学和音乐等领域。
