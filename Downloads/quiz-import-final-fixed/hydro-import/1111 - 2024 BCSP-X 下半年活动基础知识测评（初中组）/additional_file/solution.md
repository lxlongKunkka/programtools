4), 计算两个二进制数的和：11011110+101101101=
{{ select(4) }}
- A. 1 0 1 1 0 1 1 0 1 101101101
- B. 1 1 0 0 1 1 0 1 1 110011011
- C. 1 0 0 1 0 0 1 0 1 1 1001001011
- D. 1 0 1 1 0 0 1 0 1 1 1011001011

> **答案**: C
> **解析**: 本题考查二进制数的加法运算。二进制加法的规则是“逢二进一”。

计算过程如下：
1. 将两个二进制数右对齐：
      011011110 (补齐为9位，方便对齐)
    + 101101101
    -----------

2. 从右往左逐位相加：
    - 第1位：0 + 1 = 1
    - 第2位：1 + 0 = 1
    - 第3位：1 + 1 = 10，写 0 进 1
    - 第4位：1 + 1 + 1(进位) = 11，写 1 进 1
    - 第5位：1 + 0 + 1(进位) = 10，写 0 进 1
    - 第6位：0 + 1 + 1(进位) = 10，写 0 进 1
    - 第7位：1 + 1 + 1(进位) = 11，写 1 进 1
    - 第8位：1 + 0 + 1(进位) = 10，写 0 进 1
    - 第9位：0 + 1 + 1(进位) = 10，写 0 进 1
    - 第10位：最高位进位 1

3. 最终结果为：1001001011。

验证（转换为十进制）：
- 11011110 = 128 + 64 + 16 + 8 + 4 + 2 = 222
- 101101101 = 256 + 64 + 32 + 8 + 4 + 1 = 365
- 222 + 365 = 587
- 587 转换为二进制：512 + 64 + 8 + 2 + 1 = 2^9 + 2^6 + 2^3 + 2^1 + 2^0 = 1001001011

对比选项，C 选项的数值部分 1001001011 与计算结果一致。

5), 我们都知道闰年和平年。闰年的判定是“四年一闰，百年不闰，四百年又一闰”。下面哪个条件可以用来判定某一年 y 是否是闰年？
{{ select(5) }}
- A. y//4 and y//100 or y//400
- B. y%4==0 or y %100==0 or y%400==0
- C. y%4==0 and y%100!=0 or y%400==0
- D. y%400==0 and y%100!=0 and y%400==0

> **答案**: C
> **解析**: 闰年的判定规则是“四年一闰，百年不闰，四百年又一闰”。我们可以将其拆解为两个逻辑部分：

1. “四年一闰，百年不闰”：这意味着年份必须能被 4 整除，同时不能被 100 整除。在编程中，整除使用取余运算符 `%`，如果 `y % 4 == 0` 成立，说明能被 4 整除；如果 `y % 100 != 0` 成立，说明不能被 100 整除。这两个条件必须同时满足，所以用逻辑与 `and` 连接：`y % 4 == 0 and y % 100 != 0`。

2. “四百年又一闰”：这意味着如果年份能被 400 整除，它也是闰年。即 `y % 400 == 0`。

3. 综合逻辑：上述两个部分是“或”的关系，只要满足其中之一，该年份就是闰年。因此，使用逻辑或 `or` 将它们连接起来：`(y % 4 == 0 and y % 100 != 0) or (y % 400 == 0)`。

在 Python 等编程语言中，`and` 的优先级高于 `or`，所以括号可以省略，即：`y % 4 == 0 and y % 100 != 0 or y % 400 == 0`。

选项分析：
A 选项：使用了整除运算符 `//`，它返回的是商而不是余数，逻辑错误。
B 选项：逻辑为“能被 4 或 100 或 400 整除”，这会导致像 1900 年（能被 100 整除但不能被 400 整除）被错误地判定为闰年。
C 选项：完全符合闰年判定逻辑。
D 选项：逻辑自相矛盾，能被 400 整除的数一定能被 100 整除，因此 `y % 400 == 0 and y % 100 != 0` 永远为假。

6), 函数的递归和调用与下面哪个数据结构相关？
{{ select(6) }}
- A. 队列
- B. 栈
- C. 二叉树
- D. 完全图

> **答案**: B
> **解析**: 函数的调用和递归与“栈”（Stack）这一数据结构紧密相关。

1. 栈的特性：栈是一种“后进先出”（LIFO, Last-In-First-Out）的数据结构。最后被压入栈的元素最先被弹出。
2. 函数调用过程：当一个函数被调用时，系统会为该函数创建一个“栈帧”（Stack Frame），用于存储该函数的局部变量、参数以及返回地址，并将其压入系统栈中。
3. 递归的实现：在递归调用中，函数会不断调用自身。每一次调用都会产生一个新的栈帧压入栈顶。只有当最内层的函数执行完毕（触及递归边界）并返回时，对应的栈帧才会从栈顶弹出，控制权回到上一层函数。
4. 结论：这种“逐层进入、逆序退出”的机制与栈的特性完全吻合。因此，计算机底层正是利用栈来管理函数调用和递归的。

其他选项分析：
A. 队列：是“先进先出”结构，常用于广度优先搜索（BFS）或任务排队。
C. 二叉树：是一种层次结构，虽然递归常用于遍历二叉树，但实现递归的底层结构依然是栈。
D. 完全图：是图论中的概念，与函数调用的逻辑结构无关。

7), 有一天小猴子摘下了若干个桃子，当即吃掉一半，还觉得不过瘾，又多吃了一个。第二天接着吃了剩下的桃子中的一半，仍不过瘾，又多吃了一个。以后每天都是吃尚存桃子的一半零一个。到第 5 天早上小猴子再去吃桃子时，看到只剩下 5 个桃子了。问小猴子第一天共摘下了多少个桃子？
{{ select(7) }}
- A. 1 2 5 125
- B. 5 0 50
- C. 8 0 80
- D. 1 1 0 110

> **答案**: D
> **解析**: 这是一个经典的逆推问题。我们可以通过从第5天的情况往回推算，得出第1天摘下的桃子总数。

1. 题目规律：每天吃掉当前桃子数的一半再多一个。
   设第 n 天早上的桃子数为 x(n)，则第 n+1 天早上的桃子数 x(n+1) = x(n) / 2 - 1。
   
2. 逆推公式：为了求出前一天的桃子数，我们将公式变形为：
   x(n) = (x(n+1) + 1) * 2

3. 逐步逆推：
   - 已知第 5 天早上剩下：x(5) = 5 个。
   - 第 4 天早上的桃子数：x(4) = (5 + 1) * 2 = 12 个。
   - 第 3 天早上的桃子数：x(3) = (12 + 1) * 2 = 26 个。
   - 第 2 天早上的桃子数：x(2) = (26 + 1) * 2 = 54 个。
   - 第 1 天早上的桃子数：x(1) = (54 + 1) * 2 = 110 个。

4. 验证：
   - 第一天：110个，吃掉一半(55)+1个=56个，剩54个。
   - 第二天：54个，吃掉一半(27)+1个=28个，剩26个。
   - 第三天：26个，吃掉一半(13)+1个=14个，剩12个。
   - 第四天：12个，吃掉一半(6)+1个=7个，剩5个。
   - 第五天早上：剩5个，符合题意。

因此，第一天共摘了 110 个桃子。选项 D 中的 "1 1 0" 对应数字 110。

9), 如果一棵二叉树只有根结点，那么这棵二叉树的高度为 1。请问高度为 6 的完全二叉树有（ ）种不同的形态？
{{ select(9) }}
- A. 2 4 24
- B. 3 2 32
- C. 4 8 48
- D. 6 4 64

> **答案**: B
> **解析**: 1. 完全二叉树的定义：一棵深度为 h 的二叉树，除第 h 层外，其它各层 (1 到 h-1 层) 的结点数都达到最大个数（即满二叉树的状态），且第 h 层所有的结点都连续集中在最左边。
2. 确定形态的关键：对于高度为 h 的完全二叉树，其前 h-1 层的结构是完全固定的。该树的形态唯一取决于第 h 层（最后一层）结点的个数。
3. 计算第 h 层的结点范围：
   - 第 h 层最少要有 1 个结点（否则高度就变成 h-1 了）。
   - 第 h 层最多可以有 $2^{h-1}$ 个结点（此时为满二叉树）。
4. 具体到本题：
   - 高度 h = 6。
   - 第 6 层最少有 1 个结点。
   - 第 6 层最多有 $2^{6-1} = 2^5 = 32$ 个结点。
5. 结论：由于完全二叉树要求最后一层的结点必须从左到右连续排列，因此第 6 层每增加一个结点，就对应一种唯一的形态。从 1 个结点到 32 个结点，共有 32 种不同的形态。

故正确答案为 B。

10), 二分图是指能将顶点划分成两个部分，每一部分内的顶点之间没有边相连的简单无向图。那么， 16 个顶点的二分图至多有（ ）条边。
{{ select(10) }}
- A. 3 2 32
- B. 4 5 45
- C. 6 4 64
- D. 1 2 8 128

> **答案**: C
> **解析**: 1. 二分图定义：二分图是指可以将图中的顶点分为两个独立的集合 $U$ 和 $V$，使得图中的每一条边都连接一个属于 $U$ 的顶点和一个属于 $V$ 的顶点。
2. 最大边数分析：要使二分图的边数达到最大，该图必须是一个“完全二分图”。在完全二分图中，集合 $U$ 中的每一个顶点都与集合 $V$ 中的每一个顶点有一条边相连。
3. 数学建模：设集合 $U$ 的顶点数为 $n_1$，集合 $V$ 的顶点数为 $n_2$。根据题意，总顶点数 $n_1 + n_2 = 16$。此时，总边数 $E = n_1 \times n_2$。
4. 求最大值：根据数学中的均值不等式，当两个正数的和固定时，这两个数越接近（即差的绝对值越小），它们的乘积就越大。
   - 当 $n_1 = 8, n_2 = 8$ 时，$n_1 + n_2 = 16$，此时边数 $E = 8 \times 8 = 64$。
   - 如果取其他组合，如 $n_1 = 7, n_2 = 9$，则 $E = 7 \times 9 = 63$，小于 64。
5. 结论：16 个顶点的二分图至多有 64 条边。对应选项 C。

11), 一棵二叉树的中序遍历和层次遍历可以唯一的对应一个二叉树形态。现在已知某棵二叉树的中序遍历是 acebfgmdn，它的层次遍历是 abcdemnfg，请问这棵二叉树的后序遍历是（ ）？
{{ select(11) }}
- A. 𝑒 𝑐 𝑔 𝑓 𝑚 𝑛 𝑑 𝑏 𝑎 ecgfmndba
- B. 𝑎 𝑏 𝑐 𝑒 𝑑 𝑚 𝑓 𝑔 𝑛 abcedmfgn
- C. 𝑒 𝑐 𝑓 𝑔 𝑚 𝑛 𝑑 𝑏 𝑎 ecfgmndba
- D. 𝑎 𝑏 𝑑 𝑐 𝑚 𝑛 𝑒 𝑓 𝑔 abdcmnefg

> **答案**: A
> **解析**: 通过二叉树的中序遍历和层次遍历，我们可以唯一确定一棵二叉树。具体推导步骤如下：

1. **确定根节点**：
   层次遍历的第一个元素是 `a`，因此根节点是 `a`。
   在中序遍历中找到 `a`：`a | c e b f g m d n`。
   这说明 `a` 没有左子树，其右子树包含节点 `{c, e, b, f, g, m, d, n}`。

2. **分析 a 的右子树**：
   在层次遍历中，除去 `a` 后，第一个出现在右子树集合中的元素是 `b`。因此 `b` 是 `a` 的右孩子。
   在中序遍历中找到 `b`：`c e | b | f g m d n`。
   这说明 `b` 的左子树是 `{c, e}`，右子树是 `{f, g, m, d, n}`。

3. **分析 b 的左子树 {c, e}**：
   在层次遍历中，`c` 先于 `e` 出现，所以 `c` 是 `b` 的左孩子。
   在中序遍历中，`c` 在 `e` 的左边，且 `c` 后面紧跟 `e`，说明 `e` 是 `c` 的右孩子。

4. **分析 b 的右子树 {f, g, m, d, n}**：
   在层次遍历中，这些节点出现的顺序是 `d, m, n, f, g`。第一个是 `d`，所以 `d` 是 `b` 的右孩子。
   在中序遍历中找到 `d`：`f g m | d | n`。
   这说明 `d` 的左子树是 `{f, g, m}`，右子树是 `{n}`。

5. **分析 d 的子树**：
   - 右侧：只有一个节点 `n`，所以 `n` 是 `d` 的右孩子。
   - 左侧 `{f, g, m}`：在层次遍历中，`m` 最先出现，所以 `m` 是 `d` 的左孩子。在中序遍历中，`f, g` 都在 `m` 的左边，说明 `m` 没有右孩子，其左子树是 `{f, g}`。
   - 分析 `{f, g}`：在层次遍历中 `f` 先出现，是 `m` 的左孩子。在中序遍历中 `g` 在 `f` 右边，所以 `g` 是 `f` 的右孩子。

6. **构建整棵树的结构**：
   - `a` 的右孩子是 `b`
   - `b` 的左孩子是 `c`，右孩子是 `d`
   - `c` 的右孩子是 `e`
   - `d` 的左孩子是 `m`，右孩子是 `n`
   - `m` 的左孩子是 `f`
   - `f` 的右孩子是 `g`

7. **求后序遍历（左-右-根）**：
   - `c` 的子树：(左空, 右e, 根c) -> `e c`
   - `f` 的子树：(左空, 右g, 根f) -> `g f`
   - `m` 的子树：(左{g f}, 右空, 根m) -> `g f m`
   - `d` 的子树：(左{g f m}, 右n, 根d) -> `g f m n d`
   - `b` 的子树：(左{e c}, 右{g f m n d}, 根b) -> `e c g f m n d b`
   - 整棵树：(左空, 右{b子树}, 根a) -> `e c g f m n d b a`

对照选项，正确答案为 A。

13), 在一个n×m的 n 行 m 列的数字矩阵中，将这个矩阵顺时针旋转90∘，那么原矩阵中第 i 行第 j 列（i 和 j 都是从 1 开始计数）在旋转之后的矩阵中对应的行列位置是（ ）
{{ select(13) }}
- A. 第 𝑗 j 行第 𝑖 i 列
- B. 第 𝑗 + 1 j+1 行第 𝑖 − 1 i−1 列
- C. 第 𝑗 j 行第 𝑛 − 𝑖 + 1 n−i+1 列
- D. 第 𝑚 − 𝑗 + 1 m−j+1 行第 𝑖 i 列

> **答案**: C
> **解析**: 在计算机图形学和矩阵运算中，将一个 $n \times m$ 的矩阵顺时针旋转 90°，其坐标变换规律如下：

1. 矩阵维度的变化：原矩阵是 $n$ 行 $m$ 列，旋转后新矩阵变为 $m$ 行 $n$ 列。
2. 行列对应关系推导：
   - 原矩阵的“列”变成了新矩阵的“行”：原矩阵中第 $j$ 列的元素，在旋转后会出现在新矩阵的第 $j$ 行。因此，新位置的行坐标 $i' = j$。
   - 原矩阵的“行”变成了新矩阵的“列”，且顺序发生了翻转：原矩阵的第一行（$i=1$）旋转后变成了新矩阵的最后一列（第 $n$ 列）；原矩阵的最后一行（$i=n$）旋转后变成了新矩阵的第一列（第 1 列）。
   - 根据线性关系，原矩阵的第 $i$ 行对应新矩阵的第 $n - i + 1$ 列。因此，新位置的列坐标 $j' = n - i + 1$。

举例说明：
假设有一个 $2 \times 3$ 的矩阵（$n=2, m=3$）：
a11 a12 a13
a21 a22 a23

顺时针旋转 90° 后变为 $3 \times 2$ 矩阵：
a21 a11
a22 a12
a23 a13

观察原矩阵中第 1 行第 3 列的元素 $a13$（$i=1, j=3$）：
在新矩阵中，它的位置是第 3 行第 2 列。
代入公式：$i' = j = 3$，$j' = n - i + 1 = 2 - 1 + 1 = 2$。完全符合。

因此，正确答案是 C。

   - 数量分别为：红(2面)、黄(2面)、蓝(3面)、绿(3面)。

15), T(n)表示某个算法输入规模为 n 时的运算次数。如果T(1)为常数，且有递归式T(n)=2×T(2n​)+n−3，那么T(n)=
{{ select(15) }}
- A. 𝑂 ( 𝑛 ) O(n)
- B. 𝑂 ( 𝑛 2 ) O(n 2 )
- C. 𝑂 ( log ⁡ 𝑛 ) O(logn)
- D. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn)

> **答案**: D
> **解析**: 1. 识别递归式类型：
题目给出的递归式为 $T(n) = 2 \times T(n/2) + n - 3$（注：题目中的 $2n$ 显然是 $n/2$ 的笔误，在算法复杂度分析中，递归通常是将大问题分解为小问题）。这是一个典型的分治算法递归式，形式为 $T(n) = aT(n/b) + f(n)$。

2. 使用主定理 (Master Theorem) 进行分析：
主定理是解决此类递归方程的快速方法。
- $a = 2$：表示分解成了 2 个子问题。
- $b = 2$：表示每个子问题的规模是原问题的 1/2。
- $f(n) = n - 3$：表示分解和合并子问题所需的额外计算量。

3. 比较 $f(n)$ 与 $n^{\log_b a}$：
- 计算 $n^{\log_b a} = n^{\log_2 2} = n^1 = n$。
- 观察 $f(n) = n - 3$，其渐进复杂度为 $\Theta(n)$。

4. 得出结论：
根据主定理，当 $f(n) = \Theta(n^{\log_b a})$ 时（即两部分阶数相同），递归式的解为：
$T(n) = \Theta(n^{\log_b a} \log n)$
代入本题数据：
$T(n) = \Theta(n \log n)$

5. 归纳验证（代入法）：
如果我们将递归式展开：
$T(n) = 2T(n/2) + n - 3$
$T(n) = 2[2T(n/4) + n/2 - 3] + n - 3 = 4T(n/4) + 2n - 9$
$T(n) = 8T(n/8) + 3n - 21$
...
第 $k$ 层展开后：$T(n) = 2^k T(n/2^k) + k \cdot n - 3(2^k - 1)$
当 $2^k = n$ 时，$k = \log n$：
$T(n) = n \cdot T(1) + n \log n - 3(n - 1)$
在 $n$ 趋向无穷大时，最高阶项为 $n \log n$。

因此，正确答案是 D。

17), （6 分）方程a×b=(aorb)×(aandb)，在a,b都取[0,15]中的整数时，共有（ ）组解。（or 表示按位或运算； and 表示按位与运算）
{{ select(17) }}
- A. 3 2 32
- B. 1 2 8 128
- C. 1 4 6 146
- D. 2 5 6 256

> **答案**: C
> **解析**: 1. **分析位运算的基本性质**：
   对于任意两个整数 $a$ 和 $b$，在二进制表示下，按位或（or）相当于不进位的加法加上重合部分的位，按位与（and）则是重合部分的位。它们满足一个重要的恒等式：
   $a + b = (a \text{ or } b) + (a \text{ and } b)$
   这是因为对于每一位，如果两个数都是 1，则 or 为 1，and 为 1，和为 2；如果一个是 1 一个是 0，则 or 为 1，and 为 0，和为 1；如果都是 0，则 or 为 0，and 为 0，和为 0。这与直接相加的结果在每一位上的贡献是一致的。

2. **结合题目给出的方程**：
   题目要求 $a \times b = (a \text{ or } b) \times (a \text{ and } b)$。
   令 $x = a \text{ or } b$，$y = a \text{ and } b$。
   我们得到了一个方程组：
   - $a + b = x + y$
   - $a \times b = x \times y$

3. **求解方程组**：
   根据韦达定理，如果两个数的和与积确定，那么这两个数就是一个二次方程的两个根。因此，集合 $\{a, b\}$ 必须等于集合 $\{x, y\}$。
   这意味着只有两种可能的情况：
   - 情况 1：$a = x$ 且 $b = y$。即 $a = a \text{ or } b$ 且 $b = a \text{ and } b$。这在二进制上意味着 $b$ 的所有为 1 的位，在 $a$ 中也必须为 1（即 $b$ 是 $a$ 的子集，$b \subseteq a$）。
   - 情况 2：$a = y$ 且 $b = x$。即 $a = a \text{ and } b$ 且 $b = a \text{ or } b$。这在二进制上意味着 $a$ 是 $b$ 的子集（$a \subseteq b$）。

4. **计数过程**：
   $a, b$ 取值范围是 $[0, 15]$，即 4 位二进制数（$0000_2$ 到 $1111_2$）。
   - 计算 $a \subseteq b$ 的情况：对于 4 位中的每一位，$(a_i, b_i)$ 只有 3 种合法组合：$(0,0), (0,1), (1,1)$。因此共有 $3^4 = 81$ 组。
   - 计算 $b \subseteq a$ 的情况：同理，共有 $3^4 = 81$ 组。
   - 排除重复计算：当 $a = b$ 时，既满足 $a \subseteq b$ 又满足 $b \subseteq a$。对于每一位，$(a_i, b_i)$ 只有 2 种组合：$(0,0), (1,1)$。共有 $2^4 = 16$ 组。

5. **最终结果**：
   根据容斥原理，总解数为：$81 + 81 - 16 = 146$。
   选项 C 中的 "1 4 6 146" 对应结果 146。
