1), 若有定义：int a=7; float x=2.5, y=4.7，则表达式x+a%3*(int) (x+y)%2的值是：（ ）
{{ select(1) }}
- A. 0.000000
- B. 2.750000
- C. 2.500000
- D. 3.500000

> **答案**: D
> **解析**: 本题考查 C/C++ 表达式的运算优先级和数据类型转换。

1. 变量初始值：a = 7, x = 2.5, y = 4.7。
2. 表达式：x + a % 3 * (int)(x + y) % 2。
3. 按照运算符优先级进行拆解计算：
   - 第一步（括号与强制类型转换）：计算 (x + y) = 2.5 + 4.7 = 7.2。然后进行强制类型转换 (int)7.2，结果为 7。
   - 第二步（取模运算）：计算 a % 3，即 7 % 3 = 1。
   - 第三步（乘法运算）：将第二步的结果与第一步的结果相乘，即 1 * 7 = 7。
   - 第四步（取模运算）：计算 7 % 2 = 1。
   - 第五步（加法运算）：计算 x + 1，即 2.5 + 1 = 3.5。
4. 结果类型：由于 x 是 float 类型，在与整数 1 相加时会发生隐式类型转换，结果为浮点数 3.5。在 C 语言的格式化输出中，通常保留 6 位小数，即 3.500000。

因此，正确答案是 D。

3), 二进制数11 1011 1001 0111和01 0110 1110 1011进行按位或运算的结果是（ ）编者注：原题为“逻辑或”，但是根据题意应当是按位或。
{{ select(3) }}
- A. 11 1111 1101 1111 11 1111 1101 1111
- B. 11 1111 1111 1101 11 1111 1111 1101
- C. 10 1111 1111 1111 10 1111 1111 1111
- D. 11 1111 1111 1111 11 1111 1111 1111

> **答案**: D
> **解析**: 按位或（Bitwise OR）运算的规则是：在二进制表示下，对应位只要有一个为 1，结果位就为 1；只有当两个位都为 0 时，结果位才为 0。

我们将题目中的两个二进制数对齐进行逐位比较：
第一个数：11 1011 1001 0111
第二个数：01 0110 1110 1011

分段计算如下：
1. 第一组（左起）：11 OR 01 = 11
2. 第二组：1011 OR 0110 = 1111
3. 第三组：1001 OR 1110 = 1111
4. 第四组：0111 OR 1011 = 1111

将各组结果合并，得到：11 1111 1111 1111。

观察选项，选项 D 的内容虽然在排版上重复了一次，但其数值部分 `11 1111 1111 1111` 与我们的计算结果完全一致。其他选项在某些位上出现了 0，不符合按位或的运算结果。因此，正确答案为 D。

5), 设变量 x 为 float 型且已赋值，则以下语句中能将 x 中的数值保留到小数点后两位，并将第三位四舍五入的是（ ）
{{ select(5) }}
- A. x= (x*100+0. 5)/100.0;
- B. x=(int) (x*100+0. 5)/100.0;
- C. x=(x/100+0. 5）*100.0;
- D. x=x*100+0. 5/100. 0;

> **答案**: B
> **解析**: 在 C/C++ 等编程语言中，实现数值的四舍五入并保留特定小数位数的标准算法步骤如下：

1. 放大倍数：要保留两位小数，首先将原数乘以 100，使原本的百分位变为个位，千分位变为十分位。
2. 四舍五入逻辑：在放大后的数值上加上 0.5。根据四舍五入原理，如果原千分位（现在的十分位）大于等于 5，加 0.5 后会向个位进 1；如果小于 5，则不会进位。
3. 取整截断：使用 (int) 进行强制类型转换。这一步至关重要，它会直接丢弃所有的小数部分（即原千分位及之后的所有位数），只保留整数部分。
4. 缩小还原：将取整后的结果除以 100.0，将小数点向左移动两位，恢复到原来的量级。

选项分析：
A. x = (x * 100 + 0.5) / 100.0; 缺少了强制类型转换 (int)，计算结果依然是浮点数，没有截断多余的小数位。
B. x = (int)(x * 100 + 0.5) / 100.0; 正确执行了放大、加0.5、取整截断、缩小的全过程。
C. x = (x / 100 + 0.5) * 100.0; 小数点移动方向错误。
D. x = x * 100 + 0.5 / 100.0; 根据运算符优先级，会先计算 0.5 / 100.0，逻辑完全错误。

因此，正确答案是 B。

6), 由数字1,1,2,4,8,8所组成的不同的 4 位数的个数是（ ）
{{ select(6) }}
- A. 104
- B. 102
- C. 98
- D. 100

> **答案**: B
> **解析**: 这是一道排列组合问题。题目给出的数字集合为 {1, 1, 2, 4, 8, 8}，其中 1 出现了 2 次，8 出现了 2 次，2 和 4 各出现 1 次。我们需要从中选出 4 个数字组成不同的四位数。

我们可以根据选出的数字中是否有重复，分情况讨论：

1. 含有两对相同的数字：
选出的数字只能是 {1, 1, 8, 8}。
排列数：4! / (2! * 2!) = 24 / 4 = 6 种。

2. 恰好含有一对相同的数字：
- 情况 A：重复的数字是 1，即选出 {1, 1, _, _}。剩下的两个位置从 {2, 4, 8} 中选 2 个（注意这里不能再选 8，否则就归入第一种情况了）。
  选择方法：C(3, 2) = 3 种（分别是 {1,1,2,4}, {1,1,2,8}, {1,1,4,8}）。
  每种组合的排列数：4! / 2! = 12 种。
  小计：3 * 12 = 36 种。
- 情况 B：重复的数字是 8，即选出 {8, 8, _, _}。剩下的两个位置从 {1, 2, 4} 中选 2 个。
  选择方法：C(3, 2) = 3 种（分别是 {8,8,1,2}, {8,8,1,4}, {8,8,2,4}）。
  每种组合的排列数：4! / 2! = 12 种。
  小计：3 * 12 = 36 种。
该类总计：36 + 36 = 72 种。

3. 四个数字互不相同：
选出的数字只能是 {1, 2, 4, 8}。
排列数：4! = 24 种。

最后，将所有情况相加：
6 + 72 + 24 = 102 种。

因此，由数字 1,1,2,4,8,8 所组成的不同的 4 位数的个数是 102。正确答案为 B。

7), 排序的算法很多，若按排序的稳定性和不稳定性分类，则（ ）是不稳定排序。
{{ select(7) }}
- A. 冒泡排序
- B. 直接插入排序
- C. 快速排序
- D. 归并排序

> **答案**: C
> **解析**: 在计算机科学中，排序算法的“稳定性”是指：如果待排序的序列中存在两个相等的元素，在排序之后，它们的相对顺序保持不变，则称该算法是稳定的；否则，称该算法是不稳定的。

下面逐一分析选项中的算法：

A. 冒泡排序：通过相邻元素的比较和交换来实现排序。当两个元素相等时，我们不会交换它们的位置，因此它们的相对顺序保持不变。它是稳定的排序算法。

B. 直接插入排序：将每个元素插入到前面已经排好序的序列中。在寻找插入位置时，如果遇到相等的元素，会将新元素插入到相等元素的后面，从而保持相对顺序。它是稳定的排序算法。

C. 快速排序：通过选取一个基准值（pivot），将数组划分为左右两部分。在交换元素的过程中，可能会发生跨越式的交换，这会导致相等元素的相对位置发生改变。例如，序列 [5, 3, 3', 2, 8]，如果以 5 为基准进行划分，后面的 3' 可能会被交换到前面 3 的前面。因此，快速排序是不稳定的排序算法。

D. 归并排序：采用分治法，将序列分成子序列分别排序后再合并。在合并（Merge）两个有序子序列时，如果遇到相等的元素，通常规定先拷贝左侧子序列中的元素，这样就保证了相等元素的相对顺序。它是稳定的排序算法。

综上所述，选项 C 是正确答案。常见的其他不稳定排序还包括：选择排序、希尔排序、堆排序。

8), G 是一个非连通无向图（没有重边和自环），共有 28 条边，则该图至少有（ ）个顶点。
{{ select(8) }}
- A. 10
- B. 9
- C. 11
- D. 8

> **答案**: B
> **解析**: 1. **基本概念**：
   - 一个拥有 $n$ 个顶点的简单无向图（无重边和自环），其最大边数为从 $n$ 个顶点中任选 2 个顶点的组合数，即 $C(n, 2) = \frac{n(n-1)}{2}$。
   - 如果一个图是**非连通**的，意味着它至少包含两个连通分量。

2. **寻找最小顶点数**：
   - 为了使顶点数 $n$ 最小，我们应该尽可能让边分布在较少的顶点之间。
   - 对于一个非连通图，要使边数达到最大，最极端的情况是：其中 $n-1$ 个顶点构成一个完全图（边数最多），而剩下的 1 个顶点是一个孤立点（不与任何顶点相连）。
   - 此时，该非连通图的最大边数为 $C(n-1, 2) = \frac{(n-1)(n-2)}{2}$。

3. **代入计算**：
   - 题目已知边数 $E = 28$。
   - 我们需要找到最小的 $n$，使得 $C(n-1, 2) \ge 28$。
   - $\frac{(n-1)(n-2)}{2} \ge 28$
   - $(n-1)(n-2) \ge 56$
   - 我们知道 $8 \times 7 = 56$，因此当 $n-1 = 8$ 时，不等式成立。
   - 解得 $n = 9$。

4. **验证**：
   - 如果 $n = 8$，最大边数为 $C(8, 2) = 28$。但此时 8 个顶点 28 条边构成的必须是完全图 $K_8$，而完全图是连通的。因此 $n=8$ 不符合“非连通”的条件。
   - 如果 $n = 9$，我们可以取 8 个顶点构成一个完全图（28 条边），第 9 个顶点作为孤立点。此时图是非连通的，且恰好有 28 条边。

因此，该图至少有 9 个顶点。正确答案为 B。

9), 一些数字可以颠倒过来看，例如0,1,8颠倒过来还是本身， 6 颠倒过来是9,9颠倒过来看还是 6 ,其他数字颠倒过来都不构成数字。类似的，一些多位数也可以颠倒过来看，比如 106 颠倒过来是 901。假设某个城市的车牌只有 5 位数字，每一位都可以取 0 到 9。请问这个城市有多少个车牌倒过来恰好还是原来的车牌，并且车牌上的 5 位数能被 3 整除？（ ）
{{ select(9) }}
- A. 40
- B. 25
- C. 30
- D. 20

> **答案**: B
> **解析**: 这道题考查的是组合计数与数字整除特征的综合应用。我们需要找到满足以下两个条件的 5 位数车牌：
1. 颠倒过来（旋转 180 度）后与原车牌相同。
2. 车牌上的 5 位数能被 3 整除。

**第一步：确定哪些数字可以颠倒以及颠倒后的变化**
根据题意，可以颠倒的数字有：
- 0, 1, 8：颠倒后还是本身。
- 6：颠倒后变成 9。
- 9：颠倒后变成 6。
其他数字（2, 3, 4, 5, 7）颠倒后不构成数字。

**第二步：确定对称车牌的结构**
设 5 位车牌的数字分别为 $d_1, d_2, d_3, d_4, d_5$。
要使车牌颠倒后还是原来的车牌，必须满足：
- $d_3$ 颠倒后必须是本身，所以 $d_3 \in \{0, 1, 8\}$（共 3 种选择）。
- $d_1$ 与 $d_5$ 互为颠倒关系，即 $d_1 = f(d_5)$。可能的 $(d_1, d_5)$ 组合有：$(0,0), (1,1), (8,8), (6,9), (9,6)$（共 5 种选择）。
- $d_2$ 与 $d_4$ 互为颠倒关系，即 $d_2 = f(d_4)$。可能的 $(d_2, d_4)$ 组合有：$(0,0), (1,1), (8,8), (6,9), (9,6)$（共 5 种选择）。

**第三步：考虑能被 3 整除的条件**
一个数能被 3 整除，当且仅当它的各位数字之和能被 3 整除。
即：$(d_1 + d_2 + d_3 + d_4 + d_5) \pmod 3 = 0$。
根据对称关系，这可以写成：$((d_1 + d_5) + (d_2 + d_4) + d_3) \pmod 3 = 0$。

我们来分析每一部分的余数情况：
- 对于 $(d_1, d_5)$ 组合，其和 $d_1 + d_5$ 的可能值及余数：
  - (0,0) $\to$ 0 (余 0)
  - (1,1) $\to$ 2 (余 2)
  - (8,8) $\to$ 16 (余 1)
  - (6,9) $\to$ 15 (余 0)
  - (9,6) $\to$ 15 (余 0)
- 对于 $(d_2, d_4)$ 组合，情况相同，余数也有 0, 1, 2 三种可能。
- 对于 $d_3$，其可能值为 $\{0, 1, 8\}$，对应的余数分别为 $\{0, 1, 2\}$。

**第四步：计算总数**
对于任意选定的 $(d_1, d_5)$ 和 $(d_2, d_4)$，它们的和 $(d_1+d_5) + (d_2+d_4)$ 的余数 $R$ 只有三种可能：0, 1 或 2。
为了使总和被 3 整除，我们需要 $R + d_3 \equiv 0 \pmod 3$。
由于 $d_3$ 的三个可选值 $\{0, 1, 8\}$ 恰好对应了三个不同的余数 $\{0, 1, 2\}$，因此无论 $R$ 是多少，在 $\{0, 1, 8\}$ 中都**有且仅有一个** $d_3$ 能满足条件。

因此，总的车牌数量等于选择 $(d_1, d_5)$ 的方法数乘以选择 $(d_2, d_4)$ 的方法数：
$5 \times 5 \times 1 = 25$。

故正确答案为 B。

10), —次期末考试，某班有 15 人数学得满分，有 12 人语文得满分，并且有 4 人语、数都是满分，那么这个班至少有一门得满分的同学有多少人？（ ）
{{ select(10) }}
- A. 23
- B. 21
- C. 20
- D. 22

> **答案**: A
> **解析**: 这道题考察的是数学中的容斥原理（Inclusion-Exclusion Principle）。

根据容斥原理，要求两个集合的并集的大小，公式为：
两个集合的并集大小 = 集合A的大小 + 集合B的大小 - 两个集合的交集大小。

在本题中：
1. 设数学得满分的同学集合为 A，则 |A| = 15 人。
2. 设语文得满分的同学集合为 B，则 |B| = 12 人。
3. 语、数都是满分的同学（即两个集合的交集）为 A ∩ B，则 |A ∩ B| = 4 人。

我们需要计算的是“至少有一门得满分”的同学人数，即 A ∪ B 的大小：
|A ∪ B| = |A| + |B| - |A ∩ B|
|A ∪ B| = 15 + 12 - 4
|A ∪ B| = 27 - 4
|A ∪ B| = 23

所以，这个班至少有一门得满分的同学有 23 人。正确答案是 A。

11), 设 A 和 B 是两个长为 n 的有序数组，现在需要将 A 和 B 合并成一个排好序的数组，问任何以元素比较作为基本运算的归并算法，在最坏情况下至少要做多少次比较？（ ）
{{ select(11) }}
- A. 𝑛 2 n 2
- B. 𝑛 log ⁡ 𝑛 nlogn
- C. 2 𝑛 2n
- D. 2 𝑛 − 1 2n−1

> **答案**: D
> **解析**: 1. **问题背景**：题目要求将两个长度为 $n$ 的有序数组 $A$ 和 $B$ 合并为一个长度为 $2n$ 的有序数组。我们需要寻找在最坏情况下，任何基于元素比较的归并算法所需的最小比较次数。

2. **归并算法原理**：标准的归并操作（如归并排序中的 merge 步骤）使用两个指针，分别指向两个数组的当前最小元素。每次比较两个指针指向的元素，将较小者移入结果数组，并移动该指针。

3. **最坏情况分析**：
   - 在合并过程中，每进行一次比较，就能确定一个元素在最终数组中的位置。
   - 最坏的情况发生在两个数组的元素“交错”排列时。例如：$A = \{1, 3, 5\}$, $B = \{2, 4, 6\}$。
   - 在这种情况下，我们需要不断比较来决定下一个元素。
   - 当合并到只剩最后一个元素时，该元素必然是所有元素中最大的，它直接放在结果数组的最后一位，不需要再进行比较。
   - 因此，为了填满长度为 $2n$ 的数组，在最坏情况下，我们需要确定前 $2n-1$ 个元素的位置，每一次确定都需要一次比较。

4. **计算公式**：
   - 比较次数 = 总元素个数 - 1
   - 即 $n + n - 1 = 2n - 1$。

5. **选项分析**：
   - A ($n^2$)：这是冒泡排序或选择排序等简单排序算法的复杂度，不是归并有序数组的复杂度。
   - B ($n \log n$)：这是对 $n$ 个无序元素进行排序的常见复杂度。
   - C ($2n$)：忽略了最后一个元素不需要比较这一事实。
   - D ($2n - 1$)：符合最坏情况下的逻辑推导。

因此，正确答案是 D。

12), 以下哪个结构可以用来存储图（ ）
{{ select(12) }}
- A. 栈
- B. 二叉树
- C. 队列
- D. 邻接矩阵

> **答案**: D
> **解析**: 在计算机科学中，图（Graph）是一种由顶点（Vertex）和连接顶点的边（Edge）组成的数据结构。为了在程序中存储图，通常使用以下几种标准方法：

13), 以下哪些算法不属于贪心算法？（ ）
{{ select(13) }}
- A. Dijkstra 算法
- B. Floyd 算法
- C. Prim 算法
- D. Kruskal 算法

> **答案**: B
> **解析**: 贪心算法（Greedy Algorithm）是指在对问题求解时，总是做出在当前看来是最好的选择。也就是说，不从整体最优上加以考虑，算法得到的是在某种意义上的局部最优解。

A. Dijkstra 算法：用于求解带权图中的单源最短路径问题。它每次都选择当前距离源点最近的一个未访问节点，并以此节点为基础更新其他节点的距离。这种“每次选局部最优”的策略最终能推导出全局最优解，属于典型的贪心算法。

B. Floyd 算法：用于求解图中所有顶点对之间的最短路径。它的核心思想是通过中间节点 $k$ 的中转，不断更新 $i$ 到 $j$ 的最短路径。其状态转移方程为 $dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j])$。这是一种典型的动态规划（Dynamic Programming）算法，而不是贪心算法。

C. Prim 算法：用于求解图的最小生成树（MST）。它从一个顶点开始，每次选择一条连接“已在生成树中的顶点”与“不在生成树中的顶点”的最短边，属于贪心算法。

D. Kruskal 算法：同样用于求解最小生成树。它将图中所有的边按权值从小到大排序，每次选择权值最小且不与已选边构成回路的边，属于贪心算法。

综上所述，Floyd 算法属于动态规划，不属于贪心算法。因此正确答案是 B。

    *   `trans` 的第 $k$ 位为 1 表示可以取走 $k+1$ 个石子。
    *   `status` 的第 $k$ 位表示 $f[i-(k+1)]$ 的状态（1为胜，0为败）。
    *   `~status` 的第 $k$ 位为 1 则表示 $f[i-(k+1)]$ 是必败态。
    *   `~status & trans` 若不为 0，表示存在某个 $k$，使得规则可用且后继状态为必败态。

5.  **⑤ 处应填 T (status = status << 1 ^ win)**：在计算出当前 $f[i]$（即 `win`）后，需要更新 `status` 以供 $i+1$ 使用。按照定义，`status` 的第 0 位应存储 $f[i]$，第 1 位存储 $f[i-1]$，以此类推。因此将 `status` 左移一位，并将最低位设为 `win`。在 `status << 1` 后最低位必为 0，使用 `^ win` 或 `| win` 效果相同。最后，循环结束后的 `status & 1` 即代表 $f[m]$ 的状态。
