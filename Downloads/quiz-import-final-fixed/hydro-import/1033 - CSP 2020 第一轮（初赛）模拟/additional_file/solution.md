1), 十进制数 114 的相反数的 8 位二进制补码是：
{{ select(1) }}
- A. 10001110
- B. 10001101
- C. 01110010
- D. 01110011

> **答案**: A
> **解析**: 求一个负数的补码通常分为以下三个步骤：

1. 首先求出该数绝对值的二进制表示（即正数 114 的原码）：
   114 除以 2 取余数的过程：
   114 / 2 = 57 ... 0
   57 / 2 = 28 ... 1
   28 / 2 = 14 ... 0
   14 / 2 = 7 ... 0
   7 / 2 = 3 ... 1
   3 / 2 = 1 ... 1
   1 / 2 = 0 ... 1
   将余数逆序排列得到 1110010。补足 8 位，得到 01110010。

2. 求其反码：
   将原码的每一位按位取反（0 变 1，1 变 0）。
   01110010 取反后得到 10001101。

3. 求其补码：
   在反码的基础上加 1。
   10001101 + 1 = 10001110。

因此，十进制数 114 的相反数（-114）的 8 位二进制补码是 10001110。选项 A 正确。

3), 小 A 用字母 A 表示 1， B 表示 2，以此类推，用 26 表示 Z。对于 27 以上的数字，可以用两位或者更长的字符串来对应，例如 AA 对应 27， AB 对应 28， AZ 对应 52， AAA 对应 703 ……那么 BYT 字符串对应的数字是什么？
{{ select(3) }}
- A. 2018
- B. 2020
- C. 2022
- D. 2024

> **答案**: C
> **解析**: 1. 确定进制规律：题目描述的编码方式类似于 26 进制，但与标准 26 进制（0-25）不同，它的每一位取值范围是 1 到 26（A=1, B=2, ..., Z=26）。这是一种“变体 26 进制”（也称为双射 26 进制）。

2. 建立计算公式：
   - 一位字符串：$X = \text{val}(X)$
   - 两位字符串：$XY = \text{val}(X) \times 26^1 + \text{val}(Y)$
   - 三位字符串：$XYZ = \text{val}(X) \times 26^2 + \text{val}(Y) \times 26^1 + \text{val}(Z)$

3. 验证公式：
   - AA = $1 \times 26 + 1 = 27$（符合题意）
   - AAA = $1 \times 26^2 + 1 \times 26 + 1 = 676 + 26 + 1 = 703$（符合题意）

4. 计算 BYT 的值：
   - 首先确定各字母对应的数值：
     - B = 2
     - Y = 25（因为 Z 是 26，Y 是 Z 前一个字母）
     - T = 20（可以通过 A=1, E=5, J=10, O=15, T=20 快速定位）
   - 代入公式计算：
     - $2 \times 26^2 + 25 \times 26^1 + 20 \times 26^0$
     - $= 2 \times 676 + 25 \times 26 + 20$
     - $= 1352 + 650 + 20$
     - $= 2002 + 20$
     - $= 2022$

因此，BYT 字符串对应的数字是 2022。

4), UIM 拍摄了一张照片，其分辨率是4096×2160，每一个像素都是 24 位真彩色。在没有压缩的情况下，这张图片占用空间接近以下哪个值？
{{ select(4) }}
- A. 8MB
- B. 25MB
- C. 200MB
- D. 200KB

> **答案**: B
> **解析**: 计算未压缩图片的占用空间，可以按照以下步骤进行：

1. 计算总像素数：
   图片的像素总数 = 分辨率宽 × 分辨率高 = 4096 × 2160。

2. 计算总位数（bits）：
   每个像素是 24 位真彩色，所以总位数 = 4096 × 2160 × 24 bits。

3. 转换为字节（Bytes）：
   1 字节 = 8 位，所以总字节数 = (4096 × 2160 × 24) / 8 = 4096 × 2160 × 3 Bytes。

4. 转换为兆字节（MB）：
   在计算机存储单位中，1KB = 1024 Bytes，1MB = 1024 KB。
   总 MB 数 = (4096 × 2160 × 3) / (1024 × 1024)
   我们可以简化计算：
   4096 = 4 × 1024
   所以：
   (4 × 1024 × 2160 × 3) / (1024 × 1024)
   = (4 × 2160 × 3) / 1024
   = 25920 / 1024
   ≈ 25.3125 MB

计算结果约为 25.3 MB，与选项 B 最为接近。因此正确答案是 B。

5), 在一个长度为 n 的数组中找到第 k 大的数字，平均的算法时间复杂度最低的是：
{{ select(5) }}
- A. 𝑂 ( 𝑛 ) O(n)
- B. 𝑂 ( 𝑛 𝑘 ) O(nk)
- C. 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn)
- D. 𝑂 ( 𝑛 2 ) O(n 2 )

> **答案**: A
> **解析**: 1. 寻找第 k 大的数字是一个经典的“选择问题”（Selection Problem）。
2. 常见解法及其复杂度分析：
   - 排序法：先对整个数组进行排序（如使用快速排序或归并排序），然后直接访问第 k 个元素。这种方法的时间复杂度为 O(n log n)。
   - 堆排序法：建立一个大小为 k 的小顶堆来维护前 k 大的数。遍历数组的时间复杂度为 O(n log k)。
   - 快速选择算法（QuickSelect）：这是基于快速排序思想的算法。在每一轮分区（Partition）操作后，我们可以确定基准值（pivot）的最终位置。如果该位置恰好是我们要找的第 k 大的位置，则直接返回；否则，我们只需要递归地在其中一半区间继续寻找。
3. 快速选择算法的复杂度：
   - 平均情况下，每次分区都能将搜索范围缩小，其数学期望的时间复杂度为 O(n)。
   - 虽然在最坏情况下（如分区极度不平衡）复杂度会达到 O(n²)，但通过随机化选择基准值，可以使平均复杂度稳定在 O(n)。
4. 比较选项：O(n) 是所有选项中渐进复杂度最低的。因此，正确答案是 A。

6), 对于“树”这种数据结构，正确的有：①一个有 n 个顶点、n−1条边的图是树②一个树中的两个顶点之间有且只有一条简单路径③树中一定存在度数不大于 1 的顶点④树可能存在环
{{ select(6) }}
- A. ①②④
- B. ①②③
- C. ②③
- D. ①②

> **答案**: C
> **解析**: 本题考查对数据结构“树”及其图论性质的理解。我们逐一分析各陈述：

① **一个有 n 个顶点、n-1 条边的图是树**：错误。
虽然“树”一定满足“有 n 个顶点和 n-1 条边”，但反之不一定成立。一个图必须同时满足“连通性”和“无环性”才能被称为树。如果一个图不连通，它可能在某些连通分量中包含环，而总边数仍为 n-1。例如：一个由 3 个顶点组成的环（3 条边）加上 1 个孤立的顶点，总共有 4 个顶点和 3 条边，但它不是树。

② **一个树中的两个顶点之间有且只有一条简单路径**：正确。
这是树的等价定义之一。如果两个顶点之间没有路径，则图不连通；如果存在多条路径，则图中必然存在环。树作为无环连通图，任意两点间有且仅有一条简单路径。

③ **树中一定存在度数不大于 1 的顶点**：正确。
在有限树中，如果顶点数 n=1，该顶点的度数为 0（满足 ≤1）；如果 n≥2，树至少有两个度数为 1 的叶子节点。从平均度数的角度看，树的度数总和为 2(n-1)，平均度数为 (2n-2)/n = 2 - 2/n，由于平均度数小于 2，图中必然存在度数小于 2 的顶点（即度数为 0 或 1 的顶点）。

④ **树可能存在环**：错误。
根据树的定义，树是一个无环连通图。如果存在环，它就不再是树，而是普通的图。

综上所述，正确的陈述为 ② 和 ③。因此，正确选项为 C。

7), 博艾中学进行了一次信息学会考测试，其优、良、及格、不及格的试卷数量分别为10,13,14,5张。现在这些卷子混在一起，要将这些卷子按照等级分为 4 叠。分卷子的方法是，每次将一叠有不同等级答卷的卷子分为两堆，使得这两堆中没有相同等级的卷子，然后可以再分，直到分为 4 叠。要分完这些卷子，至少需要多少次“分卷子”的操作？将一堆数量为 n 的卷子分成两堆，就会产生 n 次分卷子的操作。
{{ select(7) }}
- A. 84
- B. 93
- C. 78
- D. 85

> **答案**: A
> **解析**: 这是一道关于最优二叉树（Huffman树）应用的问题。题目要求将 42 张（10+13+14+5）不同等级的卷子分为 4 叠，每次操作是将一叠卷子分为两堆，且这两堆中不能有相同等级的卷子。操作的代价是当前这叠卷子的数量 $n$。

1. **问题建模**：
   这个问题可以转化为构建一棵以 4 个等级的数量（10, 13, 14, 5）为叶子节点的二叉树。每次“分卷子”的操作对应树的一个内部节点，其代价等于该节点下所有叶子节点的权重之和。总代价即为所有内部节点的权重之和。

2. **计算总代价**：
   总代价也等于每个叶子节点的权重乘以其在树中的深度（即 $\sum w_i d_i$）。
   我们要最小化这个总代价。

3. **比较不同的分法**：
   对于 4 个叶子节点，二叉树的结构主要有两种：
   - **平衡结构（2-2 分法）**：
     先将 4 个等级分为两组，每组 2 个等级。
     例如：将 {10, 5} 和 {13, 14} 分开。
     第一步：将 42 张分为 15 张和 27 张，代价为 42。
     第二步：将 15 张分为 10 张和 5 张，代价为 15。
     第三步：将 27 张分为 13 张和 14 张，代价为 27。
     总代价 = $42 + 15 + 27 = 84$。
     （注意：在平衡结构中，每个等级的深度都是 2，因此总代价始终是 $2 \times (10+13+14+5) = 84$）。

   - **深层结构（1-3 分法）**：
     每次分出一个等级。
     为了使代价最小，应该先分出数量最多的等级。
     第一步：将 42 张分为 {14} 和 {10, 13, 5}，代价为 42。
     第二步：将 28 张（10+13+5）分为 {13} 和 {10, 5}，代价为 28。
     第三步：将 15 张（10+5）分为 {10} 和 {5}，代价为 15。
     总代价 = $42 + 28 + 15 = 85$。

4. **结论**：
   通过比较，平衡结构的代价 84 是最小的。因此，至少需要 84 次操作。

   - G 是 H 的左孩子，下标为 1 * 2 = 2。

   - A 是 D 的左孩子，下标为 3 * 2 = 6。
   - C 是 D 的右孩子，下标为 3 * 2 + 1 = 7。
   - F 是 A 的左孩子，下标为 6 * 2 = 12。
   - E 是 A 的右孩子，下标为 6 * 2 + 1 = 13。

3. **结论：**
   该二叉树结点的最大下标为 13。因此，数组的最大下标至少为 13。

9), 在 C++ 语言中，如果a=1;b=0;c=1;那么以下表达式中为 1 的是：
{{ select(9) }}
- A. a&&b||b&&c
- B. a+b>c||b
- C. !(!c&&(!a||b))
- D. a+b+c

> **答案**: C
> **解析**: 首先，我们已知变量的值为：a = 1, b = 0, c = 1。

接下来逐一分析各选项的表达式：

A. a && b || b && c
- a && b = 1 && 0 = 0
- b && c = 0 && 1 = 0
- 0 || 0 = 0
结果为 0。

B. a + b > c || b
- a + b = 1 + 0 = 1
- 1 > c 即 1 > 1，结果为 0（假）
- 0 || b 即 0 || 0 = 0
结果为 0。

C. !(!c && (!a || b))
- 括号最内层：!a = !1 = 0；b = 0。所以 (!a || b) = (0 || 0) = 0。
- 接着计算：!c = !1 = 0。
- 括号内整体：(!c && 0) = (0 && 0) = 0。
- 最外层取反：!(0) = 1。
结果为 1。

D. a + b + c
- 1 + 0 + 1 = 2
结果为 2。

综上所述，表达式结果为 1 的选项是 C。

10), 在一个初始长度为 n 的链表中连续进行 k 次操作，每次操作是读入两个数字ai​和bi​，在链表中找到元素为ai​的结点（假设一定可以找到），然后将bi​这个元素插入到这个结点前面。在最理想的情况下，链表访问的结点数量最少可能是多少（不算将要插入的结点）？
{{ select(10) }}
- A. 𝑛 n 次
- B. 𝑘 k 次
- C. 𝑛 𝑘 nk 次
- D. 𝑛 + 𝑘 n+k 次

> **答案**: B
> **解析**: 1. **题目分析**：题目要求在长度为 $n$ 的链表中进行 $k$ 次操作。每次操作需要先找到值为 $a_i$ 的结点，然后在它前面插入一个新结点 $b_i$。我们需要计算在“最理想情况下”访问结点的最少数量。

2. **链表查找特性**：在单链表中查找一个特定值的结点，必须从头结点开始逐个遍历。
   - 如果目标结点是头结点，只需访问 1 个结点。
   - 如果目标结点在链表末尾，则需要访问当前链表长度的所有结点。

3. **最理想情况**：
   - “最理想”意味着每次查找的结点 $a_i$ 都在最容易找到的位置，即链表的**头结点**。
   - 第 1 次操作：假设 $a_1$ 是头结点，访问 1 个结点即可找到。插入 $b_1$ 后，$b_1$ 成为新的头结点。
   - 第 2 次操作：假设 $a_2$ 是当前的头结点（即刚插入的 $b_1$），访问 1 个结点即可找到。插入 $b_2$ 后，$b_2$ 成为新的头结点。
   - 以此类推，每次操作如果都能在 1 次访问内找到目标结点，那么 $k$ 次操作总共访问的结点数量就是 $1 \times k = k$。

4. **结论**：无论初始长度 $n$ 是多少，只要每次要找的元素恰好都在表头，每次操作的访问次数都能达到最小值 1。因此 $k$ 次操作的最少访问总数为 $k$。

故正确答案为 B。

11), A 班有 5 名风纪委员，B 班有 4 名风纪委员，C 班有 3 名风纪委员。现在需要这些同学中选取 6 名风纪委员巡逻，如果只关注各班派出的风纪委员人数，有几种不同的方案？
{{ select(11) }}
- A. 9
- B. 12
- C. 15
- D. 18

> **答案**: D
> **解析**: 这是一道关于组合数学中“有限制的整数拆分”问题。题目要求从 A、B、C 三个班级中选出 6 名风纪委员，且只关注各班派出的人数。

1. **建立数学模型**：
   设 A 班派出 $a$ 人，B 班派出 $b$ 人，C 班派出 $c$ 人。
   根据题目条件，我们可以列出以下约束条件：
   - $a + b + c = 6$ （总人数为 6）
   - $0 \le a \le 5$ （A 班最多 5 人）
   - $0 \le b \le 4$ （B 班最多 4 人）
   - $0 \le c \le 3$ （C 班最多 3 人）
   我们需要找到满足这些条件的非负整数解 $(a, b, c)$ 的个数。

2. **分类讨论（枚举法）**：
   我们可以按照 $a$ 的取值进行分类讨论：
   - 当 $a = 5$ 时，$b + c = 1$。可能的组合有：(5, 1, 0), (5, 0, 1)，共 2 种。
   - 当 $a = 4$ 时，$b + c = 2$。可能的组合有：(4, 2, 0), (4, 1, 1), (4, 0, 2)，共 3 种。
   - 当 $a = 3$ 时，$b + c = 3$。可能的组合有：(3, 3, 0), (3, 2, 1), (3, 1, 2), (3, 0, 3)，共 4 种。
   - 当 $a = 2$ 时，$b + c = 4$。由于 $b \le 4$ 且 $c \le 3$，可能的组合有：(2, 4, 0), (2, 3, 1), (2, 2, 2), (2, 1, 3)，共 4 种。（注意：$c$ 不能取 4）
   - 当 $a = 1$ 时，$b + c = 5$。由于 $b \le 4$ 且 $c \le 3$，可能的组合有：(1, 4, 1), (1, 3, 2), (1, 2, 3)，共 3 种。
   - 当 $a = 0$ 时，$b + c = 6$。由于 $b \le 4$ 且 $c \le 3$，可能的组合有：(0, 4, 2), (0, 3, 3)，共 2 种。

3. **计算总数**：
   总方案数 = 2 + 3 + 4 + 4 + 3 + 2 = 18 种。

因此，正确的选项是 D。

12), 以下哪种排序算法的时间复杂度是O(n2)？
{{ select(12) }}
- A. 计数排序
- B. 插入排序
- C. 希尔排序
- D. 归并排序

> **答案**: B
> **解析**: 1. 插入排序（Insertion Sort）：其基本思想是将待排序的元素逐个插入到已经排好序的部分中。在最坏情况下（待排序序列是逆序的）和平均情况下，都需要进行两层循环嵌套，比较和移动次数与 n 的平方成正比，因此其时间复杂度为 O(n²)。
2. 计数排序（Counting Sort）：这是一种非比较排序算法，通过统计每个元素出现的次数来确定其位置。其时间复杂度为 O(n + k)，其中 k 是数据的范围大小，通常被视为线性复杂度。
3. 希尔排序（Shell Sort）：是插入排序的改进版，通过设置增量序列进行分组插入排序。其时间复杂度取决于增量序列的选择，通常优于 O(n²)，平均复杂度一般在 O(n^1.3) 左右。
4. 归并排序（Merge Sort）：采用分治策略，将序列递归地分成两半分别排序再合并。无论在最好、最坏还是平均情况下，其时间复杂度始终保持在 O(n log n)。

综上所述，选项中时间复杂度为 O(n²) 的算法是插入排序。

13), 已知rand（ ）可以生成一个 0 到 32767 的随机整数，如果希望得到一个范围在[a,b)的随机整数， a 和 b 均是不超过 100 的正整数且a<b，那么可行的表达式是什么？
{{ select(13) }}
- A. (rand（ ）%(b-a))+a
- B. (rand（ ）%(b-a+1))+a
- C. (rand（ ）%(b-a))+a+1
- D. (rand（ ）%(b-a+1))+a+1

> **答案**: A
> **解析**: 1. 明确目标范围：题目要求生成的随机整数范围是 $[a, b)$。在数学区间表示中，$[a, b)$ 表示包含左端点 $a$，但不包含右端点 $b$ 的连续整数集合。也就是说，生成的整数 $x$ 必须满足 $a \le x \le b-1$。

2. 计算随机数的个数：从 $a$ 到 $b-1$（包含端点）共有 $(b-1) - a + 1 = b - a$ 个整数。

3. 利用取模运算限制范围：
   - `rand()` 函数生成一个 $[0, 32767]$ 之间的随机整数。
   - `rand() % N` 的结果范围是 $[0, N-1]$。
   - 为了产生 $b-a$ 个不同的随机值，我们需要对 $b-a$ 取模，即 `rand() % (b - a)`。此时得到的结果范围是 $[0, b - a - 1]$。

4. 平移区间：
   - 我们现在有了范围在 $[0, b - a - 1]$ 的随机数，但目标范围是 $[a, b - 1]$。
   - 通过在表达式后面加上 $a$，可以将范围整体向右平移 $a$ 个单位。
   - 表达式变为：`(rand() % (b - a)) + a`。
   - 此时最小值为 $0 + a = a$，最大值为 $(b - a - 1) + a = b - 1$。这正好对应了区间 $[a, b)$。

5. 选项分析：
   - A: `(rand() % (b - a)) + a`。生成的范围是 $[a, b-1]$，即 $[a, b)$。正确。
   - B: `(rand() % (b - a + 1)) + a`。生成的范围是 $[a, b]$，即 $[a, b+1)$。错误。
   - C: `(rand() % (b - a)) + a + 1`。生成的范围是 $[a+1, b]$。错误。
   - D: `(rand() % (b - a + 1)) + a + 1`。生成的范围是 $[a+1, b+1]$。错误。

14), 一个 7 个顶点的完全图需要至少删掉多少条边才能变为森林？
{{ select(14) }}
- A. 16
- B. 21
- C. 15
- D. 6

> **答案**: C
> **解析**: 1. 理解基本概念：
   - 完全图（Complete Graph）：在一个 7 个顶点的完全图中，任意两个顶点之间都有一条边。
   - 森林（Forest）：一个没有环的图。森林中的每一个连通分量都是一棵树。
   - 树（Tree）：一个有 n 个顶点且只有 n-1 条边的连通无环图。

2. 计算完全图的边数：
   - 对于一个拥有 n 个顶点的完全图，其边数公式为 n * (n - 1) / 2。
   - 当 n = 7 时，边数 = 7 * (7 - 1) / 2 = 7 * 6 / 2 = 21 条边。

3. 计算森林的最大边数：
   - 森林是由若干棵树组成的。要使删掉的边最少，意味着我们要让剩下的森林拥有尽可能多的边。
   - 在顶点数固定的情况下，当森林只有一棵树（即连通图）时，边数最多。
   - 对于 n 个顶点的树，边数固定为 n - 1。
   - 当 n = 7 时，最大边数 = 7 - 1 = 6 条边。

4. 得出结论：
   - 需要删掉的边数 = 总边数 - 森林剩余的最大边数。
   - 需要删掉的边数 = 21 - 6 = 15 条边。

因此，至少需要删掉 15 条边才能使 7 个顶点的完全图变为森林。正确答案为 C。

    *   当遍历到第 $i$ 题时，意味着前 $i-1$ 题被烧毁，即 $K = i-1$。
    *   剩余题目范围是 $[i, n]$，题目总数为 $n-i+1$。
    *   去掉一个最小值后，参与计算平均分的题目数量为 $(n-i+1) - 1 = n-i$。
    *   平均分 = (后缀总和 - 后缀最小值) / 剩余题目数。使用 `(double)` 强制转换以确保浮点数除法精度。
4.  **④处应填 O (cnt=1;k[cnt]=i-1;)**：当计算出的当前平均分大于之前的最大平均分 `maxAverage` 时，需要更新 `maxAverage`，并将记录结果的计数器 `cnt` 重置为 1，记录当前的 $K$ 值（即 $i-1$）。
5.  **⑤处应填 R (k[++cnt]=i-1;)**：当计算出的平均分等于当前最大平均分时，说明找到了另一个符合条件的 $K$，将计数器 `cnt` 加 1，并将当前的 $K$ 值（$i-1$）存入数组 `k` 中。

注意：由于循环是从 $i=n$ 递减到 $2$ 的，记录的 $K$ 值（$i-1$）是按降序排列的。为了满足题目“升序输出”的要求，程序最后在输出数组 `k` 时应采用逆序遍历。
