# 1008 - NOIP 2010 提高组初赛试题 - 题目与解析

1), 与十六进制数 A1.2 等值的十进制数是（ ）
{{ select(1) }}
- A. 101.2
- B. 111.4
- C. 161.125
- D. 177.25

> **答案**: C
> **解析**: 将十六进制数转换为十进制数，需要按照“按权展开求和”的方法。十六进制的基数为 16。

对于十六进制数 A1.2，我们可以将其分为整数部分和小数部分分别计算：

1. 整数部分 (A1)：
   - “A”在十六进制中代表十进制的 10，它位于从右往左数第二位（十六位），权值为 16^1。
   - “1”位于个位，权值为 16^0。
   - 计算：10 × 16^1 + 1 × 16^0 = 160 + 1 = 161。

2. 小数部分 (0.2)：
   - “2”位于小数点后第一位，权值为 16^-1（即 1/16）。
   - 计算：2 × 16^-1 = 2 × (1/16) = 2/16 = 1/8。
   - 将分数 1/8 转换为十进制小数：1 ÷ 8 = 0.125。

3. 合并结果：
   - 161 + 0.125 = 161.125。

因此，十六进制数 A1.2 等值的十进制数是 161.125。正确选项为 C。

2), 一个字节（byte）由（ ）个二进制组成。
{{ select(2) }}
- A. 8
- B. 16
- C. 32
- D. 以上都有可能

> **答案**: A
> **解析**: 在计算机科学中，字节（Byte）是计算机信息技术用于计量存储容量的一种计量单位。
1. 基本定义：1个字节（Byte）固定由8个二进制位（bit）组成。
2. 换算关系：1 Byte = 8 bits。
3. 作用：字节是计算机存储空间的基本计量单位。例如，一个ASCII字符通常占用1个字节，而一个中文字符（在UTF-8编码下）通常占用3个字节。

因此，选项A是正确答案。选项B（16位）通常被称为“字”（Word，在16位系统中）或双字节；选项C（32位）通常被称为“双字”（DWord）或四字节。

3), 以下逻辑表达式的值恒为真的是（ ）
{{ select(3) }}
- A. P∨（┓P∧Q）∨(┓P∧┓Q)
- B. Q∨（┓P∧Q）∨（P∧┓Q）
- C. P∨Q∨（P∧┓Q）∨（┓P∧Q）
- D. P∨┓Q∨（P∧┓Q）∨(┓P∧┓Q)

> **答案**: A
> **解析**: 本题考查逻辑表达式的化简与逻辑代数的基本定律。我们需要找到一个表达式，无论变量 P 和 Q 取什么值（真或假），其结果始终为真。

我们逐一分析各选项：

A 选项：P ∨ (┓P ∧ Q) ∨ (┓P ∧ ┓Q)
1. 观察后两项：(┓P ∧ Q) ∨ (┓P ∧ ┓Q)。根据分配律的逆向应用（提取公因子 ┓P），可以化简为：┓P ∧ (Q ∨ ┓Q)。
2. 根据排中律，Q ∨ ┓Q 的值恒为真（1）。
3. 因此，后两项简化为 ┓P ∧ 1，即 ┓P。
4. 整个表达式简化为 P ∨ ┓P。
5. 根据排中律，P ∨ ┓P 的值恒为真。
因此，A 选项是恒为真的。

B 选项：Q ∨ (┓P ∧ Q) ∨ (P ∧ ┓Q)
1. 根据吸收律，Q ∨ (┓P ∧ Q) 等于 Q。
2. 表达式简化为 Q ∨ (P ∧ ┓Q)。
3. 根据分配律，这等于 (Q ∨ P) ∧ (Q ∨ ┓Q)，即 (Q ∨ P) ∧ 1，最终结果为 Q ∨ P。
4. 当 P 和 Q 都为假时，结果为假，因此不是恒为真。

C 选项：P ∨ Q ∨ (P ∧ ┓Q) ∨ (┓P ∧ Q)
1. 根据吸收律，P ∨ (P ∧ ┓Q) 等于 P。
2. 根据吸收律，Q ∨ (┓P ∧ Q) 等于 Q。
3. 表达式简化为 P ∨ Q。
4. 当 P 和 Q 都为假时，结果为假，因此不是恒为真。

D 选项：P ∨ ┓Q ∨ (P ∧ ┓Q) ∨ (┓P ∧ ┓Q)
1. 根据吸收律，P ∨ (P ∧ ┓Q) 等于 P。
2. 根据吸收律，┓Q ∨ (┓P ∧ ┓Q) 等于 ┓Q。
3. 表达式简化为 P ∨ ┓Q。
4. 当 P 为假且 Q 为真时，结果为假，因此不是恒为真。

综上所述，只有 A 选项的逻辑表达式值恒为真。

4), Linux 下可执行文件的默认扩展名是（ ）
{{ select(4) }}
- A. exe
- B. com
- C. dll
- D. 以上都不是

> **答案**: D
> **解析**: 在 Linux 操作系统中，文件是否可执行并不依赖于文件的后缀名（扩展名），而是取决于文件的权限属性。

1. 权限机制：Linux 使用文件权限位来控制文件的操作。一个文件如果具有“可执行”权限（通常通过 chmod +x 命令设置），系统就可以尝试运行它。
2. 扩展名的作用：在 Linux 中，扩展名（如 .sh, .py, .c）主要用于方便用户识别文件类型，对系统内核来说并不是强制要求的。
3. 选项分析：
   - A 选项 .exe 是 Windows 系统下的可执行文件后缀。
   - B 选项 .com 是早期 DOS 和 Windows 系统下的可执行文件后缀。
   - C 选项 .dll 是 Windows 系统下的动态链接库文件后缀（在 Linux 中对应的通常是 .so 文件）。
4. 结论：由于 Linux 下可执行文件没有默认的、强制性的扩展名，因此正确答案是 D。

5), 如果在某个进制下等式7*7=41成立，那么在该进制下等式12*12=（ ）也成立。
{{ select(5) }}
- A. 100
- B. 144
- C. 164
- D. 196

> **答案**: B
> **解析**: 1. **分析题目背景**：
   题目中的等式 $77=41$ 在十进制下显然不成立。根据此类编程竞赛题目的常见规律，这通常代表在某个 $x$ 进制下，乘法等式 $7 \times 7 = 41$ 成立（题目中可能省略了乘号）。

2. **确定进制**：
   设该进制为 $x$。
   将等式 $7 \times 7 = 41$ 的两边都转换为十进制：
   左边：$7 \times 7 = 49$
   右边：$4 \times x^1 + 1 \times x^0 = 4x + 1$
   建立方程：$49 = 4x + 1$
   解方程：$4x = 48$，得 $x = 12$。
   因此，该等式是在 **十二进制** 下成立的。

3. **计算目标等式**：
   题目要求在该进制下计算 $12 \times 12$ 的结果（同样，这里应理解为 $12 \times 12$ 的乘法运算）。
   
   **方法一：数值转换法**
   - 将十二进制的 $12$ 转换为十进制：$1 \times 12 + 2 = 14$。
   - 在十进制下计算乘积：$14 \times 14 = 196$。
   - 将十进制的 $196$ 转回十二进制：
     $196 \div 12 = 16 \dots 4$
     $16 \div 12 = 1 \dots 4$
     $1 \div 12 = 0 \dots 1$
     逆序排列余数，得到十二进制表示为 $144$。

   **方法二：多项式展开法**
   - 在 $x$ 进制下，数字字符串 "12" 可以表示为 $(x + 2)$。
   - 计算 $(x + 2) \times (x + 2) = x^2 + 4x + 4$。
   - 当进制 $x > 4$ 时，多项式 $1 \cdot x^2 + 4 \cdot x^1 + 4 \cdot x^0$ 直接对应进制下的数字字符串 "144"。
   - 因为我们求得 $x = 12$，满足条件，所以结果为 $144$。

4. **结论**：
   在该进制下，等式 $12 \times 12 = 144$ 成立。对应选项 B。

6), 提出“存储程序”的计算机工作原理的是（ ）
{{ select(6) }}
- A. 克劳德•香农
- B. 戈登•摩尔
- C. 查尔斯•巴比奇
- D. 冯•诺依曼

> **答案**: D
> **解析**: 冯·诺依曼（John von Neumann）在1945年发表的《关于EDVAC的报告草案》中，明确提出了“存储程序控制”的原理。这一原理的核心思想是将程序指令和数据以同等地位存储在计算机的存储器中，计算机能够自动地从存储器中取出指令并执行。这种架构被称为“冯·诺依曼架构”，是现代计算机设计的基石，因此他也被称为“现代计算机之父”。

其他选项解析：
A. 克劳德·香农（Claude Shannon）：信息论的奠基人，提出了信息熵的概念。
B. 戈登·摩尔（Gordon Moore）：英特尔创始人之一，提出了著名的“摩尔定律”（集成电路上可容纳的晶体管数目约每隔18-24个月便会增加一倍）。
C. 查尔斯·巴比奇（Charles Babbage）：设计了差分机和分析机，被誉为“通用计算机之父”，但他的设计属于机械计算机时代，并非现代意义上的电子存储程序计算机。

7), 前缀表达式+3*2+512的值是（ ）
{{ select(7) }}
- A. 23
- B. 25
- C. 37
- D. 65

> **答案**: C
> **解析**: 前缀表达式（又称波兰式）的求值规律是从右向左扫描表达式。在求值过程中，我们需要根据运算符的数量来确定操作数的划分。

1. **分析表达式结构**：
   题目给出的表达式是 `+ 3 * 2 + 5 12`。
   这里有 3 个二元运算符（`+`, `*`, `+`），根据二元运算的规则，3 个运算符需要 4 个操作数。
   观察数字序列 `3, 2, 5, 12`，正好符合 4 个操作数的要求（如果看作 5, 1, 2 则有 5 个操作数，运算符数量将不足）。

2. **从右向左计算步骤**：
   - 第一步：扫描到最右边的运算符 `+` 及其相邻的两个操作数 `5` 和 `12`。计算 `5 + 12 = 17`。此时表达式简化为 `+ 3 * 2 17`。
   - 第二步：继续向左扫描，遇到运算符 `*` 及其相邻的两个操作数 `2` 和 `17`。计算 `2 * 17 = 34`。此时表达式简化为 `+ 3 34`。
   - 第三步：最后计算剩下的运算符 `+` 及其操作数 `3` 和 `34`。计算 `3 + 34 = 37`。

3. **栈式求值过程验证**：
   - 遇到 12：入栈 [12]
   - 遇到 5：入栈 [12, 5]
   - 遇到 +：弹出 5 和 12，计算 5 + 12 = 17，结果入栈 [17]
   - 遇到 2：入栈 [17, 2]
   - 遇到 *：弹出 2 和 17，计算 2 * 17 = 34，结果入栈 [34]
   - 遇到 3：入栈 [34, 3]
   - 遇到 +：弹出 3 和 34，计算 3 + 34 = 37，结果入栈 [37]

最终结果为 37，故选 C。

8), 主存储器的存取速度比中央处理器（CPU）的工作速度慢的多，从而使得后者的效率受到影响。而根据局部性原理，CPU 所访问的存储单元通常都趋于一个较小的连续区域中。于是，为了提高系统整体的执行效率，在 CPU 中引入了（ ）
{{ select(8) }}
- A. 寄存器
- B. 高速缓存
- C. 闪存
- D. 外存

> **答案**: B
> **解析**: 本题考查计算机体系结构中的存储层次结构。

1. 背景矛盾：中央处理器（CPU）的运算速度极快，而主存储器（RAM）的存取速度相对较慢。如果 CPU 每次都直接从主存中读取数据，会因为等待数据而导致效率大幅下降。
2. 局部性原理：研究发现，程序在执行时具有“局部性原理”，即在一段时间内，CPU 所访问的存储单元往往集中在某个较小的连续区域（空间局部性），或者某些数据会被反复访问（时间局部性）。
3. 解决方案：为了弥补 CPU 与主存之间的速度差异，在 CPU 中引入了高速缓存（Cache）。高速缓存的存取速度接近 CPU，容量较小但成本较高。它利用局部性原理，预先将 CPU 可能用到的数据从主存加载到缓存中。当 CPU 需要数据时，先在缓存中查找，如果命中则直接读取，从而显著提高系统整体的执行效率。

选项分析：
A. 寄存器：位于 CPU 内部，速度最快，但数量极少，主要用于存放指令、数据地址和中间运算结果，不是为了解决主存速度慢而专门引入的缓存层。
B. 高速缓存：正确答案，专门用于解决 CPU 与主存速度不匹配的问题。
C. 闪存：属于非易失性存储器（如 U 盘、SSD），速度远慢于主存。
D. 外存：如硬盘，用于长期保存数据，速度最慢。

9), 完全二叉树的顺序存储方案，是指将完全二叉树的结点从上到下、从左到右依次存放到一个顺序结构的数组中。假定根结点存放在数组的 1 号位置上，则第 k 号结点的父结点如果存在的话，应当存放在数组中的（ ）号位置。
{{ select(9) }}
- A. 2 𝑘 2k
- B. 2 𝑘 + 1 2k+1
- C. ⌊ 𝑘 2 ⌋ ⌊ 2 k ​ ⌋
- D. 𝑘 + 1 2 2 k+1 ​

> **答案**: C
> **解析**: 在完全二叉树的顺序存储结构中，如果根结点存储在数组的 1 号位置，则结点编号与它们在树中的逻辑关系有如下规律：

1. 对于编号为 $i$ 的结点：
   - 如果它有左孩子，左孩子的编号为 $2i$。
   - 如果它有右孩子，右孩子的编号为 $2i + 1$。
2. 对于编号为 $k$ 的结点（$k > 1$）：
   - 它的父结点编号是 $k$ 除以 2 并向下取整，即 $\lfloor k/2 \rfloor$。

举例说明：
- 如果 $k=2$（根结点的左孩子），其父结点编号为 $\lfloor 2/2 \rfloor = 1$。
- 如果 $k=3$（根结点的右孩子），其父结点编号为 $\lfloor 3/2 \rfloor = 1$。
- 如果 $k=4$（编号为 2 的结点的左孩子），其父结点编号为 $\lfloor 4/2 \rfloor = 2$。
- 如果 $k=5$（编号为 2 的结点的右孩子），其父结点编号为 $\lfloor 5/2 \rfloor = 2$。

因此，选项 C 正确。选项 A 和 B 分别是求左、右孩子的位置，选项 D 的公式不符合向下取整的通用规律。

11), 元素R1​,R2​,R3​,R4​,R5​入栈的顺序为R1​,R2​,R3​,R4​,R5​。如果第 1 个出栈的是R3​，那么第 5 个出栈的可能是（ ）
{{ select(11) }}
- A. 𝑅 1 R 1 ​
- B. 𝑅 2 R 2 ​
- C. 𝑅 4 R 4 ​
- D. 𝑅 5 R 5 ​

> **答案**: B
> **解析**: 这道题考查的是栈（Stack）这一数据结构的基本性质——后进先出（LIFO, Last In First Out）。

1. **分析入栈和出栈状态**：
   - 元素入栈顺序为 $R_1, R_2, R_3, R_4, R_5$。
   - 题目指出第 1 个出栈的元素是 $R_3$。这意味着在 $R_3$ 出栈时，$R_1$ 和 $R_2$ 已经按顺序进入了栈中。
   - 此时，栈内的状态从底向上依次为：$R_1, R_2$。

2. **分析后续出栈的可能性**：
   - 由于栈是“后进先出”的，在栈内的元素中，$R_2$ 位于 $R_1$ 的上方。因此，只要 $R_1$ 和 $R_2$ 都在栈内，$R_2$ 必须在 $R_1$ 之前出栈。
   - 换句话说，在完整的出栈序列中，$R_1$ 出现的位置一定在 $R_2$ 之后。

3. **验证第 5 个出栈的元素**：
   - 如果第 5 个（即最后一个）出栈的元素是 $R_2$，那么根据“$R_1$ 必须在 $R_2$ 之后出栈”的原则，$R_1$ 必须是第 6 个出栈的元素。
   - 但题目中总共只有 5 个元素，因此 $R_2$ 不可能是第 5 个出栈的元素。

4. **验证其他选项**：
   - **A 选项 ($R_1$)**：可能是第 5 个。序列如：$R_3, R_2, R_4, R_5, R_1$。
   - **C 选项 ($R_4$)**：可能是第 5 个。序列如：$R_3, R_2, R_1, R_5, R_4$。
   - **D 选项 ($R_5$)**：可能是第 5 个。序列如：$R_3, R_2, R_1, R_4, R_5$。

**结论**：
在 GESP/CSP 等编程考级真题中，此题的题干通常为“第 5 个出栈的不可能是”。根据逻辑分析，只有 $R_2$ 是绝对不可能作为第 5 个出栈元素的。因此，正确答案选 B。

12), Pascal 语言，C 语言和 C++ 语言都属于（ ）
{{ select(12) }}
- A. 高级语言
- B. 自然语言
- C. 解释性语言
- D. 编译性语言

> **答案**: A
> **解析**: 1. **计算机语言的分类**：计算机语言按照发展历程和抽象程度，通常分为三大类：
   - **机器语言**：由二进制代码（0 和 1）组成，计算机硬件可以直接识别和执行，但人类难以阅读和编写。
   - **汇编语言**：使用助记符（如 ADD, MOV）来代替机器指令，比机器语言易于理解，但仍依赖于具体的硬件架构。
   - **高级语言**：语法接近自然语言（英语）和数学表达式，不依赖于具体的机器硬件，具有良好的可读性和可移植性。

2. **题目分析**：
   - **Pascal**：一种结构化编程语言，由尼古拉斯·沃斯设计，主要用于教学。
   - **C 语言**：一种通用的、过程式的编程语言，广泛应用于系统软件开发。
   - **C++ 语言**：在 C 语言的基础上增加了面向对象特性的编程语言。
   这三种语言都属于**高级语言**。

3. **选项辨析**：
   - **A. 高级语言**：正确。这是它们最基本的分类属性。
   - **B. 自然语言**：错误。自然语言是指人类日常交流使用的语言，如汉语、英语等。
   - **C. 解释性语言**：错误。Pascal、C 和 C++ 通常都是通过编译器将源代码一次性翻译成机器目标代码，属于编译型语言，而不是像 Python 或 JavaScript 那样逐行解释执行。
   - **D. 编译性语言**：虽然这三种语言确实都属于编译性语言，但在计算机基础知识的层级划分中（机器-汇编-高级），“高级语言”是更上位、更通用的分类标签。在 GESP/CSP 等基础等级考试中，此类题目通常考察的是对高级语言这一概念的认知。

因此，正确答案是 A。

14), 在整数的补码表示法中，以下说法正确的是（ ）
{{ select(14) }}
- A. 只有负整数的编码最高位为 1 1
- B. 在编码的位数确定后，所能表示的最小整数和最大整数的绝对值相同
- C. 整数 0 0 只有一个唯一的编码
- D. 两个用补码表示的数相加时，如果在最高位产生进位，则表示运算溢出

> **答案**: C
> **解析**: 本题考查计算机中整数的补码（Two's Complement）表示法及其特性。

A 选项：在补码表示中，最高位（符号位）为 1 的确表示负数，为 0 表示非负数（正数和 0）。虽然这个说法在逻辑上是正确的，但它并不是补码特有的优点（原码和反码的负数符号位也为 1）。在编程考试中，通常强调的是补码解决了 0 的编码问题。

B 选项：错误。在补码表示法中，由于 0 的编码唯一，其表示范围是不对称的。对于 $n$ 位二进制数，补码能表示的范围是 $-2^{n-1}$ 到 $2^{n-1}-1$。例如，8 位补码的范围是 $-128$ 到 $127$。最小整数的绝对值是 $128$，最大整数的绝对值是 $127$，两者并不相同。

C 选项：正确。这是补码最核心的特征之一。在原码和反码中，整数 0 都有两种表示方式（+0 和 -0），这给电路设计（如判断是否为 0）带来了复杂性。而补码通过溢出处理，使得 0 只有唯一的编码（所有位均为 0），简化了运算器的设计。

D 选项：错误。在补码加法运算中，最高位产生进位（Carry Out）并不代表溢出（Overflow）。溢出的判断标准通常是：两个符号相同的数相加，结果的符号与加数相反；或者看“符号位的进位”与“最高数值位的进位”是否一致（不一致则溢出）。例如，在 4 位补码中，$-1 + (-1)$ 即 $1111 + 1111 = 11110$，最高位产生了进位 1，但结果 $1110$（即 $-2$）是正确的，并未溢出。

因此，正确答案是 C。

15), 一颗二叉树的前序遍历序列是 ABCDEFG，后序遍历序列是 CBFEGDA，则根结点的左子树的结点个数可能是（ ）
{{ select(15) }}
- A. 0
- B. 2
- C. 4
- D. 6

> **答案**: B
> **解析**: 1. **确定根结点**：根据二叉树遍历的性质，前序遍历（根-左-右）的第一个字符是根，后序遍历（左-右-根）的最后一个字符是根。因此，根结点是 A。

2. **划分左右子树**：
   - 在前序序列 `A B C D E F G` 中，A 后面紧跟着 B。如果左子树不为空，B 一定是左子树的根。
   - 在后序序列 `C B F E G D A` 中，A 前面紧跟着 D。如果右子树不为空，D 一定是右子树的根。
   - 在后序遍历中，左子树的所有结点必须出现在右子树所有结点之前，且左子树的序列必须以其自身的根结点（即 B）结束。

3. **分析后序序列**：
   - 观察后序序列 `C B F E G D A`，我们可以发现 B 出现在第二个位置。
   - 因为 B 是左子树的根，在后序遍历中它必须是左子树序列的最后一个。所以左子树的后序序列只能是 `C B`。
   - 剩下的 `F E G D` 则是右子树的后序序列（以右子树根结点 D 结尾）。

4. **验证结点个数**：
   - 左子树的结点集合为 {B, C}，个数为 2。
   - 右子树的结点集合为 {D, E, F, G}，个数为 4。
   - 验证前序：左子树前序为 `B C`，右子树前序为 `D E F G`。拼接起来 `A + (B C) + (D E F G)` 正好是 `ABCDEFG`。
   - 验证后序：左子树后序为 `C B`，右子树后序为 `F E G D`。拼接起来 `(C B) + (F E G D) + A` 正好是 `CBFEGDA`。

因此，根结点的左子树结点个数是 2。

16), 在下列 HTML 语句中，可以正确产生一个指向 NOI 官方网站的超链接的是（ ）
{{ select(16) }}
- A. <a url="http://www.noi.cn">欢迎访问NOI网站</a>
- B. <a href="http://www.noi.cn">欢迎访问NOI网站</a>
- C. <a>http://www.noi.cn</a>
- D. <a name"http://www.noi.cn">欢迎访问NOI网站</a>

1. <a> 标签：代表“锚点”（anchor），用于定义超链接。
2. href 属性：全称是 Hypertext Reference（超文本引用），其值就是链接要跳转到的目标 URL。

17), 关于拓扑排序，下列说法正确的是（ ）
{{ select(17) }}
- A. 所有连通的有向图都可以实现拓扑排序
- B. 对同一个图而言，拓扑排序的结构是唯一的
- C. 拓扑排序中入度为 0 0 的结点总会排在入度大于0的结点的前面
- D. 拓扑排序结果序列中的第一个结点一定是入度等于 0 0 的点

> **答案**: D
> **解析**: 拓扑排序是对有向无环图（DAG）的一种线性排序，其基本定义是：对于图中的每一条有向边 (u, v)，在拓扑序列中，结点 u 必须出现在结点 v 之前。

下面逐一分析各选项：

A. 错误。并非所有连通的有向图都可以实现拓扑排序。如果图中存在环（即不是有向无环图），则无法进行拓扑排序。

B. 错误。拓扑排序的结果不一定是唯一的。如果在排序过程中的任意时刻，有多个结点的入度同时为 0，那么选择其中任何一个结点作为序列的下一个元素都是合法的，这会导致产生不同的拓扑序列。

C. 错误。这个说法具有误导性。虽然拓扑排序是从入度为 0 的结点开始的，但在排序过程中，一个原本入度大于 0 的结点，在其前驱结点被移除后，入度会变为 0。在最终的序列中，一个“初始入度为 0”的结点并不一定排在所有“初始入度大于 0”的结点前面。例如：图中有两条边 A->B 和 C->D，A 和 C 的初始入度为 0，B 和 D 的初始入度为 1。序列 [A, B, C, D] 是一个合法的拓扑排序，此时入度为 1 的结点 B 排在了入度为 0 的结点 C 之前。

D. 正确。根据拓扑排序的定义，如果序列中的第一个结点 X 的入度大于 0，说明至少存在一个结点 Y 使得有一条边 Y->X。按照拓扑排序的规则，Y 必须排在 X 之前。这与 X 是序列中的第一个结点相矛盾。因此，拓扑序列的第一个结点在原图中对应的入度必须为 0。

18), 一个平面的法线是指与该平面垂直的直线。过点(1,1,1),(0,3,0),(2,0,0)的平面的法线是（ ）
{{ select(18) }}
- A. 过点 ( 1 , 1 , 1 ) , ( 2 , 3 , 3 ) (1,1,1),(2,3,3) 的直线
- B. 过点 ( 1 , 1 , 1 ) , ( 3 , 2 , 1 ) (1,1,1),(3,2,1) 的直线
- C. 过点 ( 0 , 3 , 0 ) , ( − 3 , 1 , 1 ) (0,3,0),(−3,1,1) 的直线
- D. 过点 ( 2 , 0 , 0 ) , ( 5 , 2 , 1 ) (2,0,0),(5,2,1) 的直线

> **答案**: D
> **解析**: 1. **确定平面内的向量**：
   设平面上的三个点分别为 $A(1,1,1)$，$B(0,3,0)$，$C(2,0,0)$。
   我们可以求出平面内的两个不共线向量：
   $\vec{AB} = (0-1, 3-1, 0-1) = (-1, 2, -1)$
   $\vec{AC} = (2-1, 0-1, 0-1) = (1, -1, -1)$

2. **求平面的法向量**：
   平面的法向量 $\vec{n} = (x, y, z)$ 必须同时垂直于平面内的这两个向量，即它们的点积为 0：
   - $\vec{n} \cdot \vec{AB} = -x + 2y - z = 0$
   - $\vec{n} \cdot \vec{AC} = x - y - z = 0$
   
   联立方程组：
   (1) $-x + 2y - z = 0$
   (2) $x - y - z = 0$
   将两式相加得：$y - 2z = 0 \Rightarrow y = 2z$
   将 $y = 2z$ 代入 (2) 得：$x - 2z - z = 0 \Rightarrow x = 3z$
   
   取 $z = 1$，得到法向量 $\vec{n} = (3, 2, 1)$。任何与此向量成比例的向量都可以作为法向量。

3. **验证选项中的直线方向向量**：
   法线是与平面垂直的直线，其方向向量必须与平面的法向量 $\vec{n}$ 平行。
   - **选项 A**：方向向量为 $(2-1, 3-1, 3-1) = (1, 2, 2)$，与 $(3, 2, 1)$ 不平行。
   - **选项 B**：方向向量为 $(3-1, 2-1, 1-1) = (2, 1, 0)$，与 $(3, 2, 1)$ 不平行。
   - **选项 C**：方向向量为 $(-3-0, 1-3, 1-0) = (-3, -2, 1)$，与 $(3, 2, 1)$ 不平行。
   - **选项 D**：方向向量为 $(5-2, 2-0, 1-0) = (3, 2, 1)$，与 $(3, 2, 1)$ 完全一致，说明该直线与平面垂直。

因此，正确答案是 D。

19), 双向链表中有两个指针域 llink 和 rlink，分别指向该结点的前驱及后继。设 p 指向链表中的一个结点，他的左右结点均为非空。现要求删除结点 p，则下列语句序列中正确的是（ ）
{{ select(19) }}
- A. p->rlink->llink=p->rlink;p->llink->rlink=p->llink; delete p;
- B. p->llink->rlink=p->rlink;p->rlink->llink = p->llink; delete p;
- C. p->rlink->llink = p->llink;p->rlink->llink ->rlink = p->rlink; delete p;
- D. p->llink->rlink = p->rlink;p->llink->rlink->llink = p->llink; delete p;

> **答案**: B
> **解析**: 在双向链表中删除一个结点 $p$，需要修改 $p$ 的前驱结点的后继指针和 $p$ 的后继结点的前驱指针，使它们绕过 $p$ 直接相连。

具体步骤如下：
1. **修改前驱结点的后继指针**：$p$ 的前驱结点是 `p->llink`，它的后继指针是 `rlink`。我们需要让它指向 $p$ 的后继结点 `p->rlink`。对应的代码是：`p->llink->rlink = p->rlink;`
2. **修改后继结点的前驱指针**：$p$ 的后继结点是 `p->rlink`，它的前驱指针是 `llink`。我们需要让它指向 $p$ 的前驱结点 `p->llink`。对应的代码是：`p->rlink->llink = p->llink;`
3. **释放内存**：最后使用 `delete p;` 释放结点 $p$ 所占用的空间。

对比选项：
- **A 选项**：`p->rlink->llink=p->rlink` 将后继结点的前驱指向了后继结点本身，错误。
- **B 选项**：完全符合上述逻辑，是标准且正确的删除操作。
- **C 选项**：第二步 `p->rlink->llink->rlink = p->rlink` 虽然在逻辑推导上（因为第一步已经修改了 `p->rlink->llink`）可能指向正确的位置，但这种写法极不规范，且容易出错。
- **D 选项**：同理，第二步写法不规范。

因此，正确答案是 B。
