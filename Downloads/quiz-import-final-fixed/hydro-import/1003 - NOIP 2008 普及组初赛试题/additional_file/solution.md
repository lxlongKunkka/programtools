1), 微型计算机中，控制器的基本功能是（ ）
{{ select(1) }}
- A. 控制机器各个部件协调工作
- B. 实现算术运算和逻辑运算
- C. 获取外部信息
- D. 存放程序和数据

> **答案**: A
> **解析**: 在微型计算机的硬件结构中，通常由运算器、控制器、存储器、输入设备和输出设备五大部分组成。

1. 控制器（Control Unit）：是计算机的指挥中心，它的基本功能是从存储器中取出指令、分析指令（译码），并根据指令的要求向计算机的各个部件发出控制信号，从而控制机器各个部件协调一致地工作。因此，选项 A 正确。
2. 运算器（ALU）：主要负责实现算术运算（如加减乘除）和逻辑运算（如与或非）。这是选项 B 描述的功能。
3. 输入设备：负责将外部信息（如文字、图像、声音等）转换为计算机能识别的电信号并输入到计算机中。这是选项 C 描述的功能。
4. 存储器：负责存放程序和数据。这是选项 D 描述的功能。

综上所述，控制器的核心作用就是“控制”与“协调”。

2), 设A=true,B=false,C=true,D=false，以下逻辑运算表达式值为真的是（ ）
{{ select(2) }}
- A. (A∧B)∨(C∧D∨ -A)
- B. ((A∧B)∨C)∧ -D
- C. (B∨C∨D)∧D∧A
- D. A∧(D∨ C)∧B

> **答案**: B
> **解析**: 首先，我们需要明确题目中给出的变量值：A = true, B = false, C = true, D = false。
逻辑运算符的含义如下：
- ∧ 表示“与”（AND），只有两边都为真时结果才为真。
- ∨ 表示“或”（OR），只要有一边为真结果就为真。
- - 表示“非”（NOT），将真变为假，将假变为真。

接下来逐一分析各选项：

A. (A∧B)∨(C∧D∨ -A)
- A∧B = true ∧ false = false
- -A = NOT true = false
- C∧D = true ∧ false = false
- (C∧D∨ -A) = false ∨ false = false
- 最终结果：false ∨ false = false

B. ((A∧B)∨C)∧ -D
- A∧B = true ∧ false = false
- (A∧B)∨C = false ∨ true = true
- -D = NOT false = true
- 最终结果：true ∧ true = true

C. (B∨C∨D)∧D∧A
- B∨C∨D = false ∨ true ∨ false = true
- D = false
- A = true
- 最终结果：true ∧ false ∧ true = false（因为其中包含一个 false，且运算全是“与”）

D. A∧(D∨ C)∧B
- A = true
- D∨C = false ∨ true = true
- B = false
- 最终结果：true ∧ true ∧ false = false（因为其中包含一个 false，且运算全是“与”）

综上所述，只有选项 B 的计算结果为真。

4), 计算机在工作过程中，若突然停电，（ ）中的信息不会丢失。
{{ select(4) }}
- A. ROM 和 RAM
- B. CPU
- C. ROM
- D. RAM

> **答案**: C
> **解析**: 在计算机存储器中，根据掉电后数据是否丢失，可以分为易失性存储器和非易失性存储器：

1. ROM (Read-Only Memory，只读存储器)：是一种非易失性存储器。它的特点是信息一旦写入，就可以永久保存，即使在断电或突然停电的情况下，其中的数据也不会丢失。它通常用于存储计算机的启动程序（如 BIOS）或嵌入式系统的固件。

2. RAM (Random Access Memory，随机存取存储器)：是一种易失性存储器。它的读写速度快，但需要持续供电来维持数据。一旦断电，RAM 中存储的所有信息都会立即消失。

3. CPU：内部包含寄存器和高速缓存（Cache），这些部件通常由类似于 RAM 的电路构成，断电后信息也会丢失。

因此，在突然停电时，只有 ROM 中的信息能够保存下来。正确答案为 C。

5), 完全二叉树共有2N−1个结点，则它的叶节点数是（ ）
{{ select(5) }}
- A. 𝑁 − 1 N−1
- B. 𝑁 N
- C. 2 𝑁 2N
- D. 2 𝑁 − 1 2 N −1

> **答案**: B
> **解析**: 1. 二叉树的基本性质：设 $n_0$ 为叶子节点（度为 0）的数量，$n_1$ 为度为 1 的节点数量，$n_2$ 为度为 2 的节点数量。在任何二叉树中，总节点数 $n = n_0 + n_1 + n_2$。
2. 节点与边的关系：二叉树的边数等于总节点数减 1（$n-1$），同时也等于 $0 \times n_0 + 1 \times n_1 + 2 \times n_2$。由此可以推导出：$n_0 = n_2 + 1$。
3. 代入公式：将 $n_2 = n_0 - 1$ 代入总节点数公式中，得到 $n = n_0 + n_1 + (n_0 - 1) = 2n_0 + n_1 - 1$。
4. 结合题目条件：题目已知总节点数 $n = 2N - 1$。代入上式得：$2N - 1 = 2n_0 + n_1 - 1$，化简得到 $2N = 2n_0 + n_1$。
5. 完全二叉树的特性：在完全二叉树中，度为 1 的节点数 $n_1$ 只能是 0 或 1。
   - 如果 $n_1 = 1$，则 $2N = 2n_0 + 1$，此时 $n_0$ 不是整数，不符合逻辑。
   - 如果 $n_1 = 0$，则 $2N = 2n_0$，解得 $n_0 = N$。
6. 结论：该完全二叉树的叶节点数 $n_0$ 为 $N$。

举例验证：
若 $N=2$，总节点数为 $2 \times 2 - 1 = 3$。这是一个满二叉树，根节点有 2 个子节点，叶子节点数为 2（即 $N$）。
若 $N=3$，总节点数为 $2 \times 3 - 1 = 5$。其结构为根节点、第二层 2 个节点、第三层 2 个节点。此时度为 2 的节点有 2 个，叶子节点数为 3（即 $N$）。

7), 设栈 S 的初始状态为空，元素a,b,c,d,e,f依次入栈 S，出栈的序列为b,d,f,e,c,a，则栈 S 的容量至少应该是（ ）
{{ select(7) }}
- A. 6
- B. 5
- C. 4
- D. 3

> **答案**: C
> **解析**: 本题考查栈（Stack）的基本性质：后进先出（LIFO）。我们需要根据给定的入栈顺序和出栈顺序，模拟栈的操作过程，并记录栈中元素数量的最大值，这个最大值即为栈的最小容量。

具体模拟过程如下：
1. 元素 a 入栈：栈内为 [a]，当前栈大小为 1。
2. 元素 b 入栈：栈内为 [a, b]，当前栈大小为 2。
3. 元素 b 出栈（出栈序列第一个）：栈内剩余 [a]，当前栈大小为 1。
4. 元素 c 入栈：栈内为 [a, c]，当前栈大小为 2。
5. 元素 d 入栈：栈内为 [a, c, d]，当前栈大小为 3。
6. 元素 d 出栈（出栈序列第二个）：栈内剩余 [a, c]，当前栈大小为 2。
7. 元素 e 入栈：栈内为 [a, c, e]，当前栈大小为 3。
8. 元素 f 入栈：栈内为 [a, c, e, f]，当前栈大小为 4。
9. 元素 f 出栈（出栈序列第三个）：栈内剩余 [a, c, e]，当前栈大小为 3。
10. 元素 e 出栈（出栈序列第四个）：栈内剩余 [a, c]，当前栈大小为 2。
11. 元素 c 出栈（出栈序列第五个）：栈内剩余 [a]，当前栈大小为 1。
12. 元素 a 出栈（出栈序列第六个）：栈内为空 []，当前栈大小为 0。

在整个操作过程中，栈内元素数量的最大值出现在第 8 步，即元素 f 入栈后，栈内共有 4 个元素（a, c, e, f）。因此，栈 S 的容量至少应该是 4。

8), 与十进制数 28.5625 相等的四进制数是（ ）
{{ select(8) }}
- A. 123.21
- B. 131.22
- C. 130.22
- D. 130.21

> **答案**: D
> **解析**: 将十进制数转换为四进制数，需要分别处理整数部分和小数部分。

1. 整数部分转换（28）：
使用“除4取余法”：
28 ÷ 4 = 7，余数为 0
7 ÷ 4 = 1，余数为 3
1 ÷ 4 = 0，余数为 1
将余数从下往上排列，得到整数部分的四进制表示为：130。

2. 小数部分转换（0.5625）：
使用“乘4取整法”：
0.5625 × 4 = 2.25，取整数部分 2，剩余 0.25
0.25 × 4 = 1.0，取整数部分 1，剩余 0
将取出的整数部分从上往下排列，得到小数部分的四进制表示为：0.21。

3. 合并结果：
将整数部分和小数部分合并，得到 130.21。

因此，与十进制数 28.5625 相等的四进制数是 130.21。正确答案为 D。

{{ select(9) }}

> **答案**: A

2. 理解子串定义：子串是指字符串中任意连续的字符序列。
3. 计算非空子串数目：
   对于一个长度为 n 且字符各不相同的字符串，其非空子串的个数可以通过以下方式累加：
   - 长度为 1 的子串有 7 个（O, l, y, m, p, i, c）
   - 长度为 2 的子串有 6 个（Ol, ly, ym, mp, pi, ic）

   - 长度为 4 的子串有 4 个（Olym, lymp, ympi, mpic）
   - 长度为 5 的子串有 3 个（Olymp, lympi, ympic）
   - 长度为 6 的子串有 2 个（Olympi, lympic）
   - 长度为 7 的子串有 1 个（Olympic）
4. 使用求和公式：
   非空子串总数 = 1 + 2 + 3 + 4 + 5 + 6 + 7 = 28。

11), 递归过程或函数调用时，处理参数和返回地址，通常使用一种称为（ ）的数据结构。
{{ select(11) }}
- A. 队列
- B. 多维数组
- C. 线性表
- D. 栈

> **答案**: D
> **解析**: 1. 递归调用的核心逻辑：在递归或普通的函数调用过程中，系统需要保存当前函数的“现场”，包括函数的局部变量、参数以及函数执行完毕后应该返回到的地址（断点）。
2. 后进先出（LIFO）特性：函数调用的顺序是：先调用的函数后结束，后调用的函数先结束。这种“最后进入的调用最先处理完成”的逻辑，与栈（Stack）这种数据结构的“后进先出”特性完全一致。
3. 调用栈（Call Stack）：在计算机底层，系统专门开辟了一块内存区域称为“栈”，用于管理函数调用。每当发起一次调用，就会在栈顶压入一个“栈帧”；当函数返回时，再从栈顶弹出该栈帧，恢复之前的执行状态。
4. 其他选项分析：
   - A. 队列（Queue）：遵循先进先出（FIFO）原则，常用于任务调度或广度优先搜索，不符合函数调用的逻辑。
   - B. 多维数组：主要用于存储矩阵或多维数据，与函数调用的管理机制无关。
   - C. 线性表：是一个广义的概念（栈和队列都属于线性表），但在描述函数调用机制时，必须指明是具有后进先出特性的“栈”。

13), 二叉树 T，已知其先根遍历是 1243576 （数字为结点的编号，以下同），中根遍历是 2415736，则该二叉树的后根遍历是（ ）
{{ select(13) }}
- A. 4 2 5 7 6 3 1 4 2 5 7 6 3 1
- B. 4 2 7 5 6 3 1 4 2 7 5 6 3 1
- C. 7 4 2 5 6 3 1 7 4 2 5 6 3 1
- D. 4 2 7 6 5 3 1 4 2 7 6 5 3 1

> **答案**: B
> **解析**: 通过二叉树的先根遍历（根-左-右）和中根遍历（左-根-右）可以唯一确定一棵二叉树。

1. 确定根节点：先根遍历的第一个节点是 1，所以根节点是 1。
2. 划分左右子树：在中根遍历中找到 1，其左侧是 {2, 4}，右侧是 {5, 7, 3, 6}。
3. 构建左子树：
   - 左子树的中根序列是 {2, 4}，先根序列是 {2, 4}。
   - 先根序列第一个是 2，说明 2 是左子树的根。
   - 在中根序列 {2, 4} 中，2 在 4 的左边，说明 4 是 2 的右孩子。
   - 左子树的后根遍历（左-右-根）为：4 2。
4. 构建右子树：
   - 右子树的中根序列是 {5, 7, 3, 6}，先根序列是 {3, 5, 7, 6}。
   - 先根序列第一个是 3，说明 3 是右子树的根。
   - 在中根序列 {5, 7, 3, 6} 中，3 的左侧是 {5, 7}，右侧是 {6}。
   - 对于 {5, 7}：先根序列是 {5, 7}，中根序列是 {5, 7}。5 是根，7 是 5 的右孩子。其后根遍历为 7 5。
   - 对于 {6}：是叶子节点，后根遍历为 6。
   - 右子树的后根遍历（左-右-根）为：7 5 6 3。
5. 组合整棵树：
   - 后根遍历 = [左子树后根序列] + [右子树后根序列] + [根节点]
   - 后根遍历 = 4 2 7 5 6 3 1。

14), 将数组{8,23,4,16,77,−5,53,100}中的元素按从大到小的顺序排列，每次可以交换任意两个元素，最少需要交换（ ）次。
{{ select(14) }}
- A. 4
- B. 5
- C. 6
- D. 7

> **答案**: B
> **解析**: 这是一个关于数组排序和置换环（Permutation Cycles）的问题。题目要求将数组按从大到小的顺序排列，并求最少交换次数。

1. 首先，确定目标状态（从大到小排列）：
   原始数组：{8, 23, 4, 16, 77, -5, 53, 100}
   目标数组：{100, 77, 53, 23, 16, 8, 4, -5}

2. 找出每个元素当前位置与目标位置的关系，形成置换环：
   - 元素 8：当前在索引 0，目标位置是索引 5。
   - 元素 -5：当前在索引 5，目标位置是索引 7。
   - 元素 100：当前在索引 7，目标位置是索引 0。
   形成环 1：(0 -> 5 -> 7 -> 0)，长度为 3。

   - 元素 23：当前在索引 1，目标位置是索引 3。
   - 元素 16：当前在索引 3，目标位置是索引 4。
   - 元素 77：当前在索引 4，目标位置是索引 1。
   形成环 2：(1 -> 3 -> 4 -> 1)，长度为 3。

   - 元素 4：当前在索引 2，目标位置是索引 6。
   - 元素 53：当前在索引 6，目标位置是索引 2。
   形成环 3：(2 -> 6 -> 2)，长度为 2。

3. 计算最少交换次数：
   对于一个长度为 $L$ 的环，将其元素归位需要 $L-1$ 次交换。
   总交换次数 = (环1长度 - 1) + (环2长度 - 1) + (环3长度 - 1)
   总交换次数 = (3 - 1) + (3 - 1) + (2 - 1) = 2 + 2 + 1 = 5 次。

或者使用公式：最少交换次数 = 元素总数 - 置换环个数 = 8 - 3 = 5 次。

因此，最少需要交换 5 次。正确答案为 B。

20), 在 C++ 程序中，表达式200|10的值是（ ）
{{ select(20) }}
- A. 20
- B. 1
- C. 220
- D. 202

> **答案**: D
> **解析**: 在 C++ 中，符号 `|` 是按位或（Bitwise OR）运算符。它的运算规则是将参与运算的两个数转换为二进制形式，然后对对应的每一位进行逻辑或操作：只要对应的两个二进制位中有一个为 1，结果位就为 1；只有当两位都为 0 时，结果位才为 0。

具体计算步骤如下：
1. 将十进制数 200 转换为二进制：
   200 = 128 + 64 + 8 = 2^7 + 2^6 + 2^3
   二进制表示为：11001000

2. 将十进制数 10 转换为二进制：
   10 = 8 + 2 = 2^3 + 2^1
   二进制表示为：00001010（为了对齐，前面补零）

3. 进行按位或运算：
   1 1 0 0 1 0 0 0  (200)
   | 0 0 0 0 1 0 1 0  (10)
   ------------------
   1 1 0 0 1 0 1 0

4. 将结果 11001010 转换回十进制：
   1*2^7 + 1*2^6 + 0*2^5 + 0*2^4 + 1*2^3 + 0*2^2 + 1*2^1 + 0*2^0
   = 128 + 64 + 0 + 0 + 8 + 0 + 2 + 0
   = 202

因此，表达式 200|10 的值是 202。正确答案为 D。
