1), 计算机的输入设备是（ ）
{{ select(1) }}
- A. 监示器
- B. 键盘
- C. 磁盘
- D. 打印机

> **答案**: B
> **解析**: 在计算机硬件系统中，设备通常根据其功能分为输入设备、输出设备和存储设备等。

1. 输入设备：是指向计算机输入数据和信息的设备。常见的输入设备包括键盘、鼠标、扫描仪、摄像头、麦克风等。本题中的“键盘”正是最典型的输入设备，用于输入文字和控制指令。
2. 输出设备：是指将计算机处理后的结果以文字、图形、声音等形式表现出来的设备。常见的输出设备包括监视器（显示器）、打印机、音箱、投影仪等。因此，选项 A（监示器）和选项 D（打印机）都属于输出设备。
3. 存储设备：是指用于存储程序和数据的设备。选项 C（磁盘）属于存储设备，它既可以作为输入来源（读取数据），也可以作为输出目的地（写入数据），但在基础分类中不直接归类为纯粹的输入设备。

综上所述，正确答案是 B。

2), 将十进制数 12.3125 化为二进制数为（ ）
{{ select(2) }}
- A. 1 1 0 0 . 1 1100.1
- B. 1 0 1 1 . 0 1 1 1011.011
- C. 1 1 0 0 . 0 1 0 1 1100.0101
- D. 1 1 0 0 . 1 0 1 1100.101

> **答案**: C
> **解析**: 将十进制数转换为二进制数，需要分别处理整数部分和小数部分：

1. 整数部分（12）的转换：
采用“除2取余，逆序排列”法：
12 ÷ 2 = 6 ... 余 0
6 ÷ 2 = 3 ... 余 0
3 ÷ 2 = 1 ... 余 1
1 ÷ 2 = 0 ... 余 1
将余数从下往上读，得到 1100。

2. 小数部分（0.3125）的转换：
采用“乘2取整，顺序排列”法：
0.3125 × 2 = 0.625 ... 取整数部分 0
0.625 × 2 = 1.25 ... 取整数部分 1
0.25 × 2 = 0.5 ... 取整数部分 0
0.5 × 2 = 1.0 ... 取整数部分 1
将取得的整数部分从上往下读，得到 0101。

3. 合并结果：
将整数部分和小数部分合并，得到 1100.0101。

对照选项，C 选项符合计算结果。

A 选项：仅为数值 640，没有换算单位。
B 选项：使用了十进制的 1000 进行换算，这在衡量硬盘标称容量时有时会用到，但在计算机内存计算中不准确。

7), 下面循环结构中，循环体执行的次数为（ ）次。
{{ select(7) }}
- A. 无数
- B. 4 4 次
- C. 6 6 次
- D. 5 5 次

> **答案**: D
> **解析**: 虽然题目中没有给出具体的循环代码段，但根据 GESP（软件编程等级考试）一级和二级考试的常见题型，此类题目通常考察最基础的循环结构。

最常见的两种执行 5 次的循环结构如下：

1. **使用 `for` 循环（从 0 开始，不包含 5）：**
   ```cpp
   for (int i = 0; i < 5; i++) {
       // 循环体
   }
   ```
   - 变量 `i` 的取值分别为：0, 1, 2, 3, 4。
   - 当 `i` 增加到 5 时，条件 `5 < 5` 不成立，循环结束。
   - 总计执行次数：5 次。

2. **使用 `for` 循环（从 1 开始，包含 5）：**
   ```cpp
   for (int i = 1; i <= 5; i++) {
       // 循环体
   }
   ```
   - 变量 `i` 的取值分别为：1, 2, 3, 4, 5。
   - 当 `i` 增加到 6 时，条件 `6 <= 5` 不成立，循环结束。
   - 总计执行次数：5 次。

**解析总结：**
- 如果循环条件是 `i < n` 且 `i` 从 0 开始，执行次数为 `n`。
- 如果循环条件是 `i <= n` 且 `i` 从 1 开始，执行次数为 `n`。
- 在本题中，选项 D 对应的 5 次是编程中最常用于遍历（如处理 5 个元素）的循环次数。选项中的“5 5 次”应为排版重复，实际指 5 次。

8), 在 Linux 系统中，比较两个文件的有无不同之处的命令是（ ）
{{ select(8) }}
- A. cmd
- B. diff
- C. fc
- D. cmp

> **答案**: B
> **解析**: 在 Linux 系统中，用于比较文件差异的常用命令及其功能如下：

1. diff (Difference)：这是 Linux 系统中最标准、最常用的文件比较命令。它以逐行的方式比较两个文本文件，并详细列出它们之间的不同之处。它不仅能告诉你有无不同，还能显示具体的差异内容。

2. cmp (Compare)：虽然 cmp 也是 Linux 命令，但它主要用于逐字节（byte-by-byte）比较两个文件。它通常用于二进制文件的比较，并且在发现第一个不同之处时就会停止并报告位置。在日常编程和文本对比中，diff 的使用频率远高于 cmp。

3. fc (File Compare)：这是 Windows/DOS 系统中的文件比较命令，在 Linux 系统中，fc 通常是指 shell 内置的用于编辑并重新执行历史命令的工具。

4. cmd：这是 Windows 系统的命令提示符（Command Prompt）程序，不是 Linux 命令，也不具备文件比较功能。

因此，在 Linux 系统中比较两个文件有无不同之处，最合适的命令是 diff。

9), 对于入栈顺序是 a, b, c, d, e 的序列，下面（ ）不可能是合法的出栈序列
{{ select(9) }}
- A. a, b, c, d, e
- B. d, b, c, a, e
- C. b, a, d, c, e
- D. c, b, a, d, e

> **答案**: B
> **解析**: 栈（Stack）是一种后进先出（LIFO, Last-In-First-Out）的数据结构。对于入栈顺序为 a, b, c, d, e 的序列，我们可以通过模拟入栈和出栈操作来验证每个选项：

A. a, b, c, d, e：
操作：push(a), pop(a), push(b), pop(b), push(c), pop(c), push(d), pop(d), push(e), pop(e)。
这是可能的。

B. d, b, c, a, e：
操作：要先弹出 d，必须依次压入 a, b, c, d。此时栈内从底向上为 [a, b, c, d]。
1. 执行 pop() 得到 d。此时栈顶是 c。
2. 选项要求下一个出栈的是 b。但在栈中，c 在 b 的上方，必须先弹出 c 才能弹出 b。
因此，d 之后紧跟 b 是不可能的（除非 c 已经在 d 之前出栈，但本项中 d 是第一个出栈的）。
这是不可能的。

C. b, a, d, c, e：
操作：push(a), push(b), pop(b), pop(a), push(c), push(d), pop(d), pop(c), push(e), pop(e)。
这是可能的。

D. c, b, a, d, e：
操作：push(a), push(b), push(c), pop(c), pop(b), pop(a), push(d), pop(d), push(e), pop(e)。
这是可能的。

总结：在栈的操作中，如果元素 x 比元素 y 先入栈，且 x 在 y 之后出栈，那么在 y 出栈后的任何时刻，比 y 先入栈但比 x 后入栈的元素必须已经出栈，或者按照它们入栈顺序的逆序出栈。在选项 B 中，d 出栈时，a, b, c 都在栈中，出栈顺序必须是 c, b, a，不能是 b, c。

10), 线性表若采用链表存储结构，要求内存中可用存储单元地址（ ）
{{ select(10) }}
- A. 必须连续
- B. 部分地址必须连续
- C. 一定不连续
- D. 连续不连续均可

> **答案**: D
> **解析**: 1. 线性表的存储结构主要分为两种：顺序存储结构（如数组）和链式存储结构（如链表）。
2. 顺序存储结构要求逻辑上相邻的元素在物理位置上也必须相邻，因此它需要一块连续的内存空间。
3. 链式存储结构则是通过“指针”来表示元素之间的逻辑关系。每个节点包含数据域和指针域，指针域存储了下一个节点的地址。
4. 在链表中，各个节点在内存中的物理地址是不确定的。它们可以分布在内存中任意可用的位置。这些位置在物理上可以是连续的，也可以是不连续的，这完全取决于内存分配时的具体情况。
5. 链表的灵活性就在于它不需要预先分配大块连续内存，只要内存中有足够的零散空间即可。因此，内存中可用存储单元的地址连续或不连续均可。

   - 在中序遍历 LKM 中，L 在 K 的左边，M 在 K 的右边。
   - 因此，K 的左孩子是 L，右孩子是 M。

12), 一棵完全二叉树有 72 个结点，那么它的叶子结点有（ ）个。
{{ select(12) }}
- A. 30
- B. 27
- C. 36
- D. 39

> **答案**: C
> **解析**: 在二叉树中，有几个核心性质可以用来解决此类问题：

1. 结点总数公式：n = n0 + n1 + n2。其中 n0 表示叶子结点（度为 0）的数量，n1 表示度为 1 的结点数量，n2 表示度为 2 的结点数量。
2. 边数关系公式：n0 = n2 + 1。这是所有二叉树通用的性质。
3. 完全二叉树的特有性质：在完全二叉树中，度为 1 的结点（n1）最多只能有 1 个。具体来说，如果总结点数 n 是偶数，则 n1 = 1；如果总结点数 n 是奇数，则 n1 = 0。

本题中，总结点数 n = 72，是一个偶数。
根据性质 3，我们可以确定这棵完全二叉树中 n1 = 1。

接下来，我们将性质 2 代入性质 1 的公式中：
n = n0 + n1 + (n0 - 1)
n = 2 * n0 + n1 - 1

将已知数据代入：
72 = 2 * n0 + 1 - 1
72 = 2 * n0
n0 = 36

因此，这棵完全二叉树的叶子结点数为 36 个。

此外，对于完全二叉树，还有一个简便的计算公式：叶子结点数 n0 = ⌈n / 2⌉（即 n 除以 2 向上取整）。
72 / 2 = 36，结果同样为 36。

13), 对于表达式a+b*(c+d)*e的后缀形式是（ ）
{{ select(13) }}
- A. abcd*+*e+
- B. a+b*c+d*e
- C. abcd+*e*+
- D. bcda+e*+*

> **答案**: C
> **解析**: 将中缀表达式转换为后缀表达式（逆波兰表达式）的步骤如下：

1. 识别表达式中的运算符和优先级：
   题目中的表达式 a+b(c+d)e 实际上省略了乘号，完整的中缀形式为：a + b * (c + d) * e。
   运算符优先级从高到低依次为：括号 () > 乘法 * > 加法 +。

2. 按照优先级逐步转换：
   - 首先处理括号内的部分：(c + d) 转换为 cd+。
   - 此时表达式变为：a + b * cd+ * e。
   - 接下来处理乘法（从左往右）：
     - 先处理 b * cd+，得到 bcd+*。
     - 再处理 (bcd+*) * e，得到 bcd+*e*。
   - 最后处理加法：a + (bcd+*e*)，得到 abcd+*e*+。

3. 验证选项：
   - A 选项：abcd*+*e+，逻辑错误。
   - B 选项：a+bc+de，包含了中缀运算符，不是后缀形式。
   - C 选项：abcd+*e*+，与推导结果一致。
   - D 选项：bcda+e*+*，逻辑错误。

因此，正确答案是 C。

14), 以下排序算法中，不需要进行关键字比较操作的算法是（ ）
{{ select(14) }}
- A. 快速排序
- B. 冒泡排序
- C. 归并排序
- D. 基数排序

> **答案**: D
> **解析**: 1. 排序算法根据其基本原理可以分为两大类：比较类排序和非比较类排序。
2. 比较类排序：通过比较两个元素的大小来确定它们的相对次序。由于受到决策树模型的限制，这类算法的时间复杂度下界通常是 O(n log n)。
   - 快速排序（A）：通过选取基准值并与其他元素进行比较来划分区间。
   - 冒泡排序（B）：通过相邻元素的比较和交换来将最大/最小值移动到一端。
   - 归并排序（C）：在合并两个有序子序列时，需要不断比较两个序列首元素的大小。
3. 非比较类排序：不通过直接比较关键字的大小，而是利用关键字的某种特性（如数字的位数、取值范围等）进行排序。
   - 基数排序（D）：它将关键字拆分为若干位（如个位、十位、百位），按照每一位的值将元素分配到不同的“桶”中。在整个排序过程中，不需要对两个关键字进行直接的大小比较。
4. 其他常见的非比较类排序还包括计数排序（Counting Sort）和桶排序（Bucket Sort）。

因此，本题中不需要进行关键字比较操作的算法是基数排序。

15), 5 个没有区别的结点构成不同形态的二叉树的种类有（ ）个。
{{ select(15) }}
- A. 30
- B. 35
- C. 42
- D. 40

> **答案**: C
> **解析**: 本题考查的是二叉树的基本性质。

对于 $n$ 个没有区别的结点（即不考虑结点的值，只考虑树的形状），能够构成的不同形态的二叉树的种类数可以用卡特兰数（Catalan Number）来计算。

卡特兰数的通项公式为：
$C_n = \frac{1}{n+1} \binom{2n}{n} = \frac{(2n)!}{(n+1)!n!}$

$C_5 = \frac{1}{5+1} \binom{10}{5}$
$C_5 = \frac{1}{6} \times \frac{10 \times 9 \times 8 \times 7 \times 6}{5 \times 4 \times 3 \times 2 \times 1}$
$C_5 = \frac{1}{6} \times 252$
$C_5 = 42$

常见的卡特兰数前几项为：
$n=1$ 时，$C_1 = 1$
$n=2$ 时，$C_2 = 2$
$n=3$ 时，$C_3 = 5$
$n=4$ 时，$C_4 = 14$
$n=5$ 时，$C_5 = 42$

因此，5 个没有区别的结点可以构成 42 种不同形态的二叉树。故选 C。
