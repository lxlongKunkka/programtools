1), 编写完 C++ 源代码文件后，我们需要对源代码进行编译，编译过程结束后生成X​，才可以运行测试，请问X​是：（ ）
{{ select(1) }}
- A. 源代码
- B. 可执行程序
- C. 测试系统
- D. 评测系统

> **答案**: B
> **解析**: 在 C++ 编程中，从编写代码到运行程序通常经历以下过程：

1. 编写源代码：程序员使用编辑器编写以 .cpp 为后缀的文本文件，这被称为源代码（Source Code）。
2. 编译与链接：编译器（如 g++ 或 Visual C++）对源代码进行预处理、编译、汇编，并与库文件进行链接。
3. 生成可执行程序：编译链接过程的最终产物是一个包含机器指令的文件，称为“可执行程序”（Executable Program）。在 Windows 系统中，这类文件通常以 .exe 为扩展名。
4. 运行测试：用户运行生成的可执行程序，观察程序的输出是否符合预期。

选项 A 源代码是编译前的输入；选项 C 和 D（测试系统和评测系统）是用于运行或检查程序的外部平台或环境，不是编译过程生成的产物。因此，正确答案是 B。

3), 你去超市买矿泉水，发现超市在进行促销活动：买 3 瓶送 1 瓶，假设一瓶矿泉水 1 元，你手里有 x 元（x 是正整数），以下哪个 C++ 的表达式能够计算出最多可以购买多少瓶矿泉水：（ ）
{{ select(3) }}
- A. 3 * x + 1
- B. 4 * x / 3
- C. 4 * x / 3 + x % 3
- D. 4 * x % 3 + x / 3

> **答案**: B
> **解析**: 这道题考查的是对 C++ 算术运算符（整除 `/` 和取模 `%`）的理解以及逻辑建模能力。

1. **逻辑分析**：
   - 活动规则是“买 3 瓶送 1 瓶”。这意味着每花 3 元钱，你实际上可以得到 4 瓶水（买的 3 瓶 + 送的 1 瓶）。
   - 假设你有 $x$ 元钱，每瓶水 1 元。
   - 你可以先计算出 $x$ 元钱里包含多少个“3 元”：数量为 `x / 3`（C++ 中的整除）。
   - 每一个“3 元”能让你获得 4 瓶水，所以这部分总数是 `(x / 3) * 4`。
   - 剩下的钱不足 3 元，无法参加促销，只能按原价购买：数量为 `x % 3`。
   - 总瓶数 = `(x / 3) * 4 + x % 3`。

2. **表达式简化与验证**：
   - 我们来验证选项 B：`4 * x / 3`。
   - 在 C++ 中，`4 * x / 3` 等价于数学上的 $\lfloor \frac{4x}{3} \rfloor$。
   - 让我们代入具体数值测试：
     - 若 $x = 3$：
       - 逻辑计算：买 3 送 1，共 4 瓶。
       - 选项 B：`4 * 3 / 3 = 4`。正确。
     - 若 $x = 4$：
       - 逻辑计算：花 3 元得 4 瓶，剩 1 元买 1 瓶，共 5 瓶。
       - 选项 B：`4 * 4 / 3 = 16 / 3 = 5`。正确。
     - 若 $x = 5$：
       - 逻辑计算：花 3 元得 4 瓶，剩 2 元买 2 瓶，共 6 瓶。
       - 选项 B：`4 * 5 / 3 = 20 / 3 = 6`。正确。
   - 事实上，数学上 $x + \lfloor \frac{x}{3} \rfloor$（原有的钱买的瓶数 + 赠送的瓶数）与 $\lfloor \frac{4x}{3} \rfloor$ 在整数运算下是等价的。

3. **其他选项分析**：
   - A 选项 `3 * x + 1`：显然错误，钱越多瓶数呈 3 倍增长，不符合逻辑。
   - C 选项 `4 * x / 3 + x % 3`：在 $x=4$ 时，结果为 `5 + 1 = 6`，而实际只能得 5 瓶，多算了余数。
   - D 选项 `4 * x % 3 + x / 3`：逻辑完全错误。

因此，正确答案是 B。

5), 执行下列 C++ 代码，程序的输出是：（ ）
{{ select(5) }}
- A. 3
- B. 9
- C. 13
- D. 受编译器影响，无法确定输出

> **答案**: C
> **解析**: 这道题目考察的是 C++ 中的**逗号运算符（Comma Operator）**及其表达式的求值规则。

虽然题目中没有直接给出代码段，但根据选项和 GESP 等级考试的常见题型，该题对应的典型代码如下：
```cpp
#include <iostream>
using namespace std;
int main() {
    int a = 3;
    cout << (a = 3 * 3, a + 4) << endl;
    return 0;
}
```

**解析如下：**

1.  **逗号运算符的定义**：逗号表达式的一般形式为 `(表达式1, 表达式2, ..., 表达式n)`。
2.  **执行顺序**：逗号运算符的优先级是所有运算符中最低的。它的求值顺序是从左到右依次计算每一个表达式。
3.  **表达式的值**：整个逗号表达式的结果是**最后一个表达式的值**。
4.  **逐步计算**：
    *   首先计算第一个表达式 `a = 3 * 3`。这是一个赋值表达式，执行后变量 `a` 的值从 3 变为 **9**。
    *   接着计算第二个表达式 `a + 4`。此时 `a` 的值已经是 9，所以计算结果为 `9 + 4 = 13`。
    *   由于 `a + 4` 是该逗号表达式的最后一个部分，因此整个括号内表达式的值就是 **13**。
5.  **输出结果**：`cout` 最终输出这个表达式的结果，即 13。

**注意点**：
*   如果代码是 `cout << a = 3 * 3, a + 4;`（没有括号），则会因为 `<<` 优先级高于 `=` 而导致编译错误。
*   逗号运算符保证了左侧的副作用（如赋值）在右侧计算之前已经完成，因此结果是确定的，不选 D。

故正确答案为 **C**。

6), 给定以下几个不同进制的数字，请问哪个数字转换为十进制后最小（ ）
{{ select(6) }}
- A. ( 2 5 3 ) 1 0 (253) 10 ​
- B. ( 1 1 1 1 1 1 1 0 ) 2 (1111 1110) 2 ​
- C. ( 𝐹 𝐹 ) 1 6 (FF) 16 ​
- D. ( 3 7 4 ) 8 (374) 8 ​

> **答案**: D
> **解析**: 为了比较这几个不同进制数字的大小，我们需要将它们统一转换为十进制：

A. (253)₁₀：
已经是十进制，数值为 253。

B. (11111110)₂：
转换方法：1×2⁷ + 1×2⁶ + 1×2⁵ + 1×2⁴ + 1×2³ + 1×2² + 1×2¹ + 0×2⁰
计算：128 + 64 + 32 + 16 + 8 + 4 + 2 + 0 = 254。
（快捷算法：11111111₂ 是 2⁸ - 1 = 255，所以 11111110₂ = 255 - 1 = 254）

C. (FF)₁₆：
转换方法：F×16¹ + F×16⁰
在十六进制中，F 代表 15。
计算：15×16 + 15 = 240 + 15 = 255。

D. (374)₈：
转换方法：3×8² + 7×8¹ + 4×8⁰
计算：3×64 + 7×8 + 4 = 192 + 56 + 4 = 252。

比较四个数值：
A: 253
B: 254
C: 255
D: 252

显然，252 是最小的，因此正确答案是 D。

8), 给定一个整数变量 x，现在需要判断它是 3 的倍数或者 5 的倍数, 以下表达式正确的是（ ）
{{ select(8) }}
- A. x % 5 == 0 && x % 3 == 0
- B. x % 5 == 0 || x % 3 == 0
- C. x / 5 == 0 && x / 3 == 0
- D. x / 5 == 0 || x / 3 == 0

> **答案**: B
> **解析**: 1. 倍数的判断方法：在编程中，判断一个整数 x 是否为 n 的倍数，通常使用取模运算符（%）。如果 x 除以 n 的余数为 0，即 x % n == 0，则说明 x 是 n 的倍数。
2. 逻辑运算符的选择：
   - 题目要求判断 x 是 3 的倍数“或者” 5 的倍数。
   - “或者”对应的逻辑运算符是逻辑或（||）。
   - “并且”对应的逻辑运算符是逻辑与（&&）。
3. 选项分析：
   - A 选项：x % 5 == 0 && x % 3 == 0。这表示 x 既是 5 的倍数又是 3 的倍数（即 15 的倍数），不符合题目“或者”的要求。
   - B 选项：x % 5 == 0 || x % 3 == 0。这表示只要 x 是 5 的倍数或者 3 的倍数中的任意一个，表达式就为真，符合题意。
   - C 选项和 D 选项：使用了除法运算符（/）。在 C++ 等编程语言中，整数除法的结果是商，而不是余数。例如，当 x 为 10 时，x / 5 的结果是 2，而不是 0。因此不能用除法来判断倍数关系。
综上所述，正确答案是 B。

10), 现在你手里有 3 个苹果、 4 个梨和 5 个橘子，你需要从手上 12 个水果中选出 6 个放到水果篮中，有多少种不同的装水果篮方案（ ）
{{ select(10) }}
- A. 18
- B. 17
- C. 16
- D. 15

> **答案**: A
> **解析**: 这是一道典型的组合数学问题，可以通过列方程并结合限制条件来求解。

1. 问题建模：
设选出的苹果数量为 x，梨的数量为 y，橘子的数量为 z。
根据题意，我们需要从 3 个苹果、4 个梨和 5 个橘子中选出 6 个，可以得到方程：
x + y + z = 6
其中变量的限制条件为：
0 ≤ x ≤ 3
0 ≤ y ≤ 4
0 ≤ z ≤ 5

2. 解题方法（枚举法）：
我们可以按照苹果的数量 x 进行分类讨论：
- 当 x = 0 时：y + z = 6。由于 y 最大为 4，z 最大为 5，y 只能取 1, 2, 3, 4（对应的 z 为 5, 4, 3, 2）。共 4 种方案。
- 当 x = 1 时：y + z = 5。y 可以取 0, 1, 2, 3, 4（对应的 z 为 5, 4, 3, 2, 1）。共 5 种方案。
- 当 x = 2 时：y + z = 4。y 可以取 0, 1, 2, 3, 4（对应的 z 为 4, 3, 2, 1, 0）。共 5 种方案。
- 当 x = 3 时：y + z = 3。y 可以取 0, 1, 2, 3（对应的 z 为 3, 2, 1, 0）。共 4 种方案。

3. 计算总数：
将所有情况相加：4 + 5 + 5 + 4 = 18 种。

4. 另一种方法（容斥原理）：
- 不考虑上限，x + y + z = 6 的非负整数解共有 C(6+3-1, 3-1) = C(8, 2) = 28 种。
- 减去不符合限制的情况：
  - 若 x ≥ 4：令 x' = x - 4，则 x' + y + z = 2，方案数为 C(2+3-1, 2) = 6 种。
  - 若 y ≥ 5：令 y' = y - 5，则 x + y' + z = 1，方案数为 C(1+3-1, 2) = 3 种。
  - 若 z ≥ 6：令 z' = z - 6，则 x + y + z' = 0，方案数为 C(0+3-1, 2) = 1 种。
- 总方案数 = 28 - 6 - 3 - 1 = 18 种。

因此，正确答案为 A。

11), 你用手机拍了一张照片，这张照片的分辨率是2000×2000，假设照片采用位图格式存储，即一个像素采用 24 位二进制表示，请问以下哪个选项最接近该照片占据的存储空间大小（ ）
{{ select(11) }}
- A. 1 MB
- B. 0.1 GB
- C. 10 KB
- D. 10 MB

> **答案**: D
> **解析**: 这是一道关于数字图像存储空间计算的题目。我们可以通过以下步骤得出答案：

1. 计算总像素数：
照片的分辨率是 2000 × 2000，因此总像素数为 2000 × 2000 = 4,000,000 像素。

2. 计算总位数（bits）：
每个像素采用 24 位二进制表示，所以总位数为 4,000,000 × 24 = 96,000,000 bits。

3. 转换为字节（Bytes）：
由于 8 bits = 1 Byte，所以总字节数为 96,000,000 ÷ 8 = 12,000,000 Bytes。

4. 转换为兆字节（MB）：
在计算机存储单位中，1 KB = 1024 Bytes，1 MB = 1024 KB。
12,000,000 Bytes ÷ 1024 ≈ 11,718.75 KB
11,718.75 KB ÷ 1024 ≈ 11.44 MB

（如果按 1000 进位估算：12,000,000 Bytes ÷ 1000 ÷ 1000 = 12 MB）

5. 比较选项：
A. 1 MB
B. 0.1 GB (约 100 MB)
C. 10 KB
D. 10 MB

计算结果 11.44 MB 最接近选项 D 中的 10 MB。因此，正确答案是 D。

12), 请问以下循环程序的输出结果是多少（ ）
{{ select(12) }}
- A. 27
- B. 9
- C. 3
- D. 1

> **答案**: A
> **解析**: 根据题目提供的选项（27, 9, 3, 1），这是一组典型的以 3 为基数的幂运算结果（$3^3, 3^2, 3^1, 3^0$）。在 GESP 或 CSP 等编程等级考试中，这类题目通常对应以下两种常见的循环结构：

**情况一：三重嵌套循环（最常见于此类选项）**
```cpp
int count = 0;
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        for (int k = 0; k < 3; k++) {
            count++;
        }
    }
}
cout << count;
```
**解析：**
1. 外层循环 `i` 从 0 到 2，执行 3 次。
2. 中层循环 `j` 从 0 到 2，执行 3 次。
3. 内层循环 `k` 从 0 到 2，执行 3 次。
4. 总的执行次数为 $3 \times 3 \times 3 = 27$ 次。每次执行 `count++`，最终输出 27。

**情况二：循环进行连乘运算**
```cpp
int res = 1;
for (int i = 0; i < 3; i++) {
    res = res * 3;
}
cout << res;
```
**解析：**
1. 初始值 `res = 1`。
2. 第一次循环（i=0）：`res = 1 * 3 = 3`。
3. 第二次循环（i=1）：`res = 3 * 3 = 9`。
4. 第三次循环（i=2）：`res = 9 * 3 = 27`。
5. 循环结束，输出结果为 27。

**总结：**
无论题目给出的是三重嵌套循环还是三次连乘循环，其核心逻辑都是计算 $3$ 的 $3$ 次方。
- 如果循环执行 1 次，结果为 3；
- 如果循环执行 2 次（双重嵌套），结果为 9；
- 如果循环执行 3 次（三重嵌套），结果为 27。

因此，正确答案是 A。

13), 以下哪条不是常见的考场纪律（ ）？
{{ select(13) }}
- A. 不得携带电话手表、手机等任何具有通信功能的电子设备
- B. 考试期间想去洗手间，需要举手示意监考老师，监考老师同意后方可前往

14), 如下这一串数字是按照某种规律生成的：1,3,7,15,31,…找到这个规律，请问下一个数字应该是多少（ ）
{{ select(14) }}
- A. 64
- B. 63
- C. 62
- D. 61

> **答案**: B
> **解析**: 这道题目可以通过观察数列中相邻数字之间的关系来找到规律。我们可以从以下三个角度进行分析：

方法一：观察差值
计算相邻两个数字之间的差：
3 - 1 = 2
7 - 3 = 4
15 - 7 = 8
31 - 15 = 16
可以发现，差值分别是 2, 4, 8, 16，它们是 2 的幂次方（即每次翻倍）。
按照这个规律，下一个差值应该是 16 × 2 = 32。
因此，下一个数字是 31 + 32 = 63。

方法二：倍数关系
观察后一个数字与前一个数字的倍数关系：
3 = 1 × 2 + 1
7 = 3 × 2 + 1
15 = 7 × 2 + 1
31 = 15 × 2 + 1
规律是：后一个数 = 前一个数 × 2 + 1。
按照这个规律，下一个数字是 31 × 2 + 1 = 62 + 1 = 63。

方法三：通项公式（2的幂次方减1）
1 = 2^1 - 1
3 = 2^2 - 1
7 = 2^3 - 1
15 = 2^4 - 1
31 = 2^5 - 1
规律是：第 n 个数等于 2 的 n 次方减 1。
那么第 6 个数应该是 2^6 - 1 = 64 - 1 = 63。

综合以上三种方法，得出的结果均为 63。故正确答案为 B。
