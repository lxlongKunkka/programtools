C++ map 核心课程

竞赛解题实战版 · 键值对的映射神器

为什么需要map？

先思考：用vector/set解题时，处理“映射关系”是否遇到这些问题？
vector/set的痛点
✘ 统计次数：需额外存储“数+次数”，代码繁琐
✘ 映射关系：如姓名→分数，无专属存储结构
✘ 按键查找：需手动遍历，效率低（O(n)）
✘ 一键一值：无法高效建立唯一键的映射
map的核心优势
✔ 键值对存储：一键一值，天然适配映射场景
✔ 键唯一：自动去重，避免重复映射
✔ 高效查找：按键查找O(logn)，远超遍历
✔ 有序存储：默认按键升序，无需额外排序
核心结论：map是“键值对”关联容器，专为“映射关系+快速按键查找”设计，竞赛中解决统计/映射类问题的首选！
map底层基于红黑树实现，无需理解底层，只需掌握“键值对操作+选型逻辑”！

map基础语法（必掌握）

1. 基础准备

// 1. 包含头文件（必须！）
#include <map>
// 2. 使用命名空间（简化代码）
using namespace std;

2. 定义与初始化

// 方式1：空map（最常用）
map m1;                // 键int，值int
map m2;             // 键string，值int（如姓名→分数）

// 方式2：直接初始化键值对
map m3{{1, 10}, {2, 20}, {3, 30}};

// 方式3：批量插入初始化
map m4;
m4.insert({{4, 40}, {5, 50}, {6, 60}});

3. 核心操作（竞赛高频）

操作 语法 说明 时间复杂度 赋值/插入 m[key] = val 键存在则修改值，不存在则插入（值默认初始化） O(logn) 插入（不覆盖） m.insert({key, val}) 键存在则插入失败，不覆盖原有值 O(logn) 查找键 m.find(key) 返回迭代器，未找到则为m.end() O(logn) 判断键存在 m.count(key) 返回0（不存在）/1（存在） O(logn) 删除键值对 m.erase(key) / m.erase(it) 删除指定键/迭代器指向的键值对 O(logn) 遍历访问 it->first/it->second 迭代器访问键（first）和值（second） O(n) 基础判断 m.size() / m.empty() 键值对个数 / 是否为空 O(1)
易错点：用m[key]判断键是否存在会自动插入空值！应改用m.find(key) != m.end()！

map基础实战

实战1：统计整数出现次数
题目：输入若干整数（以-1结束），统计每个数出现次数，按键升序输出。
#include <iostream>
#include <map>
using namespace std;

int main() {
    map cnt; // 键=数，值=出现次数
    int num;
    
    // 输入数据
    while (cin >> num && num != -1) {
        cnt[num]++; // 键存在则值+1，不存在则插入并初始化为1
    }
    
    // 迭代器遍历
    cout << "迭代器遍历（按键升序）：" << endl;
    for (map::iterator it = cnt.begin(); it != cnt.end(); it++) {
        cout << it->first << " 出现 " << it->second << " 次" << endl;
    }
    
    // 范围for遍历（推荐，C++11+）
    cout << "\n范围for遍历：" << endl;
    for (auto& p : cnt) { // 用引用避免拷贝开销
        cout << p.first << " 出现 " << p.second << " 次" << endl;
    }
    return 0;
}
// 输入：5 2 3 2 5 5 -1
// 输出：2 出现 2 次；3 出现 1 次；5 出现 3 次
实战2：姓名→分数映射查询
题目：建立姓名到分数的映射，输入姓名查询分数，不存在则提示“未找到”。
#include <iostream>
#include <map>
#include <string>
using namespace std;

int main() {
    map score;
    // 插入映射关系
    score["张三"] = 95;
    score["李四"] = 88;
    score["王五"] = 92;
    
    string name;
    cout << "请输入要查询的姓名：";
    cin >> name;
    
    // 正确判断键是否存在（避免自动插入）
    if (score.find(name) != score.end()) {
        cout << name << " 的分数：" << score[name] << endl;
    } else {
        cout << "未找到 " << name << " 的分数" << endl;
    }
    return 0;
}
技巧：遍历map时用auto& p替代auto p，避免拷贝键值对，提升大数据量效率！

map进阶变体（竞赛高频）

1. unordered_map（无序高效版）

map
✓ 按键升序存储
✓ 查找O(logn)
✓ 底层红黑树
✓ 适合需要有序的场景
unordered_map
✗ 无序存储
✓ 查找O(1)（平均）
✓ 底层哈希表
✓ 适合仅需映射+快速查找
// unordered_map使用示例（需包含）
#include <iostream>
#include <unordered_map>
#include <string>
using namespace std;

int main() {
    unordered_map um;
    um["苹果"] = 5;
    um["香蕉"] = 3;
    um["橙子"] = 4;
    
    // 无序输出：顺序不确定
    for (auto& p : um) {
        cout << p.first << "：" << p.second << "元" << endl;
    }
    
    // 查找效率更高
    if (um.count("苹果")) {
        cout << "\n苹果价格：" << um["苹果"] << "元" << endl;
    }
    return 0;
}

2. multimap（一键多值版）

核心特性：键可重复，支持一键多值，不支持下标访问（[]）
#include <iostream>
#include <map>
#include <string>
using namespace std;

int main() {
    multimap mm;
    // 插入重复键
    mm.insert({"张三", 95});
    mm.insert({"张三", 88}); // 同一姓名多个分数
    mm.insert({"李四", 90});
    
    // 查找张三的所有分数
    string name = "张三";
    auto range = mm.equal_range(name);
    cout << name << " 的所有分数：";
    for (auto it = range.first; it != range.second; it++) {
        cout << it->second << " "; // 输出：95 88
    }
    return 0;
}

3. 自定义排序（按键降序）

// 按键降序排列的map
#include <iostream>
#include <map>
using namespace std;

int main() {
    // greater指定按键降序
    map> m{{1,10}, {2,20}, {3,30}};
    for (auto& p : m) {
        cout << p.first << "：" << p.second << endl; // 输出：3:30 2:20 1:10
    }
    return 0;
}

map/unordered_map/set/数组 选型指南

需求场景 优先选择 核心原因 效率对比（数据量1e5） 键值对+按键有序+查找 map 有序+O(logn)查找，满足排序需求 查找：~1ms 键值对+无需有序+快速查找 unordered_map O(1)平均查找，效率最高 查找：~0.1ms 仅需键+有序+去重 set 无值存储，更节省内存 查找：~1ms 键值对+键可重复 multimap 支持一键多值，有序存储 查找：~1ms 键为连续整数（1~1e5） 数组/vector O(1)访问，比map更快 访问：~0.01ms 数据量小（<1000） map/unordered_map 无明显效率差异，按代码简洁选 无明显差异
实战：map按值降序排序
题目：统计单词出现次数，按出现次数降序输出（需转vector后自定义排序）。
#include <iostream>
#include <unordered_map>
#include <vector>
#include <algorithm>
#include <string>
using namespace std;

// 自定义排序规则：按值降序，值相同按键升序
bool cmp(pair& a, pair& b) {
    if (a.second != b.second) {
        return a.second > b.second;
    }
    return a.first < b.first;
}

int main() {
    unordered_map um;
    string word;
    
    // 输入单词（以end结束）
    while (cin >> word && word != "end") {
        um[word]++;
    }
    
    // 转vector排序
    vector> vec(um.begin(), um.end());
    sort(vec.begin(), vec.end(), cmp);
    
    // 输出结果
    for (auto& p : vec) {
        cout << p.first << "：" << p.second << "次" << endl;
    }
    return 0;
}
// 输入：apple banana apple orange banana apple end
// 输出：apple：3次；banana：2次；orange：1次
选型口诀：有序映射选map，高效映射选unordered_map，连续键选数组！

map避坑指南（解题必看）

坑1：用m[key]判断键是否存在
错误代码：if (m[key]) { ... }（键不存在时自动插入，值为默认值）
解决方案：改用m.find(key) != m.end() 或 m.count(key)
坑2：遍历map未用引用
错误：for (auto p : m) { ... }（每次拷贝pair，大数据量效率低）
解决方案：用for (auto& p : m) { ... }，减少拷贝开销
坑3：multimap使用下标访问
错误：multimap mm; mm["张三"] = 95;（编译报错）
解决方案：multimap用insert()插入，equal_range()查找所有值
坑4：unordered_map自定义键无哈希函数
错误：unordered_map um;（Person为自定义结构体）
解决方案：要么改用map，要么为自定义类型编写哈希函数（竞赛少用）
坑5：盲目用map替代数组
键为连续整数（如1~1e5）时，数组访问（O(1)）比map更快
解决方案：连续整数键优先用数组/vector，非连续键用map
避坑核心：理解map的键值对特性，区分[]和insert的差异，根据场景选择容器！

综合实战 & 课程总结

综合实战：数组映射查询
题目：建立数组A[i]→B[i]的映射，输入查询值输出对应值（无则输出-1），要求高效查找。
// 用unordered_map实现高效映射查询
#include <iostream>
#include <unordered_map>
#include <vector>
using namespace std;

int main() {
    vector A = {1, 3, 5, 7, 9};
    vector B = {2, 4, 6, 8, 10};
    unordered_map um;
    
    // 建立映射
    for (int i = 0; i < A.size(); i++) {
        um[A[i]] = B[i];
    }
    
    // 查询
    int q;
    cout << "请输入查询值（输入-1结束）：" << endl;
    while (cin >> q && q != -1) {
        if (um.count(q)) {
            cout << q << " → " << um[q] << endl;
        } else {
            cout << q << " → -1" << endl;
        }
    }
    return 0;
}
  • map核心知识点
    • 基础：键值对、键唯一、[]赋值、find()查找、遍历（first/second）
    • 变体：unordered_map（无序高效）、multimap（一键多值）
    • 选型：有序选map，高效选unordered_map，连续键选数组
    • 避坑：不用[]判断存在、遍历用引用、multimap不用[]
    • 课程核心：map的价值在于“键值对映射+快速按键查找”，竞赛中精准选型才能发挥最大优势！
