# 2026-03-10 工作日志

## 主题：题目管理页批量翻译逻辑重构

---

### 背景

ProblemManager 的"批量智能处理"功能会对所有选中题目翻译+打标签。此前缺乏幂等保护，已处理过的题目重复跑会再次消耗 Token；另外翻译来源字段逻辑混乱，且 TW 版本检测方案存在根本性错误（`problem_zh_TW.md` 从未上传到 Hydro，检测永远为 false）。本次对整个流程进行梳理和重构。

---

### 工作内容

#### 1. 修复 TW 版本检测方案（根本性错误）

**原方案**：调 Hydro 文件列表 API，检查 `additional_file` 中是否有 `problem_zh_TW.md`。

**问题**：`problem_zh_TW.md` 仅在 SolveData 本地 zip 导出时生成，从未上传到 Hydro，因此检测永远为 `false`，翻译永远不会被跳过。

**新方案**：直接检查 `doc.content` 是否已是 `{"zh": ..., "en": ...}` 双语 JSON（`en` 非空），作为"已处理"的信号——这是 `processOne` 成功完成后的终态，100% 可靠。

```js
const currentParsed = this.parseContent(doc.content || '')
const alreadyTranslated = !!(currentParsed.zh && currentParsed.en)
```

#### 2. 批量处理时跳过已翻译题目（节省 Token）

已翻译说明之前已完整跑过翻译+打标签，批量时直接 `return true` 跳过整道题。

```js
if (alreadyTranslated && !force) {
  return true  // 零 API 调用
}
```

#### 3. 单题"智能处理"按钮改为强制重翻（`force=true`）

单题按钮调用 `processOne(doc, true)`，跳过已翻译检测，强制重新翻译+打标签。适用于翻译质量不满意、需要重排版等场景。

```vue
<button @click="processOne(doc, true)">智能处理</button>
```

#### 4. 翻译来源统一为 `doc.content` 整体

无论批量还是单题，都把 `doc.content` 整体发给 AI，不再区分 `en` 字段 / 纯英文判断，让 AI 自行处理重排版+翻译。

```js
const textToTranslate = doc.content || ''
```

#### 5. 修正 `newEn` 赋值逻辑

翻译结果 `en` 字段完全以 AI 返回为准，不再回退到旧的 `existingEn`：

```js
newZh = transRes.result
newEn = transRes.meta?.english || ''
```

---

### 最终流程总结

| 操作 | 已翻译题目（双语JSON） | 未翻译题目 |
|---|---|---|
| 批量处理 | 跳过（零消耗） | `doc.content` 整体发 AI → 翻译+打标签 → 自动保存 |
| 单题"智能处理"按钮 | 强制重翻+打标签 → 手动保存 | 同上 → 手动保存 |
| SolveData 生成的纯中文题 | — | `doc.content` 发 AI，AI 直接输出中文题目+打标签 |
