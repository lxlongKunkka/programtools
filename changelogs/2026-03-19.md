# 2026-03-19 工作日志

## 主题：依赖安全收尾与回归部署

---

### 1. 替换 COS SDK 风险链

- 移除 `cos-nodejs-sdk-v5` 及其高风险传递依赖
- 新增 [server/utils/cosClient.js](server/utils/cosClient.js)，实现最小可控的 COS 直传能力
- 现有图片上传与文本上传改为复用项目内实现，不再依赖旧 SDK

本次保留的能力：

- 对象上传
- `Content-Type` 透传
- 鉴权签名
- 生成公网访问地址

这样处理的原因是旧 SDK 依赖链老、风险集中，而且当前业务只用到有限上传能力，自己维护更简单也更可控。

### 2. 清理剩余运行时依赖风险

- `express` 升级到 `^4.22.1`
- `socket.io` / `socket.io-client` 升级到 `^4.8.3`
- 在 `package.json` 中增加以下 `overrides`：
  - `body-parser: 1.20.4`
  - `qs: 6.14.2`
  - `socket.io-parser: 4.2.6`

这里使用 `overrides` 的原因是安全版本需要直接落到子依赖层，不能只依赖顶层包的 semver 漂移。

### 5. 后续维护建议

- 后续如需升级上传逻辑，优先修改 [server/utils/cosClient.js](server/utils/cosClient.js)
- 不要重新引入 `cos-nodejs-sdk-v5` 这类大依赖 SDK，除非业务能力明显超出当前最小实现
- 每次部署前固定执行：
  - `npm audit --omit=dev`
  - `npm run build`
- 涉及抓题或实时通信的改动，至少回归：MNA 抓取、附件下载、COS 上传、socket 推送
