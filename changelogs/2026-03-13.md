# 2026-03-13 工作日志

## 主题：Hydro OJ AC 代码 ZIP 获取修复

---

### 背景

nflsoi.cc:10611 比赛的 `/record` 页面对部分比赛返回 403，导致无法通过记录列表获取 AC 代码。
解决方案：下载 `/contest/{id}/code` ZIP 文件，从中提取 AC 提交。

---

### 问题排查与修复过程

#### 1. ZIP 状态字段理解错误（S 字段）

- 文件名格式：`U{uid}_P{pid}_R{rid}_S{status}@{score}.{lang}.{langid}`
  - 例：`U2034_P479_R69a8d6d5ad8aaef371fd9f7f_S1@100.cc.cc14o2`
  - `S1` = Accepted，`S2` = Wrong Answer，`@` 后为得分
- **原 bug**：正则 `_S(\d+)@` 将 `S1@100` 中的 `1` 误当作分数，改为按最高分选代码
- **修复**：改为直接过滤 `_S1@`（AC 状态），不再按分数选择

#### 2. pid 别名问题（根本原因）

- 比赛页面 `href="/p/P13075?tid=..."` 中 `P13075` 是题目别名（slug）
- ZIP 文件名中存储的是真实数字 pid，如 `_P5449_`
- `P13075` 和 `5449` 是同一道题，但字符串不匹配导致查找失败
- **修复**：在抓取题目 HTML 后，扫描页面内所有链接，提取第一个 `/p/{纯数字}/` 形式的真实 pid，用真实 pid 去匹配 ZIP 条目

#### 3. zipRawNamesCache 变量未声明

- 多次编辑中一次替换失败，导致变量被使用但未声明，产生 500 错误
- **修复**：在文件顶部补充 `const zipRawNamesCache = new Map()` 声明

### 代码变更

**`server/routes/hydro_nflsoi.js`**

- 新增 `zipRawNamesCache`：缓存每场比赛 ZIP 的完整文件名列表（调试用，随 API 响应返回 `zipFiles` 字段）

- `fetchHydroNflsoiAcCode()`：加强 debug 日志，miss 时打印 `codeMap keys`
