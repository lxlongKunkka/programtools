# 2026-03-14 工作日志

## 主题：Hydro OJ 单题功能完善

---

### 新增功能

#### 1. 图片上传 COS（`hydro_nflsoi.js`）

- 新增 `hydroGetBinary(urlPath)`：带 session cookie 下载二进制文件
- 新增 `replaceHydroImages(markdown)`：正则匹配 markdown 中 `nflsoi.cc:10611` 域名图片，逐一下载并上传至 COS，替换为公网 COS 链接
- 修复 `parseHydroHTML` 中图片相对路径问题：`./docId/file/xxx` 正确解析为绝对 URL

#### 2. 附件下载（`hydro_nflsoi.js`）

- Hydro OJ 页面将附件信息嵌入 HTML 内的 JSON：`"additional_file":[{"_id":"down.zip",...}]`
- 从页面内嵌 JSON 提取 `additional_file[0]._id`，构造 `/p/{numericPid}/file/{filename}` 路径下载
- 以 `sample.zip` 文件名返回 `{ filename, base64, size }` 给前端

#### 3. `realPid` 提取修复（`hydro_nflsoi.js`）

- Hydro 页面不渲染 `<a>` 标签，原有从 `<a href>` 提取 pid 的方式失效
- 改为从页面内嵌 JSON 的 `"docId":5073` 字段提取真实数字 pid（用于 ZIP 匹配和附件下载）

#### 4. 标签提取（`hydro_nflsoi.js`）

- 单题页面新增标签提取：优先从内嵌 JSON `"tag":[...]` 字段解析，回退到 DOM `a.problem__tag-link`
- 返回值加入 `tags` 字段

#### 5. 前端 sample.zip 徽章（`SolveData.vue`）

- 标题区域新增橙色 `📦 filename (size KB)` 徽章，当 `additionalFile` 存在时显示
- 有 base64 时可点击直接下载 zip 文件；无 base64（localStorage 恢复的任务）显示为不可点击状态
- 新增 `downloadSampleZip()` 方法

#### 6. 前端标题清理（`SolveData.vue`）

- 去除数字/字母数字混合题号前缀：`35260. 平面树` → `平面树`、`#P1030. 排列` → `排列`、`P1030. 排列` → `排列`
- 纯字母题号（如 `A.`）保留字母，只去掉点号：`A. 鸭子` → `A 鸭子`

#### 7. 前端使用后端返回标签（`SolveData.vue`）

- `addProblemAsTask` 新增读取 `data.tags`：单题抓取时若 `prefetchedTags` 为空则使用后端返回的标签
- 来自比赛列表的题目仍优先使用 `prefetchedTags`，不受影响
